# Tài Liệu Học Tập AWS CI/CD

## Giới Thiệu về CI/CD trên AWS

Chào mừng bạn đến với phần học về CI/CD (Tích hợp Liên tục/Triển khai Liên tục). Sau khi đã biết cách triển khai ứng dụng đúng cách, giờ thì sao nếu chúng ta triển khai ứng dụng một cách tự động?

### CI/CD là gì?

**CI/CD** là phương pháp tự động triển khai ứng dụng lên môi trường production khi code được push lên. Điều này loại bỏ các bước triển khai thủ công và giảm thiểu lỗi do con người gây ra.

## Các Khái Niệm Chính

### Tích Hợp Liên Tục (CI - Continuous Integration)
- Tự động build và test code khi có thay đổi được push lên
- Đảm bảo chất lượng code thông qua các bước kiểm tra tự động
- Phát hiện lỗi sớm trong chu kỳ phát triển

### Triển Khai Liên Tục (CD - Continuous Deployment)
- Tự động triển khai code lên môi trường production
- Push code → Tự động triển khai lên production
- Phương pháp cách mạng trong việc phân phối phần mềm

## Tại Sao CI/CD Quan Trọng

1. **Tự Động Hóa**: Code được triển khai "một cách kỳ diệu" lên môi trường production
2. **Hiệu Quả**: Mọi developer đều nên biết công nghệ này
3. **Tiêu Chuẩn Ngành**: Phương pháp cách mạng thay đổi quy trình phát triển
4. **Quan Trọng Cho Kỳ Thi**: Kỳ thi AWS sẽ có nhiều câu hỏi về CI/CD

## Các Dịch Vụ CI/CD của AWS

AWS cung cấp nhiều dịch vụ để triển khai CI/CD pipeline:

- **AWS CodePipeline**: Điều phối quy trình CI/CD
- **AWS CodeBuild**: Biên dịch source code và chạy tests
- **AWS CodeDeploy**: Tự động hóa việc triển khai ứng dụng
- **AWS CodeCommit**: Dịch vụ quản lý source code dựa trên Git

## Mẹo Học Tập

- Hiểu rõ quy trình CI/CD hoàn chỉnh trên AWS
- Thực hành thiết lập các pipeline tự động
- Nắm vững sự khác biệt giữa CI và CD
- Chuẩn bị sẵn sàng cho các câu hỏi thi về chủ đề này

## Các Bước Tiếp Theo

Hãy bắt đầu với việc triển khai CI/CD trên AWS và xem nó thay đổi quy trình phát triển của bạn như thế nào!
