# AWS CI/CD Study Guide

## Introduction to CI/CD on AWS

Welcome to the section on CI/CD (Continuous Integration/Continuous Deployment). Now that we've seen how to properly deploy an application, how about we deploy applications automatically? 

### What is CI/CD?

**CI/CD** is the practice of automatically deploying applications to production environments when code is pushed. This eliminates manual deployment steps and reduces human error.

## Key Concepts

### Continuous Integration (CI)
- Automatically building and testing code when changes are pushed
- Ensures code quality through automated checks
- Catches bugs early in the development cycle

### Continuous Deployment (CD)
- Automatically deploying code to production environments
- Push code → Automatic deployment to production
- Revolutionary approach to software delivery

## Why CI/CD Matters

1. **Automation**: Code is "magically" deployed to production environments
2. **Efficiency**: Every developer should know this technology
3. **Industry Standard**: Revolutionary practice that transforms development workflows
4. **Exam Important**: The AWS exam will ask many questions about CI/CD

## AWS CI/CD Services

AWS provides several services for implementing CI/CD pipelines:

- **AWS CodePipeline**: Orchestrates the CI/CD workflow
- **AWS CodeBuild**: Compiles source code and runs tests
- **AWS CodeDeploy**: Automates application deployments
- **AWS CodeCommit**: Git-based source control service
