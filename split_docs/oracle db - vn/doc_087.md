
Dưới đây là tóm tắt ngắn gọn về **Oracle Scheduler trong PL/SQL & SQL** (giả sử nội dung nói về tính năng Scheduler của Oracle Database):

---

## 1. Oracle Scheduler là gì?

- Là tính năng của Oracle dùng để:
  - Lên lịch chạy **job** (thủ tục, câu lệnh PL/SQL, chương trình OS…)
  - Tự động hóa tác vụ: backup, ETL, báo cáo, dọn dẹp dữ liệu…
- Làm việc chủ yếu qua package: `DBMS_SCHEDULER`.

---

## 2. Các khái niệm chính

- **Job**: Đơn vị công việc cần chạy (ví dụ: gọi 1 procedure).
- **Program**: Định nghĩa logic công việc (PL/SQL block, stored procedure, executable…).
- **Schedule**: Định nghĩa thời gian chạy (một lần, lặp lại, theo lịch).
- **Job Class**: Nhóm job để quản lý tài nguyên, ưu tiên.
- **Window / Window Group**: Khoảng thời gian đặc biệt (ví dụ: giờ thấp điểm).

---

## 3. Tạo job đơn giản (chạy 1 lần)

### Ví dụ: Tạo procedure và job

```sql
-- 1. Tạo bảng log
CREATE TABLE job_log (
    id        NUMBER GENERATED BY DEFAULT AS IDENTITY,
    log_time  DATE,
    message   VARCHAR2(200)
);

-- 2. Tạo procedure ghi log
CREATE OR REPLACE PROCEDURE p_write_log IS
BEGIN
    INSERT INTO job_log (log_time, message)
    VALUES (SYSDATE, 'Job chạy thành công');
    COMMIT;
END;
/

-- 3. Tạo job chạy 1 lần sau 1 phút
BEGIN
    DBMS_SCHEDULER.CREATE_JOB (
        job_name        => 'job_write_log_once',
        job_type        => 'STORED_PROCEDURE',
        job_action      => 'P_WRITE_LOG',
        start_date      => SYSTIMESTAMP + INTERVAL '1' MINUTE,
        enabled         => TRUE,
        comments        => 'Job ghi log 1 lần'
    );
END;
/
```

### Kiểm tra kết quả

```sql
-- Xem trạng thái job
SELECT job_name, state, last_start_date, last_run_duration
FROM   dba_scheduler_jobs
WHERE  job_name = 'JOB_WRITE_LOG_ONCE';

-- Sau khi job chạy, xem dữ liệu log
SELECT * FROM job_log;
```

**Output mẫu (job_log):**

| ID | LOG_TIME            | MESSAGE              |
|----|---------------------|----------------------|
| 1  | 20-05-2026 10:01:05 | Job chạy thành công |

## 4. Tạo job lặp lại theo lịch

### Ví dụ: Chạy mỗi 5 phút

```sql
BEGIN
    DBMS_SCHEDULER.CREATE_JOB (
        job_name        => 'job_write_log_repeat',
        job_type        => 'STORED_PROCEDURE',
        job_action      => 'P_WRITE_LOG',
        start_date      => SYSTIMESTAMP,
        repeat_interval => 'FREQ=MINUTELY; INTERVAL=5',
        enabled         => TRUE,
        comments        => 'Job ghi log mỗi 5 phút'
    );
END;
/
```

- `repeat_interval` dùng **calendar syntax**:
  - Ví dụ khác: `FREQ=DAILY; BYHOUR=2; BYMINUTE=0; BYSECOND=0` (mỗi ngày 2:00).

---

## 5. Quản lý job

```sql
-- Tạm dừng job
BEGIN
    DBMS_SCHEDULER.DISABLE('job_write_log_repeat');
END;
/

-- Kích hoạt lại job
BEGIN
    DBMS_SCHEDULER.ENABLE('job_write_log_repeat');
END;
/

-- Chạy job ngay lập tức
BEGIN
    DBMS_SCHEDULER.RUN_JOB('job_write_log_repeat', use_current_session => FALSE);
END;
/

-- Xóa job
BEGIN
    DBMS_SCHEDULER.DROP_JOB('job_write_log_repeat');
END;
/
```

---

## 6. Xem lịch sử chạy job

```sql
SELECT job_name,
       status,
       actual_start_date,
       run_duration,
       additional_info
FROM   dba_scheduler_job_run_details
WHERE  job_name = 'JOB_WRITE_LOG_REPEAT'
ORDER  BY actual_start_date DESC;
```

---

Tóm lại, **Scheduler** trong Oracle cho phép bạn:
- Định nghĩa và lên lịch job bằng `DBMS_SCHEDULER`.
- Tự động hóa các tác vụ PL/SQL/SQL.
- Quản lý thời gian, tần suất, ưu tiên và lịch sử chạy job một cách linh hoạt.
