
Dưới đây là tóm tắt ngắn gọn về **Hierarchical Queries trong Oracle** (truy vấn phân cấp), kèm ví dụ code và input/output minh họa.

---

## 1. Khái niệm chính

- **Hierarchical query** dùng để truy vấn dữ liệu có cấu trúc cây (tree), ví dụ:
  - Cây phòng ban – nhân viên (manager → employee)
  - Cây danh mục sản phẩm (category → subcategory)
- Trong Oracle, thường dùng các mệnh đề:
  - `START WITH` – xác định nút gốc (root)
  - `CONNECT BY` – xác định quan hệ cha–con
  - `PRIOR` – chỉ ra phía “cha” trong quan hệ
  - `LEVEL` – mức độ trong cây (root = 1, con = 2, …)
  - `CONNECT_BY_ROOT`, `SYS_CONNECT_BY_PATH` – lấy đường đi trong cây

---

## 2. Cấu trúc cơ bản

```sql
SELECT  LEVEL,
        emp_id,
        emp_name,
        manager_id
FROM    employees
START WITH manager_id IS NULL        -- nút gốc (VD: CEO)
CONNECT BY PRIOR emp_id = manager_id; -- emp_id cha = manager_id con
```

**Ý nghĩa:**

- `START WITH`: chọn các dòng bắt đầu (root)
- `CONNECT BY PRIOR emp_id = manager_id`:
  - `PRIOR emp_id` là cha
  - `manager_id` là con
- `LEVEL`: cho biết cấp độ trong cây

---

## 3. Ví dụ đầy đủ với input/output

### Bảng mẫu

```sql
CREATE TABLE employees (
  emp_id      NUMBER PRIMARY KEY,
  emp_name    VARCHAR2(50),
  manager_id  NUMBER
);

INSERT INTO employees VALUES (1, 'CEO',        NULL);
INSERT INTO employees VALUES (2, 'Manager A',  1);
INSERT INTO employees VALUES (3, 'Manager B',  1);
INSERT INTO employees VALUES (4, 'Staff A1',   2);
INSERT INTO employees VALUES (5, 'Staff A2',   2);
INSERT INTO employees VALUES (6, 'Staff B1',   3);
COMMIT;
```

### Truy vấn phân cấp

```sql
SELECT  LPAD(' ', (LEVEL-1)*2) || emp_name AS tree_name,
        emp_id,
        manager_id,
        LEVEL
FROM    employees
START WITH manager_id IS NULL
CONNECT BY PRIOR emp_id = manager_id
ORDER SIBLINGS BY emp_name;
```

**Kết quả (output):**

| TREE_NAME   | EMP_ID | MANAGER_ID | LEVEL |
|------------|--------|------------|-------|
| CEO        | 1      | (null)     | 1     |
|   Manager A| 2      | 1          | 2     |
|     Staff A1| 4     | 2          | 3     |
|     Staff A2| 5     | 2          | 3     |
|   Manager B| 3      | 1          | 2     |
|     Staff B1| 6     | 3          | 3     |

## 4. Một số kỹ thuật nâng cao

### 4.1. Lấy đường đi (path) từ root đến node

```sql
SELECT  emp_id,
        emp_name,
        SYS_CONNECT_BY_PATH(emp_name, ' / ') AS path
FROM    employees
START WITH manager_id IS NULL
CONNECT BY PRIOR emp_id = manager_id;
```

**Output ví dụ:**

| EMP_ID | EMP_NAME   | PATH                         |
|--------|------------|------------------------------|
| 1      | CEO        | / CEO                        |
| 2      | Manager A  | / CEO / Manager A           |
| 4      | Staff A1   | / CEO / Manager A / Staff A1|

---

### 4.2. Lấy root của mỗi node

```sql
SELECT  emp_id,
        emp_name,
        CONNECT_BY_ROOT emp_name AS root_name
FROM    employees
START WITH manager_id IS NULL
CONNECT BY PRIOR emp_id = manager_id;
```

---

### 4.3. Tìm tất cả cấp trên (truy ngược lên)

Đảo chiều `PRIOR`:

```sql
-- Tìm tất cả cấp trên của nhân viên có emp_id = 4
SELECT  LEVEL,
        emp_id,
        emp_name
FROM    employees
START WITH emp_id = 4
CONNECT BY PRIOR manager_id = emp_id;
```

---

### 4.4. Chống vòng lặp (cycle)

```sql
SELECT  emp_id,
        emp_name
FROM    employees
START WITH manager_id IS NULL
CONNECT BY NOCYCLE PRIOR emp_id = manager_id;
```

---

## 5. So sánh nhanh: CONNECT BY vs Recursive Subquery Factoring

Từ Oracle 11gR2, có thể dùng `WITH ... CONNECT BY` hoặc **recursive subquery factoring** (`WITH ... AS (...)`):

```sql
WITH emp_tree (emp_id, emp_name, manager_id, lvl) AS (
  SELECT emp_id, emp_name, manager_id, 1
  FROM   employees
  WHERE  manager_id IS NULL
  UNION ALL
  SELECT e.emp_id, e.emp_name, e.manager_id, t.lvl + 1
  FROM   employees e
  JOIN   emp_tree t ON e.manager_id = t.emp_id
)
SELECT * FROM emp_tree;
```

---

**Tóm tắt:**  
Hierarchical queries trong Oracle dùng `START WITH` + `CONNECT BY` để duyệt dữ liệu dạng cây. `PRIOR` xác định quan hệ cha–con, `LEVEL` cho biết cấp độ, và các hàm như `SYS_CONNECT_BY_PATH`, `CONNECT_BY_ROOT` giúp hiển thị đường đi và nút gốc. Đây là công cụ mạnh để xử lý cấu trúc tổ chức, danh mục, cây phụ thuộc, v.v.
