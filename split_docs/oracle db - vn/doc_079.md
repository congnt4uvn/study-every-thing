
Dưới đây là tóm tắt ngắn gọn về chủ đề “Locking & Isolation: How to detect blocking sessions” trong Oracle Database, kèm ví dụ code. (Vì bạn chỉ đưa tiêu đề, mình sẽ dựa trên kiến thức chuẩn về Oracle.)

---

## 1. Khái niệm chính

- **Lock (khóa)**: Cơ chế Oracle dùng để đảm bảo tính nhất quán dữ liệu khi nhiều session cùng truy cập/ghi dữ liệu.
- **Blocking session**: Session đang giữ lock trên tài nguyên (row, table, …) khiến session khác phải chờ (bị block).
- **Isolation**: Mức độ “cách ly” giữa các transaction, đảm bảo transaction không thấy dữ liệu chưa commit của transaction khác.

Mục tiêu: **Phát hiện session nào đang block session khác** để có thể xử lý (kill session, tối ưu SQL, điều chỉnh ứng dụng…).

## 2. Cách phát hiện blocking sessions (Oracle)

### 2.1. Dùng `V$SESSION` và `V$LOCK`

```sql
SELECT
    s1.sid        AS blocking_sid,
    s1.serial#    AS blocking_serial,
    s1.username   AS blocking_user,
    s1.machine    AS blocking_machine,
    s2.sid        AS blocked_sid,
    s2.serial#    AS blocked_serial,
    s2.username   AS blocked_user,
    s2.machine    AS blocked_machine,
    l1.id1        AS lock_id1,
    l1.id2        AS lock_id2,
    l1.type       AS lock_type
FROM v$lock l1
JOIN v$session s1 ON l1.sid = s1.sid
JOIN v$lock l2 ON l1.id1 = l2.id1
               AND l1.id2 = l2.id2
JOIN v$session s2 ON l2.sid = s2.sid
WHERE l1.block = 1          -- session đang block
  AND l2.request > 0;       -- session đang bị block
```

**Ý nghĩa:**

- `blocking_sid` / `blocking_serial`: session đang giữ lock.
- `blocked_sid` / `blocked_serial`: session đang bị chờ lock.
- `lock_type`: loại lock (TX, TM, …).

---

### 2.2. Dùng `V$SESSION` với `BLOCKING_SESSION`

Từ các phiên bản mới, Oracle có cột `BLOCKING_SESSION`:

```sql
SELECT
    sid,
    serial#,
    username,
    blocking_session,
    wait_class,
    event,
    seconds_in_wait
FROM v$session
WHERE blocking_session IS NOT NULL;
```

**Ý nghĩa:**

- `sid`: session đang bị block.
- `blocking_session`: SID của session đang block nó.
- `event`: đang chờ cái gì (ví dụ: `enq: TX - row lock contention`).
- `seconds_in_wait`: đã chờ bao lâu.

---

### 2.3. Dùng `DBA_BLOCKERS` và `DBA_WAITERS`

```sql
SELECT * FROM dba_blockers;
SELECT * FROM dba_waiters;
```

- `DBA_BLOCKERS`: liệt kê các session đang block session khác.
- `DBA_WAITERS`: liệt kê các session đang bị block, và bởi ai.

## 3. Ví dụ minh họa (input/output)

### 3.1. Tạo tình huống block

**Session 1 (User A)**

```sql
-- Input (Session 1)
UPDATE employees
SET salary = salary + 100
WHERE employee_id = 100;

-- Không COMMIT, giữ nguyên trạng thái
```

**Session 2 (User B)**

```sql
-- Input (Session 2)
UPDATE employees
SET salary = salary + 200
WHERE employee_id = 100;
```

Session 2 sẽ bị treo (chờ lock) vì Session 1 đang giữ lock trên dòng đó.

---

### 3.2. Kiểm tra blocking bằng `V$SESSION`

**Input (chạy từ user có quyền xem v$session):**

```sql
SELECT
    sid,
    serial#,
    username,
    blocking_session,
    event,
    seconds_in_wait
FROM v$session
WHERE blocking_session IS NOT NULL;
```

**Output (ví dụ):**

| SID | SERIAL# | USERNAME | BLOCKING_SESSION | EVENT                          | SECONDS_IN_WAIT |
|-----|---------|----------|------------------|--------------------------------|-----------------|
| 45  | 1234    | USER_B   | 32               | enq: TX - row lock contention  | 120             |

Giải thích:

- SID 45 (USER_B) đang bị block.
- `BLOCKING_SESSION = 32`: session 32 (USER_A) là session đang giữ lock.

---

## 4. Xử lý session bị block

Khi đã xác định được session block, có thể kill:

```sql
ALTER SYSTEM KILL SESSION '32,5678' IMMEDIATE;
-- 32 = SID, 5678 = SERIAL#
```

---

## 5. Tóm tắt

- Blocking session xảy ra khi một session giữ lock mà session khác cần.
- Có thể phát hiện blocking bằng:
  - `V$LOCK` + `V$SESSION`
  - Cột `BLOCKING_SESSION` trong `V$SESSION`
  - View `DBA_BLOCKERS`, `DBA_WAITERS`
- Sau khi xác định, có thể:
  - Phân tích SQL gây lock
  - Tối ưu logic transaction (commit sớm hơn, giảm thời gian giữ lock)
  - Hoặc kill session trong trường hợp cần thiết.
