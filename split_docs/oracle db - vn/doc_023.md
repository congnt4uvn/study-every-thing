
Dưới đây là tóm tắt ngắn gọn về **Cursor trong PL/SQL** (Oracle), kèm ví dụ code và input/output minh họa.

---

## 1. Cursor là gì?

- Cursor là “con trỏ” trỏ đến tập kết quả của một câu lệnh `SELECT`.
- Dùng trong PL/SQL để:
  - Duyệt từng dòng (row) của kết quả.
  - Xử lý logic cho từng dòng.

Có 2 loại chính:
1. **Implicit cursor** (ngầm định)
2. **Explicit cursor** (tường minh)  
   + biến thể tiện dụng: **Cursor FOR loop**

---

## 2. Implicit Cursor (Con trỏ ngầm định)

Oracle tự động tạo khi bạn dùng:
- `INSERT`, `UPDATE`, `DELETE`
- `SELECT ... INTO ...` (trả về tối đa 1 dòng)

### Đặc điểm
- Không cần khai báo.
- Truy cập thông tin qua `SQL%ROWCOUNT`, `SQL%FOUND`, `SQL%NOTFOUND`, `SQL%ISOPEN`.

### Ví dụ 1: SELECT INTO (1 dòng)

```sql
DECLARE
    v_emp_name   employees.last_name%TYPE;
    v_emp_id     employees.employee_id%TYPE := 100;  -- INPUT
BEGIN
    SELECT last_name
    INTO   v_emp_name
    FROM   employees
    WHERE  employee_id = v_emp_id;

    DBMS_OUTPUT.PUT_LINE('Tên nhân viên: ' || v_emp_name);
    DBMS_OUTPUT.PUT_LINE('Số dòng trả về: ' || SQL%ROWCOUNT);
END;
/
```

**Input**: `v_emp_id = 100`  
**Output (ví dụ)**:
```text
Tên nhân viên: King
Số dòng trả về: 1
```

## 3. Explicit Cursor (Con trỏ tường minh)

Dùng khi:
- Câu `SELECT` trả về **nhiều dòng**.
- Cần **kiểm soát chi tiết**: OPEN, FETCH, CLOSE.

### Các bước

1. **Khai báo** cursor:
   ```sql
   CURSOR c_emp IS
       SELECT employee_id, last_name, salary
       FROM   employees
       WHERE  department_id = 10;
   ```
2. **OPEN**: thực thi SELECT, chuẩn bị tập kết quả.
3. **FETCH**: lấy từng dòng vào biến.
4. **CLOSE**: giải phóng tài nguyên.

### Ví dụ 2: Explicit cursor đầy đủ

```sql
DECLARE
    CURSOR c_emp IS
        SELECT employee_id, last_name, salary
        FROM   employees
        WHERE  department_id = 10;

    v_emp_id   employees.employee_id%TYPE;
    v_name     employees.last_name%TYPE;
    v_salary   employees.salary%TYPE;
BEGIN
    OPEN c_emp;

    LOOP
        FETCH c_emp INTO v_emp_id, v_name, v_salary;
        EXIT WHEN c_emp%NOTFOUND;

        DBMS_OUTPUT.PUT_LINE(
            'ID: ' || v_emp_id ||
            ', Tên: ' || v_name ||
            ', Lương: ' || v_salary
        );
    END LOOP;

    CLOSE c_emp;
END;
/
```

**Input**: `department_id = 10`  
**Output (ví dụ)**:
```text
ID: 200, Tên: Whalen, Lương: 4400
ID: 201, Tên: Hartstein, Lương: 13000
...
```

## 4. Cursor FOR Loop

Cú pháp rút gọn cho explicit cursor:
- Không cần `OPEN`, `FETCH`, `CLOSE` – PL/SQL tự làm.
- Mỗi vòng lặp trả về 1 record.

### 4.1. FOR loop với cursor đã khai báo

```sql
DECLARE
    CURSOR c_emp IS
        SELECT employee_id, last_name, salary
        FROM   employees
        WHERE  department_id = 10;
BEGIN
    FOR r_emp IN c_emp LOOP
        DBMS_OUTPUT.PUT_LINE(
            'ID: ' || r_emp.employee_id ||
            ', Tên: ' || r_emp.last_name ||
            ', Lương: ' || r_emp.salary
        );
    END LOOP;
END;
/
```

### 4.2. FOR loop với cursor ẩn danh (inline)

```sql
BEGIN
    FOR r_emp IN (
        SELECT employee_id, last_name, salary
        FROM   employees
        WHERE  department_id = 10
    ) LOOP
        DBMS_OUTPUT.PUT_LINE(
            'ID: ' || r_emp.employee_id ||
            ', Tên: ' || r_emp.last_name ||
            ', Lương: ' || r_emp.salary
        );
    END LOOP;
END;
/
```

**Output** tương tự ví dụ explicit cursor.

---

## 5. So sánh nhanh

| Loại cursor        | Khai báo | Điều khiển | Dùng khi nào?                                  |
|--------------------|----------|------------|-----------------------------------------------|
| Implicit cursor    | Không    | Tự động    | DML, `SELECT INTO` 1 dòng                     |
| Explicit cursor    | Có       | Thủ công   | Cần kiểm soát chi tiết, xử lý nhiều dòng      |
| Cursor FOR loop    | Có/Không | Tự động    | Duyệt nhiều dòng, code ngắn gọn, dễ đọc       |

Nếu bạn muốn, tôi có thể thêm ví dụ với cursor có tham số (parameterized cursor) hoặc ví dụ cập nhật dữ liệu trong cursor.
