## Ý chính

Cụm “What They Look For: Ability to reason about data volume and growth” nói về việc nhà tuyển dụng (hoặc người phỏng vấn) muốn đánh giá:

1. **Khả năng ước lượng dung lượng dữ liệu**  
   - Bạn có thể tính được bảng sẽ lớn đến mức nào (số dòng, dung lượng trên đĩa).
   - Bạn hiểu được tác động của kích thước dữ liệu đến hiệu năng (index, I/O, backup, v.v.).

2. **Khả năng dự đoán tốc độ tăng trưởng dữ liệu**  
   - Bạn biết dữ liệu tăng bao nhiêu mỗi ngày/tháng/năm.
   - Bạn có thể lập kế hoạch mở rộng (partitioning, thêm storage, tối ưu index, archive dữ liệu cũ).

3. **Khả năng đưa ra quyết định thiết kế dựa trên volume & growth**  
   - Chọn kiểu dữ liệu phù hợp.
   - Thiết kế partition (theo ngày/tháng/năm).
   - Lên chiến lược backup/archiving.

## Ví dụ: Ước lượng số dòng và dung lượng

Giả sử bạn có bảng giao dịch:

```sql
CREATE TABLE transactions (
    id            NUMBER,
    user_id       NUMBER,
    amount        NUMBER(10,2),
    status        VARCHAR2(20),
    created_at    DATE
);
```

### 1. Ước lượng số dòng

Giả sử hệ thống ghi nhận khoảng 100,000 giao dịch/ngày.

- 1 ngày: 100,000 dòng  
- 1 tháng (30 ngày): 3,000,000 dòng  
- 1 năm (365 ngày): 36,500,000 dòng  

Bạn có thể mô phỏng bằng một truy vấn đơn giản (chỉ để minh họa logic):

**Input (giả lập tính toán):**

```sql
WITH params AS (
    SELECT 100000 AS rows_per_day,
           365    AS days AS days_per_year
    FROM dual
)
SELECT rows_per_day,
       days_per_year,
       rows_per_day * days_per_year AS rows_per_year
FROM params;
```

**Output (kết quả logic):**

| ROWS_PER_DAY | DAYS_PER_YEAR | ROWS_PER_YEAR |
|--------------|---------------|---------------|
| 100000       | 365           | 36500000      |

### 2. Ước lượng dung lượng (rất đơn giản, mang tính minh họa)

Giả sử trung bình mỗi dòng ~ 100 bytes (con số thực tế phải tính chi tiết hơn).

- 36,500,000 dòng * 100 bytes ≈ 3.65 GB dữ liệu (chưa tính index, overhead, v.v.).

## Ví dụ: Kiểm tra kích thước bảng trong Oracle

Khi bảng đã có dữ liệu, bạn có thể dùng `DBA_SEGMENTS` (hoặc `USER_SEGMENTS`) để xem dung lượng:

```sql
SELECT segment_name,
       bytes / 1024 / 1024 AS size_mb
FROM   user_segments
WHERE  segment_name = 'TRANSACTIONS';
```

**Output (ví dụ):**

| SEGMENT_NAME | SIZE_MB |
|--------------|---------|
| TRANSACTIONS | 512     |

=> Bảng `TRANSACTIONS` đang chiếm khoảng 512 MB.

## Ví dụ: Thiết kế partition theo ngày/tháng

Khi bạn biết dữ liệu tăng rất nhanh, bạn có thể partition bảng theo ngày hoặc tháng để:

- Dễ quản lý (drop/archvie partition cũ).
- Cải thiện hiệu năng truy vấn theo thời gian.

```sql
CREATE TABLE transactions (
    id            NUMBER,
    user_id       NUMBER,
    amount        NUMBER(10,2),
    status        VARCHAR2(20),
    created_at    DATE
)
PARTITION BY RANGE (created_at) (
    PARTITION p_2025_01 VALUES LESS THAN (TO_DATE('2025-02-01','YYYY-MM-DD')),
    PARTITION p_2025_02 VALUES LESS THAN (TO_DATE('2025-03-01','YYYY-MM-DD'))
    -- thêm partition cho các tháng tiếp theo
);
```

## Tóm tắt

- “Ability to reason about data volume and growth” = khả năng:
  - Ước lượng số dòng và dung lượng dữ liệu.
  - Dự đoán tốc độ tăng trưởng theo thời gian.
  - Thiết kế cấu trúc DB (partition, index, storage, backup) dựa trên các ước lượng đó.
- Trong Oracle, bạn có thể:
  - Dùng logic/ước lượng để tính trước volume.
  - Dùng view hệ thống (`USER_SEGMENTS`, `DBA_SEGMENTS`) để kiểm tra dung lượng thực tế.
  - Thiết kế partition để xử lý dữ liệu lớn và tăng trưởng nhanh.
