
Dưới đây là tóm tắt ngắn gọn về **EXPLAIN PLAN** trong Oracle, kèm ví dụ code và kết quả minh họa.

---

## 1. Khái niệm chính

- `EXPLAIN PLAN` dùng để xem **kế hoạch thực thi dự kiến** của một câu SQL (SELECT/INSERT/UPDATE/DELETE).
- Nó **không thực thi** câu lệnh, chỉ ghi lại kế hoạch vào bảng `PLAN_TABLE`.
- Dùng để:
  - Hiểu Oracle sẽ truy cập dữ liệu như thế nào (FULL SCAN, INDEX RANGE SCAN, NESTED LOOPS, HASH JOIN…)
  - Tối ưu hiệu năng truy vấn (thêm index, chỉnh sửa SQL, hint, thống kê…)

---

## 2. Cú pháp cơ bản

```sql
EXPLAIN PLAN FOR
SELECT * 
FROM employees
WHERE department_id = 10;
```

Sau khi chạy, kế hoạch được lưu trong `PLAN_TABLE`. Để xem:

```sql
SELECT * 
FROM TABLE(DBMS_XPLAN.DISPLAY);
```

---

## 3. Ví dụ đầy đủ (có input/output)

### 3.1. Chuẩn bị dữ liệu (input)

```sql
-- Tạo bảng ví dụ
CREATE TABLE employees (
    employee_id   NUMBER PRIMARY KEY,
    first_name    VARCHAR2(50),
    last_name     VARCHAR2(50),
    department_id NUMBER,
    salary        NUMBER
);

-- Tạo index trên department_id
CREATE INDEX idx_emp_dept ON employees(department_id);

-- Chèn dữ liệu mẫu
INSERT INTO employees VALUES (1, 'A', 'Nguyen', 10, 1000);
INSERT INTO employees VALUES (2, 'B', 'Tran',   20, 2000);
INSERT INTO employees VALUES (3, 'C', 'Le',     10, 1500);
COMMIT;
```

### 3.2. EXPLAIN PLAN cho một truy vấn

```sql
EXPLAIN PLAN FOR
SELECT employee_id, first_name, salary
FROM employees
WHERE department_id = 10;
```

Xem kế hoạch:

```sql
SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

### 3.3. Kết quả minh họa (output)

Ví dụ output (rút gọn):

```text
Plan hash value: 123456789

---------------------------------------------------------
| Id  | Operation                   | Name          |
---------------------------------------------------------
|   0 | SELECT STATEMENT            |               |
|   1 |  TABLE ACCESS BY INDEX ROWID| EMPLOYEES     |
|   2 |   INDEX RANGE SCAN          | IDX_EMP_DEPT  |
---------------------------------------------------------
```

**Giải thích nhanh:**

- `INDEX RANGE SCAN IDX_EMP_DEPT`: Oracle dùng index `idx_emp_dept` để tìm các dòng có `department_id = 10`.
- `TABLE ACCESS BY INDEX ROWID`: Sau khi tìm được ROWID từ index, Oracle đọc dữ liệu thực trong bảng `EMPLOYEES`.

---

## 4. So sánh với FULL TABLE SCAN

Nếu không có index:

```sql
DROP INDEX idx_emp_dept;

EXPLAIN PLAN FOR
SELECT employee_id, first_name, salary
FROM employees
WHERE department_id = 10;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

Output có thể giống:

```text
---------------------------------------------------------
| Id  | Operation          | Name       |
---------------------------------------------------------
|   0 | SELECT STATEMENT   |            |
|   1 |  TABLE ACCESS FULL | EMPLOYEES  |
---------------------------------------------------------
```

- `TABLE ACCESS FULL`: Oracle quét toàn bộ bảng → tốn tài nguyên hơn khi bảng lớn.

---

## 5. EXPLAIN PLAN cho JOIN

```sql
EXPLAIN PLAN FOR
SELECT e.employee_id, e.first_name, d.department_name
FROM employees e
JOIN departments d
  ON e.department_id = d.department_id
WHERE d.location_id = 1000;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

Ví dụ output:

```text
--------------------------------------------------------------
| Id  | Operation           | Name        |
--------------------------------------------------------------
|   0 | SELECT STATEMENT    |             |
|   1 |  NESTED LOOPS       |             |
|   2 |   TABLE ACCESS FULL | DEPARTMENTS |
|   3 |   INDEX RANGE SCAN  | IDX_EMP_DEPT|
--------------------------------------------------------------
```

- `NESTED LOOPS`: kiểu join được dùng.
- Có thể thấy Oracle quét `DEPARTMENTS` rồi dùng index trên `EMPLOYEES`.

---

## 6. Một số lưu ý

- `EXPLAIN PLAN` cho biết **kế hoạch dự kiến**, không phải luôn 100% giống kế hoạch thực tế khi chạy (do bind variables, thống kê, adaptive plans…).
- Để xem kế hoạch thực tế đã dùng, có thể dùng:
  - `DBMS_XPLAN.DISPLAY_CURSOR` (với `SQL_ID` sau khi chạy câu lệnh).
- Thường dùng `DBMS_XPLAN.DISPLAY` với các tùy chọn:
  ```sql
  SELECT * 
  FROM TABLE(DBMS_XPLAN.DISPLAY(NULL, NULL, 'BASIC +PREDICATE'));
  ```

---

Tóm lại:  
`EXPLAIN PLAN FOR ...` + `DBMS_XPLAN.DISPLAY` là cách chuẩn trong Oracle để xem và phân tích kế hoạch thực thi, giúp tối ưu truy vấn SQL (chọn index, kiểu join, cách truy cập bảng…).
