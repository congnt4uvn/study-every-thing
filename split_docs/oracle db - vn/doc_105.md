
Dưới đây là tóm tắt ngắn gọn về chủ đề “Version Control & CI/CD: Rollback scripts” trong bối cảnh Oracle Database, kèm ví dụ code.

---

## 1. Ý chính

1. **Version Control cho database**  
   - Lưu trữ tất cả script SQL (tạo bảng, sửa bảng, tạo index, package, view, v.v.) trong hệ thống quản lý phiên bản (Git, SVN…).  
   - Mỗi thay đổi DB (migration) phải có **script forward** (deploy) và **script rollback** (undo).

2. **Rollback scripts là gì?**  
   - Là các script dùng để **hoàn tác** (revert) thay đổi database khi deploy lỗi.  
   - Được viết và lưu cùng với script deploy, gắn với cùng một version/revision.

3. **CI/CD cho Oracle Database**  
   - Pipeline CI/CD (Jenkins, GitLab CI, Azure DevOps…) sẽ:
     - Lấy code từ Git.
     - Chạy script migration (forward) lên môi trường (DEV → TEST → PROD).
     - Nếu lỗi hoặc cần revert, chạy script rollback tương ứng.
   - Có thể dùng các tool migration như **Liquibase**, **Flyway**, hoặc tự viết framework.

4. **Nguyên tắc khi viết rollback**  
   - Mỗi thay đổi phải có cách **quay lại trạng thái trước đó** (nếu có thể).  
   - Cẩn thận với dữ liệu: rollback có thể làm **mất dữ liệu** nếu không thiết kế kỹ.  
   - Test rollback trên môi trường DEV/TEST trước khi dùng ở PROD.

## 2. Ví dụ đơn giản: Script forward & rollback

### 2.1. Tạo bảng mới

**Forward script**: `V1__create_customer_table.sql`

```sql
-- Forward: tạo bảng CUSTOMER
CREATE TABLE CUSTOMER (
    ID          NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    NAME        VARCHAR2(100) NOT NULL,
    EMAIL       VARCHAR2(200),
    CREATED_AT  DATE DEFAULT SYSDATE
);
```

**Rollback script**: `V1__create_customer_table_rollback.sql`

```sql
-- Rollback: xóa bảng CUSTOMER
DROP TABLE CUSTOMER CASCADE CONSTRAINTS;
```

**Input (deploy)**: chạy forward script  
**Output**: bảng `CUSTOMER` được tạo.

**Input (rollback)**: chạy rollback script  
**Output**: bảng `CUSTOMER` bị xóa.

---

### 2.2. Thêm cột mới vào bảng

**Forward script**: `V2__add_status_to_customer.sql`

```sql
-- Forward: thêm cột STATUS
ALTER TABLE CUSTOMER
  ADD (STATUS VARCHAR2(20) DEFAULT 'ACTIVE' NOT NULL);
```

**Rollback script**: `V2__add_status_to_customer_rollback.sql`

```sql
-- Rollback: xóa cột STATUS
ALTER TABLE CUSTOMER
  DROP COLUMN STATUS;
```

## 3. Ví dụ CI/CD đơn giản (Jenkins + SQL*Plus)

### 3.1. Deploy (forward)

```bash
# Input: file forward.sql chứa các lệnh ALTER/CREATE
sqlplus user/password@//host:1521/ORCLPDB <<EOF
WHENEVER SQLERROR EXIT SQL.SQLCODE
@forward.sql
EXIT
EOF
```

**Output**:  
- Thành công: script chạy hết, Jenkins job “SUCCESS”.  
- Lỗi: SQL*Plus trả mã lỗi, Jenkins job “FAILED”.

### 3.2. Rollback khi lỗi

```bash
# Input: file rollback.sql tương ứng với version vừa deploy
sqlplus user/password@//host:1521/ORCLPDB <<EOF
WHENEVER SQLERROR EXIT SQL.SQLCODE
@rollback.sql
EXIT
EOF
```

**Output**:  
- Database quay về trạng thái trước thay đổi (trong phạm vi script rollback).

## 4. Ví dụ với Liquibase (Oracle)

**Forward changeLog**: `db.changelog-master.xml`

```xml
<databaseChangeLog
        xmlns="http://www.liquibase.org/xml/ns/dbchangelog"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xsi:schemaLocation="
          http://www.liquibase.org/xml/ns/dbchangelog
          http://www.liquibase.org/xml/ns/dbchangelog/dbchangelog-3.8.xsd">

    <changeSet id="1-create-customer" author="dev">
        <createTable tableName="CUSTOMER">
            <column name="ID" type="NUMBER">
                <constraints primaryKey="true" nullable="false"/>
            </column>
            <column name="NAME" type="VARCHAR2(100)">
                <constraints nullable="false"/>
            </column>
        </createTable>

        <!-- Rollback cho changeSet này -->
        <rollback>
            <dropTable tableName="CUSTOMER"/>
        </rollback>
    </changeSet>

</databaseChangeLog>
```

**Chạy forward**:

```bash
liquibase \
  --url=jdbc:oracle:thin:@//host:1521/ORCLPDB \
  --username=user \
  --password=pass \
  update
```

**Chạy rollback** (quay lại 1 changeSet):

```bash
liquibase \
  --url=jdbc:oracle:thin:@//host:1521/ORCLPDB \
  --username=user \
  --password=pass \
  rollbackCount 1
```

---

## 5. Tóm tắt ngắn

- Mọi thay đổi DB phải được quản lý bằng **version control** và có **script rollback** tương ứng.  
- CI/CD sẽ tự động chạy script forward; khi lỗi hoặc cần revert, chạy script rollback.  
- Cần test kỹ rollback, đặc biệt với thay đổi dữ liệu.  
- Có thể dùng tool như Liquibase/Flyway để quản lý version và rollback cho Oracle Database.
