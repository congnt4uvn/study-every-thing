
Dưới đây là tóm tắt ngắn gọn về **Materialized View** trong Oracle (thiết kế vật lý), kèm ví dụ mã và input/output.

---

## 1. Khái niệm chính

- **Materialized View (MV)** là một đối tượng lưu trữ **kết quả truy vấn** trên đĩa (giống bảng), không chỉ là định nghĩa truy vấn như view thường.
- Dùng để:
  - Tăng tốc truy vấn phức tạp (JOIN, GROUP BY, SUM, COUNT…).
  - Hỗ trợ **query rewrite**: Oracle tự động dùng MV thay cho bảng gốc nếu phù hợp.
  - Hữu ích trong **data warehouse**, báo cáo, tổng hợp dữ liệu.

---

## 2. Đặc điểm chính

- Dữ liệu trong MV **không tự động** luôn đồng bộ với bảng gốc:
  - Cần **refresh** (làm mới) theo lịch hoặc theo yêu cầu.
- Có thể:
  - `REFRESH FAST`: chỉ cập nhật phần thay đổi (cần log/constraint phù hợp).
  - `REFRESH COMPLETE`: tính lại toàn bộ.
  - `ON COMMIT`: refresh khi commit transaction.
  - `ON DEMAND`: refresh khi gọi lệnh.

---

## 3. Ví dụ cơ bản

### 3.1. Tạo bảng mẫu

```sql
-- Bảng đơn hàng
CREATE TABLE orders (
    order_id     NUMBER PRIMARY KEY,
    customer_id  NUMBER,
    order_date   DATE,
    amount       NUMBER
);

-- Dữ liệu mẫu
INSERT INTO orders VALUES (1, 101, DATE '2024-01-01', 100);
INSERT INTO orders VALUES (2, 101, DATE '2024-01-02', 200);
INSERT INTO orders VALUES (3, 102, DATE '2024-01-02', 150);
COMMIT;
```

**Input (bảng `orders`):**

| order_id | customer_id | order_date  | amount |
|----------|-------------|-------------|--------|
| 1        | 101         | 2024-01-01  | 100    |
| 2        | 101         | 2024-01-02  | 200    |
| 3        | 102         | 2024-01-02  | 150    |

---

### 3.2. Tạo Materialized View tổng hợp

Tạo MV tổng hợp doanh thu theo khách hàng:

```sql
CREATE MATERIALIZED VIEW mv_sales_by_customer
BUILD IMMEDIATE
REFRESH FAST ON DEMAND
ENABLE QUERY REWRITE
AS
SELECT
    customer_id,
    SUM(amount) AS total_amount,
    COUNT(*)    AS order_count
FROM orders
GROUP BY customer_id;
```

**Giải thích ngắn:**

- `BUILD IMMEDIATE`: xây dựng MV ngay lập tức.
- `REFRESH FAST`: cố gắng refresh nhanh (chỉ phần thay đổi).
- `ON DEMAND`: chỉ refresh khi gọi lệnh.
- `ENABLE QUERY REWRITE`: cho phép Oracle dùng MV để tối ưu truy vấn.

---

### 3.3. Xem dữ liệu trong MV

```sql
SELECT * FROM mv_sales_by_customer;
```

**Output:**

| customer_id | total_amount | order_count |
|-------------|--------------|-------------|
| 101         | 300          | 2           |
| 102         | 150          | 1           |

---

### 3.4. Cập nhật dữ liệu gốc và refresh MV

Thêm đơn hàng mới:

```sql
INSERT INTO orders VALUES (4, 101, DATE '2024-01-03', 50);
COMMIT;
```

Lúc này, nếu **chưa refresh**, MV vẫn giữ dữ liệu cũ.

Refresh MV:

```sql
EXEC DBMS_MVIEW.REFRESH('MV_SALES_BY_CUSTOMER', 'F');  -- 'F' = FAST
```

Xem lại MV:

```sql
SELECT * FROM mv_sales_by_customer;
```

**Output sau refresh:**

| customer_id | total_amount | order_count |
|-------------|--------------|-------------|
| 101         | 350          | 3           |
| 102         | 150          | 1           |

---

## 4. Ví dụ MV với REFRESH ON COMMIT

```sql
CREATE MATERIALIZED VIEW mv_sales_on_commit
BUILD IMMEDIATE
REFRESH FAST ON COMMIT
AS
SELECT
    customer_id,
    SUM(amount) AS total_amount
FROM orders
GROUP BY customer_id;
```

- Mỗi lần `COMMIT` thay đổi trên bảng `orders`, MV sẽ tự refresh.

---

## 5. Khi nào nên dùng Materialized View

- Truy vấn báo cáo phức tạp, chạy chậm nếu tính trực tiếp.
- Dữ liệu không cần real-time từng giây, chấp nhận độ trễ (theo lịch refresh).
- Muốn Oracle tự động tối ưu truy vấn thông qua **query rewrite**.

---

Nếu bạn gửi thêm nội dung chi tiết hơn (ví dụ: phần nói về partition, log, query rewrite…), tôi có thể tóm tắt sâu hơn và thêm các ví dụ nâng cao.
