
Dưới đây là tóm tắt ngắn gọn về chủ đề **Advanced PL/SQL: SAVE EXCEPTIONS** (Oracle Database), kèm ví dụ mã và input/output minh họa.

---

## 1. SAVE EXCEPTIONS là gì?

- Dùng trong **FORALL** để thực hiện **bulk DML** (INSERT/UPDATE/DELETE) trên nhiều phần tử của collection.
- Mặc định: nếu 1 bản ghi lỗi → cả câu FORALL dừng lại.
- Với `SAVE EXCEPTIONS`: PL/SQL **tiếp tục** xử lý các phần tử còn lại, và **lưu lại tất cả lỗi** vào `SQL%BULK_EXCEPTIONS`.

---

## 2. Cú pháp cơ bản

```plsql
FORALL i IN collection.FIRST .. collection.LAST SAVE EXCEPTIONS
    INSERT INTO table_name (col1, col2, ...)
    VALUES (collection(i).col1, collection(i).col2, ...);
```

- `SAVE EXCEPTIONS`: cho phép tiếp tục xử lý khi có lỗi.
- Sau khi FORALL kết thúc, nếu có lỗi, Oracle ném ra `ORA-24381: error(s) in array DML`.
- Chi tiết từng lỗi nằm trong `SQL%BULK_EXCEPTIONS`.

---

## 3. Truy cập lỗi trong SQL%BULK_EXCEPTIONS

`SQL%BULK_EXCEPTIONS` là một collection chứa:

- `SQL%BULK_EXCEPTIONS(i).ERROR_INDEX`  
  → chỉ số phần tử trong FORALL gây lỗi.
- `SQL%BULK_EXCEPTIONS(i).ERROR_CODE`  
  → mã lỗi Oracle (ví dụ: -1, -1400, -2291, …).

---

## 4. Ví dụ 1: INSERT với SAVE EXCEPTIONS

Giả sử có bảng:

```sql
CREATE TABLE employees_test (
    emp_id   NUMBER PRIMARY KEY,
    name     VARCHAR2(50) NOT NULL
);
```

### PL/SQL block

```plsql
DECLARE
    TYPE emp_rec IS RECORD (
        emp_id NUMBER,
        name   VARCHAR2(50)
    );
    TYPE emp_tab IS TABLE OF emp_rec INDEX BY PLS_INTEGER;

    l_emps emp_tab;
BEGIN
    -- Input: chuẩn bị dữ liệu (có lỗi cố ý)
    l_emps(1).emp_id := 1;  l_emps(1).name := 'A';      -- OK
    l_emps(2).emp_id := 1;  l_emps(2).name := 'B';      -- Lỗi: trùng PK
    l_emps(3).emp_id := 2;  l_emps(3).name := NULL;     -- Lỗi: NOT NULL
    l_emps(4).emp_id := 3;  l_emps(4).name := 'C';      -- OK

    FORALL i IN l_emps.FIRST .. l_emps.LAST SAVE EXCEPTIONS
        INSERT INTO employees_test (emp_id, name)
        VALUES (l_emps(i).emp_id, l_emps(i).name);

EXCEPTION
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('Tổng số lỗi: ' || SQL%BULK_EXCEPTIONS.COUNT);

        FOR j IN 1 .. SQL%BULK_EXCEPTIONS.COUNT LOOP
            DBMS_OUTPUT.PUT_LINE(
                  'Lỗi thứ ' || j
               || ' - INDEX: ' || SQL%BULK_EXCEPTIONS(j).ERROR_INDEX
               || ' - CODE: '  || SQL%BULK_EXCEPTIONS(j).ERROR_CODE
               || ' - MSG: '   || SQLERRM(-SQL%BULK_EXCEPTIONS(j).ERROR_CODE)
            );
        END LOOP;
END;
/
```

### Kết quả (output DBMS_OUTPUT dự kiến)

- Bản ghi 1: chèn thành công.
- Bản ghi 2: lỗi trùng khóa chính (ORA-00001).
- Bản ghi 3: lỗi NOT NULL (ORA-01400).
- Bản ghi 4: chèn thành công.

Ví dụ output:

```text
Tổng số lỗi: 2
Lỗi thứ 1 - INDEX: 2 - CODE: 1 - MSG: ORA-00001: unique constraint violated
Lỗi thứ 2 - INDEX: 3 - CODE: 1400 - MSG: ORA-01400: cannot insert NULL into ("EMPLOYEES_TEST"."NAME")
```

Dữ liệu trong bảng `employees_test` sau khi chạy:

```sql
SELECT * FROM employees_test ORDER BY emp_id;
```

Output:

```text
EMP_ID | NAME
------+------
1      | A
3      | C
```

## 5. Ví dụ 2: UPDATE với SAVE EXCEPTIONS

```plsql
DECLARE
    TYPE id_tab IS TABLE OF NUMBER INDEX BY PLS_INTEGER;
    l_ids id_tab;
BEGIN
    -- Input: giả sử chỉ có emp_id = 1,3 tồn tại
    l_ids(1) := 1;  -- OK
    l_ids(2) := 2;  -- Không tồn tại → không lỗi, chỉ 0 row updated
    l_ids(3) := 3;  -- OK

    FORALL i IN l_ids.FIRST .. l_ids.LAST SAVE EXCEPTIONS
        UPDATE employees_test
        SET name = name || '_UPDATED'
        WHERE emp_id = l_ids(i);

    DBMS_OUTPUT.PUT_LINE('SQL%ROWCOUNT = ' || SQL%ROWCOUNT);

EXCEPTION
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('Tổng số lỗi: ' || SQL%BULK_EXCEPTIONS.COUNT);
END;
/
```

- Ở đây nếu không có ràng buộc gây lỗi, `SAVE EXCEPTIONS` không tạo lỗi.
- `SQL%ROWCOUNT` = tổng số dòng thực sự được update (2).

---

## 6. Khi nào nên dùng SAVE EXCEPTIONS?

- Khi thực hiện **bulk DML** trên nhiều bản ghi và:
  - Muốn **tối đa hóa số bản ghi xử lý thành công**.
  - Cần **ghi nhận chi tiết các bản ghi lỗi** để xử lý sau (log, sửa dữ liệu, báo cáo).
- Thường dùng trong:
  - ETL, batch processing.
  - Import dữ liệu lớn, trong đó một số bản ghi có thể sai.

---

## 7. Mẫu xử lý lỗi chuẩn với SAVE EXCEPTIONS

```plsql
BEGIN
    FORALL i IN l_data.FIRST .. l_data.LAST SAVE EXCEPTIONS
        INSERT INTO target_table (...)
        VALUES (...);

EXCEPTION
    WHEN OTHERS THEN
        IF SQLCODE = -24381 THEN  -- ORA-24381: error(s) in array DML
            FOR j IN 1 .. SQL%BULK_EXCEPTIONS.COUNT LOOP
                -- Ghi log từng lỗi
                INSERT INTO error_log (
                    err_index,
                    err_code,
                    err_msg
                ) VALUES (
                    SQL%BULK_EXCEPTIONS(j).ERROR_INDEX,
                    SQL%BULK_EXCEPTIONS(j).ERROR_CODE,
                    SQLERRM(-SQL%BULK_EXCEPTIONS(j).ERROR_CODE)
                );
            END LOOP;
        ELSE
            RAISE; -- các lỗi khác
        END IF;
END;
/
```

---

Tóm lại, `SAVE EXCEPTIONS` là kỹ thuật quan trọng trong **Advanced PL/SQL** để xử lý **bulk DML** một cách linh hoạt, cho phép bỏ qua các bản ghi lỗi nhưng vẫn ghi nhận đầy đủ thông tin lỗi để xử lý sau.
