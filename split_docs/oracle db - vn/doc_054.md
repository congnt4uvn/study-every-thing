
Dưới đây là tóm tắt ngắn gọn bằng tiếng Việt, kèm ví dụ:

## 1. Tránh dùng DISTINCT không cần thiết

**Ý chính**  
- `DISTINCT` buộc Oracle phải loại bỏ các dòng trùng lặp (thường phải sort hoặc dùng hash), gây tốn CPU, I/O và thời gian.  
- Nếu dữ liệu đã đảm bảo không trùng (do khóa chính, ràng buộc unique, logic nghiệp vụ…), thì không cần `DISTINCT`.

**Ví dụ 1 – Không cần DISTINCT**

Giả sử `EMP` có khóa chính `EMP_ID`, nên mỗi `EMP_ID` là duy nhất.

```sql
-- Câu truy vấn kém tối ưu
SELECT DISTINCT EMP_ID, EMP_NAME
FROM EMP
WHERE DEPT_ID = 10;
```

Vì `EMP_ID` là khóa chính, mỗi dòng đã là duy nhất → bỏ `DISTINCT`:

```sql
-- Câu truy vấn tốt hơn
SELECT EMP_ID, EMP_NAME
FROM EMP
WHERE DEPT_ID = 10;
```

**Kết quả (ví dụ)**

| EMP_ID | EMP_NAME   |
|--------|------------|
| 100    | Nguyen An  |
| 101    | Tran Binh  |

Cả hai câu đều cho kết quả giống nhau, nhưng câu thứ hai nhanh hơn vì không phải loại bỏ trùng.

**Ví dụ 2 – DISTINCT thực sự cần thiết**

Nếu join có thể tạo dòng trùng:

```sql
SELECT DISTINCT C.CUST_ID, C.CUST_NAME
FROM CUSTOMERS C
JOIN ORDERS O ON C.CUST_ID = O.CUST_ID;
```

Ở đây, một khách hàng có nhiều đơn hàng → có thể lặp lại `CUST_ID`, `CUST_NAME`. `DISTINCT` là hợp lý nếu bạn chỉ muốn danh sách khách hàng duy nhất.

## 2. Dùng UNION ALL khi có thể (tránh UNION nếu không cần loại trùng)

**Ý chính**  
- `UNION` = `UNION ALL` + loại bỏ trùng (sort/hash) → chậm hơn.  
- Nếu bạn **không cần** loại bỏ các dòng trùng giữa hai tập kết quả, hãy dùng `UNION ALL`.

**Ví dụ 3 – Dùng UNION (có loại trùng)**

```sql
SELECT EMP_ID FROM EMP_2023
UNION
SELECT EMP_ID FROM EMP_2024;
```

- Kết quả: danh sách `EMP_ID` duy nhất từ cả hai bảng.  
- Oracle phải loại bỏ trùng → tốn tài nguyên.

**Ví dụ 4 – Dùng UNION ALL (không loại trùng)**

```sql
SELECT EMP_ID FROM EMP_2023
UNION ALL
SELECT EMP_ID FROM EMP_2024;
```

- Kết quả: tất cả `EMP_ID` từ 2023 và 2024, **có thể trùng**.  
- Nhanh hơn vì không phải loại bỏ trùng.

**Ví dụ Input/Output**

Bảng `EMP_2023`:

| EMP_ID |
|--------|
| 100    |
| 101    |

Bảng `EMP_2024`:

| EMP_ID |
|--------|
| 101    |
| 102    |

- Với `UNION`:

```sql
SELECT EMP_ID FROM EMP_2023
UNION
SELECT EMP_ID FROM EMP_2024;
```

Kết quả:

| EMP_ID |
|--------|
| 100    |
| 101    |
| 102    |

- Với `UNION ALL`:

```sql
SELECT EMP_ID FROM EMP_2023
UNION ALL
SELECT EMP_ID FROM EMP_2024;
```

Kết quả:

| EMP_ID |
|--------|
| 100    |
| 101    |
| 101    |  -- trùng
| 102    |

---

## 3. Tóm tắt kỹ thuật tối ưu

1. **DISTINCT**  
   - Chỉ dùng khi **thực sự cần loại bỏ trùng**.  
   - Nếu bảng/column đã đảm bảo unique → bỏ `DISTINCT`.

2. **UNION vs UNION ALL**  
   - Dùng `UNION` khi cần **kết quả duy nhất** (loại trùng).  
   - Dùng `UNION ALL` khi **không cần loại trùng** → hiệu năng tốt hơn.

Những điều chỉnh nhỏ này thường giúp giảm thời gian chạy query đáng kể trong Oracle Database.
