
Dưới đây là tóm tắt ngắn gọn + ví dụ code Oracle về việc tìm “khoảng trống” (gaps) trong chuỗi ngày và chuỗi ID.

---

## Ý chính

- “Gaps” là các giá trị bị thiếu trong một chuỗi liên tiếp:
  - Ngày bị thiếu trong một khoảng thời gian.
  - ID bị thiếu trong một dãy số (1,2,3,…).
- Trong Oracle, thường dùng:
  - `CONNECT BY` để sinh dãy số hoặc dãy ngày.
  - `MINUS` hoặc `LEFT JOIN` để so sánh dãy chuẩn với dữ liệu thực tế.
  - Hoặc dùng `LAG`/`LEAD` để phát hiện khoảng trống giữa các bản ghi liên tiếp.

---

## 1. Tìm ngày bị thiếu (missing dates)

Giả sử có bảng:

```sql
CREATE TABLE sales (
    sale_date DATE,
    amount    NUMBER
);
```

### Sinh dãy ngày và tìm ngày không có dữ liệu

```sql
-- Giả sử khoảng thời gian cần kiểm tra
VAR start_date VARCHAR2(10);
VAR end_date   VARCHAR2(10);

EXEC :start_date := '2024-01-01';
EXEC :end_date   := '2024-01-10';

-- Tìm các ngày không có bản ghi trong bảng SALES
WITH all_dates AS (
    SELECT TO_DATE(:start_date, 'YYYY-MM-DD') + LEVEL - 1 AS dt
    FROM   dual
    CONNECT BY LEVEL <= TO_DATE(:end_date, 'YYYY-MM-DD')
                      - TO_DATE(:start_date, 'YYYY-MM-DD') + 1
),
existing_dates AS (
    SELECT DISTINCT TRUNC(sale_date) AS dt
    FROM   sales
    WHERE  sale_date BETWEEN TO_DATE(:start_date, 'YYYY-MM-DD')
                         AND TO_DATE(:end_date, 'YYYY-MM-DD')
)
SELECT dt AS missing_date
FROM   all_dates
MINUS
SELECT dt
FROM   existing_dates
ORDER BY missing_date;
```

**Input (ví dụ):**

- Bảng `sales` có dữ liệu:
  - 2024-01-01, 2024-01-02, 2024-01-04, 2024-01-07
- `start_date = '2024-01-01'`, `end_date = '2024-01-07'`

**Output:**

```text
MISSING_DATE
------------
2024-01-03
2024-01-05
2024-01-06
```

## 2. Tìm ID bị thiếu (missing IDs)

Giả sử có bảng:

```sql
CREATE TABLE my_table (
    id   NUMBER PRIMARY KEY,
    data VARCHAR2(100)
);
```

### Cách 1: Dùng CONNECT BY + MINUS

```sql
WITH id_range AS (
    SELECT MIN(id) AS min_id, MAX(id) AS max_id
    FROM   my_table
),
all_ids AS (
    SELECT min_id + LEVEL - 1 AS id
    FROM   id_range
    CONNECT BY LEVEL <= (max_id - min_id + 1)
),
existing_ids AS (
    SELECT id FROM my_table
)
SELECT id AS missing_id
FROM   all_ids
MINUS
SELECT id
FROM   existing_ids
ORDER BY missing_id;
```

**Input (ví dụ):**

- Bảng `my_table` có ID: 1, 2, 4, 7

**Output:**

```text
MISSING_ID
----------
3
5
6
```

---

### Cách 2: Dùng LAG để tìm khoảng trống giữa các ID

```sql
SELECT (id_prev + 1) AS gap_start,
       (id - 1)      AS gap_end
FROM (
    SELECT id,
           LAG(id) OVER (ORDER BY id) AS id_prev
    FROM   my_table
) t
WHERE id_prev IS NOT NULL
  AND id_prev + 1 <> id
ORDER BY gap_start;
```

**Input (ví dụ):**

- Bảng `my_table` có ID: 1, 2, 4, 7

**Output:**

```text
GAP_START  GAP_END
---------  -------
3          3
5          6
```

- Nghĩa là:
  - Thiếu ID 3
  - Thiếu ID 5–6

---

## Tóm tắt

- Để tìm ngày/ID bị thiếu:
  1. Sinh dãy đầy đủ (ngày hoặc số) bằng `CONNECT BY`.
  2. So sánh với dữ liệu thực tế bằng `MINUS` hoặc `LEFT JOIN`.
  3. Hoặc dùng `LAG`/`LEAD` để tìm khoảng trống giữa các bản ghi liên tiếp.
