
Dưới đây là tóm tắt ngắn gọn về các loại index trong Oracle và ví dụ minh họa.

---

## 1. B-tree Index
- Dùng phổ biến nhất.
- Tối ưu cho:
  - Cột có tính chọn lọc cao (nhiều giá trị khác nhau).
  - Truy vấn dạng `=`, `>`, `<`, `BETWEEN`, `ORDER BY`.
- Phù hợp cho OLTP (giao dịch nhiều, cập nhật thường xuyên).

**Ví dụ:**

```sql
-- Tạo bảng mẫu
CREATE TABLE employees (
    employee_id   NUMBER PRIMARY KEY,
    first_name    VARCHAR2(50),
    last_name     VARCHAR2(50),
    department_id NUMBER,
    salary        NUMBER
);

-- Tạo B-tree index trên cột last_name
CREATE INDEX idx_emp_lastname
ON employees(last_name);
```

**Truy vấn sử dụng index (ví dụ):**

```sql
EXPLAIN PLAN FOR
SELECT * FROM employees
WHERE last_name = 'Nguyen';

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

Kết quả (rút gọn) thường sẽ có dòng:
```text
|*  1 |  INDEX RANGE SCAN | IDX_EMP_LASTNAME |
```

---

## 2. Bitmap Index
- Lưu trữ giá trị cột dưới dạng bitmap (bit 0/1).
- Tối ưu cho:
  - Cột có ít giá trị khác nhau (low cardinality), ví dụ: giới tính, trạng thái, cờ Y/N.
  - Hệ thống OLAP, data warehouse (ít cập nhật, nhiều truy vấn phân tích).
- Không phù hợp cho bảng cập nhật thường xuyên (INSERT/UPDATE/DELETE nhiều).

**Ví dụ:**

```sql
CREATE TABLE sales (
    sale_id     NUMBER PRIMARY KEY,
    customer_id NUMBER,
    status      VARCHAR2(10),  -- 'NEW', 'PAID', 'CANCEL'
    region      VARCHAR2(10)   -- 'NORTH', 'SOUTH', 'EAST', 'WEST'
);

-- Tạo bitmap index trên cột status
CREATE BITMAP INDEX idx_sales_status
ON sales(status);
```

**Truy vấn:**

```sql
SELECT status, COUNT(*)
FROM sales
GROUP BY status;
```

Bitmap index giúp tăng tốc các truy vấn thống kê, lọc theo nhiều cột low-cardinality.

---

## 3. Function-based Index
- Index trên biểu thức/hàm thay vì cột thô.
- Tối ưu khi truy vấn thường xuyên dùng hàm trên cột (UPPER, SUBSTR, tính toán…).

**Ví dụ:**

```sql
-- Index trên UPPER(last_name)
CREATE INDEX idx_emp_upper_lastname
ON employees(UPPER(last_name));
```

**Truy vấn sử dụng index:**

```sql
SELECT *
FROM employees
WHERE UPPER(last_name) = 'NGUYEN';
```

Nếu không có function-based index, Oracle khó dùng index khi có hàm trên cột.

---

## 4. Unique Index
- Đảm bảo giá trị trong cột (hoặc nhóm cột) là duy nhất.
- Thường được tạo tự động khi tạo `PRIMARY KEY` hoặc `UNIQUE` constraint.
- Dùng để:
  - Đảm bảo tính toàn vẹn dữ liệu.
  - Tăng tốc truy vấn tìm kiếm theo khóa.

**Ví dụ:**

```sql
-- Tạo unique index trực tiếp
CREATE UNIQUE INDEX idx_emp_email
ON employees(email);

-- Hoặc qua constraint (Oracle sẽ tạo unique index)
ALTER TABLE employees
ADD CONSTRAINT uq_emp_email UNIQUE (email);
```

**Thử insert trùng (ví dụ lỗi):**

```sql
INSERT INTO employees (employee_id, email)
VALUES (1, 'a@company.com');

INSERT INTO employees (employee_id, email)
VALUES (2, 'a@company.com');  -- Sẽ lỗi ORA-00001: unique constraint violated
```

## 5. Composite Index (Multi-column Index)
- Index trên nhiều cột.
- Tối ưu cho truy vấn lọc theo nhiều cột cùng lúc.
- Quy tắc quan trọng: thứ tự cột trong index rất quan trọng.
  - Index `(col1, col2)` hỗ trợ tốt:
    - `WHERE col1 = ...`
    - `WHERE col1 = ... AND col2 = ...`
  - Không tối ưu cho `WHERE col2 = ...` một mình (trừ một số trường hợp đặc biệt).

**Ví dụ:**

```sql
CREATE INDEX idx_emp_dept_salary
ON employees(department_id, salary);
```

**Truy vấn tận dụng index tốt:**

```sql
-- 1. Dùng department_id
SELECT *
FROM employees
WHERE department_id = 10;

-- 2. Dùng department_id và salary
SELECT *
FROM employees
WHERE department_id = 10
  AND salary > 2000;
```

Truy vấn chỉ dùng `salary` mà không có `department_id` sẽ ít tận dụng được index này.

---

## Tóm tắt nhanh

- **B-tree index**: chuẩn, dùng cho OLTP, cột có nhiều giá trị khác nhau, truy vấn =, >, <, BETWEEN.
- **Bitmap index**: cho cột ít giá trị, hệ thống phân tích (OLAP), không phù hợp nhiều cập nhật.
- **Function-based index**: index trên biểu thức/hàm, giúp tối ưu truy vấn có dùng hàm.
- **Unique index**: đảm bảo giá trị duy nhất, thường gắn với PRIMARY KEY/UNIQUE.
- **Composite index**: index nhiều cột, cần thiết kế đúng thứ tự cột theo mẫu truy vấn thực tế.
