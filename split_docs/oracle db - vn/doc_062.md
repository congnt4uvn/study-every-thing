
Dưới đây là tóm tắt ngắn gọn về thiết kế quan hệ trong Oracle với **PRIMARY KEY**, **FOREIGN KEY**, **UNIQUE** và ví dụ mã.

---

## 1. PRIMARY KEY (Khóa chính)

- Xác định duy nhất mỗi dòng trong bảng.  
- Không được `NULL`, không được trùng lặp.  
- Mỗi bảng thường chỉ có **1** khóa chính (có thể gồm nhiều cột – composite key).

### Ví dụ

```sql
-- Tạo bảng với khóa chính đơn cột
CREATE TABLE customers (
    customer_id   NUMBER        CONSTRAINT pk_customers PRIMARY KEY,
    name          VARCHAR2(100) NOT NULL,
    email         VARCHAR2(100)
);

-- Tạo bảng với khóa chính nhiều cột
CREATE TABLE order_items (
    order_id   NUMBER,
    line_no    NUMBER,
    product_id NUMBER,
    quantity   NUMBER,
    CONSTRAINT pk_order_items PRIMARY KEY (order_id, line_no)
);
```

**Input thử:**

```sql
INSERT INTO customers (customer_id, name) VALUES (1, 'A');
INSERT INTO customers (customer_id, name) VALUES (1, 'B');  -- trùng khóa chính
```

**Output:**

- Dòng thứ hai sẽ lỗi:  
  `ORA-00001: unique constraint (PK_CUSTOMERS) violated`

---

## 2. UNIQUE Constraint (Ràng buộc duy nhất)

- Đảm bảo giá trị trong cột (hoặc nhóm cột) là **duy nhất**.  
- Khác với PRIMARY KEY: cột UNIQUE **có thể chứa NULL** (nhiều NULL tùy cấu hình, nhưng Oracle cho phép nhiều NULL).

### Ví dụ

```sql
CREATE TABLE users (
    user_id   NUMBER        CONSTRAINT pk_users PRIMARY KEY,
    username  VARCHAR2(50)  CONSTRAINT uq_users_username UNIQUE,
    email     VARCHAR2(100) CONSTRAINT uq_users_email UNIQUE
);
```

**Input thử:**

```sql
INSERT INTO users (user_id, username, email)
VALUES (1, 'user1', 'u1@example.com');

INSERT INTO users (user_id, username, email)
VALUES (2, 'user1', 'u2@example.com');  -- trùng username
```

**Output:**

- Dòng thứ hai sẽ lỗi:  
  `ORA-00001: unique constraint (UQ_USERS_USERNAME) violated`

## 3. FOREIGN KEY (Khóa ngoại)

- Tạo mối quan hệ giữa bảng con và bảng cha.  
- Giá trị khóa ngoại phải tồn tại trong bảng cha (hoặc là NULL nếu cho phép).  
- Dùng để đảm bảo **toàn vẹn tham chiếu**.

### Ví dụ

```sql
-- Bảng cha
CREATE TABLE departments (
    dept_id   NUMBER        CONSTRAINT pk_departments PRIMARY KEY,
    dept_name VARCHAR2(100) NOT NULL
);

-- Bảng con
CREATE TABLE employees (
    emp_id    NUMBER        CONSTRAINT pk_employees PRIMARY KEY,
    emp_name  VARCHAR2(100) NOT NULL,
    dept_id   NUMBER,
    CONSTRAINT fk_employees_dept
        FOREIGN KEY (dept_id)
        REFERENCES departments (dept_id)
);
```

**Input thử:**

```sql
INSERT INTO departments (dept_id, dept_name)
VALUES (10, 'IT');

INSERT INTO employees (emp_id, emp_name, dept_id)
VALUES (1, 'Nam', 10);   -- OK, dept_id=10 tồn tại

INSERT INTO employees (emp_id, emp_name, dept_id)
VALUES (2, 'Lan', 20);   -- dept_id=20 chưa tồn tại
```

**Output:**

- Dòng thứ ba sẽ lỗi:  
  `ORA-02291: integrity constraint (FK_EMPLOYEES_DEPT) violated - parent key not found`

---

## 4. Tóm tắt ý chính

- **PRIMARY KEY**:  
  - Xác định duy nhất bản ghi, không NULL, không trùng.  
  - Thường dùng để định danh bản ghi (ID).

- **UNIQUE**:  
  - Đảm bảo giá trị không trùng, nhưng cho phép NULL.  
  - Thường dùng cho các cột như `username`, `email`, `code`.

- **FOREIGN KEY**:  
  - Liên kết bảng con với bảng cha, đảm bảo giá trị tham chiếu hợp lệ.  
  - Giữ toàn vẹn dữ liệu giữa các bảng (ví dụ: `employees.dept_id` tham chiếu `departments.dept_id`).
