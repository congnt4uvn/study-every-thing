
Dưới đây là tóm tắt ngắn gọn về **Referential Integrity** và **Cascading** trong thiết kế quan hệ (Oracle Database), kèm ví dụ mã.

---

## 1. Referential Integrity (Toàn vẹn tham chiếu)

**Ý chính:**
- Đảm bảo mối quan hệ giữa các bảng luôn hợp lệ.
- Một giá trị khóa ngoại (FOREIGN KEY) phải:
  - Hoặc trùng với một giá trị khóa chính (PRIMARY KEY) / UNIQUE ở bảng cha.
  - Hoặc là `NULL` (nếu được phép).

**Ví dụ:**

```sql
-- Bảng cha: DEPARTMENTS
CREATE TABLE departments (
    dept_id   NUMBER PRIMARY KEY,
    dept_name VARCHAR2(100) NOT NULL
);

-- Bảng con: EMPLOYEES
CREATE TABLE employees (
    emp_id    NUMBER PRIMARY KEY,
    emp_name  VARCHAR2(100) NOT NULL,
    dept_id   NUMBER,
    CONSTRAINT fk_emp_dept
        FOREIGN KEY (dept_id)
        REFERENCES departments(dept_id)
);
```

### Ví dụ dữ liệu (Input/Output)

```sql
-- Thêm phòng ban (OK)
INSERT INTO departments (dept_id, dept_name)
VALUES (10, 'IT');

-- Thêm nhân viên tham chiếu dept_id = 10 (OK)
INSERT INTO employees (emp_id, emp_name, dept_id)
VALUES (1, 'An', 10);

-- Thêm nhân viên tham chiếu dept_id = 99 (KHÔNG TỒN TẠI) -> LỖI
INSERT INTO employees (emp_id, emp_name, dept_id)
VALUES (2, 'Bình', 99);
```

**Output lỗi (mô tả):**
- ORA-02291: integrity constraint (FK_EMP_DEPT) violated - parent key not found

---

## 2. Cascading (Hành vi dây chuyền)

**Ý chính:**
- Xác định hành vi khi bản ghi ở bảng cha bị **xóa** hoặc **cập nhật**.
- Các tùy chọn phổ biến:
  - `ON DELETE CASCADE`: xóa luôn các bản ghi con liên quan.
  - `ON DELETE SET NULL`: đặt khóa ngoại ở bảng con thành `NULL`.
  - (Oracle không hỗ trợ `ON UPDATE CASCADE` trực tiếp như một số DB khác.)

### 2.1. ON DELETE CASCADE

```sql
CREATE TABLE employees (
    emp_id    NUMBER PRIMARY KEY,
    emp_name  VARCHAR2(100) NOT NULL,
    dept_id   NUMBER,
    CONSTRAINT fk_emp_dept
        FOREIGN KEY (dept_id)
        REFERENCES departments(dept_id)
        ON DELETE CASCADE
);
```

**Input/Output minh họa:**

```sql
-- Dữ liệu mẫu
INSERT INTO departments (dept_id, dept_name) VALUES (20, 'HR');
INSERT INTO employees (emp_id, emp_name, dept_id) VALUES (3, 'Cường', 20);
INSERT INTO employees (emp_id, emp_name, dept_id) VALUES (4, 'Dũng', 20);

-- Xóa phòng ban 20
DELETE FROM departments WHERE dept_id = 20;
```

**Kết quả (Output):**
- Hàng có `dept_id = 20` trong `departments` bị xóa.
- Tất cả nhân viên có `dept_id = 20` trong `employees` cũng bị xóa tự động.

---

### 2.2. ON DELETE SET NULL

```sql
CREATE TABLE employees (
    emp_id    NUMBER PRIMARY KEY,
    emp_name  VARCHAR2(100) NOT NULL,
    dept_id   NUMBER,
    CONSTRAINT fk_emp_dept
        FOREIGN KEY (dept_id)
        REFERENCES departments(dept_id)
        ON DELETE SET NULL
);
```

**Input/Output minh họa:**

```sql
-- Dữ liệu mẫu
INSERT INTO departments (dept_id, dept_name) VALUES (30, 'Finance');
INSERT INTO employees (emp_id, emp_name, dept_id) VALUES (5, 'Hà', 30);

-- Xóa phòng ban 30
DELETE FROM departments WHERE dept_id = 30;
```

**Kết quả (Output):**
- Hàng `dept_id = 30` trong `departments` bị xóa.
- Trong `employees`, bản ghi emp_id = 5 sẽ có `dept_id` được đặt thành `NULL`.

---

## 3. Ý nghĩa trong thiết kế

- **Referential integrity**: tránh dữ liệu “mồ côi” (orphan records), đảm bảo tính nhất quán.
- **Cascading**: giúp tự động hóa xử lý khi xóa/cập nhật dữ liệu ở bảng cha, giảm lỗi thủ công, nhưng cần thiết kế cẩn thận để tránh xóa dây chuyền ngoài ý muốn.

Nếu bạn cho thêm nội dung chi tiết hơn (ví dụ đoạn tài liệu cụ thể), tôi có thể tóm tắt sâu hơn theo đúng nội dung đó.
