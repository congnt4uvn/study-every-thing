
Dưới đây là tóm tắt ngắn gọn về **Query Rewrite** trong Oracle, kèm ví dụ mã và input/output minh họa.

---

## 1. Khái niệm Query Rewrite

- **Query Rewrite** là cơ chế của Oracle tự động **thay đổi (rewrite)** câu SQL do người dùng gửi, để:
  - Tận dụng **materialized view** (MV) hoặc các cấu trúc đã được tính sẵn.
  - Giảm chi phí truy vấn, tăng hiệu năng.
- Người dùng vẫn viết câu SQL bình thường; Oracle tự quyết định có rewrite hay không.

---

## 2. Điều kiện để Query Rewrite hoạt động

1. **Materialized View** phải:
   - Được tạo với `ENABLE QUERY REWRITE`.
   - Được `REFRESH` và ở trạng thái `FRESH` (dữ liệu mới).
2. Câu SQL gốc phải **logic tương đương** với dữ liệu trong MV.
3. Tham số hệ thống:
   - `QUERY_REWRITE_ENABLED = TRUE` (hoặc `FORCE`).
   - `QUERY_REWRITE_INTEGRITY` phù hợp (`ENFORCED`, `TRUSTED`, `STALE_TOLERATED`).

---

## 3. Ví dụ cơ bản với Materialized View

### 3.1. Tạo bảng dữ liệu

```sql
CREATE TABLE sales (
    sale_id      NUMBER,
    sale_date    DATE,
    product_id   NUMBER,
    amount       NUMBER
);

-- Dữ liệu mẫu
INSERT INTO sales VALUES (1, DATE '2024-01-01', 101, 100);
INSERT INTO sales VALUES (2, DATE '2024-01-01', 102, 200);
INSERT INTO sales VALUES (3, DATE '2024-01-02', 101, 150);
COMMIT;
```

### 3.2. Tạo Materialized View cho tổng doanh thu theo ngày

```sql
CREATE MATERIALIZED VIEW mv_sales_daily
BUILD IMMEDIATE
REFRESH FAST ON COMMIT
ENABLE QUERY REWRITE
AS
SELECT
    sale_date,
    SUM(amount) AS total_amount
FROM sales
GROUP BY sale_date;
```

- MV này lưu sẵn tổng `amount` theo `sale_date`.

### 3.3. Câu truy vấn người dùng

**Input (SQL người dùng):**

```sql
SELECT
    sale_date,
    SUM(amount) AS total_amount
FROM sales
GROUP BY sale_date;
```

**Hành vi Query Rewrite:**

- Oracle nhận thấy câu SQL này **tương đương** với định nghĩa của `mv_sales_daily`.
- Nếu `QUERY_REWRITE_ENABLED = TRUE`, Oracle có thể rewrite thành:

```sql
SELECT
    sale_date,
    total_amount
FROM mv_sales_daily;
```

**Output (kết quả trả về):**

| SALE_DATE   | TOTAL_AMOUNT |
|------------|--------------|
| 2024-01-01 | 300          |
| 2024-01-02 | 150          |

Kết quả giống nhau, nhưng truy vấn dùng MV sẽ nhanh hơn vì dữ liệu đã được tổng hợp sẵn.

---

## 4. Kiểm tra Query Rewrite có được dùng hay không

Dùng `EXPLAIN PLAN` hoặc `DBMS_XPLAN`:

```sql
EXPLAIN PLAN FOR
SELECT
    sale_date,
    SUM(amount) AS total_amount
FROM sales
GROUP BY sale_date;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

Trong plan, nếu thấy truy cập `MV_SALES_DAILY` thay vì `SALES`, nghĩa là Query Rewrite đã được sử dụng.

---

## 5. Cấu hình tham số hệ thống (ví dụ)

```sql
ALTER SYSTEM SET QUERY_REWRITE_ENABLED = TRUE;
ALTER SYSTEM SET QUERY_REWRITE_INTEGRITY = TRUSTED;
```

---

## 6. Ý nghĩa trong Physical Design

- Khi thiết kế vật lý (physical design), ta:
  - Xác định các truy vấn tốn kém, lặp lại nhiều.
  - Tạo **materialized view** phù hợp.
  - Bật **query rewrite** để Oracle tự động dùng MV.
- Mục tiêu: tối ưu hiệu năng mà **không cần sửa code ứng dụng** (câu SQL gốc vẫn giữ nguyên).
