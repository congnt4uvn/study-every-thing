## 1. Package trong PL/SQL là gì?

- **Package** là một nhóm logic các đối tượng PL/SQL (procedure, function, biến, type, cursor, …) được đóng gói lại.
- Gồm 2 phần:
  - **Package specification (spec)**: phần khai báo (public).
  - **Package body**: phần cài đặt (implementation).

Lợi ích chính (thường được nói trong phần “Advanced PL/SQL: Using packages for …”):

1. **Tổ chức mã nguồn**: gom các thủ tục/hàm liên quan vào một đơn vị.
2. **Ẩn chi tiết cài đặt**: chỉ lộ spec, giấu body.
3. **Tái sử dụng**: dễ dùng lại trong nhiều chương trình khác.
4. **Hiệu năng**: package được load vào shared memory, giảm chi phí parse/compile nhiều lần.
5. **Quản lý trạng thái**: có thể dùng biến package-level để lưu trạng thái trong session.

## 2. Ví dụ: Tạo package quản lý nhân viên

### 2.1. Bảng mẫu

```sql
CREATE TABLE employees (
    emp_id      NUMBER PRIMARY KEY,
    emp_name    VARCHAR2(100),
    salary      NUMBER
);

INSERT INTO employees (emp_id, emp_name, salary) VALUES (1, 'An', 1000);
INSERT INTO employees (emp_id, emp_name, salary) VALUES (2, 'Bình', 1500);
COMMIT;
```

### 2.2. Package specification

```sql
CREATE OR REPLACE PACKAGE emp_pkg AS
    -- Biến public
    g_last_emp_id   NUMBER;

    -- Procedure tăng lương
    PROCEDURE give_raise (
        p_emp_id   IN  employees.emp_id%TYPE,
        p_amount   IN  employees.salary%TYPE
    );

    -- Function lấy lương
    FUNCTION get_salary (
        p_emp_id   IN employees.emp_id%TYPE
    ) RETURN employees.salary%TYPE;
END emp_pkg;
/
```

### 2.3. Package body

```sql
CREATE OR REPLACE PACKAGE BODY emp_pkg AS

    PROCEDURE give_raise (
        p_emp_id   IN  employees.emp_id%TYPE,
        p_amount   IN  employees.salary%TYPE
    ) IS
    BEGIN
        UPDATE employees
        SET salary = salary + p_amount
        WHERE emp_id = p_emp_id;

        g_last_emp_id := p_emp_id;  -- lưu lại ID nhân viên vừa cập nhật

        COMMIT;
    END give_raise;

    FUNCTION get_salary (
        p_emp_id   IN employees.emp_id%TYPE
    ) RETURN employees.salary%TYPE IS
        v_salary employees.salary%TYPE;
    BEGIN
        SELECT salary
        INTO   v_salary
        FROM   employees
        WHERE  emp_id = p_emp_id;

        RETURN v_salary;
    EXCEPTION
        WHEN NO_DATA_FOUND THEN
            RETURN NULL;
    END get_salary;

END emp_pkg;
/
```

## 3. Ví dụ sử dụng (input/output)

### 3.1. Gọi procedure tăng lương

**Input**: tăng lương cho nhân viên `emp_id = 1` thêm `200`.

```sql
BEGIN
    emp_pkg.give_raise(p_emp_id => 1, p_amount => 200);
END;
/
```

**Kết quả (output trong bảng)**:

```sql
SELECT emp_id, emp_name, salary FROM employees WHERE emp_id = 1;
```

Kết quả:

| EMP_ID | EMP_NAME | SALARY |
|--------|----------|--------|
| 1      | An       | 1200   |

**Biến package**:

```sql
BEGIN
    DBMS_OUTPUT.PUT_LINE('Last emp id updated: ' || emp_pkg.g_last_emp_id);
END;
/
```

Output:

```text
Last emp id updated: 1
```

---

### 3.2. Gọi function lấy lương

**Input**: lấy lương của `emp_id = 2`.

```sql
SET SERVEROUTPUT ON;

DECLARE
    v_sal employees.salary%TYPE;
BEGIN
    v_sal := emp_pkg.get_salary(2);
    DBMS_OUTPUT.PUT_LINE('Salary of emp 2 = ' || v_sal);
END;
/
```

**Output** (giả sử lương hiện tại là 1500):

```text
Salary of emp 2 = 1500
```

## 4. Các ý chính thường được nhấn mạnh khi “Using packages for …”

Tùy nội dung bạn sắp gửi, phần “for” thường là:

- **for modularization**: chia nhỏ hệ thống thành các module (package) theo nghiệp vụ.
- **for encapsulation**: ẩn chi tiết cài đặt, chỉ lộ API (spec).
- **for performance**: giảm overhead parse/compile, dùng biến package-level.
- **for security**: chỉ cho phép gọi các procedure/function public, không truy cập trực tiếp bảng.
- **for maintainability**: thay đổi logic trong body mà không ảnh hưởng spec (nếu không đổi interface).
