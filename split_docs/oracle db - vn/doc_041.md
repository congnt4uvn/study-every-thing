
Dưới đây là tóm tắt ngắn gọn về `EXPLAIN PLAN` và `DBMS_XPLAN.DISPLAY` trong Oracle, kèm ví dụ.

---

## 1. Mục đích

- `EXPLAIN PLAN`: Cho Oracle “dự đoán” cách nó sẽ thực thi một câu lệnh SQL (kế hoạch thực thi – execution plan) và ghi vào bảng `PLAN_TABLE`.
- `DBMS_XPLAN.DISPLAY`: Đọc và hiển thị kế hoạch thực thi đã lưu trong `PLAN_TABLE` dưới dạng dễ đọc.

---

## 2. Quy trình cơ bản

1. Dùng `EXPLAIN PLAN` cho câu lệnh SQL.
2. Dùng `SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);` để xem kế hoạch.

---

## 3. Ví dụ đơn giản

### 3.1. Chuẩn bị bảng mẫu

```sql
CREATE TABLE employees (
    employee_id   NUMBER PRIMARY KEY,
    first_name    VARCHAR2(50),
    last_name     VARCHAR2(50),
    department_id NUMBER
);

-- Thêm index để thấy sự khác biệt trong plan
CREATE INDEX idx_emp_dept ON employees(department_id);
```

### 3.2. EXPLAIN PLAN cho một câu SELECT

```sql
EXPLAIN PLAN FOR
SELECT *
FROM employees
WHERE department_id = 10;
```

Lệnh này không chạy query thật, chỉ ghi kế hoạch vào `PLAN_TABLE`.

### 3.3. Xem kế hoạch với DBMS_XPLAN

```sql
SELECT * 
FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Output mẫu (rút gọn):**

```text
Plan hash value: 123456789

---------------------------------------------------------
| Id  | Operation                   | Name              |
---------------------------------------------------------
|   0 | SELECT STATEMENT            |                   |
|   1 |  TABLE ACCESS BY INDEX ROWID| EMPLOYEES         |
|   2 |   INDEX RANGE SCAN          | IDX_EMP_DEPT      |
---------------------------------------------------------

Predicate Information (identified by operation id):
---------------------------------------------------
   2 - access("DEPARTMENT_ID"=10)
```

Ý nghĩa chính:
- `TABLE ACCESS BY INDEX ROWID`: Lấy dữ liệu từ bảng dựa trên ROWID từ index.
- `INDEX RANGE SCAN`: Dùng index `IDX_EMP_DEPT` để tìm các dòng có `DEPARTMENT_ID = 10`.

---

## 4. Ví dụ với JOIN

```sql
EXPLAIN PLAN FOR
SELECT e.employee_id, e.last_name, d.department_name
FROM employees e
JOIN departments d
  ON e.department_id = d.department_id
WHERE d.location_id = 1700;

SELECT * 
FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Output mẫu (rút gọn):**

```text
--------------------------------------------------------------
| Id  | Operation            | Name         |
--------------------------------------------------------------
|   0 | SELECT STATEMENT     |              |
|   1 |  NESTED LOOPS        |              |
|   2 |   TABLE ACCESS FULL  | DEPARTMENTS  |
|   3 |   INDEX RANGE SCAN   | IDX_EMP_DEPT |
--------------------------------------------------------------

Predicate Information:
----------------------
   2 - filter("LOCATION_ID"=1700)
   3 - access("DEPARTMENT_ID"="D"."DEPARTMENT_ID")
```

---

## 5. Các điểm chính cần nhớ

- `EXPLAIN PLAN FOR <SQL>`: ghi kế hoạch dự kiến vào `PLAN_TABLE`.
- `SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);`: hiển thị kế hoạch đó.
- Dùng để:
  - Hiểu Oracle sẽ dùng FULL SCAN, INDEX SCAN, JOIN kiểu gì (NESTED LOOPS, HASH JOIN, …).
  - Tối ưu câu SQL (thêm index, sửa điều kiện WHERE, thay đổi JOIN, …).

Nếu bạn muốn, tôi có thể thêm ví dụ với `DBMS_XPLAN.DISPLAY_CURSOR` để xem kế hoạch thực tế sau khi query đã chạy.
