
Dưới đây là tóm tắt ngắn gọn về các khái niệm trong Explain Plan & Execution của Oracle: `FILTER`, `SORT`, `VIEW`, `MATERIALIZE`, kèm ví dụ.

---

## 1. FILTER

**Ý nghĩa**  
- `FILTER` là bước lọc dòng theo điều kiện (WHERE, HAVING, điều kiện trong subquery, function, v.v.).
- Oracle đọc dữ liệu từ các bước con, sau đó loại bỏ các dòng không thỏa điều kiện.

**Ví dụ**

```sql
EXPLAIN PLAN FOR
SELECT *
FROM employees
WHERE salary > 10000
  AND department_id IN (SELECT department_id
                        FROM departments
                        WHERE location_id = 1700);

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Kết quả (rút gọn)**

```text
| Id | Operation           | Name       |
|----|---------------------|------------|
|  0 | SELECT STATEMENT    |            |
|  1 |  FILTER             |            |
|  2 |   TABLE ACCESS FULL | EMPLOYEES  |
|  3 |   TABLE ACCESS FULL | DEPARTMENTS|
```

- `FILTER`: áp dụng điều kiện `salary > 10000` và `department_id IN (subquery)`.

---

## 2. SORT

**Ý nghĩa**  
- `SORT` là bước sắp xếp dữ liệu, xuất hiện khi dùng `ORDER BY`, `GROUP BY`, `DISTINCT`, `UNION`, `INTERSECT`, `MINUS`, v.v.
- Các biến thể thường gặp: `SORT ORDER BY`, `SORT GROUP BY`, `SORT AGGREGATE`, `SORT UNIQUE`.

**Ví dụ: ORDER BY**

```sql
EXPLAIN PLAN FOR
SELECT employee_id, salary
FROM employees
ORDER BY salary DESC;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Kết quả (rút gọn)**

```text
| Id | Operation           | Name      |
|----|---------------------|-----------|
|  0 | SELECT STATEMENT    |           |
|  1 |  SORT ORDER BY      |           |
|  2 |   TABLE ACCESS FULL | EMPLOYEES |
```

- `SORT ORDER BY`: sắp xếp kết quả theo `salary DESC`.

**Ví dụ: GROUP BY**

```sql
EXPLAIN PLAN FOR
SELECT department_id, COUNT(*)
FROM employees
GROUP BY department_id;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Kết quả (rút gọn)**

```text
| Id | Operation           | Name      |
|----|---------------------|-----------|
|  0 | SELECT STATEMENT    |           |
|  1 |  SORT GROUP BY      |           |
|  2 |   TABLE ACCESS FULL | EMPLOYEES |
```

- `SORT GROUP BY`: sắp xếp/nhóm dữ liệu để tính COUNT theo `department_id`.

---

## 3. VIEW

**Ý nghĩa**  
- `VIEW` trong plan thể hiện việc truy cập một view (view thường hoặc inline view/subquery trong FROM).
- Oracle có thể:
  - “Merge” view vào câu query chính (VIEW MERGING), hoặc
  - Giữ view như một bước riêng (VIEW).

**Ví dụ: Inline view**

```sql
EXPLAIN PLAN FOR
SELECT v.department_id, v.avg_sal
FROM (
    SELECT department_id, AVG(salary) AS avg_sal
    FROM employees
    GROUP BY department_id
) v
WHERE v.avg_sal > 10000;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Kết quả (rút gọn)**

```text
| Id | Operation           | Name      |
|----|---------------------|-----------|
|  0 | SELECT STATEMENT    |           |
|  1 |  VIEW               |           |
|  2 |   SORT GROUP BY     |           |
|  3 |    TABLE ACCESS FULL| EMPLOYEES |
```

- `VIEW`: đại diện cho subquery `(SELECT ... FROM employees GROUP BY ...) v`.

---

## 4. MATERIALIZE

**Ý nghĩa**  
- `MATERIALIZE` xuất hiện khi Oracle **lưu tạm** kết quả của một subquery/CTE (WITH) vào vùng tạm (temp segment) để dùng lại.
- Thường thấy với:
  - `WITH` (subquery factoring),
  - Subquery được dùng nhiều lần,
  - Một số hint như `MATERIALIZE`.

**Ví dụ: CTE được materialize**

```sql
EXPLAIN PLAN FOR
WITH high_paid AS (
    SELECT /*+ MATERIALIZE */
           employee_id, salary, department_id
    FROM employees
    WHERE salary > 10000
)
SELECT h.department_id, COUNT(*)
FROM high_paid h
GROUP BY h.department_id;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Kết quả (rút gọn)**

```text
| Id | Operation              | Name      |
|----|------------------------|-----------|
|  0 | SELECT STATEMENT       |           |
|  1 |  SORT GROUP BY         |           |
|  2 |   VIEW                 | HIGH_PAID |
|  3 |    TABLE ACCESS FULL   | EMPLOYEES |
|  4 |    FILTER (MATERIALIZE)|           |
```

(Tùy phiên bản, có thể thấy `TEMP TABLE TRANSFORMATION` hoặc `VIEW` + `MATERIALIZE`.)

- `MATERIALIZE`: Oracle tạo kết quả tạm cho CTE `high_paid`, sau đó dùng lại cho phần GROUP BY.

---

## Tóm tắt nhanh

- `FILTER`: Lọc dòng theo điều kiện (WHERE, HAVING, subquery).
- `SORT`: Sắp xếp/nhóm/loại trùng (ORDER BY, GROUP BY, DISTINCT, UNION…).
- `VIEW`: Bước xử lý cho view hoặc inline view/subquery trong FROM.
- `MATERIALIZE`: Lưu tạm kết quả subquery/CTE để dùng lại, thường thấy với WITH hoặc hint `MATERIALIZE`.

Nếu bạn gửi thêm đoạn `EXPLAIN PLAN` cụ thể, tôi có thể giải thích chi tiết từng dòng theo ngữ cảnh thực tế.
