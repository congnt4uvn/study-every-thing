
Dưới đây là tóm tắt ngắn gọn về **Object Types trong Oracle (PL/SQL & SQL)**, kèm ví dụ mã và input/output.

---

## 1. Object Type là gì?

- Là kiểu dữ liệu do người dùng định nghĩa (user-defined type) trong Oracle.
- Gồm:
  - **Thuộc tính (attributes)** – giống như cột.
  - **Phương thức (methods)** – hàm/thủ tục gắn với kiểu đó.
- Dùng được trong:
  - Bảng (object table, column kiểu object).
  - PL/SQL (biến, tham số, collection…).

---

## 2. Định nghĩa Object Type (spec & body)

### 2.1. Khai báo kiểu (spec)

```sql
CREATE OR REPLACE TYPE t_person AS OBJECT (
    person_id   NUMBER,
    full_name   VARCHAR2(100),
    birth_date  DATE,
    
    MEMBER FUNCTION get_age RETURN NUMBER,
    MEMBER PROCEDURE change_name(p_new_name VARCHAR2)
);
/
```

### 2.2. Cài đặt phương thức (body)

```sql
CREATE OR REPLACE TYPE BODY t_person AS

    MEMBER FUNCTION get_age RETURN NUMBER IS
    BEGIN
        RETURN TRUNC(MONTHS_BETWEEN(SYSDATE, birth_date) / 12);
    END;

    MEMBER PROCEDURE change_name(p_new_name VARCHAR2) IS
    BEGIN
        self.full_name := p_new_name;
    END;

END;
/
```

## 3. Sử dụng Object Type trong PL/SQL

### 3.1. Tạo đối tượng, gọi phương thức

**Input:** tạo một `t_person`, đổi tên, lấy tuổi.  
**Code:**

```sql
DECLARE
    v_person t_person;
    v_age    NUMBER;
BEGIN
    -- Khởi tạo object
    v_person := t_person(
        person_id  => 1,
        full_name  => 'Nguyen Van A',
        birth_date => DATE '1990-01-15'
    );

    -- Gọi procedure change_name
    v_person.change_name('Nguyen Van B');

    -- Gọi function get_age
    v_age := v_person.get_age;

    DBMS_OUTPUT.PUT_LINE('ID: ' || v_person.person_id);
    DBMS_OUTPUT.PUT_LINE('Name: ' || v_person.full_name);
    DBMS_OUTPUT.PUT_LINE('Age: ' || v_age);
END;
/
```

**Output (ví dụ):**

```text
ID: 1
Name: Nguyen Van B
Age: 34
```

## 4. Sử dụng Object Type trong bảng

### 4.1. Bảng kiểu object (object table)

```sql
CREATE TABLE person_tab OF t_person (
    CONSTRAINT pk_person_tab PRIMARY KEY (person_id)
);
```

### 4.2. Chèn dữ liệu

```sql
INSERT INTO person_tab
VALUES (t_person(1, 'Tran Thi C', DATE '1985-05-20'));

INSERT INTO person_tab
VALUES (t_person(2, 'Le Van D', DATE '2000-10-01'));

COMMIT;
```

### 4.3. Truy vấn dữ liệu

```sql
-- Lấy toàn bộ object
SELECT VALUE(p) AS person_object
FROM person_tab p;

-- Lấy từng thuộc tính
SELECT p.person_id,
       p.full_name,
       p.birth_date,
       p.get_age() AS age
FROM person_tab p;
```

**Output (ví dụ):**

```text
PERSON_ID | FULL_NAME   | BIRTH_DATE  | AGE
--------- | ----------- | ---------- | ---
1         | Tran Thi C  | 20-MAY-85  | 39
2         | Le Van D    | 01-OCT-00  | 23
```

## 5. Object Type trong cột của bảng “thường”

```sql
-- Định nghĩa kiểu địa chỉ
CREATE OR REPLACE TYPE t_address AS OBJECT (
    street  VARCHAR2(100),
    city    VARCHAR2(50),
    country VARCHAR2(50)
);
/

-- Bảng khách hàng có cột kiểu object
CREATE TABLE customers (
    customer_id NUMBER PRIMARY KEY,
    name        VARCHAR2(100),
    address     t_address
);

-- Insert
INSERT INTO customers
VALUES (
    1,
    'Pham Van E',
    t_address('123 Le Loi', 'Ho Chi Minh', 'Vietnam')
);

-- Select
SELECT c.customer_id,
       c.name,
       c.address.street,
       c.address.city,
       c.address.country
FROM customers c;
```

## 6. Điểm chính cần nhớ

- **Object Type** cho phép mô hình hóa dữ liệu theo hướng đối tượng trong Oracle.
- Có thể:
  - Định nghĩa thuộc tính + phương thức.
  - Dùng trong PL/SQL (biến, tham số).
  - Dùng trong SQL (bảng kiểu object, cột kiểu object).
- Hữu ích khi:
  - Cần đóng gói logic + dữ liệu.
  - Mô hình hóa các thực thể phức tạp (khách hàng, địa chỉ, sản phẩm…).
