
Dưới đây là tóm tắt ngắn gọn về VPD / RLS trong Oracle, kèm ví dụ code và input/output minh họa.

---

## 1. Khái niệm chính

**VPD (Virtual Private Database)** hay **RLS (Row-Level Security)** là cơ chế bảo mật của Oracle cho phép:

- Tự động **lọc dữ liệu theo từng dòng (row)** dựa trên người dùng, vai trò, hoặc ngữ cảnh.
- Người dùng vẫn truy vấn bảng bình thường (`SELECT * FROM ...`), nhưng Oracle **tự động thêm điều kiện WHERE** để chỉ trả về các dòng được phép xem.
- Chính sách được gắn vào bảng/view, không cần sửa code ứng dụng nhiều.

---

## 2. Cách hoạt động (high-level)

1. Bạn định nghĩa **hàm chính sách (policy function)** trả về một chuỗi điều kiện SQL, ví dụ:  
   ```sql
   'DEPT_ID = 10'
   ```
2. Bạn dùng `DBMS_RLS.ADD_POLICY` để gắn hàm này vào một bảng.
3. Khi người dùng chạy `SELECT/INSERT/UPDATE/DELETE` trên bảng đó, Oracle:
   - Gọi hàm chính sách.
   - Lấy chuỗi điều kiện trả về.
   - Tự động thêm vào câu lệnh SQL như một phần của `WHERE`.

## 3. Ví dụ đơn giản

### 3.1. Bảng dữ liệu

```sql
CREATE TABLE employees (
    emp_id    NUMBER,
    emp_name  VARCHAR2(100),
    dept_id   NUMBER
);

INSERT INTO employees VALUES (1, 'Alice', 10);
INSERT INTO employees VALUES (2, 'Bob',   20);
INSERT INTO employees VALUES (3, 'Carol', 10);
COMMIT;
```

Giả sử:
- User `USER10` chỉ được xem `dept_id = 10`
- User `USER20` chỉ được xem `dept_id = 20`

### 3.2. Hàm chính sách (policy function)

```sql
CREATE OR REPLACE FUNCTION emp_dept_policy (
    p_schema IN VARCHAR2,
    p_object IN VARCHAR2
) RETURN VARCHAR2
AS
    v_user VARCHAR2(30);
BEGIN
    v_user := SYS_CONTEXT('USERENV', 'SESSION_USER');

    IF v_user = 'USER10' THEN
        RETURN 'dept_id = 10';
    ELSIF v_user = 'USER20' THEN
        RETURN 'dept_id = 20';
    ELSE
        -- Không cho xem gì
        RETURN '1 = 0';
    END IF;
END;
/
```

### 3.3. Gắn policy vào bảng

```sql
BEGIN
    DBMS_RLS.ADD_POLICY(
        object_schema   => 'HR',          -- schema chứa bảng
        object_name     => 'EMPLOYEES',   -- tên bảng
        policy_name     => 'EMP_DEPT_RLS',
        function_schema => 'HR',          -- schema chứa hàm policy
        policy_function => 'EMP_DEPT_POLICY',
        statement_types => 'SELECT'       -- áp dụng cho SELECT
    );
END;
/
```

## 4. Minh họa Input / Output

### 4.1. Khi user `USER10` chạy:

**Input (SQL người dùng gõ):**
```sql
SELECT emp_id, emp_name, dept_id
FROM hr.employees;
```

**Oracle thực tế thực thi (nội bộ):**
```sql
SELECT emp_id, emp_name, dept_id
FROM hr.employees
WHERE dept_id = 10;
```

**Output:**
```text
EMP_ID | EMP_NAME | DEPT_ID
-------+----------+--------
1      | Alice    | 10
3      | Carol    | 10
```

---

### 4.2. Khi user `USER20` chạy cùng câu lệnh:

**Input:**
```sql
SELECT emp_id, emp_name, dept_id
FROM hr.employees;
```

**Oracle thực tế thực thi:**
```sql
SELECT emp_id, emp_name, dept_id
FROM hr.employees
WHERE dept_id = 20;
```

**Output:**
```text
EMP_ID | EMP_NAME | DEPT_ID
-------+----------+--------
2      | Bob      | 20
```

---

### 4.3. Khi user khác (không được phép) chạy:

**Input:**
```sql
SELECT emp_id, emp_name, dept_id
FROM hr.employees;
```

**Oracle thực tế thực thi:**
```sql
SELECT emp_id, emp_name, dept_id
FROM hr.employees
WHERE 1 = 0;
```

**Output:**
```text
No rows selected
```

---

## 5. Ý chính cần nhớ

- VPD/RLS cho phép **bảo mật theo từng dòng dữ liệu** mà không cần tạo nhiều view hay bảng riêng.
- Chính sách được định nghĩa bằng **hàm PL/SQL** trả về điều kiện `WHERE`.
- Oracle tự động áp dụng chính sách cho các câu lệnh SQL (SELECT/INSERT/UPDATE/DELETE) tùy cấu hình.
- Rất hữu ích cho các hệ thống multi-tenant, phân quyền theo phòng ban, chi nhánh, người dùng, v.v.
