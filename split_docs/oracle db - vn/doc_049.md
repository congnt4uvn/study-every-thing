
Dưới đây là tóm tắt ngắn gọn về chiến lược đánh index trên khóa ngoại (foreign key) trong Oracle, kèm ví dụ.

---

## 1. Ý chính

- Nên **tạo index trên các cột khóa ngoại** (FOREIGN KEY) trong bảng con.
- Lý do:
  - Tăng tốc các câu lệnh `JOIN` giữa bảng cha và bảng con.
  - Giảm khóa hàng/bảng và tránh chờ đợi khi:
    - `DELETE` trên bảng cha.
    - `UPDATE` khóa chính (PRIMARY KEY) trên bảng cha.
  - Giúp Oracle nhanh chóng tìm các bản ghi liên quan trong bảng con.

---

## 2. Ví dụ cấu trúc bảng

```sql
-- Bảng cha
CREATE TABLE departments (
    dept_id   NUMBER PRIMARY KEY,
    dept_name VARCHAR2(100)
);

-- Bảng con có khóa ngoại
CREATE TABLE employees (
    emp_id    NUMBER PRIMARY KEY,
    emp_name  VARCHAR2(100),
    dept_id   NUMBER,
    CONSTRAINT fk_emp_dept
        FOREIGN KEY (dept_id)
        REFERENCES departments(dept_id)
);
```

### Không có index trên khóa ngoại

- Cột `employees.dept_id` **chưa có index**.
- Khi chạy:

```sql
DELETE FROM departments
WHERE dept_id = 10;
```

Oracle phải:
- Quét toàn bộ bảng `employees` để kiểm tra xem có nhân viên nào thuộc `dept_id = 10` không.
- Có thể gây khóa (lock) lớn và chậm nếu bảng `employees` rất lớn.

---

## 3. Tạo index trên khóa ngoại

```sql
CREATE INDEX idx_emp_dept_id
    ON employees(dept_id);
```

### Lợi ích

1. **Tăng tốc JOIN**

```sql
SELECT e.emp_id, e.emp_name, d.dept_name
FROM employees e
JOIN departments d
  ON e.dept_id = d.dept_id
WHERE d.dept_id = 10;
```

- Oracle dùng `idx_emp_dept_id` để tìm nhanh các nhân viên thuộc phòng ban 10.

2. **Tăng tốc DELETE/UPDATE trên bảng cha**

```sql
-- Xóa phòng ban
DELETE FROM departments
WHERE dept_id = 10;
```

- Oracle dùng index để tìm nhanh các dòng trong `employees` có `dept_id = 10`:
  - Nếu có ràng buộc `ON DELETE CASCADE`: xóa nhanh các nhân viên liên quan.
  - Nếu không: kiểm tra nhanh xem có bản ghi con nào vi phạm ràng buộc.

---

## 4. Ví dụ minh họa input/output

### Dữ liệu mẫu

```sql
INSERT INTO departments (dept_id, dept_name) VALUES (10, 'IT');
INSERT INTO departments (dept_id, dept_name) VALUES (20, 'HR');

INSERT INTO employees (emp_id, emp_name, dept_id)
VALUES (1, 'An', 10);
INSERT INTO employees (emp_id, emp_name, dept_id)
VALUES (2, 'Bình', 10);
INSERT INTO employees (emp_id, emp_name, dept_id)
VALUES (3, 'Chi', 20);
COMMIT;
```

### Truy vấn JOIN (input)

```sql
SELECT e.emp_id, e.emp_name, d.dept_name
FROM employees e
JOIN departments d
  ON e.dept_id = d.dept_id
WHERE d.dept_id = 10
ORDER BY e.emp_id;
```

### Kết quả (output)

| EMP_ID | EMP_NAME | DEPT_NAME |
|--------|----------|-----------|
| 1      | An       | IT        |
| 2      | Bình     | IT        |

- Khi có index `idx_emp_dept_id`, truy vấn này sẽ chạy nhanh hơn trên dữ liệu lớn.

---

## 5. Khi nào nên index khóa ngoại?

- Bảng con có:
  - Nhiều bản ghi.
  - Thường xuyên:
    - JOIN với bảng cha.
    - Thực hiện `DELETE`/`UPDATE` trên bảng cha.
- Tránh index nếu:
  - Bảng rất nhỏ.
  - Cột khóa ngoại ít được dùng trong WHERE/JOIN.
  - Cần giảm số lượng index để tối ưu ghi (INSERT/UPDATE/DELETE).
