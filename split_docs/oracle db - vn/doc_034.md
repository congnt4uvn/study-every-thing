
Dưới đây là tóm tắt ngắn gọn về **Deterministic Functions** trong PL/SQL, kèm ví dụ mã và input/output.

---

## 1. Khái niệm chính

- **Deterministic function** là hàm mà:
  - Với cùng một bộ tham số đầu vào, **luôn trả về cùng một kết quả**.
- Trong PL/SQL/SQL, ta khai báo bằng từ khóa `DETERMINISTIC`.
- Lợi ích chính:
  - Cho phép Oracle **cache** kết quả, tránh tính toán lại.
  - Có thể dùng trong **function-based index**.
- Điều kiện:
  - Hàm **không phụ thuộc** vào trạng thái bên ngoài thay đổi được (ví dụ: dữ liệu bảng, `SYSDATE`, `DBMS_RANDOM`, gói có trạng thái…).

---

## 2. Cú pháp cơ bản

```sql
CREATE OR REPLACE FUNCTION ten_ham (
    p_input  IN NUMBER
) RETURN NUMBER
DETERMINISTIC
IS
    v_result NUMBER;
BEGIN
    -- Tính toán nào đó chỉ phụ thuộc vào p_input
    v_result := p_input * p_input;  -- ví dụ: bình phương
    RETURN v_result;
END;
/
```

---

## 3. Ví dụ 1: Hàm deterministic đơn giản

### Tạo hàm

```sql
CREATE OR REPLACE FUNCTION fn_square (
    p_n IN NUMBER
) RETURN NUMBER
DETERMINISTIC
IS
BEGIN
    RETURN p_n * p_n;
END;
/
```

### Gọi trong SQL (Input/Output)

```sql
SELECT
    2      AS input_value,
    fn_square(2) AS output_value
FROM dual;

-- Kết quả:
-- INPUT_VALUE | OUTPUT_VALUE
--      2      |      4
```

```sql
SELECT
    5      AS input_value,
    fn_square(5) AS output_value
FROM dual;

-- Kết quả:
-- INPUT_VALUE | OUTPUT_VALUE
--      5      |     25
```

Với cùng `p_n`, hàm luôn trả về cùng kết quả → phù hợp `DETERMINISTIC`.

---

## 4. Ví dụ 2: Hàm deterministic dùng trong function-based index

Giả sử ta có bảng khách hàng:

```sql
CREATE TABLE customers (
    customer_id   NUMBER PRIMARY KEY,
    full_name     VARCHAR2(200)
);
```

### Tạo hàm chuẩn hóa tên (deterministic)

```sql
CREATE OR REPLACE FUNCTION normalize_name (
    p_name IN VARCHAR2
) RETURN VARCHAR2
DETERMINISTIC
IS
BEGIN
    RETURN UPPER(TRIM(p_name));
END;
/
```

### Tạo function-based index

```sql
CREATE INDEX idx_customers_norm_name
    ON customers ( normalize_name(full_name) );
```

### Truy vấn sử dụng hàm (Input/Output)

```sql
SELECT customer_id, full_name
FROM   customers
WHERE  normalize_name(full_name) = normalize_name('  nguyễn văn a  ');
```

- Input: `'  nguyễn văn a  '`
- Hàm `normalize_name`:
  - `TRIM` → `'nguyễn văn a'`
  - `UPPER` → `'NGUYỄN VĂN A'`
- Output so sánh: `'NGUYỄN VĂN A'`
- Oracle có thể dùng index `idx_customers_norm_name` để tăng tốc truy vấn.

---

## 5. Ví dụ 3: Hàm *không nên* khai báo deterministic

```sql
CREATE OR REPLACE FUNCTION fn_today
RETURN DATE
DETERMINISTIC
IS
BEGIN
    RETURN SYSDATE;
END;
/
```

- Hàm này **không phải** deterministic về mặt logic:
  - Cùng “input” (không có tham số) nhưng kết quả thay đổi theo thời gian.
- Nếu khai báo `DETERMINISTIC`, Oracle có thể cache kết quả và **trả về cùng một ngày** trong cùng session/plan → sai về nghiệp vụ.

---

## 6. Khi nào nên dùng `DETERMINISTIC`

Nên dùng khi:

1. Hàm:
   - Chỉ phụ thuộc vào **tham số đầu vào**.
   - Không đọc/ghi bảng, không dùng `SYSDATE`, `SYSTIMESTAMP`, `DBMS_RANDOM`, v.v.
2. Hàm được gọi **rất nhiều lần** trong SQL (SELECT, WHERE, GROUP BY…).
3. Muốn dùng trong **function-based index** để tối ưu truy vấn.

Không nên dùng khi:

- Hàm phụ thuộc vào dữ liệu có thể thay đổi (bảng, package state, thời gian hệ thống…).

---

Tóm lại:  
**Deterministic functions** trong PL/SQL là công cụ tối ưu hóa mạnh, giúp Oracle cache kết quả và hỗ trợ function-based index, nhưng chỉ nên dùng khi hàm thực sự “định tính” (cùng input → luôn cùng output, không phụ thuộc trạng thái bên ngoài).
