
Dưới đây là tóm tắt ngắn gọn về các khái niệm PL/SQL: biến, hằng, record, và collection (associative array, nested table, VARRAY), kèm ví dụ code và kết quả.

---

## 1. Biến (Variables)

- Dùng để lưu trữ giá trị có thể thay đổi trong khối PL/SQL.
- Khai báo trong phần `DECLARE`.

**Ví dụ:**

```sql
DECLARE
    v_name   VARCHAR2(50);
    v_age    NUMBER;
BEGIN
    v_name := 'An';
    v_age  := 25;
    DBMS_OUTPUT.PUT_LINE('Name: ' || v_name);
    DBMS_OUTPUT.PUT_LINE('Age: ' || v_age);
END;
/
```

**Output:**
```
Name: An
Age: 25
```

---

## 2. Hằng (Constants)

- Giá trị không thay đổi sau khi gán.
- Phải dùng từ khóa `CONSTANT` và gán giá trị ngay khi khai báo.

**Ví dụ:**

```sql
DECLARE
    c_tax_rate CONSTANT NUMBER := 0.1;
    v_salary   NUMBER := 10000;
    v_tax      NUMBER;
BEGIN
    v_tax := v_salary * c_tax_rate;
    DBMS_OUTPUT.PUT_LINE('Tax: ' || v_tax);
END;
/
```

**Output:**
```
Tax: 1000
```

---

## 3. Record

- Kiểu dữ liệu tổng hợp, gồm nhiều trường (field) với kiểu khác nhau.
- Tương tự như “struct” trong các ngôn ngữ khác.
- Có thể:
  - Tự định nghĩa bằng `TYPE ... IS RECORD`
  - Hoặc dựa trên cấu trúc bảng (`%ROWTYPE`).

**Ví dụ tự định nghĩa record:**

```sql
DECLARE
    TYPE t_employee IS RECORD (
        emp_id   NUMBER,
        emp_name VARCHAR2(100),
        salary   NUMBER
    );

    v_emp t_employee;
BEGIN
    v_emp.emp_id   := 1;
    v_emp.emp_name := 'Bình';
    v_emp.salary   := 15000;

    DBMS_OUTPUT.PUT_LINE('ID: ' || v_emp.emp_id);
    DBMS_OUTPUT.PUT_LINE('Name: ' || v_emp.emp_name);
    DBMS_OUTPUT.PUT_LINE('Salary: ' || v_emp.salary);
END;
/
```

**Output:**
```
ID: 1
Name: Bình
Salary: 15000
```

**Ví dụ dùng `%ROWTYPE`:**

```sql
DECLARE
    v_emp employees%ROWTYPE;  -- giả sử có bảng EMPLOYEES
BEGIN
    SELECT *
    INTO   v_emp
    FROM   employees
    WHERE  employee_id = 100;

    DBMS_OUTPUT.PUT_LINE('Name: ' || v_emp.first_name || ' ' || v_emp.last_name);
END;
/
```

## 4. Collection

Collection là kiểu dữ liệu chứa nhiều phần tử cùng kiểu. Ba loại chính:

1. Associative array (index-by table)
2. Nested table
3. VARRAY

### 4.1. Associative Array

- Chỉ tồn tại trong PL/SQL (không lưu trực tiếp trong cột bảng).
- Được đánh chỉ số bởi số (BINARY_INTEGER/PLS_INTEGER) hoặc VARCHAR2.
- Linh hoạt, không cần kích thước cố định.

**Ví dụ:**

```sql
DECLARE
    TYPE t_num_tab IS TABLE OF VARCHAR2(50) INDEX BY PLS_INTEGER;
    v_names t_num_tab;
    i       PLS_INTEGER;
BEGIN
    v_names(1) := 'An';
    v_names(2) := 'Bình';
    v_names(3) := 'Chi';

    i := v_names.FIRST;
    WHILE i IS NOT NULL LOOP
        DBMS_OUTPUT.PUT_LINE('Index ' || i || ': ' || v_names(i));
        i := v_names.NEXT(i);
    END LOOP;
END;
/
```

**Output:**
```
Index 1: An
Index 2: Bình
Index 3: Chi
```

---

### 4.2. Nested Table

- Có thể dùng trong PL/SQL và làm kiểu cột trong bảng.
- Không giới hạn số phần tử (có thể “sparse” – có lỗ hổng index).
- Cần `EXTEND` để thêm phần tử khi dùng trong PL/SQL.

**Ví dụ:**

```sql
DECLARE
    TYPE t_num_list IS TABLE OF NUMBER;
    v_list t_num_list := t_num_list();  -- khởi tạo
    i      PLS_INTEGER;
BEGIN
    v_list.EXTEND(3);
    v_list(1) := 10;
    v_list(2) := 20;
    v_list(3) := 30;

    FOR i IN 1 .. v_list.COUNT LOOP
        DBMS_OUTPUT.PUT_LINE('Element ' || i || ': ' || v_list(i));
    END LOOP;
END;
/
```

**Output:**
```
Element 1: 10
Element 2: 20
Element 3: 30
```

---

### 4.3. VARRAY (Variable-size Array)

- Kích thước tối đa được định nghĩa trước (ví dụ: 10 phần tử).
- Thứ tự phần tử được giữ nguyên.
- Có thể dùng làm kiểu cột trong bảng.

**Ví dụ:**

```sql
DECLARE
    TYPE t_scores IS VARRAY(5) OF NUMBER;
    v_scores t_scores := t_scores(8, 9, 10);  -- khởi tạo với 3 phần tử
    i        PLS_INTEGER;
BEGIN
    DBMS_OUTPUT.PUT_LINE('Count: ' || v_scores.COUNT);

    FOR i IN 1 .. v_scores.COUNT LOOP
        DBMS_OUTPUT.PUT_LINE('Score ' || i || ': ' || v_scores(i));
    END LOOP;
END;
/
```

**Output:**
```
Count: 3
Score 1: 8
Score 2: 9
Score 3: 10
```

---

## Tóm tắt ý chính

- **Biến**: lưu giá trị thay đổi trong PL/SQL.
- **Hằng**: giá trị cố định, khai báo với `CONSTANT`.
- **Record**: nhóm nhiều trường khác kiểu thành một đơn vị (có thể dựa trên bảng).
- **Associative array**: mảng chỉ số linh hoạt, chỉ trong PL/SQL.
- **Nested table**: mảng có thể lưu trong bảng, không giới hạn phần tử.
- **VARRAY**: mảng có kích thước tối đa cố định, giữ thứ tự, có thể lưu trong bảng.
