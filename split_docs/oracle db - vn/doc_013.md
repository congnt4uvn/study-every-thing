
Dưới đây là tóm tắt ngắn gọn về “Advanced SQL: Set operations” trong Oracle, kèm ví dụ code và kết quả.  

---

## 1. Khái niệm chung

Set operations (phép toán tập hợp) cho phép kết hợp kết quả của nhiều câu SELECT lại với nhau, tương tự như các phép toán trên tập hợp trong toán học.

Các phép chính trong Oracle:

1. `UNION`
2. `UNION ALL`
3. `INTERSECT`
4. `MINUS` (trong chuẩn SQL mới là `EXCEPT`)

**Yêu cầu chung:**

- Số cột trong các SELECT phải bằng nhau.
- Kiểu dữ liệu tương ứng theo thứ tự cột phải tương thích.
- Thứ tự cột tương ứng phải giống nhau về ý nghĩa (ví dụ: id với id, name với name).

---

## 2. UNION

Kết hợp hai tập kết quả và **loại bỏ các dòng trùng lặp**.

```sql
-- Bảng ví dụ:
-- EMP_A(emp_id, name)
-- EMP_B(emp_id, name)

SELECT emp_id, name
FROM emp_a
UNION
SELECT emp_id, name
FROM emp_b;
```

**Đầu ra (ví dụ):**

| EMP_ID | NAME   |
|--------|--------|
| 1      | An     |
| 2      | Bình   |
| 3      | Chi    |

Nếu cùng một (emp_id, name) xuất hiện ở cả hai bảng, chỉ hiển thị một lần.

---

## 3. UNION ALL

Kết hợp hai tập kết quả **giữ nguyên tất cả các dòng**, kể cả trùng lặp.

```sql
SELECT emp_id, name
FROM emp_a
UNION ALL
SELECT emp_id, name
FROM emp_b;
```

**Đầu ra (ví dụ):**

| EMP_ID | NAME   |
|--------|--------|
| 1      | An     |
| 2      | Bình   |
| 2      | Bình   |  -- trùng vẫn giữ
| 3      | Chi    |

Dùng `UNION ALL` khi bạn cần hiệu năng tốt hơn và không cần loại bỏ trùng.

---

## 4. INTERSECT

Trả về **phần giao**: chỉ những dòng xuất hiện trong cả hai tập kết quả.

```sql
SELECT emp_id, name
FROM emp_a
INTERSECT
SELECT emp_id, name
FROM emp_b;
```

**Đầu ra (ví dụ):**

| EMP_ID | NAME   |
|--------|--------|
| 2      | Bình   |

Chỉ những nhân viên có cùng (emp_id, name) trong cả `emp_a` và `emp_b`.

---

## 5. MINUS

Trả về **phần hiệu**: các dòng có trong tập thứ nhất nhưng **không** có trong tập thứ hai.

```sql
SELECT emp_id, name
FROM emp_a
MINUS
SELECT emp_id, name
FROM emp_b;
```

**Đầu ra (ví dụ):**

| EMP_ID | NAME   |
|--------|--------|
| 1      | An     |

Nghĩa là nhân viên có trong `emp_a` nhưng không có trong `emp_b`.

---

## 6. Quy tắc sắp xếp và ORDER BY

- `ORDER BY` chỉ được viết **một lần** ở cuối toàn bộ biểu thức set operation.
- Không được đặt `ORDER BY` trong từng SELECT con (trừ khi dùng subquery).

```sql
SELECT emp_id, name
FROM emp_a
UNION
SELECT emp_id, name
FROM emp_b
ORDER BY emp_id;
```

## 7. Ví dụ tổng hợp

### Dữ liệu mẫu

```sql
-- EMP_A
-- (1, 'An')
-- (2, 'Bình')

-- EMP_B
-- (2, 'Bình')
-- (3, 'Chi')
```

### 7.1. UNION

```sql
SELECT emp_id, name FROM emp_a
UNION
SELECT emp_id, name FROM emp_b;
```

**Kết quả:**

| EMP_ID | NAME   |
|--------|--------|
| 1      | An     |
| 2      | Bình   |
| 3      | Chi    |

### 7.2. UNION ALL

```sql
SELECT emp_id, name FROM emp_a
UNION ALL
SELECT emp_id, name FROM emp_b;
```

**Kết quả:**

| EMP_ID | NAME   |
|--------|--------|
| 1      | An     |
| 2      | Bình   |
| 2      | Bình   |
| 3      | Chi    |

### 7.3. INTERSECT

```sql
SELECT emp_id, name FROM emp_a
INTERSECT
SELECT emp_id, name FROM emp_b;
```

**Kết quả:**

| EMP_ID | NAME   |
|--------|--------|
| 2      | Bình   |

### 7.4. MINUS

```sql
SELECT emp_id, name FROM emp_a
MINUS
SELECT emp_id, name FROM emp_b;
```

**Kết quả:**

| EMP_ID | NAME   |
|--------|--------|
| 1      | An     |

---

## 8. Ghi nhớ nhanh

- `UNION`: hợp, bỏ trùng.
- `UNION ALL`: hợp, giữ trùng.
- `INTERSECT`: giao.
- `MINUS`: phần còn lại của tập 1 sau khi bỏ phần chung với tập 2.

Nếu bạn gửi thêm nội dung chi tiết hơn (ví dụ: đoạn tài liệu cụ thể), tôi có thể tóm tắt sát hơn với tài liệu gốc.
