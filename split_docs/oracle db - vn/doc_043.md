
Dưới đây là tóm tắt ngắn gọn về Explain Plan & Execution trong Oracle, tập trung vào 3 kiểu join chính: Nested Loops, Hash Join, Merge Join, kèm ví dụ.

---

## 1. Explain Plan là gì?

- `EXPLAIN PLAN` cho biết **Oracle sẽ dự định** thực thi câu SQL như thế nào (kế hoạch thực thi).
- `DBMS_XPLAN.DISPLAY` cho biết **kế hoạch thực thi thực tế** (nếu lấy từ `V$SQL_PLAN`).

Ví dụ cơ bản:

```sql
EXPLAIN PLAN FOR
SELECT e.employee_id, d.department_name
FROM   employees e
       JOIN departments d ON e.department_id = d.department_id;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Output (rút gọn, ví dụ):**

```text
--------------------------------------------------------------------------------
| Id  | Operation           | Name        | Rows  | Cost |
--------------------------------------------------------------------------------
|   0 | SELECT STATEMENT    |             |   100 |   10 |
|   1 |  NESTED LOOPS       |             |   100 |   10 |
|   2 |   TABLE ACCESS FULL | EMPLOYEES   |   100 |    5 |
|   3 |   INDEX UNIQUE SCAN | PK_DEPT     |   100 |    1 |
--------------------------------------------------------------------------------
```

## 2. Nested Loops Join

### Ý tưởng chính

- Lặp qua từng dòng của bảng ngoài (outer table), với mỗi dòng sẽ tìm dòng phù hợp trong bảng trong (inner table).
- Tốt khi:
  - Bảng ngoài nhỏ.
  - Có index tốt trên bảng trong theo cột join.
- Thường dùng cho truy vấn OLTP, truy vấn trả ít dòng.

### Ví dụ

```sql
EXPLAIN PLAN FOR
SELECT e.employee_id, d.department_name
FROM   employees e
       JOIN departments d
         ON e.department_id = d.department_id
WHERE  e.employee_id < 100;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Output (rút gọn):**

```text
| Id | Operation           | Name        |
-----------------------------------------
|  0 | SELECT STATEMENT    |             |
|  1 |  NESTED LOOPS       |             |
|  2 |   TABLE ACCESS BY   | EMPLOYEES   |  -- outer
|    |   INDEX ROWID       |             |
|  3 |    INDEX RANGE SCAN | EMP_EMP_ID  |
|  4 |   INDEX UNIQUE SCAN | PK_DEPT     |  -- inner
```

**Diễn giải:**

- Oracle quét theo index `EMP_EMP_ID` để lấy các employee_id < 100.
- Với mỗi employee, dùng `PK_DEPT` (index unique) để tìm department tương ứng.

## 3. Hash Join

### Ý tưởng chính

- Xây dựng **hash table** từ bảng nhỏ (build input).
- Dùng bảng lớn (probe input) để dò (probe) vào hash table.
- Tốt khi:
  - Join trên tập dữ liệu lớn.
  - Không có index phù hợp.
  - Thường dùng trong hệ thống DW/OLAP, truy vấn quét nhiều dòng.

### Ví dụ

```sql
EXPLAIN PLAN FOR
SELECT /*+ USE_HASH(e d) */
       e.employee_id, d.department_name
FROM   employees e
       JOIN departments d
         ON e.department_id = d.department_id;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Output (rút gọn):**

```text
| Id | Operation           | Name        |
-----------------------------------------
|  0 | SELECT STATEMENT    |             |
|  1 |  HASH JOIN          |             |
|  2 |   TABLE ACCESS FULL | DEPARTMENTS |  -- build
|  3 |   TABLE ACCESS FULL | EMPLOYEES   |  -- probe
```

**Diễn giải:**

- Oracle đọc toàn bộ `DEPARTMENTS`, tạo hash table theo `department_id`.
- Sau đó quét `EMPLOYEES`, với mỗi dòng dùng `department_id` để dò trong hash table.

---

## 4. Merge Join (Sort Merge Join)

### Ý tưởng chính

- Cả hai nguồn dữ liệu được **sắp xếp theo cột join**.
- Sau đó Oracle “merge” hai tập đã sắp xếp, giống như merge trong thuật toán merge sort.
- Tốt khi:
  - Cả hai bảng đã có dữ liệu được sắp xếp sẵn (hoặc có index theo cột join).
  - Join trên tập dữ liệu lớn.
  - Điều kiện join là equality ( = ).

### Ví dụ

```sql
EXPLAIN PLAN FOR
SELECT /*+ USE_MERGE(e d) */
       e.employee_id, d.department_name
FROM   employees e
       JOIN departments d
         ON e.department_id = d.department_id;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Output (rút gọn):**

```text
| Id | Operation            | Name        |
------------------------------------------
|  0 | SELECT STATEMENT     |             |
|  1 |  MERGE JOIN          |             |
|  2 |   SORT JOIN          | EMPLOYEES   |
|  3 |    TABLE ACCESS FULL | EMPLOYEES   |
|  4 |   SORT JOIN          | DEPARTMENTS |
|  5 |    TABLE ACCESS FULL | DEPARTMENTS |
```

**Diễn giải:**

- Oracle sắp xếp `EMPLOYEES` theo `department_id`.
- Sắp xếp `DEPARTMENTS` theo `department_id`.
- Sau đó merge hai tập đã sắp xếp để tạo kết quả join.

---

## 5. So sánh nhanh 3 kiểu join

| Kiểu join      | Khi nào phù hợp                                           | Yêu cầu chính                     |
|----------------|-----------------------------------------------------------|-----------------------------------|
| Nested Loops   | Bảng ngoài nhỏ, có index tốt trên bảng trong             | Index trên cột join của bảng trong |
| Hash Join      | Tập dữ liệu lớn, không có index tốt, DW/OLAP             | Đủ RAM cho hash table             |
| Merge Join     | Cả hai bên lớn, có thể sắp xếp/đã sắp xếp theo cột join  | Sort hoặc index theo cột join     |

---

Nếu bạn gửi thêm một câu SQL cụ thể, tôi có thể phân tích Explain Plan chi tiết và minh họa kiểu join mà Oracle đang dùng.
