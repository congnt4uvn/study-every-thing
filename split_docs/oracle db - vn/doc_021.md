
Dưới đây là tóm tắt ngắn gọn về **Packages trong PL/SQL (Specification vs Body)**, kèm ví dụ mã và input/output.

---

## 1. Package là gì?

- **Package** = tập hợp logic các **procedure, function, biến, kiểu dữ liệu, cursor…** được nhóm lại trong một đơn vị.
- Gồm 2 phần:
  1. **Package Specification (Spec)**: phần khai báo (interface).
  2. **Package Body**: phần cài đặt (implementation).

---

## 2. Package Specification

- Chỉ chứa **khai báo public**:
  - Tên procedure, function.
  - Biến public, kiểu dữ liệu, cursor.
- Là phần mà **các chương trình khác nhìn thấy và gọi được**.
- Không (hoặc rất ít) chứa code thực thi chi tiết.

### Ví dụ: Package Specification

```sql
CREATE OR REPLACE PACKAGE emp_pkg IS
  -- Biến public
  g_bonus_rate CONSTANT NUMBER := 0.1;

  -- Kiểu dữ liệu
  TYPE t_emp_rec IS RECORD (
    emp_id   employees.employee_id%TYPE,
    salary   employees.salary%TYPE
  );

  -- Procedure public
  PROCEDURE give_raise(p_emp_id IN employees.employee_id%TYPE,
                       p_percent IN NUMBER);

  -- Function public
  FUNCTION get_salary(p_emp_id IN employees.employee_id%TYPE)
    RETURN NUMBER;
END emp_pkg;
/
```

---

## 3. Package Body

- Chứa **cài đặt chi tiết** cho:
  - Các procedure/function đã khai báo trong spec.
- Có thể chứa:
  - **Procedure/function private** (không khai trong spec → chỉ dùng nội bộ).
  - Biến private, cursor private.
- Người dùng bên ngoài **không gọi trực tiếp** được các thành phần private.

### Ví dụ: Package Body

```sql
CREATE OR REPLACE PACKAGE BODY emp_pkg IS

  -- Biến private (chỉ dùng trong body)
  g_min_raise CONSTANT NUMBER := 1; -- 1%

  -- Function private
  FUNCTION normalize_percent(p_percent IN NUMBER) RETURN NUMBER IS
  BEGIN
    IF p_percent < g_min_raise THEN
      RETURN g_min_raise;
    ELSE
      RETURN p_percent;
    END IF;
  END;

  -- Cài đặt procedure public
  PROCEDURE give_raise(p_emp_id IN employees.employee_id%TYPE,
                       p_percent IN NUMBER) IS
    v_percent NUMBER;
  BEGIN
    v_percent := normalize_percent(p_percent);

    UPDATE employees
       SET salary = salary * (1 + v_percent / 100)
     WHERE employee_id = p_emp_id;
  END give_raise;

  -- Cài đặt function public
  FUNCTION get_salary(p_emp_id IN employees.employee_id%TYPE)
    RETURN NUMBER IS
    v_salary employees.salary%TYPE;
  BEGIN
    SELECT salary INTO v_salary
      FROM employees
     WHERE employee_id = p_emp_id;

    RETURN v_salary;
  END get_salary;

END emp_pkg;
/
```

## 4. Cách sử dụng Package (Input/Output)

### 4.1. Gọi procedure trong package

**Input**: `p_emp_id = 100`, `p_percent = 5`

```sql
BEGIN
  emp_pkg.give_raise(p_emp_id => 100, p_percent => 5);
END;
/
```

**Output (kết quả)**:
- Lương của nhân viên `employee_id = 100` tăng 5% (hoặc tối thiểu 1% nếu truyền < 1).

---

### 4.2. Gọi function trong package

**Input**: `p_emp_id = 100`

```sql
DECLARE
  v_salary NUMBER;
BEGIN
  v_salary := emp_pkg.get_salary(100);
  DBMS_OUTPUT.PUT_LINE('Salary = ' || v_salary);
END;
/
```

**Output (hiển thị)**:
```text
Salary = 12000   -- ví dụ
```

---

## 5. Lợi ích chính của Package

1. **Đóng gói (Encapsulation)**  
   - Ẩn chi tiết cài đặt trong body, chỉ lộ interface trong spec.
   - Có thể thay đổi logic trong body mà không ảnh hưởng code gọi bên ngoài (nếu spec không đổi).

2. **Tổ chức mã tốt hơn**  
   - Nhóm các procedure/function liên quan vào cùng một package (ví dụ: `emp_pkg` cho nghiệp vụ nhân viên).

3. **Hiệu năng**  
   - Khi package được gọi, toàn bộ package có thể được nạp vào bộ nhớ → các lần gọi tiếp theo nhanh hơn.

4. **Bảo trì dễ dàng**  
   - Tách rõ phần **khai báo** (spec) và **cài đặt** (body) giúp quản lý, review, và bảo trì thuận tiện.

---

Tóm lại:  
- **Specification** = “giao diện” (những gì bên ngoài được phép dùng).  
- **Body** = “cài đặt” (chi tiết thực thi, có thể có thành phần private).  
- Dùng package để tổ chức code PL/SQL rõ ràng, dễ bảo trì, và hiệu quả hơn.
