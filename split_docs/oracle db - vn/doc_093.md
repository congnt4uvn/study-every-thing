
Dưới đây là tóm tắt ngắn gọn về **Data Masking / Redaction trong Oracle Database**, kèm ví dụ mã và input/output.

---

## 1. Khái niệm chính

### Data Masking (Che giấu dữ liệu – thường dùng cho môi trường test/dev)
- Dùng để **thay đổi vĩnh viễn** dữ liệu nhạy cảm trong bản sao CSDL (thường là test, dev).
- Mục tiêu: dữ liệu trông “thật” nhưng **không còn là dữ liệu thật** (không thể khôi phục).
- Thường dùng khi:
  - Sao chép dữ liệu từ production sang test/dev.
  - Chia sẻ dữ liệu cho đối tác, nhà thầu, QA, v.v.

### Data Redaction (Ẩn dữ liệu – runtime)
- Dùng để **ẩn hoặc che một phần dữ liệu nhạy cảm khi truy vấn**, tùy theo user/role.
- Dữ liệu trong bảng **vẫn giữ nguyên**, chỉ kết quả SELECT bị che.
- Thường dùng cho:
  - Ứng dụng production.
  - Phân quyền xem dữ liệu: admin xem full, user thường chỉ thấy che/một phần.

---

## 2. Các kiểu che giấu / ẩn dữ liệu phổ biến

- **Full redaction/masking**: che toàn bộ giá trị (ví dụ: `********`).
- **Partial redaction/masking**: che một phần (ví dụ: `0903****89`).
- **Randomization**: thay bằng giá trị ngẫu nhiên nhưng hợp lệ.
- **Substitution**: thay bằng dữ liệu từ tập dữ liệu mẫu (tên giả, địa chỉ giả).
- **Shuffling**: tráo đổi dữ liệu giữa các dòng.

## 3. Ví dụ Data Redaction (Oracle 12c+)

Giả sử có bảng:

```sql
CREATE TABLE customers (
    id          NUMBER,
    full_name   VARCHAR2(100),
    email       VARCHAR2(100),
    phone       VARCHAR2(20),
    card_number VARCHAR2(19)
);

INSERT INTO customers VALUES (1, 'Nguyen Van A', 'a@example.com', '0903123456', '4111-1111-1111-1111');
COMMIT;
```

### 3.1. Che toàn bộ số thẻ cho user thường

```sql
BEGIN
  DBMS_REDACT.ADD_POLICY(
    object_schema   => 'HR',
    object_name     => 'CUSTOMERS',
    column_name     => 'CARD_NUMBER',
    policy_name     => 'REDACT_CARD',
    function_type   => DBMS_REDACT.FULL,
    expression      => 'SYS_CONTEXT(''USERENV'',''SESSION_USER'') <> ''ADMIN_USER'''
  );
END;
/
```

**Giải thích**:  
- Nếu `SESSION_USER` khác `ADMIN_USER` → cột `CARD_NUMBER` bị che toàn bộ.

**Input (SELECT)** – user thường:

```sql
SELECT id, full_name, card_number FROM hr.customers;
```

**Output (kết quả)** – ví dụ:

| ID | FULL_NAME    | CARD_NUMBER        |
|----|--------------|--------------------|
| 1  | Nguyen Van A | ****************** |

**Input (SELECT)** – user `ADMIN_USER`:

```sql
SELECT id, full_name, card_number FROM hr.customers;
```

**Output**:

| ID | FULL_NAME    | CARD_NUMBER         |
|----|--------------|---------------------|
| 1  | Nguyen Van A | 4111-1111-1111-1111 |

---

### 3.2. Che một phần số điện thoại

```sql
BEGIN
  DBMS_REDACT.ADD_POLICY(
    object_schema   => 'HR',
    object_name     => 'CUSTOMERS',
    column_name     => 'PHONE',
    policy_name     => 'REDACT_PHONE',
    function_type   => DBMS_REDACT.PARTIAL,
    function_parameters => DBMS_REDACT.REDACT_US_SSN(
                             '0',  -- giữ lại đầu
                             '4',  -- số ký tự giữ lại đầu
                             'X',  -- ký tự che
                             '2'   -- số ký tự giữ lại cuối
                           ),
    expression      => '1=1'  -- áp dụng cho mọi user
  );
END;
/
```

**Input (SELECT)**:

```sql
SELECT phone FROM hr.customers WHERE id = 1;
```

**Output (ví dụ)**:

| PHONE       |
|------------|
| 0903XXXX56 |

## 4. Ví dụ Data Masking (dùng cho bản sao test/dev)

Data Masking thường thực hiện qua **Oracle Data Masking and Subsetting Pack** (Enterprise Manager / Cloud Control).  
Ở đây là ví dụ logic (PL/SQL đơn giản) để minh họa ý tưởng:

### 4.1. Mask email (thay domain)

```sql
UPDATE customers
SET email = REGEXP_REPLACE(email, '@.*$', '@mask.example.com');
COMMIT;
```

**Trước (input dữ liệu)**:

| ID | EMAIL            |
|----|------------------|
| 1  | a@example.com    |

**Sau (output dữ liệu)**:

| ID | EMAIL                    |
|----|--------------------------|
| 1  | a@mask.example.com       |

---

### 4.2. Mask số thẻ bằng số ngẫu nhiên

```sql
UPDATE customers
SET card_number = 
    TO_CHAR(TRUNC(DBMS_RANDOM.VALUE(4000000000000000, 4999999999999999)));
COMMIT;
```

**Trước**:

| ID | CARD_NUMBER         |
|----|---------------------|
| 1  | 4111-1111-1111-1111 |

**Sau (ví dụ)**:

| ID | CARD_NUMBER         |
|----|---------------------|
| 1  | 4923456789012345    |

(Đây là dữ liệu đã bị thay đổi vĩnh viễn trong môi trường test/dev.)

---

## 5. So sánh nhanh

| Tiêu chí              | Data Masking                         | Data Redaction                          |
|-----------------------|--------------------------------------|-----------------------------------------|
| Thời điểm áp dụng     | Khi copy/sao chép dữ liệu            | Khi chạy truy vấn (runtime)            |
| Dữ liệu gốc           | Bị thay đổi vĩnh viễn                | Không đổi, chỉ che khi hiển thị        |
| Mục đích chính        | Test/dev, chia sẻ dữ liệu            | Bảo vệ dữ liệu trong production         |
| Phụ thuộc user/role   | Thường không                         | Có (policy theo user/role/điều kiện)   |

Nếu bạn cho biết phiên bản Oracle bạn dùng (11g, 12c, 19c, …), mình có thể đưa ví dụ chính xác hơn theo phiên bản đó.
