
Dưới đây là tóm tắt ngắn gọn về **High Availability & Backup với RMAN trong Oracle** (mức khái niệm), kèm ví dụ lệnh và đầu ra mẫu.

---

## 1. Khái niệm chính

- **High Availability (HA)**: Đảm bảo hệ thống Oracle luôn sẵn sàng, giảm tối đa thời gian downtime (kế hoạch và không kế hoạch).
- **Backup & Recovery**: Sao lưu dữ liệu định kỳ và có khả năng khôi phục khi xảy ra lỗi (mất dữ liệu, hỏng đĩa, lỗi người dùng…).
- **RMAN (Recovery Manager)**:
  - Công cụ chuẩn của Oracle để backup/restore.
  - Tích hợp với database, hiểu cấu trúc dữ liệu bên trong.
  - Hỗ trợ backup vật lý (datafile, control file, archivelog…).
  - Tự động quản lý catalog backup, không cần script thủ công phức tạp.

---

## 2. Vai trò của RMAN trong High Availability

- Tạo **backup đầy đủ và nhất quán** để có thể khôi phục nhanh.
- Hỗ trợ **incremental backup** (chỉ backup phần thay đổi) → giảm thời gian backup.
- Kết hợp với các tính năng HA khác (Data Guard, RAC, Flashback) để:
  - Giảm thời gian gián đoạn khi có sự cố.
  - Cho phép khôi phục đến một thời điểm cụ thể (Point-in-Time Recovery).

---

## 3. Các loại backup chính với RMAN

1. **Full Backup**: Sao lưu toàn bộ datafile.
2. **Incremental Backup**:
   - Level 0: giống full backup, làm nền.
   - Level 1: chỉ sao lưu phần thay đổi so với level 0/1 trước đó.
3. **Control File & SPFILE Backup**: Cần thiết để khôi phục cấu trúc database.
4. **Archived Redo Log Backup**: Cần để khôi phục đến thời điểm gần nhất.

---

## 4. Ví dụ lệnh RMAN (có input/output mẫu)

### 4.1. Kết nối RMAN

**Input (shell):**
```bash
rman target /
```

**Output (mẫu):**
```text
Recovery Manager: Release 19.0.0.0.0 - Production
connected to target database: ORCL (DBID=1234567890)
RMAN>
```

---

### 4.2. Backup full database

**Input (RMAN):**
```rman
BACKUP DATABASE;
```

**Output (mẫu rút gọn):**
```text
Starting backup at 20-MAY-26
using channel ORA_DISK_1
channel ORA_DISK_1: starting full datafile backup set
channel ORA_DISK_1: finished piece 1 at 20-MAY-26
Finished backup at 20-MAY-26
```

---

### 4.3. Backup incremental

**Input (RMAN):**
```rman
BACKUP INCREMENTAL LEVEL 0 DATABASE;
BACKUP INCREMENTAL LEVEL 1 DATABASE;
```

**Ý nghĩa:**
- Level 0: backup nền.
- Level 1: chỉ backup phần thay đổi sau level 0/1 trước đó.

---

### 4.4. Backup control file và SPFILE

**Input (RMAN):**
```rman
BACKUP CURRENT CONTROLFILE;
BACKUP SPFILE;
```

---

### 4.5. Backup archived redo logs

**Input (RMAN):**
```rman
BACKUP ARCHIVELOG ALL DELETE INPUT;
```

**Ý nghĩa:**
- Sao lưu tất cả archived redo logs.
- Xóa file archived log trên đĩa sau khi backup thành công (DELETE INPUT).

---

### 4.6. Khôi phục database (mức khái niệm)

**Input (RMAN):**
```rman
STARTUP MOUNT;
RESTORE DATABASE;
RECOVER DATABASE;
ALTER DATABASE OPEN;
```

**Ý nghĩa:**
- `RESTORE`: lấy lại file từ backup.
- `RECOVER`: áp dụng redo/archivelog để đưa dữ liệu đến trạng thái nhất quán.
- `OPEN`: mở database cho người dùng.

---

## 5. Ý chính cần nhớ

- RMAN là công cụ chuẩn của Oracle để **backup & recovery**, đóng vai trò quan trọng trong chiến lược **High Availability**.
- Hỗ trợ:
  - Full & incremental backup.
  - Backup control file, SPFILE, archived redo log.
  - Khôi phục nhanh, khôi phục đến thời điểm cụ thể.
- Kết hợp với các tính năng HA khác (Data Guard, RAC, Flashback) để giảm downtime và bảo vệ dữ liệu.
