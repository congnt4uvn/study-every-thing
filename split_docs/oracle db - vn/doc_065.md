
Dưới đây là tóm tắt ngắn gọn về **Physical Design: Tablespaces** trong Oracle Database, kèm ví dụ mã và I/O minh họa.

---

## 1. Khái niệm Tablespace (khái quát)

- **Tablespace** là lớp logic dùng để nhóm và quản lý dữ liệu trong Oracle.
- Mỗi tablespace gồm một hoặc nhiều **datafile** (tập tin vật lý trên đĩa).
- Người dùng, bảng, index… được lưu trữ trong các tablespace, không trực tiếp trong datafile.
- Thiết kế tablespace là một phần của **physical design** (thiết kế vật lý) để:
  - Tối ưu hiệu năng (I/O, phân tán dữ liệu).
  - Dễ quản trị (backup, phục hồi, phân quyền).
  - Tách biệt dữ liệu theo chức năng (OLTP, báo cáo, tạm, index…).

---

## 2. Các loại tablespace thường gặp

1. **SYSTEM / SYSAUX**
   - Chứa dữ liệu hệ thống, dictionary, metadata.
   - Bắt buộc, được tạo khi tạo database.

2. **USER / APPLICATION TABLESPACES**
   - Chứa bảng, index của ứng dụng.
   - Có thể chia theo module, schema, loại dữ liệu (transaction, history, log…).

3. **TEMPORARY TABLESPACE**
   - Dùng cho sort, hash, operations tạm thời.
   - Giảm sử dụng undo/redo cho dữ liệu tạm.

4. **UNDO TABLESPACE**
   - Lưu thông tin undo để rollback, read-consistency, flashback.

5. **READ ONLY TABLESPACE**
   - Dữ liệu chỉ đọc, không cho phép ghi.
   - Hữu ích cho dữ liệu lịch sử, archive.

---

## 3. Một số quyết định thiết kế vật lý với tablespace

- **Tách dữ liệu nóng / lạnh**:
  - Dữ liệu truy cập thường xuyên (hot) có thể đặt trên storage nhanh.
  - Dữ liệu ít truy cập (cold) trên storage rẻ hơn.

- **Tách bảng và index** (tùy trường hợp):
  - Đặt bảng và index trên tablespace khác nhau để phân tán I/O.

- **Quản lý dung lượng**:
  - Dùng **AUTOEXTEND** cho datafile.
  - Theo dõi và phân bổ tablespace theo nhu cầu tăng trưởng.

- **Chính sách backup / recovery**:
  - Tablespace quan trọng có thể backup thường xuyên hơn.
  - Tablespace read-only có thể backup một lần rồi giữ nguyên.

## 4. Ví dụ mã SQL

### 4.1. Tạo tablespace mới

```sql
CREATE TABLESPACE app_data
  DATAFILE '/u01/oradata/ORCL/app_data01.dbf'
  SIZE 500M
  AUTOEXTEND ON NEXT 100M MAXSIZE 5G
  EXTENT MANAGEMENT LOCAL
  SEGMENT SPACE MANAGEMENT AUTO;
```

**Ý nghĩa chính:**
- `app_data`: tên tablespace.
- Datafile ban đầu 500MB, tự mở rộng 100MB mỗi lần, tối đa 5GB.
- Quản lý extent local, segment space auto (khuyến nghị).

---

### 4.2. Tạo temporary tablespace

```sql
CREATE TEMPORARY TABLESPACE temp_app
  TEMPFILE '/u01/oradata/ORCL/temp_app01.dbf'
  SIZE 1G
  AUTOEXTEND ON NEXT 200M MAXSIZE 10G;
```

---

### 4.3. Gán default tablespace cho user

```sql
CREATE USER app_user IDENTIFIED BY "StrongPass#1"
  DEFAULT TABLESPACE app_data
  TEMPORARY TABLESPACE temp_app
  QUOTA UNLIMITED ON app_data;

GRANT CONNECT, RESOURCE TO app_user;
```

**Kết quả (logic I/O):**
- Khi `app_user` tạo bảng mà không chỉ định tablespace, dữ liệu sẽ vào `app_data`.
- Các thao tác sort lớn của `app_user` sẽ dùng `temp_app`.

---

### 4.4. Tạo bảng trong tablespace cụ thể

```sql
CREATE TABLE customers (
  customer_id   NUMBER PRIMARY KEY,
  name          VARCHAR2(100),
  created_date  DATE
)
TABLESPACE app_data;
```

**I/O minh họa:**
- **Input**: Câu lệnh `INSERT` dữ liệu vào `customers`.
- **Output**: Oracle ghi các block dữ liệu vào datafile thuộc `app_data`.

```sql
INSERT INTO customers (customer_id, name, created_date)
VALUES (1, 'Nguyen Van A', SYSDATE);
COMMIT;
```

- Dữ liệu hàng (row) được lưu trong segment của bảng `customers` trong tablespace `app_data`.
- Datafile `/u01/oradata/ORCL/app_data01.dbf` sẽ tăng sử dụng dung lượng.

---

### 4.5. Đặt tablespace sang READ ONLY

```sql
ALTER TABLESPACE app_data READ ONLY;
-- Sau đó không thể INSERT/UPDATE/DELETE vào các đối tượng trong app_data

ALTER TABLESPACE app_data READ WRITE;
-- Cho phép ghi trở lại
```

---

## 5. Kiểm tra thông tin tablespace

```sql
SELECT tablespace_name, status, contents, logging
FROM dba_tablespaces;

SELECT tablespace_name, file_name, bytes/1024/1024 AS size_mb, autoextensible
FROM dba_data_files
ORDER BY tablespace_name;
```

**Output mẫu (rút gọn):**

| TABLESPACE_NAME | STATUS | CONTENTS | LOGGING |
|-----------------|--------|----------|---------|
| SYSTEM          | ONLINE | PERMANENT| LOGGING |
| SYSAUX          | ONLINE | PERMANENT| LOGGING |
| APP_DATA        | ONLINE | PERMANENT| LOGGING |
| TEMP_APP        | ONLINE | TEMPORARY| NOLOGGING |

---

Tóm lại, trong thiết kế vật lý Oracle, **tablespace** là đơn vị logic quan trọng để:
- Tổ chức dữ liệu.
- Tối ưu hiệu năng I/O.
- Quản lý dung lượng, backup, bảo mật và phân quyền.
