
Dưới đây là tóm tắt ngắn gọn về sự khác nhau giữa **DBMS_SCHEDULER** và **DBMS_JOB** trong Oracle, kèm ví dụ mã và input/output.

---

## 1. Ý chính

- **DBMS_JOB**:  
  - Cũ, đơn giản, đã bị khuyến nghị không dùng cho các hệ thống mới.  
  - Chủ yếu dùng để chạy các job PL/SQL theo lịch đơn giản (interval).  
  - Ít tính năng: không có chain, file watcher, event-based job, window, resource management…

- **DBMS_SCHEDULER**:  
  - Công cụ mới, mạnh hơn, thay thế cho DBMS_JOB.  
  - Hỗ trợ:
    - Lịch phức tạp (calendar expressions).
    - Job chain (chuỗi nhiều bước, điều kiện rẽ nhánh).
    - Event-based job (kích hoạt bởi event, file watcher, queue…).
    - Window, resource consumer group, job class.
    - Quản lý log, history, job run detail tốt hơn.
  - Dùng được cho:
    - PL/SQL block, stored procedure, external executable, shell script, Java, etc.

---

## 2. So sánh nhanh

| Tiêu chí                  | DBMS_JOB                         | DBMS_SCHEDULER                                  |
|---------------------------|-----------------------------------|-------------------------------------------------|
| Tình trạng                | Cũ, nên tránh dùng cho hệ mới    | Khuyến nghị dùng, nhiều tính năng               |
| Định nghĩa lịch           | Interval (biểu thức PL/SQL)      | Calendar (FREQ=DAILY, HOURLY, BYDAY, BYHOUR…)   |
| Loại job                  | PL/SQL block/procedure           | PL/SQL, external program, chain, event-based    |
| Chain / workflow          | Không                            | Có                                              |
| Event-based job           | Không                            | Có                                              |
| Quản lý log, history      | Hạn chế                           | Chi tiết, nhiều view *_SCHEDULER_*              |
| Phân quyền, resource      | Hạn chế                           | Job class, window, resource group               |

---

## 3. Ví dụ với DBMS_JOB

### 3.1. Tạo job đơn giản

**Yêu cầu**: Mỗi 5 phút ghi thời gian hiện tại vào bảng LOG_TABLE.

```sql
-- Bảng log
CREATE TABLE log_table (
    id        NUMBER GENERATED BY DEFAULT AS IDENTITY,
    log_time  DATE,
    message   VARCHAR2(200)
);

-- Thủ tục ghi log
CREATE OR REPLACE PROCEDURE p_write_log IS
BEGIN
    INSERT INTO log_table (log_time, message)
    VALUES (SYSDATE, 'Job chạy bằng DBMS_JOB');
    COMMIT;
END;
/

-- Tạo job với DBMS_JOB
DECLARE
    l_job NUMBER;
BEGIN
    DBMS_JOB.SUBMIT(
        job       => l_job,
        what      => 'p_write_log;',
        next_date => SYSDATE,                 -- chạy ngay
        interval  => 'SYSDATE + 5/1440'       -- 5 phút (5/1440 ngày)
    );
    COMMIT;
END;
/
```

**Kết quả (output)**:  
Sau một thời gian, kiểm tra:

```sql
SELECT * FROM log_table ORDER BY id;
```

Bạn sẽ thấy nhiều dòng với `log_time` cách nhau ~5 phút.

## 4. Ví dụ với DBMS_SCHEDULER

### 4.1. Tạo job tương đương (5 phút/lần)

```sql
-- Thủ tục có thể dùng lại p_write_log ở trên

BEGIN
    DBMS_SCHEDULER.CREATE_JOB (
        job_name        => 'JOB_WRITE_LOG',
        job_type        => 'STORED_PROCEDURE',
        job_action      => 'P_WRITE_LOG',
        start_date      => SYSTIMESTAMP,
        repeat_interval => 'FREQ=MINUTELY; INTERVAL=5',  -- 5 phút
        enabled         => TRUE
    );
END;
/
```

**Kiểm tra job:**

```sql
SELECT job_name, state, last_start_date, next_run_date
FROM   dba_scheduler_jobs
WHERE  job_name = 'JOB_WRITE_LOG';
```

**Kết quả (output)**:  
Một dòng với `JOB_WRITE_LOG`, trạng thái `ENABLED`, có `LAST_START_DATE` và `NEXT_RUN_DATE`.

---

### 4.2. Ví dụ job chạy mỗi ngày lúc 1h sáng

```sql
BEGIN
    DBMS_SCHEDULER.CREATE_JOB (
        job_name        => 'JOB_DAILY_1AM',
        job_type        => 'PLSQL_BLOCK',
        job_action      => q'[
            BEGIN
                INSERT INTO log_table (log_time, message)
                VALUES (SYSDATE, 'Job chạy lúc 1h sáng');
                COMMIT;
            END;
        ]',
        start_date      => SYSTIMESTAMP,
        repeat_interval => 'FREQ=DAILY; BYHOUR=1; BYMINUTE=0; BYSECOND=0',
        enabled         => TRUE
    );
END;
/
```

---

### 4.3. Ví dụ job chạy chương trình ngoài (external executable)

Giả sử đã cấu hình **external job** (extjob, credential, v.v.):

```sql
BEGIN
    DBMS_SCHEDULER.CREATE_PROGRAM (
        program_name   => 'PROG_RUN_SHELL',
        program_type   => 'EXECUTABLE',
        program_action => '/usr/local/bin/myscript.sh',
        number_of_arguments => 0,
        enabled        => TRUE
    );

    DBMS_SCHEDULER.CREATE_JOB (
        job_name        => 'JOB_RUN_SHELL_EVERY_HOUR',
        program_name    => 'PROG_RUN_SHELL',
        repeat_interval => 'FREQ=HOURLY; INTERVAL=1',
        enabled         => TRUE
    );
END;
/
```

---

## 5. Khi nào dùng cái nào?

- Hệ thống mới, cần linh hoạt, logging tốt, lịch phức tạp, chain, event-based:  
  → **Dùng DBMS_SCHEDULER**.
- Hệ thống cũ đã dùng DBMS_JOB, chưa muốn thay đổi:  
  → Có thể giữ tạm, nhưng nên lên kế hoạch chuyển sang DBMS_SCHEDULER.

---

Nếu bạn cho biết phiên bản Oracle bạn đang dùng (ví dụ 11g, 12c, 19c), tôi có thể gợi ý thêm mẫu lịch (repeat_interval) phù hợp và cách debug job cụ thể.
