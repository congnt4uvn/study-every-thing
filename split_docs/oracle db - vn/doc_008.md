
Dưới đây là tóm tắt ngắn gọn về LAG và LEAD trong Oracle, kèm ví dụ:

---

## 1. Khái niệm

**LAG** và **LEAD** là các hàm phân tích (analytic functions) dùng để:
- **LAG**: Lấy giá trị của *dòng trước* (previous row) trong một tập kết quả.
- **LEAD**: Lấy giá trị của *dòng sau* (next row) trong một tập kết quả.

Thường dùng để:
- So sánh giá trị giữa các dòng liên tiếp (ví dụ: doanh số tháng này so với tháng trước).
- Tính chênh lệch, xu hướng, hoặc tạo cột “giá trị trước/sau” mà không cần tự join bảng với chính nó.

Cú pháp cơ bản:

```sql
LAG(expr [, offset [, default]]) OVER (PARTITION BY ... ORDER BY ...)
LEAD(expr [, offset [, default]]) OVER (PARTITION BY ... ORDER BY ...)
```

- `expr`: cột hoặc biểu thức cần lấy.
- `offset`: số dòng lùi (LAG) hoặc tiến (LEAD), mặc định là 1.
- `default`: giá trị trả về nếu không có dòng tương ứng (ví dụ: dòng đầu tiên với LAG, dòng cuối cùng với LEAD).
- `PARTITION BY`: chia nhóm (ví dụ theo nhân viên, theo sản phẩm).
- `ORDER BY`: thứ tự trong mỗi nhóm để xác định “trước” và “sau”.

## 2. Ví dụ cơ bản với LAG

Giả sử có bảng `SALES`:

```sql
CREATE TABLE sales (
    sale_id     NUMBER,
    sale_date   DATE,
    amount      NUMBER
);

INSERT INTO sales (sale_id, sale_date, amount) VALUES (1, DATE '2024-01-01', 100);
INSERT INTO sales (sale_id, sale_date, amount) VALUES (2, DATE '2024-01-05', 150);
INSERT INTO sales (sale_id, sale_date, amount) VALUES (3, DATE '2024-01-10', 120);
COMMIT;
```

### Lấy giá trị dòng trước

```sql
SELECT
    sale_id,
    sale_date,
    amount,
    LAG(amount, 1, 0) OVER (ORDER BY sale_date) AS prev_amount
FROM sales
ORDER BY sale_date;
```

**Kết quả (output):**

| SALE_ID | SALE_DATE   | AMOUNT | PREV_AMOUNT |
|--------:|-------------|--------|-------------|
| 1       | 2024-01-01  | 100    | 0           |
| 2       | 2024-01-05  | 150    | 100         |
| 3       | 2024-01-10  | 120    | 150         |

- Dòng đầu tiên không có dòng trước, nên `prev_amount` = 0 (giá trị default).

### Tính chênh lệch so với dòng trước

```sql
SELECT
    sale_id,
    sale_date,
    amount,
    amount - LAG(amount, 1, amount) OVER (ORDER BY sale_date) AS diff_from_prev
FROM sales
ORDER BY sale_date;
```

**Kết quả:**

| SALE_ID | SALE_DATE   | AMOUNT | DIFF_FROM_PREV |
|--------:|-------------|--------|----------------|
| 1       | 2024-01-01  | 100    | 0              |
| 2       | 2024-01-05  | 150    | 50             |
| 3       | 2024-01-10  | 120    | -30            |

## 3. Ví dụ cơ bản với LEAD

Dùng cùng bảng `SALES`.

### Lấy giá trị dòng sau

```sql
SELECT
    sale_id,
    sale_date,
    amount,
    LEAD(amount, 1, 0) OVER (ORDER BY sale_date) AS next_amount
FROM sales
ORDER BY sale_date;
```

**Kết quả:**

| SALE_ID | SALE_DATE   | AMOUNT | NEXT_AMOUNT |
|--------:|-------------|--------|-------------|
| 1       | 2024-01-01  | 100    | 150         |
| 2       | 2024-01-05  | 150    | 120         |
| 3       | 2024-01-10  | 120    | 0           |

- Dòng cuối cùng không có dòng sau, nên `next_amount` = 0 (default).

### Tính chênh lệch với dòng sau

```sql
SELECT
    sale_id,
    sale_date,
    amount,
    LEAD(amount, 1, amount) OVER (ORDER BY sale_date) - amount AS diff_to_next
FROM sales
ORDER BY sale_date;
```

**Kết quả:**

| SALE_ID | SALE_DATE   | AMOUNT | DIFF_TO_NEXT |
|--------:|-------------|--------|--------------|
| 1       | 2024-01-01  | 100    | 50           |
| 2       | 2024-01-05  | 150    | -30          |
| 3       | 2024-01-10  | 120    | 0            |

## 4. Dùng PARTITION BY (theo nhóm)

Ví dụ bảng `EMP_SALARY`:

```sql
CREATE TABLE emp_salary (
    emp_id      NUMBER,
    dept_id     NUMBER,
    month_year  DATE,
    salary      NUMBER
);

-- Ví dụ dữ liệu (rút gọn)
INSERT INTO emp_salary VALUES (1, 10, DATE '2024-01-01', 1000);
INSERT INTO emp_salary VALUES (1, 10, DATE '2024-02-01', 1200);
INSERT INTO emp_salary VALUES (1, 10, DATE '2024-03-01', 1100);
INSERT INTO emp_salary VALUES (2, 10, DATE '2024-01-01', 900);
INSERT INTO emp_salary VALUES (2, 10, DATE '2024-02-01', 950);
COMMIT;
```

### LAG theo từng nhân viên

```sql
SELECT
    emp_id,
    month_year,
    salary,
    LAG(salary, 1, 0) OVER (
        PARTITION BY emp_id
        ORDER BY month_year
    ) AS prev_salary
FROM emp_salary
ORDER BY emp_id, month_year;
```

- `PARTITION BY emp_id`: mỗi nhân viên là một nhóm riêng, LAG chỉ so sánh trong cùng nhân viên.

---

## 5. Tóm tắt ý chính

- **LAG**: lấy giá trị của dòng trước trong một thứ tự xác định.
- **LEAD**: lấy giá trị của dòng sau trong một thứ tự xác định.
- Dùng với `OVER (PARTITION BY ... ORDER BY ...)` để:
  - So sánh giá trị giữa các dòng liên tiếp.
  - Tính chênh lệch, xu hướng theo thời gian hoặc theo nhóm (nhân viên, sản phẩm, phòng ban…).
- Giúp tránh phải tự join bảng với chính nó, câu SQL gọn và hiệu quả hơn.

Nếu bạn cho thêm nội dung chi tiết hơn (ví dụ bài gốc, use case cụ thể), tôi có thể viết lại tóm tắt sâu hơn và thêm nhiều ví dụ phù hợp hơn.
