
Dưới đây là tóm tắt ngắn gọn về các hint SQL thường dùng trong Oracle để tối ưu truy vấn, kèm ví dụ.

---

## 1. Hint `/*+ INDEX */` – Ép dùng index

**Mục đích:**  
Buộc Optimizer sử dụng một index cụ thể (hoặc ưu tiên index) thay vì full table scan.

**Cú pháp cơ bản:**
```sql
SELECT /*+ INDEX(alias index_name) */ ...
FROM   table_name alias
WHERE  ...;
```

**Ví dụ:**

Giả sử có bảng:
```sql
CREATE TABLE EMP (
    EMP_ID    NUMBER PRIMARY KEY,
    NAME      VARCHAR2(100),
    DEPT_ID   NUMBER
);

CREATE INDEX EMP_DEPT_IDX ON EMP(DEPT_ID);
```

Truy vấn:

```sql
SELECT /*+ INDEX(e EMP_DEPT_IDX) */
       e.EMP_ID, e.NAME, e.DEPT_ID
FROM   EMP e
WHERE  e.DEPT_ID = 10;
```

- **Input:** `DEPT_ID = 10`
- **Output (ví dụ):**

| EMP_ID | NAME        | DEPT_ID |
|--------|-------------|---------|
| 101    | Nguyen A    | 10      |
| 102    | Tran B      | 10      |

Hint `INDEX` hướng Optimizer dùng index `EMP_DEPT_IDX` để tìm các dòng có `DEPT_ID = 10`.

---

## 2. Hint `/*+ FULL */` – Ép full table scan

**Mục đích:**  
Buộc Optimizer đọc toàn bộ bảng (full table scan), thường hữu ích khi:
- Trả về phần lớn số dòng trong bảng
- Bảng nhỏ, full scan nhanh hơn dùng index

**Cú pháp:**
```sql
SELECT /*+ FULL(alias) */ ...
FROM   table_name alias
WHERE  ...;
```

**Ví dụ:**

```sql
SELECT /*+ FULL(e) */
       e.EMP_ID, e.NAME
FROM   EMP e
WHERE  e.NAME LIKE 'N%';
```

- Hint `FULL(e)` yêu cầu Oracle quét toàn bộ bảng `EMP` thay vì dùng index (nếu có).

---

## 3. Hint `/*+ PARALLEL */` – Chạy song song (parallel)

**Mục đích:**  
Cho phép truy vấn chạy song song trên nhiều CPU, tăng tốc với bảng lớn.

**Cú pháp:**
```sql
SELECT /*+ PARALLEL(alias, degree) */ ...
FROM   table_name alias
WHERE  ...;
```

- `degree`: số lượng process song song (ví dụ: 4, 8, 16…)

**Ví dụ:**

```sql
SELECT /*+ PARALLEL(e, 4) */
       COUNT(*)
FROM   EMP e
WHERE  e.DEPT_ID BETWEEN 1 AND 100;
```

- Oracle sẽ cố gắng chia việc đọc bảng `EMP` cho 4 process song song.
- **Output (ví dụ):**

| COUNT(*) |
|----------|
| 50000    |

---

## 4. Kết hợp nhiều hint

Bạn có thể dùng nhiều hint trong cùng một truy vấn:

```sql
SELECT /*+ FULL(e) PARALLEL(e, 8) */
       e.DEPT_ID, COUNT(*) AS CNT
FROM   EMP e
GROUP  BY e.DEPT_ID;
```

- `FULL(e)`: quét toàn bộ bảng `EMP`
- `PARALLEL(e, 8)`: quét song song với độ song song 8

---

## 5. Lưu ý khi dùng hint

- Hint chỉ là “gợi ý mạnh” cho Optimizer, nhưng có thể bị bỏ qua nếu không hợp lệ (sai alias, sai tên index…).
- Dùng hint khi:
  - Đã phân tích plan và thấy Optimizer chọn plan không tối ưu.
  - Có hiểu biết rõ về dữ liệu, index, thống kê.
- Không nên lạm dụng: tốt nhất là tối ưu schema, index, thống kê, và viết lại SQL trước khi dùng hint.

---

Tóm lại:
- `/*+ INDEX */`: ép dùng index → tốt cho truy vấn lọc ít dòng.
- `/*+ FULL */`: ép full table scan → tốt khi trả về nhiều dòng hoặc bảng nhỏ.
- `/*+ PARALLEL */`: chạy song song → tốt cho bảng lớn, truy vấn nặng.
