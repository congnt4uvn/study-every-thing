
Dưới đây là tóm tắt ngắn gọn về “High Availability & Backup” trong Oracle Database, ở mức cơ bản, kèm ví dụ minh họa.

---

## 1. High Availability (HA) – Khả dụng cao

**Mục tiêu:**  
Đảm bảo hệ thống database luôn sẵn sàng, giảm tối đa thời gian ngừng dịch vụ (downtime) khi có sự cố phần cứng, phần mềm, hoặc lỗi con người.

**Ý chính:**
- Giảm thời gian gián đoạn dịch vụ.
- Giảm mất dữ liệu khi có sự cố.
- Tự động chuyển đổi (failover) sang hệ thống dự phòng nếu hệ thống chính gặp lỗi.

**Các cơ chế HA phổ biến trong Oracle (ở mức khái niệm):**
- **Oracle RAC (Real Application Clusters):**  
  Nhiều node cùng truy cập một database, nếu 1 node chết, node khác vẫn phục vụ.
- **Data Guard:**  
  Có database chính (primary) và một hoặc nhiều database dự phòng (standby). Khi primary lỗi, có thể chuyển role cho standby.
- **Flashback Technologies:**  
  Cho phép “quay ngược thời gian” dữ liệu để sửa lỗi logic (xóa nhầm, update nhầm).

### Ví dụ khái niệm: Data Guard (chuyển đổi role)

Giả sử bạn có:
- Primary DB: `PRIM`
- Standby DB: `STBY`

Khi `PRIM` gặp sự cố, bạn có thể chuyển `STBY` thành primary:

```sql
-- Trên STBY (standby database)
ALTER DATABASE RECOVER MANAGED STANDBY DATABASE CANCEL;

ALTER DATABASE ACTIVATE STANDBY DATABASE;
```

**Kết quả (output khái niệm):**
- `STBY` trở thành database primary mới.
- Ứng dụng có thể kết nối lại vào `STBY` để tiếp tục hoạt động.

## 2. Backup – Sao lưu

**Mục tiêu:**
- Bảo vệ dữ liệu trước các sự cố: hỏng đĩa, lỗi con người, lỗi logic, ransomware, v.v.
- Cho phép khôi phục (restore) database về trạng thái trước khi xảy ra sự cố.

**Ý chính:**
- Cần có chiến lược backup định kỳ (hằng ngày, hằng tuần, v.v.).
- Kết hợp backup full, incremental, archive log.
- Lưu backup ở nơi an toàn (có thể là offsite, cloud).

**Các loại backup cơ bản trong Oracle:**
- **Physical backup:** Sao chép file dữ liệu, control file, redo log (thường dùng RMAN).
- **Logical backup:** Xuất dữ liệu dạng logic (dùng `expdp/impdp`).

---

## 3. Ví dụ với RMAN (Recovery Manager)

### 3.1. Backup toàn bộ database

**Input (lệnh RMAN):**
```bash
rman target /

RMAN> BACKUP DATABASE PLUS ARCHIVELOG;
```

**Output (kết quả mong đợi):**
- Tạo bộ backup đầy đủ của database.
- Sao lưu luôn các archive log để có thể khôi phục đến thời điểm gần nhất.

### 3.2. Backup incremental (gia tăng)

**Input:**
```bash
rman target /

RMAN> BACKUP INCREMENTAL LEVEL 1 DATABASE;
```

**Output:**
- Chỉ backup các block dữ liệu thay đổi kể từ lần backup level 0/1 trước đó.
- Giảm thời gian và dung lượng backup.

### 3.3. Khôi phục database từ backup

Giả sử database bị hỏng, bạn muốn khôi phục lại:

**Input:**
```bash
rman target /

RMAN> STARTUP MOUNT;
RMAN> RESTORE DATABASE;
RMAN> RECOVER DATABASE;
RMAN> ALTER DATABASE OPEN RESETLOGS;
```

**Output:**
- Database được khôi phục từ backup.
- Dữ liệu được phục hồi đến thời điểm gần nhất có thể (dựa trên archive log).

## 4. Flashback – Khôi phục lỗi logic nhanh

**Mục tiêu:**  
Sửa lỗi logic (ví dụ: xóa nhầm bảng, update nhầm dữ liệu) mà không cần restore toàn bộ database.

### 4.1. Flashback Table

**Tình huống:**  
Bạn lỡ xóa dữ liệu trong bảng `EMPLOYEES` lúc 10:05, muốn quay lại trạng thái lúc 10:00.

**Input:**
```sql
-- Giả sử đã bật Flashback và có đủ UNDO
FLASHBACK TABLE employees
TO TIMESTAMP TO_TIMESTAMP('2026-05-20 10:00:00', 'YYYY-MM-DD HH24:MI:SS');
```

**Output:**
- Bảng `EMPLOYEES` trở lại trạng thái như lúc 10:00.
- Không cần restore toàn bộ database.

### 4.2. Flashback Database

**Input:**
```sql
-- Đưa database về trạng thái trước đó
SHUTDOWN IMMEDIATE;
STARTUP MOUNT;
FLASHBACK DATABASE TO TIMESTAMP
  TO_TIMESTAMP('2026-05-20 10:00:00', 'YYYY-MM-DD HH24:MI:SS');
ALTER DATABASE OPEN RESETLOGS;
```

**Output:**
- Toàn bộ database quay về trạng thái tại thời điểm chỉ định.

---

## 5. Tóm tắt ngắn gọn

- **High Availability (HA):**
  - Mục tiêu: giảm downtime, giảm mất dữ liệu.
  - Công cụ chính: RAC, Data Guard, Flashback, cơ chế failover/switchover.
- **Backup:**
  - Mục tiêu: bảo vệ dữ liệu, cho phép khôi phục khi có sự cố.
  - Công cụ chính: RMAN (backup/restore/recover), expdp/impdp.
- **Kết hợp HA + Backup:**
  - HA giúp hệ thống luôn chạy.
  - Backup giúp khôi phục dữ liệu khi mọi thứ đều hỏng hoặc lỗi logic nghiêm trọng.

Nếu bạn cung cấp thêm nội dung chi tiết hơn (ví dụ: đoạn tài liệu đầy đủ), tôi có thể tóm tắt sâu hơn và đưa thêm ví dụ code phù hợp.
