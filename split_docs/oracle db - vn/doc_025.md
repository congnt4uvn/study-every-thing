## 1. Khái niệm Bulk Operations trong PL/SQL

Bulk operations giúp xử lý **nhiều dòng dữ liệu cùng lúc** thay vì xử lý từng dòng (row-by-row).  
Mục tiêu: **tăng hiệu năng**, giảm số lần chuyển đổi ngữ cảnh giữa SQL Engine và PL/SQL Engine.

Các kỹ thuật chính:

1. `BULK COLLECT` – lấy nhiều dòng dữ liệu vào collection.
2. `FORALL` – thực hiện DML (INSERT/UPDATE/DELETE/MERGE) cho nhiều dòng trong một lệnh.

---

## 2. BULK COLLECT

### 2.1. Dùng với SELECT

```sql
DECLARE
    TYPE t_emp_tab IS TABLE OF employees%ROWTYPE;
    l_emps t_emp_tab;
BEGIN
    SELECT *
    BULK COLLECT INTO l_emps
    FROM employees
    WHERE department_id = 10;

    DBMS_OUTPUT.PUT_LINE('Số dòng lấy được: ' || l_emps.COUNT);
END;
/
```

**Ý chính:**

- `BULK COLLECT INTO` cho phép lấy **nhiều dòng** vào collection `l_emps`.
- Giảm số lần “fetch” từng dòng.

**Ví dụ input/output (logic):**

- Input: bảng `employees` có 5 nhân viên thuộc `department_id = 10`.
- Output: `l_emps.COUNT = 5`.

---

### 2.2. Dùng với CURSOR

```sql
DECLARE
    CURSOR c_emp IS
        SELECT employee_id, salary
        FROM employees
        WHERE department_id = 10;

    TYPE t_emp_id_tab IS TABLE OF employees.employee_id%TYPE;
    TYPE t_sal_tab    IS TABLE OF employees.salary%TYPE;

    l_emp_ids t_emp_id_tab;
    l_sals    t_sal_tab;
BEGIN
    OPEN c_emp;
    FETCH c_emp BULK COLLECT INTO l_emp_ids, l_sals;
    CLOSE c_emp;

    DBMS_OUTPUT.PUT_LINE('Số nhân viên: ' || l_emp_ids.COUNT);
END;
/
```

---

### 2.3. LIMIT với BULK COLLECT

Dùng `LIMIT` để tránh dùng quá nhiều PGA (bộ nhớ):

```sql
DECLARE
    TYPE t_emp_tab IS TABLE OF employees%ROWTYPE;
    l_emps t_emp_tab;
    CURSOR c_emp IS
        SELECT *
        FROM employees;
BEGIN
    OPEN c_emp;
    LOOP
        FETCH c_emp BULK COLLECT INTO l_emps LIMIT 1000;

        EXIT WHEN l_emps.COUNT = 0;

        -- Xử lý 1000 dòng một lần
        FOR i IN 1 .. l_emps.COUNT LOOP
            NULL; -- xử lý gì đó
        END LOOP;
    END LOOP;
    CLOSE c_emp;
END;
/
```

## 3. FORALL – Bulk DML

`FORALL` dùng để thực hiện **DML hàng loạt** dựa trên dữ liệu trong collection.

### 3.1. Bulk INSERT

```sql
DECLARE
    TYPE t_emp_tab IS TABLE OF employees%ROWTYPE;
    l_emps t_emp_tab;
BEGIN
    -- Lấy dữ liệu nguồn
    SELECT *
    BULK COLLECT INTO l_emps
    FROM employees
    WHERE department_id = 10;

    -- Chèn sang bảng backup
    FORALL i IN l_emps.FIRST .. l_emps.LAST
        INSERT INTO employees_backup
        VALUES l_emps(i);
END;
/
```

**Ý chính:**

- `FORALL` không phải là vòng lặp; nó là lệnh bulk DML.
- Oracle tối ưu để gửi nhiều DML cùng lúc.

---

### 3.2. Bulk UPDATE

```sql
DECLARE
    TYPE t_emp_id_tab IS TABLE OF employees.employee_id%TYPE;
    TYPE t_sal_tab    IS TABLE OF employees.salary%TYPE;

    l_emp_ids t_emp_id_tab;
    l_sals    t_sal_tab;
BEGIN
    -- Lấy danh sách nhân viên và lương mới
    SELECT employee_id, salary * 1.1
    BULK COLLECT INTO l_emp_ids, l_sals
    FROM employees
    WHERE department_id = 10;

    -- Cập nhật lương hàng loạt
    FORALL i IN l_emp_ids.FIRST .. l_emp_ids.LAST
        UPDATE employees
        SET salary = l_sals(i)
        WHERE employee_id = l_emp_ids(i);
END;
/
```

---

### 3.3. Bulk DELETE

```sql
DECLARE
    TYPE t_emp_id_tab IS TABLE OF employees.employee_id%TYPE;
    l_emp_ids t_emp_id_tab;
BEGIN
    SELECT employee_id
    BULK COLLECT INTO l_emp_ids
    FROM employees
    WHERE department_id = 99;

    FORALL i IN l_emp_ids.FIRST .. l_emp_ids.LAST
        DELETE FROM employees
        WHERE employee_id = l_emp_ids(i);
END;
/
```

## 4. SAVE EXCEPTIONS trong FORALL

Cho phép tiếp tục xử lý các phần tử khác dù một số DML bị lỗi.

```sql
DECLARE
    TYPE t_emp_id_tab IS TABLE OF employees.employee_id%TYPE;
    l_emp_ids t_emp_id_tab;
BEGIN
    SELECT employee_id
    BULK COLLECT INTO l_emp_ids
    FROM employees
    WHERE department_id = 10;

    FORALL i IN l_emp_ids.FIRST .. l_emp_ids.LAST SAVE EXCEPTIONS
        UPDATE employees
        SET salary = salary * 1.1
        WHERE employee_id = l_emp_ids(i);

EXCEPTION
    WHEN OTHERS THEN
        FOR j IN 1 .. SQL%BULK_EXCEPTIONS.COUNT LOOP
            DBMS_OUTPUT.PUT_LINE(
                'Lỗi tại index ' || SQL%BULK_EXCEPTIONS(j).ERROR_INDEX ||
                ' - ORA-' || SQL%BULK_EXCEPTIONS(j).ERROR_CODE
            );
        END LOOP;
END;
/
```

**Ý chính:**

- `SAVE EXCEPTIONS` lưu lỗi vào `SQL%BULK_EXCEPTIONS`.
- Có thể xem index phần tử gây lỗi và mã lỗi.

## 5. So sánh Row-by-row vs Bulk

### Row-by-row (chậm):

```sql
FOR rec IN (SELECT * FROM employees WHERE department_id = 10) LOOP
    INSERT INTO employees_backup VALUES rec;
END LOOP;
```

### Bulk (nhanh hơn):

```sql
DECLARE
    TYPE t_emp_tab IS TABLE OF employees%ROWTYPE;
    l_emps t_emp_tab;
BEGIN
    SELECT *
    BULK COLLECT INTO l_emps
    FROM employees
    WHERE department_id = 10;

    FORALL i IN l_emps.FIRST .. l_emps.LAST
        INSERT INTO employees_backup VALUES l_emps(i);
END;
/
```

---

## 6. Ý chính cần nhớ

1. **BULK COLLECT**: lấy nhiều dòng vào collection (TABLE, VARRAY).
2. **FORALL**: thực hiện DML hàng loạt dựa trên collection.
3. Dùng **LIMIT** để kiểm soát bộ nhớ khi xử lý rất nhiều dòng.
4. **SAVE EXCEPTIONS** để không dừng toàn bộ khi một số dòng lỗi.
5. Bulk operations giúp **tăng hiệu năng đáng kể** so với xử lý từng dòng.
