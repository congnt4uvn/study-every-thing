
Dưới đây là tóm tắt ngắn gọn về “Row-level locking” trong Oracle, kèm ví dụ mã và input/output minh họa.

---

## 1. Khái niệm chính

- **Row-level locking**: Oracle khóa ở **mức dòng (row)**, không khóa toàn bộ bảng.
- Mỗi câu lệnh DML (INSERT/UPDATE/DELETE) sẽ:
  - Khóa **chỉ những dòng bị ảnh hưởng**.
  - Cho phép các session khác vẫn đọc/ghi các dòng khác trong cùng bảng.
- Điều này giúp:
  - **Tăng khả năng song song (concurrency)**.
  - Giảm xung đột so với khóa bảng (table-level lock).

---

## 2. Cách hoạt động cơ bản

- Khi một session cập nhật một dòng:
  - Dòng đó bị khóa cho đến khi **COMMIT** hoặc **ROLLBACK**.
  - Session khác:
    - Có thể **đọc** dữ liệu cũ (nhờ cơ chế **read consistency**).
    - Sẽ **bị chờ** nếu cố gắng cập nhật/xóa cùng dòng đó.

---

## 3. Ví dụ minh họa

### 3.1. Chuẩn bị bảng

```sql
CREATE TABLE employees (
    emp_id   NUMBER PRIMARY KEY,
    name     VARCHAR2(100),
    salary   NUMBER
);

INSERT INTO employees VALUES (1, 'A', 1000);
INSERT INTO employees VALUES (2, 'B', 2000);
COMMIT;
```

---

### 3.2. Ví dụ khóa dòng (2 session)

#### Session 1 (S1) – Cập nhật và giữ khóa

```sql
-- S1: cập nhật dòng emp_id = 1
UPDATE employees
SET salary = salary + 500
WHERE emp_id = 1;

-- Chưa COMMIT, dòng emp_id = 1 đang bị khóa
```

**Trạng thái:**

- Dòng `emp_id = 1` bị khóa bởi S1.
- Dòng `emp_id = 2` không bị ảnh hưởng.

#### Session 2 (S2) – Đọc và cập nhật

```sql
-- S2: đọc dữ liệu
SELECT emp_id, name, salary FROM employees ORDER BY emp_id;
```

**Output (S2 thấy dữ liệu cũ, do read consistency):**

```text
EMP_ID | NAME | SALARY
----------------------
1      | A    | 1000   -- chưa thấy +500
2      | B    | 2000
```

```sql
-- S2: thử cập nhật cùng dòng emp_id = 1
UPDATE employees
SET salary = salary + 100
WHERE emp_id = 1;
```

- Lệnh trên **bị treo/chờ** vì dòng `emp_id = 1` đang bị S1 khóa.

#### Session 1 COMMIT

```sql
COMMIT;
```

- Sau khi S1 COMMIT, khóa được giải phóng.
- Lúc này lệnh UPDATE của S2 tiếp tục thực thi.

#### Session 2 sau khi lệnh UPDATE hoàn tất

```sql
COMMIT;
```

---

### 3.3. Kiểm tra kết quả cuối cùng

```sql
SELECT emp_id, name, salary FROM employees ORDER BY emp_id;
```

**Output cuối cùng:**

- Ban đầu: `salary = 1000`
- S1 +500 → 1500
- S2 +100 → 1600

```text
EMP_ID | NAME | SALARY
----------------------
1      | A    | 1600
2      | B    | 2000
```

---

## 4. Khóa dòng tường minh (SELECT FOR UPDATE)

Dùng khi muốn **đọc và giữ khóa** dòng để cập nhật sau:

```sql
-- S1: khóa dòng để cập nhật sau
SELECT emp_id, salary
FROM employees
WHERE emp_id = 2
FOR UPDATE;

-- Dòng emp_id = 2 bị khóa cho đến khi COMMIT/ROLLBACK
```

Session khác cố gắng UPDATE/DELETE `emp_id = 2` sẽ phải chờ.

---

## 5. Tóm tắt ngắn

- Oracle dùng **row-level locking** để:
  - Khóa **chỉ các dòng bị DML tác động**.
  - Cho phép nhiều session làm việc song song trên cùng bảng.
- Dòng bị khóa cho đến khi **COMMIT/ROLLBACK**.
- Session khác:
  - Đọc được dữ liệu nhất quán (trước khi thay đổi).
  - Bị chờ nếu muốn sửa/xóa cùng dòng.
