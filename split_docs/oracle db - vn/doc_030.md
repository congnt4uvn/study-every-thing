
Dưới đây là tóm tắt ngắn gọn về **Dynamic SQL trong PL/SQL** (nâng cao), kèm ví dụ mã và input/output minh họa.

---

## 1. Dynamic SQL là gì?

- **Dynamic SQL**: Câu lệnh SQL được xây dựng như **chuỗi (string)** và chỉ được phân tích/biên dịch **khi chạy** (runtime), không cố định trong mã nguồn.
- Dùng khi:
  - Tên bảng/cột thay đổi theo tham số.
  - Cần xây dựng điều kiện WHERE linh hoạt.
  - Thực thi DDL (CREATE, ALTER, DROP) trong PL/SQL.
  - Cần câu lệnh SQL không thể viết dạng static (cố định).

---

## 2. Các cách thực hiện Dynamic SQL trong PL/SQL

### 2.1. `EXECUTE IMMEDIATE` (Native Dynamic SQL)

Cú pháp cơ bản:

```plsql
EXECUTE IMMEDIATE <sql_string>;
```

#### Ví dụ 1: DDL – tạo bảng động

**Input**: Tên bảng `'EMP_TEMP'`  
**Code**:

```plsql
DECLARE
    v_table_name VARCHAR2(30) := 'EMP_TEMP';
BEGIN
    EXECUTE IMMEDIATE '
        CREATE TABLE ' || v_table_name || ' (
            id   NUMBER,
            name VARCHAR2(100)
        )';
END;
/
```

**Output**: Tạo bảng `EMP_TEMP` trong schema hiện tại.

---

#### Ví dụ 2: DML đơn giản – UPDATE động

**Input**: Tăng lương cho `DEPTNO = 10` thêm 100

```plsql
DECLARE
    v_deptno NUMBER := 10;
    v_raise  NUMBER := 100;
BEGIN
    EXECUTE IMMEDIATE '
        UPDATE emp
           SET sal = sal + :b_raise
         WHERE deptno = :b_deptno'
    USING v_raise, v_deptno;
END;
/
```

- `:b_raise`, `:b_deptno` là **bind variables**.
- **Output**: Các dòng trong `EMP` có `DEPTNO = 10` được tăng lương 100.

---

### 2.2. Dynamic SQL với `INTO` (SELECT một dòng)

```plsql
DECLARE
    v_empno  emp.empno%TYPE := 7369;
    v_ename  emp.ename%TYPE;
    v_sal    emp.sal%TYPE;
BEGIN
    EXECUTE IMMEDIATE
        'SELECT ename, sal FROM emp WHERE empno = :b_empno'
        INTO v_ename, v_sal
        USING v_empno;

    DBMS_OUTPUT.PUT_LINE('ENAME = ' || v_ename || ', SAL = ' || v_sal);
END;
/
```

**Input**: `v_empno = 7369`  
**Output (ví dụ)**:
```text
ENAME = SMITH, SAL = 800
```

---

### 2.3. Dynamic SQL với `RETURNING INTO`

```plsql
DECLARE
    v_empno  emp.empno%TYPE := 7369;
    v_old_sal emp.sal%TYPE;
BEGIN
    EXECUTE IMMEDIATE
        'UPDATE emp
            SET sal = sal * 1.1
          WHERE empno = :b_empno
          RETURNING sal INTO :b_new_sal'
        RETURNING INTO v_old_sal
        USING v_empno;

    DBMS_OUTPUT.PUT_LINE('New salary = ' || v_old_sal);
END;
/
```

## 3. Dynamic SQL với nhiều dòng (BULK COLLECT, CURSOR)

### 3.1. SELECT nhiều dòng với `BULK COLLECT`

```plsql
DECLARE
    TYPE t_emp_tab IS TABLE OF emp%ROWTYPE;
    v_emps t_emp_tab;
    v_deptno NUMBER := 10;
BEGIN
    EXECUTE IMMEDIATE
        'SELECT * FROM emp WHERE deptno = :b_deptno'
        BULK COLLECT INTO v_emps
        USING v_deptno;

    FOR i IN 1 .. v_emps.COUNT LOOP
        DBMS_OUTPUT.PUT_LINE(
            v_emps(i).empno || ' - ' || v_emps(i).ename
        );
    END LOOP;
END;
/
```

**Output**: In ra danh sách nhân viên thuộc phòng ban 10.

---

### 3.2. Dynamic Cursor (OPEN FOR)

Dùng khi câu lệnh SELECT chỉ biết tại runtime.

```plsql
DECLARE
    TYPE ref_cursor IS REF CURSOR;
    c   ref_cursor;
    v_sql   VARCHAR2(1000);
    v_emp   emp%ROWTYPE;
    v_deptno NUMBER := 20;
BEGIN
    v_sql := 'SELECT * FROM emp WHERE deptno = :b_deptno';

    OPEN c FOR v_sql USING v_deptno;

    LOOP
        FETCH c INTO v_emp;
        EXIT WHEN c%NOTFOUND;
        DBMS_OUTPUT.PUT_LINE(v_emp.empno || ' - ' || v_emp.ename);
    END LOOP;

    CLOSE c;
END;
/
```

## 4. Dynamic PL/SQL (khối PL/SQL động)

Có thể chạy cả khối PL/SQL:

```plsql
DECLARE
    v_code VARCHAR2(4000);
BEGIN
    v_code := '
        DECLARE
            v_cnt NUMBER;
        BEGIN
            SELECT COUNT(*) INTO v_cnt FROM emp;
            DBMS_OUTPUT.PUT_LINE(''Total EMP rows: '' || v_cnt);
        END;';

    EXECUTE IMMEDIATE v_code;
END;
/
```

**Output (ví dụ)**:
```text
Total EMP rows: 14
```

## 5. Lưu ý quan trọng

1. **Bind variables** (`USING`)  
   - Giúp tránh SQL Injection, tăng hiệu năng (reuse plan).
   - Không nối trực tiếp giá trị vào chuỗi SQL nếu là input từ người dùng.

2. **SQL Injection**  
   - Không làm:
     ```plsql
     v_sql := 'SELECT * FROM emp WHERE ename = ''' || p_ename || '''';
     ```
   - Nên làm:
     ```plsql
     v_sql := 'SELECT * FROM emp WHERE ename = :b_ename';
     EXECUTE IMMEDIATE v_sql INTO v_emp USING p_ename;
     ```

3. **Hiệu năng**  
   - Dynamic SQL bị parse lại nhiều hơn static SQL.
   - Dùng khi thật sự cần linh hoạt (tên bảng, cột, điều kiện phức tạp).

4. **Quyền (Privileges)**  
   - Dynamic SQL vẫn tuân theo quyền của schema đang chạy.
   - Nếu dùng trong stored procedure, có thể liên quan đến **definer’s rights** vs **invoker’s rights**.

---

## 6. Tóm tắt ý chính

- Dynamic SQL cho phép xây dựng và thực thi câu lệnh SQL/PLSQL **tại thời điểm chạy**.
- Công cụ chính:
  - `EXECUTE IMMEDIATE` (SELECT/INSERT/UPDATE/DELETE/DDL/PLSQL).
  - `OPEN ... FOR` với REF CURSOR cho SELECT nhiều dòng.
- Hỗ trợ:
  - `INTO`, `BULK COLLECT INTO`, `RETURNING INTO`, `USING` (bind variables).
- Cần chú ý:
  - Bảo mật (SQL Injection).
  - Hiệu năng (parse nhiều).
  - Quyền truy cập.

Nếu bạn cho biết thêm nội dung chi tiết hơn (ví dụ: phần nói về DBMS_SQL, bulk operations, hay security), tôi có thể bổ sung ví dụ chuyên sâu hơn.
