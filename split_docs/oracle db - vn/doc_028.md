
Dưới đây là tóm tắt ngắn gọn về **Collections trong PL/SQL nâng cao**, kèm ví dụ mã và input/output. (Vì bạn chỉ đưa tiêu đề, mình sẽ bao quát các ý chính thường có trong chủ đề này.)

---

## 1. Các loại Collection trong PL/SQL

1. **Associative Array (Index-by Table)**
   - Chỉ tồn tại trong PL/SQL (không lưu trực tiếp trong bảng DB).
   - Chỉ số có thể là `PLS_INTEGER` hoặc `VARCHAR2`.
   - Kích thước linh hoạt, không cần khai báo trước.

2. **Nested Table**
   - Có thể dùng trong PL/SQL và lưu được trong cột bảng.
   - Không giới hạn số phần tử (trên lý thuyết).
   - Khi lưu trong DB, các phần tử được lưu trong bảng ẩn.

3. **VARRAY (Variable-size Array)**
   - Kích thước tối đa cố định khi khai báo.
   - Lưu trữ như một đối tượng trong cột bảng.
   - Thích hợp khi số phần tử tối đa đã biết và không quá lớn.

## 2. Khai báo Collection

### 2.1. Associative Array

```sql
DECLARE
  TYPE t_num_tab IS TABLE OF NUMBER INDEX BY PLS_INTEGER;
  l_numbers t_num_tab;
BEGIN
  l_numbers(1) := 10;
  l_numbers(2) := 20;
  DBMS_OUTPUT.PUT_LINE('Element 1 = ' || l_numbers(1)); -- Output: 10
END;
/
```

**Output:**
```text
Element 1 = 10
```

---

### 2.2. Nested Table

```sql
DECLARE
  TYPE t_varchar_tab IS TABLE OF VARCHAR2(50);
  l_names t_varchar_tab := t_varchar_tab();  -- khởi tạo
BEGIN
  l_names.EXTEND(2);
  l_names(1) := 'An';
  l_names(2) := 'Bình';

  FOR i IN 1 .. l_names.COUNT LOOP
    DBMS_OUTPUT.PUT_LINE('Name ' || i || ' = ' || l_names(i));
  END LOOP;
END;
/
```

**Output:**
```text
Name 1 = An
Name 2 = Bình
```

---

### 2.3. VARRAY

```sql
DECLARE
  TYPE t_score_arr IS VARRAY(5) OF NUMBER;  -- tối đa 5 phần tử
  l_scores t_score_arr := t_score_arr(8, 9, 10);
BEGIN
  FOR i IN 1 .. l_scores.COUNT LOOP
    DBMS_OUTPUT.PUT_LINE('Score ' || i || ' = ' || l_scores(i));
  END LOOP;
END;
/
```

**Output:**
```text
Score 1 = 8
Score 2 = 9
Score 3 = 10
```

## 3. Các phương thức quan trọng của Collection

Áp dụng cho Nested Table và VARRAY (Associative Array chỉ hỗ trợ một phần):

- `COUNT`  – số phần tử hiện có.
- `LIMIT`  – số phần tử tối đa (chỉ VARRAY).
- `EXTEND` – thêm phần tử.
- `TRIM`   – xóa phần tử cuối.
- `DELETE` – xóa phần tử (hoặc toàn bộ).
- `FIRST`, `LAST` – chỉ số đầu/cuối.
- `PRIOR`, `NEXT` – chỉ số trước/sau.

### Ví dụ dùng phương thức

```sql
DECLARE
  TYPE t_num_tab IS TABLE OF NUMBER;
  l_tab t_num_tab := t_num_tab();
BEGIN
  l_tab.EXTEND(3);
  l_tab(1) := 100;
  l_tab(2) := 200;
  l_tab(3) := 300;

  DBMS_OUTPUT.PUT_LINE('COUNT = ' || l_tab.COUNT); -- 3

  l_tab.TRIM;  -- xóa phần tử cuối (300)
  DBMS_OUTPUT.PUT_LINE('COUNT after TRIM = ' || l_tab.COUNT); -- 2

  l_tab.DELETE(1); -- xóa phần tử index 1
  DBMS_OUTPUT.PUT_LINE('COUNT after DELETE(1) = ' || l_tab.COUNT); -- vẫn là 2
  DBMS_OUTPUT.PUT_LINE('EXISTS(1) = ' || CASE WHEN l_tab.EXISTS(1) THEN 'Y' ELSE 'N' END);
END;
/
```

**Output (mô phỏng):**
```text
COUNT = 3
COUNT after TRIM = 2
COUNT after DELETE(1) = 2
EXISTS(1) = N
```

## 4. Collection of Records (Collection chứa RECORD)

Cho phép lưu nhiều cột logic trong một collection.

```sql
DECLARE
  TYPE t_emp_rec IS RECORD (
    emp_id   NUMBER,
    emp_name VARCHAR2(50)
  );

  TYPE t_emp_tab IS TABLE OF t_emp_rec;
  l_emps t_emp_tab := t_emp_tab();
BEGIN
  l_emps.EXTEND;
  l_emps(1).emp_id   := 1;
  l_emps(1).emp_name := 'An';

  l_emps.EXTEND;
  l_emps(2).emp_id   := 2;
  l_emps(2).emp_name := 'Bình';

  FOR i IN 1 .. l_emps.COUNT LOOP
    DBMS_OUTPUT.PUT_LINE(
      'ID=' || l_emps(i).emp_id || ', Name=' || l_emps(i).emp_name
    );
  END LOOP;
END;
/
```

**Output:**
```text
ID=1, Name=An
ID=2, Name=Bình
```

## 5. Bulk Collect & FORALL (hiệu năng)

Dùng collection để xử lý nhiều dòng dữ liệu trong một lần, tăng hiệu năng.

### 5.1. BULK COLLECT

```sql
DECLARE
  TYPE t_emp_id_tab IS TABLE OF employees.employee_id%TYPE;
  l_emp_ids t_emp_id_tab;
BEGIN
  SELECT employee_id
  BULK COLLECT INTO l_emp_ids
  FROM employees
  WHERE department_id = 10;

  DBMS_OUTPUT.PUT_LINE('Số nhân viên: ' || l_emp_ids.COUNT);
END;
/
```

### 5.2. FORALL

```sql
DECLARE
  TYPE t_emp_sal_tab IS TABLE OF employees.salary%TYPE INDEX BY PLS_INTEGER;
  l_sals t_emp_sal_tab;
BEGIN
  -- giả sử đã fill l_sals(1..N) với dữ liệu
  FORALL i IN l_sals.FIRST .. l_sals.LAST
    UPDATE employees
    SET salary = l_sals(i)
    WHERE employee_id = i;
END;
/
```

## 6. Collection trong bảng (Nested Table & VARRAY)

Ví dụ tạo kiểu và dùng trong bảng:

```sql
-- Tạo kiểu nested table
CREATE OR REPLACE TYPE t_phone_list AS TABLE OF VARCHAR2(20);
/

-- Tạo bảng dùng nested table
CREATE TABLE customers (
  customer_id NUMBER,
  name        VARCHAR2(100),
  phones      t_phone_list
) NESTED TABLE phones STORE AS phones_nt;
/

-- Insert dữ liệu
INSERT INTO customers (customer_id, name, phones)
VALUES (1, 'An', t_phone_list('0901', '0902'));
COMMIT;
```

Truy vấn:

```sql
SELECT c.customer_id, p.COLUMN_VALUE AS phone
FROM customers c
     CROSS JOIN TABLE(c.phones) p;
```

**Output (mô phỏng):**
```text
CUSTOMER_ID  PHONE
-----------  -----
1            0901
1            0902
```

### Tóm tắt ý chính

- PL/SQL hỗ trợ 3 loại collection: **Associative Array**, **Nested Table**, **VARRAY**.
- Mỗi loại có đặc điểm về chỉ số, khả năng lưu trong bảng, và giới hạn kích thước khác nhau.
- Có nhiều phương thức thao tác collection: `COUNT`, `EXTEND`, `TRIM`, `DELETE`, `FIRST`, `LAST`, `EXISTS`, v.v.
- Có thể tạo **collection of records** để mô hình hóa dữ liệu phức tạp.
- Kết hợp với **BULK COLLECT** và **FORALL** để xử lý dữ liệu hàng loạt, tăng hiệu năng.
- Nested Table và VARRAY có thể được định nghĩa như kiểu SQL và dùng trong cột bảng.
