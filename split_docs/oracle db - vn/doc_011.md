
Dưới đây là tóm tắt ngắn gọn về các khái niệm **CONNECT BY, START WITH, LEVEL** trong Oracle, kèm ví dụ.

---

## 1. Mục đích

Trong Oracle, bộ ba:

- `START WITH`
- `CONNECT BY`
- `LEVEL`

được dùng để:

- Truy vấn dữ liệu dạng **cây / phân cấp** (hierarchical query), ví dụ: cây phòng ban, cây nhân viên – quản lý, cây danh mục.
- Duyệt từ **gốc** đến **lá** hoặc ngược lại.

---

## 2. START WITH

- Xác định **điểm bắt đầu** của cây (các nút gốc).
- Thường là điều kiện chọn các bản ghi gốc (root).

**Ví dụ:**

```sql
SELECT employee_id, first_name, manager_id
FROM employees
START WITH manager_id IS NULL   -- nhân viên không có quản lý, tức là CEO
CONNECT BY PRIOR employee_id = manager_id;
```

**Ý nghĩa:**

- `START WITH manager_id IS NULL`: bắt đầu từ nhân viên không có quản lý (gốc cây).
- Sau đó dùng `CONNECT BY` để đi xuống các cấp dưới.

---

## 3. CONNECT BY

- Xác định **quan hệ cha – con** trong cây.
- Cú pháp cơ bản:

```sql
CONNECT BY [NOCYCLE] PRIOR parent_col = child_col
```

- `PRIOR` cho biết cột nào thuộc **cha**, cột nào thuộc **con**.

**Ví dụ:**

```sql
SELECT employee_id, first_name, manager_id
FROM employees
START WITH manager_id IS NULL
CONNECT BY PRIOR employee_id = manager_id;
```

- `PRIOR employee_id = manager_id` nghĩa là:
  - `employee_id` của bản ghi cha = `manager_id` của bản ghi con.

**Ví dụ có NOCYCLE (tránh vòng lặp):**

```sql
SELECT employee_id, first_name, manager_id
FROM employees
START WITH manager_id IS NULL
CONNECT BY NOCYCLE PRIOR employee_id = manager_id;
```

---

## 4. LEVEL

- `LEVEL` là một **pseudo-column** (cột ảo) trong truy vấn phân cấp.
- Cho biết **cấp độ** của mỗi dòng trong cây:
  - Gốc: `LEVEL = 1`
  - Con trực tiếp của gốc: `LEVEL = 2`
  - Cứ thế tăng dần.

**Ví dụ:**

```sql
SELECT LEVEL,
       LPAD(' ', (LEVEL-1)*2) || first_name AS emp_name,
       employee_id,
       manager_id
FROM employees
START WITH manager_id IS NULL
CONNECT BY PRIOR employee_id = manager_id
ORDER BY LEVEL, employee_id;
```

**Ý nghĩa:**

- `LEVEL`: cấp độ trong cây.
- `LPAD(' ', (LEVEL-1)*2)`: thụt lề theo cấp để nhìn rõ cấu trúc cây.

**Kết quả (ví dụ minh họa):**

| LEVEL | EMP_NAME        | EMPLOYEE_ID | MANAGER_ID |
|------:|-----------------|------------:|-----------:|
| 1     | King            | 100         | (null)     |
| 2     |   Kochhar       | 101         | 100        |
| 2     |   De Haan       | 102         | 100        |
| 3     |     Hunold      | 103         | 102        |
| 4     |       Ernst     | 104         | 103        |

---

## 5. Lọc theo LEVEL

Có thể dùng `WHERE` hoặc `CONNECT_BY_ISLEAF` để lọc:

```sql
-- Chỉ lấy tối đa 3 cấp
SELECT LEVEL, first_name, employee_id, manager_id
FROM employees
START WITH manager_id IS NULL
CONNECT BY PRIOR employee_id = manager_id
WHERE LEVEL <= 3;
```

```sql
-- Chỉ lấy các nút lá (không có cấp dưới)
SELECT LEVEL, first_name, employee_id
FROM employees
START WITH manager_id IS NULL
CONNECT BY PRIOR employee_id = manager_id
AND CONNECT_BY_ISLEAF = 1;
```

---

## 6. Ví dụ đầy đủ (input / output)

**Bảng mẫu:**

```sql
CREATE TABLE departments (
  dept_id     NUMBER PRIMARY KEY,
  dept_name   VARCHAR2(50),
  parent_id   NUMBER
);

INSERT INTO departments VALUES (1, 'Head Office', NULL);
INSERT INTO departments VALUES (2, 'HR',          1);
INSERT INTO departments VALUES (3, 'IT',          1);
INSERT INTO departments VALUES (4, 'Security',    3);
INSERT INTO departments VALUES (5, 'Support',     3);
COMMIT;
```

**Truy vấn phân cấp:**

```sql
SELECT LEVEL,
       LPAD(' ', (LEVEL-1)*2) || dept_name AS dept_tree,
       dept_id,
       parent_id
FROM departments
START WITH parent_id IS NULL
CONNECT BY PRIOR dept_id = parent_id
ORDER BY LEVEL, dept_id;
```

**Kết quả (output):**

| LEVEL | DEPT_TREE        | DEPT_ID | PARENT_ID |
|------:|------------------|--------:|----------:|
| 1     | Head Office      | 1       | (null)    |
| 2     |   HR             | 2       | 1         |
| 2     |   IT             | 3       | 1         |
| 3     |     Security     | 4       | 3         |
| 3     |     Support      | 5       | 3         |

---

### Tóm tắt ngắn

- `START WITH`: xác định nút gốc.
- `CONNECT BY`: định nghĩa quan hệ cha–con (dùng `PRIOR`).
- `LEVEL`: cho biết cấp độ trong cây, dùng để hiển thị và lọc.
