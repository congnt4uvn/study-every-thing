
Dưới đây là tóm tắt ngắn gọn về **bảo mật & người dùng trong Oracle** với **GRANT** và **REVOKE**, kèm ví dụ.

---

## 1. Khái niệm chính

- **GRANT**: Cấp quyền cho user/role (vai trò).
- **REVOKE**: Thu hồi quyền đã cấp.
- Quyền có thể là:
  - **System privileges**: quyền ở mức hệ thống (tạo user, tạo bảng, tạo session…)
  - **Object privileges**: quyền trên đối tượng cụ thể (SELECT, INSERT, UPDATE, DELETE trên bảng, view, sequence…)

---

## 2. Cấp quyền hệ thống (System Privileges)

### Cú pháp

```sql
GRANT system_privilege [, system_privilege...]
TO user_or_role [, user_or_role...]
[WITH ADMIN OPTION];
```

- `WITH ADMIN OPTION`: người được cấp có thể tiếp tục cấp quyền đó cho người khác.

### Ví dụ

#### Ví dụ 1: Cấp quyền tạo session và tạo bảng

**Input:**

```sql
-- Cấp quyền cho user SCOTT
GRANT CREATE SESSION, CREATE TABLE
TO SCOTT;
```

**Ý nghĩa (Output mong đợi):**

- User `SCOTT` có thể:
  - Đăng nhập vào database (CREATE SESSION).
  - Tạo bảng trong schema của mình (CREATE TABLE).

---

#### Ví dụ 2: Cấp quyền với ADMIN OPTION

**Input:**

```sql
GRANT CREATE USER
TO ADMIN_USER
WITH ADMIN OPTION;
```

**Ý nghĩa (Output mong đợi):**

- `ADMIN_USER` có thể:
  - Tạo user mới.
  - Cấp quyền `CREATE USER` cho user khác.

---

## 3. Cấp quyền trên đối tượng (Object Privileges)

### Cú pháp

```sql
GRANT object_privilege [, object_privilege...]
ON object_name
TO user_or_role [, user_or_role...]
[WITH GRANT OPTION];
```

- `WITH GRANT OPTION`: người được cấp có thể cấp tiếp quyền đó trên **chính đối tượng đó** cho người khác.

### Ví dụ

#### Ví dụ 3: Cấp quyền SELECT trên bảng

**Input:**

```sql
-- User HR cấp quyền SELECT trên bảng EMPLOYEES cho user SCOTT
GRANT SELECT
ON HR.EMPLOYEES
TO SCOTT;
```

**Ý nghĩa (Output mong đợi):**

- `SCOTT` có thể chạy:

```sql
SELECT * FROM HR.EMPLOYEES;
```

---

#### Ví dụ 4: Cấp quyền SELECT với GRANT OPTION

**Input:**

```sql
GRANT SELECT
ON HR.EMPLOYEES
TO SCOTT
WITH GRANT OPTION;
```

**Ý nghĩa (Output mong đợi):**

- `SCOTT` có thể:
  - SELECT trên `HR.EMPLOYEES`.
  - Cấp quyền SELECT trên `HR.EMPLOYEES` cho user khác, ví dụ:

```sql
GRANT SELECT ON HR.EMPLOYEES TO JAMES;
```

---

#### Ví dụ 5: Cấp nhiều quyền trên bảng

**Input:**

```sql
GRANT SELECT, INSERT, UPDATE
ON HR.EMPLOYEES
TO SCOTT;
```

**Ý nghĩa (Output mong đợi):**

- `SCOTT` có thể:
  - Xem dữ liệu (SELECT).
  - Thêm dữ liệu (INSERT).
  - Cập nhật dữ liệu (UPDATE) trên `HR.EMPLOYEES`.

---

## 4. Thu hồi quyền (REVOKE)

### Cú pháp – System Privileges

```sql
REVOKE system_privilege [, system_privilege...]
FROM user_or_role [, user_or_role...];
```

### Cú pháp – Object Privileges

```sql
REVOKE object_privilege [, object_privilege...]
ON object_name
FROM user_or_role [, user_or_role...];
```

---

### Ví dụ

#### Ví dụ 6: Thu hồi quyền hệ thống

**Input:**

```sql
REVOKE CREATE TABLE
FROM SCOTT;
```

**Ý nghĩa (Output mong đợi):**

- `SCOTT` không còn được phép tạo bảng mới nữa.

---

#### Ví dụ 7: Thu hồi quyền trên đối tượng

**Input:**

```sql
REVOKE SELECT
ON HR.EMPLOYEES
FROM SCOTT;
```

**Ý nghĩa (Output mong đợi):**

- `SCOTT` không còn được phép `SELECT` trên `HR.EMPLOYEES`.

---

#### Ví dụ 8: Thu hồi quyền đã cấp tiếp (GRANT OPTION)

Nếu `SCOTT` được cấp `WITH GRANT OPTION` và đã cấp cho `JAMES`, khi thu hồi:

**Input:**

```sql
REVOKE SELECT
ON HR.EMPLOYEES
FROM SCOTT;
```

**Kết quả (Output mong đợi):**

- `SCOTT` mất quyền SELECT.
- Tùy cấu hình, các quyền mà `SCOTT` đã cấp cho `JAMES` có thể cũng bị thu hồi theo (cascade).

---

## 5. Tóm tắt ý chính

- **GRANT**: dùng để cấp quyền hệ thống hoặc quyền trên đối tượng cho user/role.
- **REVOKE**: dùng để thu hồi các quyền đã cấp.
- `WITH ADMIN OPTION`: cho phép người được cấp quản lý (cấp/thu hồi) quyền hệ thống đó cho người khác.
- `WITH GRANT OPTION`: cho phép người được cấp tiếp tục cấp quyền trên **đối tượng** cho người khác.
- Quản lý đúng GRANT/REVOKE là cốt lõi của **bảo mật và phân quyền** trong Oracle Database.
