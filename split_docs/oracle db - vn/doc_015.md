
Dưới đây là tóm tắt ngắn gọn về chủ đề “Advanced SQL: Inline views, Common Table Expressions (CTE) với WITH” trong Oracle, kèm ví dụ mã và input/output minh họa.

---

## 1. Inline View (View lồng – subquery trong mệnh đề FROM)

**Ý tưởng chính**  
- Inline view là một subquery được đặt trong mệnh đề `FROM` và được xem như một bảng tạm thời.  
- Dùng để:
  - Đơn giản hóa câu SQL phức tạp.
  - Tái sử dụng kết quả trung gian trong cùng câu lệnh.
  - Áp dụng thêm điều kiện, nhóm, sắp xếp trên kết quả trung gian.

**Cú pháp cơ bản**

```sql
SELECT ...
FROM (
    SELECT ...
    FROM ...
    WHERE ...
) alias
WHERE ...;
```

### Ví dụ 1: Lọc trên kết quả đã tổng hợp

**Bài toán**: Lấy các phòng ban có tổng lương > 100000.

**Input (bảng EMPLOYEES – cột chính)**  
- EMPLOYEE_ID  
- FIRST_NAME  
- SALARY  
- DEPARTMENT_ID  

```sql
SELECT department_id, total_salary
FROM (
    SELECT department_id,
           SUM(salary) AS total_salary
    FROM employees
    GROUP BY department_id
) dept_sum
WHERE total_salary > 100000;
```

**Output (ví dụ)**

| DEPARTMENT_ID | TOTAL_SALARY |
|---------------|--------------|
| 10            | 150000       |
| 20            | 130000       |

## 2. Common Table Expression (CTE) với `WITH`

**Ý tưởng chính**  
- CTE là một subquery được đặt tên, khai báo trước câu lệnh chính bằng từ khóa `WITH`.  
- Tương tự inline view nhưng:
  - Dễ đọc hơn, đặc biệt với nhiều tầng logic.
  - Có thể tham chiếu nhiều lần trong cùng câu lệnh.
  - Hỗ trợ đệ quy (recursive CTE) trong một số hệ quản trị (Oracle 11g+).

**Cú pháp cơ bản**

```sql
WITH cte_name AS (
    SELECT ...
    FROM ...
    WHERE ...
)
SELECT ...
FROM cte_name
WHERE ...;
```

### Ví dụ 2: CTE đơn – tổng lương phòng ban

Tương đương ví dụ inline view ở trên nhưng dùng `WITH`.

```sql
WITH dept_sum AS (
    SELECT department_id,
           SUM(salary) AS total_salary
    FROM employees
    GROUP BY department_id
)
SELECT department_id, total_salary
FROM dept_sum
WHERE total_salary > 100000;
```

**Output (ví dụ)**

| DEPARTMENT_ID | TOTAL_SALARY |
|---------------|--------------|
| 10            | 150000       |
| 20            | 130000       |

## 3. Nhiều CTE trong một câu lệnh

**Ý tưởng chính**  
- Có thể khai báo nhiều CTE, phân tách bằng dấu phẩy.  
- CTE sau có thể dùng CTE trước.

```sql
WITH
  high_salary AS (
      SELECT employee_id, first_name, salary, department_id
      FROM employees
      WHERE salary > 10000
  ),
  dept_avg AS (
      SELECT department_id, AVG(salary) AS avg_salary
      FROM employees
      GROUP BY department_id
  )
SELECT h.employee_id,
       h.first_name,
       h.salary,
       d.avg_salary
FROM high_salary h
JOIN dept_avg d
  ON h.department_id = d.department_id;
```

**Output (ví dụ)**

| EMPLOYEE_ID | FIRST_NAME | SALARY | AVG_SALARY |
|-------------|------------|--------|------------|
| 101         | John       | 12000  | 9000       |
| 105         | Mary       | 15000  | 11000      |

## 4. CTE đệ quy (Recursive CTE) trong Oracle

**Ý tưởng chính**  
- Dùng để xử lý cấu trúc phân cấp (cây), ví dụ: nhân viên – quản lý, danh mục – danh mục con.  
- Gồm 2 phần:
  - Anchor member: truy vấn gốc.
  - Recursive member: truy vấn tham chiếu lại chính CTE.

**Cú pháp tổng quát (Oracle)**

```sql
WITH cte_name (col1, col2, ...) AS (
    -- Anchor member
    SELECT ...
    FROM ...
    WHERE ...

    UNION ALL

    -- Recursive member
    SELECT ...
    FROM cte_name
    JOIN ...
    ON ...
)
SELECT * FROM cte_name;
```

### Ví dụ 3: Cây phân cấp nhân viên

**Input (bảng EMPLOYEES – cột chính)**  
- EMPLOYEE_ID  
- FIRST_NAME  
- MANAGER_ID  

**Bài toán**: Lấy toàn bộ cấp dưới của một manager (ví dụ EMPLOYEE_ID = 100), kèm cấp bậc (level).

```sql
WITH emp_hierarchy (employee_id, first_name, manager_id, lvl) AS (
    -- Anchor: manager gốc
    SELECT employee_id, first_name, manager_id, 1 AS lvl
    FROM employees
    WHERE employee_id = 100

    UNION ALL

    -- Recursive: nhân viên có manager_id = employee_id của cấp trên
    SELECT e.employee_id, e.first_name, e.manager_id, eh.lvl + 1
    FROM employees e
    JOIN emp_hierarchy eh
      ON e.manager_id = eh.employee_id
)
SELECT employee_id, first_name, manager_id, lvl
FROM emp_hierarchy
ORDER BY lvl, employee_id;
```

**Output (ví dụ)**

| EMPLOYEE_ID | FIRST_NAME | MANAGER_ID | LVL |
|-------------|------------|------------|-----|
| 100         | Steven     | NULL       | 1   |
| 101         | Neena      | 100        | 2   |
| 102         | Lex        | 100        | 2   |
| 103         | Alexander  | 101        | 3   |
| 104         | Bruce      | 102        | 3   |

## 5. So sánh nhanh: Inline View vs CTE

- **Inline View**  
  - Viết trực tiếp trong `FROM`.  
  - Tốt cho truy vấn ngắn, ít tầng.  
  - Khó đọc khi lồng nhiều cấp.

- **CTE (WITH)**  
  - Đặt tên, tách logic rõ ràng.  
  - Dễ bảo trì, dễ tái sử dụng trong cùng câu lệnh.  
  - Hỗ trợ đệ quy (xử lý cây, phân cấp).  

---

Tóm lại, trong Oracle:
- Dùng **inline view** khi cần một subquery đơn giản trong `FROM`.
- Dùng **CTE với `WITH`** khi truy vấn phức tạp, cần nhiều bước trung gian, hoặc cần **đệ quy** để xử lý dữ liệu phân cấp.
