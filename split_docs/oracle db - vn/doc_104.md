
Dưới đây là tóm tắt ngắn gọn về chủ đề “Version Control & CI/CD: Migration scripts” trong bối cảnh Oracle Database, kèm ví dụ.

---

## 1. Ý chính

1. **Migration scripts là gì?**  
   - Là các file SQL (hoặc PL/SQL) mô tả thay đổi cấu trúc/ dữ liệu DB: tạo bảng, sửa cột, thêm index, seed data, v.v.  
   - Mỗi script tương ứng với một “bước tiến hóa” của schema.

2. **Version Control (Git, v.v.)**  
   - Lưu toàn bộ migration scripts trong repo (không chỉ lưu schema hiện tại).  
   - Đặt tên có thứ tự/ID để đảm bảo thứ tự chạy, ví dụ:  
     - `V001__create_users_table.sql`  
     - `V002__add_status_to_users.sql`  
   - Mỗi commit thể hiện một thay đổi DB cụ thể, giúp:
     - Theo dõi lịch sử thay đổi.
     - Rollback/khôi phục khi cần.
     - Đồng bộ giữa các môi trường (dev/test/stage/prod).

3. **CI/CD với migration scripts**  
   - Pipeline CI/CD tự động:
     1. Lấy code + migration scripts từ Git.
     2. Kết nối tới Oracle (dev/test/stage/prod).
     3. Kiểm tra các migration chưa chạy.
     4. Chạy lần lượt các script mới.
   - Đảm bảo mọi môi trường đều được cập nhật schema giống nhau, không phụ thuộc thao tác thủ công.

4. **Nguyên tắc quan trọng**  
   - Migration **bất biến**: không sửa file migration đã chạy ở môi trường khác; nếu cần sửa, tạo migration mới.  
   - Script phải **idempotent** hoặc được kiểm tra điều kiện (tồn tại hay chưa) để tránh lỗi khi chạy lại.  
   - Có cơ chế **log/metadata** trong DB để biết script nào đã chạy (bảng versioning).

## 2. Ví dụ migration script (Oracle SQL)

### 2.1. Tạo bảng người dùng

**File:** `V001__create_users_table.sql`

```sql
-- V001__create_users_table.sql
BEGIN
  EXECUTE IMMEDIATE '
    CREATE TABLE users (
      id          NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
      username    VARCHAR2(50) NOT NULL UNIQUE,
      email       VARCHAR2(100) NOT NULL UNIQUE,
      created_at  DATE DEFAULT SYSDATE
    )
  ';
EXCEPTION
  WHEN OTHERS THEN
    -- Nếu bảng đã tồn tại thì bỏ qua
    IF SQLCODE != -955 THEN -- ORA-00955: name is already used by an existing object
      RAISE;
    END IF;
END;
/
```

**Input:** chạy script trên DB chưa có bảng `users`.  
**Output:** tạo bảng `USERS` với các cột `id, username, email, created_at`.

---

### 2.2. Thêm cột `status` vào bảng `users`

**File:** `V002__add_status_to_users.sql`

```sql
-- V002__add_status_to_users.sql
DECLARE
  v_count NUMBER;
BEGIN
  SELECT COUNT(*)
  INTO v_count
  FROM user_tab_columns
  WHERE table_name = 'USERS'
    AND column_name = 'STATUS';

  IF v_count = 0 THEN
    EXECUTE IMMEDIATE '
      ALTER TABLE users
      ADD (status VARCHAR2(20) DEFAULT ''ACTIVE'' NOT NULL)
    ';
  END IF;
END;
/
```

**Input:** chạy script trên DB đã có bảng `USERS` nhưng chưa có cột `STATUS`.  
**Output:** thêm cột `STATUS` với giá trị mặc định `ACTIVE`.

## 3. Ví dụ tích hợp trong CI/CD (pseudo-code)

Giả sử dùng một tool (hoặc script) để:

1. Đọc danh sách file `Vxxx__*.sql` trong thư mục `db/migrations`.
2. So sánh với bảng `SCHEMA_VERSION` trong Oracle.
3. Chạy các file chưa có trong `SCHEMA_VERSION`.

**Bảng lưu version:**

```sql
CREATE TABLE schema_version (
  version       VARCHAR2(50) PRIMARY KEY,
  description   VARCHAR2(255),
  installed_on  DATE DEFAULT SYSDATE,
  success       CHAR(1) CHECK (success IN ('Y','N'))
);
```

**Pseudo-code (Python):**

```python
import cx_Oracle
import glob
import os

conn = cx_Oracle.connect("user", "pass", "host:1521/ORCLPDB1")
cur = conn.cursor()

# Lấy danh sách version đã chạy
cur.execute("SELECT version FROM schema_version WHERE success = 'Y'")
done_versions = {row[0] for row in cur}

# Lấy danh sách file migration
files = sorted(glob.glob("db/migrations/V*.sql"))

for path in files:
    filename = os.path.basename(path)
    version = filename.split("__")[0]  # V001, V002, ...

    if version in done_versions:
        continue  # đã chạy rồi

    with open(path, "r", encoding="utf-8") as f:
        sql = f.read()

    try:
        for stmt in sql.split("/\n"):
            if stmt.strip():
                cur.execute(stmt)
        cur.execute(
            "INSERT INTO schema_version (version, description, success) "
            "VALUES (:v, :d, 'Y')",
            v=version,
            d=filename
        )
        conn.commit()
    except Exception as e:
        cur.execute(
            "INSERT INTO schema_version (version, description, success) "
            "VALUES (:v, :d, 'N')",
            v=version,
            d=str(e)
        )
        conn.commit()
        raise

cur.close()
conn.close()
```

**Input:** repo chứa các file `V001__...sql`, `V002__...sql`, v.v.  
**Output:**  
- DB được cập nhật theo thứ tự migration.  
- Bảng `SCHEMA_VERSION` ghi nhận version đã chạy thành công/thất bại.

---

## 4. Tóm tắt ngắn

- Lưu **migration scripts** trong **version control** để theo dõi lịch sử và đồng bộ schema.  
- Dùng **CI/CD** để tự động chạy các migration mới lên Oracle ở mọi môi trường.  
- Mỗi migration là một bước nhỏ, bất biến, có thứ tự rõ ràng.  
- Có bảng metadata (như `SCHEMA_VERSION`) để biết script nào đã chạy.
