
Dưới đây là tóm tắt ngắn gọn về chủ đề:

**“Advanced PL/SQL: Using bind variables in dynamic SQL”**  
(Biến ràng buộc trong SQL động – PL/SQL nâng cao)

---

## 1. Khái niệm chính

- **SQL động (Dynamic SQL)**: Câu lệnh SQL được xây dựng dưới dạng chuỗi (string) và chỉ được biên dịch/thực thi khi chạy (runtime), thường dùng với `EXECUTE IMMEDIATE` hoặc gói `DBMS_SQL`.
- **Bind variable (biến ràng buộc)**: Biến được “gắn” vào câu SQL thay vì nối chuỗi trực tiếp.  
  Ví dụ: `WHERE emp_id = :id` thay vì `WHERE emp_id = 100`.

---

## 2. Lý do phải dùng bind variable trong dynamic SQL

1. **Hiệu năng tốt hơn**  
   - Oracle có thể **tái sử dụng plan** (kế hoạch thực thi) vì câu SQL có cùng cấu trúc, chỉ khác giá trị bind.
   - Giảm **hard parse**, giảm CPU, tăng khả năng mở rộng.

2. **Bảo mật**  
   - Giảm nguy cơ **SQL Injection** vì không nối chuỗi trực tiếp với dữ liệu người dùng.

3. **Quản lý bộ nhớ tốt hơn**  
   - Ít câu SQL khác nhau trong shared pool → ít phân mảnh, ít chiếm bộ nhớ.

---

## 3. Cách dùng bind variable với `EXECUTE IMMEDIATE`

### 3.1. Bind biến vào phần `VALUES` (INSERT, UPDATE, WHERE…)

**Ví dụ 1: SELECT INTO với bind variable**

```plsql
DECLARE
    l_sql   VARCHAR2(1000);
    l_empno NUMBER := 7369;
    l_ename VARCHAR2(100);
BEGIN
    l_sql := 'SELECT ename FROM emp WHERE empno = :id';

    EXECUTE IMMEDIATE l_sql
        INTO l_ename
        USING l_empno;

    DBMS_OUTPUT.PUT_LINE('Tên nhân viên: ' || l_ename);
END;
/
```

**Input**: `l_empno = 7369`  
**Output (ví dụ)**: `Tên nhân viên: SMITH`

---

**Ví dụ 2: INSERT với bind variable**

```plsql
DECLARE
    l_sql   VARCHAR2(1000);
    l_empno NUMBER := 9000;
    l_ename VARCHAR2(100) := 'NEW_EMP';
BEGIN
    l_sql := 'INSERT INTO emp(empno, ename) VALUES(:id, :name)';

    EXECUTE IMMEDIATE l_sql
        USING l_empno, l_ename;

    COMMIT;
END;
/
```

---

**Ví dụ 3: UPDATE với bind variable**

```plsql
DECLARE
    l_sql   VARCHAR2(1000);
    l_empno NUMBER := 7369;
    l_new_sal NUMBER := 2000;
BEGIN
    l_sql := 'UPDATE emp SET sal = :new_sal WHERE empno = :id';

    EXECUTE IMMEDIATE l_sql
        USING l_new_sal, l_empno;

    COMMIT;
END;
/
```

---

### 3.2. Bind biến vào `RETURNING INTO`

```plsql
DECLARE
    l_sql   VARCHAR2(1000);
    l_empno NUMBER := 9001;
    l_ename VARCHAR2(100) := 'TEST_EMP';
    l_rowid ROWID;
BEGIN
    l_sql := 'INSERT INTO emp(empno, ename) 
              VALUES(:id, :name)
              RETURNING ROWID INTO :rid';

    EXECUTE IMMEDIATE l_sql
        USING l_empno, l_ename
        RETURNING INTO l_rowid;

    DBMS_OUTPUT.PUT_LINE('ROWID mới: ' || l_rowid);
END;
/
```

## 4. Bind variable với `DBMS_SQL`

Dùng khi:

- Câu SQL phức tạp, số cột không cố định.
- Cần xử lý nhiều dòng (bulk) hoặc dynamic DDL.

**Ví dụ: SELECT với DBMS_SQL và bind variable**

```plsql
DECLARE
    l_cursor   INTEGER;
    l_sql      VARCHAR2(1000);
    l_empno    NUMBER := 7369;
    l_ename    VARCHAR2(100);
    l_status   INTEGER;
BEGIN
    l_sql := 'SELECT ename FROM emp WHERE empno = :id';

    l_cursor := DBMS_SQL.open_cursor;
    DBMS_SQL.parse(l_cursor, l_sql, DBMS_SQL.native);

    DBMS_SQL.bind_variable(l_cursor, ':id', l_empno);
    DBMS_SQL.define_column(l_cursor, 1, l_ename, 100);

    l_status := DBMS_SQL.execute(l_cursor);

    IF DBMS_SQL.fetch_rows(l_cursor) > 0 THEN
        DBMS_SQL.column_value(l_cursor, 1, l_ename);
        DBMS_OUTPUT.PUT_LINE('Tên nhân viên: ' || l_ename);
    END IF;

    DBMS_SQL.close_cursor(l_cursor);
END;
/
```

---

## 5. Lưu ý quan trọng

- **Không** nối trực tiếp giá trị vào chuỗi SQL, ví dụ:
  ```plsql
  -- KHÔNG NÊN
  l_sql := 'SELECT ename FROM emp WHERE empno = ' || l_empno;
  ```
  Thay bằng:
  ```plsql
  l_sql := 'SELECT ename FROM emp WHERE empno = :id';
  EXECUTE IMMEDIATE l_sql INTO l_ename USING l_empno;
  ```

- Chỉ nên nối chuỗi cho **tên bảng, tên cột, từ khóa** (nếu bắt buộc), còn **giá trị** phải dùng bind variable.
- Khi có nhiều bind variable, thứ tự trong `USING` phải khớp với thứ tự xuất hiện trong câu SQL.
