
Dưới đây là tóm tắt ngắn gọn về **Security & Users trong Oracle** với ví dụ code (có input/output minh họa).

---

## 1. Người dùng (Users)

- Mỗi user là một tài khoản đăng nhập vào Oracle.
- Mỗi user có:
  - Mật khẩu
  - Tablespace mặc định
  - Các quyền (privileges) trực tiếp hoặc thông qua role.

**Ví dụ: tạo user**

```sql
-- INPUT
CREATE USER app_user IDENTIFIED BY "StrongPass123"
  DEFAULT TABLESPACE users
  TEMPORARY TABLESPACE temp;

-- Cần cấp quyền đăng nhập
GRANT CREATE SESSION TO app_user;
```

**Kết quả (OUTPUT):**
- User `APP_USER` được tạo.
- Có thể đăng nhập nếu có quyền `CREATE SESSION`.

---

## 2. Privileges (Quyền)

Có 2 loại chính:

### 2.1. System Privileges (Quyền hệ thống)

- Cho phép thực hiện các thao tác ở mức hệ thống hoặc schema.
- Ví dụ: `CREATE SESSION`, `CREATE TABLE`, `CREATE USER`, `DROP ANY TABLE`, …

**Ví dụ: cấp system privilege**

```sql
-- INPUT
GRANT CREATE SESSION, CREATE TABLE TO app_user;
```

**Kết quả (OUTPUT):**
- `APP_USER` có thể:
  - Đăng nhập vào DB (`CREATE SESSION`)
  - Tạo bảng trong schema của chính mình (`CREATE TABLE`)

---

### 2.2. Object Privileges (Quyền trên đối tượng)

- Áp dụng cho các đối tượng cụ thể: TABLE, VIEW, SEQUENCE, PROCEDURE, …
- Ví dụ: `SELECT`, `INSERT`, `UPDATE`, `DELETE` trên một bảng cụ thể.

**Ví dụ: cấp object privilege**

```sql
-- Tạo bảng bởi user OWNER_USER
CREATE TABLE employees (
  emp_id   NUMBER PRIMARY KEY,
  emp_name VARCHAR2(100),
  salary   NUMBER
);

-- Cấp quyền SELECT và INSERT trên bảng employees cho APP_USER
GRANT SELECT, INSERT ON employees TO app_user;
```

**Kết quả (OUTPUT):**
- `APP_USER` có thể:
  - `SELECT` dữ liệu từ `OWNER_USER.EMPLOYEES`
  - `INSERT` dữ liệu vào `OWNER_USER.EMPLOYEES`
- Không thể `UPDATE` hoặc `DELETE` nếu không được cấp thêm.

---

## 3. Roles (Vai trò)

- Role là tập hợp các privileges (system + object).
- Dùng để quản lý quyền dễ dàng hơn:
  - Cấp nhiều quyền cho role
  - Sau đó chỉ cần `GRANT role` cho user.

**Ví dụ: tạo role và cấp cho user**

```sql
-- INPUT: tạo role
CREATE ROLE app_role;

-- Cấp system privileges cho role
GRANT CREATE SESSION, CREATE TABLE TO app_role;

-- Cấp object privilege cho role
GRANT SELECT, INSERT ON employees TO app_role;

-- Gán role cho user
GRANT app_role TO app_user;
```

**Kết quả (OUTPUT):**
- `APP_USER` nhận tất cả quyền gắn với `APP_ROLE`:
  - Đăng nhập
  - Tạo bảng trong schema của mình
  - SELECT/INSERT trên bảng `EMPLOYEES`.

---

## 4. Tóm tắt ý chính

- **User**: tài khoản đăng nhập, sở hữu schema.
- **System privileges**: quyền ở mức hệ thống (tạo bảng, tạo user, đăng nhập, …).
- **Object privileges**: quyền trên đối tượng cụ thể (SELECT/INSERT/UPDATE/DELETE trên một bảng, EXECUTE trên procedure, …).
- **Role**: nhóm các privileges, giúp quản lý và cấp quyền cho user một cách tập trung và dễ dàng.
