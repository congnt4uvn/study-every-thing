
Dưới đây là tóm tắt ngắn gọn về **Physical Design: Partitioning trong Oracle Database**, kèm ví dụ code. (Do bạn chỉ đưa tiêu đề, mình sẽ tóm theo nội dung chuẩn về partitioning trong Oracle.)

---

## 1. Partitioning là gì?

- Là kỹ thuật **chia một bảng hoặc index lớn** thành nhiều phần nhỏ (partition) dựa trên một khóa (partition key).
- Về logic: người dùng vẫn thấy **1 bảng**.
- Về vật lý: dữ liệu được lưu ở **nhiều phân vùng** khác nhau → dễ quản lý, tối ưu hiệu năng.

---

## 2. Lợi ích chính

1. **Hiệu năng**  
   - Query có điều kiện trên cột partition key có thể dùng **partition pruning** (bỏ qua các partition không liên quan).
   - Giảm I/O, tăng tốc độ truy vấn.

2. **Quản trị & bảo trì**  
   - Có thể **MOVE, SPLIT, MERGE, TRUNCATE** từng partition mà không ảnh hưởng toàn bảng.
   - Dễ **archiving** dữ liệu cũ (drop/truncate partition cũ).

3. **Khả năng mở rộng (scalability)**  
   - Bảng rất lớn (VLDB) vẫn quản lý được.
   - Có thể đặt partition ở **tablespace khác nhau** (phân tán I/O).

---

## 3. Các kiểu partition phổ biến

1. **Range Partitioning**  
   - Chia theo khoảng giá trị (thường dùng cho ngày/thời gian).
   - Ví dụ: mỗi tháng một partition.

2. **List Partitioning**  
   - Chia theo danh sách giá trị cụ thể (ví dụ: theo vùng, quốc gia).

3. **Hash Partitioning**  
   - Chia theo hàm băm trên cột khóa → phân bố đều dữ liệu.

4. **Composite Partitioning**  
   - Kết hợp 2 kiểu, ví dụ: RANGE-HASH, RANGE-LIST.

---

## 4. Ví dụ code

### 4.1. Range Partitioning theo ngày

```sql
CREATE TABLE sales (
    sale_id      NUMBER,
    sale_date    DATE,
    amount       NUMBER
)
PARTITION BY RANGE (sale_date) (
    PARTITION p_2024_q1 VALUES LESS THAN (DATE '2024-04-01'),
    PARTITION p_2024_q2 VALUES LESS THAN (DATE '2024-07-01'),
    PARTITION p_2024_q3 VALUES LESS THAN (DATE '2024-10-01'),
    PARTITION p_2024_q4 VALUES LESS THAN (DATE '2025-01-01')
);
```

**Input (INSERT):**

```sql
INSERT INTO sales (sale_id, sale_date, amount)
VALUES (1, DATE '2024-02-15', 1000);

INSERT INTO sales (sale_id, sale_date, amount)
VALUES (2, DATE '2024-08-10', 2000);
```

**Kết quả (phân vùng):**

- Bản ghi `sale_id = 1` → vào partition `p_2024_q1`.
- Bản ghi `sale_id = 2` → vào partition `p_2024_q3`.

**Query sử dụng partition pruning:**

```sql
SELECT *
FROM   sales
WHERE  sale_date BETWEEN DATE '2024-04-01' AND DATE '2024-06-30';
```

Oracle chỉ đọc partition `p_2024_q2` (và có thể một phần liên quan), không cần quét toàn bảng.

---

### 4.2. List Partitioning theo vùng

```sql
CREATE TABLE customers (
    customer_id   NUMBER,
    region        VARCHAR2(10),
    name          VARCHAR2(100)
)
PARTITION BY LIST (region) (
    PARTITION p_north VALUES ('NORTH'),
    PARTITION p_south VALUES ('SOUTH'),
    PARTITION p_central VALUES ('CENTRAL'),
    PARTITION p_other VALUES (DEFAULT)
);
```

**Input (INSERT):**

```sql
INSERT INTO customers VALUES (1, 'NORTH', 'A Corp');
INSERT INTO customers VALUES (2, 'SOUTH', 'B Corp');
INSERT INTO customers VALUES (3, 'WEST',  'C Corp');  -- không khớp, vào DEFAULT
```

**Kết quả:**

- `customer_id = 1` → partition `p_north`
- `customer_id = 2` → partition `p_south`
- `customer_id = 3` → partition `p_other` (DEFAULT)

---

### 4.3. Hash Partitioning

```sql
CREATE TABLE orders (
    order_id   NUMBER,
    customer_id NUMBER,
    order_date DATE
)
PARTITION BY HASH (customer_id)
PARTITIONS 4;
```

- Oracle tự chia dữ liệu vào 4 partition dựa trên hàm băm của `customer_id`.
- Mục tiêu: phân bố đều dữ liệu, tránh “hot spot”.

---

### 4.4. Composite Range-Hash Partitioning

```sql
CREATE TABLE sales_detail (
    sale_id      NUMBER,
    sale_date    DATE,
    store_id     NUMBER,
    amount       NUMBER
)
PARTITION BY RANGE (sale_date)
SUBPARTITION BY HASH (store_id)
SUBPARTITIONS 4 (
    PARTITION p_2024_q1 VALUES LESS THAN (DATE '2024-04-01'),
    PARTITION p_2024_q2 VALUES LESS THAN (DATE '2024-07-01')
);
```

- Mỗi partition theo **thời gian** lại được chia nhỏ thành 4 subpartition theo **store_id**.

---

## 5. Một số thao tác quản trị partition

```sql
-- Thêm partition mới
ALTER TABLE sales
ADD PARTITION p_2025_q1 VALUES LESS THAN (DATE '2025-04-01');

-- Xóa dữ liệu cũ bằng cách truncate partition
ALTER TABLE sales
TRUNCATE PARTITION p_2024_q1;

-- Di chuyển partition sang tablespace khác
ALTER TABLE sales
MOVE PARTITION p_2024_q2 TABLESPACE ts_archive;
```

---

## 6. Ý chính cần nhớ

- Partitioning là phần quan trọng của **physical design** cho bảng lớn.
- Chọn kiểu partition dựa trên:
  - Mẫu truy vấn (theo ngày, theo vùng, theo ID…)
  - Yêu cầu lưu trữ & archiving.
- Thiết kế tốt giúp:
  - Tăng hiệu năng (partition pruning, parallelism).
  - Dễ bảo trì (quản lý dữ liệu cũ, di chuyển, backup theo partition).

Nếu bạn gửi thêm nội dung chi tiết hơn (ví dụ: đoạn tài liệu cụ thể), mình có thể tóm tắt lại chính xác theo đúng tài liệu đó bằng tiếng Việt.
