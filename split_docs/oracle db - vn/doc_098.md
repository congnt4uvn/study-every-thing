
Dưới đây là tóm tắt ngắn gọn về “High Availability & Backup” trong Oracle Database, tập trung vào triển khai, failover và read-only replicas, kèm ví dụ lệnh (code) minh họa.

---

## 1. High Availability (HA) – Khả dụng cao

**Ý chính:**
- Mục tiêu: Giảm tối đa thời gian downtime, đảm bảo hệ thống luôn sẵn sàng.
- Thường dùng: Oracle RAC, Data Guard, Clusterware, ASM, v.v.
- Ảnh hưởng triển khai:
  - Cần nhiều node (máy chủ) hơn.
  - Cấu hình shared storage, network riêng cho interconnect.
  - Cần cơ chế giám sát, tự động failover.

**Ví dụ: Kiểm tra trạng thái instance trong RAC**

```sql
-- Trên một node trong RAC
SELECT instance_number, instance_name, status
FROM gv$instance;
```

**Output mẫu:**

| INSTANCE_NUMBER | INSTANCE_NAME | STATUS  |
|-----------------|---------------|---------|
| 1               | ORCL1         | OPEN    |
| 2               | ORCL2         | OPEN    |

## 2. Backup – Sao lưu & Phục hồi

**Ý chính:**
- Dùng RMAN (Recovery Manager) để backup/restore.
- Chiến lược: full backup, incremental backup, archivelog backup.
- Ảnh hưởng triển khai:
  - Cần không gian lưu trữ backup (disk/tape/object storage).
  - Cần lịch backup định kỳ, test restore thường xuyên.

**Ví dụ: Backup toàn bộ database với RMAN**

```bash
rman target /

RUN {
  CONFIGURE CONTROLFILE AUTOBACKUP ON;
  BACKUP DATABASE PLUS ARCHIVELOG;
}
```

**Ví dụ: Restore & Recover (khi có sự cố)**

```bash
rman target /

RUN {
  STARTUP MOUNT;
  RESTORE DATABASE;
  RECOVER DATABASE;
  ALTER DATABASE OPEN RESETLOGS;
}
```

## 3. Failover – Chuyển đổi khi lỗi

**Ý chính:**
- Failover tự động hoặc bán tự động sang standby database khi primary bị lỗi.
- Thường dùng Oracle Data Guard (Physical/Logical Standby).
- Ảnh hưởng triển khai:
  - Cần ít nhất 1 standby database (thường ở site khác).
  - Cấu hình redo transport (SYNC/ASYNC).
  - Cần cơ chế giám sát để kích hoạt failover.

**Ví dụ: Chuyển đổi vai trò (switchover) trong Data Guard**

```sql
-- Trên Primary
ALTER DATABASE SWITCHOVER TO standby_db_name;

-- Sau đó, trên Standby (giờ trở thành Primary)
ALTER DATABASE OPEN;
```

**Ví dụ: Failover (khi Primary hỏng)**

```sql
-- Trên Standby
ALTER DATABASE RECOVER MANAGED STANDBY DATABASE FINISH;
ALTER DATABASE ACTIVATE STANDBY DATABASE;
ALTER DATABASE OPEN;
```

## 4. Read-Only Replicas – Bản sao chỉ đọc

**Ý chính:**
- Dùng standby database (Data Guard) hoặc Active Data Guard để cho phép đọc.
- Mục đích: offload workload đọc (reporting, analytics) khỏi primary.
- Ảnh hưởng triển khai:
  - Cần cấu hình standby ở chế độ read-only.
  - Ứng dụng cần phân luồng: ghi vào primary, đọc nặng vào replica.

**Ví dụ: Mở standby ở chế độ read-only**

```sql
-- Trên Standby
ALTER DATABASE OPEN READ ONLY;
```

**Ví dụ: Active Data Guard (vừa apply redo, vừa cho phép đọc)**

```sql
-- Trên Standby
ALTER DATABASE RECOVER MANAGED STANDBY DATABASE DISCONNECT FROM SESSION;
ALTER DATABASE OPEN READ ONLY;
```

---

## 5. Tóm tắt ngắn

- **High Availability**: Thiết kế nhiều node, cluster, RAC để giảm downtime.
- **Backup**: Dùng RMAN, backup định kỳ + test restore để bảo vệ dữ liệu.
- **Failover**: Dùng Data Guard để chuyển sang standby khi primary lỗi.
- **Read-only replicas**: Dùng standby/Active Data Guard để chia tải đọc, tăng hiệu năng.

Nếu bạn cho biết phiên bản Oracle (ví dụ 19c, 21c) và môi trường (on-premise, OCI, AWS), tôi có thể đưa thêm các script cấu hình cụ thể hơn.
