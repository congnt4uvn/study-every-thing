
Dưới đây là tóm tắt ngắn gọn về chủ đề “Advanced PL/SQL: Overloading procedures/functions” (Nâng cao PL/SQL: Nạp chồng thủ tục/hàm) kèm ví dụ.

---

## 1. Khái niệm nạp chồng (Overloading) trong PL/SQL

- **Nạp chồng** là định nghĩa **nhiều thủ tục (procedure) hoặc hàm (function) cùng tên** trong cùng một package (hoặc scope), nhưng:
  - **Danh sách tham số khác nhau** (số lượng, kiểu dữ liệu, hoặc thứ tự).
- Trình biên dịch PL/SQL sẽ **tự động chọn** phiên bản phù hợp dựa trên **kiểu và số lượng tham số** khi bạn gọi.

---

## 2. Quy tắc chính

1. Các thủ tục/hàm **phải cùng tên**.
2. **Danh sách tham số phải khác nhau**:
   - Khác về số lượng tham số, hoặc
   - Khác về kiểu dữ liệu, hoặc
   - Khác về thứ tự kiểu dữ liệu.
3. **Kiểu trả về của hàm không đủ** để phân biệt (không được nạp chồng chỉ bằng cách đổi kiểu trả về).
4. Thường được khai báo trong **package** để dễ quản lý.

---

## 3. Ví dụ: Nạp chồng PROCEDURE

### 3.1. Tạo package specification

```sql
CREATE OR REPLACE PACKAGE emp_pkg AS
  -- Thủ tục 1: cập nhật lương theo employee_id
  PROCEDURE update_salary(p_emp_id IN NUMBER, p_new_salary IN NUMBER);

  -- Thủ tục 2: cập nhật lương theo email
  PROCEDURE update_salary(p_email IN VARCHAR2, p_new_salary IN NUMBER);
END emp_pkg;
/
```

### 3.2. Tạo package body

```sql
CREATE OR REPLACE PACKAGE BODY emp_pkg AS

  PROCEDURE update_salary(p_emp_id IN NUMBER, p_new_salary IN NUMBER) IS
  BEGIN
    UPDATE employees
       SET salary = p_new_salary
     WHERE employee_id = p_emp_id;
  END update_salary;

  PROCEDURE update_salary(p_email IN VARCHAR2, p_new_salary IN NUMBER) IS
  BEGIN
    UPDATE employees
       SET salary = p_new_salary
     WHERE email = p_email;
  END update_salary;

END emp_pkg;
/
```

### 3.3. Gọi thủ tục – ví dụ input/output

```sql
-- Input: employee_id = 101, new_salary = 5000
BEGIN
  emp_pkg.update_salary(101, 5000);
END;
/

-- Input: email = 'SKING', new_salary = 8000
BEGIN
  emp_pkg.update_salary('SKING', 8000);
END;
/
```

- **Kết quả (output)**: Lương của nhân viên tương ứng được cập nhật trong bảng `employees`.

## 4. Ví dụ: Nạp chồng FUNCTION

### 4.1. Package specification

```sql
CREATE OR REPLACE PACKAGE math_pkg AS
  FUNCTION area(p_radius IN NUMBER) RETURN NUMBER;        -- diện tích hình tròn
  FUNCTION area(p_width  IN NUMBER, p_height IN NUMBER) RETURN NUMBER; -- diện tích hình chữ nhật
END math_pkg;
/
```

### 4.2. Package body

```sql
CREATE OR REPLACE PACKAGE BODY math_pkg AS

  FUNCTION area(p_radius IN NUMBER) RETURN NUMBER IS
  BEGIN
    RETURN 3.14159 * p_radius * p_radius;
  END area;

  FUNCTION area(p_width IN NUMBER, p_height IN NUMBER) RETURN NUMBER IS
  BEGIN
    RETURN p_width * p_height;
  END area;

END math_pkg;
/
```

### 4.3. Gọi hàm – ví dụ input/output

```sql
SET SERVEROUTPUT ON;

DECLARE
  v_circle_area NUMBER;
  v_rect_area   NUMBER;
BEGIN
  -- Input: radius = 5
  v_circle_area := math_pkg.area(5);
  -- Input: width = 4, height = 6
  v_rect_area   := math_pkg.area(4, 6);

  DBMS_OUTPUT.PUT_LINE('Circle area = ' || v_circle_area);
  DBMS_OUTPUT.PUT_LINE('Rectangle area = ' || v_rect_area);
END;
/
```

**Output (ví dụ):**
```text
Circle area = 78.53975
Rectangle area = 24
```

---

## 5. Lợi ích chính

- **Tăng tính linh hoạt**: Cùng một tên hàm/thủ tục cho nhiều kiểu dữ liệu hoặc cách gọi khác nhau.
- **Dễ đọc, dễ nhớ**: Người dùng chỉ cần nhớ **một tên logic** (vd: `update_salary`, `area`).
- **Tái sử dụng**: Gom các phiên bản xử lý tương tự vào cùng một package.

---

Tóm lại, nạp chồng trong PL/SQL cho phép bạn định nghĩa nhiều phiên bản của cùng một thủ tục/hàm với danh sách tham số khác nhau, giúp code rõ ràng, linh hoạt và dễ bảo trì.
