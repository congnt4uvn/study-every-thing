
Dưới đây là tóm tắt ngắn gọn về cách thiết kế audit trail (nhật ký thay đổi) cho các bảng quan trọng trong Oracle Database, kèm ví dụ code.

---

## 1. Ý tưởng chính khi thiết kế audit trail

1. **Xác định bảng quan trọng**  
   - Ví dụ: bảng KHACH_HANG, DON_HANG, TAI_KHOAN_NGAN_HANG, v.v.

2. **Xác định loại thay đổi cần ghi nhận**  
   - INSERT (thêm mới)  
   - UPDATE (cập nhật)  
   - DELETE (xóa)

3. **Xác định thông tin cần lưu trong audit trail**  
   Thường bao gồm:
   - Tên bảng
   - Loại thao tác (I/U/D)
   - Giá trị cũ (old values) và giá trị mới (new values)
   - Người thực hiện (USER, SYS_CONTEXT)
   - Thời gian thực hiện (SYSDATE)
   - Máy trạm / ứng dụng (IP, module, OS user nếu cần)

4. **Cách triển khai phổ biến**
   - Dùng **bảng audit riêng** (ví dụ: KHACH_HANG_AUDIT)
   - Dùng **trigger** trên bảng gốc để ghi log vào bảng audit
   - Hoặc dùng **Oracle Fine-Grained Auditing (FGA)** / Unified Auditing (nếu dùng bản Enterprise, yêu cầu cấu hình DBA)

---

## 2. Thiết kế bảng audit (ví dụ)

Giả sử có bảng quan trọng:

```sql
CREATE TABLE KHACH_HANG (
    ID           NUMBER PRIMARY KEY,
    TEN          VARCHAR2(100),
    DIA_CHI      VARCHAR2(200),
    SO_DIEN_THOAI VARCHAR2(20),
    NGAY_TAO     DATE DEFAULT SYSDATE
);
```

Ta tạo bảng audit:

```sql
CREATE TABLE KHACH_HANG_AUDIT (
    AUDIT_ID        NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    KHACH_HANG_ID   NUMBER,
    OPERATION       VARCHAR2(10),      -- 'INSERT', 'UPDATE', 'DELETE'
    OLD_TEN         VARCHAR2(100),
    NEW_TEN         VARCHAR2(100),
    OLD_DIA_CHI     VARCHAR2(200),
    NEW_DIA_CHI     VARCHAR2(200),
    OLD_SDT         VARCHAR2(20),
    NEW_SDT         VARCHAR2(20),
    CHANGED_BY      VARCHAR2(30),
    CHANGED_DATE    DATE,
    MACHINE         VARCHAR2(64),
    OS_USER         VARCHAR2(64),
    MODULE          VARCHAR2(64)
);
```

## 3. Trigger ghi log audit

### Trigger AFTER INSERT/UPDATE/DELETE

```sql
CREATE OR REPLACE TRIGGER TRG_KHACH_HANG_AUDIT
AFTER INSERT OR UPDATE OR DELETE ON KHACH_HANG
FOR EACH ROW
DECLARE
    v_user    VARCHAR2(30);
    v_machine VARCHAR2(64);
    v_os_user VARCHAR2(64);
    v_module  VARCHAR2(64);
BEGIN
    v_user    := USER;
    v_machine := SYS_CONTEXT('USERENV', 'HOST');
    v_os_user := SYS_CONTEXT('USERENV', 'OS_USER');
    v_module  := SYS_CONTEXT('USERENV', 'MODULE');

    IF INSERTING THEN
        INSERT INTO KHACH_HANG_AUDIT (
            KHACH_HANG_ID, OPERATION,
            OLD_TEN, NEW_TEN,
            OLD_DIA_CHI, NEW_DIA_CHI,
            OLD_SDT, NEW_SDT,
            CHANGED_BY, CHANGED_DATE,
            MACHINE, OS_USER, MODULE
        ) VALUES (
            :NEW.ID, 'INSERT',
            NULL, :NEW.TEN,
            NULL, :NEW.DIA_CHI,
            NULL, :NEW.SO_DIEN_THOAI,
            v_user, SYSDATE,
            v_machine, v_os_user, v_module
        );

    ELSIF UPDATING THEN
        INSERT INTO KHACH_HANG_AUDIT (
            KHACH_HANG_ID, OPERATION,
            OLD_TEN, NEW_TEN,
            OLD_DIA_CHI, NEW_DIA_CHI,
            OLD_SDT, NEW_SDT,
            CHANGED_BY, CHANGED_DATE,
            MACHINE, OS_USER, MODULE
        ) VALUES (
            :OLD.ID, 'UPDATE',
            :OLD.TEN, :NEW.TEN,
            :OLD.DIA_CHI, :NEW.DIA_CHI,
            :OLD.SO_DIEN_THOAI, :NEW.SO_DIEN_THOAI,
            v_user, SYSDATE,
            v_machine, v_os_user, v_module
        );

    ELSIF DELETING THEN
        INSERT INTO KHACH_HANG_AUDIT (
            KHACH_HANG_ID, OPERATION,
            OLD_TEN, NEW_TEN,
            OLD_DIA_CHI, NEW_DIA_CHI,
            OLD_SDT, NEW_SDT,
            CHANGED_BY, CHANGED_DATE,
            MACHINE, OS_USER, MODULE
        ) VALUES (
            :OLD.ID, 'DELETE',
            :OLD.TEN, NULL,
            :OLD.DIA_CHI, NULL,
            :OLD.SO_DIEN_THOAI, NULL,
            v_user, SYSDATE,
            v_machine, v_os_user, v_module
        );
    END IF;
END;
/
```

## 4. Ví dụ thao tác + kết quả audit

### 4.1. INSERT

**Input:**

```sql
INSERT INTO KHACH_HANG (ID, TEN, DIA_CHI, SO_DIEN_THOAI)
VALUES (1, 'Nguyen Van A', 'Ha Noi', '0901234567');
COMMIT;
```

**Audit ghi nhận (truy vấn):**

```sql
SELECT KHACH_HANG_ID, OPERATION,
       OLD_TEN, NEW_TEN,
       CHANGED_BY, CHANGED_DATE
FROM KHACH_HANG_AUDIT
WHERE KHACH_HANG_ID = 1;
```

**Output (ví dụ):**

| KHACH_HANG_ID | OPERATION | OLD_TEN | NEW_TEN       | CHANGED_BY | CHANGED_DATE          |
|---------------|-----------|---------|---------------|------------|-----------------------|
| 1             | INSERT    | (null)  | Nguyen Van A  | APP_USER   | 20-MAY-2026 10:15:00  |

---

### 4.2. UPDATE

**Input:**

```sql
UPDATE KHACH_HANG
SET DIA_CHI = 'Ho Chi Minh'
WHERE ID = 1;
COMMIT;
```

**Audit ghi nhận:**

```sql
SELECT KHACH_HANG_ID, OPERATION,
       OLD_DIA_CHI, NEW_DIA_CHI,
       CHANGED_BY, CHANGED_DATE
FROM KHACH_HANG_AUDIT
WHERE KHACH_HANG_ID = 1
ORDER BY AUDIT_ID;
```

**Output (ví dụ):**

| KHACH_HANG_ID | OPERATION | OLD_DIA_CHI | NEW_DIA_CHI | CHANGED_BY | CHANGED_DATE          |
|---------------|-----------|-------------|-------------|------------|-----------------------|
| 1             | INSERT    | (null)      | Ha Noi      | APP_USER   | 20-MAY-2026 10:15:00  |
| 1             | UPDATE    | Ha Noi      | Ho Chi Minh | APP_USER   | 20-MAY-2026 10:20:00  |

---

### 4.3. DELETE

**Input:**

```sql
DELETE FROM KHACH_HANG
WHERE ID = 1;
COMMIT;
```

**Audit ghi nhận:**

```sql
SELECT KHACH_HANG_ID, OPERATION,
       OLD_TEN, OLD_DIA_CHI,
       CHANGED_BY, CHANGED_DATE
FROM KHACH_HANG_AUDIT
WHERE KHACH_HANG_ID = 1
ORDER BY AUDIT_ID;
```

**Output (ví dụ, thêm dòng DELETE):**

| KHACH_HANG_ID | OPERATION | OLD_TEN      | OLD_DIA_CHI | CHANGED_BY | CHANGED_DATE          |
|---------------|-----------|-------------|-------------|------------|-----------------------|
| 1             | INSERT    | (null)      | Ha Noi      | APP_USER   | 20-MAY-2026 10:15:00  |
| 1             | UPDATE    | Nguyen Van A| Ho Chi Minh | APP_USER   | 20-MAY-2026 10:20:00  |
| 1             | DELETE    | Nguyen Van A| Ho Chi Minh | APP_USER   | 20-MAY-2026 10:25:00  |

## 5. Một số lưu ý thiết kế

- Chỉ audit các bảng **thực sự quan trọng** để tránh phình to dữ liệu.
- Có thể:
  - Dùng **partition** cho bảng audit theo ngày/tháng.
  - Thiết lập **job dọn dẹp** (xóa log cũ hơn X tháng nếu chính sách cho phép).
- Với hệ thống lớn, cân nhắc:
  - Oracle **Unified Auditing**, **FGA** cho audit ở mức chính sách.
  - Lưu log ở **schema riêng** chỉ DBA truy cập.

---

Tóm lại, thiết kế audit trail cho bảng quan trọng thường dựa trên:
- Bảng audit riêng
- Trigger ghi nhận INSERT/UPDATE/DELETE
- Lưu đầy đủ thông tin: giá trị cũ/mới, người dùng, thời gian, nguồn truy cập.
