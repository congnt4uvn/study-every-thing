
Dưới đây là tóm tắt ngắn gọn về chiến lược dùng cột dẫn (leading column) trong composite index của Oracle, kèm ví dụ.

---

## 1. Khái niệm chính

- **Composite index**: Index gồm nhiều cột, ví dụ: `(col1, col2, col3)`.
- **Leading column**: Cột đứng đầu trong composite index (ở ví dụ trên là `col1`).
- Oracle **chỉ có thể tận dụng tốt index** nếu điều kiện WHERE bắt đầu từ **leading column** (và theo thứ tự từ trái sang phải).

---

## 2. Quy tắc sử dụng leading column

Giả sử có index:

```sql
CREATE INDEX idx_emp_dept_job
ON employees (department_id, job_id, salary);
```

- Tối ưu nhất khi truy vấn có điều kiện trên:
  - `department_id`
  - hoặc `department_id` + `job_id`
  - hoặc `department_id` + `job_id` + `salary`

Ví dụ Oracle **có thể dùng index tốt**:

```sql
SELECT *
FROM employees
WHERE department_id = 10;
```

```sql
SELECT *
FROM employees
WHERE department_id = 10
  AND job_id = 'SA_REP';
```

```sql
SELECT *
FROM employees
WHERE department_id = 10
  AND job_id = 'SA_REP'
  AND salary > 5000;
```

Nhưng nếu **không dùng leading column**:

```sql
SELECT *
FROM employees
WHERE job_id = 'SA_REP';
```

thì index `idx_emp_dept_job (department_id, job_id, salary)` **khó được dùng hiệu quả**, vì điều kiện không bắt đầu từ `department_id`.

---

## 3. Ảnh hưởng thứ tự cột trong composite index

Thứ tự cột trong index phải phản ánh **mẫu truy vấn thực tế**:

- Nếu hầu hết truy vấn lọc theo `department_id` trước, sau đó `job_id`, thì:
  ```sql
  CREATE INDEX idx_emp_dept_job
  ON employees (department_id, job_id);
  ```
- Nếu hầu hết truy vấn lọc theo `job_id` trước, sau đó `department_id`, thì nên đảo:
  ```sql
  CREATE INDEX idx_emp_job_dept
  ON employees (job_id, department_id);
  ```

---

## 4. Ví dụ đầy đủ với input/output

### Tạo bảng & dữ liệu mẫu

```sql
CREATE TABLE employees (
    employee_id   NUMBER PRIMARY KEY,
    department_id NUMBER,
    job_id        VARCHAR2(20),
    salary        NUMBER
);

INSERT INTO employees VALUES (1, 10, 'SA_REP', 6000);
INSERT INTO employees VALUES (2, 10, 'SA_MAN', 9000);
INSERT INTO employees VALUES (3, 20, 'IT_PROG', 7000);
COMMIT;
```

Tạo composite index:

```sql
CREATE INDEX idx_emp_dept_job
ON employees (department_id, job_id, salary);
```

### Truy vấn 1: Dùng leading column → index được dùng tốt

```sql
EXPLAIN PLAN FOR
SELECT *
FROM employees
WHERE department_id = 10
  AND job_id = 'SA_REP';

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Output (rút gọn, mang tính minh họa):**

```text
--------------------------------------------------------------------------------
| Id  | Operation                           | Name              |
--------------------------------------------------------------------------------
|   0 | SELECT STATEMENT                    |                   |
|   1 |  TABLE ACCESS BY INDEX ROWID        | EMPLOYEES         |
|   2 |   INDEX RANGE SCAN                  | IDX_EMP_DEPT_JOB  |
--------------------------------------------------------------------------------
```

### Truy vấn 2: Không dùng leading column → index có thể không được dùng

```sql
EXPLAIN PLAN FOR
SELECT *
FROM employees
WHERE job_id = 'SA_REP';

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Output (rút gọn, minh họa):**

```text
--------------------------------------------------------------------------------
| Id  | Operation                           | Name              |
--------------------------------------------------------------------------------
|   0 | SELECT STATEMENT                    |                   |
|   1 |  TABLE ACCESS FULL                  | EMPLOYEES         |
--------------------------------------------------------------------------------
```

Oracle phải **full table scan** vì điều kiện không bắt đầu từ `department_id`.

---

## 5. Kết luận ngắn gọn

- Composite index chỉ phát huy tối đa khi truy vấn **lọc theo leading column trước**.
- Hãy thiết kế thứ tự cột trong composite index dựa trên:
  - Cột nào thường xuất hiện trong `WHERE` nhiều nhất.
  - Thứ tự lọc trong các truy vấn thực tế.
- Nếu cần tối ưu cho nhiều mẫu truy vấn khác nhau, có thể phải tạo **nhiều index** với leading column khác nhau.
