
Dưới đây là tóm tắt ngắn gọn về chủ đề **Advanced PL/SQL: Global variables** (biến toàn cục) trong Oracle, kèm ví dụ mã và input/output minh họa.

---

## 1. Khái niệm biến toàn cục (Global variables) trong PL/SQL

Trong PL/SQL, “biến toàn cục” thường được hiểu là:

- Biến có thể được **nhiều khối PL/SQL** (procedure, function, trigger, anonymous block) **cùng truy cập**.
- Thường được cài đặt thông qua:
  1. **Package-level variables** (biến khai báo trong phần specification hoặc body của package).
  2. **Context variables** (sử dụng `DBMS_SESSION.set_context`).
  3. **Bảng cấu hình / bảng tham số** (giả lập biến toàn cục bằng dữ liệu trong bảng).

---

## 2. Biến toàn cục trong Package (cách phổ biến nhất)

### 2.1. Đặc điểm

- Khai báo trong **package specification** hoặc **package body** (ngoài bất kỳ procedure/function nào).
- **Phạm vi**: toàn bộ package.
- **Thời gian sống**: trong suốt phiên làm việc (session) của user.
- Mỗi session có **bản sao riêng** của biến (không chia sẻ giữa các session).

### 2.2. Ví dụ: Biến toàn cục trong package

```sql
-- Tạo package specification
CREATE OR REPLACE PACKAGE app_state IS
  -- Biến toàn cục (public)
  g_current_user_id   NUMBER;
  g_debug_mode        BOOLEAN := FALSE;

  PROCEDURE set_user(p_user_id NUMBER);
  FUNCTION get_user RETURN NUMBER;

  PROCEDURE enable_debug;
  PROCEDURE disable_debug;
  FUNCTION is_debug RETURN BOOLEAN;
END app_state;
/

-- Tạo package body
CREATE OR REPLACE PACKAGE BODY app_state IS

  PROCEDURE set_user(p_user_id NUMBER) IS
  BEGIN
    g_current_user_id := p_user_id;
  END;

  FUNCTION get_user RETURN NUMBER IS
  BEGIN
    RETURN g_current_user_id;
  END;

  PROCEDURE enable_debug IS
  BEGIN
    g_debug_mode := TRUE;
  END;

  PROCEDURE disable_debug IS
  BEGIN
    g_debug_mode := FALSE;
  END;

  FUNCTION is_debug RETURN BOOLEAN IS
  BEGIN
    RETURN g_debug_mode;
  END;

END app_state;
/
```

### 2.3. Cách sử dụng & kết quả (input/output)

**Input**: Gọi các procedure/function để set/get biến toàn cục.

```sql
-- Anonymous block 1: thiết lập trạng thái
BEGIN
  app_state.set_user(101);
  app_state.enable_debug;
END;
/

-- Anonymous block 2: đọc lại trạng thái
DECLARE
  v_user_id  NUMBER;
  v_debug    BOOLEAN;
BEGIN
  v_user_id := app_state.get_user;
  v_debug   := app_state.is_debug;

  DBMS_OUTPUT.put_line('User ID: ' || v_user_id);
  IF v_debug THEN
    DBMS_OUTPUT.put_line('Debug mode: ON');
  ELSE
    DBMS_OUTPUT.put_line('Debug mode: OFF');
  END IF;
END;
/
```

**Output (DBMS_OUTPUT):**

```text
User ID: 101
Debug mode: ON
```

Lưu ý: Kết quả này chỉ đúng trong **cùng một session**. Session khác sẽ có giá trị riêng.

## 3. Biến toàn cục bằng Application Context

Dùng `DBMS_SESSION.set_context` để lưu giá trị “toàn cục” cho session, truy cập được từ SQL và PL/SQL.

### 3.1. Tạo context

```sql
CREATE OR REPLACE CONTEXT app_ctx USING app_ctx_pkg;
/
```

### 3.2. Package để set context

```sql
CREATE OR REPLACE PACKAGE app_ctx_pkg IS
  PROCEDURE set_user(p_user_id NUMBER);
END app_ctx_pkg;
/

CREATE OR REPLACE PACKAGE BODY app_ctx_pkg IS
  PROCEDURE set_user(p_user_id NUMBER) IS
  BEGIN
    DBMS_SESSION.set_context('APP_CTX', 'USER_ID', p_user_id);
  END;
END app_ctx_pkg;
/
```

### 3.3. Sử dụng & output

```sql
BEGIN
  app_ctx_pkg.set_user(200);
END;
/

-- Đọc trong SQL
SELECT SYS_CONTEXT('APP_CTX', 'USER_ID') AS user_id
FROM dual;
```

**Output:**

```text
USER_ID
-------
200
```

## 4. Giả lập biến toàn cục bằng bảng cấu hình

Dùng bảng để lưu giá trị “toàn cục” dùng chung cho nhiều session.

```sql
CREATE TABLE app_config (
  config_name  VARCHAR2(30) PRIMARY KEY,
  config_value VARCHAR2(4000)
);

INSERT INTO app_config (config_name, config_value)
VALUES ('MAX_LOGIN_ATTEMPTS', '5');

COMMIT;
```

**Đọc giá trị:**

```sql
SELECT config_value
FROM app_config
WHERE config_name = 'MAX_LOGIN_ATTEMPTS';
```

**Output:**

```text
CONFIG_VALUE
------------
5
```

---

## 5. Lưu ý khi dùng biến toàn cục

- **Không thread-safe** giữa các session (mỗi session một bản sao).
- Dễ gây **phụ thuộc trạng thái** (stateful), khó debug nếu lạm dụng.
- Nên dùng cho:
  - Thông tin user hiện tại, ngữ cảnh phiên làm việc.
  - Cờ debug, cấu hình tạm thời trong session.
- Không nên dùng để chia sẻ dữ liệu giữa session → dùng bảng hoặc queue.

---

Tóm lại, trong PL/SQL, “global variables” thường được triển khai bằng **package-level variables** và/hoặc **application context**, giúp chia sẻ trạng thái trong phạm vi **một session** giữa nhiều khối PL/SQL.
