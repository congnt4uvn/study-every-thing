
Dưới đây là tóm tắt ngắn gọn về **Advanced PL/SQL: FORALL, BULK COLLECT** bằng tiếng Việt, kèm ví dụ mã và input/output.

---

## 1. Mục tiêu của FORALL và BULK COLLECT

- **BULK COLLECT**: Lấy nhiều dòng dữ liệu từ SQL vào PL/SQL collection trong **một lần** → giảm số lần chuyển đổi ngữ cảnh SQL ↔ PL/SQL.
- **FORALL**: Thực hiện nhiều câu lệnh DML (INSERT/UPDATE/DELETE) cho nhiều phần tử collection trong **một lệnh** → nhanh hơn vòng lặp FOR thông thường.

=> Dùng chung để **tối ưu hiệu năng** khi xử lý nhiều dòng dữ liệu.

---

## 2. BULK COLLECT – Lấy dữ liệu hàng loạt

### Cú pháp cơ bản

```plsql
DECLARE
    TYPE t_emp_tab IS TABLE OF employees%ROWTYPE;
    l_emps t_emp_tab;
BEGIN
    SELECT *
    BULK COLLECT INTO l_emps
    FROM employees
    WHERE department_id = 10;
END;
/
```

- `l_emps` là collection chứa nhiều dòng từ bảng `employees`.
- Thay vì `SELECT ... INTO` từng dòng, ta lấy **tất cả** (hoặc nhiều dòng) một lần.

### BULK COLLECT với LIMIT (phân lô)

Dùng khi dữ liệu rất lớn, tránh dùng quá nhiều PGA:

```plsql
DECLARE
    CURSOR c_emp IS
        SELECT employee_id, first_name, salary
        FROM employees;

    TYPE t_emp_tab IS TABLE OF c_emp%ROWTYPE;
    l_emps t_emp_tab;
BEGIN
    OPEN c_emp;
    LOOP
        FETCH c_emp BULK COLLECT INTO l_emps LIMIT 1000;  -- mỗi lần 1000 dòng
        EXIT WHEN l_emps.COUNT = 0;

        -- xử lý dữ liệu trong l_emps
        FOR i IN 1 .. l_emps.COUNT LOOP
            DBMS_OUTPUT.PUT_LINE(
                'ID=' || l_emps(i).employee_id ||
                ', Name=' || l_emps(i).first_name ||
                ', Salary=' || l_emps(i).salary
            );
        END LOOP;
    END LOOP;
    CLOSE c_emp;
END;
/
```

**Output (ví dụ)**:
```
ID=100, Name=Steven, Salary=24000
ID=101, Name=Neena, Salary=17000
...
```

## 3. FORALL – DML hàng loạt

### Cú pháp cơ bản

```plsql
DECLARE
    TYPE t_emp_id_tab IS TABLE OF employees.employee_id%TYPE;
    l_emp_ids t_emp_id_tab := t_emp_id_tab(100, 101, 102);
BEGIN
    FORALL i IN l_emp_ids.FIRST .. l_emp_ids.LAST
        UPDATE employees
        SET salary = salary * 1.1
        WHERE employee_id = l_emp_ids(i);
END;
/
```

- `FORALL` không phải là vòng lặp logic như `FOR`, mà là **chỉ thị** cho Oracle thực hiện DML hàng loạt.
- Nhanh hơn nhiều so với:

```plsql
FOR i IN l_emp_ids.FIRST .. l_emp_ids.LAST LOOP
    UPDATE employees
    SET salary = salary * 1.1
    WHERE employee_id = l_emp_ids(i);
END LOOP;
```

## 4. Kết hợp BULK COLLECT + FORALL

### Ví dụ: tăng lương cho tất cả nhân viên phòng 10

```plsql
DECLARE
    TYPE t_emp_id_tab IS TABLE OF employees.employee_id%TYPE;
    l_emp_ids t_emp_id_tab;
BEGIN
    -- 1. Lấy danh sách employee_id bằng BULK COLLECT
    SELECT employee_id
    BULK COLLECT INTO l_emp_ids
    FROM employees
    WHERE department_id = 10;

    -- 2. Cập nhật lương hàng loạt bằng FORALL
    FORALL i IN l_emp_ids.FIRST .. l_emp_ids.LAST
        UPDATE employees
        SET salary = salary * 1.05
        WHERE employee_id = l_emp_ids(i);
END;
/
```

**Input logic**:
- Bảng `employees` có nhiều nhân viên, một số thuộc `department_id = 10`.

**Output logic**:
- Tất cả nhân viên phòng 10 được tăng lương 5%.
- Không in ra màn hình, nhưng dữ liệu trong bảng đã thay đổi.

## 5. FORALL với SAVE EXCEPTIONS

Cho phép tiếp tục xử lý dù một số dòng lỗi, sau đó xem chi tiết lỗi.

```plsql
DECLARE
    TYPE t_emp_id_tab IS TABLE OF employees.employee_id%TYPE;
    l_emp_ids t_emp_id_tab := t_emp_id_tab(100, 9999, 101); -- 9999 có thể không tồn tại
BEGIN
    FORALL i IN l_emp_ids.FIRST .. l_emp_ids.LAST SAVE EXCEPTIONS
        UPDATE employees
        SET salary = salary * 1.1
        WHERE employee_id = l_emp_ids(i);

EXCEPTION
    WHEN OTHERS THEN
        FOR j IN 1 .. SQL%BULK_EXCEPTIONS.COUNT LOOP
            DBMS_OUTPUT.PUT_LINE(
                'Index=' || SQL%BULK_EXCEPTIONS(j).ERROR_INDEX ||
                ', Code=' || SQL%BULK_EXCEPTIONS(j).ERROR_CODE
            );
        END LOOP;
END;
/
```

**Output (ví dụ)**:
```
Index=2, Code=-01403
```
- Dòng thứ 2 (employee_id = 9999) bị lỗi (no data found), các dòng khác vẫn cập nhật.

---

## 6. Tóm tắt ý chính

- **BULK COLLECT**: Lấy nhiều dòng vào collection trong một lần → giảm round-trip, tăng tốc.
- **FORALL**: Thực hiện DML cho nhiều phần tử collection trong một lệnh → nhanh hơn vòng lặp FOR.
- Thường dùng **kết hợp**: `SELECT ... BULK COLLECT INTO ...` → xử lý → `FORALL ... INSERT/UPDATE/DELETE`.
- Có thể dùng `LIMIT` với BULK COLLECT để tránh dùng quá nhiều bộ nhớ.
- Có thể dùng `SAVE EXCEPTIONS` với FORALL để xử lý lỗi chi tiết từng dòng.
