---

## 1. Khái niệm Encapsulation trong PL/SQL

- **Encapsulation (đóng gói)**: gom nhóm dữ liệu (biến, kiểu dữ liệu) và logic xử lý (hàm, thủ tục) vào cùng một đơn vị (package, object type, v.v.).
- Mục tiêu:
  - Ẩn chi tiết cài đặt (implementation) khỏi người dùng.
  - Cung cấp giao diện (interface) rõ ràng qua các thủ tục/hàm public.
  - Dễ bảo trì, dễ thay đổi logic bên trong mà không ảnh hưởng code bên ngoài.
  - Tăng tính bảo mật và kiểm soát truy cập.

Trong PL/SQL, **package** là công cụ chính để thực hiện encapsulation.

## 2. Cấu trúc Package: Specification & Body

### 2.1. Package Specification (Public Interface)

- Khai báo những gì **public**: hằng số, biến, kiểu dữ liệu, thủ tục, hàm.
- Người dùng chỉ cần biết spec để gọi; không cần biết body.

```sql
CREATE OR REPLACE PACKAGE emp_api IS
  -- Hằng số public
  c_min_salary CONSTANT NUMBER := 3000;

  -- Kiểu dữ liệu public
  TYPE t_emp_rec IS RECORD (
    emp_id   employees.employee_id%TYPE,
    salary   employees.salary%TYPE
  );

  -- Thủ tục/hàm public
  PROCEDURE set_salary(p_emp_id IN employees.employee_id%TYPE,
                       p_salary IN employees.salary%TYPE);

  FUNCTION get_salary(p_emp_id IN employees.employee_id%TYPE)
    RETURN employees.salary%TYPE;
END emp_api;
/
```

### 2.2. Package Body (Implementation)

- Chứa **cài đặt** của các thủ tục/hàm đã khai báo trong spec.
- Có thể khai báo thêm biến, thủ tục/hàm **private** (chỉ dùng trong package body).

```sql
CREATE OR REPLACE PACKAGE BODY emp_api IS

  -- Biến private (không thấy từ bên ngoài)
  g_last_updated_by VARCHAR2(30) := 'SYSTEM';

  -- Hàm private
  FUNCTION validate_salary(p_salary NUMBER) RETURN BOOLEAN IS
  BEGIN
    RETURN p_salary >= c_min_salary;
  END;

  PROCEDURE set_salary(p_emp_id IN employees.employee_id%TYPE,
                       p_salary IN employees.salary%TYPE) IS
  BEGIN
    IF NOT validate_salary(p_salary) THEN
      RAISE_APPLICATION_ERROR(-20001, 'Salary below minimum');
    END IF;

    UPDATE employees
       SET salary = p_salary
     WHERE employee_id = p_emp_id;

    g_last_updated_by := USER;
  END;

  FUNCTION get_salary(p_emp_id IN employees.employee_id%TYPE)
    RETURN employees.salary%TYPE IS
    v_salary employees.salary%TYPE;
  BEGIN
    SELECT salary INTO v_salary
      FROM employees
     WHERE employee_id = p_emp_id;

    RETURN v_salary;
  END;

END emp_api;
/
```

## 3. Minh họa Input/Output khi sử dụng Package

### 3.1. Gọi thủ tục `set_salary` (Input)

```sql
-- Input: emp_id = 100, salary = 5000
BEGIN
  emp_api.set_salary(p_emp_id => 100, p_salary => 5000);
END;
/
```

**Kết quả (Output):**

- Bảng `employees` được cập nhật: `employee_id = 100` có `salary = 5000`.
- Nếu `p_salary < c_min_salary (3000)` → lỗi:

```text
ORA-20001: Salary below minimum
```

---

### 3.2. Gọi hàm `get_salary` (Input/Output)

```sql
-- Lấy lương của nhân viên 100
SET SERVEROUTPUT ON;
DECLARE
  v_sal employees.salary%TYPE;
BEGIN
  v_sal := emp_api.get_salary(100);
  DBMS_OUTPUT.PUT_LINE('Salary of emp 100 = ' || v_sal);
END;
/
```

**Output (ví dụ):**

```text
Salary of emp 100 = 5000
```

## 4. Lợi ích chính của Encapsulation trong PL/SQL

1. **Ẩn chi tiết cài đặt**  
   - Người dùng chỉ thấy spec, không thấy logic bên trong body.
   - Có thể thay đổi thuật toán, cấu trúc dữ liệu bên trong mà không ảnh hưởng code gọi.

2. **Kiểm soát truy cập**  
   - Phân biệt **public** (trong spec) và **private** (chỉ trong body).
   - Giảm rủi ro bị gọi nhầm các hàm/thủ tục nội bộ.

3. **Tổ chức mã tốt hơn**  
   - Gom nhóm các chức năng liên quan (ví dụ: `emp_api` cho nghiệp vụ nhân viên).
   - Dễ tìm, dễ đọc, dễ bảo trì.

4. **Hiệu năng**  
   - Package được nạp vào bộ nhớ (shared pool) một lần, các lần gọi sau nhanh hơn.
   - Biến package có thể giữ trạng thái trong phiên (session state).

## 5. Ví dụ Encapsulation với Biến Package (Giữ trạng thái)

```sql
CREATE OR REPLACE PACKAGE session_ctx IS
  PROCEDURE set_user_id(p_user_id IN NUMBER);
  FUNCTION get_user_id RETURN NUMBER;
END session_ctx;
/

CREATE OR REPLACE PACKAGE BODY session_ctx IS
  -- Biến private, giữ trạng thái trong session
  g_user_id NUMBER;

  PROCEDURE set_user_id(p_user_id IN NUMBER) IS
  BEGIN
    g_user_id := p_user_id;
  END;

  FUNCTION get_user_id RETURN NUMBER IS
  BEGIN
    RETURN g_user_id;
  END;
END session_ctx;
/
```

**Sử dụng (Input/Output):**

```sql
BEGIN
  session_ctx.set_user_id(123);  -- Input
END;
/

SET SERVEROUTPUT ON;
DECLARE
  v_id NUMBER;
BEGIN
  v_id := session_ctx.get_user_id;  -- Output
  DBMS_OUTPUT.PUT_LINE('Current user id = ' || v_id);
END;
/
```

**Output:**

```text
Current user id = 123
```
