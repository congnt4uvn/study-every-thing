
Dưới đây là tóm tắt ngắn gọn về “Index selectivity và cardinality” trong Oracle, kèm ví dụ mã.

---

## 1. Cardinality là gì?

- Cardinality = số lượng giá trị khác nhau (distinct) trong một cột.
- Cardinality cao: nhiều giá trị khác nhau → cột rất “đa dạng”.
- Cardinality thấp: ít giá trị khác nhau → cột lặp lại nhiều.

Ví dụ:

```sql
-- Bảng khách hàng
SELECT COUNT(*) AS total_rows,
       COUNT(DISTINCT city) AS city_cardinality
FROM customers;
```

**Output (ví dụ):**

| total_rows | city_cardinality |
|-----------:|-----------------:|
|     1,000,000 |             500 |

Cardinality của cột `city` là 500 (có 500 thành phố khác nhau).

---

## 2. Selectivity là gì?

- Selectivity = tỷ lệ số dòng được chọn / tổng số dòng.
- Selectivity càng thấp (tức là truy vấn trả về ít dòng) → index càng hữu ích.
- Công thức đơn giản (xấp xỉ):
  
  ```text
  selectivity ≈ 1 / cardinality
  ```
  
  nếu phân bố dữ liệu tương đối đều.

Ví dụ:

```sql
-- Giả sử có index trên cột city
SELECT * 
FROM customers
WHERE city = 'HANOI';
```

Nếu `city = 'HANOI'` chỉ chiếm 1,000 dòng trên 1,000,000 dòng:

- selectivity = 1,000 / 1,000,000 = 0.001 (0.1%)
- Đây là selectivity tốt → Oracle có xu hướng dùng index.

---

## 3. Khi nào index hiệu quả?

- Cột có **cardinality cao** (nhiều giá trị khác nhau).
- Điều kiện WHERE chọn **ít dòng** (selectivity thấp).
- Các cột thường xuất hiện trong:
  - WHERE
  - JOIN
  - ORDER BY / GROUP BY

Ví dụ tạo index:

```sql
CREATE INDEX idx_customers_city
ON customers (city);
```

---

## 4. Khi nào index kém hiệu quả?

- Cột có **cardinality thấp** (ví dụ cột giới tính, trạng thái Y/N).
- Điều kiện WHERE trả về **rất nhiều dòng** (selectivity cao).
- Oracle có thể bỏ qua index và dùng full table scan.

Ví dụ:

```sql
-- Cột status chỉ có 'A' hoặc 'I'
CREATE INDEX idx_customers_status
ON customers (status);

SELECT *
FROM customers
WHERE status = 'A';
```

Nếu 90% khách hàng có `status = 'A'`, selectivity ≈ 0.9 → index ít hữu ích, full scan có thể nhanh hơn.

---

## 5. Xem thống kê cardinality / selectivity trong Oracle

Oracle lưu thống kê trong `USER_TAB_COL_STATISTICS`:

```sql
SELECT column_name,
       num_distinct,      -- cardinality
       density            -- liên quan đến selectivity
FROM user_tab_col_statistics
WHERE table_name = 'CUSTOMERS';
```

**Output (ví dụ):**

| column_name | num_distinct | density  |
|-------------|--------------|----------|
| CITY        | 500          | 0.002    |
| STATUS      | 2            | 0.5      |

- `num_distinct` = cardinality.
- `density` càng nhỏ → selectivity càng tốt.

---

## 6. Ví dụ đầy đủ: so sánh dùng index và không dùng index

```sql
-- Tạo bảng demo
CREATE TABLE sales (
    id          NUMBER PRIMARY KEY,
    customer_id NUMBER,
    region      VARCHAR2(20),
    amount      NUMBER
);

-- Tạo index trên region
CREATE INDEX idx_sales_region ON sales(region);

-- Truy vấn 1: chọn 1 vùng nhỏ (selectivity thấp)
EXPLAIN PLAN FOR
SELECT * FROM sales WHERE region = 'NORTH';

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Output plan (ví dụ):**

- INDEX RANGE SCAN trên `IDX_SALES_REGION`
- TABLE ACCESS BY INDEX ROWID

```sql
-- Truy vấn 2: chọn gần như tất cả (selectivity cao)
EXPLAIN PLAN FOR
SELECT * FROM sales WHERE amount > 0;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Output plan (ví dụ):**

- FULL TABLE SCAN (Oracle bỏ qua index vì selectivity kém).

---

## 7. Tóm tắt ý chính

- **Cardinality**: số lượng giá trị distinct trong cột.
- **Selectivity**: tỷ lệ số dòng được chọn; càng thấp → index càng hữu ích.
- Index phù hợp cho:
  - Cột có cardinality cao.
  - Truy vấn lọc ít dòng (selectivity thấp).
- Cột có cardinality thấp thường không nên index, trừ khi dùng trong index composite hoặc các trường hợp đặc biệt.
