
Dưới đây là tóm tắt ngắn gọn về **Autonomous Transactions** trong PL/SQL, kèm ví dụ mã và input/output.

---

## 1. Khái niệm chính

**Autonomous transaction** là một giao dịch độc lập, tách biệt với giao dịch chính (parent transaction):

- Có **COMMIT** và **ROLLBACK** riêng.
- Không bị ảnh hưởng bởi COMMIT/ROLLBACK của giao dịch cha.
- Thường dùng cho:
  - Ghi log (audit, error log).
  - Gửi thông báo, ghi lịch sử.
  - Các thao tác cần được lưu lại ngay cả khi giao dịch chính bị rollback.

Khai báo bằng pragma:

```sql
PRAGMA AUTONOMOUS_TRANSACTION;
```

---

## 2. Cách khai báo

Có thể dùng trong:

- Procedure
- Function
- Trigger
- Anonymous block

Ví dụ:

```sql
CREATE OR REPLACE PROCEDURE log_error (
    p_module   VARCHAR2,
    p_message  VARCHAR2
) IS
    PRAGMA AUTONOMOUS_TRANSACTION;
BEGIN
    INSERT INTO error_log (log_time, module, message)
    VALUES (SYSTIMESTAMP, p_module, p_message);

    COMMIT;  -- rất quan trọng trong autonomous transaction
END;
/
```

---

## 3. Ví dụ đầy đủ (có input/output)

### 3.1. Chuẩn bị bảng log

```sql
CREATE TABLE error_log (
    id        NUMBER GENERATED BY DEFAULT AS IDENTITY,
    log_time  TIMESTAMP,
    module    VARCHAR2(100),
    message   VARCHAR2(4000)
);
```

### 3.2. Procedure autonomous để ghi log

```sql
CREATE OR REPLACE PROCEDURE log_error (
    p_module   VARCHAR2,
    p_message  VARCHAR2
) IS
    PRAGMA AUTONOMOUS_TRANSACTION;
BEGIN
    INSERT INTO error_log (log_time, module, message)
    VALUES (SYSTIMESTAMP, p_module, p_message);

    COMMIT;
END;
/
```

### 3.3. Procedure chính (giao dịch cha)

```sql
CREATE OR REPLACE PROCEDURE do_something IS
BEGIN
    -- Giao dịch chính: ví dụ chèn vào bảng nào đó
    INSERT INTO some_table (id, data)
    VALUES (1, 'test');

    -- Ghi log bằng autonomous transaction
    log_error('do_something', 'Insert into some_table started');

    -- Giả lập lỗi
    RAISE_APPLICATION_ERROR(-20001, 'Lỗi giả lập');
EXCEPTION
    WHEN OTHERS THEN
        -- Rollback giao dịch chính
        ROLLBACK;
        -- Ghi log lỗi (autonomous transaction vẫn commit)
        log_error('do_something', SQLERRM);
END;
/
```

### 3.4. Chạy và xem kết quả

**Input (gọi procedure):**

```sql
BEGIN
    do_something;
END;
/
```

Kết quả:

- Giao dịch chính bị **ROLLBACK** → dữ liệu trong `some_table` **không** được lưu.
- Các bản ghi log trong `error_log` **vẫn được COMMIT** vì chúng nằm trong autonomous transaction.

**Output (kiểm tra log):**

```sql
SELECT id, module, message
FROM error_log
ORDER BY id;
```

Ví dụ kết quả:

| ID | MODULE        | MESSAGE                               |
|----|---------------|----------------------------------------|
| 1  | do_something  | Insert into some_table started         |
| 2  | do_something  | ORA-20001: Lỗi giả lập                |

---

## 4. Lưu ý quan trọng

- Luôn **COMMIT hoặc ROLLBACK** trong autonomous transaction, nếu không sẽ giữ transaction mở.
- Không nên lạm dụng, vì:
  - Khó debug luồng giao dịch.
  - Có thể gây deadlock nếu truy cập cùng dữ liệu với giao dịch cha.
- Thường chỉ dùng cho:
  - Ghi log/audit.
  - Ghi lịch sử, thống kê, tracking.

---

Nếu bạn muốn, tôi có thể viết thêm ví dụ với **trigger** dùng autonomous transaction để ghi log audit.
