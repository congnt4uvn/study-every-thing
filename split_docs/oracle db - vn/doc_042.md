
Dưới đây là tóm tắt ngắn gọn về **Explain Plan & Execution: Full table scan vs Index range scan** trong Oracle, kèm ví dụ.

---

## 1. Explain Plan là gì?

- `EXPLAIN PLAN` cho biết **Oracle dự định** thực hiện câu SQL như thế nào (kế hoạch thực thi).
- Dùng để xem:
  - Có dùng **Full Table Scan** hay không.
  - Có dùng **Index Range Scan** hay không.
  - Thứ tự join, loại join, chi phí ước lượng,…

Xem plan bằng:

```sql
EXPLAIN PLAN FOR
SELECT * FROM employees WHERE employee_id = 100;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

---

## 2. Full Table Scan (FTS)

**Khái niệm:**

- Oracle đọc **toàn bộ** các block dữ liệu của bảng.
- Áp dụng điều kiện `WHERE` sau khi đã đọc dữ liệu.
- Thường xảy ra khi:
  - Không có index phù hợp.
  - Optimizer thấy FTS rẻ hơn dùng index (ví dụ: trả về phần lớn số dòng).
  - Dùng hàm trên cột không thể dùng index (không có function-based index).

**Đặc điểm:**

- Tốt khi:
  - Bảng nhỏ.
  - Cần đọc phần lớn dữ liệu (ví dụ > 10–20% số dòng).
- Không tốt khi:
  - Bảng lớn, chỉ cần vài dòng → tốn I/O.

**Ví dụ:**

```sql
-- Bảng lớn, không có index trên department_id
EXPLAIN PLAN FOR
SELECT * FROM employees WHERE department_id = 10;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

Kết quả (rút gọn):

```text
--------------------------------------------------------------------------------
| Id  | Operation          | Name      |
--------------------------------------------------------------------------------
|   0 | SELECT STATEMENT   |           |
|*  1 |  TABLE ACCESS FULL | EMPLOYEES |
--------------------------------------------------------------------------------
```

- `TABLE ACCESS FULL` = Full Table Scan.

---

## 3. Index Range Scan

**Khái niệm:**

- Oracle dùng **index** để tìm các giá trị trong một khoảng (range) hoặc một giá trị cụ thể.
- Chỉ đọc các entry index phù hợp, sau đó truy cập block dữ liệu tương ứng (table access by ROWID).
- Thường xảy ra khi:
  - Có index trên cột trong điều kiện `WHERE`.
  - Điều kiện là dạng:
    - `=`, `BETWEEN`, `>`, `<`, `>=`, `<=`
    - `LIKE 'ABC%'` (bắt đầu bằng hằng số, không có wildcard ở đầu).

**Đặc điểm:**

- Tốt khi:
  - Bảng lớn, chỉ cần ít dòng.
- Không tốt khi:
  - Điều kiện trả về quá nhiều dòng (gần như toàn bộ bảng) → FTS có thể rẻ hơn.

**Ví dụ:**

Tạo index:

```sql
CREATE INDEX idx_emp_dept ON employees(department_id);
```

Câu truy vấn:

```sql
EXPLAIN PLAN FOR
SELECT * FROM employees WHERE department_id = 10;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

Kết quả (rút gọn):

```text
-----------------------------------------------------------------------------------------
| Id  | Operation                           | Name          |
-----------------------------------------------------------------------------------------
|   0 | SELECT STATEMENT                    |               |
|   1 |  TABLE ACCESS BY INDEX ROWID        | EMPLOYEES     |
|*  2 |   INDEX RANGE SCAN                  | IDX_EMP_DEPT  |
-----------------------------------------------------------------------------------------
```

- `INDEX RANGE SCAN` = dùng index để tìm các dòng phù hợp.
- `TABLE ACCESS BY INDEX ROWID` = đọc dữ liệu thực tế từ bảng dựa trên ROWID lấy từ index.

---

## 4. So sánh nhanh

| Tiêu chí                 | Full Table Scan                         | Index Range Scan                          |
|--------------------------|------------------------------------------|-------------------------------------------|
| Cách đọc dữ liệu         | Đọc toàn bộ bảng                        | Đọc một phần index + các block dữ liệu   |
| Phù hợp khi              | Bảng nhỏ, trả về nhiều dòng             | Bảng lớn, trả về ít dòng                  |
| I/O                      | Nhiều (nếu bảng lớn)                    | Ít hơn (nếu chọn lọc tốt)                 |
| Phụ thuộc index          | Không                                   | Có (cần index phù hợp)                    |

## 5. Ví dụ đầy đủ: Input/Output

### Chuẩn bị dữ liệu (Input)

```sql
-- Tạo bảng
CREATE TABLE employees (
    employee_id   NUMBER PRIMARY KEY,
    first_name    VARCHAR2(50),
    last_name     VARCHAR2(50),
    department_id NUMBER,
    salary        NUMBER
);

-- Tạo index trên department_id
CREATE INDEX idx_emp_dept ON employees(department_id);

-- Giả sử đã INSERT nhiều dòng vào employees
```

### Truy vấn 1: Dùng index range scan (Input)

```sql
EXPLAIN PLAN FOR
SELECT * FROM employees WHERE department_id = 10;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Output (rút gọn):**

```text
| Id  | Operation                    | Name          |
------------------------------------------------------
|   0 | SELECT STATEMENT             |               |
|   1 |  TABLE ACCESS BY INDEX ROWID | EMPLOYEES     |
|*  2 |   INDEX RANGE SCAN           | IDX_EMP_DEPT  |
```

### Truy vấn 2: Full table scan (Input)

```sql
-- Ví dụ: điều kiện không dùng được index
EXPLAIN PLAN FOR
SELECT * FROM employees WHERE salary * 2 > 20000;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Output (rút gọn):**

```text
| Id  | Operation          | Name      |
----------------------------------------
|   0 | SELECT STATEMENT   |           |
|*  1 |  TABLE ACCESS FULL | EMPLOYEES |
```

---

## 6. Gợi ý tối ưu

- Tạo index trên các cột thường dùng trong `WHERE`, `JOIN`, `ORDER BY`.
- Tránh dùng hàm trực tiếp trên cột đã index trong `WHERE` (hoặc dùng function-based index).
- Kiểm tra kế hoạch thực thi bằng `EXPLAIN PLAN` hoặc:

```sql
SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY_CURSOR(NULL, NULL, 'ALLSTATS LAST'));
```

để xem **plan thực tế** khi câu SQL đã chạy.
