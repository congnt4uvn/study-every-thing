
Dưới đây là tóm tắt ngắn gọn về các toán tử tập hợp trong Oracle: `UNION`, `UNION ALL`, `INTERSECT`, `MINUS`, kèm ví dụ.

---

## 1. UNION

- Kết hợp **kết quả của 2 truy vấn**.
- **Loại bỏ các dòng trùng lặp**.
- Số cột và kiểu dữ liệu tương ứng giữa các SELECT phải giống nhau.

```sql
-- Ví dụ: Danh sách tất cả khách hàng từ 2 bảng, không trùng
SELECT customer_id, name
FROM customers_2023
UNION
SELECT customer_id, name
FROM customers_2024;
```

**Input (giả sử):**

`customers_2023`:

| customer_id | name   |
|------------|--------|
| 1          | An     |
| 2          | Bình   |

`customers_2024`:

| customer_id | name   |
|------------|--------|
| 2          | Bình   |
| 3          | Chi    |

**Output:**

| customer_id | name   |
|------------|--------|
| 1          | An     |
| 2          | Bình   |
| 3          | Chi    |

---

## 2. UNION ALL

- Kết hợp kết quả của 2 truy vấn.
- **Giữ nguyên các dòng trùng lặp** (không loại bỏ).
- Nhanh hơn `UNION` vì không cần bước loại trùng.

```sql
SELECT customer_id, name
FROM customers_2023
UNION ALL
SELECT customer_id, name
FROM customers_2024;
```

**Output (với dữ liệu trên):**

| customer_id | name   |
|------------|--------|
| 1          | An     |
| 2          | Bình   |
| 2          | Bình   |  -- dòng trùng vẫn giữ
| 3          | Chi    |

---

## 3. INTERSECT

- Lấy **phần giao** của 2 tập kết quả.
- Chỉ trả về các dòng **xuất hiện trong cả hai truy vấn**.
- Tự động loại bỏ trùng lặp.

```sql
-- Khách hàng xuất hiện ở cả 2023 và 2024
SELECT customer_id, name
FROM customers_2023
INTERSECT
SELECT customer_id, name
FROM customers_2024;
```

**Output (với dữ liệu trên):**

| customer_id | name   |
|------------|--------|
| 2          | Bình   |

---

## 4. MINUS (trong Oracle)

- Lấy **phần hiệu**: các dòng có trong truy vấn thứ nhất **nhưng không có** trong truy vấn thứ hai.
- Tự động loại bỏ trùng lặp.
- Trong các hệ CSDL khác thường dùng `EXCEPT` thay cho `MINUS`.

```sql
-- Khách hàng chỉ có ở 2023, không có ở 2024
SELECT customer_id, name
FROM customers_2023
MINUS
SELECT customer_id, name
FROM customers_2024;
```

**Output (với dữ liệu trên):**

| customer_id | name   |
|------------|--------|
| 1          | An     |

---

## 5. Lưu ý chung

- Các truy vấn trong `UNION / UNION ALL / INTERSECT / MINUS`:
  - Phải có **cùng số cột**.
  - Các cột tương ứng phải có **kiểu dữ liệu tương thích**.
- Có thể dùng `ORDER BY` **chỉ ở truy vấn cuối cùng**:

```sql
SELECT customer_id, name
FROM customers_2023
UNION
SELECT customer_id, name
FROM customers_2024
ORDER BY customer_id;
```

---

Tóm lại:

- `UNION`: gộp + loại trùng.
- `UNION ALL`: gộp + giữ trùng.
- `INTERSECT`: phần giao.
- `MINUS`: phần hiệu (chỉ có ở truy vấn 1, không có ở truy vấn 2).
