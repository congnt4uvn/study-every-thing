
Dưới đây là tóm tắt ngắn gọn về 3 kiểu collection nâng cao trong PL/SQL: **INDEX BY table**, **nested table**, và **VARRAY**.

---

## 1. INDEX BY Table (Associative Array)

- Là mảng liên kết (key–value) trong PL/SQL, không tồn tại ở mức schema (chỉ dùng trong PL/SQL).
- Chỉ số (index) có thể là **BINARY_INTEGER / PLS_INTEGER** hoặc **VARCHAR2**.
- Không cần `EXTEND`, có thể gán trực tiếp phần tử.
- Không thể lưu trực tiếp vào cột bảng trong database.

### Khai báo & sử dụng

```sql
DECLARE
  TYPE t_emp_tab IS TABLE OF VARCHAR2(100) INDEX BY PLS_INTEGER;
  l_emps t_emp_tab;
  i PLS_INTEGER;
BEGIN
  -- Gán giá trị
  l_emps(1) := 'Scott';
  l_emps(2) := 'Tiger';
  l_emps(10) := 'King';

  -- Duyệt bằng FIRST/NEXT
  i := l_emps.FIRST;
  WHILE i IS NOT NULL LOOP
    DBMS_OUTPUT.PUT_LINE('Index = ' || i || ', Value = ' || l_emps(i));
    i := l_emps.NEXT(i);
  END LOOP;
END;
/
```

**Output (ví dụ):**
```text
Index = 1, Value = Scott
Index = 2, Value = Tiger
Index = 10, Value = King
```

## 2. Nested Table

- Là collection có thể:
  - Dùng trong PL/SQL **và**
  - Định nghĩa ở mức schema để làm kiểu cột trong bảng.
- Chỉ số là số nguyên liên tiếp **bắt đầu từ 1**, nhưng có thể có “lỗ hổng” sau khi `DELETE`.
- Cần `EXTEND` trước khi gán phần tử (trong PL/SQL).
- Có thể lưu vào database, truy vấn bằng SQL.

### Khai báo trong PL/SQL

```sql
DECLARE
  TYPE t_num_tab IS TABLE OF NUMBER;
  l_nums t_num_tab := t_num_tab();  -- khởi tạo
BEGIN
  l_nums.EXTEND(3);
  l_nums(1) := 10;
  l_nums(2) := 20;
  l_nums(3) := 30;

  -- Xóa phần tử thứ 2
  l_nums.DELETE(2);

  -- Duyệt bằng COUNT và FIRST/NEXT
  DBMS_OUTPUT.PUT_LINE('COUNT = ' || l_nums.COUNT); -- 2

  DECLARE
    idx PLS_INTEGER;
  BEGIN
    idx := l_nums.FIRST;
    WHILE idx IS NOT NULL LOOP
      DBMS_OUTPUT.PUT_LINE('Index = ' || idx || ', Value = ' || l_nums(idx));
      idx := l_nums.NEXT(idx);
    END LOOP;
  END;
END;
/
```

**Output (ví dụ):**
```text
COUNT = 2
Index = 1, Value = 10
Index = 3, Value = 30
```

### Khai báo ở mức schema & dùng trong bảng

```sql
-- 1. Tạo kiểu nested table
CREATE OR REPLACE TYPE t_phone_list AS TABLE OF VARCHAR2(20);
/

-- 2. Tạo bảng dùng nested table
CREATE TABLE customers (
  customer_id NUMBER,
  name        VARCHAR2(100),
  phones      t_phone_list
) NESTED TABLE phones STORE AS phones_nt;
/

-- 3. Chèn dữ liệu
INSERT INTO customers (customer_id, name, phones)
VALUES (
  1,
  'Nguyen Van A',
  t_phone_list('0901', '0902', '0903')
);

COMMIT;

-- 4. Truy vấn
SELECT c.name, p.COLUMN_VALUE AS phone
FROM   customers c,
       TABLE(c.phones) p;
```

**Output (ví dụ):**
```text
NAME          PHONE
------------- ------
Nguyen Van A  0901
Nguyen Van A  0902
Nguyen Van A  0903
```

## 3. VARRAY (Variable-size Array)

- Mảng có **kích thước tối đa cố định** (định nghĩa khi tạo type).
- Phần tử được lưu **theo thứ tự**; index từ 1 đến `max_size`.
- Thích hợp khi số lượng phần tử nhỏ và giới hạn.
- Có thể dùng trong PL/SQL và làm kiểu cột trong bảng.

### Khai báo trong PL/SQL

```sql
DECLARE
  TYPE t_scores IS VARRAY(5) OF NUMBER;  -- tối đa 5 phần tử
  l_scores t_scores := t_scores();
BEGIN
  l_scores.EXTEND(3);
  l_scores(1) := 8.5;
  l_scores(2) := 9.0;
  l_scores(3) := 7.5;

  FOR i IN 1 .. l_scores.COUNT LOOP
    DBMS_OUTPUT.PUT_LINE('Index = ' || i || ', Score = ' || l_scores(i));
  END LOOP;
END;
/
```

**Output (ví dụ):**
```text
Index = 1, Score = 8.5
Index = 2, Score = 9
Index = 3, Score = 7.5
```

### Khai báo ở mức schema & dùng trong bảng

```sql
-- 1. Tạo kiểu VARRAY
CREATE OR REPLACE TYPE t_grade_list AS VARRAY(5) OF NUMBER;
/

-- 2. Tạo bảng
CREATE TABLE students (
  student_id NUMBER,
  name       VARCHAR2(100),
  grades     t_grade_list
);
/

-- 3. Chèn dữ liệu
INSERT INTO students (student_id, name, grades)
VALUES (
  1,
  'Tran Thi B',
  t_grade_list(8, 9, 7.5)
);

COMMIT;

-- 4. Truy vấn
SELECT s.name, g.COLUMN_VALUE AS grade
FROM   students s,
       TABLE(s.grades) g;
```

**Output (ví dụ):**
```text
NAME         GRADE
------------ -----
Tran Thi B   8
Tran Thi B   9
Tran Thi B   7.5
```

## So sánh nhanh (tóm tắt)

| Đặc điểm                     | INDEX BY table (Associative) | Nested table                 | VARRAY                         |
|-----------------------------|------------------------------|-----------------------------|--------------------------------|
| Dùng trong PL/SQL           | Có                           | Có                          | Có                             |
| Dùng làm kiểu cột (schema)  | Không                        | Có                          | Có                             |
| Chỉ số                      | Số / chuỗi (tự do)           | Số nguyên, bắt đầu từ 1     | Số nguyên, 1..max_size         |
| Cần `EXTEND`                | Không                        | Có                          | Có                             |
| Kích thước tối đa cố định   | Không                        | Không                        | Có (định nghĩa khi tạo type)   |
| Thứ tự phần tử quan trọng   | Không                        | Không (có thể sắp xếp lại)  | Có (giữ nguyên thứ tự)         |

Nếu bạn muốn, tôi có thể viết thêm ví dụ về thủ tục/hàm nhận và trả về các collection này.
