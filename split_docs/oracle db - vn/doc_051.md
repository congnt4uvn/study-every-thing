
Dưới đây là tóm tắt ngắn gọn bằng tiếng Việt, kèm ví dụ:

---

## Ý chính

- Khi bạn dùng **hàm trên cột đã được đánh index** trong mệnh đề `WHERE`, Oracle **không thể sử dụng index đó** (trừ khi bạn tạo **function-based index**).
- Điều này làm truy vấn chậm vì Oracle phải **full table scan** thay vì **index range scan**.
- Giải pháp:  
  - Tránh bọc cột đã index trong hàm trong `WHERE`, hoặc  
  - Tạo **function-based index** đúng với biểu thức bạn dùng.

---

## Ví dụ 1: Không dùng function-based index → index không được dùng

Giả sử có bảng:

```sql
CREATE TABLE employees (
    employee_id   NUMBER PRIMARY KEY,
    first_name    VARCHAR2(50),
    last_name     VARCHAR2(50),
    hire_date     DATE
);

CREATE INDEX idx_emp_last_name ON employees(last_name);
```

Truy vấn:

```sql
SELECT *
FROM employees
WHERE UPPER(last_name) = 'NGUYEN';
```

- Ở đây `last_name` đã có index (`idx_emp_last_name`).
- Nhưng vì bạn dùng `UPPER(last_name)`, Oracle **không dùng được** index này (trừ khi có function-based index tương ứng).
- Kết quả: thường là **full table scan**.

---

## Ví dụ 2: Dùng function-based index → index được dùng

Tạo function-based index:

```sql
CREATE INDEX idx_emp_upper_last_name
ON employees (UPPER(last_name));
```

Truy vấn:

```sql
SELECT *
FROM employees
WHERE UPPER(last_name) = 'NGUYEN';
```

- Bây giờ Oracle có thể dùng `idx_emp_upper_last_name`.
- Kế hoạch thực thi (EXPLAIN PLAN) sẽ cho thấy **INDEX RANGE SCAN** trên `idx_emp_upper_last_name`.

### Kiểm tra kế hoạch thực thi

```sql
EXPLAIN PLAN FOR
SELECT *
FROM employees
WHERE UPPER(last_name) = 'NGUYEN';

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

**Output (rút gọn, minh họa):**

```text
--------------------------------------------------------------------------------
| Id  | Operation                   | Name                     | Rows | Cost  |
--------------------------------------------------------------------------------
|   0 | SELECT STATEMENT            |                          |    1 |    2  |
|   1 |  TABLE ACCESS BY INDEX ROWID| EMPLOYEES                |    1 |    2  |
|   2 |   INDEX RANGE SCAN          | IDX_EMP_UPPER_LAST_NAME  |    1 |    1  |
--------------------------------------------------------------------------------
```

---

## Ví dụ 3: Viết lại điều kiện để dùng index sẵn có

Nếu không muốn tạo function-based index, hãy tránh dùng hàm trên cột:

Thay vì:

```sql
WHERE UPPER(last_name) = 'NGUYEN';
```

Có thể chuẩn hóa dữ liệu hoặc tham số đầu vào để không cần hàm:

```sql
-- Giả sử dữ liệu last_name luôn lưu UPPER
SELECT *
FROM employees
WHERE last_name = 'NGUYEN';
```

Lúc này index `idx_emp_last_name` sẽ được dùng.

---

## Tóm tắt

- **Không nên** dùng hàm trực tiếp trên cột đã index trong `WHERE` nếu muốn tận dụng index.
- Nếu bắt buộc phải dùng hàm, hãy tạo **function-based index** tương ứng với biểu thức đó.
- Kiểm tra bằng `EXPLAIN PLAN` / `DBMS_XPLAN.DISPLAY` để xác nhận index có được dùng hay không.
