
Dưới đây là tóm tắt ngắn gọn về các kiểu phân vùng (partition) trong thiết kế vật lý Oracle: RANGE, LIST, HASH, COMPOSITE, kèm ví dụ.

---

## 1. RANGE Partitioning (Phân vùng theo khoảng)

- Dữ liệu được chia theo khoảng giá trị của một cột (thường là ngày, số).
- Thích hợp cho dữ liệu theo thời gian (theo tháng, quý, năm).

**Ví dụ: phân vùng theo năm của cột ORDER_DATE**

```sql
CREATE TABLE orders_range (
    order_id    NUMBER,
    order_date  DATE,
    amount      NUMBER
)
PARTITION BY RANGE (order_date) (
    PARTITION p_2019 VALUES LESS THAN (DATE '2020-01-01'),
    PARTITION p_2020 VALUES LESS THAN (DATE '2021-01-01'),
    PARTITION p_2021 VALUES LESS THAN (DATE '2022-01-01'),
    PARTITION p_max  VALUES LESS THAN (MAXVALUE)
);
```

**Input (INSERT):**
```sql
INSERT INTO orders_range VALUES (1, DATE '2019-05-10', 1000);
INSERT INTO orders_range VALUES (2, DATE '2020-07-15', 2000);
```

**Kết quả (phân vùng):**
- Dòng 1 → partition `p_2019`
- Dòng 2 → partition `p_2020`

---

## 2. LIST Partitioning (Phân vùng theo danh sách giá trị)

- Dữ liệu được chia theo danh sách giá trị rời rạc (ví dụ: quốc gia, khu vực).
- Thích hợp khi có tập giá trị rõ ràng, không liên tục.

**Ví dụ: phân vùng theo COUNTRY_CODE**

```sql
CREATE TABLE customers_list (
    customer_id   NUMBER,
    name          VARCHAR2(100),
    country_code  VARCHAR2(2)
)
PARTITION BY LIST (country_code) (
    PARTITION p_us  VALUES ('US'),
    PARTITION p_uk  VALUES ('UK'),
    PARTITION p_asia VALUES ('VN', 'TH', 'SG'),
    PARTITION p_other VALUES (DEFAULT)
);
```

**Input (INSERT):**
```sql
INSERT INTO customers_list VALUES (1, 'A', 'US');
INSERT INTO customers_list VALUES (2, 'B', 'VN');
INSERT INTO customers_list VALUES (3, 'C', 'BR');
```

**Kết quả (phân vùng):**
- 'US' → `p_us`
- 'VN' → `p_asia`
- 'BR' (không nằm trong danh sách) → `p_other` (DEFAULT)

---

## 3. HASH Partitioning (Phân vùng theo băm)

- Oracle dùng hàm băm trên cột phân vùng để phân phối dữ liệu đều giữa các partition.
- Thích hợp cho cân bằng tải, truy cập ngẫu nhiên, không có quy luật rõ ràng.

**Ví dụ: phân vùng theo CUSTOMER_ID**

```sql
CREATE TABLE customers_hash (
    customer_id NUMBER,
    name        VARCHAR2(100),
    email       VARCHAR2(200)
)
PARTITION BY HASH (customer_id)
PARTITIONS 4;  -- Oracle tự tạo 4 partition p1, p2, p3, p4
```

**Input (INSERT):**
```sql
INSERT INTO customers_hash VALUES (1, 'A', 'a@test.com');
INSERT INTO customers_hash VALUES (2, 'B', 'b@test.com');
INSERT INTO customers_hash VALUES (3, 'C', 'c@test.com');
```

**Kết quả (phân vùng):**
- Oracle dùng hàm băm nội bộ, ví dụ:
  - customer_id = 1 → `SYS_P0`
  - customer_id = 2 → `SYS_P1`
  - customer_id = 3 → `SYS_P2`  
  (tên partition thực tế do Oracle đặt, nhưng dữ liệu được phân bố tương đối đều)

---

## 4. COMPOSITE Partitioning (Phân vùng kết hợp)

- Kết hợp 2 kiểu partition, phổ biến nhất:
  - RANGE-HASH
  - RANGE-LIST
- Dùng RANGE cho partition chính, HASH/LIST cho subpartition.
- Thích hợp cho bảng rất lớn, cần chia nhỏ hơn nữa trong từng khoảng.

### 4.1 RANGE-HASH

**Ví dụ: RANGE theo ORDER_DATE, trong mỗi năm HASH theo CUSTOMER_ID**

```sql
CREATE TABLE orders_comp_range_hash (
    order_id     NUMBER,
    order_date   DATE,
    customer_id  NUMBER,
    amount       NUMBER
)
PARTITION BY RANGE (order_date)
SUBPARTITION BY HASH (customer_id)
SUBPARTITIONS 4
(
    PARTITION p_2020 VALUES LESS THAN (DATE '2021-01-01'),
    PARTITION p_2021 VALUES LESS THAN (DATE '2022-01-01'),
    PARTITION p_max  VALUES LESS THAN (MAXVALUE)
);
```

**Input (INSERT):**
```sql
INSERT INTO orders_comp_range_hash
VALUES (1, DATE '2020-03-10', 101, 500);

INSERT INTO orders_comp_range_hash
VALUES (2, DATE '2021-06-20', 202, 800);
```

**Kết quả (phân vùng):**
- Dòng 1:
  - RANGE: `2020-03-10` → partition `p_2020`
  - HASH: `customer_id = 101` → 1 trong 4 subpartition của `p_2020`
- Dòng 2:
  - RANGE: `2021-06-20` → partition `p_2021`
  - HASH: `customer_id = 202` → 1 trong 4 subpartition của `p_2021`

### 4.2 RANGE-LIST (ví dụ khác)

```sql
CREATE TABLE sales_comp_range_list (
    sale_id     NUMBER,
    sale_date   DATE,
    region      VARCHAR2(10),
    amount      NUMBER
)
PARTITION BY RANGE (sale_date)
SUBPARTITION BY LIST (region)
(
    PARTITION p_2020 VALUES LESS THAN (DATE '2021-01-01') (
        SUBPARTITION p_2020_north VALUES ('NORTH'),
        SUBPARTITION p_2020_south VALUES ('SOUTH')
    ),
    PARTITION p_2021 VALUES LESS THAN (DATE '2022-01-01') (
        SUBPARTITION p_2021_north VALUES ('NORTH'),
        SUBPARTITION p_2021_south VALUES ('SOUTH')
    )
);
```

---

## Tóm tắt ý chính

- **RANGE**: chia theo khoảng giá trị (thường theo thời gian).  
- **LIST**: chia theo danh sách giá trị cụ thể (quốc gia, vùng, loại…).  
- **HASH**: chia đều dữ liệu bằng hàm băm, không phụ thuộc vào ý nghĩa giá trị.  
- **COMPOSITE**: kết hợp 2 kiểu (thường RANGE + HASH/LIST) để quản lý bảng rất lớn, tối ưu cả truy vấn theo thời gian và phân bố dữ liệu.
