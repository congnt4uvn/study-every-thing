
Dưới đây là tóm tắt ngắn gọn về **Initialization Sections trong PL/SQL** (mức nâng cao), kèm ví dụ mã và input/output.  

---

## 1. Initialization Section là gì?

Trong một khối PL/SQL (block), package, hoặc anonymous block, ta có thể có 3 phần:

```plsql
DECLARE
    -- Declaration section: khai báo biến, hằng, cursor, type...
BEGIN
    -- Execution section: các lệnh thực thi chính
EXCEPTION
    -- Exception section: xử lý lỗi (nếu có)
END;
/
```

**Initialization section** thường dùng để chỉ phần mã được chạy **một lần khi package được load vào bộ nhớ** (package initialization), hoặc phần mã chạy **ngay khi block bắt đầu** (anonymous block / procedure / function).

---

## 2. Initialization trong Package

Trong package body, phần mã **BEGIN ... END;** ở cuối chính là **package initialization section**.  
Nó chỉ chạy **một lần cho mỗi session** khi package được tham chiếu lần đầu.

### Ví dụ 1: Package có initialization section

```plsql
CREATE OR REPLACE PACKAGE pkg_config IS
    g_app_name   VARCHAR2(100);
    g_start_time DATE;
    
    PROCEDURE show_info;
END pkg_config;
/
```

```plsql
CREATE OR REPLACE PACKAGE BODY pkg_config IS

    PROCEDURE show_info IS
    BEGIN
        DBMS_OUTPUT.PUT_LINE('App: ' || g_app_name);
        DBMS_OUTPUT.PUT_LINE('Start time: ' || TO_CHAR(g_start_time, 'YYYY-MM-DD HH24:MI:SS'));
    END show_info;

BEGIN
    -- Initialization section của package
    g_app_name   := 'Order Management System';
    g_start_time := SYSDATE;
    DBMS_OUTPUT.PUT_LINE('pkg_config initialized at ' ||
                         TO_CHAR(g_start_time, 'YYYY-MM-DD HH24:MI:SS'));
END pkg_config;
/
```

#### Input / Output minh họa

```plsql
SET SERVEROUTPUT ON;

BEGIN
    -- Lần đầu tiên gọi package trong session
    pkg_config.show_info;
END;
/
```

**Output (ví dụ):**

```text
pkg_config initialized at 2026-05-20 10:15:30
App: Order Management System
Start time: 2026-05-20 10:15:30
```

Nếu bạn gọi lại:

```plsql
BEGIN
    pkg_config.show_info;
END;
/
```

**Output:**

```text
App: Order Management System
Start time: 2026-05-20 10:15:30
```

Dòng `pkg_config initialized at ...` **không xuất hiện nữa**, vì initialization section chỉ chạy **một lần** khi package được load.

## 3. Initialization trong Anonymous Block

Trong anonymous block, phần `BEGIN ...` đầu tiên chính là nơi bạn có thể coi như **initialization + execution**.  
Bạn có thể tách logic khởi tạo ra thành các procedure nội bộ.

### Ví dụ 2: Khởi tạo biến trong anonymous block

```plsql
DECLARE
    v_tax_rate NUMBER;
    v_amount   NUMBER := 1000;
    v_total    NUMBER;
BEGIN
    -- Initialization logic
    v_tax_rate := 0.1;  -- 10%

    -- Execution logic
    v_total := v_amount + v_amount * v_tax_rate;
    DBMS_OUTPUT.PUT_LINE('Total amount: ' || v_total);
END;
/
```

**Output:**

```text
Total amount: 1100
```

## 4. Initialization với biến toàn cục trong Package

Initialization section rất hữu ích để:

- Gán giá trị ban đầu cho **biến toàn cục** (global variables)
- Đọc cấu hình từ bảng
- Thiết lập môi trường (NLS, context, v.v.)

### Ví dụ 3: Đọc cấu hình từ bảng khi package load

Giả sử có bảng cấu hình:

```plsql
CREATE TABLE app_config (
    config_name  VARCHAR2(30) PRIMARY KEY,
    config_value VARCHAR2(100)
);

INSERT INTO app_config VALUES ('DEFAULT_LANGUAGE', 'VI');
INSERT INTO app_config VALUES ('MAX_LOGIN_ATTEMPTS', '5');
COMMIT;
```

Package:

```plsql
CREATE OR REPLACE PACKAGE pkg_app_settings IS
    g_default_language   VARCHAR2(2);
    g_max_login_attempts PLS_INTEGER;

    PROCEDURE show_settings;
END pkg_app_settings;
/
```

```plsql
CREATE OR REPLACE PACKAGE BODY pkg_app_settings IS

    PROCEDURE show_settings IS
    BEGIN
        DBMS_OUTPUT.PUT_LINE('Default language: ' || g_default_language);
        DBMS_OUTPUT.PUT_LINE('Max login attempts: ' || g_max_login_attempts);
    END show_settings;

BEGIN
    -- Initialization section: đọc cấu hình từ bảng
    SELECT config_value
      INTO g_default_language
      FROM app_config
     WHERE config_name = 'DEFAULT_LANGUAGE';

    SELECT TO_NUMBER(config_value)
      INTO g_max_login_attempts
      FROM app_config
     WHERE config_name = 'MAX_LOGIN_ATTEMPTS';

    DBMS_OUTPUT.PUT_LINE('pkg_app_settings initialized');
END pkg_app_settings;
/
```

**Input / Output:**

```plsql
SET SERVEROUTPUT ON;

BEGIN
    pkg_app_settings.show_settings;
END;
/
```

Output:

```text
pkg_app_settings initialized
Default language: VI
Max login attempts: 5
```

---

## 5. Điểm chính cần nhớ

1. **Initialization section trong package body** là phần `BEGIN ... END;` cuối package body.
2. Nó được thực thi **một lần cho mỗi session**, khi package được sử dụng lần đầu.
3. Dùng để:
   - Gán giá trị ban đầu cho biến toàn cục
   - Đọc cấu hình từ bảng
   - Thiết lập môi trường / context
4. Trong anonymous block, phần `BEGIN` đầu tiên vừa là nơi khởi tạo vừa là nơi thực thi, nhưng bạn có thể tổ chức code để tách logic khởi tạo cho rõ ràng.

Nếu bạn gửi thêm nội dung chi tiết hơn (ví dụ: đoạn tài liệu hoặc code cụ thể), tôi có thể tóm tắt sâu hơn đúng theo nội dung đó.
