
Dưới đây là tóm tắt ngắn gọn về **Exception Handling trong PL/SQL** (Oracle), kèm ví dụ code và input/output minh họa.

---

## 1. Khái niệm chung về Exception trong PL/SQL

- **Exception**: lỗi xảy ra trong lúc chạy (runtime error).
- Khi exception xảy ra, PL/SQL sẽ:
  1. Dừng phần còn lại của khối lệnh hiện tại.
  2. Nhảy đến phần `EXCEPTION` (nếu có) để xử lý.
  3. Nếu không được xử lý, lỗi sẽ “bong bóng” lên khối ngoài hoặc trả về cho client.

Cấu trúc khối PL/SQL:

```sql
DECLARE
    -- khai báo biến, hằng, exception
BEGIN
    -- khối lệnh chính
EXCEPTION
    -- xử lý lỗi
END;
/
```

---

## 2. Exception dựng sẵn (Predefined Exceptions)

Oracle cung cấp sẵn nhiều exception thường gặp, ví dụ:

- `NO_DATA_FOUND` – không có dòng nào được SELECT INTO.
- `TOO_MANY_ROWS` – SELECT INTO trả về nhiều hơn 1 dòng.
- `ZERO_DIVIDE` – chia cho 0.
- `VALUE_ERROR` – lỗi chuyển kiểu, tràn số, v.v.

### Ví dụ: NO_DATA_FOUND và TOO_MANY_ROWS

```sql
DECLARE
    v_name   employees.last_name%TYPE;
BEGIN
    -- Giả sử chỉ có 1 nhân viên có employee_id = 9999
    SELECT last_name
    INTO   v_name
    FROM   employees
    WHERE  employee_id = 9999;

    DBMS_OUTPUT.PUT_LINE('Tên nhân viên: ' || v_name);

EXCEPTION
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('Không tìm thấy nhân viên.');
    WHEN TOO_MANY_ROWS THEN
        DBMS_OUTPUT.PUT_LINE('Có nhiều hơn một nhân viên phù hợp.');
END;
/
```

**Input**: `employee_id = 9999`  
- Nếu không có nhân viên nào:  
  **Output**: `Không tìm thấy nhân viên.`  
- Nếu có nhiều nhân viên (trường hợp thiết kế sai):  
  **Output**: `Có nhiều hơn một nhân viên phù hợp.`  
- Nếu đúng 1 nhân viên:  
  **Output**: `Tên nhân viên: <tên>`

## 3. Exception do người dùng định nghĩa (User-defined Exceptions)

Bạn có thể tự định nghĩa exception để biểu diễn các lỗi nghiệp vụ (business logic).

### Cách dùng:

1. Khai báo exception trong phần `DECLARE`.
2. Dùng lệnh `RAISE` để kích hoạt.
3. Bắt và xử lý trong phần `EXCEPTION`.

### Ví dụ: Exception kiểm tra lương tối thiểu

```sql
DECLARE
    e_low_salary EXCEPTION;  -- exception do người dùng định nghĩa
    v_salary     employees.salary%TYPE;
BEGIN
    SELECT salary
    INTO   v_salary
    FROM   employees
    WHERE  employee_id = 100;

    IF v_salary < 3000 THEN
        RAISE e_low_salary;  -- kích hoạt exception
    END IF;

    DBMS_OUTPUT.PUT_LINE('Lương hợp lệ: ' || v_salary);

EXCEPTION
    WHEN e_low_salary THEN
        DBMS_OUTPUT.PUT_LINE('Lương thấp hơn mức cho phép (3000).');
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('Không tìm thấy nhân viên.');
END;
/
```

**Input**: `employee_id = 100`  
- Nếu `salary < 3000`:  
  **Output**: `Lương thấp hơn mức cho phép (3000).`  
- Nếu `salary >= 3000`:  
  **Output**: `Lương hợp lệ: <mức lương>`

## 4. RAISE_APPLICATION_ERROR

Dùng để:
- Tạo lỗi ứng dụng với **mã lỗi tùy chọn** (trong khoảng `-20000` đến `-20999`).
- Trả lỗi về cho client (SQL*Plus, ứng dụng Java, .NET, v.v.).
- Thường dùng trong **stored procedure, function, trigger**.

### Cú pháp:

```sql
RAISE_APPLICATION_ERROR(error_number, message [, keep_errors]);
```

- `error_number`: số âm từ `-20000` đến `-20999`.
- `message`: chuỗi mô tả lỗi (tối đa 2048 ký tự).
- `keep_errors` (tùy chọn): `TRUE` để giữ lại lỗi trước đó trong stack.

### Ví dụ: Kiểm tra lương bằng RAISE_APPLICATION_ERROR

```sql
CREATE OR REPLACE PROCEDURE update_salary (
    p_emp_id   IN employees.employee_id%TYPE,
    p_new_sal  IN employees.salary%TYPE
) AS
BEGIN
    IF p_new_sal < 3000 THEN
        RAISE_APPLICATION_ERROR(
            -20001,
            'Lương mới phải >= 3000. Giá trị hiện tại: ' || p_new_sal
        );
    END IF;

    UPDATE employees
    SET    salary = p_new_sal
    WHERE  employee_id = p_emp_id;

    IF SQL%ROWCOUNT = 0 THEN
        RAISE_APPLICATION_ERROR(
            -20002,
            'Không tìm thấy nhân viên với ID = ' || p_emp_id
        );
    END IF;
END;
/
```

**Gọi procedure và kết quả:**

```sql
-- Trường hợp lương quá thấp
BEGIN
    update_salary(100, 2000);
END;
/
```

**Output (lỗi trả về cho client):**

```text
ORA-20001: Lương mới phải >= 3000. Giá trị hiện tại: 2000
ORA-06512: at "SCOTT.UPDATE_SALARY", line ...
```

```sql
-- Trường hợp không có nhân viên
BEGIN
    update_salary(9999, 5000);
END;
/
```

**Output:**

```text
ORA-20002: Không tìm thấy nhân viên với ID = 9999
ORA-06512: at "SCOTT.UPDATE_SALARY", line ...
```

---

## 5. Tóm tắt nhanh

- **Predefined exceptions**: lỗi hệ thống phổ biến (NO_DATA_FOUND, TOO_MANY_ROWS, ZERO_DIVIDE, VALUE_ERROR, …).
- **User-defined exceptions**: tự định nghĩa để xử lý lỗi nghiệp vụ, dùng `RAISE` để kích hoạt.
- **RAISE_APPLICATION_ERROR**: tạo lỗi ứng dụng với mã lỗi riêng (-20000 đến -20999), thường dùng trong procedure/function/trigger để báo lỗi ra ngoài.

Nếu bạn muốn, tôi có thể viết thêm ví dụ với `PRAGMA EXCEPTION_INIT` để map mã lỗi Oracle sang exception do bạn đặt tên.
