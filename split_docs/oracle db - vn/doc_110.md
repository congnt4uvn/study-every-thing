
Dưới đây là tóm tắt ngắn gọn về cách xử lý **multi-tenant (nhiều khách hàng)** trong **một Oracle Database**, kèm ví dụ code.  

---

## 1. Các mô hình multi-tenant phổ biến

### 1. Một schema cho nhiều khách hàng (Shared Schema)

- Tất cả khách hàng dùng chung **các bảng**.
- Phân biệt khách hàng bằng một cột, ví dụ: `TENANT_ID`.
- Dễ quản lý, ít đối tượng, nhưng phải kiểm soát chặt chẽ quyền truy cập theo `TENANT_ID`.

**Ví dụ bảng:**

```sql
CREATE TABLE orders (
    order_id     NUMBER PRIMARY KEY,
    tenant_id    NUMBER NOT NULL,
    customer_id  NUMBER,
    amount       NUMBER,
    created_at   DATE DEFAULT SYSDATE
);
```

**Chèn dữ liệu (input):**

```sql
INSERT INTO orders (order_id, tenant_id, customer_id, amount)
VALUES (1, 1001, 501, 200000);

INSERT INTO orders (order_id, tenant_id, customer_id, amount)
VALUES (2, 1002, 601, 350000);
COMMIT;
```

**Truy vấn cho một tenant (output):**

```sql
SELECT order_id, customer_id, amount
FROM   orders
WHERE  tenant_id = 1001;
```

Kết quả (giả sử):

| ORDER_ID | CUSTOMER_ID | AMOUNT |
|----------|-------------|--------|
| 1        | 501         | 200000 |

---

### 2. Mỗi khách hàng một schema (Separate Schema per Tenant)

- Mỗi tenant có **schema riêng**: `TENANT1`, `TENANT2`, ...
- Cấu trúc bảng giống nhau, dữ liệu tách biệt.
- Tăng độ cô lập, nhưng số lượng schema lớn sẽ khó quản lý.

**Ví dụ:**

```sql
-- Schema TENANT1
CREATE TABLE tenant1.orders (
    order_id    NUMBER PRIMARY KEY,
    customer_id NUMBER,
    amount      NUMBER
);

-- Schema TENANT2
CREATE TABLE tenant2.orders (
    order_id    NUMBER PRIMARY KEY,
    customer_id NUMBER,
    amount      NUMBER
);
```

**Truy vấn:**

```sql
SELECT * FROM tenant1.orders;
SELECT * FROM tenant2.orders;
```

---

### 3. Oracle Multitenant: CDB / PDB (Container Database / Pluggable Database)

- Một **CDB** chứa nhiều **PDB**.
- Mỗi tenant có thể là một **PDB riêng** → cô lập rất cao (gần như database riêng).
- Phù hợp khi mỗi khách hàng lớn, yêu cầu bảo mật & cấu hình riêng.

**Ví dụ tạo PDB cho một tenant:**

```sql
-- Kết nối vào CDB
CONNECT sys/password@cdb AS SYSDBA;

-- Tạo PDB mới cho tenant A
CREATE PLUGGABLE DATABASE tenantA_pdb
  ADMIN USER tenant_admin IDENTIFIED BY "StrongPass123"
  FILE_NAME_CONVERT = ('/u01/app/oracle/oradata/CDB1/pdbseed/',
                       '/u01/app/oracle/oradata/CDB1/tenantA_pdb/');

-- Mở PDB
ALTER PLUGGABLE DATABASE tenantA_pdb OPEN;
```

**Kết nối vào PDB của tenant A:**

```sql
CONNECT tenant_admin/StrongPass123@tenantA_pdb;
```

### 4. Dùng VPD / RLS (Virtual Private Database / Row-Level Security)

- Áp dụng cho mô hình **shared schema**.
- Tự động thêm điều kiện `TENANT_ID` vào mọi câu lệnh SELECT/UPDATE/DELETE.
- Giúp đảm bảo mỗi tenant chỉ thấy dữ liệu của mình.

**Ví dụ:**

```sql
-- Bảng shared
CREATE TABLE orders (
    order_id     NUMBER PRIMARY KEY,
    tenant_id    NUMBER NOT NULL,
    customer_id  NUMBER,
    amount       NUMBER
);

-- Hàm policy trả về điều kiện lọc theo tenant hiện tại
CREATE OR REPLACE FUNCTION fn_tenant_policy (
    p_schema IN VARCHAR2,
    p_object IN VARCHAR2
) RETURN VARCHAR2 AS
BEGIN
    RETURN 'tenant_id = SYS_CONTEXT(''USERENV'', ''CLIENT_IDENTIFIER'')';
END;
/

-- Gán policy VPD
BEGIN
    DBMS_RLS.ADD_POLICY(
        object_schema   => 'APP',
        object_name     => 'ORDERS',
        policy_name     => 'ORDERS_TENANT_POLICY',
        function_schema => 'APP',
        policy_function => 'FN_TENANT_POLICY',
        statement_types => 'SELECT,INSERT,UPDATE,DELETE'
    );
END;
/
```

**Thiết lập tenant hiện tại (input):**

```sql
-- Ứng dụng set CLIENT_IDENTIFIER = tenant_id
EXEC DBMS_SESSION.SET_IDENTIFIER('1001');

SELECT order_id, tenant_id, amount FROM orders;
```

**Output:** chỉ thấy các dòng có `tenant_id = 1001`.

---

## 5. Tóm tắt ý chính

- Multi-tenant trong một Oracle Database có thể làm theo:
  - **Shared schema + TENANT_ID** (đơn giản, rẻ, cần VPD/RLS để bảo mật).
  - **Mỗi tenant một schema** (cô lập hơn, khó scale khi nhiều tenant).
  - **Mỗi tenant một PDB (Oracle Multitenant)** (cô lập cao nhất, phù hợp khách hàng lớn).
- Có thể kết hợp với:
  - **VPD/RLS** để bảo vệ dữ liệu theo dòng.
  - **Resource Manager** để giới hạn tài nguyên cho từng tenant.
