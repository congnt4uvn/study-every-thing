
Dưới đây là tóm tắt ngắn gọn về **Analytic (Window) Functions trong Oracle** kèm ví dụ.

---

## 1. Analytic (Window) Function là gì?

- Là các hàm SQL tính toán trên **một nhóm dòng (window)** liên quan đến dòng hiện tại.
- Không gom nhóm thành 1 dòng như `GROUP BY`, mà **giữ nguyên số dòng**.
- Cú pháp chung:

```sql
<analytic_function>([expression])
OVER (
    [PARTITION BY partition_expression]
    [ORDER BY order_expression]
    [windowing_clause]
)
```

---

## 2. Các nhóm hàm phân tích phổ biến

### 2.1. Hàm tổng hợp phân tích (SUM, AVG, COUNT, MIN, MAX, …)

Tính toán trên cửa sổ dòng, ví dụ: tổng lũy kế, trung bình trượt,…

**Ví dụ: Tổng lũy kế doanh số theo nhân viên**

```sql
SELECT
    emp_id,
    sale_date,
    amount,
    SUM(amount) OVER (
        PARTITION BY emp_id
        ORDER BY sale_date
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS running_total
FROM sales
ORDER BY emp_id, sale_date;
```

**Input (bảng `sales`):**

| emp_id | sale_date  | amount |
|--------|------------|--------|
| 101    | 2024-01-01 | 100    |
| 101    | 2024-01-05 | 200    |
| 101    | 2024-01-10 | 50     |

**Output:**

| emp_id | sale_date  | amount | running_total |
|--------|------------|--------|---------------|
| 101    | 2024-01-01 | 100    | 100           |
| 101    | 2024-01-05 | 200    | 300           |
| 101    | 2024-01-10 | 50     | 350           |

---

### 2.2. Hàm xếp hạng (RANK, DENSE_RANK, ROW_NUMBER)

Dùng để xếp hạng, đánh số dòng trong từng nhóm.

**Ví dụ: Xếp hạng lương trong từng phòng ban**

```sql
SELECT
    deptno,
    ename,
    sal,
    RANK() OVER (
        PARTITION BY deptno
        ORDER BY sal DESC
    ) AS sal_rank,
    DENSE_RANK() OVER (
        PARTITION BY deptno
        ORDER BY sal DESC
    ) AS sal_dense_rank,
    ROW_NUMBER() OVER (
        PARTITION BY deptno
        ORDER BY sal DESC
    ) AS row_num
FROM emp
ORDER BY deptno, sal DESC;
```

- `RANK`: có thể nhảy bậc (1,1,3,…)
- `DENSE_RANK`: không nhảy bậc (1,1,2,…)
- `ROW_NUMBER`: luôn 1,2,3,… (không quan tâm giá trị trùng)

---

### 2.3. Hàm LAG / LEAD (so sánh với dòng trước/sau)

Dùng để lấy giá trị ở dòng trước hoặc sau dòng hiện tại.

**Ví dụ: So sánh doanh số với ngày trước đó**

```sql
SELECT
    emp_id,
    sale_date,
    amount,
    LAG(amount, 1) OVER (
        PARTITION BY emp_id
        ORDER BY sale_date
    ) AS prev_amount,
    amount - LAG(amount, 1) OVER (
        PARTITION BY emp_id
        ORDER BY sale_date
    ) AS diff_from_prev
FROM sales
ORDER BY emp_id, sale_date;
```

**Output mẫu:**

| emp_id | sale_date  | amount | prev_amount | diff_from_prev |
|--------|------------|--------|-------------|----------------|
| 101    | 2024-01-01 | 100    | null        | null           |
| 101    | 2024-01-05 | 200    | 100         | 100            |
| 101    | 2024-01-10 | 50     | 200         | -150           |

---

### 2.4. FIRST_VALUE / LAST_VALUE

Lấy giá trị đầu tiên / cuối cùng trong cửa sổ.

**Ví dụ: Lấy mức lương cao nhất trong phòng ban cho mỗi nhân viên**

```sql
SELECT
    deptno,
    ename,
    sal,
    FIRST_VALUE(sal) OVER (
        PARTITION BY deptno
        ORDER BY sal DESC
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) AS max_sal_in_dept
FROM emp;
```

---

### 2.5. NTILE (chia nhóm phần trăm / phân vị)

Chia dữ liệu thành N nhóm gần bằng nhau (quartile, decile,…).

**Ví dụ: Chia lương thành 4 nhóm (quartile)**

```sql
SELECT
    ename,
    sal,
    NTILE(4) OVER (ORDER BY sal) AS sal_quartile
FROM emp
ORDER BY sal;
```

---

## 3. Các thành phần quan trọng trong OVER()

- `PARTITION BY`: chia dữ liệu thành các nhóm (giống `GROUP BY` nhưng không gom dòng).
- `ORDER BY`: xác định thứ tự trong mỗi partition (quan trọng cho các hàm lũy kế, LAG/LEAD, RANK,…).
- `ROWS BETWEEN ...`: định nghĩa cửa sổ:
  - `UNBOUNDED PRECEDING`: từ đầu partition
  - `CURRENT ROW`: dòng hiện tại
  - `UNBOUNDED FOLLOWING`: đến cuối partition

Ví dụ cửa sổ trượt 3 dòng (dòng hiện tại và 2 dòng trước):

```sql
AVG(amount) OVER (
    ORDER BY sale_date
    ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
) AS moving_avg_3
```

---

## 4. Khác biệt với GROUP BY

- `GROUP BY`: giảm số dòng, mỗi nhóm -> 1 dòng.
- Analytic function: **không giảm số dòng**, chỉ thêm cột tính toán.

**Ví dụ so sánh:**

```sql
-- GROUP BY: 1 dòng mỗi emp_id
SELECT emp_id, SUM(amount) total_amount
FROM sales
GROUP BY emp_id;

-- Analytic: giữ nguyên từng giao dịch, thêm tổng theo emp_id
SELECT
    emp_id,
    sale_date,
    amount,
    SUM(amount) OVER (PARTITION BY emp_id) AS total_amount_by_emp
FROM sales;
```

---

Tóm lại, **analytic/window functions** trong Oracle giúp:
- Tính toán lũy kế, trung bình trượt, so sánh với dòng trước/sau.
- Xếp hạng, phân nhóm theo phân vị.
- Phân tích dữ liệu nâng cao mà vẫn giữ nguyên chi tiết từng dòng.
