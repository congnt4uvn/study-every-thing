
Dưới đây là tóm tắt ngắn gọn về kỹ thuật SQL Tuning với **hints** trong Oracle, kèm ví dụ.

---

## 1. Ý nghĩa của SQL Hints

- **Hints** là chỉ dẫn cho Optimizer của Oracle về cách thực thi câu lệnh SQL.
- Dùng để:
  - Chọn **kế hoạch truy vấn** cụ thể (dùng index, join method, join order…).
  - Tối ưu hiệu năng trong các trường hợp Optimizer tự động chọn chưa tốt.
- **Chỉ dùng khi cần thiết**, sau khi:
  - Đã phân tích kế hoạch thực thi (EXPLAIN PLAN).
  - Đã tối ưu schema, index, thống kê (statistics).

Cú pháp chung:

```sql
SELECT /*+ HINT_NAME [tham_số] */ cột
FROM   bảng
WHERE  điều_kiện;
```

---

## 2. Một số loại hints thường dùng

### 2.1. Hint dùng Index

Buộc Optimizer dùng index cụ thể.

```sql
SELECT /*+ INDEX(emp emp_dept_idx) */
       empno, ename, deptno
FROM   emp
WHERE  deptno = 10;
```

- `emp_dept_idx`: tên index trên cột `deptno`.

**Input:**

```sql
SELECT /*+ INDEX(emp emp_dept_idx) */
       empno, ename, deptno
FROM   emp
WHERE  deptno = 10;
```

**Output (ví dụ):**

```text
EMPNO  ENAME   DEPTNO
-----  ------  ------
7369   SMITH   10
7499   ALLEN   10
7521   WARD    10
```

---

### 2.2. Hint FULL (quét toàn bảng)

Buộc quét full table (khi dữ liệu trả về lớn, index không hiệu quả).

```sql
SELECT /*+ FULL(emp) */
       empno, ename
FROM   emp
WHERE  salary > 1000;
```

---

### 2.3. Hint Join Method (NL, HASH, MERGE)

Ví dụ buộc dùng **HASH JOIN**:

```sql
SELECT /*+ USE_HASH(e d) */
       e.empno, e.ename, d.dname
FROM   emp e
JOIN   dept d ON e.deptno = d.deptno
WHERE  e.salary > 1000;
```

Buộc dùng **NESTED LOOPS**:

```sql
SELECT /*+ USE_NL(e d) */
       e.empno, e.ename, d.dname
FROM   emp e
JOIN   dept d ON e.deptno = d.deptno
WHERE  e.deptno = 10;
```

---

### 2.4. Hint Join Order

Buộc thứ tự join (thay vì để Optimizer tự quyết).

```sql
SELECT /*+ LEADING(e d) */
       e.empno, e.ename, d.dname
FROM   emp e
JOIN   dept d ON e.deptno = d.deptno
WHERE  d.location = 'HANOI';
```

---

### 2.5. Hint PARALLEL

Chạy song song để tăng tốc (cẩn thận tài nguyên).

```sql
SELECT /*+ PARALLEL(emp, 4) */
       COUNT(*)
FROM   emp
WHERE  salary > 1000;
```

---

## 3. Khi nào nên dùng hints?

- Khi:
  - Đã tối ưu index, thống kê, cấu trúc bảng.
  - Đã phân tích `EXPLAIN PLAN` và thấy kế hoạch không tối ưu.
  - Cần “ép” một kế hoạch đã biết là tốt trong môi trường cụ thể.
- Không nên:
  - Lạm dụng hints cho mọi câu SQL.
  - Dùng hints để che giấu thiết kế dữ liệu kém hoặc thiếu index.

---

## 4. Kiểm tra hiệu quả hints

Dùng `EXPLAIN PLAN` hoặc `DBMS_XPLAN.DISPLAY`:

```sql
EXPLAIN PLAN FOR
SELECT /*+ INDEX(emp emp_dept_idx) */
       empno, ename, deptno
FROM   emp
WHERE  deptno = 10;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

Bạn sẽ thấy trong kế hoạch thực thi có sử dụng `INDEX RANGE SCAN` trên `EMP_DEPT_IDX` nếu hint có hiệu lực.
