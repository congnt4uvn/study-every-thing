
Dưới đây là tóm tắt ngắn gọn về chủ đề “Locking & Isolation: V$SESSION, V$LOCK, V$SQL” trong Oracle, kèm ví dụ truy vấn (code) và kết quả mẫu.

---

## 1. Khái niệm chính

- **Locking (khóa)**  
  Oracle dùng khóa để bảo vệ dữ liệu khi nhiều session truy cập/ghi cùng lúc, tránh ghi đè hoặc đọc dữ liệu chưa commit.

- **Isolation (mức cô lập)**  
  Mức cô lập giao dịch (transaction isolation level) quyết định:
  - Có đọc được dữ liệu chưa commit không?
  - Có bị “dirty read”, “non-repeatable read”, “phantom read” không?  
  Oracle mặc định dùng **READ COMMITTED**, hỗ trợ **SERIALIZABLE** và **READ ONLY**.

- **V$SESSION**  
  View động chứa thông tin về các session đang kết nối: SID, SERIAL#, USERNAME, STATUS, SQL_ID, MACHINE, MODULE, v.v.

- **V$LOCK**  
  View động chứa thông tin về các khóa hiện tại: loại khóa, đối tượng bị khóa, session nào giữ khóa, session nào đang chờ.

- **V$SQL**  
  View động chứa thông tin về các câu SQL đã parse trong shared pool: SQL_ID, TEXT, PLAN_HASH_VALUE, số lần thực thi, v.v.

---

## 2. Dùng V$SESSION để xem session & SQL đang chạy

### Ví dụ 1: Liệt kê các session đang hoạt động

```sql
SELECT
    s.sid,
    s.serial#,
    s.username,
    s.status,
    s.machine,
    s.program,
    s.sql_id
FROM v$session s
WHERE s.username IS NOT NULL
  AND s.status = 'ACTIVE';
```

**Kết quả mẫu:**

| SID | SERIAL# | USERNAME | STATUS | MACHINE   | PROGRAM          | SQL_ID      |
|-----|---------|----------|--------|-----------|------------------|------------|
| 123 | 45678   | HR       | ACTIVE | APP-SRV01 | sqlplus.exe      | 5t9k3u1a2b |
| 130 | 98765   | SALES    | ACTIVE | APP-SRV02 | JDBC Thin Client | 9f8d2c3e4f |

---

### Ví dụ 2: Xem câu SQL hiện tại của một session

```sql
SELECT
    s.sid,
    s.serial#,
    q.sql_id,
    q.sql_text
FROM v$session s
JOIN v$sql q
  ON s.sql_id = q.sql_id
WHERE s.sid = 123;
```

**Kết quả mẫu:**

| SID | SERIAL# | SQL_ID     | SQL_TEXT                                  |
|-----|---------|-----------|-------------------------------------------|
| 123 | 45678   | 5t9k3u1a2b| UPDATE employees SET salary = salary*1.1  |

## 3. Dùng V$LOCK để xem khóa & session bị chờ

### Ý nghĩa cơ bản

- Mỗi dòng trong **V$LOCK** là một khóa mà session đang giữ hoặc đang chờ.
- Một số cột quan trọng:
  - `SID` – session ID
  - `TYPE` – loại khóa (TX = transaction, TM = table, v.v.)
  - `ID1`, `ID2` – định danh nội bộ của tài nguyên bị khóa
  - `LMODE` – mode khóa mà session đang giữ
  - `REQUEST` – mode khóa mà session đang yêu cầu (0 = không chờ)

### Ví dụ 3: Xem các session đang chờ khóa

```sql
SELECT
    l1.sid       AS waiting_sid,
    s1.username  AS waiting_user,
    l2.sid       AS holding_sid,
    s2.username  AS holding_user,
    l1.type,
    l1.id1,
    l1.id2
FROM v$lock l1
JOIN v$session s1 ON l1.sid = s1.sid
JOIN v$lock l2
  ON l1.id1 = l2.id1
 AND l1.id2 = l2.id2
 AND l1.type = l2.type
JOIN v$session s2 ON l2.sid = s2.sid
WHERE l1.request > 0      -- session đang chờ
  AND l2.lmode   > 0      -- session đang giữ khóa
  AND l1.sid    != l2.sid;
```

**Kết quả mẫu:**

| WAITING_SID | WAITING_USER | HOLDING_SID | HOLDING_USER | TYPE | ID1      | ID2 |
|-------------|--------------|-------------|--------------|------|----------|-----|
| 130         | SALES        | 123         | HR           | TX   | 65541    | 12  |

Ý nghĩa:  
- Session 130 (user SALES) đang chờ khóa transaction (TX) do session 123 (user HR) đang giữ.

## 4. Dùng V$SQL để xem chi tiết câu SQL

### Ví dụ 4: Tìm SQL theo SQL_ID

```sql
SELECT
    sql_id,
    executions,
    buffer_gets,
    rows_processed,

WHERE sql_id = '5t9k3u1a2b';
```

## 5. Kết hợp V$SESSION, V$LOCK, V$SQL để phân tích khóa & cô lập

Quy trình thường dùng khi điều tra session bị treo/chờ:

1. **Xác định session bị chờ**  
   - Từ ứng dụng hoặc `V$SESSION` (STATUS = 'ACTIVE', EVENT = 'enq: TX - row lock contention', v.v.)

2. **Dùng V$LOCK để tìm session đang giữ khóa**  
   - Truy vấn join V$LOCK như ví dụ 3.

3. **Dùng V$SESSION + V$SQL để xem câu SQL của cả session chờ và session giữ khóa**  
   - Lấy `SQL_ID` từ V$SESSION, join với V$SQL để xem `SQL_TEXT`.

### Ví dụ 5: Truy vấn tổng hợp (session chờ + session giữ khóa + SQL)

```sql
WITH lock_info AS (
    SELECT
        l1.sid  AS waiting_sid,
        l2.sid  AS holding_sid,
        l1.id1,
        l1.id2,
        l1.type
    FROM v$lock l1
    JOIN v$lock l2
      ON l1.id1 = l2.id1
     AND l1.id2 = l2.id2
     AND l1.type = l2.type
    WHERE l1.request > 0
      AND l2.lmode  > 0
      AND l1.sid   != l2.sid
)
SELECT
    lw.waiting_sid,
    sw.username      AS waiting_user,
    sw.sql_id        AS waiting_sql_id,
    qw.sql_text      AS waiting_sql_text,
    lh.holding_sid,
    sh.username      AS holding_user,
    sh.sql_id        AS holding_sql_id,
    qh.sql_text      AS holding_sql_text,
    lw.type,
    lw.id1,
    lw.id2
FROM lock_info lw
JOIN v$session sw ON lw.waiting_sid = sw.sid
JOIN v$session sh ON lw.holding_sid = sh.sid
LEFT JOIN v$sql qw ON sw.sql_id = qw.sql_id
LEFT JOIN v$sql qh ON sh.sql_id = qh.sql_id;
```

**Kết quả mẫu (rút gọn):**

| WAITING_SID | WAITING_USER | WAITING_SQL_TEXT                      | HOLDING_SID | HOLDING_USER | HOLDING_SQL_TEXT                         | TYPE | ID1   |
|-------------|--------------|----------------------------------------|-------------|--------------|-------------------------------------------|------|-------|
| 130         | SALES        | UPDATE orders SET status='CLOSED' ... | 123         | HR           | UPDATE orders SET status='OPEN' ...      | TX   | 65541 |

## 6. Isolation level trong Oracle (ngắn gọn)

Thiết lập mức cô lập cho session:

```sql
-- READ COMMITTED (mặc định)
ALTER SESSION SET ISOLATION_LEVEL = READ COMMITTED;

-- SERIALIZABLE
ALTER SESSION SET ISOLATION_LEVEL = SERIALIZABLE;

-- READ ONLY
ALTER SESSION SET ISOLATION_LEVEL = READ ONLY;
```

- **READ COMMITTED**:  
  - Không đọc dữ liệu chưa commit.  
  - Mỗi câu SELECT thấy dữ liệu mới nhất đã commit tại thời điểm bắt đầu câu SELECT.

- **SERIALIZABLE**:  
  - Giao dịch hoạt động như chụp snapshot tại thời điểm bắt đầu transaction.  
  - Có thể bị lỗi ORA-08177 nếu xung đột với transaction khác.

---

Tóm lại:  
- **V$SESSION**: biết session nào đang làm gì, đang chạy SQL nào.  
- **V$LOCK**: biết session nào giữ khóa, session nào đang chờ.  
- **V$SQL**: xem chi tiết câu SQL gây ra khóa hoặc bị khóa.  
Kết hợp 3 view này giúp phân tích vấn đề locking & isolation trong Oracle một cách hiệu quả.
