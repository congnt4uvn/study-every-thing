
Dưới đây là tóm tắt ngắn gọn về **Oracle Advanced Queuing (AQ)** trong PL/SQL & SQL, kèm ví dụ mã và input/output minh họa.

---

## 1. Khái niệm chính

- **Advanced Queuing (AQ)** là tính năng của Oracle dùng để:
  - Gửi/nhận message giữa các session, ứng dụng, hoặc hệ thống khác nhau.
  - Hỗ trợ xử lý bất đồng bộ (asynchronous processing).
  - Đảm bảo độ tin cậy (reliable messaging) nhờ cơ chế transaction của Oracle.

- **Thành phần chính**:
  - **Queue table**: Bảng lưu trữ message.
  - **Queue**: Hàng đợi logic dựa trên queue table.
  - **Message**: Dữ liệu được gửi/nhận (có thể là object type, RAW, XML, v.v.).

---

## 2. Các bước cơ bản sử dụng AQ

1. Tạo **object type** cho payload (nội dung message).
2. Tạo **queue table** dùng object type đó.
3. Tạo **queue** dựa trên queue table.
4. **Start** queue.
5. **Enqueue** (gửi) message.
6. **Dequeue** (nhận) message.

---

## 3. Ví dụ cơ bản với PL/SQL

### 3.1. Tạo object type cho message

```sql
CREATE OR REPLACE TYPE msg_type AS OBJECT (
  id   NUMBER,
  text VARCHAR2(200)
);
/
```

### 3.2. Tạo queue table

```sql
BEGIN
  DBMS_AQADM.CREATE_QUEUE_TABLE(
    queue_table        => 'msg_queue_tab',
    queue_payload_type => 'msg_type'
  );
END;
/
```

### 3.3. Tạo queue

```sql
BEGIN
  DBMS_AQADM.CREATE_QUEUE(
    queue_name  => 'msg_queue',
    queue_table => 'msg_queue_tab'
  );
END;
/
```

### 3.4. Start queue

```sql
BEGIN
  DBMS_AQADM.START_QUEUE(
    queue_name => 'msg_queue'
  );
END;
/
```

---

## 4. Enqueue (Gửi message)

### PL/SQL code

```sql
DECLARE
  l_enqueue_options    DBMS_AQ.ENQUEUE_OPTIONS_T;
  l_message_properties DBMS_AQ.MESSAGE_PROPERTIES_T;
  l_message_handle     RAW(16);
  l_payload            msg_type;
BEGIN
  l_payload := msg_type(1, 'Xin chào từ AQ');

  DBMS_AQ.ENQUEUE(
    queue_name         => 'msg_queue',
    enqueue_options    => l_enqueue_options,
    message_properties => l_message_properties,
    payload            => l_payload,
    msgid              => l_message_handle
  );

  COMMIT;
END;
/
```

**Ý nghĩa input:**

- `l_payload.id = 1`
- `l_payload.text = 'Xin chào từ AQ'`

**Kết quả (output logic):**

- Một message với payload `(1, 'Xin chào từ AQ')` được lưu vào queue `msg_queue`.
- `msgid` (l_message_handle) là ID nội bộ của message (RAW(16)).

---

## 5. Dequeue (Nhận message)

### PL/SQL code

```sql
DECLARE
  l_dequeue_options    DBMS_AQ.DEQUEUE_OPTIONS_T;
  l_message_properties DBMS_AQ.MESSAGE_PROPERTIES_T;
  l_message_handle     RAW(16);
  l_payload            msg_type;
BEGIN
  -- Thiết lập chế độ đọc: lấy message tiếp theo
  l_dequeue_options.wait      := DBMS_AQ.NO_WAIT; -- hoặc số giây chờ
  l_dequeue_options.navigation := DBMS_AQ.FIRST_MESSAGE;

  DBMS_AQ.DEQUEUE(
    queue_name         => 'msg_queue',
    dequeue_options    => l_dequeue_options,
    message_properties => l_message_properties,
    payload            => l_payload,
    msgid              => l_message_handle
  );

  DBMS_OUTPUT.PUT_LINE('ID: '   || l_payload.id);
  DBMS_OUTPUT.PUT_LINE('TEXT: ' || l_payload.text);

  COMMIT;
END;
/
```

**Output (DBMS_OUTPUT):**

```text
ID: 1
TEXT: Xin chào từ AQ
```

---

## 6. Một số điểm cần nhớ

- AQ tích hợp với **transaction**:
  - `COMMIT` sẽ xác nhận enqueue/dequeue.
  - `ROLLBACK` sẽ hủy enqueue/dequeue.
- Có thể cấu hình:
  - **Priority** (độ ưu tiên message).
  - **Delay** (trì hoãn thời điểm message có thể được đọc).
  - **Expiration** (hết hạn message).
- Hỗ trợ nhiều mô hình:
  - Point-to-point (1 gửi – 1 nhận).
  - Publish/Subscribe (1 gửi – nhiều nhận).

---

Nếu bạn muốn, tôi có thể mở rộng thêm ví dụ:
- Dùng **priority**, **delay**, **expiration**.
- Dùng **multiple consumers** (nhiều subscriber).
