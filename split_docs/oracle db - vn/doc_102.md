
Dưới đây là tóm tắt ngắn gọn về ý tưởng “Version Control & CI/CD: Storing database code in Git” cho Oracle Database, kèm ví dụ.

---

## 1. Ý chính

1. **Lưu toàn bộ mã database vào Git**  
   - Lưu script tạo schema, bảng, view, index, sequence, package, procedure, function, trigger, job…  
   - Lưu cả script migration (ALTER TABLE, thêm cột, đổi kiểu dữ liệu…).

2. **Đối xử database như source code**  
   - Mỗi thay đổi DB đều phải có commit, message rõ ràng.  
   - Có branch, pull request, code review cho thay đổi DB.  
   - Có thể rollback, xem lịch sử, so sánh phiên bản.

3. **Tự động hóa với CI/CD**  
   - Khi có commit/merge vào branch chính, pipeline CI/CD sẽ:
     - Kiểm tra cú pháp SQL/PLSQL.
     - Chạy unit test (PL/SQL unit test, SQL test).
     - Deploy script lên môi trường DEV/TEST/PROD theo thứ tự.

4. **Lợi ích chính**  
   - Truy vết được ai thay đổi gì, khi nào, tại sao.  
   - Giảm lỗi do deploy thủ công.  
   - Đồng bộ giữa code ứng dụng và schema database.  
   - Dễ dàng tạo môi trường mới (clone schema từ Git).

## 2. Cấu trúc thư mục ví dụ trong Git

```text
db/
  schema/
    tables/
      customers.sql
      orders.sql
    views/
      v_customer_orders.sql
    packages/
      pkg_order_mgmt.pks   -- spec
      pkg_order_mgmt.pkb   -- body
    triggers/
      trg_orders_bi.sql
  migrations/
    001_add_customers_table.sql
    002_add_orders_table.sql
    003_add_status_to_orders.sql
  seed/
    seed_customers.sql
    seed_orders.sql
```

## 3. Ví dụ script database lưu trong Git

### 3.1. Tạo bảng (schema script)

`db/schema/tables/customers.sql`:

```sql
CREATE TABLE customers (
    customer_id   NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    full_name     VARCHAR2(200) NOT NULL,
    email         VARCHAR2(200),
    created_at    DATE DEFAULT SYSDATE
);
```

### 3.2. Migration: thêm cột mới

`db/migrations/003_add_status_to_orders.sql`:

```sql
ALTER TABLE orders
  ADD (status VARCHAR2(20) DEFAULT 'NEW' NOT NULL);
```

**Input**: chạy script trên DB đã có bảng `orders`.  
**Output**: bảng `orders` có thêm cột `status` với default `'NEW'`.

## 4. Ví dụ CI/CD đơn giản (GitHub Actions)

`.github/workflows/db-ci.yml`:

```yaml
name: Oracle DB CI

on:
  push:
    branches: [ main ]

jobs:
  db-ci:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repo
        uses: actions/checkout@v4

      - name: Install SQL*Plus client
        run: |
          sudo apt-get update
          sudo apt-get install -y sqlplus

      - name: Run migrations
        env:
          ORA_USER: ${{ secrets.ORA_USER }}
          ORA_PASS: ${{ secrets.ORA_PASS }}
          ORA_CONN: ${{ secrets.ORA_CONN }} # ví dụ: host:port/service
        run: |
          for f in db/migrations/*.sql; do
            echo "Running $f"
            sqlplus -s "$ORA_USER/$ORA_PASS@$ORA_CONN" @"$f"
          done
```

**Input**: commit mới vào branch `main` với file SQL trong `db/migrations`.  
**Output**: GitHub Actions tự động chạy các script migration lên database mục tiêu.

---

## 5. Quy trình làm việc mẫu

1. Dev tạo branch mới: `feature/add-discount-to-orders`.  
2. Thêm file migration: `004_add_discount_to_orders.sql`.  
3. Commit & push, tạo Pull Request.  
4. CI chạy: kiểm tra, chạy test trên DB test.  
5. Review & merge vào `main`.  
6. Pipeline CD deploy script lên môi trường STG/PROD.

---

Nếu bạn muốn, tôi có thể viết thêm ví dụ cụ thể cho:
- Flyway/Liquibase quản lý migration Oracle.
- Script rollback (down migration).  
- Unit test PL/SQL (UTPLSQL).
