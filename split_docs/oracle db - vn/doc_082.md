
Dưới đây là tóm tắt ngắn gọn về cách xử lý hai lỗi phổ biến trong Oracle: `ORA-00054` và `ORA-00060`, kèm ví dụ.

---

## 1. ORA-00054: resource busy and acquire with NOWAIT specified or timeout expired

**Ý nghĩa**  
Phiên của bạn muốn khóa một tài nguyên (bảng, hàng, …) nhưng tài nguyên đang bị khóa bởi phiên khác, và bạn dùng `NOWAIT` hoặc đã hết thời gian chờ.

**Tình huống thường gặp**
- `ALTER TABLE ...` khi bảng đang bị session khác dùng.
- `DROP TABLE ...` khi có session khác đang truy cập.
- `UPDATE/DELETE ... FOR UPDATE NOWAIT`.

### Cách xử lý

1. **Tìm session đang khóa tài nguyên**

```sql
-- Tìm các khóa hiện tại
SELECT
    s.sid,
    s.serial#,
    s.username,
    l.type,
    l.id1,
    l.id2,
    l.lmode,
    l.request,
    l.block
FROM v$lock l
JOIN v$session s ON l.sid = s.sid
ORDER BY l.block DESC, s.sid;
```

**Output (ví dụ)**

| SID | SERIAL# | USERNAME | TYPE | ID1  | ID2 | LMODE | REQUEST | BLOCK |
|-----|---------|----------|------|------|-----|-------|---------|-------|
| 123 | 45678   | APPUSER  | TX   | 1234 | 0   | 6     | 0       | 1     |

- `BLOCK = 1`: session đang chặn (blocking).
- `LMODE = 6`: khóa exclusive.

2. **Xác định SQL đang chạy ở session đó**

```sql
SELECT
    s.sid,
    s.serial#,
    s.username,
    q.sql_text
FROM v$session s
JOIN v$sql q ON s.sql_id = q.sql_id
WHERE s.sid = 123;  -- thay bằng SID tìm được
```

3. **Giải phóng khóa**

- Cách tốt: yêu cầu người dùng/ứng dụng commit hoặc rollback.
- Cách mạnh tay (DBA): kill session đang khóa:

```sql
ALTER SYSTEM KILL SESSION '123,45678' IMMEDIATE;
```

4. **Thêm cơ chế chờ (không dùng NOWAIT)**

Ví dụ: thay vì:

```sql
SELECT * FROM employees FOR UPDATE NOWAIT;
```

Dùng:

```sql
SELECT * FROM employees FOR UPDATE WAIT 10;  -- chờ tối đa 10 giây
```

Hoặc trong PL/SQL, tự retry:

```plsql
DECLARE
    l_retry_count PLS_INTEGER := 5;
    l_sleep_sec   PLS_INTEGER := 2;
BEGIN
    FOR i IN 1 .. l_retry_count LOOP
        BEGIN
            UPDATE employees
               SET salary = salary * 1.1
             WHERE department_id = 10;
            COMMIT;
            EXIT; -- thành công thì thoát vòng lặp
        EXCEPTION
            WHEN OTHERS THEN
                IF SQLCODE = -54 THEN
                    DBMS_LOCK.SLEEP(l_sleep_sec); -- chờ rồi thử lại
                ELSE
                    RAISE;
                END IF;
        END;
    END LOOP;
END;
/
```

## 2. ORA-00060: deadlock detected while waiting for resource

**Ý nghĩa**  
Hai (hoặc nhiều) session khóa chéo lẫn nhau, không ai tiếp tục được. Oracle phát hiện deadlock và rollback một trong các câu lệnh, báo lỗi `ORA-00060`.

**Ví dụ đơn giản**

- Session 1:

```sql
-- Session 1
UPDATE employees SET salary = salary + 100 WHERE employee_id = 1;  -- khóa row 1
-- chưa COMMIT
```

- Session 2:

```sql
-- Session 2
UPDATE employees SET salary = salary + 200 WHERE employee_id = 2;  -- khóa row 2
-- chưa COMMIT
```

- Sau đó:

```sql
-- Session 1
UPDATE employees SET salary = salary + 100 WHERE employee_id = 2;  -- chờ row 2

-- Session 2
UPDATE employees SET salary = salary + 200 WHERE employee_id = 1;  -- chờ row 1
```

Hai session chờ nhau → deadlock → Oracle báo `ORA-00060` cho một session, rollback câu lệnh đó.

### Cách xử lý

1. **Xem trace file / alert log**

Oracle ghi chi tiết deadlock vào trace file (thường trong `user_dump_dest` hoặc `DIAGNOSTIC_DEST`).  
Trong trace có:

- Các session liên quan.
- SQL gây deadlock.
- Loại khóa, đối tượng bị khóa.

2. **Phân tích và sửa logic ứng dụng**

Mục tiêu: tránh tình huống khóa chéo.

Các nguyên tắc:

- **Khóa theo thứ tự cố định**  
  Ví dụ: luôn update bảng A trước rồi mới bảng B, hoặc luôn update theo thứ tự ID tăng dần.

  ```sql
  -- Luôn update theo thứ tự ID tăng dần
  UPDATE employees
     SET salary = salary + 100
   WHERE employee_id IN (1, 2)
   ORDER BY employee_id;
  ```

- **Giữ transaction ngắn**  
  Commit/rollback sớm, tránh giữ khóa lâu.

- **Tránh “đọc rồi update” không nhất quán**  
  Nếu cần, dùng `SELECT ... FOR UPDATE` trên tất cả các dòng sẽ update, theo cùng một thứ tự ở mọi nơi.

3. **Thêm xử lý lỗi trong ứng dụng**

Ví dụ PL/SQL:

```plsql
BEGIN
    UPDATE employees
       SET salary = salary + 100
     WHERE employee_id = 1;
    COMMIT;
EXCEPTION
    WHEN OTHERS THEN
        IF SQLCODE = -60 THEN
            -- xử lý riêng cho deadlock: log, retry, báo lỗi thân thiện...
            DBMS_OUTPUT.PUT_LINE('Deadlock xảy ra, vui lòng thử lại.');
        ELSE
            RAISE;
        END IF;
END;
/
```

---

## Tóm tắt ngắn

- `ORA-00054`: tài nguyên đang bị khóa, bạn không lấy được khóa (do NOWAIT hoặc timeout).
  - Tìm session đang khóa (`v$lock`, `v$session`), yêu cầu commit/rollback hoặc kill session.
  - Thiết kế lại: giảm thời gian giữ khóa, dùng `WAIT`, hoặc cơ chế retry.

- `ORA-00060`: deadlock (khóa chéo).
  - Xem trace file để biết SQL và session liên quan.
  - Sửa logic ứng dụng: khóa theo thứ tự cố định, transaction ngắn, dùng `SELECT FOR UPDATE` hợp lý.
  - Thêm xử lý lỗi (retry, thông báo người dùng).
