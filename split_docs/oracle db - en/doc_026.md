
Here’s a concise summary of the key ideas behind **FORALL** and **BULK COLLECT** in Advanced PL/SQL, with code examples and sample input/output.

---

## 1. What are FORALL and BULK COLLECT?

- **BULK COLLECT**: Fetches many rows from SQL into PL/SQL collections in a single operation (reduces context switches between SQL and PL/SQL).
- **FORALL**: Sends many DML operations (INSERT/UPDATE/DELETE/MERGE) from PL/SQL to SQL in a single operation using collections.

They are used together to improve performance when processing large volumes of data.

---

## 2. BULK COLLECT – Fetching Data in Bulk

### Basic pattern

```plsql
DECLARE
  TYPE t_emp_tab IS TABLE OF employees%ROWTYPE;
  l_emps t_emp_tab;
BEGIN
  SELECT *
  BULK COLLECT INTO l_emps
  FROM employees
  WHERE department_id = 10;

  DBMS_OUTPUT.PUT_LINE('Rows fetched: ' || l_emps.COUNT);
END;
/
```

**Input**: `employees` table with many rows for `department_id = 10`  
**Output (example)**:
```text
Rows fetched: 25
```

### BULK COLLECT with LIMIT (for large data sets)

```plsql
DECLARE
  TYPE t_emp_tab IS TABLE OF employees%ROWTYPE;
  l_emps t_emp_tab;
  l_limit PLS_INTEGER := 100;
BEGIN
  LOOP
    FETCH c_emp BULK COLLECT INTO l_emps LIMIT l_limit;

    EXIT WHEN l_emps.COUNT = 0;

    -- Process each batch
    FOR i IN 1 .. l_emps.COUNT LOOP
      DBMS_OUTPUT.PUT_LINE('Emp: ' || l_emps(i).employee_id);
    END LOOP;
  END LOOP;
END;
/
```

(Assumes a cursor `c_emp` is declared and opened earlier.)

## 3. FORALL – DML in Bulk

### Basic INSERT with FORALL

```plsql
DECLARE
  TYPE t_emp_id_tab IS TABLE OF employees.employee_id%TYPE;
  TYPE t_sal_tab    IS TABLE OF employees.salary%TYPE;

  l_emp_ids t_emp_id_tab := t_emp_id_tab(101, 102, 103);
  l_sals    t_sal_tab    := t_sal_tab(5000, 6000, 7000);
BEGIN
  FORALL i IN l_emp_ids.FIRST .. l_emp_ids.LAST
    INSERT INTO bonus_log (employee_id, bonus_amount)
    VALUES (l_emp_ids(i), l_sals(i) * 0.1);

  DBMS_OUTPUT.PUT_LINE('Rows inserted: ' || SQL%ROWCOUNT);
END;
/
```

**Input**:  
- `l_emp_ids = [101, 102, 103]`  
- `l_sals = [5000, 6000, 7000]`

**Output (effect)**:  
`bonus_log` table gets 3 new rows:
```text
(101, 500)
(102, 600)
(103, 700)
```
**DBMS_OUTPUT**:
```text
Rows inserted: 3
```

## 4. Combining BULK COLLECT and FORALL

Typical pattern: bulk fetch → bulk DML.

```plsql
DECLARE
  TYPE t_emp_tab IS TABLE OF employees%ROWTYPE;
  l_emps t_emp_tab;
BEGIN
  -- 1) Bulk fetch
  SELECT *
  BULK COLLECT INTO l_emps
  FROM employees
  WHERE department_id = 10;

  -- 2) Bulk update
  FORALL i IN l_emps.FIRST .. l_emps.LAST
    UPDATE employees
    SET salary = salary * 1.05
    WHERE employee_id = l_emps(i).employee_id;

  DBMS_OUTPUT.PUT_LINE('Rows updated: ' || SQL%ROWCOUNT);
END;
/
```

**Input**: All employees in department 10.  
**Output (effect)**: Their salaries are increased by 5%.  
**DBMS_OUTPUT**:
```text
Rows updated: <number_of_rows_in_dept_10>
```

## 5. FORALL with SAVE EXCEPTIONS

Allows partial success: some rows can fail while others succeed.

```plsql
DECLARE
  TYPE t_emp_id_tab IS TABLE OF employees.employee_id%TYPE;
  l_emp_ids t_emp_id_tab := t_emp_id_tab(101, 999999, 103); -- 999999 will fail

  e_bulk_errors EXCEPTION;
  PRAGMA EXCEPTION_INIT(e_bulk_errors, -24381);
BEGIN
  FORALL i IN l_emp_ids.FIRST .. l_emp_ids.LAST SAVE EXCEPTIONS
    DELETE FROM employees
    WHERE employee_id = l_emp_ids(i);

  DBMS_OUTPUT.PUT_LINE('Rows deleted: ' || SQL%ROWCOUNT);

EXCEPTION
  WHEN e_bulk_errors THEN
    DBMS_OUTPUT.PUT_LINE('Errors: ' || SQL%BULK_EXCEPTIONS.COUNT);
    FOR j IN 1 .. SQL%BULK_EXCEPTIONS.COUNT LOOP
      DBMS_OUTPUT.PUT_LINE(
        'Index: ' || SQL%BULK_EXCEPTIONS(j).ERROR_INDEX ||
        ' Error code: ' || SQL%BULK_EXCEPTIONS(j).ERROR_CODE
      );
    END LOOP;
END;
/
```

**Input**: IDs `[101, 999999, 103]`  
**Output (effect)**:  
- Rows for 101 and 103 deleted (if they exist).  
- Error for 999999 captured in `SQL%BULK_EXCEPTIONS`.

Sample **DBMS_OUTPUT**:
```text
Rows deleted: 2
Errors: 1
Index: 2 Error code: -2292
```

---

## 6. Key Takeaways

- Use **BULK COLLECT** to fetch many rows at once into collections.
- Use **FORALL** to perform many DML operations in one call.
- Combine them for high-performance batch processing.
- Use `LIMIT` with BULK COLLECT for very large data sets.
- Use `SAVE EXCEPTIONS` with FORALL to handle row-level errors without aborting the whole batch.
