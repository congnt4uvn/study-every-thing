
Here’s a concise summary of **Advanced PL/SQL: Dynamic SQL** with examples.

---

## 1. What is Dynamic SQL?

Dynamic SQL is SQL whose text is built and executed at runtime, rather than being hard-coded in the PL/SQL block.

You use it when:
- The table name, column list, or WHERE clause is not known until runtime.
- You need to execute DDL (CREATE, ALTER, DROP) or DCL (GRANT, REVOKE) from PL/SQL.
- You need very flexible queries based on user input or configuration.

---

## 2. Main Ways to Use Dynamic SQL in PL/SQL

### 2.1 `EXECUTE IMMEDIATE` (Native Dynamic SQL)

Most common and recommended for Oracle 9i+.

**Basic syntax:**

```plsql
EXECUTE IMMEDIATE dynamic_sql_string;
```

#### Example 1: DDL statement

```plsql
DECLARE
  v_sql  VARCHAR2(2000);
BEGIN
  v_sql := 'CREATE TABLE test_dyn (id NUMBER, name VARCHAR2(50))';
  EXECUTE IMMEDIATE v_sql;
END;
/
```

No direct “output” from DDL, but the table is created.

---

#### Example 2: Dynamic INSERT with bind variables

```plsql
DECLARE
  v_sql   VARCHAR2(2000);
  v_id    NUMBER := 10;
  v_name  VARCHAR2(50) := 'Scott';
BEGIN
  v_sql := 'INSERT INTO emp_temp (empno, ename) VALUES (:1, :2)';
  EXECUTE IMMEDIATE v_sql USING v_id, v_name;
END;
/
```

- Input: `v_id = 10`, `v_name = 'Scott'`
- Effect: Row inserted into `emp_temp`.

---

#### Example 3: Dynamic SELECT … INTO

```plsql
DECLARE
  v_sql   VARCHAR2(2000);
  v_empno emp.empno%TYPE := 7369;
  v_name  emp.ename%TYPE;
BEGIN
  v_sql := 'SELECT ename FROM emp WHERE empno = :1';
  EXECUTE IMMEDIATE v_sql INTO v_name USING v_empno;

  DBMS_OUTPUT.PUT_LINE('Employee name: ' || v_name);
END;
/
```

- Input: `v_empno = 7369`
- Output (example): `Employee name: SMITH`

---

### 2.2 Dynamic SQL with Returning Clause

You can get generated values (e.g., from sequences) back:

```plsql
DECLARE
  v_sql   VARCHAR2(2000);
  v_id    NUMBER;
BEGIN
  v_sql := 'INSERT INTO orders (order_id, customer_id) ' ||
           'VALUES (orders_seq.NEXTVAL, :1) ' ||
           'RETURNING order_id INTO :2';

  EXECUTE IMMEDIATE v_sql
    USING 12345
    RETURNING INTO v_id;

  DBMS_OUTPUT.PUT_LINE('New order_id: ' || v_id);
END;
/
```

- Input: `customer_id = 12345`
- Output: `New order_id: <generated value>`

---

### 2.3 Dynamic SQL with BULK COLLECT

For performance with multi-row queries:

```plsql
DECLARE
  TYPE t_emp_tab IS TABLE OF emp%ROWTYPE;
  v_emps  t_emp_tab;
  v_sql   VARCHAR2(2000);
BEGIN
  v_sql := 'SELECT * FROM emp WHERE deptno = :1';

  EXECUTE IMMEDIATE v_sql
    BULK COLLECT INTO v_emps
    USING 10;

  FOR i IN 1 .. v_emps.COUNT LOOP
    DBMS_OUTPUT.PUT_LINE(v_emps(i).empno || ' - ' || v_emps(i).ename);
  END LOOP;
END;
/
```

- Input: `deptno = 10`
- Output: List of employees in department 10.

## 3. Dynamic Cursors (REF CURSOR)

Use when you want to return a result set to the caller.

```plsql
DECLARE
  TYPE ref_cur IS REF CURSOR;
  c_emp   ref_cur;
  v_sql   VARCHAR2(2000);
  v_empno emp.empno%TYPE;
  v_ename emp.ename%TYPE;
BEGIN
  v_sql := 'SELECT empno, ename FROM emp WHERE deptno = :1';

  OPEN c_emp FOR v_sql USING 20;

  LOOP
    FETCH c_emp INTO v_empno, v_ename;
    EXIT WHEN c_emp%NOTFOUND;
    DBMS_OUTPUT.PUT_LINE(v_empno || ' - ' || v_ename);
  END LOOP;

  CLOSE c_emp;
END;
/
```

- Input: `deptno = 20`
- Output: Rows (empno, ename) for dept 20.

## 4. DBMS_SQL Package (Older, More Flexible)

Used when:
- Number or types of columns are unknown at compile time.
- You need very generic SQL processing.

Basic steps:
1. `DBMS_SQL.OPEN_CURSOR`
2. `DBMS_SQL.PARSE`
3. `DBMS_SQL.BIND_VARIABLE` (if needed)
4. `DBMS_SQL.DEFINE_COLUMN`
5. `DBMS_SQL.EXECUTE`
6. `DBMS_SQL.FETCH_ROWS`
7. `DBMS_SQL.COLUMN_VALUE`
8. `DBMS_SQL.CLOSE_CURSOR`

**Example: Generic SELECT with unknown columns**

```plsql
DECLARE
  v_cursor   INTEGER;
  v_sql      VARCHAR2(2000) := 'SELECT empno, ename FROM emp WHERE deptno = :dept';
  v_empno    NUMBER;
  v_ename    VARCHAR2(100);
  v_rows     INTEGER;
BEGIN
  v_cursor := DBMS_SQL.OPEN_CURSOR;

  DBMS_SQL.PARSE(v_cursor, v_sql, DBMS_SQL.NATIVE);
  DBMS_SQL.BIND_VARIABLE(v_cursor, ':dept', 30);

  DBMS_SQL.DEFINE_COLUMN(v_cursor, 1, v_empno);
  DBMS_SQL.DEFINE_COLUMN(v_cursor, 2, v_ename, 100);

  v_rows := DBMS_SQL.EXECUTE(v_cursor);

  LOOP
    EXIT WHEN DBMS_SQL.FETCH_ROWS(v_cursor) = 0;
    DBMS_SQL.COLUMN_VALUE(v_cursor, 1, v_empno);
    DBMS_SQL.COLUMN_VALUE(v_cursor, 2, v_ename);
    DBMS_OUTPUT.PUT_LINE(v_empno || ' - ' || v_ename);
  END LOOP;

  DBMS_SQL.CLOSE_CURSOR(v_cursor);
END;
/
```

---

## 5. Key Best Practices

1. **Use bind variables** (`:1`, `:name`) to:
   - Avoid SQL injection.
   - Improve performance (cursor sharing).

2. **Validate inputs** before building SQL strings, especially when including table/column names.

3. Prefer **`EXECUTE IMMEDIATE`** for most cases; use **`DBMS_SQL`** only when you need its extra flexibility.

4. Avoid dynamic SQL when static SQL is sufficient; static SQL is safer and faster.
