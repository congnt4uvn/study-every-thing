
Here’s a concise summary of the key PL/SQL basics, with code examples and sample inputs/outputs.

---

## 1. Anonymous Blocks

**What they are:**  
PL/SQL code that is not stored in the database; you write it, run it, and it’s gone. Used for quick scripts, tests, or one-time tasks.

**Structure:**
```plsql
DECLARE
    -- Declarations (optional)
    v_message VARCHAR2(50);
BEGIN
    -- Executable statements
    v_message := 'Hello PL/SQL';
    DBMS_OUTPUT.PUT_LINE(v_message);
EXCEPTION
    -- Exception handlers (optional)
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('Error: ' || SQLERRM);
END;
/
```

**Input:** (none; values are hard-coded)  
**Output (via `DBMS_OUTPUT`):**
```text
Hello PL/SQL
```

---

## 2. Procedures

**What they are:**  
Named PL/SQL blocks stored in the database. They perform actions (e.g., insert, update, logging). They can have parameters but do not return a value directly (no `RETURN` in the header).

**Basic syntax:**
```plsql
CREATE OR REPLACE PROCEDURE procedure_name (
    p_param1 IN  NUMBER,
    p_param2 OUT VARCHAR2
) AS
BEGIN
    -- Do something
    p_param2 := 'Result is ' || p_param1;
END;
/
```

**Example: Procedure to give a salary raise**

```plsql
CREATE OR REPLACE PROCEDURE give_raise (
    p_emp_id   IN  employees.employee_id%TYPE,
    p_percent  IN  NUMBER
) AS
BEGIN
    UPDATE employees
       SET salary = salary * (1 + p_percent / 100)
     WHERE employee_id = p_emp_id;

    DBMS_OUTPUT.PUT_LINE('Raise applied to employee ' || p_emp_id);
EXCEPTION
    WHEN NO_DATA_FOUND THEN
        DBMS_OUTPUT.PUT_LINE('Employee not found: ' || p_emp_id);
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('Error: ' || SQLERRM);
END;
/
```

**Calling the procedure from an anonymous block:**
```plsql
BEGIN
    give_raise(p_emp_id => 100, p_percent => 10);
END;
/
```

**Input:** `p_emp_id = 100`, `p_percent = 10`  
**Output (DBMS_OUTPUT):**
```text
Raise applied to employee 100
```
**Effect:** Employee 100’s salary is increased by 10%.

## 3. Functions

**What they are:**  
Named PL/SQL blocks that return a single value. Often used in SQL queries or other PL/SQL code.

**Basic syntax:**
```plsql
CREATE OR REPLACE FUNCTION function_name (
    p_param1 IN NUMBER
) RETURN NUMBER AS
    v_result NUMBER;
BEGIN
    -- Compute result
    v_result := p_param1 * 2;
    RETURN v_result;
EXCEPTION
    WHEN OTHERS THEN
        RETURN NULL;  -- or handle error appropriately
END;
/
```

**Example: Function to get annual salary**

```plsql
CREATE OR REPLACE FUNCTION get_annual_salary (
    p_emp_id IN employees.employee_id%TYPE
) RETURN NUMBER AS
    v_salary employees.salary%TYPE;
BEGIN
    SELECT salary
      INTO v_salary
      FROM employees
     WHERE employee_id = p_emp_id;

    RETURN v_salary * 12;
EXCEPTION
    WHEN NO_DATA_FOUND THEN
        RETURN NULL;
END;
/
```

**Calling the function in PL/SQL:**
```plsql
DECLARE
    v_annual NUMBER;
BEGIN
    v_annual := get_annual_salary(100);
    DBMS_OUTPUT.PUT_LINE('Annual salary: ' || v_annual);
END;
/
```

**Input:** `p_emp_id = 100`  
**Output (DBMS_OUTPUT):**
```text
Annual salary: 72000   -- example, if monthly salary is 6000
```

**Calling the function in SQL:**
```sql
SELECT employee_id,
       get_annual_salary(employee_id) AS annual_salary
  FROM employees
 WHERE employee_id = 100;
```

**Sample result:**
```text
EMPLOYEE_ID  ANNUAL_SALARY
-----------  -------------
100          72000
```

---

## 4. Key Differences

- **Anonymous block**
  - Not stored in DB
  - Used once, ad hoc
  - No name, cannot be reused directly

- **Procedure**
  - Stored, named program
  - Performs actions
  - Can have IN/OUT/IN OUT parameters
  - Does not return a value in the header (uses OUT params or side effects)

- **Function**
  - Stored, named program
  - Returns a single value (via `RETURN`)
  - Can be used in SQL (with restrictions) and PL/SQL
  - Typically has only IN parameters
