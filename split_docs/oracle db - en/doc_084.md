
Content summary (Oracle Database context):

- The statement is advising that you **don’t need to learn every single feature or tool** in the Oracle ecosystem.
- Instead, you should **focus on understanding well the features you have actually used** in practice.
- This is often relevant for:
  - Job interviews (talk in depth about what you’ve really used).
  - Certifications or exams (prioritize topics you’ve worked with).
  - Daily work (master the subset of Oracle features that matter for your role).

There’s no direct technical content to extract code from, but here are some **example areas** where you might “know the ones you’ve used” in Oracle, with simple code samples.

---

### 1. Basic SELECT (if you’ve used querying)

**Input (SQL):**
```sql
SELECT employee_id,
       first_name,
       last_name,
       salary
FROM   employees
WHERE  department_id = 10;
```

**Output (example result):**

| EMPLOYEE_ID | FIRST_NAME | LAST_NAME | SALARY |
|-------------|------------|-----------|--------|
| 100         | Steven     | King      | 24000  |
| 101         | Neena      | Kochhar   | 17000  |

Key idea: If you’ve used basic querying, be ready to explain `SELECT`, `FROM`, `WHERE`, and simple filters.

---

### 2. Creating a Table (if you’ve done DDL)

**Input (SQL):**
```sql
CREATE TABLE employees_audit (
    audit_id      NUMBER GENERATED BY DEFAULT AS IDENTITY,
    employee_id   NUMBER NOT NULL,
    action        VARCHAR2(20),
    action_date   DATE DEFAULT SYSDATE,
    CONSTRAINT employees_audit_pk PRIMARY KEY (audit_id)
);
```

**Output:**
- A new table `EMPLOYEES_AUDIT` is created in the schema.
- You can verify with:
  ```sql
  SELECT table_name
  FROM   user_tables
  WHERE  table_name = 'EMPLOYEES_AUDIT';
  ```

Key idea: If you’ve used DDL, know how to explain `CREATE TABLE`, constraints, and data types.

---

### 3. PL/SQL Block (if you’ve used PL/SQL)

**Input (PL/SQL):**
```sql
DECLARE
    v_count NUMBER;
BEGIN
    SELECT COUNT(*)
    INTO   v_count
    FROM   employees
    WHERE  department_id = 10;

    DBMS_OUTPUT.PUT_LINE('Employees in department 10: ' || v_count);
END;
/
```

**Output (DBMS_OUTPUT):**
```text
Employees in department 10: 2
```

Key idea: If you’ve used PL/SQL, be ready to explain anonymous blocks, variables, `SELECT INTO`, and `DBMS_OUTPUT`.

---

### 4. Index Creation (if you’ve worked with performance)

**Input (SQL):**
```sql
CREATE INDEX emp_dept_idx
ON employees (department_id);
```

**Output:**
- An index `EMP_DEPT_IDX` is created.
- You can check:
  ```sql
  SELECT index_name, table_name
  FROM   user_indexes
  WHERE  index_name = 'EMP_DEPT_IDX';
  ```

Key idea: If you’ve used indexes, know why and when you created them.

---

**Main takeaway:**  
In Oracle Database, you’re not expected to know every feature. Instead, be able to clearly explain and demonstrate the **specific SQL, PL/SQL, and database features you have actually used**, with concrete examples like the ones above.
