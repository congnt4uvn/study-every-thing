## Key ideas: Advanced PL/SQL – Using Packages

1. **What a package is**
   - A package is a named group of related PL/SQL elements:
     - Procedures, functions
     - Variables, constants, types, cursors
   - It has two parts:
     - **Specification (spec)**: public interface (what is visible to other schemas/code).
     - **Body**: implementation (code and private items).

2. **Why use packages**
   - **Encapsulation**: Hide implementation details; expose only what callers need.
   - **Modularity**: Group related logic (e.g., `employee_api`, `order_mgmt_pkg`).
   - **Performance**:
     - Package code is loaded into memory once per session.
     - Reduces parsing and improves execution speed.
   - **State management**:
     - Package-level variables can hold state across calls in the same session.
   - **Security & API design**:
     - Expose only safe procedures/functions.
     - Hide tables and internal logic behind a stable API.

3. **Package spec vs body**
   - **Spec**:
     - Declares public procedures, functions, types, variables.
     - Acts as a contract for callers.
   - **Body**:
     - Implements the spec.
     - Can define private procedures/functions/variables not visible outside.

4. **Advanced uses**
   - **Initialization**: Package body can have an initialization section that runs once per session.
   - **Overloading**: Multiple procedures/functions with the same name but different parameters.
   - **Persistent session state**: Use package variables to cache data or maintain counters.
   - **Private helper routines**: Keep internal logic hidden and changeable without affecting callers.

## Example 1: Basic package with procedure and function

### 1. Package specification

```sql
CREATE OR REPLACE PACKAGE math_utils_pkg AS
  -- Public function
  FUNCTION add_numbers(p_a NUMBER, p_b NUMBER) RETURN NUMBER;

  -- Public procedure
  PROCEDURE print_sum(p_a NUMBER, p_b NUMBER);
END math_utils_pkg;
/
```

### 2. Package body

```sql
CREATE OR REPLACE PACKAGE BODY math_utils_pkg AS

  FUNCTION add_numbers(p_a NUMBER, p_b NUMBER) RETURN NUMBER IS
  BEGIN
    RETURN p_a + p_b;
  END add_numbers;

  PROCEDURE print_sum(p_a NUMBER, p_b NUMBER) IS
    l_sum NUMBER;
  BEGIN
    l_sum := add_numbers(p_a, p_b);
    DBMS_OUTPUT.PUT_LINE('Sum is: ' || l_sum);
  END print_sum;

END math_utils_pkg;
/
```

### 3. Example usage (input/output)

```sql
SET SERVEROUTPUT ON;

BEGIN
  DBMS_OUTPUT.PUT_LINE('Result of add_numbers(2, 3): ' ||
                       math_utils_pkg.add_numbers(2, 3));
  math_utils_pkg.print_sum(10, 15);
END;
/
```

**Output:**

```text
Result of add_numbers(2, 3): 5
Sum is: 25
```

## Example 2: Package with state (session-level counter)

### 1. Package specification

```sql
CREATE OR REPLACE PACKAGE session_counter_pkg AS
  PROCEDURE increment_counter;
  FUNCTION get_counter RETURN NUMBER;
END session_counter_pkg;
/
```

### 2. Package body with state and initialization

```sql
CREATE OR REPLACE PACKAGE BODY session_counter_pkg AS

  -- Package-level variable (session state)
  g_counter NUMBER;

  -- Initialization section: runs once per session
  BEGIN
    g_counter := 0;
  END;

  PROCEDURE increment_counter IS
  BEGIN
    g_counter := g_counter + 1;
  END increment_counter;

  FUNCTION get_counter RETURN NUMBER IS
  BEGIN
    RETURN g_counter;
  END get_counter;

END session_counter_pkg;
/
```

### 3. Example usage (input/output)

```sql
SET SERVEROUTPUT ON;

BEGIN
  DBMS_OUTPUT.PUT_LINE('Initial counter: ' || session_counter_pkg.get_counter);
  session_counter_pkg.increment_counter;
  session_counter_pkg.increment_counter;
  DBMS_OUTPUT.PUT_LINE('After 2 increments: ' || session_counter_pkg.get_counter);
END;
/
```

**Output (within the same session):**

```text
Initial counter: 0
After 2 increments: 2
```

If you disconnect and reconnect (new session) and run again, the counter starts at 0 because the package is reinitialized per session.

## Example 3: Overloaded procedures in a package

### 1. Package specification

```sql
CREATE OR REPLACE PACKAGE log_pkg AS
  PROCEDURE write_log(p_message VARCHAR2);
  PROCEDURE write_log(p_message VARCHAR2, p_severity VARCHAR2);
END log_pkg;
/
```

### 2. Package body

```sql
CREATE OR REPLACE PACKAGE BODY log_pkg AS

  PROCEDURE write_log(p_message VARCHAR2) IS
  BEGIN
    DBMS_OUTPUT.PUT_LINE('[INFO] ' || p_message);
  END write_log;

  PROCEDURE write_log(p_message VARCHAR2, p_severity VARCHAR2) IS
  BEGIN
    DBMS_OUTPUT.PUT_LINE('[' || UPPER(p_severity) || '] ' || p_message);
  END write_log;

END log_pkg;
/
```

### 3. Example usage (input/output)

```sql
SET SERVEROUTPUT ON;

BEGIN
  log_pkg.write_log('System started');
  log_pkg.write_log('Disk space low', 'warning');
END;
/
```

**Output:**

```text
[INFO] System started
[WARNING] Disk space low
```
