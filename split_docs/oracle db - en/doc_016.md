
Here’s a concise summary of **PIVOT / UNPIVOT** in Oracle, with key ideas and code examples.

---

## 1. What PIVOT Does

**PIVOT** turns **rows into columns**.  
Typical use: convert “long” data (e.g., one row per month) into “wide” data (one row per year with 12 month columns).

### Basic Syntax

```sql
SELECT *
FROM   source_table
PIVOT (
  aggregate_function(value_column)
  FOR pivot_column IN (col_name1, col_name2, ...)
);
```

### Example: Sales by Month → Columns

**Input table: `sales`**

| year | month | amount |
|------|-------|--------|
| 2024 | JAN   | 100    |
| 2024 | FEB   | 150    |
| 2024 | MAR   | 200    |
| 2025 | JAN   | 120    |
| 2025 | FEB   | 180    |

```sql
SELECT *
FROM   sales
PIVOT (
  SUM(amount)
  FOR month IN ('JAN' AS jan, 'FEB' AS feb, 'MAR' AS mar)
)
ORDER BY year;
```

**Output**

| year | JAN | FEB | MAR |
|------|-----|-----|-----|
| 2024 | 100 | 150 | 200 |
| 2025 | 120 | 180 | null |

Key points:
- `SUM(amount)` is the aggregation.
- `FOR month IN (...)` defines which row values become columns.
- You can rename pivoted columns with `AS`.

---

## 2. What UNPIVOT Does

**UNPIVOT** turns **columns into rows**.  
Typical use: convert “wide” data back into “long” format.

### Basic Syntax

```sql
SELECT *
FROM   source_table
UNPIVOT (
  value_column
  FOR pivot_column IN (col_name1, col_name2, ...)
);
```

### Example: Month Columns → Rows

Using the previous pivoted result:

**Input table: `sales_pivoted`**

| year | JAN | FEB | MAR |
|------|-----|-----|-----|
| 2024 | 100 | 150 | 200 |
| 2025 | 120 | 180 | null |

```sql
SELECT year,
       month,
       amount
FROM   sales_pivoted
UNPIVOT (
  amount
  FOR month IN (JAN AS 'JAN', FEB AS 'FEB', MAR AS 'MAR')
)
ORDER BY year, month;
```

**Output**

| year | month | amount |
|------|-------|--------|
| 2024 | JAN   | 100    |
| 2024 | FEB   | 150    |
| 2024 | MAR   | 200    |
| 2025 | JAN   | 120    |
| 2025 | FEB   | 180    |

## 3. Common Patterns

### 3.1 PIVOT with Multiple Grouping Columns

```sql
SELECT region, year, jan, feb, mar
FROM   sales
PIVOT (
  SUM(amount)
  FOR month IN ('JAN' AS jan, 'FEB' AS feb, 'MAR' AS mar)
);
```

Here, `region` and `year` remain as row identifiers; months become columns.

---

### 3.2 UNPIVOT Multiple Measures

If you have multiple measure columns:

**Input**

| id | q1_sales | q2_sales | q1_cost | q2_cost |
|----|----------|----------|---------|---------|
| 1  | 100      | 200      | 60      | 120     |

```sql
SELECT id, quarter, metric, value
FROM   quarterly_data
UNPIVOT (
  value
  FOR (quarter, metric) IN (
    (q1_sales AS 'Q1', 'SALES'),
    (q2_sales AS 'Q2', 'SALES'),
    (q1_cost  AS 'Q1', 'COST'),
    (q2_cost  AS 'Q2', 'COST')
  )
);
```

**Output**

| id | quarter | metric | value |
|----|---------|--------|-------|
| 1  | Q1      | SALES  | 100   |
| 1  | Q2      | SALES  | 200   |
| 1  | Q1      | COST   | 60    |
| 1  | Q2      | COST   | 120   |

---

## 4. Key Points to Remember

- **PIVOT** requires an aggregate function (e.g., `SUM`, `COUNT`, `MAX`).
- The list in `IN (...)` is **static**; you must specify the values that become columns.
- **UNPIVOT** is useful for:
  - Normalizing denormalized tables.
  - Preparing data for reporting or analytics tools.
- `NULL` values often appear when a combination (e.g., year + month) has no data.
