
Here’s a concise summary and examples about **PL/SQL initialization sections**.

---

## Key Ideas

1. **Where they appear**
   - In **anonymous blocks**
   - In **procedures and functions**
   - In **packages** (package bodies)

2. **Purpose**
   - Run **once** when the block/program unit is first loaded or invoked.
   - Used to:
     - Initialize variables and constants
     - Open cursors
     - Set up application state (e.g., load configuration values)

3. **Structure of a PL/SQL block**

```plsql
DECLARE
    -- Declarations (variables, constants, cursors, types, etc.)
    v_counter NUMBER;
BEGIN
    -- Initialization section (executable statements)
    v_counter := 0;

    -- Main executable logic
    v_counter := v_counter + 1;
    DBMS_OUTPUT.PUT_LINE('Counter = ' || v_counter);
EXCEPTION
    -- Exception handlers
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('Error: ' || SQLERRM);
END;
/
```

- The **BEGIN…END** part is the executable section.
- The **first statements** in the BEGIN block often act as the **initialization section**.

---

## Initialization in Procedures/Functions

Initialization code runs each time the procedure/function is called.

```plsql
CREATE OR REPLACE PROCEDURE process_order (p_order_id IN NUMBER) IS
    v_status   VARCHAR2(20);
    v_user     VARCHAR2(30);
BEGIN
    -- Initialization section
    v_status := 'NEW';
    v_user   := USER;  -- current database user

    -- Main logic
    DBMS_OUTPUT.PUT_LINE('Processing order ' || p_order_id);
    DBMS_OUTPUT.PUT_LINE('Initial status: ' || v_status);
    DBMS_OUTPUT.PUT_LINE('User: ' || v_user);
END;
/

-- Input
BEGIN
    process_order(1001);
END;
/

-- Output (example)
-- Processing order 1001
-- Initial status: NEW
-- User: HR
```

## Package Initialization Section

Packages have a **special initialization section** that runs **once per session**, the first time anything in the package is referenced.

### Package Specification

```plsql
CREATE OR REPLACE PACKAGE app_config IS
    g_app_name   VARCHAR2(50);
    g_max_retries PLS_INTEGER;

    PROCEDURE show_config;
END app_config;
/
```

### Package Body with Initialization Section

```plsql
CREATE OR REPLACE PACKAGE BODY app_config IS

    PROCEDURE show_config IS
    BEGIN
        DBMS_OUTPUT.PUT_LINE('App Name   : ' || g_app_name);
        DBMS_OUTPUT.PUT_LINE('Max Retries: ' || g_max_retries);
    END show_config;

BEGIN
    -- Package initialization section
    g_app_name    := 'OrderSystem';
    g_max_retries := 3;
    DBMS_OUTPUT.PUT_LINE('app_config package initialized');
END app_config;
/
```

### Usage and Output

```plsql
-- First reference to the package in this session
BEGIN
    app_config.show_config;
END;
/

-- Output (example)
-- app_config package initialized
-- App Name   : OrderSystem
-- Max Retries: 3
```

- The line `app_config package initialized` appears **only once per session**, when the package is first used.
- Subsequent calls in the same session will not re-run the initialization section.

---

## Typical Uses of Initialization Sections

- Set **default values** for global variables.
- Load **configuration** from tables into package variables.
- Initialize **caches** or lookup data.
- Perform **one-time checks** (e.g., environment validation).
