
Here’s a concise summary of PL/SQL exception handling, with examples.

---

## 1. Types of Exceptions in PL/SQL

### A. Predefined Exceptions
Oracle provides many built‑in exceptions (e.g., `NO_DATA_FOUND`, `TOO_MANY_ROWS`, `ZERO_DIVIDE`).

**Example: Predefined exception**

```sql
DECLARE
    v_result NUMBER;
BEGIN
    v_result := 10 / 0;  -- division by zero
    DBMS_OUTPUT.PUT_LINE('Result: ' || v_result);
EXCEPTION
    WHEN ZERO_DIVIDE THEN
        DBMS_OUTPUT.PUT_LINE('Error: Division by zero.');
END;
/
```

**Output:**
```text
Error: Division by zero.
```

---

### B. User-Defined Exceptions
You can declare your own exceptions and raise them with `RAISE`.

**Example: User-defined exception**

```sql
DECLARE
    v_salary   NUMBER := 5000;
    e_low_sal  EXCEPTION;  -- user-defined exception
BEGIN
    IF v_salary < 10000 THEN
        RAISE e_low_sal;
    END IF;

    DBMS_OUTPUT.PUT_LINE('Salary is OK.');
EXCEPTION
    WHEN e_low_sal THEN
        DBMS_OUTPUT.PUT_LINE('Error: Salary is too low.');
END;
/
```

**Output:**
```text
Error: Salary is too low.
```

## 2. RAISE_APPLICATION_ERROR

Used to raise custom errors with your own message and a specific error number in the range `-20000` to `-20999`.

**Syntax:**
```sql
RAISE_APPLICATION_ERROR(error_number, message [, keep_errors]);
```

- `error_number`: between `-20000` and `-20999`
- `message`: custom error text
- `keep_errors` (optional): `TRUE` to add to existing error stack

**Example: RAISE_APPLICATION_ERROR**

```sql
CREATE OR REPLACE PROCEDURE update_salary (
    p_emp_id   IN employees.employee_id%TYPE,
    p_new_sal  IN employees.salary%TYPE
) AS
BEGIN
    IF p_new_sal < 0 THEN
        RAISE_APPLICATION_ERROR(
            -20001,
            'New salary cannot be negative.'
        );
    END IF;

    UPDATE employees
       SET salary = p_new_sal
     WHERE employee_id = p_emp_id;

    IF SQL%ROWCOUNT = 0 THEN
        RAISE_APPLICATION_ERROR(
            -20002,
            'Employee ID not found.'
        );
    END IF;
END;
/
```

**Example call and output:**

```sql
BEGIN
    update_salary(100, -5000);
END;
/
```

**Output (error):**
```text
ORA-20001: New salary cannot be negative.
ORA-06512: at "SCOTT.UPDATE_SALARY", line 6
ORA-06512: at line 2
```

## 3. Basic Exception Block Structure

```sql
BEGIN
    -- normal code
EXCEPTION
    WHEN predefined_exception THEN
        -- handle predefined exception
    WHEN user_defined_exception THEN
        -- handle user-defined exception
    WHEN OTHERS THEN
        -- handle all other exceptions
END;
/
```

Key ideas:
- Use predefined exceptions for common Oracle errors.
- Declare and `RAISE` user-defined exceptions for business rules.
- Use `RAISE_APPLICATION_ERROR` to return custom error codes/messages to callers.
