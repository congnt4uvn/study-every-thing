
Key idea  
Create indexes on foreign key columns to improve performance, especially for joins and DML (DELETE/UPDATE) operations on the parent table.

Why index foreign keys?

1. Faster joins  
   - Foreign keys are frequently used to join child and parent tables.  
   - Indexing the foreign key column(s) allows Oracle to quickly find matching rows.

2. Better performance on DELETE/UPDATE in parent table  
   - When you delete or update a primary key in the parent table, Oracle must check the child table for related rows.  
   - If the foreign key is not indexed, Oracle may lock or scan the entire child table, causing:
     - Full table scans
     - Lock contention
     - Slower transactions

3. Reduced locking and blocking  
   - Indexed foreign keys allow Oracle to locate and lock only the relevant child rows instead of many rows or the whole table.

When to index foreign keys

- Almost always:
  - When the foreign key is used in joins.
  - When rows in the parent table are frequently deleted or updated.
- Possibly skip:
  - If the child table is very small.
  - If the foreign key is rarely used in queries and parent rows are almost never deleted/updated.

Basic example

Tables:

```sql
-- Parent table
CREATE TABLE departments (
    dept_id   NUMBER PRIMARY KEY,
    name      VARCHAR2(100)
);

-- Child table with foreign key
CREATE TABLE employees (
    emp_id    NUMBER PRIMARY KEY,
    name      VARCHAR2(100),
    dept_id   NUMBER,
    CONSTRAINT fk_emp_dept
        FOREIGN KEY (dept_id)
        REFERENCES departments(dept_id)
);
```

Indexing the foreign key:

```sql
CREATE INDEX idx_emp_dept_id ON employees(dept_id);
```

Example: join performance

Input query (without foreign key index):

```sql
SELECT e.emp_id, e.name, d.name AS dept_name
FROM employees e
JOIN departments d
  ON e.dept_id = d.dept_id
WHERE d.dept_id = 10;
```

- Without `idx_emp_dept_id`, Oracle may:
  - Use a full scan on `employees`.
- With `idx_emp_dept_id`, Oracle can:
  - Use the index to quickly find employees in department 10.

Example: delete from parent

Input:

```sql
DELETE FROM departments
WHERE dept_id = 10;
```

Behavior:

- Without foreign key index:
  - Oracle may need to scan the entire `employees` table to check for rows with `dept_id = 10`.
  - Can cause long-running deletes and blocking.
- With `idx_emp_dept_id`:
  - Oracle uses the index to quickly find related employees.
  - Delete is faster and causes fewer locks.

Composite foreign key example

```sql
CREATE TABLE orders (
    order_id   NUMBER PRIMARY KEY,
    customer_id NUMBER,
    order_date  DATE
);

CREATE TABLE order_items (
    order_id   NUMBER,
    line_no    NUMBER,
    product_id NUMBER,
    CONSTRAINT pk_order_items PRIMARY KEY (order_id, line_no),
    CONSTRAINT fk_order_items_order
        FOREIGN KEY (order_id)
        REFERENCES orders(order_id)
);

-- Index on foreign key
CREATE INDEX idx_order_items_order_id ON order_items(order_id);
```

Summary

- Index foreign key columns to:
  - Speed up joins.
  - Improve DELETE/UPDATE performance on parent tables.
  - Reduce locking and contention.
- Create single or composite indexes that match the foreign key definition.
