
Here’s a concise summary of the key set operators in Oracle SQL, with examples and sample outputs.

---

## 1. UNION

- Combines results of two queries.
- Removes duplicate rows (per all selected columns).
- Queries must have the same number of columns and compatible data types.

```sql
-- Table: EMP_A
-- EMP_A
-- EMP_ID | NAME
-- 1      | Alice
-- 2      | Bob

-- Table: EMP_B
-- EMP_B
-- EMP_ID | NAME
-- 2      | Bob
-- 3      | Carol

SELECT EMP_ID, NAME FROM EMP_A
UNION
SELECT EMP_ID, NAME FROM EMP_B;
```

**Result:**

| EMP_ID | NAME  |
|--------|-------|
| 1      | Alice |
| 2      | Bob   |
| 3      | Carol |

Duplicates (EMP_ID=2, Bob) are removed.

---

## 2. UNION ALL

- Combines results of two queries.
- Keeps duplicates.
- Same column count and compatible types required.

```sql
SELECT EMP_ID, NAME FROM EMP_A
UNION ALL
SELECT EMP_ID, NAME FROM EMP_B;
```

**Result:**

| EMP_ID | NAME  |
|--------|-------|
| 1      | Alice |
| 2      | Bob   |
| 2      | Bob   |  -- duplicate kept
| 3      | Carol |

Use `UNION ALL` when you need all rows and want better performance (no duplicate elimination step).

---

## 3. INTERSECT

- Returns only rows that appear in **both** result sets (set intersection).
- Removes duplicates automatically.

```sql
SELECT EMP_ID, NAME FROM EMP_A
INTERSECT
SELECT EMP_ID, NAME FROM EMP_B;
```

**Result:**

| EMP_ID | NAME |
|--------|------|
| 2      | Bob  |

Only common rows between EMP_A and EMP_B are returned.

---

## 4. MINUS (Oracle-specific; in standard SQL: EXCEPT)

- Returns rows from the **first** query that do **not** appear in the second query.
- Removes duplicates automatically.

```sql
SELECT EMP_ID, NAME FROM EMP_A
MINUS
SELECT EMP_ID, NAME FROM EMP_B;
```

**Result:**

| EMP_ID | NAME  |
|--------|-------|
| 1      | Alice |

Alice exists only in EMP_A, not in EMP_B.

---

## 5. General Rules for Set Operators

1. **Column count and order** must match:
   - Same number of columns in each SELECT.
   - Data types must be compatible by position.

2. **Column names in result**:
   - Taken from the first SELECT statement.

3. **ORDER BY**:
   - Apply only once, at the end of the entire set expression.
   - Use column aliases from the first SELECT or column positions.

```sql
SELECT EMP_ID, NAME FROM EMP_A
UNION
SELECT EMP_ID, NAME FROM EMP_B
ORDER BY EMP_ID;
```

---

## 6. Practical Example with Inputs/Outputs

Assume:

**EMP_A**

| EMP_ID | NAME  | DEPT |
|--------|-------|------|
| 1      | Alice | HR   |
| 2      | Bob   | IT   |
| 4      | Dan   | IT   |

**EMP_B**

| EMP_ID | NAME  | DEPT |
|--------|-------|------|
| 2      | Bob   | IT   |
| 3      | Carol | HR   |
| 4      | Dan   | IT   |

### 6.1 All employees from both tables, no duplicates

```sql
SELECT EMP_ID, NAME, DEPT FROM EMP_A
UNION
SELECT EMP_ID, NAME, DEPT FROM EMP_B;
```

**Output:**

| EMP_ID | NAME  | DEPT |
|--------|-------|------|
| 1      | Alice | HR   |
| 2      | Bob   | IT   |
| 3      | Carol | HR   |
| 4      | Dan   | IT   |

### 6.2 All employees from both tables, including duplicates

```sql
SELECT EMP_ID, NAME, DEPT FROM EMP_A
UNION ALL
SELECT EMP_ID, NAME, DEPT FROM EMP_B;
```

**Output:**

| EMP_ID | NAME  | DEPT |
|--------|-------|------|
| 1      | Alice | HR   |
| 2      | Bob   | IT   |
| 4      | Dan   | IT   |
| 2      | Bob   | IT   |
| 3      | Carol | HR   |
| 4      | Dan   | IT   |

### 6.3 Employees common to both tables

```sql
SELECT EMP_ID, NAME, DEPT FROM EMP_A
INTERSECT
SELECT EMP_ID, NAME, DEPT FROM EMP_B;
```

**Output:**

| EMP_ID | NAME | DEPT |
|--------|------|------|
| 2      | Bob  | IT   |
| 4      | Dan  | IT   |

### 6.4 Employees only in EMP_A (not in EMP_B)

```sql
SELECT EMP_ID, NAME, DEPT FROM EMP_A
MINUS
SELECT EMP_ID, NAME, DEPT FROM EMP_B;
```

**Output:**

| EMP_ID | NAME  | DEPT |
|--------|-------|------|
| 1      | Alice | HR   |

---

**Summary:**

- `UNION` = A ∪ B (distinct)
- `UNION ALL` = A ∪ B (with duplicates)
- `INTERSECT` = A ∩ B (common rows)
- `MINUS` = A − B (rows in first, not in second)

Use them to combine or compare result sets efficiently in Oracle SQL.
