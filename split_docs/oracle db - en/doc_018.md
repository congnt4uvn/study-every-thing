
Key idea  
You want to detect “gaps” in a sequence in Oracle, such as:

- Missing dates in a date series  
- Missing IDs in a numeric sequence  

Below are concise patterns and examples.

---

## 1. Find missing IDs in a numeric sequence

Assume a table:

```sql
CREATE TABLE orders (
    order_id NUMBER PRIMARY KEY
);

INSERT INTO orders (order_id) VALUES (1);
INSERT INTO orders (order_id) VALUES (2);
INSERT INTO orders (order_id) VALUES (4);
INSERT INTO orders (order_id) VALUES (7);
COMMIT;
```

### 1.1. Find all missing IDs between min and max

```sql
WITH bounds AS (
    SELECT MIN(order_id) AS min_id,
           MAX(order_id) AS max_id
    FROM orders
),
all_ids AS (
    SELECT (min_id + LEVEL - 1) AS id
    FROM   bounds
    CONNECT BY LEVEL <= (max_id - min_id + 1)
)
SELECT id AS missing_id
FROM   all_ids a
WHERE  NOT EXISTS (
    SELECT 1
    FROM   orders o
    WHERE  o.order_id = a.id
)
ORDER BY id;
```

**Output:**

| MISSING_ID |
|-----------:|
| 3          |
| 5          |
| 6          |

---

### 1.2. Find ranges of missing IDs

```sql
WITH bounds AS (
    SELECT MIN(order_id) AS min_id,
           MAX(order_id) AS max_id
    FROM orders
),
all_ids AS (
    SELECT (min_id + LEVEL - 1) AS id
    FROM   bounds
    CONNECT BY LEVEL <= (max_id - min_id + 1)
),
missing AS (
    SELECT id,
           id - ROW_NUMBER() OVER (ORDER BY id) AS grp
    FROM   all_ids a
    WHERE  NOT EXISTS (
        SELECT 1 FROM orders o WHERE o.order_id = a.id
    )
)
SELECT MIN(id) AS start_missing,
       MAX(id) AS end_missing
FROM   missing
GROUP BY grp
ORDER BY start_missing;
```

**Output:**

| START_MISSING | END_MISSING |
|--------------:|------------:|
| 3             | 3           |
| 5             | 6           |

This shows gaps as ranges.

## 2. Find missing dates in a date sequence

Assume:

```sql
CREATE TABLE daily_sales (
    sale_date DATE PRIMARY KEY,
    amount    NUMBER
);

INSERT INTO daily_sales VALUES (DATE '2024-01-01', 100);
INSERT INTO daily_sales VALUES (DATE '2024-01-02', 150);
-- missing 2024-01-03
INSERT INTO daily_sales VALUES (DATE '2024-01-04', 200);
COMMIT;
```

### 2.1. Generate all dates and find missing ones

```sql
WITH bounds AS (
    SELECT MIN(sale_date) AS min_dt,
           MAX(sale_date) AS max_dt
    FROM daily_sales
),
all_dates AS (
    SELECT min_dt + LEVEL - 1 AS dt
    FROM   bounds
    CONNECT BY LEVEL <= (max_dt - min_dt + 1)
)
SELECT dt AS missing_date
FROM   all_dates d
WHERE  NOT EXISTS (
    SELECT 1
    FROM   daily_sales s
    WHERE  s.sale_date = d.dt
)
ORDER BY dt;
```

**Output:**

| MISSING_DATE |
|-------------:|
| 2024-01-03   |

---

### 2.2. Missing date ranges

```sql
WITH bounds AS (
    SELECT MIN(sale_date) AS min_dt,
           MAX(sale_date) AS max_dt
    FROM daily_sales
),
all_dates AS (
    SELECT min_dt + LEVEL - 1 AS dt
    FROM   bounds
    CONNECT BY LEVEL <= (max_dt - min_dt + 1)
),
missing AS (
    SELECT dt,
           dt - ROW_NUMBER() OVER (ORDER BY dt) AS grp
    FROM   all_dates d
    WHERE  NOT EXISTS (
        SELECT 1 FROM daily_sales s WHERE s.sale_date = d.dt
    )
)
SELECT MIN(dt) AS start_missing_date,
       MAX(dt) AS end_missing_date
FROM   missing
GROUP BY grp
ORDER BY start_missing_date;
```

**Output:**

| START_MISSING_DATE | END_MISSING_DATE |
|-------------------:|-----------------:|
| 2024-01-03         | 2024-01-03       |

---

Summary of technique  
1. Determine min and max of the sequence (ID or date).  
2. Generate the full range using `CONNECT BY LEVEL`.  
3. Use `NOT EXISTS` (or `LEFT JOIN ... WHERE original IS NULL`) to find missing values.  
4. Optionally group with `value - ROW_NUMBER()` to get missing ranges.
