
Here’s a concise summary of **Autonomous Transactions** in Oracle PL/SQL, with code examples and sample input/output.

---

## Key Ideas

1. **Autonomous Transaction** = a *separate*, independent transaction started from within another transaction.
2. It can **COMMIT** or **ROLLBACK** without affecting the main (parent) transaction.
3. Declared using the pragma:
   ```plsql
   PRAGMA AUTONOMOUS_TRANSACTION;
   ```
4. Common use cases:
   - Logging errors or audit information
   - Writing to debug tables
   - Sending notifications
5. Must explicitly **COMMIT** or **ROLLBACK** inside the autonomous block, or Oracle will raise an error when it ends.

---

## Basic Structure

```plsql
DECLARE
   PRAGMA AUTONOMOUS_TRANSACTION;
BEGIN
   -- independent SQL operations here
   COMMIT;  -- or ROLLBACK
END;
```

## Example 1: Error Logging with Autonomous Transaction

### Table for logging

```sql
CREATE TABLE error_log (
    id           NUMBER GENERATED BY DEFAULT AS IDENTITY,
    log_time     DATE,
    err_code     NUMBER,
    err_message  VARCHAR2(4000),
    info         VARCHAR2(4000)
);
```

### Procedure using autonomous transaction

```plsql
CREATE OR REPLACE PROCEDURE log_error (
    p_err_code    NUMBER,
    p_err_message VARCHAR2,
    p_info        VARCHAR2
) IS
   PRAGMA AUTONOMOUS_TRANSACTION;
BEGIN
   INSERT INTO error_log (log_time, err_code, err_message, info)
   VALUES (SYSDATE, p_err_code, p_err_message, p_info);

   COMMIT;  -- commit only the log, not the caller’s work
END;
/
```

### Caller procedure

```plsql
CREATE OR REPLACE PROCEDURE do_something IS
   v_dummy NUMBER;
BEGIN
   -- Simulate some work
   SELECT 1 / 0 INTO v_dummy FROM dual;  -- will raise ORA-01476
EXCEPTION
   WHEN OTHERS THEN
      log_error(SQLCODE, SQLERRM, 'Error in do_something');
      RAISE;  -- re-raise to caller
END;
/
```

### Example run (input/output)

**Input (anonymous block):**
```plsql
BEGIN
   do_something;
END;
/
```

**Output in SQL*Plus / SQL Developer:**
```text
ORA-01476: divisor is equal to zero
```

**But the log is committed:**
```sql
SELECT log_time, err_code, err_message, info
FROM error_log;
```

**Sample result:**
```text
LOG_TIME             ERR_CODE  ERR_MESSAGE                          INFO
-------------------  --------  -----------------------------------  -----------------------
2026-05-20 10:15:32  -1476     ORA-01476: divisor is equal to zero  Error in do_something
```

Even though the main transaction failed, the log row is saved.

## Example 2: Parent vs Autonomous Transaction Visibility

```sql
CREATE TABLE test_parent (
    id   NUMBER,
    txt  VARCHAR2(100)
);

CREATE TABLE test_child (
    id   NUMBER,
    txt  VARCHAR2(100)
);
```

### Autonomous procedure

```plsql
CREATE OR REPLACE PROCEDURE insert_child_autonomous (
    p_id  NUMBER,
    p_txt VARCHAR2
) IS
   PRAGMA AUTONOMOUS_TRANSACTION;
BEGIN
   INSERT INTO test_child (id, txt)
   VALUES (p_id, p_txt);

   COMMIT;  -- independent commit
END;
/
```

### Parent block

```plsql
DECLARE
BEGIN
   INSERT INTO test_parent (id, txt)
   VALUES (1, 'Parent row');

   insert_child_autonomous(1, 'Child row');

   ROLLBACK;  -- rollback parent transaction only
END;
/
```

### Check results

```sql
SELECT * FROM test_parent;
```

**Output:**
```text
(no rows)  -- parent insert rolled back
```

```sql
SELECT * FROM test_child;
```

**Output:**
```text
ID   TXT
---  ----------
1    Child row  -- autonomous insert committed
```

---

## Important Notes

- Autonomous transactions **do not see** uncommitted changes from the parent transaction.
- They have their own **transaction context** (locks, commit, rollback).
- Always include an explicit **COMMIT** or **ROLLBACK** inside the autonomous block.
- Use them mainly for **logging, auditing, and side-effect operations** that must be preserved regardless of the main transaction outcome.
