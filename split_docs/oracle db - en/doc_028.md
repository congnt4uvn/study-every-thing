
Here’s a concise summary of **Advanced PL/SQL Collections** with key ideas and code examples.

---

## 1. Types of Collections in PL/SQL

Oracle PL/SQL has **three main collection types**:

1. **Associative Arrays** (index-by tables)
2. **Nested Tables**
3. **VARRAYs** (variable-size arrays)

Each has different indexing, storage, and usage characteristics.

---

## 2. Associative Arrays

- Indexed by **PL/SQL-only** types: `PLS_INTEGER`, `BINARY_INTEGER`, or `VARCHAR2`.
- Exist only in **PL/SQL memory** (not stored in database columns).
- Sparse: indexes do not need to be consecutive.

### Example: Associative Array (Integer Index)

```sql
DECLARE
  TYPE t_num_tab IS TABLE OF NUMBER INDEX BY PLS_INTEGER;
  l_numbers t_num_tab;
  i PLS_INTEGER;
BEGIN
  -- Assign values
  l_numbers(1) := 10;
  l_numbers(5) := 50;

  -- Iterate using FIRST/NEXT
  i := l_numbers.FIRST;
  WHILE i IS NOT NULL LOOP
    DBMS_OUTPUT.PUT_LINE('Index: ' || i || ' Value: ' || l_numbers(i));
    i := l_numbers.NEXT(i);
  END LOOP;
END;
/
```

**Output:**
```
Index: 1 Value: 10
Index: 5 Value: 50
```

---

## 3. Nested Tables

- Unbounded size.
- Can be **stored in database columns**.
- Internally stored as a separate table when persisted.
- Can be **sparse** in PL/SQL (after deletions).

### Example: Nested Table Type and Usage

```sql
-- SQL-level type (can be used in table columns)
CREATE OR REPLACE TYPE t_str_tab AS TABLE OF VARCHAR2(100);
/

DECLARE
  l_names t_str_tab := t_str_tab();  -- constructor
BEGIN
  l_names.EXTEND(3);
  l_names(1) := 'Alice';
  l_names(2) := 'Bob';
  l_names(3) := 'Carol';

  -- Delete one element (sparse)
  l_names.DELETE(2);

  FOR i IN 1 .. l_names.COUNT LOOP
    IF l_names.EXISTS(i) THEN
      DBMS_OUTPUT.PUT_LINE('Index: ' || i || ' Value: ' || l_names(i));
    END IF;
  END LOOP;
END;
/
```

**Output:**
```
Index: 1 Value: Alice
Index: 3 Value: Carol
```

---

## 4. VARRAYs (Variable-Size Arrays)

- **Bounded size** (maximum number of elements defined in type).
- Stored as a **single object** (inline) with the row.
- Dense: indexes are always 1..COUNT, no gaps.

### Example: VARRAY Type and Usage

```sql
-- SQL-level VARRAY type
CREATE OR REPLACE TYPE t_num_varray AS VARRAY(5) OF NUMBER;
/

DECLARE
  l_scores t_num_varray := t_num_varray(10, 20, 30);
BEGIN
  l_scores.EXTEND;          -- add one element
  l_scores(4) := 40;

  FOR i IN 1 .. l_scores.COUNT LOOP
    DBMS_OUTPUT.PUT_LINE('Index: ' || i || ' Value: ' || l_scores(i));
  END LOOP;
END;
/
```

**Output:**
```
Index: 1 Value: 10
Index: 2 Value: 20
Index: 3 Value: 30
Index: 4 Value: 40
```

## 5. Common Collection Methods

All three collection types support methods like:

- `COUNT` – number of elements
- `LIMIT` – max size (only for VARRAYs)
- `EXTEND` – add elements
- `TRIM` – remove elements from end
- `DELETE` – remove elements (or all)
- `EXISTS(i)` – check if index `i` exists
- `FIRST`, `LAST`, `NEXT(i)`, `PRIOR(i)` – navigation

### Example: Using Collection Methods

```sql
DECLARE
  TYPE t_num_tab IS TABLE OF NUMBER;
  l_tab t_num_tab := t_num_tab();
BEGIN
  l_tab.EXTEND(3);
  l_tab(1) := 100;
  l_tab(2) := 200;
  l_tab(3) := 300;

  DBMS_OUTPUT.PUT_LINE('COUNT = ' || l_tab.COUNT);  -- 3

  l_tab.TRIM;  -- remove last element
  DBMS_OUTPUT.PUT_LINE('COUNT after TRIM = ' || l_tab.COUNT);  -- 2

  l_tab.DELETE(1);  -- delete index 1
  DBMS_OUTPUT.PUT_LINE('EXISTS(1) = ' || CASE WHEN l_tab.EXISTS(1) THEN 'Y' ELSE 'N' END);
  DBMS_OUTPUT.PUT_LINE('EXISTS(2) = ' || CASE WHEN l_tab.EXISTS(2) THEN 'Y' ELSE 'N' END);
END;
/
```

**Output:**
```
COUNT = 3
COUNT after TRIM = 2
EXISTS(1) = N
EXISTS(2) = Y
```

## 6. Bulk Operations with Collections

Collections are key for **bulk SQL operations**:

- `BULK COLLECT` – fetch many rows into a collection at once.
- `FORALL` – send many DML statements to SQL engine in one go.

### Example: BULK COLLECT + FORALL

```sql
DECLARE
  TYPE t_emp_ids IS TABLE OF employees.employee_id%TYPE;
  l_ids t_emp_ids;
BEGIN
  -- Bulk fetch
  SELECT employee_id
  BULK COLLECT INTO l_ids
  FROM employees
  WHERE department_id = 10;

  -- Bulk update
  FORALL i IN l_ids.FIRST .. l_ids.LAST
    UPDATE employees
    SET salary = salary * 1.1
    WHERE employee_id = l_ids(i);

  DBMS_OUTPUT.PUT_LINE('Updated rows: ' || SQL%ROWCOUNT);
END;
/
```

**Output (example):**
```
Updated rows: 5
```

## 7. When to Use Which Collection

- **Associative Array**  
  - In-memory processing, temporary data, lookups by key.
  - Not stored in tables.

- **Nested Table**  
  - When you need a collection type that can be **stored in a column** and can grow without a fixed upper bound.

- **VARRAY**  
  - When you need a **small, bounded list** stored with the row, and order matters.
