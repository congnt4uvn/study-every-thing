
Here’s a concise summary and examples about **deterministic functions in PL/SQL**.

---

## Key Ideas

1. **Deterministic function definition**  
   A deterministic function always returns the **same result** for the **same input parameters**, regardless of:
   - Session state
   - Data in tables (ideally)
   - Time, random values, etc.

2. **Why use `DETERMINISTIC`?**  
   - Allows Oracle to **cache** results in some contexts (especially in function-based indexes and query optimization).
   - Can improve performance when the function is called many times with the same arguments.

3. **When NOT to use it**  
   Do not mark a function as deterministic if:
   - It depends on **table data** that can change.
   - It uses **SYSDATE, SYSTIMESTAMP, DBMS_RANDOM**, sequences, or any non-repeatable source.
   - It depends on **package globals** or session state that can change.

4. **Typical use cases**
   - **Pure calculations** (math, string manipulation).
   - **Normalization** or **transformation** of input values.
   - Functions used in **function-based indexes**.

---

## Basic Deterministic Function Example

### Function definition

```sql
CREATE OR REPLACE FUNCTION normalize_name (p_name IN VARCHAR2)
  RETURN VARCHAR2
  DETERMINISTIC
IS
BEGIN
  RETURN UPPER(TRIM(p_name));
END;
/
```

- For any given `p_name`, the result is always the same → safe to be deterministic.

### Example usage (input/output)

```sql
SELECT normalize_name('  john  ') AS normalized
FROM dual;
```

**Output:**

| NORMALIZED |
|-----------|
| JOHN      |

Calling it again with `'  john  '` will always return `JOHN`.

## Deterministic Function with Function-Based Index

### 1. Create the function

```sql
CREATE OR REPLACE FUNCTION hash_email (p_email IN VARCHAR2)
  RETURN VARCHAR2
  DETERMINISTIC
IS
BEGIN
  RETURN STANDARD_HASH(LOWER(TRIM(p_email)), 'SHA256');
END;
/
```

### 2. Create a table and index

```sql
CREATE TABLE users (
  user_id   NUMBER PRIMARY KEY,
  email     VARCHAR2(320)
);

CREATE INDEX users_email_hash_idx
  ON users (hash_email(email));
```

### 3. Query using the function

```sql
SELECT user_id
FROM users
WHERE hash_email(email) = hash_email('user@example.com');
```

- Oracle can use the **function-based index** on `hash_email(email)` to speed up the search.

---

## Incorrect Use Example (Should NOT be Deterministic)

```sql
CREATE OR REPLACE FUNCTION get_today_label
  RETURN VARCHAR2
  DETERMINISTIC  -- ❌ This is wrong
IS
BEGIN
  RETURN TO_CHAR(SYSDATE, 'YYYY-MM-DD');
END;
/
```

- `SYSDATE` changes every day, so the same call at different times returns different results.
- This function must **not** be declared deterministic.

---

## Summary

- Use `DETERMINISTIC` only for **pure, repeatable** functions.
- It can improve performance, especially with **function-based indexes**.
- Never mark functions as deterministic if they depend on changing data, time, random values, or session state.
