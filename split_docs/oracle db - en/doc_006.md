
Here’s a concise summary of Oracle Analytic (Window) Functions, with code examples and sample inputs/outputs.

---

## 1. What Analytic (Window) Functions Are

Analytic (window) functions perform calculations **across a set of rows related to the current row**, without collapsing rows like `GROUP BY` does.

Key points:
- Use `OVER (...)` clause.
- Can “look” at other rows (previous, next, partition).
- Return a value **for every row**.
- Often used for running totals, rankings, moving averages, percentiles, etc.

General form:

```sql
function_name (expression) 
OVER (
  [PARTITION BY expr_list]
  [ORDER BY expr_list]
  [windowing_clause]
)
```

---

## 2. PARTITION BY and ORDER BY

- `PARTITION BY` splits data into groups (like `GROUP BY`), but rows are not collapsed.
- `ORDER BY` defines the order within each partition (important for running totals, rankings, etc.).

Example table `sales`:

| sale_id | region | sale_date  | amount |
|--------:|--------|------------|--------|
| 1       | East   | 2024-01-01 | 100    |
| 2       | East   | 2024-01-02 | 200    |
| 3       | West   | 2024-01-01 | 150    |
| 4       | West   | 2024-01-03 | 300    |

## 3. Ranking Functions

### 3.1 `ROW_NUMBER()`

Assigns a unique sequence number per partition.

```sql
SELECT
  region,
  sale_date,
  amount,
  ROW_NUMBER() OVER (
    PARTITION BY region
    ORDER BY sale_date
  ) AS rn
FROM sales
ORDER BY region, sale_date;
```

**Output:**

| region | sale_date  | amount | rn |
|--------|------------|--------|----|
| East   | 2024-01-01 | 100    | 1  |
| East   | 2024-01-02 | 200    | 2  |
| West   | 2024-01-01 | 150    | 1  |
| West   | 2024-01-03 | 300    | 2  |

### 3.2 `RANK()` and `DENSE_RANK()`

Rank rows, usually by a measure (e.g., amount).

```sql
SELECT
  region,
  amount,
  RANK()       OVER (PARTITION BY region ORDER BY amount DESC) AS rnk,
  DENSE_RANK() OVER (PARTITION BY region ORDER BY amount DESC) AS dense_rnk
FROM sales
ORDER BY region, amount DESC;
```

If two rows tie, `RANK` skips numbers, `DENSE_RANK` does not.

## 4. Aggregate Analytic Functions

Same functions as aggregates (`SUM`, `AVG`, `MIN`, `MAX`, `COUNT`), but used with `OVER`.

### 4.1 Running Total (Cumulative Sum)

```sql
SELECT
  region,
  sale_date,
  amount,
  SUM(amount) OVER (
    PARTITION BY region
    ORDER BY sale_date
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
  ) AS running_total
FROM sales
ORDER BY region, sale_date;
```

**Output:**

| region | sale_date  | amount | running_total |
|--------|------------|--------|---------------|
| East   | 2024-01-01 | 100    | 100           |
| East   | 2024-01-02 | 200    | 300           |
| West   | 2024-01-01 | 150    | 150           |
| West   | 2024-01-03 | 300    | 450           |

### 4.2 Total per Partition (Same Value on Each Row)

```sql
SELECT
  region,
  sale_date,
  amount,
  SUM(amount) OVER (PARTITION BY region) AS total_region_sales
FROM sales
ORDER BY region, sale_date;
```

**Output:**

| region | sale_date  | amount | total_region_sales |
|--------|------------|--------|--------------------|
| East   | 2024-01-01 | 100    | 300                |
| East   | 2024-01-02 | 200    | 300                |
| West   | 2024-01-01 | 150    | 450                |
| West   | 2024-01-03 | 300    | 450                |

## 5. Windowing Clauses

Control which rows are included in the calculation relative to the current row.

Common forms:
- `ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW` (from first row to current)
- `ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING` (sliding window of 3 rows)
- `RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW` (by value range, not row count)

### Example: Moving Average (3-row window)

```sql
SELECT
  sale_date,
  amount,
  AVG(amount) OVER (
    ORDER BY sale_date
    ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
  ) AS moving_avg_3
FROM sales
ORDER BY sale_date;
```

If amounts by date are: 100, 200, 150, 300

**Output (simplified):**

| sale_date  | amount | moving_avg_3 |
|------------|--------|--------------|
| 2024-01-01 | 100    | 100.0        |  (only 1 row in window)
| 2024-01-02 | 200    | 150.0        |  (100, 200)
| 2024-01-03 | 150    | 150.0        |  (100, 200, 150)
| 2024-01-04 | 300    | 216.7        |  (200, 150, 300)

## 6. Value Navigation: LAG / LEAD

Access previous or next row values.

### 6.1 `LAG` (previous row)

```sql
SELECT
  sale_date,
  amount,
  LAG(amount, 1) OVER (ORDER BY sale_date) AS prev_amount
FROM sales
ORDER BY sale_date;
```

**Output:**

| sale_date  | amount | prev_amount |
|------------|--------|-------------|
| 2024-01-01 | 100    | null        |
| 2024-01-02 | 200    | 100         |
| 2024-01-03 | 150    | 200         |
| 2024-01-04 | 300    | 150         |

### 6.2 `LEAD` (next row)

```sql
SELECT
  sale_date,
  amount,
  LEAD(amount, 1) OVER (ORDER BY sale_date) AS next_amount
FROM sales
ORDER BY sale_date;
```

## 7. FIRST_VALUE / LAST_VALUE

Get first/last value in a window.

```sql
SELECT
  sale_date,
  amount,
  FIRST_VALUE(amount) OVER (
    ORDER BY sale_date
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
  ) AS first_amount,
  LAST_VALUE(amount) OVER (
    ORDER BY sale_date
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
  ) AS last_amount_so_far
FROM sales
ORDER BY sale_date;
```

## 8. Percentile and Distribution Functions

Useful for analytics and reporting.

- `NTILE(n)` – splits ordered rows into `n` buckets.
- `PERCENT_RANK()` – relative rank between 0 and 1.
- `CUME_DIST()` – cumulative distribution.
- `PERCENTILE_CONT`, `PERCENTILE_DISC` – continuous/discrete percentiles.

Example: Quartiles with `NTILE(4)`:

```sql
SELECT
  region,
  amount,
  NTILE(4) OVER (PARTITION BY region ORDER BY amount) AS quartile
FROM sales
ORDER BY region, amount;
```

---

## 9. Key Differences vs GROUP BY

- `GROUP BY`:
  - Reduces rows (one row per group).
  - Cannot easily show detail and aggregate in the same row without subqueries.

- Analytic functions:
  - Keep all rows.
  - Allow detail + aggregate in one query/row.
  - Very powerful for reporting, dashboards, and complex analytics.
