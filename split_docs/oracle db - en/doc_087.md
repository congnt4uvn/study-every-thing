
Here’s a concise summary of Oracle Scheduler in PL/SQL/SQL, with key ideas and code examples.

---

## 1. What the Oracle Scheduler Is

Oracle Scheduler lets you:

- Run jobs automatically (once, recurring, or event-based)
- Schedule PL/SQL procedures, anonymous blocks, OS scripts, or SQL statements
- Control when and how often jobs run
- Monitor job status, logs, and history

Main objects:

- **Programs** – what to run
- **Schedules** – when to run
- **Jobs** – tie program + schedule (or define inline)
- **Job classes**, **windows**, **window groups** – advanced resource and time management

---

## 2. Creating a Simple Job (Inline)

Run a PL/SQL block every day at 2 AM:

```sql
BEGIN
  DBMS_SCHEDULER.create_job (
    job_name        => 'job_daily_purge',
    job_type        => 'PLSQL_BLOCK',
    job_action      => 'BEGIN purge_old_data; END;',
    start_date      => SYSTIMESTAMP,
    repeat_interval => 'FREQ=DAILY; BYHOUR=2; BYMINUTE=0; BYSECOND=0',
    enabled         => TRUE,
    comments        => 'Daily purge of old data');
END;
/
```

**What it does:**  
- Starts now (`start_date => SYSTIMESTAMP`)  
- Repeats daily at 02:00:00  
- Executes `purge_old_data` procedure

## 3. Creating a Program + Schedule + Job

### 3.1 Program (what to run)

```sql
BEGIN
  DBMS_SCHEDULER.create_program (
    program_name   => 'prog_monthly_report',
    program_type   => 'STORED_PROCEDURE',
    program_action => 'report_pkg.run_monthly',
    number_of_arguments => 1,
    enabled        => FALSE);

  DBMS_SCHEDULER.define_program_argument (
    program_name      => 'prog_monthly_report',
    argument_position => 1,
    argument_name     => 'p_month',
    argument_type     => 'VARCHAR2');

  DBMS_SCHEDULER.enable('prog_monthly_report');
END;
/
```

### 3.2 Schedule (when to run)

Run on the 1st of every month at 06:00:

```sql
BEGIN
  DBMS_SCHEDULER.create_schedule (
    schedule_name   => 'sched_monthly_6am',
    start_date      => SYSTIMESTAMP,
    repeat_interval => 'FREQ=MONTHLY; BYMONTHDAY=1; BYHOUR=6; BYMINUTE=0; BYSECOND=0',
    comments        => 'First day of month at 6 AM');
END;
/
```

### 3.3 Job (link program + schedule)

```sql
BEGIN
  DBMS_SCHEDULER.create_job (
    job_name      => 'job_monthly_report',
    program_name  => 'prog_monthly_report',
    schedule_name => 'sched_monthly_6am',
    enabled       => FALSE);

  DBMS_SCHEDULER.set_job_argument_value (
    job_name          => 'job_monthly_report',
    argument_position => 1,
    argument_value    => '2026-05');

  DBMS_SCHEDULER.enable('job_monthly_report');
END;
/
```

**Runtime behavior (conceptual I/O):**

- **Input:** `p_month = '2026-05'`
- **Process:** `report_pkg.run_monthly('2026-05')` runs on the 1st of each month at 06:00
- **Output:** Whatever the procedure does (e.g., inserts into a report table, sends email)

---

## 4. Running a Job On Demand

```sql
BEGIN
  DBMS_SCHEDULER.run_job (
    job_name => 'job_monthly_report',
    use_current_session => FALSE);
END;
/
```

- `use_current_session => FALSE` runs it in the background.

## 5. Checking Job Status and Logs

### 5.1 Basic job info

```sql
SELECT job_name, state, last_start_date, next_run_date
FROM   dba_scheduler_jobs
WHERE  job_name = 'JOB_MONTHLY_REPORT';
```

Example output (conceptual):

| JOB_NAME            | STATE   | LAST_START_DATE       | NEXT_RUN_DATE         |
|---------------------|---------|-----------------------|-----------------------|
| JOB_MONTHLY_REPORT  | SCHEDULED | 2026-05-01 06:00:00 | 2026-06-01 06:00:00   |

### 5.2 Run history

```sql
SELECT job_name, status, actual_start_date, run_duration, error#
FROM   dba_scheduler_job_run_details
WHERE  job_name = 'JOB_MONTHLY_REPORT'
ORDER  BY actual_start_date DESC;
```

Example output (conceptual):

| JOB_NAME           | STATUS | ACTUAL_START_DATE      | RUN_DURATION | ERROR# |
|--------------------|--------|------------------------|-------------|--------|
| JOB_MONTHLY_REPORT | SUCCEEDED | 2026-05-01 06:00:02 | +000 00:00:10 | 0      |

## 6. Enabling, Disabling, Dropping Jobs

```sql
-- Disable
BEGIN
  DBMS_SCHEDULER.disable('job_monthly_report');
END;
/

-- Enable
BEGIN
  DBMS_SCHEDULER.enable('job_monthly_report');
END;
/

-- Drop
BEGIN
  DBMS_SCHEDULER.drop_job('job_monthly_report');
END;
/
```

## 7. Scheduling a Simple SQL Statement

Example: gather stats nightly:

```sql
BEGIN
  DBMS_SCHEDULER.create_job (
    job_name        => 'job_gather_stats',
    job_type        => 'PLSQL_BLOCK',
    job_action      => 'BEGIN DBMS_STATS.gather_schema_stats(''HR''); END;',
    start_date      => SYSTIMESTAMP,
    repeat_interval => 'FREQ=DAILY; BYHOUR=1; BYMINUTE=0; BYSECOND=0',
    enabled         => TRUE);
END;
/
```
