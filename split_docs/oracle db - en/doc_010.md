
Here’s a concise summary of **Oracle hierarchical queries** with key ideas and code examples.

---

## 1. What is a Hierarchical Query?

Hierarchical queries let you query data that has a **parent–child** relationship, such as:

- Organization charts (manager → employee)  
- Category trees (category → subcategory)  
- Bill of materials (part → subpart)

In Oracle, this is done with:

```sql
START WITH ... 
CONNECT BY ...
```

---

## 2. Basic Syntax

```sql
SELECT ...
FROM   table_name
START WITH <root_condition>
CONNECT BY [NOCYCLE] PRIOR child_col = parent_col;
```

- `START WITH` defines the **root rows** (top of the hierarchy).
- `CONNECT BY` defines the **parent–child relationship**.
- `PRIOR` indicates which side is the parent.

---

## 3. Example Table

```sql
CREATE TABLE employees (
    emp_id     NUMBER PRIMARY KEY,
    emp_name   VARCHAR2(50),
    manager_id NUMBER
);

INSERT INTO employees VALUES (1, 'CEO',        NULL);
INSERT INTO employees VALUES (2, 'VP Sales',   1);
INSERT INTO employees VALUES (3, 'VP Tech',    1);
INSERT INTO employees VALUES (4, 'Sales Rep',  2);
INSERT INTO employees VALUES (5, 'Engineer',   3);
COMMIT;
```

---

## 4. Simple Hierarchical Query

Get the organization tree starting from the CEO:

```sql
SELECT emp_id,
       emp_name,
       manager_id
FROM   employees
START WITH manager_id IS NULL
CONNECT BY PRIOR emp_id = manager_id;
```

**Explanation:**

- `START WITH manager_id IS NULL` → root is the CEO.
- `CONNECT BY PRIOR emp_id = manager_id` → parent’s `emp_id` = child’s `manager_id`.

**Sample Output:**

| EMP_ID | EMP_NAME   | MANAGER_ID |
|--------|------------|------------|
| 1      | CEO        | (null)     |
| 2      | VP Sales   | 1          |
| 4      | Sales Rep  | 2          |
| 3      | VP Tech    | 1          |
| 5      | Engineer   | 3          |

(Exact order depends on data; see `ORDER SIBLINGS BY` below.)

---

## 5. LEVEL Pseudocolumn

`LEVEL` shows the depth in the hierarchy (root = 1):

```sql
SELECT LEVEL,
       emp_id,
       emp_name,
       manager_id
FROM   employees
START WITH manager_id IS NULL
CONNECT BY PRIOR emp_id = manager_id
ORDER BY LEVEL, emp_id;
```

**Sample Output:**

| LEVEL | EMP_ID | EMP_NAME   | MANAGER_ID |
|-------|--------|------------|------------|
| 1     | 1      | CEO        | (null)     |
| 2     | 2      | VP Sales   | 1          |
| 2     | 3      | VP Tech    | 1          |
| 3     | 4      | Sales Rep  | 2          |
| 3     | 5      | Engineer   | 3          |

---

## 6. Indented Tree Display

Use `LPAD` with `LEVEL` to show a tree:

```sql
SELECT LPAD(' ', 2 * (LEVEL - 1)) || emp_name AS emp_tree,
       emp_id,
       manager_id
FROM   employees
START WITH manager_id IS NULL
CONNECT BY PRIOR emp_id = manager_id
ORDER SIBLINGS BY emp_name;
```

**Sample Output:**

| EMP_TREE      | EMP_ID | MANAGER_ID |
|---------------|--------|------------|
| CEO           | 1      | (null)     |
|   VP Sales    | 2      | 1          |
|     Sales Rep | 4      | 2          |
|   VP Tech     | 3      | 1          |
|     Engineer  | 5      | 3          |

---

## 7. ORDER SIBLINGS BY

To sort children under the same parent:

```sql
SELECT LPAD(' ', 2 * (LEVEL - 1)) || emp_name AS emp_tree
FROM   employees
START WITH manager_id IS NULL
CONNECT BY PRIOR emp_id = manager_id
ORDER SIBLINGS BY emp_name;
```

`ORDER SIBLINGS BY` keeps the hierarchical structure but orders siblings.

---

## 8. Traversing Up vs Down

### Downward (parent → child)

```sql
CONNECT BY PRIOR emp_id = manager_id;
```

### Upward (child → parent)

```sql
CONNECT BY PRIOR manager_id = emp_id;
```

Example: find all managers above a given employee:

```sql
SELECT LEVEL,
       emp_id,
       emp_name
FROM   employees
START WITH emp_id = 4          -- Sales Rep
CONNECT BY PRIOR manager_id = emp_id;
```

**Sample Output:**

| LEVEL | EMP_ID | EMP_NAME |
|-------|--------|----------|
| 1     | 4      | Sales Rep|
| 2     | 2      | VP Sales |
| 3     | 1      | CEO      |

---

## 9. Avoiding Cycles: NOCYCLE

If data may contain loops:

```sql
SELECT emp_id,
       emp_name,
       LEVEL
FROM   employees
START WITH manager_id IS NULL
CONNECT BY NOCYCLE PRIOR emp_id = manager_id;
```

`NOCYCLE` prevents infinite loops.

---

## 10. SYS_CONNECT_BY_PATH

Build a full path from root to node:

```sql
SELECT emp_id,
       emp_name,
       SYS_CONNECT_BY_PATH(emp_name, ' / ') AS path
FROM   employees
START WITH manager_id IS NULL
CONNECT BY PRIOR emp_id = manager_id;
```

**Sample Output:**

| EMP_ID | EMP_NAME   | PATH                          |
|--------|------------|-------------------------------|
| 1      | CEO        | / CEO                         |
| 2      | VP Sales   | / CEO / VP Sales             |
| 4      | Sales Rep  | / CEO / VP Sales / Sales Rep |
| 3      | VP Tech    | / CEO / VP Tech              |
| 5      | Engineer   | / CEO / VP Tech / Engineer   |
