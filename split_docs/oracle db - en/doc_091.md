
Here’s a concise summary of `GRANT` and `REVOKE` in Oracle, with examples.

---

## 1. What they are

- **GRANT**: Gives privileges (permissions) or roles to users/roles.
- **REVOKE**: Removes previously granted privileges or roles.

They control **who can do what** in the database (e.g., select, insert, create tables, etc.).

---

## 2. Types of privileges

### System privileges
Allow actions on the database as a whole.

Common examples:
- `CREATE SESSION` – connect to the database
- `CREATE TABLE` – create tables in own schema
- `CREATE USER` – create new users
- `DROP USER` – drop users

**Example: grant system privilege**

```sql
-- As a DBA or user with admin rights
GRANT CREATE SESSION TO user1;
GRANT CREATE TABLE TO user1;
```

**Check result (input/output)**

```sql
-- Input
SELECT * FROM USER_SYS_PRIVS;

-- Output (example rows)
USERNAME  PRIVILEGE        ADMIN_OPTION
--------  ---------------  ------------
USER1     CREATE SESSION   NO
USER1     CREATE TABLE     NO
```

---

### Object privileges
Allow actions on specific objects (tables, views, procedures, etc.).

Common examples on tables:
- `SELECT`
- `INSERT`
- `UPDATE`
- `DELETE`
- `REFERENCES` (for foreign keys)

**Example: grant object privilege**

```sql
-- Owner of table EMP or a DBA
GRANT SELECT, INSERT ON emp TO user2;
```

**Check result**

```sql
-- Input (run as USER2)
SELECT * FROM ALL_TAB_PRIVS WHERE GRANTEE = 'USER2';

-- Output (example rows)
GRANTEE  OWNER  TABLE_NAME  PRIVILEGE
-------  -----  ----------  ---------
USER2    HR     EMP         SELECT
USER2    HR     EMP         INSERT
```

---

## 3. WITH GRANT OPTION

Allows the grantee to pass the privilege on to others.

```sql
GRANT SELECT ON emp TO user2 WITH GRANT OPTION;
```

- `user2` can now do:
  ```sql
  GRANT SELECT ON emp TO user3;
  ```

If the original grant is revoked from `user2`, the privileges `user2` gave to `user3` via that chain are also removed.

---

## 4. Granting roles

Roles are named sets of privileges.

```sql
-- Create a role
CREATE ROLE app_readonly;

-- Give privileges to the role
GRANT SELECT ON emp TO app_readonly;

-- Assign role to a user
GRANT app_readonly TO user2;
```

**Check roles**

```sql
-- Input (as USER2)
SELECT * FROM USER_ROLE_PRIVS;

-- Output (example)
USERNAME  GRANTED_ROLE   ADMIN_OPTION
--------  ------------   ------------
USER2     APP_READONLY   NO
```

---

## 5. REVOKE: removing privileges

### Revoke system privilege

```sql
REVOKE CREATE TABLE FROM user1;
```

### Revoke object privilege

```sql
REVOKE SELECT, INSERT ON emp FROM user2;
```

### Revoke role

```sql
REVOKE app_readonly FROM user2;
```

**Check after revoke**

```sql
-- Input (as USER2)
SELECT * FROM ALL_TAB_PRIVS WHERE GRANTEE = 'USER2';

-- Output (example: no rows for EMP)
no rows selected
```

---

## 6. Simple end‑to‑end example

1. **Create user and grant connect + table access**

```sql
-- As SYS or DBA
CREATE USER user_demo IDENTIFIED BY demo123
  DEFAULT TABLESPACE users
  TEMPORARY TABLESPACE temp
  QUOTA 10M ON users;

GRANT CREATE SESSION TO user_demo;
GRANT SELECT ON hr.employees TO user_demo;
```

2. **Test as `user_demo`**

```sql
-- Input
CONNECT user_demo/demo123;

SELECT employee_id, last_name FROM hr.employees WHERE ROWNUM <= 3;

-- Output (example)
EMPLOYEE_ID  LAST_NAME
-----------  ---------
100          King
101          Kochhar
102          De Haan
```

3. **Revoke and test again**

```sql
-- As HR or DBA
REVOKE SELECT ON hr.employees FROM user_demo;
```

```sql
-- Input (as user_demo)
SELECT employee_id, last_name FROM hr.employees WHERE ROWNUM <= 3;

-- Output (error)
ORA-01031: insufficient privileges
```

---

**Key idea**:  
Use `GRANT` to give users/roles the minimum privileges they need; use `REVOKE` to remove or tighten those privileges.
