
Here’s a concise summary of the key PL/SQL basics, with short code examples and sample inputs/outputs.

---

## 1. Variables

- Used to store data that can change during program execution.
- Must be declared in the `DECLARE` section (or in a package, procedure, function, etc.).

**Example:**

```sql
DECLARE
    v_employee_id   NUMBER := 100;
    v_employee_name VARCHAR2(50);
BEGIN
    v_employee_name := 'Scott';
    DBMS_OUTPUT.PUT_LINE('ID: ' || v_employee_id);
    DBMS_OUTPUT.PUT_LINE('Name: ' || v_employee_name);
END;
/
```

**Output:**

```text
ID: 100
Name: Scott
```

---

## 2. Constants

- Similar to variables, but their value cannot change after initialization.
- Use the `CONSTANT` keyword and assign a value at declaration.

**Example:**

```sql
DECLARE
    c_tax_rate CONSTANT NUMBER := 0.1;  -- 10%
    v_amount   NUMBER := 200;
    v_tax      NUMBER;
BEGIN
    v_tax := v_amount * c_tax_rate;
    DBMS_OUTPUT.PUT_LINE('Tax: ' || v_tax);
END;
/
```

**Output:**

```text
Tax: 20
```

---

## 3. Records

- Composite type that groups related fields (like a row in a table).
- Can be based on a table/rowtype or defined manually.

### 3.1 Table-based record (`%ROWTYPE`)

```sql
DECLARE
    v_emp   employees%ROWTYPE;  -- assumes EMPLOYEES table
BEGIN
    SELECT *
    INTO   v_emp
    FROM   employees
    WHERE  employee_id = 100;

    DBMS_OUTPUT.PUT_LINE('Name: ' || v_emp.first_name || ' ' || v_emp.last_name);
END;
/
```

**Output (example):**

```text
Name: Steven King
```

### 3.2 User-defined record

```sql
DECLARE
    TYPE t_employee_rec IS RECORD (
        id    NUMBER,
        name  VARCHAR2(50),
        sal   NUMBER
    );

    v_emp t_employee_rec;
BEGIN
    v_emp.id   := 1;
    v_emp.name := 'John';
    v_emp.sal  := 5000;

    DBMS_OUTPUT.PUT_LINE('Employee: ' || v_emp.id || ', ' || v_emp.name || ', ' || v_emp.sal);
END;
/
```

**Output:**

```text
Employee: 1, John, 5000
```

---

## 4. Collections

Collections are PL/SQL “arrays” (one-dimensional). Main types:

1. Associative arrays (index-by tables)
2. Nested tables
3. VARRAYs (variable-size arrays with an upper bound)

---

### 4.1 Associative Arrays

- Indexed by integer or string (PL/SQL-only, not stored directly in a table column).
- Sparse (indexes do not need to be consecutive).

**Example (integer index):**

```sql
DECLARE
    TYPE t_num_tab IS TABLE OF VARCHAR2(30) INDEX BY PLS_INTEGER;
    v_names t_num_tab;
    i       PLS_INTEGER;
BEGIN
    v_names(1) := 'Alice';
    v_names(10) := 'Bob';

    i := v_names.FIRST;
    WHILE i IS NOT NULL LOOP
        DBMS_OUTPUT.PUT_LINE('Index ' || i || ': ' || v_names(i));
        i := v_names.NEXT(i);
    END LOOP;
END;
/
```

**Output:**

```text
Index 1: Alice
Index 10: Bob
```

**Example (string index):**

```sql
DECLARE
    TYPE t_salary_tab IS TABLE OF NUMBER INDEX BY VARCHAR2(30);
    v_salary t_salary_tab;
BEGIN
    v_salary('Alice') := 4000;
    v_salary('Bob')   := 5000;

    DBMS_OUTPUT.PUT_LINE('Alice salary: ' || v_salary('Alice'));
    DBMS_OUTPUT.PUT_LINE('Bob salary: '   || v_salary('Bob'));
END;
/
```

**Output:**

```text
Alice salary: 4000
Bob salary: 5000
```

---

### 4.2 Nested Tables

- Unbounded, can be stored in database columns.
- Initially dense, can become sparse after deletions.
- Need constructor to initialize.

**Example:**

```sql
DECLARE
    TYPE t_num_list IS TABLE OF NUMBER;
    v_list t_num_list := t_num_list(10, 20, 30);  -- constructor

    i PLS_INTEGER;
BEGIN
    v_list.EXTEND;
    v_list(4) := 40;

    v_list.DELETE(2);  -- now sparse

    i := v_list.FIRST;
    WHILE i IS NOT NULL LOOP
        DBMS_OUTPUT.PUT_LINE('Index ' || i || ': ' || v_list(i));
        i := v_list.NEXT(i);
    END LOOP;
END;
/
```

**Output:**

```text
Index 1: 10
Index 3: 30
Index 4: 40
```

---

### 4.3 VARRAYs (Variable-Size Arrays)

- Have a fixed maximum size.
- Always dense (no gaps).
- Good when you know a reasonable upper bound.

**Example:**

```sql
DECLARE
    TYPE t_grade_array IS VARRAY(5) OF NUMBER;  -- max 5 elements
    v_grades t_grade_array := t_grade_array(80, 90);
BEGIN
    v_grades.EXTEND;
    v_grades(3) := 100;

    FOR i IN 1 .. v_grades.COUNT LOOP
        DBMS_OUTPUT.PUT_LINE('Grade ' || i || ': ' || v_grades(i));
    END LOOP;
END;
/
```

**Output:**

```text
Grade 1: 80
Grade 2: 90
Grade 3: 100
```

---

### Summary of Collection Types

| Type              | Indexed by         | Stored in DB column | Bounded?       | Sparse? |
|-------------------|--------------------|---------------------|----------------|---------|
| Associative array | Integer / String   | No (PL/SQL only)    | Unbounded      | Yes     |
| Nested table      | Integer (1..N)     | Yes                 | Unbounded      | Yes     |
| VARRAY            | Integer (1..N)     | Yes                 | Fixed max size | No      |

These are the core PL/SQL building blocks: variables and constants for data, records for structured rows, and collections (associative arrays, nested tables, VARRAYs) for lists/arrays of values.
