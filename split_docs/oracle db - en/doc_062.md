
Here’s a concise summary of the key ideas around **primary keys, foreign keys, and unique constraints** in an Oracle relational design, with code examples.

---

## 1. Primary Key

**Purpose:**  
Uniquely identifies each row in a table.  
- Must be **unique** and **NOT NULL**.  
- A table typically has **one primary key** (can be single or multiple columns).

**Example: Create table with primary key**

```sql
CREATE TABLE customers (
    customer_id   NUMBER        CONSTRAINT pk_customers PRIMARY KEY,
    first_name    VARCHAR2(50),
    last_name     VARCHAR2(50),
    email         VARCHAR2(100)
);
```

**Insert examples (input/output)**

```sql
-- Valid insert
INSERT INTO customers (customer_id, first_name, last_name, email)
VALUES (1, 'John', 'Doe', 'john.doe@example.com');

-- Invalid: duplicate primary key
INSERT INTO customers (customer_id, first_name, last_name, email)
VALUES (1, 'Jane', 'Smith', 'jane.smith@example.com');
```

**Output (error for second insert):**

```text
ORA-00001: unique constraint (PK_CUSTOMERS) violated
```

---

## 2. Foreign Key

**Purpose:**  
Enforces a **relationship** between two tables.  
- A foreign key in the child table references the **primary key (or unique key)** in the parent table.  
- Ensures **referential integrity** (no child row refers to a non-existent parent).

**Example: Orders referencing Customers**

```sql
CREATE TABLE orders (
    order_id     NUMBER        CONSTRAINT pk_orders PRIMARY KEY,
    customer_id  NUMBER,
    order_date   DATE          DEFAULT SYSDATE,
    CONSTRAINT fk_orders_customers
        FOREIGN KEY (customer_id)
        REFERENCES customers (customer_id)
);
```

**Insert examples (input/output)**

```sql
-- Valid: customer_id = 1 exists in customers
INSERT INTO orders (order_id, customer_id)
VALUES (100, 1);

-- Invalid: customer_id = 999 does NOT exist
INSERT INTO orders (order_id, customer_id)
VALUES (101, 999);
```

**Output (error for second insert):**

```text
ORA-02291: integrity constraint (FK_ORDERS_CUSTOMERS) violated - parent key not found
```

---

## 3. Unique Constraint

**Purpose:**  
Ensures that values in one or more columns are **unique** across the table.  
- Unlike primary key, a unique column **can be NULL** (unless also defined NOT NULL).  
- A table can have **multiple unique constraints**.

**Example: Unique email in customers**

```sql
ALTER TABLE customers
ADD CONSTRAINT uq_customers_email UNIQUE (email);
```

**Insert examples (input/output)**

```sql
-- Valid: new unique email
INSERT INTO customers (customer_id, first_name, last_name, email)
VALUES (2, 'Alice', 'Brown', 'alice.brown@example.com');

-- Invalid: duplicate email
INSERT INTO customers (customer_id, first_name, last_name, email)
VALUES (3, 'Bob', 'Green', 'alice.brown@example.com');
```

**Output (error for second insert):**

```text
ORA-00001: unique constraint (UQ_CUSTOMERS_EMAIL) violated
```

---

## 4. Composite Keys and Constraints

You can define constraints on **multiple columns**.

**Composite primary key example**

```sql
CREATE TABLE enrollment (
    student_id  NUMBER,
    course_id   NUMBER,
    grade       VARCHAR2(2),
    CONSTRAINT pk_enrollment PRIMARY KEY (student_id, course_id)
);
```

**Composite unique example**

```sql
ALTER TABLE orders
ADD CONSTRAINT uq_orders_customer_date UNIQUE (customer_id, order_date);
```

---

## 5. Naming and Inline vs. Out-of-line Constraints

- **Inline**: defined next to the column.
- **Out-of-line**: defined at the end of the column list.

**Example mixing both**

```sql
CREATE TABLE products (
    product_id   NUMBER        CONSTRAINT pk_products PRIMARY KEY,
    sku          VARCHAR2(50),
    name         VARCHAR2(100),
    CONSTRAINT uq_products_sku UNIQUE (sku)
);
```

---

**Main ideas:**
- **Primary key**: unique, not null, identifies each row.
- **Foreign key**: links tables, enforces valid references to parent rows.
- **Unique constraint**: prevents duplicate values in one or more columns.
- Together, they enforce **data integrity** and define **relationships** in an Oracle relational database.
