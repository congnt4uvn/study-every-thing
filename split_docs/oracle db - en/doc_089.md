
Here’s a concise summary of Oracle Advanced Queuing (AQ) basics in PL/SQL & SQL, with simple code examples.

---

## 1. What is Advanced Queuing (AQ)?

- Oracle AQ is a message queuing system built into the database.
- Messages are stored in database tables and processed asynchronously.
- Used for decoupling components, workflow, integration between systems, and reliable message delivery.

Key concepts:
- **Queue table**: Database table that physically stores messages.
- **Queue**: Logical FIFO (or priority-based) structure built on a queue table.
- **Message**: Data unit sent/received via a queue (can be RAW, object type, XML, etc.).
- **Enqueue**: Put a message into a queue.
- **Dequeue**: Get a message from a queue.

---

## 2. Basic Setup: Object Type, Queue Table, Queue

### 2.1 Create a message type

```sql
-- Message payload type
CREATE OR REPLACE TYPE msg_type AS OBJECT (
  id        NUMBER,
  text      VARCHAR2(200)
);
/
```

### 2.2 Create a queue table

```sql
BEGIN
  DBMS_AQADM.CREATE_QUEUE_TABLE(
    queue_table        => 'msg_queue_tab',
    queue_payload_type => 'msg_type'
  );
END;
/
```

### 2.3 Create and start a queue

```sql
BEGIN
  DBMS_AQADM.CREATE_QUEUE(
    queue_name  => 'msg_queue',
    queue_table => 'msg_queue_tab'
  );

  DBMS_AQADM.START_QUEUE(
    queue_name => 'msg_queue'
  );
END;
/
```

---

## 3. Enqueue (Send) Messages

### 3.1 Simple enqueue example

```plsql
DECLARE
  enqueue_options    DBMS_AQ.ENQUEUE_OPTIONS_T;
  message_properties DBMS_AQ.MESSAGE_PROPERTIES_T;
  msg_handle         RAW(16);
  payload            msg_type;
BEGIN
  payload := msg_type(1, 'Hello from AQ');

  DBMS_AQ.ENQUEUE(
    queue_name         => 'msg_queue',
    enqueue_options    => enqueue_options,
    message_properties => message_properties,
    payload            => payload,
    msgid              => msg_handle
  );

  COMMIT;
END;
/
```

**Conceptual input/output:**

- Input: `id = 1`, `text = 'Hello from AQ'`
- Output: Message stored in `msg_queue` with a generated `msgid` (in `msg_handle`).

---

## 4. Dequeue (Receive) Messages

### 4.1 Simple dequeue example

```plsql
DECLARE
  dequeue_options    DBMS_AQ.DEQUEUE_OPTIONS_T;
  message_properties DBMS_AQ.MESSAGE_PROPERTIES_T;
  msg_handle         RAW(16);
  payload            msg_type;
BEGIN
  -- Wait up to 5 seconds for a message
  dequeue_options.wait := 5;

  DBMS_AQ.DEQUEUE(
    queue_name         => 'msg_queue',
    dequeue_options    => dequeue_options,
    message_properties => message_properties,
    payload            => payload,
    msgid              => msg_handle
  );

  DBMS_OUTPUT.PUT_LINE('Dequeued ID: '   || payload.id);
  DBMS_OUTPUT.PUT_LINE('Dequeued Text: ' || payload.text);

  COMMIT;
END;
/
```

**Example output (DBMS_OUTPUT):**

```text
Dequeued ID: 1
Dequeued Text: Hello from AQ
```

## 5. Important Options & Features (Basics)

- **Priority**: Messages can have priorities (higher priority dequeued first).
- **Delay**: Messages can be delayed (become visible only after a time).
- **Expiration**: Messages can expire after a certain time.
- **Visibility**: Control when messages become visible (immediate vs after commit).
- **Retention**: Keep messages after dequeue for history/auditing.

Example: set priority and delay on enqueue:

```plsql
DECLARE
  enqueue_options    DBMS_AQ.ENQUEUE_OPTIONS_T;
  message_properties DBMS_AQ.MESSAGE_PROPERTIES_T;
  msg_handle         RAW(16);
  payload            msg_type;
BEGIN
  payload := msg_type(2, 'High priority, delayed 10 seconds');

  message_properties.priority := 1;      -- 1 = high priority
  message_properties.delay    := 10;     -- seconds

  DBMS_AQ.ENQUEUE(
    queue_name         => 'msg_queue',
    enqueue_options    => enqueue_options,
    message_properties => message_properties,
    payload            => payload,
    msgid              => msg_handle
  );

  COMMIT;
END;
/
```

---

## 6. Cleanup (Optional)

```plsql
BEGIN
  DBMS_AQADM.STOP_QUEUE('msg_queue');
  DBMS_AQADM.DROP_QUEUE('msg_queue');
  DBMS_AQADM.DROP_QUEUE_TABLE('msg_queue_tab', FORCE => TRUE);
END;
/
```

---

**Main ideas:**

- AQ integrates message queuing directly into Oracle DB.
- You define a payload type, create a queue table, then a queue.
- Use `DBMS_AQ.ENQUEUE` to send and `DBMS_AQ.DEQUEUE` to receive.
- Options allow control over priority, delay, expiration, and visibility.
