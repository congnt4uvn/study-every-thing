## 1. `GROUP BY`

**Purpose:**  
Aggregate rows that have the same values in one or more columns, usually used with aggregate functions like `COUNT`, `SUM`, `AVG`, `MIN`, `MAX`.

**Rules:**
- Every selected column must be either:
  - In the `GROUP BY` clause, or
  - An aggregate function result.

**Example: Count employees per department**

```sql
SELECT department_id,
       COUNT(*) AS employee_count
FROM   employees
GROUP  BY department_id;
```

**Sample output:**

| DEPARTMENT_ID | EMPLOYEE_COUNT |
|---------------|----------------|
| 10            | 3              |
| 20            | 5              |
| 30            | 2              |

---

## 2. `HAVING`

**Purpose:**  
Filter groups (not individual rows) after aggregation.  
It is like `WHERE` but for grouped results.

**Key point:**
- `WHERE` filters rows **before** grouping.
- `HAVING` filters groups **after** grouping.

**Example: Departments with more than 3 employees**

```sql
SELECT department_id,
       COUNT(*) AS employee_count
FROM   employees
GROUP  BY department_id
HAVING COUNT(*) > 3;
```

**Sample output:**

| DEPARTMENT_ID | EMPLOYEE_COUNT |
|---------------|----------------|
| 20            | 5              |

**Combined with WHERE (filter rows first, then groups):**

```sql
SELECT department_id,
       COUNT(*) AS high_paid_count
FROM   employees
WHERE  salary > 5000
GROUP  BY department_id
HAVING COUNT(*) >= 2;
```

## 3. `ORDER BY`

**Purpose:**  
Sort the result set.

**Key points:**
- Default is ascending (`ASC`); descending uses `DESC`.
- You can sort by:
  - Column names
  - Column aliases
  - Column positions (1, 2, 3, ŌĆ”) ŌĆō not recommended for readability.

**Example: Sort employees by salary descending**

```sql
SELECT employee_id,
       last_name,
       salary
FROM   employees
ORDER  BY salary DESC;
```

**Sample output:**

| EMPLOYEE_ID | LAST_NAME | SALARY |
|-------------|-----------|--------|
| 101         | King      | 24000  |
| 102         | Kochhar   | 17000  |
| 103         | De Haan   | 17000  |

**Example: Sort grouped results**

```sql
SELECT department_id,
       AVG(salary) AS avg_salary
FROM   employees
GROUP  BY department_id
ORDER  BY avg_salary DESC;
```

---

## 4. Typical Query Order

Logical processing order (how Oracle thinks about it):

1. `FROM`
2. `WHERE`
3. `GROUP BY`
4. `HAVING`
5. `SELECT`
6. `ORDER BY`

**Example combining all:**

```sql
SELECT   department_id,
         COUNT(*)        AS emp_count,
         AVG(salary)     AS avg_salary
FROM     employees
WHERE    hire_date >= DATE '2020-01-01'      -- filter rows
GROUP BY department_id                       -- group rows
HAVING   AVG(salary) > 6000                  -- filter groups
ORDER BY avg_salary DESC;                    -- sort result
```

This returns departments (hired since 2020) whose average salary is greater than 6000, sorted from highest to lowest average salary.
