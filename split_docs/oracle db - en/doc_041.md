
Here’s a concise summary and examples around:

`SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);`

---

## Key Idea

- `DBMS_XPLAN.DISPLAY` is an Oracle built‑in function used to **display the execution plan** of a SQL statement.
- It reads the plan from the **PLAN_TABLE** (usually populated by `EXPLAIN PLAN FOR ...`) and formats it in a human‑readable way.
- Wrapping it with `TABLE(...)` allows you to select its output like a normal table.

---

## Basic Workflow

1. Generate an execution plan for a SQL statement:
   ```sql
   EXPLAIN PLAN FOR
   SELECT * FROM employees WHERE department_id = 10;
   ```

2. Display the plan:
   ```sql
   SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
   ```

---

## Example: Input and Output

### Step 1: Input – Explain Plan

```sql
EXPLAIN PLAN FOR
SELECT first_name, last_name
FROM employees
WHERE department_id = 10
ORDER BY last_name;
```

### Step 2: Input – Display Plan

```sql
SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

### Example Output (simplified)

```text
PLAN_TABLE_OUTPUT
--------------------------------------------------------------------------------
Plan hash value: 123456789

--------------------------------------------------------------------------------
| Id  | Operation          | Name      | Rows  | Bytes | Cost (%CPU)| Time     |
--------------------------------------------------------------------------------
|   0 | SELECT STATEMENT   |           |    10 |   520 |     5   (0)| 00:00:01 |
|   1 |  SORT ORDER BY     |           |    10 |   520 |     5   (0)| 00:00:01 |
|   2 |   TABLE ACCESS FULL| EMPLOYEES |    10 |   520 |     4   (0)| 00:00:01 |
--------------------------------------------------------------------------------
```

Interpretation (high level):

- Oracle will do a **full table scan** on `EMPLOYEES` (step 2),
- then **sort** the result by `last_name` (step 1),
- and return the final result (step 0).

---

## Using Options with DBMS_XPLAN.DISPLAY

You can pass parameters to control what is shown:

```sql
SELECT * 
FROM TABLE(
  DBMS_XPLAN.DISPLAY(
    table_name => 'PLAN_TABLE',  -- usually default
    statement_id => NULL,        -- specific statement id if used
    format => 'TYPICAL'          -- or 'BASIC', 'ALL', 'ADVANCED'
  )
);
```

Common `format` values:

- `'BASIC'` – minimal info
- `'TYPICAL'` – default, useful details
- `'ALL'` – everything (including predicates, notes, etc.)

---

## Summary

- Use `EXPLAIN PLAN FOR <your SQL>` to store the plan.
- Use `SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);` to **view** the plan.
- The output shows operations (e.g., `TABLE ACCESS FULL`, `INDEX RANGE SCAN`), estimated rows, cost, and more, helping you understand and tune query performance.
