---

## Key Idea

For **bulk loading data** into an Oracle database, two common tools are:

1. **SQL\*Loader**  
2. **External Tables**

Both are used to efficiently load large volumes of data (e.g., from CSV or text files) into Oracle tables.

---

## 1. SQL\*Loader

**What it is:**  
A command-line utility that reads data files and loads them into Oracle tables using a control file that describes the format.

### Example

**Data file** (`data.csv`):

```text
101,John,5000
102,Mary,6000
103,Alex,5500
```

**Target table:**

```sql
CREATE TABLE employees (
  emp_id   NUMBER,
  name     VARCHAR2(50),
  salary   NUMBER
);
```

**Control file** (`load.ctl`):

```text
LOAD DATA
INFILE 'data.csv'
INTO TABLE employees
FIELDS TERMINATED BY ','
(
  emp_id   INTEGER EXTERNAL,
  name     CHAR,
  salary   INTEGER EXTERNAL
)
```

**Command to run:**

```bash
sqlldr userid=user/password@ORCL control=load.ctl log=load.log
```

**Result (simplified):**

```sql
SELECT * FROM employees;

EMP_ID | NAME | SALARY
-------+------+-------
101    | John | 5000
102    | Mary | 6000
103    | Alex | 5500
```

---

## 2. External Tables

**What it is:**  
A feature that lets Oracle treat an external file (e.g., CSV) as if it were a table. You can then `SELECT` from it and `INSERT` into real tables.

### Example

**Same data file** (`data.csv`):

```text
101,John,5000
102,Mary,6000
103,Alex,5500
```

**Directory object in Oracle:**

```sql
CREATE OR REPLACE DIRECTORY data_dir AS '/path/to/files';
GRANT READ, WRITE ON DIRECTORY data_dir TO user;
```

**External table definition:**

```sql
CREATE TABLE employees_ext (
  emp_id   NUMBER,
  name     VARCHAR2(50),
  salary   NUMBER
)
ORGANIZATION EXTERNAL (
  TYPE ORACLE_LOADER
  DEFAULT DIRECTORY data_dir
  ACCESS PARAMETERS (
    RECORDS DELIMITED BY NEWLINE
    FIELDS TERMINATED BY ','
  )
  LOCATION ('data.csv')
)
REJECT LIMIT UNLIMITED;
```

**Query external data:**

```sql
SELECT * FROM employees_ext;

-- Output:
-- EMP_ID | NAME | SALARY
-- 101    | John | 5000
-- 102    | Mary | 6000
-- 103    | Alex | 5500
```

**Load into internal table:**

```sql
INSERT INTO employees
SELECT * FROM employees_ext;

COMMIT;
```

---

## When to Use Which

- **SQL\*Loader**:  
  - Best for one-time or batch loads from the OS/command line.  
  - Very fast, lots of options for error handling and performance.

- **External Tables**:  
  - Best when you want SQL access to external files.  
  - Easy to integrate into SQL/PLSQL, can be used in views, joins, etc.
