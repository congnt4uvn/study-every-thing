
Key idea  
In Oracle, if you apply a function to an indexed column in a query, Oracle usually **cannot use the index**—unless you create a **function-based index** on that expression.

---

## 1. Problem: Function on an indexed column

Assume you have a normal index on `last_name`:

```sql
CREATE INDEX idx_emp_last_name ON employees(last_name);
```

Query:

```sql
SELECT *
FROM employees
WHERE UPPER(last_name) = 'SMITH';
```

Even though `last_name` is indexed, Oracle often **won’t use** `idx_emp_last_name` because the predicate is on `UPPER(last_name)`, not directly on `last_name`.

**Effect (simplified):**

- Input: Query with `WHERE UPPER(last_name) = 'SMITH'`
- Output: Oracle may do a **full table scan** instead of an index range scan.

---

## 2. Solution: Function-based index

Create an index on the expression used in the query:

```sql
CREATE INDEX idx_emp_last_name_upper
ON employees (UPPER(last_name));
```

Now the same query:

```sql
SELECT *
FROM employees
WHERE UPPER(last_name) = 'SMITH';
```

Oracle can use `idx_emp_last_name_upper` to quickly find matching rows.

**Effect (simplified):**

- Input: Same query with `WHERE UPPER(last_name) = 'SMITH'`
- Output: Oracle can perform an **index range scan** on `idx_emp_last_name_upper`, improving performance.

---

## 3. Another example: Date functions

Without function-based index:

```sql
SELECT *
FROM orders
WHERE TRUNC(order_date) = DATE '2024-05-01';
```

Index on `order_date` alone may not be used efficiently.

With function-based index:

```sql
CREATE INDEX idx_orders_trunc_date
ON orders (TRUNC(order_date));

SELECT *
FROM orders
WHERE TRUNC(order_date) = DATE '2024-05-01';
```

Now Oracle can use `idx_orders_trunc_date`.

---

## 4. Practical guideline

- Avoid wrapping indexed columns in functions in `WHERE`/`JOIN` conditions **unless**:
  - You create a **function-based index** on that exact expression, or
  - You can rewrite the predicate to use the column directly.

Example rewrite (no function-based index needed):

```sql
-- Instead of:
SELECT *
FROM orders
WHERE TRUNC(order_date) = DATE '2024-05-01';

-- Use range on the raw column:
SELECT *
FROM orders
WHERE order_date >= DATE '2024-05-01'
  AND order_date <  DATE '2024-05-02';
```

This can use a normal index on `order_date`.
