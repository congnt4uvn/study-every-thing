
Here’s a concise summary of the key subquery types in Oracle SQL, with examples and sample outputs.

---

## 1. Scalar Subquery

**Definition:**  
A subquery that returns exactly one value (one row, one column). It can be used wherever a single value is allowed (SELECT list, WHERE, ORDER BY, etc.).

**Example:**
```sql
-- Find each employee and the average salary of all employees
SELECT 
    e.employee_id,
    e.first_name,
    e.salary,
    (SELECT AVG(salary) FROM employees) AS avg_salary
FROM employees e;
```

**Sample output:**
| EMPLOYEE_ID | FIRST_NAME | SALARY | AVG_SALARY |
|------------:|------------|--------|-----------:|
| 100         | Steven     | 24000  | 6461.83    |
| 101         | Neena      | 17000  | 6461.83    |
| ...         | ...        | ...    | 6461.83    |

The scalar subquery `(SELECT AVG(salary) FROM employees)` returns one value used for every row.

---

## 2. Correlated Subquery

**Definition:**  
A subquery that depends on values from the outer query. It is executed once for each row of the outer query.

**Example:**
```sql
-- List employees whose salary is greater than the average salary of their department
SELECT 
    e.employee_id,
    e.first_name,
    e.department_id,
    e.salary
FROM employees e
WHERE e.salary > (
    SELECT AVG(e2.salary)
    FROM employees e2
    WHERE e2.department_id = e.department_id
);
```

Here, `e.department_id` in the subquery refers to the current row of the outer query, so the subquery is “correlated” with the outer query.

**Sample output:**
| EMPLOYEE_ID | FIRST_NAME | DEPARTMENT_ID | SALARY |
|------------:|------------|--------------:|-------:|
| 102         | Lex        | 90            | 17000  |
| 145         | John       | 50            | 8200   |
| ...         | ...        | ...           | ...    |

---

## 3. EXISTS / NOT EXISTS

**Definition:**  
- `EXISTS` checks whether the subquery returns at least one row.  
- `NOT EXISTS` checks that the subquery returns no rows.  
The subquery usually uses a correlation to the outer query.

**EXISTS Example:**
```sql
-- Find departments that have at least one employee
SELECT d.department_id, d.department_name
FROM departments d
WHERE EXISTS (
    SELECT 1
    FROM employees e
    WHERE e.department_id = d.department_id
);
```

**Sample output:**
| DEPARTMENT_ID | DEPARTMENT_NAME |
|--------------:|-----------------|
| 10            | Administration   |
| 20            | Marketing        |
| ...           | ...             |

**NOT EXISTS Example:**
```sql
-- Find departments that have no employees
SELECT d.department_id, d.department_name
FROM departments d
WHERE NOT EXISTS (
    SELECT 1
    FROM employees e
    WHERE e.department_id = d.department_id
);
```

**Sample output:**
| DEPARTMENT_ID | DEPARTMENT_NAME |
|--------------:|-----------------|
| 190           | Contracting     |

---

## 4. IN / NOT IN

**Definition:**  
- `IN` checks if a value is in a list (or in the result set of a subquery).  
- `NOT IN` checks that a value is not in that list/result set.

**IN Example (subquery):**
```sql
-- Find employees who work in departments located in 'Seattle'
SELECT e.employee_id, e.first_name, e.department_id
FROM employees e
WHERE e.department_id IN (
    SELECT d.department_id
    FROM departments d
    JOIN locations l ON d.location_id = l.location_id
    WHERE l.city = 'Seattle'
);
```

**Sample output:**
| EMPLOYEE_ID | FIRST_NAME | DEPARTMENT_ID |
|------------:|------------|--------------:|
| 201         | Michael    | 110           |
| 202         | Pat        | 110           |

**NOT IN Example:**
```sql
-- Find employees who are not in any department that has a manager
SELECT e.employee_id, e.first_name, e.department_id
FROM employees e
WHERE e.department_id NOT IN (
    SELECT d.department_id
    FROM departments d
    WHERE d.manager_id IS NOT NULL
);
```

**Important note:**  
If the subquery for `NOT IN` returns `NULL`, the comparison can become unknown and may return no rows. `NOT EXISTS` is often safer when `NULL`s are possible.

---

## Summary of Main Ideas

- **Scalar subquery**: returns a single value; used like a column or constant.
- **Correlated subquery**: references outer query columns; runs once per outer row.
- **EXISTS / NOT EXISTS**: tests presence/absence of rows; often more robust with `NULL`s.
- **IN / NOT IN**: tests membership in a set; `NOT IN` can be tricky with `NULL`s.
