
Here’s a concise summary and practical examples for **Advanced PL/SQL: `EXECUTE IMMEDIATE`**.

---

## 1. What `EXECUTE IMMEDIATE` Is

- A PL/SQL statement used for **dynamic SQL**.
- Lets you build SQL/PLSQL statements as **strings at runtime** and then execute them.
- Useful when:
  - Object names (tables, columns) are not known until runtime.
  - You need to build flexible, parameterized queries.
  - You must execute DDL (CREATE, ALTER, DROP) inside PL/SQL.

Basic form:

```plsql
EXECUTE IMMEDIATE <dynamic_string>;
```

---

## 2. Common Use Cases

### 2.1 Dynamic DDL

```plsql
BEGIN
  EXECUTE IMMEDIATE 'CREATE TABLE temp_log (id NUMBER, msg VARCHAR2(100))';
END;
/
```

No input/output variables here; just run the statement.

---

### 2.2 Dynamic DML with Bind Variables

Use `USING` to pass values safely (avoids SQL injection and improves performance).

```plsql
DECLARE
  v_sql   VARCHAR2(200);
  v_id    NUMBER := 10;
  v_name  VARCHAR2(50) := 'Scott';
BEGIN
  v_sql := 'INSERT INTO employees (employee_id, first_name) VALUES (:1, :2)';

  EXECUTE IMMEDIATE v_sql
    USING v_id, v_name;
END;
/
```

**Input:**
- `v_id = 10`
- `v_name = 'Scott'`

**Effect (Output in table):**
- New row in `employees` with `employee_id = 10`, `first_name = 'Scott'`.

---

### 2.3 Returning Values (INTO Clause)

For `SELECT` that returns a single row:

```plsql
DECLARE
  v_sql   VARCHAR2(200);
  v_name  employees.first_name%TYPE;
  v_id    employees.employee_id%TYPE := 10;
BEGIN
  v_sql := 'SELECT first_name FROM employees WHERE employee_id = :id';

  EXECUTE IMMEDIATE v_sql
    INTO v_name
    USING v_id;

  DBMS_OUTPUT.PUT_LINE('Name = ' || v_name);
END;
/
```

**Input:**
- `employee_id = 10`

**Output (DBMS_OUTPUT):**
- `Name = Scott` (for example)

---

### 2.4 Dynamic UPDATE with RETURNING

```plsql
DECLARE
  v_sql   VARCHAR2(200);
  v_id    employees.employee_id%TYPE := 10;
  v_old   employees.salary%TYPE;
BEGIN
  v_sql := 'UPDATE employees SET salary = salary * 1.1 ' ||
           'WHERE employee_id = :id RETURNING salary INTO :new_sal';

  EXECUTE IMMEDIATE v_sql
    RETURNING INTO v_old
    USING v_id;

  DBMS_OUTPUT.PUT_LINE('New salary = ' || v_old);
END;
/
```

**Input:**
- `employee_id = 10`

**Output:**
- `New salary = <updated_salary_value>`

---

### 2.5 Dynamic WHERE Conditions

```plsql
DECLARE
  v_sql   VARCHAR2(400);
  v_min   NUMBER := 3000;
  v_cnt   NUMBER;
BEGIN
  v_sql := 'SELECT COUNT(*) FROM employees WHERE salary > :min_sal';

  EXECUTE IMMEDIATE v_sql
    INTO v_cnt
    USING v_min;

  DBMS_OUTPUT.PUT_LINE('Employees with salary > ' || v_min || ' = ' || v_cnt);
END;
/
```

## 3. Using `OPEN-FOR` for Multi-Row Queries

For queries returning multiple rows, use a REF CURSOR:

```plsql
DECLARE
  TYPE ref_cur IS REF CURSOR;
  c   ref_cur;
  v_sql VARCHAR2(400);
  v_id   employees.employee_id%TYPE;
  v_name employees.first_name%TYPE;
BEGIN
  v_sql := 'SELECT employee_id, first_name FROM employees WHERE department_id = :dept';

  OPEN c FOR v_sql USING 10;  -- department_id = 10

  LOOP
    FETCH c INTO v_id, v_name;
    EXIT WHEN c%NOTFOUND;
    DBMS_OUTPUT.PUT_LINE(v_id || ' - ' || v_name);
  END LOOP;

  CLOSE c;
END;
/
```

**Input:**
- `department_id = 10`

**Output (DBMS_OUTPUT):**
- Multiple lines like: `101 - John`, `102 - Mary`, etc.

---

## 4. Key Points / Best Practices

- Always use **bind variables** (`USING`) instead of concatenating values into the SQL string.
- Use `INTO` for single-row `SELECT` statements.
- Use `RETURNING INTO` for DML that needs to return values.
- Use `OPEN-FOR` with REF CURSOR for multi-row results.
- Dynamic SQL is powerful but can be:
  - Harder to debug.
  - Vulnerable to SQL injection if you concatenate user input directly.
  - Slower if you build many different SQL strings (less chance of reuse in shared pool).

This covers the main ideas and typical patterns for advanced use of `EXECUTE IMMEDIATE` in PL/SQL.
