
Key idea  
Get the “top N rows per group” (e.g., top 3 salaries in each department) using analytic functions like `ROW_NUMBER`, `RANK`, or `DENSE_RANK`.

---

## 1. Using ROW_NUMBER (most common)

`ROW_NUMBER` assigns a unique sequence within each group, ordered by salary. Then you filter by that sequence.

```sql
-- Top 3 salaries per department
SELECT *
FROM (
    SELECT
        department_id,
        employee_id,
        salary,
        ROW_NUMBER() OVER (
            PARTITION BY department_id
            ORDER BY salary DESC
        ) AS rn
    FROM employees
) t
WHERE t.rn <= 3
ORDER BY department_id, salary DESC;
```

**Input (table `employees`):**

| employee_id | department_id | salary |
|------------:|--------------:|-------:|
| 1           | 10            | 5000   |
| 2           | 10            | 7000   |
| 3           | 10            | 6000   |
| 4           | 10            | 7000   |
| 5           | 20            | 4000   |
| 6           | 20            | 4500   |
| 7           | 20            | 4500   |
| 8           | 20            | 3000   |

**Output (top 3 per department):**

| department_id | employee_id | salary | rn |
|--------------:|------------:|-------:|---:|
| 10            | 2           | 7000   | 1 |
| 10            | 4           | 7000   | 2 |
| 10            | 3           | 6000   | 3 |
| 20            | 6           | 4500   | 1 |
| 20            | 7           | 4500   | 2 |
| 20            | 5           | 4000   | 3 |

---

## 2. Using RANK (include ties, may return more than N rows)

If you want all employees that share the same salary rank (e.g., if 3rd and 4th have same salary, include both):

```sql
SELECT *
FROM (
    SELECT
        department_id,
        employee_id,
        salary,
        RANK() OVER (
            PARTITION BY department_id
            ORDER BY salary DESC
        ) AS rnk
    FROM employees
) t
WHERE t.rnk <= 3
ORDER BY department_id, salary DESC;
```

This can return more than 3 rows per department if there are ties at the boundary.

---

## 3. Using DENSE_RANK (similar to RANK but no gaps)

`DENSE_RANK` behaves like `RANK` but doesn’t skip rank numbers when there are ties:

```sql
SELECT *
FROM (
    SELECT
        department_id,
        employee_id,
        salary,
        DENSE_RANK() OVER (
            PARTITION BY department_id
            ORDER BY salary DESC
        ) AS drnk
    FROM employees
) t
WHERE t.drnk <= 3
ORDER BY department_id, salary DESC;
```

---

**Summary**

- Use `ROW_NUMBER` when you want *exactly* N rows per group.
- Use `RANK`/`DENSE_RANK` when you want to include all ties at each rank, possibly returning more than N rows per group.
- Pattern: analytic function in a subquery → filter on the generated rank/row number.
