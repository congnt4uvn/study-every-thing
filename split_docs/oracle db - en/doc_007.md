
Here’s a concise summary and examples for the advanced SQL analytic functions: `ROW_NUMBER`, `RANK`, and `DENSE_RANK` in Oracle.

---

## 1. ROW_NUMBER

**Purpose:**  
Assigns a unique sequential number to each row within a partition, based on the specified ordering.  
No ties: even if values are equal, each row gets a different number.

**Syntax:**
```sql
ROW_NUMBER() OVER (
  [PARTITION BY partition_expression]
  ORDER BY sort_expression
)
```

**Example table: EMP_SALES**

| EMP_ID | DEPT | SALES |
|--------|------|-------|
| 101    | A    | 5000  |
| 102    | A    | 7000  |
| 103    | A    | 7000  |
| 201    | B    | 6000  |
| 202    | B    | 6000  |

**Query:**
```sql
SELECT
  EMP_ID,
  DEPT,
  SALES,
  ROW_NUMBER() OVER (
    PARTITION BY DEPT
    ORDER BY SALES DESC
  ) AS rn
FROM EMP_SALES
ORDER BY DEPT, rn;
```

**Output:**

| EMP_ID | DEPT | SALES | RN |
|--------|------|-------|----|
| 102    | A    | 7000  | 1  |
| 103    | A    | 7000  | 2  |
| 101    | A    | 5000  | 3  |
| 201    | B    | 6000  | 1  |
| 202    | B    | 6000  | 2  |

Use case: pick “top 1 per group” (e.g., best salesperson per department).

```sql
SELECT *
FROM (
  SELECT
    EMP_ID,
    DEPT,
    SALES,
    ROW_NUMBER() OVER (
      PARTITION BY DEPT
      ORDER BY SALES DESC
    ) AS rn
  FROM EMP_SALES
)
WHERE rn = 1;
```

## 2. RANK

**Purpose:**  
Assigns a rank to each row within a partition.  
Rows with the same ordering value get the same rank, and **gaps** appear after ties.

**Syntax:**
```sql
RANK() OVER (
  [PARTITION BY partition_expression]
  ORDER BY sort_expression
)
```

**Example:**
```sql
SELECT
  EMP_ID,
  DEPT,
  SALES,
  RANK() OVER (
    PARTITION BY DEPT
    ORDER BY SALES DESC
  ) AS rnk
FROM EMP_SALES
ORDER BY DEPT, rnk, EMP_ID;
```

**Output:**

| EMP_ID | DEPT | SALES | RNK |
|--------|------|-------|-----|
| 102    | A    | 7000  | 1   |
| 103    | A    | 7000  | 1   |
| 101    | A    | 5000  | 3   |
| 201    | B    | 6000  | 1   |
| 202    | B    | 6000  | 1   |

Note: After rank 1 (two rows), the next rank is 3, not 2.

Use case: ranking with ties where you want to reflect “competition ranking” (1,1,3,…).

---

## 3. DENSE_RANK

**Purpose:**  
Similar to `RANK`, but **no gaps** in ranking after ties.

**Syntax:**
```sql
DENSE_RANK() OVER (
  [PARTITION BY partition_expression]
  ORDER BY sort_expression
)
```

**Example:**
```sql
SELECT
  EMP_ID,
  DEPT,
  SALES,
  DENSE_RANK() OVER (
    PARTITION BY DEPT
    ORDER BY SALES DESC
  ) AS drnk
FROM EMP_SALES
ORDER BY DEPT, drnk, EMP_ID;
```

**Output:**

| EMP_ID | DEPT | SALES | DRNK |
|--------|------|-------|------|
| 102    | A    | 7000  | 1    |
| 103    | A    | 7000  | 1    |
| 101    | A    | 5000  | 2    |
| 201    | B    | 6000  | 1    |
| 202    | B    | 6000  | 1    |

Note: After rank 1, the next rank is 2 (no gap).

Use case: “dense” ranking for categories (1st group, 2nd group, 3rd group) without skipping numbers.

---

## 4. Quick Comparison

Using `ORDER BY SALES DESC`:

- `ROW_NUMBER`  
  - 7000 → 1, 7000 → 2, 5000 → 3  
  - Always unique, no ties.

- `RANK`  
  - 7000 → 1, 7000 → 1, 5000 → 3  
  - Ties share rank, gaps after ties.

- `DENSE_RANK`  
  - 7000 → 1, 7000 → 1, 5000 → 2  
  - Ties share rank, no gaps.
