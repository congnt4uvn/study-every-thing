
Here’s a concise summary of PL/SQL cursor basics, with code examples and sample input/output.

---

## 1. Implicit Cursors

Oracle automatically creates an **implicit cursor** for single SQL statements (like `INSERT`, `UPDATE`, `DELETE`, `SELECT ... INTO`).

### Key Points
- No need to declare or open/close.
- Use `SQL%ROWCOUNT`, `SQL%FOUND`, `SQL%NOTFOUND`, `SQL%ISOPEN`.
- Used for statements that affect or return **one row** (or you expect one row).

### Example: SELECT ... INTO (single row)

```sql
DECLARE
    v_emp_name   employees.last_name%TYPE;
    v_emp_id     employees.employee_id%TYPE := 100;  -- input
BEGIN
    SELECT last_name
    INTO   v_emp_name
    FROM   employees
    WHERE  employee_id = v_emp_id;

    DBMS_OUTPUT.PUT_LINE('Employee name: ' || v_emp_name);
    DBMS_OUTPUT.PUT_LINE('Rows processed: ' || SQL%ROWCOUNT);
END;
/
```

**Input:** `v_emp_id = 100`  
**Output (example):**
```
Employee name: King
Rows processed: 1
```

## 2. Explicit Cursors

Use an **explicit cursor** when:
- A `SELECT` returns **multiple rows**, and
- You want to process them **row by row** manually.

### Key Points
- Steps: `DECLARE` → `OPEN` → `FETCH` → `CLOSE`.
- You control the loop and fetch logic.
- Use cursor attributes: `c_emp%FOUND`, `c_emp%NOTFOUND`, `c_emp%ROWCOUNT`, `c_emp%ISOPEN`.

### Example: Explicit Cursor with Manual Loop

```sql
DECLARE
    CURSOR c_emp IS
        SELECT employee_id, last_name, salary
        FROM   employees
        WHERE  department_id = 10;  -- input condition

    v_emp_id   employees.employee_id%TYPE;
    v_name     employees.last_name%TYPE;
    v_salary   employees.salary%TYPE;
BEGIN
    OPEN c_emp;

    LOOP
        FETCH c_emp INTO v_emp_id, v_name, v_salary;
        EXIT WHEN c_emp%NOTFOUND;

        DBMS_OUTPUT.PUT_LINE(
            'ID: ' || v_emp_id ||
            ', Name: ' || v_name ||
            ', Salary: ' || v_salary
        );
    END LOOP;

    CLOSE c_emp;
END;
/
```

**Input:** `department_id = 10`  
**Output (example):**
```
ID: 200, Name: Whalen, Salary: 4400
ID: 201, Name: Hartstein, Salary: 13000
...
```

## 3. Cursor FOR Loops

A **cursor FOR loop** is a simpler way to use explicit cursors. PL/SQL:
- Declares the loop variable(s) for you.
- Opens, fetches, and closes the cursor automatically.

### Key Points
- No explicit `OPEN`, `FETCH`, `CLOSE`.
- Loop variable is a record that matches the cursor’s SELECT list.
- Less code, fewer errors.

### Example 1: Cursor FOR Loop with Named Cursor

```sql
DECLARE
    CURSOR c_emp IS
        SELECT employee_id, last_name, salary
        FROM   employees
        WHERE  department_id = 10;
BEGIN
    FOR r_emp IN c_emp LOOP
        DBMS_OUTPUT.PUT_LINE(
            'ID: ' || r_emp.employee_id ||
            ', Name: ' || r_emp.last_name ||
            ', Salary: ' || r_emp.salary
        );
    END LOOP;
END;
/
```

**Output (example):**
```
ID: 200, Name: Whalen, Salary: 4400
ID: 201, Name: Hartstein, Salary: 13000
...
```

### Example 2: Cursor FOR Loop with Inline Query

```sql
BEGIN
    FOR r_emp IN (
        SELECT employee_id, last_name, salary
        FROM   employees
        WHERE  salary > 10000
    ) LOOP
        DBMS_OUTPUT.PUT_LINE(
            'High earner: ' || r_emp.last_name ||
            ' (ID ' || r_emp.employee_id || '), Salary: ' || r_emp.salary
        );
    END LOOP;
END;
/
```

**Output (example):**
```
High earner: King (ID 100), Salary: 24000
High earner: Kochhar (ID 101), Salary: 17000
...
```

---

## Summary of Differences

- **Implicit cursor**
  - Automatic for single SQL statements.
  - Use `SQL%` attributes.
  - Good for single-row `SELECT ... INTO` and DML.

- **Explicit cursor**
  - You declare and control it.
  - Use when processing multi-row queries manually.

- **Cursor FOR loop**
  - Simplified explicit cursor.
  - Oracle handles open/fetch/close.
  - Recommended for most multi-row processing.
