
Key idea:  
Design an audit trail that records who changed what, when, and how, for critical tables (e.g., financial, security, or configuration tables). This is usually done with audit tables and triggers (or Oracle features like Flashback/Data Guard, etc.).

---

## Main Points

1. **Identify critical tables and columns**
   - Not all tables need auditing; focus on those with sensitive or business‑critical data.
   - Sometimes only specific columns need to be audited (e.g., salary, status, limits).

2. **Create dedicated audit tables**
   - Separate audit tables from main tables.
   - Include:
     - Primary key of the original row
     - Operation type (INSERT/UPDATE/DELETE)
     - Old values and/or new values
     - Who did it (user/session info)
     - When it happened (timestamp)
     - Optional: application/module, IP, reason code, etc.

3. **Use triggers to capture changes**
   - Row‑level triggers on INSERT/UPDATE/DELETE.
   - Before/after triggers depending on what you need (old vs new values).
   - Store changes in the audit table.

4. **Capture context information**
   - Use `USER`, `SYS_CONTEXT('USERENV', ...)` to get:
     - Database user
     - OS user
     - Machine name
     - Module, client identifier, etc.

5. **Performance and storage considerations**
   - Auditing can be heavy; audit only what’s necessary.
   - Consider partitioning audit tables and purging old data.
   - Avoid complex logic in triggers to reduce overhead.

6. **Security of audit data**
   - Restrict access to audit tables.
   - Prevent users from modifying or deleting audit records.
   - Possibly store audit data in a separate schema or database.

7. **Use built‑in Oracle features where appropriate**
   - Oracle Unified Auditing (for system-level and object-level actions).
   - Flashback Data Archive (for historical row versions).
   - LogMiner / GoldenGate for redo-based auditing.

## Example: Simple Audit Trail with Triggers

### 1. Base table

```sql
CREATE TABLE employees (
    emp_id       NUMBER PRIMARY KEY,
    first_name   VARCHAR2(50),
    last_name    VARCHAR2(50),
    salary       NUMBER,
    last_update  DATE
);
```

### 2. Audit table

```sql
CREATE TABLE employees_audit (
    audit_id        NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    emp_id          NUMBER,
    operation       VARCHAR2(10),      -- 'INSERT', 'UPDATE', 'DELETE'
    old_first_name  VARCHAR2(50),
    old_last_name   VARCHAR2(50),
    old_salary      NUMBER,
    new_first_name  VARCHAR2(50),
    new_last_name   VARCHAR2(50),
    new_salary      NUMBER,
    changed_by      VARCHAR2(30),
    changed_on      DATE,
    os_user         VARCHAR2(30),
    machine         VARCHAR2(64),
    module          VARCHAR2(48)
);
```

### 3. Trigger to populate audit table

```sql
CREATE OR REPLACE TRIGGER trg_employees_audit
AFTER INSERT OR UPDATE OR DELETE ON employees
FOR EACH ROW
DECLARE
    v_os_user  VARCHAR2(30);
    v_machine  VARCHAR2(64);
    v_module   VARCHAR2(48);
BEGIN
    v_os_user := SYS_CONTEXT('USERENV', 'OS_USER');
    v_machine := SYS_CONTEXT('USERENV', 'HOST');
    v_module  := SYS_CONTEXT('USERENV', 'MODULE');

    IF INSERTING THEN
        INSERT INTO employees_audit (
            emp_id, operation,
            new_first_name, new_last_name, new_salary,
            changed_by, changed_on,
            os_user, machine, module
        ) VALUES (
            :NEW.emp_id, 'INSERT',
            :NEW.first_name, :NEW.last_name, :NEW.salary,
            USER, SYSDATE,
            v_os_user, v_machine, v_module
        );

    ELSIF UPDATING THEN
        INSERT INTO employees_audit (
            emp_id, operation,
            old_first_name, old_last_name, old_salary,
            new_first_name, new_last_name, new_salary,
            changed_by, changed_on,
            os_user, machine, module
        ) VALUES (
            :OLD.emp_id, 'UPDATE',
            :OLD.first_name, :OLD.last_name, :OLD.salary,
            :NEW.first_name, :NEW.last_name, :NEW.salary,
            USER, SYSDATE,
            v_os_user, v_machine, v_module
        );

    ELSIF DELETING THEN
        INSERT INTO employees_audit (
            emp_id, operation,
            old_first_name, old_last_name, old_salary,
            changed_by, changed_on,
            os_user, machine, module
        ) VALUES (
            :OLD.emp_id, 'DELETE',
            :OLD.first_name, :OLD.last_name, :OLD.salary,
            USER, SYSDATE,
            v_os_user, v_machine, v_module
        );
    END IF;
END;
/
```

## Example: Inputs / Outputs

### Input 1: Insert

```sql
INSERT INTO employees (emp_id, first_name, last_name, salary, last_update)
VALUES (1001, 'John', 'Doe', 5000, SYSDATE);
```

**Audit row created (simplified):**

```sql
SELECT emp_id, operation, new_first_name, new_salary, changed_by
FROM employees_audit
WHERE emp_id = 1001;

-- Output:
-- EMP_ID | OPERATION | NEW_FIRST_NAME | NEW_SALARY | CHANGED_BY
-- 1001   | INSERT    | John           | 5000       | HR_USER
```

### Input 2: Update

```sql
UPDATE employees
SET salary = 5500
WHERE emp_id = 1001;
```

**Audit row created (simplified):**

```sql
SELECT emp_id, operation, old_salary, new_salary, changed_by
FROM employees_audit
WHERE emp_id = 1001
ORDER BY audit_id;

-- Output (last row):
-- EMP_ID | OPERATION | OLD_SALARY | NEW_SALARY | CHANGED_BY
-- 1001   | UPDATE    | 5000       | 5500       | HR_USER
```

### Input 3: Delete

```sql
DELETE FROM employees
WHERE emp_id = 1001;
```

**Audit row created (simplified):**

```sql
SELECT emp_id, operation, old_salary, changed_by
FROM employees_audit
WHERE emp_id = 1001
ORDER BY audit_id;

-- Output (last row):
-- EMP_ID | OPERATION | OLD_SALARY | CHANGED_BY
-- 1001   | DELETE    | 5500       | HR_USER
```

## Summary

- Design an audit trail by:
  - Choosing critical tables/columns.
  - Creating structured audit tables.
  - Using triggers (or Oracle auditing features) to capture changes.
  - Recording who, what, when, and how.
  - Managing performance, storage, and security of audit data.
