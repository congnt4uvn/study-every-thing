
Here’s a concise summary of the main SQL join types in Oracle, with examples and sample outputs.

---

## 1. INNER JOIN
Returns only rows where there is a match in **both** tables.

**Syntax:**
```sql
SELECT e.employee_id,
       e.first_name,
       d.department_name
FROM   employees e
       INNER JOIN departments d
           ON e.department_id = d.department_id;
```

**Conceptual Input:**

`employees`  

| employee_id | first_name | department_id |
|------------:|------------|--------------:|
| 101         | John       | 10            |
| 102         | Mary       | 20            |
| 103         | Alex       | 30            |

`departments`  

| department_id | department_name |
|--------------:|-----------------|
| 10            | Sales           |
| 20            | HR              |

**Output (INNER JOIN):**

| employee_id | first_name | department_name |
|------------:|------------|-----------------|
| 101         | John       | Sales           |
| 102         | Mary       | HR              |

Employee 103 is excluded because department 30 has no match.

---

## 2. LEFT OUTER JOIN
Returns **all rows from the left table**, and matching rows from the right table. Non-matching rows from the right table are `NULL`.

**Syntax:**
```sql
SELECT e.employee_id,
       e.first_name,
       d.department_name
FROM   employees e
       LEFT OUTER JOIN departments d
           ON e.department_id = d.department_id;
```

**Output (LEFT JOIN):**

| employee_id | first_name | department_name |
|------------:|------------|-----------------|
| 101         | John       | Sales           |
| 102         | Mary       | HR              |
| 103         | Alex       | NULL            |

All employees are shown; Alex has no department match.

---

## 3. RIGHT OUTER JOIN
Returns **all rows from the right table**, and matching rows from the left table. Non-matching rows from the left table are `NULL`.

**Syntax:**
```sql
SELECT e.employee_id,
       e.first_name,
       d.department_name
FROM   employees e
       RIGHT OUTER JOIN departments d
           ON e.department_id = d.department_id;
```

If `departments` has an extra row:

| department_id | department_name |
|--------------:|-----------------|
| 10            | Sales           |
| 20            | HR              |
| 40            | IT              |

**Output (RIGHT JOIN):**

| employee_id | first_name | department_name |
|------------:|------------|-----------------|
| 101         | John       | Sales           |
| 102         | Mary       | HR              |
| NULL        | NULL       | IT              |

Department 40 has no employee, so employee columns are `NULL`.

---

## 4. FULL OUTER JOIN
Returns **all rows from both tables**. Where there is no match, the missing side is `NULL`.

**Syntax:**
```sql
SELECT e.employee_id,
       e.first_name,
       d.department_name
FROM   employees e
       FULL OUTER JOIN departments d
           ON e.department_id = d.department_id;
```

**Output (FULL JOIN):**

| employee_id | first_name | department_name |
|------------:|------------|-----------------|
| 101         | John       | Sales           |
| 102         | Mary       | HR              |
| 103         | Alex       | NULL            |
| NULL        | NULL       | IT              |

Includes unmatched rows from both tables.

---

## 5. CROSS JOIN
Returns the **Cartesian product**: every row from the first table combined with every row from the second table.

**Syntax:**
```sql
SELECT e.employee_id,
       e.first_name,
       d.department_name
FROM   employees e
       CROSS JOIN departments d;
```

If `employees` has 3 rows and `departments` has 2 rows, the result has `3 × 2 = 6` rows.

**Example Output (3 employees × 2 departments):**

| employee_id | first_name | department_name |
|------------:|------------|-----------------|
| 101         | John       | Sales           |
| 101         | John       | HR              |
| 102         | Mary       | Sales           |
| 102         | Mary       | HR              |
| 103         | Alex       | Sales           |
| 103         | Alex       | HR              |

---

## 6. SELF JOIN
A table is joined **to itself**. Useful for hierarchical or relational data within the same table (e.g., employees and their managers).

**Syntax:**
```sql
SELECT e.employee_id,
       e.first_name       AS employee_name,
       m.employee_id      AS manager_id,
       m.first_name       AS manager_name
FROM   employees e
       LEFT JOIN employees m
           ON e.manager_id = m.employee_id;
```

**Conceptual Input (`employees`):**

| employee_id | first_name | manager_id |
|------------:|------------|-----------:|
| 101         | John       | NULL       |
| 102         | Mary       | 101        |
| 103         | Alex       | 101        |

**Output (SELF JOIN):**

| employee_id | employee_name | manager_id | manager_name |
|------------:|---------------|-----------:|--------------|
| 101         | John          | NULL       | NULL         |
| 102         | Mary          | 101        | John         |
| 103         | Alex          | 101        | John         |

---

**Key Ideas:**
- **INNER**: only matches.
- **LEFT/RIGHT OUTER**: all from one side + matches.
- **FULL OUTER**: all from both sides.
- **CROSS**: all combinations (Cartesian product).
- **SELF**: table joined with itself (e.g., parent–child, manager–employee).
