
Here’s a concise summary and examples about **overloading procedures/functions in PL/SQL**.

---

## Key Ideas

1. **Overloading** means having **multiple procedures or functions with the same name** but **different parameter lists** in the same scope (usually a package).
2. Overloading is resolved at **compile time** based on:
   - Number of parameters
   - Parameter data types
   - Parameter modes (IN, OUT, IN OUT) can also matter
3. Overloading is commonly used to:
   - Provide **different ways to call** the same logical operation
   - Handle **different input types** (e.g., ID vs. name)
   - Maintain **backward compatibility** when changing interfaces
4. Overloading is typically done in **packages** (specification + body).

---

## Basic Overloaded Procedures Example

### Package Specification

```sql
CREATE OR REPLACE PACKAGE emp_api AS
  -- Overload 1: add employee by ID and name
  PROCEDURE add_emp(p_emp_id   IN NUMBER,
                    p_emp_name IN VARCHAR2);

  -- Overload 2: add employee by record
  TYPE emp_rec IS RECORD (
    emp_id   NUMBER,
    emp_name VARCHAR2
  );

  PROCEDURE add_emp(p_emp IN emp_rec);
END emp_api;
/
```

### Package Body

```sql
CREATE OR REPLACE PACKAGE BODY emp_api AS

  PROCEDURE add_emp(p_emp_id   IN NUMBER,
                    p_emp_name IN VARCHAR2) IS
  BEGIN
    INSERT INTO employees (emp_id, emp_name)
    VALUES (p_emp_id, p_emp_name);
  END add_emp;

  PROCEDURE add_emp(p_emp IN emp_rec) IS
  BEGIN
    INSERT INTO employees (emp_id, emp_name)
    VALUES (p_emp.emp_id, p_emp.emp_name);
  END add_emp;

END emp_api;
/
```

### Example Calls (Inputs/Outputs)

```sql
-- Input 1: call with separate parameters
BEGIN
  emp_api.add_emp(p_emp_id   => 1001,
                  p_emp_name => 'Alice');
END;
/

-- Input 2: call with record
DECLARE
  l_emp emp_api.emp_rec;
BEGIN
  l_emp.emp_id   := 1002;
  l_emp.emp_name := 'Bob';

  emp_api.add_emp(l_emp);
END;
/
```

**Output effect** (in table `employees`):

```sql
SELECT emp_id, emp_name FROM employees WHERE emp_id IN (1001, 1002);

-- Possible result:
-- EMP_ID  EMP_NAME
-- 1001    Alice
-- 1002    Bob
```

## Overloaded Functions Example

### Package Specification

```sql
CREATE OR REPLACE PACKAGE math_utils AS
  FUNCTION area(p_radius IN NUMBER) RETURN NUMBER;      -- circle
  FUNCTION area(p_width  IN NUMBER,
                p_height IN NUMBER) RETURN NUMBER;      -- rectangle
END math_utils;
/
```

### Package Body

```sql
CREATE OR REPLACE PACKAGE BODY math_utils AS

  FUNCTION area(p_radius IN NUMBER) RETURN NUMBER IS
  BEGIN
    RETURN 3.14159 * p_radius * p_radius;
  END area;

  FUNCTION area(p_width  IN NUMBER,
                p_height IN NUMBER) RETURN NUMBER IS
  BEGIN
    RETURN p_width * p_height;
  END area;

END math_utils;
/
```

### Example Calls (Inputs/Outputs)

```sql
DECLARE
  l_circle_area    NUMBER;
  l_rectangle_area NUMBER;
BEGIN
  l_circle_area    := math_utils.area(5);        -- radius = 5
  l_rectangle_area := math_utils.area(4, 6);     -- width=4, height=6

  DBMS_OUTPUT.PUT_LINE('Circle area: '    || l_circle_area);
  DBMS_OUTPUT.PUT_LINE('Rectangle area: ' || l_rectangle_area);
END;
/
```

**Output:**

```text
Circle area: 78.53975
Rectangle area: 24
```

---

## Important Notes

- Overloaded subprograms **must differ in their parameter list**; changing only the return type is **not enough**.
- Avoid ambiguous overloads (e.g., `NUMBER` vs. `PLS_INTEGER` with NULL) that can confuse the compiler.
- Overloading is best organized inside **packages** for clarity and maintainability.
