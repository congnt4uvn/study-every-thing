
Here’s a concise summary of **pipelined table functions** in Oracle PL/SQL & SQL, with code examples.

---

## Key Ideas

- A **pipelined table function** is a PL/SQL function that:
  - Returns a **collection type** (table of records/objects).
  - Streams rows **one-by-one** using `PIPE ROW(...)` instead of building the whole collection in memory first.
  - Can be queried in SQL using `SELECT * FROM TABLE(function_name(...))`.
- Benefits:
  - Better **performance** and **memory usage** for large result sets.
  - Can be used like a **virtual table** in SQL.
  - Useful for complex transformations, parsing, or data reshaping.

---

## Basic Structure

1. Define a **record type** and a **table type**.
2. Create a function that:
   - Is declared `PIPELINED`.
   - Uses `PIPE ROW(...)` to return each row.
3. Use `TABLE(function_name(...))` in SQL.

---

## Example 1: Simple Pipelined Function (No Input)

### 1. Types and Function

```sql
-- 1. Define a record type and a table type
CREATE OR REPLACE TYPE emp_row_type AS OBJECT (
  empno   NUMBER,
  ename   VARCHAR2(100)
);
/

CREATE OR REPLACE TYPE emp_table_type AS TABLE OF emp_row_type;
/

-- 2. Pipelined function
CREATE OR REPLACE FUNCTION get_emp_pipelined
  RETURN emp_table_type PIPELINED
IS
BEGIN
  -- Example: stream employees from EMP table
  FOR r IN (SELECT empno, ename FROM emp ORDER BY empno) LOOP
    PIPE ROW (emp_row_type(r.empno, r.ename));
  END LOOP;

  RETURN;  -- no collection needed; rows already piped
END;
/
```

### 2. SQL Usage (Output)

```sql
SELECT * FROM TABLE(get_emp_pipelined);
```

**Sample output:**

| EMPNO | ENAME   |
|-------|---------|
| 7369  | SMITH   |
| 7499  | ALLEN   |
| 7521  | WARD    |
| ...   | ...     |

## Example 2: Pipelined Function with Input Parameter

### 1. Function with a Filter

```sql
CREATE OR REPLACE FUNCTION get_emp_by_dept_pipelined (
  p_deptno IN NUMBER
) RETURN emp_table_type PIPELINED
IS
BEGIN
  FOR r IN (
    SELECT empno, ename
    FROM   emp
    WHERE  deptno = p_deptno
    ORDER BY empno
  ) LOOP
    PIPE ROW (emp_row_type(r.empno, r.ename));
  END LOOP;

  RETURN;
END;
/
```

### 2. SQL Usage (Input/Output)

```sql
SELECT * 
FROM   TABLE(get_emp_by_dept_pipelined(10));
```

**Sample output:**

| EMPNO | ENAME   |
|-------|---------|
| 7782  | CLARK   |
| 7839  | KING    |
| 7934  | MILLER  |

## Example 3: Pipelined Function for String Parsing

Split a comma-separated string into rows.

### 1. Types

```sql
CREATE OR REPLACE TYPE varchar2_table AS TABLE OF VARCHAR2(4000);
/
```

### 2. Function

```sql
CREATE OR REPLACE FUNCTION split_string_pipelined (
  p_str   IN VARCHAR2,
  p_delim IN VARCHAR2 DEFAULT ','
) RETURN varchar2_table PIPELINED
IS
  l_start PLS_INTEGER := 1;
  l_pos   PLS_INTEGER;
  l_token VARCHAR2(4000);
BEGIN
  IF p_str IS NULL THEN
    RETURN;
  END IF;

  LOOP
    l_pos := INSTR(p_str, p_delim, l_start);
    IF l_pos = 0 THEN
      l_token := SUBSTR(p_str, l_start);
      PIPE ROW (l_token);
      EXIT;
    ELSE
      l_token := SUBSTR(p_str, l_start, l_pos - l_start);
      PIPE ROW (l_token);
      l_start := l_pos + LENGTH(p_delim);
    END IF;
  END LOOP;

  RETURN;
END;
/
```

### 3. SQL Usage (Input/Output)

```sql
SELECT COLUMN_VALUE AS token
FROM   TABLE(split_string_pipelined('A,B,C,D'));
```

**Output:**

| TOKEN |
|-------|
| A     |
| B     |
| C     |
| D     |

## When to Use Pipelined Table Functions

- When you need to:
  - Transform data in PL/SQL and expose it as a **queryable table**.
  - Process **large datasets** without materializing everything in memory.
  - Implement custom **ETL / parsing / aggregation** logic that SQL alone can’t easily express.
