
Here’s a concise summary and some practical examples about using bind variables in dynamic SQL in Oracle PL/SQL.

---

## Key Ideas

1. **Dynamic SQL**  
   - Built as strings and executed at runtime using:
     - `EXECUTE IMMEDIATE`
     - `DBMS_SQL` package

2. **Bind Variables**  
   - Placeholders in SQL (e.g. `:id`, `:name`) whose values are supplied at execution time.
   - Used in dynamic SQL to:
     - Improve performance (statement can be reused / shared in the library cache).
     - Reduce parsing overhead.
     - Prevent SQL injection.
     - Avoid hard-coding values into SQL strings.

3. **Why Use Bind Variables in Dynamic SQL?**
   - **Performance**:  
     `SELECT * FROM emp WHERE empno = :id`  
     can be parsed once and executed many times with different `:id` values.
   - **Security**:  
     Values are passed separately, so user input is not concatenated directly into the SQL string.
   - **Maintainability**:  
     Cleaner code, easier to read and modify.

---

## Basic Example with `EXECUTE IMMEDIATE`

### 1. Dynamic SELECT with Bind Variables

**Code:**
```plsql
DECLARE
  l_sql   VARCHAR2(200);
  l_empno NUMBER := 7369;
  l_ename VARCHAR2(100);
BEGIN
  l_sql := 'SELECT ename FROM emp WHERE empno = :id';

  EXECUTE IMMEDIATE l_sql
    INTO l_ename
    USING l_empno;

  DBMS_OUTPUT.PUT_LINE('Employee name: ' || l_ename);
END;
/
```

**Input:**
- `l_empno = 7369`

**Output (example):**
- `Employee name: SMITH`

---

### 2. Dynamic DML (INSERT/UPDATE/DELETE) with Bind Variables

**Code:**
```plsql
DECLARE
  l_sql   VARCHAR2(200);
  l_empno NUMBER := 8000;
  l_ename VARCHAR2(100) := 'NEW_EMP';
BEGIN
  l_sql := 'INSERT INTO emp (empno, ename) VALUES (:id, :name)';

  EXECUTE IMMEDIATE l_sql
    USING l_empno, l_ename;

  DBMS_OUTPUT.PUT_LINE('Row inserted: empno=' || l_empno || ', ename=' || l_ename);
END;
/
```

**Input:**
- `l_empno = 8000`
- `l_ename = 'NEW_EMP'`

**Output (example):**
- `Row inserted: empno=8000, ename=NEW_EMP`

---

### 3. Using `RETURNING INTO` with Bind Variables

**Code:**
```plsql
DECLARE
  l_sql     VARCHAR2(200);
  l_empno   NUMBER := 8001;
  l_ename   VARCHAR2(100) := 'ANOTHER_EMP';
  l_rowid   ROWID;
BEGIN
  l_sql := 'INSERT INTO emp (empno, ename) VALUES (:id, :name) RETURNING ROWID INTO :rid';

  EXECUTE IMMEDIATE l_sql
    USING  l_empno, l_ename
    RETURNING INTO l_rowid;

  DBMS_OUTPUT.PUT_LINE('Inserted rowid: ' || l_rowid);
END;
/
```

**Output (example):**
- `Inserted rowid: AAA1aAABbBBCCDD`

## Using `DBMS_SQL` with Bind Variables (Advanced)

`DBMS_SQL` is more verbose but powerful for:
- Unknown number of columns
- Dynamic column lists
- Complex dynamic queries

**Code:**
```plsql
DECLARE
  l_cursor   INTEGER;
  l_sql      VARCHAR2(200);
  l_empno    NUMBER := 7369;
  l_ename    VARCHAR2(100);
  l_status   INTEGER;
BEGIN
  l_sql := 'SELECT ename FROM emp WHERE empno = :id';

  l_cursor := DBMS_SQL.open_cursor;
  DBMS_SQL.parse(l_cursor, l_sql, DBMS_SQL.native);

  -- Bind variable
  DBMS_SQL.bind_variable(l_cursor, ':id', l_empno);

  -- Define output column
  DBMS_SQL.define_column(l_cursor, 1, l_ename, 100);

  l_status := DBMS_SQL.execute(l_cursor);

  IF DBMS_SQL.fetch_rows(l_cursor) > 0 THEN
    DBMS_SQL.column_value(l_cursor, 1, l_ename);
    DBMS_OUTPUT.PUT_LINE('Employee name: ' || l_ename);
  END IF;

  DBMS_SQL.close_cursor(l_cursor);
END;
/
```

---

## Common Patterns

1. **IN Bind Variables**  
   `USING var1, var2, ...`

2. **OUT / IN OUT Bind Variables**  
   `USING OUT var1, IN OUT var2, ...`

3. **RETURNING INTO**  
   `EXECUTE IMMEDIATE sql_stmt USING ... RETURNING INTO var1, var2, ...;`

---

## Summary

- Use `EXECUTE IMMEDIATE` or `DBMS_SQL` for dynamic SQL.
- Always use bind variables (`:param`) instead of concatenating values into SQL strings.
- Benefits:
  - Better performance (shared SQL, less parsing).
  - Stronger security (mitigates SQL injection).
  - Cleaner, more maintainable PL/SQL code.
