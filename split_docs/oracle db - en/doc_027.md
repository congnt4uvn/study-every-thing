---

## Key Idea: `SAVE EXCEPTIONS` in PL/SQL

`SAVE EXCEPTIONS` is used with **FORALL** bulk operations to:

- Continue processing all iterations even if some cause errors.
- Collect all exceptions raised during the bulk operation.
- Handle them afterward in a single exception block.

Without `SAVE EXCEPTIONS`, a single error in a bulk operation stops the entire statement.

---

## Typical Use Case

You use `FORALL ... SAVE EXCEPTIONS` when:

- Doing bulk `INSERT`, `UPDATE`, or `DELETE`.
- You want to process as many rows as possible.
- You need to know which rows failed and why.

---

## Basic Syntax

```plsql
FORALL i IN collection.FIRST .. collection.LAST SAVE EXCEPTIONS
    INSERT INTO target_table (col1, col2)
    VALUES (collection(i).col1, collection(i).col2);
```

Then in the exception section:

```plsql
EXCEPTION
    WHEN OTHERS THEN
        IF SQLCODE = -24381 THEN  -- ORA-24381: error(s) in array DML
            FOR j IN 1 .. SQL%BULK_EXCEPTIONS.COUNT LOOP
                DBMS_OUTPUT.PUT_LINE(
                    'Error #' || j ||
                    ' at index ' || SQL%BULK_EXCEPTIONS(j).ERROR_INDEX ||
                    ' code: ' || SQL%BULK_EXCEPTIONS(j).ERROR_CODE
                );
            END LOOP;
        ELSE
            RAISE;
        END IF;
END;
```

`SQL%BULK_EXCEPTIONS` is a collection that stores:

- `ERROR_INDEX`: which iteration (i) failed.
- `ERROR_CODE`: Oracle error code (e.g., -1 for ORA-00001).

## Full Example with Input/Output

### Setup

```sql
-- Target table
CREATE TABLE employees_test (
    emp_id   NUMBER PRIMARY KEY,
    name     VARCHAR2(50)
);
```

### PL/SQL Block

```plsql
DECLARE
    TYPE emp_tab IS TABLE OF employees_test%ROWTYPE;
    l_emps emp_tab := emp_tab();
BEGIN
    -- Input data (note: emp_id = 2 is duplicated to cause an error)
    l_emps.EXTEND(3);
    l_emps(1).emp_id := 1; l_emps(1).name := 'Alice';
    l_emps(2).emp_id := 2; l_emps(2).name := 'Bob';
    l_emps(3).emp_id := 2; l_emps(3).name := 'Charlie'; -- duplicate PK

    FORALL i IN l_emps.FIRST .. l_emps.LAST SAVE EXCEPTIONS
        INSERT INTO employees_test (emp_id, name)
        VALUES (l_emps(i).emp_id, l_emps(i).name);

EXCEPTION
    WHEN OTHERS THEN
        IF SQLCODE = -24381 THEN  -- ORA-24381: error(s) in array DML
            DBMS_OUTPUT.PUT_LINE('Bulk operation completed with errors:');
            FOR j IN 1 .. SQL%BULK_EXCEPTIONS.COUNT LOOP
                DBMS_OUTPUT.PUT_LINE(
                    '  Error #' || j ||
                    ' at index ' || SQL%BULK_EXCEPTIONS(j).ERROR_INDEX ||
                    ' code: ' || SQL%BULK_EXCEPTIONS(j).ERROR_CODE
                );
            END LOOP;
        ELSE
            RAISE;
        END IF;
END;
/
```

### Expected Output (DBMS_OUTPUT)

```text
Bulk operation completed with errors:
  Error #1 at index 3 code: -1
```

### Table Contents After Execution

```sql
SELECT * FROM employees_test ORDER BY emp_id;
```

**Result:**

```text
EMP_ID  NAME
------  ------
1       Alice
2       Bob
```

- Rows 1 and 2 inserted successfully.
- Row 3 failed due to primary key violation (duplicate `emp_id = 2`), but the others were not rolled back.

---

## Summary

- `SAVE EXCEPTIONS` allows **partial success** in bulk DML.
- Errors are collected in `SQL%BULK_EXCEPTIONS` instead of stopping the whole operation.
- You can inspect which iterations failed and why, while keeping successful rows.
