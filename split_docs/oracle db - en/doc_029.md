
Here’s a concise summary of the three main PL/SQL collection types, with key points and code examples (including sample inputs/outputs).

---

## 1. INDEX BY Tables (Associative Arrays)

**Key ideas:**
- In-memory, PL/SQL-only collection (not stored in DB columns).
- Indexed by `PLS_INTEGER` or `VARCHAR2` (sparse, like a hash map).
- No `LIMIT`, no `EXTEND` needed.
- Good for lookups and temporary data.

**Declaration:**
```sql
DECLARE
  TYPE t_emp_tab IS TABLE OF VARCHAR2(100) INDEX BY PLS_INTEGER;
  l_emps t_emp_tab;
BEGIN
  -- Input: assign values by index
  l_emps(1) := 'Scott';
  l_emps(2) := 'Tiger';

  -- Access by index
  DBMS_OUTPUT.PUT_LINE('Employee 1: ' || l_emps(1));
  DBMS_OUTPUT.PUT_LINE('Employee 2: ' || l_emps(2));
END;
/
```

**Output:**
```text
Employee 1: Scott
Employee 2: Tiger
```

**String-keyed example:**
```sql
DECLARE
  TYPE t_salary_tab IS TABLE OF NUMBER INDEX BY VARCHAR2(30);
  l_salaries t_salary_tab;
BEGIN
  l_salaries('SCOTT') := 3000;
  l_salaries('KING')  := 5000;

  DBMS_OUTPUT.PUT_LINE('SCOTT salary: ' || l_salaries('SCOTT'));
  DBMS_OUTPUT.PUT_LINE('KING salary: '  || l_salaries('KING'));
END;
/
```

## 2. Nested Tables

**Key ideas:**
- Unbounded, one-dimensional collection.
- Can be used in PL/SQL and stored in database columns.
- Dense when created, but can become sparse after `DELETE`.
- Need `EXTEND` to add elements in PL/SQL.

**Type and table definition (SQL):**
```sql
-- SQL type
CREATE OR REPLACE TYPE t_num_tab AS TABLE OF NUMBER;
/

-- Table using nested table column
CREATE TABLE test_nested (
  id        NUMBER,
  numbers   t_num_tab
) NESTED TABLE numbers STORE AS numbers_nt;
/
```

**PL/SQL usage:**
```sql
DECLARE
  l_nums t_num_tab := t_num_tab();  -- constructor
BEGIN
  -- Input: add elements
  l_nums.EXTEND(3);
  l_nums(1) := 10;
  l_nums(2) := 20;
  l_nums(3) := 30;

  -- Store in table
  INSERT INTO test_nested (id, numbers)
  VALUES (1, l_nums);

  COMMIT;
END;
/
```

**Querying the nested table:**
```sql
-- Input: id = 1
SELECT t.id, n.COLUMN_VALUE AS num_value
FROM   test_nested t
       CROSS JOIN TABLE(t.numbers) n
WHERE  t.id = 1;
```

**Output:**
```text
ID  NUM_VALUE
--  ---------
1   10
1   20
1   30
```

## 3. VARRAYs (Variable-Size Arrays)

**Key ideas:**
- Ordered collection with a fixed maximum size.
- Stored as a single object (good for small, bounded lists).
- Dense: no gaps in indexes.
- Can be used in PL/SQL and as column types.

**Type and table definition (SQL):**
```sql
CREATE OR REPLACE TYPE t_phone_list AS VARRAY(3) OF VARCHAR2(20);
/

CREATE TABLE customers (
  id        NUMBER,
  name      VARCHAR2(50),
  phones    t_phone_list
);
/
```

**PL/SQL usage:**
```sql
DECLARE
  l_phones t_phone_list := t_phone_list();
BEGIN
  -- Input: add up to 3 phone numbers
  l_phones.EXTEND;
  l_phones(1) := '111-111-1111';

  l_phones.EXTEND;
  l_phones(2) := '222-222-2222';

  INSERT INTO customers (id, name, phones)
  VALUES (1, 'John', l_phones);

  COMMIT;
END;
/
```

**Querying the VARRAY:**
```sql
-- Input: id = 1
SELECT c.id, c.name, p.COLUMN_VALUE AS phone
FROM   customers c
       CROSS JOIN TABLE(c.phones) p
WHERE  c.id = 1;
```

**Output:**
```text
ID  NAME  PHONE
--  ----  -------------
1   John  111-111-1111
1   John  222-222-2222
```

---

## Quick Comparison

- **INDEX BY table (associative array)**  
  - PL/SQL only, in-memory  
  - Indexed by integer or string  
  - No size limit, no `EXTEND` needed  

- **Nested table**  
  - PL/SQL + SQL (can be stored in columns)  
  - Unbounded, can be sparse  
  - Requires `EXTEND` in PL/SQL  

- **VARRAY**  
  - PL/SQL + SQL  
  - Fixed maximum size, always dense  
  - Good for small, ordered lists stored as a single unit  

These are the core advanced PL/SQL collection types used to manage sets of values in memory and in the database.
