
Here’s a concise summary of PL/SQL packages (specification vs body), with examples.

---

## 1. What is a Package?

A **package** is a container for related PL/SQL objects:
- Procedures
- Functions
- Variables / constants
- Cursors
- Types

It has two parts:
1. **Package Specification** (spec)
2. **Package Body**

---

## 2. Package Specification

- Public interface of the package.
- Declares what is visible/usable from outside the package.
- Contains:
  - Procedure/function headers (no code)
  - Public variables/constants
  - Public types, cursors

Example – package spec:

```sql
CREATE OR REPLACE PACKAGE emp_pkg AS
  -- Public constant
  c_min_salary CONSTANT NUMBER := 1000;

  -- Public variable
  g_bonus_rate NUMBER;

  -- Public type
  TYPE t_emp_rec IS RECORD (
    emp_id   NUMBER,
    emp_name VARCHAR2(100)
  );

  -- Public procedure (only header)
  PROCEDURE give_raise(p_emp_id IN NUMBER, p_amount IN NUMBER);

  -- Public function (only header)
  FUNCTION get_salary(p_emp_id IN NUMBER) RETURN NUMBER;
END emp_pkg;
/
```

You can call `emp_pkg.give_raise` or `emp_pkg.get_salary` from other PL/SQL blocks because they are declared in the spec.

---

## 3. Package Body

- Contains the **implementation** of what is declared in the spec.
- Also can contain **private** items (not visible outside) that are not in the spec.
- Must define all public procedures/functions declared in the spec.

Example – package body:

```sql
CREATE OR REPLACE PACKAGE BODY emp_pkg AS

  -- Private variable (not in spec)
  g_internal_counter NUMBER := 0;

  -- Private function (not in spec)
  FUNCTION calc_tax(p_amount IN NUMBER) RETURN NUMBER IS
  BEGIN
    RETURN p_amount * 0.1;
  END calc_tax;

  -- Implementation of public procedure
  PROCEDURE give_raise(p_emp_id IN NUMBER, p_amount IN NUMBER) IS
    v_old_salary NUMBER;
  BEGIN
    SELECT salary INTO v_old_salary
    FROM employees
    WHERE employee_id = p_emp_id;

    UPDATE employees
    SET salary = salary + p_amount
    WHERE employee_id = p_emp_id;

    g_internal_counter := g_internal_counter + 1;
  END give_raise;

  -- Implementation of public function
  FUNCTION get_salary(p_emp_id IN NUMBER) RETURN NUMBER IS
    v_salary NUMBER;
  BEGIN
    SELECT salary INTO v_salary
    FROM employees
    WHERE employee_id = p_emp_id;

    RETURN v_salary;
  END get_salary;

END emp_pkg;
/
```

## 4. Using the Package (Example with Input/Output)

Anonymous block calling the package:

```sql
DECLARE
  v_emp_id   NUMBER := 100;
  v_new_sal  NUMBER;
BEGIN
  -- Set public variable
  emp_pkg.g_bonus_rate := 0.05;

  -- Call public procedure
  emp_pkg.give_raise(p_emp_id => v_emp_id, p_amount => 500);

  -- Call public function
  v_new_sal := emp_pkg.get_salary(p_emp_id => v_emp_id);

  DBMS_OUTPUT.PUT_LINE('New salary for employee ' || v_emp_id || ' = ' || v_new_sal);
END;
/
```

**Input:**
- `v_emp_id = 100`
- `p_amount = 500`

**Output (example):**
```text
New salary for employee 100 = 6500
```
(Assuming old salary was 6000.)

---

## 5. Key Differences: Spec vs Body

- **Specification**
  - Public interface
  - What other code can see and call
  - No implementation details
  - Changes here can require recompiling dependent objects

- **Body**
  - Actual code/logic
  - Can have private items
  - You can change implementation without affecting callers (as long as spec doesn’t change)

---

## 6. Benefits of Using Packages

- Encapsulation: hide implementation details in the body.
- Modularity: group related logic together.
- Reusability: shared procedures/functions.
- Better performance: package state and cached code.
