
Here’s a concise summary of the key ideas, with code examples and sample inputs/outputs.

---

## 1. Inline Views (Subqueries in the FROM Clause)

**Concept:**  
An inline view is a subquery used in the `FROM` clause that acts like a temporary table for the outer query.

**Why use it?**
- To simplify complex queries by breaking them into steps.
- To filter/aggregate data first, then query the result.
- To avoid repeating the same subquery multiple times.

### Example 1: Aggregation in an Inline View

**Goal:** Get employees whose salary is above the average salary in their department.

```sql
SELECT e.employee_id,
       e.first_name,
       e.department_id,
       e.salary
FROM   employees e
JOIN   (
         SELECT department_id,
                AVG(salary) AS avg_sal
         FROM   employees
         GROUP  BY department_id
       ) dept_avg
ON     e.department_id = dept_avg.department_id
WHERE  e.salary > dept_avg.avg_sal;
```

**Input (employees table – sample rows):**

| employee_id | first_name | department_id | salary |
|------------:|------------|--------------:|-------:|
| 101         | John       | 10            | 5000   |
| 102         | Mary       | 10            | 7000   |
| 103         | Alex       | 20            | 4000   |
| 104         | Sara       | 20            | 6000   |

**Output (employees above dept average):**

| employee_id | first_name | department_id | salary |
|------------:|------------|--------------:|-------:|
| 102         | Mary       | 10            | 7000   |
| 104         | Sara       | 20            | 6000   |

## 2. Common Table Expressions (CTE) with `WITH`

**Concept:**  
A CTE is a named subquery defined at the start of a statement using `WITH`. It can be referenced multiple times in the main query.

**Why use it?**
- Improves readability and maintainability.
- Lets you reuse the same logical subquery.
- Supports recursive queries (hierarchies, trees).

### Basic Syntax

```sql
WITH cte_name AS (
    SELECT ...
    FROM   ...
    WHERE  ...
)
SELECT ...
FROM   cte_name
WHERE  ...;
```

---

### Example 2: Non-Recursive CTE

**Goal:** Get departments with total salary > 10000.

```sql
WITH dept_salary AS (
    SELECT department_id,
           SUM(salary) AS total_salary
    FROM   employees
    GROUP  BY department_id
)
SELECT department_id,
       total_salary
FROM   dept_salary
WHERE  total_salary > 10000;
```

**Input (employees – sample):**

| employee_id | department_id | salary |
|------------:|--------------:|-------:|
| 101         | 10            | 5000   |
| 102         | 10            | 7000   |
| 103         | 20            | 4000   |

**Intermediate (dept_salary):**

| department_id | total_salary |
|--------------:|-------------:|
| 10            | 12000        |
| 20            | 4000         |

**Output:**

| department_id | total_salary |
|--------------:|-------------:|
| 10            | 12000        |

### Example 3: Recursive CTE (Hierarchical Query)

**Goal:** Get all employees under a given manager (e.g., manager_id = 100), including multiple levels.

```sql
WITH emp_hierarchy AS (
    -- Anchor member: start from the top manager
    SELECT employee_id,
           first_name,
           manager_id,
           1 AS level_no
    FROM   employees
    WHERE  employee_id = 100

    UNION ALL

    -- Recursive member: get direct reports of previous level
    SELECT e.employee_id,
           e.first_name,
           e.manager_id,
           eh.level_no + 1 AS level_no
    FROM   employees e
    JOIN   emp_hierarchy eh
           ON e.manager_id = eh.employee_id
)
SELECT employee_id,
       first_name,
       manager_id,
       level_no
FROM   emp_hierarchy
ORDER  BY level_no, employee_id;
```

**Input (employees – simplified):**

| employee_id | first_name | manager_id |
|------------:|------------|-----------:|
| 100         | King       | NULL       |
| 101         | John       | 100        |
| 102         | Mary       | 100        |
| 103         | Alex       | 101        |

**Output:**

| employee_id | first_name | manager_id | level_no |
|------------:|------------|-----------:|---------:|
| 100         | King       | NULL       | 1        |
| 101         | John       | 100        | 2        |
| 102         | Mary       | 100        | 2        |
| 103         | Alex       | 101        | 3        |

---

## 3. Inline View vs CTE: When to Use Which

- **Inline View**
  - Quick, one-off subquery.
  - Used directly in `FROM`.
  - Good for small, local transformations.

- **CTE (`WITH`)**
  - When the same logic is reused multiple times.
  - When you want clearer, step-by-step structure.
  - Needed for recursive queries (hierarchies).
