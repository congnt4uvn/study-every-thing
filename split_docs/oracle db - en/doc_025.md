
Here’s a concise summary of **Advanced PL/SQL: Bulk Operations**, with code examples and sample input/output.

---

## 1. What Are Bulk Operations?

Bulk operations in PL/SQL let you process **many rows at once** instead of row-by-row. This reduces context switches between SQL and PL/SQL and improves performance.

Main features:

- `BULK COLLECT` – fetch multiple rows into collections in one go.
- `FORALL` – send multiple DML statements (INSERT/UPDATE/DELETE/MERGE) to SQL in one go.
- Often used together for high‑performance batch processing.

---

## 2. BULK COLLECT

### Purpose
Fetch multiple rows from a query into PL/SQL collections (nested tables, varrays, associative arrays).

### Basic Example

```sql
DECLARE
  TYPE t_emp_tab IS TABLE OF employees%ROWTYPE;
  l_emps t_emp_tab;
BEGIN
  SELECT *
  BULK COLLECT INTO l_emps
  FROM employees
  WHERE department_id = 10;

  DBMS_OUTPUT.PUT_LINE('Rows fetched: ' || l_emps.COUNT);
END;
/
```

**Input:** `department_id = 10`  
**Output (example):**
```text
Rows fetched: 5
```

### BULK COLLECT with LIMIT (for large data sets)

```sql
DECLARE
  TYPE t_emp_tab IS TABLE OF employees%ROWTYPE;
  l_emps t_emp_tab;
  CURSOR c_emp IS
    SELECT * FROM employees WHERE department_id = 10;
BEGIN
  OPEN c_emp;
  LOOP
    FETCH c_emp BULK COLLECT INTO l_emps LIMIT 1000;

    EXIT WHEN l_emps.COUNT = 0;

    -- Process batch of up to 1000 rows
    FOR i IN 1 .. l_emps.COUNT LOOP
      DBMS_OUTPUT.PUT_LINE('Emp: ' || l_emps(i).employee_id);
    END LOOP;
  END LOOP;
  CLOSE c_emp;
END;
/
```

## 3. FORALL

### Purpose
Execute the **same DML statement** for many rows using a collection of values.

### Basic INSERT Example

```sql
DECLARE
  TYPE t_emp_id_tab IS TABLE OF employees.employee_id%TYPE;
  TYPE t_sal_tab    IS TABLE OF employees.salary%TYPE;

  l_emp_ids t_emp_id_tab := t_emp_id_tab(101, 102, 103);
  l_sals    t_sal_tab    := t_sal_tab(5000, 6000, 7000);
BEGIN
  FORALL i IN l_emp_ids.FIRST .. l_emp_ids.LAST
    INSERT INTO emp_bonus (employee_id, bonus_amount)
    VALUES (l_emp_ids(i), l_sals(i) * 0.1);

  DBMS_OUTPUT.PUT_LINE('Rows inserted: ' || SQL%ROWCOUNT);
END;
/
```

**Input collections:**
- `l_emp_ids = [101, 102, 103]`
- `l_sals = [5000, 6000, 7000]`

**Resulting rows in `emp_bonus`:**

| employee_id | bonus_amount |
|-------------|--------------|
| 101         | 500          |
| 102         | 600          |
| 103         | 700          |

**Output:**
```text
Rows inserted: 3
```

### UPDATE Example with FORALL

```sql
DECLARE
  TYPE t_emp_id_tab IS TABLE OF employees.employee_id%TYPE;
  TYPE t_raise_tab  IS TABLE OF NUMBER;

  l_emp_ids t_emp_id_tab := t_emp_id_tab(101, 102, 103);
  l_raises  t_raise_tab  := t_raise_tab(0.05, 0.10, 0.08); -- 5%, 10%, 8%
BEGIN
  FORALL i IN l_emp_ids.FIRST .. l_emp_ids.LAST
    UPDATE employees
       SET salary = salary * (1 + l_raises(i))
     WHERE employee_id = l_emp_ids(i);

  DBMS_OUTPUT.PUT_LINE('Rows updated: ' || SQL%ROWCOUNT);
END;
/
```

## 4. Combining BULK COLLECT and FORALL

Typical pattern: bulk fetch → bulk DML.

```sql
DECLARE
  TYPE t_emp_tab IS TABLE OF employees%ROWTYPE;
  l_emps t_emp_tab;
BEGIN
  -- 1. Bulk fetch
  SELECT *
  BULK COLLECT INTO l_emps
  FROM employees
  WHERE department_id = 10;

  -- 2. Bulk update
  FORALL i IN 1 .. l_emps.COUNT
    UPDATE employees
       SET salary = salary * 1.05
     WHERE employee_id = l_emps(i).employee_id;

  DBMS_OUTPUT.PUT_LINE('Rows updated: ' || SQL%ROWCOUNT);
END;
/
```

**Effect:** All employees in department 10 get a 5% salary increase in bulk.

## 5. Handling Exceptions in FORALL

Use `SAVE EXCEPTIONS` to continue processing even if some rows fail.

```sql
DECLARE
  TYPE t_emp_id_tab IS TABLE OF employees.employee_id%TYPE;
  l_emp_ids t_emp_id_tab := t_emp_id_tab(101, 9999, 103); -- 9999 may not exist
BEGIN
  FORALL i IN 1 .. l_emp_ids.COUNT SAVE EXCEPTIONS
    DELETE FROM employees
    WHERE employee_id = l_emp_ids(i);

  DBMS_OUTPUT.PUT_LINE('Rows deleted: ' || SQL%ROWCOUNT);

EXCEPTION
  WHEN OTHERS THEN
    FOR j IN 1 .. SQL%BULK_EXCEPTIONS.COUNT LOOP
      DBMS_OUTPUT.PUT_LINE(
        'Error at index ' || SQL%BULK_EXCEPTIONS(j).ERROR_INDEX ||
        ' - ORA-' || SQL%BULK_EXCEPTIONS(j).ERROR_CODE
      );
    END LOOP;
END;
/
```

**Output (example):**
```text
Rows deleted: 2
Error at index 2 - ORA-02292
```

---

## 6. Key Takeaways

- Use `BULK COLLECT` to fetch many rows at once.
- Use `FORALL` to perform DML on many rows at once.
- Combine them for high‑performance batch processing.
- Use `LIMIT` with `BULK COLLECT` for very large data sets.
- Use `SAVE EXCEPTIONS` with `FORALL` to handle row-level errors.
