
Here’s a concise, high‑level summary of Oracle **tablespaces** in physical design, plus some simple code examples.

---

## Key Ideas: Tablespaces in Oracle

1. **What is a Tablespace?**  
   - A logical storage container in an Oracle database.  
   - It groups related database objects (tables, indexes, etc.).  
   - Physically, a tablespace is made up of one or more **datafiles** on disk.

2. **Logical vs Physical**  
   - **Logical level**: tables, indexes, segments, extents, blocks.  
   - **Physical level**: datafiles on disk.  
   - Tablespace is the bridge: logical objects live in a tablespace, which is stored in datafiles.

3. **Types of Tablespaces (common ones)**  
   - **SYSTEM**: core data dictionary; created with the database; must always be online.  
   - **SYSAUX**: auxiliary to SYSTEM; holds many Oracle components.  
   - **UNDO**: stores undo data for transactions (rollback, read consistency).  
   - **TEMP**: used for temporary operations (sorts, hash joins, etc.).  
   - **USER / application tablespaces**: where user/application data is stored.

4. **Why Tablespaces Matter in Physical Design**  
   - Performance: separate I/O (e.g., put heavy tables and indexes on different disks).  
   - Manageability: backup/restore at tablespace level; move datafiles; resize.  
   - Security & quotas: control which users can use which tablespaces and how much space.  
   - Availability: take tablespaces offline/online without affecting the whole database.

5. **Extent & Segment Management**  
   - Tablespaces manage how space is allocated to objects (segments) in **extents**.  
   - Modern Oracle uses **locally managed tablespaces** (LMT) with bitmaps to track extents.

## Basic SQL Examples

### 1. Create a Tablespace

**Input:**
```sql
CREATE TABLESPACE app_data
  DATAFILE '/u01/oradata/ORCL/app_data01.dbf' SIZE 500M
  AUTOEXTEND ON NEXT 50M MAXSIZE 5G
  EXTENT MANAGEMENT LOCAL
  SEGMENT SPACE MANAGEMENT AUTO;
```

**What this does:**
- Creates a tablespace `APP_DATA`.
- Uses one datafile of 500 MB, autoextends by 50 MB up to 5 GB.
- Locally managed extents, automatic segment space management.

---

### 2. Create a User and Assign Default Tablespace

**Input:**
```sql
CREATE USER app_user IDENTIFIED BY "StrongPass1"
  DEFAULT TABLESPACE app_data
  TEMPORARY TABLESPACE temp
  QUOTA 100M ON app_data;

GRANT CONNECT, RESOURCE TO app_user;
```

**What this does:**
- Creates `APP_USER` with `APP_DATA` as default tablespace.
- Limits user to 100 MB in `APP_DATA`.
- Uses `TEMP` tablespace for temporary operations.

---

### 3. Create a Table in a Specific Tablespace

**Input:**
```sql
CREATE TABLE customers (
  customer_id   NUMBER PRIMARY KEY,
  name          VARCHAR2(100),
  created_date  DATE
) TABLESPACE app_data;
```

**What this does:**
- Stores the `CUSTOMERS` table’s segment in the `APP_DATA` tablespace.

---

### 4. Check Existing Tablespaces

**Input:**
```sql
SELECT tablespace_name, status, contents, logging
FROM dba_tablespaces
ORDER BY tablespace_name;
```

**Example Output (simplified):**
```text
TABLESPACE_NAME  STATUS   CONTENTS  LOGGING
---------------  -------  --------  -------
SYSTEM           ONLINE   PERMANENT LOGGING
SYSAUX           ONLINE   PERMANENT LOGGING
UNDOTBS1         ONLINE   UNDO      LOGGING
TEMP             ONLINE   TEMPORARY NOLOGGING
APP_DATA         ONLINE   PERMANENT LOGGING
```

---

### 5. Resize a Datafile

**Input:**
```sql
ALTER DATABASE DATAFILE '/u01/oradata/ORCL/app_data01.dbf' RESIZE 800M;
```

**What this does:**
- Changes the size of the datafile (and thus available space in the tablespace).

---

### 6. Take a Tablespace Offline / Online

**Input:**
```sql
ALTER TABLESPACE app_data OFFLINE;

-- later
ALTER TABLESPACE app_data ONLINE;
```

**What this does:**
- Temporarily makes `APP_DATA` unavailable (e.g., for maintenance), then brings it back.
