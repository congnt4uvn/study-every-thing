
Here’s a concise summary and examples for the common Oracle command-line tools: **SQL\*Plus** and **SQLcl**.

---

## 1. SQL\*Plus

**What it is:**  
A classic Oracle command-line tool to connect to the database, run SQL, PL/SQL, and scripts.

**Key points:**
- Installed with Oracle Client/Database.
- Uses text-based commands.
- Good for scripting and batch jobs.
- Supports `@script.sql` to run files.

**Basic usage:**

```bash
sqlplus username/password@//host:1521/service_name
```

**Example 1 – Simple SELECT**

Input (in SQL\*Plus):

```sql
SQL> SELECT employee_id, first_name, last_name
  2  FROM employees
  3  WHERE department_id = 10;
```

Output (example):

```text
EMPLOYEE_ID FIRST_NAME  LAST_NAME
----------- ----------  ----------
        100 Steven      King
        101 Neena       Kochhar
```

**Example 2 – Run a script**

`report.sql`:

```sql
SET PAGESIZE 50
SET LINESIZE 120

SELECT department_id, COUNT(*) AS emp_count
FROM employees
GROUP BY department_id
ORDER BY department_id;

EXIT;
```

Run:

```bash
sqlplus scott/tiger@orcl @report.sql
```

---

## 2. SQLcl (SQL Command Line)

**What it is:**  
A newer Oracle command-line tool from Oracle SQL Developer team. It’s compatible with SQL\*Plus commands but adds modern features.

**Key points:**
- Supports most SQL\*Plus commands.
- Extra features: command history, tab-completion, formatting, JSON, DDL generation, scripting enhancements.
- Java-based, often downloaded separately.

**Basic usage:**

```bash
sql /nolog
SQL> CONNECT username/password@//host:1521/service_name
```

or directly:

```bash
sql username/password@//host:1521/service_name
```

**Example 1 – Same SELECT with nicer formatting**

Input (in SQLcl):

```sql
SQL> SELECT employee_id, first_name, last_name
  2  FROM employees
  3  WHERE department_id = 10;
```

Output (example, formatted):

```text
EMPLOYEE_ID  FIRST_NAME  LAST_NAME
-----------  ----------  ----------
        100  Steven      King
        101  Neena       Kochhar

2 rows selected.
```

**Example 2 – Use `INFO` and `DDL` commands (SQLcl-specific)**

Input:

```sql
SQL> INFO employees;
```

Output (example):

```text
TABLE: HR.EMPLOYEES
  COLUMNS:
    EMPLOYEE_ID NUMBER(6) PK
    FIRST_NAME  VARCHAR2(20)
    LAST_NAME   VARCHAR2(25) NOT NULL
    ...
  INDEXES:
    EMP_EMP_ID_PK (UNIQUE) ON EMPLOYEE_ID
  ...
```

Generate DDL:

```sql
SQL> DDL employees;
```

Output (example):

```sql
CREATE TABLE "HR"."EMPLOYEES"
(
  "EMPLOYEE_ID" NUMBER(6) NOT NULL ENABLE,
  "FIRST_NAME"  VARCHAR2(20),
  "LAST_NAME"   VARCHAR2(25) NOT NULL ENABLE,
  ...
);
```

---

## 3. When to Use Which

- **SQL\*Plus**:  
  - Very lightweight, always available on Oracle servers.  
  - Good for legacy scripts and batch jobs.

- **SQLcl**:  
  - Better user experience (history, completion, formatting).  
  - Extra commands (`INFO`, `DDL`, JSON support, etc.).  
  - Good for interactive work and modern scripting.
