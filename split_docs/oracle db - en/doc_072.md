
Here’s a concise summary and some practical examples around **Oracle Query Rewrite** in the context of **physical design**.

---

## Key Ideas

1. **Query Rewrite** is an Oracle optimizer feature that automatically transforms a user’s SQL query into a semantically equivalent query that can use:
   - **Materialized views**
   - **Summary tables**
   - Other precomputed structures

2. **Goal**: Improve performance without changing the application’s SQL:
   - Users query base tables.
   - Oracle rewrites the query to use faster pre-aggregated or pre-joined data.

3. **When it works**:
   - Materialized view is **fresh** (or within allowed staleness).
   - Query is **logically compatible** with the materialized view (same or compatible grouping, filters, joins).
   - Query rewrite is **enabled** at system/session level and on the materialized view.

4. **Physical Design Aspect**:
   - You design **materialized views** specifically to support common queries.
   - The optimizer then uses them automatically via query rewrite.
   - This is a key part of performance-oriented physical design (along with indexes, partitioning, etc.).

---

## Basic Setup Example

### 1. Create Base Table and Data

```sql
-- Base sales table
CREATE TABLE sales (
    sale_id      NUMBER PRIMARY KEY,
    sale_date    DATE,
    product_id   NUMBER,
    store_id     NUMBER,
    quantity     NUMBER,
    amount       NUMBER
);

INSERT INTO sales (sale_id, sale_date, product_id, store_id, quantity, amount)
VALUES (1, DATE '2024-01-01', 101, 10, 2, 50);

INSERT INTO sales (sale_id, sale_date, product_id, store_id, quantity, amount)
VALUES (2, DATE '2024-01-01', 102, 10, 1, 30);

COMMIT;
```

**Input**: Inserts two sales rows.  
**Output**: Data stored in `sales` table.

---

### 2. Create a Materialized View for Aggregation

```sql
CREATE MATERIALIZED VIEW mv_sales_daily
BUILD IMMEDIATE
REFRESH FAST ON COMMIT
ENABLE QUERY REWRITE
AS
SELECT
    TRUNC(sale_date) AS sale_day,
    product_id,
    SUM(quantity)    AS total_qty,
    SUM(amount)      AS total_amount
FROM sales
GROUP BY TRUNC(sale_date), product_id;
```

Key options:
- `REFRESH FAST ON COMMIT`: keeps it up to date quickly.
- `ENABLE QUERY REWRITE`: allows optimizer to use it for rewrite.

---

### 3. Enable Query Rewrite at Session Level

```sql
ALTER SESSION SET query_rewrite_enabled = TRUE;
ALTER SESSION SET query_rewrite_integrity = TRUSTED;
```

---

### 4. User Query (Against Base Table)

User writes:

```sql
SELECT
    TRUNC(sale_date) AS sale_day,
    product_id,
    SUM(quantity)    AS total_qty,
    SUM(amount)      AS total_amount
FROM sales
GROUP BY TRUNC(sale_date), product_id;
```

**Logical Input**: Query over `sales`.  
**What Oracle may do internally (Rewritten Query)**:

```sql
SELECT
    sale_day,
    product_id,
    total_qty,
    total_amount
FROM mv_sales_daily;
```

**Output**: Same result set, but fetched from the pre-aggregated materialized view, usually much faster.

---

## Checking If Query Rewrite Happened

Use `EXPLAIN PLAN`:

```sql
EXPLAIN PLAN FOR
SELECT
    TRUNC(sale_date) AS sale_day,
    product_id,
    SUM(quantity)    AS total_qty,
    SUM(amount)      AS total_amount
FROM sales
GROUP BY TRUNC(sale_date), product_id;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

In the plan, you should see `MV_SALES_DAILY` instead of `SALES` if rewrite occurred.

## Another Example: Join + Aggregation

### Materialized View

```sql
CREATE TABLE products (
    product_id   NUMBER PRIMARY KEY,
    category     VARCHAR2(50)
);

CREATE MATERIALIZED VIEW mv_sales_by_category
BUILD IMMEDIATE
REFRESH FAST ON COMMIT
ENABLE QUERY REWRITE
AS
SELECT
    TRUNC(s.sale_date) AS sale_day,
    p.category,
    SUM(s.amount)      AS total_amount
FROM sales s
JOIN products p ON s.product_id = p.product_id
GROUP BY TRUNC(s.sale_date), p.category;
```

### User Query

```sql
SELECT
    TRUNC(s.sale_date) AS sale_day,
    p.category,
    SUM(s.amount)      AS total_amount
FROM sales s
JOIN products p ON s.product_id = p.product_id
GROUP BY TRUNC(s.sale_date), p.category;
```

Oracle can rewrite this to:

```sql
SELECT
    sale_day,
    category,
    total_amount
FROM mv_sales_by_category;
```

---

## Summary of Main Points

- **Query rewrite** lets Oracle automatically use materialized views to speed up queries.
- It is a **physical design tool**: you design materialized views to match common query patterns.
- Requirements:
  - `ENABLE QUERY REWRITE` on the materialized view.
  - `query_rewrite_enabled` and appropriate `query_rewrite_integrity` at system/session level.
  - Logical compatibility between query and materialized view.
- Benefits: Performance gains without changing application SQL.
