
Here’s a concise summary of **Advanced SQL: Set Operations** in Oracle, with examples.

---

## 1. What are Set Operations?

Set operations combine the results of **two or more SELECT queries** into a single result set.  
In Oracle, the main set operators are:

- `UNION`
- `UNION ALL`
- `INTERSECT`
- `MINUS` (Oracle’s version of EXCEPT)

**Rules:**

- Each `SELECT` must have the **same number of columns**.
- Corresponding columns must be of **compatible data types**.
- Column names in the final result come from the **first SELECT**.

---

## 2. UNION

Combines results from both queries and **removes duplicates**.

```sql
-- Input tables
-- EMP_SALES
-- EMP_ID | NAME   | DEPT
-- 1      | Alice  | Sales
-- 2      | Bob    | Sales

-- EMP_MARKETING
-- EMP_ID | NAME   | DEPT
-- 3      | Carol  | Marketing
-- 2      | Bob    | Marketing

SELECT NAME, DEPT
FROM EMP_SALES
UNION
SELECT NAME, DEPT
FROM EMP_MARKETING;
```

**Output:**

| NAME  | DEPT      |
|-------|-----------|
| Alice | Sales     |
| Bob   | Sales     |
| Carol | Marketing |
| Bob   | Marketing |

If a row is exactly identical in both queries, it appears **once**.

---

## 3. UNION ALL

Combines results and **keeps duplicates** (faster, no distinct step).

```sql
SELECT NAME, DEPT
FROM EMP_SALES
UNION ALL
SELECT NAME, DEPT
FROM EMP_MARKETING;
```

**Output (note duplicates if any rows match exactly):**

| NAME  | DEPT      |
|-------|-----------|
| Alice | Sales     |
| Bob   | Sales     |
| Carol | Marketing |
| Bob   | Marketing |
| ...   | ...       |

If the same row appears in both queries, it appears **twice** (or more).

---

## 4. INTERSECT

Returns only rows that appear in **both** result sets (logical AND).

```sql
-- EMP_2023
-- EMP_ID | NAME
-- 1      | Alice
-- 2      | Bob
-- 3      | Carol

-- EMP_2024
-- EMP_ID | NAME
-- 2      | Bob
-- 3      | Carol
-- 4      | David

SELECT EMP_ID, NAME
FROM EMP_2023
INTERSECT
SELECT EMP_ID, NAME
FROM EMP_2024;
```

**Output:**

| EMP_ID | NAME  |
|--------|-------|
| 2      | Bob   |
| 3      | Carol |

Only rows present in **both** queries.

---

## 5. MINUS (Oracle’s EXCEPT)

Returns rows from the **first** query that are **not** in the second (A − B).

```sql
SELECT EMP_ID, NAME
FROM EMP_2023
MINUS
SELECT EMP_ID, NAME
FROM EMP_2024;
```

**Output:**

| EMP_ID | NAME  |
|--------|-------|
| 1      | Alice |

Alice is in 2023 but not in 2024.

---

## 6. Ordering and Parentheses

- `ORDER BY` can appear **only once**, at the end of the entire set expression.
- Use parentheses to control evaluation when mixing operators.

```sql
(
  SELECT NAME, DEPT FROM EMP_SALES
  UNION
  SELECT NAME, DEPT FROM EMP_MARKETING
)
MINUS
(
  SELECT NAME, DEPT FROM EMP_TERMINATED
)
ORDER BY NAME;
```

---

## 7. Typical Use Cases

- Combine similar data from multiple tables or time periods (`UNION` / `UNION ALL`).
- Find common elements between datasets (`INTERSECT`).
- Find records present in one dataset but not another (`MINUS`).
