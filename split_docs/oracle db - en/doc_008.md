
Here’s a concise summary of the key ideas around the advanced SQL analytic functions **LAG** and **LEAD** in Oracle, with examples.

---

## 1. What LAG and LEAD Do

- **LAG(expr, offset, default)**  
  Returns the value of `expr` from a **previous row** in the result set (based on a specified order).

- **LEAD(expr, offset, default)**  
  Returns the value of `expr` from a **following row** in the result set.

They are **analytic functions** used with `OVER (PARTITION BY ... ORDER BY ...)` to look “back” or “forward” across rows without self-joins or subqueries.

---

## 2. Basic Syntax

```sql
LAG ( value_expression [, offset [, default ] ] )
OVER ( [PARTITION BY partition_expression] ORDER BY order_expression )

LEAD( value_expression [, offset [, default ] ] )
OVER ( [PARTITION BY partition_expression] ORDER BY order_expression )
```

- `value_expression`: column or expression to look at.
- `offset`: how many rows back/forward (default = 1).
- `default`: value if there is no such row (default = NULL).
- `PARTITION BY`: optional; restarts the calculation per group.
- `ORDER BY`: required; defines row sequence.

---

## 3. Example Table

Assume a table:

```sql
CREATE TABLE sales (
    id         NUMBER,
    customer   VARCHAR2(50),
    sale_date  DATE,
    amount     NUMBER
);

INSERT INTO sales (id, customer, sale_date, amount) VALUES (1, 'A', DATE '2024-01-01', 100);
INSERT INTO sales (id, customer, sale_date, amount) VALUES (2, 'A', DATE '2024-01-05', 150);
INSERT INTO sales (id, customer, sale_date, amount) VALUES (3, 'A', DATE '2024-01-10', 200);
INSERT INTO sales (id, customer, sale_date, amount) VALUES (4, 'B', DATE '2024-01-02', 300);
INSERT INTO sales (id, customer, sale_date, amount) VALUES (5, 'B', DATE '2024-01-08', 250);
COMMIT;
```

---

## 4. LAG: Compare to Previous Row

### 4.1 Previous Amount per Customer

```sql
SELECT
    customer,
    sale_date,
    amount,
    LAG(amount, 1, 0) OVER (
        PARTITION BY customer
        ORDER BY sale_date
    ) AS prev_amount
FROM sales
ORDER BY customer, sale_date;
```

**Output (conceptual):**

| customer | sale_date   | amount | prev_amount |
|----------|-------------|--------|-------------|
| A        | 2024-01-01  | 100    | 0           |
| A        | 2024-01-05  | 150    | 100         |
| A        | 2024-01-10  | 200    | 150         |
| B        | 2024-01-02  | 300    | 0           |
| B        | 2024-01-08  | 250    | 300         |

### 4.2 Difference from Previous Amount

```sql
SELECT
    customer,
    sale_date,
    amount,
    amount - LAG(amount, 1, amount) OVER (
        PARTITION BY customer
        ORDER BY sale_date
    ) AS diff_from_prev
FROM sales
ORDER BY customer, sale_date;
```

**Output (conceptual):**

| customer | sale_date   | amount | diff_from_prev |
|----------|-------------|--------|----------------|
| A        | 2024-01-01  | 100    | 0              |
| A        | 2024-01-05  | 150    | 50             |
| A        | 2024-01-10  | 200    | 50             |
| B        | 2024-01-02  | 300    | 0              |
| B        | 2024-01-08  | 250    | -50            |

## 5. LEAD: Look at Next Row

### 5.1 Next Amount per Customer

```sql
SELECT
    customer,
    sale_date,
    amount,
    LEAD(amount, 1, 0) OVER (
        PARTITION BY customer
        ORDER BY sale_date
    ) AS next_amount
FROM sales
ORDER BY customer, sale_date;
```

**Output (conceptual):**

| customer | sale_date   | amount | next_amount |
|----------|-------------|--------|-------------|
| A        | 2024-01-01  | 100    | 150         |
| A        | 2024-01-05  | 150    | 200         |
| A        | 2024-01-10  | 200    | 0           |
| B        | 2024-01-02  | 300    | 250         |
| B        | 2024-01-08  | 250    | 0           |

### 5.2 Days Until Next Purchase

```sql
SELECT
    customer,
    sale_date,
    amount,
    LEAD(sale_date) OVER (
        PARTITION BY customer
        ORDER BY sale_date
    ) AS next_sale_date,
    ( LEAD(sale_date) OVER (
          PARTITION BY customer
          ORDER BY sale_date
      ) - sale_date
    ) AS days_to_next
FROM sales
ORDER BY customer, sale_date;
```

**Output (conceptual):**

| customer | sale_date   | amount | next_sale_date | days_to_next |
|----------|-------------|--------|----------------|--------------|
| A        | 2024-01-01  | 100    | 2024-01-05     | 4            |
| A        | 2024-01-05  | 150    | 2024-01-10     | 5            |
| A        | 2024-01-10  | 200    | NULL           | NULL         |
| B        | 2024-01-02  | 300    | 2024-01-08     | 6            |
| B        | 2024-01-08  | 250    | NULL           | NULL         |

## 6. Typical Use Cases

- Calculate **differences** between consecutive rows (amount, balance, etc.).
- Detect **changes** in status or value (e.g., when a value differs from previous).
- Compute **time gaps** between events (days between orders, log entries).
- Build **running comparisons** without self-joins.
