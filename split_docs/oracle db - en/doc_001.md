
Here’s a concise summary of the core SQL DML (Data Manipulation Language) statements in Oracle, with simple examples and expected results.

---

## 1. `SELECT` – Query Data

Used to read data from tables.

**Basic syntax:**
```sql
SELECT column1, column2, ...
FROM   table_name
WHERE  condition;
```

**Example:**
```sql
-- Input
SELECT employee_id, first_name, salary
FROM   employees
WHERE  department_id = 10;
```

**Output (sample):**
```text
EMPLOYEE_ID | FIRST_NAME | SALARY
----------- | ---------- | ------
        101 | John       | 5000
        102 | Mary       | 5500
```

---

## 2. `INSERT` – Add New Rows

Used to insert new records into a table.

**Insert specific columns:**
```sql
INSERT INTO employees (employee_id, first_name, last_name, salary, department_id)
VALUES (201, 'Alice', 'Brown', 6000, 20);
```

**Insert all columns (must match table structure):**
```sql
INSERT INTO employees
VALUES (202, 'Bob', 'Green', 'BGREEN', '555-1234', SYSDATE, 'IT_PROG', 4000, NULL, NULL, 60);
```

**Insert from another table:**
```sql
INSERT INTO employees_backup (employee_id, first_name, last_name, salary)
SELECT employee_id, first_name, last_name, salary
FROM   employees
WHERE  department_id = 10;
```

---

## 3. `UPDATE` – Modify Existing Rows

Used to change data in existing rows.

**Basic syntax:**
```sql
UPDATE table_name
SET    column1 = value1,
       column2 = value2
WHERE  condition;
```

**Example:**
```sql
-- Input
UPDATE employees
SET    salary = salary * 1.10
WHERE  department_id = 10;
```

**Effect (sample):**
Before:
```text
EMPLOYEE_ID | SALARY
----------- | ------
        101 | 5000
        102 | 5500
```

After:
```text
EMPLOYEE_ID | SALARY
----------- | ------
        101 | 5500
        102 | 6050
```

---

## 4. `DELETE` – Remove Rows

Used to delete rows from a table.

**Basic syntax:**
```sql
DELETE FROM table_name
WHERE  condition;
```

**Example:**
```sql
-- Input
DELETE FROM employees
WHERE  employee_id = 201;
```

**Effect:**
Row with `employee_id = 201` is removed from `employees`.

> Without a `WHERE` clause, **all rows** are deleted:
> ```sql
> DELETE FROM employees;  -- removes all rows
> ```

---

## 5. `MERGE` – Upsert (Insert or Update)

Oracle’s `MERGE` allows you to insert new rows or update existing rows in one statement, based on a match condition. Often used for syncing a target table with a source table.

**Basic syntax:**
```sql
MERGE INTO target t
USING      source s
ON         (t.key_column = s.key_column)
WHEN MATCHED THEN
  UPDATE SET t.col1 = s.col1,
             t.col2 = s.col2
WHEN NOT MATCHED THEN
  INSERT (t.key_column, t.col1, t.col2)
  VALUES (s.key_column, s.col1, s.col2);
```

**Example:**
Assume `employees` is the target and `employees_staging` is the source.

```sql
MERGE INTO employees e
USING      employees_staging s
ON         (e.employee_id = s.employee_id)
WHEN MATCHED THEN
  UPDATE SET e.salary        = s.salary,
             e.department_id = s.department_id
WHEN NOT MATCHED THEN
  INSERT (employee_id, first_name, last_name, salary, department_id)
  VALUES (s.employee_id, s.first_name, s.last_name, s.salary, s.department_id);
```

**Effect:**
- If `employee_id` exists in `employees`, that row is updated.
- If `employee_id` does not exist, a new row is inserted.

---

## 6. Transaction Control (Context)

These DML operations are usually followed by transaction control:

```sql
COMMIT;    -- Save changes permanently
ROLLBACK;  -- Undo changes since last COMMIT
```

---

**Key Ideas:**
- `SELECT` → read data  
- `INSERT` → add new rows  
- `UPDATE` → change existing rows  
- `DELETE` → remove rows  
- `MERGE` → insert or update in one step (upsert), based on a match condition
