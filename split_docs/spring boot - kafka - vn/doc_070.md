
Dưới đây là tóm tắt ngắn gọn về “Security: Encryption in transit (SSL) and at rest (disk-level, cloud-level)” trong bối cảnh Kafka + Spring Boot, kèm ví dụ code.

---

## 1. Ý chính

1. **Encryption in transit (mã hóa khi truyền dữ liệu)**  
   - Dùng **SSL/TLS** giữa:
     - Producer ↔ Kafka Broker  
     - Consumer ↔ Kafka Broker  
     - Kafka Broker ↔ Kafka Broker (inter-broker)  
   - Mục tiêu: bảo vệ dữ liệu khỏi bị nghe lén hoặc sửa đổi trên đường truyền.

2. **Encryption at rest (mã hóa khi lưu trữ)**  
   - Mã hóa dữ liệu trên **đĩa** nơi Kafka lưu log (disk-level encryption).  
   - Hoặc dùng **cloud-level encryption** (VD: AWS KMS, GCP CMEK, Azure Disk Encryption).  
   - Mục tiêu: nếu đĩa hoặc snapshot bị lộ, dữ liệu vẫn được bảo vệ.

## 2. Ví dụ cấu hình SSL cho Kafka với Spring Boot

### 2.1. application.yml (Producer & Consumer)

```yaml
spring:
  kafka:
    bootstrap-servers: broker1:9093,broker2:9093
    properties:
      security.protocol: SSL
      ssl.truststore.location: classpath:kafka-truststore.jks
      ssl.truststore.password: changeit
      ssl.keystore.location: classpath:kafka-keystore.jks
      ssl.keystore.password: changeit
      ssl.key.password: changeit

    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer

    consumer:
      group-id: demo-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      auto-offset-reset: earliest
```

### 2.2. Producer đơn giản

```java
@Service
public class DemoProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public DemoProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String topic, String key, String value) {
        kafkaTemplate.send(topic, key, value)
                     .whenComplete((result, ex) -> {
                         if (ex == null) {
                             System.out.println("Sent OK: " + result.getRecordMetadata());
                         } else {
                             System.err.println("Send failed: " + ex.getMessage());
                         }
                     });
    }
}
```

**Input (gọi service):**

```java
demoProducer.sendMessage("secure-topic", "user-1", "Xin chào Kafka qua SSL");
```

**Output (log):**

```text
Sent OK: secure-topic-0@offset 15
```

Dữ liệu `"Xin chào Kafka qua SSL"` được mã hóa trên đường truyền nhờ SSL/TLS.

## 3. Ví dụ cấu hình SSL cho Kafka Broker (in transit)

`server.properties` (trên broker):

```properties
listeners=SSL://:9093
advertised.listeners=SSL://broker1:9093
security.inter.broker.protocol=SSL

ssl.keystore.location=/opt/kafka/config/kafka.server.keystore.jks
ssl.keystore.password=changeit
ssl.key.password=changeit
ssl.truststore.location=/opt/kafka/config/kafka.server.truststore.jks
ssl.truststore.password=changeit
ssl.client.auth=required
```

## 4. Encryption at rest (mã hóa khi lưu trữ)

### 4.1. Disk-level encryption (Linux)

- Dùng LUKS/dm-crypt để mã hóa partition chứa Kafka logs, ví dụ `/var/lib/kafka`.

```bash
# Ví dụ rất rút gọn (không dùng trực tiếp trên hệ thống thật)
cryptsetup luksFormat /dev/sdb
cryptsetup open /dev/sdb kafka_data
mkfs.ext4 /dev/mapper/kafka_data
mount /dev/mapper/kafka_data /var/lib/kafka
```

Kafka vẫn ghi log bình thường, nhưng dữ liệu trên đĩa đã được mã hóa.

### 4.2. Cloud-level encryption

- **AWS**: dùng EBS với encryption enabled, hoặc S3 với SSE-KMS cho backup.  
- **GCP**: dùng CMEK cho Persistent Disk.  
- **Azure**: Azure Disk Encryption / SSE với customer-managed keys.

Không cần thay đổi code Spring Boot/Kafka; cấu hình nằm ở tầng hạ tầng (infrastructure).

---

## 5. Tóm tắt

- **Encryption in transit (SSL/TLS)**:  
  - Bảo vệ dữ liệu khi di chuyển giữa client ↔ broker ↔ broker.  
  - Cần cấu hình SSL cho cả Kafka Broker và Spring Boot (producer/consumer).

- **Encryption at rest**:  
  - Bảo vệ dữ liệu khi lưu trên đĩa hoặc cloud storage.  
  - Thường cấu hình ở mức hệ điều hành hoặc cloud provider, không cần sửa code ứng dụng.
