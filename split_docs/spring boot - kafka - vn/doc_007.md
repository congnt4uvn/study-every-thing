
Dưới đây là tóm tắt ngắn gọn về luồng dữ liệu Kafka (Producer → Broker → Consumer) và cách message được ghi vào log, kèm ví dụ Spring Boot.

---

## 1. Luồng dữ liệu Kafka: Producer → Broker → Consumer

**Ý chính:**

1. **Producer**
   - Ứng dụng gửi message đến Kafka.
   - Chọn **topic** và **partition** (theo key hoặc round-robin).
   - Gửi message dạng record: `key`, `value`, `headers`, `timestamp`.

2. **Broker**
   - Là server Kafka, lưu trữ dữ liệu.
   - Mỗi **topic** được chia thành nhiều **partition**.
   - Mỗi partition là một **log append-only**:
     - Message mới luôn được **append** (ghi thêm) vào cuối file log.
     - Mỗi message có một **offset** tăng dần (0,1,2,3,…).
   - Broker không “xóa ngay” message sau khi consumer đọc; dữ liệu được giữ theo:
     - **Retention time** (thời gian giữ)
     - hoặc **Retention size** (dung lượng tối đa).

3. **Consumer**
   - Đọc message từ topic/partition theo **offset**.
   - Consumer group:
     - Các consumer trong cùng group chia nhau các partition.
     - Mỗi partition chỉ được xử lý bởi **1 consumer** trong group tại một thời điểm.
   - Consumer lưu **offset đã đọc** (commit offset) để biết lần sau đọc từ đâu.

---

## 2. Cách message được append vào log

- Mỗi partition là một file log (hoặc nhiều segment file).
- Khi producer gửi message:
  1. Broker nhận message.
  2. Ghi message vào cuối log của partition tương ứng.
  3. Gán **offset** mới (offset tăng dần).
- Đọc message:
  - Consumer đọc theo offset: từ offset N đến N+1, N+2, …
  - Không có “update” hay “delete” từng record; chỉ có append và xóa theo chính sách retention.

## 3. Ví dụ Spring Boot + Kafka

### 3.1. Cấu hình Maven

```xml
<dependencies>
    <!-- Spring Kafka -->
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>
    <!-- Web (nếu cần REST) -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
</dependencies>
```

### 3.2. application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    consumer:
      group-id: demo-group
      auto-offset-reset: earliest
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

### 3.3. Producer (Spring Boot)

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class DemoProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private static final String TOPIC = "demo-topic";

    public DemoProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String key, String message) {
        kafkaTemplate.send(TOPIC, key, message);
    }
}
```

**Ví dụ input (REST):**

```java
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/messages")
public class MessageController {

    private final DemoProducer producer;

    public MessageController(DemoProducer producer) {
        this.producer = producer;
    }

    @PostMapping
    public String send(@RequestParam String key, @RequestParam String value) {
        producer.sendMessage(key, value);
        return "Sent: key=" + key + ", value=" + value;
    }
}
```

**Gọi API (input):**

```bash
curl -X POST "http://localhost:8080/api/messages?key=user1&value=HelloKafka"
```

**Kết quả (output logic):**

- Producer gửi record: `(key="user1", value="HelloKafka")` đến topic `demo-topic`.
- Broker append record vào cuối log của một partition, ví dụ:
  - Partition 0, offset 15: `key=user1, value=HelloKafka`.

---

### 3.4. Consumer (Spring Boot)

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class DemoConsumer {

    @KafkaListener(topics = "demo-topic", groupId = "demo-group")
    public void listen(String message) {
        System.out.println("Received message: " + message);
    }
}
```

**Khi có message mới được append vào log:**

- Consumer trong group `demo-group` sẽ đọc message theo offset tiếp theo.
- Ví dụ:
  - Offset 15: `HelloKafka`
  - Console output:
    ```text
    Received message: HelloKafka
    ```

---

## 4. Tóm tắt ngắn

- **Producer** gửi message đến **topic**.
- **Broker** lưu message trong **partition** dưới dạng **log append-only**, gán **offset** tăng dần.
- **Consumer** đọc message theo offset, không làm mất dữ liệu; retention policy mới quyết định khi nào log bị xóa.
- Spring Boot + Spring Kafka giúp triển khai producer/consumer dễ dàng với annotation `@KafkaListener` và `KafkaTemplate`.
