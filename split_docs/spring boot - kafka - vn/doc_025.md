## 1. Idempotent Producer là gì?

- Idempotent producer đảm bảo **“exactly-once” trên phía producer** (không gửi trùng bản ghi do retry).
- Kafka dùng `producerId` và `sequence number` để loại bỏ bản ghi trùng lặp.
- Bật idempotence giúp:
  - Không bị ghi trùng khi producer retry.
  - Đảm bảo **thứ tự** trong cùng một partition.

### Cấu hình idempotent producer (Spring Boot)

```yaml
spring:
  kafka:
    producer:
      bootstrap-servers: localhost:9092
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      properties:
        enable.idempotence: true
        acks: all
        retries: 5
        max.in.flight.requests.per.connection: 5
```

Hoặc Java config:

```java
@Bean
public Map<String, Object> producerConfigs() {
    Map<String, Object> props = new HashMap<>();
    props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
    props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
    props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
    props.put(ProducerConfig.ENABLE_IDEMPOTENCE_CONFIG, true);
    props.put(ProducerConfig.ACKS_CONFIG, "all");
    props.put(ProducerConfig.RETRIES_CONFIG, 5);
    props.put(ProducerConfig.MAX_IN_FLIGHT_REQUESTS_PER_CONNECTION, 5);
    return props;
}
```

## 2. Partitioning và tác động khi dùng Idempotent Producer

### 2.1. Partitioning là gì?

- Topic được chia thành nhiều **partition**.
- Thứ tự message chỉ được đảm bảo **trong cùng một partition**.
- Producer quyết định message đi vào partition nào dựa trên:
  - **Key** (hash(key) % numPartitions)
  - Hoặc custom partitioner.

### 2.2. Tác động khi dùng Idempotent Producer

- Idempotence đảm bảo:
  - Không trùng lặp và giữ thứ tự **trong từng partition**.
- Nhưng:
  - **Không đảm bảo thứ tự giữa các partition khác nhau**.
  - Nếu bạn thay đổi chiến lược partition (ví dụ đổi key, đổi số partition), thứ tự global có thể bị phá vỡ.
- Vì vậy:
  - Nếu bạn cần **thứ tự cho một entity cụ thể** (ví dụ theo userId, orderId), hãy dùng key cố định để tất cả message của entity đó vào **cùng một partition**.

## 3. Chiến lược Partitioning phổ biến

### 3.1. Dùng key (hash-based)

- Mọi message có cùng key → cùng partition → giữ được thứ tự cho key đó.

```java
kafkaTemplate.send("orders", "user-123", "OrderCreated: #1001");
kafkaTemplate.send("orders", "user-123", "OrderPaid: #1001");
```

- Cả hai message trên sẽ vào cùng một partition (ví dụ partition 2), nên consumer sẽ đọc đúng thứ tự.

**Input (producer):**

```text
Topic: orders
Key: "user-123"
Value: "OrderCreated: #1001"

Topic: orders
Key: "user-123"
Value: "OrderPaid: #1001"
```

**Output (consumer đọc từ partition 2):**

```text
1) "OrderCreated: #1001"
2) "OrderPaid: #1001"
```

### 3.2. Custom Partitioner

- Khi logic phân vùng phức tạp hơn (ví dụ: theo region, theo type).

```java
public class RegionPartitioner implements Partitioner {

    @Override
    public int partition(String topic, Object key, byte[] keyBytes,
                         Object value, byte[] valueBytes, Cluster cluster) {
        String region = (String) key; // ví dụ key = "VN", "US"
        List<PartitionInfo> partitions = cluster.partitionsForTopic(topic);
        int numPartitions = partitions.size();

        // Ví dụ: VN -> partition 0, US -> partition 1, còn lại -> partition 2
        switch (region) {
            case "VN": return 0;
            case "US": return 1;
            default:   return 2 % numPartitions;
        }
    }

    @Override
    public void close() {}

    @Override
    public void configure(Map<String, ?> configs) {}
}
```

Cấu hình:

```yaml
spring:
  kafka:
    producer:
      properties:
        partitioner.class: com.example.kafka.RegionPartitioner
        enable.idempotence: true
```

## 4. Ví dụ Spring Boot: Producer + Consumer

### 4.1. Producer Service

```java
@Service
public class OrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public OrderProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendOrderEvent(String userId, String event) {
        // userId là key -> đảm bảo tất cả event của user này vào cùng partition
        kafkaTemplate.send("orders", userId, event);
    }
}
```

**Input (REST API):**

```http
POST /orders/event
Content-Type: application/json

{
  "userId": "user-123",
  "event": "OrderCreated: #1001"
}
```

**Producer gửi:**

```java
orderProducer.sendOrderEvent("user-123", "OrderCreated: #1001");
orderProducer.sendOrderEvent("user-123", "OrderPaid: #1001");
```

### 4.2. Consumer

```java
@Service
public class OrderConsumer {

    @KafkaListener(topics = "orders", groupId = "order-service")
    public void listen(ConsumerRecord<String, String> record) {
        System.out.printf(
            "Partition=%d, Key=%s, Value=%s, Offset=%d%n",
            record.partition(), record.key(), record.value(), record.offset()
        );
    }
}
```

**Output (log consumer):**

```text
Partition=2, Key=user-123, Value=OrderCreated: #1001, Offset=15
Partition=2, Key=user-123, Value=OrderPaid: #1001, Offset=16
```

Thứ tự cho `user-123` được giữ nguyên nhờ:
- Cùng partition (do key).
- Idempotent producer đảm bảo không bị trùng/đảo thứ tự khi retry.

---

## 5. Ý chính cần nhớ

1. **Idempotent producer**:
   - Bật `enable.idempotence=true` để tránh trùng lặp và giữ thứ tự trong partition.
2. **Partitioning**:
   - Thứ tự chỉ được đảm bảo **trong cùng một partition**.
   - Dùng key (hoặc custom partitioner) để gom các message liên quan vào cùng partition.
3. **Tác động**:
   - Idempotence + partitioning đúng cách → an toàn khi retry, không trùng, giữ thứ tự cho từng entity (theo key).
   - Thay đổi số partition hoặc chiến lược partition có thể làm thay đổi mapping key → partition, ảnh hưởng đến thứ tự.
