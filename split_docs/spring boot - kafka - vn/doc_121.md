
Dưới đây là tóm tắt ngắn gọn về **Kafka Streams state & Exactly-once processing (EOS v2)**, kèm ví dụ code (Spring Boot + Kafka Streams) và mô tả input/output.

---

## 1. Khái niệm chính

1. **Stateful processing trong Kafka Streams**  
   - Nhiều phép toán trong Kafka Streams là *stateful* (có trạng thái): `aggregate`, `join`, `windowed operations`, v.v.  
   - Trạng thái được lưu trong **state store** (thường là RocksDB) và được **changelog** sang Kafka topic để có thể khôi phục.

2. **Exactly-once processing (EOS)**  
   - Đảm bảo mỗi bản ghi được xử lý **chính xác một lần**, kể cả khi có lỗi, restart, rebalance.  
   - Tránh:
     - Double-processing (xử lý lại bản ghi)
     - Double-writing (ghi trùng dữ liệu ra output topic hoặc DB)

3. **EOS v2 trong Kafka Streams**  
   - Phiên bản cải tiến của EOS (từ Kafka 2.5+).  
   - Đơn giản hơn, hiệu năng tốt hơn so với EOS v1.  
   - Sử dụng **transactional producer** + **idempotent producer** + **read-process-write** trong một transaction.  
   - Cấu hình qua `processing.guarantee=exactly_once_v2`.

---

## 2. Cách hoạt động EOS v2 (tóm tắt)

- Mỗi instance Kafka Streams có một **transactional.id** duy nhất.  
- Mỗi batch xử lý:
  1. Đọc bản ghi từ input topic.
  2. Cập nhật state store (RocksDB).
  3. Ghi kết quả ra output topic.
  4. Ghi offset đã xử lý vào transaction.
  5. **Commit transaction**: offset + output + changelog được commit atomically.  
- Nếu có lỗi:
  - Transaction bị **abort**, offset không được commit → bản ghi sẽ được xử lý lại, nhưng không gây double-write vì producer idempotent + transaction.

## 3. Cấu hình Kafka Streams với EOS v2 (Spring Boot)

### 3.1. application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    streams:
      application-id: order-processor-app
      properties:
        processing.guarantee: exactly_once_v2
        replication.factor: 1
        default.key.serde: org.apache.kafka.common.serialization.Serdes$StringSerde
        default.value.serde: org.apache.kafka.common.serialization.Serdes$StringSerde
```

## 4. Ví dụ: Aggregation stateful với EOS v2

### Bài toán

- Input topic: `orders`  
  - Key: `customerId`  
  - Value: JSON đơn giản `{ "orderId": "...", "amount": 100 }`
- Output topic: `customer-totals`  
  - Key: `customerId`  
  - Value: tổng `amount` của khách hàng đó.

### 4.1. Topology (Java, Spring Boot)

```java
import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.kstream.*;
import org.apache.kafka.common.serialization.Serdes;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class KafkaStreamsConfig {

    @Bean
    public KStream<String, String> kStream(StreamsBuilder builder) {
        KStream<String, String> orders = builder.stream("orders",
                Consumed.with(Serdes.String(), Serdes.String()));

        KTable<String, Long> customerTotals = orders
                .groupByKey()
                .aggregate(
                        () -> 0L, // initial value
                        (customerId, orderJson, currentTotal) -> {
                            long amount = extractAmount(orderJson);
                            return currentTotal + amount;
                        },
                        Materialized.with(Serdes.String(), Serdes.Long())
                );

        customerTotals
                .toStream()
                .to("customer-totals", Produced.with(Serdes.String(), Serdes.Long()));

        return orders;
    }

    private long extractAmount(String orderJson) {
        // Ví dụ đơn giản: giả sử orderJson = "100"
        // Thực tế dùng Jackson/Gson để parse JSON
        return Long.parseLong(orderJson);
    }
}
```

## 5. Input / Output minh họa

### Input (topic `orders`)

| Offset | Key (customerId) | Value (amount) |
|--------|------------------|----------------|
| 0      | C001             | "100"          |
| 1      | C001             | "50"           |
| 2      | C002             | "200"          |
| 3      | C001             | "25"           |

### Output (topic `customer-totals`)

| Offset | Key (customerId) | Value (totalAmount) | Giải thích                         |
|--------|------------------|---------------------|------------------------------------|
| 0      | C001             | 100                 | 100                                |
| 1      | C001             | 150                 | 100 + 50                           |
| 2      | C002             | 200                 | 200                                |
| 3      | C001             | 175                 | 150 + 25                           |

- Nếu ứng dụng bị crash sau khi xử lý offset 1 nhưng trước khi commit:
  - EOS v2 đảm bảo:
    - Không ghi trùng bản ghi `C001:150` nhiều lần.
    - Không bỏ sót bản ghi nào (offset được commit atomically với output).

---

## 6. Lưu ý khi dùng EOS v2

1. **Bật EOS v2**: `processing.guarantee=exactly_once_v2`.  
2. **Idempotent & transactional producer** được Kafka Streams tự cấu hình, không cần tự làm.  
3. **State store** + **changelog topic** phải được cấu hình replication factor phù hợp để đảm bảo an toàn dữ liệu.  
4. EOS có overhead, nhưng v2 đã tối ưu hơn v1; vẫn chậm hơn at-least-once, đổi lại là tính đúng đắn.
