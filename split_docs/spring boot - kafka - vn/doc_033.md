
Dưới đây là tóm tắt ngắn gọn về **Producer reliability – Idempotent Producer trong Kafka**, kèm ví dụ code (Java Spring Boot) và mô tả input/output.

---

## 1. Idempotent Producer là gì?

- Là cơ chế giúp **producer gửi message “nhiều lần nhưng chỉ được ghi 1 lần”** vào Kafka.
- Mục tiêu: **tránh ghi trùng (duplicate)** khi:
  - Producer retry gửi message.
  - Network bị lỗi, timeout, v.v.

---

## 2. Cách hoạt động: PID & Sequence Number

### PID (Producer ID)
- Khi bật idempotence, Kafka **cấp một ID duy nhất** cho mỗi producer: **PID**.
- PID được dùng để Kafka **nhận diện producer** và quản lý thứ tự message.

### Sequence Number
- Mỗi producer (PID) + mỗi partition có một **sequence number** tăng dần: `0, 1, 2, 3, ...`.
- Producer gửi message kèm:
  - `PID`
  - `sequence number`
- Broker (Kafka) lưu **sequence number cuối cùng đã chấp nhận** cho mỗi `(PID, partition)`.

### Cơ chế chống trùng
- Khi broker nhận message:
  - Nếu `sequence number == last + 1` → **chấp nhận** (message mới, đúng thứ tự).
  - Nếu `sequence number <= last` → **bỏ qua** (coi là duplicate).
  - Nếu `sequence number > last + 1` → **phát hiện mất message** (có thể error).

Nhờ đó, nếu producer retry cùng một message (do timeout, network lỗi), broker sẽ **nhận ra và không ghi trùng**.

## 3. Cấu hình Idempotent Producer trong Spring Boot

### application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      # Bật idempotence
      properties:
        enable.idempotence: true
        # Đảm bảo an toàn
        acks: all
        retries: 2147483647   # Integer.MAX_VALUE
        max.in.flight.requests.per.connection: 5
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

## 4. Ví dụ Producer (Spring Boot)

### Producer Service

```java
@Service
public class ReliableProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public ReliableProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String topic, String key, String value) {
        ListenableFuture<SendResult<String, String>> future =
                kafkaTemplate.send(topic, key, value);

        future.addCallback(
            result -> {
                if (result != null) {
                    RecordMetadata meta = result.getRecordMetadata();
                    System.out.printf(
                        "Sent OK: topic=%s partition=%d offset=%d%n",
                        meta.topic(), meta.partition(), meta.offset()
                    );
                }
            },
            ex -> {
                System.err.println("Send failed: " + ex.getMessage());
            }
        );
    }
}
```

### REST Controller để test

```java
@RestController
@RequestMapping("/api/messages")
public class MessageController {

    private final ReliableProducer producer;

    public MessageController(ReliableProducer producer) {
        this.producer = producer;
    }

    @PostMapping
    public String send(@RequestParam String key, @RequestParam String value) {
        producer.sendMessage("demo-topic", key, value);
        return "Sent request";
    }
}
```

## 5. Input / Output minh họa

### Input (HTTP request)

```http
POST /api/messages?key=user-1&value=hello
```

### Hành vi bên trong

- Producer gửi record:
  - topic: `demo-topic`
  - key: `user-1`
  - value: `hello`
  - kèm theo: `PID`, `sequence number` (Kafka tự quản lý, bạn không cần set).

Nếu vì lỗi mạng, producer **retry** cùng message:

- Lần 1: broker nhận `sequence = 10` → ghi vào log.
- Lần 2 (retry): broker nhận lại `sequence = 10` → nhận ra là **duplicate** → **bỏ qua**.

### Output (log server)

```text
Sent OK: topic=demo-topic partition=0 offset=15
```

Trên consumer, bạn chỉ thấy **1 message duy nhất**, dù producer có thể đã gửi nhiều lần.

---

## 6. Tóm tắt ý chính

- **Idempotent producer** đảm bảo: **“at-least-once + không trùng lặp”** ở mức partition.
- Dựa trên:
  - **PID**: định danh producer.
  - **Sequence number**: đảm bảo thứ tự và phát hiện duplicate.
- Trong Spring Boot:
  - Bật `enable.idempotence=true`, `acks=all`, `retries` lớn, `max.in.flight.requests.per.connection` nhỏ (≤5).
- Phù hợp khi bạn cần **đảm bảo không ghi trùng message** trong Kafka dù có retry.
