
Dưới đây là tóm tắt ngắn gọn về các kỹ thuật nâng cao trong Spring Kafka liên quan đến **seek**, **pause** và **resume** việc tiêu thụ message, kèm ví dụ code và mô tả input/output.

---

## 1. Seek – Điều khiển offset thủ công

**Ý tưởng chính**

- Mặc định, Spring Kafka sẽ đọc message theo offset hiện tại.
- Với `ConsumerSeekAware` hoặc `ConsumerSeekCallback`, bạn có thể:
  - Nhảy đến offset cụ thể (seek)
  - Đọc lại từ đầu partition
  - Bỏ qua một số message

**Trường hợp dùng**

- Reprocess (xử lý lại) message lỗi
- Bỏ qua message hỏng
- Debug / replay dữ liệu

### Ví dụ: Seek đến offset cụ thể

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.apache.kafka.common.TopicPartition;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.kafka.listener.ConsumerSeekAware;

import java.util.Map;

public class SeekListener implements ConsumerSeekAware {

    private ConsumerSeekCallback seekCallback;

    @Override
    public void registerSeekCallback(ConsumerSeekCallback callback) {
        this.seekCallback = callback;
    }

    @Override
    public void onPartitionsAssigned(Map<TopicPartition, Long> assignments,
                                     ConsumerSeekCallback callback) {
        // Ví dụ: luôn đọc lại từ offset 0 cho mọi partition
        assignments.keySet().forEach(tp -> callback.seek(tp.topic(), tp.partition(), 0L));
    }

    @KafkaListener(topics = "demo-topic", groupId = "demo-group")
    public void listen(ConsumerRecord<String, String> record) {
        System.out.printf("Received: topic=%s partition=%d offset=%d value=%s%n",
                record.topic(), record.partition(), record.offset(), record.value());
    }
}
```

**Input (Kafka messages)**  
Topic: `demo-topic`, Partition 0  
Offsets và value:

| offset | value   |
|--------|---------|
| 0      | "A"     |
| 1      | "B"     |
| 2      | "C"     |

**Output (log)**  
Mỗi lần consumer được assign partition, nó sẽ seek về 0, nên có thể đọc lại từ A, B, C (tùy cấu hình commit offset).

## 2. Pause / Resume – Tạm dừng và tiếp tục tiêu thụ

**Ý tưởng chính**

- Dùng `pause()` để tạm dừng nhận message mới từ Kafka.
- Dùng `resume()` để tiếp tục.
- Hữu ích khi:
  - Hệ thống downstream bị quá tải
  - Cần tạm dừng xử lý khi gặp lỗi tạm thời
  - Thực hiện backpressure thủ công

### 2.1. Pause/Resume qua `KafkaListenerEndpointRegistry`

Bạn có thể điều khiển listener container bằng `KafkaListenerEndpointRegistry`.

```java
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.kafka.config.KafkaListenerEndpointRegistry;
import org.springframework.kafka.listener.MessageListenerContainer;
import org.springframework.stereotype.Service;

@Service
public class ListenerControlService {

    @Autowired
    private KafkaListenerEndpointRegistry registry;

    public void pauseListener(String listenerId) {
        MessageListenerContainer container = registry.getListenerContainer(listenerId);
        if (container != null) {
            container.pause();
        }
    }

    public void resumeListener(String listenerId) {
        MessageListenerContainer container = registry.getListenerContainer(listenerId);
        if (container != null) {
            container.resume();
        }
    }
}
```

Listener:

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Component;

@Component
public class MyListener {

    @KafkaListener(id = "myListenerId", topics = "demo-topic", groupId = "demo-group")
    public void listen(String message) {
        System.out.println("Received: " + message);
    }
}
```

**Input/Output mô tả**

- Trước khi `pauseListener("myListenerId")`:
  - Kafka gửi message: `"msg1"`, `"msg2"`, `"msg3"`
  - Log:
    - `Received: msg1`
    - `Received: msg2`
    - `Received: msg3`
- Sau khi pause:
  - Kafka gửi thêm `"msg4"`, `"msg5"` → listener **không** xử lý (không log).
- Sau khi `resumeListener("myListenerId")`:
  - Consumer sẽ tiếp tục đọc từ offset hiện tại, xử lý `"msg4"`, `"msg5"`:
    - `Received: msg4`
    - `Received: msg5`

> Lưu ý: Pause không làm mất message, chỉ tạm dừng việc poll.

### 2.2. Pause/Resume theo logic trong listener

Bạn có thể pause ngay trong listener khi phát hiện điều kiện nào đó (ví dụ: downstream quá tải).

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.kafka.config.KafkaListenerEndpointRegistry;
import org.springframework.kafka.listener.MessageListenerContainer;
import org.springframework.stereotype.Component;

@Component
public class DynamicPauseListener {

    @Autowired
    private KafkaListenerEndpointRegistry registry;

    private volatile boolean overloaded = false;

    @KafkaListener(id = "dynamicListener", topics = "demo-topic", groupId = "demo-group")
    public void listen(ConsumerRecord<String, String> record) {
        System.out.println("Received: " + record.value());

        if (overloaded) {
            MessageListenerContainer container = registry.getListenerContainer("dynamicListener");
            if (container != null) {
                container.pause();
                System.out.println("Listener paused due to overload");
            }
        }
    }

    // Giả sử được gọi từ REST API hoặc scheduler
    public void setOverloaded(boolean overloaded) {
        this.overloaded = overloaded;
        if (!overloaded) {
            MessageListenerContainer container = registry.getListenerContainer("dynamicListener");
            if (container != null) {
                container.resume();
                System.out.println("Listener resumed");
            }
        }
    }
}
```

## 3. Kết hợp Seek + Pause/Resume

Bạn có thể:

- `pause()` → xử lý backlog hiện tại (ví dụ: ghi ra DB, reprocess)
- `seek()` đến offset mong muốn (ví dụ: reprocess từ offset X)
- `resume()` để tiếp tục luồng bình thường

Ví dụ pseudo-code:

```java
public void reprocessFromOffset(String listenerId, String topic, int partition, long offset) {
    MessageListenerContainer container = registry.getListenerContainer(listenerId);
    if (container != null) {
        container.pause();
        container.getContainerProperties().setConsumerRebalanceListener(
            new ConsumerSeekAware.ConsumerSeekCallback() {
                @Override
                public void seek(String t, int p, long o) {
                    // seek đến offset mong muốn
                }
            }
        );
        // ... cấu hình seek cụ thể
        container.resume();
    }
}
```

(Trong thực tế, nên dùng `ConsumerSeekAware` trong listener để quản lý seek rõ ràng hơn.)

---

## Tóm tắt ý chính

1. **Seek**:
   - Cho phép nhảy đến offset cụ thể, đọc lại hoặc bỏ qua message.
   - Dùng `ConsumerSeekAware` / `ConsumerSeekCallback`.

2. **Pause/Resume**:
   - `pause()` tạm dừng poll message mới, `resume()` tiếp tục.
   - Điều khiển qua `KafkaListenerEndpointRegistry` hoặc logic trong listener.
   - Hữu ích cho backpressure, xử lý lỗi tạm thời, bảo vệ hệ thống downstream.

3. **Kết hợp**:
   - Có thể pause, seek đến offset mong muốn, rồi resume để reprocess hoặc điều chỉnh luồng xử lý.

Nếu bạn cho mình biết version Spring Boot / Spring Kafka bạn dùng, mình có thể viết ví dụ code đầy đủ (pom.xml, config, listener) phù hợp với version đó.
