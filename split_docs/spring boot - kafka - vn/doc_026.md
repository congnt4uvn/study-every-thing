
Dưới đây là tóm tắt ngắn gọn về **Producer cơ bản trong Kafka** với **Spring Boot**, tập trung vào: gửi bất đồng bộ, callback, và future.

---

## 1. Gửi bất đồng bộ (Asynchronous send)

- Producer trong Kafka **mặc định gửi message bất đồng bộ**.
- Khi gọi `send()`, hàm **trả về ngay lập tức**, không chờ Kafka ghi xong.
- Điều này giúp **tăng throughput** và **giảm độ trễ**.

### Ví dụ với Spring Boot (`KafkaTemplate`)

```java
@Autowired
private KafkaTemplate<String, String> kafkaTemplate;

public void sendAsyncMessage() {
    String topic = "demo-topic";
    String key = "user-1";
    String value = "Xin chào Kafka";

    // Gửi bất đồng bộ
    ListenableFuture<SendResult<String, String>> future =
            kafkaTemplate.send(topic, key, value);
}
```

**Input**: topic = `demo-topic`, key = `user-1`, value = `"Xin chào Kafka"`  
**Output**: Hàm `sendAsyncMessage()` kết thúc ngay, message được gửi nền đến Kafka.

## 2. Callback (Xử lý kết quả gửi)

- Dùng callback để biết **gửi thành công hay thất bại**.
- Không chặn luồng chính, xử lý kết quả khi Kafka phản hồi.

### Ví dụ callback với `KafkaTemplate`

```java
public void sendWithCallback() {
    String topic = "demo-topic";
    String value = "Message có callback";

    ListenableFuture<SendResult<String, String>> future =
            kafkaTemplate.send(topic, value);

    future.addCallback(
        result -> {
            // Thành công
            RecordMetadata metadata = result.getRecordMetadata();
            System.out.println("Gửi thành công tới topic: " + metadata.topic()
                    + ", partition: " + metadata.partition()
                    + ", offset: " + metadata.offset());
        },
        ex -> {
            // Thất bại
            System.err.println("Gửi thất bại: " + ex.getMessage());
        }
    );
}
```

**Input**: value = `"Message có callback"`  
**Output (console)** (ví dụ):

- Thành công:
  ```text
  Gửi thành công tới topic: demo-topic, partition: 0, offset: 15
  ```
- Hoặc thất bại:
  ```text
  Gửi thất bại: TimeoutException: Failed to update metadata...
  ```

---

## 3. Future (ListenableFuture / CompletableFuture)

- `KafkaTemplate.send()` trả về `ListenableFuture<SendResult<...>>`.
- Có thể:
  - Dùng callback (như trên).
  - Hoặc **chặn** để lấy kết quả (không khuyến khích trong luồng xử lý chính).

### Ví dụ chặn để lấy kết quả (sync trên nền async)

```java
public void sendAndWait() throws ExecutionException, InterruptedException {
    String topic = "demo-topic";
    String value = "Message chờ kết quả";

    ListenableFuture<SendResult<String, String>> future =
            kafkaTemplate.send(topic, value);

    // Chặn cho đến khi có kết quả
    SendResult<String, String> result = future.get();

    RecordMetadata metadata = result.getRecordMetadata();
    System.out.println("Đã gửi tới topic: " + metadata.topic()
            + ", partition: " + metadata.partition()
            + ", offset: " + metadata.offset());
}
```

**Input**: value = `"Message chờ kết quả"`  
**Output**: Hàm chỉ kết thúc sau khi Kafka xác nhận, ví dụ:

```text
Đã gửi tới topic: demo-topic, partition: 1, offset: 20
```

---

## 4. Tóm tắt ý chính

- **Gửi bất đồng bộ**: `send()` không chờ, giúp hiệu năng cao.
- **Callback**: Dùng để xử lý **thành công/thất bại** mà không chặn luồng.
- **Future**:
  - Cho phép **chờ kết quả** (`get()`), hoặc
  - Gắn callback để xử lý kết quả khi có.

Nếu bạn gửi nội dung chi tiết hơn (config, use case cụ thể), tôi có thể viết thêm ví dụ đầy đủ với `application.yml` và consumer tương ứng.
