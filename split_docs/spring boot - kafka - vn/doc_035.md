
Dưới đây là tóm tắt ngắn gọn về độ tin cậy của Producer trong Kafka với Spring Boot, tập trung vào 3 lỗi thường gặp: `TimeoutException`, `RecordTooLargeException`, `NotEnoughReplicas`.

---

## 1. `TimeoutException`

**Ý nghĩa**  
Producer gửi message nhưng không nhận được ACK từ broker trong thời gian cấu hình (`delivery.timeout.ms`, `request.timeout.ms`).  
Thường do: broker chậm, network chậm, cluster quá tải.

**Cách xử lý trong Spring Boot**

```java
@Service
public class ReliableProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public ReliableProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String topic, String key, String value) {
        ListenableFuture<SendResult<String, String>> future =
                kafkaTemplate.send(topic, key, value);

        future.addCallback(
            result -> {
                // OUTPUT: gửi thành công
                System.out.println("Sent OK: " + result.getRecordMetadata());
            },
            ex -> {
                if (ex.getCause() instanceof org.apache.kafka.common.errors.TimeoutException) {
                    // Xử lý Timeout: log, retry, gửi sang DLQ, cảnh báo...
                    System.err.println("Timeout when sending message: " + ex.getMessage());
                } else {
                    System.err.println("Send failed: " + ex.getMessage());
                }
            }
        );
    }
}
```

**Cấu hình gợi ý (application.yml)**

```yaml
spring:
  kafka:
    producer:
      properties:
        acks: all
        delivery.timeout.ms: 120000   # 120s
        request.timeout.ms: 60000     # 60s
        linger.ms: 5
        retries: 5
        max.in.flight.requests.per.connection: 1
```

## 2. `RecordTooLargeException`

**Ý nghĩa**  
Message vượt quá kích thước tối đa cho phép (`max.request.size` của producer hoặc `message.max.bytes` / `max.message.bytes` của broker/topic).

**Cách xử lý**

1. Tăng giới hạn kích thước (nếu hợp lý).
2. Chia nhỏ message (chunking) hoặc nén (compression).
3. Đưa payload lớn lên storage khác (S3, DB…) và chỉ gửi reference.

**Ví dụ bắt lỗi trong callback**

```java
future.addCallback(
    result -> System.out.println("Sent OK"),
    ex -> {
        Throwable cause = ex.getCause();
        if (cause instanceof org.apache.kafka.common.errors.RecordTooLargeException) {
            // Xử lý message quá lớn
            System.err.println("Record too large, size exceeded: " + ex.getMessage());
            // Ví dụ: chia nhỏ, nén, hoặc lưu sang nơi khác rồi gửi link
        }
    }
);
```

**Cấu hình kích thước**

```yaml
spring:
  kafka:
    producer:
      properties:
        max.request.size: 1048576   # 1MB
        compression.type: snappy    # nén để giảm kích thước
```

**Ví dụ input gây lỗi**

- Input: JSON ~ 5MB, `max.request.size = 1MB`  
- Output: `RecordTooLargeException: The message is 5242880 bytes which is larger than the maximum configured size of 1048576.`

## 3. `NotEnoughReplicas` / `NotEnoughReplicasAfterAppend`

**Ý nghĩa**  
Không đủ replica “in-sync” (ISR) để đáp ứng yêu cầu `acks` của producer.  
Thường xảy ra khi: broker down, network partition, replica tụt lại.

**Cách xử lý**

- Đảm bảo cluster khỏe, đủ broker, monitoring ISR.
- Cân nhắc cấu hình `acks`:
  - `acks=all` (an toàn, nhưng dễ gặp lỗi khi thiếu replica).
  - `acks=1` (ít an toàn hơn, nhưng ít lỗi hơn).
- Retry với backoff, gửi sang DLQ nếu thất bại nhiều lần.

**Ví dụ bắt lỗi**

```java
future.addCallback(
    result -> System.out.println("Sent OK"),
    ex -> {
        Throwable cause = ex.getCause();
        if (cause instanceof org.apache.kafka.common.errors.NotEnoughReplicasException ||
            cause instanceof org.apache.kafka.common.errors.NotEnoughReplicasAfterAppendException) {

            System.err.println("Not enough replicas: " + ex.getMessage());
            // Chiến lược: retry với delay, cảnh báo, hoặc gửi sang DLQ
        }
    }
);
```

**Cấu hình liên quan**

```yaml
spring:
  kafka:
    producer:
      properties:
        acks: all
        retries: 10
        retry.backoff.ms: 200
```

## 4. Ví dụ tổng hợp: Producer “reliable” với Spring Boot

```java
@Service
public class ReliableKafkaProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public ReliableKafkaProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendReliable(String topic, String key, String payload) {
        kafkaTemplate.send(topic, key, payload)
            .addCallback(
                result -> {
                    // OUTPUT thành công
                    System.out.printf("SUCCESS topic=%s partition=%d offset=%d%n",
                            result.getRecordMetadata().topic(),
                            result.getRecordMetadata().partition(),
                            result.getRecordMetadata().offset());
                },
                ex -> handleFailure(topic, key, payload, ex)
            );
    }

    private void handleFailure(String topic, String key, String payload, Throwable ex) {
        Throwable cause = ex.getCause();

        if (cause instanceof org.apache.kafka.common.errors.TimeoutException) {
            System.err.println("Timeout when sending to " + topic);
            // retry / DLQ / alert
        } else if (cause instanceof org.apache.kafka.common.errors.RecordTooLargeException) {
            System.err.println("Record too large for topic " + topic);
            // split / compress / store elsewhere
        } else if (cause instanceof org.apache.kafka.common.errors.NotEnoughReplicasException ||
                   cause instanceof org.apache.kafka.common.errors.NotEnoughReplicasAfterAppendException) {
            System.err.println("Not enough replicas for topic " + topic);
            // retry with backoff / alert
        } else {
            System.err.println("Unknown send error: " + ex.getMessage());
        }
    }
}
```

**Input ví dụ**

```java
producer.sendReliable("orders", "order-123", "{ \"orderId\": 123, \"amount\": 1000 }");
```

**Output có thể**

- Thành công:
  - `SUCCESS topic=orders partition=0 offset=42`
- Thất bại do timeout:
  - `Timeout when sending to orders`
- Thất bại do record quá lớn:
  - `Record too large for topic orders`
- Thất bại do thiếu replica:
  - `Not enough replicas for topic orders`

---

**Tóm tắt ý chính**

- `TimeoutException`: broker/cluster chậm → tăng timeout, retry, monitoring.
- `RecordTooLargeException`: message quá lớn → tăng limit, nén, chia nhỏ, gửi reference.
- `NotEnoughReplicas`: thiếu replica ISR để đáp ứng `acks` → đảm bảo cluster khỏe, cấu hình `acks` hợp lý, retry.
