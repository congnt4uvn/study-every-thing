
Dưới đây là tóm tắt ngắn gọn về xử lý lỗi trong Spring Kafka Consumer, tập trung vào 3 thành phần chính: `ErrorHandler`, `SeekToCurrentErrorHandler`, `DeadLetterPublishingRecoverer`.

---

## 1. `ErrorHandler` là gì?

- Interface dùng để xử lý lỗi khi consumer đọc/ xử lý message.
- Được cấu hình trên `ConcurrentKafkaListenerContainerFactory`.
- Khi `@KafkaListener` ném exception, `ErrorHandler` sẽ được gọi.

### Ví dụ cấu hình cơ bản

```java
@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory(
        ConsumerFactory<String, String> consumerFactory,
        ErrorHandler errorHandler) {

    ConcurrentKafkaListenerContainerFactory<String, String> factory =
            new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory);
    factory.setErrorHandler(errorHandler);
    return factory;
}

@Bean
public ErrorHandler errorHandler() {
    return (thrownException, data) -> {
        System.out.println("Lỗi khi xử lý record: " + data);
        thrownException.printStackTrace();
        // Có thể log, gửi alert, v.v.
    };
}
```

**Input (message)**: `"hello"`  
**Output (log)**:
```text
Lỗi khi xử lý record: ConsumerRecord(topic = demo, value = hello, ...)
java.lang.RuntimeException: ...
```

## 2. `SeekToCurrentErrorHandler`

- Là implementation phổ biến của `ErrorHandler`.
- Khi lỗi xảy ra, nó sẽ:
  - “seek” lại offset về record lỗi (hoặc record tiếp theo tùy cấu hình),
  - cho phép retry trong cùng partition.
- Có thể cấu hình số lần retry và hành vi sau khi retry thất bại.

### Ví dụ: retry 3 lần rồi bỏ qua

```java
@Bean
public SeekToCurrentErrorHandler seekToCurrentErrorHandler() {
    // retry tối đa 3 lần, sau đó bỏ qua (skip) record lỗi
    SeekToCurrentErrorHandler errorHandler =
            new SeekToCurrentErrorHandler((record, exception) -> {
                System.out.println("Bỏ qua record sau khi retry thất bại: " + record);
            }, 3);

    return errorHandler;
}

@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory(
        ConsumerFactory<String, String> consumerFactory,
        SeekToCurrentErrorHandler errorHandler) {

    ConcurrentKafkaListenerContainerFactory<String, String> factory =
            new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory);
    factory.setErrorHandler(errorHandler);
    return factory;
}
```

**Input**: message `"bad-data"` gây `RuntimeException`  
**Output (log)**:
```text
Retry 1 cho record: bad-data
Retry 2 cho record: bad-data
Retry 3 cho record: bad-data
Bỏ qua record sau khi retry thất bại: ConsumerRecord(topic = demo, value = bad-data, ...)
```

## 3. `DeadLetterPublishingRecoverer` (DLP)

- Dùng cùng với `SeekToCurrentErrorHandler` để gửi message lỗi sang một “dead-letter topic” (DLT).
- Thay vì chỉ bỏ qua, ta lưu message lỗi lại để phân tích sau.

### Cấu hình Dead Letter Topic

Giả sử:
- Topic chính: `input-topic`
- Dead-letter topic: `input-topic.DLT`

```java
@Bean
public DeadLetterPublishingRecoverer deadLetterPublishingRecoverer(KafkaTemplate<Object, Object> template) {
    return new DeadLetterPublishingRecoverer(template, (record, ex) -> {
        // Chọn topic DLT, có thể dựa trên record hoặc exception
        return new TopicPartition(record.topic() + ".DLT", record.partition());
    });
}

@Bean
public SeekToCurrentErrorHandler seekToCurrentErrorHandler(DeadLetterPublishingRecoverer recoverer) {
    // retry 3 lần, sau đó publish sang DLT
    SeekToCurrentErrorHandler errorHandler =
            new SeekToCurrentErrorHandler(recoverer, 3);
    return errorHandler;
}

@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory(
        ConsumerFactory<String, String> consumerFactory,
        SeekToCurrentErrorHandler errorHandler) {

    ConcurrentKafkaListenerContainerFactory<String, String> factory =
            new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory);
    factory.setErrorHandler(errorHandler);
    return factory;
}
```

### Consumer chính

```java
@KafkaListener(topics = "input-topic", groupId = "demo-group")
public void listen(String message) {
    if (message.contains("error")) {
        throw new RuntimeException("Lỗi xử lý message: " + message);
    }
    System.out.println("Xử lý OK: " + message);
}
```

### Consumer cho Dead Letter Topic (tuỳ chọn)

```java
@KafkaListener(topics = "input-topic.DLT", groupId = "dlt-group")
public void listenDlt(String message) {
    System.out.println("Nhận message lỗi từ DLT: " + message);
}
```

**Input**:
- Gửi `"ok-1"` → xử lý bình thường.
- Gửi `"error-1"` → ném exception.

**Output (log)**:
```text
Xử lý OK: ok-1
Retry 1 cho record: error-1
Retry 2 cho record: error-1
Retry 3 cho record: error-1
// Sau đó DeadLetterPublishingRecoverer publish sang input-topic.DLT
Nhận message lỗi từ DLT: error-1
```

---

## Tóm tắt ý chính

1. **`ErrorHandler`**: interface chung để xử lý lỗi cho Kafka consumer.
2. **`SeekToCurrentErrorHandler`**:
   - Cho phép retry bằng cách seek lại offset.
   - Có thể cấu hình số lần retry và hành vi khi hết retry.
3. **`DeadLetterPublishingRecoverer`**:
   - Dùng với `SeekToCurrentErrorHandler` để gửi message lỗi sang Dead Letter Topic.
   - Giúp không mất dữ liệu lỗi, dễ debug và xử lý sau.
