
Dưới đây là tóm tắt ngắn gọn về **Replication trong Kafka** với ví dụ code (Spring Boot) và minh họa input/output.

---

## 1. Mô hình Leader / Follower

- Mỗi **partition** có:
  - 1 **leader replica**: nhận và xử lý mọi read/write.
  - 0..n **follower replica**: chỉ **replicate** dữ liệu từ leader.
- Producer và consumer **chỉ làm việc với leader**, không ghi trực tiếp vào follower.

---

## 2. ISR (In-Sync Replicas)

- **ISR** = tập các replica (leader + follower) đang:
  - Bắt kịp leader (không bị tụt quá xa).
  - Đang hoạt động (alive).
- Khi follower bị chậm hoặc mất kết nối, nó sẽ **bị loại khỏi ISR**.
- Khi leader chết, Kafka sẽ **chọn leader mới từ ISR** để đảm bảo dữ liệu không bị mất.

---

## 3. `min.insync.replicas`

- Cấu hình ở **broker** hoặc **topic**:
  ```properties
  min.insync.replicas=2
  ```
- Ý nghĩa: để một ghi (write) được coi là **thành công**, phải có **ít nhất N replica trong ISR** (bao gồm leader) xác nhận.
- Thường dùng kết hợp với:
  ```properties
  acks=all
  ```
- Nếu số replica trong ISR < `min.insync.replicas`:
  - Producer gửi với `acks=all` sẽ nhận **lỗi** (RecordTooLargeException / NotEnoughReplicasException tùy trường hợp).
  - Giúp **tránh mất dữ liệu** khi cluster đang thiếu replica.

---

## 4. Ví dụ cấu hình topic (CLI)

Tạo topic với replication factor = 3, yêu cầu ít nhất 2 replica in-sync:

```bash
kafka-topics.sh \
  --create \
  --bootstrap-server localhost:9092 \
  --topic demo-topic \
  --partitions 3 \
  --replication-factor 3 \
  --config min.insync.replicas=2
```

## 5. Ví dụ Producer với Spring Boot (Java)

### 5.1. Cấu hình producer (application.yml)

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      properties:
        acks: all          # yêu cầu tất cả replica trong ISR xác nhận
        retries: 3
```

### 5.2. Bean KafkaTemplate

```java
@Configuration
public class KafkaProducerConfig {

    @Bean
    public ProducerFactory<String, String> producerFactory(
            KafkaProperties properties) {
        Map<String, Object> configs = properties.buildProducerProperties();
        return new DefaultKafkaProducerFactory<>(configs);
    }

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate(
            ProducerFactory<String, String> producerFactory) {
        return new KafkaTemplate<>(producerFactory);
    }
}
```

### 5.3. Gửi message và xem kết quả (input/output)

```java
@Service
public class DemoProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public DemoProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String key, String value) {
        ListenableFuture<SendResult<String, String>> future =
                kafkaTemplate.send("demo-topic", key, value);

        future.addCallback(
            result -> {
                RecordMetadata meta = result.getRecordMetadata();
                System.out.printf(
                    "SUCCESS: topic=%s partition=%d offset=%d%n",
                    meta.topic(), meta.partition(), meta.offset()
                );
            },
            ex -> {
                System.out.println("FAILED: " + ex.getMessage());
            }
        );
    }
}
```

#### Input (gọi service):

```java
demoProducer.sendMessage("user-1", "Hello Kafka");
```

#### Output (trường hợp đủ ISR, `min.insync.replicas` thỏa mãn):

```text
SUCCESS: topic=demo-topic partition=1 offset=15
```

#### Output (trường hợp thiếu ISR, không đủ replica in-sync):

```text
FAILED: Not enough replicas to meet min.insync.replicas
```

---

## 6. Tóm tắt ý chính

- **Leader/Follower**: leader xử lý read/write, follower chỉ replicate.
- **ISR**: tập replica đang bắt kịp leader; dùng để chọn leader mới an toàn.
- **`min.insync.replicas` + `acks=all`**: đảm bảo ghi chỉ thành công khi có đủ replica in-sync, giúp tăng **độ an toàn dữ liệu** (durability) trong Kafka.
