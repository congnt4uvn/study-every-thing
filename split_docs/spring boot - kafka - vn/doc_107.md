
Dưới đây là tóm tắt ngắn gọn về “Single writer principle per key (for ordering)” trong Kafka + Spring Boot, kèm ví dụ code và input/output.

---

## 1. Ý tưởng chính

**Single writer principle per key**:  
Để **giữ đúng thứ tự (ordering)** của các message theo từng “key” (ví dụ: userId, orderId), ta đảm bảo rằng:

- Mỗi **key** chỉ được **ghi (produce)** bởi **một writer (một luồng / một service / một instance)** tại một thời điểm.
- Nhờ đó, các message cùng key sẽ:
  - Đi vào **cùng một partition** (do cơ chế partitioning theo key của Kafka).
  - Được xử lý theo **thứ tự gửi** (Kafka đảm bảo ordering trong cùng một partition).

Nếu nhiều writer cùng ghi message cho **cùng một key**, có thể dẫn đến:
- Race condition
- Thứ tự message bị đảo (không còn đúng logic nghiệp vụ).

---

## 2. Ứng dụng trong Kafka + Spring Boot

### 2.1. Producer: đảm bảo single writer per key

Ví dụ: ta có service chỉ chuyên ghi event cho từng `orderId`.  
Mỗi instance chỉ xử lý một tập `orderId` riêng (sharding theo key).

```java
@Service
public class OrderEventProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private static final String TOPIC = "order-events";

    public OrderEventProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    // Chỉ service này được phép gửi event cho orderId tương ứng
    public void sendOrderEvent(String orderId, String eventType) {
        String value = """
            {
              "orderId": "%s",
              "eventType": "%s",
              "timestamp": %d
            }
            """.formatted(orderId, eventType, System.currentTimeMillis());

        // key = orderId => tất cả event của cùng orderId vào cùng partition
        kafkaTemplate.send(TOPIC, orderId, value);
    }
}
```

**Input (gọi API / service):**

```java
orderEventProducer.sendOrderEvent("ORDER-123", "CREATED");
orderEventProducer.sendOrderEvent("ORDER-123", "PAID");
orderEventProducer.sendOrderEvent("ORDER-123", "SHIPPED");
```

**Output (trên Kafka topic `order-events`, cùng partition):**

1. Message 1: `{"orderId":"ORDER-123","eventType":"CREATED",...}`
2. Message 2: `{"orderId":"ORDER-123","eventType":"PAID",...}`
3. Message 3: `{"orderId":"ORDER-123","eventType":"SHIPPED",...}`

Thứ tự trên partition được giữ nguyên như thứ tự gửi, miễn là **chỉ một writer** gửi event cho `ORDER-123`.

---

### 2.2. Consumer: xử lý theo thứ tự

Consumer đọc từ partition sẽ nhận message theo đúng thứ tự:

```java
@Service
public class OrderEventConsumer {

    @KafkaListener(
        topics = "order-events",
        groupId = "order-service",
        containerFactory = "kafkaListenerContainerFactory"
    )
    public void consume(ConsumerRecord<String, String> record) {
        String key = record.key();      // orderId
        String value = record.value();  // JSON event

        // Xử lý tuần tự theo từng partition
        System.out.printf("Received event for key=%s, value=%s%n", key, value);
    }
}
```

**Output (log):**

```text
Received event for key=ORDER-123, value={"orderId":"ORDER-123","eventType":"CREATED",...}
Received event for key=ORDER-123, value={"orderId":"ORDER-123","eventType":"PAID",...}
Received event for key=ORDER-123, value={"orderId":"ORDER-123","eventType":"SHIPPED",...}
```

---

## 3. Tóm tắt ngắn

- **Mục tiêu**: Giữ **thứ tự message** theo từng key (userId, orderId, …).
- **Cách làm**:
  - Dùng **key** khi gửi message (KafkaTemplate.send(topic, key, value)).
  - Đảm bảo **một writer duy nhất** cho mỗi key (single writer per key).
- **Kết quả**:
  - Tất cả message cùng key vào **cùng partition**.
  - Kafka đảm bảo **ordering trong partition**, nên logic nghiệp vụ theo từng key không bị sai thứ tự.

Nếu bạn cho thêm nội dung chi tiết hơn (nhiều đoạn text hơn), mình có thể tóm tắt sâu hơn và đưa thêm pattern triển khai (sharding writer, phân vùng theo key, v.v.).
