
Dưới đây là tóm tắt ngắn gọn về tối ưu hiệu năng Producer Kafka trong Spring Boot, tập trung vào 3 tham số: `batch.size`, `linger.ms`, `compression.type`.

---

## 1. batch.size

- **Ý nghĩa**: Kích thước tối đa (bytes) của một batch message gửi tới cùng một partition.
- **Tác dụng**:
  - Tăng `batch.size` → ít request hơn, throughput cao hơn, nhưng có thể tăng độ trễ và dùng nhiều RAM hơn.
  - Quá nhỏ → không tận dụng được batching, hiệu năng thấp.
- **Gợi ý**:
  - Bắt đầu với 32KB–64KB, sau đó đo đạc và điều chỉnh.

**Cấu hình Spring Boot (application.yml)**

```yaml
spring:
  kafka:
    producer:
      bootstrap-servers: localhost:9092
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      properties:
        batch.size: 65536   # 64KB
```

---

## 2. linger.ms

- **Ý nghĩa**: Thời gian tối đa (ms) producer chờ để “gom” thêm message vào batch trước khi gửi.
- **Tác dụng**:
  - `linger.ms` lớn hơn 0 → cho phép gom nhiều message hơn → tăng throughput.
  - Nhưng cũng tăng độ trễ (latency) tối đa cho mỗi message.
- **Gợi ý**:
  - Hệ thống cần latency thấp: 0–5ms.
  - Hệ thống thiên về throughput: 10–50ms (tùy tải).

**Cấu hình Spring Boot**

```yaml
spring:
  kafka:
    producer:
      properties:
        linger.ms: 20   # chờ tối đa 20ms để gom batch
```

---

## 3. compression.type

- **Ý nghĩa**: Kiểu nén dữ liệu trước khi gửi lên broker.
- **Các giá trị phổ biến**:
  - `none` (mặc định): không nén.
  - `gzip`: nén tốt, CPU tốn hơn, latency cao hơn.
  - `snappy`: cân bằng giữa tốc độ và tỷ lệ nén.
  - `lz4`, `zstd`: thường rất tốt cho throughput, latency thấp (tùy version Kafka).
- **Tác dụng**:
  - Giảm băng thông mạng, tăng throughput, đặc biệt khi message lớn hoặc nhiều.
  - Đổi lại tốn CPU để nén/giải nén.

**Cấu hình Spring Boot**

```yaml
spring:
  kafka:
    producer:
      properties:
        compression.type: lz4
```

## 4. Ví dụ đầy đủ cấu hình Producer trong Spring Boot

**application.yml**

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      properties:
        batch.size: 65536        # 64KB
        linger.ms: 20            # chờ tối đa 20ms để gom batch
        compression.type: lz4    # nén lz4
        acks: all                # đảm bảo độ bền dữ liệu
```

**Bean KafkaTemplate**

```java
@Configuration
public class KafkaProducerConfig {

    @Bean
    public ProducerFactory<String, String> producerFactory(
            KafkaProperties kafkaProperties) {
        Map<String, Object> props = new HashMap<>(kafkaProperties.buildProducerProperties());
        return new DefaultKafkaProducerFactory<>(props);
    }

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate(
            ProducerFactory<String, String> producerFactory) {
        return new KafkaTemplate<>(producerFactory);
    }
}
```

**Gửi message**

```java
@Service
public class MyProducerService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public MyProducerService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String topic, String key, String value) {
        kafkaTemplate.send(topic, key, value)
                     .whenComplete((result, ex) -> {
                         if (ex != null) {
                             System.out.println("Send failed: " + ex.getMessage());
                         } else {
                             System.out.println("Sent to partition " +
                                 result.getRecordMetadata().partition() +
                                 " offset " + result.getRecordMetadata().offset());
                         }
                     });
    }
}
```

**Ví dụ Input/Output**

- Input (gọi service):

```java
myProducerService.sendMessage("orders", "order-123", "{ \"id\": 123, \"amount\": 1000 }");
```

- Output (log):

```text
Sent to partition 1 offset 42
```

---

## 5. Tóm tắt ý chính

- **batch.size**: tăng để gom nhiều message hơn trong 1 request → tăng throughput, nhưng tốn RAM hơn.
- **linger.ms**: cho phép chờ thêm thời gian để gom batch → tăng throughput, nhưng tăng latency tối đa.
- **compression.type**: bật nén (`snappy`, `lz4`, `zstd`, …) để giảm băng thông, tăng throughput, đổi lại tốn CPU.

Khi tối ưu, luôn:
1. Đo đạc (metrics: throughput, latency, CPU, network).
2. Thay đổi từng tham số một.
3. Kiểm tra lại dưới tải thực tế hoặc gần với thực tế.
