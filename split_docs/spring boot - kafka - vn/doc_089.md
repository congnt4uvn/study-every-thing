## 1. Message Headers trong Spring Kafka

**Ý chính:**
- Headers dùng để truyền metadata (loại sự kiện, version, correlation ID, trace ID, v.v.).
- Giúp tách dữ liệu nghiệp vụ (payload) khỏi thông tin kỹ thuật.
- Spring Kafka hỗ trợ đọc/ghi headers rất dễ.

### 1.1. Gửi message với headers

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.kafka.support.KafkaHeaders;
import org.springframework.messaging.support.MessageBuilder;
import org.springframework.messaging.Message;

@Service
public class OrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public OrderProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendOrder(String orderJson, String correlationId) {
        Message<String> message = MessageBuilder
                .withPayload(orderJson)
                .setHeader(KafkaHeaders.TOPIC, "orders")
                .setHeader("correlation-id", correlationId)
                .setHeader("event-type", "ORDER_CREATED")
                .build();

        kafkaTemplate.send(message);
    }
}
```

**Input (orderJson):**
```json
{"orderId": 123, "amount": 100000}
```

**Headers gửi kèm:**
- `correlation-id`: ví dụ `"abc-123"`
- `event-type`: `"ORDER_CREATED"`

---

### 1.2. Nhận message và đọc headers

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.messaging.handler.annotation.Header;

@Service
public class OrderConsumer {

    @KafkaListener(topics = "orders", groupId = "order-service")
    public void listen(
            String payload,
            @Header(name = "correlation-id", required = false) String correlationId,
            @Header(name = "event-type", required = false) String eventType
    ) {
        System.out.println("Payload: " + payload);
        System.out.println("CorrelationId: " + correlationId);
        System.out.println("EventType: " + eventType);
    }
}
```

**Output (log):**
```text
Payload: {"orderId":123,"amount":100000}
CorrelationId: abc-123
EventType: ORDER_CREATED
```

## 2. Correlation ID

**Ý chính:**
- Correlation ID dùng để liên kết các request/response hoặc các bước trong một flow phân tán.
- Thường được đặt trong header Kafka (`correlation-id`).
- Có thể:
  - Tạo mới ở service đầu tiên.
  - Truyền tiếp qua các service downstream.
  - Log correlation ID để dễ trace.

### 2.1. Tạo và gắn correlation ID khi gửi

```java
import java.util.UUID;

public void sendOrder(String orderJson) {
    String correlationId = UUID.randomUUID().toString();

    Message<String> message = MessageBuilder
            .withPayload(orderJson)
            .setHeader(KafkaHeaders.TOPIC, "orders")
            .setHeader("correlation-id", correlationId)
            .build();

    kafkaTemplate.send(message);
}
```

---

### 2.2. Truyền correlation ID sang message khác (chaining)

```java
@Service
public class PaymentService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public PaymentService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    @KafkaListener(topics = "orders", groupId = "payment-service")
    public void handleOrder(
            String orderJson,
            @Header(name = "correlation-id", required = false) String correlationId
    ) {
        // Xử lý order -> tạo payment
        String paymentJson = "{\"status\":\"PAID\"}";

        // Nếu không có correlation-id thì tạo mới
        if (correlationId == null) {
            correlationId = UUID.randomUUID().toString();
        }

        Message<String> paymentMessage = MessageBuilder
                .withPayload(paymentJson)
                .setHeader(KafkaHeaders.TOPIC, "payments")
                .setHeader("correlation-id", correlationId)
                .build();

        kafkaTemplate.send(paymentMessage);
    }
}
```

**Kết quả:** cùng một `correlation-id` đi qua topic `orders` → `payments`, giúp trace toàn bộ flow.

## 3. Tracing với Sleuth (Spring Cloud Sleuth)

**Ý chính:**
- Sleuth tự động:
  - Tạo `traceId`, `spanId`.
  - Gắn vào logs.
  - Truyền qua HTTP, Kafka, v.v.
- Khi dùng với Kafka, Sleuth sẽ thêm các headers như:
  - `X-B3-TraceId`
  - `X-B3-SpanId`
  - `X-B3-ParentSpanId`
- Giúp bạn xem toàn bộ call chain trong Zipkin/Jaeger (tùy cấu hình).

### 3.1. Thêm dependency (Spring Boot 2.x)

```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-sleuth</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-sleuth-zipkin</artifactId>
</dependency>
```

### 3.2. Gửi message với Sleuth (tự động)

Khi Sleuth đã bật, bạn chỉ cần dùng `KafkaTemplate` như bình thường:

```java
@Service
public class ShippingProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public ShippingProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendShippingEvent(String shippingJson) {
        kafkaTemplate.send("shipping", shippingJson);
    }
}
```

Sleuth sẽ tự:
- Lấy `traceId` hiện tại (nếu có).
- Gắn các B3 headers vào message Kafka.

**Headers ví dụ:**
- `X-B3-TraceId: 4e1f2c3d4b5a6c7d`
- `X-B3-SpanId: 8a9b0c1d2e3f4a5b`

## 4. Tracing với OpenTelemetry

**Ý chính:**
- OpenTelemetry là chuẩn mới cho metrics, logs, traces.
- Thay thế dần Sleuth trong Spring Boot 3.x.
- Dùng W3C Trace Context (`traceparent`, `tracestate`) thay vì B3 headers.

### 4.1. Thêm dependency (ví dụ Spring Boot 3.x)

```xml
<dependency>
    <groupId>io.opentelemetry.instrumentation</groupId>
    <artifactId>opentelemetry-spring-boot-starter</artifactId>
    <version>2.5.0</version> <!-- ví dụ -->
</dependency>
```

Hoặc dùng Spring Observability (Spring Boot 3) + OTel exporter.

### 4.2. Tự gắn trace context vào Kafka headers (nếu cần custom)

```java
import io.opentelemetry.api.trace.Span;
import io.opentelemetry.context.Context;
import io.opentelemetry.context.propagation.TextMapSetter;
import org.springframework.messaging.Message;
import org.springframework.messaging.support.MessageBuilder;

public class KafkaHeaderSetter implements TextMapSetter<MessageBuilder<?>> {
    @Override
    public void set(MessageBuilder<?> carrier, String key, String value) {
        carrier.setHeader(key, value);
    }
}

public Message<String> buildTracedMessage(String payload) {
    Span currentSpan = Span.current();
    MessageBuilder<String> builder = MessageBuilder.withPayload(payload);

    KafkaHeaderSetter setter = new KafkaHeaderSetter();
    // Giả sử bạn có propagator:
    // propagator.inject(Context.current(), builder, setter);

    return builder.build();
}
```

Trong thực tế, nếu dùng auto-instrumentation của OpenTelemetry cho Spring/Kafka, phần inject/extract headers sẽ được làm tự động.

## 5. Kết hợp: Headers + Correlation ID + Tracing

**Mô hình chuẩn:**
- `correlation-id`: do bạn quản lý (business correlation).
- `traceId`/`spanId`: do hệ thống tracing (Sleuth/OTel) quản lý.
- Cả hai đều nằm trong headers Kafka.

**Ví dụ flow:**
1. API Gateway nhận HTTP request → tạo `correlation-id` + bắt đầu trace.
2. Gateway gửi message Kafka:
   - Header: `correlation-id`, `traceparent` (OTel) hoặc `X-B3-TraceId` (Sleuth).
3. Service A consume message → log có `correlation-id` + traceId.
4. Service A gửi tiếp message sang topic khác, giữ nguyên `correlation-id` và trace context.
5. Bạn có thể:
   - Tìm log theo `correlation-id` để xem toàn bộ flow nghiệp vụ.
   - Dùng Zipkin/Jaeger/Tempo để xem trace theo `traceId`.
