
Dưới đây là tóm tắt ngắn gọn về chủ đề: **Tuning GC, JVM cho Kafka broker và Kafka client**, kèm ví dụ cấu hình.

---

## 1. Mục tiêu khi tuning GC/JVM cho Kafka

- Giảm **GC pause** (thời gian dừng do GC) để tránh tăng **latency** và **timeout**.
- Đảm bảo **throughput ổn định** cho cả broker và client.
- Sử dụng **GC phù hợp** (G1GC, ZGC, Shenandoah…) và cấu hình heap hợp lý.

---

## 2. GC & JVM tuning cho Kafka Broker

### 2.1. Lựa chọn GC

- Kafka (Java 11+) thường dùng:
  - **G1GC**: cân bằng giữa throughput và pause time, phù hợp đa số hệ thống.
  - **ZGC / Shenandoah**: pause rất thấp, phù hợp hệ thống latency rất nhạy cảm (nếu JVM hỗ trợ).

Ví dụ cấu hình G1GC cho broker (trong `KAFKA_HEAP_OPTS` hoặc `KAFKA_JVM_PERFORMANCE_OPTS`):

```bash
export KAFKA_HEAP_OPTS="-Xms8g -Xmx8g"
export KAFKA_JVM_PERFORMANCE_OPTS="
  -server
  -XX:+UseG1GC
  -XX:MaxGCPauseMillis=50
  -XX:InitiatingHeapOccupancyPercent=35
  -XX:+DisableExplicitGC
  -XX:+ParallelRefProcEnabled
  -XX:+ExplicitGCInvokesConcurrent
"
```

**Ý nghĩa chính:**
- `-Xms` / `-Xmx`: kích thước heap cố định (tránh resize).
- `MaxGCPauseMillis`: mục tiêu pause time (không phải cam kết tuyệt đối).
- `InitiatingHeapOccupancyPercent`: ngưỡng bắt đầu GC concurrent.

### 2.2. Kích thước heap

- Không nên dùng toàn bộ RAM cho heap; cần chừa cho:
  - Page cache (file system cache) để Kafka đọc/ghi log nhanh.
  - OS và các process khác.
- Thường: **heap 4–16GB** cho broker, tùy tải.

Ví dụ: máy 32GB RAM
- Heap: 8–12GB
- Phần còn lại cho OS + page cache.

### 2.3. Theo dõi GC

- Bật log GC để phân tích:

```bash
export KAFKA_JVM_PERFORMANCE_OPTS="$KAFKA_JVM_PERFORMANCE_OPTS
  -Xlog:gc*:file=/var/log/kafka/gc.log:time,uptime,level,tags:filecount=5,filesize=20M
"
```

**Đầu ra (ví dụ rút gọn):**

```text
[2.345s][info][gc] GC(5) Pause Young (Normal) 256M->128M(8192M) 15.6ms
[5.678s][info][gc] GC(6) Pause Young (Normal) 260M->130M(8192M) 18.2ms
```

Nếu thấy:
- Pause > 100–200ms thường xuyên → cần giảm heap, tối ưu GC, hoặc giảm load.

## 3. GC & JVM tuning cho Kafka Client (Spring Boot)

### 3.1. Đặc điểm client

- Client thường:
  - Heap nhỏ hơn broker.
  - Tập trung vào **low latency** và **ổn định**.
- G1GC vẫn là lựa chọn tốt; với Java 17+ có thể cân nhắc ZGC nếu latency cực thấp.

### 3.2. Cấu hình JVM cho Spring Boot app (client)

Ví dụ chạy Spring Boot với G1GC:

```bash
java \
  -Xms512m -Xmx512m \
  -XX:+UseG1GC \
  -XX:MaxGCPauseMillis=50 \
  -XX:InitiatingHeapOccupancyPercent=30 \
  -jar app.jar
```

## 4. Ví dụ Spring Boot Kafka Consumer/Producer

### 4.1. application.yml (client)

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    consumer:
      group-id: demo-group
      auto-offset-reset: earliest
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      properties:
        max.poll.interval.ms: 300000
        max.poll.records: 500
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      properties:
        linger.ms: 5
        batch.size: 32768
```

### 4.2. Consumer đơn giản

```java
@Service
public class DemoConsumer {

    @KafkaListener(topics = "demo-topic", groupId = "demo-group")
    public void listen(String message) {
        System.out.println("Received: " + message);
    }
}
```

**Input (message gửi vào topic `demo-topic`):**

```text
"Hello Kafka"
```

**Output (log consumer):**

```text
Received: Hello Kafka
```

### 4.3. Producer đơn giản

```java
@RestController
@RequestMapping("/api/messages")
public class DemoProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public DemoProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    @PostMapping
    public String send(@RequestBody String message) {
        kafkaTemplate.send("demo-topic", message);
        return "Sent: " + message;
    }
}
```

**Input (HTTP request):**

```http
POST /api/messages
Content-Type: text/plain

Hello Kafka
```

**Output (HTTP response):**

```text
Sent: Hello Kafka
```

## 5. Ý chính cần nhớ

1. **Broker**:
   - Dùng G1GC (hoặc ZGC/Shenandoah nếu cần latency rất thấp).
   - Đặt heap cố định, không quá lớn, chừa RAM cho page cache.
   - Bật log GC, theo dõi pause time.

2. **Client (Spring Boot)**:
   - Heap nhỏ, G1GC là đủ cho đa số trường hợp.
   - Tối ưu thêm bằng cách điều chỉnh `batch.size`, `linger.ms`, `max.poll.records`…

3. **Mục tiêu**: giảm GC pause, giữ latency ổn định, tránh full GC kéo dài gây timeout và mất ổn định hệ thống.
