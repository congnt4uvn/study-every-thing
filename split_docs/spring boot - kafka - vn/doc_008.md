
Dưới đây là tóm tắt ngắn gọn về **data flow / message format trong Kafka** khi dùng với Spring Boot, kèm ví dụ code và input/output.

---

## 1. Cấu trúc một Kafka message

Một message trong Kafka thường gồm:

1. **key**:  
   - Dùng để xác định partition (hash theo key).  
   - Có thể là `String`, `Long`, JSON, v.v.  
   - Có thể `null`.

2. **value**:  
   - Nội dung chính của message.  
   - Thường là JSON, String, Avro, v.v.

3. **headers**:  
   - Metadata bổ sung (ví dụ: trace-id, correlation-id, type…).  
   - Không ảnh hưởng đến partitioning mặc định.

4. **timestamp**:  
   - Thời gian message được tạo hoặc ghi vào Kafka.  
   - Dùng cho việc order theo thời gian, TTL, monitoring.

5. **offset**:  
   - Vị trí tuần tự của message trong một partition.  
   - Được Kafka gán khi ghi message.  
   - Dùng để commit, replay, đảm bảo at-least-once, v.v.

## 2. Ví dụ Producer (Spring Boot)

### Cấu hình Producer (Java)

```java
@Configuration
public class KafkaProducerConfig {

    @Bean
    public ProducerFactory<String, String> producerFactory() {
        Map<String, Object> config = new HashMap<>();
        config.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        config.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        config.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        return new DefaultKafkaProducerFactory<>(config);
    }

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate() {
        return new KafkaTemplate<>(producerFactory());
    }
}
```

### Gửi message với key, value, headers

```java
@Service
public class MyProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public MyProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage() {
        String topic = "my-topic";
        String key = "user-123";
        String value = "{\"event\":\"USER_CREATED\",\"userId\":123}";

        Message<String> message = MessageBuilder
                .withPayload(value)
                .setHeader(KafkaHeaders.TOPIC, topic)
                .setHeader(KafkaHeaders.MESSAGE_KEY, key)
                .setHeader("trace-id", "abc-xyz-123")
                .build();

        kafkaTemplate.send(message);
    }
}
```

### Input (ứng dụng gửi)

- `topic`: `"my-topic"`
- `key`: `"user-123"`
- `value`: `{"event":"USER_CREATED","userId":123}`
- `headers`:
  - `trace-id`: `"abc-xyz-123"`

### Output (trong Kafka)

Một record trong partition, ví dụ:

- `key`: `"user-123"`
- `value`: `{"event":"USER_CREATED","userId":123}`
- `headers`: `trace-id=abc-xyz-123`
- `timestamp`: `2026-05-20T10:15:30.000Z` (ví dụ)
- `offset`: `42` (ví dụ)

## 3. Ví dụ Consumer (Spring Boot)

### Lắng nghe message và đọc key, value, headers, offset

```java
@Service
public class MyConsumer {

    @KafkaListener(topics = "my-topic", groupId = "my-group")
    public void listen(ConsumerRecord<String, String> record) {
        String key = record.key();
        String value = record.value();
        Headers headers = record.headers();
        long timestamp = record.timestamp();
        long offset = record.offset();
        int partition = record.partition();

        System.out.println("Key: " + key);
        System.out.println("Value: " + value);
        System.out.println("Offset: " + offset);
        System.out.println("Partition: " + partition);
        System.out.println("Timestamp: " + timestamp);

        headers.forEach(h ->
                System.out.println("Header: " + h.key() + " = " + new String(h.value()))
        );
    }
}
```

### Output (log consumer)

Ví dụ log khi nhận message:

```text
Key: user-123
Value: {"event":"USER_CREATED","userId":123}
Offset: 42
Partition: 0
Timestamp: 1716209730000
Header: trace-id = abc-xyz-123
```

---

## 4. Tóm tắt ý chính

- **Kafka message** gồm: `key`, `value`, `headers`, `timestamp`, `offset`.
- **key**: quyết định partition, hỗ trợ grouping theo entity (user, order…).
- **value**: dữ liệu chính (payload).
- **headers**: metadata thêm (trace, type, correlation…).
- **timestamp**: thời điểm message, dùng cho order/time-based logic.
- **offset**: vị trí trong partition, dùng để commit và replay.
