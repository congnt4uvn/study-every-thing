
Dưới đây là bản tóm tắt ngắn gọn về **Schema Registry** trong bối cảnh Kafka + Spring Boot, tập trung vào **mục đích** và **lợi ích (compatibility, evolution)**, kèm ví dụ code.

---

## 1. Schema Registry là gì?

**Schema Registry** là một dịch vụ trung tâm dùng để:
- Lưu trữ và quản lý **schema** (thường là Avro/JSON/Protobuf) cho các message Kafka.
- Gán **ID schema** cho từng phiên bản schema.
- Đảm bảo **tính tương thích (compatibility)** giữa các phiên bản schema khi producer/consumer thay đổi.

---

## 2. Mục đích chính

1. **Chuẩn hóa cấu trúc dữ liệu**  
   - Tất cả producer/consumer cùng một topic dùng chung schema đã đăng ký.
   - Giảm lỗi do không thống nhất format dữ liệu.

2. **Tách biệt schema khỏi code**  
   - Schema được quản lý tập trung, không phụ thuộc vào từng service.
   - Dễ kiểm soát thay đổi schema trên toàn hệ thống.

3. **Hỗ trợ kiểm soát phiên bản (versioning)**  
   - Mỗi lần thay đổi schema sẽ tạo ra một **version** mới.
   - Có thể kiểm tra schema mới có tương thích với schema cũ hay không.

---

## 3. Lợi ích: Compatibility (Tương thích)

Schema Registry hỗ trợ các chế độ **compatibility** như:
- `BACKWARD`  
  Consumer mới đọc được dữ liệu cũ.
- `FORWARD`  
  Consumer cũ đọc được dữ liệu mới.
- `FULL`  
  Đảm bảo cả backward và forward.
- `NONE`  
  Không kiểm tra tương thích.

**Ý nghĩa thực tế:**
- Khi bạn thêm field mới có default value → vẫn **backward compatible**.
- Khi bạn xóa field hoặc đổi kiểu dữ liệu → có thể **breaking** nếu không cẩn thận.

---

## 4. Lợi ích: Evolution (Tiến hóa schema)

**Schema evolution** = khả năng thay đổi schema theo thời gian mà không làm hỏng hệ thống.

Ví dụ:
- V1:
  ```avro
  {
    "type": "record",
    "name": "User",
    "fields": [
      { "name": "id", "type": "string" },
      { "name": "name", "type": "string" }
    ]
  }
  ```
- V2 (thêm field mới với default):
  ```avro
  {
    "type": "record",
    "name": "User",
    "fields": [
      { "name": "id", "type": "string" },
      { "name": "name", "type": "string" },
      { "name": "email", "type": ["null", "string"], "default": null }
    ]
  }
  ```

Nếu chế độ compatibility là `BACKWARD`, V2 được chấp nhận vì:
- Consumer mới (V2) vẫn đọc được dữ liệu cũ (không có `email`, dùng default `null`).

## 5. Ví dụ với Spring Boot + Kafka + Schema Registry

### 5.1. Cấu hình Maven (Avro + Confluent)

```xml
<dependencies>
    <!-- Spring Kafka -->
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>

    <!-- Avro -->
    <dependency>
        <groupId>org.apache.avro</groupId>
        <artifactId>avro</artifactId>
    </dependency>

    <!-- Confluent Kafka Avro Serializer -->
    <dependency>
        <groupId>io.confluent</groupId>
        <artifactId>kafka-avro-serializer</artifactId>
        <version>7.6.0</version>
    </dependency>
</dependencies>
```

### 5.2. application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    properties:
      schema.registry.url: http://localhost:8081
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: io.confluent.kafka.serializers.KafkaAvroSerializer
    consumer:
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: io.confluent.kafka.serializers.KafkaAvroDeserializer
      properties:
        specific.avro.reader: true
```

### 5.3. Ví dụ schema Avro (user.avsc)

```json
{
  "type": "record",
  "name": "User",
  "namespace": "com.example.avro",
  "fields": [
    { "name": "id", "type": "string" },
    { "name": "name", "type": "string" },
    { "name": "email", "type": ["null", "string"], "default": null }
  ]
}
```

Sau khi generate class Avro (User), ta dùng trong producer/consumer.

## 6. Producer Spring Boot (ví dụ)

```java
@Service
public class UserProducer {

    private final KafkaTemplate<String, User> kafkaTemplate;

    public UserProducer(KafkaTemplate<String, User> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendUser() {
        User user = User.newBuilder()
                .setId("u1")
                .setName("Nguyen Van A")
                .setEmail("a@example.com")
                .build();

        kafkaTemplate.send("user-topic", user.getId().toString(), user);
    }
}
```

**Input (object gửi):**
```json
{
  "id": "u1",
  "name": "Nguyen Van A",
  "email": "a@example.com"
}
```

**Trên wire (Kafka message):**
- Key: `"u1"`
- Value: dữ liệu Avro nhị phân + schema ID (ví dụ `schemaId=5`) được lấy từ Schema Registry.

---

## 7. Consumer Spring Boot (ví dụ)

```java
@Service
public class UserConsumer {

    @KafkaListener(topics = "user-topic", groupId = "user-group")
    public void listen(User user) {
        System.out.println("Received user: " + user);
    }
}
```

**Output (log):**
```text
Received user: {"id": "u1", "name": "Nguyen Van A", "email": "a@example.com"}
```

Nếu sau này schema thay đổi (thêm field mới có default), consumer vẫn đọc được nhờ **schema evolution** và **compatibility** do Schema Registry đảm bảo.

---

## 8. Tóm tắt ý chính

- **Schema Registry**:
  - Lưu trữ và quản lý schema cho Kafka.
  - Gán ID cho từng version schema.
  - Kiểm tra **compatibility** khi cập nhật schema.

- **Compatibility**:
  - Đảm bảo producer/consumer cũ và mới vẫn giao tiếp được.
  - Các chế độ: `BACKWARD`, `FORWARD`, `FULL`, `NONE`.

- **Schema evolution**:
  - Cho phép thay đổi schema (thêm field, đổi default, v.v.) mà không phá vỡ hệ thống.
  - Schema Registry là trung tâm kiểm soát và xác thực các thay đổi này.

Nếu bạn muốn, tôi có thể thêm ví dụ về cách **set compatibility mode** qua API Schema Registry.
