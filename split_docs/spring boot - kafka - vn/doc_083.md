
Dưới đây là tóm tắt ngắn gọn về xử lý lỗi trong Spring Kafka với `RetryTemplate` và `RecoveringBatchErrorHandler`, kèm ví dụ code và input/output minh họa.

---

## 1. Khái niệm chính

1. **RetryTemplate (Spring Retry)**  
   - Dùng để tự động thử lại (retry) khi xử lý message Kafka bị lỗi.  
   - Có thể cấu hình:
     - Số lần retry
     - Thời gian chờ giữa các lần retry
     - Loại exception nào được retry / bỏ qua

2. **RecoveringBatchErrorHandler (Spring Kafka)**  
   - Dùng cho **batch listener** (consumer đọc nhiều record cùng lúc).  
   - Khi xử lý batch bị lỗi sau khi đã retry đủ số lần:
     - Có thể “recover” (phục hồi) bằng cách:
       - Gửi message lỗi sang topic khác (DLT – dead letter topic)
       - Log lại, lưu DB, hoặc xử lý tùy ý
   - Tránh việc consumer bị kẹt mãi ở batch lỗi.

## 2. Cấu hình `RetryTemplate` cho KafkaListener

### 2.1. Cấu hình `RetryTemplate`

```java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.retry.backoff.FixedBackOffPolicy;
import org.springframework.retry.policy.SimpleRetryPolicy;
import org.springframework.retry.support.RetryTemplate;

@Configuration
public class RetryConfig {

    @Bean
    public RetryTemplate kafkaRetryTemplate() {
        RetryTemplate retryTemplate = new RetryTemplate();

        // Chính sách retry: tối đa 3 lần
        SimpleRetryPolicy retryPolicy = new SimpleRetryPolicy();
        retryPolicy.setMaxAttempts(3);

        // Backoff: chờ 1 giây giữa các lần retry
        FixedBackOffPolicy backOffPolicy = new FixedBackOffPolicy();
        backOffPolicy.setBackOffPeriod(1000L);

        retryTemplate.setRetryPolicy(retryPolicy);
        retryTemplate.setBackOffPolicy(backOffPolicy);

        return retryTemplate;
    }
}
```

### 2.2. Gắn `RetryTemplate` vào `ConcurrentKafkaListenerContainerFactory`

```java
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.config.ConcurrentKafkaListenerContainerFactory;
import org.springframework.kafka.core.ConsumerFactory;
import org.springframework.retry.support.RetryTemplate;

@Configuration
public class KafkaConsumerConfig {

    @Autowired
    private ConsumerFactory<String, String> consumerFactory;

    @Autowired
    private RetryTemplate kafkaRetryTemplate;

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();

        factory.setConsumerFactory(consumerFactory);

        // Bật batch nếu cần (cho ví dụ RecoveringBatchErrorHandler)
        factory.setBatchListener(false); // true nếu dùng batch

        // Gắn RetryTemplate
        factory.setRetryTemplate(kafkaRetryTemplate);

        // Nếu muốn xử lý khi retry hết mà vẫn lỗi:
        factory.setRecoveryCallback(context -> {
            Object record = context.getAttribute("record");
            // TODO: gửi record sang DLT, log, v.v.
            System.out.println("Recovery after retries. Record: " + record);
            return null;
        });

        return factory;
    }
}
```

### 2.3. Ví dụ `@KafkaListener` đơn lẻ

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class SimpleConsumer {

    @KafkaListener(topics = "input-topic", groupId = "group-1")
    public void listen(String message) {
        System.out.println("Received: " + message);

        // Giả lập lỗi
        if (message.contains("error")) {
            throw new RuntimeException("Processing error");
        }

        // Xử lý bình thường
        System.out.println("Processed: " + message);
    }
}
```

**Input/Output minh họa**

- Input Kafka topic `input-topic`:
  - `"hello"`
  - `"error-message"`

- Hành vi:
  - Với `"hello"`:
    - In:  
      - `Received: hello`  
      - `Processed: hello`
  - Với `"error-message"`:
    - Lần 1: `Received: error-message` → ném `RuntimeException`
    - Lần 2 (retry): `Received: error-message` → lỗi
    - Lần 3 (retry): `Received: error-message` → lỗi
    - Sau đó gọi `RecoveryCallback`:
      - `Recovery after retries. Record: ...`

## 3. `RecoveringBatchErrorHandler` cho batch listener

Dùng khi `factory.setBatchListener(true)` và `@KafkaListener` nhận `List<ConsumerRecord<...>>` hoặc `List<String>`.

### 3.1. Cấu hình `RecoveringBatchErrorHandler`

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.listener.BatchErrorHandler;
import org.springframework.kafka.listener.RecoveringBatchErrorHandler;
import org.springframework.kafka.listener.SeekToCurrentBatchErrorHandler;
import org.springframework.kafka.core.KafkaTemplate;

import java.util.function.BiConsumer;

@Configuration
public class BatchErrorHandlerConfig {

    @Bean
    public BatchErrorHandler batchErrorHandler(KafkaTemplate<String, String> kafkaTemplate) {
        // Hàm recover: xử lý từng record lỗi
        BiConsumer<ConsumerRecord<?, ?>, Exception> recoverer = (record, ex) -> {
            System.out.println("Recovering record: " + record.value() + ", error: " + ex.getMessage());
            // Ví dụ: gửi sang DLT
            kafkaTemplate.send("dlt-topic", record.key() == null ? null : record.key().toString(),
                    record.value().toString());
        };

        // RecoveringBatchErrorHandler: sau khi retry hết, gọi recoverer
        RecoveringBatchErrorHandler handler = new RecoveringBatchErrorHandler(recoverer);

        // Có thể cấu hình số lần retry nội bộ (SeekToCurrentBatchErrorHandler)
        handler.setBatchErrorHandler(new SeekToCurrentBatchErrorHandler());

        return handler;
    }
}
```

### 3.2. Gắn vào `ConcurrentKafkaListenerContainerFactory`

```java
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.config.ConcurrentKafkaListenerContainerFactory;
import org.springframework.kafka.core.ConsumerFactory;
import org.springframework.kafka.listener.BatchErrorHandler;

@Configuration
public class BatchKafkaConsumerConfig {

    @Autowired
    private ConsumerFactory<String, String> consumerFactory;

    @Autowired
    private BatchErrorHandler batchErrorHandler;

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> batchKafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();

        factory.setConsumerFactory(consumerFactory);
        factory.setBatchListener(true); // bật batch
        factory.setBatchErrorHandler(batchErrorHandler);

        return factory;
    }
}
```

### 3.3. Ví dụ `@KafkaListener` dạng batch

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

import java.util.List;

@Service
public class BatchConsumer {

    @KafkaListener(
        topics = "batch-input-topic",
        groupId = "batch-group",
        containerFactory = "batchKafkaListenerContainerFactory"
    )
    public void listenBatch(List<String> messages) {
        System.out.println("Received batch: " + messages);

        for (String msg : messages) {
            if (msg.contains("error")) {
                throw new RuntimeException("Error in batch message: " + msg);
            }
            System.out.println("Processed: " + msg);
        }
    }
}
```

**Input/Output minh họa**

- Input batch: `["ok-1", "error-1", "ok-2"]`
- Hành vi:
  - Consumer ném lỗi khi gặp `"error-1"`.
  - `RecoveringBatchErrorHandler`:
    - Retry batch (tùy cấu hình).
    - Sau khi retry hết mà vẫn lỗi:
      - Gọi `recoverer` cho từng record trong batch:
        - Gửi `"ok-1"`, `"error-1"`, `"ok-2"` sang `dlt-topic` (hoặc tùy logic).
      - Log:  
        - `Recovering record: ok-1, error: Error in batch message: error-1`  
        - `Recovering record: error-1, error: Error in batch message: error-1`  
        - `Recovering record: ok-2, error: Error in batch message: error-1`

## 4. Tóm tắt ý chính

- **`RetryTemplate`**:
  - Dùng cho retry logic ở mức listener (đơn lẻ hoặc batch).
  - Cấu hình số lần retry, backoff, loại exception.
  - Kết hợp với `RecoveryCallback` để xử lý sau khi retry thất bại.

- **`RecoveringBatchErrorHandler`**:
  - Dùng riêng cho **batch listener**.
  - Sau khi retry batch thất bại, gọi hàm recover cho từng record.
  - Thường dùng để gửi message lỗi sang **DLT** hoặc log/ghi DB.

Nếu bạn cho biết thêm:  
- Bạn dùng Spring Kafka version nào  
- Bạn dùng batch hay single message listener  

tôi có thể viết lại ví dụ chính xác hơn theo version và use case của bạn.
