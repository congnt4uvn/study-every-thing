
Dưới đây là tóm tắt ngắn gọn về các kịch bản lỗi trong Kafka và tác động đến Producer/Consumer, kèm ví dụ code Spring Boot (Java) và mô tả input/output.

---

## 1. Lỗi phía Producer

### 1.1. Lỗi kết nối broker (broker down, network lỗi)

**Tác động:**
- Gửi message thất bại.
- Producer sẽ **retry** theo cấu hình (`retries`, `delivery.timeout.ms`).
- Có thể gặp `TimeoutException`, `NetworkException`.

**Cấu hình ví dụ:**

```java
// application.yml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      retries: 5
      properties:
        delivery.timeout.ms: 120000   # tổng thời gian cho retry
        request.timeout.ms: 30000
      acks: all
```

**Gửi message với xử lý lỗi:**

```java
@Autowired
private KafkaTemplate<String, String> kafkaTemplate;

public void sendMessage(String topic, String key, String value) {
    ListenableFuture<SendResult<String, String>> future =
            kafkaTemplate.send(topic, key, value);

    future.addCallback(
        result -> {
            // Output (thành công)
            // Partition, offset, timestamp...
            System.out.println("Sent OK: " + result.getRecordMetadata());
        },
        ex -> {
            // Output (thất bại)
            // Ví dụ: org.apache.kafka.common.errors.TimeoutException
            System.err.println("Send failed: " + ex.getMessage());
        }
    );
}
```

**Input:**  
`topic = "orders"`, `key = "order-1"`, `value = "{...json...}"`  
**Output (khi broker down):**  
Log lỗi: `Send failed: TimeoutException: Failed to update metadata after 30000 ms.`

### 1.2. Lỗi serialization

**Tác động:**
- Producer không serialize được object → `SerializationException`.
- Message không được gửi.

**Ví dụ cấu hình JSON serializer:**

```java
spring:
  kafka:
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.springframework.kafka.support.serializer.JsonSerializer
```

**Code:**

```java
public record OrderEvent(String id, String status) {}

public void sendOrder(OrderEvent event) {
    kafkaTemplate.send("orders", event.getId(), event);
}
```

Nếu `OrderEvent` không serializable đúng cách → `SerializationException`.

## 2. Lỗi phía Consumer

### 2.1. Lỗi kết nối broker

**Tác động:**
- Consumer không poll được message.
- Spring Kafka sẽ tự động retry kết nối.
- Có thể thấy log `org.apache.kafka.common.errors.TimeoutException` hoặc `DisconnectException`.

**Cấu hình cơ bản:**

```java
spring:
  kafka:
    consumer:
      bootstrap-servers: localhost:9092
      group-id: order-group
      auto-offset-reset: earliest
      properties:
        session.timeout.ms: 10000
        max.poll.interval.ms: 300000
```

---

### 2.2. Lỗi xử lý business (exception trong @KafkaListener)

**Tác động:**
- Nếu không xử lý, message có thể bị **re-deliver** (tùy config).
- Có thể gây **retry vô hạn** nếu không có cơ chế DLQ (Dead Letter Queue).

**Ví dụ listener với xử lý lỗi:**

```java
@KafkaListener(topics = "orders", groupId = "order-group")
public void listen(ConsumerRecord<String, String> record) {
    try {
        System.out.println("Input message: " + record.value());
        // Business logic
        if (record.value().contains("ERROR")) {
            throw new RuntimeException("Business error");
        }
        // Output (thành công)
        System.out.println("Processed OK: " + record.key());
    } catch (Exception e) {
        // Output (lỗi)
        System.err.println("Process failed for key " + record.key() + ": " + e.getMessage());
        // Có thể gửi sang topic DLQ
        // dlqKafkaTemplate.send("orders-dlq", record.key(), record.value());
    }
}
```

**Input:** message `"ERROR_ORDER"`  
**Output:** log lỗi, có thể gửi sang `orders-dlq`.

---

### 2.3. Lỗi deserialization

**Tác động:**
- Consumer không đọc được message → `DeserializationException`.
- Nếu không cấu hình, consumer có thể bị “chết” (stop).

**Cấu hình ErrorHandlingDeserializer:**

```java
spring:
  kafka:
    consumer:
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.springframework.kafka.support.serializer.ErrorHandlingDeserializer
      properties:
        spring.deserializer.value.delegate.class: org.springframework.kafka.support.serializer.JsonDeserializer
        spring.json.trusted.packages: "*"
```

**Xử lý record lỗi:**

```java
@KafkaListener(topics = "orders", groupId = "order-group")
public void listenWithErrorHandling(ConsumerRecord<String, String> record) {
    // Nếu deserialization lỗi, value có thể null, headers chứa thông tin lỗi
    if (record.value() == null) {
        System.err.println("Deserialization error, headers: " + record.headers());
        return;
    }
    System.out.println("Valid message: " + record.value());
}
```

## 3. Retry & Timeout (tổng quan)

### Producer
- `retries`: số lần retry gửi.
- `delivery.timeout.ms`: tổng thời gian cho việc gửi + retry.
- `request.timeout.ms`: timeout cho request tới broker.
- `acks`:
  - `0`: không chờ ack → nhanh nhưng dễ mất dữ liệu.
  - `1`: chờ leader ack.
  - `all`: chờ tất cả replica ack → an toàn hơn.

### Consumer
- `max.poll.interval.ms`: thời gian tối đa giữa 2 lần poll; xử lý quá lâu → bị kick khỏi group.
- `session.timeout.ms`: timeout cho heartbeat với broker.
- `max.poll.records`: số record tối đa mỗi lần poll (ảnh hưởng thời gian xử lý).

## 4. Mẫu code cấu hình Retry & DLQ (Spring Kafka)

```java
@Configuration
@EnableKafka
public class KafkaConfig {

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory(
            ConsumerFactory<String, String> consumerFactory,
            KafkaTemplate<String, String> kafkaTemplate) {

        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory);

        // Retry 3 lần, delay 1s
        factory.setCommonErrorHandler(new DefaultErrorHandler(
                new DeadLetterPublishingRecoverer(kafkaTemplate),
                new FixedBackOff(1000L, 3L)
        ));

        return factory;
    }
}
```

**Hành vi:**
- Nếu listener ném exception:
  - Retry tối đa 3 lần (mỗi lần cách 1s).
  - Nếu vẫn lỗi → gửi message sang topic DLQ (mặc định: `<topic>.DLT`).

---

### Tóm tắt ý chính

- Producer:
  - Lỗi kết nối → retry, timeout, `TimeoutException`.
  - Lỗi serialization → `SerializationException`, message không gửi.
  - Cần cấu hình `retries`, `delivery.timeout.ms`, `acks`.

- Consumer:
  - Lỗi kết nối → không poll được, tự retry kết nối.
  - Lỗi business → cần cơ chế retry + DLQ để tránh loop vô hạn.
  - Lỗi deserialization → dùng `ErrorHandlingDeserializer` để không “chết” consumer.
  - Cần chú ý `max.poll.interval.ms`, `session.timeout.ms`.

Nếu bạn cho thêm nội dung chi tiết hơn (log, config cụ thể), tôi có thể viết lại thành tài liệu hoàn chỉnh hơn bằng tiếng Việt.
