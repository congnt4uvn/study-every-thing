---

## 1. Mục tiêu chính

Khi dùng Spring Boot với Kafka và Confluent Schema Registry (Avro/JSON Schema/Protobuf), ta cần:

1. Cấu hình URL của Schema Registry để serializer/deserializer có thể đăng ký và lấy schema.
2. Cấu hình “subject naming strategy” để quyết định tên subject trong Schema Registry (ví dụ: `topic-name-value`, `topic-name-key`, hoặc theo tên record).

---

## 2. Cấu hình Schema Registry URL trong Spring Boot

### application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    properties:
      schema.registry.url: http://localhost:8081
      specific.avro.reader: true
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: io.confluent.kafka.serializers.KafkaAvroSerializer
    consumer:
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: io.confluent.kafka.serializers.KafkaAvroDeserializer
```

- `schema.registry.url`: URL của Schema Registry.
- `KafkaAvroSerializer` / `KafkaAvroDeserializer`: dùng Avro với Schema Registry.

---

## 3. Subject Naming Strategies

Confluent cung cấp 3 chiến lược chính:

1. **TopicNameStrategy** (mặc định)  
   - Subject: `topic-name-key` hoặc `topic-name-value`  
   - Phù hợp khi mỗi topic có schema riêng.

2. **RecordNameStrategy**  
   - Subject: tên đầy đủ của record (FQCN), ví dụ: `com.example.avro.User`  
   - Phù hợp khi muốn dùng cùng một schema cho nhiều topic.

3. **TopicRecordNameStrategy**  
   - Subject: `topic-name-com.example.avro.User`  
   - Kết hợp topic + record name.

### Cấu hình trong application.yml

```yaml
spring:
  kafka:
    properties:
      schema.registry.url: http://localhost:8081
      # Cho Producer
      value.subject.name.strategy: io.confluent.kafka.serializers.subject.TopicNameStrategy
      # Hoặc:
      # value.subject.name.strategy: io.confluent.kafka.serializers.subject.RecordNameStrategy
      # value.subject.name.strategy: io.confluent.kafka.serializers.subject.TopicRecordNameStrategy
```

> Lưu ý:  
> - `key.subject.name.strategy` cho key  
> - `value.subject.name.strategy` cho value

## 4. Ví dụ Producer/Consumer với Avro

Giả sử có Avro record `User`:

```avro
{
  "type": "record",
  "name": "User",
  "namespace": "com.example.avro",
  "fields": [
    {"name": "id", "type": "string"},
    {"name": "name", "type": "string"}
  ]
}
```

### Producer (Spring Boot)

```java
@Service
public class UserProducer {

    private final KafkaTemplate<String, com.example.avro.User> kafkaTemplate;

    public UserProducer(KafkaTemplate<String, com.example.avro.User> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendUser(String topic, String key, String id, String name) {
        com.example.avro.User user = com.example.avro.User.newBuilder()
                .setId(id)
                .setName(name)
                .build();

        kafkaTemplate.send(topic, key, user);
    }
}
```

**Input (gọi service):**

```java
userProducer.sendUser("users-topic", "user-1", "1", "Alice");
```

**Output (trong Kafka + Schema Registry):**

- Kafka topic: `users-topic`
- Message:
  - key: `"user-1"`
  - value (Avro binary) tương ứng với `{ "id": "1", "name": "Alice" }`
- Schema Registry:
  - Nếu dùng `TopicNameStrategy`:
    - subject: `users-topic-value`
  - Nếu dùng `RecordNameStrategy`:
    - subject: `com.example.avro.User`
  - Nếu dùng `TopicRecordNameStrategy`:
    - subject: `users-topic-com.example.avro.User`

---

### Consumer (Spring Boot)

```java
@Service
public class UserConsumer {

    @KafkaListener(topics = "users-topic", groupId = "user-group")
    public void listen(com.example.avro.User user) {
        System.out.println("Received user: " + user.getId() + " - " + user.getName());
    }
}
```

**Input (message từ Kafka):**  
- value: Avro binary (schema id + data)

**Output (log):**

```text
Received user: 1 - Alice
```

---

## 5. Khi nào chọn chiến lược nào?

- **TopicNameStrategy (mặc định)**  
  - Mỗi topic có schema riêng, ít chia sẻ schema giữa các topic.
- **RecordNameStrategy**  
  - Muốn tái sử dụng cùng một schema cho nhiều topic khác nhau.
  - Dễ quản lý version theo record.
- **TopicRecordNameStrategy**  
  - Cần phân biệt schema theo cả topic và record name (tránh xung đột khi cùng record dùng khác nhau trên các topic).

---

Tóm lại:  
- Cấu hình `schema.registry.url` trong `spring.kafka.properties`.  
- Chọn `subject.name.strategy` phù hợp với cách bạn tổ chức schema và topic.  
- Dùng `KafkaAvroSerializer`/`KafkaAvroDeserializer` (hoặc tương đương cho JSON Schema/Protobuf) để Spring Boot tự động đăng ký và lấy schema từ Schema Registry.
