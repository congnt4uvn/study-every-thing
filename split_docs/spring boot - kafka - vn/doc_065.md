
Dưới đây là tóm tắt ngắn gọn về “Monitoring & observability: Key broker metrics (under-replicated partitions, offline partitions)” trong Kafka, kèm ví dụ Spring Boot.

---

## 1. Khái niệm chính

### a. Under-replicated partitions (URP)
- Là các partition mà **số replica “in-sync” (ISR)** < **số replica cấu hình**.
- Nguy cơ:
  - Mất tính sẵn sàng nếu broker leader gặp sự cố.
  - Dữ liệu có thể chưa được sao chép đầy đủ.

**Ý nghĩa:**  
URP > 0 là dấu hiệu cảnh báo: cluster đang không sao chép đủ, cần kiểm tra broker chậm, network, disk, GC, v.v.

---

### b. Offline partitions
- Là các partition **không có leader** khả dụng.
- Producer/consumer **không thể đọc/ghi** vào partition đó.
- Đây là lỗi nghiêm trọng hơn URP.

**Ý nghĩa:**  
Offline partitions > 0 = **sự cố nghiêm trọng**. Cần xử lý ngay (broker chết, mất Zookeeper/Kafka Controller, lỗi cấu hình, v.v.).

---

## 2. Tại sao cần giám sát 2 metric này?

- Đảm bảo:
  - **Tính sẵn sàng** (availability) của hệ thống.
  - **Tính bền vững** (durability) của dữ liệu.
- Thường được đưa vào:
  - Dashboard (Grafana, Prometheus).
  - Alert (Slack, Email, PagerDuty).

## 3. Ví dụ giám sát bằng Spring Boot + Micrometer + Prometheus

### 3.1. Thêm dependency (Maven)

```xml
<dependencies>
    <!-- Spring Boot Actuator -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-actuator</artifactId>
    </dependency>

    <!-- Micrometer Prometheus -->
    <dependency>
        <groupId>io.micrometer</groupId>
        <artifactId>micrometer-registry-prometheus</artifactId>
    </dependency>

    <!-- Kafka client -->
    <dependency>
        <groupId>org.apache.kafka</groupId>
        <artifactId>kafka-clients</artifactId>
        <version>3.7.0</version>
    </dependency>
</dependencies>
```

### 3.2. Cấu hình Actuator Prometheus

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,prometheus
  endpoint:
    prometheus:
      enabled: true
```

Prometheus sẽ scrape tại:  
`http://<app-host>:<port>/actuator/prometheus`

## 4. Đo URP & Offline partitions bằng AdminClient

### 4.1. Tạo bean AdminClient

```java
import org.apache.kafka.clients.admin.AdminClient;
import org.apache.kafka.clients.admin.AdminClientConfig;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import java.util.Properties;

@Configuration
public class KafkaAdminConfig {

    @Bean
    public AdminClient kafkaAdminClient() {
        Properties props = new Properties();
        props.put(AdminClientConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        return AdminClient.create(props);
    }
}
```

### 4.2. Đăng ký metric Micrometer

```java
import io.micrometer.core.instrument.Gauge;
import io.micrometer.core.instrument.MeterRegistry;
import org.apache.kafka.clients.admin.*;
import org.springframework.stereotype.Component;

import javax.annotation.PostConstruct;
import java.util.Collection;
import java.util.concurrent.ExecutionException;

@Component
public class KafkaBrokerMetrics {

    private final AdminClient adminClient;
    private final MeterRegistry meterRegistry;

    public KafkaBrokerMetrics(AdminClient adminClient, MeterRegistry meterRegistry) {
        this.adminClient = adminClient;
        this.meterRegistry = meterRegistry;
    }

    @PostConstruct
    public void initMetrics() {
        Gauge.builder("kafka_under_replicated_partitions", this, KafkaBrokerMetrics::getUnderReplicatedPartitions)
                .description("Number of under-replicated partitions in the cluster")
                .register(meterRegistry);

        Gauge.builder("kafka_offline_partitions", this, KafkaBrokerMetrics::getOfflinePartitions)
                .description("Number of offline partitions in the cluster")
                .register(meterRegistry);
    }

    public double getUnderReplicatedPartitions() {
        try {
            DescribeClusterResult clusterResult = adminClient.describeCluster();
            Collection<Node> nodes = clusterResult.nodes().get();

            int totalURP = 0;
            for (Node node : nodes) {
                DescribeLogDirsResult logDirsResult =
                        adminClient.describeLogDirs(java.util.Collections.singletonList(node.id()));
                var logDirInfos = logDirsResult.all().get();

                // Cách đơn giản: dùng metric cluster-level (nếu có) hoặc tính URP qua DescribeTopics
                // Ở đây minh họa cách đơn giản hơn: dùng DescribeTopics
            }

            // Cách đơn giản hơn: dùng DescribeTopics cho tất cả topic
            var topics = adminClient.listTopics().names().get();
            var topicDesc = adminClient.describeTopics(topics).all().get();

            for (TopicDescription desc : topicDesc.values()) {
                desc.partitions().forEach(p -> {
                    int replicationFactor = p.replicas().size();
                    int isrSize = p.isr().size();
                    if (isrSize < replicationFactor) {
                        // URP: partition này under-replicated
                        // tăng biến đếm
                    }
                });
            }

            // Để đơn giản, giả sử ta chỉ đếm số partition URP:
            int urpCount = 0;
            for (TopicDescription desc : topicDesc.values()) {
                urpCount += (int) desc.partitions().stream()
                        .filter(p -> p.isr().size() < p.replicas().size())
                        .count();
            }

            return urpCount;
        } catch (InterruptedException | ExecutionException e) {
            // Trong trường hợp lỗi, trả về -1 để biết metric đang lỗi
            return -1;
        }
    }

    public double getOfflinePartitions() {
        try {
            var topics = adminClient.listTopics().names().get();
            var topicDesc = adminClient.describeTopics(topics).all().get();

            int offlineCount = 0;
            for (TopicDescription desc : topicDesc.values()) {
                offlineCount += (int) desc.partitions().stream()
                        .filter(p -> p.leader() == null || p.leader().id() == -1)
                        .count();
            }
            return offlineCount;
        } catch (InterruptedException | ExecutionException e) {
            return -1;
        }
    }
}
```

## 5. Ví dụ output Prometheus

Khi Prometheus scrape `/actuator/prometheus`, bạn có thể thấy:

```text
# HELP kafka_under_replicated_partitions Number of under-replicated partitions in the cluster
# TYPE kafka_under_replicated_partitions gauge
kafka_under_replicated_partitions 3.0

# HELP kafka_offline_partitions Number of offline partitions in the cluster
# TYPE kafka_offline_partitions gauge
kafka_offline_partitions 0.0
```

- `kafka_under_replicated_partitions = 3`: có 3 partition đang under-replicated.  
- `kafka_offline_partitions = 0`: không có partition offline (tốt).

---

## 6. Tóm tắt ngắn gọn

- **Under-replicated partitions**: số partition mà replica chưa đồng bộ đủ → cảnh báo về độ bền dữ liệu và khả năng chịu lỗi.
- **Offline partitions**: partition không có leader → không thể đọc/ghi, lỗi nghiêm trọng.
- Cần:
  - Giám sát 2 metric này liên tục.
  - Đưa vào dashboard + alert.
- Có thể dùng:
  - Spring Boot + Actuator + Micrometer + Prometheus.
  - Kafka AdminClient để tính URP & offline partitions và expose thành metric.
