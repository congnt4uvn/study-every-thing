
Dưới đây là tóm tắt ngắn gọn về **Topic, Partition, Replica** trong Kafka, kèm ví dụ code Spring Boot (Java) và input/output minh họa.

---

## 1. Topic là gì?

- **Topic**: “Tên” logic của luồng dữ liệu trong Kafka (ví dụ: `orders`, `payments`).
- Producer **ghi message** vào topic, consumer **đọc message** từ topic.
- Một topic được chia nhỏ thành nhiều **partition**.

---

## 2. Partition là gì?

- **Partition**: Đơn vị lưu trữ vật lý của topic, là một log được **append-only** (chỉ ghi thêm vào cuối).
- Mỗi message trong partition có một **offset** tăng dần (0, 1, 2, …).
- **Ordering (thứ tự)**:
  - Kafka **chỉ đảm bảo thứ tự** message **trong cùng một partition**.
  - Không đảm bảo thứ tự giữa các partition khác nhau của cùng một topic.
- Partition giúp:
  - **Scale**: nhiều consumer có thể đọc song song (mỗi consumer một partition).
  - **Throughput cao**: nhiều partition → nhiều broker xử lý song song.

---

## 3. Replica là gì?

- **Replica**: Bản sao của một partition trên các broker khác nhau.
- Có 2 loại:
  - **Leader replica**: nhận ghi/đọc chính.
  - **Follower replica**: sao chép dữ liệu từ leader, dùng cho **fault tolerance**.
- Nếu broker chứa leader bị lỗi, một follower sẽ được bầu làm leader mới → hệ thống vẫn hoạt động.

---

## 4. Mối quan hệ Topic – Partition – Replica

- Một **topic** có:
  - `N` **partitions** (ví dụ: 3).
  - Mỗi partition có `R` **replicas** (ví dụ: 2).
- Ví dụ: Topic `orders` với `3 partitions`, `2 replicas`:
  - `orders-0` (leader trên broker 1, follower trên broker 2)
  - `orders-1` (leader trên broker 2, follower trên broker 3)
  - `orders-2` (leader trên broker 3, follower trên broker 1)

---

## 5. Ordering theo key (partition key)

- Khi producer gửi message, có thể chỉ định **key**.
- Kafka dùng key để quyết định message sẽ vào **partition nào** (hash(key) → partition).
- Tất cả message có cùng key → cùng partition → **giữ được thứ tự** cho từng key.

## 6. Ví dụ Spring Boot Kafka Producer/Consumer

### 6.1. Cấu hình Maven

```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>
</dependencies>
```

### 6.2. application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
    consumer:
      group-id: order-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      auto-offset-reset: earliest
```

### 6.3. Producer: gửi message với key (đảm bảo ordering theo key)

```java
@Service
public class OrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public OrderProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendOrder(String userId, String orderJson) {
        // userId là key → tất cả order của cùng userId sẽ vào cùng 1 partition
        kafkaTemplate.send("orders", userId, orderJson);
    }
}
```

**Input (REST)**

```http
POST /orders
Content-Type: application/json

{
  "userId": "user-123",
  "orderId": "order-1",
  "amount": 100000
}
```

**Controller**

```java
@RestController
@RequestMapping("/orders")
public class OrderController {

    private final OrderProducer orderProducer;
    private final ObjectMapper objectMapper = new ObjectMapper();

    public OrderController(OrderProducer orderProducer) {
        this.orderProducer = orderProducer;
    }

    @PostMapping
    public String createOrder(@RequestBody Map<String, Object> payload) throws JsonProcessingException {
        String userId = (String) payload.get("userId");
        String orderJson = objectMapper.writeValueAsString(payload);
        orderProducer.sendOrder(userId, orderJson);
        return "Order sent to Kafka";
    }
}
```

### 6.4. Consumer: đọc message từ topic `orders`

```java
@Service
public class OrderConsumer {

    @KafkaListener(topics = "orders", groupId = "order-group")
    public void listen(String message) {
        System.out.println("Received order: " + message);
    }
}
```

**Output (log consumer)**

```text
Received order: {"userId":"user-123","orderId":"order-1","amount":100000}
Received order: {"userId":"user-123","orderId":"order-2","amount":200000}
```

Nếu cả hai order trên đều có `userId = "user-123"`, chúng sẽ vào **cùng partition**, và consumer sẽ nhận **đúng thứ tự gửi**.

---

## 7. Tóm tắt ý chính (tiếng Việt)

- **Topic**: kênh logic chứa dữ liệu (ví dụ: `orders`).
- **Partition**: chia nhỏ topic để scale, đảm bảo **thứ tự trong từng partition**.
- **Replica**: bản sao partition trên nhiều broker để **tăng độ tin cậy**.
- **Ordering**:
  - Đảm bảo trong **một partition**.
  - Dùng **key** để đảm bảo tất cả message cùng key vào cùng partition → giữ thứ tự theo key.
- Spring Boot + Kafka:
  - Producer gửi message với key.
  - Consumer đọc từ topic, nhận message theo thứ tự offset trong partition.
