
Dưới đây là tóm tắt ngắn gọn về chủ đề **theo dõi hiệu năng Producer trong Kafka** với Spring Boot, kèm ví dụ code và input/output.

---

## 1. Ý chính

Khi dùng Kafka Producer (đặc biệt trong ứng dụng Spring Boot), cần **theo dõi các metric quan trọng** để đánh giá hiệu năng và độ ổn định:

1. **record-send-rate**  
   - Tốc độ gửi bản ghi (records) từ Producer tới Kafka (records/giây).  
   - Dùng để biết Producer có đang gửi đủ nhanh, có bị nghẽn hay không.

2. **retries**  
   - Số lần Producer phải gửi lại (retry) do lỗi tạm thời (network, timeout, leader thay đổi…).  
   - Tăng cao → có vấn đề về mạng, cluster Kafka, hoặc cấu hình timeout/acks.

3. **errors**  
   - Số lỗi khi gửi message (ví dụ: timeout, serialization error, authorization error…).  
   - Tăng cao → cần kiểm tra log, cấu hình, và trạng thái cluster.

Mục tiêu:  
- Giữ **record-send-rate ổn định và đủ cao**.  
- **retries** và **errors** ở mức thấp, ổn định.  
- Dùng các metric này để cảnh báo (alert) và scale hệ thống.

## 2. Ví dụ cấu hình Producer trong Spring Boot

**application.yml**:

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      acks: all
      retries: 5
      properties:
        linger.ms: 5
        batch.size: 16384
        enable.idempotence: true
```

## 3. Gửi message với KafkaTemplate

```java
@Service
public class MyKafkaProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public MyKafkaProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String topic, String key, String value) {
        ListenableFuture<SendResult<String, String>> future =
                kafkaTemplate.send(topic, key, value);

        future.addCallback(
            result -> {
                // Gửi thành công
                RecordMetadata meta = result.getRecordMetadata();
                System.out.printf("Sent to %s-%d offset=%d%n",
                        meta.topic(), meta.partition(), meta.offset());
            },
            ex -> {
                // Gửi thất bại
                System.err.println("Send failed: " + ex.getMessage());
            }
        );
    }
}
```

**Input ví dụ:**

```java
myKafkaProducer.sendMessage("orders", "order-1", "{ \"id\": 1, \"amount\": 100000 }");
```

**Output (log):**

```text
Sent to orders-0 offset=15
```

Hoặc nếu lỗi:

```text
Send failed: TimeoutException: Failed to update metadata after 60000 ms.
```

## 4. Lấy metric Producer (record-send-rate, retries, errors)

### 4.1. Lấy metric trực tiếp từ KafkaProducer

Nếu bạn tự tạo `KafkaProducer`:

```java
@Configuration
public class KafkaProducerConfig {

    @Bean
    public KafkaProducer<String, String> rawKafkaProducer() {
        Properties props = new Properties();
        props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG,
                  "org.apache.kafka.common.serialization.StringSerializer");
        props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,
                  "org.apache.kafka.common.serialization.StringSerializer");
        props.put(ProducerConfig.ACKS_CONFIG, "all");
        props.put(ProducerConfig.RETRIES_CONFIG, 5);
        return new KafkaProducer<>(props);
    }
}
```

Lấy metric:

```java
@Component
public class ProducerMetricsLogger {

    private final KafkaProducer<String, String> producer;

    public ProducerMetricsLogger(KafkaProducer<String, String> producer) {
        this.producer = producer;
    }

    @Scheduled(fixedRate = 5000)
    public void logMetrics() {
        Map<MetricName, ? extends Metric> metrics = producer.metrics();

        metrics.forEach((name, metric) -> {
            if (name.name().equals("record-send-rate")
                    || name.name().equals("record-retry-rate")
                    || name.name().equals("record-error-rate")) {
                System.out.printf("%s = %f%n", name.name(), metric.value());
            }
        });
    }
}
```

**Output mẫu (mỗi 5 giây):**

```text
record-send-rate = 1200.500000
record-retry-rate = 3.200000
record-error-rate = 0.000000
```

- `record-send-rate`: ~1200 records/giây  
- `record-retry-rate`: ~3 retries/giây  
- `record-error-rate`: 0 lỗi/giây

### 4.2. Tích hợp với Micrometer + Actuator (Spring Boot)

Trong `build.gradle`:

```groovy
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-actuator'
    implementation 'io.micrometer:micrometer-registry-prometheus'
}
```

`application.yml`:

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,prometheus,metrics
```

Spring Boot sẽ tự export metric Kafka (nếu dùng Spring Kafka 2.5+ và Micrometer).  
Bạn có thể truy cập:

- `/actuator/metrics` – danh sách metric  
- `/actuator/metrics/kafka.producer.record.send.rate` (tên có thể khác tùy version)  

Ví dụ gọi HTTP:

```bash
curl http://localhost:8080/actuator/metrics/kafka.producer.record.send.rate
```

**Output mẫu:**

```json
{
  "name": "kafka.producer.record.send.rate",
  "measurements": [
    { "statistic": "COUNT", "value": 12000.0 },
    { "statistic": "TOTAL_TIME", "value": 10.0 }
  ],
  "availableTags": [
    { "tag": "client.id", "values": ["producer-1"] }
  ]
}
```

## 5. Cách diễn giải metric

- **record-send-rate** thấp bất thường  
  → Có thể Producer bị nghẽn CPU, network, hoặc Kafka cluster quá tải.

- **retries** tăng cao  
  → Lỗi tạm thời, cần kiểm tra:
  - `acks`, `retries`, `delivery.timeout.ms`, `request.timeout.ms`
  - Tình trạng broker, network.

- **errors** tăng  
  → Lỗi nghiêm trọng hơn:
  - Sai cấu hình topic, quyền truy cập, serializer, schema…
  - Cần xem log chi tiết và sửa cấu hình/ứng dụng.
