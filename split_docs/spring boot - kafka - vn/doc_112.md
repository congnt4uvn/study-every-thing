
Dưới đây là tóm tắt ngắn gọn về **Outbox Pattern (DB → Kafka)** trong bối cảnh **Kafka + Spring Boot**, kèm ví dụ code và luồng input/output.

---

## 1. Ý tưởng chính của Outbox Pattern

**Vấn đề**  
Khi một service vừa:
- ghi dữ liệu vào **database (DB)**,  
- vừa gửi **event lên Kafka**,  

nếu làm 2 việc này tách biệt, có thể xảy ra:
- DB ghi thành công nhưng gửi Kafka thất bại (hoặc ngược lại) → dữ liệu không nhất quán.

**Giải pháp: Outbox Pattern**

1. Khi xử lý business (ví dụ: tạo Order):
   - Ghi dữ liệu vào bảng chính (orders).
   - Đồng thời ghi một bản ghi vào bảng **outbox** trong cùng **transaction DB**.
2. Một **process khác** (hoặc thread khác) đọc bảng outbox, gửi message lên Kafka, rồi đánh dấu bản ghi outbox là đã xử lý.

→ Đảm bảo:  
- DB và “ý định gửi event” luôn nhất quán (vì cùng transaction).  
- Nếu gửi Kafka lỗi, có thể retry dựa trên dữ liệu trong outbox.

## 2. Cấu trúc bảng Outbox (ví dụ)

```sql
CREATE TABLE outbox_event (
    id           BIGSERIAL PRIMARY KEY,
    aggregate_id BIGINT NOT NULL,      -- id của entity chính (vd: order_id)
    aggregate_type VARCHAR(100) NOT NULL, -- loại entity (vd: ORDER)
    event_type   VARCHAR(100) NOT NULL,   -- loại event (vd: ORDER_CREATED)
    payload      JSONB NOT NULL,          -- dữ liệu event
    status       VARCHAR(20) NOT NULL DEFAULT 'NEW', -- NEW, SENT, FAILED
    created_at   TIMESTAMP NOT NULL DEFAULT NOW(),
    sent_at      TIMESTAMP
);
```

## 3. Luồng xử lý tổng quát

**Input (ví dụ)**:  
Request HTTP tạo order:
```http
POST /orders
Content-Type: application/json

{
  "customerId": 123,
  "items": [
    { "productId": 1, "quantity": 2 },
    { "productId": 2, "quantity": 1 }
  ]
}
```

**Bước 1 – Service xử lý business + ghi Outbox (trong 1 transaction)**

- Ghi vào bảng `orders`.
- Ghi event vào bảng `outbox_event` với status = `NEW`.

**Output DB sau bước 1 (ví dụ)**:
- Bảng `orders` có thêm 1 dòng mới.
- Bảng `outbox_event` có 1 dòng:
  ```json
  {
    "aggregate_id": 1001,
    "aggregate_type": "ORDER",
    "event_type": "ORDER_CREATED",
    "payload": {
      "orderId": 1001,
      "customerId": 123,
      "items": [...]
    },
    "status": "NEW"
  }
  ```

**Bước 2 – Outbox Processor**

- Định kỳ (hoặc streaming) đọc các bản ghi `status = 'NEW'`.
- Gửi message lên Kafka.
- Nếu gửi thành công → cập nhật `status = 'SENT'`, set `sent_at`.
- Nếu lỗi → có thể set `FAILED` hoặc giữ `NEW` để retry.

**Output Kafka (ví dụ)**:
Topic: `order-events`  
Message:
```json
{
  "eventType": "ORDER_CREATED",
  "orderId": 1001,
  "customerId": 123,
  "items": [...]
}
```

## 4. Ví dụ code Spring Boot + JPA + Kafka

### 4.1. Entity Order & OutboxEvent

```java
@Entity
@Table(name = "orders")
public class Order {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private Long customerId;

    // ... các field khác, items có thể là bảng con hoặc JSON
}
```

```java
@Entity
@Table(name = "outbox_event")
public class OutboxEvent {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private Long aggregateId;
    private String aggregateType;
    private String eventType;

    @Lob
    private String payload; // lưu JSON string

    private String status;  // NEW, SENT, FAILED

    private Instant createdAt;
    private Instant sentAt;
}
```

### 4.2. Repository

```java
public interface OrderRepository extends JpaRepository<Order, Long> {
}

public interface OutboxEventRepository extends JpaRepository<OutboxEvent, Long> {

    // Lấy các event chưa gửi
    List<OutboxEvent> findTop100ByStatusOrderByCreatedAtAsc(String status);
}
```

### 4.3. Service tạo Order (ghi DB + Outbox trong 1 transaction)

```java
@Service
public class OrderService {

    private final OrderRepository orderRepository;
    private final OutboxEventRepository outboxEventRepository;
    private final ObjectMapper objectMapper;

    public OrderService(OrderRepository orderRepository,
                        OutboxEventRepository outboxEventRepository,
                        ObjectMapper objectMapper) {
        this.orderRepository = orderRepository;
        this.outboxEventRepository = outboxEventRepository;
        this.objectMapper = objectMapper;
    }

    @Transactional
    public Order createOrder(CreateOrderRequest request) {
        // 1. Lưu Order
        Order order = new Order();
        order.setCustomerId(request.getCustomerId());
        // set thêm các field khác...
        Order savedOrder = orderRepository.save(order);

        // 2. Tạo payload event
        Map<String, Object> eventPayload = Map.of(
                "eventType", "ORDER_CREATED",
                "orderId", savedOrder.getId(),
                "customerId", savedOrder.getCustomerId()
                // thêm items...
        );

        // 3. Lưu OutboxEvent
        OutboxEvent event = new OutboxEvent();
        event.setAggregateId(savedOrder.getId());
        event.setAggregateType("ORDER");
        event.setEventType("ORDER_CREATED");
        try {
            event.setPayload(objectMapper.writeValueAsString(eventPayload));
        } catch (JsonProcessingException e) {
            throw new RuntimeException(e);
        }
        event.setStatus("NEW");
        event.setCreatedAt(Instant.now());

        outboxEventRepository.save(event);

        // 4. Trả về Order
        return savedOrder;
    }
}
```

**Input**: `CreateOrderRequest` (từ HTTP request).  
**Output**: `Order` đã được lưu, kèm 1 bản ghi outbox `NEW`.

### 4.4. Outbox Processor (gửi Kafka)

```java
@Service
public class OutboxProcessor {

    private final OutboxEventRepository outboxEventRepository;
    private final KafkaTemplate<String, String> kafkaTemplate;

    public OutboxProcessor(OutboxEventRepository outboxEventRepository,
                           KafkaTemplate<String, String> kafkaTemplate) {
        this.outboxEventRepository = outboxEventRepository;
        this.kafkaTemplate = kafkaTemplate;
    }

    // Chạy định kỳ mỗi 5 giây (ví dụ)
    @Scheduled(fixedDelay = 5000)
    @Transactional
    public void processOutbox() {
        List<OutboxEvent> events = outboxEventRepository
                .findTop100ByStatusOrderByCreatedAtAsc("NEW");

        for (OutboxEvent event : events) {
            try {
                // topic có thể dựa trên aggregateType hoặc eventType
                String topic = "order-events";

                kafkaTemplate.send(topic, event.getAggregateId().toString(), event.getPayload())
                             .get(); // get() để chờ kết quả, có thể async nếu muốn

                event.setStatus("SENT");
                event.setSentAt(Instant.now());
            } catch (Exception ex) {
                // Có thể log và set FAILED hoặc giữ NEW để retry
                event.setStatus("FAILED");
            }
        }
        // JPA sẽ flush khi transaction commit
    }
}
```

**Input**: các bản ghi `outbox_event` có `status = NEW`.  
**Output**:
- Message được gửi lên Kafka.
- Bản ghi outbox được cập nhật `status = SENT` hoặc `FAILED`.

---

## 5. Lợi ích chính

- **Đảm bảo nhất quán** giữa DB và event gửi lên Kafka (vì ghi cùng transaction).
- **Retry dễ dàng**: nếu gửi Kafka lỗi, vẫn còn dữ liệu trong outbox để gửi lại.
- **Tách biệt concern**: service business chỉ cần ghi DB + outbox; việc gửi Kafka do process khác đảm nhiệm.
- Phù hợp cho kiến trúc **microservices**, **event-driven**, cần **reliability** cao.

---

Nếu bạn muốn, tôi có thể bổ sung:
- Ví dụ cấu hình `KafkaTemplate` trong Spring Boot.
- Cách dùng Debezium để đọc outbox từ log DB thay vì job `@Scheduled`.
