
Dưới đây là tóm tắt ngắn gọn về “Spring Kafka transactions: Consuming, processing, and producing in a single transaction”, kèm ví dụ code và input/output minh họa.

---

## 1. Ý tưởng chính

- **Mục tiêu**: Đảm bảo tính *atomic* cho luồng:
  1. Consume message từ Kafka
  2. Xử lý (có thể ghi DB, logic nghiệp vụ)
  3. Produce message mới lên Kafka
  4. Commit offset

  Tất cả phải thành **một transaction**:  
  - Thành công → mọi thứ commit  
  - Lỗi → mọi thứ rollback (message sẽ được consume lại, message mới không bị gửi, offset không commit).

- **Công cụ**:
  - `KafkaTransactionManager` hoặc `ChainedKafkaTransactionManager` (kết hợp với `DataSourceTransactionManager` nếu có DB).
  - `KafkaTemplate` với `transactional.id`.
  - `@Transactional` hoặc cấu hình transaction cho listener container.

## 2. Cấu hình cơ bản

### 2.1. Cấu hình Producer hỗ trợ transaction

```java
@Bean
public Map<String, Object> producerConfigs() {
    Map<String, Object> props = new HashMap<>();
    props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
    props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
    props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
    // Bắt buộc để dùng transaction
    props.put(ProducerConfig.ENABLE_IDEMPOTENCE_CONFIG, true);
    props.put(ProducerConfig.TRANSACTIONAL_ID_CONFIG, "tx-producer-1");
    return props;
}

@Bean
public ProducerFactory<String, String> producerFactory() {
    DefaultKafkaProducerFactory<String, String> factory =
            new DefaultKafkaProducerFactory<>(producerConfigs());
    factory.setTransactionIdPrefix("tx-");
    return factory;
}

@Bean
public KafkaTemplate<String, String> kafkaTemplate() {
    return new KafkaTemplate<>(producerFactory());
}
```

---

### 2.2. Transaction Manager cho Kafka

```java
@Bean
public KafkaTransactionManager<String, String> kafkaTransactionManager(
        ProducerFactory<String, String> producerFactory) {
    return new KafkaTransactionManager<>(producerFactory);
}
```

Nếu có DB:

```java
@Bean
public PlatformTransactionManager dbTransactionManager(DataSource dataSource) {
    return new DataSourceTransactionManager(dataSource);
}

@Bean
public ChainedKafkaTransactionManager<String, String> chainedTxManager(
        KafkaTransactionManager<String, String> kafkaTxManager,
        PlatformTransactionManager dbTxManager) {
    return new ChainedKafkaTransactionManager<>(kafkaTxManager, dbTxManager);
}
```

---

### 2.3. Cấu hình Listener Container dùng transaction

```java
@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory(
        ConsumerFactory<String, String> consumerFactory,
        KafkaTransactionManager<String, String> kafkaTransactionManager) {

    ConcurrentKafkaListenerContainerFactory<String, String> factory =
            new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory);

    // Bật transaction cho listener
    factory.getContainerProperties().setTransactionManager(kafkaTransactionManager);

    // Đảm bảo offset commit trong transaction
    factory.getContainerProperties().setAckMode(ContainerProperties.AckMode.RECORD);

    return factory;
}
```

## 3. Consume – Process – Produce trong 1 transaction

### 3.1. Listener với @Transactional

```java
@Service
public class OrderService {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private final OrderRepository orderRepository; // JPA/Hibernate

    public OrderService(KafkaTemplate<String, String> kafkaTemplate,
                        OrderRepository orderRepository) {
        this.kafkaTemplate = kafkaTemplate;
        this.orderRepository = orderRepository;
    }

    @KafkaListener(topics = "orders", groupId = "order-group")
    @Transactional("chainedTxManager") // nếu dùng ChainedKafkaTransactionManager
    public void handleOrder(String message) {
        // 1. Parse message
        Order order = parseOrder(message);

        // 2. Lưu DB
        orderRepository.save(order);

        // 3. Gửi event sang topic khác
        String event = "ORDER_CREATED:" + order.getId();
        kafkaTemplate.send("order-events", order.getId().toString(), event);

        // 4. Offset sẽ được commit khi transaction commit
        // Nếu có exception, tất cả rollback
    }

    private Order parseOrder(String message) {
        // parse JSON -> Order (ví dụ)
        Order order = new Order();
        order.setId(Long.parseLong(message));
        return order;
    }
}
```

## 4. Ví dụ Input/Output

### 4.1. Input

- Kafka topic `orders` có message:

```text
Key: "101"
Value: "101"
```

### 4.2. Xử lý

1. Listener nhận `"101"`.
2. Tạo `Order{id=101}` và lưu vào DB.
3. Gửi message mới lên topic `order-events`:

```text
Key: "101"
Value: "ORDER_CREATED:101"
```
4. Commit offset của message `"101"` trong cùng transaction.

### 4.3. Output (khi thành công)

- DB:
  - Bảng `orders` có bản ghi `id = 101`.
- Kafka:
  - Topic `order-events` có message: `("101", "ORDER_CREATED:101")`.
- Consumer group `order-group`:
  - Offset cho topic `orders` đã được commit qua message `"101"`.

### 4.4. Trường hợp lỗi

- Nếu `orderRepository.save(order)` ném exception:
  - Bản ghi không được lưu vào DB.
  - Message `"ORDER_CREATED:101"` không được gửi.
  - Offset của `"101"` không commit → message sẽ được consume lại sau.

---

## 5. Tóm tắt ngắn

- Spring Kafka cho phép **consume + xử lý + produce + commit offset** trong **một transaction**.
- Dùng:
  - Producer có `transactional.id` + `KafkaTransactionManager`.
  - Listener container cấu hình transaction + `AckMode.RECORD`.
  - (Tuỳ chọn) `ChainedKafkaTransactionManager` để gộp Kafka + DB.
- Lợi ích: đảm bảo tính nhất quán giữa Kafka và các hệ thống khác (DB, service khác), tránh mất dữ liệu hoặc xử lý trùng không kiểm soát.
