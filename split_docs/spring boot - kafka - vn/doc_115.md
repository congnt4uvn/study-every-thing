
Dưới đây là tóm tắt ngắn gọn về “Multi-environment patterns: Dev/stage/prod topic naming and isolation” trong Kafka + Spring Boot, kèm ví dụ code và input/output.

---

## 1. Ý tưởng chính

1. **Tách biệt môi trường**  
   - Không dùng chung topic cho dev, stage, prod.  
   - Mỗi môi trường có **topic riêng** và thường có **cluster riêng** (hoặc ít nhất là namespace/logical separation).

2. **Quy ước đặt tên topic theo môi trường**  
   Một số pattern phổ biến:
   - `dev.<service>.<domain>`  
   - `stage.<service>.<domain>`  
   - `prod.<service>.<domain>`  
   Ví dụ:  
   - `dev.order-service.orders.created`  
   - `stage.order-service.orders.created`  
   - `prod.order-service.orders.created`

3. **Cấu hình Spring Boot theo môi trường**  
   - Dùng `application-dev.yml`, `application-stage.yml`, `application-prod.yml`.  
   - Mỗi file cấu hình:  
     - `bootstrap-servers` khác nhau (cluster khác).  
     - Tên topic khác nhau (theo prefix môi trường).

4. **Lợi ích**  
   - Không “rò rỉ” dữ liệu test sang prod.  
   - Dễ rollback, test A/B, performance test.  
   - Quản lý ACL, quota, retention theo môi trường.

## 2. Ví dụ cấu hình Spring Boot

### 2.1. application.yml (chung)

```yaml
spring:
  kafka:
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
    consumer:
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer

app:
  topics:
    orderCreated: orders.created
```

### 2.2. application-dev.yml

```yaml
spring:
  kafka:
    bootstrap-servers: dev-kafka:9092

app:
  env-prefix: dev.order-service
```

### 2.3. application-stage.yml

```yaml
spring:
  kafka:
    bootstrap-servers: stage-kafka:9092

app:
  env-prefix: stage.order-service
```

### 2.4. application-prod.yml

```yaml
spring:
  kafka:
    bootstrap-servers: prod-kafka:9092

app:
  env-prefix: prod.order-service
```

### 2.5. Ghép prefix + topic trong code

```java
import org.springframework.beans.factory.annotation.Value;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class OrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private final String topicName;

    public OrderProducer(
            KafkaTemplate<String, String> kafkaTemplate,
            @Value("${app.env-prefix}") String envPrefix,
            @Value("${app.topics.orderCreated}") String baseTopic
    ) {
        this.kafkaTemplate = kafkaTemplate;
        this.topicName = envPrefix + "." + baseTopic; 
        // Ví dụ: dev.order-service.orders.created
    }

    public void sendOrderCreated(String orderId) {
        String message = "Order created: " + orderId;
        kafkaTemplate.send(topicName, orderId, message);
    }
}
```

## 3. Ví dụ Consumer với @KafkaListener

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class OrderConsumer {

    @KafkaListener(
        topics = "#{@orderCreatedTopicName}", 
        groupId = "order-service-group"
    )
    public void handleOrderCreated(String message) {
        System.out.println("Received: " + message);
    }
}
```

Bean để resolve tên topic theo môi trường:

```java
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class TopicConfig {

    @Bean
    public String orderCreatedTopicName(
            @Value("${app.env-prefix}") String envPrefix,
            @Value("${app.topics.orderCreated}") String baseTopic
    ) {
        return envPrefix + "." + baseTopic;
    }
}
```

## 4. Ví dụ Input/Output

### 4.1. Ở môi trường dev

- Cấu hình:
  - `env-prefix = dev.order-service`
  - `orderCreated = orders.created`
- Topic thực tế: `dev.order-service.orders.created`

**Input (Producer gọi):**

```java
orderProducer.sendOrderCreated("ORD-123");
```

**Message gửi lên Kafka:**

- Topic: `dev.order-service.orders.created`
- Key: `"ORD-123"`
- Value: `"Order created: ORD-123"`

**Output (Consumer log):**

```text
Received: Order created: ORD-123
```

### 4.2. Ở môi trường prod

- Cấu hình:
  - `env-prefix = prod.order-service`
- Topic thực tế: `prod.order-service.orders.created`

Code không đổi, chỉ đổi profile Spring Boot (`--spring.profiles.active=prod`), dữ liệu sẽ đi vào topic prod.

---

## 5. Tóm tắt ngắn gọn (tiếng Việt)

- Mỗi môi trường (dev/stage/prod) cần **topic riêng** và thường **cluster riêng**.  
- Dùng **prefix môi trường** trong tên topic: `dev.*`, `stage.*`, `prod.*`.  
- Cấu hình Spring Boot theo profile để:
  - Trỏ tới Kafka cluster tương ứng.
  - Tự động ghép prefix môi trường + tên topic.  
- Producer/Consumer dùng cùng một code, chỉ thay đổi profile là tự động tách biệt dữ liệu giữa các môi trường.
