
Dưới đây là tóm tắt ngắn gọn về “Durability & consistency: Exactly-once semantics end-to-end (producer + broker + consumer)” trong Kafka + Spring Boot, kèm ví dụ code.

---

## 1. Ý nghĩa chính

**Exactly-once semantics (EOS)** end-to-end trong Kafka gồm 3 phần:  
- **Producer**: gửi mỗi message **chính xác một lần** (không mất, không trùng)  
- **Broker (Kafka)**: lưu trữ bền vững (durable), đảm bảo thứ tự trong partition  
- **Consumer**: xử lý mỗi message **chính xác một lần**, kể cả khi có lỗi hoặc restart

Để đạt EOS:
1. **Idempotent producer**: tránh gửi trùng khi retry.  
2. **Transactional producer**: gộp nhiều send + commit offset vào **một transaction**.  
3. **Consumer**: commit offset **trong cùng transaction** với việc ghi kết quả (ví dụ ghi DB, ghi sang topic khác).

## 2. Cấu hình Kafka Producer (Spring Boot)

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      # Bật idempotent producer
      properties:
        enable.idempotence: true
        acks: all
        retries: 2147483647
        max.in.flight.requests.per.connection: 5
      # Bật transaction
      transaction-id-prefix: tx-order-
```

**Ý chính**:  
- `enable.idempotence: true` → tránh duplicate do retry.  
- `transaction-id-prefix` → bật transactional producer.

---

## 3. Cấu hình Consumer + Listener Container

```yaml
spring:
  kafka:
    consumer:
      group-id: order-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      enable-auto-commit: false
    listener:
      ack-mode: MANUAL
```

**Ý chính**:  
- Tắt auto-commit → tự kiểm soát commit offset.  
- Dùng `MANUAL` để commit offset trong transaction.

## 4. Transaction Manager trong Spring

```java
import org.springframework.context.annotation.Bean;
import org.springframework.kafka.core.ProducerFactory;
import org.springframework.kafka.transaction.KafkaTransactionManager;
import org.springframework.transaction.PlatformTransactionManager;

@Bean
public KafkaTransactionManager<String, String> kafkaTransactionManager(
        ProducerFactory<String, String> producerFactory) {
    return new KafkaTransactionManager<>(producerFactory);
}
```

Nếu bạn dùng cả DB (JPA) + Kafka, có thể dùng **ChainedTransactionManager** hoặc outbox pattern, nhưng đó là bước nâng cao.

## 5. Ví dụ: Consumer xử lý exactly-once

### Luồng xử lý

1. Consumer đọc message từ topic `orders`.  
2. Xử lý nghiệp vụ (ví dụ: lưu DB, gửi sang topic `order-confirmed`).  
3. Gửi message mới + commit offset **trong cùng transaction Kafka**.  
4. Nếu lỗi → rollback, message sẽ được đọc lại, nhưng nhờ idempotent + logic nghiệp vụ, kết quả vẫn **exactly-once**.

### Code ví dụ

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.kafka.support.Acknowledgment;
import org.springframework.kafka.core.KafkaOperations;
import org.springframework.stereotype.Service;

@Service
public class OrderService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public OrderService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    @KafkaListener(topics = "orders", groupId = "order-group")
    public void handleOrder(ConsumerRecord<String, String> record,
                            Acknowledgment ack) {

        kafkaTemplate.executeInTransaction(operations -> {
            String orderId = record.key();
            String orderJson = record.value();

            // 1. Xử lý nghiệp vụ (ví dụ: lưu DB, kiểm tra tồn kho, ...)
            processOrder(orderId, orderJson);

            // 2. Gửi message sang topic khác (ví dụ: order-confirmed)
            operations.send("order-confirmed", orderId, "CONFIRMED");

            // 3. Commit offset trong transaction
            ack.acknowledge();

            return null;
        });
    }

    private void processOrder(String orderId, String orderJson) {
        // TODO: xử lý nghiệp vụ, đảm bảo idempotent (ví dụ: check nếu order đã xử lý)
    }
}
```

## 6. Ví dụ Input/Output

**Input** (message vào topic `orders`):

- Topic: `orders`  
- Partition: 0  
- Key: `"order-123"`  
- Value: `{"orderId":"order-123","amount":100000}`  

**Xử lý**:
- `processOrder("order-123", "...")`  
- Gửi message: topic `order-confirmed`, key `"order-123"`, value `"CONFIRMED"`  
- Commit offset của message `order-123` trong cùng transaction.

**Output** (message ra topic `order-confirmed`):

- Topic: `order-confirmed`  
- Key: `"order-123"`  
- Value: `"CONFIRMED"`  

Nếu service bị crash sau khi xử lý nhưng trước khi commit, transaction sẽ rollback → message sẽ được đọc lại, nhưng nhờ idempotent logic, order không bị xử lý trùng.

---

## 7. Tóm tắt ngắn

- **Durability**: Kafka lưu message bền vững trên disk, replication giữa brokers.  
- **Consistency + Exactly-once**:
  - Bật **idempotent + transactional producer**.  
  - Consumer **tự commit offset** trong transaction.  
  - Xử lý nghiệp vụ phải **idempotent** (check trùng, dùng key duy nhất, v.v.).  
- Spring Boot + Spring Kafka hỗ trợ EOS thông qua:
  - `enable.idempotence`, `transaction-id-prefix`  
  - `KafkaTemplate.executeInTransaction(...)`  
  - `KafkaTransactionManager` + manual ack.
