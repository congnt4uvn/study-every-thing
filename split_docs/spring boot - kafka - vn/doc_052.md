
Dưới đây là tóm tắt ngắn gọn về “Consumer reliability: Dealing with slow consumers, backpressure, and lag” trong Kafka + Spring Boot, kèm ví dụ code và input/output minh họa.

---

## 1. Khái niệm chính

1. **Slow consumer (consumer chậm)**  
   - Consumer xử lý message lâu → không theo kịp tốc độ producer.  
   - Dẫn đến **consumer lag** (độ trễ) tăng: khoảng cách giữa offset mới nhất trên partition và offset đã xử lý.

2. **Backpressure (chống quá tải)**  
   - Cơ chế để hệ thống “tự giảm tốc” khi downstream (consumer, DB, service khác) bị quá tải.  
   - Trong Kafka, backpressure thường được xử lý bằng:
     - Giới hạn concurrency (số thread xử lý song song).
     - Điều chỉnh `max.poll.records`, `fetch.min.bytes`, `fetch.max.wait.ms`.
     - Tạm dừng/resume consumer (`pause()/resume()`).

3. **Lag (độ trễ consumer)**  
   - Lag cao = consumer không xử lý kịp message mới.  
   - Cần monitor lag và scale thêm consumer hoặc tối ưu xử lý.

---

## 2. Chiến lược xử lý consumer chậm & lag

1. **Tối ưu logic xử lý**  
   - Giảm thời gian xử lý mỗi message (cache, batch, async I/O, giảm call external).  

2. **Tăng concurrency / scale out**  
   - Tăng `concurrency` trong Spring Kafka listener.  
   - Scale thêm instance của service (nhiều consumer group member hơn).

3. **Điều chỉnh cấu hình poll**  
   - `max.poll.records`: giảm để mỗi lần poll ít message hơn → xử lý nhanh hơn.  
   - `max.poll.interval.ms`: tăng nếu xử lý mỗi batch mất nhiều thời gian, tránh bị rebalance.  

4. **Manual commit offset**  
   - Chỉ commit offset sau khi xử lý thành công → đảm bảo at-least-once.  
   - Nếu lỗi, không commit → message sẽ được đọc lại.

5. **Pause/Resume consumer (backpressure)**  
   - Khi downstream quá tải, tạm dừng poll thêm message.  
   - Khi hệ thống hồi phục, resume.

## 3. Ví dụ Spring Boot + Spring Kafka

### 3.1. Cấu hình consumer cơ bản với backpressure nhẹ

```java
@Configuration
@EnableKafka
public class KafkaConsumerConfig {

    @Bean
    public Map<String, Object> consumerConfigs() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "order-consumer-group");

        // Giảm số record mỗi lần poll để tránh quá tải
        props.put(ConsumerConfig.MAX_POLL_RECORDS_CONFIG, 50);

        // Tăng thời gian cho phép xử lý 1 batch
        props.put(ConsumerConfig.MAX_POLL_INTERVAL_MS_CONFIG, 300000); // 5 phút

        // Tắt auto-commit để tự commit sau khi xử lý xong
        props.put(ConsumerConfig.ENABLE_AUTO_COMMIT_CONFIG, false);

        return props;
    }

    @Bean
    public ConsumerFactory<String, String> consumerFactory() {
        return new DefaultKafkaConsumerFactory<>(consumerConfigs());
    }

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory());

        // Tăng concurrency để xử lý song song (tối đa = số partition)
        factory.setConcurrency(3);

        // Cho phép manual ack
        factory.getContainerProperties().setAckMode(ContainerProperties.AckMode.MANUAL);

        return factory;
    }
}
```

### 3.2. Listener với manual commit và xử lý chậm

```java
@Component
public class OrderConsumer {

    @KafkaListener(
        topics = "orders",
        groupId = "order-consumer-group",
        containerFactory = "kafkaListenerContainerFactory"
    )
    public void consume(String message, Acknowledgment ack) {
        try {
            // Giả lập xử lý chậm
            processOrder(message); // có thể mất vài giây

            // Chỉ commit offset sau khi xử lý thành công
            ack.acknowledge();
        } catch (Exception e) {
            // Log lỗi, không commit offset -> message sẽ được đọc lại
            System.err.println("Error processing message: " + message + ", error: " + e.getMessage());
        }
    }

    private void processOrder(String message) throws InterruptedException {
        // Giả lập xử lý tốn 2 giây
        Thread.sleep(2000);
        System.out.println("Processed order: " + message);
    }
}
```

**Input (message trên topic `orders`):**
```json
{"orderId": 123, "amount": 100000, "status": "NEW"}
```

**Output (log):**
```text
Processed order: {"orderId": 123, "amount": 100000, "status": "NEW"}
```

Nếu `processOrder` ném exception, message sẽ không được commit offset và sẽ được đọc lại sau.

### 3.3. Ví dụ pause/resume để backpressure

```java
@Component
public class BackpressureOrderConsumer {

    private final AtomicInteger inFlight = new AtomicInteger(0);
    private static final int MAX_IN_FLIGHT = 100;

    @KafkaListener(
        topics = "orders",
        groupId = "order-consumer-group",
        containerFactory = "kafkaListenerContainerFactory"
    )
    public void consume(String message,
                        Acknowledgment ack,
                        Consumer<?, ?> consumer) {
        try {
            int current = inFlight.incrementAndGet();

            // Nếu số message đang xử lý vượt ngưỡng -> tạm dừng poll
            if (current > MAX_IN_FLIGHT) {
                consumer.pause(consumer.assignment());
                System.out.println("Paused consumer due to high load: " + current);
            }

            // Xử lý async hoặc chậm
            processOrder(message);

            ack.acknowledge();
        } catch (Exception e) {
            System.err.println("Error: " + e.getMessage());
        } finally {
            int current = inFlight.decrementAndGet();
            // Khi tải giảm -> resume
            if (current < MAX_IN_FLIGHT / 2) {
                consumer.resume(consumer.assignment());
                System.out.println("Resumed consumer, current in-flight: " + current);
            }
        }
    }

    private void processOrder(String message) throws InterruptedException {
        Thread.sleep(1000);
        System.out.println("Processed order: " + message);
    }
}
```

**Kịch bản:**
- Khi hệ thống đang xử lý > 100 message cùng lúc → `pause()` → không poll thêm message mới → backpressure.  
- Khi số message đang xử lý giảm < 50 → `resume()` → tiếp tục poll.

## 4. Theo dõi lag

- Dùng `kafka-consumer-groups.sh`:
  ```bash
  kafka-consumer-groups.sh \
    --bootstrap-server localhost:9092 \
    --group order-consumer-group \
    --describe
  ```
- Hoặc dùng Prometheus + Grafana, hoặc các tool như Confluent Control Center, Kafka UI.

---

## 5. Tóm tắt ngắn

- **Slow consumer** → gây **lag**; xử lý bằng tối ưu logic, tăng concurrency, scale out.  
- **Backpressure** → dùng `max.poll.records`, manual commit, pause/resume, giới hạn in-flight.  
- **Lag** cần được monitor; khi lag tăng cao, cần:
  - Tối ưu xử lý.
  - Tăng số consumer.
  - Điều chỉnh cấu hình poll và commit.
