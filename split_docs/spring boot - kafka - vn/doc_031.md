
Dưới đây là tóm tắt ngắn gọn về `compression.type` trong Kafka Producer, kèm ví dụ code và input/output minh họa.

---

## 1. `compression.type` là gì?

`compression.type` là cấu hình của Kafka Producer dùng để nén dữ liệu trước khi gửi lên broker.  
Các giá trị phổ biến:

- `none` (mặc định): không nén
- `gzip`
- `snappy`
- `lz4`
- `zstd`

---

## 2. Trade-off hiệu năng (hiệu suất vs tài nguyên)

**Mục tiêu của nén:**
- Giảm kích thước message → giảm băng thông mạng, giảm dung lượng lưu trữ trên broker.
- Có thể tăng throughput (số message/giây) khi mạng là nút thắt.

**Trade-off chính:**

1. **gzip**
   - Tỷ lệ nén: Tốt (file nhỏ)
   - CPU: Tốn CPU, chậm hơn
   - Dùng khi: Ưu tiên tiết kiệm băng thông/lưu trữ hơn là độ trễ (latency).

2. **snappy**
   - Tỷ lệ nén: Trung bình
   - CPU: Nhanh, ít tốn CPU
   - Dùng khi: Cần throughput cao, độ trễ thấp, chấp nhận file lớn hơn một chút.

3. **lz4**
   - Tỷ lệ nén: Tốt, tốc độ nén/giải nén rất nhanh
   - Thường là lựa chọn cân bằng tốt giữa tốc độ và kích thước
   - Dùng khi: Cần hiệu năng cao, message lớn, hệ thống tải nặng.

4. **zstd**
   - Tỷ lệ nén: Rất tốt (thường tốt hơn gzip)
   - CPU: Có thể tốn CPU hơn, nhưng linh hoạt (có nhiều level nén)
   - Dùng khi: Cần tối ưu kích thước mạnh, Kafka version mới (2.1+), chấp nhận chi phí CPU.

## 3. Cấu hình trong Spring Boot (Spring Kafka)

### 3.1. Cấu hình trong `application.yml`

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      properties:
        compression.type: lz4   # hoặc gzip, snappy, zstd, none
```

### 3.2. Cấu hình bằng Java Config

```java
import org.apache.kafka.clients.producer.ProducerConfig;
import org.apache.kafka.common.serialization.StringSerializer;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.core.DefaultKafkaProducerFactory;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.kafka.core.ProducerFactory;

import java.util.HashMap;
import java.util.Map;

@Configuration
public class KafkaProducerConfig {

    @Bean
    public ProducerFactory<String, String> producerFactory() {
        Map<String, Object> configProps = new HashMap<>();
        configProps.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        configProps.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        configProps.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class);

        // Cấu hình nén
        configProps.put(ProducerConfig.COMPRESSION_TYPE_CONFIG, "lz4"); 
        // hoặc "gzip", "snappy", "zstd", "none"

        return new DefaultKafkaProducerFactory<>(configProps);
    }

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate() {
        return new KafkaTemplate<>(producerFactory());
    }
}
```

## 4. Ví dụ Producer + Input/Output minh họa

### 4.1. Producer gửi message

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class MyProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public MyProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String topic, String key, String message) {
        kafkaTemplate.send(topic, key, message);
    }
}
```

**Input (từ phía ứng dụng):**

```java
myProducer.sendMessage(
    "user-events",
    "user-123",
    "{ \"userId\": \"user-123\", \"action\": \"LOGIN\", \"timestamp\": 1710000000000 }"
);
```

**Output (trên Kafka broker):**

- Broker nhận 1 record vào topic `user-events`, partition tương ứng với key `user-123`.
- Dữ liệu trong batch producer gửi lên broker được nén theo `compression.type` (ví dụ `lz4`).
- Consumer đọc message sẽ nhận được payload đã được giải nén tự động (không cần cấu hình thêm).

---

## 5. Gợi ý chọn `compression.type` theo tình huống

- **Hệ thống nhỏ, không quá tải, ưu tiên đơn giản:** `none` hoặc `snappy`
- **Băng thông mạng hạn chế, log nhiều, lưu trữ đắt:** `gzip` hoặc `zstd`
- **Hệ thống lớn, throughput cao, message nhiều và lớn:** `lz4` hoặc `zstd`
- **Cần độ trễ thấp, CPU không quá mạnh:** `snappy` hoặc `lz4`

---

Tóm lại:  
`compression.type` giúp cân bằng giữa kích thước dữ liệu, băng thông, CPU và độ trễ. Trong Spring Boot, bạn chỉ cần set `spring.kafka.producer.properties.compression.type` (YAML) hoặc `ProducerConfig.COMPRESSION_TYPE_CONFIG` (Java) để bật nén phù hợp với nhu cầu hệ thống.
