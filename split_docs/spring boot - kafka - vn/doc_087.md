
Dưới đây là tóm tắt ngắn gọn về giao dịch (transaction) trong Spring Kafka, cách dùng `@Transactional` và mô hình xử lý “exactly-once”.

---

## 1. Mục tiêu

- Đảm bảo **exactly-once processing**: mỗi message Kafka được xử lý **chính xác một lần** (không mất, không xử lý trùng).
- Đồng bộ giữa:
  - Đọc message từ Kafka (consumer)
  - Ghi dữ liệu vào DB (hoặc hệ thống khác)
  - Gửi message mới ra Kafka (producer)

---

## 2. Các khái niệm chính

1. **Transactional Producer**  
   - Kafka hỗ trợ transaction ở mức producer: nhiều record gửi trong một transaction, commit/abort toàn bộ.
   - Cần cấu hình:
     - `enable.idempotence=true`
     - `transactional.id` (duy nhất cho mỗi instance)

2. **Spring Kafka Transaction Manager**
   - `KafkaTransactionManager` hoặc `ChainedKafkaTransactionManager` (kết hợp Kafka + DB).
   - Cho phép dùng `@Transactional` để bao bọc logic xử lý message.

3. **Exactly-once với Spring Kafka**
   - Mô hình phổ biến:
     - Consumer đọc record
     - Xử lý + ghi DB
     - Gửi message mới ra Kafka
     - Commit offset Kafka trong cùng transaction
   - Nếu lỗi: rollback DB + rollback gửi message + offset không commit → message sẽ được đọc lại.

## 3. Cấu hình cơ bản

### 3.1. Cấu hình Producer hỗ trợ transaction

```java
@Bean
public Map<String, Object> producerConfigs() {
    Map<String, Object> props = new HashMap<>();
    props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
    props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
    props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class);

    // Bật idempotence + transactional
    props.put(ProducerConfig.ENABLE_IDEMPOTENCE_CONFIG, true);
    props.put(ProducerConfig.TRANSACTIONAL_ID_CONFIG, "my-tx-producer-1");
    return props;
}

@Bean
public ProducerFactory<String, String> producerFactory() {
    DefaultKafkaProducerFactory<String, String> factory =
            new DefaultKafkaProducerFactory<>(producerConfigs());
    factory.setTransactionIdPrefix("tx-");
    return factory;
}

@Bean
public KafkaTemplate<String, String> kafkaTemplate() {
    return new KafkaTemplate<>(producerFactory());
}
```

---

### 3.2. Transaction Manager cho Kafka

```java
@Bean
public KafkaTransactionManager<String, String> kafkaTransactionManager(
        ProducerFactory<String, String> producerFactory) {
    return new KafkaTransactionManager<>(producerFactory);
}
```

Nếu có DB (JPA):

```java
@Bean
public PlatformTransactionManager chainedTxManager(
        KafkaTransactionManager<String, String> kafkaTxManager,
        PlatformTransactionManager jpaTxManager) {
    return new ChainedKafkaTransactionManager<>(kafkaTxManager, jpaTxManager);
}
```

## 4. Dùng `@Transactional` với Spring Kafka

### 4.1. Transaction bao trùm: đọc Kafka → xử lý → ghi DB → gửi Kafka

```java
@Service
public class OrderService {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private final OrderRepository orderRepository;

    public OrderService(KafkaTemplate<String, String> kafkaTemplate,
                        OrderRepository orderRepository) {
        this.kafkaTemplate = kafkaTemplate;
        this.orderRepository = orderRepository;
    }

    @Transactional("chainedTxManager") // hoặc "kafkaTransactionManager" nếu chỉ Kafka
    @KafkaListener(topics = "orders", groupId = "order-service")
    public void handleOrder(String message) {
        // 1. Parse message
        Order order = parseOrder(message);

        // 2. Ghi DB
        orderRepository.save(order);

        // 3. Gửi event tiếp theo
        kafkaTemplate.send("order-events", order.getId().toString(), "ORDER_CREATED");

        // 4. Offset commit sẽ được Spring Kafka xử lý trong cùng transaction
    }

    private Order parseOrder(String message) {
        // parse JSON -> Order
        // ví dụ đơn giản:
        Order order = new Order();
        order.setId(Long.parseLong(message));
        return order;
    }
}
```

**Input**:  
- Kafka topic `orders` có message `"123"` (id đơn hàng).

**Xử lý**:
- `handleOrder("123")` được gọi.
- Lưu `Order{id=123}` vào DB.
- Gửi message `"ORDER_CREATED"` với key `"123"` lên topic `order-events`.
- Commit offset của message `"123"` trong cùng transaction.

**Output**:
- DB: có bản ghi Order mới.
- Kafka: topic `order-events` có record key `"123"`, value `"ORDER_CREATED"`.
- Nếu bất kỳ bước nào lỗi → rollback DB + không gửi message + offset không commit → message `"123"` sẽ được đọc lại.

---

## 5. Exactly-once pattern phổ biến

1. **Transactional Consumer + Producer + Offset**  
   - Dùng `KafkaTransactionManager` + `@Transactional` trên method `@KafkaListener`.
   - Commit offset trong transaction → đảm bảo:
     - Nếu DB + gửi Kafka thành công → offset commit.
     - Nếu lỗi → offset không commit → message được xử lý lại.

2. **Outbox Pattern (khi không dùng transaction Kafka)**  
   - Ghi event vào bảng “outbox” trong DB cùng transaction với dữ liệu chính.
   - Một process khác đọc outbox và gửi lên Kafka.
   - Đảm bảo exactly-once ở mức DB.

---

## 6. Lưu ý quan trọng

- Mỗi instance service cần `transactional.id` khác nhau (hoặc prefix + suffix).
- Cần bật `isolation.level=read_committed` cho consumer nếu muốn chỉ đọc message đã commit transaction.
- Transaction làm tăng độ trễ và overhead → chỉ dùng khi thực sự cần exactly-once.
- Khi kết hợp DB + Kafka, nên dùng `ChainedKafkaTransactionManager` để đảm bảo atomic giữa DB và Kafka.
