
Dưới đây là tóm tắt ngắn gọn về **Producer reliability** với **Transactional Producer** trong Kafka + Spring Boot, kèm ví dụ code và input/output.

---

## 1. Ý chính

1. **Transactional Producer** dùng để:
   - Đảm bảo **Exactly-Once Semantics (EOS)**: mỗi message được xử lý đúng 1 lần, kể cả khi có retry.
   - Hỗ trợ **transactions**: nhiều record gửi đến nhiều topic/partition được commit/abort như một đơn vị.
   - **Fencing**: ngăn “producer cũ” (bị treo, mất kết nối) ghi dữ liệu sau khi đã bị thay thế bởi producer mới.

2. **Kafka Streams / Consumers**:
   - Kafka Streams dùng transactional producer để đảm bảo EOS giữa input topic và output topic.
   - Consumer có thể đọc dữ liệu đã commit trong transaction, tránh đọc dữ liệu “nửa vời”.

---

## 2. Cấu hình Transactional Producer trong Spring Boot

### application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      # Bật idempotent + transaction
      properties:
        enable.idempotence: true
        transactional.id: "my-transactional-producer-1"
        acks: all
```

## 3. Ví dụ: Gửi message trong transaction (Spring Kafka)

### Cấu hình KafkaTemplate hỗ trợ transaction

```java
@Configuration
public class KafkaConfig {

    @Bean
    public ProducerFactory<String, String> producerFactory(
            KafkaProperties properties) {
        DefaultKafkaProducerFactory<String, String> factory =
                new DefaultKafkaProducerFactory<>(properties.buildProducerProperties());
        factory.setTransactionIdPrefix("tx-");
        return factory;
    }

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate(
            ProducerFactory<String, String> producerFactory) {
        KafkaTemplate<String, String> template = new KafkaTemplate<>(producerFactory);
        template.setTransactionalIdPrefix("tx-");
        return template;
    }
}
```

### Service gửi message với transaction

```java
@Service
public class OrderService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public OrderService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void processOrder(String orderId) {
        kafkaTemplate.executeInTransaction(operations -> {
            // Gửi nhiều message trong 1 transaction
            operations.send("orders", orderId, "ORDER_CREATED:" + orderId);
            operations.send("order-events", orderId, "EVENT_CREATED:" + orderId);

            // Nếu có lỗi -> throw exception -> transaction bị abort
            if (orderId.equals("error")) {
                throw new RuntimeException("Simulate error");
            }

            return null;
        });
    }
}
```

### Input / Output

- **Input**: gọi REST API:

```http
POST /orders
Content-Type: application/json

{
  "orderId": "123"
}
```

- **Xử lý**: `processOrder("123")` chạy transaction:
  - Gửi record 1: topic `orders`, key `123`, value `ORDER_CREATED:123`
  - Gửi record 2: topic `order-events`, key `123`, value `EVENT_CREATED:123`
  - Không có exception → transaction **commit**

- **Output** (consumer thấy):

Topic `orders`:
```text
key: 123
value: ORDER_CREATED:123
```

Topic `order-events`:
```text
key: 123
value: EVENT_CREATED:123
```

Nếu gọi với:

```http
POST /orders
Content-Type: application/json

{
  "orderId": "error"
}
```

- Trong transaction có exception → transaction **abort**
- **Không** có message nào xuất hiện ở `orders` hay `order-events`.

## 4. Fencing (chống producer cũ ghi dữ liệu)

- `transactional.id` là duy nhất cho một “logical producer”.
- Nếu một instance mới khởi động với cùng `transactional.id`, Kafka sẽ:
  - Gán **epoch** mới cho producer mới.
  - Tự động **fence** (chặn) producer cũ: mọi request từ producer cũ sẽ bị lỗi `ProducerFencedException`.
- Điều này ngăn việc:
  - Producer cũ (bị treo, timeout) quay lại và ghi dữ liệu “lạc hậu” sau khi producer mới đã tiếp quản.

---

## 5. Kafka Streams và EOS

Trong Kafka Streams:

- Cấu hình:

```properties
processing.guarantee=exactly_once_v2
```

- Kafka Streams sẽ:
  - Dùng transactional producer để ghi ra output topic.
  - Commit offset của input topic trong cùng transaction.
  - Đảm bảo: mỗi record input → xử lý → output đúng 1 lần, kể cả khi restart / retry.

---

## 6. Tóm tắt ngắn

- **Transactional Producer**:
  - Đảm bảo **exactly-once** giữa producer và Kafka.
  - Cho phép **gửi nhiều record như một transaction** (commit/abort).
  - **Fencing** bảo vệ khỏi producer cũ ghi dữ liệu sai.
- **Spring Boot + Spring Kafka**:
  - Dùng `transactional.id` + `executeInTransaction` để triển khai.
- **Kafka Streams**:
  - Dùng transaction để đảm bảo EOS giữa input topic và output topic.
