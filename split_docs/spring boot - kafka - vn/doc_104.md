## 1. Event Sourcing với Kafka

**Ý tưởng chính**

- Thay vì lưu trực tiếp trạng thái cuối cùng (state) của entity (ví dụ: Order, Account), ta lưu **chuỗi các sự kiện (events)** đã xảy ra.
- Trạng thái hiện tại được **tái dựng (rebuild)** bằng cách **replay** tất cả events liên quan.
- Kafka đóng vai trò là **event log** phân tán, bền vững, có khả năng scale.

**Lợi ích**

- Lịch sử thay đổi đầy đủ (audit log).
- Dễ dàng tái dựng state, debug, hoặc tạo view mới.
- Phù hợp với kiến trúc microservices, event-driven.

### Ví dụ: Event Sourcing cho Order

**Event model**

```java
public class OrderCreatedEvent {
    private String orderId;
    private String customerId;
    private BigDecimal amount;
    private Instant createdAt;
    // getters/setters
}

public class OrderStatusChangedEvent {
    private String orderId;
    private String oldStatus;
    private String newStatus;
    private Instant changedAt;
    // getters/setters
}
```

**Producer (Spring Boot + Kafka)**

```java
@Service
public class OrderEventProducer {

    private final KafkaTemplate<String, Object> kafkaTemplate;

    public OrderEventProducer(KafkaTemplate<String, Object> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void publishOrderCreated(OrderCreatedEvent event) {
        kafkaTemplate.send("order-events", event.getOrderId(), event);
    }

    public void publishOrderStatusChanged(OrderStatusChangedEvent event) {
        kafkaTemplate.send("order-events", event.getOrderId(), event);
    }
}
```

**Consumer: Rebuild state (Event Store / Projection)**

```java
@Service
public class OrderEventHandler {

    private final OrderViewRepository orderViewRepository;

    public OrderEventHandler(OrderViewRepository orderViewRepository) {
        this.orderViewRepository = orderViewRepository;
    }

    @KafkaListener(topics = "order-events", groupId = "order-view-builder")
    public void handleEvent(Object event) {
        if (event instanceof OrderCreatedEvent created) {
            OrderView view = new OrderView();
            view.setOrderId(created.getOrderId());
            view.setCustomerId(created.getCustomerId());
            view.setAmount(created.getAmount());
            view.setStatus("CREATED");
            orderViewRepository.save(view);
        } else if (event instanceof OrderStatusChangedEvent statusChanged) {
            OrderView view = orderViewRepository
                    .findById(statusChanged.getOrderId())
                    .orElseThrow();
            view.setStatus(statusChanged.getNewStatus());
            orderViewRepository.save(view);
        }
    }
}
```

**Input/Output minh họa**

- Input (REST – tạo order):

```http
POST /orders
Content-Type: application/json

{
  "customerId": "C123",
  "amount": 150000
}
```

- Hệ thống:
  - Tạo `OrderCreatedEvent` và publish lên topic `order-events`.
  - Consumer đọc event, tạo bản ghi `OrderView` trong DB.

- Output (REST – xem order):

```http
GET /orders/{orderId}
```

Response:

```json
{
  "orderId": "O001",
  "customerId": "C123",
  "amount": 150000,
  "status": "CREATED"
}
```

## 2. CQRS (Command Query Responsibility Segregation) với Kafka

**Ý tưởng chính**

- Tách riêng **Command side** (ghi, thay đổi dữ liệu) và **Query side** (đọc dữ liệu).
- Command side xử lý logic nghiệp vụ, phát sinh **events**.
- Query side lắng nghe events từ Kafka, xây dựng **read model** tối ưu cho truy vấn.

**Lợi ích**

- Scale đọc/ghi độc lập.
- Read model có thể tối ưu cho từng use case (SQL, NoSQL, search engine…).
- Kết hợp tốt với Event Sourcing.

### Kiến trúc đơn giản

1. **Command API (REST)** → xử lý request → validate → tạo event → gửi Kafka.
2. **Kafka** → lưu trữ events.
3. **Query service** (hoặc cùng app) → lắng nghe events → cập nhật DB đọc (read DB).
4. **Query API (REST)** → đọc từ read DB.

### Ví dụ: CQRS cho Order với Spring Boot + Kafka

**Command Controller**

```java
@RestController
@RequestMapping("/commands/orders")
public class OrderCommandController {

    private final OrderEventProducer producer;

    public OrderCommandController(OrderEventProducer producer) {
        this.producer = producer;
    }

    @PostMapping
    public ResponseEntity<String> createOrder(@RequestBody CreateOrderRequest request) {
        String orderId = UUID.randomUUID().toString();

        OrderCreatedEvent event = new OrderCreatedEvent();
        event.setOrderId(orderId);
        event.setCustomerId(request.getCustomerId());
        event.setAmount(request.getAmount());
        event.setCreatedAt(Instant.now());

        producer.publishOrderCreated(event);

        return ResponseEntity.ok(orderId);
    }

    @PostMapping("/{orderId}/status")
    public ResponseEntity<Void> changeStatus(
            @PathVariable String orderId,
            @RequestBody ChangeStatusRequest request) {

        OrderStatusChangedEvent event = new OrderStatusChangedEvent();
        event.setOrderId(orderId);
        event.setOldStatus(request.getOldStatus());
        event.setNewStatus(request.getNewStatus());
        event.setChangedAt(Instant.now());

        producer.publishOrderStatusChanged(event);

        return ResponseEntity.accepted().build();
    }
}
```

**Query Controller (đọc từ read model)**

```java
@RestController
@RequestMapping("/queries/orders")
public class OrderQueryController {

    private final OrderViewRepository orderViewRepository;

    public OrderQueryController(OrderViewRepository orderViewRepository) {
        this.orderViewRepository = orderViewRepository;
    }

    @GetMapping("/{orderId}")
    public ResponseEntity<OrderView> getOrder(@PathVariable String orderId) {
        return orderViewRepository.findById(orderId)
                .map(ResponseEntity::ok)
                .orElse(ResponseEntity.notFound().build());
    }
}
```

**Input/Output minh họa**

- Command side:

```http
POST /commands/orders
Content-Type: application/json

{
  "customerId": "C123",
  "amount": 200000
}
```

Response:

```http
200 OK
"O002"   // orderId
```

- Query side (sau khi event được xử lý và read model cập nhật):

```http
GET /queries/orders/O002
```

Response:

```json
{
  "orderId": "O002",
  "customerId": "C123",
  "amount": 200000,
  "status": "CREATED"
}
```

## 3. Kết hợp Event Sourcing + CQRS + Kafka

- **Event Sourcing**: mọi thay đổi là event, lưu trên Kafka.
- **CQRS**:
  - Command side: tạo events.
  - Query side: subscribe events, xây dựng read model.
- Kafka cung cấp:
  - Log bất biến, có thể replay.
  - Scale cao, phân tán.
  - Tách lỏng lẻo giữa các service (event-driven microservices).

Nếu bạn muốn, tôi có thể bổ sung:
- Cấu hình `application.yml` cho Kafka.
- Ví dụ dùng `Kafka Streams` để build read model.
