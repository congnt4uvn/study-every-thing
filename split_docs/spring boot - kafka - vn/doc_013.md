
Dưới đây là tóm tắt ngắn gọn về **Retention & Compaction** trong Kafka, kèm ví dụ code Spring Boot (Java) và mô tả input/output.

---

## 1. Khái niệm chính

### a. Segment rolling (chia segment)
- Log của mỗi partition được chia thành nhiều **segment file**.
- Kafka sẽ “roll” (tạo segment mới) khi:
  - Kích thước segment đạt `log.segment.bytes`, **hoặc**
  - Thời gian segment đạt `log.roll.ms`.
- Mục đích: dễ quản lý, xóa, và compact từng phần thay vì cả log.

---

### b. Retention (chính sách giữ dữ liệu)
Kafka xóa segment cũ dựa trên **thời gian** hoặc **kích thước**:

- Theo thời gian:
  - `retention.ms`: giữ message trong bao lâu.
  - Sau thời gian này, segment cũ có thể bị xóa.
- Theo kích thước:
  - `retention.bytes`: tổng dung lượng log tối đa.
  - Khi vượt quá, Kafka xóa segment cũ nhất.

> Đây là **deletion policy**: xóa hẳn segment cũ.

---

### c. Compaction vs Deletion

#### 1) Deletion policy (log.cleanup.policy=delete)
- Kafka giữ message đến khi:
  - Hết thời gian (`retention.ms`) hoặc
  - Vượt kích thước (`retention.bytes`).
- Sau đó **xóa cả segment**.
- Phù hợp cho:
  - Log sự kiện, tracking, dữ liệu không cần giữ mãi.

#### 2) Compaction policy (log.cleanup.policy=compact)
- Kafka giữ **bản ghi mới nhất cho mỗi key**.
- Các bản ghi cũ (cùng key) sẽ bị compact (loại bỏ) dần.
- Không phụ thuộc nhiều vào thời gian, mà vào key.
- Phù hợp cho:
  - Bảng trạng thái (state), cấu hình, cache, KTable.

#### 3) Kết hợp: `compact,delete`
- Vừa compact theo key, vừa xóa theo retention.
- Giữ bản ghi mới nhất trong một khoảng thời gian, sau đó xóa.

## 2. Ví dụ cấu hình Kafka topic

### a. Tạo topic với deletion policy

```bash
kafka-topics.sh --create \
  --bootstrap-server localhost:9092 \
  --topic user-events \
  --partitions 3 \
  --replication-factor 1 \
  --config retention.ms=604800000 \        # 7 ngày
  --config retention.bytes=1073741824 \    # 1 GB
  --config cleanup.policy=delete
```

- Input: producer gửi nhiều event user.
- Output: sau 7 ngày hoặc khi >1GB, segment cũ bị xóa.

---

### b. Tạo topic với compaction policy

```bash
kafka-topics.sh --create \
  --bootstrap-server localhost:9092 \
  --topic user-profile \
  --partitions 3 \
  --replication-factor 1 \
  --config cleanup.policy=compact \
  --config min.cleanable.dirty.ratio=0.5 \
  --config segment.ms=600000
```

- Input: nhiều message với cùng key (userId), value là profile mới.
- Output: sau compaction, chỉ giữ **bản ghi mới nhất** cho mỗi `userId`.

## 3. Ví dụ Spring Boot + Kafka

### a. Producer gửi dữ liệu (user profile)

```java
@Configuration
@EnableKafka
public class KafkaConfig {

    @Bean
    public ProducerFactory<String, String> producerFactory() {
        Map<String, Object> config = new HashMap<>();
        config.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        config.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        config.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        return new DefaultKafkaProducerFactory<>(config);
    }

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate() {
        return new KafkaTemplate<>(producerFactory());
    }
}
```

```java
@Service
public class UserProfileProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public UserProfileProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendUserProfile(String userId, String profileJson) {
        kafkaTemplate.send("user-profile", userId, profileJson);
    }
}
```

**Input ví dụ:**

```java
producer.sendUserProfile("user-1", "{\"name\":\"A\",\"age\":20}");
producer.sendUserProfile("user-1", "{\"name\":\"A\",\"age\":21}");
producer.sendUserProfile("user-2", "{\"name\":\"B\",\"age\":30}");
```

**Sau compaction (output logic):**
- Topic `user-profile` sẽ chỉ giữ:
  - key `user-1` → `{"name":"A","age":21}`
  - key `user-2` → `{"name":"B","age":30}`

---

### b. Consumer đọc dữ liệu

```java
@Configuration
public class KafkaConsumerConfig {

    @Bean
    public ConsumerFactory<String, String> consumerFactory() {
        Map<String, Object> config = new HashMap<>();
        config.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        config.put(ConsumerConfig.GROUP_ID_CONFIG, "user-profile-consumer");
        config.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        config.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        return new DefaultKafkaConsumerFactory<>(config);
    }

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory());
        return factory;
    }
}
```

```java
@Service
public class UserProfileConsumer {

    @KafkaListener(topics = "user-profile", groupId = "user-profile-consumer")
    public void listen(ConsumerRecord<String, String> record) {
        System.out.printf("Key=%s, Value=%s, Offset=%d%n",
                record.key(), record.value(), record.offset());
    }
}
```

## 4. Tóm tắt ngắn

- **Segment rolling**: chia log thành nhiều file nhỏ để dễ xóa/compact.
- **Retention (delete)**: xóa segment cũ theo **thời gian** hoặc **kích thước**.
- **Compaction**: giữ **bản ghi mới nhất cho mỗi key**, phù hợp cho dữ liệu trạng thái.
- Có thể dùng `cleanup.policy=delete`, `compact`, hoặc `compact,delete` tùy use case.
