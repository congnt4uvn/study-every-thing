
Dưới đây là tóm tắt ngắn gọn về **Consumer reliability: Idempotent consumers (deduplication, transactional reads)** trong Kafka + Spring Boot, kèm ví dụ code.

---

## 1. Ý tưởng chính

**Mục tiêu:** Đảm bảo consumer xử lý dữ liệu **an toàn, không trùng lặp, không mất dữ liệu**, ngay cả khi:

- Kafka gửi lại message (retries, rebalancing, network issues…)
- Consumer bị crash giữa chừng
- Có nhiều consumer cùng xử lý

**Idempotent consumer**:  
Consumer được thiết kế sao cho **xử lý cùng một message nhiều lần vẫn cho kết quả như một lần** (không gây trùng dữ liệu, không tăng số dư 2 lần, không insert record trùng…).

Hai kỹ thuật chính:

1. **Deduplication (khử trùng lặp)**  
   - Lưu lại các message đã xử lý (theo `messageId`, `key`, `offset`…)  
   - Nếu nhận lại message đã xử lý → bỏ qua.

2. **Transactional reads (đọc + ghi trong 1 transaction)**  
   - Kết hợp **đọc từ Kafka** và **ghi vào DB** trong một transaction logic.  
   - Đảm bảo: hoặc **cả hai cùng thành công**, hoặc **cùng rollback**.  
   - Tránh trường hợp: DB đã ghi nhưng offset chưa commit (hoặc ngược lại).

## 2. Ví dụ: Idempotent consumer với deduplication (Spring Boot + Kafka)

### 2.1. Cấu trúc message (có `id` duy nhất)

```java
public class OrderEvent {
    private String id;        // unique business id
    private String userId;
    private Double amount;
    // getters/setters
}
```

### 2.2. Bảng DB để lưu dấu message đã xử lý

```sql
CREATE TABLE processed_message (
    message_id VARCHAR(100) PRIMARY KEY,
    processed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 2.3. Repository

```java
public interface ProcessedMessageRepository extends JpaRepository<ProcessedMessage, String> {
}
```

```java
@Entity
@Table(name = "processed_message")
public class ProcessedMessage {
    @Id
    private String messageId;
    private Instant processedAt = Instant.now();
    // getters/setters
}
```

### 2.4. Consumer idempotent

```java
@Service
public class OrderConsumer {

    private final ProcessedMessageRepository processedRepo;
    private final OrderService orderService;

    public OrderConsumer(ProcessedMessageRepository processedRepo,
                         OrderService orderService) {
        this.processedRepo = processedRepo;
        this.orderService = orderService;
    }

    @KafkaListener(topics = "orders", groupId = "order-service")
    @Transactional  // transaction DB (và có thể kết hợp transaction Kafka nếu cấu hình)
    public void consume(OrderEvent event) {
        // 1. Kiểm tra đã xử lý chưa
        boolean exists = processedRepo.existsById(event.getId());
        if (exists) {
            // Đã xử lý rồi → bỏ qua (idempotent)
            return;
        }

        // 2. Xử lý nghiệp vụ
        orderService.processOrder(event);

        // 3. Lưu dấu đã xử lý
        ProcessedMessage pm = new ProcessedMessage();
        pm.setMessageId(event.getId());
        processedRepo.save(pm);
    }
}
```

**Input (message Kafka):**

```json
{
  "id": "order-123",
  "userId": "user-1",
  "amount": 100.0
}
```

**Output (trong DB):**

- Bảng `orders`: thêm 1 order mới
- Bảng `processed_message`: thêm 1 record với `message_id = "order-123"`

Nếu Kafka gửi lại cùng message:

- Consumer nhận lại `"order-123"`
- `existsById("order-123") == true` → bỏ qua → **không tạo order trùng**.

## 3. Transactional reads (Kafka + DB transaction)

Ý tưởng:  
- Đọc message từ Kafka  
- Xử lý + ghi DB  
- Commit offset **chỉ khi** DB commit thành công.

### 3.1. Cấu hình Kafka transaction (producer + consumer)

```yaml
spring:
  kafka:
    consumer:
      group-id: order-service
      enable-auto-commit: false
      properties:
        isolation.level: read_committed
    listener:
      ack-mode: MANUAL
    producer:
      transaction-id-prefix: tx-order-
      properties:
        enable.idempotence: true
```

### 3.2. Listener với transaction (pseudo-code)

```java
@Service
public class OrderConsumerTx {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private final OrderService orderService;

    public OrderConsumerTx(KafkaTemplate<String, String> kafkaTemplate,
                           OrderService orderService) {
        this.kafkaTemplate = kafkaTemplate;
        this.orderService = orderService;
    }

    @KafkaListener(topics = "orders", groupId = "order-service",
                   containerFactory = "kafkaListenerContainerFactory")
    @Transactional  // transaction DB
    public void consume(OrderEvent event, Acknowledgment ack) {
        // Bắt đầu transaction Kafka + DB (nếu cấu hình ChainedTransactionManager)
        orderService.processOrder(event); // ghi DB

        // Có thể gửi tiếp message sang topic khác trong cùng transaction
        kafkaTemplate.send("orders-processed", event.getId(), "OK");

        // Commit offset sau khi DB + Kafka đều OK
        ack.acknowledge();
    }
}
```

**Kết quả:**

- Nếu xử lý thành công → DB commit, offset commit → message không bị xử lý lại.
- Nếu lỗi giữa chừng → DB rollback, offset không commit → message sẽ được đọc lại → vẫn an toàn nhờ idempotent/deduplication.

---

## 4. Tóm tắt

- **Idempotent consumer**: thiết kế logic xử lý sao cho **xử lý lại không gây sai lệch**.
- **Deduplication**: lưu dấu message đã xử lý (theo `id`/`key`/`offset`) để bỏ qua message trùng.
- **Transactional reads**: kết hợp transaction Kafka + DB để đảm bảo **đọc–ghi–commit offset** là một khối nguyên tử.
- Trong Spring Boot + Kafka:
  - Dùng `@KafkaListener`, `@Transactional`, `enable-auto-commit=false`, `ack-mode=MANUAL`.
  - Có thể dùng bảng `processed_message` để khử trùng lặp.
