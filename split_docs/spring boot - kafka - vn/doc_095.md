
Dưới đây là tóm tắt ngắn gọn về **Schema Registry với Avro/Protobuf trong Kafka (Confluent)**, kèm ví dụ Spring Boot.

---

## 1. Ý tưởng chính

- **Schema Registry**: dịch vụ trung tâm lưu trữ và quản lý schema (Avro/Protobuf/JSON Schema) cho các message Kafka.
- **Mục tiêu**:
  - Tách schema khỏi code.
  - Đảm bảo **tương thích schema** (backward/forward).
  - Giảm lỗi do thay đổi cấu trúc message.
- **Avro / Protobuf**:
  - Định nghĩa cấu trúc dữ liệu (fields, type, optional/required).
  - Được compile thành class Java để dùng trong producer/consumer.
- **Confluent Schema Registry**:
  - Lưu schema theo **subject** (thường là `<topic>-value` hoặc `<topic>-key`).
  - Mỗi schema có **version**.
  - Producer/Consumer dùng **ID schema** (do registry cấp) để serialize/deserialize.

---

## 2. Luồng hoạt động

1. Bạn định nghĩa schema (Avro `.avsc` hoặc Protobuf `.proto`).
2. Producer serialize object theo schema, gửi lên Kafka:
   - Gửi kèm **schema ID** (lấy từ Schema Registry).
3. Consumer nhận message:
   - Đọc schema ID.
   - Lấy schema tương ứng từ Schema Registry.
   - Deserialize thành object Java.

---

## 3. Ví dụ với Avro + Spring Boot

### 3.1. Định nghĩa Avro schema (user.avsc)

```json
{
  "type": "record",
  "name": "User",
  "namespace": "com.example.avro",
  "fields": [
    { "name": "id", "type": "string" },
    { "name": "name", "type": "string" },
    { "name": "age", "type": ["null", "int"], "default": null }
  ]
}
```

Sau đó dùng plugin Maven/Gradle để generate class `com.example.avro.User`.

---

### 3.2. Cấu hình Maven (rút gọn)

```xml
<dependencies>
    <!-- Spring Kafka -->
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>

    <!-- Confluent Avro serializer -->
    <dependency>
        <groupId>io.confluent</groupId>
        <artifactId>kafka-avro-serializer</artifactId>
        <version>7.6.0</version>
    </dependency>
</dependencies>

<build>
    <plugins>
        <!-- Avro codegen -->
        <plugin>
            <groupId>org.apache.avro</groupId>
            <artifactId>avro-maven-plugin</artifactId>
            <version>1.11.3</version>
            <configuration>
                <sourceDirectory>${project.basedir}/src/main/avro</sourceDirectory>
                <outputDirectory>${project.basedir}/target/generated-sources/avro</outputDirectory>
            </configuration>
            <executions>
                <execution>
                    <goals>
                        <goal>schema</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>
```

---

### 3.3. application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    properties:
      schema.registry.url: http://localhost:8081
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: io.confluent.kafka.serializers.KafkaAvroSerializer
    consumer:
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: io.confluent.kafka.serializers.KafkaAvroDeserializer
      properties:
        specific.avro.reader: true
    consumer:
      group-id: user-service
```

---

### 3.4. Producer (Spring Boot)

```java
@Service
public class UserProducer {

    private final KafkaTemplate<String, User> kafkaTemplate;

    public UserProducer(KafkaTemplate<String, User> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendUser(User user) {
        kafkaTemplate.send("user-topic", user.getId().toString(), user);
    }
}
```

**Input (Java object gửi đi)**

```java
User user = User.newBuilder()
        .setId("u-1")
        .setName("Nguyen Van A")
        .setAge(25)
        .build();

userProducer.sendUser(user);
```

**Message thực tế trên Kafka (đã serialize)**  
- Binary, không đọc được bằng mắt thường.  
- Phần đầu chứa **magic byte + schema ID**, phần sau là dữ liệu Avro.

---

### 3.5. Consumer (Spring Boot)

```java
@Service
public class UserConsumer {

    @KafkaListener(topics = "user-topic", groupId = "user-service")
    public void listen(User user) {
        System.out.println("Received user: " + user);
    }
}
```

**Output (log)**

```text
Received user: {"id": "u-1", "name": "Nguyen Van A", "age": 25}
```

## 4. Ví dụ schema Protobuf (rút gọn)

`user.proto`:

```proto
syntax = "proto3";

package com.example.proto;

message User {
  string id = 1;
  string name = 2;
  int32 age = 3;
}
```

- Dùng plugin Protobuf để generate class Java.
- Thay serializer/deserializer bằng:
  - `io.confluent.kafka.serializers.protobuf.KafkaProtobufSerializer`
  - `io.confluent.kafka.serializers.protobuf.KafkaProtobufDeserializer`

---

## 5. Lợi ích chính

- Quản lý schema tập trung, có version.
- Đảm bảo tương thích khi thay đổi schema (thêm field, đổi optional/required).
- Giảm lỗi serialize/deserialize giữa nhiều service.
- Hỗ trợ nhiều format: Avro, Protobuf, JSON Schema.

Nếu bạn muốn, tôi có thể viết thêm ví dụ đầy đủ cho Protobuf hoặc cấu hình rule tương thích schema (backward/forward).
