
Dưới đây là tóm tắt ngắn gọn về các thành phần kiến trúc Kafka: **replica, ISR, controller**, kèm ví dụ (có minh họa input/output) và một chút liên hệ Spring Boot.

---

## 1. Replica (Bản sao)

**Khái niệm:**
- Mỗi **partition** trong Kafka có nhiều **replica** (bản sao dữ liệu).
- Có 1 **leader replica** và 0..n **follower replica**.
- Producer/Consumer chỉ đọc/ghi vào **leader**; follower chỉ **sao chép** dữ liệu từ leader.

**Mục đích:**
- Tăng **tính sẵn sàng** (high availability).
- Nếu leader chết, một follower có thể được bầu làm leader mới.

**Ví dụ cấu hình topic:**

```bash
# Tạo topic với 3 partition, mỗi partition có 3 replica
kafka-topics.sh --create \
  --topic demo-topic \
  --bootstrap-server localhost:9092 \
  --partitions 3 \
  --replication-factor 3
```

**Kết quả (mô phỏng):**

- Partition 0: leader = broker 1, replicas = [1,2,3]
- Partition 1: leader = broker 2, replicas = [1,2,2]
- Partition 2: leader = broker 3, replicas = [1,2,3]

---

## 2. ISR (In-Sync Replica)

**Khái niệm:**
- ISR = tập các replica **đang đồng bộ đầy đủ** với leader (không bị trễ quá ngưỡng).
- Bao gồm: **leader** + các follower theo kịp leader.
- Replica nào bị trễ quá nhiều sẽ bị **loại khỏi ISR**.

**Vai trò:**
- Khi leader chết, **chỉ replica trong ISR** mới được chọn làm leader mới → đảm bảo **không mất dữ liệu** (hoặc mất rất ít tùy cấu hình).

**Ví dụ:**

- Partition 0 có replicas: [1,2,3]
- Leader = broker 1
- ISR ban đầu: [1,2,3]

Nếu broker 3 bị chậm, không theo kịp leader:
- ISR cập nhật: [1,2]
- Replica 3 vẫn là replica, nhưng **không nằm trong ISR**.

---

## 3. Controller

**Khái niệm:**
- Là một **broker đặc biệt** trong cluster, được bầu làm **controller**.
- Có nhiệm vụ quản lý **metadata** của cluster:
  - Theo dõi broker nào sống/chết.
  - Gán leader cho các partition.
  - Cập nhật ISR, phân vùng, replica…

**Vai trò chính:**
- Khi một broker (leader) chết:
  - Controller phát hiện.
  - Chọn **leader mới** từ ISR.
  - Thông báo cho các broker khác.

**Ví dụ (luồng sự kiện):**

1. Broker 2 (leader của partition 1) bị down.
2. Controller phát hiện broker 2 mất kết nối.
3. Controller chọn broker 1 (trong ISR) làm leader mới cho partition 1.
4. Producer/Consumer tự động chuyển sang leader mới (broker 1).

## 4. Liên hệ với Spring Boot (Spring Kafka)

Trong Spring Boot, bạn không thao tác trực tiếp với replica/ISR/controller, nhưng bạn **hưởng lợi** từ cơ chế này (tự động failover, độ bền dữ liệu).

### 4.1. Producer (Spring Boot)

**application.yml:**

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      acks: all   # yêu cầu leader + ISR xác nhận -> an toàn hơn
```

**Producer code:**

```java
@Service
public class DemoProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public DemoProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String key, String value) {
        kafkaTemplate.send("demo-topic", key, value)
            .whenComplete((result, ex) -> {
                if (ex == null) {
                    System.out.println("Sent to partition " +
                        result.getRecordMetadata().partition() +
                        " offset " + result.getRecordMetadata().offset());
                } else {
                    ex.printStackTrace();
                }
            });
    }
}
```

**Input (gọi hàm):**

```java
demoProducer.sendMessage("user-1", "Hello Kafka");
```

**Output (log):**

```text
Sent to partition 0 offset 15
```

Dù leader của partition 0 có thể thay đổi (do controller bầu lại), code Spring Boot **không cần đổi**.

---

### 4.2. Consumer (Spring Boot)

```java
@Service
public class DemoConsumer {

    @KafkaListener(topics = "demo-topic", groupId = "demo-group")
    public void listen(String message) {
        System.out.println("Received: " + message);
    }
}
```

**Input (message trong topic):**

- Producer gửi: `"Hello Kafka"`

**Output (log consumer):**

```text
Received: Hello Kafka
```

Nếu leader partition thay đổi (do controller), consumer vẫn nhận được message bình thường.

---

## Tóm tắt ngắn

- **Replica**: các bản sao của partition trên nhiều broker (1 leader + nhiều follower) → tăng sẵn sàng.
- **ISR (In-Sync Replica)**: tập các replica đang đồng bộ với leader; chỉ replica trong ISR mới được bầu làm leader mới.
- **Controller**: broker đặc biệt quản lý metadata, bầu leader, cập nhật ISR, xử lý broker chết.

Spring Boot (Spring Kafka) sử dụng các cơ chế này ở tầng Kafka, giúp ứng dụng **tự động chịu lỗi** mà không cần code phức tạp.
