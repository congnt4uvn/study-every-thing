
Dưới đây là tóm tắt ngắn gọn về các kiểu serialization thường dùng với Kafka + Spring Boot, kèm ví dụ code và input/output minh họa.

---

## 1. String Serialization

**Ý chính**  
- Dùng cho dữ liệu dạng text đơn giản.  
- Dễ debug (có thể đọc trực tiếp trên console).  
- Không có schema, không kiểm soát cấu trúc dữ liệu.

**Producer config (Spring Boot `application.yml`)**

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

**Producer code**

```java
@Autowired
private KafkaTemplate<String, String> kafkaTemplate;

public void sendStringMessage() {
    String topic = "string-topic";
    String key = "user1";
    String value = "Hello Kafka";
    kafkaTemplate.send(topic, key, value);
}
```

**Consumer config**

```yaml
spring:
  kafka:
    consumer:
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      group-id: string-group
```

**Consumer code**

```java
@KafkaListener(topics = "string-topic", groupId = "string-group")
public void listenString(String message) {
    System.out.println("Received: " + message);
}
```

**Input/Output ví dụ**  
- Input gửi: `"Hello Kafka"`  
- Output nhận: `"Hello Kafka"`

## 2. JSON Serialization

**Ý chính**  
- Phù hợp cho object Java (POJO).  
- Dễ đọc, dễ debug.  
- Không có schema chặt chẽ như Avro/Protobuf, nhưng linh hoạt.

**Model**

```java
public class User {
    private String id;
    private String name;
    private int age;
    // getters/setters
}
```

**Config**

```yaml
spring:
  kafka:
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.springframework.kafka.support.serializer.JsonSerializer
    consumer:
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.springframework.kafka.support.serializer.JsonDeserializer
    properties:
      spring.json.trusted.packages: "*"
```

**Producer**

```java
@Autowired
private KafkaTemplate<String, User> kafkaTemplate;

public void sendUser() {
    User user = new User();
    user.setId("u1");
    user.setName("An");
    user.setAge(25);
    kafkaTemplate.send("user-json-topic", user.getId(), user);
}
```

**Consumer**

```java
@KafkaListener(topics = "user-json-topic", groupId = "json-group")
public void listenUser(User user) {
    System.out.println("Received user: " + user.getName());
}
```

**Input/Output (trên wire)**  
- Gửi object `User(id="u1", name="An", age=25)`  
- Kafka message value (JSON):  
  ```json
  {"id":"u1","name":"An","age":25}
  ```

## 3. Avro Serialization

**Ý chính**  
- Có schema rõ ràng, hỗ trợ schema evolution.  
- Thường dùng với Confluent Schema Registry.  
- Hiệu quả hơn JSON (nhỏ hơn, nhanh hơn).

**Schema Avro (user.avsc)**

```json
{
  "type": "record",
  "name": "UserAvro",
  "namespace": "com.example.avro",
  "fields": [
    {"name": "id", "type": "string"},
    {"name": "name", "type": "string"},
    {"name": "age", "type": "int"}
  ]
}
```

Sau khi generate class Avro (bằng plugin Maven/Gradle), ta dùng class `UserAvro`.

**Config (dùng Confluent)**

```yaml
spring:
  kafka:
    properties:
      schema.registry.url: http://localhost:8081
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: io.confluent.kafka.serializers.KafkaAvroSerializer
    consumer:
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: io.confluent.kafka.serializers.KafkaAvroDeserializer
      properties:
        specific.avro.reader: true
```

**Producer**

```java
@Autowired
private KafkaTemplate<String, UserAvro> kafkaTemplate;

public void sendAvroUser() {
    UserAvro user = UserAvro.newBuilder()
            .setId("u1")
            .setName("An")
            .setAge(25)
            .build();
    kafkaTemplate.send("user-avro-topic", user.getId(), user);
}
```

**Consumer**

```java
@KafkaListener(topics = "user-avro-topic", groupId = "avro-group")
public void listenAvroUser(UserAvro user) {
    System.out.println("Received Avro user: " + user.getName());
}
```

**Input/Output**  
- Input: object Avro `UserAvro(id="u1", name="An", age=25)`  
- Output: consumer nhận lại đúng object `UserAvro` (dạng nhị phân Avro trên wire, không đọc được bằng mắt thường).

## 4. Protobuf Serialization

**Ý chính**  
- Định dạng nhị phân, rất nhỏ gọn, hiệu năng cao.  
- Có schema `.proto`, hỗ trợ nhiều ngôn ngữ.  
- Thường dùng trong hệ thống đa ngôn ngữ (Java, Go, Node.js, v.v.).

**File `.proto`**

```proto
syntax = "proto3";

package com.example.proto;

message UserProto {
  string id = 1;
  string name = 2;
  int32 age = 3;
}
```

Generate class `UserProto` (Java) bằng protoc.

**Config (dùng Protobuf serializer)**

```yaml
spring:
  kafka:
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: io.confluent.kafka.serializers.protobuf.KafkaProtobufSerializer
    consumer:
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: io.confluent.kafka.serializers.protobuf.KafkaProtobufDeserializer
    properties:
      schema.registry.url: http://localhost:8081
```

**Producer**

```java
@Autowired
private KafkaTemplate<String, UserProto> kafkaTemplate;

public void sendProtoUser() {
    UserProto user = UserProto.newBuilder()
            .setId("u1")
            .setName("An")
            .setAge(25)
            .build();
    kafkaTemplate.send("user-proto-topic", user.getId(), user);
}
```

**Consumer**

```java
@KafkaListener(topics = "user-proto-topic", groupId = "proto-group")
public void listenProtoUser(UserProto user) {
    System.out.println("Received Proto user: " + user.getName());
}
```

**Input/Output**  
- Input: `UserProto(id="u1", name="An", age=25)`  
- Output: consumer nhận `UserProto` tương ứng (dữ liệu nhị phân Protobuf trên wire).

## 5. Thrift Serialization

**Ý chính**  
- Framework RPC + serialization của Apache.  
- Có IDL (interface definition language) giống Protobuf.  
- Ít phổ biến hơn Avro/Protobuf trong hệ Kafka hiện đại, nhưng vẫn dùng trong một số hệ thống cũ.

**File Thrift**

```thrift
namespace java com.example.thrift

struct UserThrift {
  1: string id
  2: string name
  3: i32 age
}
```

Sau khi generate class `UserThrift`, cần tự viết serializer/deserializer hoặc dùng thư viện bên thứ ba.

**Ví dụ custom serializer (rút gọn)**

```java
public class ThriftSerializer<T extends TBase> implements Serializer<T> {
    @Override
    public byte[] serialize(String topic, T data) {
        try {
            TSerializer serializer = new TSerializer(new TBinaryProtocol.Factory());
            return serializer.serialize(data);
        } catch (TException e) {
            throw new SerializationException(e);
        }
    }
}
```

**Config**

```yaml
spring:
  kafka:
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: com.example.kafka.ThriftSerializer
```

Consumer tương tự với `ThriftDeserializer`.

---

## So sánh nhanh

- **String**: đơn giản, chỉ text, không schema.  
- **JSON**: dễ đọc, linh hoạt, phù hợp cho REST + Kafka, không schema chặt.  
- **Avro**: có schema, hỗ trợ schema evolution, tích hợp tốt với Kafka + Schema Registry.  
- **Protobuf**: nhị phân, hiệu năng cao, đa ngôn ngữ, cũng có schema.  
- **Thrift**: tương tự Protobuf, nhưng ít phổ biến hơn trong Kafka mới.  

Tùy yêu cầu:  
- Hệ thống nhỏ, dễ debug → String/JSON.  
- Hệ thống lớn, cần schema, hiệu năng → Avro/Protobuf.
