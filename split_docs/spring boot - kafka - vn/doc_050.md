
Dưới đây là tóm tắt ngắn gọn về **độ tin cậy Consumer trong Kafka** với chủ đề **poison messages, DLQ, retries**, kèm ví dụ Spring Boot.

---

## 1. Khái niệm chính

1. **Poison message**  
   - Là message mà consumer **luôn luôn xử lý lỗi**, dù retry bao nhiêu lần cũng không thành công (ví dụ: dữ liệu sai format, thiếu field bắt buộc, logic không hợp lệ).
   - Nếu không xử lý đúng, consumer sẽ **kẹt** ở offset đó, không tiến được.

2. **Mục tiêu**  
   - Không để 1 message xấu làm **chặn cả consumer group**.
   - Vẫn **không mất dữ liệu**: message lỗi được chuyển sang nơi khác (DLQ) để phân tích / xử lý thủ công.

3. **Chiến lược chính**  
   - **Retry**: thử xử lý lại message vài lần (in-memory hoặc gửi sang topic retry).
   - **DLQ (Dead Letter Queue)**: nếu vẫn lỗi sau số lần retry tối đa → gửi sang topic DLQ, commit offset để consumer tiếp tục.

---

## 2. Mô hình phổ biến

### 2.1. In-memory retry + DLQ

- Consumer:
  - Nhận message → xử lý.
  - Nếu lỗi → retry trong code (ví dụ 3 lần).
  - Nếu vẫn lỗi → gửi message sang **DLQ topic** + commit offset.

**Ưu điểm**: đơn giản, không cần thêm topic retry.  
**Nhược điểm**: nếu xử lý lâu, có thể ảnh hưởng throughput.

## 3. Ví dụ với Spring Boot + Spring Kafka

### 3.1. Cấu hình Kafka Consumer với Error Handler & DLQ

```java
@Configuration
@EnableKafka
public class KafkaConfig {

    @Bean
    public ConsumerFactory<String, String> consumerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "demo-group");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.AUTO_OFFSET_RESET_CONFIG, "earliest");
        return new DefaultKafkaConsumerFactory<>(props);
    }

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate(ProducerFactory<String, String> pf) {
        return new KafkaTemplate<>(pf);
    }

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory(
            ConsumerFactory<String, String> consumerFactory,
            KafkaTemplate<String, String> kafkaTemplate) {

        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory);

        // Retry 3 lần, sau đó gửi vào DLQ
        DeadLetterPublishingRecoverer recoverer =
                new DeadLetterPublishingRecoverer(kafkaTemplate,
                        (record, ex) -> new TopicPartition("my-topic.DLQ", record.partition()));

        // 3 lần retry, delay 1s giữa mỗi lần
        SeekToCurrentErrorHandler errorHandler =
                new SeekToCurrentErrorHandler(recoverer, 3); // 3 retries

        factory.setErrorHandler(errorHandler);
        return factory;
    }
}
```

### 3.2. Consumer xử lý message

```java
@Service
public class MyKafkaConsumer {

    @KafkaListener(topics = "my-topic", groupId = "demo-group")
    public void consume(String message) {
        // Ví dụ: message phải là số nguyên dương
        int value;
        try {
            value = Integer.parseInt(message);
        } catch (NumberFormatException e) {
            // Đây là poison message (sai format)
            // Ném exception để errorHandler xử lý (retry + DLQ)
            throw new IllegalArgumentException("Invalid number format: " + message, e);
        }

        if (value < 0) {
            // Logic không hợp lệ → cũng coi là poison
            throw new IllegalStateException("Negative value not allowed: " + value);
        }

        // Xử lý bình thường
        System.out.println("Processed value: " + value);
    }
}
```

### 3.3. Producer gửi message (để test)

```java
@RestController
@RequestMapping("/produce")
public class ProducerController {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public ProducerController(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    @PostMapping
    public String send(@RequestParam String msg) {
        kafkaTemplate.send("my-topic", msg);
        return "Sent: " + msg;
    }
}
```

## 4. Ví dụ Input / Output

### 4.1. Input hợp lệ

- HTTP request:
  - `POST /produce?msg=10`
- Kafka message vào topic `my-topic`: `"10"`

**Kết quả:**

- Consumer log:
  - `Processed value: 10`
- Không có message vào DLQ.

---

### 4.2. Poison message: sai format

- HTTP request:
  - `POST /produce?msg=abc`
- Kafka message: `"abc"`

**Luồng xử lý:**

1. Consumer nhận `"abc"`.
2. `Integer.parseInt("abc")` → `NumberFormatException`.
3. Spring Kafka:
   - Retry xử lý message `"abc"` **3 lần**.
   - Lần nào cũng lỗi.
4. Sau 3 lần:
   - Gửi message `"abc"` sang topic `my-topic.DLQ`.
   - Commit offset của `"abc"` → consumer tiếp tục với message tiếp theo.

**Kết quả:**

- Topic `my-topic.DLQ` chứa message `"abc"` + metadata (topic gốc, partition, offset, exception).
- Consumer chính không bị kẹt.

---

### 4.3. Poison message: logic không hợp lệ

- HTTP request:
  - `POST /produce?msg=-5`
- Kafka message: `"-5"`

**Luồng xử lý:**

1. Parse thành `-5` thành công.
2. Check `value < 0` → ném `IllegalStateException`.
3. Retry 3 lần, vẫn lỗi.
4. Gửi `"-5"` sang `my-topic.DLQ`, commit offset.

---

## 5. Ý chính cần nhớ

- **Poison message**: message luôn gây lỗi, không thể xử lý thành công.
- **Retry**:
  - Có thể retry trong memory (error handler) hoặc qua topic retry.
  - Cần giới hạn số lần retry để tránh loop vô hạn.
- **DLQ**:
  - Topic riêng để chứa message lỗi sau khi retry thất bại.
  - Giúp không mất dữ liệu và không chặn consumer.
- **Spring Kafka** hỗ trợ:
  - `DeadLetterPublishingRecoverer` để publish vào DLQ.
  - `SeekToCurrentErrorHandler` (hoặc `DefaultErrorHandler` ở version mới) để cấu hình retry + DLQ.
