
Dưới đây là tóm tắt ngắn gọn về **sticky partitioner** trong Kafka (từ Kafka 2.4+) và lợi ích về **batching**, kèm ví dụ code và input/output.

---

## 1. Sticky Partitioner là gì?

- Là chiến lược partition mặc định mới (từ Kafka 2.4) khi **producer không chỉ định key**.
- Thay vì chọn partition ngẫu nhiên cho từng record, sticky partitioner:
  - Chọn **một partition “dính”** (sticky) cho một loạt record.
  - Gửi nhiều record liên tiếp vào cùng partition cho đến khi:
    - Batch đầy, hoặc
    - Hết thời gian linger.ms, hoặc
    - Có lỗi / partition không còn khả dụng.
  - Sau đó mới chọn partition khác.

**Mục tiêu:** Tăng hiệu quả batching → giảm số request → tăng throughput.

---

## 2. Lợi ích về batching

- Nhiều record cùng partition → Kafka producer có thể:
  - Gom nhiều record vào **một batch**.
  - Gửi **ít request hơn** đến broker.
- Kết quả:
  - Throughput cao hơn.
  - Giảm overhead mạng.
  - Độ trễ trung bình tốt hơn trong nhiều trường hợp.

---

## 3. Khi nào sticky partitioner hoạt động?

- Khi **không có key**:
  - Sticky partitioner chọn partition “dính”.
- Khi **có key**:
  - Partition vẫn được xác định bằng `hash(key) % numPartitions` như cũ.
  - Sticky partitioner không thay đổi hành vi này.

## 4. Ví dụ Kafka Producer (Java)

### Cấu hình producer sử dụng sticky partitioner (mặc định từ Kafka 2.4+)

```java
Properties props = new Properties();
props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG,
        "org.apache.kafka.common.serialization.StringSerializer");
props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,
        "org.apache.kafka.common.serialization.StringSerializer");

// Tối ưu batching
props.put(ProducerConfig.BATCH_SIZE_CONFIG, 32_768);   // 32 KB
props.put(ProducerConfig.LINGER_MS_CONFIG, 10);        // chờ tối đa 10ms để gom batch

KafkaProducer<String, String> producer = new KafkaProducer<>(props);

String topic = "demo-topic";

// Gửi 10 message không có key -> sticky partitioner sẽ “dính” vào 1 partition
for (int i = 0; i < 10; i++) {
    ProducerRecord<String, String> record =
            new ProducerRecord<>(topic, null, "message-" + i);
    producer.send(record, (metadata, exception) -> {
        if (exception == null) {
            System.out.printf("Sent: %s to partition %d, offset %d%n",
                    record.value(), metadata.partition(), metadata.offset());
        } else {
            exception.printStackTrace();
        }
    });
}

producer.flush();
producer.close();
```

### Ví dụ output (minh họa)

```text
Sent: message-0 to partition 2, offset 100
Sent: message-1 to partition 2, offset 101
Sent: message-2 to partition 2, offset 102
Sent: message-3 to partition 2, offset 103
...
Sent: message-9 to partition 2, offset 109
```

- Tất cả message không có key được gửi vào **cùng partition 2** → batching tốt.

## 5. Ví dụ Spring Boot + Spring Kafka

### application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      properties:
        linger.ms: 10
        batch.size: 32768
```

### Producer Service

```java
@Service
public class KafkaProducerService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public KafkaProducerService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendWithoutKey(String topic, String message) {
        // Không có key -> sticky partitioner hoạt động
        kafkaTemplate.send(topic, message)
                     .addCallback(
                         result -> {
                             if (result != null) {
                                 var meta = result.getRecordMetadata();
                                 System.out.printf("Sent: %s to partition %d, offset %d%n",
                                         message, meta.partition(), meta.offset());
                             }
                         },
                         ex -> ex.printStackTrace()
                     );
    }

    public void sendWithKey(String topic, String key, String message) {
        // Có key -> partition dựa trên hash(key)
        kafkaTemplate.send(topic, key, message);
    }
}
```

### Input / Output minh họa

**Input (gọi service):**

```java
producerService.sendWithoutKey("demo-topic", "hello-1");
producerService.sendWithoutKey("demo-topic", "hello-2");
producerService.sendWithoutKey("demo-topic", "hello-3");
```

**Output (log):**

```text
Sent: hello-1 to partition 1, offset 200
Sent: hello-2 to partition 1, offset 201
Sent: hello-3 to partition 1, offset 202
```

---

## 6. Tóm tắt ngắn gọn (tiếng Việt)

- **Sticky partitioner** (Kafka 2.4+):
  - Dùng khi **không có key**.
  - Gửi nhiều record liên tiếp vào **cùng partition**.
- **Lợi ích**:
  - Tối ưu **batching**.
  - Tăng throughput, giảm số request.
- **Có key** → hành vi partition như cũ (dựa trên hash key).
- Trong Spring Boot, chỉ cần cấu hình producer bình thường; sticky partitioner là mặc định (nếu dùng client mới), sau đó tối ưu thêm `batch.size`, `linger.ms` để tận dụng batching.
