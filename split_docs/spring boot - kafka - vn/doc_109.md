
Dưới đây là tóm tắt ngắn gọn về “Processing patterns: Stream processing vs simple consumers” trong Kafka + Spring Boot, kèm ví dụ code và input/output.

---

## 1. Khái niệm chính

### Simple Consumer (Tiêu thụ đơn giản)
- Đọc message từ topic, xử lý từng bản ghi, thường là **stateless** (không giữ trạng thái lâu dài).
- Thường dùng cho:
  - Ghi log, lưu DB, gửi email, gọi API khác.
  - Xử lý đơn giản, không cần join, window, aggregation phức tạp.
- Dễ triển khai, dễ debug, phù hợp cho các use case nhỏ hoặc tách biệt.

### Stream Processing (Xử lý luồng)
- Dùng **Kafka Streams** hoặc **Spring Cloud Stream** để xử lý dữ liệu như một luồng liên tục.
- Hỗ trợ:
  - Stateful processing (giữ trạng thái, store cục bộ).
  - Windowing (tính toán theo cửa sổ thời gian).
  - Aggregation (sum, count, group by).
  - Join nhiều topic.
- Phù hợp cho:
  - Real-time analytics.
  - Enrichment dữ liệu.
  - Tính toán phức tạp, pipeline nhiều bước.

---

## 2. Khi nào dùng Simple Consumer?

- Logic xử lý đơn giản: đọc → xử lý → lưu.
- Không cần:
  - Join nhiều stream.
  - Window theo thời gian.
  - Aggregation phức tạp.
- Muốn kiểm soát chi tiết việc commit offset, retry, error handling.

### Ví dụ: Simple Consumer với Spring Kafka

**Input (Kafka topic `orders`):**
```json
{"orderId": 1, "userId": 10, "amount": 100.0}
{"orderId": 2, "userId": 11, "amount": 50.0}
```

**Consumer:**
```java
// build.gradle (phần chính)
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter'
    implementation 'org.springframework.kafka:spring-kafka'
}

@Configuration
@EnableKafka
public class KafkaConfig {

    @Bean
    public Map<String, Object> consumerConfigs() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "order-consumer-group");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG,
                  StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG,
                  StringDeserializer.class);
        return props;
    }

    @Bean
    public ConsumerFactory<String, String> consumerFactory() {
        return new DefaultKafkaConsumerFactory<>(consumerConfigs());
    }

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String>
    kafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory());
        return factory;
    }
}

@Service
public class OrderConsumer {

    @KafkaListener(topics = "orders", groupId = "order-consumer-group")
    public void consume(String message) {
        System.out.println("Received order: " + message);
        // parse JSON, lưu DB, gọi service khác...
    }
}
```

**Output (log):**
```text
Received order: {"orderId": 1, "userId": 10, "amount": 100.0}
Received order: {"orderId": 2, "userId": 11, "amount": 50.0}
```

## 3. Khi nào dùng Stream Processing?

- Cần:
  - Tính toán theo thời gian (windowed count, sum).
  - Join nhiều topic (ví dụ: orders + users).
  - Aggregation real-time (thống kê, dashboard).
  - Stateful processing (giữ trạng thái giữa các message).

### Ví dụ: Kafka Streams với Spring Boot

**Mục tiêu:** Tính tổng `amount` theo `userId` từ topic `orders` và ghi ra topic `user-order-totals`.

**Input (topic `orders`):**
```json
{"orderId": 1, "userId": "u1", "amount": 100.0}
{"orderId": 2, "userId": "u1", "amount": 50.0}
{"orderId": 3, "userId": "u2", "amount": 200.0}
```

**Output (topic `user-order-totals`):**
```json
{"userId": "u1", "totalAmount": 150.0}
{"userId": "u2", "totalAmount": 200.0}
```

**build.gradle (phần chính):**
```groovy
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter'
    implementation 'org.springframework.kafka:spring-kafka'
    implementation 'org.apache.kafka:kafka-streams'
}
```

**Cấu hình Streams:**
```java
@Configuration
@EnableKafkaStreams
public class KafkaStreamsConfig {

    @Bean(name = KafkaStreamsDefaultConfiguration.DEFAULT_STREAMS_CONFIG_BEAN_NAME)
    public KafkaStreamsConfiguration kStreamsConfigs() {
        Map<String, Object> props = new HashMap<>();
        props.put(StreamsConfig.APPLICATION_ID_CONFIG, "order-streams-app");
        props.put(StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(StreamsConfig.DEFAULT_KEY_SERDE_CLASS_CONFIG,
                  Serdes.String().getClass());
        props.put(StreamsConfig.DEFAULT_VALUE_SERDE_CLASS_CONFIG,
                  Serdes.String().getClass());
        return new KafkaStreamsConfiguration(props);
    }

    @Bean
    public KStream<String, String> kStream(StreamsBuilder builder) {
        KStream<String, String> orders = builder.stream("orders");

        KTable<String, Double> totals = orders
            .map((key, value) -> {
                // parse JSON đơn giản (thực tế nên dùng ObjectMapper)
                JSONObject json = new JSONObject(value);
                String userId = json.getString("userId");
                Double amount = json.getDouble("amount");
                return KeyValue.pair(userId, amount);
            })
            .groupByKey()
            .reduce(Double::sum); // cộng dồn amount theo userId

        totals.toStream()
              .mapValues((readOnlyKey, total) -> {
                  JSONObject out = new JSONObject();
                  out.put("userId", readOnlyKey);
                  out.put("totalAmount", total);
                  return out.toString();
              })
              .to("user-order-totals", Produced.with(Serdes.String(), Serdes.String()));

        return orders;
    }
}
```

## 4. So sánh nhanh

| Tiêu chí                 | Simple Consumer                         | Stream Processing                          |
|--------------------------|------------------------------------------|--------------------------------------------|
| Độ phức tạp              | Thấp                                    | Cao hơn                                    |
| Trạng thái (state)       | Thường stateless                        | Stateful (state store, window, join)       |
| Use case                 | Xử lý đơn giản, event-driven service    | Analytics real-time, aggregation, pipeline |
| Thư viện chính           | Spring Kafka                            | Kafka Streams / Spring Cloud Stream        |
| Khả năng mở rộng logic   | Hạn chế                                  | Rất tốt cho xử lý luồng phức tạp           |
