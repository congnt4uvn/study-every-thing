
Dưới đây là tóm tắt ngắn gọn về mô hình Messaging: CDC (Change Data Capture) với Debezium + Kafka, kèm ví dụ Spring Boot.

---

## 1. CDC (Change Data Capture) là gì?

- CDC là kỹ thuật **bắt và phát hiện các thay đổi** (INSERT/UPDATE/DELETE) trên database.
- Thay vì ứng dụng tự query DB liên tục, CDC đọc **binlog/redo log** của DB và phát sự kiện ra ngoài.
- Lợi ích:
  - Giảm tải cho DB (không cần polling).
  - Dữ liệu thay đổi được đẩy **gần real-time**.
  - Dễ xây dựng kiến trúc **event-driven**.

---

## 2. Debezium + Kafka trong CDC

- **Debezium** là bộ connector CDC chạy trên **Kafka Connect**.
- Debezium:
  - Kết nối tới DB (MySQL, PostgreSQL, SQL Server, v.v.).
  - Đọc log thay đổi (binlog/redo log).
  - Gửi các thay đổi thành **Kafka messages** (record) vào các topic.
- **Kafka**:
  - Lưu trữ các sự kiện thay đổi.
  - Cho phép nhiều consumer (microservice) subscribe và xử lý.

Luồng tổng quát:

1. DB thay đổi (INSERT/UPDATE/DELETE).
2. Debezium đọc log DB.
3. Debezium gửi event vào Kafka topic (ví dụ: `dbserver1.inventory.customers`).
4. Spring Boot service (Kafka consumer) đọc event và xử lý (sync cache, search index, gửi email, v.v.).

---

## 3. Cấu trúc message Debezium gửi vào Kafka

Một record Debezium thường có dạng (rút gọn):

```json
{
  "before": {
    "id": 1,
    "name": "Old Name"
  },
  "after": {
    "id": 1,
    "name": "New Name"
  },
  "op": "u",          // c: create, u: update, d: delete, r: snapshot
  "ts_ms": 1710000000000,
  "source": {
    "db": "inventory",
    "table": "customers"
  }
}
```

- `before`: trạng thái trước khi thay đổi.
- `after`: trạng thái sau khi thay đổi.
- `op`: loại thao tác (create/update/delete).
- `source`: thông tin nguồn (DB, table, v.v.).

## 5. Spring Boot + Kafka: Consumer đọc CDC events

### 5.1. Thêm dependency

```xml
<!-- pom.xml -->
<dependencies>
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>
    <dependency>
        <groupId>com.fasterxml.jackson.core</groupId>
        <artifactId>jackson-databind</artifactId>
    </dependency>
</dependencies>
```

### 5.2. Cấu hình Kafka consumer

```yaml
# application.yml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    consumer:
      group-id: cdc-consumer-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      auto-offset-reset: earliest
```

### 5.3. DTO cho message Debezium (rút gọn)

```java
// DebeziumChangeEvent.java
public class DebeziumChangeEvent<T> {
    private T before;
    private T after;
    private String op;
    private Map<String, Object> source;
    private Long ts_ms;

    // getters/setters
}

// Customer.java
public class Customer {
    private Integer id;
    private String name;
    private String email;
    // getters/setters
}
```

### 5.4. Consumer lắng nghe topic Debezium

```java
// CdcConsumer.java
import com.fasterxml.jackson.databind.JsonNode;
import com.fasterxml.jackson.databind.ObjectMapper;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class CdcConsumer {

    private final ObjectMapper objectMapper = new ObjectMapper();

    @KafkaListener(topics = "dbserver1.inventory.customers")
    public void consume(String message) throws Exception {
        JsonNode root = objectMapper.readTree(message);

        JsonNode payload = root.get("payload"); // nếu dùng Debezium + Kafka Connect default
        if (payload == null) {
            payload = root; // trường hợp đã unwrap
        }

        DebeziumChangeEvent<Customer> event =
                objectMapper.treeToValue(payload, objectMapper.getTypeFactory()
                        .constructParametricType(DebeziumChangeEvent.class, Customer.class));

        String op = event.getOp();
        Customer before = event.getBefore();
        Customer after = event.getAfter();

        switch (op) {
            case "c": // create
            case "r": // snapshot
                handleCreate(after);
                break;
            case "u": // update
                handleUpdate(before, after);
                break;
            case "d": // delete
                handleDelete(before);
                break;
            default:
                // ignore
        }
    }

    private void handleCreate(Customer customer) {
        System.out.println("CREATE: " + customer.getId() + " - " + customer.getName());
        // ví dụ: lưu vào cache, index search, gửi email, ...
    }

    private void handleUpdate(Customer before, Customer after) {
        System.out.println("UPDATE: " + before.getId() + " from " +
                before.getName() + " to " + after.getName());
    }

    private void handleDelete(Customer customer) {
        System.out.println("DELETE: " + customer.getId());
    }
}
```

## 6. Ví dụ Input/Output

### 6.1. Thao tác trên DB

```sql
INSERT INTO customers (id, name, email)
VALUES (1, 'Alice', 'alice@example.com');

UPDATE customers
SET name = 'Alice Nguyen'
WHERE id = 1;

DELETE FROM customers
WHERE id = 1;
```

### 6.2. Message Debezium gửi vào Kafka (rút gọn)

**INSERT (op = "c")**

```json
{
  "before": null,
  "after": {
    "id": 1,
    "name": "Alice",
    "email": "alice@example.com"
  },
  "op": "c"
}
```

**UPDATE (op = "u")**

```json
{
  "before": {
    "id": 1,
    "name": "Alice",
    "email": "alice@example.com"
  },
  "after": {
    "id": 1,
    "name": "Alice Nguyen",
    "email": "alice@example.com"
  },
  "op": "u"
}
```

**DELETE (op = "d")**

```json
{
  "before": {
    "id": 1,
    "name": "Alice Nguyen",
    "email": "alice@example.com"
  },
  "after": null,
  "op": "d"
}
```

### 6.3. Output log từ Spring Boot consumer

```text
CREATE: 1 - Alice
UPDATE: 1 from Alice to Alice Nguyen
DELETE: 1
```

## 7. Ý chính cần nhớ

- CDC + Debezium + Kafka cho phép:
  - Đồng bộ dữ liệu giữa các hệ thống theo **event**.
  - Xây dựng microservice **event-driven** mà không cần ứng dụng chạm trực tiếp vào DB của nhau.
- Debezium:
  - Đọc log DB, phát sự kiện thay đổi vào Kafka.
- Spring Boot:
  - Dùng `@KafkaListener` để consume các sự kiện CDC.
  - Xử lý theo `op` (c/u/d) để cập nhật cache, search index, data warehouse, v.v.

Nếu bạn muốn, tôi có thể thêm ví dụ cấu hình Docker Compose (Kafka + Zookeeper + Debezium + MySQL) để chạy end-to-end.
