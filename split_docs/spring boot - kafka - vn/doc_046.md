
Dưới đây là tóm tắt ngắn gọn về quản lý offset và commit offset vào Kafka (`__consumer_offsets`), kèm ví dụ Spring Boot.

---

## 1. Khái niệm chính

- **Offset**: Vị trí (số thứ tự) của message trong mỗi partition.
- **Commit offset**: Lưu offset đã xử lý xong vào Kafka (topic nội bộ `__consumer_offsets`) để:
  - Khi consumer restart, có thể tiếp tục đọc từ đúng vị trí.
  - Tránh xử lý trùng hoặc mất message (tùy chiến lược).

- **Topic `__consumer_offsets`**:
  - Là topic nội bộ do Kafka quản lý.
  - Lưu offset cho từng cặp `(groupId, topic, partition)`.
  - Consumer không đọc/ghi trực tiếp; Kafka client làm việc này thông qua API commit.

---

## 2. Auto-commit vs Manual commit

### Auto-commit (mặc định)

- Thuộc tính: `enable.auto.commit=true`
- Kafka client tự động commit offset theo chu kỳ (`auto.commit.interval.ms`, mặc định 5000ms).
- Ưu điểm: Đơn giản.
- Nhược điểm: Có thể commit offset trước khi xử lý xong → nguy cơ mất message nếu consumer crash sau commit.

**Cấu hình Spring Boot (application.yml)**

```yaml
spring:
  kafka:
    consumer:
      bootstrap-servers: localhost:9092
      group-id: my-group
      enable-auto-commit: true
      auto-offset-reset: earliest
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
```

---

### Manual commit (chủ động)

- `enable.auto.commit=false`
- Bạn tự quyết định khi nào commit offset (sau khi xử lý thành công).
- Giảm nguy cơ mất message, dễ xây dựng logic “at-least-once”.

**Cấu hình Spring Boot**

```yaml
spring:
  kafka:
    consumer:
      bootstrap-servers: localhost:9092
      group-id: my-group
      enable-auto-commit: false
      auto-offset-reset: earliest
```

## 3. Ví dụ với Spring Kafka

### 3.1. Manual commit sync

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.kafka.support.Acknowledgment;
import org.springframework.stereotype.Service;

@Service
public class MyKafkaConsumer {

    @KafkaListener(topics = "my-topic", groupId = "my-group")
    public void listen(ConsumerRecord<String, String> record,
                       Acknowledgment ack) {
        // Input: message từ Kafka
        String key = record.key();
        String value = record.value();
        long offset = record.offset();
        int partition = record.partition();

        // Xử lý business logic
        System.out.printf("Received message: key=%s, value=%s, partition=%d, offset=%d%n",
                key, value, partition, offset);

        // Nếu xử lý thành công → commit offset
        ack.acknowledge();
        // Output: offset mới được commit vào __consumer_offsets
    }
}
```

**Luồng Input/Output:**

- Input:
  - Topic: `my-topic`
  - Message: `key="user1"`, `value="{\"action\":\"login\"}"`
- Xử lý:
  - In ra log, cập nhật DB, v.v.
- Output:
  - Offset của message này được commit vào `__consumer_offsets` cho group `my-group`.
  - Khi consumer restart, nó sẽ đọc từ offset tiếp theo.

---

### 3.2. Manual commit async (Java Kafka Consumer thuần)

```java
consumer.subscribe(Collections.singletonList("my-topic"));

while (true) {
    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
    for (ConsumerRecord<String, String> record : records) {
        // Xử lý message
        System.out.println("Processing: " + record.value());
    }

    // Commit async
    consumer.commitAsync((offsets, exception) -> {
        if (exception != null) {
            System.err.println("Commit failed: " + exception.getMessage());
        } else {
            System.out.println("Offsets committed: " + offsets);
        }
    });
}
```

## 4. Chiến lược xử lý & commit

- **At-most-once**:
  - Commit trước, xử lý sau → có thể mất message nếu crash sau commit.
- **At-least-once** (phổ biến):
  - Xử lý xong, commit sau → có thể xử lý trùng nếu crash trước commit.
- **Exactly-once**:
  - Cần thêm cơ chế transactional (Kafka + DB) hoặc idempotent processing.

---

## 5. Tóm tắt

- Offset được lưu trong topic nội bộ `__consumer_offsets`.
- Auto-commit: đơn giản nhưng ít kiểm soát.
- Manual commit: kiểm soát tốt hơn, phù hợp cho hệ thống cần đảm bảo xử lý tin cậy.
- Trong Spring Boot/Spring Kafka:
  - Tắt auto-commit và dùng `Acknowledgment` để commit offset sau khi xử lý thành công.
