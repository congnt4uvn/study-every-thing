## 1. Ý tưởng chính: Append-only log trong Kafka

- **Kafka topic** được lưu trên đĩa dưới dạng **append-only log** (chỉ ghi thêm, không sửa giữa chừng).
- Log được chia thành nhiều **segment file** (phân đoạn).
- Mỗi segment có:
  - **Data file**: chứa các bản ghi (record/message) nối tiếp nhau.
  - **Index file**: chứa mapping từ **offset** → **vị trí byte** trong data file.
- Kafka tận dụng **OS page cache**:
  - Dữ liệu được đọc/ghi chủ yếu thông qua page cache, giảm số lần truy cập đĩa vật lý.
  - Nhờ append-only, pattern I/O chủ yếu là **sequential write**, rất hiệu quả.

---

## 2. Segment files

- Mỗi partition của topic là một chuỗi các segment file:
  - Ví dụ: `00000000000000000000.log`, `00000000000000001000.log`, ...
- Khi segment hiện tại đạt đến:
  - **kích thước tối đa** (log.segment.bytes) hoặc
  - **thời gian tối đa** (log.roll.ms),
  → Kafka tạo segment mới.
- Việc xóa dữ liệu cũ (retention) thường là xóa **cả segment** thay vì xóa từng record.

---

## 3. Index files

- Mỗi segment có một **index file**:
  - Ví dụ: `00000000000000000000.index`.
- Index chứa các entry: `(relativeOffset, positionInLogFile)`.
- Khi consumer yêu cầu đọc từ một offset:
  - Kafka dùng index để **nhảy nhanh** đến vị trí byte tương ứng trong data file.
  - Sau đó đọc tuần tự các record tiếp theo.

---

## 4. Page cache

- Kafka không tự quản lý cache phức tạp, mà dựa vào **OS page cache**:
  - Khi ghi: dữ liệu được ghi vào page cache, OS flush xuống đĩa.
  - Khi đọc: nếu dữ liệu còn trong cache, đọc rất nhanh (không cần I/O đĩa).
- Append-only log giúp:
  - Ghi tuần tự → OS dễ tối ưu.
  - Đọc gần thời gian thực (near real-time) → dữ liệu thường vẫn nằm trong cache.

## 5. Ví dụ Spring Boot + Kafka

### 5.1. Cấu hình Maven

```xml
<dependencies>
    <!-- Spring Kafka -->
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>
    <!-- Web (nếu cần REST API) -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
</dependencies>
```

### 5.2. application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
    consumer:
      group-id: demo-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      auto-offset-reset: earliest
```

## 6. Producer: gửi message vào log

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/messages")
public class MessageController {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private static final String TOPIC = "demo-topic";

    public MessageController(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    @PostMapping
    public String sendMessage(@RequestBody String message) {
        kafkaTemplate.send(TOPIC, message);
        return "Sent: " + message;
    }
}
```

### Input/Output ví dụ

- **HTTP Request (input)**:
  - `POST /api/messages`
  - Body: `"Xin chao Kafka"`

- **Kafka side (append-only log)**:
  - Bản ghi `"Xin chao Kafka"` được **append** vào segment file hiện tại của `demo-topic` (partition cụ thể).
  - Index file được cập nhật với offset mới.

- **HTTP Response (output)**:
  - `"Sent: Xin chao Kafka"`

## 7. Consumer: đọc từ log theo offset

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class MessageConsumer {

    @KafkaListener(topics = "demo-topic", groupId = "demo-group")
    public void listen(String message) {
        System.out.println("Received: " + message);
    }
}
```

### Input/Output ví dụ

- **Input**: các message đã được ghi vào log của `demo-topic`.
- **Cơ chế bên trong Kafka**:
  - Consumer yêu cầu đọc từ offset X.
  - Broker dùng **index file** để tìm vị trí byte trong segment.
  - Đọc record từ data file (thường qua page cache).
- **Output (console)**:
  - `Received: Xin chao Kafka`
  - `Received: Message 2`
  - ...

---

## 8. Liên hệ giữa code và cơ chế log

- Mỗi lần `kafkaTemplate.send(...)`:
  - Kafka broker **append** message vào log (segment file).
  - Cập nhật index file.
  - OS page cache giữ dữ liệu nóng để đọc nhanh.
- Mỗi lần `@KafkaListener` nhận message:
  - Consumer đọc tuần tự từ log theo offset.
  - Nhờ index + page cache, việc đọc rất hiệu quả, dù dữ liệu nằm trên đĩa.

---
