## 1. Gửi message async với Spring Kafka

Spring Kafka dùng `KafkaTemplate` để gửi message.  
Gửi async trả về `ListenableFuture` / `CompletableFuture` để xử lý kết quả.

### Ví dụ cấu hình `KafkaTemplate`

```java
@Configuration
public class KafkaProducerConfig {

    @Bean
    public ProducerFactory<String, String> producerFactory() {
        Map<String, Object> config = new HashMap<>();
        config.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        config.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        config.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        // Retry ở level Kafka producer
        config.put(ProducerConfig.RETRIES_CONFIG, 3);
        config.put(ProducerConfig.ACKS_CONFIG, "all");
        return new DefaultKafkaProducerFactory<>(config);
    }

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate() {
        return new KafkaTemplate<>(producerFactory());
    }
}
```

## 2. Gửi async với callback

### Ví dụ gửi async + callback

```java
@Service
public class KafkaProducerService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public KafkaProducerService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessageAsync(String topic, String key, String value) {
        ListenableFuture<SendResult<String, String>> future =
                kafkaTemplate.send(topic, key, value);

        future.addCallback(
            result -> {
                // Callback khi gửi thành công
                RecordMetadata meta = result.getRecordMetadata();
                System.out.printf("Sent OK: topic=%s partition=%d offset=%d%n",
                        meta.topic(), meta.partition(), meta.offset());
            },
            ex -> {
                // Callback khi gửi thất bại
                System.err.println("Send FAILED: " + ex.getMessage());
                // Có thể log, gửi alert, hoặc trigger retry logic
            }
        );
    }
}
```

### Input/Output ví dụ

- Input:
  ```java
  producerService.sendMessageAsync("orders", "order-123", "{ \"id\": 123, \"status\": \"NEW\" }");
  ```

- Output (log):
  - Thành công:
    ```
    Sent OK: topic=orders partition=0 offset=15
    ```
  - Thất bại:
    ```
    Send FAILED: TimeoutException: Failed to update metadata after 60000 ms.
    ```

## 3. Xử lý lỗi & retry ở tầng ứng dụng

Ngoài retry của Kafka producer (`retries`), bạn có thể dùng `RetryTemplate` của Spring để retry logic gửi.

### Ví dụ dùng `RetryTemplate`

```java
@Configuration
public class RetryConfig {

    @Bean
    public RetryTemplate retryTemplate() {
        RetryTemplate retryTemplate = new RetryTemplate();

        SimpleRetryPolicy retryPolicy = new SimpleRetryPolicy();
        retryPolicy.setMaxAttempts(3); // số lần thử

        FixedBackOffPolicy backOffPolicy = new FixedBackOffPolicy();
        backOffPolicy.setBackOffPeriod(2000); // 2s giữa mỗi lần retry

        retryTemplate.setRetryPolicy(retryPolicy);
        retryTemplate.setBackOffPolicy(backOffPolicy);

        return retryTemplate;
    }
}
```

```java
@Service
public class ReliableKafkaProducerService {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private final RetryTemplate retryTemplate;

    public ReliableKafkaProducerService(KafkaTemplate<String, String> kafkaTemplate,
                                        RetryTemplate retryTemplate) {
        this.kafkaTemplate = kafkaTemplate;
        this.retryTemplate = retryTemplate;
    }

    public void sendWithRetry(String topic, String key, String value) {
        retryTemplate.execute(context -> {
            ListenableFuture<SendResult<String, String>> future =
                    kafkaTemplate.send(topic, key, value);

            // Chờ kết quả để biết thành công/thất bại
            SendResult<String, String> result = future.get(); // có thể ném exception

            System.out.println("Sent OK with retry: " + result.getRecordMetadata());
            return null;
        }, context -> {
            // Recovery callback: sau khi retry hết mà vẫn fail
            System.err.println("Send FAILED after retries. Last error: "
                    + context.getLastThrowable().getMessage());
            // Có thể lưu DB, gửi vào dead-letter topic, v.v.
            return null;
        });
    }
}
```

### Input/Output ví dụ

- Input:
  ```java
  reliableKafkaProducerService.sendWithRetry("payments", "pay-1", "{ \"amount\": 1000 }");
  ```

- Output (log, giả sử lần 1 fail, lần 2 OK):
  ```
  Send FAILED attempt 1: TimeoutException...
  (đợi 2s)
  Sent OK with retry: payments-0@offset 42
  ```

## 4. Tóm tắt ý chính

1. **Gửi async**: dùng `KafkaTemplate.send(...)` → trả về `ListenableFuture`/`CompletableFuture`.
2. **Callback**:
   - `onSuccess`: log topic/partition/offset, phục vụ tracking.
   - `onFailure`: log lỗi, trigger alert, hoặc custom retry.
3. **Xử lý lỗi**:
   - Ở level Kafka producer: `retries`, `acks`, `delivery.timeout.ms`, v.v.
   - Ở level ứng dụng: dùng `RetryTemplate` hoặc thư viện retry khác.
4. **Retry**:
   - Có thể kết hợp retry của Kafka + retry của ứng dụng.
   - Sau khi retry hết mà vẫn fail → recovery: lưu DB, gửi dead-letter topic, thông báo hệ thống khác.

Nếu bạn gửi nội dung chi tiết hơn (code/config hiện tại), tôi có thể chỉnh lại ví dụ cho sát với project của bạn.
