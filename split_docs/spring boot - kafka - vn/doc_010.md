
Dưới đây là tóm tắt ngắn gọn về luồng dữ liệu trong Kafka, cách consumer đọc từ offset và cơ chế zero-copy (sendfile), kèm ví dụ code Spring Boot.

---

## 1. Data flow & Fetching trong Kafka

- **Producer** gửi message vào **topic** (chia thành nhiều **partition**).
- Mỗi message trong partition có một **offset** tăng dần (0, 1, 2, 3, …).
- **Consumer** đọc dữ liệu từ Kafka bằng cách:
  - Gửi request `Fetch` đến broker.
  - Broker trả về batch record bắt đầu từ offset mà consumer yêu cầu.
- Consumer có thể:
  - Tự quản lý offset (manual commit).
  - Hoặc để Kafka client tự commit offset (auto commit).

---

## 2. Offset – Cách consumer đọc dữ liệu

- Offset là “vị trí” trong partition.
- Consumer nhớ offset đã đọc đến đâu để:
  - Không đọc lại dữ liệu cũ (trừ khi muốn).
  - Tiếp tục đọc từ nơi dừng lại sau khi restart.

### Ví dụ: Consumer Spring Boot (Spring Kafka)

**pom.xml (phần chính)**

```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>
</dependencies>
```

**application.yml**

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    consumer:
      group-id: my-group
      auto-offset-reset: earliest   # nếu chưa có offset thì đọc từ đầu
      enable-auto-commit: false     # tự quản lý commit offset
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
```

**Consumer với manual commit**

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.kafka.support.Acknowledgment;
import org.springframework.stereotype.Service;

@Service
public class MyKafkaConsumer {

    @KafkaListener(topics = "my-topic", groupId = "my-group")
    public void listen(ConsumerRecord<String, String> record, Acknowledgment ack) {
        // Input: message từ Kafka
        System.out.println("Partition: " + record.partition());
        System.out.println("Offset: " + record.offset());
        System.out.println("Key: " + record.key());
        System.out.println("Value: " + record.value());

        // Xử lý business logic ở đây
        // ...

        // Sau khi xử lý thành công -> commit offset
        ack.acknowledge();
    }
}
```

**Input (message trong Kafka)**  
Topic: `my-topic`, Partition: 0  
Record:

```text
Key: user-1
Value: {"action":"login","time":"2026-05-20T10:00:00Z"}
Offset: 15
```

**Output (log của ứng dụng)**

```text
Partition: 0
Offset: 15
Key: user-1
Value: {"action":"login","time":"2026-05-20T10:00:00Z"}
```

## 3. Zero-copy transfer (sendfile) trong Kafka

- Bình thường, dữ liệu đi từ **disk → kernel → user space → kernel → network** (nhiều lần copy).
- Kafka dùng **zero-copy** với system call `sendfile()`:
  - Dữ liệu đi từ **disk → kernel → network** (bỏ qua user space).
  - Giảm số lần copy, giảm CPU, tăng throughput.
- Áp dụng khi broker gửi dữ liệu cho consumer:
  - Broker đọc log segment trên disk.
  - Dùng `sendfile()` để đẩy trực tiếp ra socket network.

### Minh họa luồng dữ liệu

**Không zero-copy:**

1. Disk → Kernel buffer
2. Kernel → User space (process Kafka)
3. User space → Kernel (socket buffer)
4. Kernel → Network

**Với zero-copy (sendfile):**

1. Disk → Kernel buffer
2. Kernel → Network (bỏ qua user space)

---

## 4. Tóm tắt ý chính

- Consumer đọc dữ liệu từ Kafka dựa trên **offset** trong từng partition.
- Offset giúp consumer biết đã đọc đến đâu và có thể tiếp tục sau khi restart.
- Kafka broker dùng **zero-copy (sendfile)** để gửi dữ liệu từ disk ra network:
  - Giảm overhead copy dữ liệu.
  - Tăng hiệu năng và throughput.
- Trong Spring Boot (Spring Kafka), có thể:
  - Dùng `@KafkaListener` để nhận message.
  - Tự quản lý commit offset để kiểm soát chính xác việc đọc/ghi.
