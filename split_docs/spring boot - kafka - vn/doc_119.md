## 1. Khái niệm cơ bản Kafka Streams

### 1.1. Topology là gì?

- **Topology**: là “sơ đồ xử lý” dữ liệu trong Kafka Streams.  
- Gồm:
  - **Source processors**: đọc dữ liệu từ topic Kafka.
  - **Stream processors**: xử lý, transform, filter, join, aggregate…
  - **Sink processors**: ghi kết quả ra topic Kafka khác.
- Có thể hình dung như một **đồ thị có hướng** (DAG) của các bước xử lý.

---

## 2. Kafka Streams DSL vs Processor API

### 2.1. Kafka Streams DSL (Domain Specific Language)

- Cung cấp API mức cao, dễ dùng:
  - `stream()`, `map()`, `filter()`, `groupByKey()`, `aggregate()`, `join()`, `to()`…
- Phù hợp cho:
  - Các pipeline xử lý chuẩn: transform, filter, aggregate, join.
  - Phát triển nhanh, code ngắn gọn, dễ đọc.
- Hạn chế:
  - Ít linh hoạt hơn khi cần logic rất phức tạp, stateful đặc biệt, hoặc tương tác thấp-level với record.

### 2.2. Processor API (Low-level)

- Cho phép bạn định nghĩa:
  - `Processor`, `Transformer`, `ProcessorContext`, `StateStore`…
- Phù hợp cho:
  - Logic xử lý phức tạp, custom state store, truy cập metadata, điều khiển luồng xử lý chi tiết.
- Hạn chế:
  - Code dài hơn, phức tạp hơn, khó bảo trì hơn so với DSL.

### 2.3. Kết hợp DSL + Processor API

- Có thể dùng DSL cho phần lớn pipeline, và chèn Processor API ở chỗ cần custom:
  - `transform()`, `process()` trong DSL để dùng custom processor/transformer.

## 3. Ví dụ với Spring Boot + Kafka Streams (DSL)

### 3.1. Mục tiêu

- Input topic: `input-text` (mỗi record là 1 dòng text).
- Xử lý:
  - Tách từ, chuyển về chữ thường, đếm số lần xuất hiện.
- Output topic: `word-count-output` (key = từ, value = số lần xuất hiện).

### 3.2. Cấu hình Maven (pom.xml – phần chính)

```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter</artifactId>
    </dependency>

    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>

    <dependency>
        <groupId>org.apache.kafka</groupId>
        <artifactId>kafka-streams</artifactId>
    </dependency>
</dependencies>
```

### 3.3. Cấu hình application.yml

```yaml
spring:
  application:
    name: kafka-streams-demo
  kafka:
    bootstrap-servers: localhost:9092
    streams:
      application-id: word-count-app
      properties:
        default.key.serde: org.apache.kafka.common.serialization.Serdes$StringSerde
        default.value.serde: org.apache.kafka.common.serialization.Serdes$StringSerde
```

### 3.4. Topology với DSL

```java
import org.apache.kafka.common.serialization.Serdes;
import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.kstream.*;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import java.util.Arrays;

@Configuration
public class WordCountTopology {

    @Bean
    public KStream<String, String> kStream(StreamsBuilder builder) {
        KStream<String, String> textLines = builder.stream("input-text");

        KTable<String, Long> wordCounts = textLines
                // tách dòng thành từ
                .flatMapValues(value -> Arrays.asList(value.toLowerCase().split("\\W+")))
                // key = từ
                .selectKey((key, word) -> word)
                // group theo key
                .groupByKey(Grouped.with(Serdes.String(), Serdes.String()))
                // đếm
                .count(Materialized.as("word-counts-store"));

        // ghi kết quả ra topic
        wordCounts.toStream()
                .to("word-count-output", Produced.with(Serdes.String(), Serdes.Long()));

        return textLines;
    }
}
```

### 3.5. Input / Output ví dụ

**Input topic `input-text`:**

| key   | value                         |
|-------|-------------------------------|
| null  | "Hello Kafka Kafka Streams"   |
| null  | "Hello Spring Boot"           |

**Output topic `word-count-output`:** (thứ tự có thể khác, đây là ví dụ)

| key     | value (count) |
|---------|---------------|
| "hello" | 1             |
| "kafka" | 1             |
| "kafka" | 2             |
| "streams" | 1           |
| "hello" | 2             |
| "spring" | 1            |
| "boot"   | 1            |

## 4. Ví dụ Processor API (custom Processor)

Ví dụ: log từng record trước khi đếm (chỉ minh họa cách dùng Processor API).

```java
import org.apache.kafka.streams.processor.api.*;
import org.apache.kafka.streams.processor.PunctuationType;
import org.apache.kafka.streams.processor.api.ProcessorSupplier;
import org.apache.kafka.streams.state.KeyValueStore;

public class LoggingProcessor implements Processor<String, String, String, String> {

    private ProcessorContext<String, String> context;

    @Override
    public void init(ProcessorContext<String, String> context) {
        this.context = context;
    }

    @Override
    public void process(Record<String, String> record) {
        System.out.println("Received record: key=" + record.key() + ", value=" + record.value());
        // forward record không thay đổi
        context.forward(record);
    }

    @Override
    public void close() {
    }
}
```

Kết hợp với DSL:

```java
@Bean
public KStream<String, String> kStreamWithProcessor(StreamsBuilder builder) {
    KStream<String, String> textLines = builder.stream("input-text");

    KStream<String, String> processed = textLines.process(() -> new LoggingProcessor());

    // tiếp tục xử lý bằng DSL như ví dụ word count
    processed
        .flatMapValues(value -> Arrays.asList(value.toLowerCase().split("\\W+")))
        .selectKey((key, word) -> word)
        .groupByKey()
        .count()
        .toStream()
        .to("word-count-output");

    return processed;
}
```

**Input/Output** logic vẫn giống ví dụ DSL, chỉ thêm bước log record.

## 5. Tóm tắt ý chính

- **Topology**: mô tả luồng xử lý dữ liệu (source → processor → sink).
- **DSL**:
  - Dùng cho hầu hết use case: transform, filter, aggregate, join.
  - Code ngắn, dễ đọc, dễ bảo trì.
- **Processor API**:
  - Dùng khi cần kiểm soát chi tiết, logic phức tạp, custom state.
- Có thể **kết hợp DSL + Processor API** trong cùng một topology để vừa dễ phát triển, vừa linh hoạt.
