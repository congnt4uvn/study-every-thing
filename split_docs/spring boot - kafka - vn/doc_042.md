
Dưới đây là tóm tắt ngắn gọn về cấu hình `auto.offset.reset` trong Kafka Consumer (Spring Boot), kèm ví dụ code và input/output.

---

## 1. `auto.offset.reset` là gì?

Thuộc tính `auto.offset.reset` quyết định **consumer sẽ đọc từ đâu** khi:

- Không tìm thấy offset đã commit trước đó, hoặc  
- Offset không hợp lệ (ví dụ: dữ liệu cũ đã bị xóa do retention).

Các giá trị chính:

1. `earliest`  
   - Đọc từ **bản ghi đầu tiên còn tồn tại** trong partition.  
   - Thích hợp khi bạn muốn **đọc lại toàn bộ dữ liệu** hiện có.

2. `latest`  
   - Đọc từ **bản ghi tiếp theo sẽ được ghi vào** (bỏ qua dữ liệu cũ).  
   - Thích hợp cho các dịch vụ chỉ quan tâm **dữ liệu mới**.

3. `none`  
   - Nếu **không có offset** đã commit → **ném lỗi** (Exception).  
   - Dùng khi bạn muốn **bắt buộc phải có offset** (ví dụ: hệ thống đã chạy trước đó, không được phép đọc lại từ đầu).

---

## 2. Cấu hình trong Spring Boot (`application.yml`)

```yaml
spring:
  kafka:
    consumer:
      bootstrap-servers: localhost:9092
      group-id: my-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      auto-offset-reset: earliest   # hoặc latest, none
```

---

## 3. Ví dụ Consumer đơn giản với `@KafkaListener`

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class MyKafkaConsumer {

    @KafkaListener(topics = "demo-topic", groupId = "my-group")
    public void listen(String message) {
        System.out.println("Received message: " + message);
    }
}
```

---

## 4. Ví dụ Input/Output theo từng cấu hình

Giả sử topic `demo-topic` có các message theo thứ tự:

1. `"msg-1"`
2. `"msg-2"`
3. `"msg-3"`

Consumer group `my-group` **chưa từng chạy trước đó** (chưa có offset).

### a) `auto.offset.reset = earliest`

- Input (producer gửi trước khi consumer chạy):
  - `"msg-1"`, `"msg-2"`, `"msg-3"`
- Khi consumer khởi động:
  - Output:
    - `Received message: msg-1`
    - `Received message: msg-2`
    - `Received message: msg-3`

### b) `auto.offset.reset = latest`

- Input (producer gửi trước khi consumer chạy):
  - `"msg-1"`, `"msg-2"`, `"msg-3"`
- Consumer khởi động sau đó, rồi producer gửi thêm:
  - `"msg-4"`, `"msg-5"`
- Output:
  - Consumer **không nhận** `msg-1, msg-2, msg-3`
  - Chỉ nhận:
    - `Received message: msg-4`
    - `Received message: msg-5`

### c) `auto.offset.reset = none`

- Consumer group `my-group` chưa có offset.
- Khi consumer khởi động:
  - Ném lỗi kiểu:
    - `org.apache.kafka.clients.consumer.NoOffsetForPartitionException`
  - Không đọc message nào cho đến khi bạn xử lý lỗi hoặc thiết lập offset bằng cách khác.

---

## 5. Khi nào dùng giá trị nào?

- Dùng `earliest` khi:
  - Dịch vụ mới, cần **đọc lại toàn bộ dữ liệu hiện có**.
- Dùng `latest` khi:
  - Chỉ quan tâm **dữ liệu mới**, không cần dữ liệu cũ.
- Dùng `none` khi:
  - Hệ thống yêu cầu **tính chặt chẽ về offset**, không được tự ý đọc từ đầu hoặc từ cuối nếu thiếu offset.
