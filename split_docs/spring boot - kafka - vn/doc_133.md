
Dưới đây là tóm tắt ngắn gọn về “Upgrades & compatibility: Rolling upgrades, protocol versions, feature flags (KIP-based)” trong Kafka + Spring Boot, kèm ví dụ code.

---

## 1. Rolling Upgrades (Nâng cấp cuốn chiếu)

**Ý chính:**
- Nâng cấp từng broker trong cluster, không dừng toàn bộ hệ thống.
- Đảm bảo **tương thích ngược** (backward compatible) giữa các phiên bản Kafka.
- Thứ tự thường dùng:
  1. Nâng cấp client (ứng dụng Spring Boot).
  2. Nâng cấp broker từng node một.
  3. Sau khi toàn cluster đã nâng cấp, có thể bật các tính năng mới.

**Ví dụ quy trình:**
1. Dừng broker 1 → nâng cấp → khởi động lại.
2. Kiểm tra cluster ổn định.
3. Lặp lại cho broker 2, 3, …

---

## 2. Protocol Versions (Phiên bản giao thức)

**Ý chính:**
- Kafka client và broker giao tiếp qua **giao thức có version**.
- Client mới có thể nói chuyện với broker cũ (và ngược lại) nếu vẫn hỗ trợ version giao thức tương ứng.
- Điều này cho phép nâng cấp dần dần mà không cần đồng bộ tất cả thành cùng một version ngay lập tức.

**Trong Spring Boot (Spring Kafka):**

```xml
<!-- pom.xml -->
<dependency>
    <groupId>org.springframework.kafka</groupId>
    <artifactId>spring-kafka</artifactId>
    <version>3.1.0</version> <!-- ví dụ -->
</dependency>
```

```properties
# application.properties
spring.kafka.bootstrap-servers=localhost:9092
spring.kafka.properties.client.id=my-spring-client
spring.kafka.properties.api.version.request=true
spring.kafka.properties.api.version.fallback.ms=0
```

- `api.version.request=true`: client tự động đàm phán version giao thức với broker.

**Input/Output ví dụ:**

- Input: Producer gửi message `"Hello Kafka"` đến topic `demo-topic`.
- Output: Broker nhận message, lưu vào log; Consumer đọc message `"Hello Kafka"` bình thường dù broker đang trong quá trình nâng cấp (rolling upgrade), miễn là version giao thức tương thích.

## 3. Feature Flags (KIP-based)

**Ý chính:**
- Nhiều tính năng mới của Kafka được đề xuất qua **KIP (Kafka Improvement Proposal)**.
- Một số tính năng được bật/tắt qua **feature flags** hoặc config, giúp:
  - Nâng cấp trước, bật tính năng sau.
  - Dễ rollback nếu có vấn đề.

**Ví dụ: KIP-500 (loại bỏ Zookeeper)**  
- Các tính năng liên quan có thể được bật khi cluster đã ở version hỗ trợ, nhưng không bắt buộc bật ngay khi nâng cấp.

**Ví dụ config (server.properties):**

```properties
# Ví dụ (mang tính minh họa)
inter.broker.protocol.version=3.6
log.message.format.version=3.6
# Một số feature có thể được bật sau khi toàn cluster đã lên 3.6
```

## 4. Ví dụ Spring Boot + Kafka (Producer/Consumer)

### Producer

```java
// ProducerConfig
@Configuration
public class KafkaProducerConfig {

    @Bean
    public ProducerFactory<String, String> producerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        // Đàm phán version giao thức
        props.put("api.version.request", "true");
        return new DefaultKafkaProducerFactory<>(props);
    }

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate() {
        return new KafkaTemplate<>(producerFactory());
    }
}
```

```java
// Service gửi message
@Service
public class DemoProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public DemoProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String msg) {
        kafkaTemplate.send("demo-topic", msg);
    }
}
```

**Input:**  
`sendMessage("Xin chào Kafka trong quá trình rolling upgrade")`

**Output:**  
- Message được gửi thành công đến `demo-topic` dù một số broker đang nâng cấp, miễn là cluster vẫn có quorum và version giao thức tương thích.

---

### Consumer

```java
@Configuration
@EnableKafka
public class KafkaConsumerConfig {

    @Bean
    public ConsumerFactory<String, String> consumerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "demo-group");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        // Đàm phán version giao thức
        props.put("api.version.request", "true");
        return new DefaultKafkaConsumerFactory<>(props);
    }

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory());
        return factory;
    }
}
```

```java
// Listener
@Service
public class DemoConsumer {

    @KafkaListener(topics = "demo-topic", groupId = "demo-group")
    public void listen(String message) {
        System.out.println("Received: " + message);
    }
}
```

**Output khi Producer gửi:**  
`Xin chào Kafka trong quá trình rolling upgrade`  
Consumer log:  
`Received: Xin chào Kafka trong quá trình rolling upgrade`

## Tóm tắt ngắn

- **Rolling upgrades**: Nâng cấp từng broker, không dừng hệ thống; cần đảm bảo tương thích giữa các version.
- **Protocol versions**: Client/broker dùng giao thức có version; Spring Kafka hỗ trợ đàm phán version tự động.
- **Feature flags (KIP-based)**: Tính năng mới được bật/tắt qua config, cho phép nâng cấp trước, bật tính năng sau, dễ rollback.
