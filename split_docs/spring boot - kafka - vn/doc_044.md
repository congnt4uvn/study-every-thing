
Dưới đây là tóm tắt ngắn gọn về các config consumer Kafka trong Spring Boot, kèm ví dụ code và mô tả input/output.

---

## 1. `max.poll.records`
- **Ý nghĩa**: Số lượng bản ghi (records) tối đa mà consumer lấy về trong mỗi lần `poll()`.
- **Tác dụng**:
  - Tăng giá trị → mỗi lần poll lấy nhiều message hơn → throughput cao hơn nhưng xử lý mỗi batch lâu hơn.
  - Giảm giá trị → mỗi lần poll ít message → độ trễ thấp hơn cho từng message nhưng tổng throughput có thể giảm.

**Ví dụ cấu hình (application.yml)**:
```yaml
spring:
  kafka:
    consumer:
      bootstrap-servers: localhost:9092
      group-id: my-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      properties:
        max.poll.records: 50
```

**Ví dụ xử lý (Java)**:
```java
@KafkaListener(topics = "orders", groupId = "my-group")
public void listen(List<String> messages) {
    // Input: tối đa 50 messages mỗi lần gọi (theo max.poll.records)
    for (String msg : messages) {
        System.out.println("Received: " + msg);
    }
    // Output: log ra console các message nhận được
}
```

## 2. `max.poll.interval.ms`
- **Ý nghĩa**: Thời gian tối đa giữa hai lần `poll()` liên tiếp.
- **Tác dụng**:
  - Nếu consumer không gọi `poll()` trong khoảng thời gian này, Kafka coi consumer bị “chết” và sẽ **revoke partition**.
  - Dùng để bảo vệ cluster khi consumer xử lý quá lâu mà không poll thêm.

**Ví dụ**:
```yaml
spring:
  kafka:
    consumer:
      properties:
        max.poll.interval.ms: 300000  # 5 phút
```

**Tình huống**:
- Nếu xử lý 1 batch mất 7 phút > 5 phút:
  - Kafka sẽ kick consumer ra khỏi group.
  - Partition được assign cho consumer khác.
  - Có thể dẫn đến xử lý trùng (reprocessing) nếu offset chưa commit.

---

## 3. `session.timeout.ms`
- **Ý nghĩa**: Thời gian timeout của session giữa consumer và Kafka broker.
- **Tác dụng**:
  - Nếu broker không nhận được heartbeat từ consumer trong khoảng thời gian này, nó coi consumer đã chết.
  - Phải **lớn hơn** `heartbeat.interval.ms`.

**Ví dụ**:
```yaml
spring:
  kafka:
    consumer:
      properties:
        session.timeout.ms: 10000  # 10 giây
```

---

## 4. `heartbeat.interval.ms`
- **Ý nghĩa**: Khoảng thời gian giữa các lần consumer gửi heartbeat tới broker.
- **Tác dụng**:
  - Giúp broker biết consumer vẫn “sống”.
  - Thường nhỏ hơn `session.timeout.ms` (ví dụ: 1/3).

**Ví dụ**:
```yaml
spring:
  kafka:
    consumer:
      properties:
        heartbeat.interval.ms: 3000  # 3 giây
```

---

## Mối quan hệ giữa các config

- `heartbeat.interval.ms` < `session.timeout.ms`
  - Ví dụ: `heartbeat.interval.ms = 3s`, `session.timeout.ms = 10s`.
- `max.poll.interval.ms` liên quan đến **thời gian xử lý batch**:
  - Thời gian xử lý 1 batch phải < `max.poll.interval.ms`.
- `max.poll.records` ảnh hưởng đến:
  - Thời gian xử lý 1 batch.
  - Số message nhận được mỗi lần poll.

---

## Ví dụ đầy đủ Spring Boot (Java)

**application.yml**:
```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    consumer:
      group-id: my-group
      auto-offset-reset: earliest
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      properties:
        max.poll.records: 100
        max.poll.interval.ms: 300000      # 5 phút
        session.timeout.ms: 10000         # 10 giây
        heartbeat.interval.ms: 3000       # 3 giây
```

**Consumer code**:
```java
@KafkaListener(topics = "orders", groupId = "my-group")
public void consume(List<String> messages) {
    // Input: tối đa 100 messages mỗi lần (max.poll.records)
    System.out.println("Batch size: " + messages.size());
    for (String msg : messages) {
        // Giả sử xử lý mỗi message
        System.out.println("Processing: " + msg);
    }
    // Output: log số lượng và nội dung message
}
```

**Ví dụ Input/Output**:

- Topic `orders` có 3 message:
  - `"order-1"`, `"order-2"`, `"order-3"`
- `max.poll.records = 100` → 1 batch nhận đủ 3 message.

**Output console**:
```text
Batch size: 3
Processing: order-1
Processing: order-2
Processing: order-3
```

---

Tóm lại:
- Dùng `max.poll.records` để điều chỉnh kích thước batch.
- Dùng `max.poll.interval.ms` để giới hạn thời gian xử lý batch.
- Dùng `session.timeout.ms` + `heartbeat.interval.ms` để kiểm soát việc phát hiện consumer “chết” và rebalance.
