
Dưới đây là tóm tắt ngắn gọn về **Topics, Partitions, Ordering** trong Kafka, kèm ví dụ Spring Boot.

---

## 1. Topic và Partition

- **Topic**: “Tên” của luồng dữ liệu (ví dụ: `orders`, `payments`).
- **Partition**: Mỗi topic được chia thành nhiều partition.
  - Mỗi partition là một log **append-only** (ghi nối tiếp).
  - Mỗi message trong partition có một **offset** tăng dần.

---

## 2. Bảo đảm thứ tự (Ordering Guarantees)

- Kafka **chỉ đảm bảo thứ tự trong phạm vi 1 partition**.
- Không có bảo đảm thứ tự **giữa các partition khác nhau** của cùng một topic.
- Nếu bạn cần giữ thứ tự cho một “nhóm” dữ liệu (ví dụ: theo `userId`, `orderId`), bạn phải:
  - Dùng **cùng key** khi gửi message → các message cùng key sẽ vào **cùng partition** → giữ được thứ tự.

---

## 3. Ví dụ Producer (Spring Boot) – Gửi theo key để giữ thứ tự

**pom.xml** (phần chính cho Kafka + Spring Boot):

```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>
</dependencies>
```

**Cấu hình Producer**:

```java
@Configuration
@EnableKafka
public class KafkaProducerConfig {

    @Bean
    public ProducerFactory<String, String> producerFactory() {
        Map<String, Object> config = new HashMap<>();
        config.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        config.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        config.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        return new DefaultKafkaProducerFactory<>(config);
    }

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate() {
        return new KafkaTemplate<>(producerFactory());
    }
}
```

**Service gửi message với key (ví dụ: userId)**:

```java
@Service
public class OrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public OrderProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendOrder(String userId, String orderJson) {
        // userId là key → tất cả order của cùng userId sẽ vào cùng partition
        kafkaTemplate.send("orders-topic", userId, orderJson);
    }
}
```

**Input ví dụ**:

```java
orderProducer.sendOrder("user-1", "{ \"orderId\": 1, \"item\": \"A\" }");
orderProducer.sendOrder("user-1", "{ \"orderId\": 2, \"item\": \"B\" }");
orderProducer.sendOrder("user-2", "{ \"orderId\": 10, \"item\": \"X\" }");
```

**Kết quả (logic)**:

- Các message có `key = "user-1"` sẽ vào **cùng partition**, nên consumer sẽ đọc theo thứ tự:
  1. `{ "orderId": 1, "item": "A" }`
  2. `{ "orderId": 2, "item": "B" }`
- Message của `user-2` có thể ở partition khác, thứ tự giữa `user-1` và `user-2` **không được đảm bảo**.

---

## 4. Ví dụ Consumer (Spring Boot) – Đọc theo thứ tự trong partition

```java
@Service
public class OrderConsumer {

    @KafkaListener(topics = "orders-topic", groupId = "order-group")
    public void consume(ConsumerRecord<String, String> record) {
        String key = record.key();      // userId
        String value = record.value();  // orderJson
        long offset = record.offset();  // offset trong partition

        System.out.printf(
            "Partition=%d, Offset=%d, Key=%s, Value=%s%n",
            record.partition(), offset, key, value
        );
    }
}
```

**Output ví dụ (console)**:

```text
Partition=0, Offset=0, Key=user-1, Value={ "orderId": 1, "item": "A" }
Partition=0, Offset=1, Key=user-1, Value={ "orderId": 2, "item": "B" }
Partition=1, Offset=0, Key=user-2, Value={ "orderId": 10, "item": "X" }
```

- Trong **mỗi partition**, offset tăng dần → thứ tự được giữ.
- Giữa partition 0 và 1, không có bảo đảm thứ tự tổng thể.

---

## 5. Ý chính cần nhớ

1. **Topic** chứa nhiều **partition**.
2. Kafka **chỉ đảm bảo thứ tự trong 1 partition**.
3. Dùng **key** nhất quán để đưa các message cần giữ thứ tự vào **cùng partition**.
4. Consumer đọc theo offset tăng dần trong partition → giữ được thứ tự đó.
