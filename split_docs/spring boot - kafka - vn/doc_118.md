
Dưới đây là tóm tắt ngắn gọn về các khái niệm cơ bản trong Kafka Streams: **KStream, KTable, GlobalKTable**, kèm ví dụ code (Java/Spring Boot style) và input/output minh họa.

---

## 1. KStream – Luồng sự kiện (Stream)

- Là **dòng các record bất biến**, liên tục, theo thời gian.
- Mỗi record thường có dạng **(key, value, timestamp)**.
- Phù hợp cho:
  - Xử lý sự kiện theo thời gian thực (real-time event processing).
  - Các phép biến đổi như `map`, `filter`, `flatMap`, `branch`, `join` với stream khác.

### Ví dụ code (Java – Kafka Streams API)

```java
StreamsBuilder builder = new StreamsBuilder();

// Đọc từ topic "input-topic" thành KStream
KStream<String, String> inputStream = builder.stream("input-topic");

// Ví dụ: lọc các message có value chứa "error"
KStream<String, String> errorStream = inputStream
        .filter((key, value) -> value != null && value.contains("error"));

// Ghi kết quả ra topic "error-topic"
errorStream.to("error-topic", Produced.with(Serdes.String(), Serdes.String()));
```

**Input (topic `input-topic`):**
- key: "user1", value: "login success"
- key: "user2", value: "error: invalid password"
- key: "user3", value: "error: timeout"

**Output (topic `error-topic`):**
- key: "user2", value: "error: invalid password"
- key: "user3", value: "error: timeout"

## 2. KTable – Bảng trạng thái (Table)

- Là **view dạng bảng** được xây dựng từ một topic dạng **changelog**.
- Mỗi key chỉ có **một giá trị hiện tại** (giống như một bản ghi trong bảng database).
- Khi có record mới với cùng key, giá trị sẽ **ghi đè** (update).
- Phù hợp cho:
  - Lưu trạng thái (stateful processing).
  - Tính toán tổng hợp (aggregation) như `count`, `sum`, `reduce`.
  - Join với KStream hoặc KTable khác.

### Ví dụ code: đếm số message theo key

```java
StreamsBuilder builder = new StreamsBuilder();

KStream<String, String> inputStream = builder.stream("input-topic");

// Đếm số message theo key -> KTable
KTable<String, Long> counts = inputStream
        .groupByKey()
        .count(Materialized.as("counts-store"));

// Ghi kết quả ra topic "counts-topic"
counts.toStream().to("counts-topic", Produced.with(Serdes.String(), Serdes.Long()));
```

**Input (topic `input-topic`):**
- ("user1", "a")
- ("user1", "b")
- ("user2", "x")
- ("user1", "c")

**Trạng thái KTable (counts):**
- user1 → 3
- user2 → 1

**Output (topic `counts-topic`):**
- ("user1", 1)
- ("user1", 2)
- ("user2", 1)
- ("user1", 3)

> Lưu ý: KTable luôn giữ **giá trị cuối cùng** cho mỗi key (state hiện tại).

## 3. GlobalKTable – Bảng toàn cục (Global Table)

- Giống KTable nhưng:
  - **Mỗi instance** của ứng dụng Kafka Streams sẽ **chứa toàn bộ dữ liệu** của bảng.
  - Dữ liệu được **replicate** đến tất cả các instance.
- Phù hợp cho:
  - Tra cứu (lookup) dữ liệu tham chiếu (reference data) nhỏ đến trung bình.
  - Join với KStream mà không phụ thuộc vào partitioning.

### Ví dụ code: join KStream với GlobalKTable

Giả sử:
- Topic `user-events`: stream các sự kiện của user.
- Topic `user-profiles`: chứa thông tin profile user (id, name, email, …) → dùng làm GlobalKTable.

```java
StreamsBuilder builder = new StreamsBuilder();

// KStream các sự kiện user
KStream<String, String> userEvents = builder.stream("user-events");

// GlobalKTable chứa profile user
GlobalKTable<String, String> userProfiles = builder.globalTable("user-profiles");

// Join KStream với GlobalKTable theo key
KStream<String, String> enrichedEvents = userEvents.join(
        userProfiles,
        (eventKey, eventValue) -> eventKey, // cách map key để lookup trong GlobalKTable
        (eventValue, profileValue) -> "event=" + eventValue + ", profile=" + profileValue
);

// Ghi kết quả ra topic
enrichedEvents.to("enriched-user-events");
```

**Input (topic `user-profiles` – GlobalKTable):**
- key: "user1", value: "{name: 'A', email: 'a@example.com'}"
- key: "user2", value: "{name: 'B', email: 'b@example.com'}"

**Input (topic `user-events` – KStream):**
- key: "user1", value: "login"
- key: "user2", value: "purchase"
- key: "user1", value: "logout"

**Output (topic `enriched-user-events`):**
- key: "user1", value: "event=login, profile={name: 'A', email: 'a@example.com'}"
- key: "user2", value: "event=purchase, profile={name: 'B', email: 'b@example.com'}"
- key: "user1", value: "event=logout, profile={name: 'A', email: 'a@example.com'}"

## 4. Tóm tắt so sánh

| Khái niệm      | Kiểu dữ liệu | Đặc điểm chính                                      | Use case chính                          |
|----------------|-------------|-----------------------------------------------------|-----------------------------------------|
| **KStream**    | Stream      | Dòng sự kiện liên tục, không có trạng thái cuối cùng | Xử lý event real-time, transform, filter |
| **KTable**     | Table       | Bảng trạng thái, mỗi key có 1 giá trị hiện tại      | Aggregation, stateful processing, join  |
| **GlobalKTable** | Table     | Bảng trạng thái được replicate toàn bộ trên mọi instance | Lookup dữ liệu tham chiếu, join linh hoạt |

Nếu bạn muốn, tôi có thể chuyển các ví dụ trên sang cấu hình Spring Boot (application.yml, @Configuration, @Bean) cụ thể cho Kafka Streams.
