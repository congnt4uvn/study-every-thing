
Dưới đây là tóm tắt ngắn gọn về các loại container consumer trong Spring Kafka, kèm ví dụ code và input/output minh họa.

---

## 1. Hai loại container chính

### `KafkaMessageListenerContainer`
- Container **đơn luồng** (single-threaded).
- Mỗi container thường gắn với **một partition** (hoặc một nhóm nhỏ partition).
- Phù hợp khi:
  - Bạn muốn **kiểm soát chi tiết** việc gán partition.
  - Cần xử lý tuần tự, không cần nhiều luồng.

### `ConcurrentMessageListenerContainer`
- Container **đa luồng** (multi-threaded).
- Bọc nhiều `KafkaMessageListenerContainer` bên trong.
- Thuộc tính chính: `concurrency` = số luồng / số container con.
- Phù hợp khi:
  - Cần **tăng throughput** bằng cách xử lý song song.
  - Topic có nhiều partition và bạn muốn tận dụng đa luồng.

---

## 2. Cách hoạt động (tóm tắt)

- Mỗi `KafkaMessageListenerContainer`:
  - Tạo một consumer Kafka.
  - Lắng nghe message từ partition được gán.
  - Gọi `MessageListener` (hoặc `@KafkaListener`) khi có message.

- `ConcurrentMessageListenerContainer`:
  - Tạo **N** container con (N = `concurrency`).
  - Mỗi container con là một `KafkaMessageListenerContainer`.
  - Các container con chia nhau các partition của topic.

## 3. Ví dụ cấu hình với `ConcurrentMessageListenerContainer`

### 3.1. Cấu hình ConsumerFactory & ContainerFactory

```java
@Configuration
@EnableKafka
public class KafkaConsumerConfig {

    @Bean
    public Map<String, Object> consumerConfigs() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "my-group");
        props.put(ConsumerConfig.AUTO_OFFSET_RESET_CONFIG, "earliest");
        return props;
    }

    @Bean
    public ConsumerFactory<String, String> consumerFactory() {
        return new DefaultKafkaConsumerFactory<>(consumerConfigs());
    }

    // Dùng cho @KafkaListener
    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory());
        factory.setConcurrency(3); // 3 luồng / 3 container con
        factory.getContainerProperties().setPollTimeout(3000);
        return factory;
    }
}
```

### 3.2. Consumer với `@KafkaListener`

```java
@Component
public class MyKafkaConsumer {

    @KafkaListener(topics = "my-topic", groupId = "my-group")
    public void listen(String message) {
        System.out.println("Thread: " + Thread.currentThread().getName()
                + " | Received: " + message);
    }
}
```

**Input (message gửi vào Kafka topic `my-topic`):**
- Partition 0: `"msg-1"`, `"msg-2"`
- Partition 1: `"msg-3"`
- Partition 2: `"msg-4"`, `"msg-5"`

**Output (log console, ví dụ):**
```text
Thread: consumer-0-C-1 | Received: msg-1
Thread: consumer-0-C-1 | Received: msg-2
Thread: consumer-0-C-2 | Received: msg-3
Thread: consumer-0-C-3 | Received: msg-4
Thread: consumer-0-C-3 | Received: msg-5
```

- Mỗi thread xử lý một hoặc nhiều partition.
- Thứ tự **trong cùng một partition** vẫn được giữ nguyên.

## 4. Ví dụ dùng trực tiếp `KafkaMessageListenerContainer`

Khi bạn muốn tự tạo container đơn lẻ, không dùng `@KafkaListener`:

```java
@Configuration
public class SingleContainerConfig {

    @Bean
    public KafkaMessageListenerContainer<String, String> singleContainer(
            ConsumerFactory<String, String> consumerFactory) {

        ContainerProperties containerProps =
                new ContainerProperties("my-topic"); // có thể set partition cụ thể

        containerProps.setMessageListener((MessageListener<String, String>) record -> {
            System.out.println("Single container received: " + record.value()
                    + " from partition: " + record.partition());
        });

        return new KafkaMessageListenerContainer<>(consumerFactory, containerProps);
    }

    @Bean
    public ApplicationRunner runner(KafkaMessageListenerContainer<String, String> container) {
        return args -> container.start(); // tự start container
    }
}
```

**Input:**
- Gửi `"hello"` vào `my-topic`, partition 0.

**Output:**
```text
Single container received: hello from partition: 0
```

---

## 5. Khi nào dùng loại nào?

- Dùng `ConcurrentMessageListenerContainer` (thường qua `@KafkaListener`):
  - Hầu hết các use case trong ứng dụng Spring Boot.
  - Cần scale theo số partition, tăng throughput.

- Dùng `KafkaMessageListenerContainer`:
  - Cần cấu hình chi tiết, manual (ví dụ: gán partition cụ thể, start/stop container theo logic riêng).
  - Use case đặc biệt, ít partition, cần kiểm soát chặt.

Nếu bạn gửi nội dung chi tiết hơn (ví dụ bài viết đầy đủ), tôi có thể tóm tắt sâu hơn từng phần và thêm các ví dụ nâng cao (ack thủ công, error handler, batch listener, v.v.).
