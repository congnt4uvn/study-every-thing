
Dưới đây là tóm tắt ngắn gọn về **chiến lược partitioning trong Kafka với Spring Boot**, tập trung vào:

- Default partitioner  
- Custom partitioner  
- Ví dụ code + input/output minh họa  

---

## 1. Khái niệm Partitioning trong Kafka

- Mỗi **topic** được chia thành nhiều **partition**.  
- Partition quyết định:  
  - Thứ tự message (order) trong cùng một partition  
  - Khả năng song song (parallelism) khi consumer đọc dữ liệu  
- **Partitioner** là thành phần quyết định **message sẽ đi vào partition nào**.

---

## 2. Default Partitioner (Mặc định)

### Cách hoạt động

Kafka default partitioner (từ Kafka 2.x trở lên) thường hoạt động như sau:

1. **Có key**:
   - Sử dụng **hash(key)** để xác định partition:
     ```text
     partition = hash(key) % numberOfPartitions
     ```
   - Kết quả: cùng một key luôn vào cùng một partition → đảm bảo **ordering theo key**.

2. **Không có key**:
   - Sử dụng **round-robin** hoặc cơ chế phân phối đều giữa các partition → giúp cân bằng tải.

### Ví dụ với Spring Boot (Default Partitioner)

**application.yml** (chỉ cấu hình cơ bản, không đụng đến partitioner → dùng default)

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

**Producer gửi message với key**

```java
@Service
public class DefaultPartitionProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public DefaultPartitionProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendWithKey(String key, String value) {
        kafkaTemplate.send("my-topic", key, value);
    }

    public void sendWithoutKey(String value) {
        kafkaTemplate.send("my-topic", value); // key = null
    }
}
```

### Input/Output minh họa

Giả sử topic `my-topic` có **3 partition**: 0, 1, 2.

**Input 1:**

```java
sendWithKey("user-1", "Hello A");
sendWithKey("user-1", "Hello B");
sendWithKey("user-2", "Hello C");
sendWithoutKey("No key 1");
sendWithoutKey("No key 2");
```

**Output (logic phân phối, ví dụ):**

- `key = "user-1"` → hash("user-1") % 3 = 1 → partition 1  
  - "Hello A" → partition 1  
  - "Hello B" → partition 1 (giữ được thứ tự theo key)
- `key = "user-2"` → hash("user-2") % 3 = 0 → partition 0  
  - "Hello C" → partition 0
- Không có key → được phân phối lần lượt (round-robin)  
  - "No key 1" → partition 2  
  - "No key 2" → partition 0 (ví dụ)

## 3. Custom Partitioner (Tự định nghĩa)

Khi nào cần custom partitioner:

- Muốn **điều khiển logic phân phối** message theo:
  - Range (ví dụ: userId 0–9999 → partition 0, 10000–19999 → partition 1, …)
  - Loại dữ liệu (event type, region, priority, …)
  - Yêu cầu business đặc biệt (ví dụ: tất cả VIP user vào partition riêng)

### Bước 1: Tạo class Custom Partitioner

```java
import org.apache.kafka.clients.producer.Partitioner;
import org.apache.kafka.common.Cluster;

import java.util.Map;

public class UserIdRangePartitioner implements Partitioner {

    @Override
    public int partition(String topic,
                         Object key,
                         byte[] keyBytes,
                         Object value,
                         byte[] valueBytes,
                         Cluster cluster) {

        int numPartitions = cluster.partitionsForTopic(topic).size();

        if (key == null) {
            // Nếu không có key, cho vào partition cuối cùng
            return numPartitions - 1;
        }

        // Giả sử key là userId dạng số (String)
        int userId;
        try {
            userId = Integer.parseInt(key.toString());
        } catch (NumberFormatException e) {
            // Nếu key không phải số, cho vào partition 0
            return 0;
        }

        // Ví dụ: chia range theo userId
        // userId % (numPartitions - 1) để dành 1 partition cuối cho key null
        return userId % (numPartitions - 1);
    }

    @Override
    public void close() {
        // Không cần làm gì
    }

    @Override
    public void configure(Map<String, ?> configs) {
        // Có thể đọc config nếu cần
    }
}
```

### Bước 2: Cấu hình Spring Boot dùng custom partitioner

**application.yml**

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      properties:
        partitioner.class: com.example.kafka.partition.UserIdRangePartitioner
```

> Lưu ý: thay `com.example.kafka.partition.UserIdRangePartitioner` bằng package thực tế của bạn.

### Bước 3: Producer sử dụng custom partitioner

```java
@Service
public class CustomPartitionProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public CustomPartitionProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendUserEvent(String userId, String event) {
        kafkaTemplate.send("user-topic", userId, event);
    }

    public void sendWithoutKey(String event) {
        kafkaTemplate.send("user-topic", event); // key = null
    }
}
```

### Input/Output minh họa

Giả sử topic `user-topic` có **4 partition**: 0, 1, 2, 3  
- Partition 0–2: dùng cho userId có key là số  
- Partition 3: dành cho message **không có key**

**Input:**

```java
sendUserEvent("1", "Login");
sendUserEvent("2", "Logout");
sendUserEvent("5", "Purchase");
sendWithoutKey("System event");
sendUserEvent("abc", "Invalid user");
```

**Output (theo logic partitioner):**

- `userId = "1"` → 1 % (4 - 1) = 1 % 3 = 1 → partition 1  
- `userId = "2"` → 2 % 3 = 2 → partition 2  
- `userId = "5"` → 5 % 3 = 2 → partition 2  
- `key = null` → partition 3 (dành riêng cho key null)  
- `key = "abc"` (không parse được số) → partition 0

## 4. So sánh nhanh

| Tiêu chí                 | Default Partitioner                         | Custom Partitioner                                      |
|--------------------------|---------------------------------------------|---------------------------------------------------------|
| Cấu hình                 | Không cần thêm gì                           | Cần viết class + cấu hình `partitioner.class`          |
| Logic phân phối          | Hash theo key, round-robin nếu không có key | Tùy ý (range, type, region, priority, …)               |
| Đảm bảo order theo key   | Có (trong cùng partition)                   | Có, nếu bạn thiết kế logic phù hợp                     |
| Độ phức tạp              | Thấp                                        | Cao hơn, cần test kỹ                                   |

---

Nếu bạn cho biết thêm:  
- Số lượng partition  
- Kiểu key/value (userId, region, type, …)  
mình có thể viết một custom partitioner cụ thể hơn đúng với use case của bạn.
