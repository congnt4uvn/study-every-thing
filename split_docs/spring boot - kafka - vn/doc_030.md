
Dưới đây là tóm tắt ngắn gọn về `enable.idempotence` và khái niệm Exactly-Once Semantics (EOS) trong Kafka, kèm ví dụ Spring Boot.

---

## 1. `enable.idempotence` là gì?

- Là cấu hình cho **Kafka Producer** để đảm bảo **mỗi message chỉ được ghi đúng 1 lần** vào partition, ngay cả khi producer retry.
- Khi bật `enable.idempotence=true`, Kafka:
  - Gán **Producer ID (PID)** cho producer.
  - Gắn **sequence number** cho từng record.
  - Broker dùng PID + sequence number để **loại bỏ bản ghi trùng** (do retry).

### Tác dụng chính

- Tránh **duplicate message** do retry.
- Đảm bảo **at-least-once + không trùng lặp trên partition** → gần với **exactly-once** ở phía producer.

### Cấu hình quan trọng đi kèm

Khi `enable.idempotence=true`, Kafka sẽ tự động điều chỉnh:

- `acks=all`
- `retries` = Integer.MAX_VALUE (rất nhiều lần)
- `max.in.flight.requests.per.connection <= 5` (để giữ thứ tự)

---

## 2. Exactly-Once Semantics (EOS) là gì?

- EOS trong Kafka là đảm bảo: **mỗi message được xử lý và ghi ra kết quả đúng 1 lần**, kể cả khi có lỗi, retry, restart.
- EOS đầy đủ thường cần:
  1. **Idempotent producer** (`enable.idempotence=true`)
  2. **Transactions** (giao dịch) giữa:
     - đọc từ topic input
     - xử lý
     - ghi ra topic output
     - commit offset
  3. Consumer sử dụng **read-process-write** trong cùng 1 transaction.

## 3. Ví dụ cấu hình Producer trong Spring Boot

### `application.yml`

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      properties:
        enable.idempotence: true
        acks: all
        max.in.flight.requests.per.connection: 5
        retries: 2147483647
```

### Producer bean

```java
@Configuration
public class KafkaProducerConfig {

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate(
            ProducerFactory<String, String> producerFactory) {
        return new KafkaTemplate<>(producerFactory);
    }
}
```

### Gửi message

```java
@Service
public class MyProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public MyProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String key, String value) {
        kafkaTemplate.send("my-topic", key, value)
                     .whenComplete((result, ex) -> {
                         if (ex == null) {
                             System.out.println("Sent: " + value +
                                     " to partition " + result.getRecordMetadata().partition() +
                                     " offset " + result.getRecordMetadata().offset());
                         } else {
                             System.err.println("Send failed: " + ex.getMessage());
                         }
                     });
    }
}
```

**Input**:  
`sendMessage("user-1", "order-created-123")`

**Output (log)**:  
`Sent: order-created-123 to partition 0 offset 15`

Nếu producer retry do lỗi mạng, broker vẫn chỉ ghi **1 bản** với offset 15 (không trùng).

## 4. Ví dụ EOS với Transaction trong Spring Kafka

### Cấu hình transaction

```yaml
spring:
  kafka:
    producer:
      properties:
        enable.idempotence: true
      transaction-id-prefix: tx-  # bật transactional producer
    consumer:
      group-id: my-group
      enable-auto-commit: false
```

### Config

```java
@Configuration
public class KafkaTxConfig {

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate(
            ProducerFactory<String, String> pf) {
        KafkaTemplate<String, String> template = new KafkaTemplate<>(pf);
        template.setTransactionIdPrefix("tx-");
        return template;
    }
}
```

### Xử lý read → process → write với transaction

```java
@Service
public class TxService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public TxService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    @KafkaListener(topics = "input-topic", groupId = "my-group")
    public void listen(List<String> messages, Acknowledgment ack) {
        kafkaTemplate.executeInTransaction(operations -> {
            for (String msg : messages) {
                String processed = "processed-" + msg;
                operations.send("output-topic", processed);
            }
            // commit offset trong transaction (Spring Kafka hỗ trợ)
            ack.acknowledge();
            return null;
        });
    }
}
```

**Input**:  
`input-topic`: `"msg-1", "msg-2"`

**Xử lý**:
- Đọc `"msg-1", "msg-2"`
- Gửi `"processed-msg-1", "processed-msg-2"` vào `output-topic`
- Commit offset cùng transaction

**Kết quả**:
- Nếu lỗi giữa chừng → **không commit offset, không ghi output**.
- Khi chạy lại → xử lý lại từ đầu nhưng vẫn đảm bảo **mỗi message chỉ tạo ra đúng 1 output**.

---

## 5. Tóm tắt ý chính

- `enable.idempotence=true`:
  - Đảm bảo producer không ghi trùng message vào cùng partition khi retry.
  - Tự động yêu cầu `acks=all`, `retries` cao, `max.in.flight` giới hạn.
- EOS đầy đủ:
  - Kết hợp idempotent producer + transaction + commit offset trong transaction.
  - Đảm bảo **exactly-once** cho pipeline đọc → xử lý → ghi.
