## 1. `KafkaTemplate` là gì?

- Là class trung tâm trong Spring Kafka dùng để **gửi (produce) message** lên Kafka.
- Được cấu hình như một Spring bean, sử dụng `ProducerFactory` bên dưới.
- Hỗ trợ:
  - Gửi message theo topic, key, value.
  - Gửi đồng bộ hoặc bất đồng bộ (async).
  - Nhận callback khi gửi thành công/thất bại.

---

## 2. Cấu hình cơ bản

### 2.1. Maven dependency

```xml
<dependency>
    <groupId>org.springframework.kafka</groupId>
    <artifactId>spring-kafka</artifactId>
</dependency>
```

### 2.2. Cấu hình ProducerFactory & KafkaTemplate

```java
@Configuration
public class KafkaProducerConfig {

    @Bean
    public ProducerFactory<String, String> producerFactory() {
        Map<String, Object> configProps = new HashMap<>();
        configProps.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        configProps.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        configProps.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        return new DefaultKafkaProducerFactory<>(configProps);
    }

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate() {
        return new KafkaTemplate<>(producerFactory());
    }
}
```

## 3. Gửi message với `KafkaTemplate`

### 3.1. Gửi đơn giản (fire-and-forget)

```java
@Service
public class SimpleProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public SimpleProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String topic, String message) {
        kafkaTemplate.send(topic, message);
    }
}
```

**Input (ví dụ):**

```java
simpleProducer.sendMessage("demo-topic", "Xin chào Kafka");
```

**Output (trên Kafka):**

- Topic: `demo-topic`
- Partition: (Kafka tự chọn)
- Key: `null`
- Value: `"Xin chào Kafka"`

---

### 3.2. Gửi với key

```java
public void sendMessageWithKey(String topic, String key, String message) {
    kafkaTemplate.send(topic, key, message);
}
```

**Input:**

```java
sendMessageWithKey("user-topic", "user-123", "User created");
```

**Output (trên Kafka):**

- Topic: `user-topic`
- Key: `"user-123"` (dùng để xác định partition)
- Value: `"User created"`

---

### 3.3. Gửi bất đồng bộ với callback

```java
public void sendWithCallback(String topic, String message) {
    ListenableFuture<SendResult<String, String>> future =
            kafkaTemplate.send(topic, message);

    future.addCallback(
        result -> {
            RecordMetadata metadata = result.getRecordMetadata();
            System.out.printf("Sent to topic=%s partition=%d offset=%d%n",
                    metadata.topic(), metadata.partition(), metadata.offset());
        },
        ex -> {
            System.err.println("Failed to send message: " + ex.getMessage());
        }
    );
}
```

**Input:**

```java
sendWithCallback("log-topic", "Log message");
```

**Output (console):**

- Thành công:

  ```
  Sent to topic=log-topic partition=0 offset=15
  ```

- Thất bại:

  ```
  Failed to send message: <error message>
  ```

---

### 3.4. Gửi đồng bộ (chờ kết quả)

```java
public void sendSync(String topic, String message) throws ExecutionException, InterruptedException {
    SendResult<String, String> result =
            kafkaTemplate.send(topic, message).get(); // block

    System.out.println("Offset: " + result.getRecordMetadata().offset());
}
```

## 4. Gửi object (JSON)

### 4.1. Cấu hình serializer JSON

```java
@Bean
public ProducerFactory<String, MyEvent> jsonProducerFactory() {
    Map<String, Object> configProps = new HashMap<>();
    configProps.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
    configProps.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
    configProps.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, JsonSerializer.class);
    return new DefaultKafkaProducerFactory<>(configProps);
}

@Bean
public KafkaTemplate<String, MyEvent> jsonKafkaTemplate() {
    return new KafkaTemplate<>(jsonProducerFactory());
}
```

### 4.2. Gửi object

```java
public class MyEvent {
    private String id;
    private String content;
    // getters/setters
}

@Service
public class EventProducer {

    private final KafkaTemplate<String, MyEvent> jsonKafkaTemplate;

    public EventProducer(KafkaTemplate<String, MyEvent> jsonKafkaTemplate) {
        this.jsonKafkaTemplate = jsonKafkaTemplate;
    }

    public void sendEvent(MyEvent event) {
        jsonKafkaTemplate.send("event-topic", event.getId(), event);
    }
}
```

**Input:**

```java
MyEvent event = new MyEvent();
event.setId("evt-1");
event.setContent("Nội dung sự kiện");
eventProducer.sendEvent(event);
```

**Output (trên Kafka – dạng JSON):**

```json
{
  "id": "evt-1",
  "content": "Nội dung sự kiện"
}
```

---

## 5. Ý chính cần nhớ

- `KafkaTemplate` là API chính trong Spring Kafka để **produce message**.
- Cần cấu hình:
  - `ProducerFactory` (bootstrap servers, serializer, v.v.).
  - `KafkaTemplate` bean.
- Hỗ trợ:
  - Gửi message với topic, key, value.
  - Gửi async với callback hoặc sync với `.get()`.
  - Gửi object (JSON) với `JsonSerializer`.
