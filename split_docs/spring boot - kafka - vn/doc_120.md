
Dưới đây là tóm tắt ngắn gọn về **Kafka Streams state**: state store, changelog topic, và fault tolerance, kèm ví dụ code (Java/Spring Boot style) và input/output minh họa.

---

## 1. State trong Kafka Streams là gì?

- Kafka Streams không chỉ xử lý từng record độc lập, mà còn có thể **lưu trạng thái (state)** để:
  - Đếm số lượng (count)
  - Tổng hợp (aggregate)
  - Join theo thời gian (windowed join)
- Trạng thái này được lưu trong **State Store** (bộ nhớ cục bộ trên mỗi instance).

---

## 2. State Store

- Là nơi Kafka Streams lưu dữ liệu trạng thái cục bộ:
  - Thường dùng **RocksDB** (on-disk key-value store) + cache in-memory.
  - Có thể là:
    - `KeyValueStore<K, V>`
    - `WindowStore<K, V>`
    - `SessionStore<K, V>`
- Truy cập được:
  - Bên trong topology (trong processor/transformer).
  - Từ bên ngoài qua **Interactive Queries** (REST API, v.v.).

### Ví dụ: Đếm số lượng theo key (Word Count)

#### Topology (Java)

```java
@Bean
public KStream<String, String> kStream(StreamsBuilder builder) {
    KStream<String, String> textLines = builder.stream("input-topic");

    KTable<String, Long> wordCounts = textLines
            .flatMapValues(value -> Arrays.asList(value.toLowerCase().split("\\W+")))
            .groupBy((key, word) -> word)
            .count(Materialized.as("word-count-store")); // state store tên "word-count-store"

    wordCounts.toStream().to("output-topic", Produced.with(Serdes.String(), Serdes.Long()));

    return textLines;
}
```

- `count(Materialized.as("word-count-store"))` tạo một **state store** tên `word-count-store`.
- Store này lưu: `word -> currentCount`.

#### Input / Output

- Input topic: `input-topic`
  - `"hello kafka"`
  - `"hello streams"`
- State store (word-count-store) sau khi xử lý:
  - `hello -> 2`
  - `kafka -> 1`
  - `streams -> 1`
- Output topic: `output-topic`
  - `("hello", 1)`
  - `("kafka", 1)`
  - `("hello", 2)`
  - `("streams", 1)`

## 3. Changelog Topic

- Mỗi state store có một **changelog topic** tương ứng trong Kafka.
- Mọi thay đổi trong state store (put/update/delete) đều được ghi vào changelog topic.
- Changelog topic:
  - Được **partition** tương ứng với task/partition của input.
  - Thường có **replication factor > 1** để đảm bảo an toàn dữ liệu.

### Vai trò

- Khi instance bị lỗi hoặc restart:
  - Kafka Streams **rebuild state store** bằng cách:
    - Load snapshot (nếu có)
    - Replay changelog topic để khôi phục trạng thái mới nhất.

---

## 4. Fault Tolerance (Chịu lỗi)

Kafka Streams đảm bảo fault tolerance cho state bằng:

1. **State store cục bộ + changelog topic**:
   - State cục bộ nhanh (local).
   - Changelog topic là nguồn sự thật (source of truth) để khôi phục.

2. **Rebalancing**:
   - Khi một instance chết, các task của nó được chuyển sang instance khác.
   - Instance mới:
     - Đọc changelog topic tương ứng.
     - Xây lại state store.
     - Tiếp tục xử lý từ offset mới nhất.

3. **Exactly-once processing (tùy chọn)**:
   - Cấu hình `processing.guarantee=exactly_once_v2` để tránh double-processing khi failover.

## 5. Ví dụ Spring Boot + Kafka Streams (có state store)

### application.yml

```yaml
spring:
  kafka:
    streams:
      application-id: word-count-app
      bootstrap-servers: localhost:9092
      properties:
        processing.guarantee: exactly_once_v2
```

### Config + Topology

```java
@Configuration
@EnableKafkaStreams
public class KafkaStreamsConfig {

    @Bean
    public KStream<String, String> kStream(StreamsBuilder builder) {
        KStream<String, String> textLines = builder.stream("input-topic");

        KTable<String, Long> wordCounts = textLines
                .flatMapValues(value -> Arrays.asList(value.toLowerCase().split("\\W+")))
                .groupBy((key, word) -> word)
                .count(Materialized.as("word-count-store"));

        wordCounts.toStream().to("output-topic", Produced.with(Serdes.String(), Serdes.Long()));

        return textLines;
    }
}
```

## 6. Truy vấn state store (Interactive Query)

Bạn có thể expose REST API để đọc state hiện tại (ví dụ: đếm của một từ):

```java
@RestController
public class WordCountController {

    private final InteractiveQueryService interactiveQueryService;

    public WordCountController(InteractiveQueryService interactiveQueryService) {
        this.interactiveQueryService = interactiveQueryService;
    }

    @GetMapping("/count/{word}")
    public Long getCount(@PathVariable String word) {
        ReadOnlyKeyValueStore<String, Long> store =
                interactiveQueryService.getQueryableStore("word-count-store", QueryableStoreTypes.keyValueStore());
        Long count = store.get(word);
        return count == null ? 0L : count;
    }
}
```

### Ví dụ request/response

- Request: `GET /count/hello`
- Response: `2`

## Tóm tắt ý chính

- **State store**: Lưu trạng thái cục bộ (key-value, window, session) cho các phép toán stateful (count, aggregate, join).
- **Changelog topic**: Ghi lại mọi thay đổi của state store vào Kafka để có thể khôi phục.
- **Fault tolerance**:
  - Khi instance lỗi, state được rebuild từ changelog topic.
  - Hỗ trợ exactly-once để tránh xử lý trùng.
- Spring Boot + Kafka Streams:
  - Dễ cấu hình state store.
  - Có thể truy vấn state qua REST (Interactive Queries).
