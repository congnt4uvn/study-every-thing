
Dưới đây là tóm tắt ngắn gọn về **Spring Kafka consumer: Concurrency và partition assignment**, kèm ví dụ code và input/output minh họa.

---

## 1. Khái niệm chính

1. **Consumer group & partition**
   - Mỗi topic có nhiều **partition**.
   - Mỗi consumer thuộc một **consumer group**.
   - **Một partition chỉ được xử lý bởi tối đa 1 consumer trong cùng một group** tại một thời điểm.
   - Nếu số consumer > số partition → một số consumer sẽ “nhàn rỗi” (không nhận message).

2. **`concurrency` trong Spring Kafka**
   - Thuộc tính `concurrency` (trong `@KafkaListener` hoặc container factory) tạo ra **nhiều consumer thread** trong cùng một consumer group.
   - Mỗi thread là một Kafka consumer riêng, cùng `groupId`.
   - Số thread hiệu quả tối đa = số partition của topic.
   - Nếu `concurrency` > số partition → một số thread không nhận message.

3. **Partition assignment**
   - Kafka broker sẽ **tự động phân phối partition** cho các consumer trong group.
   - Khi số consumer thay đổi (thêm/bớt), Kafka sẽ **rebalance** lại việc gán partition.
   - Có thể **chỉ định partition cụ thể** cho listener nếu muốn kiểm soát chặt (ví dụ: chỉ đọc partition 0,1).

## 2. Ví dụ cơ bản với `@KafkaListener` và `concurrency`

### Cấu hình consumer

```java
@Configuration
@EnableKafka
public class KafkaConsumerConfig {

    @Bean
    public Map<String, Object> consumerConfigs() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "demo-group");
        props.put(ConsumerConfig.AUTO_OFFSET_RESET_CONFIG, "earliest");
        return props;
    }

    @Bean
    public ConsumerFactory<String, String> consumerFactory() {
        return new DefaultKafkaConsumerFactory<>(consumerConfigs());
    }

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory());
        // Có thể set concurrency mặc định ở đây
        factory.setConcurrency(3); // 3 consumer thread
        return factory;
    }
}
```

### Listener với concurrency

```java
@Component
public class DemoConsumer {

    @KafkaListener(
        topics = "demo-topic",
        groupId = "demo-group",
        concurrency = "3" // 3 consumer thread cho listener này
    )
    public void listen(String message,
                       @Header(KafkaHeaders.RECEIVED_PARTITION_ID) int partition,
                       @Header(KafkaHeaders.OFFSET) long offset,
                       @Header(KafkaHeaders.RECEIVED_MESSAGE_KEY) String key) {

        String threadName = Thread.currentThread().getName();
        System.out.printf(
            "Thread=%s | Partition=%d | Offset=%d | Key=%s | Message=%s%n",
            threadName, partition, offset, key, message
        );
    }
}
```

### Input/Output minh họa

Giả sử:
- Topic `demo-topic` có **3 partition**: 0, 1, 2.
- `concurrency = 3` → 3 consumer thread trong group `demo-group`.
- Producer gửi 6 message:

```text
Key=k1, Value="A"
Key=k2, Value="B"
Key=k3, Value="C"
Key=k4, Value="D"
Key=k5, Value="E"
Key=k6, Value="F"
```

Kafka sẽ phân phối message theo partition (ví dụ):

- Partition 0: (k1, "A"), (k4, "D")
- Partition 1: (k2, "B"), (k5, "E")
- Partition 2: (k3, "C"), (k6, "F")

Output console (minh họa):

```text

  | Partition=2 | Offset=0 | Key=k3 | Message=C
```

Mỗi thread xử lý một hoặc nhiều partition, nhưng **một partition không bị xử lý bởi nhiều thread trong cùng group**.

## 3. Chỉ định partition cụ thể cho listener

Nếu muốn kiểm soát partition assignment ở mức Spring (thay vì để Kafka tự phân phối):

```java
@Component
public class SpecificPartitionConsumer {

    @KafkaListener(
        topicPartitions = @TopicPartition(
            topic = "demo-topic",
            partitions = {"0", "1"} // chỉ đọc partition 0 và 1
        ),
        groupId = "demo-group-specific",
        concurrency = "2"
    )
    public void listenSpecific(String message,
                               @Header(KafkaHeaders.RECEIVED_PARTITION_ID) int partition) {
        System.out.printf("Specific listener | Partition=%d | Message=%s%n", partition, message);
    }
}
```

- Consumer này **chỉ nhận message từ partition 0 và 1** của `demo-topic`.
- Partition 2 sẽ không được listener này xử lý (có thể do consumer khác xử lý).

---

## 4. Các điểm cần nhớ

1. `concurrency` = số consumer thread trong Spring → tối đa nên bằng hoặc nhỏ hơn tổng số partition mà listener đọc.
2. Kafka đảm bảo **một partition chỉ có một consumer trong cùng group** → đảm bảo thứ tự message trong partition.
3. Tăng `concurrency` giúp tăng throughput, nhưng:
   - Không vượt quá số partition.
   - Cần đủ tài nguyên (CPU, connection, memory).
4. Có thể:
   - Để Kafka tự gán partition (thông thường).
   - Hoặc dùng `topicPartitions` để chỉ định partition cụ thể.

Nếu bạn gửi nội dung chi tiết hơn (ví dụ đoạn text gốc), tôi có thể tóm tắt sát hơn từng ý và bổ sung thêm ví dụ nâng cao (seek, assign, manual ack, v.v.).
