
Dưới đây là tóm tắt ngắn gọn về 3 config consumer Kafka: `fetch.min.bytes`, `fetch.max.wait.ms`, `max.partition.fetch.bytes` trong bối cảnh Spring Boot.

---

## 1. `fetch.min.bytes`

**Ý nghĩa**  
- Số byte tối thiểu mà broker phải gom đủ trước khi trả về một batch dữ liệu cho consumer.  
- Giúp **giảm số lần request** (tăng throughput) nhưng có thể **tăng độ trễ** nếu dữ liệu ít.

**Ví dụ cấu hình (application.yml)**

```yaml
spring:
  kafka:
    consumer:
      properties:
        fetch.min.bytes: 1048576   # 1 MB
```

**Tác động (input → output)**  
- Input: `fetch.min.bytes = 1_000_000`, tốc độ sinh message thấp.  
- Output: Consumer sẽ **chờ lâu hơn** để đủ 1MB dữ liệu mới trả về, giảm số lần poll nhưng tăng latency.

---

## 2. `fetch.max.wait.ms`

**Ý nghĩa**  
- Thời gian tối đa broker sẽ **chờ** để gom đủ `fetch.min.bytes`.  
- Nếu hết thời gian này mà chưa đủ dữ liệu, broker vẫn trả về những gì đang có.

**Ví dụ cấu hình**

```yaml
spring:
  kafka:
    consumer:
      properties:
        fetch.max.wait.ms: 500   # 500 ms
```

**Tác động (input → output)**  
- Input: `fetch.min.bytes = 1MB`, `fetch.max.wait.ms = 500ms`.  
- Output:  
  - Nếu trong 500ms gom đủ ≥ 1MB → trả về batch ≥ 1MB.  
  - Nếu không đủ → sau 500ms vẫn trả về batch nhỏ hơn 1MB (giảm latency).

---

## 3. `max.partition.fetch.bytes`

**Ý nghĩa**  
- Số byte tối đa **mỗi partition** có thể trả về trong một lần fetch.  
- Giới hạn kích thước batch per partition, tránh **OOM** (Out Of Memory) ở consumer.

**Ví dụ cấu hình**

```yaml
spring:
  kafka:
    consumer:
      properties:
        max.partition.fetch.bytes: 1048576   # 1 MB mỗi partition
```

**Tác động (input → output)**  
- Input: `max.partition.fetch.bytes = 1MB`, message lớn 2MB.  
- Output:  
  - Consumer **không thể** fetch message 2MB (vì vượt limit) → có thể gây lỗi hoặc không consume được message đó.  
  - Cần đảm bảo `max.partition.fetch.bytes` ≥ `max.message.bytes` của topic.

## Ví dụ Spring Boot Consumer (Java)

```java
import org.apache.kafka.clients.consumer.ConsumerConfig;
import org.apache.kafka.common.serialization.StringDeserializer;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.config.ConcurrentKafkaListenerContainerFactory;
import org.springframework.kafka.core.ConsumerFactory;
import org.springframework.kafka.core.DefaultKafkaConsumerFactory;

import java.util.HashMap;
import java.util.Map;

@Configuration
public class KafkaConsumerConfig {

    @Bean
    public ConsumerFactory<String, String> consumerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "demo-group");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);

        // Các config chính
        props.put(ConsumerConfig.FETCH_MIN_BYTES_CONFIG, 1_048_576);        // 1MB
        props.put(ConsumerConfig.FETCH_MAX_WAIT_MS_CONFIG, 500);           // 500ms
        props.put(ConsumerConfig.MAX_PARTITION_FETCH_BYTES_CONFIG, 1_048_576); // 1MB/partition

        return new DefaultKafkaConsumerFactory<>(props);
    }

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String>
    kafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory());
        return factory;
    }
}
```

**Consumer sử dụng:**

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class DemoConsumer {

    @KafkaListener(topics = "demo-topic", groupId = "demo-group")
    public void listen(String message) {
        System.out.println("Received: " + message);
    }
}
```

## Tóm tắt nhanh

- `fetch.min.bytes`:  
  - Tăng → ít request hơn, throughput cao hơn, nhưng latency cao hơn.
- `fetch.max.wait.ms`:  
  - Giới hạn thời gian chờ để gom đủ `fetch.min.bytes`.  
  - Tăng → chấp nhận chờ lâu hơn để có batch lớn.
- `max.partition.fetch.bytes`:  
  - Giới hạn kích thước dữ liệu mỗi partition trong một lần fetch.  
  - Phải đủ lớn để chứa message lớn nhất, tránh lỗi không consume được.
