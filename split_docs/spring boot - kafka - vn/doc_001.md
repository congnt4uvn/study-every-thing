
Dưới đây là tóm tắt ngắn gọn về kiến trúc Kafka (broker, topic, partition) kèm ví dụ Spring Boot.

---

## 1. Khái niệm chính trong Kafka

### 1.1 Broker
- Là một server Kafka.
- Nhiều broker tạo thành một Kafka cluster.
- Nhiệm vụ:
  - Lưu trữ dữ liệu (messages) trong các partition.
  - Xử lý yêu cầu từ producer và consumer.
  - Đảm bảo tính phân tán và chịu lỗi (replication).

### 1.2 Topic
- Là “kênh” logic để gửi/nhận message.
- Producer ghi message vào topic, consumer đọc message từ topic.
- Một topic được chia thành nhiều partition để:
  - Tăng khả năng song song (parallelism).
  - Tăng khả năng mở rộng (scalability).

### 1.3 Partition
- Là đơn vị lưu trữ vật lý của một topic.
- Mỗi partition là một log được ghi nối tiếp (append-only).
- Mỗi message trong partition có một offset (chỉ số) duy nhất.
- Cho phép:
  - Nhiều consumer trong cùng một consumer group đọc song song.
  - Phân tán dữ liệu trên nhiều broker.

---

## 2. Luồng dữ liệu cơ bản

1. Producer gửi message đến một topic.
2. Kafka phân bổ message vào một partition (theo key hoặc round-robin).
3. Message được lưu trên broker tương ứng.
4. Consumer đọc message từ các partition của topic đó, theo offset.

## 3. Ví dụ Spring Boot + Kafka

### 3.1 Cấu hình Maven (pom.xml)

```xml
<dependencies>
    <!-- Spring Boot Web (nếu cần REST API) -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>

    <!-- Spring for Apache Kafka -->
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>
</dependencies>
```

### 3.2 Cấu hình application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
    consumer:
      group-id: demo-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
```

Giả sử bạn đã tạo topic: `demo-topic` với 3 partition.

---

### 3.3 Producer (gửi message)

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class KafkaProducerService {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private static final String TOPIC = "demo-topic";

    public KafkaProducerService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    // Gửi message không có key (Kafka sẽ phân bổ partition)
    public void sendMessage(String message) {
        kafkaTemplate.send(TOPIC, message);
    }

    // Gửi message có key (cùng key -> cùng partition)
    public void sendMessageWithKey(String key, String message) {
        kafkaTemplate.send(TOPIC, key, message);
    }
}
```

#### Ví dụ Input/Output

- Input (REST API):

```http
POST /api/send
Content-Type: application/json

{
  "key": "user-1",
  "message": "Xin chào Kafka"
}
```

- Xử lý:
  - `KafkaProducerService.sendMessageWithKey("user-1", "Xin chào Kafka")`
  - Message được ghi vào topic `demo-topic`, partition được chọn dựa trên key `"user-1"`.

- Output (log trên broker/consumer):
  - Topic: `demo-topic`
  - Partition: ví dụ `1`
  - Offset: ví dụ `15`
  - Value: `"Xin chào Kafka"`

---

### 3.4 Consumer (nhận message)

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class KafkaConsumerService {

    @KafkaListener(topics = "demo-topic", groupId = "demo-group")
    public void listen(String message) {
        System.out.println("Nhận message: " + message);
    }
}
```

#### Ví dụ Output

Khi producer gửi:

```java
producerService.sendMessage("Hello Kafka");
```

Consumer log:

```text
Nhận message: Hello Kafka
```

---

## 4. Tóm tắt ngắn gọn

- **Broker**: server Kafka, lưu trữ và xử lý message.
- **Topic**: kênh logic để gửi/nhận message.
- **Partition**: chia nhỏ topic, là log tuần tự, giúp tăng song song và mở rộng.
- Spring Boot dùng `KafkaTemplate` để gửi message và `@KafkaListener` để nhận message từ topic/partition trong Kafka.
