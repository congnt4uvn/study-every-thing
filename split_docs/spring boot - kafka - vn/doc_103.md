
Dưới đây là tóm tắt ngắn gọn về **Event-driven architecture** và **Request/Response** trong bối cảnh Kafka + Spring Boot, kèm ví dụ code và input/output.

---

## 1. Khái niệm chính

### a) Request/Response (Yêu cầu/Phản hồi)
- Mô hình truyền thống (HTTP, REST).
- Client gửi request → Server xử lý → trả response ngay lập tức.
- Đặc điểm:
  - Đồng bộ (synchronous): client chờ kết quả.
  - Phù hợp cho: truy vấn dữ liệu, thao tác CRUD, khi cần phản hồi tức thì.
  - Ràng buộc chặt giữa client và server (coupling cao hơn).

### b) Event-driven Architecture (Kiến trúc hướng sự kiện)
- Hệ thống giao tiếp qua **event** (sự kiện) được publish lên message broker (Kafka).
- Producer phát sự kiện → Kafka topic → Consumer lắng nghe và xử lý.
- Đặc điểm:
  - Thường là bất đồng bộ (asynchronous).
  - Giảm coupling: producer không cần biết consumer là ai.
  - Phù hợp cho: xử lý nền (background), tích hợp nhiều service, audit log, event sourcing, streaming.

---

## 2. So sánh nhanh

| Tiêu chí                 | Request/Response                 | Event-driven (Kafka)                          |
|--------------------------|----------------------------------|----------------------------------------------|
| Kiểu giao tiếp           | Đồng bộ                         | Bất đồng bộ (chủ yếu)                        |
| Cơ chế                   | HTTP/REST, gRPC                 | Kafka topics (publish/subscribe)             |
| Phụ thuộc giữa service   | Coupling cao hơn                | Coupling thấp hơn                            |
| Thời gian phản hồi       | Ngay lập tức                    | Có thể trễ (tùy consumer)                    |
| Trường hợp sử dụng       | API CRUD, query, form submit    | Xử lý nền, tích hợp hệ thống, event sourcing |

## 3. Ví dụ với Spring Boot + Kafka

### 3.1. Event-driven với Kafka

#### Producer: gửi sự kiện khi tạo đơn hàng

**Input (HTTP request từ client):**
```http
POST /orders
Content-Type: application/json

{
  "orderId": "O123",
  "userId": "U1",
  "amount": 100000
}
```

**Controller + Producer:**
```java
@RestController
@RequestMapping("/orders")
public class OrderController {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public OrderController(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    @PostMapping
    public ResponseEntity<String> createOrder(@RequestBody OrderRequest request) {
        // 1. Lưu đơn hàng vào DB (giả sử đã làm)
        // 2. Gửi event sang Kafka
        String event = """
            {
              "orderId": "%s",
              "userId": "%s",
              "amount": %d,
              "status": "CREATED"
            }
            """.formatted(request.getOrderId(), request.getUserId(), request.getAmount());

        kafkaTemplate.send("order-created-topic", request.getOrderId(), event);

        // Response cho client (request/response bình thường)
        return ResponseEntity.ok("Order created and event published");
    }
}
```

**Output (HTTP response cho client):**
```http
200 OK
Order created and event published
```

**Event trong Kafka topic `order-created-topic`:**
```json
{
  "orderId": "O123",
  "userId": "U1",
  "amount": 100000,
  "status": "CREATED"
}
```

#### Consumer: lắng nghe và xử lý event

```java
@Service
public class OrderCreatedListener {

    @KafkaListener(topics = "order-created-topic", groupId = "payment-service")
    public void handleOrderCreated(String message) {
        // Parse JSON, xử lý thanh toán, gửi email, v.v.
        System.out.println("Received order event: " + message);
    }
}
```

**Kết quả (output log):**
```text
Received order event: {"orderId":"O123","userId":"U1","amount":100000,"status":"CREATED"}
```

Ở đây:
- API `/orders` vẫn là **request/response** với client.
- Giữa các service nội bộ (Order → Payment, Notification, …) dùng **event-driven** qua Kafka.

### 3.2. Request/Response thuần túy (không Kafka)

**Controller:**
```java
@RestController
@RequestMapping("/orders")
public class OrderController {

    @PostMapping
    public ResponseEntity<OrderResponse> createOrder(@RequestBody OrderRequest request) {
        // Xử lý đồng bộ: validate, lưu DB, tính toán, v.v.
        OrderResponse response = new OrderResponse(
                request.getOrderId(),
                request.getUserId(),
                request.getAmount(),
                "CREATED"
        );
        return ResponseEntity.ok(response);
    }
}
```

**Input:**
```http
POST /orders
Content-Type: application/json

{
  "orderId": "O123",
  "userId": "U1",
  "amount": 100000
}
```

**Output:**
```http
200 OK
Content-Type: application/json

{
  "orderId": "O123",
  "userId": "U1",
  "amount": 100000,
  "status": "CREATED"
}
```

## 4. Khi nào dùng cái nào?

- **Chỉ Request/Response**:
  - Ứng dụng đơn giản, ít service.
  - Cần phản hồi ngay, không có nhiều xử lý nền phức tạp.

- **Kết hợp Request/Response + Event-driven (Kafka)**:
  - Microservices, nhiều hệ thống cần tích hợp.
  - Cần mở rộng (scalable), xử lý nền, audit, log, analytics.
  - Muốn giảm phụ thuộc giữa các service.
