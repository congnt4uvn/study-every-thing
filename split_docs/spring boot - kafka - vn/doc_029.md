
Dưới đây là tóm tắt ngắn gọn về 3 config quan trọng của Kafka Producer trong Spring Boot: `retries`, `delivery.timeout.ms`, `max.in.flight.requests.per.connection`.

---

## 1. `retries`
- **Ý nghĩa**: Số lần Kafka Producer sẽ thử gửi lại (retry) khi gửi message thất bại (ví dụ lỗi tạm thời trên broker).
- **Tác dụng**:
  - Tăng độ tin cậy (message ít bị mất hơn).
  - Có thể làm tăng độ trễ (latency) vì phải thử lại nhiều lần.
- **Lưu ý**: Kết hợp với `delivery.timeout.ms` để giới hạn tổng thời gian gửi + retry.

**Ví dụ cấu hình trong Spring Boot (`application.yml`):**
```yaml
spring:
  kafka:
    producer:
      bootstrap-servers: localhost:9092
      retries: 5
      properties:
        delivery.timeout.ms: 120000   # 120 giây
        max.in.flight.requests.per.connection: 1
```

---

## 2. `delivery.timeout.ms`
- **Ý nghĩa**: Thời gian tối đa (tính bằng ms) cho toàn bộ quá trình gửi 1 record:
  - Gồm: xếp hàng, batch, gửi, retry…
- **Nếu quá thời gian này**: Producer coi là thất bại và trả lỗi cho ứng dụng.
- **Quan hệ với `retries` và `request.timeout.ms`**:
  - `delivery.timeout.ms` ≈ tổng “budget” thời gian cho tất cả lần retry.
  - Nếu `retries` lớn nhưng `delivery.timeout.ms` quá nhỏ → thực tế không retry được nhiều.

**Ví dụ:**
- `retries = 5`
- `delivery.timeout.ms = 120000` (120s)  
→ Producer có thể retry nhiều lần trong 120s, nếu vẫn không gửi được thì báo lỗi.

---

## 3. `max.in.flight.requests.per.connection`
- **Ý nghĩa**: Số request gửi song song (chưa được ack) trên mỗi connection tới broker.
- **Tác dụng**:
  - Giá trị cao (ví dụ 5): tăng throughput (gửi nhanh hơn).
  - Giá trị thấp (ví dụ 1): đảm bảo **thứ tự** (ordering) khi kết hợp với `retries > 0`.
- **Lưu ý quan trọng**:
  - Nếu `retries > 0` và `max.in.flight.requests.per.connection > 1`, có thể bị **đảo thứ tự message** khi retry.
  - Để đảm bảo thứ tự tuyệt đối: đặt `max.in.flight.requests.per.connection = 1`.

## Ví dụ code Producer trong Spring Boot

**Cấu hình Producer (Java config):**
```java
import org.apache.kafka.clients.producer.ProducerConfig;
import org.apache.kafka.common.serialization.StringSerializer;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.core.DefaultKafkaProducerFactory;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.kafka.core.ProducerFactory;

import java.util.HashMap;
import java.util.Map;

@Configuration
public class KafkaProducerConfig {

    @Bean
    public ProducerFactory<String, String> producerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class);

        // Các config chính
        props.put(ProducerConfig.RETRIES_CONFIG, 5);
        props.put(ProducerConfig.DELIVERY_TIMEOUT_MS_CONFIG, 120000); // 120s
        props.put(ProducerConfig.MAX_IN_FLIGHT_REQUESTS_PER_CONNECTION, 1); // đảm bảo thứ tự

        return new DefaultKafkaProducerFactory<>(props);
    }

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate() {
        return new KafkaTemplate<>(producerFactory());
    }
}
```

**Service gửi message:**
```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class MyProducerService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public MyProducerService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String topic, String key, String value) {
        kafkaTemplate.send(topic, key, value)
            .whenComplete((result, ex) -> {
                if (ex != null) {
                    System.out.println("Gửi thất bại: " + ex.getMessage());
                } else {
                    System.out.println("Gửi thành công, offset = " +
                        result.getRecordMetadata().offset());
                }
            });
    }
}
```

## Ví dụ Input/Output

**Input (gọi service):**
```java
myProducerService.sendMessage("orders", "order-1", "Create order #1");
```

**Output (log có thể thấy):**
- Trường hợp thành công:
  ```text
  Gửi thành công, offset = 15
  ```
- Trường hợp broker lỗi tạm thời:
  - Producer sẽ tự động retry tối đa 5 lần trong 120s.
  - Nếu vẫn lỗi:
    ```text
    Gửi thất bại: org.apache.kafka.common.errors.TimeoutException: 
    Expiring 1 record(s) for orders-0:120000 ms has passed since batch creation
    ```

---

### Tóm tắt ngắn
- `retries`: Số lần thử gửi lại khi lỗi → tăng độ tin cậy, có thể tăng latency.
- `delivery.timeout.ms`: Thời gian tối đa cho toàn bộ quá trình gửi + retry → quá thời gian thì báo lỗi.
- `max.in.flight.requests.per.connection`: Số request song song → cao thì nhanh, nhưng có thể mất thứ tự; đặt = 1 để giữ thứ tự khi có retry.
