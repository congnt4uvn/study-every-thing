
Dưới đây là tóm tắt ngắn gọn về xử lý lỗi trong Spring Kafka, tập trung vào lỗi deserialize với `ErrorHandlingDeserializer`, kèm ví dụ code và input/output.

---

## 1. Mục tiêu của `ErrorHandlingDeserializer`

- Trong Kafka, khi consumer nhận message, nó phải **deserialize** key/value (ví dụ từ JSON → object Java).
- Nếu dữ liệu bị lỗi (sai format JSON, sai schema, v.v.), deserializer mặc định sẽ ném exception và **consumer có thể bị dừng**.
- `ErrorHandlingDeserializer` của Spring Kafka giúp:
  - Không làm chết consumer.
  - Chuyển lỗi deserialization thành một **record đặc biệt** để bạn xử lý (log, gửi sang DLT, bỏ qua, v.v.).

---

## 2. Cách cấu hình `ErrorHandlingDeserializer`

### 2.1. application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    consumer:
      group-id: my-group
      auto-offset-reset: earliest

      # Dùng ErrorHandlingDeserializer cho key/value
      key-deserializer: org.springframework.kafka.support.serializer.ErrorHandlingDeserializer
      value-deserializer: org.springframework.kafka.support.serializer.ErrorHandlingDeserializer

      # Deserializer thực sự (bên trong)
      properties:
        spring.deserializer.key.delegate.class: org.apache.kafka.common.serialization.StringDeserializer
        spring.deserializer.value.delegate.class: org.springframework.kafka.support.serializer.JsonDeserializer

        # Cấu hình JsonDeserializer
        spring.json.trusted.packages: "*"
        spring.json.value.default.type: com.example.demo.dto.MyMessage
```

**Ý nghĩa:**

- `ErrorHandlingDeserializer` là lớp “bọc ngoài”.
- `delegate.class` là deserializer thật (String, Json, Avro, …).
- Nếu delegate ném lỗi → `ErrorHandlingDeserializer` sẽ tạo ra `DeserializationException` và gói trong `ConsumerRecord`.

## 3. Xử lý lỗi trong @KafkaListener

### 3.1. Listener bình thường (message OK)

```java
@KafkaListener(topics = "my-topic", groupId = "my-group")
public void listen(MyMessage message) {
    // message đã deserialize OK
    System.out.println("Received: " + message);
}
```

**Input Kafka (OK):**

```json
{"id": 1, "content": "Hello"}
```

**Output log:**

```text
Received: MyMessage{id=1, content='Hello'}
```

---

### 3.2. Listener nhận cả header lỗi

Khi deserialize lỗi, `ErrorHandlingDeserializer` sẽ:

- Không ném exception trực tiếp.
- Đặt thông tin lỗi vào **header**: `KafkaHeaders.DLT_EXCEPTION_FQCN`, `KafkaHeaders.DLT_EXCEPTION_MESSAGE`, `KafkaHeaders.DLT_EXCEPTION_STACKTRACE`, v.v.
- Value có thể là `null`.

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.springframework.kafka.support.KafkaHeaders;
import org.springframework.messaging.handler.annotation.Header;
import org.springframework.messaging.handler.annotation.Payload;

@KafkaListener(topics = "my-topic", groupId = "my-group")
public void listenWithErrorHandling(
        @Payload(required = false) MyMessage message,
        @Header(KafkaHeaders.DLT_EXCEPTION_MESSAGE) String exceptionMessage,
        @Header(KafkaHeaders.DLT_EXCEPTION_FQCN) String exceptionClass,
        ConsumerRecord<?, ?> record) {

    if (message == null) {
        // Deserialize lỗi
        System.out.println("Deserialization error: " + exceptionClass + " - " + exceptionMessage);
        System.out.println("Raw value: " + record.value());
        // TODO: log, gửi sang DLT, bỏ qua, ...
    } else {
        // Deserialize OK
        System.out.println("Received: " + message);
    }
}
```

**Input Kafka (LỖI JSON):**

```text
{"id": "abc", "content": "Hello"}   // id phải là số, nhưng lại là string
```

**Output log (ví dụ):**

```text
Deserialization error: org.springframework.kafka.support.serializer.DeserializationException - 
JSON parse error: Cannot deserialize value of type `java.lang.Long` from String "abc"; ...

Raw value: {"id": "abc", "content": "Hello"}
```

## 4. Kết hợp với Error Handler / DLT

Bạn có thể dùng `DefaultErrorHandler` để:

- Retry.
- Gửi message lỗi sang **Dead Letter Topic (DLT)**.

### 4.1. Cấu hình `DefaultErrorHandler` với DLT

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.kafka.listener.DefaultErrorHandler;
import org.springframework.kafka.listener.DeadLetterPublishingRecoverer;
import org.springframework.util.backoff.FixedBackOff;

@Configuration
public class KafkaErrorConfig {

    @Bean
    public DefaultErrorHandler errorHandler(KafkaTemplate<Object, Object> template) {
        DeadLetterPublishingRecoverer recoverer =
                new DeadLetterPublishingRecoverer(template); // topic DLT mặc định: <topic>.DLT

        // Retry 2 lần, mỗi lần cách nhau 1s
        FixedBackOff backOff = new FixedBackOff(1000L, 2L);

        DefaultErrorHandler errorHandler = new DefaultErrorHandler(recoverer, backOff);

        // Có thể thêm logic bỏ qua một số exception
        // errorHandler.addNotRetryableExceptions(DeserializationException.class);

        return errorHandler;
    }
}
```

**Luồng:**

1. Deserialization lỗi → `ErrorHandlingDeserializer` tạo record với header lỗi.
2. Listener ném exception (hoặc bạn chủ động ném).
3. `DefaultErrorHandler` xử lý:
   - Retry theo backoff.
   - Nếu vẫn lỗi → gửi record sang `<topic>.DLT`.

---

## 5. Tóm tắt ý chính

- `ErrorHandlingDeserializer` giúp **không làm chết consumer** khi deserialize lỗi.
- Nó bọc deserializer thật (String, Json, Avro, …) và chuyển lỗi thành **DeserializationException** trong header.
- Bạn có thể:
  - Đọc header lỗi trong `@KafkaListener` để log/xử lý.
  - Kết hợp với `DefaultErrorHandler` + `DeadLetterPublishingRecoverer` để gửi message lỗi sang DLT.
- Cấu hình chủ yếu qua:
  - `spring.kafka.consumer.key-deserializer` / `value-deserializer` = `ErrorHandlingDeserializer`
  - `spring.deserializer.*.delegate.class` = deserializer thật.
