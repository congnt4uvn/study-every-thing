## 1. Mục tiêu chính

Khi tích hợp Kafka Streams với Spring (Spring Boot / Spring Kafka), ta thường cần:

1. **Giám sát trạng thái thread** của Kafka Streams (RUNNING, ERROR, DEAD, …).  
2. **Theo dõi độ trễ (lag)** giữa consumer và producer (bao xa so với offset mới nhất).  
3. **Theo dõi kích thước state store** (số bản ghi, dung lượng, v.v.) để biết ứng dụng có đang phình to bất thường hay không.

Spring giúp ta:
- Dễ dàng truy cập `KafkaStreams` instance.
- Expose metrics qua Actuator / Micrometer (Prometheus, Grafana).
- Đăng ký listener để theo dõi thay đổi trạng thái.

## 2. Giám sát trạng thái thread Kafka Streams

### Ý chính
- Kafka Streams có nhiều thread xử lý.
- Mỗi thread có trạng thái riêng (CREATED, RUNNING, PENDING_SHUTDOWN, DEAD, …).
- Ta có thể đăng ký `StateListener` hoặc `StateRestoreListener` để log, cảnh báo, hoặc cập nhật health check.

### Ví dụ: Đăng ký StateListener trong Spring Boot

```java
import org.apache.kafka.streams.KafkaStreams;
import org.apache.kafka.streams.KafkaStreams.State;
import org.apache.kafka.streams.KafkaStreams.StateListener;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class KafkaStreamsMonitoringConfig {

    @Bean
    public StateListener kafkaStreamsStateListener() {
        return new StateListener() {
            @Override
            public void onChange(State newState, State oldState) {
                System.out.println("Kafka Streams state changed from " + oldState + " to " + newState);
                // Ở đây có thể:
                // - Gửi metric
                // - Gửi alert
                // - Cập nhật health indicator
            }
        };
    }
}
```

**Input**: Thay đổi trạng thái nội bộ của Kafka Streams (ví dụ: từ REBALANCING sang RUNNING).  
**Output (log)**:
```text
Kafka Streams state changed from REBALANCING to RUNNING
```

Nếu dùng Spring Kafka Streams (`@EnableKafkaStreams`), bạn có thể inject `KafkaStreams` và set listener:

```java
import org.springframework.kafka.config.StreamsBuilderFactoryBean;
import org.springframework.beans.factory.annotation.Autowired;

@Configuration
public class KafkaStreamsConfig {

    @Autowired
    void configureStateListener(StreamsBuilderFactoryBean factoryBean,
                                StateListener stateListener) {
        factoryBean.setStateListener(stateListener);
    }
}
```

## 3. Giám sát lag (độ trễ) của Kafka Streams

### Ý chính
- Lag = `endOffset (trên broker)` – `currentCommittedOffset (consumer)`.
- Lag cao => ứng dụng xử lý không kịp dữ liệu mới.
- Có thể dùng:
  - Metrics sẵn có của Kafka (`records-lag-max`, `records-lag`).
  - Hoặc tự tính bằng cách lấy end offsets từ Kafka và committed offsets từ consumer group.

### Ví dụ: Lấy lag qua metrics (Micrometer + Actuator)

Trong Spring Boot, nếu bật Actuator và Micrometer Kafka, bạn có thể expose metrics:

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,prometheus,metrics
```

Kafka Streams sẽ expose metrics như:
- `kafka_consumer_records_lag_max`
- `kafka_consumer_records_lag`

**Input**: Prometheus scrape endpoint `/actuator/prometheus`.  
**Output (metric)**:
```text
kafka_consumer_records_lag_max{client_id="my-streams-app-1",topic="input-topic",partition="0"}  12
```

Nghĩa là partition 0 của `input-topic` đang có lag tối đa 12 bản ghi.

### Ví dụ: Tự tính lag (Java thuần)

```java
import org.apache.kafka.clients.admin.AdminClient;
import org.apache.kafka.clients.admin.ListConsumerGroupOffsetsResult;
import org.apache.kafka.clients.consumer.OffsetAndMetadata;
import org.apache.kafka.common.TopicPartition;
import org.apache.kafka.clients.consumer.KafkaConsumer;

import java.util.Map;
import java.util.Properties;

public class LagCalculator {

    public static void main(String[] args) throws Exception {
        String groupId = "my-streams-app";
        String topic = "input-topic";

        Properties adminProps = new Properties();
        adminProps.put("bootstrap.servers", "localhost:9092");
        try (AdminClient adminClient = AdminClient.create(adminProps)) {

            ListConsumerGroupOffsetsResult offsetsResult =
                    adminClient.listConsumerGroupOffsets(groupId);

            Map<TopicPartition, OffsetAndMetadata> committedOffsets =
                    offsetsResult.partitionsToOffsetAndMetadata().get();

            Properties consumerProps = new Properties();
            consumerProps.put("bootstrap.servers", "localhost:9092");
            consumerProps.put("group.id", "lag-checker");
            consumerProps.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
            consumerProps.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");

            try (KafkaConsumer<String, String> consumer = new KafkaConsumer<>(consumerProps)) {
                for (TopicPartition tp : committedOffsets.keySet()) {
                    if (!tp.topic().equals(topic)) continue;

                    consumer.assign(java.util.Collections.singleton(tp));
                    consumer.seekToEnd(java.util.Collections.singleton(tp));
                    long endOffset = consumer.position(tp);
                    long committed = committedOffsets.get(tp).offset();
                    long lag = endOffset - committed;

                    System.out.printf("Partition %s lag = %d (end=%d, committed=%d)%n",
                            tp.partition(), lag, endOffset, committed);
                }
            }
        }
    }
}
```

**Output (console)**:
```text
Partition 0 lag = 15 (end=120, committed=105)
Partition 1 lag = 0 (end=200, committed=200)
```

## 4. Giám sát kích thước state store

### Ý chính
- Kafka Streams dùng state store (RocksDB, in-memory) để lưu trạng thái (aggregation, join, window, …).
- Cần theo dõi:
  - Số bản ghi trong store.
  - Dung lượng (nếu có thể).
- Có thể truy cập store qua `KafkaStreams#store` hoặc expose metrics.

### Ví dụ: Lấy số bản ghi trong state store

Giả sử bạn có một `KeyValueStore<String, Long>` tên `my-store`:

```java
import org.apache.kafka.streams.KafkaStreams;
import org.apache.kafka.streams.state.QueryableStoreTypes;
import org.apache.kafka.streams.state.ReadOnlyKeyValueStore;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class StoreMonitoringController {

    private final KafkaStreams kafkaStreams;

    public StoreMonitoringController(KafkaStreams kafkaStreams) {
        this.kafkaStreams = kafkaStreams;
    }

    @GetMapping("/store-size")
    public long getStoreSize() {
        ReadOnlyKeyValueStore<String, Long> store =
                kafkaStreams.store("my-store", QueryableStoreTypes.keyValueStore());

        long count = 0;
        try (var iter = store.all()) {
            while (iter.hasNext()) {
                iter.next();
                count++;
            }
        }
        return count;
    }
}
```

**Input**: HTTP GET `/store-size`  
**Output (JSON)**:
```json
12345
```
Nghĩa là state store `my-store` hiện có 12.345 bản ghi.

### Ví dụ: Expose metrics state store qua Micrometer

Kafka Streams thường expose metrics như:
- `kafka_streams_stream_metrics_records_processed_total`
- `kafka_streams_stream_metrics_commit_total`
- Và metrics liên quan RocksDB (nếu dùng RocksDB).

Bạn có thể cấu hình Prometheus để scrape và vẽ dashboard (Grafana) cho:
- Số bản ghi xử lý.
- Số commit.
- Dung lượng RocksDB (nếu metric có sẵn).

## 5. Tích hợp với Spring Boot Actuator

### Ý chính
- Dùng Actuator để expose health, metrics, info.
- Có thể tạo custom `HealthIndicator` cho Kafka Streams.

### Ví dụ: HealthIndicator cho Kafka Streams

```java
import org.apache.kafka.streams.KafkaStreams;
import org.springframework.boot.actuate.health.Health;
import org.springframework.boot.actuate.health.HealthIndicator;
import org.springframework.stereotype.Component;

@Component
public class KafkaStreamsHealthIndicator implements HealthIndicator {

    private final KafkaStreams kafkaStreams;

    public KafkaStreamsHealthIndicator(KafkaStreams kafkaStreams) {
        this.kafkaStreams = kafkaStreams;
    }

    @Override
    public Health health() {
        KafkaStreams.State state = kafkaStreams.state();
        if (state == KafkaStreams.State.RUNNING) {
            return Health.up()
                    .withDetail("state", state.toString())
                    .build();
        } else {
            return Health.down()
                    .withDetail("state", state.toString())
                    .build();
        }
    }
}
```

**Input**: HTTP GET `/actuator/health`  
**Output (JSON)**:
```json
{
  "status": "UP",
  "components": {
    "kafkaStreamsHealthIndicator": {
      "status": "UP",
      "details": {
        "state": "RUNNING"
      }
    }
  }
}
```

## 6. Tóm tắt ngắn gọn

- **Thread states**: Dùng `StateListener` + Spring (`StreamsBuilderFactoryBean`) để log, alert, và health check trạng thái Kafka Streams.
- **Lag**:
  - Dùng metrics sẵn có (`records-lag-max`, `records-lag`) qua Micrometer/Actuator.
  - Hoặc tự tính bằng AdminClient + KafkaConsumer.
- **Store sizes**:
  - Truy vấn state store qua `kafkaStreams.store(...)` để đếm số bản ghi.
  - Expose endpoint REST hoặc metrics để giám sát.
- **Spring Boot Actuator**:
  - Tạo `HealthIndicator` cho Kafka Streams.
  - Expose metrics/health qua `/actuator/*` để tích hợp Prometheus, Grafana.

Nếu bạn cho biết phiên bản Spring Boot / Spring Kafka / Kafka Streams bạn dùng, tôi có thể điều chỉnh code cho đúng API cụ thể.
