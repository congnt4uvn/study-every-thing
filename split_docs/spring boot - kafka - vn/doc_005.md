## 1. Kafka Broker là gì?

**Kafka Broker** là server trong cluster Kafka, chịu trách nhiệm:

1. **Request Handling (Xử lý yêu cầu)**  
   - Nhận và xử lý request từ:
     - Producer (gửi message)
     - Consumer (đọc message)
     - Admin client (tạo topic, xóa topic, thay đổi cấu hình…)
   - Định tuyến request đến partition/leader phù hợp.

2. **Log Management (Quản lý log)**  
   - Lưu trữ dữ liệu của các topic dưới dạng **log** (file trên đĩa).
   - Mỗi partition là một log được append liên tục.
   - Quản lý:
     - Segment file
     - Retention (giữ dữ liệu theo thời gian/kích thước)
     - Index để truy cập nhanh theo offset.

3. **Replication (Sao chép dữ liệu)**  
   - Mỗi partition có:
     - **Leader**: broker chính nhận read/write.
     - **Follower**: broker sao chép dữ liệu từ leader.
   - Đảm bảo:
     - Tính sẵn sàng (high availability)
     - Tính bền vững (durability) khi một broker bị lỗi.

4. **Quotas (Hạn mức)**  
   - Giới hạn tài nguyên cho:
     - Producer (tốc độ gửi)
     - Consumer (tốc độ đọc)
     - Client theo user/client-id
   - Tránh một client “chiếm” hết tài nguyên cluster.

## 2. Ví dụ cấu hình Kafka Broker (server.properties)

```properties
# ID của broker
broker.id=1

# Địa chỉ listener
listeners=PLAINTEXT://localhost:9092

# Thư mục lưu log (dữ liệu topic/partition)
log.dirs=/var/lib/kafka/logs

# Cấu hình replication mặc định
default.replication.factor=3
min.insync.replicas=2

# Retention log
log.retention.hours=168   # giữ 7 ngày
log.segment.bytes=1073741824  # 1GB/segment

# Quotas
producer.quota.window.num=11
consumer.quota.window.num=11
```

## 3. Ví dụ Spring Boot Producer/Consumer

### 3.1. Cấu hình Maven

```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>
</dependencies>
```

### 3.2. application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
    consumer:
      group-id: demo-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
```

### 3.3. Producer (gửi message)

```java
@Service
public class DemoProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public DemoProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String topic, String key, String value) {
        kafkaTemplate.send(topic, key, value);
    }
}
```

**Input (REST):**

```http
POST /api/send
Content-Type: application/json

{
  "topic": "demo-topic",
  "key": "user-1",
  "value": "Xin chào Kafka"
}
```

**Controller:**

```java
@RestController
@RequestMapping("/api")
public class KafkaController {

    private final DemoProducer producer;

    public KafkaController(DemoProducer producer) {
        this.producer = producer;
    }

    @PostMapping("/send")
    public String send(@RequestBody MessageRequest request) {
        producer.sendMessage(request.getTopic(), request.getKey(), request.getValue());
        return "Sent";
    }
}

class MessageRequest {
    private String topic;
    private String key;
    private String value;
    // getters/setters
}
```

**Output (HTTP Response):**

```http
200 OK
Sent
```

---

### 3.4. Consumer (nhận message)

```java
@Service
public class DemoConsumer {

    @KafkaListener(topics = "demo-topic", groupId = "demo-group")
    public void listen(String message) {
        System.out.println("Received message: " + message);
    }
}
```

**Khi Producer gửi:**

```text
Topic: demo-topic
Key: user-1
Value: "Xin chào Kafka"
```

**Output trên consumer:**

```text
Received message: Xin chào Kafka
```

---

## 4. Liên hệ với vai trò Broker

- Khi `KafkaTemplate.send(...)` được gọi:
  - Broker **xử lý request** từ producer.
  - Ghi message vào **log** của partition tương ứng.
  - **Replicate** message sang các follower.
  - Áp dụng **quota** nếu producer gửi quá nhanh.

Tóm lại, Kafka Broker là “trái tim” của Kafka, chịu trách nhiệm nhận/ghi/nhân bản dữ liệu và quản lý tài nguyên cho toàn bộ cluster.
