
Dưới đây là tóm tắt ngắn gọn về “Monitoring & Observability” cho Kafka/Spring Boot, cùng ví dụ cấu hình:

---

## 1. Ý chính

- **Prometheus**: Thu thập metrics (số liệu) từ Kafka, Spring Boot (qua Actuator) để lưu trữ và query.
- **Grafana**: Vẽ dashboard, biểu đồ từ dữ liệu Prometheus (giám sát latency, throughput, lag, error…).
- **Kafka Manager** (hoặc CMAK): Công cụ web để:
  - Xem cluster, topic, partition, consumer group
  - Theo dõi consumer lag, trạng thái broker
- **Confluent Control Center**: Công cụ thương mại của Confluent:
  - Giám sát Kafka, ksqlDB, Connect
  - Theo dõi end-to-end latency, throughput, schema, alert…

Mục tiêu: **quan sát được hệ thống Kafka + ứng dụng Spring Boot** (metrics, log, trace) để:
- Phát hiện sớm lỗi
- Tối ưu hiệu năng
- Đảm bảo độ tin cậy

## 2. Ví dụ: Spring Boot + Prometheus + Grafana

### 2.1. Thêm dependency (Maven)

```xml
<dependencies>
    <!-- Actuator để expose metrics -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-actuator</artifactId>
    </dependency>

    <!-- Micrometer Prometheus registry -->
    <dependency>
        <groupId>io.micrometer</groupId>
        <artifactId>micrometer-registry-prometheus</artifactId>
    </dependency>

    <!-- Kafka -->
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>
</dependencies>
```

### 2.2. Cấu hình `application.yml`

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,prometheus
  endpoint:
    prometheus:
      enabled: true
  metrics:
    tags:
      application: kafka-demo-service

server:
  port: 8080
```

- Prometheus sẽ scrape metrics tại:  
  `http://localhost:8080/actuator/prometheus`

### 2.3. Ví dụ producer Kafka + metrics custom

```java
import io.micrometer.core.instrument.Counter;
import io.micrometer.core.instrument.MeterRegistry;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class KafkaProducerService {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private final Counter messageSentCounter;

    public KafkaProducerService(KafkaTemplate<String, String> kafkaTemplate,
                                MeterRegistry meterRegistry) {
        this.kafkaTemplate = kafkaTemplate;
        this.messageSentCounter = Counter.builder("kafka_messages_sent_total")
                .description("Tổng số message đã gửi lên Kafka")
                .tag("topic", "demo-topic")
                .register(meterRegistry);
    }

    public void sendMessage(String message) {
        kafkaTemplate.send("demo-topic", message);
        messageSentCounter.increment();
    }
}
```

**Input (REST)**:  
`POST /send?msg=hello`

**Output (Kafka)**:  
- Message `"hello"` được gửi lên topic `demo-topic`  
- Metrics Prometheus (ví dụ):

```text
# HELP kafka_messages_sent_total Tổng số message đã gửi lên Kafka
# TYPE kafka_messages_sent_total counter
kafka_messages_sent_total{application="kafka-demo-service",topic="demo-topic",} 1.0
```

### 2.4. Cấu hình Prometheus (ví dụ `prometheus.yml`)

```yaml
scrape_configs:
  - job_name: 'kafka-demo-service'
    metrics_path: '/actuator/prometheus'
    static_configs:
      - targets: ['host.docker.internal:8080']  # hoặc 'localhost:8080'
```

### 2.5. Grafana

- Thêm **Prometheus** làm Data Source.
- Import dashboard có sẵn (Spring Boot / JVM / Kafka) hoặc tự tạo:
  - `kafka_messages_sent_total`
  - `kafka_consumer_records_lag_max`
  - `jvm_memory_used_bytes`
  - `http_server_requests_seconds_count` …

## 3. Kafka Manager (CMAK)

- Chạy Kafka Manager (Docker):

```yaml
version: '3'
services:
  kafka-manager:
    image: hlebalbau/kafka-manager:latest
    environment:
      ZK_HOSTS: "zookeeper:2181"
    ports:
      - "9000:9000"
```

- Truy cập: `http://localhost:9000`
- Chức năng chính:
  - Xem danh sách topic, partition, replication factor
  - Xem consumer group, consumer lag
  - Thực hiện một số thao tác quản trị (tạo/xóa topic, rebalance…)

---

## 4. Confluent Control Center (tóm tắt)

- Đi kèm Confluent Platform / Confluent Cloud.
- Cho phép:
  - Giám sát cluster Kafka, Connect, ksqlDB
  - Theo dõi throughput, latency, error rate
  - Quản lý schema, alert, SLA
- Thường dùng trong môi trường production lớn, cần quan sát end-to-end.
