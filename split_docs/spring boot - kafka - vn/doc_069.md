
Dưới đây là tóm tắt ngắn gọn về **Security: Authorization (ACLs, resource patterns)** trong Kafka, kèm ví dụ code (Kafka + Spring Boot) và input/output minh họa.

---

## 1. Khái niệm chính

1. **Authorization trong Kafka**  
   - Kiểm soát *ai* được phép làm *gì* trên *tài nguyên nào* (topic, group, cluster, transactionalId, …).  
   - Dựa trên **ACLs (Access Control Lists)**.

2. **ACL (Access Control List)**  
   - Mỗi ACL định nghĩa:
     - Principal (người dùng / service): `User:alice`
     - Resource (tài nguyên): topic, group, cluster, …
     - Operation (hành động): READ, WRITE, CREATE, DELETE, ALTER, DESCRIBE, …
     - Permission: ALLOW hoặc DENY
   - ACL được lưu trên Kafka (Zookeeper hoặc KRaft, tùy phiên bản).

3. **Resource Patterns**  
   - Cách chỉ định tài nguyên mà ACL áp dụng:
     - `LITERAL`: chính xác tên tài nguyên (vd: topic `orders`)
     - `PREFIXED`: áp dụng cho tất cả tài nguyên bắt đầu với prefix (vd: `orders-` → `orders-1`, `orders-2`, …)
   - Giúp quản lý ACL dễ hơn khi có nhiều topic/group theo cùng một pattern.

## 2. Ví dụ ACL với Kafka CLI

### 2.1. Cho phép user `alice` đọc topic `orders`

```bash
kafka-acls.sh \
  --authorizer-properties zookeeper.connect=localhost:2181 \
  --add \
  --allow-principal User:alice \
  --operation READ \
  --topic orders
```

**Ý nghĩa**:  
- `User:alice` được **READ** từ topic `orders`.

**Input**: client `alice` gọi consumer đọc từ topic `orders`.  
**Output**: consumer đọc được message (nếu có).

---

### 2.2. Cho phép user `order-service` ghi vào tất cả topic bắt đầu bằng `orders-`

```bash
kafka-acls.sh \
  --authorizer-properties zookeeper.connect=localhost:2181 \
  --add \
  --allow-principal User:order-service \
  --operation WRITE \
  --topic orders- \
  --resource-pattern-type prefixed
```

**Ý nghĩa**:  
- `User:order-service` được **WRITE** vào mọi topic có tên bắt đầu bằng `orders-` (vd: `orders-created`, `orders-updated`).

**Input**: producer với principal `User:order-service` gửi message đến `orders-created`.  
**Output**: gửi thành công; gửi đến topic khác không có prefix `orders-` sẽ bị từ chối.

## 3. Cấu hình Authorization trong Kafka Broker

Trong `server.properties`:

```properties
# Bật cơ chế bảo mật
listeners=SASL_PLAINTEXT://:9092
security.inter.broker.protocol=SASL_PLAINTEXT
sasl.mechanism.inter.broker.protocol=PLAIN
sasl.enabled.mechanisms=PLAIN

# Bật authorizer
authorizer.class.name=kafka.security.authorizer.AclAuthorizer
allow.everyone.if.no.acl.found=false
super.users=User:admin
```

## 4. Spring Boot + Kafka: cấu hình client với SASL (để dùng ACL)

### 4.1. application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    properties:
      security.protocol: SASL_PLAINTEXT
      sasl.mechanism: PLAIN
      sasl.jaas.config: >
        org.apache.kafka.common.security.plain.PlainLoginModule required
        username="alice"
        password="alice-secret";
    consumer:
      group-id: order-consumer-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

Ở đây, principal sẽ là `User:alice` (tùy cấu hình trên broker).

## 5. Ví dụ code Spring Boot

### 5.1. Producer gửi message

```java
@Service
public class OrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public OrderProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendOrder(String orderId, String payload) {
        kafkaTemplate.send("orders", orderId, payload);
    }
}
```

**Input**:  
```java
orderProducer.sendOrder("order-1", "{ \"id\": \"order-1\", \"amount\": 100 }");
```

**Output**:  
- Nếu ACL cho phép `User:alice` WRITE vào topic `orders` → message được gửi thành công.  
- Nếu không có ACL phù hợp → lỗi `TopicAuthorizationException` hoặc `Not authorized to access topics`.

---

### 5.2. Consumer nhận message

```java
@Service
public class OrderConsumer {

    @KafkaListener(topics = "orders", groupId = "order-consumer-group")
    public void listen(ConsumerRecord<String, String> record) {
        System.out.println("Received order: key=" + record.key() + ", value=" + record.value());
    }
}
```

**Input**: message đã được gửi vào topic `orders`.  
**Output** (log):  
```text
Received order: key=order-1, value={ "id": "order-1", "amount": 100 }
```

Nếu ACL không cho phép `User:alice` READ từ topic `orders` → consumer không thể subscribe, sẽ nhận lỗi authorization.

---

## 6. Tóm tắt ngắn

- **Authorization** trong Kafka được thực hiện bằng **ACLs**.  
- ACL xác định: principal + resource + operation + ALLOW/DENY.  
- **Resource patterns** (LITERAL, PREFIXED) giúp áp dụng ACL cho nhiều topic/group theo pattern.  
- Spring Boot chỉ là client; việc cho phép/không cho phép phụ thuộc vào ACL trên Kafka broker.  
- Cần cấu hình **SASL + authorizer** trên broker và **SASL** trên Spring Boot client để ACL hoạt động.
