
Dưới đây là tóm tắt ngắn gọn về **Consumer Group Model** với hai khái niệm chính: **Static Membership** và **Cooperative Rebalancing** trong Kafka + Spring Boot.

---

## 1. Consumer Group Model (mô hình nhóm consumer)

- Một **consumer group** gồm nhiều consumer cùng đọc từ một hoặc nhiều topic.
- Mỗi **partition** của topic chỉ được xử lý bởi **tối đa 1 consumer** trong cùng một group.
- Khi số consumer thay đổi (thêm/bớt instance), Kafka sẽ **rebalance** để phân phối lại partition.

---

## 2. Static Membership (thành viên tĩnh)

### Ý tưởng chính
- Mặc định, Kafka dùng **dynamic membership**: khi một consumer rời group (do crash, restart, network issue), group sẽ rebalance.
- **Static membership** cho phép mỗi consumer có một **member.id cố định** (ví dụ: theo hostname, instance id), giúp:
  - Hạn chế rebalance không cần thiết khi **restart** hoặc **tạm mất kết nối ngắn**.
  - Giữ phân vùng (partition assignment) ổn định hơn.

### Cách dùng trong Spring Boot (Spring Kafka)

```java
import org.apache.kafka.clients.consumer.ConsumerConfig;
import org.apache.kafka.common.serialization.StringDeserializer;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.config.ConcurrentKafkaListenerContainerFactory;
import org.springframework.kafka.core.ConsumerFactory;
import org.springframework.kafka.core.DefaultKafkaConsumerFactory;

import java.util.HashMap;
import java.util.Map;

@Configuration
public class KafkaConsumerConfig {

    @Bean
    public ConsumerFactory<String, String> consumerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "order-group");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);

        // Static membership: gán group.instance.id cố định cho mỗi instance
        props.put(ConsumerConfig.GROUP_INSTANCE_ID_CONFIG, "order-service-instance-1");
        // Ví dụ: mỗi pod/container set giá trị khác nhau qua ENV

        return new DefaultKafkaConsumerFactory<>(props);
    }

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory());
        return factory;
    }
}
```

### Ví dụ Input/Output

- **Input**: Topic `orders`, 3 partition (0,1,2), group `order-group` có 3 instance:
  - instance-1: `group.instance.id = order-service-instance-1`
  - instance-2: `group.instance.id = order-service-instance-2`
  - instance-3: `group.instance.id = order-service-instance-3`

- **Lần đầu**:
  - P0 → instance-1  
  - P1 → instance-2  
  - P2 → instance-3  

- **Restart instance-2** (trong thời gian ngắn):
  - Với static membership: Kafka nhận ra instance-2 quay lại với cùng `group.instance.id`, **không cần rebalance toàn group**, partition P1 vẫn gán cho instance-2.
  - Giảm downtime và tránh “pause” xử lý.

---

## 3. Cooperative Rebalancing (rebalance hợp tác)

### Ý tưởng chính
- Cách cũ: **eager rebalancing** – khi rebalance, tất cả consumer **ngừng xử lý**, trả lại toàn bộ partition, rồi phân phối lại → gây “stop-the-world”.
- **Cooperative rebalancing**:
  - Consumer **từ từ** nhả partition (incremental).
  - Cho phép **chuyển giao partition dần dần**, giảm thời gian gián đoạn.
  - Hữu ích khi có nhiều consumer, nhiều partition, hoặc xử lý lâu.

### Cấu hình trong Spring Boot (Spring Kafka)

```java
import org.apache.kafka.clients.consumer.ConsumerConfig;
import org.apache.kafka.clients.consumer.CooperativeStickyAssignor;
import org.apache.kafka.common.serialization.StringDeserializer;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.config.ConcurrentKafkaListenerContainerFactory;
import org.springframework.kafka.core.ConsumerFactory;
import org.springframework.kafka.core.DefaultKafkaConsumerFactory;

import java.util.Arrays;
import java.util.HashMap;
import java.util.Map;

@Configuration
public class KafkaCooperativeConsumerConfig {

    @Bean
    public ConsumerFactory<String, String> cooperativeConsumerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "payment-group");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);

        // Dùng cooperative rebalancing
        props.put(
            ConsumerConfig.PARTITION_ASSIGNMENT_STRATEGY_CONFIG,
            Arrays.asList(CooperativeStickyAssignor.class.getName())
        );

        return new DefaultKafkaConsumerFactory<>(props);
    }

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> cooperativeKafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(cooperativeConsumerFactory());
        return factory;
    }
}
```

### Ví dụ hành vi

- **Trước khi scale**: 2 consumer (C1, C2), 4 partition (P0–P3)
  - C1: P0, P1  
  - C2: P2, P3  

- **Thêm C3**:
  - Cooperative rebalance:
    - Bước 1: C1 nhả P1, C2 nhả P3 (từng phần).
    - Bước 2: P1, P3 được gán cho C3.
    - Trong quá trình này, C1 vẫn xử lý P0, C2 vẫn xử lý P2 → **ít downtime hơn**.

---

## 4. Kết hợp trong thực tế

- **Static membership**:
  - Dùng khi service có **số instance cố định hoặc scale không quá linh hoạt**, muốn giảm rebalance khi restart.
- **Cooperative rebalancing**:
  - Dùng cho hệ thống **nhiều consumer, nhiều partition**, cần **tối thiểu gián đoạn** khi scale in/out.

Có thể kết hợp cả hai:
- `group.instance.id` để có static membership.
- `CooperativeStickyAssignor` để có cooperative rebalancing.
