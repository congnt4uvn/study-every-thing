
Dưới đây là tóm tắt ngắn gọn về **Replication trong Kafka** với trọng tâm là **rack awareness** và **cross-AZ replication**, kèm ví dụ cấu hình và luồng dữ liệu.  

---

## 1. Khái niệm chính

### a. Replication trong Kafka
- Mỗi **partition** có 1 **leader** và 0+ **follower**.
- Dữ liệu được **replicate** từ leader sang các follower.
- Mục tiêu: **tăng độ sẵn sàng (high availability)** và **chống mất dữ liệu** khi broker/AZ bị lỗi.

---

## 2. Rack Awareness

**Rack awareness** giúp Kafka phân bố các replica của một partition lên **nhiều rack khác nhau** (hoặc nhiều AZ khác nhau) để tránh single point of failure.

### Cách hoạt động
- Mỗi broker được gán một `rack` (hoặc AZ).
- Khi tạo topic, Kafka cố gắng:
  - Đặt các replica của cùng một partition trên **các rack khác nhau**.
  - Giảm rủi ro mất toàn bộ partition nếu một rack/AZ bị down.

### Cấu hình broker (server.properties)

```properties
# Ví dụ: 3 broker ở 3 AZ khác nhau
broker.id=1
broker.rack=az-1

broker.id=2
broker.rack=az-2

broker.id=3
broker.rack=az-3
```

### Tạo topic với replication factor

```bash
kafka-topics.sh \
  --create \
  --topic orders \
  --partitions 3 \
  --replication-factor 3 \
  --bootstrap-server broker1:9092
```

Với `broker.rack` đã cấu hình, Kafka sẽ cố gắng đặt 3 replica của mỗi partition trên 3 AZ khác nhau.

## 3. Cross-AZ Replication Patterns

**Cross-AZ** = triển khai Kafka cluster trong nhiều **Availability Zone** (AZ) trong cùng một region (hoặc nhiều region).

### Mục tiêu
- Chịu lỗi khi **một AZ bị down**.
- Đảm bảo **dữ liệu vẫn còn** và **cluster vẫn hoạt động**.

### Các pattern phổ biến

1. **Single cluster, multi-AZ**
   - Một Kafka cluster trải rộng trên nhiều AZ.
   - Dùng **rack awareness** để phân bố replica.
   - Ưu điểm: đơn giản, latency thấp trong cùng region.
   - Nhược: nếu network giữa AZ có vấn đề, có thể gây ra leader election, tăng latency.

2. **Multi-cluster, cross-region / cross-AZ replication**
   - Mỗi region/AZ có một Kafka cluster riêng.
   - Dùng **MirrorMaker 2 / Kafka Connect** để replicate dữ liệu giữa các cluster.
   - Ưu điểm: cách ly lỗi tốt, phù hợp DR (disaster recovery).
   - Nhược: phức tạp hơn, cần xử lý vấn đề **duplicate** và **ordering**.

## 4. Ví dụ cấu hình Spring Boot + Kafka (Producer/Consumer)

### application.yml (Spring Boot)

```yaml
spring:
  kafka:
    bootstrap-servers: broker1:9092,broker2:9092,broker3:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      acks: all              # chờ tất cả replica trong ISR xác nhận
      retries: 3
    consumer:
      group-id: order-service
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      auto-offset-reset: earliest
```

### Producer (Spring Boot)

```java
@Service
public class OrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public OrderProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendOrder(String orderId, String payload) {
        kafkaTemplate.send("orders", orderId, payload)
                     .whenComplete((result, ex) -> {
                         if (ex == null) {
                             System.out.println("Sent to partition " +
                                 result.getRecordMetadata().partition() +
                                 " offset " + result.getRecordMetadata().offset());
                         } else {
                             ex.printStackTrace();
                         }
                     });
    }
}
```

**Input ví dụ:**

```java
orderProducer.sendOrder("order-123", "{ \"id\": \"order-123\", \"amount\": 100000 }");
```

**Output (log):**

```text
Sent to partition 1 offset 42
```

Dữ liệu `"order-123"` sẽ được ghi vào **leader** của partition tương ứng, sau đó replicate sang các follower ở **các AZ khác** (nhờ rack awareness).

---

### Consumer (Spring Boot)

```java
@Service
public class OrderConsumer {

    @KafkaListener(topics = "orders", groupId = "order-service")
    public void handleOrder(String message) {
        System.out.println("Received order: " + message);
        // xử lý business logic
    }
}
```

**Input (từ Kafka):**

```json
{ "id": "order-123", "amount": 100000 }
```

**Output (log):**

```text
Received order: { "id": "order-123", "amount": 100000 }
```

---

## 5. Ý chính cần nhớ

- **Replication**: bảo vệ dữ liệu, tăng độ sẵn sàng.
- **Rack awareness**: phân bố replica trên nhiều rack/AZ để tránh mất dữ liệu khi một rack/AZ hỏng.
- **Cross-AZ patterns**:
  - Single cluster, multi-AZ (dùng rack awareness).
  - Multi-cluster, replicate bằng MirrorMaker/Kafka Connect cho DR.
- Spring Boot chỉ cần cấu hình đúng `bootstrap-servers` và `acks` để tận dụng replication của Kafka.
