
Here’s a concise summary of Spring Kafka consumer container types and how to use them.

---

## 1. Two Main Container Types

### `KafkaMessageListenerContainer<K, V>`
- Single-threaded: processes records from assigned partitions in **one thread**.
- Lower throughput, simpler behavior.
- You typically don’t create this directly; it’s used under the hood.

### `ConcurrentMessageListenerContainer<K, V>`
- Wraps multiple `KafkaMessageListenerContainer` instances.
- Provides **concurrency**: can run multiple consumer threads in parallel.
- You usually use this via `ConcurrentKafkaListenerContainerFactory` and `@KafkaListener`.

---

## 2. Typical Setup with `ConcurrentMessageListenerContainer`

### Maven dependency

```xml
<dependency>
    <groupId>org.springframework.kafka</groupId>
    <artifactId>spring-kafka</artifactId>
</dependency>
```

### Consumer configuration

```java
import org.apache.kafka.clients.consumer.ConsumerConfig;
import org.apache.kafka.common.serialization.StringDeserializer;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.config.ConcurrentKafkaListenerContainerFactory;
import org.springframework.kafka.core.ConsumerFactory;
import org.springframework.kafka.core.DefaultKafkaConsumerFactory;

import java.util.HashMap;
import java.util.Map;

@Configuration
public class KafkaConsumerConfig {

    @Bean
    public ConsumerFactory<String, String> consumerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "my-group");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.AUTO_OFFSET_RESET_CONFIG, "earliest");
        return new DefaultKafkaConsumerFactory<>(props);
    }

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String>
    kafkaListenerContainerFactory() {

        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory());

        // Number of concurrent consumer threads
        factory.setConcurrency(3);

        // Optional: manual ack, error handlers, etc.
        // factory.getContainerProperties().setAckMode(ContainerProperties.AckMode.MANUAL);

        return factory;
    }
}
```

### Listener using `@KafkaListener`

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class MyKafkaConsumer {

    @KafkaListener(topics = "my-topic", groupId = "my-group")
    public void listen(String message) {
        System.out.println("Received message: " + message);
    }
}
```

#### Example Input / Output

- **Input**: Messages produced to topic `my-topic`:
  - `"hello"`
  - `"world"`

- **Output (console)**:
  ```text
  Received message: hello
  Received message: world
  ```

With `concurrency = 3`, up to 3 threads can process messages in parallel (subject to partition count).

## 3. Using `KafkaMessageListenerContainer` Directly (Single Thread)

You can create it manually if you need fine-grained control:

```java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.core.ConsumerFactory;
import org.springframework.kafka.listener.ContainerProperties;
import org.springframework.kafka.listener.KafkaMessageListenerContainer;
import org.springframework.kafka.listener.MessageListener;

@Configuration
public class SingleContainerConfig {

    @Bean
    public KafkaMessageListenerContainer<String, String> singleContainer(
            ConsumerFactory<String, String> consumerFactory) {

        ContainerProperties containerProps = new ContainerProperties("my-topic");
        containerProps.setGroupId("single-group");

        containerProps.setMessageListener((MessageListener<String, String>) record -> {
            System.out.println("Single-container received: " + record.value());
        });

        return new KafkaMessageListenerContainer<>(consumerFactory, containerProps);
    }
}
```

#### Example Input / Output

- **Input**: Messages to `my-topic`: `"A"`, `"B"`
- **Output**:
  ```text
  Single-container received: A
  Single-container received: B
  ```
All in a single thread.

---

## 4. When to Use Which

- Use **`ConcurrentMessageListenerContainer`** (via `ConcurrentKafkaListenerContainerFactory` + `@KafkaListener`) for:
  - Most applications
  - Higher throughput
  - Easy scaling with `setConcurrency(n)`

- Use **`KafkaMessageListenerContainer`** directly when:
  - You need a **single-threaded** consumer with explicit control
  - You want to manage container lifecycle and properties manually
