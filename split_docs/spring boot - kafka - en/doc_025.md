## Key Ideas

1. **Idempotent Producer**
   - Ensures that messages are written to Kafka **exactly once per producer session**, even if retries happen.
   - Prevents **duplicate messages** caused by network errors, timeouts, or broker failures.
   - Achieved by enabling `enable.idempotence=true` in the producer configuration.
   - Works by assigning a **producer ID (PID)** and **sequence numbers** per partition; the broker rejects out-of-order or duplicate records.

2. **Partitioning Implications**
   - Idempotence is **tracked per partition**. Each partition keeps sequence numbers for a given producer.
   - If you change the partitioning strategy (e.g., different keys, custom partitioner), the **same logical message** may go to different partitions, which affects:
     - Ordering guarantees (ordering is only per partition).
     - Idempotence tracking (sequence numbers are per partition).
   - For strong guarantees (ordering + idempotence), you typically:
     - Use a **stable key** so the same key always maps to the same partition.
     - Avoid changing the partitioner logic in incompatible ways.
   - With **sticky partitioning** (Kafka’s default for no-key records), batches are sent to a chosen partition for a while, then switch. Idempotence still works, but ordering is only guaranteed within each partition.

3. **Spring Boot / Spring Kafka**
   - Spring Kafka integrates Kafka’s idempotent producer by setting the right properties.
   - You configure idempotence and partitioning via `ProducerFactory` and `KafkaTemplate` or application properties.

## Basic Idempotent Producer Configuration (Spring Boot)

**application.yml:**

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      properties:
        enable.idempotence: true
        acks: all
        retries: 2147483647
        max.in.flight.requests.per.connection: 5
```

**Explanation:**
- `enable.idempotence: true` – turns on idempotent producer.
- `acks: all` – waits for all replicas; required for idempotence.
- `max.in.flight.requests.per.connection: 5` – must be ≤ 5 for idempotence.

## Example: Sending Messages with a Stable Key (Ordering + Idempotence)

**Producer code (Spring Kafka):**

```java
@Service
public class OrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public OrderProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendOrder(String orderId, String payload) {
        // orderId is the key -> stable partition mapping
        kafkaTemplate.send("orders-topic", orderId, payload)
                     .addCallback(
                         result -> System.out.println("Sent: " + payload + " to partition " +
                                                      result.getRecordMetadata().partition()),
                         ex -> System.err.println("Failed to send: " + ex.getMessage())
                     );
    }
}
```

**Input:**

```java
orderProducer.sendOrder("order-123", "status=CREATED");
orderProducer.sendOrder("order-123", "status=PAID");
orderProducer.sendOrder("order-123", "status=SHIPPED");
```

**Output (consumer view, per partition):**

- All messages for `order-123` go to the **same partition**.
- They are seen in order: `CREATED -> PAID -> SHIPPED`.
- If a retry happens, duplicates are suppressed by the broker due to idempotence.

## Custom Partitioner Example

If you need a custom partitioning strategy, you must keep it **stable** to preserve ordering and idempotence behavior.

**Custom partitioner:**

```java
public class RegionPartitioner implements org.apache.kafka.clients.producer.Partitioner {

    @Override
    public int partition(String topic, Object key, byte[] keyBytes,
                         Object value, byte[] valueBytes, Cluster cluster) {
        int numPartitions = cluster.partitionsForTopic(topic).size();
        String region = (String) key; // e.g., "EU", "US", "APAC"
        return Math.abs(region.hashCode()) % numPartitions;
    }

    @Override
    public void close() {}

    @Override
    public void configure(Map<String, ?> configs) {}
}
```

**application.yml:**

```yaml
spring:
  kafka:
    producer:
      properties:
        enable.idempotence: true
        partitioner.class: com.example.kafka.RegionPartitioner
```

**Input:**

```java
kafkaTemplate.send("user-events", "EU", "user=1,action=login");
kafkaTemplate.send("user-events", "EU", "user=1,action=logout");
```

**Output:**
- Both messages with key `"EU"` go to the same partition.
- Idempotence ensures no duplicates on retries.
- Ordering is preserved for `"EU"` events.

## Summary

- Idempotent producer:
  - Eliminates duplicates caused by retries.
  - Requires specific producer settings (`enable.idempotence=true`, `acks=all`, etc.).
- Partitioning:
  - Idempotence and ordering are **per partition**.
  - Use stable keys and stable partitioning logic to maintain consistent behavior.
- Spring Boot / Spring Kafka:
  - Configure idempotence via properties.
  - Use `KafkaTemplate` with keys to control partitioning and leverage idempotent guarantees.
