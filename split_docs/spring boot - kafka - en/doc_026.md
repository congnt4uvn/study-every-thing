
Key ideas about Kafka producer basics in Spring Boot:

1. Asynchronous send  
   - Kafka producers send messages asynchronously: `send()` returns immediately.  
   - The actual network I/O happens in the background, so your app doesn’t block.

2. Futures (Kafka `Future<RecordMetadata>`)  
   - `KafkaTemplate.send(...)` returns a `ListenableFuture<SendResult<K,V>>`.  
   - You can:
     - Ignore it (fire-and-forget).
     - Block on it (synchronous behavior).
     - Attach callbacks (non-blocking).

3. Callbacks  
   - Callbacks let you react when the send succeeds or fails:
     - On success: inspect topic, partition, offset.
     - On failure: log, retry, or trigger compensating actions.

---

### Spring Boot Example

**Producer configuration (auto via Spring Boot)**  
Spring Boot auto-configures `KafkaTemplate` if you include `spring-kafka` and set basic properties:

```yaml
# application.yml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

**Producer service with async send + callback**

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.kafka.support.SendResult;
import org.springframework.stereotype.Service;
import org.springframework.util.concurrent.ListenableFuture;
import org.springframework.util.concurrent.ListenableFutureCallback;

@Service
public class MyKafkaProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public MyKafkaProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendAsync(String topic, String key, String value) {
        ListenableFuture<SendResult<String, String>> future =
                kafkaTemplate.send(topic, key, value);

        future.addCallback(new ListenableFutureCallback<>() {
            @Override
            public void onSuccess(SendResult<String, String> result) {
                System.out.printf(
                    "Message sent successfully: topic=%s partition=%d offset=%d key=%s value=%s%n",
                    result.getRecordMetadata().topic(),
                    result.getRecordMetadata().partition(),
                    result.getRecordMetadata().offset(),
                    key,
                    value
                );
            }

            @Override
            public void onFailure(Throwable ex) {
                System.err.printf(
                    "Failed to send message: key=%s value=%s error=%s%n",
                    key,
                    value,
                    ex.getMessage()
                );
                // Optionally: retry, send to DLQ, etc.
            }
        });
    }

    // Synchronous style using the Future (blocking)
    public void sendSync(String topic, String key, String value) throws Exception {
        SendResult<String, String> result =
                kafkaTemplate.send(topic, key, value).get(); // blocks

        System.out.printf(
            "Sync send: topic=%s partition=%d offset=%d%n",
            result.getRecordMetadata().topic(),
            result.getRecordMetadata().partition(),
            result.getRecordMetadata().offset()
        );
    }
}
```

### Example Input / Output

**Input (application code):**

```java
myKafkaProducer.sendAsync("orders", "order-123", "{ \"id\": 123, \"status\": \"NEW\" }");
```

**Possible console output (success):**

```text
Message sent successfully: topic=orders partition=0 offset=15 key=order-123 value={ "id": 123, "status": "NEW" }
```

**Possible console output (failure):**

```text
Failed to send message: key=order-123 value={ "id": 123, "status": "NEW" } error=TimeoutException: Failed to update metadata after 60000 ms.
```

Summary:  
- Kafka producers in Spring Boot send messages asynchronously.  
- `KafkaTemplate.send()` returns a future you can ignore, block on, or attach callbacks to.  
- Callbacks are the standard way to handle success/failure without blocking.
