
Here’s a concise summary of the main ideas behind **Spring Kafka transactions** where you **consume, process, and produce** in a single transaction, plus code examples.

---

## Key Ideas

1. **Goal**:  
   Ensure that:
   - A message is **consumed** from Kafka,
   - **Processed** (e.g., business logic, DB updates),
   - And **new messages are produced** to Kafka  
   all within **one atomic transaction**.

2. **Exactly-once semantics (EOS)**:  
   Using Kafka transactions + Spring’s transaction support, you can avoid:
   - Duplicated processing
   - Lost messages
   - Inconsistent state between Kafka and your application/DB

3. **Core components**:
   - `KafkaTemplate` with a **transactional producer** (`transactionIdPrefix`).
   - `@KafkaListener` with a **transactional container**.
   - Optional: integration with a **database transaction** via Spring’s `ChainedKafkaTransactionManager` or similar.

4. **Pattern**:
   - Start transaction
   - Consume record(s)
   - Run business logic (and DB operations if needed)
   - Produce new record(s)
   - Commit transaction  
   If anything fails → **rollback** (offsets not committed, produced messages not visible).

## Basic Configuration Example

### 1. Producer Factory & KafkaTemplate (Transactional)

```java
@Bean
public ProducerFactory<String, String> producerFactory() {
    Map<String, Object> props = new HashMap<>();
    props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
    props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
    props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
    // Enable idempotence and transactions
    props.put(ProducerConfig.ENABLE_IDEMPOTENCE_CONFIG, true);
    props.put(ProducerConfig.TRANSACTIONAL_ID_CONFIG, "my-tx-producer-");
    return new DefaultKafkaProducerFactory<>(props);
}

@Bean
public KafkaTemplate<String, String> kafkaTemplate() {
    KafkaTemplate<String, String> template = new KafkaTemplate<>(producerFactory());
    template.setTransactionIdPrefix("my-tx-producer-");
    return template;
}
```

### 2. Consumer Factory & Listener Container (Transactional)

```java
@Bean
public ConsumerFactory<String, String> consumerFactory() {
    Map<String, Object> props = new HashMap<>();
    props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
    props.put(ConsumerConfig.GROUP_ID_CONFIG, "my-tx-group");
    props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
    props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
    props.put(ConsumerConfig.ISOLATION_LEVEL_CONFIG, "read_committed"); // important
    return new DefaultKafkaConsumerFactory<>(props);
}

@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory(
        ConsumerFactory<String, String> consumerFactory,
        KafkaTemplate<String, String> kafkaTemplate) {

    ConcurrentKafkaListenerContainerFactory<String, String> factory =
            new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory);

    // Enable transactions for the listener container
    factory.getContainerProperties().setTransactionManager(
            new KafkaTransactionManager<>(producerFactory())
    );

    // Or: factory.getContainerProperties().setTransactionManager(kafkaTransactionManager);

    return factory;
}
```

## Transactional Listener Example

### Listener that consumes, processes, and produces in one transaction

```java
@Service
public class MyKafkaService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public MyKafkaService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    @KafkaListener(topics = "input-topic", groupId = "my-tx-group",
                   containerFactory = "kafkaListenerContainerFactory")
    @Transactional  // Spring-managed transaction
    public void handleMessage(String inputMessage) {
        // 1. Consume: inputMessage is the consumed record value

        // 2. Process: business logic
        String processed = inputMessage.toUpperCase(); // example transformation

        // 3. Produce: send result to another topic
        kafkaTemplate.send("output-topic", processed);

        // If an exception is thrown here, the whole transaction is rolled back:
        // - offsets are not committed
        // - produced message is not visible to consumers
    }
}
```

### Example Input/Output

- **Input topic**: `input-topic`  
  Message: `"hello world"`

- **Processing**:
  - `processed = "HELLO WORLD"`

- **Output topic**: `output-topic`  
  Message produced: `"HELLO WORLD"`

If an exception occurs after consuming `"hello world"` but before commit:
- The offset for `"hello world"` is **not committed**.
- `"HELLO WORLD"` is **not visible** on `output-topic`.
- On retry, `"hello world"` will be reprocessed.

## Kafka + Database in One Logical Transaction (Pattern)

You can combine Kafka and DB operations using a **chained transaction manager**:

```java
@Bean
public PlatformTransactionManager kafkaTransactionManager(ProducerFactory<String, String> pf) {
    return new KafkaTransactionManager<>(pf);
}

@Bean
public PlatformTransactionManager chainedTxManager(
        PlatformTransactionManager kafkaTxManager,
        PlatformTransactionManager dbTxManager) {
    return new ChainedKafkaTransactionManager<>(kafkaTxManager, dbTxManager);
}
```

Then:

```java
@KafkaListener(topics = "input-topic", containerFactory = "kafkaListenerContainerFactory")
@Transactional("chainedTxManager")
public void handleMessageWithDb(String inputMessage) {
    // DB operation
    myRepository.save(new MyEntity(inputMessage));

    // Kafka produce
    kafkaTemplate.send("output-topic", "saved: " + inputMessage);
}
```

If DB save fails → Kafka send is rolled back; if Kafka send fails → DB save is rolled back.

---

## Summary

- Use **transactional producer** (`transactionIdPrefix`) and **transactional listener containers**.
- Configure consumer with `isolation.level=read_committed`.
- Wrap **consume → process → produce** in a single Spring transaction (`@Transactional`).
- Optionally chain Kafka and DB transaction managers for consistent state across both.
