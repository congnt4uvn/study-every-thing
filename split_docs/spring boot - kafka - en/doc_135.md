## Likely Key Points & Main Ideas

1. **Managed Kafka vs. Self‑Hosted**
   - Managed Kafka (e.g., Confluent Cloud, AWS MSK, Azure Event Hubs for Kafka) offloads:
     - Cluster provisioning
     - Scaling
     - Patching and upgrades
     - Monitoring and some security aspects
   - You focus on producers/consumers and integration, not cluster ops.

2. **Networking**
   - Private connectivity options:
     - VPC peering / PrivateLink (AWS)
     - Private Service Connect (GCP)
     - VNet peering / Private Endpoint (Azure)
   - Restricting public access:
     - Disable public endpoints or limit via IP allowlists.
   - DNS and bootstrap servers:
     - Use cloud‑provided bootstrap servers (e.g., `pkc-xxxxx.us-central1.gcp.confluent.cloud:9092`).
     - Ensure your app’s network (VPC/VNet/subnet) can resolve and reach those endpoints.

3. **Security**
   - **Encryption in transit**: TLS/SSL for all Kafka connections.
   - **Authentication**:
     - SASL mechanisms: SASL_SSL with SCRAM, OAUTHBEARER, or custom cloud provider auth.
     - Cloud‑specific auth (e.g., IAM roles for MSK, service principals for Azure).
   - **Authorization**:
     - Kafka ACLs (topic‑level, consumer group‑level).
     - Role‑based access control (RBAC) in managed platforms (e.g., Confluent Cloud roles).

4. **IAM Integration**
   - Use cloud IAM instead of static credentials where possible:
     - AWS: IAM roles (EC2, ECS, EKS, Lambda) + MSK IAM auth.
     - GCP: Service accounts + workload identity.
     - Azure: Managed identities + Event Hubs / Kafka.
   - Principle of least privilege:
     - Separate roles for producers, consumers, admins.
   - Centralized credential management:
     - Store secrets in Secret Manager / Parameter Store / Key Vault.
     - Rotate credentials automatically.

5. **Spring Boot Integration**
   - Use `spring-kafka` with:
     - Proper `bootstrap-servers`
     - SASL/TLS configuration
     - Externalized configuration via `application.yml` or environment variables.
   - Use Spring’s `KafkaTemplate` for producing and `@KafkaListener` for consuming.

## Example: Spring Boot + Managed Kafka (SASL_SSL)

### `application.yml`

```yaml
spring:
  kafka:
    bootstrap-servers: pkc-xxxxx.us-central1.gcp.confluent.cloud:9092
    properties:
      security.protocol: SASL_SSL
      sasl.mechanism: PLAIN
      sasl.jaas.config: >
        org.apache.kafka.common.security.plain.PlainLoginModule required
        username="${KAFKA_API_KEY}"
        password="${KAFKA_API_SECRET}";
    consumer:
      group-id: demo-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      auto-offset-reset: earliest
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

Environment variables (managed by your IAM/secret system):

```bash
export KAFKA_API_KEY="your-api-key"
export KAFKA_API_SECRET="your-api-secret"
```

### Producer Example

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class OrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public OrderProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendOrder(String orderId, String payload) {
        kafkaTemplate.send("orders", orderId, payload);
    }
}
```

**Input (method call):**

```java
orderProducer.sendOrder("order-123", "{\"item\":\"book\",\"qty\":1}");
```

**Output (Kafka message on topic `orders`):**

- **Topic**: `orders`
- **Key**: `order-123`
- **Value**: `{"item":"book","qty":1}`

### Consumer Example

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class OrderConsumer {

    @KafkaListener(topics = "orders", groupId = "demo-group")
    public void handleOrder(String message) {
        // Process the order
        System.out.println("Received order: " + message);
    }
}
```

**Input (Kafka message on `orders`):**

- **Key**: `order-123`
- **Value**: `{"item":"book","qty":1}`

**Output (application log):**

```text
Received order: {"item":"book","qty":1}
```

## Example: AWS MSK with IAM Authentication (Spring Boot)

Assuming:
- Your app runs on EC2/ECS/EKS with an IAM role that has permissions to access MSK.
- MSK is configured for IAM auth.

### `application.yml`

```yaml
spring:
  kafka:
    bootstrap-servers: b-1.msk-cluster.xxxxxx.c2.kafka.us-east-1.amazonaws.com:9098
    properties:
      security.protocol: SASL_SSL
      sasl.mechanism: AWS_MSK_IAM
      sasl.jaas.config: software.amazon.msk.auth.iam.IAMLoginModule required;
      sasl.client.callback.handler.class: software.amazon.msk.auth.iam.IAMClientCallbackHandler
    consumer:
      group-id: msk-demo-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

IAM role is attached to the compute resource; no static username/password needed. IAM policies control which topics can be accessed.
