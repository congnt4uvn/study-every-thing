## Key Ideas

1. **Async send**  
   - Spring Kafka’s `KafkaTemplate` sends messages asynchronously.  
   - `send()` returns a `ListenableFuture` / `CompletableFuture` that completes when the broker acknowledges the message.

2. **Callbacks**  
   - You can attach success and failure callbacks to handle results of async sends.  
   - On success: log metadata (topic, partition, offset).  
   - On failure: log error, trigger alerts, or send to a dead-letter topic.

3. **Error handling**  
   - Handle send failures in the callback’s failure branch.  
   - Optionally wrap send logic in your own service that can:
     - Retry
     - Persist failed messages
     - Notify other systems

4. **Retries**  
   - Two main layers:
     - **Producer-level retries** (Kafka client config: `retries`, `delivery.timeout.ms`, `acks`, etc.).  
     - **Application-level retries** (Spring Retry, manual retry loops, or re-queueing messages).

## Basic Async Send with Callback

### Configuration (application.yml)

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      # Producer-level retry example
      properties:
        retries: 5
        acks: all
        delivery.timeout.ms: 120000
```

### Producer Service

```java
import org.apache.kafka.clients.producer.RecordMetadata;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.kafka.support.SendResult;
import org.springframework.stereotype.Service;
import org.springframework.util.concurrent.ListenableFuture;
import org.springframework.util.concurrent.ListenableFutureCallback;

@Service
public class MyKafkaProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private static final String TOPIC = "my-topic";

    public MyKafkaProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String key, String value) {
        ListenableFuture<SendResult<String, String>> future =
                kafkaTemplate.send(TOPIC, key, value);

        future.addCallback(new ListenableFutureCallback<>() {
            @Override
            public void onSuccess(SendResult<String, String> result) {
                RecordMetadata meta = result.getRecordMetadata();
                System.out.printf(
                    "Sent message successfully: key=%s value=%s topic=%s partition=%d offset=%d%n",
                    key, value, meta.topic(), meta.partition(), meta.offset()
                );
            }

            @Override
            public void onFailure(Throwable ex) {
                System.err.printf(
                    "Failed to send message: key=%s value=%s error=%s%n",
                    key, value, ex.getMessage()
                );
                // Custom error handling: log, alert, save to DB, send to DLT, etc.
            }
        });
    }
}
```

### Example Input / Output

**Input (your application code):**

```java
myKafkaProducer.sendMessage("user-123", "User created");
```

**Possible success log output:**

```text
Sent message successfully: key=user-123 value=User created topic=my-topic partition=0 offset=42
```

**Possible failure log output:**

```text
Failed to send message: key=user-123 value=User created error=TimeoutException: Failed to update metadata...
```

## Async Send Using CompletableFuture (Spring Kafka 3+)

```java
import org.springframework.kafka.support.SendResult;
import java.util.concurrent.CompletableFuture;

public void sendMessageV2(String key, String value) {
    CompletableFuture<SendResult<String, String>> future =
            kafkaTemplate.send(TOPIC, key, value);

    future.whenComplete((result, ex) -> {
        if (ex == null) {
            var meta = result.getRecordMetadata();
            System.out.printf(
                "Sent: key=%s value=%s topic=%s partition=%d offset=%d%n",
                key, value, meta.topic(), meta.partition(), meta.offset()
            );
        } else {
            System.err.printf(
                "Send failed: key=%s value=%s error=%s%n",
                key, value, ex.getMessage()
            );
            // Error handling / retry logic here
        }
    });
}
```

## Application-Level Retry Example (Manual)

```java
public void sendWithRetry(String key, String value, int maxAttempts) {
    int attempt = 0;
    while (attempt < maxAttempts) {
        attempt++;
        try {
            CompletableFuture<SendResult<String, String>> future =
                    kafkaTemplate.send(TOPIC, key, value);
            future.join(); // wait for result (still async from caller’s perspective if you run this in another thread)
            System.out.printf("Send succeeded on attempt %d%n", attempt);
            return;
        } catch (Exception ex) {
            System.err.printf("Send failed on attempt %d: %s%n", attempt, ex.getMessage());
            if (attempt == maxAttempts) {
                // Final failure handling: persist, alert, etc.
            }
        }
    }
}
```

**Input:**

```java
sendWithRetry("user-123", "User created", 3);
```

**Output (example):**

```text
Send failed on attempt 1: TimeoutException...
Send failed on attempt 2: TimeoutException...
Send succeeded on attempt 3
```
