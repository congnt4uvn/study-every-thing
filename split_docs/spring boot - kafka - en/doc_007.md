
Here’s a concise summary of the key ideas around Kafka data flow with Spring Boot, plus code examples.

---

## Core Concepts

**Data flow:**  
`Producer → Broker (Kafka cluster) → Consumer`

1. **Producer**
   - Sends messages to a **topic**.
   - Each topic is split into **partitions**.
   - Messages are appended to the end of a partition’s **log** (an ordered, immutable sequence).

2. **Broker**
   - A Kafka server that stores topic partitions.
   - For each partition, messages are stored in a log file on disk.
   - Messages are identified by an **offset** (a monotonically increasing number).
   - Kafka does not delete messages after consumption; it keeps them based on **retention policy** (time/size).

3. **Consumer**
   - Reads messages from a topic partition in order of offsets.
   - Tracks its position using **consumer offsets** (stored in Kafka, usually in `__consumer_offsets`).
   - Multiple consumers in the same **consumer group** share partitions (each partition is consumed by only one consumer in a group).

4. **Log Append Behavior**
   - When a producer sends a message:
     - Kafka chooses a partition (by key or round-robin).
     - The message is appended to the end of that partition’s log.
     - The message gets the next offset.
   - Consumers read sequentially from the log; this gives strong ordering per partition.

## Spring Boot + Kafka: Basic Producer & Consumer

### Maven Dependencies

```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>
</dependencies>
```

### Configuration (application.yml)

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
    consumer:
      group-id: demo-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      auto-offset-reset: earliest
```

## Producer Example (Spring Boot)

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class SimpleProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private static final String TOPIC = "demo-topic";

    public SimpleProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String key, String value) {
        kafkaTemplate.send(TOPIC, key, value);
    }
}
```

### Example Input → Broker

```java
// Example calls
simpleProducer.sendMessage("user-1", "Hello Kafka");
simpleProducer.sendMessage("user-2", "Another message");
```

**What happens internally:**

- Messages are sent to `demo-topic`.
- Kafka chooses partitions (e.g., based on key `"user-1"` / `"user-2"`).
- Each message is appended to the end of the chosen partition’s log:
  - Partition 0: offset 0 → "Hello Kafka"
  - Partition 1: offset 0 → "Another message"  
  (actual partition assignment depends on configuration)

## Consumer Example (Spring Boot)

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class SimpleConsumer {

    @KafkaListener(topics = "demo-topic", groupId = "demo-group")
    public void listen(String message) {
        System.out.println("Received message: " + message);
    }
}
```

### Example Output (Consumer)

Given the two messages above, the consumer prints:

```text
Received message: Hello Kafka
Received message: Another message
```

(Per-partition order is guaranteed; cross-partition order is not.)

---

## Key Takeaways

- Kafka stores messages in **append-only logs** per partition.
- Producers append messages; consumers read them sequentially by **offset**.
- Spring Boot with Spring Kafka simplifies producing and consuming messages with annotations and templates.
- Message order is guaranteed **within a partition**, not across the whole topic.
