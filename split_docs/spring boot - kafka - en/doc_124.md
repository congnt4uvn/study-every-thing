
Here’s a concise summary and some practical examples for **using Kafka Streams with Spring Boot** via **Spring Integration / Spring for Apache Kafka**.

---

## Key Ideas

1. **Kafka Streams** is a Java library for building stream-processing applications on top of Kafka (stateful/stateless processing, windowing, joins, etc.).
2. **Spring Boot + Spring Kafka** simplifies:
   - Creating and configuring `StreamsBuilder` / `KafkaStreams`
   - Managing application lifecycle (start/stop)
   - Externalizing configuration via `application.yml`
3. You typically:
   - Define a **topology** using `StreamsBuilder` (e.g., `KStream`, `KTable`).
   - Configure **SerDes** (serializers/deserializers).
   - Use Spring Boot auto-configuration or a `@Configuration` class to create the `KafkaStreams` bean.
4. Integration patterns:
   - **Pure Kafka Streams**: Use `StreamsBuilder` directly.
   - **Spring Cloud Stream**: Use functional style (`Function<KStream<..>, KStream<..>>`) with Kafka Streams binder.
   - **Spring Integration**: Use channels and adapters to bridge Kafka Streams with other Spring Integration flows.

## Basic Setup (Spring Boot + Kafka Streams)

**Gradle dependencies (example):**
```groovy
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter'
    implementation 'org.springframework.kafka:spring-kafka'
    implementation 'org.apache.kafka:kafka-streams:3.7.0'
}
```

**application.yml:**
```yaml
spring:
  application:
    name: kafka-streams-demo
  kafka:
    bootstrap-servers: localhost:9092
    streams:
      application-id: word-count-app
      properties:
        default.key.serde: org.apache.kafka.common.serialization.Serdes$StringSerde
        default.value.serde: org.apache.kafka.common.serialization.Serdes$StringSerde
```

## Example 1: Simple Kafka Streams Topology in Spring Boot

**Goal:** Read from `input-topic`, convert messages to uppercase, write to `output-topic`.

**Configuration class:**
```java
import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.kstream.KStream;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class KafkaStreamsConfig {

    @Bean
    public KStream<String, String> kStream(StreamsBuilder builder) {
        KStream<String, String> stream = builder.stream("input-topic");
        stream
            .mapValues(value -> value.toUpperCase())
            .to("output-topic");
        return stream;
    }
}
```

**Input / Output example:**

- Input to `input-topic`:
  - Key: `"user1"`, Value: `"hello kafka"`
  - Key: `"user2"`, Value: `"spring boot integration"`

- Output from `output-topic`:
  - Key: `"user1"`, Value: `"HELLO KAFKA"`
  - Key: `"user2"`, Value: `"SPRING BOOT INTEGRATION"`

## Example 2: Word Count with Kafka Streams

**Goal:** Count words from a text stream and continuously output counts.

```java
import org.apache.kafka.common.serialization.Serdes;
import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.kstream.*;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import java.util.Arrays;

@Configuration
public class WordCountTopology {

    @Bean
    public KStream<String, String> wordCountStream(StreamsBuilder builder) {
        KStream<String, String> textLines = builder.stream("text-input");

        KTable<String, Long> wordCounts = textLines
            .flatMapValues(value -> Arrays.asList(value.toLowerCase().split("\\W+")))
            .groupBy((key, word) -> word, Grouped.with(Serdes.String(), Serdes.String()))
            .count(Materialized.as("word-counts-store"));

        wordCounts
            .toStream()
            .to("word-count-output", Produced.with(Serdes.String(), Serdes.Long()));

        return textLines;
    }
}
```

**Input / Output example:**

- Input to `text-input`:
  - Value: `"Hello Kafka Kafka"`
  - Value: `"Hello Spring"`

- Output from `word-count-output` (order may vary):
  - Key: `"hello"`, Value: `1`
  - Key: `"kafka"`, Value: `1`
  - Key: `"kafka"`, Value: `2`
  - Key: `"hello"`, Value: `2`
  - Key: `"spring"`, Value: `1`

(Counts are updated as new messages arrive.)

## Example 3: Using Spring Cloud Stream Functional Style (Kafka Streams Binder)

If you use **Spring Cloud Stream** with Kafka Streams binder, you can define a function:

**build.gradle:**
```groovy
dependencies {
    implementation 'org.springframework.cloud:spring-cloud-starter-stream-kafka'
    implementation 'org.springframework.cloud:spring-cloud-stream-binder-kafka-streams'
}
```

**application.yml:**
```yaml
spring:
  cloud:
    stream:
      function:
        definition: process
      bindings:
        process-in-0:
          destination: input-topic
        process-out-0:
          destination: output-topic
      kafka:
        streams:
          binder:
            application-id: functional-streams-app
            configuration:
              default.key.serde: org.apache.kafka.common.serialization.Serdes$StringSerde
              default.value.serde: org.apache.kafka.common.serialization.Serdes$StringSerde
```

**Function bean:**
```java
import org.apache.kafka.streams.kstream.KStream;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import java.util.function.Function;

@Configuration
public class StreamFunctions {

    @Bean
    public Function<KStream<String, String>, KStream<String, String>> process() {
        return input -> input.mapValues(v -> "processed-" + v);
    }
}
```

**Input / Output example:**

- Input to `input-topic`: Value: `"msg1"`
- Output from `output-topic`: Value: `"processed-msg1"`

## Example 4: Bridging Kafka Streams with Spring Integration

You can use Spring Integration to connect Kafka Streams to other systems (HTTP, JMS, etc.).

**Example: send Kafka Streams output to a Spring Integration channel:**

```java
import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.kstream.KStream;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.integration.channel.DirectChannel;
import org.springframework.messaging.MessageChannel;
import org.springframework.messaging.support.MessageBuilder;

@Configuration
public class StreamsIntegrationConfig {

    @Bean
    public MessageChannel processedChannel() {
        return new DirectChannel();
    }

    @Bean
    public KStream<String, String> integrationStream(StreamsBuilder builder,
                                                     MessageChannel processedChannel) {
        KStream<String, String> stream = builder.stream("raw-input");

        stream.foreach((key, value) -> {
            processedChannel.send(
                MessageBuilder.withPayload(value.toUpperCase())
                              .setHeader("kafkaKey", key)
                              .build()
            );
        });

        return stream;
    }
}
```

**Input / Output example:**

- Input to `raw-input`: Key: `"id1"`, Value: `"hello"`
- Message on `processedChannel`:
  - Payload: `"HELLO"`
  - Header `kafkaKey`: `"id1"`

From there, Spring Integration can route to HTTP, file, database, etc.

## Summary

- Use **Spring Boot + Spring Kafka** to configure and run Kafka Streams topologies easily.
- Define processing logic with `StreamsBuilder` and `KStream`/`KTable`.
- Optionally use **Spring Cloud Stream** for a functional, binder-based approach.
- Use **Spring Integration** to connect Kafka Streams with other systems via channels and adapters.
- Inputs/outputs are just Kafka records; Spring helps with configuration, lifecycle, and integration.
