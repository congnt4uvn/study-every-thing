
Here’s a concise summary and practical view of:

**“Durability & consistency: Exactly-once semantics end-to-end (producer + broker + consumer)”**  
in Kafka with Spring Boot.

---

## Key Ideas

1. **Durability**  
   - Messages are written to disk on Kafka brokers and replicated across multiple brokers.  
   - This ensures data is not lost even if a broker fails.

2. **Consistency & Exactly-Once Semantics (EOS)**  
   - EOS means each message is processed **once and only once** from producer → Kafka → consumer.  
   - Kafka supports EOS using:
     - **Idempotent producers** (no duplicates on broker)
     - **Transactions** (grouping send + consume operations atomically)
   - End-to-end EOS:  
     - Producer sends messages in a transaction  
     - Consumer reads and processes messages and commits offsets in the same transaction  
     - Either all operations succeed or none do.

3. **Spring Boot + Spring Kafka**  
   - Spring Kafka simplifies EOS by:
     - Configuring transactional producers
     - Managing consumer offset commits within the same transaction
   - You typically use:
     - `transactional.id` for producer
     - `KafkaTransactionManager` or `ChainedKafkaTransactionManager`
     - `@Transactional` or container-managed transactions

## Basic Configuration (Spring Boot / Spring Kafka)

**application.yml**

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      # Enable idempotent producer
      properties:
        enable.idempotence: true
      # Required for EOS
      transaction-id-prefix: tx-producer-
    consumer:
      group-id: my-eos-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      # Read committed messages only
      properties:
        isolation.level: read_committed
    listener:
      # Let Spring manage transactions for listeners
      ack-mode: RECORD
```

## Producer with Transactions (Spring Kafka)

**Producer service**

```java
@Service
public class EosProducerService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public EosProducerService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    // Send multiple messages atomically
    public void sendInTransaction(List<String> messages) {
        kafkaTemplate.executeInTransaction(operations -> {
            for (String msg : messages) {
                operations.send("input-topic", msg);
            }
            return null;
        });
    }
}
```

**Input → Output example**

- Input to service: `["order-1", "order-2", "order-3"]`
- Behavior:
  - All three messages are sent in one Kafka transaction.
  - If any send fails, **none** of the messages are visible to consumers.
- Output on Kafka:
  - Either:
    - `input-topic` contains `order-1`, `order-2`, `order-3`  
    - Or none of them (if transaction aborted).

## Consumer with Exactly-Once Processing

**Transactional listener (consume → process → produce → commit offsets atomically)**

```java
@EnableKafka
@Configuration
public class KafkaConfig {

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory(
            ConsumerFactory<String, String> consumerFactory,
            KafkaTemplate<String, String> kafkaTemplate) {

        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();

        factory.setConsumerFactory(consumerFactory);
        factory.getContainerProperties().setAckMode(ContainerProperties.AckMode.RECORD);

        // Enable Kafka transactions for the listener
        factory.getContainerProperties().setTransactionManager(
                new KafkaTransactionManager<>(kafkaTemplate.getProducerFactory())
        );

        return factory;
    }
}
```

```java
@Service
public class EosConsumerService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public EosConsumerService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    @KafkaListener(topics = "input-topic", groupId = "my-eos-group")
    public void process(String message) {
        // 1. Read message from input-topic
        // 2. Process it (e.g., transform, validate, call DB, etc.)
        String processed = "processed-" + message;

        // 3. Send result to output-topic in the same transaction
        kafkaTemplate.send("output-topic", processed);

        // Offsets are committed as part of the same transaction
    }
}
```

**End-to-end behavior**

- Input to Kafka:  
  - Producer sends `order-1` to `input-topic` in a transaction.
- Consumer processing:
  - Reads `order-1`
  - Produces `processed-order-1` to `output-topic`
  - Commits offset for `order-1`
- If a failure occurs:
  - Before commit: transaction is aborted →  
    - `processed-order-1` is not visible  
    - offset is not committed → message will be reprocessed
  - After commit: message is processed exactly once (no duplicates).

---

## Summary

- **Durability**: Kafka persists and replicates messages.
- **Consistency & EOS**: Use idempotent producers + transactions + `read_committed` consumers.
- **Spring Boot**:  
  - Configure `transaction-id-prefix` and `enable.idempotence` for producers.  
  - Use `KafkaTransactionManager` and transactional listeners.  
  - This gives end-to-end exactly-once processing: each message is processed once, with no duplicates and no loss.
