## Key Points: Kafka Architecture (Network Model)

1. **Transport**:  
   - Kafka uses **TCP** as its underlying transport protocol.
   - This ensures reliable, ordered delivery of data between clients (producers/consumers) and brokers.

2. **Protocol Type**:  
   - Kafka uses a **binary protocol**.
   - Messages and control data are encoded in a compact binary format, which is efficient and fast to parse.

3. **Communication Pattern**:  
   - Kafka uses a **request/response** model:
     - Clients send requests (e.g., Produce, Fetch).
     - Brokers send back responses (e.g., acknowledgments, records).
   - This is similar to HTTP in pattern, but implemented over a custom binary protocol on TCP.

4. **Batching**:  
   - Kafka heavily relies on **batching**:
     - Producers send records in batches to reduce network overhead.
     - Brokers write and replicate data in batches.
     - Consumers fetch records in batches.
   - Batching improves throughput and reduces per-message overhead.

## Simple Spring Boot + Kafka Example

### 1. Maven Dependencies (pom.xml)

```xml
<dependencies>
    <!-- Spring Boot Web (optional, for REST endpoints) -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>

    <!-- Spring Kafka -->
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>
</dependencies>
```

---

### 2. Configuration (application.yml)

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      properties:
        linger.ms: 5        # small delay to allow batching
        batch.size: 16384   # batch size in bytes
    consumer:
      group-id: demo-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      auto-offset-reset: earliest
```

These producer properties (`linger.ms`, `batch.size`) directly relate to **batching** in Kafka’s network model.

---

### 3. Producer Example (Spring Boot Service)

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class KafkaProducerService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public KafkaProducerService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String topic, String key, String value) {
        kafkaTemplate.send(topic, key, value);
    }
}
```

**Input example (logical):**

- Topic: `demo-topic`
- Key: `user-123`
- Value: `{"event":"login","timestamp":"2026-05-20T10:00:00Z"}`

**Network behavior:**

- The producer buffers this record in memory.
- If more messages arrive quickly, they are **batched** together.
- The batch is sent over **TCP** using Kafka’s **binary request/response** protocol.

### 4. Consumer Example (Spring Boot Listener)

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Component;

@Component
public class KafkaConsumerListener {

    @KafkaListener(topics = "demo-topic", groupId = "demo-group")
    public void listen(String message) {
        System.out.println("Received message: " + message);
    }
}
```

**Output example (console):**

```text
Received message: {"event":"login","timestamp":"2026-05-20T10:00:00Z"}
```

Internally, the consumer sends a **Fetch** request over TCP, receives a **batch** of records in a binary response, and Spring Kafka deserializes them into `String` messages.

### Summary in One Sentence

Kafka’s network architecture is built on TCP with a custom binary request/response protocol that emphasizes batching at every stage (producer, broker, consumer) to achieve high throughput and efficient communication, and Spring Boot with Spring Kafka provides a convenient way to interact with this model via configuration and simple producer/consumer code.
