
Here’s a concise summary and some practical examples around monitoring Kafka broker metrics in a Spring Boot context.

---

## Key Kafka Broker Metrics

1. **Under-Replicated Partitions (URP)**  
   - **What it is**: Partitions where the number of in-sync replicas (ISR) is less than the configured replication factor.  
   - **Why it matters**: Indicates potential risk of data loss if a broker fails; replication is lagging.

2. **Offline Partitions**  
   - **What it is**: Partitions that have no leader at all.  
   - **Why it matters**: Messages cannot be produced or consumed from these partitions; indicates a serious availability issue.

---

## How to Monitor These Metrics

### 1. Using Kafka JMX Metrics

Kafka exposes metrics via JMX. Common MBeans:

- Under-replicated partitions:
  - `kafka.server:type=ReplicaManager,name=UnderReplicatedPartitions`
- Offline partitions:
  - `kafka.controller:type=KafkaController,name=OfflinePartitionsCount`

You can scrape these with Prometheus (via JMX exporter) and visualize/alert in Grafana.

---

## Spring Boot Example: Reading Kafka JMX Metrics

Assume:
- Kafka broker exposes JMX on `service:jmx:rmi:///jndi/rmi://localhost:9999/jmxrmi`.

### Maven Dependencies

```xml
<dependencies>
    <!-- Spring Boot Actuator for metrics endpoints -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-actuator</artifactId>
    </dependency>

    <!-- Micrometer Prometheus registry (optional, for Prometheus) -->
    <dependency>
        <groupId>io.micrometer</groupId>
        <artifactId>micrometer-registry-prometheus</artifactId>
    </dependency>

    <!-- JMX access -->
    <dependency>
        <groupId>org.springframework</groupId>
        <artifactId>spring-context</artifactId>
    </dependency>
</dependencies>
```

### Configuration: Connect to Kafka JMX

```java
import javax.management.MBeanServerConnection;
import javax.management.ObjectName;
import javax.management.remote.JMXConnector;
import javax.management.remote.JMXConnectorFactory;
import javax.management.remote.JMXServiceURL;
import java.io.IOException;

public class KafkaJmxClient {

    private final MBeanServerConnection mbeanConn;

    public KafkaJmxClient(String jmxUrl) throws IOException {
        JMXServiceURL serviceURL = new JMXServiceURL(jmxUrl);
        JMXConnector connector = JMXConnectorFactory.connect(serviceURL, null);
        this.mbeanConn = connector.getMBeanServerConnection();
    }

    public int getUnderReplicatedPartitions() throws Exception {
        ObjectName name = new ObjectName("kafka.server:type=ReplicaManager,name=UnderReplicatedPartitions");
        return (Integer) mbeanConn.getAttribute(name, "Value");
    }

    public int getOfflinePartitionsCount() throws Exception {
        ObjectName name = new ObjectName("kafka.controller:type=KafkaController,name=OfflinePartitionsCount");
        return (Integer) mbeanConn.getAttribute(name, "Value");
    }
}
```

### Expose as Spring Boot Metrics (Micrometer)

```java
import io.micrometer.core.instrument.Gauge;
import io.micrometer.core.instrument.MeterRegistry;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class KafkaMetricsConfig {

    @Bean
    public KafkaJmxClient kafkaJmxClient() throws Exception {
        return new KafkaJmxClient("service:jmx:rmi:///jndi/rmi://localhost:9999/jmxrmi");
    }

    @Bean
    public void kafkaMetrics(KafkaJmxClient client, MeterRegistry registry) {
        Gauge.builder("kafka_under_replicated_partitions", client, c -> {
            try {
                return c.getUnderReplicatedPartitions();
            } catch (Exception e) {
                return -1.0;
            }
        }).description("Number of under-replicated partitions").register(registry);

        Gauge.builder("kafka_offline_partitions", client, c -> {
            try {
                return c.getOfflinePartitionsCount();
            } catch (Exception e) {
                return -1.0;
            }
        }).description("Number of offline partitions").register(registry);
    }
}
```

## Example Inputs / Outputs

### Example 1: Normal State

- Kafka cluster healthy.
- JMX values:
  - `UnderReplicatedPartitions = 0`
  - `OfflinePartitionsCount = 0`

Spring Boot `/actuator/metrics/kafka_under_replicated_partitions`:

```json
{
  "name": "kafka_under_replicated_partitions",
  "measurements": [
    { "statistic": "VALUE", "value": 0.0 }
  ],
  "availableTags": []
}
```

Spring Boot `/actuator/metrics/kafka_offline_partitions`:

```json
{
  "name": "kafka_offline_partitions",
  "measurements": [
    { "statistic": "VALUE", "value": 0.0 }
  ],
  "availableTags": []
}
```

### Example 2: Problem Detected

- One broker down, replication lagging.
- JMX values:
  - `UnderReplicatedPartitions = 5`
  - `OfflinePartitionsCount = 1`

Actuator output (simplified):

```json
{
  "name": "kafka_under_replicated_partitions",
  "measurements": [
    { "statistic": "VALUE", "value": 5.0 }
  ]
}
```

```json
{
  "name": "kafka_offline_partitions",
  "measurements": [
    { "statistic": "VALUE", "value": 1.0 }
  ]
}
```

You would typically set alerts such as:
- URP > 0 for more than X minutes.
- OfflinePartitionsCount > 0 immediately (critical).
