
Here’s a concise summary and examples around the Kafka consumer config `auto.offset.reset` in a Spring Boot / Kafka context.

---

## 1. What `auto.offset.reset` Does

`auto.offset.reset` controls **what a consumer does when there is no valid offset** for a partition (e.g., first time a group reads a topic, or the stored offset is out of range).

Possible values:

1. **`earliest`**  
   - Behavior: Start reading from the **beginning** of the partition (the oldest available message).  
   - Use when: You want to process **all existing messages** in the topic for a new consumer group.

2. **`latest`**  
   - Behavior: Start reading from the **end** of the partition (only new messages).  
   - Use when: You only care about **new messages** that arrive after the consumer starts.

3. **`none`**  
   - Behavior: If no offset is found for the consumer group, **throw an error**.  
   - Use when: You want to **fail fast** if offsets are missing, instead of silently starting at earliest or latest.

## 2. Spring Boot Configuration Examples

### application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    consumer:
      group-id: my-consumer-group
      auto-offset-reset: earliest   # or latest, or none
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
```

### application.properties

```properties
spring.kafka.bootstrap-servers=localhost:9092
spring.kafka.consumer.group-id=my-consumer-group
spring.kafka.consumer.auto-offset-reset=earliest  # or latest, or none
spring.kafka.consumer.key-deserializer=org.apache.kafka.common.serialization.StringDeserializer
spring.kafka.consumer.value-deserializer=org.apache.kafka.common.serialization.StringDeserializer
```

## 3. Java Consumer Factory Configuration

```java
import org.apache.kafka.clients.consumer.ConsumerConfig;
import org.apache.kafka.common.serialization.StringDeserializer;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.core.DefaultKafkaConsumerFactory;
import org.springframework.kafka.core.ConsumerFactory;

import java.util.HashMap;
import java.util.Map;

@Configuration
public class KafkaConsumerConfig {

    @Bean
    public ConsumerFactory<String, String> consumerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "my-consumer-group");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);

        // Choose one:
        props.put(ConsumerConfig.AUTO_OFFSET_RESET_CONFIG, "earliest"); // or "latest", "none"

        return new DefaultKafkaConsumerFactory<>(props);
    }
}
```

## 4. Example: Input / Output Behavior

Assume topic `test-topic` has messages:

- Offset 0: `"msg-0"`
- Offset 1: `"msg-1"`
- Offset 2: `"msg-2"`

Consumer group `group-A` has **no stored offsets** yet.

### Case 1: `auto.offset.reset=earliest`

- On first start, consumer begins at offset 0.
- **Output consumed:**
  - `"msg-0"`
  - `"msg-1"`
  - `"msg-2"`
  - plus any new messages that arrive.

### Case 2: `auto.offset.reset=latest`

- On first start, consumer begins at the end (after offset 2).
- **Output consumed:**
  - It will **not** read `"msg-0"`, `"msg-1"`, `"msg-2"`.
  - It will only read **new messages** produced after the consumer starts, e.g. `"msg-3"`, `"msg-4"`.

### Case 3: `auto.offset.reset=none`

- On first start, since there is no offset for `group-A`, the consumer throws an exception:
  - Example: `org.apache.kafka.clients.consumer.NoOffsetForPartitionException`
- **Output consumed:** none; the application fails unless you handle the error.

## 5. Simple Spring Kafka Listener Example

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class MyKafkaConsumer {

    @KafkaListener(topics = "test-topic", groupId = "my-consumer-group")
    public void listen(String message) {
        System.out.println("Received: " + message);
    }
}
```

- How it behaves depends on `spring.kafka.consumer.auto-offset-reset`:
  - `earliest`: prints all existing messages on first run.
  - `latest`: prints only new messages produced after the listener starts.
  - `none`: fails if no offset exists for `my-consumer-group`.
