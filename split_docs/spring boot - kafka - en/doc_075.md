
Here’s a concise summary and practical examples around **Spring Kafka producers and serializers** (String, JSON, Avro, custom).

---

## 1. String Serializer

**Use when**: Your key/value are plain strings.

**Config (application.yml)**

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

**Producer code**

```java
@Service
public class StringProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public StringProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String topic, String key, String value) {
        kafkaTemplate.send(topic, key, value);
    }
}
```

**Example input / output**

- Input to method:  
  `sendMessage("text-topic", "user1", "Hello Kafka")`
- Kafka message on topic `text-topic`:  
  `key = "user1"`, `value = "Hello Kafka"` (both as UTF‑8 strings)

## 2. JSON Serializer

**Use when**: You send Java objects as JSON.

**POJO**

```java
public class UserEvent {
    private String userId;
    private String action;
    // getters/setters/constructors
}
```

**Config**

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.springframework.kafka.support.serializer.JsonSerializer
```

**Producer code**

```java
@Service
public class JsonProducer {

    private final KafkaTemplate<String, UserEvent> kafkaTemplate;

    public JsonProducer(KafkaTemplate<String, UserEvent> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendUserEvent(UserEvent event) {
        kafkaTemplate.send("user-events", event.getUserId(), event);
    }
}
```

**Example input / output**

- Input object:

  ```java
  new UserEvent("u123", "LOGIN");
  ```

- Kafka message value (JSON):

  ```json
  {
    "userId": "u123",
    "action": "LOGIN"
  }
  ```

## 3. Avro Serializer

**Use when**: You need schema-based, compact, strongly-typed messages (often with Schema Registry).

**Avro schema (user-event.avsc)**

```json
{
  "type": "record",
  "name": "UserEvent",
  "namespace": "com.example.avro",
  "fields": [
    {"name": "userId", "type": "string"},
    {"name": "action", "type": "string"}
  ]
}
```

Generate Avro class (e.g., `com.example.avro.UserEvent`) via Avro plugin.

**Config (Confluent example)**

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    properties:
      schema.registry.url: http://localhost:8081
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: io.confluent.kafka.serializers.KafkaAvroSerializer
```

**Producer code**

```java
@Service
public class AvroProducer {

    private final KafkaTemplate<String, com.example.avro.UserEvent> kafkaTemplate;

    public AvroProducer(KafkaTemplate<String, com.example.avro.UserEvent> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendUserEvent(String userId, String action) {
        com.example.avro.UserEvent event =
            com.example.avro.UserEvent.newBuilder()
                .setUserId(userId)
                .setAction(action)
                .build();

        kafkaTemplate.send("user-events-avro", userId, event);
    }
}
```

**Example input / output**

- Input to method:  
  `sendUserEvent("u123", "LOGIN")`
- Kafka message: Avro binary payload with schema ID stored in Schema Registry.

## 4. Custom Serializer

**Use when**: You need a special binary format (e.g., encryption, compression, legacy format).

**Custom serializer**

```java
public class CustomUserEventSerializer implements Serializer<UserEvent> {

    @Override
    public byte[] serialize(String topic, UserEvent data) {
        if (data == null) return null;
        // Simple example: "userId|action" as bytes
        String payload = data.getUserId() + "|" + data.getAction();
        return payload.getBytes(StandardCharsets.UTF_8);
    }
}
```

**Config**

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: com.example.kafka.CustomUserEventSerializer
```

**Producer code**

```java
@Service
public class CustomProducer {

    private final KafkaTemplate<String, UserEvent> kafkaTemplate;

    public CustomProducer(KafkaTemplate<String, UserEvent> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendUserEvent(UserEvent event) {
        kafkaTemplate.send("user-events-custom", event.getUserId(), event);
    }
}
```

**Example input / output**

- Input object:

  ```java
  new UserEvent("u123", "LOGIN");
  ```

- Kafka message value (raw bytes):  
  UTF‑8 string `"u123|LOGIN"` as a byte array.

---

## Key Ideas

- **Serializer choice defines how Java objects become bytes** on the wire.
- **StringSerializer**: simple text messages.
- **JsonSerializer**: human-readable, flexible, good for many microservices.
- **AvroSerializer**: schema-based, compact, good for strongly-typed, evolving contracts.
- **Custom Serializer**: full control over binary format (e.g., encryption, compression, legacy protocols).
