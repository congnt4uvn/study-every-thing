
Here’s a concise summary of Spring Kafka consumer `AckMode` options, with code examples and what they imply for message processing.

---

## 1. What is `AckMode`?

`AckMode` controls **when offsets are committed** for messages consumed from Kafka. It’s configured on the `ContainerProperties` used by `ConcurrentKafkaListenerContainerFactory`.

Main modes:

- `BATCH`
- `RECORD`
- `MANUAL`
- `MANUAL_IMMEDIATE`

---

## 2. `BATCH` AckMode

**Idea:** Commit offsets **once per batch** of records returned by the poll.

- Offsets are committed after the listener processes the entire batch.
- Good for batch processing and higher throughput.
- If processing fails mid-batch, some records may be reprocessed.

### Example: Batch listener

```java
@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> batchFactory(
        ConsumerFactory<String, String> consumerFactory) {

    ConcurrentKafkaListenerContainerFactory<String, String> factory =
            new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory);
    factory.setBatchListener(true);

    factory.getContainerProperties().setAckMode(ContainerProperties.AckMode.BATCH);
    return factory;
}

@KafkaListener(topics = "orders", containerFactory = "batchFactory")
public void listenBatch(List<String> messages) {
    // INPUT: a batch of messages, e.g. ["m1", "m2", "m3"]
    for (String msg : messages) {
        // process each message
    }
    // OUTPUT: offsets for all messages in this batch are committed together
}
```

---

## 3. `RECORD` AckMode

**Idea:** Commit offsets **after each record** is processed.

- Safer: each message is acknowledged individually.
- More overhead: more frequent commits.

### Example: Record-by-record listener

```java
@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> recordFactory(
        ConsumerFactory<String, String> consumerFactory) {

    ConcurrentKafkaListenerContainerFactory<String, String> factory =
            new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory);
    factory.getContainerProperties().setAckMode(ContainerProperties.AckMode.RECORD);
    return factory;
}

@KafkaListener(topics = "payments", containerFactory = "recordFactory")
public void listenRecord(String message) {
    // INPUT: "payment-123"
    // process message
    // OUTPUT: offset for this single record is committed after method returns
}
```

---

## 4. `MANUAL` AckMode

**Idea:** Application code decides **when** to acknowledge, but commit is **asynchronous** (may be delayed).

- You call `Acknowledgment.acknowledge()` yourself.
- Commit may be grouped with others depending on configuration.

### Example: Manual ack (async commit)

```java
@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> manualFactory(
        ConsumerFactory<String, String> consumerFactory) {

    ConcurrentKafkaListenerContainerFactory<String, String> factory =
            new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory);
    factory.getContainerProperties().setAckMode(ContainerProperties.AckMode.MANUAL);
    return factory;
}

@KafkaListener(topics = "invoices", containerFactory = "manualFactory")
public void listenManual(String message, Acknowledgment ack) {
    // INPUT: "invoice-001"
    // 1. validate
    // 2. save to DB
    // 3. call external service
    if (/* all good */ true) {
        ack.acknowledge(); // mark offset to be committed
        // OUTPUT: offset will be committed asynchronously
    } else {
        // do not ack -> message may be redelivered
    }
}
```

---

## 5. `MANUAL_IMMEDIATE` AckMode

**Idea:** Application code decides **when**, and commit happens **immediately**.

- You call `ack.acknowledge()`, and the offset is committed right away.
- Strongest control and safety, but more commit overhead.

### Example: Manual immediate ack

```java
@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> manualImmediateFactory(
        ConsumerFactory<String, String> consumerFactory) {

    ConcurrentKafkaListenerContainerFactory<String, String> factory =
            new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory);
    factory.getContainerProperties()
           .setAckMode(ContainerProperties.AckMode.MANUAL_IMMEDIATE);
    return factory;
}

@KafkaListener(topics = "shipments", containerFactory = "manualImmediateFactory")
public void listenManualImmediate(String message, Acknowledgment ack) {
    // INPUT: "shipment-xyz"
    // process message
    ack.acknowledge(); // OUTPUT: offset committed immediately
}
```

---

## 6. When to use which?

- **BATCH**: high throughput, batch processing, some reprocessing acceptable.
- **RECORD**: simple per-message processing, want auto-commit after each record.
- **MANUAL**: need custom control over when to mark messages as processed, but can tolerate slight delay in commit.
- **MANUAL_IMMEDIATE**: need strict “process-then-commit-now” semantics (e.g., exactly-once-like flows with external systems).
