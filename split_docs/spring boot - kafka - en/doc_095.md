
Here’s a concise summary and some practical examples around using Avro/Protobuf with Confluent Schema Registry in a Spring Boot + Kafka setup.

---

## Key Ideas

1. **Schema Registry**  
   - Central service to store and manage message schemas (Avro, Protobuf, JSON Schema).  
   - Ensures producers and consumers agree on the structure of messages.  
   - Supports **schema evolution** (backward/forward compatibility).

2. **Avro / Protobuf with Kafka**  
   - Messages are serialized using Avro or Protobuf instead of plain JSON/String.  
   - The actual schema is stored in Schema Registry; Kafka messages contain a schema ID + binary payload.  
   - Producers/consumers use the Schema Registry to fetch the correct schema for (de)serialization.

3. **Spring Boot Integration**  
   - Use Confluent serializers/deserializers (`KafkaAvroSerializer`, `KafkaAvroDeserializer`, `KafkaProtobufSerializer`, etc.).  
   - Configure `schema.registry.url` and value/key serializer/deserializer in Spring Boot `application.yml` or `application.properties`.  
   - Use Spring Kafka’s `KafkaTemplate` and `@KafkaListener` with Avro/Protobuf types.

---

## Example: Avro with Spring Boot + Confluent Schema Registry

### 1. Avro Schema (user.avsc)

```json
{
  "type": "record",
  "name": "User",
  "namespace": "com.example.avro",
  "fields": [
    { "name": "id", "type": "string" },
    { "name": "name", "type": "string" },
    { "name": "age", "type": "int" }
  ]
}
```

Generate Java classes using the Avro Maven plugin (or Gradle equivalent). This will create `com.example.avro.User`.

### 2. Maven Dependencies

```xml
<dependencies>
    <!-- Spring Kafka -->
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>

    <!-- Confluent Avro serializer/deserializer -->
    <dependency>
        <groupId>io.confluent</groupId>
        <artifactId>kafka-avro-serializer</artifactId>
        <version>7.6.0</version>
    </dependency>

    <!-- Avro -->
    <dependency>
        <groupId>org.apache.avro</groupId>
        <artifactId>avro</artifactId>
        <version>1.11.3</version>
    </dependency>
</dependencies>
```

### 3. Spring Boot Configuration (application.yml)

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    properties:
      schema.registry.url: http://localhost:8081
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: io.confluent.kafka.serializers.KafkaAvroSerializer
    consumer:
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: io.confluent.kafka.serializers.KafkaAvroDeserializer
      properties:
        specific.avro.reader: true
    listener:
      ack-mode: record
```

### 4. Producer Example

```java
import com.example.avro.User;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class UserProducer {

    private final KafkaTemplate<String, User> kafkaTemplate;

    public UserProducer(KafkaTemplate<String, User> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendUser(User user) {
        kafkaTemplate.send("users-topic", user.getId().toString(), user);
    }
}
```

**Input (Java object):**

```java
User user = User.newBuilder()
    .setId("u-123")
    .setName("Alice")
    .setAge(30)
    .build();

userProducer.sendUser(user);
```

**What is sent to Kafka (conceptually):**

- Binary payload:  
  - Magic byte (0)  
  - Schema ID (from Schema Registry)  
  - Avro-encoded bytes for `{id: "u-123", name: "Alice", age: 30}`

### 5. Consumer Example

```java
import com.example.avro.User;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class UserConsumer {

    @KafkaListener(topics = "users-topic", groupId = "user-service")
    public void consume(User user) {
        // Input: Avro User object deserialized using Schema Registry
        System.out.println("Received user: " + user.getName() + ", age: " + user.getAge());
    }
}
```

**Output (console):**

```text
Received user: Alice, age: 30
```

## Example: Protobuf with Spring Boot + Schema Registry

### 1. Protobuf Schema (user.proto)

```proto
syntax = "proto3";

package com.example.proto;

message User {
  string id = 1;
  string name = 2;
  int32 age = 3;
}
```

Generate Java classes using the Protobuf plugin (e.g., `com.example.proto.UserOuterClass.User`).

### 2. Maven Dependencies

```xml
<dependencies>
    <!-- Spring Kafka -->
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>

    <!-- Confluent Protobuf serializer/deserializer -->
    <dependency>
        <groupId>io.confluent</groupId>
        <artifactId>kafka-protobuf-serializer</artifactId>
        <version>7.6.0</version>
    </dependency>

    <!-- Protobuf -->
    <dependency>
        <groupId>com.google.protobuf</groupId>
        <artifactId>protobuf-java</artifactId>
        <version>3.25.3</version>
    </dependency>
</dependencies>
```

### 3. Spring Boot Configuration (application.yml)

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    properties:
      schema.registry.url: http://localhost:8081
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: io.confluent.kafka.serializers.protobuf.KafkaProtobufSerializer
    consumer:
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: io.confluent.kafka.serializers.protobuf.KafkaProtobufDeserializer
      properties:
        specific.protobuf.value.type: com.example.proto.UserOuterClass$User
```

### 4. Producer Example (Protobuf)

```java
import com.example.proto.UserOuterClass.User;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class UserProtoProducer {

    private final KafkaTemplate<String, User> kafkaTemplate;

    public UserProtoProducer(KafkaTemplate<String, User> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendUser(User user) {
        kafkaTemplate.send("users-proto-topic", user.getId(), user);
    }
}
```

**Input (Java object):**

```java
User user = User.newBuilder()
    .setId("u-456")
    .setName("Bob")
    .setAge(25)
    .build();

userProtoProducer.sendUser(user);
```

### 5. Consumer Example (Protobuf)

```java
import com.example.proto.UserOuterClass.User;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class UserProtoConsumer {

    @KafkaListener(topics = "users-proto-topic", groupId = "user-proto-service")
    public void consume(User user) {
        System.out.println("Received proto user: " + user.getName() + ", age: " + user.getAge());
    }
}
```

**Output (console):**

```text
Received proto user: Bob, age: 25
```

---

## Summary

- Use **Confluent Schema Registry** to centrally manage Avro/Protobuf schemas for Kafka messages.
- Configure Spring Boot’s Kafka producer/consumer to use Confluent serializers/deserializers.
- Messages carry a schema ID; the actual schema is fetched from Schema Registry, enabling:
  - Strong typing in Java (Avro/Protobuf classes).
  - Safe schema evolution.
  - Reduced payload size compared to self-describing JSON.
