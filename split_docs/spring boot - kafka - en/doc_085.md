
Here‚Äôs a concise summary and practical examples around **Spring Kafka error handling for deserialization** using `ErrorHandlingDeserializer`.

---

## Key Ideas

1. **Problem**:  
   When Kafka consumers fail to deserialize messages (e.g., invalid JSON, wrong schema), the default behavior is often to fail the consumer and potentially stop processing.

2. **Solution**:  
   Use Spring Kafka‚Äôs `ErrorHandlingDeserializer` to:
   - Catch deserialization exceptions.
   - Route problematic records to an error handler (e.g., Dead Letter Topic).
   - Allow the consumer to continue processing other messages.

3. **How it works**:
   - You wrap your real deserializer (e.g., `JsonDeserializer`) with `ErrorHandlingDeserializer`.
   - On deserialization failure, the record value becomes `null`, and the exception is stored in headers.
   - Your error handler (e.g., `DefaultErrorHandler`) or listener can inspect headers and decide what to do.

---

## Basic Configuration (application.yml)

```yaml
spring:
  kafka:
    consumer:
      bootstrap-servers: localhost:9092
      group-id: my-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.springframework.kafka.support.serializer.ErrorHandlingDeserializer
      properties:
        spring.deserializer.value.delegate.class: org.springframework.kafka.support.serializer.JsonDeserializer
        spring.json.trusted.packages: "*"
```

- `ErrorHandlingDeserializer` wraps `JsonDeserializer`.
- On error, the exception is stored in headers like `kafka_dlt-exception-message`, etc.

## Consumer Configuration with Error Handler

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.config.ConcurrentKafkaListenerContainerFactory;
import org.springframework.kafka.core.ConsumerFactory;
import org.springframework.kafka.listener.DefaultErrorHandler;
import org.springframework.kafka.listener.DeadLetterPublishingRecoverer;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.util.backoff.FixedBackOff;

@Configuration
public class KafkaConsumerConfig {

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, MyEvent> kafkaListenerContainerFactory(
            ConsumerFactory<String, MyEvent> consumerFactory,
            KafkaTemplate<Object, Object> kafkaTemplate) {

        ConcurrentKafkaListenerContainerFactory<String, MyEvent> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory);

        // Send failed records to a dead-letter topic
        DeadLetterPublishingRecoverer recoverer = new DeadLetterPublishingRecoverer(kafkaTemplate,
                (record, ex) -> new org.apache.kafka.common.TopicPartition("my-topic.DLT", record.partition()));

        DefaultErrorHandler errorHandler =
                new DefaultErrorHandler(recoverer, new FixedBackOff(1000L, 3L)); // retry 3 times

        factory.setCommonErrorHandler(errorHandler);
        return factory;
    }
}
```

## Listener Example

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.messaging.handler.annotation.Header;
import org.springframework.kafka.support.KafkaHeaders;
import org.springframework.stereotype.Service;

@Service
public class MyKafkaListener {

    @KafkaListener(topics = "my-topic", groupId = "my-group")
    public void listen(ConsumerRecord<String, MyEvent> record,
                       @Header(KafkaHeaders.DLT_EXCEPTION_MESSAGE) String exceptionMessage) {

        MyEvent value = record.value(); // may be null if deserialization failed

        if (value == null) {
            // Handle deserialization error (e.g., log, metrics)
            System.out.println("Deserialization error: " + exceptionMessage);
        } else {
            // Normal processing
            System.out.println("Received: " + value);
        }
    }
}
```

## Example Input / Output

### Input message (valid JSON)

Topic: `my-topic`  
Key: `"user-1"`  
Value:
```json
{
  "id": 1,
  "name": "Alice"
}
```

- `JsonDeserializer` succeeds.
- Listener receives `MyEvent{id=1, name="Alice"}`.
- Output (log):
  ```text
  Received: MyEvent{id=1, name='Alice'}
  ```

### Input message (invalid JSON)

Topic: `my-topic`  
Key: `"user-2"`  
Value:
```json
{
  "id": "not-a-number",
  "name": "Bob"
}
```

- `JsonDeserializer` throws exception.
- `ErrorHandlingDeserializer` catches it, sets `value = null`, adds exception info to headers.
- `DefaultErrorHandler` retries; after retries, sends to `my-topic.DLT`.
- Listener for main topic may see `null` value (if configured to receive) or the record is handled by the error handler only.
- Listener for DLT (if you create one) can inspect the failed record and headers.

## DLT Listener Example

```java
@Service
public class DltListener {

    @KafkaListener(topics = "my-topic.DLT", groupId = "my-group-dlt")
    public void listenDlt(ConsumerRecord<String, byte[]> record,
                          @Header(KafkaHeaders.DLT_EXCEPTION_MESSAGE) String exceptionMessage) {
        System.out.println("DLT record key=" + record.key());
        System.out.println("DLT exception=" + exceptionMessage);
        // Optionally store to DB, alert, etc.
    }
}
```

---

**In summary**:  
`ErrorHandlingDeserializer` lets you safely handle deserialization failures in Spring Kafka by wrapping your real deserializer, capturing exceptions in headers, and integrating with error handlers and dead-letter topics so your consumer can keep running and you can inspect or reroute bad messages.
