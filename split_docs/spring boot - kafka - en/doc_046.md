## Key Ideas

1. **Offsets**  
   - An offset is the position of a consumer in a Kafka partition (like a cursor).
   - It tells Kafka ŌĆ£up to which message has this consumer processed?ŌĆØ

2. **`__consumer_offsets` topic**  
   - Kafka stores committed offsets in an internal topic called `__consumer_offsets`.
   - This is how Kafka remembers consumer progress across restarts and rebalances.

3. **Auto vs Manual Commit (Spring Kafka)**  
   - **Auto commit**: Kafka client periodically commits offsets automatically.
   - **Manual commit**: Your code decides when to commit (after successful processing).

4. **At-least-once vs At-most-once**  
   - **At-least-once**: Process first, then commit. Possible duplicates if reprocessing after failure.
   - **At-most-once**: Commit first, then process. No duplicates, but possible data loss if failure occurs after commit.

5. **Spring Kafka Listener Container**  
   - You configure offset commit behavior via `ContainerProperties` or `application.yml`.
   - You can use:
     - `AckMode` (e.g., `MANUAL`, `BATCH`, `RECORD`)
     - `Acknowledgment` object in listener methods to commit manually.

## Basic Configuration Examples

### 1. Auto Commit (Simple, but less control)

**application.yml:**
```yaml
spring:
  kafka:
    consumer:
      bootstrap-servers: localhost:9092
      group-id: my-group
      auto-offset-reset: earliest
      enable-auto-commit: true
      properties:
        # commit interval in ms
        "auto.commit.interval.ms": 5000
```

**Listener:**
```java
@KafkaListener(topics = "my-topic", groupId = "my-group")
public void listen(String message) {
    // Process message
    System.out.println("Received: " + message);
    // Offsets are committed automatically in the background
}
```

**Behavior:**  
- Kafka commits offsets every `auto.commit.interval.ms` milliseconds.
- If the app crashes between commits, some messages may be reprocessed.

### 2. Manual Commit per Record (At-least-once)

**application.yml:**
```yaml
spring:
  kafka:
    consumer:
      bootstrap-servers: localhost:9092
      group-id: my-group
      enable-auto-commit: false
    listener:
      ack-mode: MANUAL_IMMEDIATE
```

**Listener with manual acknowledgment:**
```java
@KafkaListener(topics = "my-topic", groupId = "my-group")
public void listen(String message, Acknowledgment ack) {
    try {
        // 1. Process the message
        System.out.println("Processing: " + message);

        // 2. Commit offset only after successful processing
        ack.acknowledge();
    } catch (Exception e) {
        // Do not acknowledge -> offset not committed
        // Message will be redelivered after restart or rebalance
        System.err.println("Error processing: " + message);
    }
}
```

**Input/Output example:**

- Topic `my-topic` has messages: `["A", "B", "C"]`
- Consumer starts with no committed offset.

Run 1:
- Receives `"A"` ŌåÆ prints `Processing: A` ŌåÆ `ack.acknowledge()` ŌåÆ offset for `"A"` committed to `__consumer_offsets`.
- Receives `"B"` ŌåÆ error occurs ŌåÆ no `ack.acknowledge()` ŌåÆ offset for `"B"` not committed.

Restart app:
- Kafka sees last committed offset is after `"A"`.
- It re-delivers `"B"` and `"C"`.

### 3. Batch Listener with Manual Commit

**application.yml:**
```yaml
spring:
  kafka:
    consumer:
      bootstrap-servers: localhost:9092
      group-id: my-batch-group
      enable-auto-commit: false
    listener:
      type: batch
      ack-mode: MANUAL
```

**Listener:**
```java
@KafkaListener(topics = "my-topic", groupId = "my-batch-group")
public void listenBatch(List<String> messages, Acknowledgment ack) {
    try {
        // Process all messages in the batch
        for (String msg : messages) {
            System.out.println("Batch processing: " + msg);
        }
        // Commit offsets for the whole batch
        ack.acknowledge();
    } catch (Exception e) {
        // No commit -> entire batch may be redelivered
        System.err.println("Error in batch: " + e.getMessage());
    }
}
```

**Behavior:**  
- Offsets are committed after the whole batch is processed.
- Good for throughput, but if failure occurs, the whole batch may be reprocessed.

---

### 4. At-most-once (Commit Before Processing)

Not common in Spring Kafka, but conceptually:

```java
@KafkaListener(topics = "my-topic", groupId = "my-group")
public void listen(String message, Acknowledgment ack) {
    // Commit first
    ack.acknowledge();

    // Then process
    System.out.println("Processing (at-most-once): " + message);
}
```

**Behavior:**  
- If the app crashes during processing, the message is lost (offset already committed).
- No duplicates, but possible data loss.

- You normally donŌĆÖt read `__consumer_offsets` directly in applications.

## Summary

- Offsets track consumer progress and are stored in `__consumer_offsets`.
- Use **auto commit** for simplicity, **manual commit** for reliability and control.
- Choose commit strategy based on your guarantees:
  - At-least-once (most common): process ŌåÆ commit.
  - At-most-once: commit ŌåÆ process.
- In Spring Boot/Spring Kafka, control this via `enable-auto-commit`, `ack-mode`, and `Acknowledgment` in `@KafkaListener`.
