
Here’s a concise summary and practical examples for **Spring Kafka batch listeners** using `@KafkaListener` with `List<ConsumerRecord>`.

---

## Key Ideas

1. **Batch listener**:  
   Instead of receiving one message at a time, the listener method receives a **list of records**:
   ```java
   @KafkaListener(topics = "my-topic")
   public void listen(List<ConsumerRecord<String, String>> records) { ... }
   ```

2. **Configuration requirement**:  
   To use batch listeners, you must enable batch mode in the **container factory**:
   ```java
   factory.setBatchListener(true);
   ```
   and configure `max.poll.records` to control batch size.

3. **Benefits**:
   - Higher throughput (process multiple records in one call).
   - Can do **bulk operations** (e.g., batch DB insert, batch processing).
   - Fewer network round-trips and less overhead per record.

4. **Trade-offs**:
   - Higher **latency** for individual messages (wait to form a batch).
   - Error handling is more complex (one bad record in a batch).
   - You must handle partial failures and retries carefully.

5. **Payload types**:
   - You can receive:
     - `List<ConsumerRecord<K, V>>`
     - `List<V>` (just the values)
     - `List<YourDto>` (with message conversion)
   - You can also get headers and metadata via `@Header` and `@Headers`.

## Basic Configuration Example

### Kafka Consumer Config

```java
@Configuration
@EnableKafka
public class KafkaConsumerConfig {

    @Bean
    public Map<String, Object> consumerConfigs() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "batch-group");
        // Control batch size
        props.put(ConsumerConfig.MAX_POLL_RECORDS_CONFIG, 50); // up to 50 records per poll
        return props;
    }

    @Bean
    public ConsumerFactory<String, String> consumerFactory() {
        return new DefaultKafkaConsumerFactory<>(consumerConfigs());
    }

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> batchFactory() {
        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory());
        factory.setBatchListener(true); // IMPORTANT: enable batch mode
        factory.getContainerProperties().setAckMode(ContainerProperties.AckMode.BATCH);
        return factory;
    }
}
```

## Batch Listener Examples

### 1. Raw `ConsumerRecord` Batch

```java
@Component
public class RawBatchListener {

    @KafkaListener(
        topics = "orders",
        containerFactory = "batchFactory"
    )
    public void listen(List<ConsumerRecord<String, String>> records) {
        System.out.println("Received batch size: " + records.size());

        for (ConsumerRecord<String, String> record : records) {
            System.out.printf(
                "Partition=%d, Offset=%d, Key=%s, Value=%s%n",
                record.partition(),
                record.offset(),
                record.key(),
                record.value()
            );
        }

        // Example: bulk processing
        List<String> values = records.stream()
                                     .map(ConsumerRecord::value)
                                     .toList();
        // e.g. saveAll(values) to DB
    }
}
```

**Example input (Kafka messages):**

Topic: `orders`  
Partition 0:
- offset 10: key=`order-1`, value=`{"id":1,"amount":100}`
- offset 11: key=`order-2`, value=`{"id":2,"amount":200}`

**Example output (logs):**
```text
Received batch size: 2
Partition=0, Offset=10, Key=order-1, Value={"id":1,"amount":100}
Partition=0, Offset=11, Key=order-2, Value={"id":2,"amount":200}
```

### 3. Batch with DTO Conversion

Assume JSON messages like:
```json
{"id": 1, "amount": 100.0}
```

DTO:

```java
public class OrderDto {
    private Long id;
    private Double amount;
    // getters/setters
}
```

Configure a `JsonDeserializer` and use:

```java
@Component
public class DtoBatchListener {

    @KafkaListener(
        topics = "order-json",
        containerFactory = "batchFactory"
    )
    public void listen(List<OrderDto> orders) {
        System.out.println("Received order batch: " + orders.size());
        // e.g. orderService.saveAll(orders);
    }
}
```

### 4. Batch Listener with Headers

```java
@Component
public class BatchWithHeadersListener {

    @KafkaListener(
        topics = "events",
        containerFactory = "batchFactory"
    )
    public void listen(
        List<String> payloads,
        @Header(KafkaHeaders.RECEIVED_PARTITION_ID) List<Integer> partitions,
        @Header(KafkaHeaders.OFFSET) List<Long> offsets
    ) {
        for (int i = 0; i < payloads.size(); i++) {
            System.out.printf(
                "Partition=%d, Offset=%d, Value=%s%n",
                partitions.get(i),
                offsets.get(i),
                payloads.get(i)
            );
        }
    }
}
```

## Error Handling & Retries (High-Level)

- Use `SeekToCurrentErrorHandler` (older) or `DefaultErrorHandler` (newer) to handle failures.
- For batch listeners, you can:
  - Retry the whole batch.
  - Or split and handle individual records (more complex).
- Consider **dead-letter topics** for messages that keep failing.
