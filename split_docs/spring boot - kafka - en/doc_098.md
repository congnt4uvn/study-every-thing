
Here’s a concise summary and practical examples around **Spring + Schema Registry** focusing on:

- How to configure the **Schema Registry URL**
- How to configure **subject naming strategies**

Assumptions:  
- Using **Confluent Schema Registry**  
- Using **Spring Boot** with **spring-kafka** and **Confluent serializers**  
- Using **Avro** as an example

---

## 1. Configure Schema Registry URL

You need to tell the Avro serializer/deserializer where the Schema Registry is.

### application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    properties:
      schema.registry.url: http://localhost:8081
      specific.avro.reader: true   # if using generated SpecificRecord classes
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: io.confluent.kafka.serializers.KafkaAvroSerializer
    consumer:
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: io.confluent.kafka.serializers.KafkaAvroDeserializer
      group-id: my-avro-group
```

This configures:

- Kafka broker: `localhost:9092`
- Schema Registry: `http://localhost:8081`
- Producer uses `KafkaAvroSerializer`
- Consumer uses `KafkaAvroDeserializer`

## 2. Subject Naming Strategies

Schema Registry stores schemas under **subjects**. The subject name strategy controls how those names are generated.

Common strategies (value side):

- `io.confluent.kafka.serializers.subject.TopicNameStrategy` (default)  
  Subject: `<topic>-value`
- `io.confluent.kafka.serializers.subject.RecordNameStrategy`  
  Subject: `<full.avro.RecordName>`
- `io.confluent.kafka.serializers.subject.TopicRecordNameStrategy`  
  Subject: `<topic>-<full.avro.RecordName>`

You can configure them separately for **key** and **value**.

### Example: TopicNameStrategy (default)

```yaml
spring:
  kafka:
    properties:
      schema.registry.url: http://localhost:8081
      # these are actually defaults, shown for clarity
      key.subject.name.strategy: io.confluent.kafka.serializers.subject.TopicNameStrategy
      value.subject.name.strategy: io.confluent.kafka.serializers.subject.TopicNameStrategy
```

Resulting subjects in Schema Registry for topic `user-events`:

- Key subject: `user-events-key`
- Value subject: `user-events-value`

---

### Example: RecordNameStrategy

Useful when the same Avro type is used across multiple topics and you want **one schema per record type**, not per topic.

```yaml
spring:
  kafka:
    properties:
      schema.registry.url: http://localhost:8081
      value.subject.name.strategy: io.confluent.kafka.serializers.subject.RecordNameStrategy
```

Resulting subject for Avro record `com.example.avro.User`:

- Value subject: `com.example.avro.User`

---

### Example: TopicRecordNameStrategy

Useful when you want to distinguish the same record type across topics.

```yaml
spring:
  kafka:
    properties:
      schema.registry.url: http://localhost:8081
      value.subject.name.strategy: io.confluent.kafka.serializers.subject.TopicRecordNameStrategy
```

For topic `user-events` and record `com.example.avro.User`:

- Value subject: `user-events-com.example.avro.User`

## 3. Spring Boot Producer Example

### Avro schema (user.avsc)

```json
{
  "type": "record",
  "name": "User",
  "namespace": "com.example.avro",
  "fields": [
    { "name": "id", "type": "string" },
    { "name": "name", "type": "string" }
  ]
}
```

Generate Java class `com.example.avro.User` via Avro plugin.

### Producer configuration (Java-based alternative)

```java
@Configuration
public class KafkaProducerConfig {

    @Bean
    public ProducerFactory<String, com.example.avro.User> producerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put("schema.registry.url", "http://localhost:8081");
        props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG,
                  org.apache.kafka.common.serialization.StringSerializer.class);
        props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,
                  io.confluent.kafka.serializers.KafkaAvroSerializer.class);

        // Example: use RecordNameStrategy
        props.put("value.subject.name.strategy",
                  "io.confluent.kafka.serializers.subject.RecordNameStrategy");

        return new DefaultKafkaProducerFactory<>(props);
    }

    @Bean
    public KafkaTemplate<String, com.example.avro.User> kafkaTemplate() {
        return new KafkaTemplate<>(producerFactory());
    }
}
```

### Sending a message

```java
@Service
public class UserProducer {

    private final KafkaTemplate<String, com.example.avro.User> kafkaTemplate;

    public UserProducer(KafkaTemplate<String, com.example.avro.User> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendUser() {
        com.example.avro.User user = com.example.avro.User.newBuilder()
                .setId("u-123")
                .setName("Alice")
                .build();

        kafkaTemplate.send("user-events", user.getId(), user);
    }
}
```

**Input (Java object):**

```java
User{id="u-123", name="Alice"}
```

**On the wire (conceptual):**

- Binary Avro payload with schema ID from Schema Registry
- Schema stored under subject depending on your strategy, e.g.:
  - `user-events-value` (TopicNameStrategy)
  - `com.example.avro.User` (RecordNameStrategy)
  - `user-events-com.example.avro.User` (TopicRecordNameStrategy)

## 4. Spring Boot Consumer Example

```java
@Configuration
public class KafkaConsumerConfig {

    @Bean
    public ConsumerFactory<String, com.example.avro.User> consumerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put("schema.registry.url", "http://localhost:8081");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "my-avro-group");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG,
                  org.apache.kafka.common.serialization.StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG,
                  io.confluent.kafka.serializers.KafkaAvroDeserializer.class);
        props.put("specific.avro.reader", true);

        // Must match producer’s subject naming strategy
        props.put("value.subject.name.strategy",
                  "io.confluent.kafka.serializers.subject.RecordNameStrategy");

        return new DefaultKafkaConsumerFactory<>(props);
    }

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, com.example.avro.User> kafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, com.example.avro.User> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory());
        return factory;
    }
}
```

### Listener

```java
@Service
public class UserConsumer {

    @KafkaListener(topics = "user-events", groupId = "my-avro-group")
    public void listen(com.example.avro.User user) {
        // Input to listener (deserialized):
        // User{id="u-123", name="Alice"}
        System.out.println("Received user: " + user);
    }
}
```

---

**Key ideas:**

- Configure `schema.registry.url` in Spring Kafka properties.
- Use Confluent Avro serializers/deserializers.
- Choose a **subject naming strategy** that matches your schema reuse and topic design:
  - Per topic: `TopicNameStrategy`
  - Per record type: `RecordNameStrategy`
  - Per topic + record type: `TopicRecordNameStrategy`
- Producer and consumer must use compatible strategies and schemas.
