
Here’s a concise overview of Spring Kafka consumers using the `@KafkaListener` annotation, with code examples and sample input/output.

---

## 1. What `@KafkaListener` Does

- Marks a method as a Kafka message listener.
- Spring creates a consumer that:
  - Subscribes to one or more topics.
  - Deserializes messages.
  - Invokes your method whenever a record arrives.
- Works with Spring Boot auto-configuration (via `spring-kafka`).

---

## 2. Basic Example

### Configuration (application.yml)

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    consumer:
      group-id: my-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
```

### Listener

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class SimpleKafkaConsumer {

    @KafkaListener(topics = "my-topic", groupId = "my-group")
    public void listen(String message) {
        System.out.println("Received message: " + message);
    }
}
```

**Input (Kafka message):**

- Topic: `my-topic`
- Key: `null`
- Value: `"Hello Kafka"`

**Output (application log):**

```text
Received message: Hello Kafka
```

---

## 3. Listening to Multiple Topics

```java
@KafkaListener(topics = {"topic1", "topic2"}, groupId = "multi-group")
public void listenMultiple(String message) {
    System.out.println("Received from topic1 or topic2: " + message);
}
```

## 4. Accessing Full `ConsumerRecord`

Use this when you need headers, key, partition, offset, etc.

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;

@KafkaListener(topics = "my-topic", groupId = "my-group")
public void listenRecord(ConsumerRecord<String, String> record) {
    System.out.printf(
        "Topic: %s, Partition: %d, Offset: %d, Key: %s, Value: %s%n",
        record.topic(), record.partition(), record.offset(),
        record.key(), record.value()
    );
}
```

**Input:**

- Topic: `my-topic`
- Partition: `1`
- Offset: `42`
- Key: `"user-1"`
- Value: `"User created"`

**Output:**

```text
Topic: my-topic, Partition: 1, Offset: 42, Key: user-1, Value: User created
```

## 5. Consuming JSON into a POJO

### POJO

```java
public class UserEvent {
    private String userId;
    private String action;

    // getters and setters
}
```

### Configuration (JSON deserializer)

```yaml
spring:
  kafka:
    consumer:
      group-id: user-events-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.springframework.kafka.support.serializer.JsonDeserializer
      properties:
        spring.json.trusted.packages: "*"
```

### Listener

```java
@KafkaListener(topics = "user-events", groupId = "user-events-group")
public void listenUserEvent(UserEvent event) {
    System.out.println("User: " + event.getUserId() + ", Action: " + event.getAction());
}
```

**Input (Kafka message value):**

```json
{"userId": "u123", "action": "LOGIN"}
```

**Output:**

```text
User: u123, Action: LOGIN
```

## 6. Using `@Payload`, `@Header`, etc.

```java
import org.springframework.kafka.support.KafkaHeaders;
import org.springframework.messaging.handler.annotation.Header;
import org.springframework.messaging.handler.annotation.Payload;

@KafkaListener(topics = "orders", groupId = "order-group")
public void listenOrder(
        @Payload String orderJson,
        @Header(KafkaHeaders.RECEIVED_KEY) String key,
        @Header(KafkaHeaders.RECEIVED_PARTITION) int partition) {

    System.out.printf("Key: %s, Partition: %d, Order: %s%n", key, partition, orderJson);
}
```

## 7. Concurrency (Multiple Consumer Threads)

```java
@KafkaListener(
    topics = "my-topic",
    groupId = "concurrent-group",
    concurrency = "3"
)
public void listenConcurrent(String message) {
    System.out.println("Thread: " + Thread.currentThread().getName() +
                       " received: " + message);
}
```

- `concurrency = "3"` creates 3 consumer threads in the same group.
- Kafka will distribute partitions among them.

---

## 8. Manual Acknowledgment (Manual Commit)

Enable manual ack in config:

```yaml
spring:
  kafka:
    listener:
      ack-mode: MANUAL
```

Listener:

```java
import org.springframework.kafka.support.Acknowledgment;

@KafkaListener(topics = "manual-topic", groupId = "manual-group")
public void listenManual(String message, Acknowledgment ack) {
    try {
        // process message
        System.out.println("Processing: " + message);
        ack.acknowledge(); // commit offset after successful processing
    } catch (Exception e) {
        // handle error (no commit -> message can be reprocessed)
    }
}
```

---

## 9. Error Handling (Basic)

Use `try/catch` inside the listener or configure an error handler:

```java
@KafkaListener(topics = "error-topic", groupId = "error-group")
public void listenWithErrorHandling(String message) {
    try {
        if (message.contains("fail")) {
            throw new RuntimeException("Simulated error");
        }
        System.out.println("Processed: " + message);
    } catch (Exception e) {
        System.err.println("Error processing message: " + message);
        // optionally send to a dead-letter topic
    }
}
```

---

**Summary of Key Ideas**

- `@KafkaListener` is the main way to define Kafka consumers in Spring Boot.
- You can:
  - Listen to one or multiple topics.
  - Receive raw values, POJOs, or full `ConsumerRecord`.
  - Access headers, keys, partitions, offsets.
  - Control concurrency and acknowledgment.
  - Integrate JSON deserialization and error handling.
