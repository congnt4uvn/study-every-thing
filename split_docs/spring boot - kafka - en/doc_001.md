
Here’s a concise summary of the key Kafka architecture components, plus simple Spring Boot code examples.

---

## 1. Broker
- A **Kafka broker** is a server that stores messages and serves client requests (producers/consumers).
- A Kafka cluster is made of multiple brokers for scalability and fault tolerance.

**Example (conceptual):**
- Broker 1: stores partitions 0, 1 of topic `orders`
- Broker 2: stores partitions 2, 3 of topic `orders`

---

## 2. Topic
- A **topic** is a named stream of records (like a category or feed name).
- Producers write messages to a topic; consumers read from a topic.
- Topics are split into partitions.

**Example topics:**
- `orders`
- `payments`
- `user-events`

---

## 3. Partition
- A **partition** is an ordered, immutable sequence of records within a topic.
- Each record in a partition has an **offset** (its position).
- Partitions allow parallelism: multiple consumers can read different partitions in parallel.

**Example:**
Topic: `orders`  
Partitions: `orders-0`, `orders-1`, `orders-2`  
- Message 1 → `orders-0`, offset 0  
- Message 2 → `orders-1`, offset 0  
- Message 3 → `orders-0`, offset 1  

---

## Spring Boot + Kafka Example

### Maven Dependency (pom.xml)
```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
        <version>3.1.0</version> <!-- adjust as needed -->
    </dependency>
</dependencies>
```

### application.yml
```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
    consumer:
      group-id: demo-group
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
```

### Producer Example

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class OrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private static final String TOPIC = "orders";

    public OrderProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendOrder(String orderId, String payload) {
        // key = orderId, value = payload
        kafkaTemplate.send(TOPIC, orderId, payload);
    }
}
```

**Input (method call):**
```java
orderProducer.sendOrder("order-123", "{ \"item\": \"book\", \"qty\": 2 }");
```

**Output (in Kafka):**
- Topic: `orders`
- Partition: chosen by Kafka (e.g., `orders-1` based on key hash)
- Record:
  - key: `"order-123"`
  - value: `"{ \"item\": \"book\", \"qty\": 2 }"`
  - offset: e.g., `15`

---

### Consumer Example

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class OrderConsumer {

    @KafkaListener(topics = "orders", groupId = "demo-group")
    public void listen(String message) {
        System.out.println("Received order: " + message);
    }
}
```

**Input (from Kafka topic `orders`):**
- value: `"{ \"item\": \"book\", \"qty\": 2 }"`

**Output (application log):**
```text
Received order: { "item": "book", "qty": 2 }
```

---

**Core idea:**  
- **Broker**: the Kafka server.  
- **Topic**: named stream of messages.  
- **Partition**: ordered subset of a topic enabling scalability and parallel consumption.
