
Here’s a concise summary and some practical examples around Kafka producer `enable.idempotence` and exactly-once semantics (EOS) in a Spring Boot context.

---

## 1. Key Concepts

### Idempotent Producer (`enable.idempotence=true`)
- Guarantees **no duplicates** and **no out-of-order messages** per partition, even on retries.
- Achieved by:
  - Assigning a **producer ID (PID)**.
  - Using **sequence numbers** per partition.
- Works at the **producer → broker** level only (does not cover consumer or processing logic).

**Key effects when enabled:**
- `acks` is forced to `all`
- `retries` is set to a high value (Integer.MAX_VALUE)
- `max.in.flight.requests.per.connection` is limited (≤ 5) to preserve ordering

---

### Exactly-Once Semantics (EOS)
- Goal: each message is **processed and written to output topics exactly once**, even with failures.
- Requires:
  1. **Idempotent producer** (`enable.idempotence=true`)
  2. **Transactions** (producer uses a `transactional.id`)
  3. **Read-process-write** pattern (e.g., consume from topic A, process, produce to topic B)
- EOS is end-to-end: consumer + processing + producer.

---

## 2. Basic Producer Config (Plain Kafka Java)

```java
Properties props = new Properties();
props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class.getName());
props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class.getName());

// Idempotent producer
props.put(ProducerConfig.ENABLE_IDEMPOTENCE_CONFIG, "true");
// Optional explicit settings (often auto-set when idempotence is true)
props.put(ProducerConfig.ACKS_CONFIG, "all");
props.put(ProducerConfig.RETRIES_CONFIG, Integer.toString(Integer.MAX_VALUE));
props.put(ProducerConfig.MAX_IN_FLIGHT_REQUESTS_PER_CONNECTION, "5");

KafkaProducer<String, String> producer = new KafkaProducer<>(props);

ProducerRecord<String, String> record =
        new ProducerRecord<>("input-topic", "key1", "value1");

producer.send(record, (metadata, exception) -> {
    if (exception == null) {
        System.out.printf("Sent to topic=%s partition=%d offset=%d%n",
                metadata.topic(), metadata.partition(), metadata.offset());
    } else {
        exception.printStackTrace();
    }
});

producer.flush();
producer.close();
```

**Input:**  
Topic: `input-topic`  
Message: key=`"key1"`, value=`"value1"`

**Output (example log):**
```text
Sent to topic=input-topic partition=0 offset=42
```

Even if the producer retries due to a transient error, the broker will store the record **once**.

## 3. EOS with Transactions (Plain Kafka Java)

```java
Properties props = new Properties();
props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class.getName());
props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class.getName());

// Enable idempotence + transactions
props.put(ProducerConfig.ENABLE_IDEMPOTENCE_CONFIG, "true");
props.put(ProducerConfig.TRANSACTIONAL_ID_CONFIG, "my-tx-producer-1");

KafkaProducer<String, String> producer = new KafkaProducer<>(props);
producer.initTransactions();

try {
    producer.beginTransaction();

    // Example: read from input-topic, process, write to output-topic
    // (pseudo-code for consumer part)
    // ConsumerRecord<String, String> inputRecord = ...
    String processedValue = "processed-value1";

    ProducerRecord<String, String> outRecord =
            new ProducerRecord<>("output-topic", "key1", processedValue);

    producer.send(outRecord);

    // Optionally send offsets to transaction (for EOS with consumer)
    // producer.sendOffsetsToTransaction(offsets, consumerGroupId);

    producer.commitTransaction();
} catch (Exception e) {
    producer.abortTransaction();
} finally {
    producer.close();
}
```

**Input:**  
Read from `input-topic` (e.g., key=`"key1"`, value=`"value1"`)

**Output:**  
Write to `output-topic` (key=`"key1"`, value=`"processed-value1"`) **exactly once**, even if failures occur during send/commit.

## 4. Spring Boot / Spring Kafka Configuration

### application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      properties:
        enable.idempotence: true
        # For transactions / EOS
        transactional.id: my-tx-producer-1
```

### Producer Factory & KafkaTemplate (Java Config)

```java
@Configuration
public class KafkaConfig {

    @Bean
    public ProducerFactory<String, String> producerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, StringSerializer.class);
        props.put(ProducerConfig.ENABLE_IDEMPOTENCE_CONFIG, true);
        props.put(ProducerConfig.TRANSACTIONAL_ID_CONFIG, "my-tx-producer-1");
        return new DefaultKafkaProducerFactory<>(props);
    }

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate() {
        return new KafkaTemplate<>(producerFactory());
    }
}
```

### Sending Messages with Transactions in Spring

```java
@Service
public class MyKafkaService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public MyKafkaService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendWithTransaction(String key, String value) {
        kafkaTemplate.executeInTransaction(operations -> {
            operations.send("output-topic", key, value);
            // you can send multiple records here atomically
            return null;
        });
    }
}
```

**Input:**  
`sendWithTransaction("user-1", "order-created")`

**Output (Kafka):**  
Topic: `output-topic`  
Message: key=`"user-1"`, value=`"order-created"`  
Written atomically as part of a transaction; if anything fails inside `executeInTransaction`, the send is rolled back.

---

## 5. Summary

- `enable.idempotence=true`:
  - Prevents duplicates and preserves order per partition at the producer level.
- EOS:
  - Combines idempotent producer + transactions + read-process-write pattern.
  - Ensures each input is processed and output exactly once, even with failures.
- In Spring Boot:
  - Configure `enable.idempotence` and `transactional.id`.
  - Use `KafkaTemplate.executeInTransaction(...)` for transactional sends.
