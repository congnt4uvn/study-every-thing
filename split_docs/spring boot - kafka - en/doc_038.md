
**Summary: Kafka Producer Performance Monitoring (Spring Boot)**

Key idea: To ensure high performance and reliability, you should monitor Kafka producer metrics such as:

- **record-send-rate**: How many records are sent per second  
- **retries**: How often the producer retries sending messages  
- **errors**: How many send failures occur  

These metrics help you detect bottlenecks, network issues, and configuration problems early.

---

## 1. Enabling Producer Metrics in Spring Boot

Spring Boot’s `KafkaTemplate` exposes Kafka producer metrics via Micrometer (if you use Spring Boot Actuator).

**Gradle dependencies:**
```groovy
implementation 'org.springframework.boot:spring-boot-starter-actuator'
implementation 'org.springframework.boot:spring-boot-starter-web'
implementation 'org.springframework.kafka:spring-kafka'
implementation 'io.micrometer:micrometer-registry-prometheus' // example registry
```

**application.yml:**
```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,prometheus,metrics
  metrics:
    tags:
      application: kafka-producer-app
```

## 2. Accessing Metrics Programmatically

You can read producer metrics from the `KafkaTemplate` or from Micrometer’s `MeterRegistry`.

### 2.1 From KafkaTemplate (raw Kafka metrics)

```java
import org.apache.kafka.clients.producer.Producer;
import org.apache.kafka.common.Metric;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Component;

import java.util.Map;

@Component
public class ProducerMetricsLogger {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public ProducerMetricsLogger(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void logProducerMetrics() {
        Producer<String, String> producer = (Producer<String, String>) kafkaTemplate
                .getProducerFactory()
                .createProducer();

        Map<org.apache.kafka.common.MetricName, ? extends Metric> metrics = producer.metrics();

        metrics.forEach((name, metric) -> {
            if (name.name().equals("record-send-rate")
                    || name.name().contains("retry")
                    || name.name().contains("error")) {
                System.out.printf("Metric: %s | Value: %s%n", name.name(), metric.metricValue());
            }
        });

        producer.close();
    }
}
```

**Example output (console):**
```text
Metric: record-send-rate | Value: 523.4
Metric: record-retry-rate | Value: 1.2
Metric: record-error-rate | Value: 0.0
```

### 2.2 From Micrometer / Actuator

Spring Boot exposes metrics via `/actuator/metrics`.

**Example: record-send-rate**

HTTP request:
```bash
curl "http://localhost:8080/actuator/metrics/kafka.producer.record.send.rate"
```

Example JSON response:
```json

  "name": "kafka.producer.record.send.rate",
  "measurements": [
```

**Example: retries**

## 3. Interpreting the Metrics

- **record-send-rate**  
  - High and stable: producer is sending efficiently  
  - Sudden drop: possible network, broker, or backpressure issue

- **retries** (e.g., `record-retry-rate`)  
  - Low: normal  
  - Increasing: network instability, broker overload, or wrong `acks`/timeout settings

- **errors** (e.g., `record-error-rate`)  
  - Should be near zero  
  - Non-zero: misconfiguration (topic not found, auth issues), broker problems, or serialization errors

## 4. Simple Producer Example (for context)

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class OrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public OrderProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendOrder(String orderId, String payload) {
        kafkaTemplate.send("orders", orderId, payload)
                .whenComplete((result, ex) -> {
                    if (ex != null) {
                        System.err.println("Send failed: " + ex.getMessage());
                    } else {
                        System.out.println("Sent to partition " +
                                result.getRecordMetadata().partition() +
                                " offset " + result.getRecordMetadata().offset());
                    }
                });
    }
}
```

**Example input:**
```java
orderProducer.sendOrder("order-123", "{ \"id\": \"order-123\", \"amount\": 99.99 }");
```

**Example output (log):**
```text
Sent to partition 0 offset 42
```

You can then watch how `record-send-rate`, `retries`, and `errors` change as you send more messages.
