
Here‚Äôs a concise summary of Spring Kafka consumer error handling, with code examples and I/O behavior.

---

## 1. Core Idea

Spring Kafka provides several mechanisms to handle errors in consumers:

1. `ErrorHandler` (record-based, single-record listeners)
2. `SeekToCurrentErrorHandler` (batch or record listeners; retries by seeking offsets)
3. `DeadLetterPublishingRecoverer` (publishes failed records to a dead-letter topic after retries are exhausted)

These are typically configured on the `ConcurrentKafkaListenerContainerFactory`.

---

## 2. Basic `ErrorHandler`

Used for simple, single-record listeners. You can log, skip, or seek manually.

### Configuration

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.apache.kafka.clients.consumer.Consumer;
import org.springframework.kafka.listener.ErrorHandler;
import org.springframework.kafka.config.ConcurrentKafkaListenerContainerFactory;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class KafkaConfig {

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory(
            org.springframework.kafka.core.ConsumerFactory<String, String> consumerFactory) {

        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory);

        factory.setErrorHandler(new ErrorHandler() {
            @Override
            public void handle(Exception thrownException,
                               ConsumerRecord<?, ?> data) {
                // Custom handling
                System.err.println("Error processing record: " + data);
                System.err.println("Exception: " + thrownException.getMessage());
                // You could log, send metrics, etc.
            }
        });

        return factory;
    }
}
```

### Example Listener

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class MyConsumer {

    @KafkaListener(topics = "input-topic", groupId = "group1")
    public void listen(String message) {
        if (message.contains("fail")) {
            throw new RuntimeException("Simulated failure");
        }
        System.out.println("Processed: " + message);
    }
}
```

### Example I/O

- Input message: `"hello"`  
  - Output: `Processed: hello`  
  - No error handler invoked.

- Input message: `"fail-this"`  
  - Listener throws `RuntimeException("Simulated failure")`  
  - `ErrorHandler.handle(...)` is called with the exception and the failing record.  
  - Depending on your commit/ack mode, the record may be retried or skipped.

## 3. `SeekToCurrentErrorHandler` (Retry + Seek)

This handler seeks the consumer back to the failed record‚Äôs offset so it can be retried a configurable number of times. After retries are exhausted, you can either skip or send to a dead-letter topic.

### Basic Retry Configuration

```java
import org.springframework.kafka.listener.SeekToCurrentErrorHandler;
import org.springframework.util.backoff.FixedBackOff;

@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory(
        org.springframework.kafka.core.ConsumerFactory<String, String> consumerFactory) {

    ConcurrentKafkaListenerContainerFactory<String, String> factory =
            new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory);

    // Retry 3 times with 1 second backoff
    SeekToCurrentErrorHandler errorHandler =
            new SeekToCurrentErrorHandler(new FixedBackOff(1000L, 3L));

    factory.setErrorHandler(errorHandler);
    return factory;
}
```

### Behavior

- Input message: `"fail-this"`  
  - Listener throws exception.  
  - Handler seeks back to the same offset.  
  - Message is retried up to 3 times (with 1s delay).  
  - After retries, the handler will skip the record (by default) and move on.

## 4. `DeadLetterPublishingRecoverer` (Dead-Letter Topic)

Used with `SeekToCurrentErrorHandler` to send failed records to a dead-letter topic after retries.

### Configuration with Dead-Letter Topic

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.kafka.listener.DeadLetterPublishingRecoverer;
import org.springframework.kafka.listener.SeekToCurrentErrorHandler;
import org.springframework.util.backoff.FixedBackOff;
import org.apache.kafka.common.TopicPartition;

@Bean
public DeadLetterPublishingRecoverer deadLetterPublishingRecoverer(
        KafkaTemplate<Object, Object> template) {

    // Default: send to <original-topic>.DLT, same partition
    return new DeadLetterPublishingRecoverer(template, (record, ex) ->
            new TopicPartition(record.topic() + ".DLT", record.partition()));
}

@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory(
        org.springframework.kafka.core.ConsumerFactory<String, String> consumerFactory,
        DeadLetterPublishingRecoverer recoverer) {

    ConcurrentKafkaListenerContainerFactory<String, String> factory =
            new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory);

    SeekToCurrentErrorHandler errorHandler =
            new SeekToCurrentErrorHandler(recoverer, new FixedBackOff(1000L, 3L));

    factory.setErrorHandler(errorHandler);
    return factory;
}
```

### Example I/O

Assume:
- Main topic: `input-topic`
- DLT: `input-topic.DLT`

Listener:

```java
@KafkaListener(topics = "input-topic", groupId = "group1")
public void listen(String message) {
    if (message.contains("fail")) {
        throw new RuntimeException("Simulated failure");
    }
    System.out.println("Processed: " + message);
}
```

Flow:

1. Input message: `"ok-1"`  
   - Output: `Processed: ok-1`  
   - No DLT.

2. Input message: `"fail-1"`  
   - Listener throws exception.  
   - `SeekToCurrentErrorHandler` retries 3 times.  
   - All retries fail.  
   - `DeadLetterPublishingRecoverer` publishes the record to `input-topic.DLT`.

DLT consumer:

```java
@KafkaListener(topics = "input-topic.DLT", groupId = "dlt-group")
public void dltListener(String message) {
    System.out.println("DLT received: " + message);
    // Here you can log, alert, or manually inspect
}
```

DLT Output:

- Message on `input-topic.DLT`: `"fail-1"`  
- Console: `DLT received: fail-1`

## 5. When to Use What

- Use `ErrorHandler` for:
  - Simple logging or custom handling without retries.
- Use `SeekToCurrentErrorHandler` for:
  - Automatic retries with backoff.
- Use `SeekToCurrentErrorHandler` + `DeadLetterPublishingRecoverer` for:
  - Retries + guaranteed capture of permanently failing messages in a DLT.
