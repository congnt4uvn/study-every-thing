
Here’s a concise summary and practical guide for using Spring Boot with Confluent Schema Registry via `KafkaAvroSerializer` / `KafkaAvroDeserializer`.

---

## Key Ideas

1. **Avro + Schema Registry**  
   - Messages are encoded as Avro and their schemas are stored in Confluent Schema Registry.  
   - Producers use `KafkaAvroSerializer` to serialize Avro objects and register/fetch schemas.  
   - Consumers use `KafkaAvroDeserializer` to deserialize bytes back into Avro objects using the schema from the registry.

2. **Spring Boot Integration**  
   - Configure Kafka producer/consumer properties to use Confluent serializers/deserializers.  
   - Use Spring Kafka’s `KafkaTemplate` for producing and `@KafkaListener` for consuming.  
   - You can use either:
     - **SpecificRecord** (generated Avro classes), or  
     - **GenericRecord** (schema-driven, no generated classes).

3. **Schema Evolution**  
   - Schema Registry enforces compatibility (e.g., backward/forward).  
   - You can evolve schemas (add optional fields, etc.) without breaking existing consumers.

## Basic Setup

### Maven Dependencies

```xml
<dependencies>
    <!-- Spring Kafka -->
    <dependency>
        <groupId>org.springframework.kafka</groupId>
        <artifactId>spring-kafka</artifactId>
    </dependency>

    <!-- Confluent Avro serializer/deserializer -->
    <dependency>
        <groupId>io.confluent</groupId>
        <artifactId>kafka-avro-serializer</artifactId>
        <version>7.6.0</version> <!-- match your Confluent platform -->
    </dependency>

    <!-- Avro (for code generation / GenericRecord) -->
    <dependency>
        <groupId>org.apache.avro</groupId>
        <artifactId>avro</artifactId>
        <version>1.11.3</version>
    </dependency>
</dependencies>
```

## Example Avro Schema

`user.avsc`:

```json
{
  "type": "record",
  "name": "User",
  "namespace": "com.example.avro",
  "fields": [
    { "name": "id", "type": "string" },
    { "name": "name", "type": "string" },
    { "name": "age", "type": ["null", "int"], "default": null }
  ]
}
```

Generate Java classes (SpecificRecord) via Avro plugin or build step.  
Resulting class: `com.example.avro.User`.

## Spring Boot Configuration (application.yml)

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    properties:
      schema.registry.url: http://localhost:8081

    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: io.confluent.kafka.serializers.KafkaAvroSerializer

    consumer:
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: io.confluent.kafka.serializers.KafkaAvroDeserializer
      properties:
        specific.avro.reader: true  # use generated classes
      group-id: user-service-group
```

## Producer Example (SpecificRecord)

```java
import com.example.avro.User;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class UserProducer {

    private final KafkaTemplate<String, User> kafkaTemplate;

    public UserProducer(KafkaTemplate<String, User> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendUser(User user) {
        kafkaTemplate.send("users-topic", user.getId().toString(), user);
    }
}
```

### Example Input → Output

**Input (Java object):**

```java
User user = User.newBuilder()
        .setId("u-123")
        .setName("Alice")
        .setAge(30)
        .build();

userProducer.sendUser(user);
```

**On the wire (Kafka message):**

- **Key**: `"u-123"` (String)  
- **Value**: Avro binary (includes schema ID from Schema Registry)  

Schema Registry stores a subject like `users-topic-value` with the `User` schema.

## Consumer Example (SpecificRecord)

```java
import com.example.avro.User;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class UserConsumer {

    @KafkaListener(topics = "users-topic", groupId = "user-service-group")
    public void listen(User user) {
        // user is already deserialized Avro SpecificRecord
        System.out.println("Received user: " + user.getId() + ", " + user.getName());
    }
}
```

### Example Input → Output

**Input (Kafka message):**  
- Key: `"u-123"`  
- Value: Avro bytes with schema ID

**Output (in consumer):**

```text
Received user: u-123, Alice
```

## Using GenericRecord Instead of SpecificRecord

If you don’t generate Java classes:

**Config:**

```yaml
spring:
  kafka:
    consumer:
      properties:
        specific.avro.reader: false
```

**Listener:**

```java
import org.apache.avro.generic.GenericRecord;

@KafkaListener(topics = "users-topic", groupId = "generic-consumer")
public void listen(GenericRecord record) {
    String id = record.get("id").toString();
    String name = record.get("name").toString();
    System.out.println("Received user: " + id + ", " + name);
}
```

---

## Main Takeaways

- Use `KafkaAvroSerializer` / `KafkaAvroDeserializer` with Schema Registry to handle Avro schemas centrally.
- Configure Spring Kafka producer/consumer to use these serializers.
- Choose between SpecificRecord (generated classes) or GenericRecord.
- Schema Registry enables safe schema evolution and compatibility across services.
