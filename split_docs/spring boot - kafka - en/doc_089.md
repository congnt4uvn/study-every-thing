
Here’s a concise summary and some practical code examples around advanced Spring Kafka usage with message headers, correlation IDs, and tracing (Sleuth / OpenTelemetry).

---

## 1. Message Headers in Spring Kafka

Kafka messages can carry metadata in headers (e.g., correlation IDs, content type, tenant ID). Spring Kafka makes headers easy to set and read.

### Producer: Setting Headers

```java
import org.apache.kafka.clients.producer.ProducerRecord;
import org.apache.kafka.common.header.internals.RecordHeader;
import org.springframework.kafka.core.KafkaTemplate;

@Service
public class OrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public OrderProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendOrder(String orderId, String payload, String correlationId) {
        ProducerRecord<String, String> record =
                new ProducerRecord<>("orders-topic", orderId, payload);

        record.headers()
              .add(new RecordHeader("correlation-id", correlationId.getBytes()))
              .add(new RecordHeader("content-type", "application/json".getBytes()));

        kafkaTemplate.send(record);
    }
}
```

**Input example:**

```java
sendOrder("order-123", "{\"item\":\"book\",\"qty\":1}", "corr-abc-123");
```

**Kafka message produced:**

- Topic: `orders-topic`
- Key: `order-123`
- Value: `{"item":"book","qty":1}`
- Headers:
  - `correlation-id = corr-abc-123`
  - `content-type = application/json`

### Consumer: Reading Headers

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.messaging.handler.annotation.Header;

@Service
public class OrderConsumer {

    @KafkaListener(topics = "orders-topic", groupId = "order-service")
    public void listen(
            String payload,
            @Header(name = "correlation-id", required = false) String correlationId,
            ConsumerRecord<String, String> record) {

        // correlationId may be null if not set
        System.out.println("Received payload: " + payload);
        System.out.println("Correlation ID: " + correlationId);
        System.out.println("Key: " + record.key());
    }
}
```

## 2. Correlation IDs

Correlation IDs let you track a logical request across multiple services and topics.

### Pattern

1. Generate a correlation ID at the entry point (e.g., HTTP request).
2. Put it in:
   - Logging MDC (Mapped Diagnostic Context).
   - Kafka message headers.
3. Downstream services read it from headers and propagate it further.

### Example: HTTP → Kafka with Correlation ID

```java
import org.slf4j.MDC;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/orders")
public class OrderController {

    private final OrderProducer orderProducer;

    public OrderController(OrderProducer orderProducer) {
        this.orderProducer = orderProducer;
    }

    @PostMapping
    public String createOrder(@RequestBody String orderJson,
                              @RequestHeader(value = "X-Correlation-Id", required = false) String corrId) {

        String correlationId = corrId != null ? corrId : java.util.UUID.randomUUID().toString();
        MDC.put("correlationId", correlationId);

        orderProducer.sendOrder("order-123", orderJson, correlationId);

        MDC.clear();
        return correlationId;
    }
}
```

**HTTP Input:**

```http
POST /orders
X-Correlation-Id: corr-abc-123
Content-Type: application/json

{"item":"book","qty":1}
```

**Output (HTTP response):**

```text
corr-abc-123
```

Logs in producer service will include `correlationId=corr-abc-123` if logging pattern is configured to show MDC.

## 3. Tracing with Spring Cloud Sleuth (legacy) or OpenTelemetry

### 3.1 With Spring Cloud Sleuth (pre-Spring Boot 3)

Sleuth automatically:

- Creates trace/span IDs for incoming HTTP requests.
- Propagates them via Kafka headers.
- Adds them to logs.

**Setup (Spring Boot 2.x example):**

```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-sleuth</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-sleuth-zipkin</artifactId>
</dependency>
```

**Behavior:**

- When you send a message with `KafkaTemplate`, Sleuth adds headers like:
  - `X-B3-TraceId`
  - `X-B3-SpanId`
  - `X-B3-ParentSpanId`
- Consumers with Sleuth will:
  - Read these headers.
  - Continue the trace.
  - Add trace IDs to logs.

No extra code is required for basic propagation; just use `KafkaTemplate` and `@KafkaListener` as usual.

---

### 3.2 With OpenTelemetry (Spring Boot 3+)

Spring Boot 3 uses Micrometer Tracing + OpenTelemetry instead of Sleuth.

**Dependencies (Gradle example):**

```groovy
implementation 'io.micrometer:micrometer-tracing-bridge-otel'
implementation 'io.opentelemetry:opentelemetry-exporter-otlp'
implementation 'org.springframework.kafka:spring-kafka'
```

**Configuration (application.yml):**

```yaml
management:
  tracing:
    sampling:
      probability: 1.0  # 100% for demo; lower in production
```

**Kafka instrumentation:**

- Use `KafkaTemplate` and `@KafkaListener` as usual.
- Micrometer Tracing + OTel instrumentation (via Spring Boot autoconfiguration) will:
  - Create spans for Kafka send/receive.
  - Inject/extract trace context into/from Kafka headers (e.g., `traceparent`).

**Example: Custom Span Around Kafka Send**

```java
import io.micrometer.tracing.Tracer;
import org.springframework.stereotype.Service;

@Service
public class TracedOrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private final Tracer tracer;

    public TracedOrderProducer(KafkaTemplate<String, String> kafkaTemplate, Tracer tracer) {
        this.kafkaTemplate = kafkaTemplate;
        this.tracer = tracer;
    }

    public void sendTracedOrder(String orderId, String payload) {
        var span = tracer.nextSpan().name("send-order").start();
        try (var scope = tracer.withSpan(span)) {
            kafkaTemplate.send("orders-topic", orderId, payload);
        } finally {
            span.end();
        }
    }
}
```

On the consumer side, the trace context is automatically restored when the message is received, so logs and spans are linked across services.

## 4. Putting It All Together

Typical advanced setup:

1. **Headers**: Use Kafka headers to carry correlation IDs, tenant IDs, content type, etc.
2. **Correlation ID**:
   - Generate at the edge (HTTP).
   - Store in MDC for logging.
   - Put into Kafka headers.
   - Read and propagate in downstream services.
3. **Tracing**:
   - Use Sleuth (Boot 2) or Micrometer Tracing + OpenTelemetry (Boot 3).
   - Let the framework inject/extract trace context into Kafka headers.
   - Use a tracing backend (Zipkin, Jaeger, Tempo, etc.) to visualize traces.

This combination gives you:

- Structured logs with correlation IDs.
- End-to-end distributed traces across HTTP and Kafka.
- Clear visibility into message flows between microservices.
