## Key Ideas: Kafka Topics, Partitions, and Ordering

1. **Topic**  
   - A logical category or feed name to which messages are published (e.g., `"orders"`, `"payments"`).

2. **Partition**  
   - Each topic is split into one or more **partitions**.  
   - A partition is an **ordered, append-only log** of messages.  
   - Partitions enable **scalability** and **parallelism** (multiple consumers can read different partitions in parallel).

3. **Ordering Guarantees**  
   - Kafka **only guarantees ordering within a single partition**.  
   - Messages in the same partition are read in the exact order they were written.  
   - Across different partitions of the same topic, **no global ordering** is guaranteed.  
   - To preserve order for related messages, you must ensure they go to the **same partition** (usually by using a key).

## Example: Producing Ordered Messages with Spring Boot + Kafka

### Producer Configuration (Spring Boot `application.yml`)

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

### Producer Code (Java)

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class OrderProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private static final String TOPIC = "orders";

    public OrderProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    // All events for the same orderId will go to the same partition (ordering preserved per orderId)
    public void sendOrderEvent(String orderId, String event) {
        kafkaTemplate.send(TOPIC, orderId, event);
    }
}
```

### Example Inputs (Producer Calls)

```java
orderProducer.sendOrderEvent("order-123", "CREATED");
orderProducer.sendOrderEvent("order-123", "PAID");
orderProducer.sendOrderEvent("order-123", "SHIPPED");

orderProducer.sendOrderEvent("order-456", "CREATED");
orderProducer.sendOrderEvent("order-456", "CANCELLED");
```

Because the **key** is the `orderId`, all messages for `"order-123"` go to the same partition, and all messages for `"order-456"` go to (possibly another) single partition. Ordering is preserved **per orderId**.

## Consumer Example (Spring Boot)

```java
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.stereotype.Service;

@Service
public class OrderConsumer {

    @KafkaListener(topics = "orders", groupId = "order-service")
    public void consume(String message) {
        System.out.println("Received: " + message);
    }
}
```

### Example Output (Consumer Logs)

For partition 0 (e.g., `order-123`):

```text
Received: CREATED
Received: PAID
Received: SHIPPED
```

For partition 1 (e.g., `order-456`):

```text
Received: CREATED
Received: CANCELLED
```

Note: The **relative order** within each partition is preserved, but messages from different partitions may interleave when viewed together.
