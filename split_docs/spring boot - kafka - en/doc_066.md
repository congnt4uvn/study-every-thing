
Here’s a concise summary and some practical examples around **monitoring & observability for Kafka producers/consumers and lag** in a Spring Boot context.

---

## Key Points

1. **Producer Metrics**
   - Track how your producer behaves: throughput, latency, errors, retries, batch sizes.
   - Common metrics:
     - `record-send-rate`, `byte-rate`
     - `request-latency-avg`, `request-latency-max`
     - `record-error-rate`, `record-retry-rate`
   - Use these to detect slow brokers, network issues, or misconfiguration.

2. **Consumer Metrics**
   - Monitor how consumers process messages: throughput, poll latency, commit behavior.
   - Common metrics:
     - `records-consumed-rate`, `bytes-consumed-rate`
     - `poll-latency-avg`
     - `commit-latency-avg`
   - Helps identify slow consumers, backpressure, or processing bottlenecks.

3. **Lag Monitoring**
   - **Consumer lag** = difference between the latest offset in a partition and the offset last committed by a consumer group.
   - High or growing lag indicates:
     - Consumer is too slow
     - Not enough consumer instances
     - Processing errors or frequent restarts
   - Lag is critical for **SLA** and **real-time** guarantees.

4. **Spring Boot Integration**
   - Spring Boot + Spring Kafka expose Kafka metrics via Micrometer.
   - You can export metrics to Prometheus, Grafana, etc.
   - Lag can be monitored via:
     - Kafka tools (e.g., `kafka-consumer-groups.sh`)
     - Metrics from the broker (e.g., `kafka_consumergroup_lag` in Prometheus)
     - Custom code using `AdminClient` or `KafkaConsumer` APIs.

## Example: Spring Boot Producer with Metrics

**build.gradle (relevant parts)**

```groovy
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter'
    implementation 'org.springframework.kafka:spring-kafka'
    implementation 'io.micrometer:micrometer-registry-prometheus'
}
```

**application.yml**

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
management:
  endpoints:
    web:
      exposure:
        include: health,info,prometheus
  metrics:
    tags:
      application: kafka-demo
```

**Producer code**

```java
@Service
public class DemoProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public DemoProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String topic, String key, String value) {
        kafkaTemplate.send(topic, key, value)
            .whenComplete((result, ex) -> {
                if (ex != null) {
                    // error metric will be recorded by Kafka client
                    System.err.println("Send failed: " + ex.getMessage());
                } else {
                    System.out.println("Sent to " + result.getRecordMetadata());
                }
            });
    }
}
```

**Example input/output**

- Input (call from REST controller):

```http
POST /send
Content-Type: application/json

{
  "topic": "orders",
  "key": "order-123",
  "value": "created"
}
```

- Output (HTTP):

```http
200 OK
{"status":"sent"}
```

- Metrics (Prometheus scrape at `/actuator/prometheus` – simplified):

```text
kafka_producer_record_send_total{client_id="producer-1",topic="orders"} 100
kafka_producer_record_error_total{client_id="producer-1",topic="orders"} 2
kafka_producer_record_send_rate{client_id="producer-1"} 10.5
```

## Example: Spring Boot Consumer with Metrics

**Consumer configuration**

```yaml
spring:
  kafka:
    consumer:
      bootstrap-servers: localhost:9092
      group-id: order-service
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      auto-offset-reset: latest
```

**Consumer code**

```java
@Service
public class DemoConsumer {

    @KafkaListener(topics = "orders", groupId = "order-service")
    public void listen(String message) {
        long start = System.currentTimeMillis();
        try {
            // process message
            System.out.println("Received: " + message);
        } finally {
            long duration = System.currentTimeMillis() - start;
            // You can record custom processing time metric if needed
        }
    }
}
```

**Example input/output**

- Input: message produced to topic `orders` with value `"created"`.
- Output (consumer log):

```text
Received: created
```

- Metrics (Prometheus – simplified):

```text
kafka_consumer_records_consumed_total{client_id="consumer-1",topic="orders"} 100
kafka_consumer_poll_latency_avg{client_id="consumer-1"} 5.2
```

## Example: Lag Monitoring via AdminClient (Java)

You can compute lag per partition by comparing **end offsets** with **committed offsets**.

```java
@Component
public class LagMonitor {

    private final AdminClient adminClient;

    public LagMonitor() {
        Properties props = new Properties();
        props.put(AdminClientConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        this.adminClient = AdminClient.create(props);
    }

    public Map<TopicPartition, Long> getLag(String groupId) throws Exception {
        Map<TopicPartition, Long> lagMap = new HashMap<>();

        // 1. Get committed offsets for the group
        ListConsumerGroupOffsetsResult offsetsResult =
                adminClient.listConsumerGroupOffsets(groupId);
        Map<TopicPartition, OffsetAndMetadata> committed =
                offsetsResult.partitionsToOffsetAndMetadata().get();

        // 2. Get end offsets from the broker
        try (KafkaConsumer<byte[], byte[]> consumer =
                     new KafkaConsumer<>(consumerProps())) {

            consumer.assign(committed.keySet());
            consumer.seekToEnd(committed.keySet());

            for (TopicPartition tp : committed.keySet()) {
                long endOffset = consumer.position(tp);
                long committedOffset = committed.get(tp).offset();
                long lag = endOffset - committedOffset;
                lagMap.put(tp, lag);
            }
        }

        return lagMap;
    }

    private Properties consumerProps() {
        Properties props = new Properties();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "lag-monitor");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG,
                  "org.apache.kafka.common.serialization.ByteArrayDeserializer");
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG,
                  "org.apache.kafka.common.serialization.ByteArrayDeserializer");
        return props;
    }
}
```

**Example output**

```text
TopicPartition{topic='orders', partition=0} -> lag=120
TopicPartition{topic='orders', partition=1} -> lag=45
```

## Typical Prometheus/Grafana Lag Metric

If you use a Kafka exporter, you might see metrics like:

```text
kafka_consumergroup_lag{consumergroup="order-service",topic="orders",partition="0"} 120
```

You can alert when:

- `lag > 1000` for more than 5 minutes
- or `lag` is continuously increasing.

---

**Summary**

- Monitor **producer metrics** to ensure messages are sent reliably and quickly.
- Monitor **consumer metrics** to ensure messages are processed in time.
- Track **consumer lag** to know if your system is keeping up with incoming data.
- In Spring Boot, use **Spring Kafka + Micrometer** to expose metrics and integrate with Prometheus/Grafana.
- For deeper control, use **AdminClient** or Kafka tools to compute and alert on lag.
