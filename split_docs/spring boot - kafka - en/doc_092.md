
Here’s a concise summary of common Kafka serialization formats in a Spring Boot context, with simple code examples and sample inputs/outputs.

---

## 1. String Serialization

**Use when:** Messages are plain text (e.g., logs, simple events).

**Producer config:**
```java
props.put(ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG,
          org.apache.kafka.common.serialization.StringSerializer.class);
props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,
          org.apache.kafka.common.serialization.StringSerializer.class);
```

**Send message:**
```java
kafkaTemplate.send("text-topic", "user-1", "Hello Kafka");
```

**Input (producer value):**
```text
"Hello Kafka"
```

**Output (on consumer side):**
```text
"Hello Kafka"
```

---

## 2. JSON Serialization

**Use when:** You have simple POJOs and want human-readable messages.

**Example POJO:**
```java
public class UserEvent {
    private String userId;
    private String action;
    // getters/setters
}
```

**Spring Boot config (application.yml):**
```yaml
spring:
  kafka:
    producer:
      value-serializer: org.springframework.kafka.support.serializer.JsonSerializer
    consumer:
      value-deserializer: org.springframework.kafka.support.serializer.JsonDeserializer
      properties:
        spring.json.trusted.packages: "*"
```

**Send message:**
```java
UserEvent event = new UserEvent();
event.setUserId("u123");
event.setAction("LOGIN");

kafkaTemplate.send("user-events", event);
```

**Input (object):**
```java
UserEvent{ userId="u123", action="LOGIN" }
```

**Output (on wire as JSON):**
```json
{"userId":"u123","action":"LOGIN"}
```

---

## 3. Avro Serialization

**Use when:** You need schema evolution, compact binary format, and strong typing.

**Avro schema (user_event.avsc):**
```json
{
  "type": "record",
  "name": "UserEvent",
  "namespace": "com.example.avro",
  "fields": [
    {"name": "userId", "type": "string"},
    {"name": "action", "type": "string"}
  ]
}
```

Generate classes with Avro plugin, then:

**Producer config (with Confluent Schema Registry):**
```java
props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,
          io.confluent.kafka.serializers.KafkaAvroSerializer.class);
props.put("schema.registry.url", "http://localhost:8081");
```

**Send message:**
```java
UserEvent event = UserEvent.newBuilder()
    .setUserId("u123")
    .setAction("LOGIN")
    .build();

kafkaTemplate.send("user-events-avro", event);
```

**Input (object):**
```java
UserEvent{ userId="u123", action="LOGIN" }
```

**Output (on wire):**
- Binary Avro payload (not human-readable)
- Schema ID stored in Schema Registry

---

## 4. Protobuf Serialization

**Use when:** You want compact binary format, strong typing, and language-agnostic schemas (often used in gRPC ecosystems).

**Protobuf schema (user_event.proto):**
```proto
syntax = "proto3";

package com.example.proto;

message UserEvent {
  string userId = 1;
  string action = 2;
}
```

Generate Java classes, then:

**Producer config (with Protobuf serializer):**
```java
props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,
          io.confluent.kafka.serializers.protobuf.KafkaProtobufSerializer.class);
props.put("schema.registry.url", "http://localhost:8081");
```

**Send message:**
```java
UserEvent event = UserEvent.newBuilder()
    .setUserId("u123")
    .setAction("LOGIN")
    .build();

kafkaTemplate.send("user-events-proto", event);
```

**Input (object):**
```java
UserEvent{ userId="u123", action="LOGIN" }
```

**Output (on wire):**
- Compact Protobuf binary
- Schema managed via Schema Registry (if used)

---

## 5. Thrift Serialization

**Use when:** Your system already uses Apache Thrift for cross-language services.

**Thrift IDL (user_event.thrift):**
```thrift
namespace java com.example.thrift

struct UserEvent {
  1: string userId,
  2: string action
}
```

Generate Java classes, then implement custom serializer:

```java
public class ThriftSerializer<T extends TBase> implements Serializer<T> {
    @Override
    public byte[] serialize(String topic, T data) {
        try (ByteArrayOutputStream out = new ByteArrayOutputStream()) {
            TIOStreamTransport transport = new TIOStreamTransport(out);
            TProtocol protocol = new TBinaryProtocol(transport);
            data.write(protocol);
            return out.toByteArray();
        } catch (Exception e) {
            throw new SerializationException("Error serializing Thrift", e);
        }
    }
}
```

**Producer config:**
```java
props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,
          "com.example.kafka.ThriftSerializer");
```

**Input (object):**
```java
UserEvent{ userId="u123", action="LOGIN" }
```

**Output (on wire):**
- Thrift binary format

---

## Quick Comparison

- **String**: Easiest, no schema, text only.
- **JSON**: Human-readable, flexible, no strict schema by default.
- **Avro**: Binary, schema-based, good for schema evolution, common in Kafka.
- **Protobuf**: Binary, schema-based, widely used with gRPC.
- **Thrift**: Binary, schema-based, good if you already use Thrift elsewhere.
