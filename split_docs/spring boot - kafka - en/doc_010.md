
Here’s a concise summary and some practical examples around Kafka data flow, consumer offsets, and zero-copy (sendfile) in a Spring Boot context.

---

## Key Ideas

1. **Data Flow in Kafka**
   - Producers write messages to **topics**, which are split into **partitions**.
   - Messages in each partition are stored in an **append-only log** with a sequential **offset**.
   - Consumers read messages **sequentially by offset** from each partition.

2. **Fetching & Offsets (Consumers)**
   - Each consumer keeps track of the **offset** it has read up to.
   - Kafka brokers send data to consumers in **batches** starting from the requested offset.
   - Offsets can be:
     - **Automatically committed** (periodically by the client), or
     - **Manually committed** (after processing is confirmed).

3. **Zero-Copy Transfer (sendfile)**
   - Kafka uses the OS-level `sendfile` system call to send data from disk to network **without copying** it into user-space.
   - This is called **zero-copy** and:
     - Reduces CPU usage.
     - Increases throughput.
   - The broker reads data from the log segment file and streams it directly to the consumer’s socket.

## Spring Boot + Kafka: Basic Consumer with Offsets

### Maven Dependency

```xml
<dependency>
    <groupId>org.springframework.kafka</groupId>
    <artifactId>spring-kafka</artifactId>
</dependency>
```

### Basic Configuration (application.yml)

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    consumer:
      group-id: my-group
      auto-offset-reset: earliest
      enable-auto-commit: false   # we’ll commit manually
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
```

### Consumer with Manual Offset Commit

```java
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.apache.kafka.clients.consumer.Consumer;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.kafka.support.Acknowledgment;
import org.springframework.stereotype.Service;

@Service
public class MyKafkaConsumer {

    @KafkaListener(topics = "my-topic", containerFactory = "kafkaListenerContainerFactory")
    public void listen(ConsumerRecord<String, String> record,
                       Acknowledgment ack,
                       Consumer<String, String> consumer) {

        // Input: message from Kafka
        String key = record.key();
        String value = record.value();
        long offset = record.offset();
        int partition = record.partition();

        System.out.printf("Received message: key=%s, value=%s, partition=%d, offset=%d%n",
                key, value, partition, offset);

        // Process the message (business logic here)
        // ...

        // Output: after successful processing, commit offset
        ack.acknowledge();  // commits offset for this record
    }
}
```

### Listener Container Factory (to enable manual ack)

```java
import org.apache.kafka.clients.consumer.ConsumerConfig;
import org.apache.kafka.common.serialization.StringDeserializer;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.kafka.config.ConcurrentKafkaListenerContainerFactory;
import org.springframework.kafka.core.DefaultKafkaConsumerFactory;

import java.util.HashMap;
import java.util.Map;

@Configuration
public class KafkaConsumerConfig {

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "my-group");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class);
        props.put(ConsumerConfig.ENABLE_AUTO_COMMIT_CONFIG, false);

        DefaultKafkaConsumerFactory<String, String> consumerFactory =
                new DefaultKafkaConsumerFactory<>(props);

        ConcurrentKafkaListenerContainerFactory<String, String> factory =
                new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory);
        factory.getContainerProperties().setAckMode(
                org.springframework.kafka.listener.ContainerProperties.AckMode.MANUAL_IMMEDIATE
        );
        return factory;
    }
}
```

## Example: Producer + Consumer I/O

### Producer (Input to Kafka)

```java
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;

@Service
public class MyKafkaProducer {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public MyKafkaProducer(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    public void sendMessage(String key, String value) {
        kafkaTemplate.send("my-topic", key, value);
        System.out.printf("Sent message: key=%s, value=%s%n", key, value);
    }
}
```

**Example Input:**

```java
myKafkaProducer.sendMessage("user-1", "Hello Kafka");
myKafkaProducer.sendMessage("user-2", "Another message");
```

**Example Output (Consumer logs):**

```text
Received message: key=user-1, value=Hello Kafka, partition=0, offset=0
Received message: key=user-2, value=Another message, partition=0, offset=1
```

Offsets `0` and `1` show the sequential reading from the partition log.

---

## Where Zero-Copy Fits

- You don’t configure `sendfile` directly in Spring Boot; it’s handled by the **Kafka broker**.
- When the consumer fetches data:
  1. Consumer requests messages from offset N.
  2. Broker reads from the log segment file.
  3. Broker uses `sendfile` to stream bytes from disk to the network socket.
- Result: high-throughput, low-CPU data transfer from Kafka to your Spring Boot consumer.
