
Here’s a concise overview of the main Kafka Streams operations, with simple Spring Boot–style examples (Java) and example inputs/outputs.

---

## 1. `map` / `mapValues`

**Purpose:** Transform records (key, value) into new (key, value) pairs.

- `map`: can change both key and value  
- `mapValues`: changes only the value

**Example:**

```java
KStream<String, String> input = builder.stream("input-topic");

// mapValues: uppercase the value
KStream<String, String> uppercased = input.mapValues(value -> value.toUpperCase());

// map: change key and value
KStream<String, String> rekeyed = input.map((key, value) -> KeyValue.pair(value, key));
```

**Example Input (topic `input-topic`):**
- ("user1", "hello")
- ("user2", "world")

**Output of `uppercased`:**
- ("user1", "HELLO")
- ("user2", "WORLD")

---

## 2. `filter` / `filterNot`

**Purpose:** Keep or remove records based on a condition.

**Example:**

```java
KStream<String, String> input = builder.stream("input-topic");

// Keep only messages where value length > 5
KStream<String, String> longMessages =
        input.filter((key, value) -> value.length() > 5);
```

**Example Input:**
- ("user1", "hi")
- ("user2", "welcome")
- ("user3", "kafka")

**Output of `longMessages`:**
- ("user2", "welcome")   // length 7 > 5

---

## 3. `groupByKey` / `groupBy`

**Purpose:** Group records by key (or by a new key) to prepare for aggregations.

- `groupByKey`: group using existing key  
- `groupBy`: group using a new key extractor

**Example:**

```java
KStream<String, String> input = builder.stream("input-topic");

// Group by existing key
KGroupedStream<String, String> groupedByKey = input.groupByKey();

// Group by value (e.g., group all same values together)
KGroupedStream<String, String> groupedByValue =
        input.groupBy((key, value) -> value);
```

**Example Input:**
- ("user1", "A")
- ("user1", "B")
- ("user2", "A")

`groupByKey` groups by user; `groupBy` example groups by value "A" or "B".

---

## 4. `aggregate` / `count` / `reduce`

**Purpose:** Build stateful aggregations over grouped data.

- `count`: count records per key  
- `reduce`: combine values with a function  
- `aggregate`: fully custom aggregation type

**Example: count messages per user**

```java
KStream<String, String> input = builder.stream("input-topic");

KTable<String, Long> messageCounts = input
        .groupByKey()
        .count(Materialized.as("message-counts-store"));
```

**Example Input:**
- ("user1", "msg1")
- ("user1", "msg2")
- ("user2", "msg3")

**Output (KTable `messageCounts`):**
- user1 → 2
- user2 → 1

**Example: custom aggregate (concatenate values)**

```java
KTable<String, String> aggregated = input
        .groupByKey()
        .aggregate(
            () -> "", // initializer
            (key, value, agg) -> agg + "|" + value, // adder
            Materialized.as("concat-store")
        );
```

---

## 5. `join` (KStream–KStream, KStream–KTable, KTable–KTable)

**Purpose:** Combine records from two streams/tables with the same key.

### KStream–KStream Join (within a time window)

```java
KStream<String, String> left = builder.stream("left-topic");
KStream<String, String> right = builder.stream("right-topic");

KStream<String, String> joined = left.join(
        right,
        (leftValue, rightValue) -> leftValue + "-" + rightValue, // joiner
        JoinWindows.ofTimeDifferenceWithNoGrace(Duration.ofMinutes(5))
);
```

**Example Input:**

`left-topic`:
- ("user1", "L1") at t=10:00

`right-topic`:
- ("user1", "R1") at t=10:03

**Output (joined):**
- ("user1", "L1-R1")  // within 5-minute window

### KStream–KTable Join

```java
KStream<String, String> events = builder.stream("events-topic");
KTable<String, String> userProfiles = builder.table("user-profile-topic");

KStream<String, String> enriched = events.join(
        userProfiles,
        (eventValue, profileValue) -> eventValue + " for " + profileValue
);
```

**Example Input:**

`user-profile-topic` (KTable):
- ("user1", "Alice")

`events-topic`:
- ("user1", "Login")

**Output:**
- ("user1", "Login for Alice")

---

## 6. Windowed Operations

**Purpose:** Aggregate or join data over time windows (e.g., per 1 minute, per hour).

Common window types:
- `Tumbling`: fixed, non-overlapping windows  
- `Hopping`: fixed windows that can overlap  
- `Session`: dynamic windows based on inactivity gaps

### Example: Count events per user per 1-minute window

```java
KStream<String, String> input = builder.stream("input-topic");

TimeWindows windows = TimeWindows.ofSizeWithNoGrace(Duration.ofMinutes(1));

KTable<Windowed<String>, Long> countsPerMinute = input
        .groupByKey()
        .windowedBy(windows)
        .count(Materialized.as("per-minute-counts"));
```

**Example Input (user1 events):**
- t=10:00:10 → ("user1", "e1")
- t=10:00:30 → ("user1", "e2")
- t=10:01:05 → ("user1", "e3")

**Output windows:**
- Window [10:00:00–10:01:00), key user1 → count 2
- Window [10:01:00–10:02:00), key user1 → count 1

## Minimal Spring Boot Kafka Streams Configuration Example

```java
@Configuration
@EnableKafkaStreams
public class KafkaStreamsConfig {

    @Bean(name = KafkaStreamsDefaultConfiguration.DEFAULT_STREAMS_CONFIG_BEAN_NAME)
    public KafkaStreamsConfiguration kStreamsConfigs() {
        Map<String, Object> props = new HashMap<>();
        props.put(StreamsConfig.APPLICATION_ID_CONFIG, "demo-app");
        props.put(StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(StreamsConfig.DEFAULT_KEY_SERDE_CLASS_CONFIG, Serdes.String().getClass());
        props.put(StreamsConfig.DEFAULT_VALUE_SERDE_CLASS_CONFIG, Serdes.String().getClass());
        return new KafkaStreamsConfiguration(props);
    }

    @Bean
    public KStream<String, String> kStream(StreamsBuilder builder) {
        KStream<String, String> input = builder.stream("input-topic");

        // Example pipeline: filter -> mapValues -> groupByKey -> count
        KTable<String, Long> counts = input
                .filter((k, v) -> v != null && v.length() > 3)
                .mapValues(String::toUpperCase)
                .groupByKey()
                .count(Materialized.as("counts-store"));

        counts.toStream().to("output-topic", Produced.with(Serdes.String(), Serdes.Long()));
        return input;
    }
}
```

**Example Input (`input-topic`):**
- ("user1", "hi")
- ("user1", "hello")
- ("user2", "world")

**Processing:**
- filter: remove "hi"
- mapValues: "hello" → "HELLO", "world" → "WORLD"
- groupByKey + count

**Output (`output-topic`):**
- ("user1", 1)
- ("user2", 1)

---

These are the core Kafka Streams operations you’ll use most often: transform (`map`), filter, group, aggregate, join, and apply time windows for time-based analytics.
