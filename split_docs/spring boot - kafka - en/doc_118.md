
Here’s a concise summary of the Kafka Streams basics: **KStream**, **KTable**, and **GlobalKTable**, with simple Spring Boot examples.

---

## 1. KStream

**Concept**

- A **KStream** is a stream of **records/events** (append-only log).
- Each record is processed **independently**.
- Good for event-driven processing, transformations, filtering, branching, etc.

**Typical use cases**

- Real-time processing of events (clicks, orders, logs).
- Stateless or stateful transformations (map, filter, join, windowed aggregations).

**Spring Boot example (Kafka Streams)**

```java
import org.apache.kafka.streams.kstream.KStream;
import org.apache.kafka.streams.kstream.Consumed;
import org.apache.kafka.streams.kstream.Produced;
import org.apache.kafka.streams.StreamsBuilder;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class KStreamConfig {

    @Bean
    public KStream<String, String> kStream(StreamsBuilder builder) {
        KStream<String, String> inputStream = builder.stream(
                "input-topic",
                Consumed.with(org.apache.kafka.common.serialization.Serdes.String(),
                              org.apache.kafka.common.serialization.Serdes.String())
        );

        // Example: filter and transform
        KStream<String, String> processed = inputStream
                .filter((key, value) -> value != null && value.contains("important"))
                .mapValues(value -> value.toUpperCase());

        processed.to(
                "output-topic",
                Produced.with(org.apache.kafka.common.serialization.Serdes.String(),
                              org.apache.kafka.common.serialization.Serdes.String())
        );

        return inputStream;
    }
}
```

**Example I/O**

- Input topic `input-topic`:
  - `("user1", "this is important")`
  - `("user2", "ignore this")`

- Output topic `output-topic`:
  - `("user1", "THIS IS IMPORTANT")`  
  (`user2` record is filtered out)

## 2. KTable

**Concept**

- A **KTable** is a **view of the latest state** per key (like a changelog-backed table).
- Each key has **one current value**; new records with the same key **overwrite** the previous value.
- Good for representing entities, configurations, aggregates.

**Typical use cases**

- Maintaining current account balance, user profile, product info.
- Aggregations: counts, sums, etc. (results are KTables).
- Joining streams with state (e.g., orders stream with customer table).

**Spring Boot example: stream → table aggregation**

```java
import org.apache.kafka.streams.kstream.KStream;
import org.apache.kafka.streams.kstream.KTable;
import org.apache.kafka.streams.kstream.Materialized;
import org.apache.kafka.streams.kstream.Consumed;
import org.apache.kafka.streams.kstream.Produced;
import org.apache.kafka.streams.StreamsBuilder;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class KTableConfig {

    @Bean
    public KTable<String, Long> wordCountTable(StreamsBuilder builder) {
        KStream<String, String> textLines = builder.stream(
                "text-input",
                Consumed.with(org.apache.kafka.common.serialization.Serdes.String(),
                              org.apache.kafka.common.serialization.Serdes.String())
        );

        KTable<String, Long> wordCounts = textLines
                .flatMapValues(value -> java.util.Arrays.asList(value.toLowerCase().split("\\W+")))
                .groupBy((key, word) -> word)
                .count(Materialized.as("word-counts-store"));

        // Publish the current counts as a changelog stream
        wordCounts.toStream().to(
                "word-count-output",
                Produced.with(org.apache.kafka.common.serialization.Serdes.String(),
                              org.apache.kafka.common.serialization.Serdes.Long())
        );

        return wordCounts;
    }
}
```

**Example I/O**

- Input topic `text-input`:
  - `("k1", "Hello Kafka")`
  - `("k2", "Hello Streams")`

- Internal KTable state (word → count):
  - After first record: `hello=1`, `kafka=1`
  - After second record: `hello=2`, `kafka=1`, `streams=1`

- Output topic `word-count-output` (changelog):
  - `("hello", 1)`
  - `("kafka", 1)`
  - `("hello", 2)`
  - `("streams", 1)`

## 3. GlobalKTable

**Concept**

- A **GlobalKTable** is a KTable that is **fully replicated** on **every** Kafka Streams instance.
- Every application instance has a **complete copy** of the table.
- Good for **small to medium reference data** that you want to join with any key (not just partition-aligned).

**Typical use cases**

- Reference/lookup data: product catalog, user metadata, configuration.
- Joining a KStream with a table where the join key is not the same as the stream’s partition key.

**Spring Boot example: KStream join with GlobalKTable**

```java
import org.apache.kafka.streams.kstream.*;
import org.apache.kafka.streams.StreamsBuilder;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class GlobalKTableConfig {

    @Bean
    public KStream<String, String> joinWithGlobalTable(StreamsBuilder builder) {
        // Orders stream: key = orderId, value = customerId
        KStream<String, String> orders = builder.stream("orders");

        // Customers table: key = customerId, value = customerName
        GlobalKTable<String, String> customers = builder.globalTable("customers");

        KStream<String, String> enrichedOrders = orders.join(
                customers,
                // Map stream record to the key used in GlobalKTable
                (orderId, customerId) -> customerId,
                // Join function: (orderValue, customerValue) -> result
                (customerId, customerName) -> "customer=" + customerName + ", orderId=" + customerId
        );

        enrichedOrders.to("enriched-orders");

        return enrichedOrders;
    }
}
```

**Example I/O**

- Topic `customers` (GlobalKTable):
  - `("c1", "Alice")`
  - `("c2", "Bob")`

- Topic `orders` (KStream):
  - `("o100", "c1")`
  - `("o101", "c2")`

- Output topic `enriched-orders`:
  - `("o100", "customer=Alice, orderId=c1")`
  - `("o101", "customer=Bob, orderId=c2")`

## Quick Comparison

- **KStream**
  - Model: event stream
  - Semantics: every record is an event
  - Use for: event processing, transformations, windowed aggregations

- **KTable**
  - Model: changelog of a table (latest value per key)
  - Semantics: current state per key
  - Use for: stateful aggregates, entity state, stream–table joins

- **GlobalKTable**
  - Model: fully replicated table on each instance
  - Semantics: same as KTable, but globally available
  - Use for: reference/lookup data, non-partition-aligned joins
