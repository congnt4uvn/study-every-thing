
Here’s a concise summary of the three Kafka consumer configs and how they affect behavior, with Spring Boot / Java examples.

---

## 1. `fetch.min.bytes`

**What it does:**  
Minimum amount of data (in bytes) the broker should collect before returning a fetch response to the consumer.

- **Low value (e.g., 1):**  
  - Consumer gets messages quickly (low latency)  
  - More network calls (less efficient)
- **High value (e.g., 1 MB):**  
  - Consumer waits until enough data is available (higher latency)  
  - Fewer, larger fetches (more efficient throughput)

**Example:**

```properties
# application.properties
spring.kafka.consumer.properties.fetch.min.bytes=1048576  # 1 MB
```

**Effect (conceptual):**

- Input: Broker has only 100 KB of data available.  
- Config: `fetch.min.bytes=1MB`.  
- Output: Broker waits (up to `fetch.max.wait.ms`) before responding, hoping to accumulate 1 MB.

---

## 2. `fetch.max.wait.ms`

**What it does:**  
Maximum time (in milliseconds) the broker will wait before sending data, even if `fetch.min.bytes` is not reached.

- Works together with `fetch.min.bytes`:
  - Broker returns data when:
    - It has at least `fetch.min.bytes` **OR**
    - It has waited `fetch.max.wait.ms`, whichever comes first.

**Example:**

```properties
spring.kafka.consumer.properties.fetch.max.wait.ms=500  # 0.5 seconds
```

**Effect (conceptual):**

- Input: `fetch.min.bytes=1MB`, but only 200 KB arrives in 500 ms.  
- Output: Broker sends 200 KB after 500 ms instead of waiting indefinitely.

---

## 3. `max.partition.fetch.bytes`

**What it does:**  
Maximum amount of data (in bytes) the consumer will fetch from **one partition** in a single request.

- Protects consumer from:
  - Very large batches from a single partition
  - Excessive memory usage
- If a single message is larger than this value, the consumer **cannot** consume it.

**Example:**

```properties
spring.kafka.consumer.properties.max.partition.fetch.bytes=1048576  # 1 MB per partition
```

**Effect (conceptual):**

- Input: Partition has 5 MB of data.  
- Config: `max.partition.fetch.bytes=1MB`.  
- Output: Consumer fetches up to 1 MB from that partition per request (multiple fetches needed to read all data).

---

## Combined Example (Spring Boot)

```properties
# application.properties
spring.kafka.bootstrap-servers=localhost:9092
spring.kafka.consumer.group-id=my-group
spring.kafka.consumer.auto-offset-reset=earliest

# Tuning fetch behavior
spring.kafka.consumer.properties.fetch.min.bytes=1048576          # 1 MB
spring.kafka.consumer.properties.fetch.max.wait.ms=500            # 0.5 s
spring.kafka.consumer.properties.max.partition.fetch.bytes=2097152 # 2 MB per partition
```

### Simple Consumer Example (Java)

```java
import org.apache.kafka.clients.consumer.ConsumerConfig;
import org.apache.kafka.clients.consumer.KafkaConsumer;
import org.apache.kafka.common.serialization.StringDeserializer;

import java.time.Duration;
import java.util.Collections;
import java.util.Properties;

public class SimpleConsumer {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "my-group");
        props.put(ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class.getName());
        props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, StringDeserializer.class.getName());

        // Fetch tuning
        props.put(ConsumerConfig.FETCH_MIN_BYTES_CONFIG, 1048576);          // 1 MB
        props.put(ConsumerConfig.FETCH_MAX_WAIT_MS_CONFIG, 500);           // 0.5 s
        props.put(ConsumerConfig.MAX_PARTITION_FETCH_BYTES_CONFIG, 2097152); // 2 MB

        try (KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props)) {
            consumer.subscribe(Collections.singletonList("my-topic"));

            while (true) {
                var records = consumer.poll(Duration.ofMillis(1000));
                System.out.println("Fetched " + records.count() + " records");
                records.forEach(record ->
                    System.out.printf("partition=%d, offset=%d, key=%s, value=%s%n",
                            record.partition(), record.offset(), record.key(), record.value())
                );
            }
        }
    }
}
```

---

### Summary

- `fetch.min.bytes`: Controls **minimum batch size** per fetch → latency vs throughput.
- `fetch.max.wait.ms`: Controls **maximum wait time** to reach `fetch.min.bytes`.
- `max.partition.fetch.bytes`: Limits **max data per partition per fetch** → memory protection and batch size per partition.
