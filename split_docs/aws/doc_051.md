


Dưới đây là tóm tắt nội dung:

- EC2 có thể gắn 2 loại “ổ đĩa” chính:
  - Ổ mạng (EBS, EFS…) – hiệu suất tốt nhưng vẫn có giới hạn.
  - Ổ gắn trực tiếp vào máy chủ vật lý – gọi là **EC2 Instance Store**.

- **EC2 Instance Store**:
  - Là ổ đĩa vật lý gắn trực tiếp vào máy chủ chạy EC2 Instance.
  - Cho hiệu suất I/O và thông lượng rất cao, phù hợp khi cần **hiệu năng đĩa cực lớn**.
  - Tuy nhiên là **kho lưu trữ tạm thời**:
    - Dừng hoặc terminate instance → dữ liệu trên Instance Store **mất hoàn toàn**.
    - Nếu phần cứng máy chủ bị lỗi → dữ liệu trên Instance Store cũng mất.
  - Vì vậy **không dùng để lưu trữ lâu dài/bền vững**.
  - Phù hợp cho:
    - Cache, bộ nhớ đệm.
    - Dữ liệu tạm thời, dữ liệu có thể tái tạo.

- **So sánh hiệu năng**:
  - Một số loại instance như I3 có Instance Store với IOPS rất cao:
    - Đọc ngẫu nhiên: tới ~3,3 triệu IOPS.
    - Ghi ngẫu nhiên: tới ~1,4 triệu IOPS.
  - Trong khi đó EBS gp2/gp3 chỉ khoảng vài chục nghìn IOPS (ví dụ ~32.000 IOPS).

- **Lưu ý cho kỳ thi**:
  - Khi đề bài nói đến ổ đĩa hiệu năng cực cao, gắn trực tiếp với phần cứng của EC2 → nghĩ ngay đến **EC2 Instance Store**.
  - Nếu cần lưu trữ lâu dài, bền vững → dùng **EBS**.
  - Khi dùng Instance Store, bạn phải tự chịu trách nhiệm backup/replication dữ liệu.
