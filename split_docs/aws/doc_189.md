


Người kể chuyện giải thích về **Elastic Beanstalk Extensions (.ebextensions)**:

1. **Khái niệm & mục đích**  
   - Khi tạo file zip để deploy lên Elastic Beanstalk, ngoài mã nguồn, ta có thể thêm các file cấu hình gọi là **EB extensions**.  
   - Những gì thường cấu hình trong giao diện (UI) của Elastic Beanstalk có thể được cấu hình bằng code thông qua các file này.

2. **Cấu trúc & định dạng file**  
   - Các file cấu hình phải nằm trong thư mục **`.ebextensions/`** ở **thư mục gốc** của mã nguồn.  
   - Nội dung file có thể ở dạng **YAML hoặc JSON**.  
   - Dù dùng YAML/JSON, **đuôi file phải kết thúc bằng `.config`** (ví dụ: `logging.config`, `environment.config`).

3. **Chức năng chính**  
   - Có thể **sửa đổi các giá trị mặc định** bằng cách dùng phần **`option_settings`** (sẽ được giải thích chi tiết hơn sau).  
   - Có thể **thêm tài nguyên AWS** thông qua EB extensions, ví dụ:  
     - RDS  
     - ElastiCache  
     - DynamoDB  
     - Và các tài nguyên khác mà bình thường phải tạo qua console.

4. **Lưu ý quan trọng về vòng đời tài nguyên**  
   - Mọi tài nguyên được tạo và quản lý bởi EB extensions **sẽ bị xóa khi môi trường Elastic Beanstalk bị xóa**.  
   - Ví dụ: nếu ElastiCache được tạo như một phần của môi trường EB, khi xóa môi trường EB thì ElastiCache đó cũng bị xóa.

5. **Ví dụ thực hành**  
   - Người kể chuyện tạo thư mục `nodejs-v3-ebextensions`, bên trong có thư mục **`.ebextensions`**.  
   - Trong đó tạo file **`environment.config`** (đuôi `.config`, nằm trong `.ebextensions/`, đúng yêu cầu).  
   - File được viết ở định dạng **YAML**, sử dụng **`option_settings`** để cấu hình **biến môi trường** cho ứng dụng Elastic Beanstalk, ví dụ:  
     - `DB_URL`  
     - `DB_USER`  
   - Đây là ví dụ cho trường hợp muốn kết nối tới một RDS bên ngoài (ví dụ Postgres RDS), ta sẽ đặt các biến môi trường này trong file `.ebextensions`.

6. **Triển khai & kiểm tra**  
   - Nén mã nguồn cùng thư mục `.ebextensions` thành file zip `nodejs-v3-ebextensions.zip`.  
   - Upload và deploy phiên bản này lên môi trường Elastic Beanstalk (môi trường dev).  
   - Sau khi deploy xong, vào phần **Configuration → Environment properties** để kiểm tra:  
     - Thấy `DB_URL` và `DB_USER` đã được thiết lập.  
     - Các giá trị này không được đặt bằng tay trong console, mà đến từ file `.ebextensions`.

7. **Kết luận**  
   - EB extensions cho phép quản lý cấu hình và tài nguyên bằng code (infrastructure/config as code).  
   - Bài giảng mới chỉ giới thiệu phần cơ bản, nhưng như vậy là đủ cho mục đích ôn thi.
