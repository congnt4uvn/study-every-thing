


Nội dung trên giải thích về tính năng **Connection Draining / Deregistration Delay** trong AWS Elastic Load Balancing:

- Tên gọi:
  - Với **Classic Load Balancer**: gọi là **Connection Draining** (Kết nối thoát).
  - Với **Application Load Balancer (ALB)** và **Network Load Balancer (NLB)**: gọi là **Deregistration Delay** (Độ trễ hủy đăng ký).

- Mục đích:
  - Khi một instance EC2 bị **deregister** (gỡ khỏi load balancer) hoặc bị đánh dấu **unhealthy**, tính năng này cho phép:
    - Các **kết nối / request đang xử lý** trên instance đó được hoàn thành.
    - Trong thời gian đó, **ELB không gửi thêm request mới** đến instance đang bị rút.

- Cách hoạt động:
  - Ví dụ có 3 EC2 instance, một instance được đặt ở trạng thái “draining”.
  - Người dùng đã kết nối tới instance đó sẽ có **một khoảng thời gian** (draining timeout) để hoàn tất request.
  - Sau khi tất cả request hiện tại hoàn thành, **mọi kết nối tới instance đó sẽ bị ngắt**.
  - Người dùng mới gửi request đến ELB sẽ được phân phối sang **các instance khác** (không phải instance đang draining).

- Cấu hình:
  - Thời gian Connection Draining / Deregistration Delay có thể đặt từ **1 đến 3600 giây**.
  - **Mặc định**: 300 giây (5 phút).
  - Đặt **0** nghĩa là **tắt** tính năng này (không có giai đoạn draining).

- Cách chọn giá trị:
  - **Request rất ngắn** (ví dụ < 1 giây): có thể đặt timeout thấp, như **30 giây**, để instance rút khỏi hệ thống nhanh hơn.
  - **Request dài** (upload, xử lý lâu, long-lived connections): nên đặt timeout **cao hơn** để tránh cắt ngang request.
  - Đánh đổi: timeout càng cao thì instance **mất nhiều thời gian hơn** mới thực sự được rút khỏi load balancer.
