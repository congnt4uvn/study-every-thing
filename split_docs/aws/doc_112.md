


Dưới đây là tóm tắt ngắn gọn nội dung:

1. Lý do nói về VPC  
   - Giải thích VPC và subnet để dẫn tới kiến trúc 3 tầng (three-tier architecture) trên AWS.  
   - Giúp hiểu rõ cách chia mạng con công cộng – riêng tư trước khi vào kiến trúc thực tế.

2. Kiến trúc 3 tầng điển hình  
   - Tầng 1 (Presentation):  
     - Người dùng truy cập ứng dụng web qua Internet.  
     - Dùng Elastic Load Balancer (ELB) đặt trong các public subnet, trải trên nhiều AZ.  
     - Người dùng truy cập ELB thông qua DNS (Route 53).  
   - Tầng 2 (Application):  
     - Các EC2 instance nằm trong Auto Scaling Group.  
     - Đặt trong private subnet (không truy cập trực tiếp từ Internet, chỉ từ ELB).  
     - ELB chuyển traffic từ public subnet vào private subnet thông qua route table.  
     - Lớp compute được tách biệt trong private subnet → an toàn hơn.  
   - Tầng 3 (Data):  
     - Một lớp private subnet khác, thường gọi là data subnet.  
     - Chứa Amazon RDS (database) để đọc/ghi dữ liệu.  
     - EC2 kết nối tới RDS.  
     - Có thể thêm ElastiCache để cache dữ liệu từ RDS hoặc lưu/truy xuất session data của ứng dụng web.

3. LAMP Stack trên EC2  
   - LAMP = Linux + Apache + MySQL + PHP:  
     - Linux: Hệ điều hành cho EC2.  
     - Apache: Web server chạy trên EC2.  
     - MySQL: Database (có thể dùng MySQL trên RDS).  
     - PHP: Logic ứng dụng, chạy trên EC2.  
   - Có thể thêm Redis/Memcached (ElastiCache) để caching.  
   - Dùng EBS gắn vào EC2 để lưu trữ dữ liệu cục bộ.

4. Kiến trúc WordPress trên AWS (phiên bản đơn giản)  
   - Dùng kiến trúc tương tự:  
     - Tầng load balancer + tầng application (EC2).  
   - Người dùng upload hình ảnh qua ELB tới EC2.  
   - Các EC2 cần chia sẻ file (hình ảnh) với nhau → dùng EFS:  
     - EFS là network file system, tạo mount point ở mỗi AZ.  
     - Tất cả EC2 có thể đọc/ghi chung dữ liệu trên EFS.  
   - Đây là sơ đồ đơn giản, chưa vẽ phần database.  
   - Trên trang AWS có kiến trúc WordPress đầy đủ (có NAT Gateway, Internet Gateway, Auto Scaling, nhiều subnet, Aurora, EFS, v.v.).  
   - Phần duy nhất người học chưa được học là CloudFront và S3 (sẽ học sau).

5. Kết luận  
   - Người học hiện đã nắm được khá nhiều về kiến trúc giải pháp trên AWS (VPC, subnet, ELB, ASG, RDS, ElastiCache, EFS…).  
   - Những kiến thức này rất hữu ích cho cả thực tế phát triển và kỳ thi.  
   - Giảng viên khuyến khích dành thời gian tự xem lại sơ đồ kiến trúc để hiểu sâu hơn.
