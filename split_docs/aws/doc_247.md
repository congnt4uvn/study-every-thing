


Dưới đây là tóm tắt nội dung:

1. Tổng quan Amazon EventBridge  
   - Có nhiều thành phần: Rules (quy tắc), Schedules (lịch), Pipes, Schema Registry (kho lược đồ), event bus mặc định và tùy chỉnh, nguồn sự kiện đối tác, API destinations.

2. Quy tắc EventBridge với Event Pattern  
   - Tạo một rule để phản ứng với sự kiện “EC2 Instance State-change Notification”.  
   - Xem cấu trúc và ví dụ sự kiện trong schema.  
   - Lọc sự kiện theo trường `state` (ví dụ: “stopping”, “terminated”) dựa trên các ví dụ trong schema.  
   - Chọn target là SNS Topic để nhận thông báo khi EC2 bị tắt hoặc chấm dứt.  
   - EventBridge tự tạo IAM role để gửi message vào SNS.  
   - Sau khi tạo rule, mỗi lần EC2 tắt/ngừng, sẽ có thông báo gửi qua SNS.

3. Lịch trình EventBridge (EventBridge Scheduler)  
   - Tạo schedule riêng (không dùng event pattern).  
   - Ví dụ: schedule tên “InvokeLambdaEveryHour”.  
   - Chọn kiểu: one-time hoặc recurring (lặp lại).  
   - Dùng cron hoặc rate (ví dụ: mỗi 1 giờ).  
   - Chọn hành động: chạy ECS task, gửi vào Kinesis Firehose, hoặc invoke Lambda function.  
   - Sau khi cấu hình, EventBridge sẽ tự động thực thi theo lịch.

4. Event Bus (kênh sự kiện)  
   - Event bus mặc định: nhận tất cả sự kiện do AWS phát ra.  
   - Có thể tạo event bus tùy chỉnh để gửi các sự kiện riêng của ứng dụng.  
   - Có thể lưu trữ, xem lại các sự kiện đã xảy ra.

5. Nguồn sự kiện đối tác (Partner Event Sources)  
   - Nhận sự kiện từ bên thứ ba (ví dụ Auth0) trực tiếp vào EventBridge.  
   - Từ đó có thể kích hoạt Lambda hoặc các tác vụ khác sau khi sự kiện xảy ra (như user login thành công).  
   - Có nhiều loại đối tác khác nhau được hỗ trợ.

6. API Destinations  
   - Cho phép EventBridge gửi sự kiện ra các endpoint HTTP bên ngoài.  
   - Hữu ích để tích hợp với hạ tầng hoặc hệ thống ngoài AWS.

7. Schema Registry (Kho lược đồ)  
   - Lưu trữ lược đồ của các sự kiện AWS và lược đồ tùy chỉnh.  
   - Giúp hiểu rõ cấu trúc, trường dữ liệu của từng loại sự kiện.

8. Quản lý quy tắc  
   - Có thể bật/tắt các rule trong phần Rules của EventBridge khi cần.

Toàn bộ nội dung tập trung vào cách dùng EventBridge để:
- Bắt sự kiện (EC2 state change) và gửi thông báo qua SNS.  
- Lập lịch thực thi (ví dụ gọi Lambda mỗi giờ).  
- Tích hợp với hệ thống nội bộ và bên thứ ba thông qua event bus, partner sources, API destinations và schema registry.
