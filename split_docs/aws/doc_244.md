


Dưới đây là tóm tắt nội dung:

- Giảng viên vào CloudWatch Alarms để tạo một báo thức (alarm) cho EC2.
- Tạo nhanh một EC2 instance loại t2, chỉ cần nó chạy, không cần cấu hình thêm.
- Mục tiêu: tạo CloudWatch Alarm giám sát CPU, nếu CPU lên 95–100% trong một khoảng thời gian thì tự động chấm dứt (terminate) instance.
- Trong CloudWatch:
  - Chọn metric cho EC2 instance vừa tạo (dựa trên Instance ID).
  - Chờ khoảng 5 phút để metric CPU xuất hiện (do không bật detailed monitoring).
  - Chọn metric CPUUtilization, chọn cách tính (Average/Max/…).
  - Đặt period 5 phút, điều kiện ngưỡng: ví dụ CPU > 95% trong 3 trên 3 khoảng (tức ~15 phút).
- Phần hành động (Actions):
  - Chọn loại hành động là EC2 action.
  - Cấu hình: khi alarm ở trạng thái ALARM thì terminate instance.
  - Đặt tên alarm, ví dụ “Terminate EC2 on high CPU”, rồi tạo alarm.
- Lúc mới tạo, alarm ở trạng thái “Insufficient data” vì chưa đủ dữ liệu.
- Để thử nhanh mà không phải làm CPU cao 15 phút, sử dụng API/CLI:
  - Lệnh `aws cloudwatch set-alarm-state` với:
    - `--alarm-name` là tên alarm,
    - `--state-value` là `ALARM`,
    - `--state-reason` là lý do (ví dụ “test”).
- Sau khi chạy lệnh:
  - Trên console CloudWatch, alarm chuyển sang trạng thái ALARM.
  - Trong phần History, thấy log: từ OK sang ALARM, và hành động terminate EC2 được thực thi thành công.
  - Trên EC2 console, instance chuyển sang trạng thái shutting down/terminated.
- Kết luận: minh họa cách tạo CloudWatch Alarm cho CPU EC2 và cấu hình để tự động chấm dứt instance khi CPU cao kéo dài, đồng thời dùng API để mô phỏng trạng thái ALARM và kiểm tra hành động.
