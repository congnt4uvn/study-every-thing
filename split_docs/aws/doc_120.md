


Dưới đây là tóm tắt nội dung:

- Bật versioning (lập phiên bản) cho S3 bucket trong phần Properties → Bucket Versioning → Enable.  
- Sau khi bật versioning, mọi lần upload ghi đè cùng một file sẽ không xóa file cũ mà tạo thêm một phiên bản mới (version) của đối tượng.

Ví dụ với file `index.html`:
- Ban đầu nội dung là “I love coffee”.
- Sửa file thành “I REALLY love coffee” rồi upload lại lên cùng key `index.html`.
- Nhờ versioning, S3 tạo thêm một version mới cho `index.html`.
- Khi bật “Show versions”, ta thấy:
  - Version cũ (trước khi bật versioning) có Version ID rỗng.
  - Version mới (sau khi bật versioning) có một Version ID cụ thể.

Khôi phục nội dung cũ:
- Để quay lại nội dung “I love coffee”, ta chọn version mới (với Version ID) và xóa phiên bản đó.
- Đây là “permanent delete” (xóa vĩnh viễn) một version cụ thể, không thể hoàn tác.
- Sau khi xóa version mới, version cũ trở thành phiên bản hiện tại, trang web hiển thị lại “I love coffee”.

Xóa đối tượng khi tắt “Show versions”:
- Khi tắt “Show versions” và xóa một file (ví dụ `coffee.jpg`), S3 không xóa hẳn dữ liệu mà:
  - Thêm một “delete marker” (điểm đánh dấu xóa) cho key đó.
  - Từ góc nhìn bình thường, đối tượng coi như đã bị xóa (không còn thấy trong danh sách, truy cập URL báo 404).
- Khi bật lại “Show versions”, ta thấy:
  - Các phiên bản cũ của `coffee.jpg` vẫn tồn tại.
  - Có thêm một delete marker là phiên bản mới nhất.

Khôi phục đối tượng đã “xóa”:
- Để khôi phục `coffee.jpg`, ta xóa vĩnh viễn delete marker.
- Khi delete marker bị xóa, phiên bản trước đó của `coffee.jpg` lại trở thành phiên bản hiện tại.
- Reload trang web, hình ảnh `coffee.jpg` xuất hiện trở lại.

Ý chính:
- Versioning cho phép:
  - Lưu nhiều phiên bản của cùng một file.
  - Khôi phục lại phiên bản cũ bằng cách xóa phiên bản mới hơn.
  - “Xóa” file ở chế độ bình thường chỉ thêm delete marker, có thể khôi phục bằng cách xóa delete marker đó.
