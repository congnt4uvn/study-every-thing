


Dưới đây là tóm tắt nội dung về CloudFormation DeletionPolicy:

1. Khái niệm chung  
- DeletionPolicy là thuộc tính gán cho từng resource trong template CloudFormation.  
- Nó quyết định điều gì xảy ra với resource khi:
  - Resource bị xóa khỏi template, hoặc  
  - Cả stack CloudFormation bị xóa.  
- Mặc định: DeletionPolicy = Delete → khi xóa stack thì resource cũng bị xóa.

2. DeletionPolicy = Delete  
- Ví dụ EC2 Instance: nếu đặt DeletionPolicy: Delete (hoặc không đặt gì), khi xóa stack thì EC2 instance bị xóa.  
- Với S3 Bucket:
  - Nếu DeletionPolicy = Delete, việc xóa chỉ thành công nếu bucket trống.  
  - Nếu bucket còn dữ liệu, việc xóa stack sẽ thất bại.  
  - Cách xử lý:
    - Xóa thủ công toàn bộ object trong bucket rồi xóa stack, hoặc  
    - Dùng custom resource để tự động xóa dữ liệu trong bucket trước khi xóa bucket.

3. DeletionPolicy = Retain  
- Dùng khi muốn giữ lại resource ngay cả khi stack bị xóa.  
- Ví dụ DynamoDB Table:
  - Mặc định sẽ bị xóa cùng stack.  
  - Nếu đặt DeletionPolicy: Retain, khi xóa stack, bảng và dữ liệu vẫn tồn tại.  
- Áp dụng được cho mọi loại resource.  
- Ví dụ Security Group với DeletionPolicy: Retain:
  - Khi xóa stack, Security Group không bị xóa, phải xóa thủ công nếu không cần nữa.

4. DeletionPolicy = Snapshot  
- Tạo một snapshot cuối cùng trước khi xóa resource.  
- Hỗ trợ cho:
  - EBS Volume  
  - ElastiCache Cluster / ReplicationGroup  
  - RDS DBInstance, DBCluster  
  - Redshift, Neptune, DocumentDB, …  
- Cách hoạt động:
  - Resource (ví dụ RDS instance, EBS volume) bị xóa.  
  - Trước đó CloudFormation tạo một snapshot cuối cùng để backup.  

5. Ví dụ demo trong bài  
- Template có:
  - Một Security Group với DeletionPolicy: Retain.  
  - Một EBS Volume với DeletionPolicy: Snapshot.  
- Khi tạo stack:
  - EBS volume và Security Group được tạo.  
- Khi xóa stack:
  - Security Group: thao tác delete bị bỏ qua (do Retain), SG vẫn tồn tại → phải xóa thủ công nếu muốn.  
  - EBS Volume: bị xóa, nhưng trước đó CloudFormation tạo một snapshot mới (thấy được trong danh sách snapshots).  
  - Nếu muốn dọn sạch hoàn toàn, cần xóa snapshot và Security Group thủ công.

Ý chính: DeletionPolicy cho phép bạn kiểm soát việc xóa resource (xóa hẳn, giữ lại, hay chụp snapshot trước khi xóa) để bảo vệ dữ liệu và backup khi làm việc với CloudFormation.
