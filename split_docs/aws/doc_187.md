


Nội dung trên nói về:

1. EB CLI là gì  
- Ngoài AWS CLI thông thường, còn có một công cụ riêng cho Elastic Beanstalk gọi là EB CLI (Elastic Beanstalk CLI).  
- EB CLI giúp thao tác với Elastic Beanstalk qua dòng lệnh dễ dàng hơn, hỗ trợ các lệnh như: `eb create`, `eb status`, `eb health`, `eb events`, `eb logs`, `eb open`, `eb deploy`, `eb config`, `eb terminate`, v.v.  
- Các lệnh này cho phép bạn làm hầu hết những gì đã làm trên giao diện console của Elastic Beanstalk nhưng bằng CLI.

2. Khi nào nên dùng EB CLI  
- EB CLI rất hữu ích khi bạn muốn tự động hóa pipeline phát triển và triển khai.  
- Không bắt buộc phải biết EB CLI cho kỳ thi Developer, nhưng quan trọng hơn cho kỳ thi DevOps.  
- Trong khóa học hiện tại (và kỳ thi Developer), giảng viên không đi sâu vào EB CLI, chỉ giới thiệu để bạn biết nó tồn tại.

3. Quy trình triển khai ứng dụng với Elastic Beanstalk  
- Để triển khai một ứng dụng lên Beanstalk, bạn phải mô tả các dependency:  
  - Python: dùng `requirements.txt`  
  - Node.js: dùng `package.json`  
- Sau đó:  
  1. Đóng gói toàn bộ mã nguồn (kèm file mô tả dependency) thành một file `.zip`.  
  2. Upload file `.zip` đó lên Elastic Beanstalk.  
  3. Việc upload này tạo ra một “application version” mới.  
  4. Từ application version đó, bạn có thể triển khai (deploy) qua console hoặc qua CLI/EB CLI.

4. Cách Beanstalk xử lý phía backend  
- Khi bạn upload file `.zip`, thực chất nó được lưu vào Amazon S3.  
- Elastic Beanstalk tham chiếu đến file trong S3 như một “application bundle”.  
- Khi triển khai, Beanstalk lấy bundle này, phân phối xuống các EC2 instance trong môi trường.  
- Trên mỗi EC2 instance, Beanstalk:  
  - Cài đặt các dependency dựa trên `requirements.txt` (Python) hoặc `package.json` (Node.js).  
  - Khởi động ứng dụng của bạn.

5. Phạm vi khóa học  
- Việc cài đặt và sử dụng EB CLI chi tiết nằm ngoài phạm vi khóa học và kỳ thi Developer mà giảng viên đang chuẩn bị cho bạn.  
- Giảng viên chỉ muốn bạn biết EB CLI tồn tại và có thể tham khảo thêm trong tài liệu nếu muốn.
