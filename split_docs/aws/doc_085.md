


Dưới đây là tóm tắt nội dung:

- Hướng dẫn thực hành tạo một cụm Amazon ElastiCache.
- Lựa chọn engine:
  - Có thể dùng Valkey (thay thế Redis), Redis OSS hoặc Memcached.
  - Trong ví dụ chọn Redis.
- Lựa chọn kiểu triển khai:
  - Có serverless và node-based cluster.
  - Chọn node-based để dễ hiểu cách hoạt động, khôi phục từ backup và áp dụng best practice.
- Chế độ cấu hình:
  - Có các cấu hình gợi ý: production, dev/test, demo hoặc tự cấu hình.
  - Chọn tự cấu hình để xem hết các tùy chọn.
  - Cluster mode bị tắt: chỉ 1 shard, 1 primary, tối đa 5 read replica.
  - Nếu cần nhiều shard trên nhiều server thì bật cluster mode.
- Đặt tên cụm là `DemoCluster`.
- Vị trí:
  - Chạy trên AWS Cloud, có thể chạy on-premises với AWS Outposts.
- Tùy chọn Multi-AZ:
  - Multi-AZ giúp tăng HA và failover.
  - Trong ví dụ tắt Multi-AZ để giảm chi phí.
  - Auto failover giữ ở trạng thái bật.
- Thiết lập cluster:
  - Chọn phiên bản engine, port, parameter group.
  - Chọn loại node: dùng instance loại micro (t2.micro, t3.micro, t4g.micro).
  - Ưu tiên t2/t3 vì nằm trong free tier.
  - Số read replica để 0 để tiết kiệm chi phí (nhiều replica hơn khi dùng Multi-AZ).
- Mạng:
  - Tạo subnet group `my-first-subnet-group` để chỉ định subnet nào ElastiCache được phép chạy.
  - Chọn VPC và subnet tương ứng.
  - Có thể chỉ định AZ cho replica, nhưng ví dụ này không dùng Multi-AZ nên không quan trọng.
- Bảo mật & mã hóa:
  - Tùy chọn mã hóa at-rest (khi lưu trữ) – nếu bật cần chỉ định KMS key.
  - Tùy chọn mã hóa in-transit (giữa client và server):
    - Nếu bật, có thêm kiểm soát truy cập.
    - Có thể dùng Redis AUTH (mật khẩu/token) hoặc User Group (Access Control List).
  - Trong ví dụ tắt mã hóa in-transit.
  - Có thể cấu hình Security Group để kiểm soát ứng dụng nào được truy cập cụm.
- Sao lưu & bảo trì:
  - Tùy chọn bật/tắt backup.
  - Cấu hình maintenance window để nâng cấp minor version.
  - Bật log (slow log, engine log) và gửi lên CloudWatch nếu muốn.
  - Gắn tag cho tài nguyên.
- Sau khi cấu hình:
  - Review cấu hình, nhấn Create để tạo cụm.
  - Khi cụm sẵn sàng, có thể dùng:
    - Primary endpoint để đọc/ghi.
    - Reader endpoint (nếu có replica) để đọc.
  - Việc kết nối thực tế cần viết code (client Redis), nên không demo chi tiết trong bài.
- Từ console có thể xem:
  - Thông tin cụm, node, metrics, logs, network security, v.v.
  - Giao diện và cách quản lý rất giống RDS nhưng dành cho Redis/Memcached.
- Xóa cụm:
  - Chọn cụm → Actions → Delete.
  - Chọn có/không giữ backup (ví dụ chọn không).
  - Nhập tên cụm để xác nhận, xóa là xong.

Bài giảng kết thúc với việc nhấn mạnh quy trình tạo – cấu hình – quan sát – xóa một cụm ElastiCache Redis.
