


Dưới đây là tóm tắt nội dung:

- Trong AWS Lambda có 2 khái niệm quan trọng: **event object (đối tượng sự kiện)** và **context object (đối tượng ngữ cảnh)**.  
- **Event object**:
  - Là dữ liệu đầu vào mà Lambda sẽ xử lý, ở dạng JSON.
  - Được tạo và gửi bởi dịch vụ gọi Lambda (EventBridge, SQS, SNS, …).
  - Chứa thông tin về sự kiện: nguồn, vùng (region), nội dung thông điệp, v.v.
  - Tùy runtime sẽ được chuyển thành kiểu tương ứng (ví dụ Python → dict).
- **Context object**:
  - Chứa metadata về lần gọi hàm và môi trường runtime.
  - Ví dụ: AWS request ID, tên hàm, ARN hàm, log group, log stream, giới hạn bộ nhớ (MB), v.v.
  - Được truyền cùng với event vào handler để bạn có thể dùng trong mã (logging, theo dõi, tối ưu, …).

- Trong Python handler thường có dạng: `def handler(event, context):`
  - `event`: dùng để lấy dữ liệu sự kiện cần xử lý.
  - `context`: dùng để lấy thông tin về lần gọi và môi trường.

- Kết luận:  
  - **Event** = dữ liệu nghiệp vụ mà hàm xử lý.  
  - **Context** = thông tin kỹ thuật/metadata về lần thực thi hàm.  
  - Cần phân biệt rõ để chọn đúng nơi lấy thông tin khi làm bài thi hoặc viết code.
