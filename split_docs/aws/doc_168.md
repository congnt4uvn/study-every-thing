


Người dẫn chuyện hướng dẫn quy trình triển khai một dịch vụ ECS trên AWS Fargate với ứng dụng demo nginx:

1. **Tạo Task Definition (Định nghĩa nhiệm vụ)**  
   - Vào phần Task Definitions, tạo task mới tên `nginxdemos-hello`.  
   - Sử dụng image trên Docker Hub: `nginxdemos/hello`.  
   - Chọn **Fargate** (serverless) làm loại compute, không dùng EC2 cho đơn giản.  
   - Hệ điều hành: Linux.  
   - Cấu hình tài nguyên Fargate: 0.5 vCPU và 1 GB RAM (có thể tăng tới 16 vCPU và 120 GB RAM).  
   - Task role (IAM role cho container gọi dịch vụ AWS): để trống vì demo không cần; nhưng nhấn mạnh đây là phần rất quan trọng nếu container cần truy cập AWS.  
   - Task execution role: dùng mặc định, ECS sẽ tự tạo nếu chưa có.  
   - Cấu hình container:  
     - Tên: `nginxdemos-hello`.  
     - Image: `nginxdemos/hello`.  
     - Port mapping: map cổng 80 host → 80 container.  
     - Các tùy chọn khác (resource limits, env vars, logging) giữ mặc định.  
   - Storage: dùng bộ nhớ tạm thời mặc định của Fargate (khoảng 20+ GB).  
   - Tạo task definition xong.

2. **Tạo Service ECS từ Task Definition**  
   - Vào Clusters → chọn cluster demo → mục Services → Create service.  
   - Chọn task definition family `nginxdemos-hello` và revision mới nhất.  
   - Đặt tên service (giữ mặc định hoặc đổi).  
   - Compute configuration:  
     - Dùng Fargate, chiến lược capacity provider mặc định.  
     - Platform version: latest.  
   - Deployment configuration:  
     - Kiểu: **Replica**.  
     - Số lượng task mong muốn ban đầu: 1 (có thể tăng lên 4 để có 4 container).  
     - Bật tính năng AZ rebalancing (ECS tự cân bằng task giữa các AZ).  

3. **Cấu hình mạng và bảo mật**  
   - Chọn các subnet sẵn có.  
   - Tạo security group mới, cho phép HTTP (port 80) từ mọi nơi (0.0.0.0/0) để truy cập nginx.  
   - Bật gán **Public IP** cho task.

4. **Cấu hình Load Balancer**  
   - Bật load balancing, chọn **Application Load Balancer (ALB)**.  
   - Tạo ALB mới, tên `DemoALBForECS`, listener trên port 80.  
   - Tạo target group mới `nginxdemosTG` trên port 80, gắn với ALB.  
   - Không thay đổi VPC, không bật autoscaling cho service trong demo.

5. **Triển khai và kiểm tra**  
   - Tạo service, ECS khởi chạy 1 task Fargate.  
   - Kiểm tra trong service:  
     - Desired tasks = 1, Running tasks = 1, trạng thái Active.  
     - Service gắn với target group `nginxdemosTG`.  
   - Vào Target Group:  
     - Thấy target là IP private của container.  
     - Target group gắn với ALB `DemoALBForECS`.  
   - Vào ALB, copy DNS name, mở trên trình duyệt:  
     - Hiện trang chào mừng nginx → xác nhận hệ thống hoạt động.  
   - Kiểm tra task details:  
     - Thông tin revision, private IP, container, logs của nginx.  
   - Kiểm tra tab Events của service để xem các sự kiện: task được khởi chạy, đăng ký vào target group, đạt trạng thái ổn định.  
   - Truy cập thêm đường dẫn `/test` để thấy URI thay đổi, chứng minh nginx hoạt động đúng.

6. **Scale out (mở rộng)**  
   - Cập nhật service, tăng Desired tasks từ 1 lên 3 (ví dụ 1 task mỗi AZ).  
   - ECS/Fargate tự động cung cấp thêm 2 task mới.  
   - Khi các task chuyển sang trạng thái Running, ALB phân phối traffic giữa nhiều container.  
   - Làm mới trang nhiều lần sẽ thấy địa chỉ IP backend thay đổi, chứng minh load balancing và khả năng mở rộng của ECS.

7. **Scale in (thu nhỏ) và tiết kiệm chi phí**  
   - Cập nhật service, đặt Desired tasks = 0 → dừng toàn bộ container nhưng vẫn giữ service cấu hình.  
   - Đảm bảo Auto Scaling Group (nếu có EC2) cũng đặt desired capacity = 0 để không còn instance chạy.  
   - Kiểm tra lại trong service: không còn task chạy; xem Events để biết ECS đã dừng task như thế nào.

8. **Kết luận**  
   - Đã trình diễn:  
     - Tạo cluster ECS.  
     - Tạo task definition dùng Fargate.  
     - Tạo service ECS với ALB, security group, subnet.  
     - Deploy ứng dụng nginx demo, kiểm tra hoạt động.  
     - Scale out/in số lượng task để minh họa khả năng mở rộng và tối ưu chi phí.
