


Nội dung trên là một bài hướng dẫn thực hành sử dụng AWS Systems Manager Parameter Store:

1. Truy cập Parameter Store  
   - Vào AWS Console, tìm “Parameter Store” trong Systems Manager → Application Management → Parameter Store.  
   - Đây là nơi lưu trữ cấu hình/secret dưới dạng tham số.

2. Tạo tham số dạng String (ví dụ DB URL)  
   - Chọn “Create parameter”.  
   - Đặt tên theo dạng phân cấp, ví dụ:  
     - `/my-app/dev/db-url`  
       - `my-app`: tên ứng dụng  
       - `dev`: môi trường  
       - `db-url`: tên tham số  
   - Tier: chọn Standard (tối đa 10.000 tham số, giá trị ≤ 4 KB, không chia sẻ cross-account).  
     - Advanced: tối đa 100.000 tham số, giá trị ≤ 8 KB, có thể chia sẻ cross-account.  
   - Type: String (có thể là String, StringList, SecureString).  
   - Data type: Text (có thể là text hoặc AWS EC2 Image).  
   - Nhập giá trị, ví dụ: `dev.database.stephantheteacher.com`.  
   - Tạo tham số, có thể xem version và lịch sử thay đổi.

3. Tạo tham số dạng SecureString (ví dụ DB password)  
   - Tạo tham số mới: `/my-app/dev/db-password`.  
   - Type: SecureString (dùng cho mật khẩu/secret).  
   - Chọn KMS key để mã hóa:  
     - Dùng key mặc định `alias/aws/ssm` hoặc  
     - Dùng custom key (ví dụ key “Tutorial”).  
   - Nhập giá trị, ví dụ: `devpassword`.  
   - Sau khi tạo, giá trị được mã hóa và ẩn; muốn xem phải “Show decrypted value” và có quyền KMS phù hợp.

4. Tạo thêm tham số cho môi trường Prod  
   - Ví dụ:  
     - `/my-app/prod/db-url` → `prod.stephantheteacher.com:3306`  
     - `/my-app/prod/db-password` → SecureString, dùng cùng KMS key, giá trị `prodpassword`.  
   - Kết quả: có 4 tham số, được tổ chức theo hệ thống phân cấp `/my-app/dev/...` và `/my-app/prod/...`.

5. Truy cập tham số bằng AWS CLI (CloudShell)  
   - Mở CloudShell (CLI đã cài và cấu hình sẵn).  

   a) Lệnh `get-parameters`  
   - Ví dụ:  
     ```bash
     aws ssm get-parameters \
       --names "/my-app/dev/db-url" "/my-app/dev/db-password"
     ```  
   - Kết quả:  
     - `db-url` (String) trả về giá trị rõ.  
     - `db-password` (SecureString) trả về giá trị đã mã hóa.  

   - Để giải mã SecureString:  
     ```bash
     aws ssm get-parameters \
       --names "/my-app/dev/db-url" "/my-app/dev/db-password" \
       --with-decryption
     ```  
   - Khi đó `db-password` được trả về dưới dạng giá trị đã giải mã (nếu có quyền KMS).

   b) Lệnh `get-parameters-by-path`  
   - Lấy tất cả tham số dưới một “path”:  
     ```bash
     aws ssm get-parameters-by-path \
       --path "/my-app/dev"
     ```  
     → trả về tất cả tham số trong namespace `/my-app/dev` (db-url, db-password).  

   - Lấy toàn bộ tham số cho cả app:  
     ```bash
     aws ssm get-parameters-by-path \
       --path "/my-app" \
       --recursive
     ```  
     → trả về cả dev và prod (4 tham số).  

   - Có thể thêm `--with-decryption` để giải mã tất cả SecureString.

6. Kết luận  
   - Parameter Store cho phép:  
     - Tổ chức cấu hình/secret theo hệ thống phân cấp.  
     - Phân biệt môi trường (dev/prod).  
     - Lưu trữ giá trị thường (String) và secret (SecureString) với KMS.  
     - Truy cập linh hoạt qua CLI (get-parameters, get-parameters-by-path, recursive, with-decryption).  
   - Bài giảng kết thúc bằng việc nhấn mạnh sức mạnh của Parameter Store trong quản lý cấu hình và secret.
