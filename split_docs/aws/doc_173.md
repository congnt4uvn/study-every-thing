


Dưới đây là tóm tắt nội dung:

1. **Chọn loại hạ tầng cho ECS Task Definition**  
   - Có thể chạy trên **AWS Fargate**, **EC2**, hoặc cả hai.  
   - Với **Fargate**: phải chọn cấu hình CPU và RAM theo các mức tương thích Fargate; **network mode bắt buộc là AWS VPC**.  
   - Với **EC2**: có thể nhập tự do giá trị CPU/RAM; có thêm lựa chọn network mode nâng cao.

2. **IAM Role cho ECS**  
   - **Task Role**: IAM role mà container sử dụng để gọi API AWS (ví dụ truy cập S3, DynamoDB…). Rất quan trọng trong kỳ thi.  
   - **Task Execution Role**: IAM role dành cho ECS agent để thực hiện các thao tác như kéo image từ registry, ghi log… Đây là role “chuẩn” cho ECS.

3. **Container trong Task Definition**  
   - Một task definition có thể chứa nhiều container, nhưng **ít nhất một container phải là “essential”**.  
   - Nếu container essential bị lỗi/dừng → **toàn bộ task dừng**.  
   - Container không essential có thể dừng mà task vẫn tiếp tục.

4. **Image & Private Registry**  
   - Container image có thể lấy từ public registry hoặc **private registry**.  
   - Khi dùng private registry, cần cấu hình **ARN của Secret Manager** để ECS dùng secret đó xác thực và kéo image.

5. **Port Mapping**  
   - Định nghĩa các cổng container, giao thức (HTTP, HTTP2, gRPC, hoặc none), tên cổng.  
   - Có thể khai báo nhiều port mapping nếu ứng dụng mở nhiều cổng.

6. **Giới hạn tài nguyên**  
   - Có thể đặt giới hạn CPU, memory (hard/soft limit) cho từng container, hữu ích khi nhiều container chạy cùng trên một host.

7. **Environment Variables**  
   - Có thể khai báo trực tiếp (key–value).  
   - Hoặc lấy từ **AWS Secrets Manager** / **SSM Parameter Store** bằng ARN.  
   - Có thể nạp từ **file trên S3** chứa danh sách biến môi trường.

8. **Logging**  
   - Log có thể gửi tới: **CloudWatch Logs**, Splunk, Firehose, Kinesis Streams, OpenSearch, S3…  
   - Thông qua **AWS FireLens** hoặc trực tiếp CloudWatch.  
   - Với CloudWatch: cấu hình log group, region, stream prefix, và tùy chọn tự tạo log group.

9. **Health Check & Timeout**  
   - Health check đảm bảo container vẫn “healthy”.  
   - **Start timeout**: nếu container không khởi động kịp → bị kill.  
   - **Stop timeout**: thời gian chờ để container tắt “gracefully” trước khi bị kill.

10. **Storage & Volume**  
    - Có thể thêm storage cho container:  
      - **Bind mount** (gắn thư mục host).  
      - **EFS** (Amazon Elastic File System).  
    - Định nghĩa **container mount points**: đường dẫn trong container và volume tương ứng.  
    - Có thể gắn volume từ container khác, hoặc từ EFS/host.

11. **Monitoring & Tracing**  
    - Có thể bật thu thập trace gửi tới **AWS X-Ray** thông qua sidecar **AWS Distro for OpenTelemetry** (ADOT).  
    - CPU/memory sẽ tự điều chỉnh để thêm sidecar.  
    - Có thể gửi metrics tới **CloudWatch** hoặc **Amazon Managed Service for Prometheus**.

12. **Hoàn tất Task Definition**  
    - Sau khi cấu hình xong, nhấn **Create**.  
    - Có thể xem lại cấu hình dưới dạng **JSON**, hoặc tạo **revision mới** và chỉnh sửa dần.  
    - Bài giảng minh họa cách cấu hình đầy đủ ECS Task Definition bằng AWS Console.
