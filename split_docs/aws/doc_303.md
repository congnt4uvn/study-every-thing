


Dưới đây là tóm tắt nội dung:

1. Tạo hàm Lambda  
   - Tạo hàm tên `lambda-version-demo` với runtime Python 3.8.  
   - Viết code đơn giản trả về chuỗi `"this is version one"` (phiên bản 1).  
   - Deploy và test, kết quả đúng như mong đợi.

2. Xuất bản (publish) phiên bản  
   - Từ hàm đang chạy tốt, chọn “Actions” → “Publish new version”.  
   - Đặt mô tả, publish → tạo ra **Version 1**.  
   - Khi xem Version 1: không thể chỉnh sửa code (chỉ test được), vì version đã publish là bất biến.

3. Tạo phiên bản mới  
   - Quay lại **$LATEST** (bản chưa publish), sửa code thành `"this is version two"`.  
   - Deploy, test → kết quả `"this is version two"`.  
   - Publish → tạo **Version 2**.  
   - Version 1 và Version 2 tồn tại song song, mỗi version có code riêng, không chỉnh sửa trực tiếp được.

4. Tạo alias (bí danh)  
   - Alias dùng để trỏ đến một version cụ thể hoặc `$LATEST`, giúp quản lý môi trường (dev/test/prod).

   a. Alias `dev`  
   - Tạo alias `dev` trỏ đến **$LATEST** (bản mới nhất chưa publish).  
   - Dùng cho developer để luôn chạy code mới nhất.

   b. Alias `test`  
   - Tạo alias `test` trỏ đến **Version 2**.  
   - Test alias này cho kết quả `"this is version two"`.

   c. Alias `prod`  
   - Tạo alias `prod` trỏ đến **Version 1** (phiên bản ổn định).  
   - Test alias `prod` cho kết quả `"this is version one"`.

5. Alias có trọng số (weighted alias) – triển khai dần (canary / gradual rollout)  
   - Chỉnh sửa alias `prod` để trỏ đồng thời:  
     - Version 1 với một trọng số.  
     - Version 2 với một trọng số khác.  
   - Ví dụ thực tế: 90% traffic → Version 1, 10% → Version 2.  
   - Trong demo, dùng 50% – 50% để dễ quan sát.  
   - Sau khi lưu, test alias `prod` nhiều lần:  
     - Có lần trả `"this is version one"`,  
     - Có lần trả `"this is version two"`,  
     → chứng minh traffic được phân phối giữa hai version.

6. Chuyển hẳn sang version mới  
   - Khi đã hài lòng với Version 2 trong môi trường production:  
     - Chỉnh sửa alias `prod`, bỏ weighted routing,  
     - Trỏ 100% traffic sang **Version 2**.  
   - Như vậy alias `prod` được nâng cấp từ Version 1 lên Version 2.

7. Ý nghĩa  
   - **Version**: ảnh chụp bất biến của code + config tại một thời điểm.  
   - **Alias**: tên logic (dev/test/prod) trỏ đến version cụ thể, có thể dùng weighted routing để triển khai dần.  
   - Cách này giúp Lambda phát triển an toàn, kiểm soát được rollout và rollback.
