


Nội dung trên là một đoạn demo thực hành về cách sử dụng AMI (Amazon Machine Image) trong AWS EC2:

1. **Khởi chạy EC2 ban đầu**
   - Chọn Amazon Linux 2, loại máy `t2.micro`.
   - Chọn sẵn key pair, security group (nhóm bảo mật) đã tạo trước.
   - Trong phần Advanced details → User data:
     - Dùng script cài đặt Apache HTTPD (web server).
     - Bỏ dòng cuối cùng tạo file index (chỉ cài HTTPD, chưa tạo nội dung web).
   - Khởi chạy instance, chờ 1–2 phút để user data chạy xong.
   - Sau khi Apache cài xong, truy cập IP public bằng HTTP sẽ thấy trang test mặc định của Apache.

2. **Tạo AMI từ instance đã cấu hình**
   - Chuột phải vào instance → “Create image”.
   - Đặt tên, ví dụ “demo image”, giữ nguyên các thiết lập khác.
   - AMI được tạo và hiển thị trong mục AMI, trạng thái ban đầu là “pending”, chờ đến khi hoàn tất.

3. **Khởi chạy instance mới từ AMI**
   - Từ trang AMI, chọn AMI “demo image” → Launch instance.
   - Hoặc trong giao diện tạo instance, chọn tab “My AMIs” → chọn AMI vừa tạo.
   - Thiết lập lại key pair, network, security group như ý.
   - Trong phần User data lần này:
     - Không cài lại HTTPD (vì AMI đã có sẵn).
     - Chỉ dùng script ngắn để tạo file index (ví dụ ghi nội dung “Hello World”).
   - Khởi chạy instance, do HTTPD đã cài sẵn nên thời gian khởi động và sẵn sàng phục vụ nhanh hơn.
   - Truy cập IP public sẽ thấy nội dung “Hello World”.

4. **Ý nghĩa của AMI**
   - AMI cho phép:
     - Lưu lại trạng thái một instance (đã cài sẵn phần mềm, cấu hình).
     - Khởi chạy nhanh nhiều instance mới từ cùng một cấu hình.
   - Rất hữu ích khi có nhiều phần mềm, bảo mật, cấu hình phức tạp cần cài đặt trước: chỉ làm một lần, đóng gói thành AMI, sau đó khởi chạy nhanh các instance mới.

5. **Dọn dẹp**
   - Sau demo, giảng viên chọn cả hai instance và terminate (chấm dứt) để tránh tốn chi phí.
