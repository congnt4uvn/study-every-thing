


Dưới đây là tóm tắt nội dung:

1. Tạo hàm Lambda:
- Tạo một hàm Lambda mới tên “Lambda S3” với runtime Python 3.8.
- Hàm này sẽ được gọi mỗi khi có đối tượng mới được thêm vào một bucket S3.

2. Tạo bucket S3 và cấu hình sự kiện:
- Vào dịch vụ S3, tạo một bucket mới (ví dụ: events3-demo-stephane), đảm bảo cùng region với Lambda (ví dụ: Ireland).
- Trong phần Properties của bucket, tìm mục Event notifications.
- Tạo một Event notification mới:
  - Đặt tên (ví dụ: call-lambda).
  - Prefix và suffix để mặc định (tức là áp dụng cho mọi đối tượng).
  - Event type: chọn “All object create events”.
  - Destination: chọn Lambda, rồi chọn hàm Lambda S3 vừa tạo.
- Lưu cấu hình, từ đó S3 sẽ gửi sự kiện đến Lambda khi có object mới.

3. Kiểm tra cấu hình Lambda:
- Quay lại trang Lambda, refresh để thấy S3 đã được liệt kê là trigger của hàm.
- Sửa code Lambda để in ra biến `event` (log nội dung sự kiện S3 gửi vào), rồi deploy.

4. Quyền gọi Lambda (resource-based policy):
- Vào tab Configuration → Permissions của Lambda.
- Cuộn xuống phần Resource-based policy.
- Thấy một statement cho phép S3 bucket (events3-demo-stephane) gọi hàm Lambda S3.
- Có thể xem chi tiết JSON policy để thấy rõ S3 là principal được phép invoke hàm.

5. Kiểm thử tích hợp:
- Vào S3 Management Console, upload một file (ví dụ: coffee.jpeg) lên bucket.
- Upload thành công sẽ kích hoạt sự kiện gửi đến Lambda.
- Vào tab Monitor của Lambda → “View logs in CloudWatch”.
- Mở log stream tương ứng với lần invoke do S3 gửi.
- Trong log, xem nội dung `event`:
  - `eventSource` là `aws:s3`.
  - Thông tin region, bucket name (events3-demo-stephane), object key (coffee.jpeg), size, ETag, v.v.

6. Ý nghĩa:
- Từ thông tin trong event, Lambda có thể dùng SDK (boto3) để gọi S3 API, lấy object vừa upload và xử lý (ví dụ: resize ảnh, phân tích dữ liệu, v.v.).
- Đây là một ví dụ về cơ chế gọi Lambda bất đồng bộ (asynchronous invocation) thông qua S3 events.
- Đồng thời minh họa cách hoạt động của resource-based policy cho phép dịch vụ khác (S3) invoke Lambda.
