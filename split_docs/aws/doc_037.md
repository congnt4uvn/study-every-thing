


Nội dung trên nói về các cách kết nối vào máy chủ (EC2) để quản trị, chủ yếu là dùng SSH:

1. SSH là gì và dùng khi nào  
- SSH là tiện ích dòng lệnh để truy cập an toàn vào máy chủ Linux.  
- Dùng để “đi vào bên trong” máy chủ và thực hiện bảo trì, thao tác.

2. Các cách kết nối theo hệ điều hành:
- Mac / Linux: dùng trực tiếp lệnh `ssh` trong terminal.  
- Windows:
  - Windows 10 trở lên: có thể dùng SSH (tương tự Mac/Linux).  
  - Windows dưới 10: dùng phần mềm PuTTY (thực hiện chức năng giống SSH).

3. EC2 Instance Connect:
- Là cách kết nối mới, dùng ngay trên trình duyệt web, không cần cài đặt gì.  
- Hoạt động trên mọi hệ điều hành (Mac, Linux, mọi phiên bản Windows).  
- Hiện tại chỉ hỗ trợ loại máy Amazon Linux 2, nên trong hướng dẫn giảng viên dùng Amazon Linux 2.  
- Giảng viên sẽ chủ yếu dùng EC2 Instance Connect trong các bài sau.

4. Hướng dẫn tiếp theo:
- Nếu dùng Mac/Linux: xem bài “SSH trên Mac/Linux”.  
- Nếu dùng Windows: xem bài “PuTTY” hoặc “SSH trên Windows 10”.  

5. Vấn đề thường gặp:
- SSH là phần gây rắc rối nhiều nhất cho học viên (lỗi security group, lệnh sai, lỗi chính tả, v.v.).  
- Có bài hướng dẫn khắc phục sự cố riêng, nên nếu gặp lỗi hãy xem lại.  
- EC2 Instance Connect thường giúp giải quyết nhiều vấn đề SSH.

6. Lời khuyên:
- Chỉ cần một phương pháp kết nối hoạt động là đủ, không cần tất cả.  
- Nếu không cách nào hoạt động cũng không sao, vì khóa học chỉ mang tính giới thiệu và không dùng SSH quá nhiều.
