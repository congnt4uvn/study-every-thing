


Nội dung trên là một hướng dẫn thực hành về cách tạo và sử dụng Amazon S3 bucket:

1. Tạo bucket  
- Chọn Region (ví dụ: eu-west-1 – Ireland).  
- Chọn loại bucket: “General purpose” (đa năng) thay vì “Directory”.  
- Giải thích về namespace:  
  - Global namespace: tên bucket phải duy nhất trên toàn hệ thống AWS.  
  - Account Regional namespace: AWS tự thêm hậu tố chứa số tài khoản + region, giúp bạn có thể đặt tên đơn giản (demo, test, …) mà vẫn luôn khả dụng.  
- Đặt tên bucket (ví dụ stephane-demo-s3-v12) sao cho không trùng.

2. Cấu hình bảo mật và các tùy chọn  
- Object ownership: tắt ACL (mặc định, khuyến nghị).  
- Public access: bật chặn truy cập công khai để đảm bảo chỉ chủ tài khoản truy cập được.  
- Versioning: tạm thời tắt, sẽ bật sau.  
- Tagging: không dùng.  
- Encryption: bật mã hóa phía máy chủ (SSE-S3) với khóa do S3 quản lý, bật Bucket Key.  
- Giữ hầu hết cài đặt mặc định, chỉ thay tên bucket.

3. Xem danh sách bucket  
- Sau khi tạo, bucket xuất hiện trong danh sách tất cả bucket trên mọi region.  
- Có thể tìm nhanh bằng ô tìm kiếm.

4. Upload và xem file  
- Vào bucket, chọn Upload, chọn file coffee.jpg (100 KB).  
- Sau khi upload, file xuất hiện trong tab Objects.  
- Nhấp vào file để xem chi tiết: kích thước, loại, URL đối tượng.  
- Nhấn nút “Open” trong console: file mở được vì AWS tạo một URL đã ký sẵn (pre-signed URL) chứa chữ ký và thông tin xác thực của bạn.  
- Khi dùng “Public URL” (URL đối tượng thông thường) thì bị lỗi AccessDenied vì bucket đang chặn truy cập công khai.  
- Giải thích:  
  - Public URL: chỉ dùng được nếu đối tượng/bucket cho phép public.  
  - Pre-signed URL: dài, phức tạp, chứa chữ ký, chỉ người có URL và có quyền mới truy cập được.

5. Thư mục (folder) trong S3  
- Tạo folder tên images trong bucket.  
- Upload file beach.jpg vào folder images.  
- Cấu trúc hiển thị giống các dịch vụ lưu trữ đám mây (Google Drive, Dropbox…).  
- Có thể xóa folder images: thao tác này xóa luôn các file bên trong; cần gõ “delete permanently” để xác nhận.

6. Kết luận  
- Bài học đã hướng dẫn:  
  - Tạo bucket S3 với các cài đặt bảo mật/mã hóa cơ bản.  
  - Upload file, mở file bằng pre-signed URL.  
  - Phân biệt URL công khai và URL đã ký sẵn.  
  - Tạo, quản lý, xóa thư mục và đối tượng trong S3.
