


Nội dung trên nói về S3 Access Logs (Nhật ký truy cập S3):

1. Mục đích  
   - Dùng cho kiểm tra (auditing): ghi lại mọi lần truy cập vào bucket S3.  
   - Ghi lại cả yêu cầu được cho phép (authorized) lẫn bị từ chối (denied).  
   - Mọi yêu cầu đến bucket S3 sẽ được ghi thành file log trong một bucket S3 khác.

2. Phân tích dữ liệu  
   - Các file log này có thể được phân tích bằng các công cụ như Amazon Athena.  
   - Định dạng log có chuẩn riêng, có thể xem chi tiết tại tài liệu AWS (URL được nhắc đến).

3. Yêu cầu về bucket ghi log  
   - Bucket dùng để lưu log (target logging bucket) phải nằm trong cùng Region AWS với bucket được ghi log.  
   - Khi bật Access Logging cho bucket, tất cả request đến bucket đó sẽ được ghi vào bucket ghi log.

4. Cảnh báo quan trọng  
   - Không được dùng cùng một bucket vừa làm bucket ứng dụng (bucket đang được theo dõi) vừa làm bucket ghi log.  
   - Nếu làm vậy sẽ tạo vòng lặp ghi log vô hạn:  
     - Ghi log → log lại chính file log → tiếp tục log…  
     - Dung lượng bucket tăng theo cấp số nhân, chi phí rất lớn.  

Tóm lại: S3 Access Logs cho phép ghi lại mọi truy cập vào bucket để phục vụ kiểm tra và phân tích, nhưng phải cấu hình bucket ghi log riêng (cùng Region) để tránh vòng lặp log vô hạn và chi phí cao.
