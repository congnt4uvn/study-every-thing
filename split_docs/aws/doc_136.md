


Dưới đây là tóm tắt nội dung:

1. Tạo S3 bucket  
   - Tạo bucket tên `stephane-v3-event-notification` (region Ireland).  
   - Mở bucket, vào tab Properties, phần Event notifications.

2. Hai cách nhận sự kiện từ S3  
   - Cách 1: Tạo Event notification (gửi đến Lambda/SNS/SQS).  
   - Cách 2: Bật tích hợp Amazon EventBridge để gửi mọi sự kiện của bucket vào EventBridge.  
   - Trong demo, chọn cách 1 (Event notification) vì đơn giản hơn.

3. Cấu hình Event notification  
   - Tạo notification tên `DemoEventNotification`.  
   - Không dùng prefix/suffix filter.  
   - Chọn loại sự kiện: tất cả sự kiện tạo object (ObjectCreated).  
   - Có thể thêm các loại khác như xóa object, restore object, v.v.

4. Chọn đích (destination)  
   - Có 3 lựa chọn: Lambda, SNS topic, SQS queue.  
   - Demo chọn SQS, nhưng trước tiên phải tạo queue và cấp quyền.

5. Tạo SQS queue và cấp quyền  
   - Vào SQS, tạo queue tên `DemoS3Notification`.  
   - Quay lại S3, chọn queue này làm đích.  
   - Lúc lưu cấu hình bị lỗi: SQS chưa cho phép S3 gửi message.  
   - Vào phần Access policy của queue, dùng Policy Generator:  
     - Loại: SQS queue policy.  
     - Effect: Allow.  
     - Principal: * (rất “dễ dãi”).  
     - Action: `SendMessage`.  
     - Resource: ARN của queue.  
   - Lưu policy mới.  
   - Quay lại S3, lưu lại Event notification → thành công.  
   - S3 gửi một message test vào SQS để kiểm tra kết nối.

6. Kiểm tra hoạt động của Event notification  
   - Vào SQS, “Send and receive messages” → Poll for messages → thấy message test từ S3.  
   - Xóa message test.  
   - Upload một file lên S3 (ví dụ `coffee.jpeg`).  
   - Kiểm tra trong bucket: file `coffee.jpeg` đã được tạo.  
   - Poll messages trong SQS: có message mới.  
   - Nội dung message:  
     - `eventName`: `ObjectCreated:Put`.  
     - `s3.object.key`: `coffee.jpeg`.  
   - Điều này chứng minh mỗi khi object được tạo, S3 gửi event vào SQS.

7. Kết luận  
   - S3 Event Notifications cho phép gửi sự kiện đến:  
     - SQS queue  
     - SNS topic  
     - Lambda function  
   - Hoặc gửi tất cả sự kiện của bucket đến Amazon EventBridge để xử lý nâng cao và fan-out đến nhiều đích khác.
