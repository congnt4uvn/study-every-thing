


Dưới đây là tóm tắt nội dung:

1. Mục tiêu  
   - Thực hành tạo một hàm AWS Lambda (Node.js) có phụ thuộc (dependencies) và tích hợp AWS X-Ray để theo dõi, đồng thời truy cập Amazon S3.

2. Chuẩn bị môi trường  
   - Sử dụng AWS CloudShell vì đã cài sẵn Node.js và NPM.  
   - Tạo thư mục `lambda`, cd vào đó.  
   - Cài trình soạn thảo nano: `sudo yum install -y nano`.  
   - Tạo file `index.js` bằng `nano index.js` và dán mã nguồn mẫu vào.

3. Cài đặt phụ thuộc  
   - Trong `index.js` có `require('aws-xray-sdk-core')` và dùng AWS SDK để gọi S3 `ListBuckets`.  
   - Cài dependency: `npm install aws-xray-sdk`.  
   - Sau khi cài, thư mục có: `index.js`, `node_modules`, `package-lock.json`.

4. Đóng gói và phân quyền  
   - Chỉnh sửa quyền file (dùng `chmod`).  
   - Nén toàn bộ nội dung thư mục thành file `function.zip` (chứa `index.js`, `node_modules`, v.v.) để upload lên Lambda.

5. Tạo IAM Role cho Lambda  
   - Vào IAM → tạo role cho dịch vụ Lambda.  
   - Gán policy “AWSLambdaBasicExecutionRole” (vai trò thực thi cơ bản).  
   - Đặt tên, ví dụ: `lambda-demo-with-dependencies`.  
   - Sao chép ARN của role để dùng trong lệnh CLI tạo Lambda.

6. Tạo hàm Lambda bằng CLI  
   - Dùng AWS CLI trong CloudShell để tạo hàm Lambda, chỉ định:  
     - File zip (`function.zip`)  
     - Runtime Node.js  
     - Handler (`index.handler` hoặc tương tự)  
     - IAM role ARN vừa tạo  
   - Sau khi chạy lệnh, hàm Lambda được tạo; vào console Lambda sẽ thấy mã nguồn và `node_modules`.

7. Cách upload khác  
   - Có thể upload file zip thủ công qua console Lambda, hoặc upload zip lên S3 rồi trỏ Lambda tới S3.

8. Kiểm thử hàm Lambda  
   - Tạo một test event đơn giản trong console Lambda.  
   - Lần đầu chạy bị lỗi “Access Denied” vì Lambda chưa có quyền S3 `ListBuckets`.

9. Bật X-Ray và bổ sung quyền  
   - Trong cấu hình Lambda, bật “Active tracing” (X-Ray).  
   - Điều này thêm quyền X-Ray vào role.  
   - Vào IAM, mở role của Lambda:  
     - Đã có: Lambda basic execution + X-Ray permissions.  
     - Gắn thêm policy “AmazonS3ReadOnlyAccess” để cho phép `ListBuckets`.  
   - Chờ một lúc cho quyền có hiệu lực, chạy lại test:  
     - Lúc đầu vẫn có thể “Access Denied” do độ trễ.  
     - Sau vài lần, hàm chạy thành công và trả về danh sách S3 buckets.

10. Quan sát với AWS X-Ray  
    - Vào console X-Ray, xem Service Map:  
      - Thấy client → Lambda → S3.  
      - Một số request màu xanh (thành công), một số đỏ/vàng (lỗi).  
      - Các lỗi tương ứng với giai đoạn chưa có quyền S3 (403).  
    - Xem chi tiết trace:  
      - Trace lỗi: thấy Lambda gọi S3 `ListBuckets`, nhận 403 AccessDenied.  
      - Trace thành công: thấy thời gian thực thi `ListBuckets`, thời gian phản hồi, v.v.  
    - X-Ray giúp phân tích lỗi quyền và hiệu năng của Lambda.

11. Kết luận  
    - Thực hành này minh họa:  
      - Cách đóng gói hàm Lambda Node.js kèm dependencies.  
      - Cách upload bằng CLI hoặc console.  
      - Cách cấu hình IAM role và quyền S3.  
      - Cách bật và sử dụng AWS X-Ray để theo dõi, debug và phân tích hiệu năng Lambda.
