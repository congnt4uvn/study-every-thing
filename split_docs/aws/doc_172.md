


Dưới đây là tóm tắt nội dung về **ECS Task Definition (định nghĩa nhiệm vụ ECS)** trong bài giảng:

1. **Khái niệm chung về Task Definition**
   - Được định nghĩa dưới dạng JSON (có thể tạo qua Console UI).
   - Cho ECS biết cách chạy **một hoặc nhiều container Docker**.
   - Một task definition có thể chứa tối đa **10 containers**.

2. **Các thông tin quan trọng trong Task Definition**
   - Tên image (Docker image).
   - Cấu hình port:
     - Container port.
     - Host port (khi dùng EC2).
   - Cấu hình CPU, Memory.
   - Biến môi trường (environment variables).
   - Thông tin mạng (networking).
   - IAM Role gắn với task.
   - Cấu hình logging (ví dụ CloudWatch Logs).

3. **Container port vs Host port (kiểu khởi chạy EC2)**
   - Ví dụ: container chạy Apache HTTP trên **container port 80**.
   - Host port có thể là 80 hoặc 8080, không bắt buộc giống container port.
   - Truy cập từ Internet: request đến **host port** trên EC2 instance → được chuyển tiếp vào **container port**.

4. **Dynamic Host Port Mapping với EC2 + ALB**
   - Trong task definition chỉ đặt **container port = 80**, **host port = 0** (không cố định).
   - ECS sẽ gán **host port ngẫu nhiên** cho mỗi task trên EC2.
   - ALB tích hợp với ECS service sẽ tự động biết mỗi task đang lắng nghe trên host port nào (nhờ dynamic host port mapping).
   - Cơ chế này **chỉ hỗ trợ với ALB**, không hỗ trợ Classic Load Balancer.
   - Về bảo mật:
     - Security group của EC2 instance phải cho phép **tất cả các port** từ security group của ALB (vì host port là ngẫu nhiên).

5. **Fargate và port**
   - Mỗi ECS task trên Fargate có **một IP riêng** (ENI riêng).
   - Không có host (EC2) nên chỉ cần cấu hình **container port**.
   - ALB kết nối trực tiếp đến IP của task trên **cùng một port** (ví dụ 80).
   - Security group:
     - SG của ENI (task) cho phép port 80 từ SG của ALB.
     - SG của ALB cho phép 80/443 từ Internet.

6. **IAM Role cho ECS Task**
   - IAM Role được gán **ở cấp Task Definition**, không phải cấp Service.
   - Mỗi task chạy từ task definition đó sẽ **assume** IAM role tương ứng.
   - Ví dụ:
     - Task definition A gán role cho phép truy cập S3 → mọi task từ A đều truy cập được S3.
     - Task definition B gán role khác cho phép truy cập DynamoDB → service khác dùng B sẽ có quyền DynamoDB.
   - Câu hỏi thường gặp trong exam: “IAM role cho ECS task được cấu hình ở đâu?” → **Trong Task Definition**.

7. **Environment Variables (biến môi trường)**
   - Có thể:
     - Hard-code trực tiếp trong task definition (dùng cho giá trị không nhạy cảm, ví dụ URL).
     - Lấy từ:
       - **SSM Parameter Store**.
       - **AWS Secrets Manager** (cho giá trị nhạy cảm: API key, DB password…).
   - ECS sẽ fetch và resolve các giá trị này **khi task khởi chạy**, rồi inject vào container dưới dạng biến môi trường.
   - Có thể load biến môi trường hàng loạt từ **file trong S3** (bulk load).

8. **Chia sẻ dữ liệu giữa các containers trong cùng một task**
   - Một task có thể chứa nhiều containers (ví dụ: app container + sidecar container).
   - Sidecar thường dùng cho:
     - Thu thập/logging.
     - Metrics, monitoring, forwarding…
   - Để chia sẻ file/dữ liệu giữa các containers:
     - Định nghĩa **volume** trong task.
     - Gắn volume đó vào nhiều containers (mount point, ví dụ `/var/logs`).
     - App container ghi log/metrics vào volume, sidecar container đọc từ đó.
   - Cách hoạt động:
     - Với **EC2 launch type**:
       - Volume kiểu bind mount → dùng **ephemeral storage** của EC2 instance.
       - Dữ liệu gắn với vòng đời EC2 instance.
     - Với **Fargate**:
       - Dùng **ephemeral storage** của Fargate task.
       - Dữ liệu gắn với vòng đời task.
       - Fargate cung cấp từ **20 GB đến 200 GB** storage dùng chung cho các containers trong task.

9. **Ý nghĩa chính cho kỳ thi**
   - Nắm rõ:
     - Task definition chứa gì.
     - Sự khác nhau giữa EC2 vs Fargate về port và networking.
     - Dynamic host port mapping và ALB.
     - IAM role được gán ở **task definition**.
     - Cách dùng SSM/Secrets Manager/S3 cho environment variables.
     - Cách chia sẻ dữ liệu giữa containers bằng volumes, đặc biệt cho sidecar pattern.
