


Dưới đây là tóm tắt nội dung:

1. Tạo Network Load Balancer (NLB)  
   - Tạo NLB tên DemoNLB, loại Internet-facing, IPv4.  
   - Chọn VPC hiện có và kích hoạt tất cả các Availability Zone (AZ).  
   - Mỗi AZ có một subnet và NLB sẽ nhận một địa chỉ IPv4 cố định do AWS cấp (có thể thay bằng Elastic IP nếu muốn).

2. Gắn Security Group cho NLB  
   - Tạo security group mới: demo-sg-nlb.  
   - Inbound rule: cho phép HTTP (port 80) từ mọi nơi.  
   - Gắn security group này vào NLB (bỏ security group mặc định).  
   - Ý tưởng: giống ALB, NLB cũng có thể gắn security group để kiểm soát lưu lượng vào.

3. Cấu hình Listener và Target Group  
   - Listener: giao thức TCP, port 80.  
   - Tạo target group mới: demo-tg-nlb, loại target là EC2 instances.  
   - Giao thức target: TCP port 80.  
   - Health check: dùng HTTP (vì ứng dụng HTTP trên EC2),  
     - Healthy threshold: 2  
     - Timeout: 2 giây  
     - Interval: 5 giây  
   - Đăng ký 2 EC2 instances vào target group (trạng thái pending).

4. Vấn đề health check và cách khắc phục  
   - Ban đầu, truy cập DNS của NLB không được.  
   - Kiểm tra target group thấy instances “unhealthy” do health check fail.  
   - Kiểm tra security group của EC2:  
     - Đang cho phép HTTP từ security group của ALB, nhưng chưa cho phép từ NLB.  
   - Sửa inbound rule của EC2:  
     - Thêm rule cho phép HTTP từ security group demo-sg-nlb (của NLB).  
   - Sau đó health check pass, instances chuyển sang trạng thái healthy.

5. Kiểm tra hoạt động của NLB  
   - Truy cập DNS của NLB trên trình duyệt:  
     - Nhận được “Hello World” từ một IP (instance).  
     - Refresh nhiều lần thấy IP thay đổi → chứng tỏ NLB đang phân phối tải giữa 2 instance.

6. Dọn dẹp tài nguyên  
   - Để tránh tốn chi phí:  
     - Xóa NLB (DemoNLB).  
     - Tùy chọn: xóa target group demo-tg-nlb.  
     - Tùy chọn: xóa security group của NLB (thực hành tốt nhưng không bắt buộc).

Nội dung chính: Hướng dẫn từng bước tạo NLB, gắn security group, cấu hình listener/target group, xử lý lỗi health check do thiếu rule security group, và xác nhận NLB cân bằng tải thành công giữa các EC2 instances.
