


Nội dung trên là một hướng dẫn thực hành triển khai ứng dụng với **AWS Amplify**:

1. **Khởi tạo ứng dụng với Amplify**
   - Trong AWS Amplify Console, tạo một ứng dụng mới.
   - Mở tài liệu hướng dẫn, sao chép một repository mẫu (Amplify VT React) về GitHub của riêng bạn.
   - Quay lại Amplify, chọn nhà cung cấp là GitHub, cấp quyền để Amplify đọc repo của bạn.
   - Chọn repo mẫu vừa sao chép, bật cài đặt build, để Amplify tự tạo và dùng service role mới.
   - Nhấn “Save and deploy” để Amplify bắt đầu thiết lập.

2. **Hậu trường hạ tầng (Infrastructure)**
   - Amplify dùng **CDK** và **CloudFormation** để tạo hạ tầng cần thiết.
   - Trong CloudFormation, bạn sẽ thấy các stack Amplify và các nested stack được tạo tự động.
   - Ứng dụng được build xong, Amplify cung cấp một domain để truy cập ứng dụng (ví dụ app To‑Do).

3. **Ứng dụng To‑Do**
   - Ví dụ ứng dụng là một To‑Do app (việc cần làm: vượt qua kỳ thi, luyện tập câu hỏi).
   - Có thể thêm, lưu dữ liệu To‑Do; toàn bộ được lưu trữ bởi Amplify.

4. **Data Manager & DynamoDB**
   - Trong Amplify UI, chọn nhánh production để xem thêm thông tin.
   - Vào **Data Manager** để xem dữ liệu ứng dụng: các bản ghi To‑Do đã thêm xuất hiện ở đây.
   - Phía sau, Amplify dùng **DynamoDB**: có một bảng To‑Do, các item tương ứng với dữ liệu trong Data Manager.
   - Có thể:
     - Tạo To‑Do mới trực tiếp trong Data Manager.
     - Chỉnh sửa To‑Do.
     - Tải dữ liệu dưới dạng CSV.

5. **Quản lý người dùng (Auth)**
   - Phần **User management** dùng để tạo và quản lý người dùng, bảo mật ứng dụng.
   - Cần tạo nhánh xác thực (auth branch) và đẩy code thiết lập auth.
   - Amplify tích hợp sâu với **Amazon Cognito** để quản lý người dùng và xác thực.

6. **Lưu trữ (Storage)**
   - Phần **Storage** cho biết loại lưu trữ backend (ví dụ Amazon S3).
   - Có thể tạo backend storage mới và cấu hình theo nhu cầu.

7. **Functions (Lambda)**
   - Các **functions** trong Amplify là các hàm backend được hỗ trợ bởi **AWS Lambda**.
   - Tất cả được trừu tượng hóa trong Amplify Console, giúp bạn tập trung vào code, còn hạ tầng do Amplify lo.

8. **UI Library & Figma-to-React**
   - Amplify cung cấp **UI Library**, có thể dùng các component UI hoặc tính năng **Figma-to-React** trực tiếp từ console.

9. **CI/CD và cấu hình nâng cao**
   - Khi thay đổi code và push lên GitHub, Amplify tự động build và deploy (CI/CD).
   - Có thể cấu hình:
     - Domain tùy chỉnh
     - Build notifications
     - Build settings
     - Environment variables
   - Tương tự các dịch vụ như Netlify, nhưng mọi thứ chạy trong tài khoản AWS của bạn, dữ liệu ở chế độ riêng tư.

10. **Xóa ứng dụng và dọn dẹp**
    - Trong **App settings → General**, có thể xóa ứng dụng Amplify (gõ “delete”), tất cả tài nguyên liên quan sẽ bị xóa.
    - Vào GitHub, xóa repository đã dùng (nhập tên repo để xác nhận).

Kết luận: Amplify là một “cửa hàng tất cả trong một” trên AWS để xây dựng, triển khai và quản lý ứng dụng (frontend + backend, auth, data, storage, functions) với CI/CD và hạ tầng được tự động hóa.
