


Dưới đây là tóm tắt nội dung:

- Serverless không có nghĩa là “không có server”, mà là lập trình viên không phải quản lý hay cung cấp (provision) server.  
- Ban đầu serverless chủ yếu là FaaS (Function as a Service), ví dụ AWS Lambda.  
- Hiện nay, serverless được hiểu rộng hơn: bất kỳ dịch vụ nào được quản lý hoàn toàn, tự mở rộng, bạn chỉ dùng và trả tiền theo mức sử dụng, không quản lý hạ tầng (database, messaging, storage, v.v.).  

Ví dụ kiến trúc serverless trên AWS:  
- Người dùng truy cập nội dung tĩnh từ S3 (có thể qua CloudFront).  
- Đăng nhập và quản lý danh tính bằng Amazon Cognito.  
- Gọi API REST qua API Gateway.  
- API Gateway gọi các hàm AWS Lambda.  
- Lambda đọc/ghi dữ liệu vào DynamoDB.  

Các dịch vụ AWS thường gặp trong kiến trúc serverless:  
- Lambda, DynamoDB, Cognito, API Gateway, S3  
- SNS, SQS (nhắn tin, hàng đợi, tự mở rộng, không quản lý server)  
- Kinesis Data Firehose (tự mở rộng theo throughput, trả tiền theo sử dụng)  
- Aurora Serverless (database tự mở rộng, không quản lý server)  
- Step Functions  
- AWS Fargate (chạy container không cần quản lý hạ tầng ECS)  

Phần tiếp theo của khóa học sẽ tập trung vào AWS Lambda và các dịch vụ serverless liên quan, vì kỳ thi sẽ kiểm tra rất nhiều kiến thức về serverless.
