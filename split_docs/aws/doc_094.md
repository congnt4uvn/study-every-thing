


Dưới đây là tóm tắt ngắn gọn nội dung:

1. Mục tiêu  
   - Có một tài nguyên AWS (ALB, CloudFront, …) với hostname do AWS cấp.  
   - Muốn ánh xạ hostname đó sang domain của mình (ví dụ: myapp.mydomain.com).

2. CNAME Record  
   - CNAME cho phép trỏ một hostname này đến một hostname khác.  
   - Ví dụ: app.mydomain.com → some-alb.amazonaws.com.  
   - Chỉ dùng được cho subdomain (non‑root), như app.mydomain.com.  
   - Không được dùng CNAME ở “đỉnh zone” (zone apex), tức là không được đặt CNAME cho mydomain.com.

3. Alias Record (bản ghi bí danh – riêng của Route 53)  
   - Cho phép trỏ một tên miền đến một tài nguyên AWS cụ thể:  
     app.mydomain.com → ALB / CloudFront / API Gateway / S3 website / Elastic Beanstalk / Global Accelerator / VPC Interface Endpoint / bản ghi Route 53 khác cùng hosted zone.  
   - Hoạt động được cho cả root domain và subdomain:  
     - mydomain.com (zone apex)  
     - app.mydomain.com  
   - Luôn là loại A hoặc AAAA (IPv4/IPv6).  
   - TTL không tự đặt được, Route 53 quản lý.  
   - Miễn phí truy vấn (không tính phí như bản ghi DNS thường).  
   - Hỗ trợ health check tích hợp.  
   - Tự động cập nhật nếu IP của tài nguyên (ví dụ ALB) thay đổi.  
   - Không thể đặt alias trực tiếp đến DNS name của EC2 instance.

4. Ví dụ thực hành trong bài  
   - Tạo CNAME:  
     - myapp.stephanetheteacher.com → ALB DNS name.  
     - Truy cập myapp.stephanetheteacher.com sẽ đi qua ALB đến EC2 và trả về “Hello World…”.  
   - Tạo Alias record:  
     - myalias.stephanetheteacher.com (A – Alias) → ALB trong region eu-central-1.  
     - Truy cập myalias.stephanetheteacher.com cũng ra kết quả “Hello World…”.  
   - Thử tạo CNAME cho root domain stephanetheteacher.com → ALB:  
     - Bị từ chối: “CNAME not allowed at apex of zone”.  
   - Giải pháp:  
     - Tạo A – Alias cho root domain stephanetheteacher.com → ALB.  
     - Truy cập stephanetheteacher.com hoạt động bình thường.

5. Ý chính cần nhớ (hay bị hỏi trong exam)  
   - CNAME: chỉ dùng cho subdomain, không dùng cho root domain.  
   - Alias (Route 53): dùng được cho cả root và subdomain, chỉ trỏ đến tài nguyên AWS được hỗ trợ, miễn phí, có health check, tự động theo dõi IP thay đổi.
