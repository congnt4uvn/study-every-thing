
Dưới đây là tóm tắt nội dung bài giảng:

1. **Bối cảnh dự án**
   - Trong thư mục `section-6` hiện có thư mục `v1-spring-boot` chứa 3 microservice: `accounts`, `cards`, `loans`.
   - Các service này đang quản lý cấu hình chỉ bằng Spring Boot.
   - Giảng viên không muốn sửa cấu hình Spring Cloud trong `v1-spring-boot`, nên tạo thêm một phiên bản mới.

2. **Tạo phiên bản mới v2**
   - Sao chép thư mục `v1-spring-boot`, dán cùng cấp và đổi tên thành `v2-spring-cloud-config`.
   - Xóa thư mục ẩn `.idea` vì nó thuộc workspace IntelliJ cũ.
   - Từ giờ, `v1` là tham chiếu cho cách làm với Spring Boot, `v2` là tham chiếu cho cách làm với Spring Cloud Config.

3. **Tạo dự án Config Server**
   - Truy cập `start.spring.io`.
   - Chọn:
     - Language: Java
     - Project: Maven
     - Spring Boot version: 3.1.2 (hoặc bản ổn định mới nhất)
   - Group: `com.eazybytes`
   - Artifact: `configserver`
   - Name: `configserver`
   - Description: “Config server for EazyBank Microservices”
   - Packaging: `jar`

4. **Thêm dependencies**
   - Thêm:
     - `Config Server` (dùng cho server cấu hình tập trung)
     - `Actuator` (giám sát, health, metrics, v.v.)
   - Lưu ý:
     - `Config Client` sẽ được dùng trong các microservice (`accounts`, `cards`, `loans`) để kết nối tới Config Server.
   - Mặc định có thêm `spring-boot-starter-test`.

5. **Phiên bản Spring Boot vs Spring Cloud**
   - Trong `pom.xml`:
     - Spring Boot: 3.1.2
     - Spring Cloud: 2022.0.3
   - Lý do: Spring Boot và Spring Cloud là hai dự án khác nhau, có hệ version riêng.
   - Khi dùng `start.spring.io`, trang sẽ tự map phiên bản Spring Boot với phiên bản Spring Cloud tương thích và thêm BOM tương ứng.
   - Nếu không dùng `start.spring.io`, có thể vào trang Spring Cloud để xem bảng mapping giữa phiên bản Spring Boot và Spring Cloud.

6. **Fun fact về tên phiên bản Spring Cloud**
   - Các tên như Dalston, Edgware, Finchley, Greenwich, Hoxton, Ilford, Jubilee, Kilburn…
   - Đây là tên các ga tàu điện ngầm / trạm trong thành phố London, sắp xếp theo thứ tự alphabet (d, e, f, g, h, i, j, k).

7. **Chuẩn bị dự án trong workspace**
   - Tải dự án `configserver` về.
   - Copy vào thư mục `v2-spring-cloud-config`.
   - Giải nén, xóa file `.zip`.
   - Mở `v2-spring-cloud-config` bằng IntelliJ.
   - Load tất cả các dự án Maven: `accounts`, `cards`, `configserver`, `loans`.

8. **Cấu hình Config Server**
   - Mở lớp main `ConfigServerApplication` (tên tương tự).
   - Thêm annotation: `@EnableConfigServer` để bật chức năng Config Server.
   - Đổi `application.properties` thành `application.yml` (vì khóa học dùng YAML).
   - Trong `application.yml`, cấu hình:
     ```yaml
     server:
       port: 8071
     ```
   - Cổng 8071 sẽ được dùng cố định cho Config Server trong suốt khóa học.

9. **Lưu trữ cấu hình tập trung**
   - Config Server hiện chưa biết đọc cấu hình từ đâu.
   - Cần di chuyển cấu hình của các microservice vào một nơi tập trung và cấu hình đường dẫn đó trong Config Server.
   - Các lựa chọn lưu trữ cấu hình:
     - Classpath của Config Server
     - File system (thư mục trên máy / server)
     - GitHub (phổ biến nhất)
     - Database
     - Cloud storage (ví dụ AWS S3)
   - Trong khóa học, sẽ trình bày 3 cách:
     1. Lưu trong classpath của Config Server.
     2. Lưu trong file system.
     3. Lưu trong GitHub.

Bài giảng kết thúc ở bước chuẩn bị Config Server (port 8071, @EnableConfigServer) và giới thiệu các cách lưu trữ cấu hình sẽ được hướng dẫn chi tiết ở các bài tiếp theo.
