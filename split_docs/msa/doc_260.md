
Dưới đây là tóm tắt nội dung bài giảng:

1. **Chuyển từ Eureka sang Kubernetes Discovery cho Gateway Server**
   - Mở `pom.xml` của Gateway Server.
   - Giữ lại (hoặc bỏ) dependency Eureka client, nhưng **thêm** dependency mới:  
     `spring-cloud-starter-kubernetes-discovery-client`.
   - Cập nhật thẻ version (từ `s14` lên `s17`), tải lại Maven và build lại dự án.

2. **Cập nhật GatewayserverApplication**
   - Mở lớp `GatewayserverApplication`.
   - Thêm annotation `@EnableDiscoveryClient` để bật discovery client.

3. **Thay đổi cấu hình route trong Gateway**
   - Trước đây URI dùng dạng:
     - `lb://ACCOUNTS`
     - `lb://LOANS`
     - `lb://CARDS`
   - `lb` = load balancer (Spring Cloud LoadBalancer), dùng cho client-side load balancing với Eureka.
   - Vì không dùng Eureka nữa, không muốn Gateway thực hiện client-side load balancing.
   - Cập nhật URI sang URL thực tế của từng microservice, ví dụ:
     - Accounts: `http://accounts:8080`
     - Loans: `http://loans:8090`
     - Cards: `http://cards:9000`
   - Lưu ý: không hard-code hostname cụ thể; chỉ cần service name (accounts, loans, cards) trùng với tên service trong Kubernetes thì Gateway sẽ hoạt động.

4. **Cập nhật `application.yml` của Gateway**
   - Xóa các thuộc tính liên quan đến Eureka, ví dụ:
     - `spring.cloud.discovery.enabled`
     - `spring.cloud.discovery.lower-case-service-id`
     - Và các thuộc tính `eureka.*` khác.
   - Thêm thuộc tính cho Kubernetes discovery:
     - `spring.cloud.kubernetes.discovery.enabled=true`
     - `spring.cloud.kubernetes.discovery.all-namespaces=true`
   - Ý nghĩa: yêu cầu Gateway sử dụng Kubernetes Discovery trong cluster để định tuyến đến các microservice (accounts, loans, cards).

5. **Xử lý lỗi liên quan đến Health Indicator**
   - Khi bật Kubernetes discovery, sẽ gặp lỗi do bean health indicator mặc định.
   - Để tránh lỗi, thêm thuộc tính:
     - `spring.cloud.discovery.client.health-indicator.enabled=false`
   - Lý do: mặc định Spring tạo một health indicator bean, nhưng thư viện Kubernetes discovery client không tạo bean cần thiết tương ứng, gây lỗi.
   - Thuộc tính này là workaround tạm thời; nếu sau này issue được fix trong Spring Cloud Kubernetes, có thể xóa thuộc tính này.

6. **Chuẩn bị build và triển khai**
   - Sẽ tạo Docker image cho tất cả microservice với tag `s17`.
   - Việc build image và setup cluster (Kubernetes) sẽ được thực hiện “ngầm”:
     - Thiết lập cluster với Kafka, Keycloak, Grafana, Prometheus.
   - Cập nhật Helm chart của Easy Bank với tag mới `s17`.
   - Dùng cùng Helm chart để triển khai toàn bộ microservice lên Kubernetes.

Tóm lại: bài giảng hướng dẫn chuyển Gateway Server từ Eureka sang Kubernetes Discovery, chỉnh lại dependency, annotation, route URI, cấu hình `application.yml`, thêm workaround cho health indicator, rồi chuẩn bị build Docker image và cập nhật Helm chart để triển khai lên Kubernetes.
