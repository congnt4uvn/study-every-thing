
Dưới đây là tóm tắt nội dung:

1. Tạo và xem cụm Kubernetes trên Google Cloud  
   - Sau khoảng 5 phút, cụm Kubernetes trên Google Cloud được tạo thành công (biểu tượng dấu tích xanh).  
   - Nhấp vào tên cụm để xem chi tiết: vùng, quốc gia, thông tin cấu hình.  
   - Xem danh sách node trong cụm, mỗi node hiển thị CPU, RAM, disk được gán.

2. Xem chi tiết node và pod  
   - Nhấp vào từng node để xem chi tiết: mức sử dụng CPU, RAM, disk.  
   - Cuộn xuống sẽ thấy danh sách các pod đang chạy trên node đó.  
   - Google Cloud tự động cài các pod hệ thống (như gke-metrics-agent) để thu thập metrics và quản lý cụm.

3. Kết nối cụm từ máy cục bộ  
   - Từ giao diện GKE, nhấp vào menu ba chấm → Connect để lấy lệnh gcloud.  
   - Dán và chạy lệnh trong terminal để cập nhật kubeconfig và kết nối tới cụm GKE.  
   - Dùng `kubectl get nodes` để xác nhận: thấy 3 node tương ứng với cụm trên cloud.

4. Quản lý nhiều cụm bằng context  
   - Trước đó đã kết nối cụm Kubernetes cục bộ (Docker Desktop).  
   - Thông qua context trong Kubernetes (xem trong Docker Desktop → Kubernetes), có thể chuyển giữa:  
     - Context Docker Desktop (cụm local, 1 node).  
     - Context GKE (cụm trên Google Cloud, 3 node).  
   - Khi đổi context và chạy `kubectl get nodes`, kết quả thay đổi theo cụm tương ứng.

5. Áp dụng manifest và Helm chart lên cụm GKE  
   - Điều hướng đến thư mục `section_17` (dùng lại code, chart, manifest cũ).  
   - Cài Discovery Server:  
     - Vào thư mục `kubernetes`.  
     - Chạy `kubectl apply -f kubernetes-discoveryserver.yaml`.  
     - Ban đầu bị lỗi timeout do mạng, chạy lại lệnh gcloud connect rồi apply thành công.

6. Cài đặt các Helm chart hạ tầng  
   - Vào thư mục `helm` và lần lượt cài:  
     - Keycloak: `helm install keycloak keycloak`  
     - Kafka: `helm install kafka kafka`  
     - Prometheus: `helm install prometheus kube-prometheus`  
     - Loki: `helm install loki grafana-loki`  
     - Tempo: `helm install tempo grafana-tempo`  
     - Grafana: `helm install grafana grafana`  
   - Tất cả được cài vào cụm Kubernetes trên cloud.

7. Triển khai ứng dụng EasyBank (microservices)  
   - Từ thư mục `helm`, chuyển sang thư mục `environments`.  
   - Cài đặt môi trường production:  
     - `helm install easybank prod-env` (tên release: easybank, chart: prod-env).  
   - Lệnh này triển khai toàn bộ các microservice của EasyBank lên cụm GKE.  
   - Tác giả nhấn mạnh quy trình giống hệt khi làm với cụm local, chỉ khác là cụm cloud có tài nguyên lớn hơn.

8. Kết luận  
   - Sau khi khởi động các deployment, cần chờ 5–10 phút để mọi thứ ổn định.  
   - Bài giảng tiếp theo sẽ kiểm tra và xác thực tất cả deployment có chạy thành công hay không.
