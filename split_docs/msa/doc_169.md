
Bài giảng mô tả cách tận dụng Grafana kết hợp Prometheus để giám sát hệ sinh thái Java (đặc biệt là các microservice Spring Boot):

1. **Sử dụng các dashboard có sẵn từ cộng đồng**
   - Truy cập trang Prometheus → tab Visualization → chọn Grafana → vào trang “Grafana Dashboards”.
   - Tại đây có rất nhiều dashboard được cộng đồng và team Grafana xây dựng sẵn cho nhiều hệ thống: Jira, MongoDB, Kubernetes, Datadog, v.v.
   - Tập trung vào hệ sinh thái Java:
     - Tìm kiếm “JVM”, chọn dashboard có lượt tải cao (ví dụ hơn 5 triệu lượt).
     - Sao chép URL endpoint của dashboard.
   - Vào Grafana:
     - Đăng nhập (mặc định: admin/admin, có thể bỏ qua bước đổi mật khẩu).
     - Vào “New” → “Import” → dán URL → chọn data source là Prometheus → Import.
   - Dashboard JVM Micrometer hiển thị nhiều thông tin:
     - Uptime, start time.
     - Heap / non-heap usage.
     - Thời lượng, lỗi, CPU usage, event logs.
     - GC pressure, chi tiết bộ nhớ JVM (Eden, Survivor, Tenured Gen).
   - Có thể chọn từng microservice (accounts, cards, …) để xem metric tương ứng.

2. **Import thêm dashboard Spring Boot**
   - Trên trang Grafana Dashboards, tìm “Spring Boot”.
   - Chọn dashboard “Spring Boot 2.1 System Monitor” (khoảng gần 500k lượt tải).
   - Sao chép URL endpoint, quay lại Grafana → Import → chọn Prometheus → Import.
   - Dashboard này hiển thị metric Spring Boot trong khoảng thời gian chọn (mặc định 2 giờ, có thể đổi 15 phút).
   - Có thể chọn microservice khác nhau để xem metric tương ứng.

3. **Tạo dashboard tùy chỉnh riêng**
   - Vào Grafana → “New dashboard”.
   - Lưu dashboard với tên, ví dụ: “easybank”, lưu trong thư mục General.
   - Thêm một “Row”:
     - Đặt tên row là “Accounts Microservice”.
   - Thêm panel đầu tiên (visualization):
     - Đặt tên panel: “Uptime”.
     - Data source: Prometheus.
     - Metric: `process_uptime_seconds`.
     - Lọc theo label `application = accounts-microservice` (hoặc tương đương).
     - Giữ kiểu biểu đồ mặc định “Time series”.
   - Thêm panel thứ hai:
     - Data source: Prometheus.
     - Metric: `up`.
     - Lọc theo label `job = accounts`.
     - Đặt tiêu đề panel: “Up”.
     - Chọn kiểu biểu đồ “Gauge” (thước đo) để thấy dịch vụ đang up (1) hay down (0).
   - Sắp xếp panel vào trong row “Accounts Microservice”.
   - Có thể thêm nhiều row và panel khác để tạo dashboard tùy chỉnh cho các microservice.

4. **Ý nghĩa và vai trò**
   - Grafana + Prometheus cho phép:
     - Giám sát chi tiết từng microservice (uptime, CPU, memory, GC, lỗi, …).
     - Tái sử dụng dashboard cộng đồng, tiết kiệm thời gian.
     - Tạo dashboard riêng phù hợp nhu cầu đội ngũ.
   - Thông thường, team platform/operations sẽ đi sâu hơn vào Grafana, còn developer nắm được mức cơ bản như trong bài là đủ.
   - Các dashboard này giúp bất kỳ thành viên nào trong team nhanh chóng nắm được tình trạng tổng thể của hệ thống microservice.

Kết luận: Bài giảng hướng dẫn cách import dashboard có sẵn (JVM, Spring Boot) và cách tự tạo dashboard trong Grafana để giám sát microservice Java/Spring Boot bằng Prometheus.
