
Dưới đây là tóm tắt nội dung bài giảng:

1. **Container là gì?**  
   - Container là một môi trường được cô lập (nhưng lỏng lẻo) bên trong máy chủ, máy ảo hoặc máy cục bộ.  
   - Trong container, ta đóng gói ứng dụng (ví dụ microservice) cùng toàn bộ thư viện và dependencies cần thiết.  
   - Nhờ vậy, container chạy nhất quán trên mọi môi trường (local, data center, cloud) mà không cần cài đặt thủ công thêm.  
   - Mọi thứ (code, thư viện, server cần thiết) đều nằm trong container.

2. **Image và Container (so sánh với Java):**  
   - Image (hình ảnh container/Docker image) là gói phần mềm chứa ứng dụng và dependencies.  
   - Từ một image, có thể tạo ra nhiều container.  
   - So sánh:  
     - Image giống như **class** trong Java.  
     - Container giống như **object** được tạo ra từ class.  
   - Container Docker là “phiên bản đang chạy” của Docker image.

3. **Container hóa phần mềm (Software Containerization):**  
   - Là phương pháp **ảo hóa ở cấp độ hệ điều hành (OS-level virtualization)**.  
   - Nhiều container có thể chạy trên cùng một máy hoặc máy ảo, chia sẻ chung kernel của hệ điều hành host.  
   - Mỗi container có cảm giác như đang chạy trên hệ điều hành riêng của nó.  
   - Khác với ảo hóa truyền thống (hypervisor/VM):  
     - VM: ảo hóa **phần cứng**, mỗi VM có hệ điều hành riêng.  
     - Container: ảo hóa **hệ điều hành**, nhiều container dùng chung kernel.

4. **Docker là gì?**  
   - Docker là một nền tảng / sản phẩm mã nguồn mở để:  
     - Chuyển đổi ứng dụng (web/app) thành Docker image.  
     - Từ image đó tạo và chạy container.  
     - Tự động hóa việc triển khai, scale, và quản lý ứng dụng.  
   - Mối quan hệ:  
     - **Containerization**: là khái niệm.  
     - **Docker**: là nền tảng triển khai khái niệm containerization.  

5. **Cách container hoạt động bên trong (Linux namespaces & cgroups):**  
   - Container dựa trên ảo hóa hệ điều hành: nhiều container chạy trên cùng một kernel.  
   - **Namespaces** (không gian tên):  
     - Tính năng của Linux tạo ra môi trường biệt lập trong cùng một OS.  
     - Mỗi container có namespace riêng cho: process, network, storage, user, v.v.  
     - Process trong một container chỉ “thấy” và tương tác với tài nguyên trong namespace của nó → tạo sự cô lập giữa các container.  
   - **cgroups (control groups)**:  
     - Dùng để **kiểm soát và giới hạn** việc sử dụng tài nguyên: CPU, RAM, disk, network.  
     - Ngăn một container chiếm hết tài nguyên, đảm bảo phân bổ công bằng giữa nhiều container.

6. **Docker trên Linux, macOS, Windows:**  
   - Trên **Linux**: Docker chạy trực tiếp trên hệ điều hành, dùng luôn các tính năng của Linux (namespaces, cgroups).  
   - Trên **macOS/Windows**:  
     - Cài đặt sẽ gồm:  
       - Docker client (CLI) trên hệ điều hành host (macOS/Windows).  
       - Một máy ảo Linux nhẹ chạy nền, bên trong đó có Docker server/daemon.  
     - Docker client trên macOS/Windows giao tiếp với Docker server chạy trong VM Linux.  
   - Khi chạy `docker version` trên máy:  
     - Phần **Client**: OS có thể là darwin/arm64 (macOS) hoặc Windows.  
     - Phần **Server**: OS là Linux (do chạy trong VM Linux).

7. **Kết luận:**  
   - Container: môi trường cô lập để chạy ứng dụng cùng dependencies.  
   - Containerization: ảo hóa hệ điều hành, nhiều container dùng chung kernel.  
   - Docker: nền tảng giúp tạo image, chạy container, và quản lý vòng đời container.  
   - Bên trong, Docker dựa vào namespaces (cô lập) và cgroups (quản lý tài nguyên) của Linux.
