
Dưới đây là tóm tắt ngắn gọn nội dung bạn cung cấp:

1. **Mục tiêu**  
   - Tích hợp dịch vụ cho vay (loans) và dịch vụ thẻ (cards) với Spring Cloud Config Server.
   - Đảm bảo các microservice lấy cấu hình từ Config Server thay vì giữ trong chính ứng dụng.

2. **Thay đổi cho microservice Loans**  
   - Trong `resources`:
     - Xóa các file cấu hình YAML không cần thiết (prod, qa, …).
     - Gộp cấu hình vào `application.yml`.
     - Thêm:
       - `spring.config.import` trỏ đến URL endpoint của Config Server.
       - `spring.application.name=loans` (trùng với tên file cấu hình trên Config Server, ví dụ `loans-prod.yml`).
       - Kích hoạt profile `prod` mặc định.
   - Trong `pom.xml`:
     - Thêm dependency `spring-cloud-starter-config`.
     - Cấu hình version Spring Cloud: `2022.0.3`.
     - Thêm phần `dependencyManagement` để quản lý version Spring Cloud.
   - Sau khi chỉnh sửa:
     - Build lại ứng dụng.
     - Chạy loans service trên cổng `8090`.
     - Dùng Postman gọi các API:
       - `/build-info` → trả về version `1.0` từ profile prod.
       - `/contact-info` → trả về thông tin liên hệ từ profile prod.
     - Kết luận: Loans đã tích hợp thành công với Config Server.

3. **Thay đổi cho microservice Cards**  
   - Trong `resources`:
     - Xóa các file YAML không cần thiết.
     - Trong `application.yml`:
       - `spring.application.name=cards` (trùng với tên file cấu hình trên Config Server).
       - Kích hoạt profile `prod`.
       - Thêm `spring.config.import` trỏ đến Config Server.
   - Trong `pom.xml`:
     - Thêm `spring-cloud-starter-config`.
     - Thêm version Spring Cloud và `dependencyManagement` tương tự loans.
   - Sau khi chỉnh sửa:
     - Build lại ứng dụng.
     - Chạy cards service trên cổng `9000`.
     - Dùng Postman:
       - `/build-info` → trả về version `1.2`.
       - `/contact-info` → trả về thông tin liên hệ từ profile prod.
     - Kết luận: Cards cũng tích hợp thành công với Config Server.

4. **Kết quả chung**  
   - Tất cả microservice hiện đang lấy cấu hình từ Spring Cloud Config Server.
   - Cấu hình đang được lưu trong classpath của Config Server (các file cấu hình nằm trong chính project Config Server).
   - Cách này phù hợp với một số dự án, nhưng có nhược điểm:
     - Ai có quyền truy cập mã Config Server sẽ thấy toàn bộ cấu hình (kể cả thông tin nhạy cảm).
   - Bài học tiếp theo sẽ hướng dẫn:
     - Cách tách cấu hình ra khỏi Config Server bằng cách lưu trong một vị trí hệ thống file riêng (file system), để tăng tính tách biệt và bảo mật.

Nếu bạn muốn, tôi có thể giúp bạn viết mẫu `application.yml` và `pom.xml` tương ứng cho loans và cards.
