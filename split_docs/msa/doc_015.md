
Dưới đây là tóm tắt ngắn gọn nội dung:

1. **Bối cảnh**  
   - Đã có CSDL H2 với 2 bảng: `customers` và `accounts`.  
   - Cấu trúc này sẽ được giữ nguyên khi chuyển sang MySQL.  
   - Mục tiêu: viết code Java (Spring Data JPA) để thao tác CRUD với các bảng.

2. **Tạo lớp siêu thực thể – `BaseEntity`**  
   - Tạo package `entity` và lớp `BaseEntity`.  
   - Lớp này chứa 4 trường “siêu dữ liệu” dùng chung cho mọi bảng:
     - `createdAt` (LocalDateTime)  
     - `createdBy` (String)  
     - `updatedAt` (LocalDateTime)  
     - `updatedBy` (String)  
   - Dùng Lombok:
     - `@Getter`, `@Setter`, `@ToString` để tự sinh getter, setter, toString.  
   - Dùng JPA:
     - `@MappedSuperclass` để báo cho Spring Data JPA đây là lớp cha cho các entity khác.  
     - Có thể dùng `@Column` nếu tên trường khác tên cột.  
     - Với `createdAt`, `createdBy`: `updatable = false` để không cập nhật khi update bản ghi.  
     - Với `updatedAt`, `updatedBy`: `insertable = false` để không set khi insert bản ghi mới.

3. **Tạo entity `Customer`**  
   - Lớp `Customer` trong package `entity`.  
   - Dùng `@Entity` (và `@Table` nếu tên bảng khác tên lớp).  
   - Trường dữ liệu:
     - `private long customerId;` (khóa chính)  
     - `private String name;`  
     - `private String email;`  
     - `private String mobileNumber;`  
   - Kế thừa `BaseEntity`.  
   - Lombok:
     - `@Getter`, `@Setter`, `@ToString`, `@AllArgsConstructor`, `@NoArgsConstructor`.  
   - JPA:
     - `@Id` trên `customerId`.  
     - `@GeneratedValue` + `@GenericGenerator` với `strategy = GenerationType.AUTO` và `strategy = "native"` để Spring tự sinh khóa chính theo kiểu native của DB.  
     - Có thể dùng `@Column(name = "customer_id")`, `@Column(name = "mobile_number")` nếu tên cột có gạch dưới.

4. **Tạo entity `Accounts`**  
   - Sao chép từ `Customer` và chỉnh sửa thành `Accounts`.  
   - Kế thừa `BaseEntity`.  
   - Trường chính:
     - `private long accountNumber;` (khóa chính, `@Id`)  
     - `private String accountType;`  
     - `private String branchAddress;`  
     - Có thêm `customerId` để liên kết với bảng `customers` (không phải khóa chính).  
   - Dùng `@Column(name = "account_number")`, `@Column(name = "account_type")`, `@Column(name = "branch_address")` nếu cần.  
   - Không dùng `@GeneratedValue` cho `accountNumber` vì số tài khoản sẽ được sinh thủ công trong service (thường là số 10 chữ số, không phải 1,2,3,…).

5. **Tạo các Repository**  
   - Tạo package `repository`: `com.easybytes.accounts.repository`.  

   **CustomerRepository**  
   - Interface `CustomerRepository`.  
   - Dùng `@Repository`.  
   - `extends JpaRepository<Customer, Long>`.  
   - Nhờ `JpaRepository`, tự động có các phương thức CRUD: `save`, `findById`, `findAll`, `delete`,… mà không cần viết SQL.

   **AccountsRepository**  
   - Interface `AccountsRepository`.  
   - `extends JpaRepository<Accounts, Long>`.  
   - Tương tự, dùng các phương thức có sẵn để thao tác với bảng `accounts`.

6. **Kết luận**  
   - Đã tạo:
     - Lớp siêu thực thể `BaseEntity` chứa các cột metadata dùng chung.  
     - Hai entity `Customer` và `Accounts` ánh xạ 2 bảng trong DB.  
     - Hai repository `CustomerRepository` và `AccountsRepository` để thao tác CRUD với DB thông qua Spring Data JPA.  
   - Nhờ Spring Data JPA + Lombok, giảm rất nhiều code boilerplate (getter/setter, constructor, SQL CRUD).
