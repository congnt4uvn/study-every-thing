
Dưới đây là tóm tắt nội dung:

1. Mục tiêu  
   - Đã phát triển Edge Server cho hệ thống microservices và test trong môi trường local.  
   - Bước tiếp theo: build Docker image cho tất cả project trong thư mục `section9`, cập nhật `docker-compose` và kiểm thử toàn bộ hệ thống bằng container.

2. Bật health / readiness / liveness cho microservices  
   - Cần bật các URL health, readiness, liveness cho 3 dịch vụ: `accounts`, `loans`, `cards`.  
   - Lý do: trong `docker-compose`, gateway/edge server chỉ nên khởi động sau khi 3 dịch vụ này “healthy”.  
   - Các thuộc tính health đã được định nghĩa sẵn trong Config Server (`application.yml` của config server).  
   - Thao tác:
     - Sao chép các thuộc tính health từ `application.yml` của Config Server.
     - Dán vào:
       - `application.yml` của `accounts` (phần `management` và `endpoints`).
       - `application.yml` của `cards`.
       - `application.yml` của `loans`.
     - Bật các probe liên quan đến health dưới phần `endpoints` (ví dụ: bật `health`, `liveness`, `readiness` = `true`).

3. Chuẩn bị trước khi build Docker  
   - Dừng tất cả instance đang chạy local.  
   - Mở `pom.xml` của từng microservice, đảm bảo tag version có hậu tố `S9` (để phân biệt image của Section 9).  
   - Kiểm tra Docker Desktop:
     - Không có image hoặc container cũ liên quan đến các microservice (eazybytes).

4. Build Docker image bằng Maven + Jib  
   - Mở terminal tại thư mục `section9`.  
   - Lần lượt vào từng project và chạy lệnh:
     - `accounts`:
       - `cd accounts`
       - `mvn compile jib:dockerBuild`
     - Quay lại thư mục cha, rồi:
     - `loans`:
       - `cd loans`
       - `mvn compile jib:dockerBuild`
     - `cards`:
       - `cd cards`
       - `mvn compile jib:dockerBuild`
     - `configserver`:
       - `cd configserver`
       - `mvn compile jib:dockerBuild`
     - `eurekaserver`:
       - `cd eurekaserver`
       - `mvn compile jib:dockerBuild`
     - `gateway/edge server`:
       - `cd gateway` (hoặc tên tương ứng)
       - `mvn compile jib:dockerBuild`
   - Tổng cộng tạo 6 Docker image.  
   - Jib giúp build image nhanh hơn so với Buildpacks.

5. Kiểm tra image đã tạo  
   - Chạy `docker images` để xác nhận có 6 image với tag `S9`.  
   - Có thể kiểm tra thêm trong Docker Desktop (tab Images, lọc theo `S9`).

6. Đẩy image lên Docker Hub  
   - Cú pháp:
     - `docker image push <docker-username>/<image-name>:S9`
   - Ví dụ:
     - `docker image push eazybytes/accounts:S9`
   - Thực hiện tương tự cho các image còn lại (loans, cards, configserver, eurekaserver, gateway).  
   - Khuyến khích thường xuyên build và push image theo từng phần (section) để dễ kiểm thử các thay đổi sau này.

Nội dung chính: bật health endpoints cho các microservice, build Docker image bằng Maven + Jib cho tất cả dịch vụ trong Section 9, kiểm tra image, và đẩy lên Docker Hub để phục vụ cho việc chạy hệ thống bằng Docker Compose.
