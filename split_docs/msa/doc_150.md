
Dưới đây là tóm tắt nội dung:

1. **Thêm phụ thuộc Redis Reactive**  
   - Thêm dependency `spring-boot-starter-data-redis-reactive` vào dự án Gateway để dùng Redis làm backend cho RateLimiter.  
   - Chạy lại Maven để tải dependency.

2. **Tạo 2 bean trong GatewayserverApplication**  
   - **Bean KeyResolver (userKeyResolver)**  
     - Lấy giá trị header `username` từ request.  
     - Nếu không có, gán giá trị mặc định là `"anonymous"`.  
     - Đây là “key” mà RateLimiter sẽ dùng để giới hạn theo từng người dùng.  
   - **Bean RedisRateLimiter**  
     - Khởi tạo `new RedisRateLimiter(replenishRate, burstCapacity, requestedTokens)`  
     - Ở ví dụ: mỗi giây thêm 1 token, burst capacity = 1, mỗi request tốn 1 token.  
     - Kết quả: mỗi user chỉ được 1 request/giây.

3. **Cấu hình Route để dùng RequestRateLimiter**  
   - Trong cấu hình route của Gateway (cho service Cards), thêm filter `RequestRateLimiter`.  
   - Dùng lambda để:  
     - `setRateLimiter()` → truyền bean `RedisRateLimiter`.  
     - `setKeyResolver()` → truyền bean `userKeyResolver`.  
   - Như vậy, tất cả API của service Cards qua Gateway sẽ bị giới hạn theo cấu hình này.

4. **Khởi động Redis bằng Docker**  
   - Đảm bảo Docker daemon đang chạy.  
   - Chạy lệnh:  
     `docker run -p 6379:6379 --name eazyredis -d redis`  
   - Redis chạy nền, Gateway sẽ dùng Redis để lưu bucket/token cho từng key (username).

5. **Cấu hình Redis trong application.yml của Gateway**  
   - Thêm cấu hình dưới `spring:` → `data:` → `redis:`  
     - `host` (server)  
     - `port` (thường 6379)  
     - `timeout`  
     - `connect-timeout`  
   - Lưu file, build lại, rồi khởi động Gatewayserver.

6. **Chuẩn bị môi trường chạy demo**  
   - Đang chạy: Config Server, Eureka Server, CardsApplication.  
   - Không cần Accounts và Loans cho demo này.  
   - Khởi động Gatewayserver.

7. **Kiểm thử RateLimiter bằng Apache Benchmark (ab)**  
   - Cài đặt Apache Benchmark (ab).  
   - Chạy lệnh ví dụ:  
     `ab -n 10 -c 2 -v 3 http://gateway-url/cards/contact-info`  
     - `-n 10`: gửi 10 request.  
     - `-c 2`: 2 request đồng thời.  
     - `-v 3`: log chi tiết.  
   - Kết quả:  
     - 10 request hoàn tất trong ~0.5 giây.  
     - 1 request thành công (HTTP 200).  
     - 9 request bị từ chối với HTTP 429 (Too Many Requests).  
     - Điều này chứng minh RateLimiter hoạt động đúng: chỉ 1 request/giây cho mỗi key (user).

8. **Giải thích thêm về KeyResolver**  
   - Vì không gửi header `username`, key mặc định là `"anonymous"`.  
   - Tất cả request trong demo đều bị xem là cùng một user (`anonymous`), nên bị giới hạn chung.  
   - Trong thực tế, có thể thay đổi logic KeyResolver để giới hạn theo user thực, IP, API key, v.v.

Kết luận: Đã triển khai thành công RateLimiter trong Spring Cloud Gateway sử dụng Redis, cấu hình giới hạn 1 request/giây cho mỗi user key, và kiểm chứng bằng Apache Benchmark với kết quả 200 cho request đầu tiên và 429 cho các request vượt hạn mức.
