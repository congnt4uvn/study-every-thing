
Dưới đây là tóm tắt ngắn gọn nội dung:

1. **@RestController vs @Controller + @ResponseBody**  
   - Khi xây dựng REST API với các annotation như `@GetMapping`, `@PostMapping`, nên dùng `@RestController` trên lớp để Spring Boot tự hiểu và expose các method thành REST API.  
   - Nếu trong cùng một controller vừa có REST API vừa có chức năng Spring MVC (trả về HTML, view…), thì nên dùng `@Controller`.  
   - Khi đó, với các method trả JSON, cần thêm `@ResponseBody` để báo cho Spring trả về dữ liệu (JSON) thay vì view.  
   - `@RestController` thực chất là kết hợp của `@Controller` + `@ResponseBody`.

2. **@ResponseBody**  
   - Dùng khi lớp dùng `@Controller` nhưng method cần trả về JSON (hoặc dữ liệu) thay vì HTML/UI.  
   - Trong ví dụ microservice, vì chỉ làm REST API nên dùng `@RestController` và không cần `@ResponseBody` riêng lẻ.

3. **ResponseEntity**  
   - Là một lớp (không phải annotation) của Spring, dùng để trả về đầy đủ **status code**, **headers** và **body**.  
   - Là generic type: `ResponseEntity<CustomerDto>`, `ResponseEntity<String>`,…  
   - Nên dùng khi muốn kiểm soát chi tiết response gửi về client.

4. **@ControllerAdvice và @ExceptionHandler**  
   - `@ControllerAdvice` dùng để định nghĩa **xử lý ngoại lệ toàn cục** cho các controller.  
   - Bên trong lớp có `@ControllerAdvice`, các method đánh dấu `@ExceptionHandler(SomeException.class)` sẽ được gọi khi exception tương ứng xảy ra trong controller.  
   - `@RestControllerAdvice` là kết hợp của `@ControllerAdvice` + `@ResponseBody`, tương tự quan hệ giữa `@RestController` và `@Controller`.  
   - Trong ví dụ, tác giả dùng `@ControllerAdvice` và vẫn hoạt động tốt; nếu muốn chắc chắn trả JSON cho exception thì có thể dùng `@RestControllerAdvice`.

5. **RequestEntity**  
   - Tương tự `ResponseEntity` nhưng cho **request**.  
   - Dùng khi muốn nhận cả **request body** và **request headers** trong một tham số method.  
   - Trong microservice ví dụ chưa dùng vì không có nhu cầu lấy headers, chỉ cần body.

6. **@RequestHeader và @RequestBody**  
   - `@RequestHeader`: lấy giá trị header từ request làm tham số method.  
   - `@RequestBody`: lấy toàn bộ nội dung body của request (thường là JSON) và map vào object/DTO.

7. **Cách trình bày khi đi phỏng vấn / xây REST API**  
   - Giải thích theo trình tự:  
     1) Tạo lớp controller, gắn `@RestController` (hoặc `@Controller` + `@ResponseBody`).  
     2) Định nghĩa các method với `@GetMapping`, `@PostMapping`,…  
     3) Dùng `@RequestBody`, `@RequestHeader`, `RequestEntity` để nhận dữ liệu từ client.  
     4) Dùng `ResponseEntity` để trả về status, headers, body.  
     5) Định nghĩa xử lý lỗi toàn cục với `@ControllerAdvice` / `@RestControllerAdvice` và `@ExceptionHandler`.  

Các annotation và lớp này là “bộ công cụ” quan trọng khi xây dựng REST API/Microservices với Spring Boot.
