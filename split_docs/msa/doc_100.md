
Bài giảng trên nói về việc:

1. Cập nhật Docker Compose:
   - Đổi tag image của tất cả microservice từ `S6` sang `S7` vì đã build image mới.
   - Xóa dependency `rabbit` trong `config-server` vì không còn dùng Spring Cloud Bus, nếu không Docker Compose sẽ báo lỗi “depends_on service undefined”.

2. Chạy Docker Compose:
   - Điều hướng vào thư mục `docker-compose/default`.
   - Chạy `docker compose up` (lần đầu không dùng `-d` để xem log).
   - Thấy các container database khởi tạo trước, sau đó đến các microservice (accounts, loans, cards).
   - Giải thích lợi ích chạy ở chế độ detached (`-d`) để terminal rảnh cho lệnh khác.

3. Lỗi kết nối database của loans và cards:
   - Trong log của microservice loans/cards báo không kết nối được DB.
   - Nguyên nhân: trong cấu hình microservice dùng port 3307/3308, trong khi các container nằm trong cùng mạng Docker, phải dùng port nội bộ 3306 (port mà MySQL lắng nghe trong container), không phải port đã publish ra ngoài.
   - Giải thích:
     - Các container giao tiếp với nhau qua mạng nội bộ Docker, không cần port mapping.
     - Port mapping (3306, 3307, 3308) chỉ cần khi muốn truy cập DB từ bên ngoài (ví dụ SQL Electron).
   - Kết luận:
     - Trong cấu hình microservice: dùng port 3306 cho các DB.
     - Có thể bỏ port mapping nếu chỉ giao tiếp nội bộ; giữ lại nếu cần truy cập từ ngoài để kiểm tra dữ liệu.

4. Dừng và tạo lại container:
   - Dùng `Ctrl + C` chỉ dừng, không xóa container.
   - Dùng `docker compose down` để remove container.
   - Chạy lại `docker compose up -d` để khởi động tất cả dịch vụ ở chế độ detached.

5. Tạo file Docker Compose cho môi trường prod và qa:
   - Copy toàn bộ nội dung file `docker-compose` default sang file `docker-compose` cho `prod` và `qa`.
   - Mở `common-config.yml`:
     - Copy nội dung profile `default` sang profile `prod`, đổi `spring.profiles.active` thành `prod`.
     - Tương tự cho `qa`, đổi `spring.profiles.active` thành `qa`.

6. Xác thực các container và API:
   - Trong Docker Desktop: thấy 7 container (DBs, config-server, accounts, loans, cards) đều chạy.
   - Kiểm tra log của cards và loans: sau khi sửa port, kết nối DB thành công, ứng dụng khởi động.
   - Test REST API:
     - Gọi API tạo account → thành công.
     - Gọi API tạo card → thành công.
     - Gọi API tạo loan → thành công.

7. Kiểm tra dữ liệu bằng SQL Electron:
   - Kết nối đến các DB (accounts, customers, loans, cards) qua các port đã publish (3306/3307/3308).
   - Mở các bảng và xác nhận dữ liệu đã được lưu sau khi gọi API.

8. Trường hợp dùng DB bên ngoài Docker:
   - Nếu dự án dùng DB dev hoặc DB trên cloud (không nằm trong Docker/local):
     - Không cần định nghĩa các container DB trong Docker Compose.
     - Vào từng microservice, cấu hình trực tiếp `datasource URL` trỏ đến host/IP/public endpoint của DB ngoài.
   - Khi đó microservice sẽ kết nối trực tiếp DB bên ngoài mà không cần container DB.

Tóm lại, bài giảng tập trung vào:
- Cập nhật image/tag đúng trong Docker Compose.
- Hiểu cơ chế mạng Docker (port nội bộ vs port publish).
- Cách sửa lỗi kết nối DB giữa microservice và MySQL trong Docker.
- Tạo cấu hình cho nhiều môi trường (default, prod, qa).
- Xác thực end-to-end: API → DB (qua SQL Electron).
