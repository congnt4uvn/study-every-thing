
Bài giảng nói về việc cấu hình Spring Cloud Config Server để đọc cấu hình từ **hệ thống tệp (file system)** thay vì **classpath**.

Nội dung chính:

1. **Lý do dùng file system**  
   - Toàn bộ file cấu hình (các file .yml/.properties) được đặt ngay trên máy chủ nơi deploy microservice.  
   - Quản trị viên máy chủ có thể áp dụng các chính sách bảo mật: chỉ Config Server được phép truy cập thư mục đó, người khác không xem được nội dung.

2. **Chuẩn bị thư mục cấu hình**  
   - Tác giả copy tất cả file cấu hình từ thư mục `config` của project sang một thư mục trên máy:  
     - Ví dụ trên macOS: `users/eazybytes/documents/config`  
   - Thư mục này chứa các file như `loans-prod.yml`, `accounts-prod.yml`, `cards-prod.yml`,...

3. **Cấu hình lại Config Server**  
   - Mở `application.yml` của Spring Cloud Config Server.  
   - Trước đó `spring.cloud.config.server.native.search-locations` (hoặc tương tự) đang dùng `classpath:`.  
   - Thay bằng cấu hình kiểu `file:` trỏ tới thư mục trên hệ thống tệp.  
   - Cú pháp:  
     - Trên macOS/Linux:  
       `file:///users/eazybytes/documents/config`  
       (sau `file:` là `///`, rồi đến đường dẫn, giữa các thư mục dùng `/`).  
     - Trên Windows:  
       Ví dụ: `file:///C:/config` (ổ C, thư mục `config`).

4. **Khởi động lại hệ thống**  
   - Dừng tất cả microservice và Config Server.  
   - Build lại, rồi khởi động **Config Server trước**, sau đó lần lượt khởi động các microservice: accounts, loans, cards.

5. **Kiểm tra hoạt động**  
   - Gọi URL của Config Server để lấy cấu hình, ví dụ profile `prod` của loans.  
   - Thấy đường dẫn file được log là:  
     `user/eazybytes/documents/config/loans-prod.yml`  
     → xác nhận Config Server đang đọc từ file system.  
   - Kiểm tra thêm với services khác (cards, accounts) để chắc chắn tất cả đều lấy cấu hình đúng.  
   - Gọi một API của cards (ví dụ `/contact-info`) và thấy các thuộc tính `prod` được trả về, chứng tỏ microservice đã tích hợp đúng với Config Server.

6. **Kết luận**  
   - Thay đổi duy nhất: trong Config Server, đổi `search-locations` từ `classpath:` sang `file:` + đường dẫn thư mục.  
   - Profile dùng cho Config Server là profile gốc (default) của Spring Cloud Config Server.  
   - Nhờ vậy, Config Server đọc cấu hình từ hệ thống tệp thay vì từ classpath.
