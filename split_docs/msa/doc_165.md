
Bài giảng mô tả cách tích hợp Micrometer và Prometheus vào hệ thống microservices để thu thập và giám sát metrics:

1. **Thêm dependency Micrometer–Prometheus**
   - Mở `pom.xml` của từng microservice (account, loans, cards, gateway, Eureka, config server…).
   - Thêm dependency:
     - `groupId`: `io.micrometer`
     - `artifactId`: `micrometer-registry-prometheus`
   - Dependency này giúp Micrometer xuất các metrics actuator dưới định dạng Prometheus hiểu được.  
   - Nếu sau này đổi sang hệ thống giám sát khác, chỉ cần đổi `artifactId`.

2. **Cấu hình trong `application.yml`**
   - Trong mỗi service, thêm cấu hình dưới `management`:
     - `management.metrics.tags.application: ${spring.application.name}`
   - `spring.application.name` (ví dụ: `accounts`) sẽ được gán làm tag `application`.
   - Mục đích: giúp Micrometer/Prometheus nhóm và phân biệt metrics theo từng microservice.  
     Nếu không có tag này, khó biết metric thuộc service nào.

3. **Build và khởi động hệ thống**
   - Thực hiện `clean build`.
   - Khởi động lần lượt:
     1. Config Server (cổng 8071)
     2. Eureka Server (cổng 8070)
     3. Các microservice: Accounts (8080), Loans (8090), Cards (9000)
     4. Gateway Server (8072)

4. **Kiểm tra metrics qua Actuator**
   - Truy cập endpoint actuator của từng service:
     - Ví dụ: `http://localhost:8080/actuator/metrics`
   - Có thể xem chi tiết từng metric:
     - Ví dụ: `http://localhost:8080/actuator/metrics/system.cpu.usage`
     - Hoặc `http://localhost:8080/actuator/metrics/process.uptime`
   - Thấy được giá trị như mức sử dụng CPU, thời gian uptime, v.v.

5. **Expose metrics cho Prometheus**
   - Khi thêm dependency Micrometer–Prometheus, mỗi service tự động có endpoint:
     - `/actuator/prometheus`
   - Ví dụ:
     - `http://localhost:8080/actuator/prometheus` (accounts)
     - `http://localhost:8090/actuator/prometheus` (loans)
     - `http://localhost:9000/actuator/prometheus` (cards)
     - Tương tự cho Eureka, Config Server, Gateway…
   - Endpoint này trả về metrics ở định dạng Prometheus.
   - Prometheus sẽ định kỳ (5s, 10s, 1 phút… tùy cấu hình) gọi các URL này để “scrape” metrics từ tất cả instances.

6. **Ý nghĩa tổng thể**
   - Không cần viết thêm nhiều logic Java; chỉ cần:
     - Thêm dependency Micrometer–Prometheus.
     - Thêm cấu hình tag `application`.
   - Kết quả: toàn bộ microservices đều xuất metrics actuator ở định dạng Prometheus, giúp hệ thống giám sát tập trung và phân biệt rõ metrics của từng service.
