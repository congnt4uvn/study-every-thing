
Dưới đây là tóm tắt nội dung bài giảng:

1. Dọn dẹp cấu hình cũ trong microservice Accounts  
   - Xóa các file YAML riêng cho `prod` và `qa` trong microservice tài khoản vì giờ đã đưa cấu hình lên Config Server.  
   - Xóa các thuộc tính liên quan đến profile Spring Boot và kích hoạt profile trong `application.yml`, chỉ giữ lại các thuộc tính cần thiết (như cổng server, cấu hình datasource…).

2. Đặt tên ứng dụng (application name)  
   - Trong `application.yml` của microservice Accounts, thêm:
     - `spring.application.name: accounts`  
   - Tên này phải trùng với tên file cấu hình trên Config Server (`accounts.yml`, `accounts-prod.yml`, `accounts-qa.yml`), để Config Server biết phải trả về bộ cấu hình nào.

3. Cấu hình profile mặc định  
   - Trong `application.yml`, thêm:
     - `spring.profiles.active: prod`  
   - Nghĩa là khi microservice Accounts khởi động, mặc định sẽ dùng profile `prod`.  
   - Có thể override giá trị này từ bên ngoài (command line args, biến môi trường, JVM system properties).

4. Thêm dependency Spring Cloud Config Client  
   - Mở `pom.xml` của microservice Accounts.  
   - Thêm dependency Config Client (từ start.spring.io, dependency “Config Client”).  
   - Thêm thuộc tính phiên bản Spring Cloud (ví dụ `spring-cloud.version`) trong phần `<properties>`.  
   - Thêm phần `dependencyManagement` cho Spring Cloud (groupId, artifactId, version).  
   - Đảm bảo plugin liên quan được cấu hình đúng.  
   - Sau đó chạy “Load Maven Changes” để tải các dependency.

5. Cấu hình kết nối tới Config Server  
   - Trong `application.yml` của Accounts, thêm:
     - `spring.config.import: optional:configserver:http://localhost:8071`  
   - `configserver:` là prefix để Spring hiểu đây là Config Server.  
   - `optional:` cho phép ứng dụng vẫn khởi động nếu không kết nối được Config Server (chỉ log cảnh báo).  
   - Nếu cấu hình là “sống còn” với ứng dụng, có thể bỏ `optional:` để bắt buộc kết nối.

6. Khởi động và kiểm tra với profile `prod`  
   - Khởi động microservice Accounts (port 8080).  
   - Trong log console, thấy Spring báo profile đang kích hoạt là `prod` và đang kết nối Config Server để tải cấu hình tương ứng (`accounts-prod.yml`).  
   - Dùng Postman gọi các REST API:
     - Endpoint trả thông tin contact/product: thấy dữ liệu lấy từ cấu hình `prod`.  
     - Endpoint `build-info`: trả về version `1.0` như cấu hình trong `accounts-prod.yml`.  
     - Thông tin JAVA_HOME vẫn lấy từ biến môi trường hệ thống, không liên quan Config Server.

7. Thử kích hoạt profile `qa` từ bên ngoài  
   - Trong IDE (Run/Debug Configurations), thêm program argument:
     - `--spring.profiles.active=qa`  
   - Dừng ứng dụng, khởi động lại.  
   - Lần này ứng dụng sẽ tải cấu hình từ `accounts-qa.yml` trên Config Server.  
   - Kiểm tra lại:
     - `build-info` trả về version `2.0`.  
     - Endpoint contact trả về thông tin contact của profile `qa`.  
   - Điều này chứng minh microservice Accounts đã tích hợp thành công với Config Server và lấy cấu hình theo profile.

8. Bước tiếp theo  
   - Áp dụng các thay đổi tương tự cho microservice Cards và Loans.  
   - Đảm bảo chúng cũng kết nối được với Config Server và kiểm tra các REST API tương ứng.  
   - Trong bài giảng tiếp theo, giảng viên sẽ lướt nhanh phần thay đổi code cho Loans và Cards để người học nắm rõ.
