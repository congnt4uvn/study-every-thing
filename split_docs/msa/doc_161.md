
Dưới đây là tóm tắt ngắn gọn nội dung:

1. Chuẩn bị workspace  
   - Sao chép mã từ phần trước sang thư mục mới, đổi tên thành `section-11`.  
   - Mở project trong IntelliJ, bật annotation processing.  
   - Mở tất cả `pom.xml` của các microservice và đổi tag Docker image từ `s10` sang `s11`.  

2. Điều chỉnh cấu hình gateway  
   - Mở `application.yml` của API Gateway.  
   - Tăng `response-timeout` từ 2 giây lên 10 giây để tránh lỗi timeout khi chạy nhiều container trên máy local (tài nguyên hạn chế).  
   - Lưu ý: nếu sau này vẫn bị timeout, có thể tiếp tục tăng thời gian theo nhu cầu test.

3. Giới thiệu Grafana + Loki + Promtail  
   - Tham chiếu tài liệu chính thức của Grafana.  
   - Yêu cầu: đã cài Docker và Docker Compose (đã làm ở phần trước).  
   - Tài liệu hướng dẫn tạo thư mục riêng và tải 3 file YAML:
     - `docker-compose.yml`
     - `promtail-local-config.yaml`
     - `loki-config.yaml`  
   - Trong khóa học, thay vì tạo thư mục mới, sẽ đặt các file này trong thư mục `docker-compose` của workspace microservices.

4. Phân tích `docker-compose.yml` mẫu của Grafana/Loki  
   - Định nghĩa network `loki` (tương tự network `EazyBank` đã dùng trước đó).  
   - Các service chính:
     - `read` (Loki read component)  
       - Image Loki, chạy với command `-target=read`.  
       - Expose cổng 3101.  
       - Dùng `volumes` để mount file `loki-config.yaml` từ host vào container tại `/etc/loki/config.yaml`.  
       - Có `healthcheck`, `depends_on` minio, gắn vào network `loki` với alias `loki`.  
       - Dùng anchor `&loki-dns` để lưu cấu hình network vào một biến YAML.

     - `write` (Loki write component)  
       - Cùng image Loki, command `-target=write`.  
       - Expose cổng 3102.  
       - Dùng `<<: *loki-dns` (merge key) để tái sử dụng cấu hình network đã lưu trong anchor `&loki-dns`.  

     - `promtail`  
       - Image promtail.  
       - Mount file `promtail-local-config.yaml` từ host vào `/etc/promtail/config.yml` trong container (read-only `:ro`).  
       - Command sử dụng file `config.yml` này.  
       - `depends_on` gateway, dùng network `loki`.  

     - `minio`  
       - Dùng làm storage backend cho Loki.  
       - Có `entrypoint` tạo thư mục `data/loki-data`, `data/loki-ruler` trên host.  
       - Dùng `volumes` để mount thư mục `.data/minio` trên host vào `/data` trong container, nơi lưu log thực tế.  
       - Có biến môi trường, cổng, healthcheck, network `loki`.  

     - `grafana`  
       - Image Grafana.  
       - Biến môi trường, `depends_on` gateway.  
       - `entrypoint` tạo thư mục `provisioning/datasources` và file `ds.yml` để cấu hình datasource kiểu `loki`.  
       - Trong datasource, URL trỏ tới Loki (cổng 3100).  
       - Expose cổng 3000, healthcheck, network `loki`.  

     - `gateway` (Nginx)  
       - Image Nginx, `depends_on` `read` và `write`.  
       - `entrypoint` tạo file cấu hình Nginx để định tuyến:
         - Một số path chuyển tới Loki write (cổng 3100).  
         - Một số path chuyển tới Loki read.  
       - Cấu hình cổng, healthcheck, network.  

     - `flog`  
       - Ứng dụng mẫu tạo log ngẫu nhiên.  
       - Trong bài học này không dùng vì sẽ dùng chính các microservice để tạo log, nên không cần copy service này vào file `docker-compose` của dự án.

   - Giải thích YAML:
     - `&name` là anchor: tạo “biến” trong YAML.  
     - `*name` là alias: tham chiếu lại anchor.  
     - `<<: *anchor` là merge: nhập toàn bộ nội dung anchor vào chỗ đó.

5. Phân tích `promtail-local-config.yaml`  
   - Cấu hình promtail:
     - HTTP lắng nghe cổng 9080, gRPC cổng 0.  
     - `clients`: gửi log tới Loki qua URL cổng 3100.  
     - `scrape_configs`:  
       - `job_name` (ví dụ `flog_scrap`, có thể đổi tùy ý).  
       - Đọc log từ các container Docker thông qua socket.  
       - `scrape_interval` 5 giây.  
       - Dùng regex để đọc mọi loại log, gán nhãn `container` cho log.  
   - Đây là phần thường do DevOps/Platform team quản lý, nhưng hiểu cơ bản giúp tự triển khai hoặc hỗ trợ team.

6. Phân tích `loki-config.yaml`  
   - Loki lắng nghe cổng 3100.  
   - Cấu hình storage:
     - Sử dụng Minio (cổng 9000) làm backend.  
     - Định nghĩa bucket `loki-data`, key, secret, v.v.  
   - Trong môi trường thực tế có thể thay Minio bằng S3 hoặc dịch vụ storage cloud khác bằng cách chỉnh file này theo tài liệu Loki.

7. Kết luận  
   - Đã giải thích ý nghĩa 3 file YAML: `docker-compose.yml`, `promtail-local-config.yaml`, `loki-config.yaml`.  
   - Bước tiếp theo (ở bài giảng sau): tích hợp các cấu hình này vào file `docker-compose` của dự án microservices hiện tại để triển khai Grafana + Loki + Promtail cùng hệ thống.
