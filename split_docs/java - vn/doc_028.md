## 1. Wrapping Exceptions (Bọc ngoại lệ)

**Ý chính**  
- Khi gọi thư viện hoặc tầng thấp hơn, nên **bọc (wrap)** ngoại lệ gốc vào ngoại lệ có ý nghĩa hơn cho tầng hiện tại.  
- Giúp:  
  - Không mất thông tin gốc (stack trace, message).  
  - Giao diện API rõ ràng, dễ hiểu hơn.

**Ví dụ**

```java
class UserNotFoundException extends RuntimeException {
    public UserNotFoundException(String message, Throwable cause) {
        super(message, cause); // giữ nguyên nguyên nhân gốc
    }
}

class UserRepository {
    public String findUserNameById(int id) throws SQLException {
        // Giả sử lỗi DB
        throw new SQLException("Database connection failed");
    }
}

class UserService {
    private final UserRepository repo = new UserRepository();

    public String getUserName(int id) {
        try {
            return repo.findUserNameById(id);
        } catch (SQLException e) {
            // Wrap ngoại lệ SQL thành ngoại lệ domain
            throw new UserNotFoundException("Không tìm thấy user với id = " + id, e);
        }
    }
}

public class Main {
    public static void main(String[] args) {
        UserService service = new UserService();
        service.getUserName(1);
    }
}
```

**Output (stack trace rút gọn)**

```
Exception in thread "main" UserNotFoundException: Không tìm thấy user với id = 1
    at UserService.getUserName(UserService.java:...)
Caused by: java.sql.SQLException: Database connection failed
    at UserRepository.findUserNameById(UserRepository.java:...)
```

## 2. Custom Exceptions (Ngoại lệ tùy chỉnh)

**Ý chính**  
- Tạo ngoại lệ riêng để:  
  - Diễn tả rõ ràng lỗi thuộc domain (business logic).  
  - Phân biệt các loại lỗi khác nhau.  
- Đặt tên rõ ràng: `InvalidOrderException`, `InsufficientBalanceException`, v.v.

**Ví dụ**

```java
class InsufficientBalanceException extends Exception {
    public InsufficientBalanceException(String message) {
        super(message);
    }
}

class BankAccount {
    private double balance;

    public BankAccount(double balance) {
        this.balance = balance;
    }

    public void withdraw(double amount) throws InsufficientBalanceException {
        if (amount > balance) {
            throw new InsufficientBalanceException(
                "Số dư không đủ. Số dư hiện tại: " + balance + ", yêu cầu: " + amount
            );
        }
        balance -= amount;
    }

    public double getBalance() {
        return balance;
    }
}

public class Main2 {
    public static void main(String[] args) {
        BankAccount account = new BankAccount(100.0);
        try {
            account.withdraw(150.0);
        } catch (InsufficientBalanceException e) {
            System.out.println("Lỗi: " + e.getMessage());
        }
        System.out.println("Số dư cuối: " + account.getBalance());
    }
}
```

**Input/Output**

- Input logic: tạo tài khoản với 100, rút 150.  
- Output:

```
Lỗi: Số dư không đủ. Số dư hiện tại: 100.0, yêu cầu: 150.0
Số dư cuối: 100.0
```

## 3. Avoid Swallowing Exceptions (Tránh nuốt ngoại lệ)

**Ý chính**  
- “Nuốt” ngoại lệ = bắt ngoại lệ nhưng **không làm gì** (hoặc chỉ comment).  
- Hậu quả:  
  - Lỗi bị che giấu, khó debug.  
  - Hệ thống có thể ở trạng thái không nhất quán.  
- Luôn:  
  - Log lại ngoại lệ, hoặc  
  - Ném lại (rethrow), hoặc  
  - Chuyển thành ngoại lệ khác có ý nghĩa hơn.

**Ví dụ xấu (nuốt ngoại lệ)**

```java
try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    // Không làm gì cả -> nuốt ngoại lệ
}
System.out.println("Tiếp tục chạy như không có gì xảy ra...");
```

- Lỗi chia cho 0 bị bỏ qua, chương trình tiếp tục chạy, có thể gây lỗi logic.

**Ví dụ tốt**

```java
try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    // Log và ném lại
    System.err.println("Lỗi toán học: " + e.getMessage());
    throw e; // hoặc throw new RuntimeException("Lỗi tính toán", e);
}
```

**Output**

```
Lỗi toán học: / by zero
Exception in thread "main" java.lang.ArithmeticException: / by zero
    at ...
```

## Tóm tắt ngắn

1. **Wrapping**: Bọc ngoại lệ thấp (SQL, IO, v.v.) thành ngoại lệ có ý nghĩa với tầng hiện tại, luôn giữ `cause`.
2. **Custom Exceptions**: Tạo ngoại lệ riêng cho các lỗi domain, đặt tên rõ ràng, giúp API dễ hiểu và dễ xử lý.
3. **Avoid Swallow**: Không được bắt ngoại lệ rồi bỏ qua; luôn log, xử lý, hoặc ném lại để không che giấu lỗi.
