
Dưới đây là tóm tắt ngắn gọn về **Exceptions & Error Handling: Checked vs Unchecked exceptions** bằng tiếng Việt, kèm ví dụ mã và input/output.

---

## 1. Khái niệm chung

- **Exception**: Lỗi xảy ra trong lúc chương trình đang chạy (runtime), ví dụ: chia cho 0, truy cập mảng ngoài phạm vi, file không tồn tại…
- **Error Handling**: Cách xử lý lỗi để chương trình không bị “crash” đột ngột, thường dùng `try-catch-finally` hoặc khai báo `throws`.

---

## 2. Checked Exceptions

- Được **kiểm tra tại thời điểm biên dịch** (compile-time).
- Bắt buộc:
  - Hoặc phải **bắt** (`try-catch`)
  - Hoặc phải **khai báo** trong chữ ký phương thức (`throws`).
- Thường là lỗi có thể dự đoán và xử lý được (ví dụ: file không tồn tại, lỗi IO, lỗi SQL).

**Ví dụ:**

```java
import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class CheckedExample {
    public static void main(String[] args) {
        try {
            readFile("data.txt");
        } catch (IOException e) {
            System.out.println("Không đọc được file: " + e.getMessage());
        }
    }

    // Checked exception: phải khai báo throws hoặc try-catch
    public static void readFile(String fileName) throws IOException {
        BufferedReader br = new BufferedReader(new FileReader(fileName));
        String line = br.readLine();
        System.out.println("Nội dung dòng đầu: " + line);
        br.close();
    }
}
```

**Input/Output ví dụ:**

- Input: file `data.txt` tồn tại, nội dung dòng đầu là `Hello`
  - Output:
    ```text
    Nội dung dòng đầu: Hello
    ```

- Input: file `data.txt` không tồn tại
  - Output:
    ```text
    Không đọc được file: data.txt (No such file or directory)
    ```

---

## 3. Unchecked Exceptions (Runtime Exceptions)

- **Không bị kiểm tra tại compile-time**.
- Là các lớp con của `RuntimeException`.
- Không bắt buộc phải `try-catch` hoặc `throws`, nhưng có thể dùng nếu muốn.
- Thường là lỗi lập trình (bug) như:
  - `NullPointerException`
  - `ArrayIndexOutOfBoundsException`
  - `ArithmeticException` (chia cho 0)
  - `IllegalArgumentException`, …

**Ví dụ:**

```java
public class UncheckedExample {
    public static void main(String[] args) {
        int result = divide(10, 0); // gây ArithmeticException
        System.out.println("Kết quả: " + result);
    }

    public static int divide(int a, int b) {
        return a / b; // Unchecked exception nếu b = 0
    }
}
```

**Input/Output ví dụ:**

- Input: `a = 10`, `b = 0`
  - Output (thực tế là chương trình ném exception và dừng):
    ```text
    Exception in thread "main" java.lang.ArithmeticException: / by zero
        at UncheckedExample.divide(UncheckedExample.java:8)
        at UncheckedExample.main(UncheckedExample.java:4)
    ```

---

## 4. So sánh Checked vs Unchecked

| Tiêu chí                     | Checked Exceptions                         | Unchecked Exceptions (Runtime)              |
|-----------------------------|--------------------------------------------|--------------------------------------------|
| Kiểm tra lúc biên dịch      | Có                                         | Không                                      |
| Bắt buộc xử lý              | Có (`try-catch` hoặc `throws`)             | Không                                      |
| Thường dùng cho             | Lỗi có thể dự đoán, phụ thuộc môi trường   | Lỗi lập trình, bug logic                   |
| Ví dụ                       | `IOException`, `SQLException`              | `NullPointerException`, `ArithmeticException` |

---

## 5. Khi nào dùng cái nào?

- **Checked**:
  - Khi lỗi có thể xảy ra do môi trường bên ngoài (file, mạng, DB…).
  - Muốn buộc người dùng API phải xử lý lỗi.

- **Unchecked**:
  - Khi lỗi là do lập trình sai (bug).
  - Không muốn “làm ồn” chữ ký phương thức với quá nhiều `throws`.
  - Thường dùng cho `IllegalArgumentException`, `IllegalStateException`, v.v.
