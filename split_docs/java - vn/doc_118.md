
Dưới đây là tóm tắt ngắn gọn về **Unit Test & Integration Test với Mockito / các framework mocking** trong Java, kèm ví dụ mã và input/output.

---

## 1. Khái niệm chính

### Unit Test
- Kiểm thử **từng đơn vị nhỏ** (class, method) một cách **cô lập**.
- Các phụ thuộc (dependency) được **mock** để không gọi thật (DB, API, file, …).
- Mục tiêu: logic của class đó có đúng không?

### Integration Test
- Kiểm thử **nhiều thành phần kết hợp** với nhau (service + repository + DB thật hoặc in-memory).
- Ít hoặc không dùng mock, tập trung vào **luồng hoạt động thực tế**.
- Mục tiêu: các module tích hợp với nhau có hoạt động đúng không?

### Mockito / Mocking Frameworks
- Dùng để tạo **mock object** thay cho dependency thật.
- Cho phép:
  - Định nghĩa hành vi giả (`when(...).thenReturn(...)`)
  - Kiểm tra tương tác (`verify(...)`)
- Các framework phổ biến: **Mockito**, **EasyMock**, **JMock**, …

## 2. Ví dụ Unit Test với Mockito

Giả sử ta có service tính tổng số tiền trong giỏ hàng, phụ thuộc vào `PriceService` để lấy giá sản phẩm.

### Code chính

```java
public class PriceService {
    public int getPrice(String productId) {
        // Giả sử gọi DB hoặc API ngoài
        throw new UnsupportedOperationException("Real call not implemented");
    }
}

public class CartService {
    private final PriceService priceService;

    public CartService(PriceService priceService) {
        this.priceService = priceService;
    }

    public int calculateTotal(List<String> productIds) {
        int total = 0;
        for (String id : productIds) {
            total += priceService.getPrice(id);
        }
        return total;
    }
}
```

### Unit Test với Mockito

```java
import org.junit.jupiter.api.Test;
import org.mockito.Mockito;

import java.util.Arrays;
import java.util.List;

import static org.junit.jupiter.api.Assertions.assertEquals;
import static org.mockito.Mockito.*;

class CartServiceTest {

    @Test
    void calculateTotal_shouldReturnSumOfPrices() {
        // 1. Tạo mock cho PriceService
        PriceService priceServiceMock = Mockito.mock(PriceService.class);

        // 2. Định nghĩa hành vi giả
        when(priceServiceMock.getPrice("A")).thenReturn(100);
        when(priceServiceMock.getPrice("B")).thenReturn(200);

        // 3. Tạo CartService với mock
        CartService cartService = new CartService(priceServiceMock);

        // 4. Input
        List<String> productIds = Arrays.asList("A", "B");

        // 5. Gọi method cần test
        int result = cartService.calculateTotal(productIds);

        // 6. Kiểm tra output
        assertEquals(300, result); // Output mong đợi: 300

        // 7. Verify tương tác với mock
        verify(priceServiceMock, times(1)).getPrice("A");
        verify(priceServiceMock, times(1)).getPrice("B");
        verifyNoMoreInteractions(priceServiceMock);
    }
}
```

**Input:** `["A", "B"]`  
**Mock behavior:**  
- `getPrice("A")` → 100  
- `getPrice("B")` → 200  

**Output thực tế:** `300`  
**Ý nghĩa:** Đảm bảo logic cộng tổng trong `CartService` đúng, không cần DB/API thật.

## 3. Ví dụ Integration Test (ít dùng mock)

Giả sử dùng Spring Boot, test tích hợp `UserService` + `UserRepository` + DB in-memory (H2).

### Entity & Repository

```java
@Entity
public class User {
    @Id @GeneratedValue
    private Long id;
    private String name;

    // getters/setters
}

public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByName(String name);
}
```

### Service

```java
@Service
public class UserService {
    @Autowired
    private UserRepository userRepository;

    public User createUser(String name) {
        User user = new User();
        user.setName(name);
        return userRepository.save(user);
    }

    public Optional<User> findByName(String name) {
        return userRepository.findByName(name);
    }
}
```

### Integration Test

```java
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.transaction.annotation.Transactional;

import java.util.Optional;

import static org.junit.jupiter.api.Assertions.*;

@SpringBootTest
@Transactional
class UserServiceIntegrationTest {

    @Autowired
    private UserService userService;

    @Test
    void createAndFindUser_shouldWorkWithRealDatabase() {
        // Input
        String inputName = "Alice";

        // Gọi service thật, repository thật, DB in-memory
        User created = userService.createUser(inputName);

        // Kiểm tra output tạo user
        assertNotNull(created.getId());
        assertEquals(inputName, created.getName());

        // Gọi lại để tìm user
        Optional<User> found = userService.findByName(inputName);

        // Kiểm tra output tìm kiếm
        assertTrue(found.isPresent());
        assertEquals(created.getId(), found.get().getId());
        assertEquals(inputName, found.get().getName());
    }
}
```

**Input:** tên user `"Alice"`  
**Output:**  
- User được lưu vào DB với `id` khác null  
- Tìm lại theo tên trả về đúng user vừa tạo

## 4. Khi nào dùng Unit Test + Mockito, khi nào dùng Integration Test?

- **Unit Test + Mockito**:
  - Khi muốn test **logic thuần** của một class.
  - Khi dependency là **DB, API ngoài, message queue, file**, … → nên mock.
  - Nhanh, chạy nhiều, feedback sớm.

- **Integration Test**:
  - Khi muốn đảm bảo **các lớp làm việc với nhau** đúng (service + repository + config).
  - Khi cần kiểm tra **mapping JPA, transaction, cấu hình Spring**, …
  - Chậm hơn, nhưng phản ánh gần với môi trường thật.
