## 1. Class Loading (Quá trình nạp lớp)

**Ý chính:**
- JVM không nạp tất cả class ngay từ đầu, mà nạp **khi cần** (lazy loading).
- Quá trình gồm 3 bước chính:
  1. **Loading**: Tìm và đọc file `.class` (từ file, JAR, network…).
  2. **Linking**: Kiểm tra, chuẩn bị bộ nhớ cho static fields, và (tùy chọn) resolve các tham chiếu.
  3. **Initialization**: Chạy các khối `static` và gán giá trị ban đầu cho static fields.

**ClassLoader:**
- Có nhiều loại: 
  - **Bootstrap ClassLoader**: nạp các class core của JDK (`java.lang.*`, `java.util.*`…).
  - **Platform / Extension ClassLoader**: nạp các thư viện nền tảng.
  - **Application ClassLoader**: nạp các class trong ứng dụng (từ classpath).

**Ví dụ: xem ClassLoader của một class**

```java
public class ClassLoaderDemo {
    public static void main(String[] args) {
        ClassLoader cl = ClassLoaderDemo.class.getClassLoader();
        System.out.println("ClassLoader của ClassLoaderDemo: " + cl);

        ClassLoader stringCl = String.class.getClassLoader();
        System.out.println("ClassLoader của String: " + stringCl);
    }
}
```

**Output (ví dụ):**
```text
ClassLoader của ClassLoaderDemo: jdk.internal.loader.ClassLoaders$AppClassLoader@4e0e2f2a
ClassLoader của String: null
```

`String` có `ClassLoader = null` vì nó được nạp bởi **Bootstrap ClassLoader** (thường hiển thị là `null`).

## 2. Classpath (Đường dẫn tìm class)

**Ý chính:**
- `classpath` là danh sách đường dẫn (thư mục, file JAR) mà JVM dùng để **tìm file `.class`**.
- Nếu class không nằm trong `classpath`, sẽ bị lỗi `ClassNotFoundException` hoặc `NoClassDefFoundError`.
- Thiết lập bằng:
  - Biến môi trường `CLASSPATH`, hoặc
  - Tham số `-cp` / `-classpath` khi chạy `java`.

**Ví dụ: cấu trúc thư mục**

```text
project/
 ├─ src/
 │   └─ com/example/Main.java
 └─ lib/
     └─ mylib.jar
```

**Compile:**

```bash
javac -d out src/com/example/Main.java
```

**Run với classpath:**

```bash
java -cp out:lib/mylib.jar com.example.Main   # Linux/macOS
java -cp out;lib\mylib.jar com.example.Main  # Windows
```

Nếu bỏ `lib/mylib.jar` khỏi classpath mà `Main` dùng class trong `mylib.jar`, sẽ bị lỗi `ClassNotFoundException`.

## 3. Modules (Java 9+)

**Ý chính:**
- Từ Java 9, giới thiệu **Java Platform Module System (JPMS)**.
- Mục tiêu:
  - Tổ chức code thành **module** (thay vì chỉ package + classpath).
  - Kiểm soát **tính đóng gói**: module nào export package nào, module nào được phép dùng.
  - Giảm “classpath hell” (xung đột thư viện, thiếu phụ thuộc…).

**Khái niệm chính:**
- Mỗi module có file `module-info.java` ở root của module.
- Dùng:
  - `module` để khai báo tên module.
  - `exports` để cho phép module khác dùng package.
  - `requires` để khai báo phụ thuộc vào module khác.

**Ví dụ: module đơn giản**

Cấu trúc:

```text
app/
 ├─ src/
 │   └─ com.example.app/
 │       ├─ module-info.java
 │       └─ com/example/app/Main.java
 └─ lib/
     └─ com.example.lib.jar (module khác)
```

`module-info.java`:

```java
module com.example.app {
    requires com.example.lib;      // phụ thuộc vào module com.example.lib
    exports com.example.app;       // export package com.example.app
}
```

`Main.java`:

```java
package com.example.app;

import com.example.lib.Util;

public class Main {
    public static void main(String[] args) {
        String result = Util.upper("hello");
        System.out.println(result);
    }
}
```

Giả sử `com.example.lib` là module có class:

```java
package com.example.lib;

public class Util {
    public static String upper(String s) {
        return s.toUpperCase();
    }
}
```

**Compile & Run với module:**

```bash
# Compile
javac --module-path lib -d out $(find src -name "*.java")

# Run
java --module-path out:lib --module com.example.app/com.example.app.Main
```

**Output:**
```text
HELLO
```

## Tóm tắt ngắn

- **Class loading**: JVM nạp class theo nhu cầu, qua các `ClassLoader` (bootstrap, platform, application), gồm các bước loading → linking → initialization.
- **Classpath**: danh sách đường dẫn để JVM tìm `.class`/JAR; cấu hình sai dễ gây `ClassNotFoundException`.
- **Modules (Java 9+)**: thêm tầng tổ chức trên classpath, dùng `module-info.java` để khai báo `requires` và `exports`, giúp quản lý phụ thuộc và đóng gói tốt hơn.
