
Dưới đây là tóm tắt ngắn gọn về **Saga pattern cho giao dịch phân tán** trong Java, kèm ví dụ code và input/output minh họa.

---

## 1. Saga pattern là gì?

- Dùng để xử lý **giao dịch phân tán** trong hệ thống microservices.
- Thay vì một **giao dịch lớn, đồng bộ** (2PC), Saga chia thành **chuỗi các bước (local transactions)**.
- Mỗi bước:
  - Thực hiện một **local transaction** trong một service.
  - Nếu thất bại, sẽ gọi **compensation (giao dịch bù trừ)** để rollback logic nghiệp vụ.

---

## 2. Hai kiểu Saga chính

1. **Choreography (dựa trên event)**
   - Không có “orchestrator” trung tâm.
   - Các service giao tiếp qua **event** (message broker).
   - Mỗi service lắng nghe event, thực hiện việc của mình, rồi phát event tiếp theo.

2. **Orchestration (dựa trên orchestrator)**
   - Có một **Saga orchestrator** điều phối toàn bộ luồng.
   - Orchestrator gọi từng service theo thứ tự, và gọi **compensation** nếu có lỗi.

---

## 3. Ví dụ đơn giản: Đặt hàng (Order Saga)

Quy trình:
1. Tạo đơn hàng (Order Service).
2. Trừ tồn kho (Inventory Service).
3. Trừ tiền (Payment Service).

Nếu bước 2 hoặc 3 lỗi:
- Hủy đơn hàng.
- Hoàn lại tồn kho (nếu đã trừ).
- Hoàn tiền (nếu đã trừ).

### 3.1. Ví dụ Orchestration Saga (Java, pseudo-code)

#### Orchestrator

```java
public class OrderSagaOrchestrator {

    private final OrderService orderService;
    private final InventoryService inventoryService;
    private final PaymentService paymentService;

    public OrderSagaOrchestrator(OrderService orderService,
                                 InventoryService inventoryService,
                                 PaymentService paymentService) {
        this.orderService = orderService;
        this.inventoryService = inventoryService;
        this.paymentService = paymentService;
    }

    public String createOrderSaga(String productId, int quantity, String userId) {
        String orderId = null;
        boolean inventoryReserved = false;
        boolean paymentCompleted = false;

        try {
            // Step 1: Create order
            orderId = orderService.createOrder(productId, quantity, userId);

            // Step 2: Reserve inventory
            inventoryService.reserve(productId, quantity);
            inventoryReserved = true;

            // Step 3: Process payment
            paymentService.pay(userId, productId, quantity);
            paymentCompleted = true;

            // Saga success
            orderService.markOrderCompleted(orderId);
            return "SUCCESS: orderId=" + orderId;

        } catch (Exception e) {
            // Compensation logic
            if (paymentCompleted) {
                paymentService.refund(userId, productId, quantity);
            }
            if (inventoryReserved) {
                inventoryService.release(productId, quantity);
            }
            if (orderId != null) {
                orderService.cancelOrder(orderId);
            }
            return "FAILED: " + e.getMessage();
        }
    }
}
```

#### Service giả lập

```java
public class OrderService {
    public String createOrder(String productId, int quantity, String userId) {
        // Tạo order trong DB local
        String orderId = "ORD-" + System.currentTimeMillis();
        System.out.println("Order created: " + orderId);
        return orderId;
    }

    public void markOrderCompleted(String orderId) {
        System.out.println("Order completed: " + orderId);
    }

    public void cancelOrder(String orderId) {
        System.out.println("Order cancelled: " + orderId);
    }
}

public class InventoryService {
    public void reserve(String productId, int quantity) {
        System.out.println("Inventory reserved: " + productId + " x " + quantity);
        // Có thể ném exception để mô phỏng lỗi
    }

    public void release(String productId, int quantity) {
        System.out.println("Inventory released: " + productId + " x " + quantity);
    }
}

public class PaymentService {
    public void pay(String userId, String productId, int quantity) {
        System.out.println("Payment done for user " + userId);
        // Ví dụ: mô phỏng lỗi
        if ("errorUser".equals(userId)) {
            throw new RuntimeException("Payment failed");
        }
    }

    public void refund(String userId, String productId, int quantity) {
        System.out.println("Payment refunded for user " + userId);
    }
}
```

#### Input / Output minh họa

**Input 1 (thành công):**

```java
public static void main(String[] args) {
    OrderService orderService = new OrderService();
    InventoryService inventoryService = new InventoryService();
    PaymentService paymentService = new PaymentService();

    OrderSagaOrchestrator orchestrator =
        new OrderSagaOrchestrator(orderService, inventoryService, paymentService);

    String result = orchestrator.createOrderSaga("P001", 2, "userA");
    System.out.println("Result: " + result);
}
```

**Output 1:**

```text
Order created: ORD-1716200000000
Inventory reserved: P001 x 2
Payment done for user userA
Order completed: ORD-1716200000000
Result: SUCCESS: orderId=ORD-1716200000000
```

## 4. Ý chính cần nhớ

- Saga pattern:
  - Giải quyết **giao dịch phân tán** trong microservices.
  - Dùng **chuỗi local transactions + compensation** thay cho 2PC.
- Có 2 cách triển khai:
  - **Choreography**: dựa trên event, không có trung tâm.
  - **Orchestration**: có orchestrator điều phối.
- Ưu điểm:
  - Tăng **tính chịu lỗi**, **tính mở rộng**, phù hợp kiến trúc microservices.
- Nhược điểm:
  - Logic nghiệp vụ phức tạp hơn, phải thiết kế **compensation** cẩn thận.
