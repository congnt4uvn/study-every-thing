
Dưới đây là tóm tắt ngắn gọn về **Spring Data & Persistence – Entity mapping** với ví dụ mã và input/output.

---

## 1. @Entity và @Table

- `@Entity`: Đánh dấu một class là **thực thể (entity)** để JPA/Hibernate ánh xạ với bảng trong database.
- `@Table`: (tuỳ chọn) Dùng để chỉ rõ **tên bảng** trong database nếu khác với tên class.

```java
import jakarta.persistence.*;

@Entity
@Table(name = "users") // nếu bỏ @Table, mặc định tên bảng là "user" hoặc "User" tuỳ config
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String username;

    // getter/setter
}
```

**Ý nghĩa:**
- Class `User` sẽ được ánh xạ với bảng `users`.
- Trường `id` là khóa chính, tự tăng.

## 2. Quan hệ @OneToMany (1 - N)

Ví dụ: Một `User` có nhiều `Post`.

```java
import jakarta.persistence.*;
import java.util.List;

@Entity
@Table(name = "users")
public class User {

    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String username;

    @OneToMany(mappedBy = "user", cascade = CascadeType.ALL)
    private List<Post> posts;

    // getter/setter
}

@Entity
@Table(name = "posts")
public class Post {

    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;

    @ManyToOne
    @JoinColumn(name = "user_id") // cột khóa ngoại trong bảng posts
    private User user;

    // getter/setter
}
```

### Ví dụ Input/Output (REST API)

**Input (POST /users)**

```json
{
  "username": "alice",
  "posts": [
    { "title": "Bài viết 1" },
    { "title": "Bài viết 2" }
  ]
}
```

**Output (GET /users/1)**

```json
{
  "id": 1,
  "username": "alice",
  "posts": [
    { "id": 1, "title": "Bài viết 1" },
    { "id": 2, "title": "Bài viết 2" }
  ]
}
```

## 3. Quan hệ @ManyToMany (N - N)

Ví dụ: `Student` học nhiều `Course`, và `Course` có nhiều `Student`.

```java
import jakarta.persistence.*;
import java.util.Set;

@Entity
@Table(name = "students")
public class Student {

    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;

    @ManyToMany
    @JoinTable(
        name = "student_course", // bảng trung gian
        joinColumns = @JoinColumn(name = "student_id"),
        inverseJoinColumns = @JoinColumn(name = "course_id")
    )
    private Set<Course> courses;

    // getter/setter
}

@Entity
@Table(name = "courses")
public class Course {

    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;

    @ManyToMany(mappedBy = "courses")
    private Set<Student> students;

    // getter/setter
}
```

### Ví dụ Input/Output

**Input (POST /students)**

```json
{
  "name": "Nam",
  "courses": [
    { "id": 1 },
    { "id": 2 }
  ]
}
```

**Output (GET /students/1)**

```json
{
  "id": 1,
  "name": "Nam",
  "courses": [
    { "id": 1, "title": "Java Cơ Bản" },
    { "id": 2, "title": "Spring Boot" }
  ]
}
```

---

## 4. Tóm tắt ý chính

- **@Entity**: Đánh dấu class là entity để lưu trong database.
- **@Table**: Chỉ định tên bảng tương ứng.
- **@OneToMany / @ManyToOne**: Quan hệ 1 - N (ví dụ: User – Post).
- **@ManyToMany**: Quan hệ N - N (ví dụ: Student – Course) với bảng trung gian.
- Các annotation này giúp Spring Data JPA tự động ánh xạ object Java ↔ bảng/quan hệ trong database, giảm code SQL thủ công.
