
Dưới đây là tóm tắt ngắn gọn về **Code Coverage tools (JaCoCo) trong Java** bằng tiếng Việt, kèm ví dụ:

---

## 1. JaCoCo là gì?

- **JaCoCo** (Java Code Coverage) là công cụ đo **mức độ bao phủ mã nguồn** khi chạy test (JUnit, v.v.).
- Giúp biết **phần nào của code đã được test**, phần nào **chưa được chạm tới**.
- Thường dùng trong:
  - Dự án Maven/Gradle
  - CI/CD (Jenkins, GitLab CI, GitHub Actions, …)

---

## 2. Các loại Code Coverage chính

JaCoCo có thể đo:

- **Line coverage**: Bao nhiêu dòng code đã được thực thi.
- **Branch coverage**: Bao nhiêu nhánh (if/else, switch) đã được đi qua.
- **Instruction coverage**: Bao nhiêu lệnh bytecode đã chạy.
- **Method/Class coverage**: Bao nhiêu method/class đã được gọi.

---

## 3. Tích hợp JaCoCo với Maven (ví dụ)

**pom.xml**:

```xml
<build>
    <plugins>
        <plugin>
            <groupId>org.jacoco</groupId>
            <artifactId>jacoco-maven-plugin</artifactId>
            <version>0.8.11</version>
            <executions>
                <!-- Gắn agent để thu thập coverage khi test -->
                <execution>
                    <goals>
                        <goal>prepare-agent</goal>
                    </goals>
                </execution>

                <!-- Tạo báo cáo sau khi test -->
                <execution>
                    <id>report</id>
                    <phase>test</phase>
                    <goals>
                        <goal>report</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>
```

Chạy:

```bash
mvn test
```

JaCoCo sẽ tạo báo cáo (thường ở: `target/site/jacoco/index.html`).

## 4. Ví dụ code + test + coverage

### 4.1. Class cần test

```java
public class Calculator {

    public int add(int a, int b) {
        return a + b;
    }

    public String checkSign(int x) {
        if (x > 0) {
            return "positive";
        } else if (x < 0) {
            return "negative";
        } else {
            return "zero";
        }
    }
}
```

### 4.2. Test với JUnit

```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {

    @Test
    void testAdd() {
        Calculator c = new Calculator();
        assertEquals(5, c.add(2, 3));      // input: 2,3 -> output: 5
        assertEquals(0, c.add(-1, 1));     // input: -1,1 -> output: 0
    }

    @Test
    void testCheckSign() {
        Calculator c = new Calculator();
        assertEquals("positive", c.checkSign(10));   // input: 10 -> "positive"
        assertEquals("negative", c.checkSign(-5));   // input: -5 -> "negative"
        assertEquals("zero", c.checkSign(0));        // input: 0 -> "zero"
    }
}
```

### 4.3. Kết quả coverage (ví dụ)

Sau khi chạy `mvn test`, JaCoCo có thể báo:

- **Line coverage**: 100% (tất cả dòng trong `Calculator` đã chạy)
- **Branch coverage**: 100% (các nhánh `>0`, `<0`, `==0` đều được test)

Nếu bạn chỉ test `checkSign(10)` mà không test `-5` và `0`, thì:

- Line coverage có thể ~66%
- Branch coverage thấp (vì chưa đi qua hết các nhánh if/else)

---

## 5. Ý nghĩa thực tế

- Code coverage **không đảm bảo** code không có bug, nhưng:
  - Giúp phát hiện **phần code chưa được test**.
  - Hỗ trợ cải thiện **chất lượng test**.
- Thường đặt **ngưỡng coverage tối thiểu** (ví dụ: 80%) trong pipeline CI để đảm bảo dự án luôn được test đủ.

Nếu bạn muốn, tôi có thể thêm ví dụ cấu hình Gradle hoặc ví dụ báo cáo HTML của JaCoCo.
