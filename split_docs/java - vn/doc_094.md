
Dưới đây là tóm tắt ngắn gọn về 3 mẫu thiết kế hành vi trong Java: Strategy, Observer, Template Method, kèm ví dụ code và input/output minh họa.

---

## 1. Strategy Pattern (Chiến lược)

**Ý chính:**
- Định nghĩa một họ (family) các thuật toán, đóng gói từng thuật toán, và cho phép thay đổi thuật toán lúc runtime.
- Tách phần “cách làm” (strategy) khỏi phần “ngữ cảnh sử dụng” (context).

**Khi dùng:**
- Khi bạn có nhiều cách xử lý khác nhau cho cùng một nhiệm vụ (ví dụ: nhiều cách tính phí, nhiều cách sắp xếp, nhiều cách nén dữ liệu…).
- Muốn tránh dùng nhiều `if-else` hoặc `switch` để chọn thuật toán.

### Ví dụ: Tính giá sau giảm (discount)

```java
// Strategy: interface chung cho các chiến lược giảm giá
interface DiscountStrategy {
    double applyDiscount(double price);
}

// Concrete Strategies
class NoDiscount implements DiscountStrategy {
    @Override
    public double applyDiscount(double price) {
        return price;
    }
}

class PercentageDiscount implements DiscountStrategy {
    private final double percent; // ví dụ: 0.1 = 10%

    public PercentageDiscount(double percent) {
        this.percent = percent;
    }

    @Override
    public double applyDiscount(double price) {
        return price * (1 - percent);
    }
}

class FixedAmountDiscount implements DiscountStrategy {
    private final double amount;

    public FixedAmountDiscount(double amount) {
        this.amount = amount;
    }

    @Override
    public double applyDiscount(double price) {
        return Math.max(0, price - amount);
    }
}

// Context: sử dụng chiến lược
class PriceCalculator {
    private DiscountStrategy discountStrategy;

    public PriceCalculator(DiscountStrategy discountStrategy) {
        this.discountStrategy = discountStrategy;
    }

    public void setDiscountStrategy(DiscountStrategy discountStrategy) {
        this.discountStrategy = discountStrategy;
    }

    public double calculate(double originalPrice) {
        return discountStrategy.applyDiscount(originalPrice);
    }
}

public class StrategyDemo {
    public static void main(String[] args) {
        double price = 100.0;

        PriceCalculator calculator = new PriceCalculator(new NoDiscount());
        System.out.println("No discount: " + calculator.calculate(price));

        calculator.setDiscountStrategy(new PercentageDiscount(0.2)); // 20%
        System.out.println("20% discount: " + calculator.calculate(price));

        calculator.setDiscountStrategy(new FixedAmountDiscount(15));
        System.out.println("15 fixed discount: " + calculator.calculate(price));
    }
}
```

**Output (ví dụ):**
```
No discount: 100.0
20% discount: 80.0
15 fixed discount: 85.0
```

## 2. Observer Pattern (Quan sát)

**Ý chính:**
- Thiết lập quan hệ 1-n: khi một đối tượng (Subject) thay đổi trạng thái, tất cả các Observer liên quan sẽ được thông báo tự động.
- Giảm sự phụ thuộc chặt chẽ giữa các thành phần (loose coupling).

**Khi dùng:**
- Khi nhiều phần trong hệ thống cần được cập nhật khi một trạng thái thay đổi (UI, log, cache, notification…).
- Ví dụ: hệ thống event, GUI, hệ thống pub/sub.

### Ví dụ: Hệ thống thông báo giá cổ phiếu

```java
import java.util.ArrayList;
import java.util.List;

// Observer
interface StockObserver {
    void onPriceChanged(String symbol, double price);
}

// Subject
class StockMarket {
    private final List<StockObserver> observers = new ArrayList<>();
    private double price;

    public void addObserver(StockObserver observer) {
        observers.add(observer);
    }

    public void removeObserver(StockObserver observer) {
        observers.remove(observer);
    }

    public void setPrice(double price) {
        this.price = price;
        notifyObservers();
    }

    private void notifyObservers() {
        for (StockObserver observer : observers) {
            observer.onPriceChanged("ABC", price);
        }
    }
}

// Concrete Observers
class MobileApp implements StockObserver {
    @Override
    public void onPriceChanged(String symbol, double price) {
        System.out.println("MobileApp: Giá " + symbol + " mới = " + price);
    }
}

class WebDashboard implements StockObserver {
    @Override
    public void onPriceChanged(String symbol, double price) {
        System.out.println("WebDashboard: Cập nhật " + symbol + " = " + price);
    }
}

public class ObserverDemo {
    public static void main(String[] args) {
        StockMarket market = new StockMarket();

        StockObserver mobile = new MobileApp();
        StockObserver web = new WebDashboard();

        market.addObserver(mobile);
        market.addObserver(web);

        market.setPrice(10.5);
        market.setPrice(11.2);
    }
}
```

**Output (ví dụ):**
```
MobileApp: Giá ABC mới = 10.5
WebDashboard: Cập nhật ABC = 10.5
MobileApp: Giá ABC mới = 11.2
WebDashboard: Cập nhật ABC = 11.2
```

## 3. Template Method Pattern (Phương thức khuôn mẫu)

**Ý chính:**
- Định nghĩa “khung” (template) của một thuật toán trong lớp abstract, cho phép các lớp con override một số bước cụ thể.
- Một phần thuật toán cố định, một phần linh hoạt.

**Khi dùng:**
- Khi nhiều lớp có quy trình chung, nhưng một vài bước trong quy trình khác nhau.
- Ví dụ: quy trình xử lý file, quy trình thanh toán, quy trình game loop…

### Ví dụ: Quy trình tạo báo cáo

```java
// Template
abstract class ReportGenerator {

    // Template method: final để không bị override
    public final void generateReport() {
        fetchData();
        formatData();
        printReport();
    }

    protected abstract void fetchData();   // bước tùy biến
    protected abstract void formatData();  // bước tùy biến

    // bước chung
    protected void printReport() {
        System.out.println("In báo cáo ra màn hình...");
    }
}

// Concrete classes
class SalesReportGenerator extends ReportGenerator {
    @Override
    protected void fetchData() {
        System.out.println("Lấy dữ liệu doanh số từ DB...");
    }

    @Override
    protected void formatData() {
        System.out.println("Định dạng dữ liệu doanh số theo bảng...");
    }
}

class InventoryReportGenerator extends ReportGenerator {
    @Override
    protected void fetchData() {
        System.out.println("Lấy dữ liệu tồn kho từ API...");
    }

    @Override
    protected void formatData() {
        System.out.println("Định dạng dữ liệu tồn kho theo biểu đồ...");
    }
}

public class TemplateMethodDemo {
    public static void main(String[] args) {
        ReportGenerator salesReport = new SalesReportGenerator();
        ReportGenerator inventoryReport = new InventoryReportGenerator();

        System.out.println("=== Sales Report ===");
        salesReport.generateReport();

        System.out.println("\n=== Inventory Report ===");
        inventoryReport.generateReport();
    }
}
```

**Output (ví dụ):**
```
=== Sales Report ===
Lấy dữ liệu doanh số từ DB...
Định dạng dữ liệu doanh số theo bảng...
In báo cáo ra màn hình...

=== Inventory Report ===
Lấy dữ liệu tồn kho từ API...
Định dạng dữ liệu tồn kho theo biểu đồ...
In báo cáo ra màn hình...
```

## Tóm tắt ngắn gọn

- **Strategy**: Thay đổi thuật toán linh hoạt thông qua interface chiến lược; tránh `if-else` lớn.
- **Observer**: Một đối tượng thay đổi → tự động thông báo cho nhiều observer; dùng cho event, notification.
- **Template Method**: Định nghĩa khung thuật toán trong lớp abstract, cho phép lớp con tùy biến một số bước.

Nếu bạn muốn, tôi có thể thêm ví dụ phức tạp hơn (ví dụ trong web service, game, hoặc hệ thống thanh toán).
