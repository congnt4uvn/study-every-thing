
Dưới đây là tóm tắt ngắn gọn về 4 tính chất OOP trong Java, kèm ví dụ mã và input/output.  

---

## 1. Encapsulation (Đóng gói)

**Ý chính**  
- Ẩn dữ liệu bên trong lớp, chỉ cho truy cập qua phương thức (getter/setter).  
- Bảo vệ dữ liệu, tránh sửa trực tiếp từ bên ngoài.

**Ví dụ**

```java
class BankAccount {
    private double balance; // dữ liệu bị ẩn

    public BankAccount(double balance) {
        this.balance = balance;
    }

    public double getBalance() { // getter
        return balance;
    }

    public void deposit(double amount) { // setter có kiểm soát
        if (amount > 0) {
            balance += amount;
        }
    }
}

public class Main {
    public static void main(String[] args) {
        BankAccount acc = new BankAccount(1000);
        acc.deposit(500);
        System.out.println(acc.getBalance());
    }
}
```

**Output**

```
1500.0
```

## 2. Inheritance (Kế thừa)

**Ý chính**  
- Lớp con kế thừa thuộc tính và phương thức của lớp cha.  
- Giúp tái sử dụng mã, mở rộng chức năng.

**Ví dụ**

```java
class Animal {
    public void sound() {
        System.out.println("Some sound");
    }
}

class Dog extends Animal {
    // Dog tự động có phương thức sound() từ Animal
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.sound();
    }
}
```

**Output**

```
Some sound
```

## 3. Polymorphism (Đa hình)

**Ý chính**  
- Cùng một phương thức nhưng hành vi khác nhau tùy đối tượng.  
- Thường dùng với kế thừa + override.

**Ví dụ**

```java
class Animal {
    public void sound() {
        System.out.println("Some sound");
    }
}

class Dog extends Animal {
    @Override
    public void sound() {
        System.out.println("Woof");
    }
}

class Cat extends Animal {
    @Override
    public void sound() {
        System.out.println("Meow");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog();
        Animal a2 = new Cat();

        a1.sound();
        a2.sound();
    }
}
```

**Output**

```
Woof
Meow
```

## 4. Abstraction (Trừu tượng)

**Ý chính**  
- Ẩn chi tiết triển khai, chỉ lộ ra “cái gì làm” chứ không phải “làm như thế nào”.  
- Dùng `abstract class` hoặc `interface`.

**Ví dụ với abstract class**

```java
abstract class Shape {
    abstract double area(); // chỉ khai báo, không triển khai
}

class Circle extends Shape {
    private double r;

    public Circle(double r) {
        this.r = r;
    }

    @Override
    double area() {
        return Math.PI * r * r;
    }
}

public class Main {
    public static void main(String[] args) {
        Shape s = new Circle(2);
        System.out.println(s.area());
    }
}
```

**Output (xấp xỉ)**

```
12.566370614359172
```

---

**Tóm tắt cực ngắn**  
- Encapsulation: Ẩn dữ liệu, dùng getter/setter.  
- Inheritance: Lớp con kế thừa lớp cha.  
- Polymorphism: Cùng interface, nhiều cách triển khai.  
- Abstraction: Chỉ lộ ra phần “cần dùng”, giấu chi tiết bên trong.
