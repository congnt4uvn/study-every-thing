
Dưới đây là tóm tắt ngắn gọn về kiến trúc Hexagonal / Clean Architecture (dùng trong Java), kèm ví dụ code và input/output minh họa.

---

## 1. Ý tưởng chính

**Mục tiêu chung** của Hexagonal Architecture (Ports & Adapters) và Clean Architecture:

- Tách **core business (domain)** khỏi:
  - Framework (Spring, Jakarta…)
  - Database
  - UI (REST, Web, CLI…)
  - Hệ thống bên ngoài (API khác, message queue…)
- Giúp:
  - Dễ test (test domain mà không cần DB, web server…)
  - Dễ thay thế công nghệ (đổi DB, đổi framework, đổi giao thức)
  - Code rõ ràng: business không phụ thuộc vào “hạ tầng”.

---

## 2. Cấu trúc cơ bản

### Hexagonal Architecture (Ports & Adapters)

- **Domain (Core)**:  
  - Entities, business logic, use cases.
- **Ports (Interfaces)**:
  - Định nghĩa “cổng” mà domain dùng để nói chuyện với bên ngoài.
  - Ví dụ: `UserRepository`, `PaymentGateway`.
- **Adapters**:
  - **Inbound adapters**: nhận request từ bên ngoài → gọi vào use case.
    - Ví dụ: REST controller, CLI, message listener.
  - **Outbound adapters**: triển khai các port để kết nối ra ngoài.
    - Ví dụ: adapter dùng JPA, JDBC, REST client, Kafka client.

**Nguyên tắc phụ thuộc**:  
Domain → chỉ phụ thuộc vào **interfaces (ports)**, không phụ thuộc vào adapter cụ thể.

---

### Clean Architecture (vòng tròn đồng tâm)

Các vòng điển hình:

1. **Entities (Domain Model)**  
   - Quy tắc nghiệp vụ cốt lõi, độc lập với use case cụ thể.
2. **Use Cases (Application)**  
   - Luồng nghiệp vụ cụ thể (create order, transfer money…).
3. **Interface Adapters**  
   - Chuyển đổi dữ liệu giữa use case và thế giới bên ngoài (DTO, mappers, controllers, presenters).
4. **Frameworks & Drivers**  
   - Spring, DB, UI, external services.

**Luật**:  
Mũi tên phụ thuộc chỉ được đi **từ ngoài vào trong**.  
Code ở vòng trong không được import code ở vòng ngoài.

## 3. Ví dụ Java đơn giản

### 3.1. Domain (Entity + Use Case + Port)

```java
// Domain entity
public class Account {
    private final String id;
    private int balance;

    public Account(String id, int balance) {
        this.id = id;
        this.balance = balance;
    }

    public void deposit(int amount) {
        if (amount <= 0) throw new IllegalArgumentException("Amount must be > 0");
        this.balance += amount;
    }

    public int getBalance() {
        return balance;
    }

    public String getId() {
        return id;
    }
}

// Outbound port (interface)
public interface AccountRepository {
    Account findById(String id);
    void save(Account account);
}

// Use case (application service)
public class DepositMoneyUseCase {
    private final AccountRepository accountRepository;

    public DepositMoneyUseCase(AccountRepository accountRepository) {
        this.accountRepository = accountRepository;
    }

    public int deposit(String accountId, int amount) {
        Account account = accountRepository.findById(accountId);
        if (account == null) {
            throw new IllegalArgumentException("Account not found");
        }
        account.deposit(amount);
        accountRepository.save(account);
        return account.getBalance();
    }
}
```

- `AccountRepository` là **port**.
- `DepositMoneyUseCase` là **use case** (core logic).
- Không có code Spring, không có code DB ở đây.

---

### 3.2. Outbound Adapter (ví dụ: In-memory, có thể thay bằng JPA)

```java
import java.util.HashMap;
import java.util.Map;

public class InMemoryAccountRepository implements AccountRepository {
    private final Map<String, Account> storage = new HashMap<>();

    @Override
    public Account findById(String id) {
        return storage.get(id);
    }

    @Override
    public void save(Account account) {
        storage.put(account.getId(), account);
    }
}
```

Có thể thay `InMemoryAccountRepository` bằng `JpaAccountRepository` mà không đổi code domain.

---

### 3.3. Inbound Adapter (REST Controller với Spring)

```java
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/accounts")
public class AccountController {

    private final DepositMoneyUseCase depositMoneyUseCase;

    public AccountController(DepositMoneyUseCase depositMoneyUseCase) {
        this.depositMoneyUseCase = depositMoneyUseCase;
    }

    @PostMapping("/{id}/deposit")
    public DepositResponse deposit(
            @PathVariable String id,
            @RequestBody DepositRequest request
    ) {
        int newBalance = depositMoneyUseCase.deposit(id, request.amount());
        return new DepositResponse(id, newBalance);
    }

    public record DepositRequest(int amount) {}
    public record DepositResponse(String accountId, int newBalance) {}
}
```

## 4. Ví dụ Input / Output (REST)

Giả sử đã có account:

```java
Account acc = new Account("A123", 1000);
accountRepository.save(acc);
```

### Request

```http
POST /accounts/A123/deposit
Content-Type: application/json

{
  "amount": 500
}
```

### Response

```http
200 OK
Content-Type: application/json

{
  "accountId": "A123",
  "newBalance": 1500
}
```

## 5. Điểm chính cần nhớ

- **Hexagonal / Clean Architecture**:
  - Tách **business** khỏi **infrastructure**.
  - Dùng **ports (interfaces)** và **adapters (implementations)**.
  - Phụ thuộc **từ ngoài vào trong**, domain không biết gì về framework/DB.
- Lợi ích:
  - Dễ test, dễ bảo trì, dễ thay đổi công nghệ.
  - Code rõ ràng, tập trung vào nghiệp vụ.

Nếu bạn muốn, tôi có thể giúp bạn thiết kế cấu trúc package cụ thể cho một project Spring Boot theo kiểu Hexagonal/Clean.
