
Dưới đây là tóm tắt ngắn gọn về các lớp atomic trong `java.util.concurrent` và khái niệm CAS, kèm ví dụ code và input/output.

---

## 1. Atomic classes là gì?

- Nằm trong package `java.util.concurrent.atomic`
- Cung cấp các thao tác **an toàn trong môi trường đa luồng** (thread-safe) mà **không cần dùng synchronized**.
- Các thao tác như tăng/giảm, gán giá trị, so sánh và đặt giá trị… được thực hiện **nguyên tử** (atomic) nhờ cơ chế **CAS (Compare-And-Set)** ở mức CPU.

---

## 2. `AtomicInteger`

- Dùng để thay thế `int` trong môi trường đa luồng.
- Các phương thức chính:
  - `get()`: lấy giá trị hiện tại
  - `set(int newValue)`: đặt giá trị mới
  - `incrementAndGet()`: tăng 1 rồi trả về giá trị mới
  - `getAndIncrement()`: trả về giá trị cũ rồi mới tăng 1
  - `addAndGet(int delta)`: cộng thêm delta rồi trả về
  - `compareAndSet(int expect, int update)`: nếu giá trị hiện tại == expect thì gán thành update, trả về true/false

### Ví dụ: Đếm số lần gọi trong nhiều luồng

```java
import java.util.concurrent.atomic.AtomicInteger;

public class AtomicIntegerExample {
    private static AtomicInteger counter = new AtomicInteger(0);

    public static void main(String[] args) throws InterruptedException {
        Runnable task = () -> {
            for (int i = 0; i < 1000; i++) {
                counter.incrementAndGet(); // tăng an toàn trong đa luồng
            }
        };

        Thread t1 = new Thread(task);
        Thread t2 = new Thread(task);

        t1.start();
        t2.start();

        t1.join();
        t2.join();

        System.out.println("Final counter = " + counter.get());
    }
}
```

**Input:** không có input từ người dùng, chương trình tự chạy 2 luồng, mỗi luồng tăng 1000 lần.  
**Output mong đợi:**
```text
Final counter = 2000
```

Nếu dùng `int` bình thường mà không đồng bộ, kết quả có thể < 2000 do race condition.

## 3. `AtomicReference<T>`

- Dùng để lưu trữ **tham chiếu đến một object** một cách atomic.
- Hữu ích khi bạn muốn cập nhật một object (hoặc trạng thái) theo kiểu **so sánh rồi gán** (CAS).
- Các phương thức chính:
  - `get()`
  - `set(T newValue)`
  - `compareAndSet(T expect, T update)`

### Ví dụ: Cập nhật trạng thái người dùng

```java
import java.util.concurrent.atomic.AtomicReference;

class UserStatus {
    final String name;
    final String status;

    UserStatus(String name, String status) {
        this.name = name;
        this.status = status;
    }

    @Override
    public String toString() {
        return name + " - " + status;
    }
}

public class AtomicReferenceExample {
    public static void main(String[] args) {
        AtomicReference<UserStatus> ref =
                new AtomicReference<>(new UserStatus("An", "OFFLINE"));

        UserStatus oldStatus = ref.get();
        UserStatus newStatus = new UserStatus("An", "ONLINE");

        boolean updated = ref.compareAndSet(oldStatus, newStatus);

        System.out.println("Updated: " + updated);
        System.out.println("Current: " + ref.get());
    }
}
```

**Input:** không có input từ người dùng.  
**Output ví dụ:**
```text
Updated: true
Current: An - ONLINE
```

Nếu trong lúc đó một luồng khác đã thay đổi `ref`, `compareAndSet` sẽ trả về `false` và không cập nhật.

## 4. Khái niệm CAS (Compare-And-Set / Compare-And-Swap)

- **Ý tưởng:**  
  “Chỉ cập nhật giá trị nếu nó vẫn là giá trị mà tôi mong đợi (expect). Nếu đã bị thay đổi bởi luồng khác, thì không cập nhật.”
- Hoạt động ở mức CPU, thường là một lệnh máy duy nhất, nên:
  - Không cần `synchronized`
  - Tránh lock, giảm contention
- Mô hình logic:

```java
boolean compareAndSet(expectedValue, newValue) {
    if (currentValue == expectedValue) {
        currentValue = newValue;
        return true;
    } else {
        return false;
    }
}
```

### Ví dụ minh họa CAS với `AtomicInteger`

```java
import java.util.concurrent.atomic.AtomicInteger;

public class CASExample {
    public static void main(String[] args) {
        AtomicInteger value = new AtomicInteger(10);

        // Giả sử ta mong đợi giá trị hiện tại là 10
        boolean success = value.compareAndSet(10, 20);
        System.out.println("First CAS success: " + success + ", value = " + value.get());

        // Lần này mong đợi là 10 nhưng hiện tại đã là 20
        boolean success2 = value.compareAndSet(10, 30);
        System.out.println("Second CAS success: " + success2 + ", value = " + value.get());
    }
}
```

**Output:**
```text
First CAS success: true, value = 20
Second CAS success: false, value = 20
```

---

## Tóm tắt ý chính

- `AtomicInteger` và `AtomicReference` là các lớp atomic giúp thao tác dữ liệu **an toàn trong đa luồng** mà không cần `synchronized`.
- Chúng dựa trên cơ chế **CAS (Compare-And-Set)**:
  - So sánh giá trị hiện tại với giá trị mong đợi.
  - Nếu giống nhau thì cập nhật; nếu không thì thất bại.
- Thích hợp cho các trường hợp:
  - Bộ đếm, số liệu thống kê, ID tự tăng.
  - Cập nhật trạng thái/đối tượng chia sẻ giữa nhiều luồng.
