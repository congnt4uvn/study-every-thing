
Dưới đây là tóm tắt ngắn gọn về Spring Boot Actuator, tập trung vào health, metrics, info và custom endpoints, kèm ví dụ code và input/output mẫu.

---

## 1. Spring Boot Actuator là gì?

Spring Boot Actuator cung cấp các endpoint giúp:
- Giám sát ứng dụng (health, metrics)
- Xem thông tin build/app (info)
- Tạo endpoint tùy chỉnh (custom endpoints)
- Tích hợp với các hệ thống monitoring (Prometheus, Grafana, v.v.)

Để dùng Actuator, thêm dependency:

```xml
<!-- pom.xml -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

Cấu hình bật endpoint trong `application.properties`:

```properties
management.endpoints.web.exposure.include=health,info,metrics,custom
management.endpoint.health.show-details=always
```

## 2. `/actuator/health` – Health Endpoint

Dùng để kiểm tra tình trạng ứng dụng (up/down, DB, disk, v.v.).

### Ví dụ: gọi endpoint

**Request:**

```http
GET /actuator/health
```

**Response (JSON):**

```json
{
  "status": "UP",
  "components": {
    "db": {
      "status": "UP",
      "details": {
        "database": "H2",
        "validationQuery": "isValid()"
      }
    },
    "diskSpace": {
      "status": "UP",
      "details": {
        "total": 499963174912,
        "free": 1234567890,
        "threshold": 10485760
      }
    }
  }
}
```

### Custom Health Indicator

```java
import org.springframework.boot.actuate.health.Health;
import org.springframework.boot.actuate.health.HealthIndicator;
import org.springframework.stereotype.Component;

@Component
public class MyServiceHealthIndicator implements HealthIndicator {

    @Override
    public Health health() {
        boolean serviceOk = checkMyService(); // logic kiểm tra
        if (serviceOk) {
            return Health.up()
                    .withDetail("service", "OK")
                    .build();
        } else {
            return Health.down()
                    .withDetail("service", "NOT OK")
                    .build();
        }
    }

    private boolean checkMyService() {
        // TODO: kiểm tra thực tế
        return true;
    }
}
```

**Request:**

```http
GET /actuator/health
```

**Response (thêm custom):**

```json
{
  "status": "UP",
  "components": {
    "myServiceHealthIndicator": {
      "status": "UP",
      "details": {
        "service": "OK"
      }
    }
  }
}
```

## 3. `/actuator/metrics` – Metrics Endpoint

Dùng để xem các số liệu như:
- `jvm.memory.used`
- `http.server.requests`
- `system.cpu.usage`
- v.v.

### Liệt kê tất cả metrics

**Request:**

```http
GET /actuator/metrics
```

**Response (ví dụ):**

```json
{
  "names": [
    "jvm.memory.used",
    "jvm.memory.max",
    "system.cpu.usage",
    "http.server.requests"
  ]
}
```

### Xem chi tiết một metric

**Request:**

```http
GET /actuator/metrics/http.server.requests
```

**Response (ví dụ):**

```json
{
  "name": "http.server.requests",
  "measurements": [
    {
      "statistic": "COUNT",
      "value": 42.0
    }
  ],
  "availableTags": [
    {
      "tag": "uri",
      "values": ["/hello", "/actuator/health"]
    }
  ]
}
```

### Tạo custom metric

```java
import io.micrometer.core.instrument.MeterRegistry;
import org.springframework.stereotype.Component;

@Component
public class MyMetricService {

    private final MeterRegistry meterRegistry;

    public MyMetricService(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
    }

    public void processSomething() {
        // logic xử lý
        meterRegistry.counter("my.custom.counter").increment();
    }
}
```

**Sau khi gọi `processSomething()` nhiều lần:**

**Request:**

```http
GET /actuator/metrics/my.custom.counter
```

**Response (ví dụ):**

```json
{
  "name": "my.custom.counter",
  "measurements": [
    {
      "statistic": "COUNT",
      "value": 5.0
    }
  ],
  "availableTags": []
}
```

## 4. `/actuator/info` – Info Endpoint

Dùng để hiển thị thông tin ứng dụng (version, mô tả, contact, v.v.).

### Cấu hình info

```properties
info.app.name=Demo Application
info.app.description=Demo Spring Boot Actuator
info.app.version=1.0.0
```

**Request:**

```http
GET /actuator/info
```

**Response:**

```json
{
  "app": {
    "name": "Demo Application",
    "description": "Demo Spring Boot Actuator",
    "version": "1.0.0"
  }
}
```

### Info Contributor (custom)

```java
import org.springframework.boot.actuate.info.Info;
import org.springframework.boot.actuate.info.InfoContributor;
import org.springframework.stereotype.Component;

import java.util.Map;

@Component
public class MyInfoContributor implements InfoContributor {

    @Override
    public void contribute(Info.Builder builder) {
        builder.withDetail("custom", Map.of(
                "author", "Nguyen Van A",
                "env", "production"
        ));
    }
}
```

**Response (mở rộng):**

```json
{
  "app": {
    "name": "Demo Application",
    "description": "Demo Spring Boot Actuator",
    "version": "1.0.0"
  },
  "custom": {
    "author": "Nguyen Van A",
    "env": "production"
  }
}
```

## 5. Custom Actuator Endpoints

Bạn có thể tạo endpoint riêng dưới `/actuator/...`.

### Ví dụ: custom endpoint đơn giản

```java
import org.springframework.boot.actuate.endpoint.annotation.Endpoint;
import org.springframework.boot.actuate.endpoint.annotation.ReadOperation;
import org.springframework.stereotype.Component;

import java.util.Map;

@Component
@Endpoint(id = "mycustom")
public class MyCustomEndpoint {

    @ReadOperation
    public Map<String, Object> customInfo() {
        return Map.of(
                "status", "OK",
                "message", "This is my custom actuator endpoint"
        );
    }
}
```

Cấu hình expose:

```properties
management.endpoints.web.exposure.include=health,info,metrics,mycustom
```

**Request:**

```http
GET /actuator/mycustom
```

**Response:**

```json
{
  "status": "OK",
  "message": "This is my custom actuator endpoint"
}
```

### Custom endpoint với input (path param)

```java
import org.springframework.boot.actuate.endpoint.annotation.Endpoint;
import org.springframework.boot.actuate.endpoint.annotation.ReadOperation;
import org.springframework.boot.actuate.endpoint.annotation.Selector;
import org.springframework.stereotype.Component;

import java.util.Map;

@Component
@Endpoint(id = "greet")
public class GreetEndpoint {

    @ReadOperation
    public Map<String, String> greet(@Selector String name) {
        return Map.of("greeting", "Hello " + name);
    }
}
```

**Request:**

```http
GET /actuator/greet/John
```

**Response:**

```json
{
  "greeting": "Hello John"
}
```

## Tóm tắt ý chính

- **Actuator**: module giúp giám sát và quản lý ứng dụng Spring Boot.
- **Health** (`/actuator/health`): kiểm tra tình trạng hệ thống, có thể mở rộng bằng `HealthIndicator`.
- **Metrics** (`/actuator/metrics`): cung cấp số liệu runtime, có thể tạo metric riêng bằng Micrometer.
- **Info** (`/actuator/info`): hiển thị thông tin ứng dụng, cấu hình qua `info.*` hoặc `InfoContributor`.
- **Custom endpoints**: tạo endpoint quản trị riêng với `@Endpoint`, `@ReadOperation`, v.v., và expose qua `management.endpoints.web.exposure.include`.
