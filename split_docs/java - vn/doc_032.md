
Dưới đây là tóm tắt ngắn gọn về: **`synchronized`, khóa nội tại (intrinsic lock), và tính tái nhập (reentrancy)** trong Java, kèm ví dụ mã và input/output.

---

## 1. `synchronized` là gì?

- Từ khóa dùng để **đồng bộ hóa** truy cập vào tài nguyên dùng chung (biến, đối tượng, phương thức).
- Đảm bảo **chỉ một thread** được vào vùng code được bảo vệ tại một thời điểm (trên cùng một lock).

Có 3 cách dùng chính:

1. `synchronized` trên **method instance**  
2. `synchronized` trên **method static**  
3. `synchronized` trên **khối code** (`synchronized (lockObject) { ... }`)

---

## 2. Intrinsic Lock (khóa nội tại)

- Mỗi đối tượng trong Java đều có một **monitor lock** (intrinsic lock).
- Khi dùng `synchronized`:
  - `synchronized (obj) { ... }` → thread phải **chiếm lock** của `obj`.
  - `synchronized` method instance → dùng lock của **this**.
  - `synchronized` method static → dùng lock của **Class object** (vd: `MyClass.class`).

Nếu một thread đã giữ lock, thread khác muốn vào vùng `synchronized` dùng cùng lock sẽ **phải chờ**.

---

## 3. Reentrancy (tính tái nhập)

- Lock trong Java là **reentrant**:  
  Nếu một thread **đã giữ** lock, nó có thể **vào lại** vùng `synchronized` dùng cùng lock **mà không bị chặn**.
- Điều này cho phép:
  - Một method `synchronized` gọi **method `synchronized` khác** trên cùng đối tượng.
  - Tránh deadlock tự gây ra bởi chính thread đó.

---

## 4. Ví dụ 1: `synchronized` method instance

```java
public class Counter {
    private int count = 0;

    // synchronized method: lock trên this
    public synchronized void increment() {
        count++;
    }

    public synchronized int getCount() {
        return count;
    }

    public static void main(String[] args) throws InterruptedException {
        Counter counter = new Counter();

        Runnable task = () -> {
            for (int i = 0; i < 1000; i++) {
                counter.increment();
            }
        };

        Thread t1 = new Thread(task);
        Thread t2 = new Thread(task);

        t1.start();
        t2.start();

        t1.join();
        t2.join();

        System.out.println("Final count = " + counter.getCount());
    }
}
```

**Input:** không có input từ người dùng, chương trình tự chạy.  
**Output mong đợi:**  
Vì `increment()` được đồng bộ, kết quả ổn định:

```text
Final count = 2000
```

Nếu bỏ `synchronized`, có thể thấy kết quả nhỏ hơn 2000 do race condition.

## 5. Ví dụ 2: `synchronized` block với lock riêng

```java
public class SafeList {
    private final Object lock = new Object();
    private final java.util.List<Integer> list = new java.util.ArrayList<>();

    public void add(int value) {
        synchronized (lock) {
            list.add(value);
        }
    }

    public int size() {
        synchronized (lock) {
            return list.size();
        }
    }

    public static void main(String[] args) throws InterruptedException {
        SafeList safeList = new SafeList();

        Runnable task = () -> {
            for (int i = 0; i < 1000; i++) {
                safeList.add(i);
            }
        };

        Thread t1 = new Thread(task);
        Thread t2 = new Thread(task);

        t1.start();
        t2.start();

        t1.join();
        t2.join();

        System.out.println("List size = " + safeList.size());
    }
}
```

**Output mong đợi:**

```text
List size = 2000
```

## 6. Ví dụ 3: Reentrancy (tái nhập)

```java
public class ReentrantExample {
    public synchronized void methodA() {
        System.out.println(Thread.currentThread().getName() + " vào methodA");
        methodB(); // gọi method synchronized khác
    }

    public synchronized void methodB() {
        System.out.println(Thread.currentThread().getName() + " vào methodB");
    }

    public static void main(String[] args) {
        ReentrantExample ex = new ReentrantExample();

        Thread t = new Thread(ex::methodA, "Thread-1");
        t.start();
    }
}
```

**Giải thích:**

- `methodA()` và `methodB()` đều `synchronized` trên cùng đối tượng `ex` (lock là `ex`).
- `Thread-1` vào `methodA()` → giữ lock `ex`.
- Trong `methodA()`, nó gọi `methodB()` → **vẫn là cùng lock `ex`**.
- Vì lock là **reentrant**, `Thread-1` được phép vào `methodB()` mà không bị chặn.

**Output ví dụ:**

```text
Thread-1 vào methodA
Thread-1 vào methodB
```

Nếu lock **không** reentrant, thread sẽ tự khóa chính nó (deadlock), nhưng trong Java điều này không xảy ra.

---

## 7. Tóm tắt ngắn

- `synchronized` dùng để **đồng bộ hóa** truy cập tài nguyên dùng chung.
- Mỗi đối tượng có một **intrinsic lock**; `synchronized` dùng lock này để đảm bảo **mutual exclusion** (loại trừ lẫn nhau).
- Lock trong Java là **reentrant**: cùng một thread có thể **vào nhiều lần** các vùng `synchronized` dùng cùng lock mà không bị chặn.
