## 1. Kiến thức cơ bản về Cloud & DevOps (AWS / GCP / Azure)

Khi làm Java trong môi trường Cloud & DevOps, bạn cần nắm:

1. **Khái niệm cơ bản về Cloud**
   - IaaS, PaaS, SaaS.
   - Vùng (Region), Zone, VPC/Network, Security Group/Firewall.
   - Dịch vụ compute: EC2 (AWS), Compute Engine (GCP), VM (Azure).
   - Dịch vụ storage: S3 (AWS), Cloud Storage (GCP), Blob Storage (Azure).
   - Dịch vụ database: RDS/Aurora, Cloud SQL, Azure SQL, v.v.

2. **Triển khai ứng dụng Java lên Cloud**
   - Đóng gói ứng dụng Java (JAR/ WAR).
   - Dùng Docker container.
   - Deploy lên:
     - AWS: ECS, EKS, Elastic Beanstalk, Lambda.
     - GCP: Cloud Run, GKE, App Engine.
     - Azure: App Service, AKS, Functions.

3. **DevOps cơ bản**
   - CI/CD: build, test, deploy tự động (Jenkins, GitHub Actions, GitLab CI, Azure DevOps).
   - Quản lý cấu hình: environment variables, secrets.
   - Monitoring & Logging: CloudWatch (AWS), Stackdriver/Cloud Logging (GCP), Azure Monitor.

## 2. Ví dụ: Ứng dụng Java đơn giản (REST API) để deploy lên Cloud

### 2.1. Code Java (Spring Boot REST API)

```java
// File: src/main/java/com/example/demo/DemoApplication.java
package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.web.bind.annotation.*;

@SpringBootApplication
@RestController
public class DemoApplication {

    @GetMapping("/hello")
    public String hello(@RequestParam(defaultValue = "world") String name) {
        return "Hello, " + name + "!";
    }

    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
```

**Cách chạy local:**

```bash
mvn spring-boot:run
# hoặc
mvn clean package
java -jar target/demo-0.0.1-SNAPSHOT.jar
```

**Input (HTTP request):**

```http
GET http://localhost:8080/hello?name=Java
```

**Output (HTTP response):**

```text
Hello, Java!
```

## 3. Ví dụ Docker hóa ứng dụng Java để đưa lên Cloud

### 3.1. Dockerfile

```dockerfile
# Sử dụng JDK để build
FROM maven:3.9-eclipse-temurin-17 AS build
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn clean package -DskipTests

# Sử dụng JRE nhẹ để chạy
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY --from=build /app/target/demo-0.0.1-SNAPSHOT.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

**Build image:**

```bash
docker build -t my-java-app:1.0 .
```

**Run container:**

```bash
docker run -p 8080:8080 my-java-app:1.0
```

**Input:**

```http
GET http://localhost:8080/hello?name=Cloud
```

**Output:**

```text
Hello, Cloud!
```

## 4. Ví dụ cấu hình CI/CD đơn giản (GitHub Actions)

File: `.github/workflows/maven.yml`

```yaml
name: Java CI

on:
  push:
    branches: [ "main" ]
  pull_request:
    branches: [ "main" ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Build with Maven
        run: mvn -B clean package --file pom.xml
```

Pipeline này:
- Tự động build khi push lên branch `main`.
- Đảm bảo code Java build được trước khi deploy lên AWS/GCP/Azure.

## 5. Tóm tắt ý chính

- Cần có **kiến thức cơ bản về ít nhất một cloud provider** (AWS, GCP, Azure): compute, storage, database, networking.
- Biết cách **đóng gói và deploy ứng dụng Java** (thường qua Docker) lên các dịch vụ cloud.
- Nắm **DevOps cơ bản**: CI/CD, quản lý cấu hình, logging/monitoring.
- Ví dụ thực tế: Spring Boot REST API, Dockerfile, pipeline CI đơn giản.

Nếu bạn cho biết cụ thể bạn muốn tập trung vào AWS, GCP hay Azure, tôi có thể đưa thêm ví dụ chi tiết cho provider đó (ví dụ file cấu hình deploy, lệnh CLI, v.v.).
