
Dưới đây là tóm tắt ngắn gọn về **Exceptions & Error Handling: try-with-resources, AutoCloseable** kèm ví dụ mã và input/output.

---

## 1. Khái niệm chính

1. **try-with-resources**  
   - Cú pháp đặc biệt của `try` trong Java (từ Java 7).  
   - Dùng để làm việc với các tài nguyên cần đóng (file, stream, connection, …).  
   - Tự động gọi `close()` sau khi khối `try` kết thúc, **kể cả khi có exception**.  
   - Giúp tránh rò rỉ tài nguyên (resource leak) và code gọn hơn.

2. **AutoCloseable**  
   - Interface có một phương thức duy nhất: `void close() throws Exception;`  
   - Bất kỳ class nào implement `AutoCloseable` đều có thể dùng trong `try-with-resources`.  
   - `java.io.Closeable` cũng tương tự, nhưng `AutoCloseable` là interface tổng quát hơn.

---

## 2. Cú pháp try-with-resources

```java
try (ResourceType resource = new ResourceType()) {
    // sử dụng resource
} catch (Exception e) {
    // xử lý lỗi
}
```

- Có thể khai báo nhiều resource:

```java
try (ResourceA a = new ResourceA();
     ResourceB b = new ResourceB()) {
    // dùng a, b
}
```

## 3. Ví dụ với File (thực tế)

### Ví dụ: Đọc file bằng try-with-resources

```java
import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class ReadFileExample {
    public static void main(String[] args) {
        String filePath = "input.txt";

        try (BufferedReader br = new BufferedReader(new FileReader(filePath))) {
            String line;
            while ((line = br.readLine()) != null) {
                System.out.println("Read line: " + line);
            }
        } catch (IOException e) {
            System.out.println("Có lỗi khi đọc file: " + e.getMessage());
        }
    }
}
```

**Giả sử file `input.txt` có nội dung:**
```text
Hello
Java
try-with-resources
```

**Output:**
```text
Read line: Hello
Read line: Java
Read line: try-with-resources
```

- `BufferedReader` và `FileReader` sẽ được tự động đóng sau khi khối `try` kết thúc.

## 4. Tự tạo class implement AutoCloseable

### Ví dụ: Tài nguyên tự định nghĩa

```java
class MyResource implements AutoCloseable {
    private String name;

    public MyResource(String name) {
        this.name = name;
        System.out.println("Mở resource: " + name);
    }

    public void doWork() {
        System.out.println("Đang làm việc với: " + name);
    }

    @Override
    public void close() throws Exception {
        System.out.println("Đóng resource: " + name);
    }
}

public class AutoCloseableExample {
    public static void main(String[] args) {
        try (MyResource r = new MyResource("Resource1")) {
            r.doWork();
        } catch (Exception e) {
            System.out.println("Có lỗi: " + e.getMessage());
        }
    }
}
```

**Output (dự kiến):**
```text
Mở resource: Resource1
Đang làm việc với: Resource1
Đóng resource: Resource1
```

- Dù có exception trong `doWork()` thì `close()` vẫn được gọi.

## 5. So sánh với try/finally truyền thống

**Cách cũ (dễ quên đóng):**

```java
BufferedReader br = null;
try {
    br = new BufferedReader(new FileReader("input.txt"));
    // đọc file
} catch (IOException e) {
    // xử lý
} finally {
    if (br != null) {
        try {
            br.close();
        } catch (IOException e) {
            // xử lý lỗi khi đóng
        }
    }
}
```

**Cách mới (try-with-resources):**

```java
try (BufferedReader br = new BufferedReader(new FileReader("input.txt"))) {
    // đọc file
} catch (IOException e) {
    // xử lý
}
```

- Code ngắn hơn, ít lỗi hơn, tự động đóng tài nguyên.

---

## 6. Ý chính cần nhớ

- Dùng `try-with-resources` khi làm việc với tài nguyên cần đóng (file, stream, socket, connection DB, …).
- Class dùng trong `try-with-resources` phải implement `AutoCloseable` (hoặc `Closeable`).
- `close()` được gọi tự động, giúp tránh rò rỉ tài nguyên và code sạch hơn.
