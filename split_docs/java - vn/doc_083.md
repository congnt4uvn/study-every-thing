
Dưới đây là tóm tắt ngắn gọn về cache trong JPA/Hibernate, tập trung vào 2 loại: **First-level cache** và **Second-level cache**, kèm ví dụ code và input/output.

---

## 1. First-level cache (L1 cache)

**Ý chính:**

- Gắn với **EntityManager** (JPA) hoặc **Session** (Hibernate).
- **Luôn luôn bật**, không cần cấu hình.
- Phạm vi: **một phiên làm việc** (một EntityManager/Session).
- Trong cùng một EntityManager, khi `find()`/`get()` cùng một entity (cùng class + id), Hibernate sẽ:
  - Lần đầu: truy vấn DB, lưu entity vào cache.
  - Lần sau: lấy từ cache, **không truy vấn DB nữa**.

### Ví dụ (JPA)

```java
EntityManager em = entityManagerFactory.createEntityManager();
em.getTransaction().begin();

// Lần 1: load từ DB
User user1 = em.find(User.class, 1L);

// Lần 2: cùng EntityManager, cùng id => lấy từ L1 cache
User user2 = em.find(User.class, 1L);

System.out.println(user1 == user2); // true

em.getTransaction().commit();
em.close();
```

**Input:**  
- Gọi `em.find(User.class, 1L)` hai lần trong cùng một `EntityManager`.

**Output (hành vi):**  
- SQL SELECT chỉ chạy **1 lần**.  
- `user1 == user2` là `true`.

## 2. Second-level cache (L2 cache)

**Ý chính:**

- Gắn với **SessionFactory/EntityManagerFactory** (toàn bộ ứng dụng).
- Phạm vi: **nhiều Session/EntityManager khác nhau**, thậm chí nhiều request.
- **Không tự động bật**: cần cấu hình + chọn provider (Ehcache, Infinispan, Caffeine, Hazelcast, v.v.).
- Lưu entity (và có thể là collection, query result) để giảm truy vấn DB giữa các phiên làm việc khác nhau.

### 2.1. Cấu hình Hibernate (ví dụ dùng Ehcache)

`application.properties` (Spring Boot):

```properties
spring.jpa.properties.hibernate.cache.use_second_level_cache=true
spring.jpa.properties.hibernate.cache.use_query_cache=true
spring.jpa.properties.hibernate.cache.region.factory_class=org.hibernate.cache.ehcache.EhCacheRegionFactory
spring.jpa.properties.javax.persistence.sharedCache.mode=ENABLE_SELECTIVE
```

Entity:

```java
import javax.persistence.*;
import org.hibernate.annotations.Cache;
import org.hibernate.annotations.CacheConcurrencyStrategy;

@Entity
@Cacheable
@Cache(usage = CacheConcurrencyStrategy.READ_WRITE)
public class User {
    @Id
    private Long id;

    private String name;
    // getters/setters
}
```

### 2.2. Ví dụ hành vi L2 cache

```java
// Session/EntityManager 1
EntityManager em1 = emf.createEntityManager();
em1.getTransaction().begin();

User u1 = em1.find(User.class, 1L); // Lần đầu: load từ DB, đưa vào L1 + L2
em1.getTransaction().commit();
em1.close();

// Session/EntityManager 2
EntityManager em2 = emf.createEntityManager();
em2.getTransaction().begin();

User u2 = em2.find(User.class, 1L); 
// Lần này: lấy từ L2 cache (nếu cache hit), không cần query DB

em2.getTransaction().commit();
em2.close();
```

**Input:**  
- `em1.find(User.class, 1L)` trong Session 1.  
- `em2.find(User.class, 1L)` trong Session 2 (khác).

**Output (hành vi):**  
- SQL SELECT chạy **1 lần** (ở Session 1).  
- Session 2 lấy từ **L2 cache**, không query DB (nếu cache được cấu hình đúng).

## 3. So sánh nhanh

| Đặc điểm                 | First-level cache (L1)              | Second-level cache (L2)                     |
|--------------------------|-------------------------------------|--------------------------------------------|
| Gắn với                  | EntityManager/Session               | EntityManagerFactory/SessionFactory        |
| Phạm vi                  | Một phiên làm việc                  | Nhiều phiên, toàn ứng dụng                 |
| Mặc định                 | Luôn bật                            | Phải cấu hình                              |
| Mục đích                 | Tránh query lặp trong 1 session     | Giảm query giữa nhiều session/request      |
| Cấu hình                 | Không cần                           | Cần chọn provider + cấu hình               |

---

## 4. Lưu ý quan trọng

- L1 cache: bạn **không tắt được**, nó là một phần của cơ chế persistence context.
- L2 cache:
  - Phải chọn chiến lược `CacheConcurrencyStrategy` phù hợp (`READ_ONLY`, `NONSTRICT_READ_WRITE`, `READ_WRITE`, `TRANSACTIONAL`).
  - Không nên cache dữ liệu thay đổi liên tục nếu không cần thiết.
  - Cần chú ý đến **invalidations** (khi update/delete) để tránh dữ liệu cũ.

Nếu bạn cho mình biết framework bạn dùng (Spring Boot, Java EE, plain Hibernate), mình có thể đưa ví dụ cấu hình chi tiết hơn.
