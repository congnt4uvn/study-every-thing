
Dưới đây là tóm tắt ngắn gọn về **Storage trong Cloud & DevOps (S3, GCS)**, kèm ví dụ code Java và input/output minh họa.

---

## 1. Khái niệm chung

- **Object Storage**: S3 (AWS) và GCS (Google Cloud Storage) đều là dịch vụ lưu trữ dạng object.
- Dùng để:
  - Lưu file (ảnh, video, log, backup…)
  - Lưu artifact build (JAR, WAR, Docker layer…)
  - Lưu dữ liệu tĩnh cho web/app
- Đặc điểm:
  - Lưu trong **bucket**
  - Mỗi file là một **object** có **key (tên)** và **metadata**
  - Truy cập qua HTTP/HTTPS, SDK, CLI
  - Tích hợp tốt với CI/CD, hệ thống log, data lake…

---

## 2. Amazon S3 – Key points

- **Bucket**: không gian lưu trữ, tên global unique.
- **Object**: file + metadata, được định danh bằng `bucket + key`.
- **Tính năng chính**:
  - Versioning (lưu nhiều phiên bản)
  - Lifecycle rules (tự động chuyển lớp lưu trữ, xóa sau X ngày)
  - Access control (IAM, bucket policy, ACL)
  - Static website hosting
- Dùng nhiều trong DevOps:
  - Lưu artifact build
  - Lưu log (CloudWatch → S3)
  - Lưu backup DB

### Ví dụ Java với AWS S3

**Maven dependency:**

```xml
<dependency>
    <groupId>software.amazon.awssdk</groupId>
    <artifactId>s3</artifactId>
    <version>2.25.0</version> <!-- phiên bản ví dụ -->
</dependency>
```

#### 2.1. Upload file lên S3

```java
import software.amazon.awssdk.auth.credentials.ProfileCredentialsProvider;
import software.amazon.awssdk.regions.Region;
import software.amazon.awssdk.services.s3.S3Client;
import software.amazon.awssdk.services.s3.model.PutObjectRequest;

import java.nio.file.Paths;

public class S3UploadExample {
    public static void main(String[] args) {
        Region region = Region.AP_SOUTHEAST_1; // ví dụ: Singapore
        S3Client s3 = S3Client.builder()
                .region(region)
                .credentialsProvider(ProfileCredentialsProvider.create("default"))
                .build();

        String bucketName = "my-demo-bucket";
        String key = "demo-folder/hello.txt";
        String filePath = "/path/to/local/hello.txt";

        PutObjectRequest putRequest = PutObjectRequest.builder()
                .bucket(bucketName)
                .key(key)
                .build();

        s3.putObject(putRequest, Paths.get(filePath));

        System.out.println("Uploaded " + filePath + " to s3://" + bucketName + "/" + key);
    }
}
```

**Input:**
- File local: `/path/to/local/hello.txt` (ví dụ nội dung: `Xin chào S3`)

**Output (console):**
```text
Uploaded /path/to/local/hello.txt to s3://my-demo-bucket/demo-folder/hello.txt
```

Object sẽ xuất hiện trong bucket `my-demo-bucket` với key `demo-folder/hello.txt`.

#### 2.2. Download file từ S3

```java
import software.amazon.awssdk.services.s3.model.GetObjectRequest;
import java.nio.file.Paths;

public class S3DownloadExample {
    public static void main(String[] args) {
        S3Client s3 = S3Client.create();

        String bucketName = "my-demo-bucket";
        String key = "demo-folder/hello.txt";
        String downloadPath = "/tmp/hello-downloaded.txt";

        GetObjectRequest getRequest = GetObjectRequest.builder()
                .bucket(bucketName)
                .key(key)
                .build();

        s3.getObject(getRequest, Paths.get(downloadPath));

        System.out.println("Downloaded to " + downloadPath);
    }
}
```

## 3. Google Cloud Storage (GCS) – Key points

- Tương tự S3: bucket + object.
- Tích hợp chặt với:
  - GKE, Cloud Run, Cloud Functions
  - BigQuery, Dataflow, AI/ML
- Tính năng:
  - Storage class (Standard, Nearline, Coldline, Archive)
  - Lifecycle management
  - IAM & ACL
  - Signed URL để chia sẻ file tạm thời

### Ví dụ Java với GCS

**Maven dependency:**

```xml
<dependency>
    <groupId>com.google.cloud</groupId>
    <artifactId>google-cloud-storage</artifactId>
    <version>2.40.0</version> <!-- phiên bản ví dụ -->
</dependency>
```

#### 3.1. Upload file lên GCS

```java
import com.google.cloud.storage.BlobInfo;
import com.google.cloud.storage.Storage;
import com.google.cloud.storage.StorageOptions;

import java.nio.file.Files;
import java.nio.file.Paths;

public class GcsUploadExample {
    public static void main(String[] args) throws Exception {
        Storage storage = StorageOptions.getDefaultInstance().getService();

        String bucketName = "my-demo-bucket-gcs";
        String objectName = "demo-folder/hello.txt";
        String filePath = "/path/to/local/hello.txt";

        byte[] bytes = Files.readAllBytes(Paths.get(filePath));

        BlobInfo blobInfo = BlobInfo.newBuilder(bucketName, objectName).build();
        storage.create(blobInfo, bytes);

        System.out.println("Uploaded " + filePath + " to gs://" + bucketName + "/" + objectName);
    }
}
```

**Input:**
- File local: `/path/to/local/hello.txt` (ví dụ nội dung: `Xin chào GCS`)

**Output (console):**
```text
Uploaded /path/to/local/hello.txt to gs://my-demo-bucket-gcs/demo-folder/hello.txt
```

#### 3.2. Download file từ GCS

```java
import com.google.cloud.storage.Blob;
import com.google.cloud.storage.Storage;
import com.google.cloud.storage.StorageOptions;

import java.nio.file.Files;
import java.nio.file.Paths;

public class GcsDownloadExample {
    public static void main(String[] args) throws Exception {
        Storage storage = StorageOptions.getDefaultInstance().getService();

        String bucketName = "my-demo-bucket-gcs";
        String objectName = "demo-folder/hello.txt";
        String downloadPath = "/tmp/hello-gcs-downloaded.txt";

        Blob blob = storage.get(bucketName, objectName);
        byte[] content = blob.getContent();

        Files.write(Paths.get(downloadPath), content);

        System.out.println("Downloaded to " + downloadPath);
    }
}
```

## 4. S3 vs GCS – Tóm tắt so sánh nhanh

- **Giống nhau**:
  - Object storage, bucket/object, HTTP API, SDK đa ngôn ngữ.
  - Hỗ trợ versioning, lifecycle, IAM, public/private access.
- **Khác nhau**:
  - S3 thuộc AWS, GCS thuộc GCP.
  - Tên API, SDK, CLI khác nhau.
  - Tích hợp dịch vụ xung quanh khác (EC2/Lambda vs GCE/Cloud Run…).

---

## 5. Ý nghĩa trong DevOps

- Lưu artifact build (JAR, WAR, image layer, Helm chart…)
- Lưu log, metrics, backup, snapshot.
- Lưu file cấu hình, static assets cho web.
- Tích hợp trong pipeline CI/CD:
  - Bước build → upload artifact lên S3/GCS
  - Bước deploy → tải artifact từ S3/GCS

Nếu bạn muốn, tôi có thể viết thêm ví dụ:
- Tạo bucket bằng Java
- Liệt kê object trong bucket
- Tạo URL tạm thời (presigned URL / signed URL) để tải file.
