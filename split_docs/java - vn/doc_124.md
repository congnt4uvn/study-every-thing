## 1. Công cụ & Hệ sinh thái trong Java

### Maven và Gradle

Đây là 2 công cụ build phổ biến nhất trong Java, dùng để:

- Quản lý vòng đời build (lifecycle)
- Quản lý dependencies (thư viện bên ngoài)
- Quản lý profiles (cấu hình khác nhau cho dev/test/prod)

---

## 2. Maven

### a. Vòng đời (Lifecycle) cơ bản

Một số phase quan trọng:

- `clean` – Xóa thư mục build cũ (`target`)
- `compile` – Biên dịch mã nguồn
- `test` – Chạy test
- `package` – Đóng gói (jar/war)
- `install` – Cài vào local repository
- `deploy` – Đẩy lên remote repository

Chạy lệnh:

```bash
mvn clean package
```

**Ý nghĩa**: Xóa build cũ, sau đó biên dịch, test, và đóng gói.

### b. Quản lý dependencies

Trong `pom.xml`:

```xml
<dependencies>
    <!-- Thư viện JUnit để test -->
    <dependency>
        <groupId>org.junit.jupiter</groupId>
        <artifactId>junit-jupiter</artifactId>
        <version>5.10.0</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

Maven sẽ tự động tải thư viện về và thêm vào classpath.

### c. Profiles (cấu hình môi trường)

Dùng để thay đổi cấu hình theo môi trường (dev, test, prod):

```xml
<profiles>
    <profile>
        <id>dev</id>
        <properties>
            <env.name>development</env.name>
        </properties>
    </profile>

    <profile>
        <id>prod</id>
        <properties>
            <env.name>production</env.name>
        </properties>
    </profile>
</profiles>
```

Chạy với profile:

```bash
mvn clean package -Pdev
mvn clean package -Pprod
```

## 3. Gradle

### a. Vòng đời (Tasks)

Gradle dùng “task” thay vì “phase”:

- `clean` – Xóa thư mục build
- `build` – Biên dịch, test, đóng gói
- `test` – Chạy test
- `assemble` – Đóng gói (không chạy test)

Chạy lệnh:

```bash
gradle clean build
```

### b. Quản lý dependencies

Trong `build.gradle` (Groovy DSL):

```groovy
plugins {
    id 'java'
}

repositories {
    mavenCentral()
}

dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter:5.10.0'
}
```

Gradle sẽ tải thư viện từ Maven Central.

### c. Profiles tương đương (Gradle)

Gradle không có “profile” như Maven, nhưng có thể dùng:

- `-P` project properties
- Nhiều file cấu hình (ví dụ: `application-dev.yml`, `application-prod.yml`)
- Hoặc `if (project.hasProperty('env')) { ... }` trong `build.gradle`

Ví dụ:

```groovy
if (project.hasProperty('env') && project.env == 'prod') {
    println "Building for PRODUCTION"
} else {
    println "Building for DEVELOPMENT"
}
```

Chạy:

```bash
gradle build -Penv=prod
gradle build -Penv=dev
```

## 4. Ví dụ đơn giản: Chương trình Java + Maven

### Cấu trúc

```text
my-app/
 ├─ src/
 │   └─ main/
 │       └─ java/
 │           └─ com/example/App.java
 └─ pom.xml
```

`App.java`:

```java
package com.example;

public class App {
    public static void main(String[] args) {
        System.out.println("Xin chào Maven!");
    }
}
```

`pom.xml` tối thiểu:

```xml
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
                             http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <artifactId>my-app</artifactId>
    <version>1.0-SNAPSHOT</version>
    <packaging>jar</packaging>

    <properties>
        <maven.compiler.source>17</maven.compiler.source>
        <maven.compiler.target>17</maven.compiler.target>
    </properties>
</project>
```

### Build & chạy

```bash
mvn clean package
java -cp target/my-app-1.0-SNAPSHOT.jar com.example.App
```

**Output:**

```text
Xin chào Maven!
```
