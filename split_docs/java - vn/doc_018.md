## Java 19

### 2. Virtual Threads (Preview – JEP 425, hoàn thiện ở Java 21)
Luồng nhẹ, tạo hàng chục nghìn luồng mà không tốn nhiều tài nguyên như Thread truyền thống.

```java
public class VirtualThreadDemo {
    public static void main(String[] args) throws Exception {
        try (var executor = java.util.concurrent.Executors.newVirtualThreadPerTaskExecutor()) {
            for (int i = 0; i < 5; i++) {
                int id = i;
                executor.submit(() -> {
                    System.out.println("Hello from virtual thread " + id);
                    Thread.sleep(100);
                    return null;
                });
            }
        }
    }
}
```

**Output (ví dụ):**
```text
Hello from virtual thread 0
Hello from virtual thread 1
Hello from virtual thread 2
Hello from virtual thread 3
Hello from virtual thread 4
```

## Java 21 (LTS – rất quan trọng)

### 3. Virtual Threads (JEP 444 – chính thức)
Dùng để xử lý I/O đồng thời lớn (web server, service gọi API, DB…).

```java
public class VirtualThreadServer {
    public static void main(String[] args) throws Exception {
        try (var executor = java.util.concurrent.Executors.newVirtualThreadPerTaskExecutor()) {
            for (int i = 0; i < 100_000; i++) {
                int id = i;
                executor.submit(() -> {
                    // giả lập xử lý request
                    Thread.sleep(10);
                    return "Done " + id;
                });
            }
        }
        System.out.println("All tasks submitted");
    }
}
```

**Ý chính:** Có thể tạo hàng chục/hàng trăm nghìn virtual thread mà vẫn nhẹ.

### 4. Pattern Matching cho `switch` (JEP 441)

Cho phép `switch` theo kiểu (type), kết hợp với `instanceof` một cách gọn hơn.

```java
static String format(Object obj) {
    return switch (obj) {
        case Integer i -> "Số nguyên: " + i;
        case String s  -> "Chuỗi: " + s.toUpperCase();
        case null      -> "Giá trị null";
        default        -> "Kiểu khác: " + obj;
    };
}

public static void main(String[] args) {
    System.out.println(format(10));
    System.out.println(format("hello"));
    System.out.println(format(null));
}
```

**Output:**
```text
Số nguyên: 10
Chuỗi: HELLO
Giá trị null
```

### 5. Record Patterns (JEP 440)

Giúp “bóc tách” (deconstruct) record trực tiếp trong pattern.

```java
record Point(int x, int y) {}

static void printPoint(Object obj) {
    switch (obj) {
        case Point(int x, int y) -> System.out.println("Point: x=" + x + ", y=" + y);
        default -> System.out.println("Không phải Point");
    }
}

public static void main(String[] args) {
    printPoint(new Point(3, 4));
    printPoint("abc");
}
```

**Output:**
```text
Point: x=3, y=4
Không phải Point
```

### 6. Sequenced Collections (JEP 431)

Thêm interface mới: `SequencedCollection`, `SequencedSet`, `SequencedMap`  
→ Hỗ trợ thao tác đầu/cuối (first/last) một cách thống nhất.

```java
import java.util.*;

public class SequencedDemo {
    public static void main(String[] args) {
        SequencedCollection<String> list = (SequencedCollection<String>) new ArrayList<String>();
        list.addFirst("b");
        list.addLast("c");
        list.addFirst("a");

        System.out.println(list);          // [a, b, c]
        System.out.println(list.getFirst()); // a
        System.out.println(list.getLast());  // c
    }
}
```

### 7. String Templates (Preview – JEP 430)

Giúp nội suy chuỗi (string interpolation) an toàn hơn, linh hoạt hơn.

```java
// Cần bật preview khi compile/run
// javac --release 21 --enable-preview Main.java
// java --enable-preview Main

import static java.lang.StringTemplate.STR;

public class TemplateDemo {
    public static void main(String[] args) {
        int x = 10;
        int y = 20;
        String s = STR."Tổng của \{x} và \{y} là \{x + y}";
        System.out.println(s);
    }
}
```

**Output:**
```text
Tổng của 10 và 20 là 30
```

### 8. Structured Concurrency (Preview – JEP 453)

Quản lý nhiều task bất đồng bộ theo “khối” (scope) rõ ràng.

```java
import java.time.Duration;
import java.util.concurrent.*;

public class StructuredDemo {
    static String fetchUser() throws InterruptedException {
        Thread.sleep(100);
        return "User A";
    }

    static String fetchOrders() throws InterruptedException {
        Thread.sleep(150);
        return "3 orders";
    }

    public static void main(String[] args) throws Exception {
        try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
            var userFuture = scope.fork(() -> fetchUser());
            var ordersFuture = scope.fork(() -> fetchOrders());

            scope.join();          // chờ cả hai
            scope.throwIfFailed(); // ném exception nếu có task fail

            String user = userFuture.resultNow();
            String orders = ordersFuture.resultNow();

            System.out.println(user + " - " + orders);
        }
    }
}
```

**Output:**
```text
User A - 3 orders
```

## Tóm tắt ý chính

- **Java 18**: Simple Web Server để test nhanh HTTP.
- **Java 19–20**: Chuẩn bị, preview cho các tính năng lớn.
- **Java 21 (LTS)**:
  - Virtual Threads: xử lý đồng thời cực lớn, nhẹ.
  - Pattern Matching cho `switch`: code ngắn, rõ, theo kiểu.
  - Record Patterns: bóc tách record trực tiếp trong `switch`/`instanceof`.
  - Sequenced Collections: thao tác đầu/cuối cho List/Set/Map thống nhất.
  - String Templates (preview): nội suy chuỗi an toàn, linh hoạt.
  - Structured Concurrency (preview): quản lý nhiều task bất đồng bộ theo scope.

Nếu bạn cho biết bạn đang dùng Java phiên bản nào và làm loại ứng dụng gì (web, desktop, microservice…), tôi có thể gợi ý cụ thể nên dùng tính năng nào và ví dụ sát hơn.
