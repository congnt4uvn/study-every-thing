---

## 1. Git là gì?

Git là hệ thống quản lý phiên bản (Version Control System) giúp bạn:

- Lưu lại lịch sử thay đổi mã nguồn  
- Làm việc theo nhánh (branch)  
- Hợp nhất (merge) hoặc sắp xếp lại lịch sử (rebase)  
- Hợp tác qua Pull Request (thường trên GitHub, GitLab, Bitbucket)

---

## 2. Branching (Tạo và làm việc với nhánh)

**Ý chính:**

- Mỗi nhánh là một “dòng phát triển” độc lập.  
- Thường có nhánh `main`/`master` và các nhánh tính năng như `feature/login`.

**Lệnh cơ bản:**

```bash
# Tạo nhánh mới
git branch feature/login

# Chuyển sang nhánh đó
git checkout feature/login
# hoặc (Git mới)
git switch feature/login

# Tạo và chuyển nhánh cùng lúc
git checkout -b feature/login
```

**Ví dụ:**

- Đầu vào: `git branch`  
- Đầu ra (ví dụ):

```text
  feature/login
* main
```

Dấu `*` chỉ nhánh hiện tại.

---

## 3. Merging (Hợp nhất nhánh)

**Ý chính:**

- Dùng để đưa thay đổi từ một nhánh (ví dụ `feature/login`) vào nhánh khác (ví dụ `main`).  
- Có thể tạo commit merge hoặc fast-forward.

**Quy trình:**

```bash
# 1. Chuyển sang nhánh đích (ví dụ: main)
git checkout main

# 2. Hợp nhất nhánh feature/login vào main
git merge feature/login
```

**Kết quả:**

- Nếu không có xung đột: Git tạo một commit merge (hoặc fast-forward).  
- Nếu có xung đột: Git yêu cầu bạn sửa file bị conflict, sau đó:

```bash
git add <file_conflict_da_sua>
git commit
```

---

## 4. Rebasing (Sắp xếp lại lịch sử)

**Ý chính:**

- `rebase` “đặt lại” các commit của bạn lên trên một nhánh khác, tạo lịch sử thẳng, sạch hơn.  
- Thường dùng để cập nhật nhánh feature theo nhánh `main` mới nhất.

**Quy trình:**

```bash
# Đang ở nhánh feature/login
git checkout feature/login

# Rebase nhánh feature/login lên main
git rebase main
```

**Kết quả:**

- Lịch sử commit của `feature/login` được “phát lại” trên đầu `main`.  
- Nếu có xung đột: sửa file, rồi:

```bash
git add <file_conflict_da_sua>
git rebase --continue
```

---

## 5. Pull Requests (PR)

**Ý chính:**

- Pull Request là yêu cầu hợp nhất code từ một nhánh vào nhánh khác (thường trên GitHub/GitLab).  
- Dùng để review code, chạy CI, thảo luận trước khi merge.

**Quy trình điển hình:**

1. Tạo nhánh mới, code, commit.  
2. Đẩy nhánh lên remote:

   ```bash
   git push -u origin feature/login
   ```

3. Lên GitHub/GitLab → tạo Pull Request từ `feature/login` vào `main`.  
4. Reviewer xem code, comment, yêu cầu sửa.  
5. Sau khi OK → nhấn “Merge” PR.

**Ví dụ (dòng lệnh để cập nhật PR):**

```bash
# Sau khi sửa theo review
git add .
git commit -m "Fix review comments"
git push
```

PR sẽ tự cập nhật với commit mới.

---

## 6. Tóm tắt ngắn

- **Git**: công cụ quản lý phiên bản phổ biến.  
- **Branching**: tách nhánh để phát triển tính năng độc lập.  
- **Merging**: hợp nhất nhánh, có thể tạo commit merge.  
- **Rebasing**: sắp xếp lại lịch sử, giúp lịch sử thẳng và sạch.  
- **Pull Requests**: quy trình hợp tác, review và merge code trên nền tảng như GitHub.
