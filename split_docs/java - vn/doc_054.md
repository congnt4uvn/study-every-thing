
Dưới đây là tóm tắt ngắn gọn về nội dung “Performance Tuning với Profiling tools” trong Java, kèm ví dụ:

---

## 1. Ý chính

- **Mục tiêu**: Tối ưu hiệu năng ứng dụng Java (giảm CPU, RAM, GC, thời gian phản hồi).
- **Cách làm**: Dùng **profiling tools** để:
  - Theo dõi **CPU usage** (method nào tốn thời gian).
  - Theo dõi **Memory usage** (object nào chiếm nhiều RAM, rò rỉ bộ nhớ).
  - Phân tích **Garbage Collection (GC)**.
  - Xem **thread** (deadlock, contention).
- **Công cụ phổ biến** (chỉ cần nắm vững ít nhất 1 cái):
  - **VisualVM** (miễn phí, đi kèm JDK hoặc tải thêm).
  - **Java Mission Control (JMC)** (phân tích flight recording, hiệu năng thấp).
  - **YourKit** (thương mại, giao diện mạnh).
  - **JProfiler** (thương mại, nhiều tính năng profiling chi tiết).

---

## 2. Quy trình cơ bản khi dùng profiler

1. Chạy ứng dụng Java (local hoặc server).
2. Mở profiler (VD: VisualVM).
3. **Attach** vào process Java đang chạy.
4. Bật **CPU profiling** hoặc **Memory profiling**.
5. Thực hiện các thao tác gây tải (load test, dùng thật).
6. Xem:
   - Method nào chiếm nhiều thời gian.
   - Class/object nào chiếm nhiều bộ nhớ.
   - GC có chạy quá nhiều không.
7. Tối ưu code (thuật toán, cấu trúc dữ liệu, cache, giảm allocation…).
8. Chạy lại profiler để kiểm tra hiệu quả.

---

## 3. Ví dụ đơn giản: Code gây vấn đề hiệu năng

### 3.1. Ví dụ CPU-bound (thuật toán chậm)

```java
public class CpuHeavyExample {
    public static void main(String[] args) {
        long start = System.currentTimeMillis();
        long sum = 0;

        // Vòng lặp rất lớn, thuật toán O(n^2) không cần thiết
        for (int i = 0; i < 50_000; i++) {
            for (int j = 0; j < 50_000; j++) {
                sum += (i + j) % 3;
            }
        }

        long end = System.currentTimeMillis();
        System.out.println("Sum = " + sum);
        System.out.println("Time = " + (end - start) + " ms");
    }
}
```

**Input**: không có (chỉ chạy chương trình).  
**Output (ví dụ)**:
```text
Sum = 249999750000
Time = 12345 ms
```

Khi dùng profiler (VisualVM/JProfiler/YourKit/JMC), bạn sẽ thấy:
- Method `main` (hoặc block lặp) chiếm gần như toàn bộ CPU.
- Có thể tối ưu bằng cách:
  - Giảm độ phức tạp thuật toán.
  - Dùng công thức toán học thay vì lặp lồng nhau nếu có thể.

---

### 3.2. Ví dụ Memory leak / allocation nhiều

```java
import java.util.ArrayList;
import java.util.List;

public class MemoryLeakExample {
    private static final List<byte[]> storage = new ArrayList<>();

    public static void main(String[] args) throws InterruptedException {
        while (true) {
            // Mỗi vòng lặp thêm 1MB vào list, không bao giờ xóa
            storage.add(new byte[1024 * 1024]);
            System.out.println("Allocated 1MB, total = " + storage.size() + " MB");

            Thread.sleep(500);
        }
    }
}
```

**Input**: không có.  
**Output (ví dụ)**:
```text
Allocated 1MB, total = 1 MB
Allocated 1MB, total = 2 MB
Allocated 1MB, total = 3 MB
...
Exception in thread "main" java.lang.OutOfMemoryError: Java heap space
```

Khi dùng profiler:
- Biểu đồ heap tăng liên tục.
- Tab **Memory** cho thấy `byte[]` và `ArrayList` chiếm phần lớn bộ nhớ.
- Có thể sửa bằng:
  - Không giữ reference lâu (xóa khỏi list, dùng weak reference, hoặc thiết kế lại logic).

---

## 4. Cách dùng VisualVM (ví dụ nhanh)

1. Cài VisualVM (nhiều JDK đã có sẵn `jvisualvm`).
2. Chạy ứng dụng Java.
3. Mở VisualVM → chọn process ứng dụng.
4. Vào tab:
   - **Monitor**: xem CPU, heap, GC.
   - **Sampler** hoặc **Profiler**:
     - CPU: xem method nào tốn thời gian.
     - Memory: xem class nào chiếm nhiều bộ nhớ, tạo nhiều object.
5. Dựa trên kết quả để chỉnh sửa code.

---

## 5. Tóm tắt

- Performance tuning trong Java **không nên đoán mò**, mà nên dùng **profiling tools**.
- Biết dùng ít nhất **1 công cụ** (VisualVM, JMC, YourKit, JProfiler) để:
  - Phân tích CPU, Memory, GC, Thread.
  - Tìm “nút thắt cổ chai” (bottleneck).
- Sau khi tối ưu, **profiling lại** để xác nhận hiệu quả.
