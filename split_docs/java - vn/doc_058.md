
Dưới đây là tóm tắt ngắn gọn về Spring Core: Bean Scope, Lifecycle, @PostConstruct, @PreDestroy, kèm ví dụ code và input/output.

---

## 1. Bean Scope (Phạm vi của Bean)

Spring định nghĩa bean và quản lý vòng đời của chúng. Scope quyết định **bean được tạo bao nhiêu lần** và **sống ở đâu**.

Các scope thường dùng:

1. **singleton** (mặc định)  
   - Chỉ tạo **1 instance** cho mỗi Spring Container.  
   - Mọi chỗ `@Autowired` cùng type sẽ dùng chung 1 object.

2. **prototype**  
   - Mỗi lần `getBean()` hoặc `@Autowired` sẽ tạo **một instance mới**.

3. (Trong web) **request**, **session**, **application**  
   - `request`: 1 bean cho mỗi HTTP request  
   - `session`: 1 bean cho mỗi HTTP session  
   - `application`: 1 bean cho toàn bộ ServletContext

### Ví dụ: Bean scope

```java
import org.springframework.context.annotation.Scope;
import org.springframework.stereotype.Component;

@Component
@Scope("singleton") // hoặc "prototype"
public class MyService {
}
```

#### Test scope

```java
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class ScopeTest {
    public static void main(String[] args) {
        AnnotationConfigApplicationContext context =
                new AnnotationConfigApplicationContext(AppConfig.class);

        MyService s1 = context.getBean(MyService.class);
        MyService s2 = context.getBean(MyService.class);

        System.out.println("s1 == s2 ? " + (s1 == s2));

        context.close();
    }
}
```

**Output (với @Scope("singleton")):**
```text
s1 == s2 ? true
```

**Output (với @Scope("prototype")):**
```text
s1 == s2 ? false
```

## 2. Bean Lifecycle (Vòng đời của Bean)

Vòng đời cơ bản của một bean trong Spring:

1. Tạo instance (constructor)
2. Inject dependencies (setter, constructor, field)
3. Gọi các callback khởi tạo (init):
   - `@PostConstruct`
   - hoặc `InitializingBean.afterPropertiesSet()`
   - hoặc `init-method` trong XML / @Bean
4. Bean sẵn sàng sử dụng
5. Khi container shutdown:
   - Gọi các callback hủy (destroy):
     - `@PreDestroy`
     - hoặc `DisposableBean.destroy()`
     - hoặc `destroy-method`

---

## 3. @PostConstruct (Khởi tạo sau khi inject xong)

- Đánh dấu method sẽ được gọi **sau khi Spring inject xong tất cả dependencies**.
- Dùng cho logic khởi tạo: mở kết nối, load dữ liệu, validate config,…

```java
import jakarta.annotation.PostConstruct;
import jakarta.annotation.PreDestroy;
import org.springframework.stereotype.Component;

@Component
public class MyBean {

    public MyBean() {
        System.out.println("Constructor: MyBean created");
    }

    @PostConstruct
    public void init() {
        System.out.println("@PostConstruct: init() called");
        // Ví dụ: mở kết nối, load cache, ...
    }

    @PreDestroy
    public void cleanup() {
        System.out.println("@PreDestroy: cleanup() called");
        // Ví dụ: đóng kết nối, giải phóng tài nguyên, ...
    }
}
```

### Chạy thử

```java
import org.springframework.context.annotation.AnnotationConfigApplicationContext;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;

@Configuration
@ComponentScan(basePackages = "com.example")
public class AppConfig {
}

public class LifecycleTest {
    public static void main(String[] args) {
        AnnotationConfigApplicationContext context =
                new AnnotationConfigApplicationContext(AppConfig.class);

        MyBean bean = context.getBean(MyBean.class);
        System.out.println("Bean is ready to use");

        context.close(); // trigger @PreDestroy
    }
}
```

**Output (với scope singleton):**
```text
Constructor: MyBean created
@PostConstruct: init() called
Bean is ready to use
@PreDestroy: cleanup() called
```

## 4. @PreDestroy (Dọn dẹp trước khi bean bị hủy)

- Đánh dấu method sẽ được gọi **trước khi bean bị hủy** (khi container shutdown).
- Dùng để:
  - Đóng connection (DB, message queue, file,…)
  - Giải phóng tài nguyên (thread pool, socket,…)

Lưu ý:

- Với **singleton**, `@PreDestroy` được gọi khi `context.close()`.
- Với **prototype**, Spring **không tự động** gọi `@PreDestroy`; bạn phải tự quản lý vòng đời.

## 5. Ví dụ đầy đủ: singleton vs prototype + lifecycle

```java
@Component
@Scope("prototype")
public class PrototypeBean {

    public PrototypeBean() {
        System.out.println("PrototypeBean: constructor");
    }

    @PostConstruct
    public void init() {
        System.out.println("PrototypeBean: @PostConstruct");
    }

    @PreDestroy
    public void destroy() {
        System.out.println("PrototypeBean: @PreDestroy");
    }
}

public class PrototypeLifecycleTest {
    public static void main(String[] args) {
        AnnotationConfigApplicationContext context =
                new AnnotationConfigApplicationContext(AppConfig.class);

        PrototypeBean p1 = context.getBean(PrototypeBean.class);
        PrototypeBean p2 = context.getBean(PrototypeBean.class);

        System.out.println("p1 == p2 ? " + (p1 == p2));

        context.close(); // @PreDestroy của prototype thường không được gọi
    }
}
```

**Output (thường thấy):**
```text
PrototypeBean: constructor
PrototypeBean: @PostConstruct
PrototypeBean: constructor
PrototypeBean: @PostConstruct
p1 == p2 ? false
```

(Không có log `@PreDestroy` vì Spring không quản lý destroy cho prototype.)

---

### Tóm tắt ý chính

- **Bean scope** quyết định số lượng instance và phạm vi sống (singleton, prototype, request, session,…).
- **Lifecycle** gồm: tạo bean → inject dependency → init (@PostConstruct) → sử dụng → destroy (@PreDestroy).
- `@PostConstruct`: chạy sau khi inject xong, dùng để khởi tạo nâng cao.
- `@PreDestroy`: chạy trước khi bean bị hủy, dùng để dọn dẹp tài nguyên.
- Singleton: Spring quản lý đầy đủ init + destroy.  
- Prototype: Spring chỉ quản lý tạo + init, không tự destroy.
