
Dưới đây là tóm tắt ngắn gọn về **Spring Data JPA** với ví dụ mã và input/output.

---

## 1. Spring Data JPA là gì?

Spring Data JPA giúp làm việc với cơ sở dữ liệu (persistence) dễ dàng hơn bằng cách:
- Tự động tạo các phương thức CRUD (Create, Read, Update, Delete).
- Hỗ trợ viết query bằng cách đặt tên method (query methods).
- Cho phép viết query tùy chỉnh với annotation `@Query`.

---

## 2. CrudRepository

`CrudRepository` cung cấp các hàm CRUD cơ bản.

### Ví dụ Entity

```java
import jakarta.persistence.*;

@Entity
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;
    private String email;

    // getters, setters, constructors
}
```

### Ví dụ Repository với CrudRepository

```java
import org.springframework.data.repository.CrudRepository;

public interface UserRepository extends CrudRepository<User, Long> {
}
```

### Các method chính (mặc định có sẵn)

- `save(S entity)` – lưu mới hoặc cập nhật
- `findById(ID id)` – tìm theo id
- `findAll()` – lấy tất cả
- `deleteById(ID id)` – xóa theo id
- `count()` – đếm số bản ghi

#### Ví dụ sử dụng + Input/Output

```java
@Autowired
private UserRepository userRepository;

public void demoCrud() {
    // INPUT: tạo user mới
    User u = new User();
    u.setName("An");
    u.setEmail("an@example.com");

    // SAVE
    User saved = userRepository.save(u);
    System.out.println(saved.getId()); 
    // OUTPUT: id được sinh, ví dụ: 1

    // FIND
    User found = userRepository.findById(1L).orElse(null);
    System.out.println(found.getName());
    // OUTPUT: "An"

    // DELETE
    userRepository.deleteById(1L);
}
```

## 3. JpaRepository

`JpaRepository` mở rộng `CrudRepository` và `PagingAndSortingRepository`, cung cấp thêm:
- Phân trang, sắp xếp
- Một số method tiện ích khác

### Ví dụ Repository với JpaRepository

```java
import org.springframework.data.jpa.repository.JpaRepository;

public interface UserRepository extends JpaRepository<User, Long> {
}
```

### Một số method thêm

- `List<User> findAll()` – trả về `List` thay vì `Iterable`
- `List<User> findAll(Sort sort)`
- `Page<User> findAll(Pageable pageable)`
- `void flush()`
- `User saveAndFlush(User entity)`

#### Ví dụ phân trang + Input/Output

```java
import org.springframework.data.domain.Page;
import org.springframework.data.domain.PageRequest;

public void demoPaging() {
    PageRequest pageRequest = PageRequest.of(0, 2); // page 0, size 2
    Page<User> page = userRepository.findAll(pageRequest);

    System.out.println(page.getTotalElements()); // tổng số bản ghi
    System.out.println(page.getContent().size()); // số bản ghi trong trang hiện tại
}
```

## 4. Query Methods (đặt tên method để tạo query)

Spring Data JPA tự sinh SQL dựa trên tên method.

### Ví dụ

```java
public interface UserRepository extends JpaRepository<User, Long> {

    // Tìm theo tên
    List<User> findByName(String name);

    // Tìm theo email chứa chuỗi
    List<User> findByEmailContaining(String keyword);

    // Tìm theo tên và email
    List<User> findByNameAndEmail(String name, String email);

    // Tìm theo tên, sắp xếp theo email
    List<User> findByNameOrderByEmailAsc(String name);
}
```

#### Ví dụ sử dụng + Input/Output

```java
public void demoQueryMethods() {
    List<User> users = userRepository.findByName("An");
    // INPUT: name = "An"
    // OUTPUT: danh sách User có name = "An"

    List<User> gmailUsers = userRepository.findByEmailContaining("@gmail.com");
    // INPUT: keyword = "@gmail.com"
    // OUTPUT: tất cả User có email chứa "@gmail.com"
}
```

## 5. @Query – viết query tùy chỉnh

Khi query method không đủ, dùng `@Query` để viết JPQL hoặc native SQL.

### Ví dụ JPQL

```java
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;

public interface UserRepository extends JpaRepository<User, Long> {

    @Query("SELECT u FROM User u WHERE u.email = :email")
    User findByEmailExact(@Param("email") String email);

    @Query("SELECT u FROM User u WHERE u.name LIKE %:name%")
    List<User> searchByName(@Param("name") String name);
}
```

### Ví dụ native query

```java
    @Query(value = "SELECT * FROM user WHERE email LIKE %:domain%", nativeQuery = true)
    List<User> findByEmailDomain(@Param("domain") String domain);
```

#### Ví dụ sử dụng + Input/Output

```java
public void demoCustomQuery() {
    User u = userRepository.findByEmailExact("an@example.com");
    // INPUT: email = "an@example.com"
    // OUTPUT: User có email đúng bằng "an@example.com" hoặc null

    List<User> users = userRepository.searchByName("An");
    // INPUT: name = "An"
    // OUTPUT: User có name chứa "An", ví dụ "An", "Anh", "An Nguyen"
}
```

---

## Tóm tắt ý chính

- **CrudRepository**: cung cấp các thao tác CRUD cơ bản.
- **JpaRepository**: mở rộng CrudRepository, hỗ trợ phân trang, sắp xếp, thêm tiện ích.
- **Query methods**: chỉ cần đặt tên method theo quy ước (`findBy...`, `findBy...And...`, `...Containing`, `...OrderBy...`) là Spring tự sinh query.
- **@Query**: dùng để viết query tùy chỉnh (JPQL hoặc native SQL) khi query methods không đủ linh hoạt.
