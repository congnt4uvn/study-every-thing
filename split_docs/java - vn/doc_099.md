## 1. Event-driven Architecture (Kiến trúc hướng sự kiện)

**Ý chính:**
- Hệ thống được xây dựng xoay quanh **sự kiện (event)**.
- Thành phần **phát sự kiện (producer/publisher)** và **xử lý sự kiện (consumer/subscriber)** được tách rời (loose coupling).
- Thường dùng trong hệ thống phân tán, microservices, xử lý bất đồng bộ (asynchronous).

**Đặc điểm:**
- **Asynchronous**: Producer không chờ consumer xử lý xong.
- **Scalable**: Dễ mở rộng số lượng consumer.
- **Decoupled**: Producer không cần biết ai sẽ xử lý sự kiện.

### Ví dụ đơn giản với Spring (ApplicationEvent)

```java
// 1. Định nghĩa Event
public class UserCreatedEvent extends ApplicationEvent {
    private final String username;

    public UserCreatedEvent(Object source, String username) {
        super(source);
        this.username = username;
    }

    public String getUsername() {
        return username;
    }
}

// 2. Publisher (phát sự kiện)
@Component
public class UserService {
    private final ApplicationEventPublisher publisher;

    public UserService(ApplicationEventPublisher publisher) {
        this.publisher = publisher;
    }

    public void createUser(String username) {
        // logic tạo user (giả lập)
        System.out.println("User created in DB: " + username);

        // phát sự kiện
        publisher.publishEvent(new UserCreatedEvent(this, username));
    }
}

// 3. Listener (lắng nghe & xử lý sự kiện)
@Component
public class WelcomeEmailListener {

    @EventListener
    public void handleUserCreated(UserCreatedEvent event) {
        System.out.println("Sending welcome email to: " + event.getUsername());
    }
}
```

**Input/Output minh họa:**

```java
// Gọi từ main hoặc controller
userService.createUser("alice");
```

**Output console:**
```text
User created in DB: alice
Sending welcome email to: alice
```

## 2. Messaging (Nhắn tin / Hàng đợi thông điệp)

**Ý chính:**
- Các thành phần giao tiếp với nhau thông qua **message** (thông điệp) qua **message broker** (RabbitMQ, Kafka, ActiveMQ, JMS...).
- Hỗ trợ **queue** (hàng đợi) hoặc **topic** (pub/sub).
- Thường dùng để:
  - Xử lý nền (background processing)
  - Tách rời các service
  - Tăng độ tin cậy (reliability) và khả năng chịu tải (load)

**Đặc điểm:**
- **Reliable**: Message có thể được lưu, retry khi lỗi.
- **Asynchronous**: Gửi và nhận không cần đồng bộ.
- **Buffering**: Hàng đợi giúp “đệm” khi hệ thống bị tải cao.

### Ví dụ với JMS (Java Message Service)

#### 2.1. Producer (Gửi message)

```java
import javax.jms.*;

public class OrderProducer {

    private final ConnectionFactory connectionFactory;
    private final Queue orderQueue;

    public OrderProducer(ConnectionFactory connectionFactory, Queue orderQueue) {
        this.connectionFactory = connectionFactory;
        this.orderQueue = orderQueue;
    }

    public void sendOrder(String orderJson) throws JMSException {
        try (Connection connection = connectionFactory.createConnection();
             Session session = connection.createSession(false, Session.AUTO_ACKNOWLEDGE)) {

            MessageProducer producer = session.createProducer(orderQueue);
            TextMessage message = session.createTextMessage(orderJson);
            producer.send(message);

            System.out.println("Sent order: " + orderJson);
        }
    }
}
```

**Input:**

```java
orderProducer.sendOrder("{\"orderId\": 123, \"item\": \"Book\", \"quantity\": 2}");
```

**Output console (producer):**
```text
Sent order: {"orderId": 123, "item": "Book", "quantity": 2}
```

#### 2.2. Consumer (Nhận message)

```java
import javax.jms.*;

public class OrderConsumer implements MessageListener {

    @Override
    public void onMessage(Message message) {
        try {
            if (message instanceof TextMessage) {
                String orderJson = ((TextMessage) message).getText();
                System.out.println("Received order: " + orderJson);

                // xử lý đơn hàng
                // ...
                System.out.println("Order processed.");
            }
        } catch (JMSException e) {
            e.printStackTrace();
        }
    }
}
```

**Output console (consumer):**
```text
Received order: {"orderId": 123, "item": "Book", "quantity": 2}
Order processed.
```

## 3. Tóm tắt so sánh

- **Event-driven architecture**:
  - Tập trung vào **sự kiện** trong hệ thống.
  - Thường dùng pattern **publish/subscribe**.
  - Phù hợp cho **microservices**, xử lý bất đồng bộ, phản ứng theo sự kiện.

- **Messaging**:
  - Tập trung vào **message** qua **broker** (queue/topic).
  - Dùng để **kết nối các hệ thống**, xử lý nền, tăng độ tin cậy.
  - Là cơ chế nền tảng để triển khai **event-driven** trong hệ thống phân tán.

Nếu bạn muốn, tôi có thể thêm ví dụ với Kafka hoặc RabbitMQ trong Java.
