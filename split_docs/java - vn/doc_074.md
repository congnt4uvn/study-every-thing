
Dưới đây là tóm tắt ngắn gọn về **Authentication** và **Authorization** trong **Spring Security**, kèm ví dụ code và input/output.

---

## 1. Khái niệm chính

### Authentication (Xác thực)
- Mục tiêu: **Xác minh bạn là ai**.
- Kiểm tra thông tin đăng nhập: username/password, token, OAuth2, v.v.
- Kết quả: Nếu thành công, Spring Security tạo ra một **Authentication object** chứa thông tin user (principal, roles, …) và lưu trong **SecurityContext**.

### Authorization (Phân quyền)
- Mục tiêu: **Kiểm tra bạn được phép làm gì**.
- Dựa trên roles/permissions của user đã được xác thực.
- Quyết định cho phép hay từ chối truy cập vào URL, method, resource.

---

## 2. Luồng xử lý trong Spring Security (đơn giản)

1. Client gửi request (ví dụ: POST /login với username/password).
2. **Authentication**: Spring Security kiểm tra thông tin → nếu đúng, tạo `Authentication` và lưu vào `SecurityContext`.
3. Client tiếp tục gọi API khác (ví dụ: GET /admin).
4. **Authorization**: Spring Security kiểm tra roles/authorities trong `Authentication` → quyết định cho phép hay từ chối.

---

## 3. Ví dụ cấu hình Spring Security (Java Config)

### 3.1. Cấu hình Security (Spring Boot 3 / Spring Security 6)

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable())
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/public/**").permitAll()          // không cần login
                .requestMatchers("/admin/**").hasRole("ADMIN")      // cần role ADMIN
                .anyRequest().authenticated()                       // các request khác cần login
            )
            .formLogin(form -> form
                .loginPage("/login")        // trang login custom (nếu có)
                .permitAll()
            )
            .logout(logout -> logout.permitAll());

        return http.build();
    }

    // Tạo user trong memory để demo
    @Bean
    public UserDetailsService userDetailsService() {
        UserDetails user = User.withUsername("user")
            .password("{noop}password") // {noop} = không mã hóa, chỉ dùng demo
            .roles("USER")
            .build();

        UserDetails admin = User.withUsername("admin")
            .password("{noop}admin123")
            .roles("ADMIN")
            .build();

        return new InMemoryUserDetailsManager(user, admin);
    }
}
```

### 3.2. Ví dụ Controller

```java
@RestController
public class DemoController {

    @GetMapping("/public/hello")
    public String publicHello() {
        return "Public Hello";
    }

    @GetMapping("/user/profile")
    public String userProfile(Authentication authentication) {
        return "Hello " + authentication.getName();
    }

    @GetMapping("/admin/dashboard")
    public String adminDashboard() {
        return "Admin Dashboard";
    }
}
```

## 4. Ví dụ Input/Output

### 4.1. Truy cập endpoint public (không cần Authentication)

**Request:**
```http
GET /public/hello HTTP/1.1
Host: localhost:8080
```

**Response:**
```http
HTTP/1.1 200 OK
Content-Type: text/plain

Public Hello
```

---

### 4.2. Truy cập endpoint cần Authentication (chưa login)

**Request:**
```http
GET /user/profile HTTP/1.1
Host: localhost:8080
```

**Response (Spring Security redirect đến trang login):**
```http
HTTP/1.1 302 Found
Location: /login
```

---

### 4.3. Login với user thường

**Request:**
```http
POST /login HTTP/1.1
Content-Type: application/x-www-form-urlencoded

username=user&password=password
```

- Nếu đúng: Authentication thành công, tạo session, lưu `Authentication` vào `SecurityContext`.

Sau đó gọi:

```http
GET /user/profile HTTP/1.1
Cookie: JSESSIONID=...
```

**Response:**
```http
HTTP/1.1 200 OK
Content-Type: text/plain

Hello user
```

---

### 4.4. User thường truy cập endpoint chỉ cho ADMIN

```http
GET /admin/dashboard HTTP/1.1
Cookie: JSESSIONID=...
```

**Response (Authorization fail):**
```http
HTTP/1.1 403 Forbidden
```

- Ở đây: Authentication đã OK (user đã login), nhưng Authorization không đủ quyền (không có role ADMIN).

---

## 5. Tóm tắt ngắn

- **Authentication (Xác thực)**: kiểm tra danh tính (login, token, …).  
- **Authorization (Phân quyền)**: kiểm tra quyền truy cập (ROLE_USER, ROLE_ADMIN, …).  
- Spring Security:
  - Đầu tiên **Authentication** → tạo `Authentication` object.
  - Sau đó **Authorization** → dựa trên roles/authorities để cho phép hoặc từ chối truy cập.
