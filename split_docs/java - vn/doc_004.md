
Dưới đây là tóm tắt ngắn gọn về **Enum** và **Annotation** trong Java, kèm ví dụ mã và input/output.

## 1. Enum trong Java

### Khái niệm
- `enum` là kiểu dữ liệu đặc biệt dùng để định nghĩa một tập hằng số (constant) cố định.
- Giúp code rõ ràng, tránh dùng “magic number” hoặc string rời rạc.

### Ví dụ cơ bản

```java
public enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}

public class EnumExample {
    public static void main(String[] args) {
        Day today = Day.MONDAY;

        if (today == Day.MONDAY) {
            System.out.println("Start of the week");
        }

        // Duyệt qua tất cả giá trị enum
        for (Day d : Day.values()) {
            System.out.println(d);
        }
    }
}
```

**Output (kết quả in ra):**
```text
Start of the week
MONDAY
TUESDAY
WEDNESDAY
THURSDAY
FRIDAY
SATURDAY
SUNDAY
```

### Enum có field, constructor, method

```java
public enum Status {
    NEW(0),
    IN_PROGRESS(1),
    DONE(2);

    private final int code;

    Status(int code) {
        this.code = code;
    }

    public int getCode() {
        return code;
    }
}

public class StatusTest {
    public static void main(String[] args) {
        Status s = Status.IN_PROGRESS;
        System.out.println(s);              // IN_PROGRESS
        System.out.println(s.getCode());    // 1
    }
}
```

**Output:**
```text
IN_PROGRESS
1
```

## 2. Annotation trong Java

### Khái niệm
- Annotation là “siêu thông tin” (metadata) gắn lên:
  - class, method, field, parameter, v.v.
- Dùng cho:
  - Cấu hình (VD: Spring, JPA)
  - Kiểm tra compile-time (VD: `@Override`)
  - Xử lý runtime (reflection)

---

## 3. Annotation dựng sẵn (built-in)

Một số annotation thường dùng:

### `@Override`
- Đánh dấu method override từ superclass hoặc interface.

```java
class Parent {
    void sayHello() {
        System.out.println("Hello from Parent");
    }
}

class Child extends Parent {
    @Override
    void sayHello() {
        System.out.println("Hello from Child");
    }
}
```

**Output:**
```text
Hello from Child
```

### `@Deprecated`
- Đánh dấu phần code không nên dùng nữa.

```java
class OldApi {
    @Deprecated
    public void oldMethod() {
        System.out.println("Old method");
    }
}

class UseApi {
    public static void main(String[] args) {
        OldApi api = new OldApi();
        api.oldMethod(); // Compiler sẽ cảnh báo
    }
}
```

**Output:**
```text
Old method
```

### `@SuppressWarnings`
- Tắt một số cảnh báo của compiler.

```java
import java.util.ArrayList;
import java.util.List;

public class SuppressExample {
    @SuppressWarnings("unchecked")
    public static void main(String[] args) {
        List list = new ArrayList(); // raw type
        list.add("Hello");
        System.out.println(list.get(0));
    }
}
```

## 4. Tạo custom annotation

### Bước 1: Định nghĩa annotation

```java
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;
import java.lang.annotation.ElementType;

// Chỉ tồn tại ở runtime
@Retention(RetentionPolicy.RUNTIME)
// Chỉ dùng cho method
@Target(ElementType.METHOD)
public @interface MyAnnotation {
    String value();          // thuộc tính bắt buộc
    int version() default 1; // thuộc tính có giá trị mặc định
}
```

### Bước 2: Sử dụng annotation

```java
public class Service {

    @MyAnnotation(value = "doSomething", version = 2)
    public void doSomething() {
        System.out.println("Doing something...");
    }

    @MyAnnotation("simpleMethod") // dùng default version = 1
    public void simpleMethod() {
        System.out.println("Simple method");
    }
}
```

### Bước 3: Đọc annotation bằng reflection

```java
import java.lang.reflect.Method;

public class AnnotationReader {
    public static void main(String[] args) throws Exception {
        Class<Service> clazz = Service.class;

        for (Method m : clazz.getDeclaredMethods()) {
            if (m.isAnnotationPresent(MyAnnotation.class)) {
                MyAnnotation ann = m.getAnnotation(MyAnnotation.class);
                System.out.println("Method: " + m.getName());
                System.out.println("  value  = " + ann.value());
                System.out.println("  version= " + ann.version());
            }
        }
    }
}
```

**Output (có thể):**
```text
Method: doSomething
  value  = doSomething
  version= 2
Method: simpleMethod
  value  = simpleMethod
  version= 1
```

## Tóm tắt ý chính

- **Enum**
  - Dùng để định nghĩa tập hằng số có số lượng cố định.
  - Có thể có field, constructor, method.
  - Giúp code rõ ràng, type-safe.

- **Annotation**
  - Là metadata gắn lên code.
  - Built-in annotation phổ biến: `@Override`, `@Deprecated`, `@SuppressWarnings`, …
  - Có thể tự tạo custom annotation với:
    - `@Retention` (thời điểm tồn tại: SOURCE, CLASS, RUNTIME)
    - `@Target` (nơi được phép gắn: METHOD, TYPE, FIELD, …)
  - Có thể đọc và xử lý annotation bằng reflection (runtime) hoặc annotation processor (compile-time).
