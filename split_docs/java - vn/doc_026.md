
Dưới đây là tóm tắt ngắn gọn về chủ đề:

## 1. Khái niệm chính

**Immutable collection (bộ sưu tập bất biến)**  
- Không thể thay đổi sau khi được tạo.  
- Không thể thêm, xóa, sửa phần tử.  
- Thường an toàn khi dùng trong môi trường đa luồng (thread-safe về mặt cấu trúc).  
- Ví dụ: `List.of(...)`, `Set.of(...)`, `Map.of(...)` (Java 9+), hoặc thư viện như Guava `ImmutableList`.

**Unmodifiable collection (bộ sưu tập không cho sửa trực tiếp)**  
- Là một “view” (lớp bọc) không cho phép sửa **qua chính view đó**, nhưng **dữ liệu gốc bên dưới vẫn có thể thay đổi** nếu bạn còn giữ tham chiếu đến collection gốc.  
- Dùng `Collections.unmodifiableList`, `unmodifiableSet`, `unmodifiableMap`, …  
- Không đảm bảo bất biến hoàn toàn, chỉ chặn thao tác sửa qua wrapper.

---

## 2. So sánh nhanh

| Đặc điểm                         | Immutable                          | Unmodifiable (wrapper)                         |
|----------------------------------|------------------------------------|-----------------------------------------------|
| Có thể thay đổi dữ liệu bên dưới? | Không                              | Có (nếu còn tham chiếu đến collection gốc)    |
| Thao tác thêm/xóa/sửa            | Không cho phép                     | Không cho phép qua wrapper                    |
| An toàn với đa luồng (về cấu trúc)| Thường có                          | Không đảm bảo (vì collection gốc có thể đổi)  |
| Cách tạo                         | Tạo mới, copy dữ liệu              | Bọc (wrap) collection hiện có                 |

## 3. Ví dụ: Immutable collections (Java 9+)

### 3.1. List bất biến

```java
import java.util.List;

public class ImmutableExample {
    public static void main(String[] args) {
        List<String> immutableList = List.of("A", "B", "C");

        System.out.println("immutableList = " + immutableList);

        // Thử sửa
        try {
            immutableList.add("D");
        } catch (UnsupportedOperationException e) {
            System.out.println("Không thể thêm phần tử vào immutableList");
        }
    }
}
```

**Output (kết quả):**
```
immutableList = [A, B, C]
Không thể thêm phần tử vào immutableList
```

### 3.2. Map bất biến

```java
import java.util.Map;

public class ImmutableMapExample {
    public static void main(String[] args) {
        Map<String, Integer> immutableMap = Map.of(
                "one", 1,
                "two", 2,
                "three", 3
        );

        System.out.println("immutableMap = " + immutableMap);

        try {
            immutableMap.put("four", 4);
        } catch (UnsupportedOperationException e) {
            System.out.println("Không thể thêm phần tử vào immutableMap");
        }
    }
}
```

**Output:**
```
immutableMap = {one=1, two=2, three=3}
Không thể thêm phần tử vào immutableMap
```

## 4. Ví dụ: Unmodifiable collections (wrapper)

### 4.1. Unmodifiable List

```java
import java.util.ArrayList;
import java.util.Collections;
import java.util.List;

public class UnmodifiableExample {
    public static void main(String[] args) {
        List<String> original = new ArrayList<>();
        original.add("A");
        original.add("B");

        List<String> unmodifiable = Collections.unmodifiableList(original);

        System.out.println("original    = " + original);
        System.out.println("unmodifiable = " + unmodifiable);

        // Thử sửa qua wrapper
        try {
            unmodifiable.add("C");
        } catch (UnsupportedOperationException e) {
            System.out.println("Không thể thêm phần tử qua unmodifiable");
        }

        // Nhưng vẫn có thể sửa qua original
        original.add("C");

        System.out.println("Sau khi sửa original:");
        System.out.println("original    = " + original);
        System.out.println("unmodifiable = " + unmodifiable);
    }
}
```

**Output:**
```
original    = [A, B]
unmodifiable = [A, B]
Không thể thêm phần tử qua unmodifiable
Sau khi sửa original:
original    = [A, B, C]
unmodifiable = [A, B, C]
```

→ `unmodifiable` thay đổi theo vì nó chỉ là “view” của `original`.

## 5. Khi nào dùng cái nào?

- **Dùng immutable** khi:
  - Muốn dữ liệu thực sự không đổi sau khi tạo.
  - Cần an toàn hơn trong môi trường đa luồng.
  - Muốn tránh side-effect khi truyền collection cho nhiều nơi.

- **Dùng unmodifiable** khi:
  - Muốn chặn việc sửa collection từ bên ngoài API (không cho caller sửa trực tiếp).
  - Nhưng vẫn chấp nhận (hoặc cần) thay đổi collection gốc ở bên trong.

---

## 6. Tóm tắt ngắn

- **Immutable**: không thể thay đổi, không có cách nào sửa dữ liệu sau khi tạo → an toàn hơn, rõ ràng hơn.  
- **Unmodifiable**: chỉ chặn thao tác sửa qua wrapper, nhưng dữ liệu gốc vẫn có thể bị thay đổi nếu còn tham chiếu → cẩn thận khi dùng, dễ gây hiểu nhầm nếu không nắm rõ.
