
Dưới đây là tóm tắt ngắn gọn về **Contract Testing** (kiểm thử hợp đồng), đặc biệt trong bối cảnh **microservices**:

---

## 1. Contract Testing là gì?

- Là kiểu kiểm thử đảm bảo **dịch vụ cung cấp (provider)** và **dịch vụ sử dụng (consumer)** tuân thủ cùng một **hợp đồng (contract)** về:
  - Cấu trúc request/response (URL, method, header, body, status code…)
  - Kiểu dữ liệu, field bắt buộc, field tùy chọn
- Mục tiêu: nếu provider thay đổi, **không làm hỏng** các consumer đang dùng API.

---

## 2. Tại sao quan trọng với Microservices?

Trong kiến trúc microservices:
- Có nhiều service giao tiếp qua HTTP/REST, gRPC, message queue…
- Thay đổi ở một service có thể làm **vỡ** các service khác.
- Contract testing giúp:
  - Phát hiện sớm lỗi tương thích (breaking changes)
  - Giảm phụ thuộc vào test tích hợp (integration test) nặng nề
  - Tự động kiểm tra tương thích trong CI/CD

---

## 3. Cách tiếp cận phổ biến

1. **Consumer-Driven Contract Testing (CDCT)**  
   - Consumer định nghĩa hợp đồng mong muốn (contract).
   - Provider phải đáp ứng hợp đồng đó.
   - Công cụ phổ biến: **Pact**, Spring Cloud Contract.

2. **Provider-Driven**  
   - Provider định nghĩa schema (OpenAPI/Swagger, JSON Schema…).
   - Consumer phải tuân theo.

---

## 4. Ví dụ với Java + Spring + Pact (REST)

### 4.1. Consumer: viết contract (test phía consumer)

Giả sử có service `OrderService` gọi sang `PaymentService`.

**Test phía consumer (OrderService)**:

```java
// Maven dependency (pom.xml)
// <dependency>
//   <groupId>au.com.dius.pact.consumer</groupId>
//   <artifactId>junit5</artifactId>
//   <version>4.6.3</version>
//   <scope>test</scope>
// </dependency>

import au.com.dius.pact.consumer.dsl.PactDslWithProvider;
import au.com.dius.pact.consumer.junit5.PactConsumerTestExt;
import au.com.dius.pact.consumer.junit5.PactTestFor;
import au.com.dius.pact.core.model.RequestResponsePact;
import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.extension.ExtendWith;

import java.io.IOException;
import java.net.HttpURLConnection;
import java.net.URL;

import static org.assertj.core.api.Assertions.assertThat;

@ExtendWith(PactConsumerTestExt.class)
@PactTestFor(providerName = "PaymentService", port = "8089")
public class PaymentConsumerPactTest {

    @au.com.dius.pact.consumer.junit5.Pact(consumer = "OrderService")
    public RequestResponsePact createPact(PactDslWithProvider builder) {
        return builder
            .given("Payment exists for order 123")
            .uponReceiving("a request to get payment by orderId")
                .path("/payments/123")
                .method("GET")
            .willRespondWith()
                .status(200)
                .header("Content-Type", "application/json")
                .body("{\"orderId\": \"123\", \"status\": \"PAID\"}")
            .toPact();
    }

    @Test
    void testGetPayment() throws IOException {
        URL url = new URL("http://localhost:8089/payments/123");
        HttpURLConnection connection = (HttpURLConnection) url.openConnection();
        connection.setRequestMethod("GET");

        int status = connection.getResponseCode();
        assertThat(status).isEqualTo(200);
        // Có thể đọc body và assert thêm
    }
}
```

- **Kết quả (output)**:  
  - Test chạy, tạo ra file contract (Pact file), ví dụ:  
    `target/pacts/orderservice-paymentservice.json`
  - File này mô tả:  
    - Request: `GET /payments/123`  
    - Response: `200`, body `{"orderId":"123","status":"PAID"}`

### 4.2. Provider: verify contract (test phía provider)

Ở `PaymentService`, ta dùng contract do consumer tạo để kiểm tra xem API thực tế có đúng không.

```java
// Maven dependency (pom.xml)
// <dependency>
//   <groupId>au.com.dius.pact.provider</groupId>
//   <artifactId>junit5</artifactId>
//   <version>4.6.3</version>
//   <scope>test</scope>
// </dependency>

import au.com.dius.pact.provider.junit5.PactVerificationContext;
import au.com.dius.pact.provider.junit5.PactVerificationInvocationContextProvider;
import au.com.dius.pact.provider.junit5.PactVerificationTest;
import au.com.dius.pact.provider.junit5.PactVerificationTestContext;
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.TestTemplate;
import org.junit.jupiter.api.extension.ExtendWith;

@PactVerificationTest(provider = "PaymentService")
public class PaymentProviderPactTest {

    @BeforeEach
    void before(PactVerificationContext context) {
        // Khởi động PaymentService (embedded server, test profile, v.v.)
        context.setTarget(new au.com.dius.pact.provider.junit5.http.HttpTestTarget("localhost", 8080));
    }

    @TestTemplate
    @ExtendWith(PactVerificationInvocationContextProvider.class)
    void pactVerificationTestTemplate(PactVerificationContext context) {
        context.verifyInteraction();
    }
}
```

- **Input**: file contract `orderservice-paymentservice.json` (từ consumer).
- **Output**:
  - Nếu provider trả đúng như contract → test PASS.
  - Nếu khác (ví dụ trả 404, hoặc thiếu field `status`) → test FAIL, báo lỗi chi tiết.

---

## 5. Lợi ích chính

- Phát hiện sớm lỗi tương thích giữa microservices.
- Giảm số lượng test tích hợp end-to-end phức tạp.
- Hỗ trợ CI/CD: mỗi lần build, provider/consumer đều được kiểm tra theo contract.
- Tài liệu sống (living documentation) cho API.

---

Tóm lại, **Contract Testing** là kỹ thuật kiểm thử quan trọng trong microservices, giúp đảm bảo các service giao tiếp đúng như thỏa thuận, giảm rủi ro khi thay đổi code và tăng độ tin cậy của hệ thống phân tán.
