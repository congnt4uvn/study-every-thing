
Dưới đây là tóm tắt ngắn gọn về các trạng thái Entity trong JPA/Hibernate, kèm ví dụ code và input/output minh họa.

---

## 1. Các trạng thái của Entity

Trong JPA/Hibernate, một entity (đối tượng được ánh xạ với bảng DB) có thể ở 4 trạng thái chính:

1. **Transient (tạm thời / chưa quản lý)**
2. **Persistent (đang được quản lý)**
3. **Detached (bị tách / không còn được quản lý)**
4. **Removed (đã đánh dấu xóa)**

---

## 2. Transient

- Đối tượng **mới tạo bằng `new`**, chưa gắn với `EntityManager`/`Session`.
- Chưa có bản ghi tương ứng trong DB.
- Thay đổi trên object **không** được tự động lưu xuống DB.

```java
// Entity ví dụ
@Entity
public class User {
    @Id @GeneratedValue
    private Long id;
    private String name;
    // getter/setter
}

// Transient
User user = new User();  // trạng thái: TRANSIENT
user.setName("Alice");
```

**Trạng thái:**  
- `id == null`  
- Không có row trong DB.

---

## 3. Persistent

- Đối tượng đang được `EntityManager` quản lý.
- Mọi thay đổi trên entity sẽ được tự động đồng bộ với DB khi:
  - `flush()`
  - hoặc `commit` transaction.

### Cách trở thành Persistent

1. **persist()** (với entity mới):

```java
EntityManager em = ...;
em.getTransaction().begin();

User user = new User();      // TRANSIENT
user.setName("Alice");

em.persist(user);            // giờ là PERSISTENT
// user.getId() có thể được gán sau khi flush/commit

em.getTransaction().commit();
```

**Kết quả (output trong DB):**

```sql
INSERT INTO user (name) VALUES ('Alice');
```

2. **find() / getReference()** (load từ DB):

```java
User user = em.find(User.class, 1L); // nếu tồn tại -> PERSISTENT
```

---

## 4. Detached

- Entity **từng** được quản lý, nhưng **không còn** gắn với `EntityManager` hiện tại.
- Thường xảy ra khi:
  - `EntityManager` bị đóng (`em.close()`),
  - hoặc entity bị detach thủ công (`em.detach(entity)`),
  - hoặc sau khi kết thúc transaction (tùy cách dùng).

- Thay đổi trên entity **không tự động** lưu xuống DB nữa.

```java
EntityManager em = ...;
em.getTransaction().begin();

User user = em.find(User.class, 1L); // PERSISTENT
em.getTransaction().commit();
em.close();                          // EntityManager đóng

user.setName("Bob");                 // user giờ là DETACHED
// Thay đổi này không tự lưu vào DB
```

### Gắn lại (merge) Detached entity

```java
EntityManager em2 = ...;
em2.getTransaction().begin();

User managedUser = em2.merge(user);  // managedUser: PERSISTENT
// user (cũ) vẫn DETACHED

em2.getTransaction().commit();
```

**Kết quả (output trong DB):**

```sql
UPDATE user SET name = 'Bob' WHERE id = 1;
```

---

## 5. Removed

- Entity đang được quản lý (PERSISTENT) và bị đánh dấu xóa bằng `remove()`.
- Sau khi `flush`/`commit`, bản ghi tương ứng trong DB sẽ bị xóa.
- Sau khi xóa, entity thường được xem là Detached (không còn quản lý).

```java
EntityManager em = ...;
em.getTransaction().begin();

User user = em.find(User.class, 1L); // PERSISTENT
em.remove(user);                     // trạng thái: REMOVED

em.getTransaction().commit();
```

**Kết quả (output trong DB):**

```sql
DELETE FROM user WHERE id = 1;
```

---

## 6. Tóm tắt nhanh

| Trạng thái   | Đặc điểm chính                                                | Cách chuyển sang |
|-------------|----------------------------------------------------------------|------------------|
| Transient   | Mới `new`, chưa gắn EM, chưa có trong DB                       | `persist()`      |
| Persistent  | Được EM quản lý, tự sync với DB khi flush/commit               | `find()`, `persist()`, `merge()` |
| Detached    | Từng được quản lý, giờ không còn; thay đổi không tự lưu DB     | `detach()`, `close()`, hết transaction |
| Removed     | Được quản lý và đã đánh dấu xóa; sẽ xóa khỏi DB khi commit     | `remove()`       |
