
Dưới đây là tóm tắt ngắn gọn về các collection concurrent trong `java.util.concurrent` và ví dụ code (có input/output):

---

## 1. `ConcurrentHashMap`

- Map an toàn trong môi trường đa luồng (thread-safe).
- Cho phép nhiều thread đọc/ghi cùng lúc hiệu quả hơn `Collections.synchronizedMap`.
- Không bị `ConcurrentModificationException` khi duyệt trong khi đang cập nhật.

### Ví dụ

```java
import java.util.concurrent.ConcurrentHashMap;

public class ConcurrentHashMapExample {
    public static void main(String[] args) {
        ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

        // Input: put dữ liệu
        map.put("A", 1);
        map.put("B", 2);

        // Cập nhật an toàn
        map.compute("A", (k, v) -> v == null ? 1 : v + 10); // A = 11

        // Duyệt map trong khi cập nhật
        map.forEach((k, v) -> {
            System.out.println(k + " = " + v);
            map.put("C", 3); // vẫn an toàn, không ném ConcurrentModificationException
        });

        System.out.println("Final map: " + map);
    }
}
```

**Output (có thể khác thứ tự):**
```
A = 11
B = 2
Final map: {A=11, B=2, C=3}
```

## 2. `CopyOnWriteArrayList`

- List an toàn cho đa luồng, tối ưu cho **nhiều đọc, ít ghi**.
- Mỗi lần ghi (add/remove) sẽ tạo một bản copy mới của mảng bên trong.
- Đọc không cần lock, rất nhanh; nhưng ghi tốn chi phí.

### Ví dụ

```java
import java.util.List;
import java.util.concurrent.CopyOnWriteArrayList;

public class CopyOnWriteArrayListExample {
    public static void main(String[] args) {
        List<String> list = new CopyOnWriteArrayList<>();
        
        // Input: thêm phần tử
        list.add("A");
        list.add("B");

        // Duyệt trong khi thêm phần tử
        for (String s : list) {
            System.out.println("Read: " + s);
            list.add("C"); // không gây ConcurrentModificationException
        }

        System.out.println("Final list: " + list);
    }
}
```

**Output:**
```
Read: A
Read: B
Final list: [A, B, C, C]
```
(Lần duyệt đầu chỉ thấy A, B; C được thêm nhưng không xuất hiện trong vòng for hiện tại.)

## 3. `BlockingQueue`

- Hàng đợi an toàn đa luồng, hỗ trợ **chặn** (block) khi:
  - `put()` vào queue đầy (bounded queue).
  - `take()` từ queue rỗng.
- Rất hữu ích cho mô hình Producer–Consumer.

### 3.1 `ArrayBlockingQueue`

- Cài đặt bằng mảng, kích thước cố định (bounded).
- Thích hợp khi biết trước số lượng tối đa phần tử.

```java
import java.util.concurrent.ArrayBlockingQueue;
import java.util.concurrent.BlockingQueue;

public class ArrayBlockingQueueExample {
    public static void main(String[] args) throws InterruptedException {
        BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(2);

        // Producer
        new Thread(() -> {
            try {
                System.out.println("Put 1");
                queue.put(1); // ok
                System.out.println("Put 2");
                queue.put(2); // ok
                System.out.println("Put 3 (sẽ block cho đến khi có chỗ trống)");
                queue.put(3); // block cho đến khi consumer lấy bớt
                System.out.println("Put 3 done");
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }).start();

        // Consumer
        Thread.sleep(1000);
        System.out.println("Take: " + queue.take()); // lấy 1
        Thread.sleep(1000);
        System.out.println("Take: " + queue.take()); // lấy 2
        Thread.sleep(1000);
        System.out.println("Take: " + queue.take()); // lấy 3
    }
}
```

**Output (thứ tự thời gian):**
```
Put 1
Put 2
Put 3 (sẽ block cho đến khi có chỗ trống)
Take: 1
Put 3 done
Take: 2
Take: 3
```

---

### 3.2 `LinkedBlockingQueue`

- Cài đặt bằng danh sách liên kết (linked list).
- Có thể:
  - Không giới hạn (unbounded) – mặc định.
  - Hoặc giới hạn kích thước (bounded) nếu truyền capacity.
- Thường linh hoạt hơn `ArrayBlockingQueue` khi không biết trước số lượng phần tử.

```java
import java.util.concurrent.BlockingQueue;
import java.util.concurrent.LinkedBlockingQueue;

public class LinkedBlockingQueueExample {
    public static void main(String[] args) throws InterruptedException {
        BlockingQueue<String> queue = new LinkedBlockingQueue<>();

        // Producer
        queue.put("Task1");
        queue.put("Task2");

        // Consumer
        System.out.println("Take: " + queue.take());
        System.out.println("Take: " + queue.take());
    }
}
```

**Output:**
```
Take: Task1
Take: Task2
```

---

### Tóm tắt ý chính

- `ConcurrentHashMap`: Map thread-safe, hiệu năng tốt, cho phép đọc/ghi song song.
- `CopyOnWriteArrayList`: List thread-safe, tối ưu cho nhiều đọc, ít ghi; mỗi lần ghi tạo bản copy.
- `BlockingQueue`:
  - `ArrayBlockingQueue`: queue giới hạn kích thước, dùng mảng.
  - `LinkedBlockingQueue`: queue có thể giới hạn hoặc không, dùng linked list.
- Tất cả đều thuộc `java.util.concurrent`, thiết kế để làm việc an toàn và hiệu quả trong môi trường đa luồng.
