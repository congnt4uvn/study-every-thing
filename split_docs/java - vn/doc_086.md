
Dưới đây là tóm tắt ngắn gọn về các “pitfall” thường gặp trong JPA/Hibernate: N+1, cascade, orphan removal, kèm ví dụ code và input/output minh họa.

---

## 1. Vấn đề N+1 Query

**Ý chính**  
- Xảy ra khi bạn load 1 danh sách entity (1 query), nhưng mỗi entity lại trigger thêm 1 query để load quan hệ (N query).  
- Tổng cộng: **N+1 query**, gây chậm, tốn tài nguyên.

**Ví dụ**  
Giả sử có `Author` – `Book` (1-n):

```java
@Entity
public class Author {
    @Id
    private Long id;

    private String name;

    @OneToMany(mappedBy = "author", fetch = FetchType.LAZY)
    private List<Book> books;
}

@Entity
public class Book {
    @Id
    private Long id;

    private String title;

    @ManyToOne
    private Author author;
}
```

Code gây N+1:

```java
List<Author> authors = em.createQuery("SELECT a FROM Author a", Author.class)
                         .getResultList();

for (Author a : authors) {
    // Mỗi lần gọi a.getBooks() sẽ bắn thêm 1 query
    System.out.println(a.getName() + " has " + a.getBooks().size() + " books");
}
```

**Input (giả sử)**  
- Bảng `author`: 3 dòng (A1, A2, A3)  
- Bảng `book`: nhiều sách cho mỗi author

**Output (log SQL)**  
- 1 query: `SELECT * FROM author`  
- + 3 query: `SELECT * FROM book WHERE author_id = ?` (cho A1, A2, A3)  
→ Tổng: 4 query (N+1)

**Cách khắc phục (JOIN FETCH)**

```java
List<Author> authors = em.createQuery(
        "SELECT a FROM Author a LEFT JOIN FETCH a.books",
        Author.class
).getResultList();

for (Author a : authors) {
    System.out.println(a.getName() + " has " + a.getBooks().size() + " books");
}
```

**Output (log SQL)**  
- 1 query duy nhất với `JOIN` để lấy cả author + books.

## 2. Cascade (CascadeType)

**Ý chính**  
- Cascade quyết định **hành động trên entity cha** sẽ tự động áp dụng cho **entity con** hay không.  
- Các loại thường dùng:  
  - `PERSIST` – lưu cha thì tự lưu con  
  - `MERGE` – merge cha thì merge con  
  - `REMOVE` – xóa cha thì xóa con  
  - `ALL` – tất cả các loại trên  
- Dùng sai có thể:  
  - Xóa nhầm dữ liệu  
  - Lưu quá nhiều entity không mong muốn

**Ví dụ**

```java
@Entity
public class Order {
    @Id
    @GeneratedValue
    private Long id;

    private String customerName;

    @OneToMany(mappedBy = "order", cascade = CascadeType.ALL)
    private List<OrderItem> items = new ArrayList<>();
}

@Entity
public class OrderItem {
    @Id
    @GeneratedValue
    private Long id;

    private String productName;

    @ManyToOne
    private Order order;
}
```

**Input (code)**

```java
Order order = new Order();
order.setCustomerName("Nguyen Van A");

OrderItem item1 = new OrderItem();
item1.setProductName("Laptop");
item1.setOrder(order);

order.getItems().add(item1);

em.persist(order);
```

**Output (SQL)**  
- `INSERT INTO order ...`  
- `INSERT INTO order_item ...`  
→ Nhờ `cascade = CascadeType.ALL`, chỉ cần `persist(order)` là `OrderItem` cũng được lưu.

**Pitfall**  
- Nếu dùng `cascade = CascadeType.REMOVE` ở quan hệ không phải “sở hữu” (ví dụ User – Role dùng chung), xóa 1 User có thể xóa luôn Role dùng chung.

## 3. Orphan Removal

**Ý chính**  
- `orphanRemoval = true` dùng cho quan hệ “cha – con” mà **cha là chủ sở hữu vòng đời** của con.  
- Khi một entity con bị **loại khỏi collection** của cha, JPA sẽ **tự động xóa** con khỏi DB.  
- Khác với `cascade = REMOVE`:  
  - `cascade = REMOVE`: xóa cha → xóa con  
  - `orphanRemoval = true`: bỏ con khỏi cha → xóa con

**Ví dụ**

```java
@Entity
public class Parent {
    @Id
    @GeneratedValue
    private Long id;

    private String name;

    @OneToMany(mappedBy = "parent",
               cascade = CascadeType.ALL,
               orphanRemoval = true)
    private List<Child> children = new ArrayList<>();
}

@Entity
public class Child {
    @Id
    @GeneratedValue
    private Long id;

    private String name;

    @ManyToOne
    private Parent parent;
}
```

**Input (code)**

```java
Parent p = em.find(Parent.class, 1L);
Child c = p.getChildren().get(0);

// Bỏ child khỏi danh sách
p.getChildren().remove(c);
```

**Output (SQL)**  
- `DELETE FROM child WHERE id = ?`  
→ Child bị xóa khỏi DB vì bị “mồ côi” (orphan).

**Pitfall**  
- Nếu bạn chỉ muốn “ngắt liên kết” mà không muốn xóa record, **không nên** dùng `orphanRemoval = true`.  
- Cần cẩn thận khi thao tác với collection (remove, clear) vì có thể xóa dữ liệu thật trong DB.

---

## Tóm tắt nhanh

1. **N+1 Query**  
   - Vấn đề: quá nhiều query khi load quan hệ LAZY.  
   - Giải pháp: dùng `JOIN FETCH`, `EntityGraph`, batch fetching.

2. **Cascade**  
   - Điều khiển việc tự động propagate `persist`, `merge`, `remove` từ cha sang con.  
   - Dùng đúng cho quan hệ “sở hữu”; tránh dùng bừa bãi ở quan hệ nhiều-nhiều dùng chung.

3. **Orphan Removal**  
   - Tự động xóa entity con khi bị loại khỏi quan hệ với cha.  
   - Phù hợp cho quan hệ “cha sở hữu hoàn toàn vòng đời con” (ví dụ: Order – OrderItem).  
   - Cẩn thận vì thao tác trên collection có thể xóa dữ liệu trong DB.
