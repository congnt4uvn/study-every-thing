
Dưới đây là tóm tắt ngắn gọn về các bộ gom rác (Garbage Collector) phổ biến trong Java, kèm ví dụ code minh họa cách quan sát GC (gián tiếp qua log).

---

## 1. Các bộ gom rác phổ biến trong Java

### 1. Serial GC
- Đặc điểm:
  - Dùng **một luồng** để gom rác.
  - Đơn giản, phù hợp cho ứng dụng nhỏ, máy đơn nhân (single-core) hoặc môi trường có tài nguyên hạn chế.
  - Có **stop-the-world** (tạm dừng toàn bộ ứng dụng khi GC chạy).
- Cách bật:
  ```bash
  java -XX:+UseSerialGC MyApp
  ```

---

### 2. Parallel GC (Throughput Collector)
- Đặc điểm:
  - Dùng **nhiều luồng** để gom rác, tối ưu **throughput** (tổng công việc xử lý).
  - Vẫn có stop-the-world, nhưng thời gian gom rác nhanh hơn nhờ chạy song song.
  - Thường là GC mặc định trong nhiều phiên bản Java cũ (Java 8).
- Cách bật:
  ```bash
  java -XX:+UseParallelGC MyApp
  ```

---

### 3. CMS (Concurrent Mark-Sweep) – *Legacy*
- Đặc điểm:
  - Gom rác **song song với ứng dụng** (concurrent) ở nhiều pha, giảm thời gian dừng.
  - Tập trung giảm **GC pause** (độ trễ).
  - Đã bị **đánh dấu là legacy** và bị loại bỏ trong các phiên bản Java mới (từ Java 14 trở đi).
- Cách bật (trên các bản Java còn hỗ trợ):
  ```bash
  java -XX:+UseConcMarkSweepGC MyApp
  ```

---

### 4. G1 GC (Garbage-First)
- Đặc điểm:
  - Thiết kế cho **heap lớn**, giảm pause, thay thế CMS.
  - Chia heap thành nhiều **region nhỏ**, gom rác theo vùng có nhiều rác trước (“Garbage-First”).
  - Hỗ trợ **predictable pause time** (cấu hình mục tiêu thời gian dừng).
- Cách bật (thường là mặc định từ Java 9+):
  ```bash
  java -XX:+UseG1GC MyApp
  ```

---

### 5. ZGC
- Đặc điểm:
  - **Low-latency GC**: thời gian dừng rất nhỏ (thường < 10ms), ngay cả với heap rất lớn (nhiều GB).
  - Hầu hết các pha GC chạy **concurrent** với ứng dụng.
  - Phù hợp cho hệ thống yêu cầu **độ trễ cực thấp**.
- Cách bật (tùy phiên bản JDK, thường từ Java 11+):
  ```bash
  java -XX:+UseZGC MyApp
  ```

---

### 6. Shenandoah
- Đặc điểm:
  - Cũng là **low-pause GC**, tương tự ZGC.
  - Tập trung vào **giảm pause time**, không phụ thuộc nhiều vào kích thước heap.
  - Được phát triển bởi Red Hat, tích hợp vào một số bản OpenJDK.
- Cách bật (tùy bản JDK):
  ```bash
  java -XX:+UseShenandoahGC MyApp
  ```

## Ví dụ code Java để quan sát GC (gián tiếp)

Code dưới đây tạo nhiều đối tượng để kích hoạt GC. Bạn có thể bật log GC để xem hoạt động của từng collector.

```java
public class GCDemo {
    private static final int _1MB = 1024 * 1024;

    public static void main(String[] args) {
        for (int i = 0; i < 1000; i++) {
            byte[] data = new byte[_1MB]; // tạo mảng 1MB
            if (i % 100 == 0) {
                System.out.println("Đã tạo " + i + " MB dữ liệu");
            }
        }
        System.out.println("Kết thúc chương trình");
    }
}
```

### Chạy với G1 GC và bật log GC

```bash
javac GCDemo.java

java \
  -XX:+UseG1GC \
  -Xms128m -Xmx128m \
  -Xlog:gc* \
  GCDemo
```

**Input:**  
Chương trình không nhận input từ người dùng, chỉ chạy vòng lặp tạo dữ liệu.

**Output (ví dụ, rút gọn):**
```text
Đã tạo 0 MB dữ liệu
Đã tạo 100 MB dữ liệu
Đã tạo 200 MB dữ liệu
...
Kết thúc chương trình
[0.123s][info][gc] GC(0) Pause Young (Normal) (G1 Evacuation Pause)  ...
[0.456s][info][gc] GC(1) Pause Young (Normal) (G1 Evacuation Pause)  ...
...
```

Nếu bạn đổi collector, ví dụ dùng ZGC:

```bash
java \
  -XX:+UseZGC \
  -Xms128m -Xmx128m \
  -Xlog:gc* \
  GCDemo
```

Bạn sẽ thấy log GC khác, thể hiện cách ZGC hoạt động (nhiều pha concurrent, pause rất ngắn).

## Tóm tắt ngắn

- **Serial GC**: đơn luồng, đơn giản, phù hợp ứng dụng nhỏ.
- **Parallel GC**: đa luồng, tối ưu throughput, vẫn có pause.
- **CMS**: giảm pause, concurrent, đã legacy.
- **G1 GC**: mặc định mới, chia heap thành region, cân bằng giữa throughput và pause.
- **ZGC**: low-latency, pause rất nhỏ, phù hợp hệ thống lớn, yêu cầu độ trễ thấp.
- **Shenandoah**: low-pause GC, tương tự ZGC, tập trung giảm pause bất kể kích thước heap.
