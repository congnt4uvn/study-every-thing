## 1. Compute trong Cloud là gì?

- “Compute” là tài nguyên xử lý (máy ảo / server) trên cloud.
- Thay vì mua server vật lý, bạn thuê máy ảo từ nhà cung cấp cloud.
- Ví dụ:
  - **AWS**: EC2 (Elastic Compute Cloud)
  - **GCP**: GCE (Google Compute Engine)
  - **Azure**: Virtual Machines

---

## 2. Đặc điểm chính của dịch vụ Compute

1. **Máy ảo (VM)**  
   - Bạn chọn:
     - Số CPU, RAM
     - Ổ đĩa (SSD/HDD)
     - Hệ điều hành (Linux, Windows)
   - Có thể khởi động, dừng, xóa giống như server thật.

2. **Thanh toán theo mức sử dụng**  
   - Trả tiền theo giờ/giây chạy máy.
   - Có các loại giá:
     - On-demand (theo nhu cầu)
     - Reserved (đặt trước, rẻ hơn)
     - Spot/Preemptible (rất rẻ nhưng có thể bị thu hồi).

3. **Tự động mở rộng (Auto Scaling)**  
   - Tự tăng/giảm số lượng máy ảo theo tải (CPU, request, v.v.).
   - Giúp tiết kiệm chi phí và đảm bảo hiệu năng.

4. **Tích hợp với các dịch vụ khác**  
   - Load Balancer (cân bằng tải)
   - Storage (S3, GCS, EBS…)
   - Network (VPC, firewall, security group)

## 3. Ví dụ: Khởi chạy một ứng dụng Java trên EC2

### 3.1. Ứng dụng Java đơn giản

**File: HelloServer.java**

```java
import com.sun.net.httpserver.HttpServer;
import com.sun.net.httpserver.HttpHandler;
import com.sun.net.httpserver.HttpExchange;

import java.io.IOException;
import java.io.OutputStream;
import java.net.InetSocketAddress;

public class HelloServer {
    public static void main(String[] args) throws IOException {
        int port = 8080; // cổng server
        HttpServer server = HttpServer.create(new InetSocketAddress(port), 0);
        server.createContext("/hello", new HelloHandler());
        server.setExecutor(null); // dùng executor mặc định
        System.out.println("Server started at http://0.0.0.0:" + port + "/hello");
        server.start();
    }

    static class HelloHandler implements HttpHandler {
        @Override
        public void handle(HttpExchange exchange) throws IOException {
            String response = "Xin chào từ EC2!";
            exchange.sendResponseHeaders(200, response.getBytes().length);
            try (OutputStream os = exchange.getResponseBody()) {
                os.write(response.getBytes());
            }
        }
    }
}
```

### 3.2. Cách chạy trên EC2 (ý tưởng)

1. Tạo một EC2 instance (Ubuntu chẳng hạn).
2. SSH vào máy:
   ```bash
   ssh -i your-key.pem ubuntu@<EC2_PUBLIC_IP>
   ```
3. Cài Java:
   ```bash
   sudo apt update
   sudo apt install -y openjdk-17-jdk
   ```
4. Copy file `HelloServer.java` lên EC2, rồi biên dịch & chạy:
   ```bash
   javac HelloServer.java
   java HelloServer
   ```

5. Mở Security Group cho cổng 8080 (inbound rule).
6. Truy cập từ máy bạn:

**Input (HTTP request):**
```http
GET http://<EC2_PUBLIC_IP>:8080/hello
```

**Output (HTTP response body):**
```text
Xin chào từ EC2!
```

## 4. Ví dụ tương tự trên GCE (Google Compute Engine)

Ý tưởng tương tự:

1. Tạo VM trên GCE.
2. SSH vào:
   ```bash
   gcloud compute ssh my-vm --zone=asia-southeast1-b
   ```
3. Cài Java, chạy cùng code `HelloServer.java`.
4. Mở firewall rule cho cổng 8080.
5. Gửi request đến `http://<GCE_EXTERNAL_IP>:8080/hello`.

---

## 5. Tóm tắt ý chính

- **Compute** = máy ảo trên cloud (EC2, GCE, Azure VM).
- Bạn có thể:
  - Chọn cấu hình CPU/RAM/OS.
  - Trả tiền theo mức sử dụng.
  - Tự động scale theo tải.
- Ứng dụng Java có thể chạy trên các VM này giống như trên server vật lý.
- Truy cập qua IP public + cổng ứng dụng (ví dụ 8080).
