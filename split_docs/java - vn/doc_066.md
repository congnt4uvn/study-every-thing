
Dưới đây là tóm tắt ngắn gọn về Spring MVC REST Controller, kèm ví dụ code và input/output.

---

## 1. @RestController là gì?

- Kết hợp `@Controller` + `@ResponseBody`.
- Dùng cho REST API: trả về dữ liệu (JSON/XML) thay vì trả về view (JSP, Thymeleaf…).
- Mọi method trong class được đánh dấu `@RestController` sẽ tự động trả về body response.

```java
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {
    // ...
}
```

---

## 2. @RequestMapping

- Dùng để mapping URL và HTTP method đến method trong controller.
- Có thể đặt ở class và/hoặc method.
- Ở class: định nghĩa prefix chung cho tất cả endpoint.
- Ở method: định nghĩa path cụ thể và method (GET, POST, …).

```java
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api") // prefix chung
public class ApiController {

    @RequestMapping(value = "/hello", method = RequestMethod.GET)
    public String hello() {
        return "Xin chào";
    }
}
```

**Input (HTTP request):**

```http
GET /api/hello HTTP/1.1
Host: localhost:8080
```

**Output (HTTP response):**

```http
HTTP/1.1 200 OK
Content-Type: text/plain;charset=UTF-8

Xin chào
```

## 3. @GetMapping, @PostMapping, @PutMapping, @DeleteMapping

- Là shortcut của `@RequestMapping(method = RequestMethod.GET/POST/PUT/DELETE)`.
- Giúp code ngắn gọn, dễ đọc.

### 3.1. @GetMapping

Dùng cho HTTP GET (lấy dữ liệu).

```java
@RestController
@RequestMapping("/api/users")
public class UserController {

    // Lấy danh sách user
    @GetMapping
    public List<String> getUsers() {
        return List.of("An", "Bình", "Chi");
    }

    // Lấy user theo id (path variable)
    @GetMapping("/{id}")
    public String getUserById(@PathVariable int id) {
        return "User có id = " + id;
    }
}
```

**Input:**

```http
GET /api/users HTTP/1.1
Host: localhost:8080
```

**Output (JSON):**

```json
["An", "Bình", "Chi"]
```

**Input:**

```http
GET /api/users/5 HTTP/1.1
Host: localhost:8080
```

**Output:**

```json
"User có id = 5"
```

---

### 3.2. @PostMapping

Dùng cho HTTP POST (tạo mới dữ liệu).

```java
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/users")
public class UserController {

    @PostMapping
    public User createUser(@RequestBody User user) {
        // giả sử đã lưu user vào DB
        user.setId(1L);
        return user;
    }
}

class User {
    private Long id;
    private String name;

    // getters/setters
}
```

**Input:**

```http
POST /api/users HTTP/1.1
Host: localhost:8080
Content-Type: application/json

{
  "name": "An"
}
```

**Output:**

```json
{
  "id": 1,
  "name": "An"
}
```

---

### 3.3. @PutMapping và @DeleteMapping

- `@PutMapping`: cập nhật dữ liệu.
- `@DeleteMapping`: xóa dữ liệu.

```java
@RestController
@RequestMapping("/api/users")
public class UserController {

    @PutMapping("/{id}")
    public String updateUser(@PathVariable Long id, @RequestBody User user) {
        return "Đã cập nhật user " + id + " với tên " + user.getName();
    }

    @DeleteMapping("/{id}")
    public String deleteUser(@PathVariable Long id) {
        return "Đã xóa user " + id;
    }
}
```

**Input (PUT):**

```http
PUT /api/users/1 HTTP/1.1
Host: localhost:8080
Content-Type: application/json

{
  "name": "An mới"
}
```

**Output (PUT):**

```json
"Đã cập nhật user 1 với tên An mới"
```

**Input (DELETE):**

```http
DELETE /api/users/1 HTTP/1.1
Host: localhost:8080
```

**Output (DELETE):**

```json
"Đã xóa user 1"
```

---

## Tóm tắt ý chính

- `@RestController`: tạo REST API, trả về JSON/XML trực tiếp.
- `@RequestMapping`: mapping URL + HTTP method, dùng cho class và method.
- `@GetMapping`, `@PostMapping`, `@PutMapping`, `@DeleteMapping`: phiên bản rút gọn của `@RequestMapping` cho từng HTTP method.
- Thường kết hợp với:
  - `@PathVariable` để lấy dữ liệu từ path.
  - `@RequestParam` để lấy query param.
  - `@RequestBody` để nhận JSON từ body request.
