
Dưới đây là tóm tắt ngắn gọn về các “synchronizer” trong `java.util.concurrent`:  
**CountDownLatch, CyclicBarrier, Semaphore, Phaser** – kèm ví dụ code và input/output minh họa.

---

## 1. CountDownLatch

**Ý tưởng chính**  
- Dùng để **chờ một số lượng sự kiện** xảy ra xong rồi mới tiếp tục.  
- Có một “đồng hồ đếm ngược” với giá trị ban đầu `count`.  
- Mỗi lần gọi `countDown()` → `count` giảm 1.  
- Thread gọi `await()` sẽ **chờ đến khi `count` về 0**.

**Trường hợp dùng**  
- Main thread chờ nhiều worker thread hoàn thành xong rồi mới xử lý tiếp.  
- Khởi tạo tài nguyên xong hết mới cho hệ thống chạy.

**Ví dụ**

```java
import java.util.concurrent.CountDownLatch;

public class CountDownLatchExample {
    public static void main(String[] args) throws InterruptedException {
        int numWorkers = 3;
        CountDownLatch latch = new CountDownLatch(numWorkers);

        for (int i = 1; i <= numWorkers; i++) {
            int id = i;
            new Thread(() -> {
                System.out.println("Worker " + id + " bắt đầu");
                try {
                    Thread.sleep(1000L * id); // giả lập xử lý
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
                System.out.println("Worker " + id + " xong");
                latch.countDown(); // báo đã xong
            }).start();
        }

        System.out.println("Main chờ các worker...");
        latch.await(); // chờ đến khi countDown đủ 3 lần
        System.out.println("Tất cả worker xong, main tiếp tục");
    }
}
```

**Output (minh họa)**

```
Main chờ các worker...
Worker 1 bắt đầu
Worker 2 bắt đầu
Worker 3 bắt đầu
Worker 1 xong
Worker 2 xong
Worker 3 xong
Tất cả worker xong, main tiếp tục
```

## 2. CyclicBarrier

**Ý tưởng chính**  
- Dùng để **đồng bộ nhiều thread tại một “điểm hẹn” (barrier)**.  
- Khi đủ `N` thread gọi `await()` → tất cả cùng được “mở khóa” và tiếp tục.  
- “Cyclic” nghĩa là **có thể dùng lại nhiều lần** (sau mỗi lần đủ N thread, barrier reset).

**Trường hợp dùng**  
- N thread xử lý từng “giai đoạn” (phase) của thuật toán song song, và phải chờ nhau sau mỗi giai đoạn.

**Ví dụ**

```java
import java.util.concurrent.BrokenBarrierException;
import java.util.concurrent.CyclicBarrier;

public class CyclicBarrierExample {
    public static void main(String[] args) {
        int numWorkers = 3;

        CyclicBarrier barrier = new CyclicBarrier(numWorkers, () -> {
            // Hành động chạy khi đủ 3 thread đến barrier
            System.out.println("== Tất cả worker đã đến barrier, sang phase mới ==");
        });

        for (int i = 1; i <= numWorkers; i++) {
            int id = i;
            new Thread(() -> {
                try {
                    System.out.println("Worker " + id + " xử lý phase 1");
                    Thread.sleep(500L * id);
                    System.out.println("Worker " + id + " chờ tại barrier");
                    barrier.await(); // chờ các worker khác

                    System.out.println("Worker " + id + " xử lý phase 2");
                } catch (InterruptedException | BrokenBarrierException e) {
                    Thread.currentThread().interrupt();
                }
            }).start();
        }
    }
}
```

**Output (minh họa)**

```
Worker 1 xử lý phase 1
Worker 2 xử lý phase 1
Worker 3 xử lý phase 1
Worker 1 chờ tại barrier
Worker 2 chờ tại barrier
Worker 3 chờ tại barrier
== Tất cả worker đã đến barrier, sang phase mới ==
Worker 1 xử lý phase 2
Worker 2 xử lý phase 2
Worker 3 xử lý phase 2
```

## 3. Semaphore

**Ý tưởng chính**  
- Dùng để **giới hạn số thread được truy cập đồng thời** vào một tài nguyên.  
- Có một số “permit” (giấy phép).  
- `acquire()` → xin permit (nếu hết thì chờ).  
- `release()` → trả lại permit.

**Trường hợp dùng**  
- Giới hạn số kết nối đồng thời đến DB, số thread truy cập vào vùng code nhất định, số client truy cập API,…

**Ví dụ**

```java
import java.util.concurrent.Semaphore;

public class SemaphoreExample {
    public static void main(String[] args) {
        int maxConcurrent = 2;
        Semaphore semaphore = new Semaphore(maxConcurrent);

        for (int i = 1; i <= 5; i++) {
            int id = i;
            new Thread(() -> {
                try {
                    System.out.println("Task " + id + " chờ permit");
                    semaphore.acquire(); // xin permit
                    System.out.println("Task " + id + " đang chạy (có permit)");
                    Thread.sleep(1000L);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                } finally {
                    System.out.println("Task " + id + " trả permit");
                    semaphore.release(); // trả permit
                }
            }).start();
        }
    }
}
```

**Output (minh họa)**  
Chỉ tối đa 2 task chạy “đang chạy (có permit)” cùng lúc:

```
Task 1 chờ permit
Task 1 đang chạy (có permit)
Task 2 chờ permit
Task 2 đang chạy (có permit)
Task 3 chờ permit
Task 4 chờ permit
Task 5 chờ permit
Task 1 trả permit
Task 3 đang chạy (có permit)
Task 2 trả permit
Task 4 đang chạy (có permit)
...
```

## 4. Phaser

**Ý tưởng chính**  
- Tương tự `CyclicBarrier` nhưng **linh hoạt hơn**:  
  - Hỗ trợ **nhiều phase** (giai đoạn) một cách tự nhiên.  
  - Cho phép **đăng ký/huỷ đăng ký** số lượng party (thread) động (`register()`, `arriveAndDeregister()`).  
- Mỗi phase: các party gọi `arriveAndAwaitAdvance()` để chờ nhau.

**Trường hợp dùng**  
- Bài toán nhiều giai đoạn, số thread có thể thay đổi theo thời gian.  
- Workflow phức tạp hơn barrier đơn giản.

**Ví dụ**

```java
import java.util.concurrent.Phaser;

public class PhaserExample {
    public static void main(String[] args) {
        int numWorkers = 3;
        Phaser phaser = new Phaser(numWorkers); // đăng ký 3 party ban đầu

        for (int i = 1; i <= numWorkers; i++) {
            int id = i;
            new Thread(() -> {
                System.out.println("Worker " + id + " phase 0");
                phaser.arriveAndAwaitAdvance(); // kết thúc phase 0

                System.out.println("Worker " + id + " phase 1");
                phaser.arriveAndAwaitAdvance(); // kết thúc phase 1

                System.out.println("Worker " + id + " phase 2 và rời phaser");
                phaser.arriveAndDeregister(); // xong việc, rời phaser
            }).start();
        }

        // Main có thể theo dõi phase
        while (!phaser.isTerminated()) {
            System.out.println("Main: current phase = " + phaser.getPhase()
                    + ", registered parties = " + phaser.getRegisteredParties());
            try {
                Thread.sleep(500L);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
            if (phaser.getRegisteredParties() == 0) {
                phaser.forceTermination();
            }
        }
        System.out.println("Phaser terminated");
    }
}
```

**Output (minh họa, rút gọn)**

```
Worker 1 phase 0
Worker 2 phase 0
Worker 3 phase 0
Main: current phase = 0, registered parties = 3
Worker 1 phase 1
Worker 2 phase 1
Worker 3 phase 1
Main: current phase = 1, registered parties = 3
Worker 1 phase 2 và rời phaser
Worker 2 phase 2 và rời phaser
Worker 3 phase 2 và rời phaser
Main: current phase = 2, registered parties = 0
Phaser terminated
```

## Tóm tắt so sánh nhanh

- **CountDownLatch**  
  - Một lần dùng (không reset).  
  - Chờ đến khi đếm về 0.  
  - Phù hợp: chờ nhiều task khởi tạo/hoàn thành.

- **CyclicBarrier**  
  - Dùng lại nhiều lần (cyclic).  
  - N thread gặp nhau tại barrier, rồi cùng tiếp tục.  
  - Phù hợp: thuật toán chia phase cố định, số thread cố định.

- **Semaphore**  
  - Giới hạn số thread truy cập tài nguyên.  
  - Phù hợp: pool kết nối, giới hạn truy cập.

- **Phaser**  
  - Nhiều phase, số thread có thể thay đổi.  
  - Linh hoạt hơn CyclicBarrier.  
  - Phù hợp: workflow nhiều giai đoạn, dynamic.
