
Dưới đây là tóm tắt ngắn gọn về 3 mẫu thiết kế cấu trúc trong Java: **Adapter, Facade, Decorator**, kèm ví dụ code và input/output.

---

## 1. Adapter Pattern (Mẫu Bộ Chuyển Đổi)

**Mục đích:**  
Chuyển đổi *giao diện* của một lớp hiện có sang một giao diện khác mà client mong muốn.  
Giúp tái sử dụng code cũ mà không cần sửa trực tiếp.

**Khi dùng:**

- Có lớp đã tồn tại nhưng giao diện không phù hợp với hệ thống mới.
- Muốn “bọc” (wrap) một lớp để nó trông giống như một interface khác.

### Ví dụ Java

Giả sử hệ thống cần interface `MediaPlayer` nhưng ta chỉ có lớp `AdvancedPlayer` với phương thức khác tên.

```java
// Giao diện mà client mong muốn
interface MediaPlayer {
    void play(String fileName);
}

// Lớp hiện có, giao diện không khớp
class AdvancedPlayer {
    public void playMp3File(String file) {
        System.out.println("Playing MP3 file: " + file);
    }
}

// Adapter: chuyển AdvancedPlayer -> MediaPlayer
class MediaPlayerAdapter implements MediaPlayer {
    private AdvancedPlayer advancedPlayer;

    public MediaPlayerAdapter(AdvancedPlayer advancedPlayer) {
        this.advancedPlayer = advancedPlayer;
    }

    @Override
    public void play(String fileName) {
        advancedPlayer.playMp3File(fileName);
    }
}

// Client code
public class AdapterDemo {
    public static void main(String[] args) {
        MediaPlayer player = new MediaPlayerAdapter(new AdvancedPlayer());
        player.play("song.mp3");
    }
}
```

**Output:**
```text
Playing MP3 file: song.mp3
```

## 2. Facade Pattern (Mẫu Mặt Tiền)

**Mục đích:**  
Cung cấp một *giao diện đơn giản* cho một hệ thống phức tạp (nhiều lớp, nhiều bước).  
Client chỉ làm việc với Facade, không cần biết chi tiết bên trong.

**Khi dùng:**

- Hệ thống có nhiều lớp, nhiều bước khởi tạo/phối hợp.
- Muốn giảm độ phức tạp cho client, chỉ expose vài phương thức đơn giản.

### Ví dụ Java

Hệ thống xem phim gồm nhiều bước: bật đèn, bật máy chiếu, bật âm thanh…

```java
class Light {
    void dim() { System.out.println("Lights dimmed"); }
}

class Projector {
    void on() { System.out.println("Projector on"); }
}

class SoundSystem {
    void on() { System.out.println("Sound system on"); }
}

// Facade: đơn giản hóa việc xem phim
class HomeTheaterFacade {
    private Light light;
    private Projector projector;
    private SoundSystem soundSystem;

    public HomeTheaterFacade(Light light, Projector projector, SoundSystem soundSystem) {
        this.light = light;
        this.projector = projector;
        this.soundSystem = soundSystem;
    }

    public void watchMovie(String movie) {
        System.out.println("Get ready to watch: " + movie);
        light.dim();
        projector.on();
        soundSystem.on();
        System.out.println("Movie started!");
    }
}

// Client code
public class FacadeDemo {
    public static void main(String[] args) {
        HomeTheaterFacade theater = new HomeTheaterFacade(
                new Light(), new Projector(), new SoundSystem()
        );
        theater.watchMovie("Inception");
    }
}
```

**Output:**
```text
Get ready to watch: Inception
Lights dimmed
Projector on
Sound system on
Movie started!
```

## 3. Decorator Pattern (Mẫu Trang Trí)

**Mục đích:**  
Thêm *hành vi hoặc chức năng mới* cho đối tượng **một cách linh hoạt**, bằng cách “bọc” nó trong các lớp decorator, thay vì kế thừa cứng.

**Khi dùng:**

- Muốn thêm/bớt tính năng động (runtime), không muốn tạo quá nhiều subclass.
- Cần kết hợp nhiều “topping”/“option” khác nhau cho một đối tượng.

### Ví dụ Java

Ví dụ về hệ thống đặt đồ uống (coffee) với nhiều loại topping.

```java
// Component
interface Beverage {
    String getDescription();
    double cost();
}

// Concrete Component
class Espresso implements Beverage {
    @Override
    public String getDescription() {
        return "Espresso";
    }

    @Override
    public double cost() {
        return 2.0;
    }
}

// Decorator cơ sở
abstract class BeverageDecorator implements Beverage {
    protected Beverage beverage;

    public BeverageDecorator(Beverage beverage) {
        this.beverage = beverage;
    }
}

// Concrete Decorators
class MilkDecorator extends BeverageDecorator {
    public MilkDecorator(Beverage beverage) {
        super(beverage);
    }

    @Override
    public String getDescription() {
        return beverage.getDescription() + ", Milk";
    }

    @Override
    public double cost() {
        return beverage.cost() + 0.5;
    }
}

class SugarDecorator extends BeverageDecorator {
    public SugarDecorator(Beverage beverage) {
        super(beverage);
    }

    @Override
    public String getDescription() {
        return beverage.getDescription() + ", Sugar";
    }

    @Override
    public double cost() {
        return beverage.cost() + 0.2;
    }
}

// Client code
public class DecoratorDemo {
    public static void main(String[] args) {
        Beverage coffee = new Espresso();                 // Espresso
        coffee = new MilkDecorator(coffee);              // + Milk
        coffee = new SugarDecorator(coffee);             // + Sugar

        System.out.println(coffee.getDescription());
        System.out.println("Total cost: " + coffee.cost());
    }
}
```

**Output:**
```text
Espresso, Milk, Sugar
Total cost: 2.7
```

## Tóm tắt ngắn gọn

- **Adapter**:  
  - Chuyển đổi *giao diện* của một lớp sang giao diện khác mà client cần.  
  - Dùng khi có code cũ/giao diện không khớp.

- **Facade**:  
  - Cung cấp *giao diện đơn giản* cho một hệ thống phức tạp.  
  - Dùng để ẩn bớt chi tiết, giảm độ phức tạp cho client.

- **Decorator**:  
  - Thêm chức năng cho đối tượng bằng cách *bọc* nó, không cần sửa lớp gốc.  
  - Dùng khi cần mở rộng linh hoạt, kết hợp nhiều “tính năng phụ” khác nhau.

Nếu bạn muốn, tôi có thể thêm ví dụ nâng cao hơn hoặc so sánh 3 mẫu này trong cùng một sơ đồ.
