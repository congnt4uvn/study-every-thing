
Dưới đây là tóm tắt ngắn gọn về các loại Lock trong `java.util.concurrent`: `ReentrantLock`, `ReadWriteLock`, `StampedLock`, kèm ví dụ code và input/output minh họa.

---

## 1. ReentrantLock

**Ý chính:**
- Là lock “tái nhập” (reentrant): cùng một thread có thể lock nhiều lần mà không bị kẹt.
- Cung cấp các tính năng nâng cao hơn `synchronized`:
  - `tryLock()` (thử lock, không chờ vô hạn)
  - `lockInterruptibly()` (có thể bị interrupt khi đang chờ)
  - Hỗ trợ fairness (công bằng) theo thứ tự FIFO.

**Ví dụ đơn giản:**

```java
import java.util.concurrent.locks.ReentrantLock;

public class Counter {
    private int count = 0;
    private final ReentrantLock lock = new ReentrantLock();

    public void increment() {
        lock.lock();              // lấy lock
        try {
            count++;
        } finally {
            lock.unlock();        // luôn luôn unlock trong finally
        }
    }

    public int getCount() {
        return count;
    }

    public static void main(String[] args) throws InterruptedException {
        Counter counter = new Counter();

        Runnable task = () -> {
            for (int i = 0; i < 1000; i++) {
                counter.increment();
            }
        };

        Thread t1 = new Thread(task);
        Thread t2 = new Thread(task);
        t1.start();
        t2.start();
        t1.join();
        t2.join();

        System.out.println("Final count = " + counter.getCount());
    }
}
```

**Input:** 2 thread, mỗi thread tăng `count` 1000 lần  
**Output mong đợi:**  
`Final count = 2000`  
(nếu không dùng lock, kết quả có thể < 2000 do race condition)

## 2. ReadWriteLock (ReentrantReadWriteLock)

**Ý chính:**
- Cho phép:
  - Nhiều thread **đọc** cùng lúc (read lock chia sẻ).
  - Chỉ 1 thread **ghi** tại một thời điểm (write lock độc quyền).
- Phù hợp khi số lượng đọc nhiều hơn ghi → tăng hiệu năng so với lock độc quyền.

**Ví dụ:**

```java
import java.util.concurrent.locks.ReadWriteLock;
import java.util.concurrent.locks.ReentrantReadWriteLock;

public class SharedData {
    private int value = 0;
    private final ReadWriteLock rwLock = new ReentrantReadWriteLock();

    public void write(int newValue) {
        rwLock.writeLock().lock();
        try {
            value = newValue;
            System.out.println(Thread.currentThread().getName() +
                               " ghi value = " + value);
        } finally {
            rwLock.writeLock().unlock();
        }
    }

    public int read() {
        rwLock.readLock().lock();
        try {
            int v = value;
            System.out.println(Thread.currentThread().getName() +
                               " đọc value = " + v);
            return v;
        } finally {
            rwLock.readLock().unlock();
        }
    }

    public static void main(String[] args) {
        SharedData data = new SharedData();

        // Thread ghi
        Thread writer = new Thread(() -> {
            for (int i = 1; i <= 3; i++) {
                data.write(i);
                try { Thread.sleep(50); } catch (InterruptedException ignored) {}
            }
        }, "Writer");

        // Nhiều thread đọc
        Runnable readerTask = () -> {
            for (int i = 0; i < 5; i++) {
                data.read();
                try { Thread.sleep(30); } catch (InterruptedException ignored) {}
            }
        };

        Thread r1 = new Thread(readerTask, "Reader-1");
        Thread r2 = new Thread(readerTask, "Reader-2");

        writer.start();
        r1.start();
        r2.start();
    }
}
```

**Output (mang tính minh họa, thứ tự có thể khác):**
- Nhiều dòng `Reader-1 đọc value = ...` và `Reader-2 đọc value = ...` có thể xuất hiện song song.
- `Writer ghi value = ...` sẽ đảm bảo độc quyền khi ghi.

## 3. StampedLock

**Ý chính:**
- Cung cấp 3 kiểu lock:
  1. **Write lock** (độc quyền, giống writeLock).
  2. **Read lock** (chia sẻ, giống readLock).
  3. **Optimistic read** (đọc lạc quan, không chặn, rất nhanh, nhưng phải **xác thực**).
- Không reentrant (không tái nhập).
- Thích hợp cho trường hợp đọc rất nhiều, ghi ít, cần hiệu năng cao.

**Ví dụ với optimistic read:**

```java
import java.util.concurrent.locks.StampedLock;

public class Point {
    private double x, y;
    private final StampedLock lock = new StampedLock();

    // Ghi: di chuyển điểm
    public void move(double deltaX, double deltaY) {
        long stamp = lock.writeLock();   // lấy write lock
        try {
            x += deltaX;
            y += deltaY;
        } finally {
            lock.unlockWrite(stamp);
        }
    }

    // Đọc: tính khoảng cách đến gốc (0,0) với optimistic read
    public double distanceFromOrigin() {
        long stamp = lock.tryOptimisticRead(); // không chặn
        double currentX = x;
        double currentY = y;

        // Kiểm tra xem trong lúc đọc có ai ghi không
        if (!lock.validate(stamp)) {
            // Nếu có ghi, chuyển sang read lock “thật”
            stamp = lock.readLock();
            try {
                currentX = x;
                currentY = y;
            } finally {
                lock.unlockRead(stamp);
            }
        }
        return Math.sqrt(currentX * currentX + currentY * currentY);
    }

    public static void main(String[] args) {
        Point p = new Point();
        p.move(3, 4);
        double d = p.distanceFromOrigin();
        System.out.println("Distance = " + d);
    }
}
```

**Input:** `move(3, 4)` rồi gọi `distanceFromOrigin()`  
**Output:**  
`Distance = 5.0` (vì √(3² + 4²) = 5)

## Tóm tắt so sánh

- **ReentrantLock**
  - Lock độc quyền, tái nhập.
  - Thay thế linh hoạt cho `synchronized`.
- **ReadWriteLock (ReentrantReadWriteLock)**
  - Tách read lock / write lock.
  - Nhiều đọc song song, ghi độc quyền.
- **StampedLock**
  - Hỗ trợ read, write, và optimistic read.
  - Không reentrant, dùng cho trường hợp đọc rất nhiều, cần tối ưu hiệu năng.
