
Dưới đây là tóm tắt ngắn gọn về Spring AOP, tập trung vào ý chính, kèm ví dụ code và input/output.

---

## 1. AOP là gì? (Aspect-Oriented Programming)

- Dùng để xử lý **cross-cutting concerns** (các mối quan tâm cắt ngang) như:
  - Logging (ghi log)
  - Transaction (giao dịch)
  - Security (bảo mật)
  - Caching
- Thay vì viết lặp lại ở nhiều class, ta tách ra thành **Aspect** và để Spring tự “chen” vào các điểm phù hợp.

---

## 2. @Aspect

- Là annotation để đánh dấu một class là **Aspect**.
- Bên trong class @Aspect, ta định nghĩa:
  - **Pointcut**: chỉ ra *nơi* sẽ áp dụng
  - **Advice**: chỉ ra *làm gì* và *khi nào* (trước/sau/quanh method)

```java
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.aspectj.lang.annotation.Pointcut;
import org.springframework.stereotype.Component;

@Aspect
@Component
public class LoggingAspect {

    // Pointcut: tất cả method trong package service
    @Pointcut("execution(* com.example.service.*.*(..))")
    public void serviceMethods() {}

    // Advice: chạy TRƯỚC khi method trong service được gọi
    @Before("serviceMethods()")
    public void logBefore() {
        System.out.println("[LOG] Method trong service sắp được gọi");
    }
}
```

---

## 3. Pointcut

- Dùng để **chỉ định vị trí** áp dụng AOP (method nào, class nào, package nào).
- Thường dùng biểu thức `execution(...)`.

Ví dụ:

```java
// Tất cả method public trong package service
@Pointcut("execution(public * com.example.service.*.*(..))")
public void allPublicServiceMethods() {}

// Tất cả method tên là 'save' trong mọi class
@Pointcut("execution(* *..*Service.save(..))")
public void allSaveMethods() {}
```

## 4. Các loại Advice

### 4.1 @Before

Chạy **trước** khi method mục tiêu được gọi.

```java
@Before("allPublicServiceMethods()")
public void beforeAdvice() {
    System.out.println("[Before] Chuẩn bị gọi method service");
}
```

**Input/Output ví dụ**

```java
// Service
public class UserService {
    public String getUserName(int id) {
        return "User_" + id;
    }
}

// Gọi:
userService.getUserName(10);
```

**Console output:**

```text
[Before] Chuẩn bị gọi method service
```

---

### 4.2 @AfterReturning

Chạy **sau khi method chạy thành công** (không ném exception).

```java
import org.aspectj.lang.annotation.AfterReturning;

@AfterReturning(pointcut = "allPublicServiceMethods()", returning = "result")
public void afterReturningAdvice(Object result) {
    System.out.println("[AfterReturning] Kết quả trả về: " + result);
}
```

**Input/Output ví dụ**

```java
// Gọi:
String name = userService.getUserName(10);
System.out.println("Client nhận: " + name);
```

**Console output:**

```text
[Before] Chuẩn bị gọi method service
[AfterReturning] Kết quả trả về: User_10
Client nhận: User_10
```

---

### 4.3 @AfterThrowing

Chạy khi method **ném exception**.

```java
import org.aspectj.lang.annotation.AfterThrowing;

@AfterThrowing(pointcut = "allPublicServiceMethods()", throwing = "ex")
public void afterThrowingAdvice(Exception ex) {
    System.out.println("[AfterThrowing] Có lỗi xảy ra: " + ex.getMessage());
}
```

**Input/Output ví dụ**

```java
public String getUserName(int id) {
    if (id < 0) {
        throw new IllegalArgumentException("ID không hợp lệ");
    }
    return "User_" + id;
}

// Gọi:
userService.getUserName(-1);
```

**Console output:**

```text
[Before] Chuẩn bị gọi method service
[AfterThrowing] Có lỗi xảy ra: ID không hợp lệ
Exception in thread "main" java.lang.IllegalArgumentException: ID không hợp lệ
...
```

---

### 4.4 @After (finally)

Chạy **sau cùng**, dù thành công hay lỗi (giống `finally` trong try-catch-finally).

```java
import org.aspectj.lang.annotation.After;

@After("allPublicServiceMethods()")
public void afterAdvice() {
    System.out.println("[After] Method service đã kết thúc (thành công hoặc lỗi)");
}
```

---

### 4.5 @Around

- Bao quanh toàn bộ quá trình gọi method.
- Cho phép:
  - Chặn/cho phép thực thi
  - Sửa input
  - Sửa output
  - Đo thời gian chạy

```java
import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.Around;

@Around("allPublicServiceMethods()")
public Object aroundAdvice(ProceedingJoinPoint pjp) throws Throwable {
    long start = System.currentTimeMillis();
    System.out.println("[Around] Bắt đầu: " + pjp.getSignature());

    Object result = pjp.proceed(); // gọi method thật

    long end = System.currentTimeMillis();
    System.out.println("[Around] Kết thúc: " + pjp.getSignature()
            + ", thời gian: " + (end - start) + "ms");

    return result;
}
```

**Input/Output ví dụ**

```java
// Gọi:
userService.getUserName(5);
```

**Console output (thứ tự có thể):**

```text
[Around] Bắt đầu: String com.example.service.UserService.getUserName(int)
[Before] Chuẩn bị gọi method service
[AfterReturning] Kết quả trả về: User_5
[After] Method service đã kết thúc (thành công hoặc lỗi)
[Around] Kết thúc: String com.example.service.UserService.getUserName(int), thời gian: 2ms
```

## 5. Tóm tắt ý chính

- **Cross-cutting concerns**: logic dùng chung (log, bảo mật, transaction) cắt ngang nhiều lớp.
- **@Aspect**: đánh dấu class chứa logic AOP.
- **Pointcut**: định nghĩa *nơi* áp dụng (method nào, package nào).
- **Advice**: định nghĩa *khi nào* và *làm gì*:
  - `@Before`: trước method
  - `@AfterReturning`: sau khi thành công
  - `@AfterThrowing`: khi có exception
  - `@After`: luôn chạy sau cùng
  - `@Around`: bao quanh toàn bộ, linh hoạt nhất

Nếu bạn muốn, tôi có thể viết một ví dụ Spring Boot hoàn chỉnh (pom.xml, cấu hình, service, aspect) để bạn chạy thử.
