
Dưới đây là tóm tắt ngắn gọn về các tính năng mới trong Java liên quan đến **Project Loom**: **virtual threads** và **structured concurrency**.

---

## 1. Virtual Threads (Luồng ảo)

**Ý chính:**

- Là loại thread mới trong Java, rất nhẹ, được quản lý bởi JVM (không phải OS).
- Cho phép tạo **hàng chục nghìn đến hàng triệu** thread mà không tốn nhiều tài nguyên như platform thread truyền thống.
- Mục tiêu: đơn giản hóa lập trình đồng thời (concurrency) theo kiểu “mỗi request một thread” mà vẫn hiệu quả.

**Đặc điểm:**

- API vẫn dùng `java.lang.Thread`, nhưng tạo bằng `Thread.ofVirtual()`.
- Thích hợp cho các tác vụ I/O (network, file, DB) – khi chờ I/O, virtual thread được “park” và không chiếm OS thread.
- Giảm nhu cầu dùng callback phức tạp, CompletableFuture, reactive framework… trong nhiều trường hợp.

**Ví dụ 1: Tạo virtual thread đơn giản**

```java
public class VirtualThreadExample {
    public static void main(String[] args) throws InterruptedException {
        Thread vThread = Thread.ofVirtual().start(() -> {
            System.out.println("Hello from virtual thread!");
        });

        vThread.join(); // chờ thread kết thúc
    }
}
```

**Output:**
```
Hello from virtual thread!
```

---

**Ví dụ 2: Tạo nhiều virtual thread xử lý song song**

```java
import java.util.stream.IntStream;

public class ManyVirtualThreads {
    public static void main(String[] args) {
        IntStream.range(0, 5).forEach(i -> {
            Thread.ofVirtual().start(() -> {
                System.out.println("Task " + i + " running in " + Thread.currentThread());
                try {
                    Thread.sleep(1000); // giả lập I/O hoặc tác vụ chậm
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
                System.out.println("Task " + i + " done");
            });
        });

        // Chờ cho dễ nhìn (trong thực tế nên join hoặc dùng executor)
        try { Thread.sleep(2000); } catch (InterruptedException e) {}
    }
}
```

**Output (thứ tự có thể khác):**
```
Task 0 running in VirtualThread[#...]
Task 1 running in VirtualThread[#...]
Task 2 running in VirtualThread[#...]
Task 3 running in VirtualThread[#...]
Task 4 running in VirtualThread[#...]
Task 0 done
Task 1 done
Task 2 done
Task 3 done
Task 4 done
```

**Ví dụ 3: Dùng ExecutorService với virtual threads**

```java
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class VirtualThreadExecutor {
    public static void main(String[] args) {
        try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {
            for (int i = 0; i < 5; i++) {
                int taskId = i;
                executor.submit(() -> {
                    System.out.println("Task " + taskId + " in " + Thread.currentThread());
                    Thread.sleep(500);
                    return taskId;
                });
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

## 2. Structured Concurrency (Tính đồng thời có cấu trúc)

**Ý chính:**

- Mục tiêu: quản lý các tác vụ song song theo **cấu trúc khối** (block-structured), giống như `try-with-resources`.
- Giúp:
  - Dễ **hủy** (cancel) toàn bộ nhóm task khi có lỗi.
  - Dễ **chờ** tất cả task hoàn thành.
  - Tránh “thread bị rò rỉ” (orphan threads) chạy ngoài vòng đời logic của request.

**Khái niệm:**

- Dùng `StructuredTaskScope` (trong `java.util.concurrent`) để:
  - Fork (tạo) nhiều task con.
  - Join (chờ) chúng.
  - Xử lý kết quả hoặc lỗi theo một nơi tập trung.

---

**Ví dụ 4: Structured concurrency với StructuredTaskScope**

```java
import java.time.Duration;
import java.util.concurrent.*;

public class StructuredConcurrencyExample {

    static String fetchUser() throws InterruptedException {
        Thread.sleep(500); // giả lập gọi service
        return "User A";
    }

    static String fetchOrders() throws InterruptedException {
        Thread.sleep(800); // giả lập gọi service
        return "Orders: [1, 2, 3]";
    }

    public static void main(String[] args) throws Exception {
        try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
            Future<String> userFuture = scope.fork(() -> fetchUser());
            Future<String> ordersFuture = scope.fork(() -> fetchOrders());

            // Chờ tất cả task (hoặc dừng nếu có task lỗi)
            scope.join();
            scope.throwIfFailed(); // ném exception nếu có task nào thất bại

            String user = userFuture.resultNow();
            String orders = ordersFuture.resultNow();

            System.out.println("Result: " + user + " - " + orders);
        }
    }
}
```

**Output:**
```
Result: User A - Orders: [1, 2, 3]
```

**Giải thích ngắn:**

- `ShutdownOnFailure`: nếu một task lỗi, scope sẽ hủy các task còn lại.
- `scope.join()`: chờ tất cả task kết thúc (hoặc bị hủy).
- `scope.throwIfFailed()`: nếu có lỗi, ném exception ra ngoài.

## 3. Tóm tắt ý chính

- **Virtual Threads**:
  - Thread cực nhẹ, số lượng rất lớn.
  - Thích hợp cho ứng dụng server, I/O-bound.
  - Giữ được mô hình lập trình “mỗi request một thread” nhưng hiệu quả hơn.

- **Structured Concurrency**:
  - Tổ chức các tác vụ song song theo khối, dễ quản lý vòng đời.
  - Hỗ trợ hủy nhóm task, gom lỗi, và chờ kết quả một cách rõ ràng.
  - Giúp code đồng thời dễ đọc, dễ bảo trì, ít bug hơn.

Nếu bạn muốn, tôi có thể viết thêm ví dụ cụ thể cho một API web (REST) dùng virtual threads + structured concurrency.
