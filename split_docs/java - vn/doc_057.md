
Dưới đây là tóm tắt ngắn gọn về các annotation Spring Core liên quan đến IoC/DI, kèm ví dụ code và input/output.

---

## 1. Khái niệm chung: IoC & DI

- **IoC (Inversion of Control)**: Thay vì code tự tạo và quản lý object, Spring Container sẽ làm việc đó.
- **DI (Dependency Injection)**: Spring “tiêm” (inject) các phụ thuộc (dependency) vào class thông qua constructor, setter, hoặc field.

---

## 2. Các stereotype annotation chính

### 2.1 `@Component`
- Annotation chung cho một bean do Spring quản lý.
- Dùng cho các class “thường” không thuộc các tầng cụ thể (service, repository, controller).

```java
import org.springframework.stereotype.Component;

@Component
public class EmailValidator {
    public boolean isValid(String email) {
        return email != null && email.contains("@");
    }
}
```

**Sử dụng (DI qua constructor):**

```java
import org.springframework.stereotype.Service;

@Service
public class UserRegistrationService {

    private final EmailValidator emailValidator;

    public UserRegistrationService(EmailValidator emailValidator) {
        this.emailValidator = emailValidator;
    }

    public String register(String email) {
        if (!emailValidator.isValid(email)) {
            return "Email không hợp lệ";
        }
        // logic lưu user...
        return "Đăng ký thành công";
    }
}
```

**Input/Output (giả sử gọi từ controller):**

- Input: `email = "test@example.com"`
- Output: `"Đăng ký thành công"`

---

### 2.2 `@Service`
- Dùng cho **tầng service** (business logic).
- Về mặt kỹ thuật giống `@Component`, nhưng mang ý nghĩa rõ ràng về mặt kiến trúc.

```java
import org.springframework.stereotype.Service;

@Service
public class PaymentService {
    public String pay(long amount) {
        // logic thanh toán
        return "Đã thanh toán: " + amount;
    }
}
```

**Input/Output:**

- Input: `amount = 100000`
- Output: `"Đã thanh toán: 100000"`

---

### 2.3 `@Repository`
- Dùng cho **tầng truy cập dữ liệu** (DAO).
- Ngoài việc là một `@Component`, Spring có thể:
  - Chuyển đổi exception của JPA/JDBC thành `DataAccessException`.

```java
import org.springframework.stereotype.Repository;
import java.util.*;

@Repository
public class UserRepository {
    private final Map<Long, String> data = new HashMap<>();

    public void save(Long id, String name) {
        data.put(id, name);
    }

    public String findById(Long id) {
        return data.get(id);
    }
}
```

**Input/Output:**

- `save(1L, "An")` → lưu user
- `findById(1L)` → `"An"`

---

### 2.4 `@Controller`
- Dùng cho **tầng web (MVC)**, xử lý HTTP request, trả về view (hoặc dữ liệu).
- Thường dùng với `@RequestMapping`, `@GetMapping`, `@PostMapping`,...

```java
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.*;

@Controller
@RequestMapping("/users")
public class UserController {

    private final UserRegistrationService registrationService;

    public UserController(UserRegistrationService registrationService) {
        this.registrationService = registrationService;
    }

    @GetMapping("/register")
    @ResponseBody
    public String register(@RequestParam String email) {
        return registrationService.register(email);
    }
}
```

**HTTP Input/Output:**

- Request: `GET /users/register?email=test@example.com`
- Response body: `"Đăng ký thành công"`

> Nếu muốn trả JSON thay vì view, thường dùng `@RestController` (tương đương `@Controller + @ResponseBody`).

## 3. Cấu hình bằng Java: `@Configuration` & `@Bean`

### 3.1 `@Configuration`
- Đánh dấu class là **class cấu hình** Spring (thay cho XML).
- Chứa các method tạo bean với `@Bean`.

```java
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Bean;

@Configuration
public class AppConfig {

    @Bean
    public EmailValidator emailValidator() {
        return new EmailValidator(); // tự tạo object
    }

    @Bean
    public UserRegistrationService userRegistrationService(EmailValidator emailValidator) {
        // Spring sẽ inject EmailValidator vào đây
        return new UserRegistrationService(emailValidator);
    }
}
```

### 3.2 `@Bean`
- Đánh dấu một method trả về object sẽ được Spring quản lý như một bean.
- Thường dùng khi:
  - Cần cấu hình thư viện bên ngoài (không thể dùng `@Component`).
  - Cần kiểm soát cách khởi tạo bean.

**Ví dụ sử dụng bean:**

```java
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class MainApp {
    public static void main(String[] args) {
        var context = new AnnotationConfigApplicationContext(AppConfig.class);

        UserRegistrationService service = context.getBean(UserRegistrationService.class);
        String result = service.register("abc@example.com");
        System.out.println(result);
    }
}
```

**Console Output:**

```text
Đăng ký thành công
```

---

## 4. Tóm tắt nhanh

- `@Component`: Bean chung, không thuộc tầng cụ thể.
- `@Service`: Bean tầng service (business logic).
- `@Repository`: Bean tầng data access, hỗ trợ chuyển đổi exception.
- `@Controller`: Bean tầng web MVC, xử lý HTTP request.
- `@Configuration`: Class cấu hình, chứa định nghĩa bean.
- `@Bean`: Method tạo bean, Spring quản lý lifecycle.

Tất cả đều phục vụ cho **IoC/DI**: Spring tạo, quản lý và inject các object thay cho bạn.
