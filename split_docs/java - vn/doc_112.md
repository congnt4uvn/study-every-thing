
Dưới đây là tóm tắt ngắn gọn về khái niệm CI/CD và pipeline trong Cloud & DevOps, kèm ví dụ code (giả sử dùng Java + Maven).

---

## 1. CI/CD là gì?

- **CI (Continuous Integration – Tích hợp liên tục)**  
  - Lập trình viên đẩy (push) code thường xuyên lên repository (Git).  
  - Mỗi lần push, hệ thống tự động:
    - Build (biên dịch) code
    - Chạy test (unit test, integration test)
  - Mục tiêu: phát hiện lỗi sớm, tránh “vỡ” build.

- **CD (Continuous Delivery/Deployment – Triển khai liên tục)**  
  - **Continuous Delivery**: sau khi build + test thành công, tạo ra artifact (file .jar/.war, image Docker…) sẵn sàng deploy. Việc deploy có thể cần người bấm nút “Approve”.
  - **Continuous Deployment**: sau khi build + test thành công, hệ thống tự động deploy lên môi trường (staging/production) không cần thao tác tay.

---

## 2. Pipeline là gì?

**Pipeline** là chuỗi các bước tự động từ lúc code được commit cho đến khi chạy trên môi trường thật.  
Thường gồm 3 giai đoạn chính:

1. **Build**  
   - Biên dịch code (Java → bytecode)  
   - Quản lý dependency (Maven/Gradle)  
   - Tạo artifact (.jar, .war, Docker image)

2. **Test**  
   - Chạy unit test, integration test, UI test…  
   - Nếu test fail → dừng pipeline

3. **Deploy**  
   - Triển khai artifact lên môi trường:
     - Dev / Test / Staging / Production  
   - Có thể dùng: Docker, Kubernetes, cloud (AWS, GCP, Azure), v.v.

---

## 3. Ví dụ pipeline đơn giản (Jenkinsfile cho Java + Maven)

```groovy
pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                // Biên dịch project Java bằng Maven
                sh 'mvn clean package -DskipTests'
            }
        }

        stage('Test') {
            steps {
                // Chạy test
                sh 'mvn test'
            }
        }

        stage('Deploy') {
            steps {
                // Ví dụ: copy file .jar lên server (giả lập)
                sh 'scp target/myapp-1.0.jar user@server:/opt/apps/'
            }
        }
    }
}
```

**Dòng chảy (input/output) của pipeline:**

1. **Input**: Code Java trong Git repo  
2. **Build stage**  
   - Input: Source code  
   - Output: `target/myapp-1.0.jar`
3. **Test stage**  
   - Input: Code + test + jar  
   - Output: Kết quả test (pass/fail, report)
4. **Deploy stage**  
   - Input: `myapp-1.0.jar` đã pass test  
   - Output: Ứng dụng chạy trên server (dev/staging/prod)

## 4. Ví dụ code Java + test (để CI chạy)

**File Java: `Calculator.java`**

```java
public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}
```

**Unit test: `CalculatorTest.java` (JUnit)**

```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    void testAdd() {
        Calculator calc = new Calculator();
        int result = calc.add(2, 3);
        assertEquals(5, result); // mong đợi 2 + 3 = 5
    }
}
```

**Input test:**  
- Hàm `add(2, 3)`

**Output test:**  
- Kết quả thực tế: `5`  
- So sánh với mong đợi `5` → test **pass** → pipeline tiếp tục.

---

## 5. Tóm tắt ý chính

- CI/CD giúp:
  - Tự động hóa build, test, deploy
  - Giảm lỗi do thao tác tay
  - Rút ngắn thời gian đưa tính năng mới lên môi trường thật
- Pipeline thường gồm 3 stage chính:
  - **Build** → tạo artifact
  - **Test** → đảm bảo chất lượng
  - **Deploy** → đưa ứng dụng lên môi trường chạy thực tế  
- Java thường dùng Maven/Gradle + Jenkins/GitLab CI/GitHub Actions để xây dựng pipeline CI/CD.
