
Dưới đây là tóm tắt ngắn gọn về **sealed classes** trong Java (Java 17+), kèm ví dụ mã và input/output.

---

## 1. Sealed class là gì?

- `sealed` class/interface cho phép **giới hạn** những lớp/concrete type nào được phép kế thừa/implement.
- Mục tiêu:
  - Kiểm soát chặt chẽ hệ phân cấp kế thừa.
  - Dễ dàng hơn cho pattern matching, switch expression, kiểm tra tính đầy đủ (exhaustiveness).
- Chỉ những lớp được liệt kê trong `permits` mới được kế thừa/implement sealed class.

Cú pháp cơ bản:

```java
public sealed class Shape permits Circle, Rectangle, Square {
    // ...
}
```

---

## 2. Các loại lớp con được phép

Các lớp con của một `sealed` class **phải** là một trong ba loại:

1. `final` – không cho phép lớp khác kế thừa tiếp.
2. `sealed` – tiếp tục giới hạn thêm lớp con.
3. `non-sealed` – bỏ giới hạn, cho phép kế thừa tự do từ lớp đó.

Ví dụ:

```java
public sealed class Shape permits Circle, Rectangle, WeirdShape {}

public final class Circle extends Shape {
    double radius;
}

public sealed class Rectangle extends Shape permits FilledRectangle {
    double width;
    double height;
}

public final class FilledRectangle extends Rectangle {
    String color;
}

public non-sealed class WeirdShape extends Shape {
    // từ đây có thể có nhiều lớp con tự do
}
```

## 3. Ví dụ sử dụng với pattern matching (Java 17+)

### 3.1. Tính diện tích với sealed class

```java
public sealed class Shape permits Circle, Rectangle, Square {}

public final class Circle extends Shape {
    public final double radius;
    public Circle(double radius) { this.radius = radius; }
}

public final class Rectangle extends Shape {
    public final double width;
    public final double height;
    public Rectangle(double width, double height) {
        this.width = width;
        this.height = height;
    }
}

public final class Square extends Shape {
    public final double side;
    public Square(double side) { this.side = side; }
}
```

Hàm tính diện tích:

```java
public class ShapeUtils {
    public static double area(Shape shape) {
        if (shape instanceof Circle c) {
            return Math.PI * c.radius * c.radius;
        } else if (shape instanceof Rectangle r) {
            return r.width * r.height;
        } else if (shape instanceof Square s) {
            return s.side * s.side;
        } else {
            throw new IllegalStateException("Unknown shape: " + shape);
        }
    }
}
```

### 3.2. Input/Output minh họa

```java
public class Main {
    public static void main(String[] args) {
        Shape s1 = new Circle(2.0);
        Shape s2 = new Rectangle(3.0, 4.0);
        Shape s3 = new Square(5.0);

        System.out.println("Area s1 = " + ShapeUtils.area(s1));
        System.out.println("Area s2 = " + ShapeUtils.area(s2));
        System.out.println("Area s3 = " + ShapeUtils.area(s3));
    }
}
```

**Output (xấp xỉ):**

```text
Area s1 = 12.566370614359172
Area s2 = 12.0
Area s3 = 25.0
```

## 4. Lợi ích chính

- **Kiểm soát hệ phân cấp**: Chỉ cho phép một số lớp cụ thể kế thừa/implement.
- **An toàn hơn**: Dễ đảm bảo đã xử lý đầy đủ tất cả các trường hợp (exhaustive) trong `switch`/pattern matching.
- **Thiết kế API rõ ràng**: Người dùng thư viện biết chính xác những kiểu nào có thể xuất hiện.

---

Tóm lại, **sealed classes** giúp bạn thiết kế hệ thống kiểu (type system) chặt chẽ hơn, dễ bảo trì, và hỗ trợ tốt cho các tính năng mới như pattern matching trong Java 17+.
