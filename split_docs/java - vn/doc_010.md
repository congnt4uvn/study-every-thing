
Dưới đây là tóm tắt ngắn gọn về **Java Records** (tính năng mới trong Java):

---

## 1. Record là gì?

- `record` là một kiểu khai báo lớp đặc biệt, được giới thiệu từ **Java 14 (preview) / Java 16 (chính thức)**.  
- Dùng để biểu diễn **các “data carrier”** – lớp chỉ dùng để chứa dữ liệu, ít hành vi (giống DTO, Value Object).
- Java tự động sinh:
  - `private final` field cho từng thành phần
  - Constructor
  - `getter` (dưới dạng accessor)
  - `equals()`, `hashCode()`
  - `toString()`

---

## 2. Cú pháp cơ bản

```java
public record Point(int x, int y) {}
```

Tương đương (về ý nghĩa) với một lớp như:

```java
public final class Point {
    private final int x;
    private final int y;

    public Point(int x, int y) {
        this.x = x;
        this.y = y;
    }

    public int x() { return x; }
    public int y() { return y; }

    @Override
    public boolean equals(Object o) { /* auto-generated */ }

    @Override
    public int hashCode() { /* auto-generated */ }

    @Override
    public String toString() { /* auto-generated */ }
}
```

---

## 3. Cách dùng cơ bản (ví dụ + input/output)

### Ví dụ 1: Tạo và sử dụng record

```java
public record Person(String name, int age) {}

public class Main {
    public static void main(String[] args) {
        Person p = new Person("An", 25);

        System.out.println(p.name()); // accessor
        System.out.println(p.age());
        System.out.println(p);        // toString()
    }
}
```

**Output:**

```text
An
25
Person[name=An, age=25]
```

---

### Ví dụ 2: So sánh equals/hashCode

```java
public record Person(String name, int age) {}

public class Main {
    public static void main(String[] args) {
        Person p1 = new Person("An", 25);
        Person p2 = new Person("An", 25);
        Person p3 = new Person("Bình", 30);

        System.out.println(p1.equals(p2));
        System.out.println(p1.equals(p3));
    }
}
```

**Output:**

```text
true
false
```

## 4. Constructor trong record

Bạn có thể:

### 4.1. Compact constructor (kiểm tra dữ liệu)

```java
public record Person(String name, int age) {
    public Person {
        if (age < 0) {
            throw new IllegalArgumentException("Age must be >= 0");
        }
    }
}
```

### 4.2. Canonical constructor đầy đủ

```java
public record Person(String name, int age) {
    public Person(String name, int age) {
        if (name == null || name.isBlank()) {
            throw new IllegalArgumentException("Name is required");
        }
        this.name = name;
        this.age = age;
    }
}
```

---

## 5. Đặc điểm quan trọng

- **Bất biến (immutable)**: các field là `final`, không có setter.
- **Final**: record là `final`, không thể kế thừa từ record.
- Có thể:
  - Thêm **method** bình thường
  - Thêm **static field/method**
  - Implement **interface**
- Không thể:
  - Kế thừa class khác (nhưng có thể extends `java.lang.Record` ngầm định)
  - Khai báo field “bình thường” trùng với component (nhưng có thể thêm field phụ nếu cần, ít dùng).

## 6. Ví dụ thực tế: DTO cho API

```java
public record UserDto(long id, String username, String email) {}

public class Main {
    public static void main(String[] args) {
        UserDto user = new UserDto(1L, "user1", "user1@example.com");
        System.out.println(user);
    }
}
```

**Output:**

```text
UserDto[id=1, username=user1, email=user1@example.com]
```

---

## 7. Khi nào nên dùng record?

- Khi bạn cần:
  - Lớp chỉ để **chứa dữ liệu** (DTO, Value Object, response model…)
  - So sánh theo **giá trị** (value-based), không phải theo tham chiếu.
  - Code ngắn gọn, ít boilerplate (không muốn tự viết getter, equals, hashCode, toString).

---

**Tóm tắt:**  
`record` giúp định nghĩa các lớp dữ liệu bất biến một cách ngắn gọn, tự động sinh constructor, accessor, `equals`, `hashCode`, `toString`, rất phù hợp cho DTO, value object và các cấu trúc dữ liệu đơn giản trong Java hiện đại.
