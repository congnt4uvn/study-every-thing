## 1. Mục tiêu của lớp “thread-safe”

- **Thread-safe** nghĩa là: nhiều luồng (threads) cùng truy cập/ sửa dữ liệu **mà không gây lỗi** (race condition, data corruption).
- Hành vi của lớp phải **đúng và nhất quán** trong mọi trường hợp chạy song song.

---

## 2. Các chiến lược chính để thiết kế lớp thread-safe

### 2.1. Tránh chia sẻ trạng thái (Immutability)

- Lớp **bất biến (immutable)**: trạng thái không thay đổi sau khi tạo.
- Không cần đồng bộ (synchronization) vì không có thay đổi dữ liệu.

**Ví dụ: lớp bất biến**

```java
public final class Money {
    private final int amount;
    private final String currency;

    public Money(int amount, String currency) {
        this.amount = amount;
        this.currency = currency;
    }

    public int getAmount() {
        return amount;
    }

    public String getCurrency() {
        return currency;
    }
}
```

- Nhiều luồng đọc `Money` cùng lúc → **an toàn**.

---

### 2.2. Đồng bộ hóa truy cập (Synchronization)

- Dùng `synchronized`, `ReentrantLock`, hoặc các cấu trúc trong `java.util.concurrent`.
- Đảm bảo chỉ **một luồng** được sửa dữ liệu tại một thời điểm.

**Ví dụ: bộ đếm thread-safe với synchronized**

```java
public class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++; // chỉ 1 thread được vào mỗi lần
    }

    public synchronized int getCount() {
        return count;
    }
}
```

**Input/Output minh họa**

```java
public class TestCounter {
    public static void main(String[] args) throws InterruptedException {
        Counter counter = new Counter();

        Runnable task = () -> {
            for (int i = 0; i < 1000; i++) {
                counter.increment();
            }
        };

        Thread t1 = new Thread(task);
        Thread t2 = new Thread(task);
        t1.start();
        t2.start();
        t1.join();
        t2.join();

        System.out.println("Final count = " + counter.getCount());
    }
}
```

- **Kỳ vọng output**:  
  `Final count = 2000`  
  (Nếu không thread-safe, có thể < 2000).

### 2.3. Sử dụng lớp thread-safe có sẵn

- Dùng các lớp trong `java.util.concurrent`:
  - `ConcurrentHashMap`, `CopyOnWriteArrayList`, `BlockingQueue`, `AtomicInteger`, …
- Giảm việc tự viết đồng bộ phức tạp.

**Ví dụ: dùng AtomicInteger**

```java
import java.util.concurrent.atomic.AtomicInteger;

public class AtomicCounter {
    private final AtomicInteger count = new AtomicInteger(0);

    public void increment() {
        count.incrementAndGet();
    }

    public int getCount() {
        return count.get();
    }
}
```

### 2.4. Bảo vệ toàn bộ trạng thái (Encapsulation + Invariants)

- Tất cả dữ liệu dùng chung phải:
  - **private**
  - chỉ truy cập qua các phương thức đã được đồng bộ.
- Giữ **bất biến (invariant)**: các điều kiện luôn đúng sau mỗi thao tác.

**Ví dụ: tài khoản ngân hàng đơn giản**

```java
public class BankAccount {
    private int balance; // chỉ truy cập qua các method synchronized

    public BankAccount(int initialBalance) {
        this.balance = initialBalance;
    }

    public synchronized void deposit(int amount) {
        if (amount < 0) throw new IllegalArgumentException();
        balance += amount;
    }

    public synchronized void withdraw(int amount) {
        if (amount < 0 || amount > balance) throw new IllegalArgumentException();
        balance -= amount;
    }

    public synchronized int getBalance() {
        return balance;
    }
}
```

### 2.5. Tránh deadlock và liveness issues

- **Deadlock**: hai (hoặc nhiều) luồng chờ nhau mãi mãi.
- Quy tắc:
  - Khóa (lock) theo **thứ tự cố định**.
  - Hạn chế giữ lock quá lâu.
  - Tránh gọi code bên ngoài (callback) khi đang giữ lock.

---

### 2.6. Visibility & `volatile`

- Nhiều luồng có thể **không thấy** cập nhật mới nếu không có cơ chế nhớ (memory visibility).
- `volatile` đảm bảo:
  - đọc/ghi biến luôn thấy giá trị mới nhất.
  - nhưng **không** đảm bảo thao tác phức tạp là atomic (ví dụ `count++`).

**Ví dụ: cờ dừng với volatile**

```java
public class StopTask implements Runnable {
    private volatile boolean running = true;

    public void stop() {
        running = false;
    }

    @Override
    public void run() {
        while (running) {
            // làm việc
        }
        System.out.println("Stopped");
    }
}
```

## 3. Mẫu thiết kế (Patterns) thường dùng

- **Thread confinement**: dữ liệu chỉ dùng trong 1 thread → không cần đồng bộ.
- **Immutable objects**: như trên.
- **Monitor pattern**: dùng `synchronized` để bảo vệ toàn bộ trạng thái của đối tượng.
- **Producer–Consumer**: dùng `BlockingQueue` để chia sẻ công việc giữa threads.

**Ví dụ: Producer–Consumer với BlockingQueue**

```java
import java.util.concurrent.*;

public class ProducerConsumerExample {
    public static void main(String[] args) {
        BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(10);

        Runnable producer = () -> {
            try {
                for (int i = 1; i <= 5; i++) {
                    queue.put(i);
                    System.out.println("Produced: " + i);
                }
                queue.put(-1); // tín hiệu kết thúc
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        };

        Runnable consumer = () -> {
            try {
                while (true) {
                    int value = queue.take();
                    if (value == -1) break;
                    System.out.println("Consumed: " + value);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        };

        new Thread(producer).start();
        new Thread(consumer).start();
    }
}
```

**Output (ví dụ)**

```
Produced: 1
Consumed: 1
Produced: 2
Consumed: 2
...
Produced: 5
Consumed: 5
```

## 4. Ý chính cần nhớ

1. Cố gắng **thiết kế bất biến** hoặc **không chia sẻ trạng thái** nếu có thể.  
2. Nếu phải chia sẻ, dùng:
   - `synchronized`, `Lock`, hoặc
   - các lớp trong `java.util.concurrent`.
3. Bảo vệ **toàn bộ trạng thái** của đối tượng, giữ bất biến.  
4. Chú ý **visibility** (dùng `volatile` hoặc đồng bộ) và tránh **deadlock**.  
5. Ưu tiên dùng **thư viện có sẵn** thay vì tự viết cơ chế đồng bộ phức tạp.
