## 1. Khái niệm chính

### JPQL (Java Persistence Query Language)
- Ngôn ngữ truy vấn giống SQL nhưng làm việc với **entity** (class) thay vì bảng.
- Viết truy vấn dạng **string**.
- Dễ đọc, dễ viết cho các truy vấn đơn giản và trung bình.
- Khó bảo trì khi truy vấn phức tạp, dễ lỗi cú pháp (vì là string, không được kiểm tra compile-time).

### Criteria API
- Cung cấp cách xây dựng truy vấn **dạng hướng đối tượng** (type-safe).
- Dùng class, method, builder thay vì string.
- Được kiểm tra **type** ở compile-time, dễ refactor.
- Truy vấn phức tạp, động (dynamic query) sẽ dễ quản lý hơn, nhưng code dài và khó đọc hơn.

---

## 2. Khi nào dùng JPQL?

- Truy vấn **cố định**, không quá phức tạp.
- Cần code **ngắn gọn, dễ đọc**.
- Ví dụ: tìm user theo tên, lọc đơn giản, join không quá phức tạp.

### Ví dụ JPQL

Giả sử có entity:

```java
@Entity
public class User {
    @Id
    private Long id;

    private String name;
    private Integer age;
}
```

#### a) Truy vấn đơn giản

**Code:**

```java
String jpql = "SELECT u FROM User u WHERE u.name = :name";
TypedQuery<User> query = entityManager.createQuery(jpql, User.class);
query.setParameter("name", "Alice");
List<User> users = query.getResultList();
```

**Input:**
- name = "Alice"

**Output (giả sử DB có 2 bản ghi):**
- users = `[User{id=1, name="Alice", age=25}, User{id=5, name="Alice", age=30}]`

#### b) Truy vấn với điều kiện và sắp xếp

```java
String jpql = "SELECT u FROM User u WHERE u.age > :age ORDER BY u.name ASC";
TypedQuery<User> query = entityManager.createQuery(jpql, User.class);
query.setParameter("age", 20);
List<User> users = query.getResultList();
```

## 3. Khi nào dùng Criteria API?

- Cần **truy vấn động** (dynamic query), ví dụ:
  - Chỉ filter theo name nếu người dùng nhập name.
  - Chỉ filter theo age nếu người dùng nhập age.
- Cần **type-safe**, dễ refactor khi đổi tên field.
- Truy vấn phức tạp, nhiều điều kiện kết hợp.

### Ví dụ Criteria API

#### a) Truy vấn tương đương với JPQL: `SELECT u FROM User u WHERE u.name = :name`

```java
CriteriaBuilder cb = entityManager.getCriteriaBuilder();
CriteriaQuery<User> cq = cb.createQuery(User.class);
Root<User> user = cq.from(User.class);

cq.select(user)
  .where(cb.equal(user.get("name"), "Alice"));

TypedQuery<User> query = entityManager.createQuery(cq);
List<User> users = query.getResultList();
```

**Input:**
- name = "Alice"

**Output:**
- Tương tự ví dụ JPQL ở trên.

#### b) Truy vấn động: filter theo name và age nếu có

```java
public List<User> searchUsers(String name, Integer minAge) {
    CriteriaBuilder cb = entityManager.getCriteriaBuilder();
    CriteriaQuery<User> cq = cb.createQuery(User.class);
    Root<User> user = cq.from(User.class);

    List<Predicate> predicates = new ArrayList<>();

    if (name != null && !name.isEmpty()) {
        predicates.add(cb.equal(user.get("name"), name));
    }

    if (minAge != null) {
        predicates.add(cb.greaterThanOrEqualTo(user.get("age"), minAge));
    }

    cq.select(user).where(predicates.toArray(new Predicate[0]));

    return entityManager.createQuery(cq).getResultList();
}
```

**Input ví dụ:**
- name = "Alice"
- minAge = 25

**Output:**
- Danh sách User có name = "Alice" và age ≥ 25.

## 4. So sánh nhanh

| Tiêu chí                 | JPQL                                  | Criteria API                                  |
|--------------------------|----------------------------------------|-----------------------------------------------|
| Cú pháp                  | String                                | Hướng đối tượng (API)                        |
| Độ dễ đọc                | Dễ, giống SQL                         | Khó hơn, code dài                             |
| Kiểm tra type            | Không (runtime)                       | Có (compile-time)                             |
| Truy vấn động            | Khó, phải tự build string             | Rất phù hợp                                   |
| Truy vấn đơn giản        | Rất phù hợp                           | Dùng được nhưng hơi “nặng”                    |
| Bảo trì, refactor        | Dễ lỗi khi đổi tên field              | An toàn hơn, IDE hỗ trợ tốt hơn              |

---

## 5. Kết luận

- **Dùng JPQL** cho:
  - Truy vấn **cố định**, đơn giản, trung bình.
  - Muốn code ngắn, dễ đọc, dễ hiểu.

- **Dùng Criteria API** cho:
  - Truy vấn **động**, nhiều điều kiện tùy chọn.
  - Cần **type-safe**, dễ refactor, truy vấn phức tạp.
