
Dưới đây là tóm tắt ngắn gọn về **kiến trúc phân lớp (Layered Architecture)** trong Java, với ví dụ code và input/output.

---

## 1. Kiến trúc phân lớp là gì?

Kiến trúc phân lớp chia ứng dụng thành các tầng (layer) rõ ràng, mỗi tầng có trách nhiệm riêng:

1. **Presentation Layer (Tầng trình bày / UI)**  
   - Nhận request từ người dùng (web, REST API, GUI, …)  
   - Gửi response lại cho người dùng  
   - Không chứa logic nghiệp vụ phức tạp

2. **Service Layer (Tầng dịch vụ / Business Logic)**  
   - Chứa logic nghiệp vụ chính  
   - Xử lý dữ liệu, kiểm tra điều kiện, gọi đến repository  
   - Là “cầu nối” giữa Presentation và Repository

3. **Repository Layer (Tầng truy cập dữ liệu / Data Access)**  
   - Làm việc với database (CRUD)  
   - Ẩn chi tiết truy vấn (SQL, JPA, JDBC, …)  
   - Trả về entity hoặc DTO cho Service

## 2. Ví dụ đơn giản với Java (Spring Boot)

Giả sử ta có chức năng quản lý User (người dùng).

### 2.1. Entity (User)

```java
// User.java
public class User {
    private Long id;
    private String name;
    private String email;

    // constructors, getters, setters
}
```

---

### 2.2. Repository Layer

```java
// UserRepository.java
public interface UserRepository {
    User findById(Long id);
    void save(User user);
}
```

Ví dụ triển khai giả lập (in-memory):

```java
// InMemoryUserRepository.java
import java.util.HashMap;
import java.util.Map;

public class InMemoryUserRepository implements UserRepository {
    private final Map<Long, User> storage = new HashMap<>();

    @Override
    public User findById(Long id) {
        return storage.get(id);
    }

    @Override
    public void save(User user) {
        storage.put(user.getId(), user);
    }
}
```

---

### 2.3. Service Layer

```java
// UserService.java
public class UserService {
    private final UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    // Logic nghiệp vụ: không cho phép email trùng
    public void registerUser(User user) {
        // ví dụ đơn giản: kiểm tra email đã tồn tại hay chưa
        // (ở đây bỏ qua phần tìm theo email, chỉ minh họa)
        if (user.getEmail() == null || user.getEmail().isEmpty()) {
            throw new IllegalArgumentException("Email không được để trống");
        }
        userRepository.save(user);
    }

    public User getUser(Long id) {
        User user = userRepository.findById(id);
        if (user == null) {
            throw new IllegalArgumentException("Không tìm thấy user với id = " + id);
        }
        return user;
    }
}
```

---

### 2.4. Presentation Layer (ví dụ REST Controller)

```java
// UserController.java (giả lập, không dùng annotation để đơn giản)
public class UserController {
    private final UserService userService;

    public UserController(UserService userService) {
        this.userService = userService;
    }

    // Giả lập API: POST /users
    public String createUser(Long id, String name, String email) {
        User user = new User();
        user.setId(id);
        user.setName(name);
        user.setEmail(email);

        userService.registerUser(user);
        return "User created with id = " + id;
    }

    // Giả lập API: GET /users/{id}
    public String getUser(Long id) {
        User user = userService.getUser(id);
        return "User: id=" + user.getId() + ", name=" + user.getName() + ", email=" + user.getEmail();
    }
}
```

## 3. Ví dụ Input/Output (giả lập)

### 3.1. Tạo user

**Input (gọi hàm controller):**

```java
UserRepository repo = new InMemoryUserRepository();
UserService service = new UserService(repo);
UserController controller = new UserController(service);

String resultCreate = controller.createUser(1L, "Nguyen Van A", "a@example.com");
System.out.println(resultCreate);
```

**Output:**

```text
User created with id = 1
```

---

### 3.2. Lấy thông tin user

**Input:**

```java
String resultGet = controller.getUser(1L);
System.out.println(resultGet);
```

**Output:**

```text
User: id=1, name=Nguyen Van A, email=a@example.com
```

---

## 4. Ý chính cần nhớ

- **Layered Architecture** chia ứng dụng thành các tầng:  
  - Presentation: giao tiếp với người dùng / client  
  - Service: xử lý logic nghiệp vụ  
  - Repository: truy cập và lưu trữ dữ liệu  
- Mỗi tầng chỉ nên gọi tầng “bên dưới” nó (Presentation → Service → Repository).  
- Giúp code dễ bảo trì, dễ test, dễ mở rộng.
