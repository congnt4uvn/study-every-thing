
Dưới đây là tóm tắt ngắn gọn về **CompletableFuture** trong Java, tập trung vào: **pipeline bất đồng bộ, composition, và xử lý lỗi**, kèm ví dụ code + input/output.

---

## 1. CompletableFuture là gì?

- Là API trong Java để xử lý **bất đồng bộ** (async) và **lập trình phản ứng** (reactive style).
- Cho phép:
  - Chạy task ở background.
  - Ghép (compose) nhiều bước xử lý.
  - Xử lý lỗi một cách rõ ràng.

---

## 2. Async Pipelines (Chuỗi xử lý bất đồng bộ)

Dùng các method như `supplyAsync`, `thenApply`, `thenAccept`, `thenRun` để tạo pipeline.

### Ví dụ: Pipeline đơn giản

```java
import java.util.concurrent.CompletableFuture;

public class AsyncPipelineExample {
    public static void main(String[] args) {
        CompletableFuture<Void> pipeline = CompletableFuture
                .supplyAsync(() -> {
                    // Input giả định: không có, chỉ là bắt đầu pipeline
                    System.out.println("Bước 1: Lấy dữ liệu...");
                    sleep(500);
                    return 10; // Output bước 1
                })
                .thenApply(data -> {
                    // Input: 10
                    System.out.println("Bước 2: Xử lý dữ liệu: " + data);
                    return data * 2; // Output: 20
                })
                .thenAccept(result -> {
                    // Input: 20
                    System.out.println("Bước 3: In kết quả: " + result);
                });

        pipeline.join(); // Chờ pipeline hoàn thành
    }

    private static void sleep(long ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) { }
    }
}
```

**Dòng chảy Input/Output:**

- Bước 1: không input → output: `10`
- Bước 2: input: `10` → output: `20`
- Bước 3: input: `20` → output: `void` (chỉ in ra)

**Output console (ví dụ):**
```
Bước 1: Lấy dữ liệu...
Bước 2: Xử lý dữ liệu: 10
Bước 3: In kết quả: 20
```

## 3. Composition (Ghép nhiều CompletableFuture)

Có hai kiểu chính:

1. **thenCompose**: dùng khi bước sau **phụ thuộc** vào kết quả bước trước (chuỗi).
2. **thenCombine / allOf / anyOf**: dùng khi chạy **song song** rồi ghép kết quả.

### 3.1. thenCompose – Chuỗi phụ thuộc

```java
import java.util.concurrent.CompletableFuture;

public class ThenComposeExample {
    public static void main(String[] args) {
        CompletableFuture<String> result = getUserId("alice")      // Input: "alice"
                .thenCompose(userId -> getUserProfile(userId));    // Input: userId

        System.out.println("Kết quả cuối: " + result.join());
    }

    static CompletableFuture<String> getUserId(String username) {
        return CompletableFuture.supplyAsync(() -> {
            System.out.println("Lấy userId cho: " + username);
            return "user-123"; // giả lập
        });
    }

    static CompletableFuture<String> getUserProfile(String userId) {
        return CompletableFuture.supplyAsync(() -> {
            System.out.println("Lấy profile cho: " + userId);
            return "Profile của " + userId;
        });
    }
}
```

**Input:** `"alice"`  
**Output console (ví dụ):**
```
Lấy userId cho: alice
Lấy profile cho: user-123
Kết quả cuối: Profile của user-123
```

---

### 3.2. thenCombine – Chạy song song rồi ghép

```java
import java.util.concurrent.CompletableFuture;

public class ThenCombineExample {
    public static void main(String[] args) {
        CompletableFuture<Integer> priceFuture = getPrice();
        CompletableFuture<Integer> discountFuture = getDiscount();

        CompletableFuture<Integer> finalPrice = priceFuture
                .thenCombine(discountFuture, (price, discount) -> {
                    // Input: price, discount
                    System.out.println("Giá gốc: " + price + ", giảm: " + discount);
                    return price - discount; // Output: finalPrice
                });

        System.out.println("Giá cuối cùng: " + finalPrice.join());
    }

    static CompletableFuture<Integer> getPrice() {
        return CompletableFuture.supplyAsync(() -> 100);
    }

    static CompletableFuture<Integer> getDiscount() {
        return CompletableFuture.supplyAsync(() -> 20);
    }
}
```

**Input:** không có (giả lập)  
**Output console:**
```
Giá gốc: 100, giảm: 20
Giá cuối cùng: 80
```

### 3.3. allOf / anyOf – Nhiều task song song

```java
import java.util.List;
import java.util.concurrent.CompletableFuture;
import java.util.stream.Collectors;

public class AllOfExample {
    public static void main(String[] args) {
        List<CompletableFuture<Integer>> futures = List.of(1, 2, 3, 4, 5).stream()
                .map(i -> CompletableFuture.supplyAsync(() -> i * 10))
                .collect(Collectors.toList());

        CompletableFuture<Void> all = CompletableFuture.allOf(
                futures.toArray(new CompletableFuture[0])
        );

        CompletableFuture<List<Integer>> result = all.thenApply(v ->
                futures.stream()
                        .map(CompletableFuture::join)
                        .collect(Collectors.toList())
        );

        System.out.println("Kết quả: " + result.join());
    }
}
```

**Input:** `[1, 2, 3, 4, 5]`  
**Output:** `Kết quả: [10, 20, 30, 40, 50]`

## 4. Error Handling (Xử lý lỗi)

Các method chính:

- `exceptionally` – bắt lỗi và trả về giá trị fallback.
- `handle` – luôn chạy, nhận cả kết quả và exception.
- `whenComplete` – chỉ để side-effect, không thay đổi kết quả.

### 4.1. exceptionally – Giá trị fallback

```java
import java.util.concurrent.CompletableFuture;

public class ExceptionallyExample {
    public static void main(String[] args) {
        CompletableFuture<Integer> future = CompletableFuture
                .supplyAsync(() -> {
                    System.out.println("Tính toán...");
                    if (true) throw new RuntimeException("Lỗi tính toán");
                    return 42;
                })
                .exceptionally(ex -> {
                    System.out.println("Gặp lỗi: " + ex.getMessage());
                    return -1; // fallback
                });

        System.out.println("Kết quả: " + future.join());
    }
}
```

**Output console:**
```
Tính toán...
Gặp lỗi: Lỗi tính toán
Kết quả: -1
```

---

### 4.2. handle – Xử lý cả thành công và thất bại

```java
import java.util.concurrent.CompletableFuture;

public class HandleExample {
    public static void main(String[] args) {
        CompletableFuture<String> future = CompletableFuture
                .supplyAsync(() -> {
                    if (true) throw new RuntimeException("Lỗi IO");
                    return "OK";
                })
                .handle((result, ex) -> {
                    if (ex != null) {
                        System.out.println("Có lỗi: " + ex.getMessage());
                        return "DEFAULT"; // giá trị thay thế
                    }
                    return result;
                });

        System.out.println("Kết quả: " + future.join());
    }
}
```

**Output console:**
```
Có lỗi: Lỗi IO
Kết quả: DEFAULT
```

## 5. Tóm tắt ý chính

- **CompletableFuture** hỗ trợ:
  - Tạo **pipeline bất đồng bộ** với `supplyAsync`, `thenApply`, `thenAccept`, `thenRun`.
  - **Composition**:
    - `thenCompose` cho chuỗi phụ thuộc.
    - `thenCombine`, `allOf`, `anyOf` cho chạy song song và ghép kết quả.
  - **Xử lý lỗi**:
    - `exceptionally` để trả fallback khi lỗi.
    - `handle` để xử lý cả kết quả và exception.
    - `whenComplete` để log/side-effect mà không đổi kết quả.

Nếu bạn muốn, tôi có thể viết một ví dụ hoàn chỉnh hơn (end-to-end) mô phỏng một service gọi API, xử lý dữ liệu, và xử lý lỗi bằng CompletableFuture.
