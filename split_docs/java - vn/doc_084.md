## 1. Fetch Strategies (Chiến lược tải dữ liệu)

### 1.1. `EAGER` vs `LAZY`

- `EAGER`: Quan hệ được tải **ngay lập tức** cùng với entity chính.
- `LAZY`: Quan hệ được tải **khi cần dùng** (khi truy cập getter).

```java
@Entity
public class Author {
    @Id
    private Long id;

    private String name;

    // Mặc định là LAZY với @OneToMany
    @OneToMany(mappedBy = "author", fetch = FetchType.LAZY)
    private List<Book> books;
}

@Entity
public class Book {
    @Id
    private Long id;

    private String title;

    @ManyToOne(fetch = FetchType.EAGER) // Mặc định @ManyToOne là EAGER
    private Author author;
}
```

**Ý chính:**
- Nên ưu tiên `LAZY` cho hầu hết các quan hệ để tránh tải dữ liệu thừa.
- `EAGER` dễ gây ra vấn đề **N+1 query** và hiệu năng kém.

## 2. N+1 Query Problem (Vấn đề N+1 truy vấn)

Khi bạn lấy danh sách entity cha, và mỗi lần truy cập quan hệ con lại sinh thêm 1 query.

```java
// Service
List<Author> authors = em.createQuery("SELECT a FROM Author a", Author.class)
                         .getResultList();

for (Author a : authors) {
    // Mỗi lần gọi a.getBooks() có thể sinh thêm 1 query (nếu LAZY và không join fetch)
    System.out.println(a.getName() + " has " + a.getBooks().size() + " books");
}
```

**Nếu có 1 query lấy 10 Author + 10 query lấy Book cho từng Author → 11 query (N+1).**

---

## 3. Giải pháp: `JOIN FETCH`

Dùng `JOIN FETCH` để tải luôn quan hệ cần thiết trong 1 query.

```java
// JPQL
List<Author> authors = em.createQuery(
    "SELECT DISTINCT a FROM Author a " +
    "LEFT JOIN FETCH a.books", Author.class)
    .getResultList();
```

**Kết quả:**
- Chỉ 1 query SQL với `JOIN` để lấy cả Author và Book.
- Tránh N+1 query.

**Ví dụ SQL sinh ra (minh họa):**
```sql
SELECT a.id, a.name, b.id, b.title, b.author_id
FROM author a
LEFT JOIN book b ON b.author_id = a.id;
```

## 4. Batch Fetching (Tải theo lô)

Batch fetching giúp Hibernate **gom nhiều ID lại** để load trong 1 query, giảm số lượng query khi dùng `LAZY`.

### 4.1. Cấu hình bằng annotation

```java
@Entity
@BatchSize(size = 10)
public class Author {
    @Id
    private Long id;

    private String name;

    @OneToMany(mappedBy = "author", fetch = FetchType.LAZY)
    @BatchSize(size = 5)
    private List<Book> books;
}
```

Hoặc trong `application.properties`:

```properties
spring.jpa.properties.hibernate.default_batch_fetch_size=10
```

### 4.2. Cách hoạt động

Giả sử bạn load 20 `Author`, sau đó truy cập `getBooks()` của từng Author:

- Không có batch: 20 query `SELECT * FROM book WHERE author_id = ?`
- Có batch (size = 5): Hibernate gom 5 ID mỗi lần:

```sql
SELECT * FROM book WHERE author_id IN (?, ?, ?, ?, ?);
```

→ 4 query thay vì 20.

---

## 5. So sánh: JOIN FETCH vs Batch Fetch

- `JOIN FETCH`:
  - Tốt khi bạn **biết chắc** cần quan hệ đó ngay.
  - Có thể gây **bùng nổ số dòng** (cartesian product) nếu join nhiều collection.
- `Batch Fetch`:
  - Tốt khi bạn **truy cập dần dần** các quan hệ LAZY.
  - Giảm số query mà không làm phình to 1 query duy nhất.

Thường dùng:
- `LAZY` + `JOIN FETCH` cho các use case cụ thể.
- `BatchSize`/`default_batch_fetch_size` để tối ưu truy cập LAZY nói chung.

## 6. Ví dụ tổng hợp

### Entity

```java
@Entity
public class Author {
    @Id
    private Long id;

    private String name;

    @OneToMany(mappedBy = "author", fetch = FetchType.LAZY)
    @BatchSize(size = 5)
    private List<Book> books;
}

@Entity
public class Book {
    @Id
    private Long id;

    private String title;

    @ManyToOne(fetch = FetchType.LAZY)
    private Author author;
}
```

### Use case 1: Dùng `JOIN FETCH`

```java
// Lấy Author và Book trong 1 query
List<Author> authors = em.createQuery(
    "SELECT DISTINCT a FROM Author a " +
    "LEFT JOIN FETCH a.books", Author.class)
    .getResultList();

for (Author a : authors) {
    System.out.println(a.getName() + " -> " + a.getBooks().size() + " books");
}
```

**Output (console):**
```text
Author A -> 3 books
Author B -> 2 books
Author C -> 0 books
```

### Use case 2: Dùng Batch Fetch (không join fetch)

```java
// Chỉ lấy Author
List<Author> authors = em.createQuery(
    "SELECT a FROM Author a", Author.class)
    .getResultList();

// Khi cần mới truy cập books (LAZY + batch)
for (Author a : authors) {
    System.out.println(a.getName() + " -> " + a.getBooks().size() + " books");
}
```

**Với batch size = 5:**
- Hibernate sẽ sinh query kiểu:
  ```sql
  SELECT * FROM book WHERE author_id IN (?, ?, ?, ?, ?);
  ```
- Lặp lại cho các nhóm 5 Author.

---

## 7. Ý chính cần nhớ

1. Dùng `LAZY` cho hầu hết quan hệ, tránh `EAGER` tràn lan.
2. Dùng `JOIN FETCH` trong JPQL khi cần load quan hệ cụ thể trong 1 query.
3. Bật `BatchSize` hoặc `default_batch_fetch_size` để giảm N+1 query cho các quan hệ LAZY.
4. Tránh join fetch quá nhiều collection trong 1 query (dễ gây bùng nổ dữ liệu).
