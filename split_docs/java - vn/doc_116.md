
Dưới đây là tóm tắt ngắn gọn về “Managed DBs (RDS, Cloud SQL)” trong bối cảnh Cloud & DevOps, kèm ví dụ code Java.

---

## 1. Managed Database là gì?

- **Managed DB**: Dịch vụ cơ sở dữ liệu được nhà cung cấp cloud quản lý (AWS RDS, GCP Cloud SQL, Azure Database…).
- Nhà cung cấp chịu trách nhiệm:
  - Cài đặt, cấu hình DB
  - Backup, restore
  - Tự động cập nhật bản vá (patching)
  - High availability, failover
  - Monitoring, scaling
- Dev/DevOps chủ yếu tập trung vào:
  - Thiết kế schema
  - Tối ưu query
  - Kết nối ứng dụng với DB
  - Quản lý tài khoản & quyền truy cập

---

## 2. AWS RDS & GCP Cloud SQL – Ý chính

### AWS RDS
- Hỗ trợ nhiều engine: MySQL, PostgreSQL, MariaDB, SQL Server, Oracle, Aurora…
- Tính năng:
  - Multi-AZ (HA)
  - Read replicas
  - Tự động backup, snapshot
  - Tự động scale storage

### GCP Cloud SQL
- Hỗ trợ: MySQL, PostgreSQL, SQL Server.
- Tính năng:
  - High availability
  - Automated backups
  - Point-in-time recovery
  - Kết nối qua Cloud SQL Proxy hoặc private IP

---

## 3. Lợi ích chính trong DevOps

- **Giảm công việc vận hành**: Không phải tự cài DB, thiết lập replication, backup.
- **Tự động hóa**: Dễ tích hợp vào CI/CD (Terraform, CloudFormation, gcloud…).
- **Khả năng mở rộng**: Scale up/down dễ dàng.
- **Độ tin cậy cao**: HA, backup, monitoring tích hợp sẵn.

## 4. Kết nối Java với Managed DB (ví dụ MySQL trên RDS/Cloud SQL)

Giả sử bạn có một DB MySQL trên RDS hoặc Cloud SQL với thông tin:

- Host: `mydb.example.com`
- Port: `3306`
- Database: `myapp`
- User: `app_user`
- Password: `secret123`

### 4.1. Maven dependency (MySQL)

```xml
<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
    <version>8.0.33</version>
</dependency>
```

### 4.2. Kết nối đơn giản bằng JDBC

```java
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.ResultSet;

public class SimpleJdbcExample {
    public static void main(String[] args) throws Exception {
        String url = "jdbc:mysql://mydb.example.com:3306/myapp";
        String user = "app_user";
        String password = "secret123";

        try (Connection conn = DriverManager.getConnection(url, user, password)) {
            String sql = "SELECT id, name FROM users WHERE id = ?";
            try (PreparedStatement ps = conn.prepareStatement(sql)) {
                ps.setInt(1, 1); // input: id = 1
                try (ResultSet rs = ps.executeQuery()) {
                    while (rs.next()) {
                        int id = rs.getInt("id");
                        String name = rs.getString("name");
                        System.out.println("User: " + id + " - " + name);
                    }
                }
            }
        }
    }
}
```

**Input**: `id = 1`  
**Output (ví dụ)**:
```text
User: 1 - Nguyen Van A
```

## 5. Ví dụ với Spring Boot (application.properties)

```properties
spring.datasource.url=jdbc:mysql://mydb.example.com:3306/myapp
spring.datasource.username=app_user
spring.datasource.password=secret123
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
```

Repository:

```java
import org.springframework.data.jpa.repository.JpaRepository;

public interface UserRepository extends JpaRepository<User, Long> {
}
```

Controller:

```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;
import java.util.List;

@RestController
public class UserController {
    private final UserRepository userRepository;

    public UserController(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    @GetMapping("/users")
    public List<User> getUsers() {
        return userRepository.findAll();
    }
}
```

**Request (input)**:
```http
GET /users
```

**Response (output – ví dụ)**:
```json
[
  { "id": 1, "name": "Nguyen Van A" },
  { "id": 2, "name": "Tran Thi B" }
]
```

## 6. Điểm cần lưu ý trong DevOps

- Quản lý **secrets** (username/password) bằng:
  - AWS Secrets Manager, SSM Parameter Store
  - GCP Secret Manager
- Dùng **VPC / private IP** để kết nối an toàn.
- Thiết lập **monitoring & alerting** (CPU, connections, slow queries).
- Tự động hóa tạo DB bằng:
  - Terraform, CloudFormation, Pulumi, v.v.
