
Dưới đây là tóm tắt ngắn gọn về cấu hình trong Spring Boot với `@ConfigurationProperties` và `@Value`, kèm ví dụ code và input/output.

---

## 1. `@ConfigurationProperties`

**Mục đích:**  
Dùng để map (ánh xạ) nhiều cấu hình trong `application.properties` hoặc `application.yml` vào một class Java (POJO). Phù hợp khi có nhiều thuộc tính liên quan.

### Ví dụ cấu hình (input)

**application.properties:**
```properties
app.name=My Spring App
app.version=1.0.0
app.security.enabled=true
app.security.roles[0]=ADMIN
app.security.roles[1]=USER
```

### Class cấu hình

```java
import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.stereotype.Component;

import java.util.List;

@Component
@ConfigurationProperties(prefix = "app")
public class AppProperties {

    private String name;
    private String version;
    private Security security;

    public static class Security {
        private boolean enabled;
        private List<String> roles;

        // getters & setters
        public boolean isEnabled() { return enabled; }
        public void setEnabled(boolean enabled) { this.enabled = enabled; }

        public List<String> getRoles() { return roles; }
        public void setRoles(List<String> roles) { this.roles = roles; }
    }

    // getters & setters
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public String getVersion() { return version; }
    public void setVersion(String version) { this.version = version; }

    public Security getSecurity() { return security; }
    public void setSecurity(Security security) { this.security = security; }
}
```

### Sử dụng trong service/controller

```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class AppController {

    private final AppProperties appProperties;

    public AppController(AppProperties appProperties) {
        this.appProperties = appProperties;
    }

    @GetMapping("/info")
    public String info() {
        return "Name: " + appProperties.getName()
                + ", Version: " + appProperties.getVersion()
                + ", Security enabled: " + appProperties.getSecurity().isEnabled()
                + ", Roles: " + appProperties.getSecurity().getRoles();
    }
}
```

### Output (khi gọi `GET /info`)

```text
Name: My Spring App, Version: 1.0.0, Security enabled: true, Roles: [ADMIN, USER]
```

**Ưu điểm:**

- Quản lý nhiều thuộc tính liên quan trong một class.
- Hỗ trợ type-safe (kiểu dữ liệu rõ ràng: boolean, int, List, object lồng nhau…).
- Dễ test, dễ tái sử dụng.

## 2. `@Value`

**Mục đích:**  
Dùng để inject (tiêm) giá trị cấu hình đơn lẻ vào field, constructor, hoặc method. Phù hợp cho các giá trị lẻ, đơn giản.

### Ví dụ cấu hình (input)

**application.properties:**
```properties
app.title=Demo App
app.timeout=5000
app.default-role=USER
```

### Sử dụng `@Value`

```java
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

@Component
public class ValueExample {

    @Value("${app.title}")
    private String title;

    @Value("${app.timeout}")
    private int timeout;

    // Giá trị mặc định nếu không tìm thấy key
    @Value("${app.unknown:defaultValue}")
    private String unknown;

    public String info() {
        return "Title: " + title
                + ", Timeout: " + timeout
                + ", Unknown: " + unknown;
    }
}
```

### Controller dùng `ValueExample`

```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class ValueController {

    private final ValueExample valueExample;

    public ValueController(ValueExample valueExample) {
        this.valueExample = valueExample;
    }

    @GetMapping("/value-info")
    public String valueInfo() {
        return valueExample.info();
    }
}
```

### Output (khi gọi `GET /value-info`)

```text
Title: Demo App, Timeout: 5000, Unknown: defaultValue
```

**Ưu điểm:**

- Nhanh, đơn giản cho vài thuộc tính.
- Có thể dùng biểu thức SpEL (Spring Expression Language) nếu cần.

---

## 3. So sánh nhanh

- `@ConfigurationProperties`
  - Dùng cho: nhóm nhiều thuộc tính liên quan.
  - Ưu: type-safe, dễ tổ chức, dễ test.
  - Thường dùng với: `@Component` hoặc `@EnableConfigurationProperties`.

- `@Value`
  - Dùng cho: thuộc tính đơn lẻ, nhanh gọn.
  - Ưu: đơn giản, dễ dùng trực tiếp trong field/constructor.

---

Tóm lại:  
- Nhiều cấu hình có cấu trúc → dùng `@ConfigurationProperties`.  
- Một vài giá trị đơn lẻ, nhanh gọn → dùng `@Value`.
