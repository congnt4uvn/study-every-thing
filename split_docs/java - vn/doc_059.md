## 1. ApplicationContext

**Ý chính:**
- Là “trái tim” của Spring, container quản lý bean.
- Chịu trách nhiệm:
  - Tạo, cấu hình, quản lý vòng đời bean.
  - Dependency Injection (DI).
  - Tải cấu hình từ XML, Java config, annotation.
  - Cung cấp các dịch vụ như: event, i18n, resource loading.

**Ví dụ đơn giản với Java Config**

```java
// 1. Định nghĩa bean
public class HelloService {
    public String sayHello(String name) {
        return "Hello, " + name;
    }
}

// 2. Cấu hình Spring bằng Java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class AppConfig {

    @Bean
    public HelloService helloService() {
        return new HelloService();
    }
}

// 3. Sử dụng ApplicationContext
import org.springframework.context.ApplicationContext;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class MainApp {
    public static void main(String[] args) {
        ApplicationContext context =
                new AnnotationConfigApplicationContext(AppConfig.class);

        HelloService helloService = context.getBean(HelloService.class);
        String result = helloService.sayHello("Spring");

        System.out.println(result);
    }
}
```

**Input/Output:**

- Input: gọi `helloService.sayHello("Spring")`
- Output console:
```text
Hello, Spring
```

## 2. Profiles

**Ý chính:**
- Dùng để tách cấu hình theo môi trường: `dev`, `test`, `prod`,…
- Cho phép bật/tắt các bean tùy theo profile đang active.
- Kích hoạt profile bằng:
  - `spring.profiles.active` (property, env var, JVM arg)
  - Hoặc trong code.

**Ví dụ: cấu hình bean theo profile**

```java
public interface DataSourceConfig {
    String getUrl();
}

import org.springframework.context.annotation.Profile;
import org.springframework.stereotype.Component;

@Component
@Profile("dev")
class DevDataSourceConfig implements DataSourceConfig {
    @Override
    public String getUrl() {
        return "jdbc:h2:mem:devdb";
    }
}

@Component
@Profile("prod")
class ProdDataSourceConfig implements DataSourceConfig {
    @Override
    public String getUrl() {
        return "jdbc:mysql://prod-db:3306/app";
    }
}
```

**Kích hoạt profile (ví dụ chạy bằng JVM arg):**

```bash
# Chạy với profile dev
java -Dspring.profiles.active=dev -jar app.jar

# Chạy với profile prod
java -Dspring.profiles.active=prod -jar app.jar
```

**Sử dụng bean:**

```java
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

@Component
public class AppRunner {

    @Autowired
    private DataSourceConfig dataSourceConfig;

    public void run() {
        System.out.println("DB URL: " + dataSourceConfig.getUrl());
    }
}
```

**Output:**

- Khi `spring.profiles.active=dev`:
  ```text
  DB URL: jdbc:h2:mem:devdb
  ```
- Khi `spring.profiles.active=prod`:
  ```text
  DB URL: jdbc:mysql://prod-db:3306/app
  ```

## 3. Property Sources

**Ý chính:**
- Dùng để quản lý cấu hình (key–value) từ nhiều nguồn:
  - `application.properties` / `application.yml`
  - Biến môi trường, tham số JVM, file ngoài, v.v.
- Truy cập bằng:
  - `@Value("${property.name}")`
  - `Environment`
  - `@ConfigurationProperties`

**Ví dụ với `application.properties`**

`src/main/resources/application.properties`:

```properties
app.name=My Spring App
app.version=1.0.0
server.port=8080
```

**Sử dụng `@Value` để inject property:**

```java
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

@Component
public class AppInfo {

    @Value("${app.name}")
    private String appName;

    @Value("${app.version}")
    private String appVersion;

    public String info() {
        return appName + " - version " + appVersion;
    }
}
```

**Sử dụng `Environment`:**

```java
import org.springframework.core.env.Environment;
import org.springframework.stereotype.Component;

@Component
public class EnvReader {

    private final Environment env;

    public EnvReader(Environment env) {
        this.env = env;
    }

    public String getPort() {
        return env.getProperty("server.port");
    }
}
```

**Output ví dụ:**

```java
AppInfo appInfo = context.getBean(AppInfo.class);
System.out.println(appInfo.info());

EnvReader envReader = context.getBean(EnvReader.class);
System.out.println("Server port: " + envReader.getPort());
```

Kết quả:

```text
My Spring App - version 1.0.0
Server port: 8080
```

---

## Tóm tắt ngắn

- **ApplicationContext**: container Spring quản lý bean, DI, lifecycle, event, resource.
- **Profiles**: cho phép cấu hình khác nhau theo môi trường (`dev`, `test`, `prod`), quyết định bean nào được tạo.
- **Property Sources**: quản lý cấu hình dạng key–value từ nhiều nguồn; inject vào bean bằng `@Value`, `Environment`, `@ConfigurationProperties`.
