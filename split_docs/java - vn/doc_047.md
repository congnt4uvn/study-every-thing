
Dưới đây là tóm tắt ngắn gọn về **Memory Model trong Java: Heap vs Stack**, kèm ví dụ mã và input/output.

---

## 1. Stack (Ngăn xếp)

- Lưu trữ:
  - Biến cục bộ (local variables)
  - Tham chiếu đến đối tượng (không phải bản thân đối tượng)
  - Thông tin lời gọi hàm (call stack frame)
- Đặc điểm:
  - Mỗi thread có **stack riêng**
  - Tốc độ rất nhanh
  - Vùng nhớ được cấp phát và thu hồi **tự động** khi hàm kết thúc
  - Không dùng cho dữ liệu lớn sống lâu

### Ví dụ Stack

```java
public class StackExample {
    public static void main(String[] args) {
        int x = 10;          // x nằm trên stack
        int y = 20;          // y nằm trên stack
        int result = add(x, y); // x, y, result đều là biến trên stack
        System.out.println(result);
    }

    static int add(int a, int b) { // a, b trên stack của hàm add
        int sum = a + b;           // sum trên stack
        return sum;
    }
}
```

**Input:** (chương trình không nhận input từ người dùng)  
**Output:**
```text
30
```

## 2. Heap (Đống)

- Lưu trữ:
  - Các **đối tượng** (object)
  - Các **instance variables** bên trong đối tượng
  - Mảng (array)
- Đặc điểm:
  - Dùng chung giữa các thread (cần đồng bộ khi truy cập)
  - Được quản lý bởi **Garbage Collector**
  - Dùng cho dữ liệu sống lâu, chia sẻ giữa nhiều hàm

### Ví dụ Heap

```java
class Person {
    String name;  // thuộc tính nằm trên heap (bên trong object)
    int age;

    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

public class HeapExample {
    public static void main(String[] args) {
        Person p = new Person("An", 25);
        // p là biến tham chiếu trên stack
        // new Person(...) là object trên heap

        System.out.println(p.name + " - " + p.age);
    }
}
```

**Output:**
```text
An - 25
```

## 3. Tương tác giữa Stack và Heap

- Biến tham chiếu (reference) nằm trên **stack**
- Đối tượng mà nó trỏ tới nằm trên **heap**

```java
public class StackHeapDemo {
    public static void main(String[] args) {
        int a = 5;                 // a trên stack
        Integer b = 10;            // b là reference trên stack, object Integer trên heap
        int[] arr = new int[3];    // arr là reference trên stack, mảng trên heap

        arr[0] = 1;
        arr[1] = 2;
        arr[2] = 3;

        System.out.println("a = " + a);
        System.out.println("b = " + b);
        System.out.println("arr[0] = " + arr[0]);
    }
}
```

**Output:**
```text
a = 5
b = 10
arr[0] = 1
```

---

## 4. Tóm tắt nhanh

- **Stack**
  - Dùng cho: biến cục bộ, tham số hàm, tham chiếu
  - Vòng đời: tồn tại trong thời gian hàm đang chạy
  - Nhanh, tự động thu hồi

- **Heap**
  - Dùng cho: đối tượng, mảng, dữ liệu sống lâu
  - Vòng đời: tồn tại cho đến khi không còn tham chiếu, GC sẽ thu hồi
  - Chia sẻ giữa các thread, chậm hơn stack

Nếu bạn gửi thêm nội dung chi tiết hơn (ví dụ đoạn tài liệu đầy đủ), mình có thể tóm tắt sát hơn với nội dung gốc.
