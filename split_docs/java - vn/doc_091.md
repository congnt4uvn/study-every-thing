
Dưới đây là tóm tắt ngắn gọn về 3 nguyên tắc trong thiết kế hệ thống & kiến trúc: **DRY, KISS, YAGNI**, kèm ví dụ Java.

---

## 1. DRY – Don’t Repeat Yourself (Đừng lặp lại chính mình)

**Ý chính**  
- Tránh lặp lại logic, code, hoặc kiến thức ở nhiều nơi.  
- Giúp dễ bảo trì, giảm bug: sửa 1 nơi thay vì nhiều nơi.

**Ví dụ xấu (vi phạm DRY)**

```java
public class OrderService {

    public double calculateTotalWithTax(double amount) {
        double tax = amount * 0.1; // 10%
        return amount + tax;
    }

    public double calculateDiscountedTotalWithTax(double amount, double discount) {
        double discounted = amount - discount;
        double tax = discounted * 0.1; // lặp lại logic tính thuế
        return discounted + tax;
    }
}
```

**Ví dụ tốt (áp dụng DRY)**

```java
public class OrderService {

    private static final double TAX_RATE = 0.1;

    private double applyTax(double amount) {
        return amount + amount * TAX_RATE;
    }

    public double calculateTotalWithTax(double amount) {
        return applyTax(amount);
    }

    public double calculateDiscountedTotalWithTax(double amount, double discount) {
        double discounted = amount - discount;
        return applyTax(discounted);
    }
}
```

**Input/Output mẫu**

```java
public static void main(String[] args) {
    OrderService service = new OrderService();
    System.out.println(service.calculateTotalWithTax(100));              // 110.0
    System.out.println(service.calculateDiscountedTotalWithTax(100, 20)); // 88.0
}
```

## 2. KISS – Keep It Simple, Stupid (Giữ mọi thứ đơn giản)

**Ý chính**  
- Thiết kế và code càng đơn giản càng tốt.  
- Tránh phức tạp hóa vấn đề khi không cần thiết.  
- Code đơn giản dễ đọc, dễ debug, dễ bảo trì.

**Ví dụ xấu (quá phức tạp)**

```java
public class MathUtils {

    public static int max(int a, int b) {
        if (a > b) {
            return a;
        } else if (b > a) {
            return b;
        } else {
            // xử lý trường hợp bằng nhau một cách rườm rà
            return Integer.valueOf(a).compareTo(Integer.valueOf(b)) >= 0 ? a : b;
        }
    }
}
```

**Ví dụ tốt (KISS)**

```java
public class MathUtils {

    public static int max(int a, int b) {
        return a >= b ? a : b;
    }
}
```

**Input/Output mẫu**

```java
public static void main(String[] args) {
    System.out.println(MathUtils.max(3, 5));  // 5
    System.out.println(MathUtils.max(10, 10)); // 10
}
```

## 3. YAGNI – You Aren’t Gonna Need It (Bạn sẽ không cần nó đâu)

**Ý chính**  
- Đừng viết tính năng, lớp, hoặc abstraction cho những thứ “có thể” cần trong tương lai.  
- Chỉ implement khi có yêu cầu thực tế.  
- Giảm code thừa, giảm độ phức tạp.

**Ví dụ xấu (vi phạm YAGNI – thiết kế quá sớm)**

```java
public interface PaymentStrategy {
    void pay(double amount);
}

public class CreditCardPayment implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        // ...
    }
}

// Hiện tại hệ thống chỉ dùng duy nhất CreditCardPayment,
// nhưng đã tạo interface, pattern phức tạp dù chưa cần.
```

**Ví dụ tốt (áp dụng YAGNI – đơn giản trước)**

```java
public class PaymentService {

    public void payByCreditCard(double amount) {
        // xử lý thanh toán thẻ
    }
}
```

Khi sau này thực sự cần thêm PayPal, BankTransfer…, lúc đó mới refactor sang Strategy pattern.

**Input/Output mẫu**

```java
public static void main(String[] args) {
    PaymentService service = new PaymentService();
    service.payByCreditCard(250_000);
    // Output (giả sử):
    // "Thanh toán 250000.0 bằng thẻ tín dụng thành công"
}
```

## Tóm tắt ngắn

- **DRY**: Tránh lặp code/logic → trích xuất thành method, class dùng chung.  
- **KISS**: Ưu tiên giải pháp đơn giản, rõ ràng → dễ đọc, dễ sửa.  
- **YAGNI**: Không xây tính năng/kiến trúc “dự phòng” khi chưa có nhu cầu thực tế.

Áp dụng 3 nguyên tắc này trong Java (và mọi ngôn ngữ) giúp hệ thống:  
- Ít bug hơn  
- Dễ bảo trì, dễ mở rộng  
- Code sạch, rõ ràng, dễ hiểu cho cả team.
