
Dưới đây là tóm tắt ngắn gọn về chủ đề: **Logging trong Java với SLF4J, Logback, log levels, correlation IDs**.

---

## 1. SLF4J là gì?

- **SLF4J (Simple Logging Facade for Java)** là một *facade* (lớp trung gian) cho logging.
- Bạn code dựa trên API của SLF4J, còn việc ghi log thực tế sẽ do một implementation như **Logback** đảm nhiệm.
- Ưu điểm: dễ thay đổi thư viện logging mà không phải sửa code ứng dụng.

---

## 2. Logback là gì?

- **Logback** là một implementation phổ biến cho SLF4J.
- Cấu hình bằng file `logback.xml` hoặc `logback-spring.xml` (trong Spring Boot).
- Hỗ trợ:
  - Ghi log ra console, file, rolling file (xoay vòng file log),
  - Định dạng log, pattern,
  - Mức log khác nhau cho từng package/class.

---

## 3. Log levels (các mức log chính)

Các mức log thường dùng (từ ít quan trọng đến nghiêm trọng):

- `TRACE` – chi tiết nhất, dùng để debug sâu.
- `DEBUG` – thông tin phục vụ debug trong môi trường dev/test.
- `INFO` – thông tin hoạt động bình thường (startup, shutdown, sự kiện chính).
- `WARN` – cảnh báo, có thể là vấn đề nhưng hệ thống vẫn chạy.
- `ERROR` – lỗi nghiêm trọng, thường cần xử lý ngay.

**Nguyên tắc:**  
- Prod: thường bật từ `INFO` trở lên (hoặc `WARN` trở lên).  
- Dev: có thể bật `DEBUG` hoặc `TRACE` để điều tra lỗi.

---

## 4. Correlation ID là gì?

- **Correlation ID** là một mã định danh duy nhất gắn với mỗi request (hoặc luồng xử lý).
- Mục đích:
  - Dễ dàng truy vết một request xuyên suốt nhiều service/log.
  - Hữu ích trong hệ thống microservices, phân tán.
- Thường được:
  - Lấy từ header HTTP (ví dụ: `X-Correlation-ID`), hoặc
  - Tự sinh (UUID) nếu không có.

## 5. Ví dụ code với SLF4J + Logback

### 5.1. Thêm dependency (Maven)

```xml
<dependencies>
    <!-- SLF4J API -->
    <dependency>
        <groupId>org.slf4j</groupId>
        <artifactId>slf4j-api</artifactId>
        <version>2.0.13</version>
    </dependency>

    <!-- Logback implementation -->
    <dependency>
        <groupId>ch.qos.logback</groupId>
        <artifactId>logback-classic</artifactId>
        <version>1.5.6</version>
    </dependency>
</dependencies>
```

---

### 5.2. Sử dụng logger trong code

```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class UserService {
    private static final Logger log = LoggerFactory.getLogger(UserService.class);

    public User findUserById(String id) {
        log.info("Finding user with id={}", id);   // INFO
        try {
            // giả lập xử lý
            if ("0".equals(id)) {
                throw new IllegalArgumentException("Invalid id");
            }
            User user = new User(id, "Alice");
            log.debug("Found user: {}", user);     // DEBUG
            return user;
        } catch (Exception e) {
            log.error("Error while finding user with id={}", id, e); // ERROR
            throw e;
        }
    }
}
```

**Ví dụ output (console):**

```text
2026-05-20 10:00:01.123  INFO  [main] c.example.UserService - Finding user with id=123
2026-05-20 10:00:01.456 DEBUG [main] c.example.UserService - Found user: User{id='123', name='Alice'}
```

## 6. Cấu hình Logback cơ bản (`logback.xml`)

```xml
<configuration>
    <!-- Appender ghi ra console -->
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} %-5level [%thread] %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>

    <!-- Mức log mặc định -->
    <root level="INFO">
        <appender-ref ref="CONSOLE" />
    </root>

    <!-- Mức log riêng cho package -->
    <logger name="com.example" level="DEBUG" />
</configuration>
```

## 7. Correlation ID với MDC (Mapped Diagnostic Context)

SLF4J/Logback hỗ trợ **MDC** để gắn thêm thông tin (như correlation ID) vào log.

### 7.1. Gắn correlation ID vào MDC

```java
import org.slf4j.MDC;

public void handleRequest(String correlationId) {
    try {
        MDC.put("correlationId", correlationId); // gắn vào context
        log.info("Handling request");
        // xử lý...
    } finally {
        MDC.clear(); // luôn clear để tránh rò rỉ giữa thread
    }
}
```

### 7.2. Cấu hình pattern để in correlation ID

```xml
<encoder>
    <pattern>%d{HH:mm:ss.SSS} %-5level [%thread] [%X{correlationId}] %logger{36} - %msg%n</pattern>
</encoder>
```

**Ví dụ output:**

```text
10:05:00.100 INFO  [http-nio-8080-exec-1] [abc-123] c.example.UserController - Handling request
10:05:00.200 DEBUG [http-nio-8080-exec-1] [abc-123] c.example.UserService    - Found user: User{id='123', name='Alice'}
```

Ở đây:
- `abc-123` là **correlationId**,
- Giúp bạn lọc tất cả log liên quan đến cùng một request.

---

## Tóm tắt ý chính

- **SLF4J**: API chung cho logging, tách biệt code với implementation.
- **Logback**: implementation mạnh mẽ, cấu hình bằng XML, hỗ trợ nhiều appender & pattern.
- **Log levels**: `TRACE`, `DEBUG`, `INFO`, `WARN`, `ERROR` – dùng đúng mức để log có ý nghĩa và dễ quản lý.
- **Correlation ID + MDC**: gắn ID duy nhất cho mỗi request, đưa vào log để dễ truy vết trong hệ thống phân tán.
