
Dưới đây là tóm tắt ngắn gọn về **Filters** và **Interceptors** trong Spring MVC/Web, kèm ví dụ code và input/output minh họa.

---

## 1. Filter là gì?

- Là thành phần của **Servlet** (thuộc Java EE), chạy **trước khi** request vào Servlet (hoặc Spring DispatcherServlet) và **sau khi** response trả về.
- Thường dùng cho:
  - Logging request/response
  - Kiểm tra/bổ sung header
  - Nén dữ liệu (GZIP)
  - Xử lý encoding (UTF-8)
- Không phụ thuộc Spring, hoạt động ở mức **web container** (Tomcat, Jetty,…).

### Ví dụ Filter đơn giản

```java
import jakarta.servlet.*;
import jakarta.servlet.http.HttpServletRequest;
import java.io.IOException;

public class LoggingFilter implements Filter {

    @Override
    public void init(FilterConfig filterConfig) { }

    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
                         FilterChain chain) throws IOException, ServletException {
        HttpServletRequest httpReq = (HttpServletRequest) request;
        System.out.println("Filter - Request URI: " + httpReq.getRequestURI());

        // Cho request đi tiếp
        chain.doFilter(request, response);

        System.out.println("Filter - Response đã được xử lý xong");
    }

    @Override
    public void destroy() { }
}
```

#### Đăng ký Filter trong Spring Boot

```java
import org.springframework.boot.web.servlet.FilterRegistrationBean;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class FilterConfig {

    @Bean
    public FilterRegistrationBean<LoggingFilter> loggingFilter() {
        FilterRegistrationBean<LoggingFilter> registration = new FilterRegistrationBean<>();
        registration.setFilter(new LoggingFilter());
        registration.addUrlPatterns("/*"); // áp dụng cho mọi URL
        registration.setOrder(1);          // thứ tự chạy
        return registration;
    }
}
```

#### Input/Output minh họa

- **Request**: `GET /api/hello`
- **Console log**:

```text
Filter - Request URI: /api/hello
Filter - Response đã được xử lý xong
```

## 2. HandlerInterceptor là gì?

- Là thành phần của **Spring MVC**, chạy xung quanh **Handler (Controller)**.
- Có thể truy cập:
  - `HandlerMethod` (thông tin method controller)
  - `ModelAndView`
  - `HttpServletRequest` / `HttpServletResponse`
- Thường dùng cho:
  - Kiểm tra quyền (authorization)
  - Kiểm tra login
  - Logging ở mức controller
  - Thêm dữ liệu chung vào `Model`

### Các method chính

```java
boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler)
void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView)
void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex)
```

- `preHandle`: chạy **trước** khi controller thực thi. Trả về `false` để chặn request.
- `postHandle`: chạy **sau** khi controller chạy xong, **trước** khi render view.
- `afterCompletion`: chạy **sau cùng**, sau khi view đã render (dùng để dọn dẹp, logging lỗi,…).

### Ví dụ Interceptor kiểm tra login

```java
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import org.springframework.web.servlet.HandlerInterceptor;
import org.springframework.web.servlet.ModelAndView;

public class AuthInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                             Object handler) throws Exception {
        Object user = request.getSession().getAttribute("USER");
        if (user == null) {
            response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            response.getWriter().write("Unauthorized");
            return false; // chặn không cho vào controller
        }
        return true; // cho phép vào controller
    }

    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response,
                           Object handler, ModelAndView modelAndView) {
        System.out.println("postHandle - Controller đã xử lý xong");
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
                                Object handler, Exception ex) {
        System.out.println("afterCompletion - Request hoàn tất");
    }
}
```

#### Đăng ký Interceptor

```java
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.InterceptorRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class WebConfig implements WebMvcConfigurer {

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new AuthInterceptor())
                .addPathPatterns("/api/**")      // áp dụng cho /api/*
                .excludePathPatterns("/login");  // bỏ qua /login
    }
}
```

#### Controller ví dụ

```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {

    @GetMapping("/api/hello")
    public String hello() {
        return "Hello, user!";
    }
}
```

#### Input/Output minh họa

1. **Chưa login (session không có USER)**  
   - Request: `GET /api/hello`
   - Response:
     - Status: `401 Unauthorized`
     - Body: `Unauthorized`

2. **Đã login (session có USER)**  
   - Request: `GET /api/hello`
   - Response:
     - Status: `200 OK`
     - Body: `Hello, user!`
   - Console log:
     ```text
     postHandle - Controller đã xử lý xong
     afterCompletion - Request hoàn tất
     ```

## 3. So sánh nhanh Filter vs Interceptor

| Tiêu chí                 | Filter                               | Interceptor (HandlerInterceptor)          |
|--------------------------|--------------------------------------|-------------------------------------------|
| Thuộc về                 | Servlet (Java EE)                    | Spring MVC                                |
| Mức độ                   | Thấp (web container)                 | Cao (controller level)                    |
| Biết về Controller?      | Không                                | Có (handler, method, annotation, …)       |
| Dùng cho                 | Encoding, GZIP, CORS, logging chung  | Auth, logging business, thêm dữ liệu model|
| Đăng ký                  | `web.xml` hoặc `FilterRegistrationBean` | `WebMvcConfigurer.addInterceptors`     |

---

## 4. Khi nào dùng cái nào?

- **Dùng Filter** khi:
  - Xử lý ở mức rất chung cho mọi request (kể cả static resources).
  - Không cần biết controller nào sẽ xử lý.
- **Dùng Interceptor** khi:
  - Cần logic gắn với **controller** (auth, logging theo endpoint, thêm attribute vào model,…).
  - Cần truy cập `HandlerMethod`, `ModelAndView`.

Nếu bạn muốn, tôi có thể viết thêm ví dụ về:
- Filter xử lý UTF-8
- Interceptor logging chi tiết thời gian xử lý từng endpoint.
