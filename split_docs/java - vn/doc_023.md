
Dưới đây là tóm tắt ngắn gọn về 3 cấu trúc dữ liệu trong Java: `TreeMap`, `PriorityQueue`, `ConcurrentHashMap`, kèm ví dụ code và input/output.

---

## 1. `TreeMap`

**Ý chính:**
- Lưu trữ cặp key–value, giống `HashMap` nhưng **các key được sắp xếp** (theo thứ tự tự nhiên hoặc `Comparator`).
- Dựa trên cây đỏ–đen (Red-Black Tree).
- Tìm kiếm, thêm, xóa: O(log n).
- Không cho phép key `null`.

**Khi dùng:**
- Cần map có thứ tự theo key (ví dụ: sắp xếp theo tên, theo số, theo thời gian).

**Ví dụ:**

```java
import java.util.*;

public class TreeMapExample {
    public static void main(String[] args) {
        TreeMap<Integer, String> map = new TreeMap<>();

        // Input: thêm phần tử
        map.put(3, "Ba");
        map.put(1, "Mot");
        map.put(2, "Hai");

        // Duyệt map
        for (Map.Entry<Integer, String> e : map.entrySet()) {
            System.out.println(e.getKey() + " -> " + e.getValue());
        }

        // Lấy phần tử nhỏ nhất và lớn nhất
        System.out.println("firstKey: " + map.firstKey());
        System.out.println("lastKey: " + map.lastKey());
    }
}
```

**Output:**
```
1 -> Mot
2 -> Hai
3 -> Ba
firstKey: 1
lastKey: 3
```

## 2. `PriorityQueue`

**Ý chính:**
- Hàng đợi ưu tiên: phần tử được lấy ra theo **độ ưu tiên**, không phải theo thứ tự thêm vào.
- Mặc định là **min-heap**: phần tử nhỏ nhất được lấy ra trước.
- Thêm: `offer/add`, lấy đầu: `poll`, xem đầu: `peek`.
- Độ phức tạp: thêm/lấy O(log n).

**Khi dùng:**
- Cần luôn lấy phần tử nhỏ nhất/lớn nhất nhanh (ví dụ: top-k, thuật toán Dijkstra, xử lý task theo độ ưu tiên).

**Ví dụ (min-heap mặc định):**

```java
import java.util.*;

public class PriorityQueueExample {
    public static void main(String[] args) {
        PriorityQueue<Integer> pq = new PriorityQueue<>();

        // Input: thêm phần tử
        pq.offer(5);
        pq.offer(1);
        pq.offer(3);

        // Lấy phần tử theo thứ tự ưu tiên (nhỏ -> lớn)
        while (!pq.isEmpty()) {
            System.out.println(pq.poll());
        }
    }
}
```

**Output:**
```
1
3
5
```

**Ví dụ max-heap (ưu tiên số lớn):**

```java
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
maxHeap.offer(5);
maxHeap.offer(1);
maxHeap.offer(3);

while (!maxHeap.isEmpty()) {
    System.out.println(maxHeap.poll());
}
```

**Output:**
```
5
3
1
```

## 3. `ConcurrentHashMap`

**Ý chính:**
- Map an toàn trong môi trường **đa luồng** (thread-safe).
- Cho phép nhiều luồng đọc/ghi đồng thời hiệu quả hơn `Collections.synchronizedMap`.
- Không khóa toàn bộ map, mà khóa theo từng phần (segment/bucket) → hiệu năng tốt hơn.
- Không cho phép key hoặc value `null`.

**Khi dùng:**
- Dùng trong ứng dụng đa luồng (web server, hệ thống xử lý song song) khi nhiều thread cùng truy cập/ cập nhật map.

**Ví dụ đơn giản:**

```java
import java.util.concurrent.*;

public class ConcurrentHashMapExample {
    public static void main(String[] args) {
        ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

        // Thread 1: thêm dữ liệu
        Thread t1 = new Thread(() -> {
            map.put("A", 1);
            map.put("B", 2);
        });

        // Thread 2: cập nhật dữ liệu
        Thread t2 = new Thread(() -> {
            map.put("B", 3);
            map.put("C", 4);
        });

        t1.start();
        t2.start();

        try {
            t1.join();
            t2.join();
        } catch (InterruptedException e) {
            e.printStackTrace();
        }

        // In kết quả cuối cùng
        map.forEach((k, v) -> System.out.println(k + " -> " + v));
    }
}
```

**Output (thứ tự có thể khác nhau, nhưng giá trị hợp lệ):**
```
A -> 1
B -> 3
C -> 4
```

---

## Tóm tắt so sánh

- **TreeMap**
  - Map có thứ tự theo key.
  - Dựa trên cây, O(log n).
  - Không thread-safe.

- **PriorityQueue**
  - Hàng đợi ưu tiên (heap).
  - Lấy phần tử theo ưu tiên (min hoặc max).
  - Không thread-safe.

- **ConcurrentHashMap**
  - Map không có thứ tự.
  - Tối ưu cho đa luồng, thread-safe.
  - Hiệu năng tốt hơn synchronized map trong môi trường concurrent.
