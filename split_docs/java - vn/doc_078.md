
Dưới đây là tóm tắt nội dung, tập trung vào các ý chính về SQL & RDBMS, kèm ví dụ code và input/output.  

---

## 1. Kiến thức chính cần nắm

1. **SQL & RDBMS (Hệ quản trị CSDL quan hệ)**  
   - Hiểu bảng (table), hàng (row), cột (column), khóa chính (primary key), khóa ngoại (foreign key).  
   - Biết cách thiết kế và truy vấn dữ liệu trong các hệ như MySQL, PostgreSQL, SQL Server, Oracle,...

2. **Joins (kết bảng)**  
   - `INNER JOIN`: chỉ lấy các bản ghi khớp ở cả hai bảng.  
   - `LEFT JOIN`: lấy tất cả bản ghi bên trái, và phần khớp (nếu có) bên phải.  
   - `RIGHT JOIN`, `FULL OUTER JOIN` (tùy hệ CSDL).  

3. **Subqueries (truy vấn lồng)**  
   - Truy vấn nằm trong `SELECT`, `FROM`, hoặc `WHERE`.  
   - Dùng để lọc, tính toán, hoặc thay thế join trong một số trường hợp.

4. **GROUP BY & HAVING**  
   - `GROUP BY`: nhóm dữ liệu theo một hoặc nhiều cột.  
   - Hàm tổng hợp: `COUNT`, `SUM`, `AVG`, `MIN`, `MAX`.  
   - `HAVING`: lọc trên kết quả đã nhóm (khác với `WHERE` là lọc trước khi nhóm).

5. **Window Functions (nếu có thể)**  
   - Ví dụ: `ROW_NUMBER()`, `RANK()`, `DENSE_RANK()`, `SUM() OVER(...)`, `AVG() OVER(...)`.  
   - Cho phép tính toán trên “cửa sổ” dữ liệu mà không gom nhóm thành 1 dòng (khác với `GROUP BY`).

## 2. Ví dụ minh họa

Giả sử có 2 bảng:

```sql
-- Bảng khách hàng
CREATE TABLE customers (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    city VARCHAR(100)
);

-- Bảng đơn hàng
CREATE TABLE orders (
    id INT PRIMARY KEY,
    customer_id INT,
    amount DECIMAL(10,2),
    order_date DATE,
    FOREIGN KEY (customer_id) REFERENCES customers(id)
);
```

### 2.1. INNER JOIN

**Mục tiêu:** Lấy danh sách đơn hàng kèm tên khách hàng.

```sql
SELECT
    o.id AS order_id,
    c.name AS customer_name,
    o.amount
FROM orders o
INNER JOIN customers c ON o.customer_id = c.id;
```

**Input (dữ liệu mẫu)**

`customers`:

| id | name   | city     |
|----|--------|----------|
| 1  | An     | Hanoi    |
| 2  | Bình   | HCM      |

`orders`:

| id | customer_id | amount | order_date  |
|----|-------------|--------|------------|
| 10 | 1           | 100.00 | 2024-01-01 |
| 11 | 2           | 200.00 | 2024-01-02 |

**Output**

| order_id | customer_name | amount |
|----------|---------------|--------|
| 10       | An            | 100.00 |
| 11       | Bình          | 200.00 |

---

### 2.2. LEFT JOIN

**Mục tiêu:** Lấy tất cả khách hàng, kể cả khách chưa có đơn hàng.

```sql
SELECT
    c.name,
    o.id AS order_id,
    o.amount
FROM customers c
LEFT JOIN orders o ON c.id = o.customer_id;
```

Nếu khách hàng id=3 chưa có đơn hàng, họ vẫn xuất hiện với `order_id` = `NULL`.

---

### 2.3. Subquery trong WHERE

**Mục tiêu:** Lấy các đơn hàng có số tiền lớn hơn mức trung bình của tất cả đơn hàng.

```sql
SELECT *
FROM orders
WHERE amount > (
    SELECT AVG(amount) FROM orders
);
```

**Ý chính:** Subquery tính `AVG(amount)` rồi dùng kết quả để lọc.

---

### 2.4. GROUP BY & HAVING

**Mục tiêu:** Tổng số tiền đơn hàng theo khách hàng, chỉ lấy khách có tổng > 150.

```sql
SELECT
    c.name,
    SUM(o.amount) AS total_amount
FROM customers c
JOIN orders o ON c.id = o.customer_id
GROUP BY c.name
HAVING SUM(o.amount) > 150;
```

**Giải thích:**

- `GROUP BY c.name`: nhóm theo khách hàng.  
- `SUM(o.amount)`: tổng tiền mỗi khách.  
- `HAVING SUM(o.amount) > 150`: chỉ giữ nhóm có tổng > 150.

---

### 2.5. Window Function: SUM() OVER

**Mục tiêu:** Tính tổng lũy kế (running total) số tiền theo ngày.

```sql
SELECT
    order_date,
    amount,
    SUM(amount) OVER (
        ORDER BY order_date
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS running_total
FROM orders
ORDER BY order_date;
```

**Input (dữ liệu mẫu)**

| id | amount | order_date  |
|----|--------|------------|
| 10 | 100.00 | 2024-01-01 |
| 11 | 50.00  | 2024-01-02 |
| 12 | 70.00  | 2024-01-03 |

**Output**

| order_date  | amount | running_total |
|-------------|--------|---------------|
| 2024-01-01  | 100.00 | 100.00        |
| 2024-01-02  | 50.00  | 150.00        |
| 2024-01-03  | 70.00  | 220.00        |

---

### 2.6. Window Function: ROW_NUMBER()

**Mục tiêu:** Đánh số thứ tự đơn hàng theo từng khách hàng.

```sql
SELECT
    o.id AS order_id,
    o.customer_id,
    o.amount,
    ROW_NUMBER() OVER (
        PARTITION BY o.customer_id
        ORDER BY o.order_date
    ) AS order_index_for_customer
FROM orders o;
```

**Ý chính:**

- `PARTITION BY customer_id`: chia dữ liệu theo từng khách.  
- `ROW_NUMBER()`: đánh số 1,2,3,... trong từng nhóm khách.

---

## 3. Tóm tắt ngắn gọn

- Cần nắm vững SQL trên RDBMS: thiết kế bảng, khóa, quan hệ.  
- Thành thạo **JOIN** để kết hợp dữ liệu từ nhiều bảng.  
- Biết dùng **subquery** để lọc, tính toán, hoặc thay thế join.  
- Sử dụng **GROUP BY + HAVING** để tổng hợp và lọc dữ liệu đã nhóm.  
- Nếu có thể, học **window functions** để làm các thống kê nâng cao (running total, ranking, phân tích theo nhóm) mà không làm mất chi tiết từng dòng.
