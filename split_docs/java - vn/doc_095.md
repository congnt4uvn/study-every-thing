
Dưới đây là tóm tắt ngắn gọn về **Dependency Injection (DI)** và **Inversion of Control (IoC)** trong Java, kèm ví dụ code và input/output.

---

## 1. Inversion of Control (IoC) là gì?

- IoC là nguyên tắc thiết kế: **thay vì class tự tạo và quản lý các phụ thuộc (dependencies), một “container” hoặc framework sẽ làm việc đó**.
- Mục tiêu:
  - Giảm sự phụ thuộc chặt chẽ giữa các class (loose coupling).
  - Dễ test, dễ mở rộng, dễ bảo trì.
- Framework IoC phổ biến: **Spring**, **CDI (Jakarta EE)**, v.v.

**Không có IoC (tự tạo phụ thuộc):**

```java
class EmailService {
    void send(String message) {
        System.out.println("Sending email: " + message);
    }
}

class UserController {
    private EmailService emailService;

    public UserController() {
        // Tự tạo phụ thuộc
        this.emailService = new EmailService();
    }

    public void registerUser(String name) {
        // logic đăng ký
        emailService.send("Welcome " + name);
    }
}

public class Main {
    public static void main(String[] args) {
        UserController controller = new UserController();
        controller.registerUser("An");
    }
}
```

**Output:**
```
Sending email: Welcome An
```

- Ở đây `UserController` **tự new EmailService**, nên khó thay thế, khó test.

## 2. Dependency Injection (DI) là gì?

- DI là **cách cụ thể để thực hiện IoC**.
- Thay vì class tự tạo dependency, **dependency được “tiêm” (inject) từ bên ngoài**:
  - Qua **constructor**
  - Qua **setter**
  - Hoặc qua **field** (thường dùng với framework như Spring)

### 2.1. Constructor Injection (khuyến khích)

```java
class EmailService {
    void send(String message) {
        System.out.println("Sending email: " + message);
    }
}

class UserController {
    private final EmailService emailService;

    // Dependency được truyền từ bên ngoài
    public UserController(EmailService emailService) {
        this.emailService = emailService;
    }

    public void registerUser(String name) {
        emailService.send("Welcome " + name);
    }
}

public class Main {
    public static void main(String[] args) {
        // Tạo dependency
        EmailService emailService = new EmailService();
        // Inject vào UserController
        UserController controller = new UserController(emailService);

        controller.registerUser("An");
    }
}
```

**Output:**
```
Sending email: Welcome An
```

- `UserController` không còn phụ thuộc vào việc **tự new EmailService**.
- Dễ thay thế `EmailService` bằng mock khi test.

---

### 2.2. Interface + DI (tăng tính linh hoạt)

```java
interface MessageService {
    void send(String message);
}

class EmailService implements MessageService {
    public void send(String message) {
        System.out.println("Email: " + message);
    }
}

class SmsService implements MessageService {
    public void send(String message) {
        System.out.println("SMS: " + message);
    }
}

class UserController {
    private final MessageService messageService;

    public UserController(MessageService messageService) {
        this.messageService = messageService;
    }

    public void registerUser(String name) {
        messageService.send("Welcome " + name);
    }
}

public class Main {
    public static void main(String[] args) {
        // Có thể chọn implementation khác nhau
        MessageService emailService = new EmailService();
        MessageService smsService = new SmsService();

        UserController controller1 = new UserController(emailService);
        UserController controller2 = new UserController(smsService);

        controller1.registerUser("An");
        controller2.registerUser("Bình");
    }
}
```

**Output:**
```
Email: Welcome An
SMS: Welcome Bình
```

- Chỉ cần thay implementation, không cần sửa `UserController`.

## 3. DI với Spring (IoC Container)

Spring cung cấp **IoC Container** để tự động:
- Tạo object (bean)
- Quản lý vòng đời
- Inject dependencies

```java
import org.springframework.stereotype.Service;
import org.springframework.stereotype.Controller;
import org.springframework.beans.factory.annotation.Autowired;

interface MessageService {
    void send(String message);
}

@Service
class EmailService implements MessageService {
    public void send(String message) {
        System.out.println("Email: " + message);
    }
}

@Controller
class UserController {
    private final MessageService messageService;

    @Autowired  // Spring sẽ tự inject bean phù hợp
    public UserController(MessageService messageService) {
        this.messageService = messageService;
    }

    public void registerUser(String name) {
        messageService.send("Welcome " + name);
    }
}
```

Khi chạy ứng dụng Spring:
- Spring scan các class `@Service`, `@Controller`
- Tạo bean `EmailService`, `UserController`
- Tự động inject `MessageService` vào `UserController`.

---

## 4. Tóm tắt ý chính

- **IoC**: Đảo ngược quyền kiểm soát tạo và quản lý object từ code của bạn sang framework/container.
- **DI**: Cách thực hiện IoC bằng cách **tiêm dependencies từ bên ngoài** thay vì tự tạo bên trong class.
- Lợi ích:
  - Giảm coupling giữa các class.
  - Dễ test (mock/stub).
  - Dễ mở rộng, thay thế implementation.
  - Dễ quản lý cấu hình trong hệ thống lớn (system design & architecture).
