
Dưới đây là tóm tắt ngắn gọn về `Optional` trong Java 8+ (bằng tiếng Việt), kèm ví dụ code và input/output.

---

## 1. `Optional` là gì?

- `Optional<T>` là một “container” có thể:
  - chứa 1 giá trị kiểu `T`, hoặc
  - rỗng (không có giá trị).
- Mục đích: giảm lỗi `NullPointerException`, giúp code rõ ràng hơn khi xử lý giá trị có thể null.

---

## 2. Cách tạo `Optional`

```java
Optional<String> o1 = Optional.of("Hello");        // không được null
Optional<String> o2 = Optional.ofNullable(null);   // có thể null
Optional<String> o3 = Optional.empty();            // rỗng
```

- `Optional.of(value)`: ném `NullPointerException` nếu `value == null`.
- `Optional.ofNullable(value)`: cho phép `value` là null.
- `Optional.empty()`: tạo Optional rỗng.

---

## 3. Lấy giá trị từ `Optional`

### 3.1 `get()` và `isPresent()`

```java
Optional<String> opt = Optional.of("Java");

if (opt.isPresent()) {
    System.out.println(opt.get());
}
```

**Output:**
```text
Java
```

> Lưu ý: `get()` trên Optional rỗng sẽ ném `NoSuchElementException`. Thường nên dùng các phương thức an toàn hơn bên dưới.

### 3.2 `orElse`, `orElseGet`, `orElseThrow`

```java
Optional<String> opt1 = Optional.ofNullable(null);

// orElse: trả về giá trị mặc định
String result1 = opt1.orElse("Giá trị mặc định");
System.out.println(result1);

// orElseGet: nhận Supplier, chỉ gọi khi rỗng
String result2 = opt1.orElseGet(() -> "Tạo giá trị khi rỗng");
System.out.println(result2);

// orElseThrow: ném exception nếu rỗng
try {
    String result3 = opt1.orElseThrow(() -> new IllegalArgumentException("Không có giá trị"));
} catch (Exception e) {
    System.out.println(e.getMessage());
}
```

**Output:**
```text
Giá trị mặc định
Tạo giá trị khi rỗng
Không có giá trị
```

## 4. `map` và `flatMap`

Dùng để biến đổi giá trị bên trong `Optional` mà không cần kiểm tra null.

```java
Optional<String> opt = Optional.of("java");

// map: biến đổi giá trị nếu có
Optional<String> upper = opt.map(String::toUpperCase);
System.out.println(upper.orElse("Rỗng"));

// Ví dụ với flatMap (thường dùng khi map trả về Optional)
Optional<String> opt2 = Optional.of("  hello  ");
Optional<String> trimmedUpper = opt2
        .map(String::trim)
        .map(String::toUpperCase);

System.out.println(trimmedUpper.orElse("Rỗng"));
```

**Output:**
```text
JAVA
HELLO
```

## 5. `ifPresent` và `ifPresentOrElse` (Java 9+)

```java
Optional<String> opt = Optional.of("Optional");

opt.ifPresent(value -> System.out.println("Có giá trị: " + value));

Optional<String> emptyOpt = Optional.empty();
emptyOpt.ifPresentOrElse(
        v -> System.out.println("Có giá trị: " + v),
        () -> System.out.println("Không có giá trị")
);
```

**Output:**
```text
Có giá trị: Optional
Không có giá trị
```

## 6. Ví dụ thực tế: tránh `NullPointerException`

### Trước khi có Optional (cách cũ)

```java
public String getUserCity(User user) {
    if (user != null && user.getAddress() != null && user.getAddress().getCity() != null) {
        return user.getAddress().getCity();
    }
    return "Unknown";
}
```

### Với Optional

```java
public Optional<Address> getAddress() { ... }  // giả sử trả về Optional
public Optional<String> getCity() { ... }

public String getUserCity(User user) {
    return Optional.ofNullable(user)
            .flatMap(User::getAddress)
            .flatMap(Address::getCity)
            .orElse("Unknown");
}
```

---

## 7. Lưu ý khi dùng `Optional`

- Dùng `Optional` cho:
  - Kết quả của method có thể không có giá trị.
- Không nên dùng `Optional` cho:
  - Field trong entity (đặc biệt JPA).
  - Tham số method (thường dùng `@Nullable` hoặc document rõ).
- Tránh lạm dụng `get()`; ưu tiên `orElse`, `orElseGet`, `orElseThrow`, `map`, `flatMap`, `ifPresent`.
