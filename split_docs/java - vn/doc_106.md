
Dưới đây là tóm tắt ngắn gọn về khái niệm **Latency** và **Throughput** trong hệ thống phân tán (và Java):

---

## 1. Khái niệm chính

### Latency (Độ trễ)
- Là **thời gian** để xử lý xong **một yêu cầu**.
- Đơn vị thường dùng: milliseconds (ms), microseconds (µs).
- Ví dụ: từ lúc client gửi request đến lúc nhận được response.

### Throughput (Thông lượng)
- Là **số lượng yêu cầu** hệ thống xử lý được trong **một đơn vị thời gian**.
- Đơn vị thường dùng: requests/second (req/s), transactions/second (TPS).

> Hệ thống tốt thường muốn: **latency thấp** và **throughput cao**, nhưng hai yếu tố này thường phải đánh đổi với nhau.

---

## 2. Ví dụ Java đơn giản (đo latency & throughput)

### Ví dụ: xử lý một tác vụ giả lập (sleep 100ms)

```java
public class LatencyThroughputDemo {

    // Mô phỏng xử lý 1 request (mất ~100ms)
    public static void handleRequest() {
        try {
            Thread.sleep(100); // giả lập xử lý tốn thời gian
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }

    public static void main(String[] args) {
        int totalRequests = 20;

        long start = System.currentTimeMillis();

        for (int i = 0; i < totalRequests; i++) {
            long reqStart = System.currentTimeMillis();
            handleRequest();
            long reqEnd = System.currentTimeMillis();

            long latency = reqEnd - reqStart;
            System.out.println("Request " + i + " latency: " + latency + " ms");
        }

        long end = System.currentTimeMillis();
        long totalTime = end - start;

        double throughput = (totalRequests * 1000.0) / totalTime;

        System.out.println("Total time: " + totalTime + " ms");
        System.out.println("Throughput: " + throughput + " requests/second");
    }
}
```

### Đầu ra (ví dụ)

```
Request 0 latency: 101 ms
Request 1 latency: 100 ms
...
Request 19 latency: 100 ms
Total time: 2010 ms
Throughput: 9.95 requests/second
```

- **Latency** mỗi request ~100ms.
- **Throughput** ~10 request/giây.

## 3. Tăng throughput bằng đa luồng (có thể ảnh hưởng latency)

```java
import java.util.concurrent.*;

public class ParallelThroughputDemo {

    public static void handleRequest() {
        try {
            Thread.sleep(100); // giả lập xử lý tốn 100ms
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }

    public static void main(String[] args) throws InterruptedException {
        int totalRequests = 20;
        int threadCount = 10;

        ExecutorService pool = Executors.newFixedThreadPool(threadCount);

        long start = System.currentTimeMillis();

        CountDownLatch latch = new CountDownLatch(totalRequests);

        for (int i = 0; i < totalRequests; i++) {
            pool.submit(() -> {
                long reqStart = System.currentTimeMillis();
                handleRequest();
                long reqEnd = System.currentTimeMillis();
                System.out.println("Latency: " + (reqEnd - reqStart) + " ms");
                latch.countDown();
            });
        }

        latch.await();
        long end = System.currentTimeMillis();
        pool.shutdown();

        long totalTime = end - start;
        double throughput = (totalRequests * 1000.0) / totalTime;

        System.out.println("Total time: " + totalTime + " ms");
        System.out.println("Throughput: " + throughput + " requests/second");
    }
}
```

### Đầu ra (ví dụ)

```
Latency: 101 ms
Latency: 100 ms
...
Total time: 310 ms
Throughput: 64.5 requests/second
```

- Latency mỗi request vẫn ~100ms.
- **Throughput tăng mạnh** vì xử lý song song nhiều request.

---

## 4. Ý chính cần nhớ

1. **Latency** = thời gian xử lý 1 request.  
2. **Throughput** = số request xử lý được mỗi giây.  
3. Tăng **song song/đa luồng** → tăng throughput, nhưng có thể:
   - Tăng độ trễ nếu quá tải (queue, contention).
   - Tăng sử dụng CPU, memory.
4. Thiết kế hệ thống phân tán cần cân bằng giữa **latency thấp** và **throughput cao** tùy mục tiêu thực tế.
