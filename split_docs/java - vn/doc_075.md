
Dưới đây là tóm tắt ngắn gọn về **Spring Security Filter Chain** (chuỗi bộ lọc bảo mật trong Spring Security), kèm ví dụ code và input/output.

---

## 1. Spring Security Filter Chain là gì?

- Là **chuỗi các Servlet Filter** được Spring Security đăng ký vào ứng dụng web.
- Mọi request HTTP đi vào ứng dụng đều đi qua chuỗi filter này trước khi đến Controller.
- Mỗi filter xử lý một phần công việc bảo mật: xác thực (authentication), phân quyền (authorization), CSRF, CORS, session, v.v.

---

## 2. Cách hoạt động (luồng xử lý cơ bản)

1. Client gửi request HTTP (ví dụ: `GET /api/users`).
2. Request đi qua **SecurityFilterChain**:
   - Filter kiểm tra token / session / cookie.
   - Nếu có thông tin xác thực hợp lệ → tạo `Authentication` và lưu vào `SecurityContext`.
   - Nếu không hợp lệ → trả về 401/403 hoặc chuyển hướng đến trang login.
3. Nếu được phép, request tiếp tục đến Controller.
4. Response trả về cho client.

---

## 3. Cấu hình Security Filter Chain (Spring Security 5.7+)

### 3.1. Cấu hình cơ bản với `SecurityFilterChain`

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable()) // tắt CSRF (ví dụ cho API)
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/public/**").permitAll()   // cho phép truy cập không cần login
                .anyRequest().authenticated()                // các request khác phải login
            )
            .httpBasic(Customizer.withDefaults()); // dùng HTTP Basic Auth

        return http.build();
    }
}
```

**Ý chính:**

- `SecurityFilterChain` định nghĩa **quy tắc bảo mật** cho các URL.
- `authorizeHttpRequests` dùng để cấu hình **phân quyền**.
- `httpBasic()` bật xác thực HTTP Basic (username/password qua header).

## 4. Ví dụ: HTTP Basic Auth – Input/Output

### 4.1. Thêm user trong memory

```java
@Bean
public UserDetailsService userDetailsService() {
    UserDetails user = User.withUsername("user")
            .password("{noop}password") // {noop} = không mã hóa (chỉ demo)
            .roles("USER")
            .build();

    return new InMemoryUserDetailsManager(user);
}
```

### 4.2. Request không có thông tin xác thực

**Input (HTTP request):**

```http
GET /api/hello HTTP/1.1
Host: localhost:8080
```

**Output (HTTP response):**

```http
HTTP/1.1 401 Unauthorized
WWW-Authenticate: Basic realm="Realm"
```

---

### 4.3. Request có thông tin xác thực đúng

**Input (HTTP request):**

```http
GET /api/hello HTTP/1.1
Host: localhost:8080
Authorization: Basic dXNlcjpwYXNzd29yZA==   // "user:password" base64
```

**Controller:**

```java
@RestController
public class HelloController {

    @GetMapping("/api/hello")
    public String hello() {
        return "Xin chào, bạn đã xác thực thành công!";
    }
}
```

**Output (HTTP response):**

```http
HTTP/1.1 200 OK
Content-Type: text/plain

Xin chào, bạn đã xác thực thành công!
```

## 5. Ví dụ: Phân quyền theo URL

```java
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/admin/**").hasRole("ADMIN")
            .requestMatchers("/user/**").hasAnyRole("USER", "ADMIN")
            .requestMatchers("/public/**").permitAll()
            .anyRequest().authenticated()
        )
        .formLogin(Customizer.withDefaults()); // dùng form login

    return http.build();
}
```

**Ý chính:**

- `/admin/**` → chỉ user có role `ADMIN`.
- `/user/**` → user có role `USER` hoặc `ADMIN`.
- `/public/**` → ai cũng truy cập được.
- Các URL khác → phải đăng nhập.

---

## 6. Tóm tắt ý chính

- **Security Filter Chain** là chuỗi filter xử lý bảo mật cho mọi request.
- Được cấu hình qua `SecurityFilterChain` bean trong Spring.
- Cho phép:
  - Xác thực (Basic, Form Login, JWT, OAuth2, v.v.).
  - Phân quyền theo URL, role, method.
  - Bật/tắt CSRF, CORS, session, v.v.
- Mỗi request phải “đi qua” chuỗi filter này trước khi đến Controller.
