
Dưới đây là tóm tắt ngắn gọn về 3 mẫu thiết kế khởi tạo (Creational Patterns) trong Java: Singleton, Factory, Builder, kèm ví dụ code và input/output minh họa.

---

## 1. Singleton Pattern

**Ý tưởng chính**  
Đảm bảo **chỉ có duy nhất một instance** của một class trong toàn bộ chương trình, và cung cấp một điểm truy cập toàn cục (global access point) đến instance đó.

**Khi dùng**  
- Cấu hình hệ thống (Config)
- Logger
- Connection pool
- Cache toàn cục

**Ví dụ code (Eager Initialization)**

```java
public class ConfigManager {
    // Chỉ tạo 1 instance duy nhất
    private static final ConfigManager INSTANCE = new ConfigManager();

    // Ẩn constructor
    private ConfigManager() {}

    // Cung cấp điểm truy cập toàn cục
    public static ConfigManager getInstance() {
        return INSTANCE;
    }

    public String getAppName() {
        return "MyApp";
    }
}

public class Main {
    public static void main(String[] args) {
        ConfigManager c1 = ConfigManager.getInstance();
        ConfigManager c2 = ConfigManager.getInstance();

        System.out.println(c1.getAppName());
        System.out.println(c1 == c2); // kiểm tra có cùng instance không
    }
}
```

**Output**

```text
MyApp
true
```

## 2. Factory Method Pattern

**Ý tưởng chính**  
Định nghĩa một **interface hoặc class** có nhiệm vụ **tạo đối tượng**, nhưng để cho các subclass hoặc logic bên trong quyết định **tạo object cụ thể nào**.  
→ Giúp **ẩn chi tiết khởi tạo**, giảm phụ thuộc vào `new` trực tiếp.

**Khi dùng**  
- Khi có nhiều loại đối tượng cùng implement một interface (ví dụ: `Shape`, `Payment`, `Notification`)
- Khi logic tạo object phức tạp hoặc phụ thuộc vào tham số (type, config, môi trường)

**Ví dụ code**

```java
// Product interface
interface Shape {
    void draw();
}

// Concrete products
class Circle implements Shape {
    @Override
    public void draw() {
        System.out.println("Drawing Circle");
    }
}

class Rectangle implements Shape {
    @Override
    public void draw() {
        System.out.println("Drawing Rectangle");
    }
}

// Factory
class ShapeFactory {
    public static Shape getShape(String type) {
        if ("CIRCLE".equalsIgnoreCase(type)) {
            return new Circle();
        } else if ("RECTANGLE".equalsIgnoreCase(type)) {
            return new Rectangle();
        }
        throw new IllegalArgumentException("Unknown shape type: " + type);
    }
}

public class Main {
    public static void main(String[] args) {
        Shape s1 = ShapeFactory.getShape("CIRCLE");
        Shape s2 = ShapeFactory.getShape("RECTANGLE");

        s1.draw();
        s2.draw();
    }
}
```

**Input (logic)**  
- Gọi `ShapeFactory.getShape("CIRCLE")`
- Gọi `ShapeFactory.getShape("RECTANGLE")`

**Output**

```text
Drawing Circle
Drawing Rectangle
```

## 3. Builder Pattern

**Ý tưởng chính**  
Tách rời **quá trình xây dựng (build)** một object phức tạp khỏi **biểu diễn (representation)** của nó.  
→ Giúp tạo object với **nhiều tham số**, đặc biệt là tham số tùy chọn, mà không cần constructor quá dài.

**Khi dùng**  
- Class có nhiều field (nhiều tham số, nhiều optional)
- Muốn code khởi tạo **dễ đọc**, tránh “telescoping constructor” (constructor nhiều tham số khó nhớ thứ tự)

**Ví dụ code (Builder kiểu phổ biến trong Java)**

```java
public class User {
    private final String username;   // bắt buộc
    private final String email;      // bắt buộc
    private final int age;           // tùy chọn
    private final String address;    // tùy chọn

    private User(Builder builder) {
        this.username = builder.username;
        this.email = builder.email;
        this.age = builder.age;
        this.address = builder.address;
    }

    public static class Builder {
        private final String username;
        private final String email;
        private int age;
        private String address;

        public Builder(String username, String email) {
            this.username = username;
            this.email = email;
        }

        public Builder age(int age) {
            this.age = age;
            return this; // cho phép chain
        }

        public Builder address(String address) {
            this.address = address;
            return this;
        }

        public User build() {
            return new User(this);
        }
    }

    @Override
    public String toString() {
        return "User{username='" + username + "', email='" + email +
               "', age=" + age + ", address='" + address + "'}";
    }
}

public class Main {
    public static void main(String[] args) {
        User user1 = new User.Builder("alice", "alice@example.com")
                .age(25)
                .address("Hanoi")
                .build();

        User user2 = new User.Builder("bob", "bob@example.com")
                .build(); // không set age, address

        System.out.println(user1);
        System.out.println(user2);
    }
}
```

**Output (ví dụ)**

```text
User{username='alice', email='alice@example.com', age=25, address='Hanoi'}
User{username='bob', email='bob@example.com', age=0, address='null'}
```

## Tóm tắt ngắn

- **Singleton**:  
  - Một class chỉ có **1 instance duy nhất**.  
  - Dùng cho: config, logger, cache, resource chung.

- **Factory**:  
  - Cung cấp **phương thức tạo object** mà không lộ chi tiết `new`.  
  - Dùng khi có nhiều loại object cùng interface, hoặc logic tạo phức tạp.

- **Builder**:  
  - Dùng để **xây dựng object phức tạp** với nhiều tham số, đặc biệt là tham số tùy chọn.  
  - Giúp code khởi tạo **dễ đọc, linh hoạt**, tránh constructor dài.
