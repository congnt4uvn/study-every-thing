
Dưới đây là tóm tắt ngắn gọn về **Generics trong Java: wildcards (? extends, ? super) và type erasure**, kèm ví dụ code và input/output.

---

## 1. Wildcards trong Generics

### 1.1. `?` là gì?

`?` là **wildcard** – đại diện cho “một kiểu bất kỳ” trong generics.  
Dùng nhiều trong **tham số phương thức** (method parameters), không dùng để khai báo biến kiểu (type parameter).

---

### 1.2. `? extends T` – Upper bounded wildcard

- Nghĩa: “một kiểu bất kỳ **là T hoặc con của T**”.
- Thường dùng khi **chỉ đọc** dữ liệu (không thêm phần tử vào list).

```java
import java.util.*;

class Animal {
    void sound() { System.out.println("Animal sound"); }
}

class Dog extends Animal {
    @Override
    void sound() { System.out.println("Dog barks"); }
}

public class DemoExtends {
    // Chỉ đọc từ list, không thêm mới
    public static void printSounds(List<? extends Animal> animals) {
        for (Animal a : animals) {
            a.sound();
        }
        // animals.add(new Animal()); // LỖI: không được thêm
    }

    public static void main(String[] args) {
        List<Dog> dogs = new ArrayList<>();
        dogs.add(new Dog());
        dogs.add(new Dog());

        printSounds(dogs);
    }
}
```

**Output:**
```text
Dog barks
Dog barks
```

**Ý chính:**

- `List<? extends Animal>` có thể nhận `List<Dog>`, `List<Cat>`, …
- Bạn **có thể đọc** phần tử dưới dạng `Animal`.
- Bạn **không thể thêm** phần tử (trừ `null`) vì không biết chính xác kiểu con là gì.

---

### 1.3. `? super T` – Lower bounded wildcard

- Nghĩa: “một kiểu bất kỳ **là T hoặc cha của T**”.
- Thường dùng khi **chỉ ghi (thêm)** dữ liệu vào collection.

```java
import java.util.*;

class Animal {
    @Override
    public String toString() { return "Animal"; }
}

class Dog extends Animal {
    @Override
    public String toString() { return "Dog"; }
}

public class DemoSuper {
    // Chỉ thêm Dog (hoặc con của Dog) vào list
    public static void addDogs(List<? super Dog> list) {
        list.add(new Dog());
        // list.add(new Animal()); // LỖI: chỉ chắc chắn list chấp nhận Dog
    }

    public static void main(String[] args) {
        List<Animal> animals = new ArrayList<>();
        addDogs(animals);

        for (Object o : animals) {
            System.out.println(o);
        }
    }
}
```

**Output:**
```text
Dog
```

**Ý chính:**

- `List<? super Dog>` có thể là `List<Dog>`, `List<Animal>`, `List<Object>`.
- Bạn **có thể thêm** `Dog` vào list.
- Khi lấy ra, chỉ chắc chắn là `Object` (hoặc kiểu cha), nên phải cast nếu cần.

---

### 1.4. Quy tắc nhớ nhanh: PECS

**PECS** = *Producer Extends, Consumer Super*

- Nếu collection **sản xuất dữ liệu** (bạn chỉ đọc): dùng `? extends T`.
- Nếu collection **tiêu thụ dữ liệu** (bạn chỉ thêm): dùng `? super T`.

---

## 2. Type Erasure (Xóa kiểu tại runtime)

### 2.1. Khái niệm

- Generics trong Java chỉ tồn tại ở **compile-time**.
- Khi chạy (runtime), thông tin kiểu generic bị **xóa bỏ** (erased).
- Ví dụ: `List<String>` và `List<Integer>` đều trở thành `List` khi chạy.

```java
import java.util.*;

public class ErasureDemo {
    public static void main(String[] args) {
        List<String> list1 = new ArrayList<>();
        List<Integer> list2 = new ArrayList<>();

        System.out.println(list1.getClass() == list2.getClass());
        System.out.println(list1.getClass().getName());
    }
}
```

**Output:**
```text
true
java.util.ArrayList
```

**Ý chính:**

- `list1` và `list2` có **cùng class** tại runtime.
- Không thể phân biệt `List<String>` hay `List<Integer>` bằng reflection.

---

### 2.2. Hệ quả của type erasure

1. **Không thể tạo mảng generic trực tiếp**

```java
// List<String>[] arr = new List<String>[10]; // LỖI
List<String>[] arr = (List<String>[]) new List[10]; // phải cast, có warning
```

2. **Không thể dùng kiểu generic trong instanceof**

```java
List<String> list = new ArrayList<>();
// if (list instanceof List<String>) {} // LỖI
if (list instanceof List) {             // OK
    System.out.println("It's a List");
}
```

3. **Không thể tạo đối tượng với type parameter**

```java
class Box<T> {
    // T value = new T(); // LỖI: không biết T là gì khi runtime
}
```

---

## 3. Tóm tắt ngắn

- `? extends T`: dùng khi **chỉ đọc**, chấp nhận `T` và các lớp con.
- `? super T`: dùng khi **chỉ ghi**, chấp nhận `T` và các lớp cha.
- **PECS**: Producer Extends, Consumer Super.
- **Type erasure**: generics bị xóa kiểu khi runtime → không phân biệt `List<String>` và `List<Integer>` khi chạy, dẫn đến một số hạn chế (không `new T()`, không `instanceof` với tham số kiểu, không mảng generic an toàn).
