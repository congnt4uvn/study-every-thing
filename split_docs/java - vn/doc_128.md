
Dưới đây là tóm tắt nội dung về **Tools & Ecosystem – Monitoring** trong Java, kèm ví dụ minh họa.

---

## 1. Khái niệm chính

### a. Metrics (Chỉ số)
- Là các số liệu định lượng về hệ thống: CPU, RAM, số request/giây, thời gian phản hồi, số lỗi, v.v.
- Dùng để:
  - Theo dõi hiệu năng
  - Cảnh báo khi vượt ngưỡng (alerting)
  - Tối ưu hệ thống

**Ví dụ (Spring Boot + Micrometer + Prometheus):**

```java
// build.gradle (trích đoạn)
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-actuator'
    implementation 'io.micrometer:micrometer-registry-prometheus'
}
```

```yaml
# application.yml
management:
  endpoints:
    web:
      exposure:
        include: health,info,prometheus
```

**Input:** Gửi HTTP GET đến `/actuator/prometheus`  
**Output (một phần):**
```text
# HELP http_server_requests_seconds_max  
# TYPE http_server_requests_seconds_max gauge
http_server_requests_seconds_max{exception="None",method="GET",outcome="SUCCESS",status="200",uri="/hello",} 0.0123
```

### b. Logs (Nhật ký)
- Ghi lại các sự kiện xảy ra trong ứng dụng: lỗi, cảnh báo, thông tin debug, audit.
- Dùng để:
  - Điều tra lỗi (troubleshooting)
  - Theo dõi hành vi người dùng/hệ thống
  - Kết hợp với ELK để tìm kiếm & phân tích

**Ví dụ log trong Java (SLF4J + Logback):**

```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class UserService {
    private static final Logger log = LoggerFactory.getLogger(UserService.class);

    public User findUser(String id) {
        log.info("Finding user with id={}", id);
        try {
            // giả lập xử lý
            if ("123".equals(id)) {
                return new User("123", "Alice");
            } else {
                throw new RuntimeException("User not found");
            }
        } catch (Exception e) {
            log.error("Error finding user id={}", id, e);
            throw e;
        }
    }
}
```

**Output log (ví dụ):**
```text
INFO  c.example.UserService - Finding user with id=123
ERROR c.example.UserService - Error finding user id=999
java.lang.RuntimeException: User not found
    at c.example.UserService.findUser(UserService.java:15)
    ...
```

### c. Tracing (Theo dõi phân tán)
- Theo dõi luồng xử lý của một request đi qua nhiều service (microservices).
- Mỗi request có:
  - **Trace ID**: định danh toàn bộ luồng
  - **Span**: từng đoạn công việc trong luồng
- Dùng để:
  - Tìm “nút thắt cổ chai” (bottleneck)
  - Hiểu luồng gọi giữa các service
  - Debug trong hệ thống phân tán

**Ví dụ (Spring Cloud Sleuth + Zipkin):**

```java
// build.gradle (trích đoạn)
dependencies {
    implementation 'org.springframework.cloud:spring-cloud-starter-sleuth'
    implementation 'org.springframework.cloud:spring-cloud-sleuth-zipkin'
}
```

```java
@RestController
public class HelloController {

    @GetMapping("/hello")
    public String hello() {
        // Sleuth tự động tạo traceId, spanId và gửi sang Zipkin
        return "Hello";
    }
}
```

**Kết quả:**  
- Khi gọi `/hello`, bạn có thể xem trace trong giao diện Zipkin:
  - Input: HTTP GET `/hello`
  - Output: Trace hiển thị thời gian xử lý, các span, service liên quan.

## 2. Các công cụ phổ biến

### ELK Stack
- **Elasticsearch**: lưu trữ & tìm kiếm log.
- **Logstash**: thu thập, xử lý, chuyển log.
- **Kibana**: giao diện để xem, tìm kiếm, vẽ biểu đồ log.

**Luồng cơ bản:**
Ứng dụng Java → ghi log → Logstash → Elasticsearch → Kibana (dashboard, search).

---

### Prometheus
- Hệ thống thu thập metrics (pull từ các endpoint như `/actuator/prometheus`).
- Lưu trữ time-series data.
- Hỗ trợ alerting (Alertmanager).

**Ví dụ cấu hình Prometheus (prometheus.yml):**
```yaml
scrape_configs:
  - job_name: 'my-java-app'
    metrics_path: '/actuator/prometheus'
    static_configs:
      - targets: ['my-java-app:8080']
```

---

### Grafana
- Công cụ vẽ dashboard, biểu đồ từ nhiều nguồn dữ liệu (Prometheus, Elasticsearch, v.v.).
- Dùng để:
  - Tạo dashboard theo dõi CPU, RAM, request/giây, error rate, latency, v.v.

---

### Zipkin / Jaeger
- Hệ thống quản lý **distributed tracing**.
- Thu thập trace từ các service (qua HTTP, Kafka, v.v.).
- Giao diện để:
  - Xem từng trace
  - Xem thời gian từng span
  - Phát hiện service chậm

**Ví dụ cấu hình Zipkin trong Spring Boot (application.yml):**
```yaml
spring:
  zipkin:
    base-url: http://zipkin:9411
  sleuth:
    sampler:
      probability: 1.0   # 100% request được trace (chỉ nên dùng trong dev)
```

---

## 3. Tóm tắt ngắn gọn

- **Metrics**: số liệu định lượng (CPU, request, latency, error rate) → dùng Prometheus + Grafana.
- **Logs**: nhật ký sự kiện, lỗi → dùng ELK (Elasticsearch, Logstash, Kibana).
- **Tracing**: theo dõi luồng request qua nhiều service → dùng Zipkin hoặc Jaeger.
- Trong Java (đặc biệt Spring Boot), có thể:
  - Dùng **Actuator + Micrometer** để xuất metrics.
  - Dùng **SLF4J/Logback** để ghi log.
  - Dùng **Spring Cloud Sleuth** để tạo trace và gửi sang Zipkin/Jaeger.
