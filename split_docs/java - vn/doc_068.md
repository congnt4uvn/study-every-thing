
Dưới đây là tóm tắt ngắn gọn về xử lý Exception trong Spring MVC/Web với `@ControllerAdvice` và `@ExceptionHandler`, kèm ví dụ code và input/output.

---

## 1. Mục tiêu chính

- Tách riêng logic xử lý lỗi khỏi controller.
- Xử lý tập trung (global) các exception xảy ra trong ứng dụng web.
- Trả về response rõ ràng, có cấu trúc (JSON, view, status code).

---

## 2. `@ExceptionHandler` trong Controller

Dùng để xử lý exception ngay trong một controller cụ thể.

```java
@Controller
@RequestMapping("/users")
public class UserController {

    @GetMapping("/{id}")
    @ResponseBody
    public User getUser(@PathVariable Long id) {
        if (id <= 0) {
            throw new IllegalArgumentException("ID phải > 0");
        }
        // giả sử không tìm thấy user
        throw new UserNotFoundException("Không tìm thấy user với id = " + id);
    }

    // Xử lý IllegalArgumentException trong controller này
    @ExceptionHandler(IllegalArgumentException.class)
    @ResponseBody
    public ResponseEntity<String> handleIllegalArgument(IllegalArgumentException ex) {
        return ResponseEntity
                .badRequest()
                .body("Lỗi tham số: " + ex.getMessage());
    }
}
```

### Ví dụ request/response

**Request:**

```http
GET /users/0
```

**Response:**

```http
HTTP/1.1 400 Bad Request
Content-Type: text/plain

Lỗi tham số: ID phải > 0
```

## 3. `@ControllerAdvice` – Xử lý Exception toàn cục (Global)

Dùng để định nghĩa các handler cho exception áp dụng cho nhiều controller (toàn ứng dụng).

```java
@ControllerAdvice
public class GlobalExceptionHandler {

    // Xử lý UserNotFoundException cho toàn bộ controller
    @ExceptionHandler(UserNotFoundException.class)
    @ResponseBody
    public ResponseEntity<Map<String, Object>> handleUserNotFound(UserNotFoundException ex) {
        Map<String, Object> body = new HashMap<>();
        body.put("error", "USER_NOT_FOUND");
        body.put("message", ex.getMessage());

        return ResponseEntity
                .status(HttpStatus.NOT_FOUND)
                .body(body);
    }

    // Xử lý mọi Exception còn lại
    @ExceptionHandler(Exception.class)
    @ResponseBody
    public ResponseEntity<Map<String, Object>> handleGeneralException(Exception ex) {
        Map<String, Object> body = new HashMap<>();
        body.put("error", "INTERNAL_ERROR");
        body.put("message", "Đã xảy ra lỗi hệ thống");

        return ResponseEntity
                .status(HttpStatus.INTERNAL_SERVER_ERROR)
                .body(body);
    }
}
```

### Ví dụ request/response

**Request:**

```http
GET /users/123   // giả sử ném UserNotFoundException
```

**Response:**

```http
HTTP/1.1 404 Not Found
Content-Type: application/json

{
  "error": "USER_NOT_FOUND",
  "message": "Không tìm thấy user với id = 123"
}
```

## 4. `@RestControllerAdvice` (cho REST API)

Tương tự `@ControllerAdvice` nhưng mặc định thêm `@ResponseBody` cho tất cả method (thường dùng cho REST API).

```java
@RestControllerAdvice
public class RestGlobalExceptionHandler {

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<Map<String, Object>> handleValidation(MethodArgumentNotValidException ex) {
        Map<String, Object> body = new HashMap<>();
        body.put("error", "VALIDATION_ERROR");
        body.put("message", ex.getBindingResult().getFieldError().getDefaultMessage());
        return ResponseEntity.badRequest().body(body);
    }
}
```

---

## 5. Ý chính cần nhớ

- `@ExceptionHandler`:
  - Đặt trong controller hoặc trong `@ControllerAdvice`.
  - Chỉ định loại exception cần xử lý.
- `@ControllerAdvice` / `@RestControllerAdvice`:
  - Xử lý exception toàn cục cho nhiều controller.
  - Thích hợp cho API lớn, cần chuẩn hóa format lỗi.
- Có thể:
  - Trả về view (tên trang lỗi).
  - Trả về JSON/XML với `@ResponseBody` hoặc dùng `@RestControllerAdvice`.
  - Thiết lập HTTP status code phù hợp (`400`, `404`, `500`, …).
