
Dưới đây là tóm tắt ngắn gọn về **Unit & Integration Testing với JUnit 5** kèm ví dụ mã và input/output.

---

## 1. Khái niệm chính

- **Unit Test**:  
  Kiểm thử từng đơn vị nhỏ (class, method) một cách độc lập.  
  Mục tiêu: đảm bảo logic nhỏ hoạt động đúng, nhanh, chạy thường xuyên.

- **Integration Test**:  
  Kiểm thử sự kết hợp giữa nhiều thành phần (service + repository + DB, API, …).  
  Mục tiêu: đảm bảo các module làm việc đúng với nhau.

- **JUnit 5**:  
  Framework test cho Java, gồm:
  - `junit-jupiter-api`: viết test
  - `junit-jupiter-engine`: chạy test
  - Hỗ trợ annotation hiện đại, test linh hoạt.

---

## 2. Các annotation cơ bản trong JUnit 5

- `@Test` – đánh dấu một method là test.
- `@BeforeEach` – chạy trước mỗi test (setup).
- `@AfterEach` – chạy sau mỗi test (cleanup).
- `@BeforeAll` – chạy một lần trước tất cả test (static).
- `@AfterAll` – chạy một lần sau tất cả test (static).
- `@DisplayName` – đặt tên hiển thị cho test.
- `@Disabled` – bỏ qua test.

## 3. Ví dụ Unit Test đơn giản

### Class cần test

```java
public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }

    public int divide(int a, int b) {
        if (b == 0) throw new IllegalArgumentException("b must not be zero");
        return a / b;
    }
}
```

### Test với JUnit 5

```java
import org.junit.jupiter.api.*;

import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {

    private Calculator calculator;

    @BeforeEach
    void setUp() {
        calculator = new Calculator();
    }

    @Test
    @DisplayName("Cộng hai số dương")
    void testAddTwoPositiveNumbers() {
        // Input
        int a = 2;
        int b = 3;

        // Thực thi
        int result = calculator.add(a, b);

        // Output mong đợi
        assertEquals(5, result);
    }

    @Test
    @DisplayName("Chia cho 0 phải ném exception")
    void testDivideByZero() {
        // Input
        int a = 10;
        int b = 0;

        // Kiểm tra exception
        Exception ex = assertThrows(IllegalArgumentException.class,
                () -> calculator.divide(a, b));

        assertEquals("b must not be zero", ex.getMessage());
    }
}
```

**Kết quả chạy test (output điển hình):**

- `testAddTwoPositiveNumbers` → PASSED  
- `testDivideByZero` → PASSED  

Trong IDE (IntelliJ/Eclipse) sẽ hiển thị 2 test xanh.

## 4. Assertions thường dùng

- `assertEquals(expected, actual)`
- `assertNotNull(object)`
- `assertTrue(condition)`
- `assertFalse(condition)`
- `assertThrows(Exception.class, executable)`

Ví dụ:

```java
assertEquals(5, calculator.add(2, 3));
assertThrows(IllegalArgumentException.class, () -> calculator.divide(1, 0));
```

## 5. Ví dụ Integration Test (Spring Boot + JUnit 5)

Giả sử có service gọi repository truy cập DB.

### Service

```java
@Service
public class UserService {
    @Autowired
    private UserRepository userRepository;

    public User createUser(String name) {
        User user = new User();
        user.setName(name);
        return userRepository.save(user);
    }
}
```

### Integration Test

```java
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.context.SpringBootTest;
import static org.assertj.core.api.Assertions.assertThat;

@SpringBootTest // khởi động context Spring, dùng DB test, bean thật
class UserServiceIntegrationTest {

    @Autowired
    private UserService userService;

    @Test
    void createUser_shouldPersistToDatabase() {
        // Input
        String name = "Alice";

        // Thực thi
        User saved = userService.createUser(name);

        // Output mong đợi
        assertThat(saved.getId()).isNotNull();
        assertThat(saved.getName()).isEqualTo("Alice");
    }
}
```

**Ý nghĩa:**

- Đây là **integration test** vì:
  - Dùng `@SpringBootTest` → load toàn bộ Spring context.
  - Gọi `UserService` thật, `UserRepository` thật, DB (thường là H2 in-memory hoặc test DB).
- Kiểm tra luồng: Service → Repository → DB.

---

## 6. Phân biệt nhanh Unit vs Integration Test

- **Unit Test**
  - Không dùng DB thật, không gọi network.
  - Thường dùng mock (Mockito, …).
  - Nhanh, số lượng nhiều, chạy liên tục.

- **Integration Test**
  - Dùng nhiều thành phần thật (Spring context, DB test).
  - Chậm hơn, ít hơn.
  - Đảm bảo hệ thống hoạt động đúng khi các module kết hợp.

---
