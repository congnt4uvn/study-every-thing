
Dưới đây là bản tóm tắt ngắn gọn (tiếng Việt) về cách `hashCode()` và `equals()` hoạt động, hợp đồng (contract) giữa chúng và các “bẫy” thường gặp, kèm ví dụ code.

---

## 1. Mối quan hệ giữa `equals()` và `hashCode()`

**Hợp đồng (contract) cơ bản:**

1. Nếu `a.equals(b) == true` thì **bắt buộc** `a.hashCode() == b.hashCode()`.
2. Nếu `a.hashCode() != b.hashCode()` thì chắc chắn `a.equals(b) == false`.
3. Nếu `a.hashCode() == b.hashCode()` thì **chưa chắc** `a.equals(b) == true` (có thể xảy ra collision).

**Ý nghĩa:**

- `equals()` dùng để so sánh **bằng nhau về mặt logic**.
- `hashCode()` dùng để phân tán đối tượng vào các “bucket” trong các cấu trúc như `HashMap`, `HashSet`.

---

## 2. Tại sao phải override cả hai?

Khi bạn override `equals()` mà **không** override `hashCode()`:

- Hai đối tượng có thể được coi là bằng nhau theo `equals()`,
- Nhưng có `hashCode()` khác nhau (do dùng implementation mặc định từ `Object`),
- Dẫn đến `HashSet`, `HashMap` không hoạt động đúng (không tìm thấy phần tử, trùng key không bị phát hiện, v.v.).

## 3. Ví dụ sai (chỉ override `equals()`)

```java
import java.util.HashSet;
import java.util.Objects;
import java.util.Set;

class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Chỉ override equals, KHÔNG override hashCode
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Person)) return false;
        Person person = (Person) o;
        return age == person.age &&
               Objects.equals(name, person.name);
    }

    // hashCode() vẫn dùng từ Object (mặc định)
}

public class Demo {
    public static void main(String[] args) {
        Set<Person> set = new HashSet<>();
        Person p1 = new Person("An", 20);
        Person p2 = new Person("An", 20);

        set.add(p1);
        System.out.println("set contains p2? " + set.contains(p2));
    }
}
```

**Kỳ vọng (logic):** `p1` và `p2` “bằng nhau” → `contains(p2)` nên là `true`.  
**Thực tế:** Có thể in ra `false` vì `hashCode()` khác nhau → `HashSet` không tìm thấy.

**Output có thể:**
```text
set contains p2? false
```

## 4. Ví dụ đúng (override cả `equals()` và `hashCode()`)

```java
import java.util.HashSet;
import java.util.Objects;
import java.util.Set;

class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // equals: so sánh theo name và age
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Person)) return false;
        Person person = (Person) o;
        return age == person.age &&
               Objects.equals(name, person.name);
    }

    // hashCode: dựa trên cùng các field với equals
    @Override
    public int hashCode() {
        return Objects.hash(name, age);
    }
}

public class Demo {
    public static void main(String[] args) {
        Set<Person> set = new HashSet<>();
        Person p1 = new Person("An", 20);
        Person p2 = new Person("An", 20);

        set.add(p1);
        System.out.println("set contains p2? " + set.contains(p2));
        System.out.println("set size = " + set.size());
    }
}
```

**Output:**
```text
set contains p2? true
set size = 1
```

- `p1` và `p2` được coi là cùng một phần tử trong `HashSet`.

## 5. Các “bẫy” thường gặp

1. **Dùng field mutable trong `equals()`/`hashCode()`**  
   - Nếu object được dùng làm key trong `HashMap`/`HashSet` mà bạn thay đổi field ảnh hưởng đến `hashCode()`, object có thể “mất” khỏi map/set.
   - Ví dụ: đổi `name` sau khi đã put vào `HashMap`.

2. **Không nhất quán giữa `equals()` và `hashCode()`**  
   - `equals()` dùng một tập field, `hashCode()` dùng tập field khác → vi phạm contract.

3. **Vi phạm tính chất của `equals()`**  
   - Phải đảm bảo:  
     - Tự phản xạ (reflexive): `x.equals(x)` luôn `true`.  
     - Đối xứng (symmetric): `x.equals(y)` == `y.equals(x)`.  
     - Bắt cầu (transitive): nếu `x.equals(y)` và `y.equals(z)` thì `x.equals(z)`.  
     - Nhất quán (consistent): kết quả không đổi nếu object không đổi.  
     - Không so sánh bằng `null`: `x.equals(null)` luôn `false`.

## 6. Ví dụ với `HashMap` (key custom)

```java
import java.util.HashMap;
import java.util.Map;
import java.util.Objects;

class Point {
    private int x;
    private int y;

    public Point(int x, int y) {
        this.x = x;
        this.y = y;
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Point)) return false;
        Point point = (Point) o;
        return x == point.x && y == point.y;
    }

    @Override
    public int hashCode() {
        return Objects.hash(x, y);
    }
}

public class DemoMap {
    public static void main(String[] args) {
        Map<Point, String> map = new HashMap<>();
        Point p1 = new Point(1, 2);
        Point p2 = new Point(1, 2);

        map.put(p1, "A");
        System.out.println("Value with p2: " + map.get(p2));
        System.out.println("Map size = " + map.size());
    }
}
```

**Output:**
```text
Value with p2: A
Map size = 1
```

- `p1` và `p2` được coi là cùng một key.

---

## 7. Tóm tắt ngắn

- Luôn override **cả** `equals()` và `hashCode()` khi so sánh logic các object.
- Các field dùng trong `equals()` phải **trùng** với các field dùng trong `hashCode()`.
- Tránh dùng field dễ thay đổi (mutable) trong `hashCode()` nếu object được dùng làm key trong `HashMap`/`HashSet`.
- Vi phạm contract sẽ làm các collection dựa trên hash (HashMap, HashSet, LinkedHashMap, v.v.) hoạt động sai hoặc khó debug.
