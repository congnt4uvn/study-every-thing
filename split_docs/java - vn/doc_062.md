
Dưới đây là tóm tắt ngắn gọn về `application.yml` và `application.properties` trong Spring Boot, kèm ví dụ code và input/output.

---

## 1. Mục đích của `application.yml` và `application.properties`

- Cả hai đều dùng để:
  - Cấu hình ứng dụng Spring Boot (port, database, logging, profile, v.v.)
  - Tách biệt cấu hình khỏi code (dễ thay đổi theo môi trường: dev, test, prod)
- Khác nhau chủ yếu ở **định dạng**:
  - `application.properties`: dạng key=value
  - `application.yml`: dạng YAML, hỗ trợ cấu trúc lồng nhau, dễ đọc hơn

## 2. Ví dụ cấu hình cơ bản

### 2.1. Dùng `application.properties`

```properties
# application.properties
server.port=8081

spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=root
spring.datasource.password=123456

logging.level.root=INFO
logging.level.com.example.demo=DEBUG
```

**Ý nghĩa:**
- Ứng dụng chạy trên port `8081`
- Kết nối MySQL với DB `mydb`
- Log mặc định `INFO`, riêng package `com.example.demo` là `DEBUG`

---

### 2.2. Dùng `application.yml`

```yaml
# application.yml
server:
  port: 8081

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb
    username: root
    password: 123456

logging:
  level:
    root: INFO
    com.example.demo: DEBUG
```

**Ý nghĩa:** giống hệt file `.properties` ở trên, chỉ khác cách viết.

## 3. Ví dụ binding cấu hình vào class Java

### 3.1. Cấu hình trong `application.yml`

```yaml
app:
  name: Demo Application
  version: 1.0.0
  features:
    enableSignup: true
    maxUsers: 100
```

### 3.2. Class Java đọc cấu hình

```java
// AppProperties.java
import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.stereotype.Component;

@Component
@ConfigurationProperties(prefix = "app")
public class AppProperties {
    private String name;
    private String version;
    private Features features;

    public static class Features {
        private boolean enableSignup;
        private int maxUsers;

        // getters & setters
        public boolean isEnableSignup() { return enableSignup; }
        public void setEnableSignup(boolean enableSignup) { this.enableSignup = enableSignup; }
        public int getMaxUsers() { return maxUsers; }
        public void setMaxUsers(int maxUsers) { this.maxUsers = maxUsers; }
    }

    // getters & setters
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public String getVersion() { return version; }
    public void setVersion(String version) { this.version = version; }
    public Features getFeatures() { return features; }
    public void setFeatures(Features features) { this.features = features; }
}
```

### 3.3. Sử dụng trong Controller (Input/Output)

```java
// DemoController.java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class DemoController {

    private final AppProperties appProperties;

    public DemoController(AppProperties appProperties) {
        this.appProperties = appProperties;
    }

    @GetMapping("/info")
    public AppInfoResponse getInfo() {
        AppInfoResponse res = new AppInfoResponse();
        res.setName(appProperties.getName());
        res.setVersion(appProperties.getVersion());
        res.setEnableSignup(appProperties.getFeatures().isEnableSignup());
        res.setMaxUsers(appProperties.getFeatures().getMaxUsers());
        return res;
    }

    public static class AppInfoResponse {
        private String name;
        private String version;
        private boolean enableSignup;
        private int maxUsers;

        // getters & setters
        public String getName() { return name; }
        public void setName(String name) { this.name = name; }
        public String getVersion() { return version; }
        public void setVersion(String version) { this.version = version; }
        public boolean isEnableSignup() { return enableSignup; }
        public void setEnableSignup(boolean enableSignup) { this.enableSignup = enableSignup; }
        public int getMaxUsers() { return maxUsers; }
        public void setMaxUsers(int maxUsers) { this.maxUsers = maxUsers; }
    }
}
```

**Input (HTTP request):**

```http
GET /info HTTP/1.1
Host: localhost:8081
```

**Output (JSON response):**

```json
{
  "name": "Demo Application",
  "version": "1.0.0",
  "enableSignup": true,
  "maxUsers": 100
}
```

## 4. Profiles: `application-dev.yml`, `application-prod.yml`

- Dùng để cấu hình theo môi trường:

```yaml
# application.yml
spring:
  profiles:
    active: dev
```

```yaml
# application-dev.yml
server:
  port: 8080
```

```yaml
# application-prod.yml
server:
  port: 80
```

Chạy với profile `dev` → port 8080  
Chạy với profile `prod` (ví dụ: `--spring.profiles.active=prod`) → port 80

---

## 5. Tóm tắt ý chính

- `application.properties` và `application.yml` đều dùng để cấu hình Spring Boot.
- `.yml` hỗ trợ cấu trúc lồng nhau, dễ đọc; `.properties` đơn giản, key=value.
- Có thể bind cấu hình vào class Java bằng `@ConfigurationProperties`.
- Hỗ trợ profiles để tách cấu hình theo môi trường (`dev`, `test`, `prod`).
- Cấu hình ảnh hưởng trực tiếp đến hành vi ứng dụng (port, DB, logging, v.v.).
