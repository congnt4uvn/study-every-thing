
Dưới đây là tóm tắt ngắn gọn về **Java 8+ Streams API** (tập trung vào: toán tử trung gian, toán tử kết thúc, và cơ chế đánh giá lười – lazy evaluation), kèm ví dụ code và input/output.

---

## 1. Streams API là gì?

- Là một API trong Java 8+ giúp xử lý tập dữ liệu (Collection, array, …) theo kiểu **hàm** (functional style).
- Cho phép:
  - Lọc (filter)
  - Biến đổi (map)
  - Sắp xếp (sorted)
  - Gom nhóm, tổng hợp (collect, reduce)
- Không thay đổi dữ liệu gốc, thường tạo ra kết quả mới.

---

## 2. Toán tử trung gian (Intermediate Operations)

- Trả về **Stream mới**.
- **Không thực thi ngay** (lazy) – chỉ được “xếp hàng”.
- Ví dụ: `filter`, `map`, `sorted`, `distinct`, `limit`, `skip`, `peek`, …

Ví dụ:

```java
import java.util.Arrays;
import java.util.List;
import java.util.stream.Stream;

public class IntermediateExample {
    public static void main(String[] args) {
        List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);

        Stream<Integer> stream = numbers.stream()
                .filter(n -> {
                    System.out.println("filter: " + n);
                    return n % 2 == 1; // giữ số lẻ
                })
                .map(n -> {
                    System.out.println("map: " + n);
                    return n * 10;
                });

        System.out.println("Chưa có toán tử terminal, nên chưa chạy gì thêm.");
        
        // Toán tử terminal
        System.out.println("Bắt đầu terminal operation:");
        stream.forEach(System.out::println);
    }
}
```

**Input:**

```text
numbers = [1, 2, 3, 4, 5]
```

**Output (console):**

```text
Chưa có toán tử terminal, nên chưa chạy gì thêm.
Bắt đầu terminal operation:
filter: 1
map: 1
10
filter: 2
filter: 3
map: 3
30
filter: 4
filter: 5
map: 5
50
```

Lưu ý: `filter` và `map` chỉ thực sự chạy khi gọi `forEach` (toán tử terminal).

## 3. Toán tử kết thúc (Terminal Operations)

- **Kết thúc** pipeline stream.
- Thực sự **thực thi** toàn bộ chuỗi toán tử trung gian.
- Không trả về Stream nữa, mà trả về:
  - Giá trị (ví dụ: `int`, `Optional<T>`, …)
  - Collection
  - Hoặc không trả về gì (`void` với `forEach`)

Ví dụ các toán tử terminal phổ biến:

- `forEach` – duyệt từng phần tử
- `collect` – gom về `List`, `Set`, `Map`, …
- `reduce` – gộp thành một giá trị
- `count`, `min`, `max`, `anyMatch`, `allMatch`, `noneMatch`, `findFirst`, `findAny`, …

Ví dụ:

```java
import java.util.Arrays;
import java.util.List;
import java.util.stream.Collectors;

public class TerminalExample {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("An", "Bình", "Anh", "Bảo");

        // Lọc tên bắt đầu bằng 'A' và gom về List
        List<String> result = names.stream()
                .filter(name -> name.startsWith("A"))
                .collect(Collectors.toList());

        System.out.println("Kết quả: " + result);

        // Đếm số phần tử
        long count = names.stream()
                .filter(name -> name.length() == 3)
                .count();

        System.out.println("Số tên có 3 ký tự: " + count);
    }
}
```

**Input:**

```text
names = ["An", "Bình", "Anh", "Bảo"]
```

**Output:**

```text
Kết quả: [An, Anh]
Số tên có 3 ký tự: 1
```

## 4. Lazy Evaluation (Đánh giá lười)

- Các toán tử trung gian **không chạy ngay** khi được khai báo.
- Chỉ khi có **toán tử terminal**, toàn bộ pipeline mới được thực thi.
- Stream xử lý **từng phần tử theo chuỗi** (pipeline), không phải làm xong toàn bộ `filter` rồi mới `map`.

Ví dụ minh họa lazy + pipeline:

```java
import java.util.Arrays;
import java.util.List;

public class LazyEvaluationExample {
    public static void main(String[] args) {
        List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);

        int sum = numbers.stream()
                .peek(n -> System.out.println("peek: " + n)) // xem từng phần tử
                .filter(n -> {
                    System.out.println("filter: " + n);
                    return n % 2 == 0; // số chẵn
                })
                .map(n -> {
                    System.out.println("map: " + n);
                    return n * 2;
                })
                .reduce(0, Integer::sum); // terminal

        System.out.println("Tổng: " + sum);
    }
}
```

**Input:**

```text
numbers = [1, 2, 3, 4, 5]
```

**Output (rút gọn):**

```text
peek: 1
filter: 1
peek: 2
filter: 2
map: 2
peek: 3
filter: 3
peek: 4
filter: 4
map: 4
peek: 5
filter: 5
Tổng: 12
```

Ta thấy:
- Mỗi phần tử đi qua `peek -> filter -> map` rồi mới đến phần tử tiếp theo.
- Không có terminal (`reduce`) thì không có dòng nào được in.

---

## 5. Tóm tắt ý chính

- **Stream**: Cung cấp cách xử lý dữ liệu theo pipeline, không làm thay đổi nguồn.
- **Intermediate operations**:
  - Trả về Stream.
  - Lazy (chỉ “đăng ký” thao tác).
  - Ví dụ: `filter`, `map`, `sorted`, `distinct`, `limit`, `skip`, `peek`.
- **Terminal operations**:
  - Kết thúc pipeline, thực thi toàn bộ.
  - Không trả về Stream.
  - Ví dụ: `forEach`, `collect`, `reduce`, `count`, `min`, `max`, `findFirst`, …
- **Lazy evaluation**:
  - Toàn bộ chuỗi intermediate chỉ chạy khi có terminal.
  - Xử lý từng phần tử theo chuỗi, giúp tối ưu hiệu năng.
