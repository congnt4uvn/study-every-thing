
Dưới đây là tóm tắt ngắn gọn về nội dung Spring MVC/Web: Request/Response, DTO, Validation.

---

## 1. Request/Response Mapping (Ánh xạ request/response)

Trong Spring MVC, ta dùng `@Controller` hoặc `@RestController` và các annotation như `@GetMapping`, `@PostMapping` để ánh xạ URL đến method xử lý.

**Ví dụ:**

```java
@RestController
@RequestMapping("/api/users")
public class UserController {

    // GET /api/users/1
    @GetMapping("/{id}")
    public UserResponse getUser(@PathVariable Long id) {
        // Giả sử lấy user từ DB
        UserResponse user = new UserResponse(id, "Nguyen Van A", "a@example.com");
        return user; // Trả về JSON
    }

    // POST /api/users
    @PostMapping
    public UserResponse createUser(@RequestBody UserRequest request) {
        // Giả sử lưu user vào DB và trả về user đã tạo
        UserResponse user = new UserResponse(1L, request.getName(), request.getEmail());
        return user;
    }
}
```

**Ví dụ request/response (JSON):**

- Request (POST `/api/users`):

```json
{
  "name": "Nguyen Van A",
  "email": "a@example.com"
}
```

- Response:

```json
{
  "id": 1,
  "name": "Nguyen Van A",
  "email": "a@example.com"
}
```

## 2. DTO (Data Transfer Object)

DTO là các class dùng để truyền dữ liệu giữa client và server (request/response), tách biệt với entity trong database.

**Ví dụ DTO:**

```java
public class UserRequest {
    private String name;
    private String email;

    // getters/setters
}

public class UserResponse {
    private Long id;
    private String name;
    private String email;

    public UserResponse(Long id, String name, String email) {
        this.id = id;
        this.name = name;
        this.email = email;
    }

    // getters/setters
}
```

Mục đích:
- Không lộ trực tiếp entity DB ra ngoài.
- Dễ kiểm soát dữ liệu vào/ra (validation, format, ẩn field nhạy cảm).

## 3. Validation (@Valid, @NotNull, ...)

Spring hỗ trợ validate dữ liệu request bằng Bean Validation (JSR-380) với các annotation như:
- `@NotNull`, `@NotBlank`, `@Size`, `@Email`, `@Min`, `@Max`, …

Kết hợp với `@Valid` trong controller để tự động kiểm tra dữ liệu.

**Ví dụ DTO có validation:**

```java
import jakarta.validation.constraints.Email;
import jakarta.validation.constraints.NotBlank;
import jakarta.validation.constraints.Size;

public class UserRequest {

    @NotBlank(message = "Tên không được để trống")
    @Size(min = 2, max = 50, message = "Tên phải từ 2 đến 50 ký tự")
    private String name;

    @NotBlank(message = "Email không được để trống")
    @Email(message = "Email không hợp lệ")
    private String email;

    // getters/setters
}
```

**Controller dùng @Valid:**

```java
@RestController
@RequestMapping("/api/users")
public class UserController {

    @PostMapping
    public UserResponse createUser(@Valid @RequestBody UserRequest request) {
        // Nếu dữ liệu không hợp lệ, Spring sẽ tự động trả lỗi 400
        UserResponse user = new UserResponse(1L, request.getName(), request.getEmail());
        return user;
    }
}
```

**Ví dụ input sai và output lỗi:**

- Request (POST `/api/users`):

```json
{
  "name": "",
  "email": "khong-phai-email"
}
```

- Response (ví dụ):

```json
{
  "timestamp": "2026-05-20T10:00:00",
  "status": 400,
  "errors": [
    "Tên không được để trống",
    "Email không hợp lệ"
  ]
}
```

---

### Tóm tắt ý chính

1. **Request/Response mapping**: Dùng `@RestController`, `@GetMapping`, `@PostMapping`,… để ánh xạ URL → method, trả về JSON/XML.
2. **DTO**: Class chuyên dùng để nhận request và trả response, tách biệt với entity, giúp dễ quản lý dữ liệu và bảo mật.
3. **Validation**: Dùng `@Valid` trong controller + các annotation như `@NotNull`, `@NotBlank`, `@Email`,… trong DTO để tự động kiểm tra dữ liệu đầu vào và trả lỗi 400 nếu không hợp lệ.
