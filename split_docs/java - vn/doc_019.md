## 1. Pattern Matching & Records (hoàn thiện dần)

### a. Pattern Matching cho `switch` (Java 21/22+)

Giúp viết `switch` ngắn gọn, an toàn kiểu hơn.

```java
static String formatObject(Object obj) {
    return switch (obj) {
        case Integer i -> "Số nguyên: " + i;
        case String s  -> "Chuỗi: " + s.toUpperCase();
        case null      -> "Giá trị null";
        default        -> "Kiểu khác: " + obj.toString();
    };
}

public static void main(String[] args) {
    System.out.println(formatObject(10));        // Số nguyên: 10
    System.out.println(formatObject("hello"));   // Chuỗi: HELLO
    System.out.println(formatObject(null));      // Giá trị null
}
```

**Ý chính:**

- `switch` hiểu được kiểu (type) của biến.
- Không cần ép kiểu thủ công (`(String) obj`).
- Hỗ trợ `null` trong `switch`.

### b. Record Patterns (Java 21/22+)

Giúp “bóc tách” record trực tiếp trong pattern.

```java
record Point(int x, int y) {}

static String describe(Object o) {
    return switch (o) {
        case Point(int x, int y) when x == y -> "Điểm trên đường chéo: " + x;
        case Point(int x, int y)             -> "Điểm (" + x + ", " + y + ")";
        default                              -> "Không phải Point";
    };
}

public static void main(String[] args) {
    System.out.println(describe(new Point(5, 5)));  // Điểm trên đường chéo: 5
    System.out.println(describe(new Point(3, 7)));  // Điểm (3, 7)
}
```

**Ý chính:**

- Dễ dàng truy xuất trường của `record` trong `switch`.
- Có thể dùng `when` để thêm điều kiện.

## 2. Virtual Threads (Project Loom – ổn định từ Java 21, tiếp tục cải tiến 22–25)

Virtual thread giúp tạo hàng triệu thread nhẹ, đơn giản hóa lập trình đồng thời.

```java
public static void main(String[] args) throws Exception {
    try (var executor = java.util.concurrent.Executors.newVirtualThreadPerTaskExecutor()) {
        for (int i = 0; i < 5; i++) {
            int id = i;
            executor.submit(() -> {
                System.out.println("Task " + id + " chạy trên " + Thread.currentThread());
                Thread.sleep(1000);
                return null;
            });
        }
    }
}
```

**Output (ví dụ):**

```
Task 0 chạy trên VirtualThread[#23]/runnable
Task 1 chạy trên VirtualThread[#24]/runnable
Task 2 chạy trên VirtualThread[#25]/runnable
Task 3 chạy trên VirtualThread[#26]/runnable
Task 4 chạy trên VirtualThread[#27]/runnable
```

**Ý chính:**

- Tạo thread rẻ, số lượng rất lớn.
- Code vẫn dùng API `Thread`, `ExecutorService` quen thuộc.
- Phù hợp cho server, I/O-bound.

## 3. Structured Concurrency (dự kiến hoàn thiện dần 22–25)

Giúp quản lý nhiều tác vụ song song như một “khối” logic.

```java
import java.util.concurrent.*;
import java.util.concurrent.StructuredTaskScope;

String fetchUser() throws InterruptedException {
    Thread.sleep(500);
    return "User A";
}

String fetchOrders() throws InterruptedException {
    Thread.sleep(800);
    return "3 orders";
}

String getUserInfo() throws Exception {
    try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
        var userFuture   = scope.fork(this::fetchUser);
        var ordersFuture = scope.fork(this::fetchOrders);

        scope.join();          // chờ tất cả
        scope.throwIfFailed(); // ném lỗi nếu có

        return userFuture.resultNow() + " - " + ordersFuture.resultNow();
    }
}

public static void main(String[] args) throws Exception {
    System.out.println(new Main().getUserInfo());
    // Output: User A - 3 orders
}
```

**Ý chính:**

- Nhóm các task song song vào một scope.
- Dễ hủy toàn bộ khi một task lỗi.
- Giúp code đồng thời dễ đọc, dễ quản lý.

## 4. Foreign Function & Memory API (Project Panama – dần ổn định 22–25)

Thay thế `JNI` bằng API an toàn, dễ dùng để gọi code native (C/C++) và quản lý bộ nhớ ngoài heap.

Ví dụ đơn giản (mang tính minh họa, API có thể thay đổi):

```java
import java.lang.foreign.*;
import java.lang.invoke.MethodHandle;

public class PanamaExample {
    public static void main(String[] args) throws Throwable {
        Linker linker = Linker.nativeLinker();
        SymbolLookup stdlib = linker.defaultLookup();

        MethodHandle strlen = linker.downcallHandle(
                stdlib.find("strlen").orElseThrow(),
                FunctionDescriptor.of(ValueLayout.JAVA_LONG, ValueLayout.ADDRESS)
        );

        try (Arena arena = Arena.ofConfined()) {
            MemorySegment cString = arena.allocateUtf8String("Xin chào");
            long len = (long) strlen.invoke(cString);
            System.out.println("Độ dài chuỗi: " + len); // Độ dài chuỗi: 8 (tùy encoding)
        }
    }
}
```

**Ý chính:**

- Gọi hàm C trực tiếp từ Java.
- Quản lý bộ nhớ native an toàn hơn, tránh lỗi JNI phức tạp.
- Hiệu năng tốt cho các thư viện native (ML, đồ họa, xử lý tín hiệu…).

## 5. String Templates (dự kiến Java 22–23+)

Giúp nội suy chuỗi (string interpolation) an toàn, linh hoạt.

```java
import static java.lang.StringTemplate.STR;

public class TemplateExample {
    public static void main(String[] args) {
        int a = 5, b = 7;
        String s = STR."Tổng của \{a} + \{b} = \{a + b}";
        System.out.println(s); // Tổng của 5 + 7 = 12
    }
}
```

**Ý chính:**

- Viết chuỗi có biến chèn vào giống nhiều ngôn ngữ khác.
- Có thể kết hợp với “template processor” để chống SQL injection, XSS…

## 6. Vectơ API (Vector API – tiếp tục incubator/preview 22–25)

Hỗ trợ tính toán SIMD (song song trên vector) để tăng hiệu năng.

```java
import jdk.incubator.vector.*;

public class VectorSum {
    static final VectorSpecies<Float> SPECIES = FloatVector.SPECIES_PREFERRED;

    public static float sum(float[] a, float[] b) {
        int i = 0;
        int length = a.length;
        float[] c = new float[length];

        for (; i < SPECIES.loopBound(length); i += SPECIES.length()) {
            var va = FloatVector.fromArray(SPECIES, a, i);
            var vb = FloatVector.fromArray(SPECIES, b, i);
            var vc = va.add(vb);
            vc.intoArray(c, i);
        }
        // phần còn lại
        for (; i < length; i++) {
            c[i] = a[i] + b[i];
        }

        float sum = 0;
        for (float v : c) sum += v;
        return sum;
    }
}
```

**Ý chính:**

- Tận dụng SIMD của CPU để tăng tốc tính toán số học.
- API cấp cao, JVM tối ưu cho từng kiến trúc CPU.

## 7. Các cải tiến khác (22–25, xu hướng chung)

- **Hiệu năng JVM**: cải thiện JIT, GC (G1, ZGC, Shenandoah), giảm độ trễ.
- **Bảo mật & TLS**: cập nhật thuật toán, cấu hình mặc định an toàn hơn.
- **Ngôn ngữ**: tiếp tục hoàn thiện pattern matching, sealed types, records.
- **Tooling**: `jlink`, `jpackage`, profiling, observability tốt hơn.

### Tóm tắt ngắn

- Java 22–25 tập trung vào:
  - **Đơn giản hóa code**: pattern matching, record patterns, string templates.
  - **Đồng thời hiện đại**: virtual threads, structured concurrency.
  - **Hiệu năng & native**: Foreign Function & Memory API, Vector API, tối ưu JVM.
- Mục tiêu: viết code **ngắn hơn, an toàn hơn, hiệu năng cao hơn**, đặc biệt cho ứng dụng server, cloud, và tính toán hiệu năng cao.
