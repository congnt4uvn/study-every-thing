
Dưới đây là tóm tắt ngắn gọn về **Index trong SQL & RDBMS** (Clustered vs Non-Clustered), kèm ví dụ minh họa.

---

## 1. Khái niệm cơ bản

### Index là gì?
- Index là cấu trúc dữ liệu giúp **tăng tốc truy vấn** (SELECT) bằng cách cho phép DB tìm dữ liệu nhanh hơn, thay vì quét toàn bộ bảng.
- Đổi lại, **INSERT/UPDATE/DELETE sẽ chậm hơn** một chút vì phải cập nhật index.

---

## 2. Clustered Index

### Đặc điểm
- Quyết định **thứ tự vật lý** của dữ liệu trong bảng.
- Mỗi bảng **chỉ có 1 clustered index**.
- Thường được tạo trên **primary key**.
- Dữ liệu trong bảng được lưu theo thứ tự của clustered index.

### Khi nên dùng
- Cột thường dùng để:
  - Tìm kiếm theo **range** (BETWEEN, >, <, ORDER BY).
  - Là **khóa chính** (id, mã khách hàng, …).
- Bảng có nhiều truy vấn đọc theo thứ tự, phân trang, sắp xếp.

### Ví dụ (SQL Server / MySQL InnoDB)

```sql
-- Tạo bảng
CREATE TABLE Customers (
    CustomerID INT PRIMARY KEY,      -- Mặc định sẽ là clustered index (SQL Server)
    Name VARCHAR(100),
    City VARCHAR(50)
);

-- Hoặc tạo clustered index tường minh
CREATE CLUSTERED INDEX IX_Customers_CustomerID
ON Customers(CustomerID);
```

**Truy vấn sử dụng tốt clustered index:**

```sql
SELECT * 
FROM Customers
WHERE CustomerID BETWEEN 100 AND 200
ORDER BY CustomerID;
```

- DB có thể đọc dữ liệu liên tục theo thứ tự `CustomerID` → rất nhanh.

---

## 3. Non-Clustered Index

### Đặc điểm
- Không thay đổi thứ tự vật lý của dữ liệu.
- Là một cấu trúc riêng, chứa:
  - Giá trị của cột được index.
  - Con trỏ (pointer) đến dòng dữ liệu thật (clustered key hoặc row id).
- Một bảng có thể có **nhiều non-clustered index**.

### Khi nên dùng
- Cột thường xuất hiện trong:
  - `WHERE`
  - `JOIN`
  - `ORDER BY`
  - `GROUP BY`
- Các cột được truy vấn nhiều nhưng **không phải** là khóa chính.

### Ví dụ

```sql
-- Tạo non-clustered index trên cột City
CREATE NONCLUSTERED INDEX IX_Customers_City
ON Customers(City);
```

**Truy vấn hưởng lợi từ non-clustered index:**

```sql
SELECT CustomerID, Name, City
FROM Customers
WHERE City = 'Hanoi';
```

- DB dùng index `IX_Customers_City` để tìm nhanh tất cả khách hàng ở Hà Nội.

---

## 4. So sánh nhanh

| Tiêu chí                 | Clustered Index                          | Non-Clustered Index                         |
|--------------------------|------------------------------------------|---------------------------------------------|
| Số lượng trên 1 bảng    | Tối đa 1                                 | Nhiều                                       |
| Ảnh hưởng dữ liệu vật lý| Có – sắp xếp lại dữ liệu                 | Không – chỉ là cấu trúc riêng               |
| Thường dùng cho         | Primary key, cột tìm kiếm theo range     | Cột lọc, join, group, order                 |
| Tốc độ truy vấn         | Rất nhanh cho truy vấn theo key/range    | Nhanh cho điều kiện phù hợp với index       |
| Ảnh hưởng ghi (write)   | Tăng chi phí khi insert/update/delete    | Cũng tăng chi phí, càng nhiều index càng chậm|

## 5. Ví dụ đầy đủ: Input/Output

### Tạo bảng và index

```sql
CREATE TABLE Orders (
    OrderID INT PRIMARY KEY,          -- Clustered index
    CustomerID INT,
    OrderDate DATE,
    TotalAmount DECIMAL(10,2)
);

-- Non-clustered index trên CustomerID
CREATE NONCLUSTERED INDEX IX_Orders_CustomerID
ON Orders(CustomerID);
```

### Dữ liệu mẫu (Input)

```sql
INSERT INTO Orders (OrderID, CustomerID, OrderDate, TotalAmount) VALUES
(1,  101, '2024-01-01', 100.00),
(2,  102, '2024-01-02', 200.00),
(3,  101, '2024-01-03', 150.00),
(4,  103, '2024-01-04', 300.00);
```

### Truy vấn 1: Tìm theo OrderID (dùng clustered index)

```sql
SELECT * 
FROM Orders
WHERE OrderID = 3;
```

**Output (kết quả):**

| OrderID | CustomerID | OrderDate   | TotalAmount |
|--------:|-----------:|------------|------------:|
| 3       | 101        | 2024-01-03 | 150.00      |

---

### Truy vấn 2: Tìm theo CustomerID (dùng non-clustered index)

```sql
SELECT OrderID, OrderDate, TotalAmount
FROM Orders
WHERE CustomerID = 101;
```

**Output (kết quả):**

| OrderID | OrderDate   | TotalAmount |
|--------:|------------|------------:|
| 1       | 2024-01-01 | 100.00      |
| 3       | 2024-01-03 | 150.00      |

---

## 6. Tóm tắt ý chính

- **Index** giúp tăng tốc truy vấn đọc, nhưng làm chậm thao tác ghi.
- **Clustered index**:
  - Sắp xếp dữ liệu vật lý.
  - Mỗi bảng chỉ có 1.
  - Thường dùng cho primary key, truy vấn theo range.
- **Non-clustered index**:
  - Không sắp xếp dữ liệu vật lý.
  - Có thể có nhiều.
  - Dùng cho cột thường xuất hiện trong WHERE/JOIN/ORDER BY/GROUP BY.
- Cần **cân bằng** giữa số lượng index và hiệu năng ghi/đọc.

Nếu bạn muốn, tôi có thể thêm ví dụ cụ thể cho MySQL, SQL Server hoặc PostgreSQL riêng biệt.
