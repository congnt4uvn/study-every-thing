
Dưới đây là tóm tắt ngắn gọn về các khái niệm: vòng đời Thread, Runnable, Callable, Future trong Java, kèm ví dụ code và input/output.

---

## 1. Vòng đời (Lifecycle) của Thread

Một `Thread` trong Java thường đi qua các trạng thái chính:

1. **New**:  
   - Thread vừa được tạo bằng `new Thread(...)` nhưng chưa `start()`.
2. **Runnable**:  
   - Đã gọi `start()`, sẵn sàng chạy (hoặc đang chạy) do JVM và OS quản lý.
3. **Blocked / Waiting / Timed Waiting**:  
   - Tạm dừng chờ lock, chờ notify, hoặc chờ hết thời gian (vd: `sleep`, `wait`, `join`).
4. **Terminated (Dead)**:  
   - Kết thúc sau khi `run()` hoàn thành hoặc ném exception không bắt.

### Ví dụ đơn giản về Thread

```java
public class SimpleThread extends Thread {
    @Override
    public void run() {
        System.out.println("Thread đang chạy: " + Thread.currentThread().getName());
    }

    public static void main(String[] args) {
        Thread t = new SimpleThread(); // New
        t.start();                     // Runnable -> Running -> Terminated
    }
}
```

**Output (ví dụ):**
```text
Thread đang chạy: Thread-0
```

## 2. Runnable

- `Runnable` là một interface chức năng với 1 phương thức: `void run()`.
- Không trả về kết quả, không ném checked exception.
- Dùng với `Thread` hoặc `ExecutorService`.

### Ví dụ Runnable

```java
public class RunnableExample {
    public static void main(String[] args) {
        Runnable task = () -> {
            System.out.println("Chạy trong Runnable: " + Thread.currentThread().getName());
        };

        Thread t = new Thread(task);
        t.start();
    }
}
```

**Output (ví dụ):**
```text
Chạy trong Runnable: Thread-0
```

## 3. Callable

- `Callable<V>` giống `Runnable` nhưng:
  - Có kiểu trả về `V`.
  - Có thể ném checked exception.
- Thường dùng với `ExecutorService` để nhận kết quả thông qua `Future`.

### Ví dụ Callable

```java
import java.util.concurrent.*;

public class CallableExample {
    public static void main(String[] args) throws Exception {
        Callable<Integer> task = () -> {
            // Giả sử tính tổng từ 1 đến 5
            int sum = 0;
            for (int i = 1; i <= 5; i++) {
                sum += i;
            }
            return sum; // trả về 15
        };

        ExecutorService executor = Executors.newSingleThreadExecutor();
        Future<Integer> future = executor.submit(task);

        // Lấy kết quả (có thể block)
        Integer result = future.get();
        System.out.println("Kết quả: " + result);

        executor.shutdown();
    }
}
```

**Input logic:** tính tổng 1 + 2 + 3 + 4 + 5  
**Output:**
```text
Kết quả: 15
```

## 4. Future

- `Future<V>` đại diện cho kết quả của một tác vụ bất đồng bộ (asynchronous).
- Được trả về khi submit `Callable` (hoặc `Runnable`) cho `ExecutorService`.
- Các phương thức chính:
  - `get()` – chờ và lấy kết quả.
  - `get(timeout, unit)` – chờ tối đa một khoảng thời gian.
  - `isDone()` – kiểm tra đã xong chưa.
  - `cancel(boolean mayInterruptIfRunning)` – hủy tác vụ.

### Ví dụ Future với kiểm tra isDone

```java
import java.util.concurrent.*;

public class FutureExample {
    public static void main(String[] args) throws Exception {
        Callable<String> task = () -> {
            Thread.sleep(1000); // giả lập tác vụ tốn 1 giây
            return "Hoàn thành";
        };

        ExecutorService executor = Executors.newSingleThreadExecutor();
        Future<String> future = executor.submit(task);

        System.out.println("Tác vụ đã xong chưa? " + future.isDone());

        String result = future.get(); // chờ đến khi xong
        System.out.println("Kết quả: " + result);
        System.out.println("Tác vụ đã xong chưa? " + future.isDone());

        executor.shutdown();
    }
}
```

**Output (ví dụ):**
```text
Tác vụ đã xong chưa? false
Kết quả: Hoàn thành
Tác vụ đã xong chưa? true
```

---

## Tóm tắt ý chính

- **Thread lifecycle**: New → Runnable → (Blocked/Waiting) → Terminated.
- **Runnable**: không trả về kết quả, không ném checked exception, dùng cho tác vụ đơn giản.
- **Callable**: có giá trị trả về, có thể ném checked exception, dùng cho tác vụ cần kết quả.
- **Future**: đại diện cho kết quả tương lai của một tác vụ bất đồng bộ, cho phép lấy kết quả, kiểm tra trạng thái, hoặc hủy tác vụ.
