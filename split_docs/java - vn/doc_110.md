---

## 1. Khái niệm chính

1. **Container**  
   - Là môi trường chạy ứng dụng nhẹ, độc lập, chia sẻ kernel của hệ điều hành.  
   - Đóng gói: code + thư viện + cấu hình → chạy đồng nhất trên mọi máy.

2. **Image**  
   - Là “khuôn mẫu” (template) chỉ đọc để tạo ra container.  
   - Gồm nhiều lớp (layers) được build từ Dockerfile.  
   - Một image có thể tạo nhiều container.

3. **Dockerfile**  
   - Là file cấu hình (text) mô tả cách build image.  
   - Gồm các lệnh như: `FROM`, `COPY`, `RUN`, `CMD`, `EXPOSE`, v.v.

---

## 2. Luồng làm việc cơ bản với Docker

1. Viết **Dockerfile**  
2. Dùng `docker build` để tạo **image**  
3. Dùng `docker run` để tạo và chạy **container** từ image

---

## 3. Ví dụ: Ứng dụng Java đơn giản với Docker

Giả sử ta có một ứng dụng Java (Spring Boot hoặc Java main) đóng gói thành file JAR.

### 3.1. Cấu trúc thư mục

```text
my-java-app/
├─ Dockerfile
└─ target/
   └─ my-app-1.0.0.jar
```

### 3.2. Dockerfile ví dụ

```dockerfile
# 1. Chọn image nền có sẵn JDK
FROM openjdk:17-jdk-slim

# 2. Thiết lập thư mục làm việc trong container
WORKDIR /app

# 3. Copy file JAR từ máy host vào container
COPY target/my-app-1.0.0.jar app.jar

# 4. Expose port (ví dụ app chạy port 8080)
EXPOSE 8080

# 5. Lệnh mặc định khi container chạy
CMD ["java", "-jar", "app.jar"]
```

### 3.3. Build image

```bash
# Cú pháp
docker build -t <ten-image>:<tag> .

# Ví dụ
docker build -t my-java-app:1.0 .
```

**Input:** Dockerfile + source đã build thành JAR  
**Output:** Image mới tên `my-java-app:1.0`

Kiểm tra:

```bash
docker images
```

---

### 3.4. Chạy container

```bash
# Cú pháp
docker run [options] <image> [command]

# Ví dụ: map port 8080 của host sang 8080 của container
docker run -d -p 8080:8080 --name my-java-container my-java-app:1.0
```

**Input:** Image `my-java-app:1.0`  
**Output:** Container `my-java-container` đang chạy, truy cập app tại `http://localhost:8080`

Xem log:

```bash
docker logs my-java-container
```

Dừng container:

```bash
docker stop my-java-container
```

Xóa container:

```bash
docker rm my-java-container
```

---

## 4. Một số lệnh Docker cơ bản

```bash
# Liệt kê container đang chạy
docker ps

# Liệt kê tất cả container (kể cả đã dừng)
docker ps -a

# Liệt kê images
docker images

# Xóa image
docker rmi my-java-app:1.0
```

---

## 5. Tóm tắt ý chính

- **Container**: môi trường chạy ứng dụng nhẹ, nhất quán, dễ deploy.  
- **Image**: mẫu chỉ đọc để tạo container, được build từ Dockerfile.  
- **Dockerfile**: file mô tả cách build image (chọn base image, copy file, cài đặt, lệnh chạy).  
- Quy trình:  
  1) Viết Dockerfile  
  2) `docker build` → tạo image  
  3) `docker run` → tạo & chạy container từ image.
