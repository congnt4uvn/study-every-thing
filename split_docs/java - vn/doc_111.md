
Dưới đây là tóm tắt ngắn gọn về Kubernetes (ở mức cơ bản) bằng tiếng Việt, kèm ví dụ code và input/output.

---

## 1. Kubernetes là gì?

Kubernetes là hệ thống **orchestration** dùng để:
- Triển khai (deploy) ứng dụng container (thường là Docker)
- Tự động scale (tăng/giảm số lượng instance)
- Tự phục hồi (self-healing) khi container bị lỗi
- Cung cấp cơ chế networking, load balancing, cấu hình, secret,…

Ba khái niệm cơ bản: **Pod**, **Deployment**, **Service**.

---

## 2. Pod

- **Đơn vị nhỏ nhất** trong Kubernetes.
- Chứa **1 hoặc nhiều container** chạy chung network, storage.
- Thường bạn không tạo Pod trực tiếp trong môi trường production, mà dùng Deployment để quản lý.

### Ví dụ Pod (YAML)

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: hello-pod
spec:
  containers:
    - name: hello-container
      image: nginx:1.25
      ports:
        - containerPort: 80
```

**Input (lệnh tạo Pod):**
```bash
kubectl apply -f hello-pod.yaml
```

**Output (kiểm tra Pod):**
```bash
kubectl get pods
```

Ví dụ output:
```text
NAME         READY   STATUS    RESTARTS   AGE
hello-pod    1/1     Running   0          10s
```

## 3. Deployment

- Quản lý **nhiều Pod giống nhau** (replicas).
- Đảm bảo số lượng Pod luôn như mong muốn (self-healing).
- Hỗ trợ **rolling update** (cập nhật phiên bản không downtime).

### Ví dụ Deployment (YAML)

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: hello-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: hello-app
  template:
    metadata:
      labels:
        app: hello-app
    spec:
      containers:
        - name: hello-container
          image: nginx:1.25
          ports:
            - containerPort: 80
```

**Input (tạo Deployment):**
```bash
kubectl apply -f hello-deployment.yaml
```

**Output (kiểm tra Deployment & Pod):**
```bash
kubectl get deployments
kubectl get pods -l app=hello-app
```

Ví dụ output:
```text
NAME               READY   UP-TO-DATE   AVAILABLE   AGE
hello-deployment   3/3     3            3           20s

NAME                                READY   STATUS    RESTARTS   AGE
hello-deployment-7c8b9c7d4b-abcde   1/1     Running   0          20s
hello-deployment-7c8b9c7d4b-fghij   1/1     Running   0          20s
hello-deployment-7c8b9c7d4b-klmno   1/1     Running   0          20s
```

## 4. Service

- Dùng để **expose** (mở) Pod/Deployment ra:
  - Cho các Pod khác trong cluster
  - Hoặc cho bên ngoài cluster (tùy loại Service)
- Cung cấp **IP ổn định** và **load balancing** giữa các Pod.

Các loại phổ biến:
- `ClusterIP` (mặc định): chỉ truy cập **trong cluster**
- `NodePort`: mở port trên mỗi node, truy cập từ bên ngoài
- `LoadBalancer`: dùng load balancer của cloud (AWS, GCP, Azure,…)

### Ví dụ Service (ClusterIP)

```yaml
apiVersion: v1
kind: Service
metadata:
  name: hello-service
spec:
  selector:
    app: hello-app
  ports:
    - protocol: TCP
      port: 80        # port của Service
      targetPort: 80  # port trong container
  type: ClusterIP
```

**Input (tạo Service):**
```bash
kubectl apply -f hello-service.yaml
```

**Output (kiểm tra Service):**
```bash
kubectl get service hello-service
```

Ví dụ output:
```text
NAME            TYPE        CLUSTER-IP      EXTERNAL-IP   PORT(S)   AGE
hello-service   ClusterIP   10.96.123.45    <none>        80/TCP    10s
```

Truy cập từ một Pod khác trong cluster:
```bash
curl http://hello-service
```

---

## 5. Luồng tổng quát

1. Bạn viết file YAML cho **Deployment** (quy định image, số replicas, port…).
2. Kubernetes tạo và quản lý các **Pod** tương ứng.
3. Bạn tạo **Service** để:
   - Cố định endpoint (DNS/IP) cho các Pod
   - Cân bằng tải giữa các Pod
4. Người dùng/ứng dụng khác gọi vào Service, Service sẽ chuyển tiếp request đến các Pod.
