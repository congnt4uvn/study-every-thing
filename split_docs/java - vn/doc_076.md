
Dưới đây là tóm tắt ngắn gọn về **Spring Security – Bảo mật ở mức method** với `@PreAuthorize` và `@Secured`, kèm ví dụ code và input/output.

---

## 1. Khái niệm chung

- **Bảo mật mức method**: kiểm soát quyền truy cập ngay trên các method của service/controller.
- Hai cách phổ biến:
  - `@PreAuthorize` (linh hoạt, dùng SpEL – Spring Expression Language)
  - `@Secured` (đơn giản, chỉ kiểm tra role)

Để bật bảo mật method, thường dùng (Spring Security 6+):

```java
@Configuration
@EnableMethodSecurity(prePostEnabled = true, securedEnabled = true)
public class SecurityConfig {
}
```

---

## 2. @PreAuthorize

### Đặc điểm

- Được đánh trên method (service, controller).
- Dùng biểu thức SpEL để kiểm tra:
  - Role: `hasRole('ADMIN')`, `hasAnyRole('ADMIN', 'USER')`
  - Authority: `hasAuthority('SCOPE_read')`
  - Tham số method: `#id`, `#user`, …
  - Thông tin user hiện tại: `authentication`, `principal`, `#username == authentication.name`, …

### Ví dụ 1: Kiểm tra role

```java
@Service
public class UserService {

    // Chỉ ADMIN mới được gọi
    @PreAuthorize("hasRole('ADMIN')")
    public String deleteUser(Long id) {
        // logic xóa user
        return "User " + id + " deleted";
    }
}
```

**Input (giả lập)**  
- User hiện tại có role: `ROLE_ADMIN`  
- Gọi: `userService.deleteUser(10L)`

**Output**  
- Trả về: `"User 10 deleted"`

**Nếu user không có ROLE_ADMIN**  
- Ném exception: `AccessDeniedException` (HTTP 403 nếu là REST API)

---

### Ví dụ 2: Dùng tham số method

```java
@Service
public class DocumentService {

    // Chỉ cho phép user truy cập tài liệu của chính mình hoặc ADMIN
    @PreAuthorize("#owner == authentication.name or hasRole('ADMIN')")
    public String getDocument(String owner, Long docId) {
        return "Document " + docId + " of " + owner;
    }
}
```

**Input**  
- User hiện tại: `username = "alice"`, role: `ROLE_USER`
- Gọi: `documentService.getDocument("alice", 5L)`

**Output**  
- `"Document 5 of alice"` (được phép)

**Input khác**  
- User hiện tại: `username = "bob"`, role: `ROLE_USER`
- Gọi: `documentService.getDocument("alice", 5L)`

**Output**  
- `AccessDeniedException` (vì `bob` ≠ `alice` và không có ROLE_ADMIN)

---

## 3. @Secured

### Đặc điểm

- Đơn giản hơn, chỉ kiểm tra role (không dùng SpEL).
- Giá trị là mảng String, ví dụ: `"ROLE_ADMIN"`, `"ROLE_USER"`.

### Ví dụ

```java
@Service
public class OrderService {

    // Chỉ ADMIN mới được gọi
    @Secured("ROLE_ADMIN")
    public String approveOrder(Long orderId) {
        return "Order " + orderId + " approved";
    }

    // ADMIN hoặc MANAGER đều được
    @Secured({"ROLE_ADMIN", "ROLE_MANAGER"})
    public String viewAllOrders() {
        return "All orders";
    }
}
```

**Input**  
- User hiện tại: role: `ROLE_ADMIN`  
- Gọi: `orderService.approveOrder(100L)`

**Output**  
- `"Order 100 approved"`

**Nếu user chỉ có ROLE_USER**  
- Ném `AccessDeniedException` (HTTP 403)

---

## 4. So sánh nhanh

| Annotation     | Ưu điểm                              | Hạn chế                          |
|----------------|--------------------------------------|----------------------------------|
| `@PreAuthorize`| Linh hoạt, dùng SpEL, truy cập tham số, principal | Cú pháp phức tạp hơn            |
| `@Secured`     | Đơn giản, dễ đọc                    | Chỉ kiểm tra role, ít linh hoạt |

---

Nếu bạn muốn, tôi có thể thêm ví dụ đầy đủ với controller + cấu hình security + request HTTP (input/output) để bạn chạy thử.
