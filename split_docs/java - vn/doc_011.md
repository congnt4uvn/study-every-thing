
Dưới đây là tóm tắt ngắn gọn về **switch expressions** trong Java (các phiên bản mới, từ Java 14 trở đi):

---

## 1. switch cũ vs switch mới

### switch cũ (statement)
- Dùng `switch` như một **câu lệnh** (statement), không trả về giá trị.
- Cần `break` để tránh “rơi xuống” (fall-through).
- Dễ lỗi nếu quên `break`.

```java
int day = 3;
String dayName;

switch (day) {
    case 1:
        dayName = "Monday";
        break;
    case 2:
        dayName = "Tuesday";
        break;
    case 3:
        dayName = "Wednesday";
        break;
    default:
        dayName = "Unknown";
}

System.out.println(dayName);
// Output: Wednesday
```

---

## 2. switch expression (mới)

### Đặc điểm chính
- `switch` giờ có thể dùng như **biểu thức** (expression) → trả về giá trị.
- Dùng toán tử `->` thay cho `:` + `break`.
- Hạn chế fall-through, code ngắn gọn, an toàn hơn.
- Có thể gán trực tiếp vào biến hoặc dùng trong `return`.

### Ví dụ cơ bản

```java
int day = 3;

String dayName = switch (day) {
    case 1 -> "Monday";
    case 2 -> "Tuesday";
    case 3 -> "Wednesday";
    case 4 -> "Thursday";
    case 5 -> "Friday";
    case 6 -> "Saturday";
    case 7 -> "Sunday";
    default -> "Unknown";
};

System.out.println(dayName);
// Output: Wednesday
```

---

## 3. Gộp nhiều case

```java
int day = 6;

String type = switch (day) {
    case 1, 2, 3, 4, 5 -> "Weekday";
    case 6, 7 -> "Weekend";
    default -> "Invalid";
};

System.out.println(type);
// Output: Weekend
```

---

## 4. Khối lệnh phức tạp với `yield`

Khi logic trong mỗi nhánh phức tạp hơn, dùng `{}` và `yield` để trả về giá trị:

```java
int score = 85;

String grade = switch (score / 10) {
    case 10, 9 -> "A";
    case 8 -> {
        String msg = "B (Good)";
        yield msg;  // trả về giá trị cho switch expression
    }
    case 7 -> "C";
    case 6 -> "D";
    default -> "F";
};

System.out.println(grade);
// Output: B (Good)
```

---

## 5. Dùng trong `return` hoặc biểu thức khác

```java
public static String getDayType(int day) {
    return switch (day) {
        case 1, 2, 3, 4, 5 -> "Weekday";
        case 6, 7 -> "Weekend";
        default -> "Invalid";
    };
}

public static void main(String[] args) {
    System.out.println(getDayType(2)); // Output: Weekday
    System.out.println(getDayType(7)); // Output: Weekend
    System.out.println(getDayType(9)); // Output: Invalid
}
```

---

## 6. Lợi ích chính

- Code ngắn gọn, dễ đọc.
- Tránh lỗi quên `break`.
- Dùng được như biểu thức → gán biến, return trực tiếp.
- Hỗ trợ gộp nhiều case, khối lệnh phức tạp với `yield`.

Nếu bạn gửi thêm nội dung chi tiết hơn (ví dụ: đoạn tài liệu gốc), mình có thể tóm tắt sát hơn và bổ sung thêm ví dụ nâng cao.
