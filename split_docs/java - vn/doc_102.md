## 1. Load Balancing (Cân bằng tải)

**Ý chính:**
- Phân phối lưu lượng (request) từ client đến nhiều server backend.
- Mục tiêu: tăng **tính sẵn sàng**, **hiệu năng**, **khả năng mở rộng**, và **độ tin cậy**.
- Load balancer thường đứng giữa client và nhiều instance server.

**Các chiến lược phổ biến:**
1. **Round Robin**: lần lượt gửi request đến từng server theo vòng tròn.
2. **Least Connections**: gửi request đến server đang có ít kết nối nhất.
3. **IP Hash / Consistent Hashing**: dựa trên IP hoặc key để luôn gửi cùng client đến cùng server (hữu ích cho session sticky).
4. **Random**: chọn server ngẫu nhiên.

### Ví dụ đơn giản: Round Robin Load Balancer (Java)

```java
import java.util.List;
import java.util.concurrent.atomic.AtomicInteger;

class Server {
    private final String name;

    public Server(String name) {
        this.name = name;
    }

    public String handleRequest(String request) {
        return "Server " + name + " processed: " + request;
    }
}

class RoundRobinLoadBalancer {
    private final List<Server> servers;
    private final AtomicInteger index = new AtomicInteger(0);

    public RoundRobinLoadBalancer(List<Server> servers) {
        this.servers = servers;
    }

    public Server getNextServer() {
        int i = Math.abs(index.getAndIncrement() % servers.size());
        return servers.get(i);
    }

    public String dispatch(String request) {
        Server server = getNextServer();
        return server.handleRequest(request);
    }
}

public class Demo {
    public static void main(String[] args) {
        List<Server> servers = List.of(
                new Server("A"),
                new Server("B"),
                new Server("C")
        );

        RoundRobinLoadBalancer lb = new RoundRobinLoadBalancer(servers);

        // Input: 5 request liên tiếp
        for (int i = 1; i <= 5; i++) {
            String req = "Request-" + i;
            String res = lb.dispatch(req);
            System.out.println("Input: " + req + " -> Output: " + res);
        }
    }
}
```

**Output (ví dụ):**
```
Input: Request-1 -> Output: Server A processed: Request-1
Input: Request-2 -> Output: Server B processed: Request-2
Input: Request-3 -> Output: Server C processed: Request-3
Input: Request-4 -> Output: Server A processed: Request-4
Input: Request-5 -> Output: Server B processed: Request-5
```

## 2. Vertical Scaling vs Horizontal Scaling

### 2.1 Vertical Scaling (Scale Up)

**Ý chính:**
- Tăng sức mạnh **một máy**: thêm CPU, RAM, SSD nhanh hơn, v.v.
- Ưu điểm:
  - Đơn giản: ít thay đổi kiến trúc.
  - Không cần phân tán dữ liệu phức tạp.
- Nhược điểm:
  - Có **giới hạn phần cứng**.
  - Single point of failure (nếu máy đó chết, hệ thống chết).
  - Chi phí phần cứng cao khi lên cấu hình rất mạnh.

**Ví dụ:**
- Từ 4 CPU, 8GB RAM → nâng lên 16 CPU, 64GB RAM cho 1 server chạy Java app.

### 2.2 Horizontal Scaling (Scale Out)

**Ý chính:**
- Thêm **nhiều máy** (nhiều instance) vào hệ thống.
- Thường kết hợp với **load balancer**.
- Ưu điểm:
  - Khả năng mở rộng gần như không giới hạn (thêm máy).
  - Tăng tính sẵn sàng (nhiều node, 1 node chết vẫn chạy).
- Nhược điểm:
  - Kiến trúc phức tạp hơn: đồng bộ dữ liệu, session, cache, v.v.
  - Cần cơ chế phân phối tải, discovery, v.v.

**Ví dụ kiến trúc:**

Client → Load Balancer → [App Server 1, App Server 2, App Server 3] → Database

## 3. Ví dụ Java: Mô phỏng Horizontal Scaling với nhiều instance

Giả sử ta có nhiều instance của cùng một service, load balancer sẽ phân phối request.

```java
import java.util.List;
import java.util.concurrent.atomic.AtomicInteger;

class AppInstance {
    private final String id;

    public AppInstance(String id) {
        this.id = id;
    }

    public String process(String userId) {
        // Giả sử xử lý logic cho userId
        return "Instance " + id + " handled user " + userId;
    }
}

class SimpleLoadBalancer {
    private final List<AppInstance> instances;
    private final AtomicInteger index = new AtomicInteger(0);

    public SimpleLoadBalancer(List<AppInstance> instances) {
        this.instances = instances;
    }

    public String handleRequest(String userId) {
        int i = Math.abs(index.getAndIncrement() % instances.size());
        return instances.get(i).process(userId);
    }
}

public class HorizontalScalingDemo {
    public static void main(String[] args) {
        List<AppInstance> instances = List.of(
                new AppInstance("1"),
                new AppInstance("2"),
                new AppInstance("3")
        );

        SimpleLoadBalancer lb = new SimpleLoadBalancer(instances);

        // Input: nhiều userId
        String[] userIds = {"u1", "u2", "u3", "u4", "u5", "u6"};
        for (String userId : userIds) {
            String result = lb.handleRequest(userId);
            System.out.println("Input: " + userId + " -> Output: " + result);
        }
    }
}
```

**Output (ví dụ):**
```
Input: u1 -> Output: Instance 1 handled user u1
Input: u2 -> Output: Instance 2 handled user u2
Input: u3 -> Output: Instance 3 handled user u3
Input: u4 -> Output: Instance 1 handled user u4
Input: u5 -> Output: Instance 2 handled user u5
Input: u6 -> Output: Instance 3 handled user u6
```

## 4. Tóm tắt ngắn gọn

- **Load Balancing**: phân phối request đến nhiều server để tăng hiệu năng, độ tin cậy, và khả năng mở rộng. Có nhiều thuật toán: Round Robin, Least Connections, IP Hash, v.v.
- **Vertical Scaling (Scale Up)**: tăng tài nguyên cho 1 máy (CPU, RAM). Đơn giản nhưng có giới hạn và rủi ro single point of failure.
- **Horizontal Scaling (Scale Out)**: thêm nhiều máy/instance. Khả năng mở rộng tốt, độ sẵn sàng cao, nhưng kiến trúc phức tạp hơn, thường cần load balancer.
