
Here’s a concise summary of Spring MVC exception handling with `@ControllerAdvice` and `@ExceptionHandler`, plus code examples.

---

## Key Ideas

1. **`@ExceptionHandler`**  
   - Placed on a method in a controller (or in a `@ControllerAdvice` class).  
   - Handles specific exception types thrown in controller methods.  
   - Can return a view name, a `ResponseEntity`, or any object (for REST, usually JSON).

2. **`@ControllerAdvice`**  
   - A global component that applies to multiple controllers.  
   - Centralizes exception handling logic instead of duplicating it in each controller.  
   - Methods inside it can use `@ExceptionHandler` to handle exceptions across the whole application.

3. **Flow**  
   - A controller method throws an exception.  
   - Spring looks for a matching `@ExceptionHandler` method:  
     - First in the same controller.  
     - Then in any `@ControllerAdvice` classes.  
   - The handler method builds the HTTP response (status, body, view, etc.).

---

## Basic Example: Local `@ExceptionHandler` in a Controller

```java
// Custom exception
public class UserNotFoundException extends RuntimeException {
    public UserNotFoundException(String message) {
        super(message);
    }
}
```

```java
@Controller
@RequestMapping("/users")
public class UserController {

    @GetMapping("/{id}")
    @ResponseBody
    public String getUser(@PathVariable("id") Long id) {
        if (id == 1L) {
            return "User: Alice";
        } else {
            throw new UserNotFoundException("User with id " + id + " not found");
        }
    }

    @ExceptionHandler(UserNotFoundException.class)
    @ResponseBody
    public ResponseEntity<String> handleUserNotFound(UserNotFoundException ex) {
        return ResponseEntity
                .status(HttpStatus.NOT_FOUND)
                .body(ex.getMessage());
    }
}
```

### Example Input / Output

- **Request**  
  `GET /users/1`  
  **Response**  
  `200 OK`  
  Body:  
  `User: Alice`

- **Request**  
  `GET /users/2`  
  (throws `UserNotFoundException`)  
  **Response**  
  `404 Not Found`  
  Body:  
  `User with id 2 not found`

## Global Exception Handling with `@ControllerAdvice`

```java
@ControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(UserNotFoundException.class)
    @ResponseBody
    public ResponseEntity<Map<String, Object>> handleUserNotFound(UserNotFoundException ex) {
        Map<String, Object> body = new HashMap<>();
        body.put("error", "User Not Found");
        body.put("message", ex.getMessage());
        body.put("timestamp", Instant.now().toString());

        return ResponseEntity.status(HttpStatus.NOT_FOUND).body(body);
    }

    @ExceptionHandler(Exception.class)
    @ResponseBody
    public ResponseEntity<Map<String, Object>> handleGenericException(Exception ex) {
        Map<String, Object> body = new HashMap<>();
        body.put("error", "Internal Server Error");
        body.put("message", ex.getMessage());

        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body(body);
    }
}
```

### Example Input / Output (Global Handler)

- **Request**  
  `GET /users/999`  
  (controller throws `UserNotFoundException`)  

  **Response**  
  `404 Not Found`  
  Body (JSON-like):  
  ```json
  {
    "error": "User Not Found",
    "message": "User with id 999 not found",
    "timestamp": "2026-05-20T12:34:56Z"
  }
  ```

- **Request**  
  `GET /some-endpoint`  
  (controller throws a generic `RuntimeException`)  

  **Response**  
  `500 Internal Server Error`  
  Body:  
  ```json
  {
    "error": "Internal Server Error",
    "message": "Some unexpected error"
  }
  ```

---

## Summary

- Use `@ExceptionHandler` to map exceptions to custom responses.
- Use `@ControllerAdvice` to centralize and reuse exception handling across controllers.
- Handlers can return different response types (views, JSON, `ResponseEntity`) and set HTTP status codes.
