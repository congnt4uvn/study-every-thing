
Here’s a concise summary of the key concurrent collections in `java.util.concurrent`, with simple code examples and sample input/output.

---

## 1. `ConcurrentHashMap<K,V>`

**Main idea:**  
A thread-safe, high-performance map for concurrent read/write access without locking the whole map.

**Key points:**
- Multiple threads can read and write concurrently.
- Better scalability than `Collections.synchronizedMap`.
- Iterators are weakly consistent (may not see all updates, but never throw `ConcurrentModificationException`).

**Example:**

```java
import java.util.concurrent.ConcurrentHashMap;

public class ConcurrentHashMapExample {
    public static void main(String[] args) {
        ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

        // Put values
        map.put("apple", 1);
        map.put("banana", 2);

        // Atomic update
        map.compute("apple", (k, v) -> v == null ? 1 : v + 1);

        // Get value
        System.out.println("apple = " + map.get("apple"));
        System.out.println("banana = " + map.get("banana"));

        // Iterate (safe under concurrent modification)
        map.forEach((k, v) -> System.out.println(k + " -> " + v));
    }
}
```

**Sample output (order may vary):**
```
apple = 2
banana = 2
apple -> 2
banana -> 2
```

## 2. `CopyOnWriteArrayList<E>`

**Main idea:**  
A thread-safe list where every write (add/remove) creates a new copy of the underlying array.

**Key points:**
- Very efficient for **many reads, few writes**.
- Iterators never throw `ConcurrentModificationException` and see a snapshot of the list at creation time.
- Writes are expensive (copying the array).

**Example:**

```java
import java.util.List;
import java.util.concurrent.CopyOnWriteArrayList;

public class CopyOnWriteArrayListExample {
    public static void main(String[] args) {
        List<String> list = new CopyOnWriteArrayList<>();
        list.add("A");
        list.add("B");
        list.add("C");

        // Snapshot iterator
        for (String s : list) {
            System.out.println("Iterating: " + s);
            if (s.equals("B")) {
                list.add("D"); // Safe: iterator sees old snapshot
            }
        }

        System.out.println("Final list: " + list);
    }
}
```

**Sample output:**
```
Iterating: A
Iterating: B
Iterating: C
Final list: [A, B, C, D]
```

## 3. `BlockingQueue<E>`

**Main idea:**  
A thread-safe queue that blocks on `put` when full and on `take` when empty. Used for producer–consumer patterns.

**Common implementations:**
- `ArrayBlockingQueue<E>`:  
  - Bounded, backed by an array, fixed capacity.
  - Good when you want strict upper bound on queue size.
- `LinkedBlockingQueue<E>`:  
  - Usually optionally bounded (can be unbounded or have a large capacity).
  - Backed by linked nodes, often better for high concurrency.

### 3.1 `ArrayBlockingQueue<E>`

**Example (producer–consumer):**

```java
import java.util.concurrent.ArrayBlockingQueue;
import java.util.concurrent.BlockingQueue;

public class ArrayBlockingQueueExample {
    public static void main(String[] args) throws InterruptedException {
        BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(2);

        // Producer
        new Thread(() -> {
            try {
                for (int i = 1; i <= 3; i++) {
                    System.out.println("Producing: " + i);
                    queue.put(i); // blocks if queue is full
                    Thread.sleep(100);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }).start();

        // Consumer
        new Thread(() -> {
            try {
                for (int i = 1; i <= 3; i++) {
                    Integer value = queue.take(); // blocks if queue is empty
                    System.out.println("Consuming: " + value);
                    Thread.sleep(200);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }).start();
    }
}
```

**Sample output (timing-dependent, order similar to):**
```
Producing: 1
Consuming: 1
Producing: 2
Producing: 3   // may block until consumer takes
Consuming: 2
Consuming: 3
```

### 3.2 `LinkedBlockingQueue<E>`

**Example:**

```java
import java.util.concurrent.BlockingQueue;
import java.util.concurrent.LinkedBlockingQueue;

public class LinkedBlockingQueueExample {
    public static void main(String[] args) throws InterruptedException {
        BlockingQueue<String> queue = new LinkedBlockingQueue<>(); // unbounded by default

        queue.put("task1");
        queue.put("task2");

        System.out.println(queue.take()); // "task1"
        System.out.println(queue.take()); // "task2"
    }
}
```

**Sample output:**
```
task1
task2
```

---

**Summary:**
- `ConcurrentHashMap`: high-performance concurrent map, fine-grained locking, weakly consistent iterators.
- `CopyOnWriteArrayList`: best for many reads and few writes; snapshot iterators.
- `BlockingQueue` (`ArrayBlockingQueue`, `LinkedBlockingQueue`): blocking operations for producer–consumer patterns, with bounded/unbounded options.
