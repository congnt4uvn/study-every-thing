
**Summary: Contract Testing (e.g., with Microservices)**

**Main Idea:**  
Contract testing verifies that two services (usually a client and a provider microservice) can communicate correctly by checking that both sides follow a shared “contract” (API expectations: endpoints, request/response formats, status codes, etc.).  
It focuses on **integration between services** without needing full end-to-end environments.

---

## Key Points

1. **What is a Contract?**  
   - A formal description of how a service can be called:
     - URL / endpoint
     - HTTP method (GET, POST, etc.)
     - Request body, headers, query params
     - Response body, status codes, headers

2. **Why Use Contract Testing?**
   - Detects breaking changes between microservices early.
   - Reduces need for heavy end-to-end tests.
   - Allows teams to develop services independently but safely.

3. **Types of Contract Testing**
   - **Consumer-driven**: The client (consumer) defines expectations; the provider must satisfy them.
   - **Provider-driven**: The provider defines the contract; consumers must adapt.

4. **Typical Flow**
   1. Consumer writes tests describing how it calls the provider.
   2. These tests generate a contract file (e.g., JSON).
   3. Provider runs tests against this contract to ensure it still matches.

---

## Java Example with Pact (Consumer-Driven Contract Testing)

### Scenario
- **Consumer**: `OrderService` calls `UserService` to get user details.
- **Provider**: `UserService` REST API.

### Consumer-Side Test (Java + Pact + JUnit)

**build.gradle (dependencies example)**

```groovy
dependencies {
    testImplementation 'au.com.dius.pact.consumer:junit5:4.6.6'
    testImplementation 'org.junit.jupiter:junit-jupiter:5.10.0'
    testImplementation 'org.springframework.boot:spring-boot-starter-web'
}
```

**Consumer test: defines the contract**

```java
import au.com.dius.pact.consumer.dsl.PactDslWithProvider;
import au.com.dius.pact.consumer.junit5.PactConsumerTestExt;
import au.com.dius.pact.consumer.junit5.PactTestFor;
import au.com.dius.pact.core.model.RequestResponsePact;
import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.extension.ExtendWith;

import java.util.Map;

import static io.pactfoundation.consumer.dsl.LambdaDsl.newJsonBody;
import static org.assertj.core.api.Assertions.assertThat;

@ExtendWith(PactConsumerTestExt.class)
@PactTestFor(providerName = "UserService", port = "8081")
class OrderServiceUserClientPactTest {

    @au.com.dius.pact.consumer.junit5.Pact(consumer = "OrderService")
    public RequestResponsePact createPact(PactDslWithProvider builder) {
        return builder
            .given("User with id 1 exists")
            .uponReceiving("A request for user 1")
                .path("/users/1")
                .method("GET")
            .willRespondWith()
                .status(200)
                .headers(Map.of("Content-Type", "application/json"))
                .body(newJsonBody(o -> {
                    o.numberType("id", 1);
                    o.stringType("name", "Alice");
                    o.stringType("email", "alice@example.com");
                }).build())
            .toPact();
    }

    @Test
    void testUserClient() {
        // This URL points to the mock server started by Pact
        UserClient client = new UserClient("http://localhost:8081");

        User user = client.getUserById(1L);

        assertThat(user.getId()).isEqualTo(1L);
        assertThat(user.getName()).isEqualTo("Alice");
    }
}
```

**UserClient (consumer code calling provider)**

```java
import org.springframework.web.client.RestTemplate;

public class UserClient {
    private final String baseUrl;
    private final RestTemplate restTemplate = new RestTemplate();

    public UserClient(String baseUrl) {
        this.baseUrl = baseUrl;
    }

    public User getUserById(Long id) {
        return restTemplate.getForObject(baseUrl + "/users/" + id, User.class);
    }
}

class User {
    private Long id;
    private String name;
    private String email;

    // getters/setters
}
```

### Example Input/Output (from the contract)

**Request (from consumer to provider)**

- Method: `GET`  
- URL: `/users/1`  
- Headers: `Accept: application/json`

**Expected Response**

```http
HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 1,
  "name": "Alice",
  "email": "alice@example.com"
}
```

Pact will generate a contract file (e.g., `OrderService-UserService.json`) describing this interaction.

## Provider-Side Verification (Java + Pact)

The provider (`UserService`) runs tests to verify it matches the contract.

**build.gradle (provider test dependency)**

```groovy
dependencies {
    testImplementation 'au.com.dius.pact.provider:junit5:4.6.6'
    testImplementation 'org.springframework.boot:spring-boot-starter-test'
}
```

**Provider verification test**

```java
import au.com.dius.pact.provider.junit5.PactVerificationContext;
import au.com.dius.pact.provider.junit5.PactVerificationInvocationContextProvider;
import au.com.dius.pact.provider.junit5.PactVerificationTest;
import au.com.dius.pact.provider.junit5.PactVerificationTestContext;
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.TestTemplate;
import org.junit.jupiter.api.extension.ExtendWith;
import org.springframework.boot.test.context.SpringBootTest;

@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.DEFINED_PORT, properties = {
    "server.port=8080"
})
@PactVerificationTest(provider = "UserService", pactSource = "path:../pacts/OrderService-UserService.json")
class UserServicePactProviderTest {

    @BeforeEach
    void before(PactVerificationContext context) {
        context.setTarget(new au.com.dius.pact.provider.junit5.http.HttpTestTarget("localhost", 8080));
    }

    @TestTemplate
    @ExtendWith(PactVerificationInvocationContextProvider.class)
    void pactVerificationTestTemplate(PactVerificationTestContext context) {
        context.verifyInteraction();
    }
}
```

If the provider’s real `/users/1` endpoint returns the expected JSON and status code, the test passes; otherwise, it fails, signaling a contract break.

---

**In short:**  
Contract testing ensures microservices agree on how they talk to each other by testing against a shared contract, catching integration issues early and enabling safer independent development.
