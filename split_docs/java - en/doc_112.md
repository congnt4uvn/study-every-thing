
Here’s a concise summary of CI/CD basics with Java-focused examples.

---

## Key Concepts

### 1. CI/CD (Continuous Integration / Continuous Delivery/Deployment)
- **Continuous Integration (CI)**:  
  Developers frequently merge code into a shared repository. Each merge triggers:
  - **Build** (compile, package)
  - **Automated tests**
- **Continuous Delivery/Deployment (CD)**:  
  After CI passes, the application is:
  - **Delivered** to a staging environment (Continuous Delivery), and/or  
  - **Automatically deployed** to production (Continuous Deployment).

---

## 2. Pipelines

A **pipeline** is an automated sequence of steps that code goes through from commit to deployment.

Typical stages:
1. **Source**: Get code from Git (GitHub, GitLab, etc.).
2. **Build**: Compile and package the application.
3. **Test**: Run unit/integration tests.
4. **Deploy**: Deploy to staging/production.

---

## 3. Build Stage (Java Example)

**Goal**: Compile and package the Java application.

**Example: Maven build command**

```bash
mvn clean package
```

**Typical output:**
- Input: Java source files in `src/main/java`
- Output: JAR/WAR file in `target/`, e.g.:
  - `target/myapp-1.0.0.jar`

**Sample `pom.xml` snippet:**

```xml
<project>
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.example</groupId>
  <artifactId>myapp</artifactId>
  <version>1.0.0</version>
  <packaging>jar</packaging>

  <dependencies>
    <!-- your dependencies -->
  </dependencies>

  <build>
    <plugins>
      <!-- e.g. maven-compiler-plugin -->
    </plugins>
  </build>
</project>
```

---

## 4. Test Stage (Java Example)

**Goal**: Automatically verify code correctness.

**Example: JUnit test**

```java
// src/test/java/com/example/CalculatorTest.java
import static org.junit.jupiter.api.Assertions.*;
import org.junit.jupiter.api.Test;

class Calculator {
    int add(int a, int b) { return a + b; }
}

public class CalculatorTest {

    @Test
    void testAdd() {
        Calculator calc = new Calculator();
        int result = calc.add(2, 3);
        assertEquals(5, result); // expected output: 5
    }
}
```

**Command to run tests (Maven):**

```bash
mvn test
```

**Input/Output:**
- Input: Test classes in `src/test/java`
- Output: Test report, e.g. `target/surefire-reports/`  
  - If all tests pass: build continues  
  - If any test fails: pipeline stops

---

## 5. Deploy Stage (Java Example)

**Goal**: Automatically deploy the built artifact to an environment.

### Example: Simple deployment script

```bash
#!/bin/bash
# deploy.sh

JAR_FILE="target/myapp-1.0.0.jar"
REMOTE_USER="deploy"
REMOTE_HOST="example.com"
REMOTE_DIR="/opt/myapp"

# Copy JAR to server
scp "$JAR_FILE" "$REMOTE_USER@$REMOTE_HOST:$REMOTE_DIR/"

# Restart service (example using systemd)
ssh "$REMOTE_USER@$REMOTE_HOST" "sudo systemctl restart myapp"
```

**Input/Output:**
- Input: Built JAR file
- Output: Updated application running on the server

---

## 6. Example CI Pipeline (GitHub Actions for Java)

```yaml
# .github/workflows/ci-cd.yml
name: Java CI/CD

on:
  push:
    branches: [ "main" ]

jobs:
  build-test-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Build with Maven
        run: mvn -B clean package

      - name: Run tests
        run: mvn test

      - name: Deploy (example)
        if: success()
        run: |
          chmod +x deploy.sh
          ./deploy.sh
```

**Pipeline flow:**
1. Trigger: push to `main`
2. Build: `mvn clean package`
3. Test: `mvn test`
4. Deploy: run `deploy.sh` if previous steps succeed

---

**In summary**:  
CI/CD pipelines automate the **build → test → deploy** flow. For Java, this usually means using tools like Maven/Gradle to compile and test, then deploying the resulting JAR/WAR to servers or cloud environments, all triggered automatically by code changes.
