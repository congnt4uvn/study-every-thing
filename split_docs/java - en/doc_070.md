## 1. Spring Data JPA Overview

Spring Data JPA simplifies working with relational databases using JPA.  
You define **repository interfaces**, and Spring generates the implementation at runtime.

---

## 2. `CrudRepository`

Basic CRUD operations (Create, Read, Update, Delete).

```java
import org.springframework.data.repository.CrudRepository;

public interface UserRepository extends CrudRepository<User, Long> {
}
```

Key methods (inherited):

- `save(S entity)` – insert or update
- `findById(ID id)` – find one
- `findAll()` – find all
- `deleteById(ID id)` – delete by id
- `count()` – number of records

**Example usage:**

```java
@Autowired
private UserRepository userRepository;

public void demoCrud() {
    // Input: new User(null, "Alice", "alice@example.com")
    User saved = userRepository.save(new User(null, "Alice", "alice@example.com"));
    Long id = saved.getId();

    Optional<User> found = userRepository.findById(id);
    Iterable<User> all = userRepository.findAll();
    userRepository.deleteById(id);
}
```

**Example output (conceptual):**

- After `save`: user row inserted with generated `id = 1`
- `findById(1)` → `User{id=1, name='Alice', email='alice@example.com'}`

## 3. `JpaRepository`

Extends `PagingAndSortingRepository` and `CrudRepository`.  
Adds JPA-specific and convenience methods.

```java
import org.springframework.data.jpa.repository.JpaRepository;

public interface UserRepository extends JpaRepository<User, Long> {
}
```

Extra methods include:

- `List<S> findAll()` (returns `List` instead of `Iterable`)
- `List<S> findAll(Sort sort)`
- `Page<S> findAll(Pageable pageable)`
- `void flush()`
- `S saveAndFlush(S entity)`

**Example usage with paging and sorting:**

```java
public void demoJpa() {
    // Input: page 0, size 5, sort by name ascending
    Pageable pageable = PageRequest.of(0, 5, Sort.by("name").ascending());
    Page<User> page = userRepository.findAll(pageable);

    // Output (conceptual):
    // page.getContent() -> List<User> of first 5 users sorted by name
    // page.getTotalElements() -> total number of users
}
```

## 4. Query Methods (Derived Queries)

Spring can derive queries from method names.

```java
public interface UserRepository extends JpaRepository<User, Long> {

    // SELECT u FROM User u WHERE u.email = ?1
    User findByEmail(String email);

    // SELECT u FROM User u WHERE u.name LIKE %?1%
    List<User> findByNameContaining(String keyword);

    // SELECT u FROM User u WHERE u.age >= ?1 AND u.age <= ?2
    List<User> findByAgeBetween(int min, int max);

    // SELECT u FROM User u WHERE u.active = true
    List<User> findByActiveTrue();
}
```

**Example usage:**

```java
public void demoQueryMethods() {
    // Input: "alice@example.com"
    User u = userRepository.findByEmail("alice@example.com");
    // Output: User with that email or null

    // Input: "Al"
    List<User> users = userRepository.findByNameContaining("Al");
    // Output: all users whose name contains "Al"
}
```

## 5. `@Query` Annotation (Custom Queries)

Use `@Query` when method-name derivation is not enough or you need custom JPQL/SQL.

```java
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;

public interface UserRepository extends JpaRepository<User, Long> {

    // JPQL query
    @Query("SELECT u FROM User u WHERE u.email = :email")
    User findUserByEmail(@Param("email") String email);

    // JPQL with multiple parameters
    @Query("SELECT u FROM User u WHERE u.age > :age AND u.active = true")
    List<User> findActiveUsersOlderThan(@Param("age") int age);

    // Native SQL query
    @Query(value = "SELECT * FROM users u WHERE u.name LIKE %:keyword%", 
           nativeQuery = true)
    List<User> searchByName(@Param("keyword") String keyword);
}
```

**Example usage:**

```java
public void demoCustomQuery() {
    // Input: "bob@example.com"
    User u = userRepository.findUserByEmail("bob@example.com");
    // Output: User with that email

    // Input: age = 30
    List<User> users = userRepository.findActiveUsersOlderThan(30);
    // Output: all active users older than 30
}
```

## 6. Example Entity

For completeness, a simple `User` entity:

```java
import jakarta.persistence.*;

@Entity
@Table(name = "users")
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;

    @Column(unique = true)
    private String email;

    private Integer age;

    private boolean active;

    // getters/setters, constructors omitted for brevity
}
```

---

**Main ideas:**

- `CrudRepository` → basic CRUD operations.
- `JpaRepository` → CRUD + paging, sorting, and JPA-specific utilities.
- Query methods → automatically derived from method names.
- `@Query` → custom JPQL or native SQL when you need more control.
