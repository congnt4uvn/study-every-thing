
Here’s a concise summary of the key ideas, with code examples and sample I/O.

---

## 1. Atomic Classes in `java.util.concurrent.atomic`

Atomic classes provide **lock-free, thread-safe** operations on single variables using **CAS (Compare-And-Set)** at the CPU level.

Common ones:
- `AtomicInteger` – for `int` values
- `AtomicLong` – for `long` values
- `AtomicBoolean` – for `boolean` values
- `AtomicReference<T>` – for object references

They avoid explicit `synchronized` blocks and can be faster under contention.

---

## 2. `AtomicInteger`

A thread-safe integer that supports atomic operations like increment, decrement, add, compare-and-set, etc.

### Common methods
- `get()` – read the current value
- `set(int newValue)` – set a new value
- `incrementAndGet()` – increment then return new value
- `getAndIncrement()` – return current value then increment
- `addAndGet(int delta)` – add delta then return new value
- `compareAndSet(int expect, int update)` – CAS operation

### Example: Thread-safe counter

```java
import java.util.concurrent.atomic.AtomicInteger;

public class AtomicCounterExample {
    private static AtomicInteger counter = new AtomicInteger(0);

    public static void main(String[] args) throws InterruptedException {
        Runnable task = () -> {
            for (int i = 0; i < 1000; i++) {
                counter.incrementAndGet(); // atomic increment
            }
        };

        Thread t1 = new Thread(task);
        Thread t2 = new Thread(task);

        t1.start();
        t2.start();
        t1.join();
        t2.join();

        System.out.println("Final counter value: " + counter.get());
    }
}
```

**Expected output:**
```text
Final counter value: 2000
```

Without `AtomicInteger`, using a plain `int` and `counter++`, the result could be less than 2000 due to race conditions.

---

## 3. `AtomicReference<T>`

Holds a reference to an object and allows atomic updates to that reference.

### Common methods
- `get()`
- `set(T newValue)`
- `compareAndSet(T expect, T update)`

### Example: Atomic update of a reference

```java
import java.util.concurrent.atomic.AtomicReference;

class User {
    final String name;
    final int age;

    User(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

public class AtomicReferenceExample {
    public static void main(String[] args) {
        AtomicReference<User> userRef =
                new AtomicReference<>(new User("Alice", 20));

        User oldUser = userRef.get();
        User newUser = new User("Alice", 21);

        boolean updated = userRef.compareAndSet(oldUser, newUser);

        System.out.println("Updated: " + updated);
        System.out.println("Current user age: " + userRef.get().age);
    }
}
```

**Possible output:**
```text
Updated: true
Current user age: 21
```

If another thread had changed `userRef` between `get()` and `compareAndSet()`, `updated` would be `false`.

---

## 4. CAS (Compare-And-Set) Concept

**CAS** is the low-level operation behind atomic classes. It works like this:

- Check if the current value equals an expected value.
- If equal, set it to a new value.
- The whole check-and-set is done atomically by the CPU.

In Java, this is exposed as `compareAndSet(expectedValue, newValue)`.

### Example: Manual CAS loop with `AtomicInteger`

```java
import java.util.concurrent.atomic.AtomicInteger;

public class CasLoopExample {
    public static void main(String[] args) {
        AtomicInteger value = new AtomicInteger(0);

        while (true) {
            int current = value.get();
            int next = current + 10;

            // Try to update from current to next
            if (value.compareAndSet(current, next)) {
                System.out.println("Updated successfully to: " + next);
                break;
            } else {
                System.out.println("CAS failed, retrying...");
            }
        }
    }
}
```

**Typical output:**
```text
Updated successfully to: 10
```

In a multi-threaded environment, you might see some `"CAS failed, retrying..."` lines before success.

---

### Key Takeaways

- Atomic classes provide **lock-free, thread-safe** operations on single variables.
- `AtomicInteger` is for primitive `int`; `AtomicReference<T>` is for object references.
- They rely on **CAS** to update values safely without using `synchronized`.
- Useful for counters, flags, and simple shared state in concurrent programs.
