
Here’s a concise summary of **immutable vs unmodifiable collections** in Java, with examples.

---

## Key Ideas

### 1. Unmodifiable Collection
- A **view** or wrapper around another collection.
- You **cannot modify it through that reference** (add/remove/set throws `UnsupportedOperationException`).
- But if the **underlying collection changes**, the unmodifiable view reflects those changes.
- So it is **not truly immutable**.

**How to create:**
- `Collections.unmodifiableList(...)`
- `Collections.unmodifiableSet(...)`
- `Collections.unmodifiableMap(...)`, etc.

**Example:**

```java
import java.util.*;

public class UnmodifiableExample {
    public static void main(String[] args) {
        List<String> original = new ArrayList<>();
        original.add("A");
        original.add("B");

        List<String> unmodifiable = Collections.unmodifiableList(original);

        System.out.println("Original:    " + original);
        System.out.println("Unmodifiable:" + unmodifiable);

        // Try to modify via unmodifiable reference
        try {
            unmodifiable.add("C");
        } catch (UnsupportedOperationException e) {
            System.out.println("Cannot modify unmodifiable list via its reference");
        }

        // Modify the original list
        original.add("C");

        System.out.println("After modifying original:");
        System.out.println("Original:    " + original);
        System.out.println("Unmodifiable:" + unmodifiable);
    }
}
```

**Sample Output:**

```
Original:    [A, B]
Unmodifiable:[A, B]
Cannot modify unmodifiable list via its reference
After modifying original:
Original:    [A, B, C]
Unmodifiable:[A, B, C]
```

Note how the unmodifiable list changes when the original list changes.

---

### 2. Immutable Collection
- The **data cannot change at all** after creation.
- No method can change its contents (no add/remove/set that works).
- Also, it does **not reflect changes** from any external collection.
- Truly **read-only and stable**.

**How to create (Java 9+):**
- `List.of(...)`
- `Set.of(...)`
- `Map.of(...)`, `Map.ofEntries(...)`

**Example:**

```java
import java.util.*;

public class ImmutableExample {
    public static void main(String[] args) {
        List<String> immutable = List.of("A", "B", "C");

        System.out.println("Immutable: " + immutable);

        // Try to modify
        try {
            immutable.add("D");
        } catch (UnsupportedOperationException e) {
            System.out.println("Cannot modify immutable list");
        }

        // Try to change an element
        try {
            immutable.set(0, "X");
        } catch (UnsupportedOperationException e) {
            System.out.println("Cannot change elements in immutable list");
        }
    }
}
```

**Sample Output:**

```
Immutable: [A, B, C]
Cannot modify immutable list
Cannot change elements in immutable list
```

---

### 3. Key Differences

1. **Source of truth**
   - Unmodifiable: wraps another collection; underlying data can still change.
   - Immutable: owns its data; no external changes are reflected.

2. **Safety**
   - Unmodifiable: safe only if the underlying collection is not modified elsewhere.
   - Immutable: safe by design; contents never change.

3. **Typical usage**
   - Unmodifiable: when you want to **expose a read-only view** of a mutable collection.
   - Immutable: when you want **guaranteed read-only data** (e.g., constants, configuration, thread-safe shared data).

---

### 4. Example: Unmodifiable vs Immutable

```java
import java.util.*;

public class CompareExample {
    public static void main(String[] args) {
        List<String> base = new ArrayList<>();
        base.add("A");
        base.add("B");

        List<String> unmodifiable = Collections.unmodifiableList(base);
        List<String> immutable = List.copyOf(base); // creates an immutable copy

        System.out.println("Base:        " + base);
        System.out.println("Unmodifiable:" + unmodifiable);
        System.out.println("Immutable:   " + immutable);

        // Modify base
        base.add("C");

        System.out.println("\nAfter modifying base:");
        System.out.println("Base:        " + base);
        System.out.println("Unmodifiable:" + unmodifiable); // reflects change
        System.out.println("Immutable:   " + immutable);    // does NOT change
    }
}
```

**Sample Output:**

```
Base:        [A, B]
Unmodifiable:[A, B]
Immutable:   [A, B]

After modifying base:
Base:        [A, B, C]
Unmodifiable:[A, B, C]
Immutable:   [A, B]
```

- `unmodifiable` changes with `base`.
- `immutable` stays the same.
