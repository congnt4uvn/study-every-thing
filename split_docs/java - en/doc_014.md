
Here’s a concise summary of Java’s pattern matching features for `instanceof` and `switch` (Java 16+), with code examples and sample I/O.

---

## 1. Pattern Matching for `instanceof` (Java 16+)

**Idea:**  
You can combine type check and cast in one step. If the `instanceof` check succeeds, the variable is automatically cast to that type in the `if` block.

### Before (old style)

```java
Object obj = "Hello";

if (obj instanceof String) {
    String s = (String) obj;  // manual cast
    System.out.println(s.toUpperCase());
}
```

**Output:**
```
HELLO
```

### After (pattern matching)

```java
Object obj = "Hello";

if (obj instanceof String s) {  // pattern variable 's'
    System.out.println(s.toUpperCase());
}
```

**Output:**
```
HELLO
```

**Key points:**
- `obj instanceof String s` both checks type and declares `s` as a `String`.
- `s` is only in scope where the compiler can guarantee the check is true (inside the `if` block, etc.).
- Reduces boilerplate and avoids unsafe casts.

---

## 2. Pattern Matching for `switch` (Java 17+ preview, Java 21+ standard)

**Idea:**  
`switch` can match on types and conditions, not just constant values. You can use patterns in `case` labels.

### Example: Type-based `switch`

```java
public class PatternSwitchExample {
    public static String formatObject(Object obj) {
        return switch (obj) {
            case String s      -> "String of length " + s.length();
            case Integer i     -> "Integer doubled: " + (i * 2);
            case Double d      -> "Double rounded: " + Math.round(d);
            case null          -> "It is null";
            default            -> "Unknown type: " + obj;
        };
    }

    public static void main(String[] args) {
        System.out.println(formatObject("Java"));
        System.out.println(formatObject(10));
        System.out.println(formatObject(3.14));
        System.out.println(formatObject(null));
        System.out.println(formatObject(true));
    }
}
```

**Output:**
```
String of length 4
Integer doubled: 20
Double rounded: 3
It is null
Unknown type: true
```

**Key points:**
- `case String s` matches when the object is a `String` and binds it to `s`.
- `case null` is allowed and must be handled explicitly if needed.
- `switch` can now be an expression that returns a value.

---

### Example: Guarded patterns (`when` / `&&` style conditions)

```java
public class GuardedPatternExample {
    public static String classifyNumber(Object obj) {
        return switch (obj) {
            case Integer i && i > 0  -> "Positive integer: " + i;
            case Integer i && i == 0 -> "Zero";
            case Integer i           -> "Negative integer: " + i;
            case null                -> "Null value";
            default                  -> "Not an integer";
        };
    }

    public static void main(String[] args) {
        System.out.println(classifyNumber(5));
        System.out.println(classifyNumber(0));
        System.out.println(classifyNumber(-3));
        System.out.println(classifyNumber("test"));
        System.out.println(classifyNumber(null));
    }
}
```

**Output:**
```
Positive integer: 5
Zero
Negative integer: -3
Not an integer
Null value
```

**Key points:**
- You can add extra conditions to a pattern (e.g., `Integer i && i > 0`).
- This allows more precise matching inside `switch`.

---

## Main Ideas (Summary)

- **Pattern matching for `instanceof`:**
  - Combines type check and cast: `if (obj instanceof String s)`.
  - Reduces boilerplate and improves readability and safety.

- **Pattern matching for `switch`:**
  - `switch` can match on **types** and **conditions**, not just constants.
  - Supports `case Type var` patterns and guarded patterns (`case Integer i && i > 0`).
  - `switch` can be used as an expression that returns a value.
  - Makes complex type-based logic more concise and expressive.
