
Here’s a concise summary of the main Java collection types you listed, with small code examples and sample input/output.

---

## 1. `HashSet`

- Stores **unique elements** (no duplicates).
- **No guaranteed order** of elements.
- Backed by a **hash table** → fast `add`, `remove`, `contains` (average O(1)).

```java
import java.util.HashSet;
import java.util.Set;

public class HashSetExample {
    public static void main(String[] args) {
        Set<String> set = new HashSet<>();

        set.add("Apple");
        set.add("Banana");
        set.add("Apple"); // duplicate, will be ignored

        System.out.println(set);
        System.out.println("Contains 'Banana'? " + set.contains("Banana"));
    }
}
```

**Possible output** (order not guaranteed):

```text
[Banana, Apple]
Contains 'Banana'? true
```

---

## 2. `TreeSet`

- Stores **unique elements**.
- Elements are kept in **sorted (ascending) order**.
- Backed by a **balanced tree** (e.g., Red-Black tree) → operations O(log n).
- Elements must be **Comparable** or you must provide a **Comparator**.

```java
import java.util.Set;
import java.util.TreeSet;

public class TreeSetExample {
    public static void main(String[] args) {
        Set<Integer> set = new TreeSet<>();

        set.add(5);
        set.add(1);
        set.add(3);

        System.out.println(set);
        System.out.println("First: " + ((TreeSet<Integer>) set).first());
        System.out.println("Last: " + ((TreeSet<Integer>) set).last());
    }
}
```

**Output:**

```text
[1, 3, 5]
First: 1
Last: 5
```

## 3. `HashMap`

- Stores **key–value pairs**.
- Keys are **unique**; values can be duplicated.
- **No guaranteed order** of keys.
- Backed by a **hash table** → fast `put`, `get`, `remove` (average O(1)).

```java
import java.util.HashMap;
import java.util.Map;

public class HashMapExample {
    public static void main(String[] args) {
        Map<String, Integer> map = new HashMap<>();

        map.put("Alice", 25);
        map.put("Bob", 30);
        map.put("Alice", 26); // overwrites previous value for "Alice"

        System.out.println(map);
        System.out.println("Alice's age: " + map.get("Alice"));
        System.out.println("Contains key 'Bob'? " + map.containsKey("Bob"));
    }
}
```

**Possible output** (order not guaranteed):

```text
{Bob=30, Alice=26}
Alice's age: 26
Contains key 'Bob'? true
```

---

## 4. `LinkedHashMap`

- Also stores **key–value pairs** with **unique keys**.
- Maintains **insertion order** (or access order if configured).
- Slightly more overhead than `HashMap`, but predictable iteration order.

```java
import java.util.LinkedHashMap;
import java.util.Map;

public class LinkedHashMapExample {
    public static void main(String[] args) {
        Map<String, Integer> map = new LinkedHashMap<>();

        map.put("First", 1);
        map.put("Second", 2);
        map.put("Third", 3);

        System.out.println(map);

        // Example: iterating in insertion order
        for (Map.Entry<String, Integer> entry : map.entrySet()) {
            System.out.println(entry.getKey() + " -> " + entry.getValue());
        }
    }
}
```

**Output:**

```text
{First=1, Second=2, Third=3}
First -> 1
Second -> 2
Third -> 3
```

---

## Quick Comparison

- **HashSet**: Unique elements, fast, **no order**.
- **TreeSet**: Unique elements, **sorted order**, slower than HashSet.
- **HashMap**: Key–value, fast, **no order**.
- **LinkedHashMap**: Key–value, fast, **insertion order** preserved.
