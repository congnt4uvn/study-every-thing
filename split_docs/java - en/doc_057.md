
Here’s a concise summary of the key Spring Core IoC/DI annotations, with simple code examples and expected behavior.

---

## 1. IoC & DI (Inversion of Control & Dependency Injection)

- **IoC**: The container (Spring) creates and manages objects instead of you using `new`.
- **DI**: The container injects dependencies into your classes (via constructor, setter, or field).

Example:

```java
@Component
public class MessageService {
    public String getMessage() {
        return "Hello, Spring!";
    }
}

@Component
public class MessagePrinter {

    private final MessageService messageService;

    // Constructor injection
    public MessagePrinter(MessageService messageService) {
        this.messageService = messageService;
    }

    public void print() {
        System.out.println(messageService.getMessage());
    }
}
```

Configuration and run:

```java
@Configuration
@ComponentScan(basePackages = "com.example")
public class AppConfig {
}

public class Main {
    public static void main(String[] args) {
        AnnotationConfigApplicationContext ctx =
                new AnnotationConfigApplicationContext(AppConfig.class);

        MessagePrinter printer = ctx.getBean(MessagePrinter.class);
        printer.print();  // Output: Hello, Spring!
    }
}
```

**Input**: none  
**Output (console)**: `Hello, Spring!`

## 2. Stereotype Annotations

These mark classes as Spring-managed beans and give them a role.

### @Component
- Generic Spring bean.
- Used for any class that should be managed by Spring.

```java
@Component
public class EmailSender {
    public void send(String to, String text) {
        System.out.println("Sending to " + to + ": " + text);
    }
}
```

---

### @Service
- Specialization of `@Component`.
- Indicates **service/business logic** layer.
- Semantically clearer than plain `@Component`.

```java
@Service
public class OrderService {
    public String placeOrder(String item) {
        return "Order placed for: " + item;
    }
}
```

Usage:

```java
@RestController
public class OrderController {

    private final OrderService orderService;

    public OrderController(OrderService orderService) {
        this.orderService = orderService;
    }

    @GetMapping("/order")
    public String order(String item) {
        return orderService.placeOrder(item);
    }
}
```

**Input (HTTP GET)**: `/order?item=Book`  
**Output (HTTP response)**: `Order placed for: Book`

---

### @Repository
- Specialization of `@Component`.
- Indicates **data access layer** (DAO).
- Enables automatic translation of persistence exceptions to Spring’s `DataAccessException`.

```java
@Repository
public class UserRepository {

    private final Map<Long, String> users = new HashMap<>();

    public UserRepository() {
        users.put(1L, "Alice");
        users.put(2L, "Bob");
    }

    public String findNameById(Long id) {
        return users.get(id);
    }
}
```

---

### @Controller
- Specialization of `@Component`.
- Used in **Spring MVC** for web controllers that return views (e.g., JSP, Thymeleaf).

```java
@Controller
public class HomeController {

    @GetMapping("/home")
    public String home(Model model) {
        model.addAttribute("message", "Welcome!");
        return "home"; // view name
    }
}
```

**Input (HTTP GET)**: `/home`  
**Output**: Renders `home.html` (or `home.jsp`) with `message = "Welcome!"`.

> In REST APIs, you often use `@RestController` (which is `@Controller + @ResponseBody`).

## 3. Configuration & Bean Definition

### @Configuration
- Marks a class as a **Java-based configuration** class.
- Replaces XML configuration.
- Methods annotated with `@Bean` define beans.

```java
@Configuration
public class AppConfig {

    @Bean
    public String appName() {
        return "MyApp";
    }
}
```

### @Bean
- Declares a bean that Spring should manage.
- Used inside `@Configuration` classes.

Example with dependency:

```java
public class GreetingService {
    private final String appName;

    public GreetingService(String appName) {
        this.appName = appName;
    }

    public String greet(String name) {
        return "Hello " + name + " from " + appName;
    }
}

@Configuration
public class AppConfig {

    @Bean
    public String appName() {
        return "MyApp";
    }

    @Bean
    public GreetingService greetingService(String appName) {
        return new GreetingService(appName);
    }
}
```

Run:

```java
public class Main {
    public static void main(String[] args) {
        AnnotationConfigApplicationContext ctx =
                new AnnotationConfigApplicationContext(AppConfig.class);

        GreetingService service = ctx.getBean(GreetingService.class);
        System.out.println(service.greet("John"));
    }
}
```

**Input**: `"John"` (method argument)  
**Output (console)**: `Hello John from MyApp`

---

## Summary of Roles

- `@Component`: Generic Spring-managed bean.
- `@Service`: Business/service layer bean.
- `@Repository`: Data access layer bean, with exception translation.
- `@Controller`: Web MVC controller (handles HTTP requests, returns views).
- `@Configuration`: Java config class, defines beans.
- `@Bean`: Method-level bean definition inside `@Configuration`.

All of these work together to enable IoC/DI: Spring creates and wires your objects automatically.
