## Key Ideas

1. **Unit Testing**
   - Tests a single class or method in isolation.
   - External dependencies (databases, web services, other classes) are replaced with **mocks**.
   - Frameworks: **Mockito**, EasyMock, etc.
   - Goal: Verify logic of one unit without side effects.

2. **Integration Testing**
   - Tests how multiple components work together (e.g., service + repository + database).
   - Often uses real implementations or test containers (e.g., in-memory DB).
   - Less mocking; focuses on real interactions.

3. **Mockito / Mocking Frameworks**
   - Create fake objects (mocks) to:
     - Control their behavior (what they return).
     - Verify interactions (which methods were called, with what arguments).
   - Useful for:
     - Isolating the class under test.
     - Simulating error conditions or edge cases.
     - Speeding up tests (no real DB/network).

## Simple Unit Test with Mockito

### Production Code

```java
// Service that depends on a repository
public class UserService {
    private final UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    public String getUserDisplayName(Long id) {
        User user = userRepository.findById(id);
        if (user == null) {
            return "Unknown User";
        }
        return user.getFirstName() + " " + user.getLastName();
    }
}

public interface UserRepository {
    User findById(Long id);
}

public class User {
    private Long id;
    private String firstName;
    private String lastName;

    // constructor, getters, setters omitted for brevity
}
```

### Unit Test with Mockito (JUnit 5)

```java
import org.junit.jupiter.api.Test;
import org.mockito.Mockito;

import static org.junit.jupiter.api.Assertions.assertEquals;
import static org.mockito.Mockito.*;

class UserServiceTest {

    @Test
    void getUserDisplayName_returnsFullName_whenUserExists() {
        // Arrange (given)
        UserRepository mockRepo = Mockito.mock(UserRepository.class);
        UserService userService = new UserService(mockRepo);

        User user = new User(1L, "John", "Doe");
        when(mockRepo.findById(1L)).thenReturn(user);

        // Act (when)
        String result = userService.getUserDisplayName(1L);

        // Assert (then)
        assertEquals("John Doe", result);
        verify(mockRepo, times(1)).findById(1L);
    }

    @Test
    void getUserDisplayName_returnsUnknown_whenUserNotFound() {
        // Arrange
        UserRepository mockRepo = Mockito.mock(UserRepository.class);
        UserService userService = new UserService(mockRepo);

        when(mockRepo.findById(2L)).thenReturn(null);

        // Act
        String result = userService.getUserDisplayName(2L);

        // Assert
        assertEquals("Unknown User", result);
        verify(mockRepo).findById(2L);
    }
}
```

### Example Inputs/Outputs (Conceptual)

- Input: `getUserDisplayName(1L)`  
  Mock behavior: `findById(1L)` returns `User(1, "John", "Doe")`  
  Output: `"John Doe"`

- Input: `getUserDisplayName(2L)`  
  Mock behavior: `findById(2L)` returns `null`  
  Output: `"Unknown User"`

## Simple Integration Test (No Mocking, Using In-Memory DB)

```java
// Example with Spring Boot + H2 (conceptual)
@SpringBootTest
@AutoConfigureTestDatabase
class UserServiceIntegrationTest {

    @Autowired
    private UserService userService;

    @Autowired
    private UserRepository userRepository; // real implementation

    @Test
    void getUserDisplayName_worksWithRealDatabase() {
        // Arrange: save real data in test DB
        User user = new User(1L, "Alice", "Smith");
        userRepository.save(user);

        // Act
        String result = userService.getUserDisplayName(1L);

        // Assert
        assertEquals("Alice Smith", result);
    }
}
```

Here, the test uses the real `UserRepository` and a real (test) database, so it’s an **integration test**, not a pure unit test.
