
Here’s a concise summary of the key ideas, with simple Java-focused examples.

---

## 1. Core Concepts

### Docker Image
- A **read-only template** that contains everything needed to run an app: code, runtime (e.g., JDK), libraries, and OS-level dependencies.
- Built from a **Dockerfile**.
- You can think of it as a “packaged application snapshot.”

### Docker Container
- A **running instance** of an image.
- Lightweight, isolated environment.
- You can start, stop, and remove containers without affecting the image.

### Dockerfile
- A **text file with instructions** to build a Docker image.
- Defines:
  - Base image (e.g., `openjdk:17-jdk-alpine`)
  - How to copy your app into the image
  - How to run it

---

## 2. Simple Java Example

Assume you have a simple Java app:

**File: `Hello.java`**
```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello from Docker + Java!");
    }
}
```

Compile it locally:

```bash
javac Hello.java
```

This creates `Hello.class`.

---

## 3. Dockerfile Example (Java)

**File: `Dockerfile`**
```dockerfile
# 1. Use a base image with Java installed
FROM openjdk:17-jdk-alpine

# 2. Set working directory inside the container
WORKDIR /app

# 3. Copy compiled class into the container
COPY Hello.class /app/

# 4. Command to run when container starts
CMD ["java", "Hello"]
```

---

## 4. Build the Image

From the directory containing `Hello.class` and `Dockerfile`:

```bash
docker build -t java-hello .
```

- `-t java-hello` names the image `java-hello`.
- `.` means “use the current directory as build context.”

---

## 5. Run the Container

```bash
docker run --rm java-hello
```

**Expected output:**
```text
Hello from Docker + Java!
```

- `--rm` removes the container after it exits.

---

## 6. Another Example: Spring Boot JAR

If you have a Spring Boot app packaged as `app.jar`:

**Dockerfile**
```dockerfile
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY target/app.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

Build and run:

```bash
docker build -t my-spring-app .
docker run -p 8080:8080 my-spring-app
```

- Access: `http://localhost:8080`

---

## 7. Key Commands Summary

- Build image: `docker build -t <name> .`
- List images: `docker images`
- Run container: `docker run <image>`
- List containers: `docker ps` (running), `docker ps -a` (all)
- Stop container: `docker stop <container-id>`
- Remove container: `docker rm <container-id>`
- Remove image: `docker rmi <image-id>`

These are the basics of using Docker (images, containers, Dockerfile) with Java applications.
