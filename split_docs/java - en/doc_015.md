
Here’s a concise summary of the key ideas behind Java’s Project Loom, plus code examples with sample input/output.

---

## 1. Virtual Threads (Lightweight Threads)

**Main idea:**  
Virtual threads are very lightweight threads managed by the JVM, not by the OS. They let you create **thousands or millions** of concurrent tasks without the heavy cost of traditional platform threads.

**Key points:**
- Introduced as a preview in Java 19, more stable in later versions.
- Same programming model as normal threads (blocking I/O, `synchronized`, etc.).
- Great for server apps that handle many concurrent requests.
- You still use `Runnable`, `Callable`, `ExecutorService`, etc., but backed by virtual threads.

### Basic Example: Creating Virtual Threads

```java
public class VirtualThreadExample {
    public static void main(String[] args) throws InterruptedException {
        Thread vThread = Thread.ofVirtual().start(() -> {
            System.out.println("Hello from virtual thread: " + Thread.currentThread());
        });

        vThread.join(); // wait for completion
    }
}
```

**Sample output:**
```
Hello from virtual thread: VirtualThread[#21]/runnable@ForkJoinPool-1-worker-1
```

### Example: Many Virtual Threads

```java
import java.util.concurrent.*;

public class ManyVirtualThreads {
    public static void main(String[] args) throws InterruptedException {
        try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {
            for (int i = 0; i < 5; i++) {
                int id = i;
                executor.submit(() -> {
                    System.out.println("Task " + id + " running in " + Thread.currentThread());
                    Thread.sleep(1000); // blocking call is OK
                    System.out.println("Task " + id + " finished");
                    return null;
                });
            }
        } // executor.close() waits for tasks to finish
    }
}
```

**Sample output (order may vary):**
```

Task 0 finished
```

## 2. Structured Concurrency

**Main idea:**  
Structured concurrency treats multiple concurrent tasks as a **single unit of work**. It helps you:
- Start several tasks together.
- Wait for them as a group.
- Cancel remaining tasks if one fails.
- Ensure no “orphan” threads are left running.

**Key points:**
- Introduced as an incubator API (`StructuredTaskScope`) in Java 19+.
- Makes concurrent code easier to reason about and safer.
- Similar to “scopes” in other languages (e.g., Go, Kotlin coroutines).

### Example: Run Tasks in Parallel and Wait for All

```java
import java.time.Duration;
import java.util.concurrent.*;
import jdk.incubator.concurrent.StructuredTaskScope;

public class StructuredConcurrencyExample {
    public static void main(String[] args) throws InterruptedException, ExecutionException {
        try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
            Future<String> userFuture = scope.fork(() -> fetchUser());
            Future<String> orderFuture = scope.fork(() -> fetchOrders());

            scope.join();           // wait for both
            scope.throwIfFailed();  // throw if any failed

            String user = userFuture.resultNow();
            String orders = orderFuture.resultNow();

            System.out.println("User: " + user);
            System.out.println("Orders: " + orders);
        }
    }

    static String fetchUser() throws InterruptedException {
        Thread.sleep(500); // simulate I/O
        return "User(id=1, name=Alice)";
    }

    static String fetchOrders() throws InterruptedException {
        Thread.sleep(800); // simulate I/O
        return "Orders[order1, order2]";
    }
}
```

**Sample output:**
```
User: User(id=1, name=Alice)
Orders: Orders[order1, order2]
```

---

### Example: Cancel Others on Failure

```java
import java.util.concurrent.*;
import jdk.incubator.concurrent.StructuredTaskScope;

public class StructuredFailureExample {
    public static void main(String[] args) {
        try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
            Future<String> fast = scope.fork(() -> {
                Thread.sleep(200);
                return "Fast result";
            });

            Future<String> failing = scope.fork(() -> {
                Thread.sleep(300);
                throw new RuntimeException("Something went wrong");
            });

            scope.join(); // wait for both to finish or fail

            try {
                scope.throwIfFailed(); // will throw because one task failed
            } catch (Exception e) {
                System.out.println("Scope failed: " + e.getMessage());
            }

            // The other task is cancelled automatically
            System.out.println("Fast task done? " + fast.state());
            System.out.println("Failing task done? " + failing.state());
        }
    }
}
```

**Sample output (simplified):**
```
Scope failed: java.lang.RuntimeException: Something went wrong
Fast task done? CANCELLED
Failing task done? FAILED
```

## Summary

- **Virtual threads**:  
  - Extremely cheap threads managed by the JVM.  
  - Let you write simple blocking code while scaling to many concurrent tasks.

- **Structured concurrency**:  
  - Groups related tasks into a scope.  
  - Makes starting, waiting, error handling, and cancellation of concurrent tasks safer and clearer.

These features aim to make high‑concurrency Java code **simpler, more readable, and more scalable** without forcing you into complex async callback styles.
