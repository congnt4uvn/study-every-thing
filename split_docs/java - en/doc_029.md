
Here are the key ideas about **exceptions & error handling** with **try-with-resources** and **AutoCloseable** in Java, plus examples.

---

## 1. `try-with-resources` (TWR)

**Purpose:**  
Automatically closes resources (like files, streams, DB connections) when the `try` block finishes, **even if an exception occurs**.

**Requirements:**

- Resources must implement **`AutoCloseable`** (or `Closeable`).
- Resources are declared in the parentheses of `try(...)`.

### Basic Example

```java
import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class TryWithResourcesExample {
    public static void main(String[] args) {
        // Input: a file "test.txt" containing: Hello\nWorld
        try (BufferedReader br = new BufferedReader(new FileReader("test.txt"))) {
            String line;
            while ((line = br.readLine()) != null) {
                System.out.println(line);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
        // Output:
        // Hello
        // World
        // br is automatically closed here
    }
}
```

**Key points:**

- No need for a `finally` block to close `br`.
- If an exception happens while reading, `br.close()` is still called automatically.

---

## 2. `AutoCloseable` Interface

**Purpose:**  
Allows a class to be used in `try-with-resources`.  
Any class that implements `AutoCloseable` must define a `close()` method.

### Custom Resource Example

```java
class MyResource implements AutoCloseable {
    public MyResource() {
        System.out.println("MyResource: opened");
    }

    public void doWork() {
        System.out.println("MyResource: working");
    }

    @Override
    public void close() {
        System.out.println("MyResource: closed");
    }
}

public class AutoCloseableExample {
    public static void main(String[] args) {
        try (MyResource r = new MyResource()) {
            r.doWork();
        }
        // Output:
        // MyResource: opened
        // MyResource: working
        // MyResource: closed
    }
}
```

**Key points:**

- `close()` is called automatically at the end of the `try` block.
- You can put multiple resources in one `try`:

```java
try (MyResource r1 = new MyResource();
     MyResource r2 = new MyResource()) {
    r1.doWork();
    r2.doWork();
}
```

---

## 3. Exception Handling with TWR

- If an exception occurs in the `try` block and another occurs during `close()`, the one from `close()` becomes a **suppressed exception**.
- You can inspect suppressed exceptions:

```java
try (MyResource r = new MyResource()) {
    throw new RuntimeException("Main exception");
} catch (Exception e) {
    System.out.println("Caught: " + e.getMessage());
    for (Throwable t : e.getSuppressed()) {
        System.out.println("Suppressed: " + t);
    }
}
```

**Key idea:**  
TWR simplifies code, reduces resource leaks, and makes exception handling around resources safer and cleaner.
