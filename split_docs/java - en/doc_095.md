
Here’s a concise summary of the key ideas behind **Dependency Injection (DI)** and **Inversion of Control (IoC)** in Java, with simple code examples.

---

## 1. Inversion of Control (IoC)

**Idea:**  
Instead of a class creating and controlling its own dependencies, the control is “inverted” and given to an external container or framework (like Spring).  
The framework decides **what** objects to create, **when**, and **how** they are wired together.

- Without IoC: Your code controls object creation.
- With IoC: A container/framework controls object creation and wiring.

---

## 2. Dependency Injection (DI)

**Idea:**  
DI is a way to implement IoC.  
Instead of a class creating its dependencies with `new`, those dependencies are **injected** from the outside (via constructor, setter, or field).

Benefits:
- Loose coupling
- Easier testing (mock dependencies)
- More flexible and maintainable design

---

## 3. Example Without DI (Tightly Coupled)

```java
class EmailService {
    public void sendEmail(String message, String to) {
        System.out.println("Email sent to " + to + " with message: " + message);
    }
}

class NotificationController {
    private EmailService emailService;

    public NotificationController() {
        // Directly creating dependency (tight coupling)
        this.emailService = new EmailService();
    }

    public void send(String msg, String user) {
        emailService.sendEmail(msg, user);
    }
}

public class Main {
    public static void main(String[] args) {
        NotificationController controller = new NotificationController();
        controller.send("Hello!", "user@example.com");
    }
}
```

**Sample Output:**
```
Email sent to user@example.com with message: Hello!
```

Problems:
- `NotificationController` is tightly coupled to `EmailService`.
- Hard to replace `EmailService` with another implementation (e.g., SMS, mock for tests).

---

## 4. Example With DI (Constructor Injection)

### Step 1: Define an interface

```java
interface MessageService {
    void sendMessage(String message, String to);
}
```

### Step 2: Implement the interface

```java
class EmailService implements MessageService {
    @Override
    public void sendMessage(String message, String to) {
        System.out.println("Email sent to " + to + " with message: " + message);
    }
}

class SmsService implements MessageService {
    @Override
    public void sendMessage(String message, String to) {
        System.out.println("SMS sent to " + to + " with message: " + message);
    }
}
```

### Step 3: Inject dependency via constructor

```java
class NotificationController {
    private final MessageService messageService;

    // Dependency is injected from outside
    public NotificationController(MessageService messageService) {
        this.messageService = messageService;
    }

    public void send(String msg, String user) {
        messageService.sendMessage(msg, user);
    }
}
```

### Step 4: Wire them together (manual DI)

```java
public class Main {
    public static void main(String[] args) {
        // Choose implementation at configuration time
        MessageService service = new EmailService(); // or new SmsService();
        NotificationController controller = new NotificationController(service);

        controller.send("Hello!", "user@example.com");
    }
}
```

**Sample Output (with EmailService):**
```
Email sent to user@example.com with message: Hello!
```

**Sample Output (with SmsService):**
```
SMS sent to user@example.com with message: Hello!
```

Now `NotificationController` doesn’t care *how* the message is sent; it just uses `MessageService`.

---

## 5. DI with a Framework (IoC Container Example – Spring)

Using Spring annotations:

```java
import org.springframework.stereotype.Service;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

interface MessageService {
    void sendMessage(String message, String to);
}

@Service
class EmailService implements MessageService {
    @Override
    public void sendMessage(String message, String to) {
        System.out.println("Email sent to " + to + " with message: " + message);
    }
}

@Component
class NotificationController {
    private final MessageService messageService;

    @Autowired  // Spring injects the dependency
    public NotificationController(MessageService messageService) {
        this.messageService = messageService;
    }

    public void send(String msg, String user) {
        messageService.sendMessage(msg, user);
    }
}
```

Spring’s IoC container:
- Creates `EmailService` and `NotificationController`.
- Injects `EmailService` into `NotificationController` automatically.

---

## 6. Key Takeaways

- **IoC**: The framework controls object creation and wiring instead of your code.
- **DI**: A pattern to achieve IoC by injecting dependencies rather than creating them inside classes.
- **Result**: More modular, testable, and maintainable systems.
