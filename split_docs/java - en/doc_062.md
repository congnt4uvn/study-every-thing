
Here’s a concise summary and examples about using `application.yml` and `application.properties` in Spring Boot.

---

## 1. Purpose

Both `application.yml` and `application.properties` are configuration files used by Spring Boot to:

- Set server settings (port, context path, etc.)
- Configure data sources (DB URL, username, password)
- Define custom application properties
- Configure logging, profiles, etc.

Spring Boot automatically loads them from `src/main/resources/`.

---

## 2. File Formats

### `application.properties`

- Simple key-value pairs
- Dot notation for hierarchy

```properties
# application.properties
server.port=8081
spring.application.name=my-app

spring.datasource.url=jdbc:mysql://localhost:3306/testdb
spring.datasource.username=testuser
spring.datasource.password=secret

app.title=My Application
app.feature.enabled=true
```

### `application.yml`

- YAML format (indentation-based)
- More readable for nested structures

```yaml
# application.yml
server:
  port: 8081

spring:
  application:
    name: my-app
  datasource:
    url: jdbc:mysql://localhost:3306/testdb
    username: testuser
    password: secret

app:
  title: My Application
  feature:
    enabled: true
```

Both files configure the same things; you typically choose one style.

---

## 3. Using Profiles

You can have different configs for different environments (dev, test, prod).

### Properties

```properties
# application.properties
spring.profiles.active=dev
```

```properties
# application-dev.properties
server.port=8080

# application-prod.properties
server.port=80
```

### YAML

```yaml
# application.yml
spring:
  profiles:
    active: dev
```

```yaml
# application-dev.yml
server:
  port: 8080

# application-prod.yml
server:
  port: 80
```

## 4. Reading Values in Code

### 4.1 Using `@Value`

```java
// application.yml
app:
  title: My Application
  feature:
    enabled: true
```

```java
// ExampleService.java
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Service;

@Service
public class ExampleService {

    @Value("${app.title}")
    private String title;

    @Value("${app.feature.enabled}")
    private boolean featureEnabled;

    public String getInfo() {
        return "Title: " + title + ", featureEnabled: " + featureEnabled;
    }
}
```

**Example REST controller to show output:**

```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class InfoController {

    private final ExampleService exampleService;

    public InfoController(ExampleService exampleService) {
        this.exampleService = exampleService;
    }

    @GetMapping("/info")
    public String info() {
        return exampleService.getInfo();
    }
}
```

**Sample HTTP request/response:**

- **Request:** `GET http://localhost:8081/info`
- **Response body:**  
  `Title: My Application, featureEnabled: true`

### 4.2 Using `@ConfigurationProperties` (good for groups)

```yaml
# application.yml
app:
  title: My Application
  feature:
    enabled: true
    max-users: 100
```

```java
// AppProperties.java
import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.stereotype.Component;

@Component
@ConfigurationProperties(prefix = "app")
public class AppProperties {

    private String title;
    private Feature feature = new Feature();

    public static class Feature {
        private boolean enabled;
        private int maxUsers;

        // getters and setters
        public boolean isEnabled() { return enabled; }
        public void setEnabled(boolean enabled) { this.enabled = enabled; }
        public int getMaxUsers() { return maxUsers; }
        public void setMaxUsers(int maxUsers) { this.maxUsers = maxUsers; }
    }

    // getters and setters
    public String getTitle() { return title; }
    public void setTitle(String title) { this.title = title; }
    public Feature getFeature() { return feature; }
    public void setFeature(Feature feature) { this.feature = feature; }
}
```

```java
// AppController.java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class AppController {

    private final AppProperties appProperties;

    public AppController(AppProperties appProperties) {
        this.appProperties = appProperties;
    }

    @GetMapping("/app-config")
    public String getConfig() {
        return "Title: " + appProperties.getTitle()
                + ", enabled: " + appProperties.getFeature().isEnabled()
                + ", maxUsers: " + appProperties.getFeature().getMaxUsers();
    }
}
```

**Sample HTTP request/response:**

- **Request:** `GET http://localhost:8081/app-config`
- **Response body:**  
  `Title: My Application, enabled: true, maxUsers: 100`

---

## 5. Key Points

- `application.properties` and `application.yml` serve the same purpose; choose one style.
- YAML is better for complex/nested configuration.
- Use profiles (`application-dev`, `application-prod`) for environment-specific settings.
- Access configuration via `@Value` for single values or `@ConfigurationProperties` for grouped settings.
