
Here’s a concise summary of Hexagonal / Clean Architecture basics, with Java-style examples.

---

## Core Ideas

1. **Separation of concerns**
   - Split your system into layers so business logic is independent from frameworks, databases, and UI.
   - Typical layers:
     - **Domain (Core)**: Entities, value objects, business rules.
     - **Application (Use Cases)**: Orchestrates domain logic, defines input/output ports.
     - **Adapters (Infrastructure / UI)**: Implement ports to talk to DB, web, messaging, etc.

2. **Dependency rule**
   - Dependencies always point **inward**:
     - Adapters → Application → Domain
   - Inner layers know nothing about outer layers (no imports from web, DB, etc. in domain).

3. **Ports and Adapters (Hexagonal)**
   - **Ports**: Interfaces defined by the core (application/domain) that describe what the outside world can do with it.
   - **Adapters**: Implementations of those interfaces for specific technologies (e.g., REST controller, JPA repository).

4. **Testability and flexibility**
   - Because the core depends only on interfaces, you can:
     - Swap databases, UIs, frameworks with minimal changes.
     - Unit test business logic without infrastructure.

## Simple Java Example

### 1. Domain Layer (Core Business Logic)

```java
// Domain entity
public class Account {
    private final String id;
    private int balance;

    public Account(String id, int initialBalance) {
        this.id = id;
        this.balance = initialBalance;
    }

    public void deposit(int amount) {
        if (amount <= 0) throw new IllegalArgumentException("Amount must be positive");
        this.balance += amount;
    }

    public void withdraw(int amount) {
        if (amount <= 0) throw new IllegalArgumentException("Amount must be positive");
        if (amount > balance) throw new IllegalStateException("Insufficient funds");
        this.balance -= amount;
    }

    public int getBalance() {
        return balance;
    }

    public String getId() {
        return id;
    }
}
```

### 2. Application Layer (Use Case + Ports)

```java
// Input port (use case interface)
public interface TransferUseCase {
    void transfer(String fromAccountId, String toAccountId, int amount);
}

// Output port (driven port for persistence)
public interface AccountRepository {
    Account findById(String id);
    void save(Account account);
}

// Use case implementation (application service)
public class TransferService implements TransferUseCase {

    private final AccountRepository accountRepository;

    public TransferService(AccountRepository accountRepository) {
        this.accountRepository = accountRepository;
    }

    @Override
    public void transfer(String fromAccountId, String toAccountId, int amount) {
        Account from = accountRepository.findById(fromAccountId);
        Account to = accountRepository.findById(toAccountId);

        from.withdraw(amount);
        to.deposit(amount);

        accountRepository.save(from);
        accountRepository.save(to);
    }
}
```

- `TransferUseCase` and `AccountRepository` are **ports**.
- `TransferService` is the **core application logic**.

### 3. Infrastructure Adapter (e.g., JPA Repository)

```java
// JPA entity (could be same or mapped from domain)
import jakarta.persistence.Entity;
import jakarta.persistence.Id;

@Entity
public class AccountEntity {
    @Id
    private String id;
    private int balance;

    // getters/setters omitted for brevity
}
```

```java
// Spring Data JPA repository
import org.springframework.data.jpa.repository.JpaRepository;

public interface SpringDataAccountJpaRepository
        extends JpaRepository<AccountEntity, String> {
}
```

```java
// Adapter implementing the domain port
public class AccountRepositoryJpaAdapter implements AccountRepository {

    private final SpringDataAccountJpaRepository jpaRepository;

    public AccountRepositoryJpaAdapter(SpringDataAccountJpaRepository jpaRepository) {
        this.jpaRepository = jpaRepository;
    }

    @Override
    public Account findById(String id) {
        AccountEntity entity = jpaRepository.findById(id)
                .orElseThrow(() -> new IllegalArgumentException("Account not found"));
        return new Account(entity.getId(), entity.getBalance());
    }

    @Override
    public void save(Account account) {
        AccountEntity entity = new AccountEntity();
        entity.setId(account.getId());
        entity.setBalance(account.getBalance());
        jpaRepository.save(entity);
    }
}
```

### 4. Driving Adapter (e.g., REST Controller)

```java
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/accounts")
public class TransferController {

    private final TransferUseCase transferUseCase;

    public TransferController(TransferUseCase transferUseCase) {
        this.transferUseCase = transferUseCase;
    }

    @PostMapping("/transfer")
    public void transfer(@RequestBody TransferRequest request) {
        transferUseCase.transfer(
                request.getFromAccountId(),
                request.getToAccountId(),
                request.getAmount()
        );
    }
}

class TransferRequest {
    private String fromAccountId;
    private String toAccountId;
    private int amount;
    // getters/setters
}
```

## Example Input/Output (REST)

**HTTP Request (Input)**

```http
POST /accounts/transfer
Content-Type: application/json

{
  "fromAccountId": "A1",
  "toAccountId": "A2",
  "amount": 100
}
```

**Behavior (Internal)**

- `TransferController` → calls `TransferUseCase.transfer(...)`
- `TransferService`:
  - loads accounts via `AccountRepository`
  - applies domain rules (`withdraw`, `deposit`)
  - saves updated accounts

**HTTP Response (Output)**

```http
200 OK
```

(Or an error status if validation/business rules fail.)

---

## Key Takeaways

- Hexagonal / Clean Architecture:
  - Centered on **domain and use cases**.
  - Uses **ports (interfaces)** and **adapters (implementations)**.
  - Keeps business logic independent from frameworks and infrastructure.
  - Improves testability, maintainability, and ability to change technologies.
