## Key Ideas

- **Maven and Gradle** are popular Java build tools.
- They manage:
  - **Lifecycle**: how code is compiled, tested, packaged, and deployed.
  - **Dependencies**: external libraries your project needs.
  - **Profiles**: different configurations for environments (dev, test, prod).

---

## Maven

### 1. Lifecycle (simplified)

Common phases (run with `mvn <phase>`):

- `clean` – delete previous build output  
- `compile` – compile source code  
- `test` – run unit tests  
- `package` – create JAR/WAR  
- `install` – install artifact to local repository  
- `deploy` – deploy to remote repository  

Example command:

```bash
mvn clean package
```

**Input**: `pom.xml` + source code  
**Output**: `target/my-app-1.0-SNAPSHOT.jar`

---

### 2. Dependencies

Defined in `pom.xml`:

```xml
<dependencies>
    <dependency>
        <groupId>org.springframework</groupId>
        <artifactId>spring-context</artifactId>
        <version>5.3.30</version>
    </dependency>
</dependencies>
```

Maven downloads this library and makes it available on the classpath.

---

### 3. Profiles

Use profiles for different environments:

```xml
<profiles>
    <profile>
        <id>dev</id>
        <properties>
            <env.name>development</env.name>
        </properties>
    </profile>

    <profile>
        <id>prod</id>
        <properties>
            <env.name>production</env.name>
        </properties>
    </profile>
</profiles>
```

Run with a profile:

```bash
mvn clean package -Pdev
```

## Gradle

### 1. Lifecycle (simplified)

Gradle uses **tasks** instead of fixed phases.

Common tasks (Java plugin):

- `clean` – delete build directory  
- `compileJava` – compile Java sources  
- `test` – run tests  
- `jar` – create JAR  
- `build` – runs tests and creates artifacts  

Example command:

```bash
gradle clean build
```

**Input**: `build.gradle` + source code  
**Output**: `build/libs/my-app-1.0-SNAPSHOT.jar`

---

### 2. Dependencies

Defined in `build.gradle`:

```groovy
plugins {
    id 'java'
}

repositories {
    mavenCentral()
}

dependencies {
    implementation 'org.springframework:spring-context:5.3.30'
    testImplementation 'junit:junit:4.13.2'
}
```

Gradle resolves and downloads these libraries automatically.

---

### 3. Profiles (via Gradle properties)

Gradle doesn’t have “profiles” like Maven, but you can simulate them with properties:

`build.gradle`:

```groovy
def env = project.hasProperty('env') ? project.getProperty('env') : 'dev'

task printEnv {
    doLast {
        println "Current environment: $env"
    }
}
```

Run with:

```bash
gradle printEnv -Penv=prod
```

**Output**:

```text
Current environment: prod
```

---

**In summary**:  
- Use **Maven or Gradle** to automate compile–test–package–deploy steps.  
- Declare **dependencies** so libraries are downloaded and managed for you.  
- Use **profiles/properties** to switch configuration between environments.
