## 1. `@Entity` ŌĆō Marking a Persistent Class

`@Entity` tells JPA/Spring Data that a class should be mapped to a database table.

```java
import jakarta.persistence.*;

@Entity
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String username;

    // getters/setters
}
```

- This creates a table (e.g., `user`) with columns `id`, `username`.
- Spring Data repositories can now persist and query `User`.

**Example repository:**

```java
import org.springframework.data.jpa.repository.JpaRepository;

public interface UserRepository extends JpaRepository<User, Long> {
    User findByUsername(String username);
}
```

**Example usage (input ŌåÆ output):**

```java
// Input: save a new user
User user = new User();
user.setUsername("alice");
userRepository.save(user);

// Input: find by username
User found = userRepository.findByUsername("alice");
System.out.println(found.getId() + " " + found.getUsername());
```

**Output (console):**
```text
1 alice
```

## 2. `@Table` ŌĆō Customizing Table Name and Schema

`@Table` customizes how the entity maps to a database table.

```java
import jakarta.persistence.*;

@Entity
@Table(name = "app_users", schema = "public")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "user_name", nullable = false, unique = true)
    private String username;

    // getters/setters
}
```

- Entity: `User`
- Table: `public.app_users`
- Column: `user_name` instead of `username`

## 3. Relationships

### 3.1 `@OneToMany` ŌĆō One-to-Many Relationship

Example: One `User` has many `Post`s.

```java
import jakarta.persistence.*;
import java.util.List;

@Entity
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String username;

    @OneToMany(mappedBy = "author", cascade = CascadeType.ALL, orphanRemoval = true)
    private List<Post> posts;

    // getters/setters
}

@Entity
public class Post {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;

    @ManyToOne
    @JoinColumn(name = "author_id") // FK to User
    private User author;

    // getters/setters
}
```

**Input (saving data):**

```java
User user = new User();
user.setUsername("bob");

Post p1 = new Post();
p1.setTitle("First Post");
p1.setAuthor(user);

Post p2 = new Post();
p2.setTitle("Second Post");
p2.setAuthor(user);

user.setPosts(List.of(p1, p2));

userRepository.save(user); // cascades to posts
```

**Database result (simplified):**

- `user` table:
  - `id = 1`, `username = "bob"`
- `post` table:
  - `id = 1`, `title = "First Post"`, `author_id = 1`
  - `id = 2`, `title = "Second Post"`, `author_id = 1`

### 3.2 `@ManyToMany` ŌĆō Many-to-Many Relationship

Example: `Student` can enroll in many `Course`s, and each `Course` can have many `Student`s.

```java
import jakarta.persistence.*;
import java.util.Set;

@Entity
public class Student {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;

    @ManyToMany
    @JoinTable(
        name = "student_course",
        joinColumns = @JoinColumn(name = "student_id"),
        inverseJoinColumns = @JoinColumn(name = "course_id")
    )
    private Set<Course> courses;

    // getters/setters
}

@Entity
public class Course {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;

    @ManyToMany(mappedBy = "courses")
    private Set<Student> students;

    // getters/setters
}
```

**Input (saving data):**

```java
Student s1 = new Student();
s1.setName("Alice");

Student s2 = new Student();
s2.setName("Bob");

Course c1 = new Course();
c1.setTitle("Math");

Course c2 = new Course();
c2.setTitle("Physics");

s1.setCourses(Set.of(c1, c2));
s2.setCourses(Set.of(c1));

studentRepository.save(s1);
studentRepository.save(s2);
```

**Join table `student_course` (simplified):**

| student_id | course_id |
|-----------:|----------:|
| 1          | 1         |
| 1          | 2         |
| 2          | 1         |

---

### 3.3 Other Common Relationships

- `@ManyToOne` ŌĆō Many entities reference one parent (e.g., many `Post`s to one `User`).
- `@OneToOne` ŌĆō One-to-one mapping (e.g., `User` Ōåö `UserProfile`).

---

## Core Ideas Summary

- `@Entity`: marks a class as a JPA entity mapped to a table.
- `@Table`: customizes table name, schema, and constraints.
- Relationships:
  - `@OneToMany` / `@ManyToOne`: parentŌĆōchild relationships (e.g., UserŌĆōPost).
  - `@ManyToMany`: many-to-many via a join table (e.g., StudentŌĆōCourse).
- Spring Data repositories (`JpaRepository`) provide CRUD and query methods for these entities.
