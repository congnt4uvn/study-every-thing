
Here’s a concise summary of the key ideas, with code examples and sample I/O.

---

## 1. What is AOP (Aspect-Oriented Programming)?

- AOP is a way to separate **cross‑cutting concerns** from your main business logic.
- Cross‑cutting concerns are things that affect many parts of the app, such as:
  - Logging
  - Security checks
  - Transaction management
  - Performance monitoring

Instead of repeating this code everywhere, you put it in **aspects**.

---

## 2. @Aspect

- `@Aspect` marks a class as an **aspect**.
- An aspect contains:
  - **Pointcuts**: where to apply
  - **Advice**: what to do

```java
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.springframework.stereotype.Component;

@Aspect
@Component
public class LoggingAspect {

    @Before("execution(* com.example.service.*.*(..))")
    public void logBefore() {
        System.out.println("[LOG] Method in service package is about to run");
    }
}
```

**Effect (conceptual I/O):**

- **Input**: Call `userService.getUser(1)`
- **Output (console)**:
  ```text
  [LOG] Method in service package is about to run
  ```
  then the normal result of `getUser(1)` is returned.

---

## 3. Pointcuts

- A **pointcut** defines *where* the advice should run.
- Common syntax: `execution(modifiers-pattern? return-type-pattern declaring-type-pattern? method-name-pattern(param-pattern) throws-pattern?)`

Example: all methods in `com.example.service` package:

```java
@Pointcut("execution(* com.example.service.*.*(..))")
public void serviceMethods() {}
```

Use it in advice:

```java
@Before("serviceMethods()")
public void logBeforeService() {
    System.out.println("[LOG] Service method called");
}
```

---

## 4. Advice Types

Advice is the **action** that runs at a join point (e.g., method call). Main types:

### 4.1 @Before

Runs **before** the method.

```java
@Before("execution(* com.example.service.UserService.*(..))")
public void beforeUserService() {
    System.out.println("[BEFORE] UserService method");
}
```

**I/O example:**

- Call: `userService.createUser("Alice")`
- Console:
  ```text
  [BEFORE] UserService method
  ```
- Then `createUser("Alice")` executes normally.

---

### 4.2 @AfterReturning

Runs **after** the method successfully returns.

```java
import org.aspectj.lang.annotation.AfterReturning;

@AfterReturning(
    pointcut = "execution(* com.example.service.UserService.getUser(..))",
    returning = "result"
)
public void afterReturningGetUser(Object result) {
    System.out.println("[AFTER RETURNING] Result: " + result);
}
```

**I/O example:**

- Call: `userService.getUser(1)`
- Suppose it returns: `User{id=1, name='Alice'}`
- Console:
  ```text
  [AFTER RETURNING] Result: User{id=1, name='Alice'}
  ```

---

### 4.3 @AfterThrowing

Runs when the method **throws an exception**.

```java
import org.aspectj.lang.annotation.AfterThrowing;

@AfterThrowing(
    pointcut = "execution(* com.example.service.UserService.*(..))",
    throwing = "ex"
)
public void afterThrowing(Exception ex) {
    System.out.println("[AFTER THROWING] Exception: " + ex.getMessage());
}
```

**I/O example:**

- Call: `userService.getUser(-1)`  
- Method throws `IllegalArgumentException("Invalid id")`
- Console:
  ```text
  [AFTER THROWING] Exception: Invalid id
  ```

---

### 4.4 @After (finally)

Runs **after** the method finishes, whether it returns normally or throws an exception (like `finally` in try/catch).

```java
import org.aspectj.lang.annotation.After;

@After("execution(* com.example.service.UserService.*(..))")
public void afterUserService() {
    System.out.println("[AFTER] UserService method finished");
}
```

---

### 4.5 @Around

- Most powerful advice.
- Wraps the method call: can run **before and after**, modify arguments, or even skip the method.
- Uses `ProceedingJoinPoint` to call the original method.

```java
import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.Around;

@Around("execution(* com.example.service.UserService.*(..))")
public Object measureExecutionTime(ProceedingJoinPoint pjp) throws Throwable {
    long start = System.currentTimeMillis();
    System.out.println("[AROUND] Starting: " + pjp.getSignature());

    Object result = pjp.proceed(); // call the original method

    long end = System.currentTimeMillis();
    System.out.println("[AROUND] Finished: " + pjp.getSignature() +
                       " in " + (end - start) + " ms");
    return result;
}
```

**I/O example:**

- Call: `userService.getUser(1)`
- Console:
  ```text
  [AROUND] Starting: User com.example.service.UserService.getUser(..)
  [AROUND] Finished: User com.example.service.UserService.getUser(..) in 12 ms
  ```
- Return value: same as original `getUser(1)`.

---

## 5. Summary

- **AOP**: separates cross‑cutting concerns from business logic.
- **@Aspect**: marks a class that contains cross‑cutting logic.
- **Pointcuts**: define *where* advice applies (e.g., which methods).
- **Advice types**:
  - `@Before`: run before method
  - `@AfterReturning`: after successful return
  - `@AfterThrowing`: after exception
  - `@After`: always after (success or failure)
  - `@Around`: wrap the method, can control execution and measure, modify, or block it.
