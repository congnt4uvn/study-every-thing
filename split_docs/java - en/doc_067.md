
Here’s a concise summary of the key ideas about Spring MVC / Web: request/response mapping, DTOs, and validation.

---

## 1. Request/Response Mapping

Spring MVC maps HTTP requests to controller methods using annotations like `@GetMapping`, `@PostMapping`, etc.

### Example: Simple Controller

```java
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/users")
public class UserController {

    // GET /api/users/1
    @GetMapping("/{id}")
    public UserResponse getUser(@PathVariable Long id) {
        // Normally you’d fetch from DB
        return new UserResponse(id, "Alice", "alice@example.com");
    }

    // POST /api/users
    @PostMapping
    public UserResponse createUser(@RequestBody UserRequest request) {
        // Normally you’d save to DB and return saved entity
        return new UserResponse(1L, request.getName(), request.getEmail());
    }
}
```

### Example Request/Response

**Request:**

```http
POST /api/users
Content-Type: application/json

{
  "name": "Bob",
  "email": "bob@example.com"
}
```

**Response:**

```http
HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 1,
  "name": "Bob",
  "email": "bob@example.com"
}
```

## 2. DTOs (Data Transfer Objects)

DTOs are simple objects used to carry data between layers (e.g., controller ↔ service).  
They help separate API models from internal domain models.

### Example DTOs

```java
public class UserRequest {
    private String name;
    private String email;

    // getters and setters
}

public class UserResponse {
    private Long id;
    private String name;
    private String email;

    public UserResponse(Long id, String name, String email) {
        this.id = id;
        this.name = name;
        this.email = email;
    }

    // getters
}
```

## 3. Validation with `@Valid`, `@NotNull`, etc.

Spring integrates Bean Validation (Jakarta Validation) to validate DTOs.  
You annotate fields with constraints and use `@Valid` in controller methods.

### Example: Validated DTO

```java
import jakarta.validation.constraints.*;

public class UserRequest {

    @NotBlank(message = "Name is required")
    private String name;

    @Email(message = "Email must be valid")
    @NotBlank(message = "Email is required")
    private String email;

    @Min(value = 18, message = "Age must be at least 18")
    private Integer age;

    // getters and setters
}
```

### Example: Controller with `@Valid`

```java
import org.springframework.validation.annotation.Validated;
import jakarta.validation.Valid;

@RestController
@RequestMapping("/api/users")
@Validated
public class UserController {

    @PostMapping
    public UserResponse createUser(@Valid @RequestBody UserRequest request) {
        // If validation fails, Spring throws MethodArgumentNotValidException
        return new UserResponse(1L, request.getName(), request.getEmail());
    }
}
```

### Example Invalid Request / Error Response

**Request:**

```http
POST /api/users
Content-Type: application/json

{
  "name": "",
  "email": "not-an-email",
  "age": 15
}
```

**Typical JSON error response (simplified):**

```json
{
  "timestamp": "2026-05-20T10:00:00Z",
  "status": 400,
  "errors": [
    "Name is required",
    "Email must be valid",
    "Age must be at least 18"
  ]
}
```

---

### Main Ideas

- Spring MVC maps HTTP requests to controller methods using annotations.
- DTOs are used to structure request and response data cleanly.
- Validation is done via annotations (`@NotNull`, `@NotBlank`, `@Email`, `@Min`, etc.) and triggered with `@Valid` in controller method parameters.
- On validation failure, Spring automatically returns a 400 Bad Request with error details (which you can customize).
