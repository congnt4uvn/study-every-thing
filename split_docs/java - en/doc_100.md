
Here’s a concise summary and some concrete Java examples around **designing scalable REST APIs**.

---

## Key Ideas for Scalable REST APIs

1. **Statelessness**
   - Each request contains all information needed (no server-side session).
   - Makes horizontal scaling (adding more servers) easier.

2. **Resource-Oriented Design**
   - Use nouns for resources, HTTP methods for actions:
     - `GET /users` – list users  
     - `GET /users/{id}` – get one user  
     - `POST /users` – create user  
     - `PUT /users/{id}` – replace user  
     - `PATCH /users/{id}` – update part of user  
     - `DELETE /users/{id}` – delete user

3. **Proper Use of HTTP Status Codes**
   - `200 OK`, `201 Created`, `204 No Content`
   - `400 Bad Request`, `401 Unauthorized`, `403 Forbidden`, `404 Not Found`
   - `500 Internal Server Error`, `503 Service Unavailable`

4. **Pagination, Filtering, Sorting**
   - Avoid returning huge lists.
   - Use query params: `GET /users?page=1&size=20&sort=name,asc`

5. **Versioning**
   - Keep backward compatibility.
   - Common patterns:
     - URL: `/api/v1/users`
     - Header: `Accept: application/vnd.myapp.v1+json`

6. **Security**
   - Use HTTPS.
   - Use tokens (e.g., JWT, OAuth2) instead of sessions.
   - Validate and sanitize inputs.

7. **Caching**
   - Use HTTP caching headers: `ETag`, `Cache-Control`, `Last-Modified`.
   - Cache frequently requested data (in-memory, Redis, etc.).

8. **Rate Limiting & Throttling**
   - Protect against abuse and overload.
   - Limit requests per user/IP/token.

9. **Observability**
   - Logging, metrics, tracing.
   - Health checks: `/actuator/health` (Spring Boot) or similar.

10. **Scalability Architecture**
    - Load balancer in front of multiple stateless API instances.
    - Separate database, cache, message queues.
    - Use asynchronous processing for heavy tasks.

## Simple Java REST API Example (Spring Boot)

### 1. Basic Controller

```java
// Input DTO
public class CreateUserRequest {
    private String name;
    private String email;

    // getters and setters
}

// Output DTO
public class UserResponse {
    private Long id;
    private String name;
    private String email;

    // constructor, getters, setters
}
```

```java
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.net.URI;
import java.util.*;

@RestController
@RequestMapping("/api/v1/users")
public class UserController {

    // In-memory store for example only
    private final Map<Long, UserResponse> users = new HashMap<>();
    private long idSequence = 1L;

    // GET /api/v1/users?page=0&size=2
    @GetMapping
    public ResponseEntity<List<UserResponse>> listUsers(
            @RequestParam(defaultValue = "0") int page,
            @RequestParam(defaultValue = "10") int size) {

        List<UserResponse> all = new ArrayList<>(users.values());
        int from = page * size;
        int to = Math.min(from + size, all.size());

        if (from >= all.size()) {
            return ResponseEntity.ok(Collections.emptyList());
        }

        List<UserResponse> pageContent = all.subList(from, to);
        return ResponseEntity.ok(pageContent);
    }

    // GET /api/v1/users/{id}
    @GetMapping("/{id}")
    public ResponseEntity<UserResponse> getUser(@PathVariable Long id) {
        UserResponse user = users.get(id);
        if (user == null) {
            return ResponseEntity.notFound().build();
        }
        return ResponseEntity.ok(user);
    }

    // POST /api/v1/users
    @PostMapping
    public ResponseEntity<UserResponse> createUser(@RequestBody CreateUserRequest request) {
        Long newId = idSequence++;
        UserResponse user = new UserResponse(newId, request.getName(), request.getEmail());
        users.put(newId, user);

        URI location = URI.create("/api/v1/users/" + newId);
        return ResponseEntity.created(location).body(user);
    }

    // DELETE /api/v1/users/{id}
    @DeleteMapping("/{id}")
    public ResponseEntity<Void> deleteUser(@PathVariable Long id) {
        if (!users.containsKey(id)) {
            return ResponseEntity.notFound().build();
        }
        users.remove(id);
        return ResponseEntity.noContent().build();
    }
}
```

### 2. Example Requests and Responses

#### Create User

**Request**

```http
POST /api/v1/users
Content-Type: application/json

{
  "name": "Alice",
  "email": "alice@example.com"
}
```

**Response**

```http
HTTP/1.1 201 Created
Location: /api/v1/users/1
Content-Type: application/json

{
  "id": 1,
  "name": "Alice",
  "email": "alice@example.com"
}
```

#### Get User

**Request**

```http
GET /api/v1/users/1
Accept: application/json
```

**Response**

```http
HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 1,
  "name": "Alice",
  "email": "alice@example.com"
}
```

#### List Users with Pagination

**Request**

```http
GET /api/v1/users?page=0&size=2
Accept: application/json
```

**Response**

```http
HTTP/1.1 200 OK
Content-Type: application/json

[
  {
    "id": 1,
    "name": "Alice",
    "email": "alice@example.com"
  },
  {
    "id": 2,
    "name": "Bob",
    "email": "bob@example.com"
  }
]
```
