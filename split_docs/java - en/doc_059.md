
Here’s a concise summary of the key ideas around **Spring Core: `ApplicationContext`, profiles, and property sources`**, with code examples and sample input/output.

---

## 1. ApplicationContext

**What it is:**  
`ApplicationContext` is the central interface in Spring for:
- Creating and managing beans (IoC container)
- Dependency injection
- Handling configuration (XML, Java config, annotations)
- Integrating with Spring features (AOP, events, etc.)

**Typical usage:**

### Java-based configuration

```java
// AppConfig.java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class AppConfig {

    @Bean
    public MessageService messageService() {
        return new MessageService();
    }
}

// MessageService.java
public class MessageService {
    public String getMessage() {
        return "Hello from Spring!";
    }
}

// Main.java
import org.springframework.context.ApplicationContext;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class Main {
    public static void main(String[] args) {
        ApplicationContext ctx =
                new AnnotationConfigApplicationContext(AppConfig.class);

        MessageService service = ctx.getBean(MessageService.class);
        System.out.println(service.getMessage());
    }
}
```

**Output:**
```text
Hello from Spring!
```

## 2. Profiles

**What they are:**  
Profiles let you define different bean configurations for different environments, such as:
- `dev`
- `test`
- `prod`

You can activate a profile to load only the beans for that environment.

### Example: Profile-specific beans

```java
// DataSourceConfig.java
import org.springframework.context.annotation.*;

@Configuration
public class DataSourceConfig {

    @Bean
    @Profile("dev")
    public String devDataSource() {
        return "Connected to DEV database";
    }

    @Bean
    @Profile("prod")
    public String prodDataSource() {
        return "Connected to PROD database";
    }
}

// MainProfiles.java
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class MainProfiles {
    public static void main(String[] args) {
        AnnotationConfigApplicationContext ctx =
                new AnnotationConfigApplicationContext();

        // Activate a profile
        ctx.getEnvironment().setActiveProfiles("dev");
        ctx.register(DataSourceConfig.class);
        ctx.refresh();

        String ds = ctx.getBean(String.class);
        System.out.println(ds);
    }
}
```

**Output when `dev` is active:**
```text
Connected to DEV database
```

If you change to:
```java
ctx.getEnvironment().setActiveProfiles("prod");
```

**Output:**
```text
Connected to PROD database
```

**Other ways to set active profiles:**
- JVM argument: `-Dspring.profiles.active=dev`
- Environment variable: `SPRING_PROFILES_ACTIVE=dev`
- In `application.properties` (Spring Boot):  
  `spring.profiles.active=dev`

## 3. Property Sources

**What they are:**  
Property sources provide external configuration (key–value pairs) from:
- `.properties` files
- `.yml` files (Spring Boot)
- Environment variables
- System properties
- Custom sources

They allow you to keep configuration outside the code and inject values into beans.

### Example: Using `@PropertySource` and `@Value`

**File:** `app.properties`
```properties
app.name=My Spring App
app.version=1.0.0
```

**Java config:**

```java
// AppConfigProperties.java
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.*;

@Configuration
@PropertySource("classpath:app.properties")
public class AppConfigProperties {

    @Value("${app.name}")
    private String appName;

    @Value("${app.version}")
    private String appVersion;

    @Bean
    public AppInfo appInfo() {
        return new AppInfo(appName, appVersion);
    }
}

// AppInfo.java
public class AppInfo {
    private final String name;
    private final String version;

    public AppInfo(String name, String version) {
        this.name = name;
        this.version = version;
    }

    public String info() {
        return name + " v" + version;
    }
}

// MainProperties.java
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class MainProperties {
    public static void main(String[] args) {
        AnnotationConfigApplicationContext ctx =
                new AnnotationConfigApplicationContext(AppConfigProperties.class);

        AppInfo appInfo = ctx.getBean(AppInfo.class);
        System.out.println(appInfo.info());
    }
}
```

**Output:**
```text
My Spring App v1.0.0
```

### Example: Environment and PropertySources

You can also access properties via `Environment`:

```java
import org.springframework.context.annotation.*;
import org.springframework.core.env.Environment;

@Configuration
@PropertySource("classpath:app.properties")
public class EnvConfig {

    @Bean
    public String appInfo(Environment env) {
        String name = env.getProperty("app.name");
        String version = env.getProperty("app.version");
        return name + " v" + version;
    }
}
```

---

## Summary of Main Ideas

- **ApplicationContext**: Core Spring container that manages beans and configuration.
- **Profiles**: Allow environment-specific bean definitions (`dev`, `test`, `prod`), activated via code, JVM args, env vars, or properties.
- **Property Sources**: External configuration (files, env vars, etc.) injected into beans using `@PropertySource`, `@Value`, or `Environment`.

These three together give you:
- Centralized bean management (`ApplicationContext`)
- Flexible environment-specific configuration (profiles)
- Externalized, easily changeable settings (property sources).
