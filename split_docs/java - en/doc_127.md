## Key Point

- In the Java ecosystem, **API tools** like **Postman** and **curl** are commonly used to:
  - Test REST APIs
  - Debug HTTP requests/responses
  - Inspect headers, status codes, and payloads
- These tools are external to Java but are essential when developing and testing Java-based web services (e.g., Spring Boot APIs).

---

## Example: Testing a Java REST API

Assume you have a Java Spring Boot endpoint:

```java
// GET /hello
@GetMapping("/hello")
public String hello() {
    return "Hello, World!";
}
```

### Using curl

**Command (input):**
```bash
curl -X GET http://localhost:8080/hello
```

**Output:**
```text
Hello, World!
```

---

Assume a POST endpoint:

```java
// POST /sum
@PostMapping("/sum")
public int sum(@RequestBody List<Integer> numbers) {
    return numbers.stream().mapToInt(Integer::intValue).sum();
}
```

### Using curl

**Command (input):**
```bash
curl -X POST http://localhost:8080/sum \
  -H "Content-Type: application/json" \
  -d "[1, 2, 3, 4]"
```

**Output:**
```text
10
```

---

### Using Postman (conceptual)

For the same `/sum` endpoint in Postman:

1. Method: `POST`
2. URL: `http://localhost:8080/sum`
3. Headers: `Content-Type: application/json`
4. Body (raw, JSON):
   ```json
   [1, 2, 3, 4]
   ```

**Response body:**
```text
10
```

---

**Main idea:**  
Postman and curl are key tools in the Java ecosystem for sending HTTP requests to Java APIs, verifying behavior, and debugging during development.
