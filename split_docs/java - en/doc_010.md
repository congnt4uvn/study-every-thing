
Here’s a concise summary of Java **records**, plus code examples with inputs/outputs.

---

## What are records?

- **Records** are a special kind of class introduced in **Java 16** (preview earlier).
- They are meant for **immutable data carriers** (like simple DTOs).
- The compiler automatically generates:
  - `private final` fields
  - constructor
  - `equals()`, `hashCode()`
  - `toString()`
  - accessors (no `getX()`, just `x()`)

---

## Basic syntax

```java
public record Point(int x, int y) { }
```

This single line is roughly equivalent to:

```java
public final class Point {
    private final int x;
    private final int y;

    public Point(int x, int y) {
        this.x = x;
        this.y = y;
    }

    public int x() { return x; }
    public int y() { return y; }

    @Override
    public boolean equals(Object o) { /* auto-generated */ }

    @Override
    public int hashCode() { /* auto-generated */ }

    @Override
    public String toString() { /* auto-generated */ }
}
```

## Example with input/output

### Record definition

```java
public record Person(String name, int age) { }
```

### Usage

```java
public class Main {
    public static void main(String[] args) {
        Person p1 = new Person("Alice", 30);
        Person p2 = new Person("Alice", 30);
        Person p3 = new Person("Bob", 25);

        System.out.println(p1.name());   // accessor
        System.out.println(p1.age());

        System.out.println(p1);          // toString
        System.out.println(p1.equals(p2));
        System.out.println(p1.equals(p3));
    }
}
```

**Output:**

```text
Alice
30
Person[name=Alice, age=30]
true
false
```

## Custom behavior in records

You can still add methods and validation.

```java
public record Rectangle(int width, int height) {
    public Rectangle {
        if (width <= 0 || height <= 0) {
            throw new IllegalArgumentException("Width and height must be positive");
        }
    }

    public int area() {
        return width * height;
    }
}
```

### Usage

```java
public class Main {
    public static void main(String[] args) {
        Rectangle r = new Rectangle(3, 4);
        System.out.println(r.area());

        // This will throw IllegalArgumentException
        Rectangle bad = new Rectangle(0, 5);
    }
}
```

**Output:**

```text
12
Exception in thread "main" java.lang.IllegalArgumentException: Width and height must be positive
    at Rectangle.<init>(Rectangle.java:...)
    ...
```

---

## Key points

- Records are **final**, fields are **final**, and they are best for **immutable data**.
- They reduce boilerplate for simple data classes.
- They work well with pattern matching and newer Java features.
- Not suitable when:
  - You need mutable fields.
  - You need complex inheritance (records cannot extend other classes, only `java.lang.Record`).
