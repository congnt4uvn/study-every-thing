## Key Concepts

1. **`@RestController`**
   - Combines `@Controller` + `@ResponseBody`.
   - Every handler method returns data directly (usually JSON) instead of rendering a view (like JSP/Thymeleaf).

2. **`@RequestMapping`**
   - General mapping annotation for HTTP requests.
   - Can be used at class level (base path) and method level (specific path + HTTP method).
   - Can specify: `path`, `method`, `params`, `headers`, `produces`, `consumes`, etc.

3. **Shortcut Annotations**
   - `@GetMapping` Ôćĺ `@RequestMapping(method = RequestMethod.GET)`
   - `@PostMapping` Ôćĺ `@RequestMapping(method = RequestMethod.POST)`
   - `@PutMapping` Ôćĺ `@RequestMapping(method = RequestMethod.PUT)`
   - `@DeleteMapping` Ôćĺ `@RequestMapping(method = RequestMethod.DELETE)`
   - `@PatchMapping` Ôćĺ `@RequestMapping(method = RequestMethod.PATCH)`

These are specialized, more readable versions of `@RequestMapping` for common HTTP methods.

## Basic REST Controller Example

```java
import org.springframework.web.bind.annotation.*;

import java.util.*;

@RestController
@RequestMapping("/api/users") // base path for all methods in this controller
public class UserController {

    // In-memory "database"
    private Map<Long, String> users = new HashMap<>();

    public UserController() {
        users.put(1L, "Alice");
        users.put(2L, "Bob");
    }

    // GET /api/users
    @GetMapping
    public Map<Long, String> getAllUsers() {
        return users;
    }

    // GET /api/users/{id}
    @GetMapping("/{id}")
    public String getUserById(@PathVariable Long id) {
        return users.get(id); // returns null if not found (in real code, handle 404)
    }

    // POST /api/users
    @PostMapping
    public String createUser(@RequestParam String name) {
        long newId = users.keySet().stream().max(Long::compareTo).orElse(0L) + 1;
        users.put(newId, name);
        return "Created user with id " + newId;
    }

    // PUT /api/users/{id}
    @PutMapping("/{id}")
    public String updateUser(@PathVariable Long id, @RequestParam String name) {
        users.put(id, name);
        return "Updated user " + id;
    }

    // DELETE /api/users/{id}
    @DeleteMapping("/{id}")
    public String deleteUser(@PathVariable Long id) {
        users.remove(id);
        return "Deleted user " + id;
    }
}
```

## Example Requests and Responses

### 1. Get all users

**Request**

```http
GET /api/users
Accept: application/json
```

**Response (JSON)**

```json
{
  "1": "Alice",
  "2": "Bob"
}
```

---

### 2. Get user by ID

**Request**

```http
GET /api/users/1
Accept: text/plain
```

**Response**

```text
Alice
```

---

### 3. Create a new user

**Request**

```http
POST /api/users?name=Charlie
Accept: text/plain
```

**Response**

```text
Created user with id 3
```

---

### 4. Update a user

**Request**

```http
PUT /api/users/2?name=BobUpdated
Accept: text/plain
```

**Response**

```text
Updated user 2
```

---

### 5. Delete a user

**Request**

```http
DELETE /api/users/1
Accept: text/plain
```

**Response**

```text
Deleted user 1
```

---

**Main idea:**  
Use `@RestController` to build REST APIs in Spring MVC. Map HTTP requests to Java methods using `@RequestMapping` or the more specific `@GetMapping`, `@PostMapping`, etc., and return data (usually JSON) directly to the client.
