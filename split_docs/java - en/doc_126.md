## Key Ideas

1. **SLF4J (Simple Logging Facade for Java)**  
   - A *logging facade* (abstraction) that lets you write logging code independent of the underlying logging implementation.  
   - You code against `org.slf4j.Logger`, and at runtime you plug in a backend like Logback, Log4j2, etc.

2. **Logback**  
   - A popular logging implementation that works very well with SLF4J.  
   - Configured usually via `logback.xml` (or `logback-spring.xml` in Spring Boot).  
   - Supports advanced features: rolling files, different appenders, pattern layouts, async logging, etc.

3. **Log Levels**  
   Common levels (from most to least severe):
   - `ERROR` – serious problems; the application might not continue correctly.
   - `WARN` – something unexpected, but the app can still run.
   - `INFO` – high-level application events (startup, shutdown, key business events).
   - `DEBUG` – detailed information for debugging.
   - `TRACE` – very fine-grained details, usually too verbose for production.

   You typically:
   - Use `INFO` for business events.
   - Use `DEBUG`/`TRACE` during development or when diagnosing issues.
   - Use `WARN`/`ERROR` for problems.

4. **Correlation IDs**  
   - A unique ID attached to each request or transaction.  
   - Included in every log line related to that request.  
   - Makes it easy to trace a single request across multiple services and log files.  
   - Often stored in MDC (Mapped Diagnostic Context) so it’s automatically added to log patterns.

## Basic SLF4J + Logback Example

### Maven Dependencies (example)

```xml
<dependencies>
    <!-- SLF4J API -->
    <dependency>
        <groupId>org.slf4j</groupId>
        <artifactId>slf4j-api</artifactId>
        <version>2.0.13</version>
    </dependency>

    <!-- Logback implementation -->
    <dependency>
        <groupId>ch.qos.logback</groupId>
        <artifactId>logback-classic</artifactId>
        <version>1.5.6</version>
    </dependency>
</dependencies>
```

### Java Logging Usage

```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class LoggingExample {
    private static final Logger log = LoggerFactory.getLogger(LoggingExample.class);

    public static void main(String[] args) {
        log.trace("This is a TRACE message");
        log.debug("This is a DEBUG message");
        log.info("Application started with args: {}", (Object) args);
        log.warn("This is a WARN message");
        log.error("This is an ERROR message");
    }
}
```

### Example `logback.xml`

```xml
<configuration>
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <!-- %d = date, %-5level = log level, %logger = logger name, %msg = message -->
            <pattern>%d{yyyy-MM-dd HH:mm:ss} %-5level [%thread] %logger - %msg%n</pattern>
        </encoder>
    </appender>

    <root level="INFO">
        <appender-ref ref="CONSOLE" />
    </root>
</configuration>
```

### Example Output

Running `LoggingExample` with `INFO` root level:

```text
2026-05-20 10:00:00 INFO  [main] LoggingExample - Application started with args: []
2026-05-20 10:00:00 WARN  [main] LoggingExample - This is a WARN message
2026-05-20 10:00:00 ERROR [main] LoggingExample - This is an ERROR message
```

`TRACE` and `DEBUG` messages are not shown because the root level is `INFO`.

## Correlation ID with MDC

### Java Code

```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.slf4j.MDC;

import java.util.UUID;

public class CorrelationIdExample {
    private static final Logger log = LoggerFactory.getLogger(CorrelationIdExample.class);

    public static void main(String[] args) {
        // Generate a correlation ID for this "request"
        String correlationId = UUID.randomUUID().toString();
        MDC.put("correlationId", correlationId);

        try {
            log.info("Handling request");
            doWork();
        } finally {
            // Clean up MDC to avoid leaking data between threads/requests
            MDC.clear();
        }
    }

    private static void doWork() {
        log.debug("Doing some work inside the request");
    }
}
```

### `logback.xml` with Correlation ID

```xml
<configuration>
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <!-- Include correlationId from MDC: %X{correlationId} -->
            <pattern>%d{HH:mm:ss.SSS} %-5level [%thread] [%X{correlationId}] %logger - %msg%n</pattern>
        </encoder>
    </appender>

    <root level="DEBUG">
        <appender-ref ref="CONSOLE" />
    </root>
</configuration>
```

### Example Output

```text
10:05:00.123 INFO  [main] [f3a1c0b2-9d4e-4a1b-8c2e-123456789abc] CorrelationIdExample - Handling request
10:05:00.124 DEBUG [main] [f3a1c0b2-9d4e-4a1b-8c2e-123456789abc] CorrelationIdExample - Doing some work inside the request
```

All log lines for that request share the same `correlationId`, making it easy to trace.
