
Here’s a concise overview of the main Java collection interfaces you listed, with small code examples and sample input/output.

---

## 1. `List`
- Ordered collection (sequence).
- Allows duplicate elements.
- Access by index.

**Common implementations:** `ArrayList`, `LinkedList`

```java
import java.util.*;

public class ListExample {
    public static void main(String[] args) {
        List<String> names = new ArrayList<>();
        names.add("Alice");
        names.add("Bob");
        names.add("Alice"); // duplicate allowed

        System.out.println(names);      // Output: [Alice, Bob, Alice]
        System.out.println(names.get(1)); // Output: Bob
    }
}
```

---

## 2. `Set`
- Unordered (no guaranteed order, except some implementations).
- No duplicate elements.

**Common implementations:** `HashSet`, `LinkedHashSet`, `TreeSet`

```java
import java.util.*;

public class SetExample {
    public static void main(String[] args) {
        Set<Integer> numbers = new HashSet<>();
        numbers.add(10);
        numbers.add(20);
        numbers.add(10); // duplicate ignored

        System.out.println(numbers); // Possible output: [20, 10]
        System.out.println(numbers.contains(20)); // Output: true
    }
}
```

---

## 3. `Map`
- Key–value pairs.
- Keys are unique; values can be duplicated.
- Not a `Collection`, but part of the collections framework.

**Common implementations:** `HashMap`, `LinkedHashMap`, `TreeMap`

```java
import java.util.*;

public class MapExample {
    public static void main(String[] args) {
        Map<String, Integer> ages = new HashMap<>();
        ages.put("Alice", 25);
        ages.put("Bob", 30);
        ages.put("Alice", 26); // overwrites previous value for "Alice"

        System.out.println(ages); // Output: {Alice=26, Bob=30} (order not guaranteed)
        System.out.println(ages.get("Alice")); // Output: 26
    }
}
```

---

## 4. `Queue`
- Typically FIFO (First-In-First-Out).
- Used for processing elements in order.

**Common implementations:** `LinkedList`, `PriorityQueue`

```java
import java.util.*;

public class QueueExample {
    public static void main(String[] args) {
        Queue<String> queue = new LinkedList<>();
        queue.add("first");
        queue.add("second");
        queue.add("third");

        System.out.println(queue);      // Output: [first, second, third]
        System.out.println(queue.poll()); // Output: first (removed)
        System.out.println(queue);      // Output: [second, third]
    }
}
```

---

## 5. `Deque` (Double-Ended Queue)
- Elements can be added/removed from both ends.
- Can be used as a queue (FIFO) or stack (LIFO).

**Common implementations:** `ArrayDeque`, `LinkedList`

```java
import java.util.*;

public class DequeExample {
    public static void main(String[] args) {
        Deque<Integer> deque = new ArrayDeque<>();
        deque.addFirst(1); // front
        deque.addLast(2);  // back
        deque.addLast(3);

        System.out.println(deque);        // Output: [1, 2, 3]
        System.out.println(deque.removeLast()); // Output: 3
        System.out.println(deque);        // Output: [1, 2]
    }
}
```

---

**Summary of main ideas:**
- `List`: ordered, index-based, duplicates allowed.
- `Set`: no duplicates, no index, order depends on implementation.
- `Map`: key–value pairs, unique keys.
- `Queue`: process elements in FIFO order.
- `Deque`: double-ended queue, supports both FIFO and LIFO patterns.
