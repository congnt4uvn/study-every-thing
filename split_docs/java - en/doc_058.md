
Here’s a concise summary of the key ideas, with code examples and sample I/O where it makes sense.

---

## 1. Bean Scopes

Bean scope defines **how long** a bean lives and **how it is shared**.

Common scopes (Spring Core / typical Spring app):

1. **singleton** (default)  
   - One instance per Spring container.  
   - Every `getBean()` returns the same object.

2. **prototype**  
   - New instance every time you request it.

3. **request** (web)  
   - One instance per HTTP request.

4. **session** (web)  
   - One instance per HTTP session.

### Example: Singleton vs Prototype

```java
import org.springframework.context.annotation.Scope;
import org.springframework.stereotype.Component;

@Component
@Scope("singleton") // default; could be omitted
public class SingletonBean {
}

@Component
@Scope("prototype")
public class PrototypeBean {
}
```

Test code:

```java
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class ScopeTest {
    public static void main(String[] args) {
        AnnotationConfigApplicationContext ctx =
                new AnnotationConfigApplicationContext("com.example");

        SingletonBean s1 = ctx.getBean(SingletonBean.class);
        SingletonBean s2 = ctx.getBean(SingletonBean.class);

        PrototypeBean p1 = ctx.getBean(PrototypeBean.class);
        PrototypeBean p2 = ctx.getBean(PrototypeBean.class);

        System.out.println("Singleton: " + (s1 == s2));
        System.out.println("Prototype: " + (p1 == p2));

        ctx.close();
    }
}
```

**Output:**

```text
Singleton: true
Prototype: false
```

## 2. Bean Lifecycle

Spring bean lifecycle (simplified):

1. **Instantiation** – Spring creates the object.
2. **Dependency Injection** – Spring sets fields/constructor args.
3. **Initialization** –  
   - `@PostConstruct` methods  
   - or `InitializingBean.afterPropertiesSet()`  
   - or custom init-method.
4. **Usage** – Bean is used in the application.
5. **Destruction** –  
   - `@PreDestroy` methods  
   - or `DisposableBean.destroy()`  
   - or custom destroy-method (for singleton beans when context closes).

---

## 3. `@PostConstruct`

- Marks a method to run **after dependency injection** but **before** the bean is used.
- Good for initialization that needs injected dependencies.

```java
import jakarta.annotation.PostConstruct;
import org.springframework.stereotype.Component;

@Component
public class MyService {

    private final MyRepository repo;

    public MyService(MyRepository repo) {
        this.repo = repo;
    }

    @PostConstruct
    public void init() {
        System.out.println("MyService initialized with repo = " + repo);
        // e.g., warm up cache, validate config, etc.
    }
}
```

**Behavior:**  
When the application context starts and creates `MyService`, it injects `MyRepository`, then calls `init()` once.

---

## 4. `@PreDestroy`

- Marks a method to run **before the bean is destroyed** (usually when the application context is closed).
- Used for cleanup: closing connections, releasing resources, etc.
- Works reliably for **singleton** beans; prototype beans are not automatically destroyed by the container.

```java
import jakarta.annotation.PreDestroy;
import org.springframework.stereotype.Component;

@Component
public class ConnectionManager {

    @PreDestroy
    public void cleanup() {
        System.out.println("ConnectionManager cleanup: closing connections");
        // close DB connections, thread pools, etc.
    }
}
```

**Behavior:**  
When the Spring context is closed (e.g., on application shutdown), `cleanup()` is called.

## 5. Combined Example

```java
import jakarta.annotation.PostConstruct;
import jakarta.annotation.PreDestroy;
import org.springframework.context.annotation.Scope;
import org.springframework.stereotype.Component;

@Component
@Scope("singleton")
public class ExampleBean {

    public ExampleBean() {
        System.out.println("Constructor: ExampleBean created");
    }

    @PostConstruct
    public void init() {
        System.out.println("@PostConstruct: ExampleBean initialized");
    }

    @PreDestroy
    public void destroy() {
        System.out.println("@PreDestroy: ExampleBean will be destroyed");
    }
}
```

Runner:

```java
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class LifecycleDemo {
    public static void main(String[] args) {
        AnnotationConfigApplicationContext ctx =
                new AnnotationConfigApplicationContext("com.example");

        ExampleBean bean = ctx.getBean(ExampleBean.class);
        System.out.println("Using bean: " + bean);

        ctx.close();
    }
}
```

**Possible output:**

```text
Constructor: ExampleBean created
@PostConstruct: ExampleBean initialized
Using bean: com.example.ExampleBean@5e2de80c
@PreDestroy: ExampleBean will be destroyed
```

---

**In summary:**

- **Bean scope** controls how many instances exist and where they live (singleton, prototype, request, session).
- **Lifecycle**: create → inject dependencies → initialize (`@PostConstruct`) → use → destroy (`@PreDestroy`).
- `@PostConstruct` is for initialization after injection; `@PreDestroy` is for cleanup before the bean is removed.
