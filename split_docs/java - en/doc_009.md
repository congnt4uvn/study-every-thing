
Here’s a concise summary of the Java `var` feature and how it works.

---

## What is `var` in Java?

- Introduced in **Java 10**.
- Allows **local variable type inference**.
- The compiler infers the variable’s type from the initializer.
- It does **not** make Java dynamically typed; types are still checked at compile time.

---

## Where can you use `var`?

You can use `var` for:

1. **Local variables with initializers**
2. **For-each loop variables**
3. **Traditional for-loop index variables**

You **cannot** use `var` for:
- Fields (class/instance variables)
- Method parameters
- Method return types
- Variables without an initializer

---

## Basic examples

### 1. Local variable with initializer

```java
public class VarExample {
    public static void main(String[] args) {
        var message = "Hello, Java 10";  // inferred as String
        var number = 42;                 // inferred as int
        var pi = 3.14;                   // inferred as double
        var list = java.util.List.of(1, 2, 3); // inferred as List<Integer>

        System.out.println(message);
        System.out.println(number);
        System.out.println(pi);
        System.out.println(list);
    }
}
```

**Output:**
```
Hello, Java 10
42
3.14
[1, 2, 3]
```

---

### 2. For-each loop

```java
import java.util.List;

public class VarForEach {
    public static void main(String[] args) {
        var names = List.of("Alice", "Bob", "Charlie"); // List<String>

        for (var name : names) { // name is inferred as String
            System.out.println(name.toUpperCase());
        }
    }
}
```

**Output:**
```
ALICE
BOB
CHARLIE
```

---

### 3. Traditional for loop

```java
public class VarForLoop {
    public static void main(String[] args) {
        for (var i = 0; i < 3; i++) { // i is inferred as int
            System.out.println("i = " + i);
        }
    }
}
```

**Output:**
```
i = 0
i = 1
i = 2
```

---

## Invalid usages (won’t compile)

```java
// 1. No initializer
var x;              // ERROR: cannot use 'var' without initializer

// 2. As a field
class Test {
    var field = 10; // ERROR: 'var' not allowed for fields
}

// 3. As parameter or return type
var method(var param) { // ERROR: not allowed
    return param;
}
```

---

## When is `var` useful?

- When the type is **obvious** from the right-hand side:
  ```java
  var stream = list.stream(); // clear from context
  ```
- When the type is **long or verbose**:
  ```java
  Map<String, List<Integer>> map = new HashMap<>();
  var map2 = new HashMap<String, List<Integer>>();
  ```

Use `var` to improve readability, but avoid it when it makes the code harder to understand.
