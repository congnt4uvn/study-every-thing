
Here’s a concise summary of Kubernetes basics with simple Java-related examples.

---

## 1. Pods

- Smallest deployable unit in Kubernetes.
- Usually runs **one container** (e.g., a Java Spring Boot app), but can run multiple tightly coupled containers.
- Pods are **ephemeral**: they can be killed and recreated.

### Example Pod (running a Java app)

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: java-app-pod
spec:
  containers:
    - name: java-app
      image: myrepo/java-app:1.0
      ports:
        - containerPort: 8080
```

**Behavior (conceptual I/O):**

- Input: `kubectl apply -f pod.yaml`
- Output: Pod created, app accessible (internally) at `java-app-pod:8080` (via cluster DNS or direct IP).

---

## 2. Deployments

- A **higher-level controller** that manages Pods.
- Ensures a desired number of replicas (e.g., 3 instances of your Java app).
- Handles **rolling updates** and **rollbacks**.

### Example Deployment (Java app with 3 replicas)

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: java-app-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: java-app
  template:
    metadata:
      labels:
        app: java-app
    spec:
      containers:
        - name: java-app
          image: myrepo/java-app:1.0
          ports:
            - containerPort: 8080
```

**Behavior (conceptual I/O):**

- Input: `kubectl apply -f deployment.yaml`
- Output:
  - 3 Pods created, all running the same Java app.
  - If one Pod dies, Deployment creates a new one automatically.

---

## 3. Services

- Provide a **stable network endpoint** to access Pods.
- Load-balance traffic across Pods that match a label selector.
- Types:
  - `ClusterIP` (default, internal only)
  - `NodePort` (exposes on each node’s IP and a port)
  - `LoadBalancer` (cloud load balancer, external access)

### Example Service (exposing the Java Deployment)

```yaml
apiVersion: v1
kind: Service
metadata:
  name: java-app-service
spec:
  type: ClusterIP
  selector:
    app: java-app
  ports:
    - port: 80        # service port
      targetPort: 8080  # container port
```

**Behavior (conceptual I/O):**

- Input: `kubectl apply -f service.yaml`
- Output:
  - Service created at `java-app-service:80`.
  - Requests to `java-app-service:80` are load-balanced to one of the Pods’ `8080` ports.

---

## 4. Simple Java Application Example

### Java (Spring Boot) REST Controller

```java
@RestController
public class HelloController {

    @GetMapping("/hello")
    public String hello() {
        return "Hello from Kubernetes Java app!";
    }
}
```

**Example HTTP I/O (inside cluster):**

- Request: `GET http://java-app-service/hello`
- Response: `Hello from Kubernetes Java app!`

---

## Key Ideas

- **Pod**: runs your Java container; ephemeral, not directly stable for clients.
- **Deployment**: manages multiple Pods, scaling and updates.
- **Service**: stable access point and load balancer for Pods.
