## 1. What is `CompletableFuture`?

`CompletableFuture` is a powerful class for asynchronous programming in Java. It lets you:

- Run tasks asynchronously (non-blocking)
- Chain operations into pipelines
- Combine multiple async tasks
- Handle errors in a functional style

---

## 2. Async Pipelines

You can create a pipeline of operations that run one after another, asynchronously.

### Example: Simple async pipeline

```java
import java.util.concurrent.CompletableFuture;

public class AsyncPipelineExample {
    public static void main(String[] args) {
        CompletableFuture<Void> pipeline =
            CompletableFuture.supplyAsync(() -> {
                // Step 1: async computation
                System.out.println("Step 1: Fetching data...");
                sleep(500);
                return 10;
            })
            .thenApply(result -> {
                // Step 2: transform result
                System.out.println("Step 2: Transforming data...");
                return result * 2;
            })
            .thenAccept(finalResult -> {
                // Step 3: consume result
                System.out.println("Step 3: Final result = " + finalResult);
            });

        // Wait for pipeline to finish (only in demo / main)
        pipeline.join();
    }

    private static void sleep(long ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
    }
}
```

**Possible console output:**

```
Step 1: Fetching data...
Step 2: Transforming data...
Step 3: Final result = 20
```

Key methods:
- `supplyAsync(Supplier)` – start async computation that returns a value
- `thenApply(Function)` – transform the result
- `thenAccept(Consumer)` – consume the result (no return)

## 3. Composition (Combining Futures)

You can combine multiple async tasks in different ways.

### 3.1 `thenCompose` – dependent tasks (flatMap)

Use when the second async task depends on the result of the first.

```java
import java.util.concurrent.CompletableFuture;

public class ThenComposeExample {
    public static void main(String[] args) {
        CompletableFuture<String> userFuture =
            CompletableFuture.supplyAsync(() -> {
                System.out.println("Fetching userId...");
                sleep(300);
                return 42; // userId
            })
            .thenCompose(userId -> getUserNameAsync(userId));

        String userName = userFuture.join();
        System.out.println("User name: " + userName);
    }

    private static CompletableFuture<String> getUserNameAsync(int userId) {
        return CompletableFuture.supplyAsync(() -> {
            System.out.println("Fetching user name for id " + userId + "...");
            sleep(300);
            return "Alice-" + userId;
        });
    }

    private static void sleep(long ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
    }
}
```

**Possible output:**

```
Fetching userId...
Fetching user name for id 42...
User name: Alice-42
```

### 3.2 `thenCombine` – independent tasks, combine results

Use when two tasks can run in parallel and you want to merge their results.

```java
import java.util.concurrent.CompletableFuture;

public class ThenCombineExample {
    public static void main(String[] args) {
        CompletableFuture<Integer> priceFuture =
            CompletableFuture.supplyAsync(() -> {
                System.out.println("Fetching price...");
                sleep(400);
                return 100;
            });

        CompletableFuture<Integer> discountFuture =
            CompletableFuture.supplyAsync(() -> {
                System.out.println("Fetching discount...");
                sleep(300);
                return 20;
            });

        CompletableFuture<Integer> finalPriceFuture =
            priceFuture.thenCombine(discountFuture, (price, discount) -> {
                System.out.println("Combining price and discount...");
                return price - discount;
            });

        Integer finalPrice = finalPriceFuture.join();
        System.out.println("Final price: " + finalPrice);
    }

    private static void sleep(long ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
    }
}
```

**Possible output:**

```
Fetching price...
Fetching discount...
Combining price and discount...
Final price: 80
```

### 3.3 `allOf` / `anyOf` – many tasks

- `allOf` waits for all futures to complete.
- `anyOf` completes when the first future completes.

```java
import java.util.List;
import java.util.concurrent.CompletableFuture;
import java.util.stream.Collectors;
import java.util.stream.IntStream;

public class AllOfExample {
    public static void main(String[] args) {
        List<CompletableFuture<Integer>> futures =
            IntStream.range(1, 4)
                     .mapToObj(i -> CompletableFuture.supplyAsync(() -> {
                         System.out.println("Task " + i + " running...");
                         sleep(200 * i);
                         return i * 10;
                     }))
                     .collect(Collectors.toList());

        CompletableFuture<Void> all =
            CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]));

        CompletableFuture<List<Integer>> resultsFuture = all.thenApply(v ->
            futures.stream()
                   .map(CompletableFuture::join)
                   .collect(Collectors.toList())
        );

        List<Integer> results = resultsFuture.join();
        System.out.println("Results: " + results);
    }

    private static void sleep(long ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
    }
}
```

**Possible output:**

```
Task 1 running...
Task 2 running...
Task 3 running...
Results: [10, 20, 30]
```

## 4. Error Handling

`CompletableFuture` supports functional-style error handling.

### 4.1 `exceptionally` – recover from error

```java
import java.util.concurrent.CompletableFuture;

public class ExceptionallyExample {
    public static void main(String[] args) {
        CompletableFuture<Integer> future =
            CompletableFuture.supplyAsync(() -> {
                System.out.println("Computing...");
                sleep(200);
                if (true) {
                    throw new RuntimeException("Something went wrong");
                }
                return 42;
            })
            .exceptionally(ex -> {
                System.out.println("Error: " + ex.getMessage());
                // fallback value
                return -1;
            });

        Integer result = future.join();
        System.out.println("Result: " + result);
    }

    private static void sleep(long ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
    }
}
```

**Possible output:**

```
Computing...
Error: java.lang.RuntimeException: Something went wrong
Result: -1
```

### 4.2 `handle` – handle success or failure

```java
import java.util.concurrent.CompletableFuture;

public class HandleExample {
    public static void main(String[] args) {
        CompletableFuture<String> future =
            CompletableFuture.supplyAsync(() -> {
                System.out.println("Loading data...");
                sleep(200);
                if (Math.random() < 0.5) {
                    throw new RuntimeException("Random failure");
                }
                return "OK";
            })
            .handle((result, ex) -> {
                if (ex != null) {
                    System.out.println("Error: " + ex.getMessage());
                    return "DEFAULT";
                } else {
                    System.out.println("Success: " + result);
                    return result;
                }
            });

        String finalResult = future.join();
        System.out.println("Final result: " + finalResult);
    }

    private static void sleep(long ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
    }
}
```

**Possible outputs (two cases):**

Case 1 (success):

```
Loading data...
Success: OK
Final result: OK
```

Case 2 (failure):

```
Loading data...
Error: java.lang.RuntimeException: Random failure
Final result: DEFAULT
```

### 4.3 `whenComplete` – side effects on completion

`whenComplete` lets you observe completion (success or failure) without changing the result.

```java
import java.util.concurrent.CompletableFuture;

public class WhenCompleteExample {
    public static void main(String[] args) {
        CompletableFuture<Integer> future =
            CompletableFuture.supplyAsync(() -> {
                System.out.println("Starting task...");
                sleep(200);
                return 5;
            })
            .whenComplete((result, ex) -> {
                if (ex != null) {
                    System.out.println("Task failed: " + ex.getMessage());
                } else {
                    System.out.println("Task completed with result: " + result);
                }
            });

        Integer result = future.join();
        System.out.println("Final result: " + result);
    }

    private static void sleep(long ms) {
        try { Thread.sleep(ms); } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
    }
}
```

**Possible output:**

```
Starting task...
Task completed with result: 5
Final result: 5
```

## 5. Key Takeaways

- Use `supplyAsync` / `runAsync` to start async tasks.
- Build async pipelines with `thenApply`, `thenAccept`, `thenRun`.
- Compose dependent tasks with `thenCompose`.
- Combine independent tasks with `thenCombine`, `allOf`, `anyOf`.
- Handle errors with `exceptionally`, `handle`, and `whenComplete`.
- Use `join()` or `get()` only at the edges (e.g., in `main` or controller) to avoid blocking the whole pipeline.
