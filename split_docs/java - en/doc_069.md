
Here’s a concise summary of **Filters** and **Interceptors** in a Spring MVC / Web context, with code examples.

---

## 1. Servlet Filters

**What they are**

- Part of the **Servlet specification** (not Spring-specific).
- Work at the **web container level** (Tomcat, Jetty, etc.).
- Can wrap the request/response before they reach the DispatcherServlet.
- Good for **cross-cutting concerns** that are not specific to Spring MVC:
  - Logging raw HTTP requests
  - Character encoding
  - Security checks
  - GZIP compression

**Execution order**

- Filter → DispatcherServlet → Handler (Controller) → View → DispatcherServlet → Filter

**Basic example**

### Filter implementation

```java
import jakarta.servlet.*;
import jakarta.servlet.http.HttpServletRequest;
import java.io.IOException;

public class LoggingFilter implements Filter {

    @Override
    public void init(FilterConfig filterConfig) {
        // initialization if needed
    }

    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
                         FilterChain chain) throws IOException, ServletException {

        HttpServletRequest httpRequest = (HttpServletRequest) request;
        System.out.println("[Filter] Request URI: " + httpRequest.getRequestURI());

        // continue the filter chain
        chain.doFilter(request, response);

        System.out.println("[Filter] Response committed: " + response.isCommitted());
    }

    @Override
    public void destroy() {
        // cleanup if needed
    }
}
```

### Registering the filter in Spring Boot

```java
import org.springframework.boot.web.servlet.FilterRegistrationBean;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class FilterConfig {

    @Bean
    public FilterRegistrationBean<LoggingFilter> loggingFilter() {
        FilterRegistrationBean<LoggingFilter> registration = new FilterRegistrationBean<>();
        registration.setFilter(new LoggingFilter());
        registration.addUrlPatterns("/*"); // apply to all URLs
        registration.setOrder(1);          // order among multiple filters
        return registration;
    }
}
```

### Example input/output

**Request**

```http
GET /api/hello HTTP/1.1
Host: localhost:8080
```

**Console output**

```text
[Filter] Request URI: /api/hello
[Filter] Response committed: true
```

## 2. Spring Handler Interceptors

**What they are**

- Part of **Spring MVC** (not the Servlet spec).
- Work at the **handler (controller) level**.
- Can access:
  - Handler method information
  - ModelAndView
  - Path variables, etc.
- Good for **web-layer cross-cutting concerns**:
  - Authentication/authorization at controller level
  - Logging controller execution time
  - Adding common model attributes
  - Locale, theme, etc.

**Execution order**

- Filter → DispatcherServlet → Interceptor → Controller → Interceptor → View → DispatcherServlet → Filter

**Key methods**

- `preHandle(...)` – before controller method
- `postHandle(...)` – after controller method, before view rendering
- `afterCompletion(...)` – after view rendering (for cleanup, logging)

**Basic example**

### Interceptor implementation

```java
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import org.springframework.web.servlet.HandlerInterceptor;
import org.springframework.web.servlet.ModelAndView;

public class LoggingInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                             Object handler) throws Exception {
        long startTime = System.currentTimeMillis();
        request.setAttribute("startTime", startTime);
        System.out.println("[Interceptor] preHandle: " + request.getRequestURI());
        return true; // return false to stop the request
    }

    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response,
                           Object handler, ModelAndView modelAndView) throws Exception {
        System.out.println("[Interceptor] postHandle: " + request.getRequestURI());
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
                                Object handler, Exception ex) throws Exception {
        long startTime = (Long) request.getAttribute("startTime");
        long endTime = System.currentTimeMillis();
        System.out.println("[Interceptor] afterCompletion: " + request.getRequestURI()
                + " took " + (endTime - startTime) + " ms");
    }
}
```

### Registering the interceptor

```java
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.InterceptorRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class WebConfig implements WebMvcConfigurer {

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new LoggingInterceptor())
                .addPathPatterns("/api/**")   // apply to /api/** paths
                .excludePathPatterns("/api/health"); // exclude some paths
    }
}
```

### Simple controller for testing

```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {

    @GetMapping("/api/hello")
    public String hello() throws InterruptedException {
        Thread.sleep(100); // simulate work
        return "Hello, Spring!";
    }
}
```

## 3. When to Use Which

- **Use Filters** when:
  - You need to work with **raw HTTP requests/responses**.
  - The logic is **not specific to Spring MVC** (e.g., encoding, low-level security).
  - You want it to apply to **all** servlets, not just Spring’s DispatcherServlet.

- **Use Interceptors** when:
  - You need access to **handler (controller) information**.
  - You want to modify the **ModelAndView** or use Spring MVC features.
  - You’re implementing **web-layer cross-cutting concerns** (logging, auth, metrics) at the controller level.
