
Here’s a concise summary of the key ideas and usage of the three Java collections, with code examples and sample I/O.

---

## 1. `TreeMap<K,V>`

**Main ideas:**
- Map implementation that stores keys in **sorted order** (by natural order or a custom `Comparator`).
- Operations like `get`, `put`, `remove` are **O(log n)**.
- Useful when you need a map that is always ordered (e.g., range queries, first/last key).

**Key features:**
- Sorted by key.
- Methods like `firstKey()`, `lastKey()`, `subMap()`, `headMap()`, `tailMap()`.

**Example:**

```java
import java.util.*;

public class TreeMapExample {
    public static void main(String[] args) {
        TreeMap<Integer, String> map = new TreeMap<>();
        map.put(3, "C");
        map.put(1, "A");
        map.put(2, "B");
        map.put(5, "E");
        map.put(4, "D");

        System.out.println("Map: " + map);
        System.out.println("First key: " + map.firstKey());
        System.out.println("Last key: " + map.lastKey());
        System.out.println("SubMap(2 to 4): " + map.subMap(2, true, 4, true));
    }
}
```

**Possible output:**
```
Map: {1=A, 2=B, 3=C, 4=D, 5=E}
First key: 1
Last key: 5
SubMap(2 to 4): {2=B, 3=C, 4=D}
```

## 2. `PriorityQueue<E>`

**Main ideas:**
- A queue where elements are ordered by **priority**, not insertion order.
- By default, it’s a **min-heap** (smallest element has highest priority).
- Operations like `offer`, `poll`, `peek` are **O(log n)** for insertion/removal, `O(1)` for peek.

**Key features:**
- Not ordered for iteration; only the head (top priority) is guaranteed.
- Can use a custom `Comparator` for max-heap or custom priority.

**Min-heap example (default):**

```java
import java.util.*;

public class PriorityQueueMinExample {
    public static void main(String[] args) {
        PriorityQueue<Integer> pq = new PriorityQueue<>();
        pq.offer(5);
        pq.offer(1);
        pq.offer(3);

        while (!pq.isEmpty()) {
            System.out.println(pq.poll());
        }
    }
}
```

**Output (ascending):**
```
1
3
5
```

**Max-heap example (custom comparator):**

```java
import java.util.*;

public class PriorityQueueMaxExample {
    public static void main(String[] args) {
        PriorityQueue<Integer> pq = new PriorityQueue<>(Comparator.reverseOrder());
        pq.offer(5);
        pq.offer(1);
        pq.offer(3);

        while (!pq.isEmpty()) {
            System.out.println(pq.poll());
        }
    }
}
```

**Output (descending):**
```
5
3
1
```

## 3. `ConcurrentHashMap<K,V>`

**Main ideas:**
- Thread-safe implementation of `Map` for **concurrent** access.
- Allows multiple threads to read and write with **better performance** than synchronizing a normal `HashMap`.
- Uses internal partitioning and non-blocking techniques (implementation details vary by Java version).

**Key features:**
- No need to manually synchronize for basic operations (`get`, `put`, `remove`).
- Iterators are **weakly consistent** (they reflect some but not necessarily all updates during iteration).
- Good for high-concurrency environments (servers, multi-threaded apps).

**Example (simple concurrent usage):**

```java
import java.util.Map;
import java.util.concurrent.*;

public class ConcurrentHashMapExample {
    public static void main(String[] args) throws InterruptedException {
        ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

        Runnable writer = () -> {
            for (int i = 0; i < 5; i++) {
                map.put("key" + i, i);
            }
        };

        Runnable reader = () -> {
            for (int i = 0; i < 5; i++) {
                Integer value = map.get("key" + i);
                System.out.println(Thread.currentThread().getName() +
                        " read key" + i + " = " + value);
            }
        };

        Thread t1 = new Thread(writer, "Writer");
        Thread t2 = new Thread(reader, "Reader-1");
        Thread t3 = new Thread(reader, "Reader-2");

        t1.start();
        t2.start();
        t3.start();

        t1.join();
        t2.join();
        t3.join();

        System.out.println("Final map: " + map);
    }
}
```

**Sample output (order may vary due to threads):**
```
Reader-1 read key0 = null
Reader-2 read key0 = null
Reader-1 read key1 = null
Reader-2 read key1 = null
...
Final map: {key0=0, key1=1, key2=2, key3=3, key4=4}
```

(Readers may see `null` if they read before the writer has inserted the value.)

---

### Quick Comparison

- **TreeMap**: Sorted map, log-time operations, good for ordered/range queries.
- **PriorityQueue**: Heap-based priority structure, good for always getting min/max quickly.
- **ConcurrentHashMap**: High-performance thread-safe map for concurrent environments.
