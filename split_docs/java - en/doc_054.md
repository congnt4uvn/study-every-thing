
**Key Idea**

For Java performance tuning, you should know how to use at least one **profiling tool** to find bottlenecks (slow methods, high CPU usage, memory leaks, excessive allocations).

Common Java profilers:

- **VisualVM** (free, comes with many JDKs)
- **Java Mission Control (JMC)**
- **YourKit**
- **JProfiler**

Knowing one well (e.g., VisualVM) is usually enough for interviews and real work.

---

## What a Java Profiler Helps You Do

1. **CPU Profiling** – find which methods use the most CPU time.  
2. **Memory Profiling** – find memory leaks and heavy allocations.  
3. **Thread Analysis** – see deadlocks, blocked threads, thread states.  
4. **GC Analysis** – see how often garbage collection runs and how long it takes.

## Example: Using VisualVM to Find a Slow Method

### Sample Code (CPU bottleneck)

```java
public class PerformanceDemo {

    public static void main(String[] args) {
        long start = System.currentTimeMillis();
        int result = slowComputation(40);
        long end = System.currentTimeMillis();

        System.out.println("Result: " + result);
        System.out.println("Time (ms): " + (end - start));
    }

    // Intentionally slow (recursive Fibonacci)
    private static int slowComputation(int n) {
        if (n <= 1) return n;
        return slowComputation(n - 1) + slowComputation(n - 2);
    }
}
```

### Example Output (without optimization)

```
Result: 102334155
Time (ms): 2500
```

Using **VisualVM**:

1. Run the program: `java PerformanceDemo`.
2. Open **VisualVM**.
3. Attach to the running `PerformanceDemo` process.
4. Go to **Profiler** → **CPU** → **Start**.
5. Let it run, then **Stop** profiling.

You will see that `slowComputation` takes almost all CPU time.  
This tells you where to optimize.

### Optimized Version

```java
import java.util.HashMap;
import java.util.Map;

public class PerformanceDemoOptimized {

    private static final Map<Integer, Integer> cache = new HashMap<>();

    public static void main(String[] args) {
        long start = System.currentTimeMillis();
        int result = fastComputation(40);
        long end = System.currentTimeMillis();

        System.out.println("Result: " + result);
        System.out.println("Time (ms): " + (end - start));
    }

    private static int fastComputation(int n) {
        if (n <= 1) return n;
        if (cache.containsKey(n)) return cache.get(n);

        int value = fastComputation(n - 1) + fastComputation(n - 2);
        cache.put(n, value);
        return value;
    }
}
```

### Example Output (after optimization)

```
Result: 102334155
Time (ms): 5
```

A profiler would now show much less CPU time spent in `fastComputation`.

## Example: Memory Leak Detection (Conceptual)

```java
import java.util.ArrayList;
import java.util.List;

public class MemoryLeakDemo {

    private static final List<byte[]> storage = new ArrayList<>();

    public static void main(String[] args) throws InterruptedException {
        while (true) {
            // Allocate 1 MB repeatedly
            storage.add(new byte[1024 * 1024]);
            Thread.sleep(100);
        }
    }
}
```

If you run this and attach a profiler (VisualVM, JProfiler, YourKit):

- You’ll see **heap usage constantly growing**.
- Heap dump will show many `byte[]` objects referenced from `storage`.
- This indicates a memory leak due to never clearing the list.

---

## Summary

- For Java performance tuning, **learn at least one profiler** (VisualVM, JMC, YourKit, JProfiler).
- Use it to:
  - Identify **CPU hotspots** (slow methods).
  - Detect **memory leaks** and heavy allocations.
  - Analyze **threads** and **GC behavior**.
- Profilers guide you to the real bottlenecks so you optimize the right code.
