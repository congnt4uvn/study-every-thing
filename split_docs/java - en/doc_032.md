## 1. `synchronized` and Intrinsic Locks

- Every Java object has an **intrinsic lock** (also called a monitor lock).
- The `synchronized` keyword uses this intrinsic lock to ensure **mutual exclusion**:
  - Only one thread at a time can hold the lock for a given object.
  - Other threads trying to enter a synchronized block/method on the same object must wait.

### Forms of `synchronized`

1. **Synchronized instance method**

```java
class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++; // lock is on "this"
    }

    public synchronized int getCount() {
        return count; // lock is on "this"
    }
}
```

- Lock target: `this` (the current instance).

2. **Synchronized static method**

```java
class GlobalCounter {
    private static int count = 0;

    public static synchronized void increment() {
        count++; // lock is on GlobalCounter.class
    }

    public static synchronized int getCount() {
        return count; // lock is on GlobalCounter.class
    }
}
```

- Lock target: `GlobalCounter.class` (the Class object).

3. **Synchronized block**

```java
class SafeList {
    private final List<Integer> list = new ArrayList<>();
    private final Object lock = new Object();

    public void add(int value) {
        synchronized (lock) {
            list.add(value);
        }
    }

    public int size() {
        synchronized (lock) {
            return list.size();
        }
    }
}
```

- Lock target: any chosen object (`lock` here).

## 2. Reentrancy

- Java‚Äôs intrinsic locks are **reentrant**:
  - A thread that already holds a lock can **acquire it again** without blocking.
  - This allows synchronized methods to call other synchronized methods on the same object safely.

### Reentrancy Example

```java
class ReentrantExample {
    public synchronized void methodA() {
        System.out.println("In methodA");
        methodB(); // same thread re-enters the lock
    }

    public synchronized void methodB() {
        System.out.println("In methodB");
    }
}

public class Main {
    public static void main(String[] args) {
        ReentrantExample obj = new ReentrantExample();

        Thread t = new Thread(obj::methodA);
        t.start();
    }
}
```

**Possible output:**
```
In methodA
In methodB
```

- `methodA` acquires the lock on `obj`.
- Inside `methodA`, calling `methodB` tries to acquire the same lock.
- Because the same thread already holds the lock, it is allowed to re-enter.

## 3. Simple Multi-threaded Example with Input/Output

```java
class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public synchronized int getCount() {
        return count;
    }
}

public class Main {
    public static void main(String[] args) throws InterruptedException {
        Counter counter = new Counter();

        Runnable task = () -> {
            for (int i = 0; i < 1000; i++) {
                counter.increment();
            }
        };

        Thread t1 = new Thread(task);
        Thread t2 = new Thread(task);

        t1.start();
        t2.start();

        t1.join();
        t2.join();

        System.out.println("Final count: " + counter.getCount());
    }
}
```

**Expected output:**
```
Final count: 2000
```

- Without `synchronized`, the result could be less than 2000 due to race conditions.
- With `synchronized`, intrinsic locks ensure only one thread increments at a time.

---

**Key points:**

- `synchronized` uses an object‚Äôs intrinsic lock to provide mutual exclusion.
- You can synchronize methods or blocks, on `this`, a class object, or any chosen lock object.
- Java intrinsic locks are **reentrant**, allowing the same thread to acquire the same lock multiple times.
