
Here’s a concise summary of unit & integration testing in Spring Boot, focusing on the three main test annotations and showing simple examples.

---

## 1. `@SpringBootTest` – Full Integration Test

**Purpose:**  
Loads the full Spring application context. Used for end-to-end or integration tests (e.g., service + repository + DB).

**Key points:**
- Starts the whole Spring Boot context.
- Can start an embedded web server if needed.
- Slower than slice tests, but more realistic.

**Example: Service + Repository Integration**

```java
// src/test/java/com/example/demo/OrderServiceIntegrationTest.java
@SpringBootTest
class OrderServiceIntegrationTest {

    @Autowired
    private OrderService orderService;

    @Test
    void createOrder_savesAndReturnsOrder() {
        // Input
        OrderRequest request = new OrderRequest("Book", 2);

        // Action
        Order order = orderService.createOrder(request);

        // Output assertions
        assertNotNull(order.getId());
        assertEquals("Book", order.getProductName());
        assertEquals(2, order.getQuantity());
    }
}
```

**Input:** `OrderRequest("Book", 2)`  
**Expected Output:** `Order` with non-null `id`, `productName = "Book"`, `quantity = 2`.

---

## 2. `@WebMvcTest` – Web Layer (Controller) Test

**Purpose:**  
Tests only the web layer (controllers, `@Controller`, `@RestController`). No full context, no DB. Other beans are usually mocked.

**Key points:**
- Loads only MVC components.
- Uses `MockMvc` to simulate HTTP requests.
- Fast and focused on controller behavior.

**Example: Controller Test with MockMvc**

```java
// src/main/java/com/example/demo/HelloController.java
@RestController
@RequestMapping("/api")
public class HelloController {

    @GetMapping("/hello")
    public String hello(@RequestParam String name) {
        return "Hello, " + name;
    }
}
```

```java
// src/test/java/com/example/demo/HelloControllerTest.java
@WebMvcTest(HelloController.class)
class HelloControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @Test
    void hello_returnsGreeting() throws Exception {
        // Input: HTTP GET /api/hello?name=John
        mockMvc.perform(get("/api/hello")
                .param("name", "John"))
            // Expected HTTP status
            .andExpect(status().isOk())
            // Expected response body
            .andExpect(content().string("Hello, John"));
    }
}
```

**Input (HTTP):** `GET /api/hello?name=John`  
**Output (HTTP):** `200 OK`, body: `"Hello, John"`.

---

## 3. `@DataJpaTest` – JPA Repository Test

**Purpose:**  
Tests the persistence layer (JPA repositories). Uses an in-memory database by default (e.g., H2).

**Key points:**
- Loads only JPA-related components.
- Configures in-memory DB (unless overridden).
- Good for testing queries and entity mappings.

**Example: Repository Test**

```java
// src/main/java/com/example/demo/Book.java
@Entity
public class Book {
    @Id @GeneratedValue
    private Long id;
    private String title;
    private String author;
    // getters/setters/constructors
}
```

```java
// src/main/java/com/example/demo/BookRepository.java
public interface BookRepository extends JpaRepository<Book, Long> {
    List<Book> findByAuthor(String author);
}
```

```java
// src/test/java/com/example/demo/BookRepositoryTest.java
@DataJpaTest
class BookRepositoryTest {

    @Autowired
    private TestEntityManager entityManager;

    @Autowired
    private BookRepository bookRepository;

    @Test
    void findByAuthor_returnsMatchingBooks() {
        // Input: persist some books
        Book b1 = new Book(null, "Spring in Action", "Craig");
        Book b2 = new Book(null, "Spring Boot Up & Running", "Mark");
        Book b3 = new Book(null, "Another Spring Book", "Craig");

        entityManager.persist(b1);
        entityManager.persist(b2);
        entityManager.persist(b3);
        entityManager.flush();

        // Action
        List<Book> result = bookRepository.findByAuthor("Craig");

        // Output assertions
        assertEquals(2, result.size());
        assertTrue(result.stream().allMatch(b -> b.getAuthor().equals("Craig")));
    }
}
```

**Input (DB state):** 3 books, 2 with author `"Craig"`  
**Output:** `findByAuthor("Craig")` returns a list of 2 books, all with author `"Craig"`.

---

## Summary of When to Use What

- **`@SpringBootTest`**  
  Use when you need the **full application**: multiple layers working together, real configuration, possibly external systems (or test containers).

- **`@WebMvcTest`**  
  Use when you want to test **only controllers** and HTTP behavior, mocking services and repositories.

- **`@DataJpaTest`**  
  Use when you want to test **JPA repositories and database interactions** in isolation with an in-memory DB.
