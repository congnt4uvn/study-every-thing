
Here’s a concise summary and some practical Java examples related to **Cloud & DevOps basics: Compute (EC2, GCE, etc.)**.

---

## Key Ideas

1. **Compute in the Cloud**
   - Services like **AWS EC2** and **Google Compute Engine (GCE)** provide **virtual machines (VMs)** on demand.
   - You choose:
     - CPU, memory, storage
     - Operating system (Linux/Windows)
     - Network settings (VPC, subnets, firewalls/security groups)
   - You pay for what you use (per second/minute/hour).

2. **DevOps Perspective**
   - Compute instances are:
     - **Provisioned automatically** (Infrastructure as Code: Terraform, CloudFormation, etc.).
     - **Configured automatically** (Ansible, Chef, Puppet, cloud-init).
     - **Used to run applications** (Java apps, microservices, etc.).
   - Goal: repeatable, automated, scalable deployments.

3. **Java on Cloud Compute**
   - You typically:
     1. Build your Java app (JAR/WAR).
     2. Copy it to the VM (EC2/GCE).
     3. Run it with `java -jar` or in a container (Docker).
   - You can also **control compute resources via Java SDKs** (e.g., AWS SDK, Google Cloud Client Libraries).

---

## Example 1: Simple Java App to Run on EC2/GCE

**File:** `HelloCloud.java`

```java
public class HelloCloud {
    public static void main(String[] args) {
        String instanceName = System.getenv("INSTANCE_NAME");
        if (instanceName == null || instanceName.isEmpty()) {
            instanceName = "unknown-instance";
        }

        System.out.println("Hello from cloud compute instance: " + instanceName);
    }
}
```

### How to run on a VM

1. Compile locally:
   ```bash
   javac HelloCloud.java
   jar cfe hello-cloud.jar HelloCloud HelloCloud.class
   ```

2. On your EC2/GCE instance:
   ```bash
   export INSTANCE_NAME="ec2-01"
   java -jar hello-cloud.jar
   ```

**Sample Output:**
```text
Hello from cloud compute instance: ec2-01
```

## Example 2: Starting an EC2 Instance with Java (AWS SDK v2)

> You need AWS credentials configured and the AWS SDK dependency.

**Maven dependency:**

```xml
<dependency>
    <groupId>software.amazon.awssdk</groupId>
    <artifactId>ec2</artifactId>
    <version>2.25.0</version> <!-- or latest -->
</dependency>
```

**Java code:**

```java
import software.amazon.awssdk.regions.Region;
import software.amazon.awssdk.services.ec2.Ec2Client;
import software.amazon.awssdk.services.ec2.model.*;

public class LaunchEc2Example {
    public static void main(String[] args) {
        Region region = Region.US_EAST_1;

        try (Ec2Client ec2 = Ec2Client.builder().region(region).build()) {
            RunInstancesRequest runRequest = RunInstancesRequest.builder()
                    .imageId("ami-0123456789abcdef0") // example AMI ID
                    .instanceType(InstanceType.T2_MICRO)
                    .minCount(1)
                    .maxCount(1)
                    .build();

            RunInstancesResponse response = ec2.runInstances(runRequest);

            String instanceId = response.instances().get(0).instanceId();
            System.out.println("Launched EC2 instance with ID: " + instanceId);
        }
    }
}
```

**Sample Output:**
```text
Launched EC2 instance with ID: i-0abc123def4567890
```

## Example 3: Calling a Service Running on a Compute Instance

Assume you have a Java service running on an EC2/GCE instance at `http://<instance-ip>:8080/hello`.

**Client code:**

```java
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;

public class CallCloudService {
    public static void main(String[] args) throws Exception {
        String url = "http://203.0.113.10:8080/hello"; // example public IP

        HttpClient client = HttpClient.newHttpClient();
        HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create(url))
                .GET()
                .build();

        HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());

        System.out.println("Status: " + response.statusCode());
        System.out.println("Body: " + response.body());
    }
}
```

**Sample Output:**
```text
Status: 200
Body: Hello from my Java service on EC2!
```

### Summary

- **Compute (EC2, GCE)** = on-demand virtual machines in the cloud.
- Used to **run Java applications**, often managed via **DevOps practices** (automation, IaC, CI/CD).
- Java can:
  - Run on these instances (standard JVM).
  - Programmatically manage them via cloud SDKs (e.g., AWS EC2 API).
