
Here’s a concise summary and example around **code coverage tools in Java**, focusing on **JaCoCo**.

---

## Key Points

- **Code coverage** measures how much of your code is executed by tests (e.g., unit tests).
- **JaCoCo** is a popular Java code coverage tool that:
  - Integrates with **Maven**, **Gradle**, and IDEs.
  - Reports coverage metrics like:
    - Line coverage
    - Branch coverage
    - Method/class coverage
  - Generates reports in **HTML**, **XML**, etc.
- It helps you:
  - Identify untested code.
  - Improve test quality.
  - Enforce minimum coverage thresholds in CI.

---

## Simple Example (Maven + JaCoCo)

### 1. `pom.xml` configuration

```xml
<build>
    <plugins>
        <!-- Surefire runs the tests -->
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-surefire-plugin</artifactId>
            <version>3.2.5</version>
        </plugin>

        <!-- JaCoCo plugin -->
        <plugin>
            <groupId>org.jacoco</groupId>
            <artifactId>jacoco-maven-plugin</artifactId>
            <version>0.8.11</version>
            <executions>
                <!-- Attach JaCoCo agent when tests run -->
                <execution>
                    <goals>
                        <goal>prepare-agent</goal>
                    </goals>
                </execution>

                <!-- Generate coverage report after tests -->
                <execution>
                    <id>report</id>
                    <phase>test</phase>
                    <goals>
                        <goal>report</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>
```

### 2. Sample Java class

```java
// src/main/java/com/example/Calculator.java
package com.example;

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }

    public int divide(int a, int b) {
        if (b == 0) {
            throw new IllegalArgumentException("b must not be zero");
        }
        return a / b;
    }
}
```

### 3. Sample JUnit test

```java
// src/test/java/com/example/CalculatorTest.java
package com.example;

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {

    @Test
    void testAdd() {
        Calculator calc = new Calculator();
        assertEquals(5, calc.add(2, 3));   // input: (2,3) -> expected output: 5
    }

    @Test
    void testDivide() {
        Calculator calc = new Calculator();
        assertEquals(2, calc.divide(6, 3)); // input: (6,3) -> expected output: 2
    }
}
```

### 4. Running tests with coverage

Command:

```bash
mvn test
```

- **Inputs**: The test methods call:
  - `add(2, 3)` → expected output `5`
  - `divide(6, 3)` → expected output `2`
- **Outputs**:
  - Test results (pass/fail) in console.
  - Coverage report generated at:
    - `target/site/jacoco/index.html`

Open `target/site/jacoco/index.html` in a browser to see:

- Percentage of lines covered (e.g., 100% for `Calculator` if all methods are tested).
- Which lines/branches are covered (green) or not covered (red).

---

## Typical Use in Projects

- Run in CI (e.g., GitHub Actions, Jenkins) to:
  - Fail builds if coverage is below a threshold.
  - Track coverage trends over time.
- Combine with other testing types (unit, integration, system tests) to ensure broad coverage of the codebase.
