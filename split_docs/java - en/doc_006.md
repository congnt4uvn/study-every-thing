
Here’s a concise summary of Java 8+ Streams, with key ideas and code examples.

---

## 1. What is the Streams API?

- A **Stream** is a sequence of elements supporting **functional-style operations**.
- It does **not** store data; it processes data from a source (collections, arrays, I/O, etc.).
- Operations are **chained** to form a processing pipeline.

---

## 2. Intermediate vs Terminal Operations

### Intermediate Operations
- Return another `Stream`.
- Are **lazy**: they don’t run until a terminal operation is called.
- Examples: `map`, `filter`, `sorted`, `distinct`, `limit`, `skip`.

### Terminal Operations
- Produce a **result** or a **side effect**.
- Trigger the execution of the entire pipeline.
- Examples: `collect`, `forEach`, `reduce`, `count`, `findFirst`, `anyMatch`.

---

## 3. Lazy Evaluation

- Intermediate operations are **not executed immediately**.
- The stream processes elements **only when** a terminal operation is invoked.
- This allows optimizations like **short-circuiting** (e.g., `findFirst`, `anyMatch`).

---

## 4. Basic Example: Filter, Map, Collect

```java
import java.util.*;
import java.util.stream.*;

public class StreamExample {
    public static void main(String[] args) {
        List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5, 6);

        // Pipeline:
        // 1) filter even numbers (intermediate)
        // 2) square them (intermediate)
        // 3) collect to a list (terminal)
        List<Integer> result = numbers.stream()
                .filter(n -> n % 2 == 0)   // intermediate
                .map(n -> n * n)          // intermediate
                .collect(Collectors.toList()); // terminal

        System.out.println(result);
    }
}
```

**Input:**
```text
numbers = [1, 2, 3, 4, 5, 6]
```

**Output:**
```text
[4, 16, 36]
```

## 5. Demonstrating Laziness

```java
import java.util.Arrays;
import java.util.List;

public class LazyStreamExample {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie");

        System.out.println("Creating stream...");

        // No output from filter/map yet (lazy)
        names.stream()
                .filter(name -> {
                    System.out.println("filter: " + name);
                    return name.length() > 3;
                })
                .map(name -> {
                    System.out.println("map: " + name);
                    return name.toUpperCase();
                })
                .forEach(result -> System.out.println("forEach: " + result));
    }
}
```

**Output (order shows laziness and element-by-element processing):**
```text
Creating stream...
filter: Alice
map: Alice
forEach: ALICE
filter: Bob
filter: Charlie
map: Charlie
forEach: CHARLIE
```

Note:
- `filter` and `map` run only when `forEach` (terminal) is called.
- Each element goes through the whole pipeline before the next element starts.

---

## 6. Example with a Terminal Reduction

```java
import java.util.Arrays;
import java.util.List;

public class ReduceExample {
    public static void main(String[] args) {
        List<Integer> nums = Arrays.asList(1, 2, 3, 4);

        int sum = nums.stream()
                .reduce(0, (a, b) -> a + b); // terminal

        System.out.println(sum);
    }
}
```

**Input:**
```text
nums = [1, 2, 3, 4]
```

**Output:**
```text
10
```

---

**Key ideas:**
- Streams provide a **declarative**, functional way to process data.
- **Intermediate operations** build the pipeline and are **lazy**.
- **Terminal operations** execute the pipeline and produce a result or side effect.
