
Here’s a concise summary of Spring Boot configuration properties with `@ConfigurationProperties` and `@Value`, plus code examples.

---

## 1. `@ConfigurationProperties`

**Purpose:**  
Bind a group of related properties from `application.properties` or `application.yml` to a Java class.

**Key points:**
- Good for structured, grouped configuration.
- Supports type-safe binding (e.g., int, boolean, lists, nested objects).
- Usually used with `@ConfigurationProperties(prefix = "some.prefix")`.
- Class must be a Spring bean (`@Component`, `@Configuration`, or registered via `@EnableConfigurationProperties`).

### Example: Properties file

```properties
# application.properties
app.name=MyApp
app.version=1.0.0
app.features.enabled=true
app.limits.max-users=100
app.limits.timeout-seconds=30
```

### Example: Java class with `@ConfigurationProperties`

```java
import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.stereotype.Component;

@Component
@ConfigurationProperties(prefix = "app")
public class AppProperties {

    private String name;
    private String version;
    private Features features = new Features();
    private Limits limits = new Limits();

    public static class Features {
        private boolean enabled;
        // getters and setters
        public boolean isEnabled() { return enabled; }
        public void setEnabled(boolean enabled) { this.enabled = enabled; }
    }

    public static class Limits {
        private int maxUsers;
        private int timeoutSeconds;
        // getters and setters
        public int getMaxUsers() { return maxUsers; }
        public void setMaxUsers(int maxUsers) { this.maxUsers = maxUsers; }
        public int getTimeoutSeconds() { return timeoutSeconds; }
        public void setTimeoutSeconds(int timeoutSeconds) { this.timeoutSeconds = timeoutSeconds; }
    }

    // getters and setters
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public String getVersion() { return version; }
    public void setVersion(String version) { this.version = version; }
    public Features getFeatures() { return features; }
    public void setFeatures(Features features) { this.features = features; }
    public Limits getLimits() { return limits; }
    public void setLimits(Limits limits) { this.limits = limits; }
}
```

### Using it in a controller

```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class AppController {

    private final AppProperties appProperties;

    public AppController(AppProperties appProperties) {
        this.appProperties = appProperties;
    }

    @GetMapping("/info")
    public String info() {
        return String.format(
            "Name: %s, Version: %s, Features enabled: %s, Max users: %d",
            appProperties.getName(),
            appProperties.getVersion(),
            appProperties.getFeatures().isEnabled(),
            appProperties.getLimits().getMaxUsers()
        );
    }
}
```

**Example HTTP call / output:**

- **Request:** `GET /info`
- **Response body (text):**  
  `Name: MyApp, Version: 1.0.0, Features enabled: true, Max users: 100`

## 2. `@Value`

**Purpose:**  
Inject a single property value (or expression) directly into a field, constructor parameter, or method.

**Key points:**
- Good for simple, one-off values.
- Uses SpEL (Spring Expression Language) syntax: `@Value("${property.name:default}")`.
- Less structured than `@ConfigurationProperties`.

### Example: Using `@Value` for simple properties

```properties
# application.properties
app.title=My Spring App
app.page-size=20
```

```java
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

@Component
public class ValueExample {

    @Value("${app.title}")
    private String title;

    @Value("${app.page-size:10}") // default 10 if not set
    private int pageSize;

    @Value("${unknown.property:default-value}")
    private String withDefault;

    public String info() {
        return String.format("Title: %s, Page size: %d, Default: %s",
                title, pageSize, withDefault);
    }
}
```

### Using it in a controller

```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class ValueController {

    private final ValueExample valueExample;

    public ValueController(ValueExample valueExample) {
        this.valueExample = valueExample;
    }

    @GetMapping("/value-info")
    public String valueInfo() {
        return valueExample.info();
    }
}
```

**Example HTTP call / output:**

- **Request:** `GET /value-info`
- **Response body (text):**  
  `Title: My Spring App, Page size: 20, Default: default-value`

---

## 3. When to use which?

- Use **`@ConfigurationProperties`** when:
  - You have multiple related settings.
  - You want type-safe, structured configuration.
  - You may reuse the configuration in many places.

- Use **`@Value`** when:
  - You need a single simple value.
  - You want quick injection without creating a separate config class.
