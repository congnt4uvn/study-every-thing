## What is `Optional`?

- `Optional<T>` is a container object that may or may not hold a non-null value.
- Its main goal is to **avoid `NullPointerException`** and make the possibility of ŌĆ£no valueŌĆØ explicit in the type system.
- Instead of returning `null`, methods can return `Optional<T>`.

## Creating Optionals

```java
import java.util.Optional;

public class OptionalCreation {
    public static void main(String[] args) {
        Optional<String> nonEmpty = Optional.of("Hello");
        Optional<String> empty1   = Optional.empty();
        Optional<String> maybe    = Optional.ofNullable(null); // or a value

        System.out.println(nonEmpty.isPresent()); // true
        System.out.println(empty1.isPresent());   // false
        System.out.println(maybe.isPresent());    // false
    }
}
```

**Output:**
```
true
false
false
```

## Accessing Values Safely

### `orElse`, `orElseGet`, `orElseThrow`

```java
public class OptionalAccess {
    public static void main(String[] args) {
        Optional<String> name = Optional.ofNullable(null);

        String result1 = name.orElse("default");
        String result2 = name.orElseGet(() -> "computed default");
        // String result3 = name.orElseThrow(); // throws NoSuchElementException

        System.out.println(result1); // default
        System.out.println(result2); // computed default
    }
}
```

## Transforming Values: `map` and `flatMap`

```java
public class OptionalMap {
    public static void main(String[] args) {
        Optional<String> name = Optional.of("Alice");

        Optional<Integer> length = name.map(String::length);
        System.out.println(length.orElse(-1)); // 5

        // flatMap example: Optional<Optional<Integer>> -> Optional<Integer>
        Optional<Optional<Integer>> nested = Optional.of(Optional.of(10));
        Optional<Integer> flat = nested.flatMap(x -> x);
        System.out.println(flat.orElse(0)); // 10
    }
}
```

## Conditional Execution: `ifPresent` and `ifPresentOrElse` (Java 9+)

```java
public class OptionalIfPresent {
    public static void main(String[] args) {
        Optional<String> name = Optional.of("Bob");
        Optional<String> empty = Optional.empty();

        name.ifPresent(n -> System.out.println("Name: " + n));
        // Output: Name: Bob

        empty.ifPresentOrElse(
            n -> System.out.println("Name: " + n),
            () -> System.out.println("No name")
        );
        // Output: No name
    }
}
```

## Filtering Values: `filter`

```java
public class OptionalFilter {
    public static void main(String[] args) {
        Optional<Integer> age = Optional.of(20);

        age.filter(a -> a >= 18)
           .ifPresent(a -> System.out.println("Adult: " + a));

        // If age < 18, nothing is printed
    }
}
```

**Output:**
```
Adult: 20
```

## Typical Use Case: Avoiding Null Checks

Without `Optional`:

```java
public String getUserCity(User user) {
    if (user != null && user.getAddress() != null && user.getAddress().getCity() != null) {
        return user.getAddress().getCity();
    }
    return "Unknown";
}
```

With `Optional`:

```java
public String getUserCity(Optional<User> userOpt) {
    return userOpt
        .map(User::getAddress)
        .map(Address::getCity)
        .orElse("Unknown");
}
```

---

## Key Points

- `Optional` represents a value that may be present or absent.
- Encourages **explicit handling** of missing values instead of using `null`.
- Main methods:
  - Creation: `of`, `ofNullable`, `empty`
  - Access: `get` (avoid if possible), `orElse`, `orElseGet`, `orElseThrow`
  - Functional: `map`, `flatMap`, `filter`, `ifPresent`, `ifPresentOrElse`
- Best used as **return types**, not as fields or method parameters in most designs.
