## Key Ideas

### 1. Event-Driven Architecture (EDA)

- **Core concept**: Components communicate by **emitting and reacting to events**, not by calling each other directly.
- **Loose coupling**: Producers don’t know who consumes the events; consumers subscribe to events they care about.
- **Asynchronous**: Events are usually processed asynchronously, improving scalability and responsiveness.
- **Typical flow**:
  1. Something happens → an **event** is created (e.g., `OrderCreatedEvent`).
  2. Event is **published** to an event bus or broker.
  3. One or more **event handlers** consume and process it.

#### Simple Java Example (In-Memory Event Bus)

```java
// Event
class OrderCreatedEvent {
    private final String orderId;
    public OrderCreatedEvent(String orderId) { this.orderId = orderId; }
    public String getOrderId() { return orderId; }
}

// Listener interface
interface OrderCreatedListener {
    void onOrderCreated(OrderCreatedEvent event);
}

// Simple event bus
class EventBus {
    private final List<OrderCreatedListener> listeners = new ArrayList<>();

    public void register(OrderCreatedListener listener) {
        listeners.add(listener);
    }

    public void publish(OrderCreatedEvent event) {
        for (OrderCreatedListener listener : listeners) {
            listener.onOrderCreated(event);
        }
    }
}

// Usage
public class Main {
    public static void main(String[] args) {
        EventBus bus = new EventBus();

        // Subscriber
        bus.register(event ->
            System.out.println("Handling order: " + event.getOrderId())
        );

        // Publisher
        bus.publish(new OrderCreatedEvent("ORD-123"));
    }
}
```

**Sample Output:**
```
Handling order: ORD-123
```

### 2. Messaging Architecture

- **Core concept**: Components communicate by **sending messages** via a **message broker** (e.g., RabbitMQ, Kafka, ActiveMQ).
- **Decoupling**: Sender and receiver don’t need to be online at the same time; the broker buffers messages.
- **Patterns**:
  - **Point-to-point (queue)**: One consumer gets each message.
  - **Publish/subscribe (topic)**: Multiple consumers can receive the same message.
- **Reliability**: Messages can be persisted, retried, and processed later.

#### Java Example with JMS (Point-to-Point Queue)

```java
import javax.jms.*;
import org.apache.activemq.ActiveMQConnectionFactory;

public class JmsExample {
    public static void main(String[] args) throws Exception {
        // 1. Connection to broker
        ConnectionFactory factory =
            new ActiveMQConnectionFactory("tcp://localhost:61616");
        Connection connection = factory.createConnection();
        connection.start();

        Session session = connection.createSession(false, Session.AUTO_ACKNOWLEDGE);
        Destination queue = session.createQueue("order.queue");

        // 2. Producer: send a message
        MessageProducer producer = session.createProducer(queue);
        TextMessage message = session.createTextMessage("Order ID: ORD-123");
        producer.send(message);
        System.out.println("Sent: " + message.getText());

        // 3. Consumer: receive a message
        MessageConsumer consumer = session.createConsumer(queue);
        Message received = consumer.receive(1000); // wait up to 1s

        if (received instanceof TextMessage) {
            TextMessage textMessage = (TextMessage) received;
            System.out.println("Received: " + textMessage.getText());
        }

        session.close();
        connection.close();
    }
}
```

**Sample Output (single run, same process):**
```
Sent: Order ID: ORD-123
Received: Order ID: ORD-123
```

---

## Relationship Between the Two

- **Event-driven** is about **reacting to events**; **messaging** is often the **technical mechanism** used to deliver those events.
- Many systems implement EDA **on top of** a messaging infrastructure (e.g., events published to Kafka topics, consumed by multiple services).
