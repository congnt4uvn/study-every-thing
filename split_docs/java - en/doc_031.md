
Here’s a concise summary of the key ideas, with code examples and sample I/O.

## 1. Thread Lifecycle (Basics)

A Java thread typically goes through these main states:

1. **New** – Thread object created, not started yet.
2. **Runnable** – `start()` called; eligible to run by the CPU.
3. **Running** – Actually executing `run()` (conceptually part of Runnable).
4. **Blocked/Waiting/Timed Waiting** – Waiting for a monitor lock, `wait()`, `sleep()`, `join()`, etc.
5. **Terminated** – `run()` method finished or thread stopped due to an error.

You can see a thread’s state with `getState()`.

```java
public class ThreadLifecycleDemo {
    public static void main(String[] args) throws InterruptedException {
        Thread t = new Thread(() -> {
            System.out.println("Thread is running");
        });

        System.out.println("State after creation: " + t.getState()); // NEW
        t.start();
        System.out.println("State after start: " + t.getState());    // RUNNABLE or TERMINATED (depends on timing)

        t.join(); // wait for thread to finish
        System.out.println("State after join: " + t.getState());     // TERMINATED
    }
}
```

**Sample output (one possible run):**
```
State after creation: NEW
Thread is running
State after start: RUNNABLE
State after join: TERMINATED
```

## 2. `Runnable`

- Represents a task that **does not return a result** and **cannot throw checked exceptions**.
- Has a single method: `void run()`.

### Creating and starting a thread with `Runnable`

```java
public class RunnableExample {
    public static void main(String[] args) {
        Runnable task = () -> {
            System.out.println("Hello from Runnable! Current thread: " +
                               Thread.currentThread().getName());
        };

        Thread thread = new Thread(task);
        thread.start();

        System.out.println("Main thread: " + Thread.currentThread().getName());
    }
}
```

**Possible output:**
```
Main thread: main
Hello from Runnable! Current thread: Thread-0
```

---

## 3. `Callable<V>`

- Similar to `Runnable`, but:
  - Returns a value of type `V`.
  - Can throw checked exceptions.
- Has a single method: `V call() throws Exception`.

You usually use `Callable` with an `ExecutorService` and get the result via a `Future`.

### Example with `Callable`

```java
import java.util.concurrent.*;

public class CallableExample {
    public static void main(String[] args) throws Exception {
        Callable<Integer> task = () -> {
            // Simulate some computation
            Thread.sleep(500);
            return 42;
        };

        ExecutorService executor = Executors.newSingleThreadExecutor();
        Future<Integer> future = executor.submit(task);

        System.out.println("Doing other work in main...");

        Integer result = future.get(); // waits for the result
        System.out.println("Result from Callable: " + result);

        executor.shutdown();
    }
}
```

**Sample output:**
```
Doing other work in main...
Result from Callable: 42
```

## 4. `Future<V>`

- Represents the **result of an asynchronous computation**.
- Returned by `ExecutorService.submit(...)`.
- Key methods:
  - `V get()` – waits (blocks) until the result is available.
  - `V get(long timeout, TimeUnit unit)` – waits up to a timeout.
  - `boolean isDone()` – checks if computation is finished.
  - `boolean cancel(boolean mayInterruptIfRunning)` – attempts to cancel.

### Example: checking `isDone()` and using timeout

```java
import java.util.concurrent.*;

public class FutureExample {
    public static void main(String[] args) throws Exception {
        ExecutorService executor = Executors.newSingleThreadExecutor();

        Callable<String> task = () -> {
            Thread.sleep(2000); // simulate long task
            return "Task completed";
        };

        Future<String> future = executor.submit(task);

        System.out.println("Is done right away? " + future.isDone());

        try {
            // Wait at most 1 second
            String result = future.get(1, TimeUnit.SECONDS);
            System.out.println("Result: " + result);
        } catch (TimeoutException e) {
            System.out.println("Timeout: task not finished yet");
        }

        // Now wait without timeout
        String finalResult = future.get();
        System.out.println("Final result: " + finalResult);
        System.out.println("Is done now? " + future.isDone());

        executor.shutdown();
    }
}
```

**Sample output:**
```
Is done right away? false
Timeout: task not finished yet
Final result: Task completed
Is done now? true
```

---

### Core Ideas Summary

- **Thread lifecycle**: New → Runnable → (Running) → Waiting/Blocked → Terminated.
- **Runnable**: task with `run()`, no return value, no checked exceptions.
- **Callable**: task with `call()`, returns a value, can throw checked exceptions.
- **Future**: handle to an asynchronous computation; lets you wait for results, check completion, or cancel.
