
Here’s a concise overview of notable features introduced or planned around Java 22–25, with short code examples where possible. (Some items for 23–25 are based on current JEPs / previews and may change.)

---

## Java 22 (released)

### 1. Unnamed Variables and Patterns (Preview)
You can use `_` for variables or pattern parts you don’t care about.

```java
// Unnamed variable
for (int _ = 0; _ < 10; _++) {  // here _ is still used, but idea is "I don't care about the name"
    System.out.println("Hello");
}

// Unnamed pattern
if (obj instanceof Point(int x, _)) {  // ignore y
    System.out.println("x = " + x);
}
```

**Output (example):**
```
Hello
Hello
...
x = 10
```

---

### 2. Statements Before `super(...)` (Preview)
You can run some code before calling `super(...)` in a constructor.

```java
class Base {
    Base(String msg) {
        System.out.println("Base: " + msg);
    }
}

class Child extends Base {
    Child(String msg) {
        System.out.println("Preparing...");
        super(msg.toUpperCase());  // allowed in preview
        System.out.println("Done.");
    }
}

public class Main {
    public static void main(String[] args) {
        new Child("hello");
    }
}
```

**Output:**
```
Preparing...
Base: HELLO
Done.
```

### 3. String Templates (Second Preview)
Safer, more powerful string interpolation.

```java
import static java.lang.StringTemplate.RAW;

public class Main {
    public static void main(String[] args) {
        int x = 10;
        int y = 20;
        StringTemplate st = RAW."x = \{x}, y = \{y}, sum = \{x + y}";
        String result = st.interpolate();  // or use processors
        System.out.println(result);
    }
}
```

**Output:**
```
x = 10, y = 20, sum = 30
```

---

### 4. Structured Concurrency (Second Preview)
Treat a group of tasks as a single unit, simplifying concurrent code.

```java
import java.util.concurrent.*;
import jdk.incubator.concurrent.StructuredTaskScope;

public class Main {
    static String fetchUser() throws InterruptedException {
        Thread.sleep(200);
        return "Alice";
    }

    static int fetchScore() throws InterruptedException {
        Thread.sleep(300);
        return 42;
    }

    public static void main(String[] args) throws Exception {
        try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
            var userFuture  = scope.fork(Main::fetchUser);
            var scoreFuture = scope.fork(Main::fetchScore);

            scope.join();           // wait for both
            scope.throwIfFailed();  // propagate error if any

            String user = userFuture.get();
            int score   = scoreFuture.get();
            System.out.println(user + " -> " + score);
        }
    }
}
```

**Output:**
```
Alice -> 42
```

### 5. Scoped Values (Second Preview)
A safer alternative to thread-local variables, especially with virtual threads.

```java
import jdk.incubator.concurrent.ScopedValue;

public class Main {
    static final ScopedValue<String> USER = ScopedValue.newInstance();

    public static void main(String[] args) {
        ScopedValue.where(USER, "Alice").run(() -> {
            doWork();
        });
    }

    static void doWork() {
        System.out.println("Current user: " + USER.get());
    }
}
```

**Output:**
```
Current user: Alice
```

### 6. Foreign Function & Memory API (Third Preview)
Call native code and manage off-heap memory safely (replacing JNI in many cases).

```java
import java.lang.foreign.*;
import java.lang.invoke.MethodHandle;

public class Main {
    public static void main(String[] args) throws Throwable {
        Linker linker = Linker.nativeLinker();
        SymbolLookup stdlib = linker.defaultLookup();

        // Example: call C's "strlen"
        MethodHandle strlen = linker.downcallHandle(
            stdlib.find("strlen").orElseThrow(),
            FunctionDescriptor.of(ValueLayout.JAVA_LONG, ValueLayout.ADDRESS)
        );

        try (Arena arena = Arena.ofConfined()) {
            MemorySegment cString = arena.allocateUtf8String("Hello");
            long len = (long) strlen.invoke(cString);
            System.out.println("Length = " + len);
        }
    }
}
```

**Output:**
```
Length = 5
```

## Java 23 (expected / in development)

(Names and details may change; this is based on current proposals.)

### 1. String Templates – Final
String templates are expected to become a standard feature (no longer preview), making interpolation more ergonomic and safe.

### 2. Structured Concurrency – Final
Structured concurrency is expected to be finalized, giving a stable API for managing concurrent tasks.

### 3. Scoped Values – Final
Scoped values likely become permanent, recommended over `ThreadLocal` for many use cases.

### 4. More Pattern Matching Enhancements
Further refinements to pattern matching for `switch`, records, and sealed types.

```java
sealed interface Shape permits Circle, Rectangle {}
record Circle(double r) implements Shape {}
record Rectangle(double w, double h) implements Shape {}

static double area(Shape s) {
    return switch (s) {
        case Circle(double r)      -> Math.PI * r * r;
        case Rectangle(double w, double h) -> w * h;
    };
}
```

**Input/Output example:**

```java
public static void main(String[] args) {
    System.out.println(area(new Circle(2)));
    System.out.println(area(new Rectangle(3, 4)));
}
```

**Output:**
```
12.566370614359172
12.0
```

## Java 24–25 (longer-term / roadmap ideas)

These are *not final*; they are directions and possible features:

### 1. Further Virtual Thread Improvements
- Better debugging and profiling support.
- More libraries optimized for virtual threads.

```java
public class Main {
    public static void main(String[] args) throws Exception {
        try (var executor = java.util.concurrent.Executors.newVirtualThreadPerTaskExecutor()) {
            for (int i = 0; i < 1_000_000; i++) {
                int id = i;
                executor.submit(() -> {
                    // lightweight task
                    if (id == 0) System.out.println("First task running");
                });
            }
        }
    }
}
```

**Output (example):**
```
First task running
```

### 2. More Foreign Function & Memory API Enhancements
- Easier bindings to popular native libraries.
- Better performance and safety checks.

### 3. Language Cleanups and Quality-of-Life
- More pattern matching forms.
- Possible new small syntax improvements (e.g., more concise declarations, better generics ergonomics).
- Continued deprecation/removal of old, unsafe APIs.

---

## Summary of Main Ideas (22–25)

- **Java 22**: Focus on modern language constructs (string templates, unnamed patterns, pre-`super` statements) and powerful concurrency tools (structured concurrency, scoped values), plus a mature Foreign Function & Memory API.
- **Java 23**: Expected to **stabilize** these features (string templates, structured concurrency, scoped values) and refine pattern matching.
- **Java 24–25**: Likely to continue improving **virtual threads**, **FFM API**, and **pattern matching**, and to clean up older parts of the language and runtime.
