
Here’s a concise overview of the main Java synchronizers in `java.util.concurrent`, with simple code examples and sample I/O where it makes sense.

---

## 1. CountDownLatch

**Idea:**  
Allows one or more threads to wait until a set of operations in other threads completes.  
You set an initial count; each `countDown()` reduces it. When it reaches 0, waiting threads are released.

**Typical use:**  
- Start multiple worker threads, wait for all to finish before continuing.

**Key methods:**
- `new CountDownLatch(int count)`
- `await()` – waits until count reaches 0
- `countDown()` – decrements the count

**Example:**

```java
import java.util.concurrent.CountDownLatch;

public class CountDownLatchExample {
    public static void main(String[] args) throws InterruptedException {
        int workers = 3;
        CountDownLatch latch = new CountDownLatch(workers);

        for (int i = 1; i <= workers; i++) {
            final int id = i;
            new Thread(() -> {
                System.out.println("Worker " + id + " started");
                try {
                    Thread.sleep(500L * id); // simulate work
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
                System.out.println("Worker " + id + " finished");
                latch.countDown();
            }).start();
        }

        System.out.println("Main thread waiting for workers...");
        latch.await(); // waits until count == 0
        System.out.println("All workers finished. Main continues.");
    }
}
```

**Possible output (order may vary):**
```
Main thread waiting for workers...
Worker 1 started
Worker 2 started
Worker 3 started
Worker 1 finished
Worker 2 finished
Worker 3 finished
All workers finished. Main continues.
```

## 2. CyclicBarrier

**Idea:**  
Makes a group of threads wait for each other at a common barrier point.  
Once all parties reach the barrier, they all proceed. The barrier is *reusable* (cyclic).

**Typical use:**  
- Multiple threads performing phases of work in lockstep (e.g., simulation steps).

**Key methods:**
- `new CyclicBarrier(int parties)`
- `new CyclicBarrier(int parties, Runnable barrierAction)`
- `await()` – waits until all parties have called `await()`

**Example:**

```java
import java.util.concurrent.BrokenBarrierException;
import java.util.concurrent.CyclicBarrier;

public class CyclicBarrierExample {
    public static void main(String[] args) {
        int parties = 3;
        CyclicBarrier barrier = new CyclicBarrier(parties, 
            () -> System.out.println("All reached barrier. Proceeding to next phase."));

        for (int i = 1; i <= parties; i++) {
            final int id = i;
            new Thread(() -> {
                try {
                    System.out.println("Thread " + id + " doing phase 1 work");
                    Thread.sleep(300L * id);
                    System.out.println("Thread " + id + " waiting at barrier");
                    barrier.await(); // phase 1 barrier

                    System.out.println("Thread " + id + " doing phase 2 work");
                } catch (InterruptedException | BrokenBarrierException e) {
                    Thread.currentThread().interrupt();
                }
            }).start();
        }
    }
}
```

**Possible output (order may vary):**
```
Thread 1 doing phase 1 work
Thread 2 doing phase 1 work
Thread 3 doing phase 1 work
Thread 1 waiting at barrier
Thread 2 waiting at barrier
Thread 3 waiting at barrier
All reached barrier. Proceeding to next phase.
Thread 1 doing phase 2 work
Thread 2 doing phase 2 work
Thread 3 doing phase 2 work
```

## 3. Semaphore

**Idea:**  
Controls access to a limited number of permits (resources).  
Threads acquire permits before proceeding and release them when done.

**Typical use:**  
- Limit concurrent access to a resource (e.g., max 3 threads accessing a service).

**Key methods:**
- `new Semaphore(int permits)`
- `acquire()` / `acquire(int permits)` – blocks until permits available
- `release()` / `release(int permits)` – returns permits

**Example:**

```java
import java.util.concurrent.Semaphore;

public class SemaphoreExample {
    public static void main(String[] args) {
        int maxConcurrent = 2;
        Semaphore semaphore = new Semaphore(maxConcurrent);

        for (int i = 1; i <= 5; i++) {
            final int id = i;
            new Thread(() -> {
                try {
                    System.out.println("Task " + id + " waiting for permit");
                    semaphore.acquire();
                    System.out.println("Task " + id + " acquired permit");
                    Thread.sleep(1000); // simulate work
                    System.out.println("Task " + id + " releasing permit");
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                } finally {
                    semaphore.release();
                }
            }).start();
        }
    }
}
```

**Possible output (order may vary, but only 2 tasks run at once):**
```
Task 1 waiting for permit
Task 2 waiting for permit
Task 3 waiting for permit
Task 4 waiting for permit
Task 5 waiting for permit
Task 1 acquired permit
Task 2 acquired permit
Task 1 releasing permit
Task 3 acquired permit
Task 2 releasing permit
Task 4 acquired permit
Task 3 releasing permit
Task 5 acquired permit
Task 4 releasing permit
Task 5 releasing permit
```

## 4. Phaser

**Idea:**  
A more flexible, dynamic version of `CyclicBarrier`.  
Supports multiple phases and allows parties (threads) to register/deregister during runtime.

**Typical use:**  
- Complex multi-phase tasks where the number of participating threads can change.

**Key methods:**
- `new Phaser()` or `new Phaser(int parties)`
- `register()` / `bulkRegister(int parties)` – add parties
- `arrive()` / `arriveAndAwaitAdvance()` – signal arrival, optionally wait for others
- `arriveAndDeregister()` – signal arrival and leave the phaser

**Example:**

```java
import java.util.concurrent.Phaser;

public class PhaserExample {
    public static void main(String[] args) {
        Phaser phaser = new Phaser(1); // main thread registered

        for (int i = 1; i <= 3; i++) {
            phaser.register(); // register new party
            final int id = i;
            new Thread(() -> {
                System.out.println("Thread " + id + " phase 0 work");
                phaser.arriveAndAwaitAdvance(); // wait for others

                System.out.println("Thread " + id + " phase 1 work");
                phaser.arriveAndAwaitAdvance(); // wait for others

                System.out.println("Thread " + id + " done, deregistering");
                phaser.arriveAndDeregister();
            }).start();
        }

        // main participates in phases too
        System.out.println("Main phase 0 work");
        phaser.arriveAndAwaitAdvance(); // end of phase 0

        System.out.println("Main phase 1 work");
        phaser.arriveAndAwaitAdvance(); // end of phase 1

        // main deregisters
        phaser.arriveAndDeregister();
        System.out.println("Main done");
    }
}
```

**Possible output (order may vary, but phases are synchronized):**
```
Main phase 0 work
Thread 1 phase 0 work
Thread 2 phase 0 work
Thread 3 phase 0 work
Main phase 1 work
Thread 1 phase 1 work
Thread 2 phase 1 work
Thread 3 phase 1 work
Thread 1 done, deregistering
Thread 2 done, deregistering
Thread 3 done, deregistering
Main done
```

---

## Summary of Differences

- **CountDownLatch**: One-time countdown; once it reaches zero, it cannot be reset.
- **CyclicBarrier**: Reusable barrier for a fixed number of threads; all wait for each other.
- **Semaphore**: Controls how many threads can access a resource concurrently.
- **Phaser**: Multi-phase, dynamic barrier; threads can join/leave and synchronize across multiple phases.
