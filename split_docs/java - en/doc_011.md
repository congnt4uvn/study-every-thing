
Here’s a concise summary of newer Java **switch expressions** and how they differ from traditional `switch` statements, with code examples and sample I/O.

---

## 1. Switch as an Expression (returns a value)

**Old style (statement only):**

```java
int numLetters;
String day = "MONDAY";

switch (day) {
    case "MONDAY":
    case "FRIDAY":
    case "SUNDAY":
        numLetters = 6;
        break;
    case "TUESDAY":
        numLetters = 7;
        break;
    case "THURSDAY":
    case "SATURDAY":
        numLetters = 8;
        break;
    case "WEDNESDAY":
        numLetters = 9;
        break;
    default:
        throw new IllegalArgumentException("Unknown day: " + day);
}

System.out.println(numLetters);
```

**Output:**

```
6
```

**New style (expression, introduced in Java 14):**

```java
String day = "MONDAY";

int numLetters = switch (day) {
    case "MONDAY", "FRIDAY", "SUNDAY" -> 6;
    case "TUESDAY"                    -> 7;
    case "THURSDAY", "SATURDAY"       -> 8;
    case "WEDNESDAY"                  -> 9;
    default -> throw new IllegalArgumentException("Unknown day: " + day);
};

System.out.println(numLetters);
```

**Output:**

```
6
```

Key idea: `switch` can now directly produce a value and be assigned to a variable.

---

## 2. Arrow syntax (`->`) and no fall-through

- New `case` syntax uses `->` instead of `:`.
- No implicit fall-through; each case is self-contained.
- Multiple labels can be combined with commas.

Example:

```java
int dayNumber = 3;

String type = switch (dayNumber) {
    case 1, 7 -> "Weekend";
    case 2, 3, 4, 5, 6 -> "Weekday";
    default -> "Invalid";
};

System.out.println(type);
```

**Input:** `dayNumber = 3`  
**Output:**

```
Weekday
```

---

## 3. Using `yield` for multi-statement cases

If a case needs more than one statement, use a block and `yield` to return a value.

```java
int score = 85;

String grade = switch (score / 10) {
    case 10, 9 -> "A";
    case 8 -> {
        String msg = "B (good)";
        yield msg;  // return value from this case
    }
    case 7 -> "C";
    case 6 -> "D";
    default -> "F";
};

System.out.println(grade);
```

**Input:** `score = 85`  
**Output:**

```
B (good)
```

---

## 4. Works with enums and other types

Switch expressions work especially well with enums:

```java
enum Operation { PLUS, MINUS, MULTIPLY, DIVIDE }

static int apply(Operation op, int x, int y) {
    return switch (op) {
        case PLUS      -> x + y;
        case MINUS     -> x - y;
        case MULTIPLY  -> x * y;
        case DIVIDE    -> {
            if (y == 0) throw new ArithmeticException("Divide by zero");
            yield x / y;
        }
    };
}

public static void main(String[] args) {
    System.out.println(apply(Operation.PLUS, 3, 4));
    System.out.println(apply(Operation.DIVIDE, 10, 2));
}
```

**Output:**

```
7
5
```

---

## 5. Main benefits

- **Expression form**: `switch` can return a value directly.
- **Safer**: No accidental fall-through; clearer logic.
- **More concise**: Arrow syntax and multiple labels.
- **More powerful**: `yield` allows complex logic while still returning a value.

These features are available starting from **Java 14** (previewed earlier in Java 12/13).
