
Here’s a concise summary of the Java memory model focusing on **heap vs stack**, with simple code examples and sample outputs.

---

## 1. Stack Memory

**Key ideas:**
- Used for **method calls**, **local variables**, and **reference variables**.
- Follows **LIFO (Last-In, First-Out)**.
- Memory is automatically managed: when a method finishes, its stack frame is removed.
- Very fast, small, and **thread-specific** (each thread has its own stack).

**Example: local variables on the stack**

```java
public class StackExample {
    public static void main(String[] args) {
        int x = 10;          // x is on the stack
        int y = add(x, 5);   // x and y are on the stack
        System.out.println(y);
    }

    static int add(int a, int b) { // a and b are on the stack
        int sum = a + b;           // sum is on the stack
        return sum;
    }
}
```

**Input:** (no input)  
**Output:**
```text
15
```

When `add` finishes, its local variables (`a`, `b`, `sum`) are removed from the stack.

---

## 2. Heap Memory

**Key ideas:**
- Used for **objects** and **arrays** created with `new`.
- Shared by **all threads**.
- Managed by the **Garbage Collector (GC)**: objects are freed when no references point to them.
- Larger but slower to access than the stack.

**Example: objects on the heap**

```java
class Person {
    String name;  // field stored in the object on the heap
    int age;

    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

public class HeapExample {
    public static void main(String[] args) {
        Person p1 = new Person("Alice", 30); // Person object on heap, p1 is a reference on stack
        Person p2 = p1;                      // p2 is another reference on stack to the same heap object

        System.out.println(p1.name + ", " + p1.age);
        System.out.println(p2.name + ", " + p2.age);

        p1.age = 31; // modifies the same heap object

        System.out.println(p1.age);
        System.out.println(p2.age);
    }
}
```

**Output:**
```text
Alice, 30
Alice, 30
31
31
```

Both `p1` and `p2` point to the **same object on the heap**.

---

## 3. How Stack and Heap Work Together

- **Primitive local variables** (e.g., `int`, `double`, `boolean`) live directly on the **stack**.
- **Reference variables** live on the **stack**, but they point to objects on the **heap**.
- When a method returns, its stack frame is removed, but objects on the heap remain if still referenced elsewhere.

**Example combining both:**

```java
class Box {
    int value;
    Box(int value) {
        this.value = value;
    }
}

public class StackHeapDemo {
    public static void main(String[] args) {
        int a = 5;              // primitive on stack
        Box box = new Box(10);  // box reference on stack, Box object on heap

        modify(a, box);

        System.out.println("a after modify: " + a);
        System.out.println("box.value after modify: " + box.value);
    }

    static void modify(int x, Box b) {
        x = 100;        // changes only local copy on stack
        b.value = 200;  // changes heap object
    }
}
```

**Output:**
```text
a after modify: 5
box.value after modify: 200
```

- `a` is unchanged because primitives are passed by value (copied on the stack).
- `box.value` changes because the method has a copy of the **reference**, still pointing to the same heap object.

---

## 4. Summary

- **Stack**
  - Stores: method calls, local variables, references.
  - Scope: per thread.
  - Lifetime: until method returns.
  - Fast, automatically cleaned.

- **Heap**
  - Stores: objects and arrays.
  - Scope: shared across threads.
  - Lifetime: until no references remain (GC).
  - Larger, slower, managed by garbage collector.
