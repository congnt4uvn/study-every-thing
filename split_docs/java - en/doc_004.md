
Here’s a concise summary of Java enums and annotations, with code examples and sample inputs/outputs.

---

## 1. Enums in Java

**What they are:**  
Enums define a fixed set of constant values (type-safe). They are special classes.

### Basic Enum

```java
public enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}
```

### Using an Enum

```java
public class EnumExample {
    public static void main(String[] args) {
        Day today = Day.MONDAY;

        if (today == Day.MONDAY) {
            System.out.println("Start of the week!");
        }

        // Loop through all values
        for (Day d : Day.values()) {
            System.out.println(d);
        }
    }
}
```

**Sample Output:**
```
Start of the week!
MONDAY
TUESDAY
WEDNESDAY
THURSDAY
FRIDAY
SATURDAY
SUNDAY
```

### Enum with Fields and Methods

```java
public enum Status {
    NEW(0),
    IN_PROGRESS(1),
    DONE(2);

    private final int code;

    Status(int code) {
        this.code = code;
    }

    public int getCode() {
        return code;
    }
}

class StatusTest {
    public static void main(String[] args) {
        Status s = Status.IN_PROGRESS;
        System.out.println(s);              // IN_PROGRESS
        System.out.println(s.getCode());    // 1
    }
}
```

**Key points:**
- Enums are type-safe constants.
- Can have fields, constructors, and methods.
- `values()` returns all constants; `valueOf("NAME")` converts a string to an enum constant.

## 2. Annotations in Java

**What they are:**  
Metadata attached to code (classes, methods, fields, etc.) that can be used by the compiler or at runtime.

### Common Built-in Annotations

1. `@Override` – method overrides a superclass method.
2. `@Deprecated` – marks code as obsolete.
3. `@SuppressWarnings` – tells compiler to ignore specific warnings.
4. `@FunctionalInterface` – marks an interface with exactly one abstract method.

#### Example: @Override

```java
class Parent {
    void sayHello() {
        System.out.println("Hello from Parent");
    }
}

class Child extends Parent {
    @Override
    void sayHello() {
        System.out.println("Hello from Child");
    }
}

class TestOverride {
    public static void main(String[] args) {
        Parent p = new Child();
        p.sayHello();
    }
}
```

**Output:**
```
Hello from Child
```

#### Example: @Deprecated

```java
class OldApi {
    @Deprecated
    public void oldMethod() {
        System.out.println("Old method");
    }
}

class UseOldApi {
    public static void main(String[] args) {
        OldApi api = new OldApi();
        api.oldMethod(); // compiler shows deprecation warning
    }
}
```

## 3. Custom Annotations

You can define your own annotations with `@interface`.

### Defining a Custom Annotation

```java
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;
import java.lang.annotation.ElementType;

// Annotation available at runtime, applicable to methods
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface RunTimes {
    int value();  // parameter of the annotation
}
```

### Using the Custom Annotation

```java
public class TaskRunner {

    @RunTimes(3)
    public void doTask() {
        System.out.println("Task executed");
    }
}
```

### Reading the Annotation via Reflection

```java
import java.lang.reflect.Method;

public class AnnotationProcessor {
    public static void main(String[] args) throws Exception {
        TaskRunner runner = new TaskRunner();

        for (Method m : TaskRunner.class.getDeclaredMethods()) {
            if (m.isAnnotationPresent(RunTimes.class)) {
                RunTimes rt = m.getAnnotation(RunTimes.class);
                int times = rt.value();
                for (int i = 0; i < times; i++) {
                    m.invoke(runner);
                }
            }
        }
    }
}
```

**Output:**
```
Task executed
Task executed
Task executed
```

**Key points:**
- Custom annotations are defined with `@interface`.
- `@Retention` controls whether the annotation is available at source, class, or runtime.
- `@Target` controls where the annotation can be used (class, method, field, etc.).
- Reflection can read runtime annotations and act on them.

---

**Main Ideas Summary:**
- **Enums**: define a fixed set of constants, can have fields/methods, provide type safety.
- **Built-in annotations**: give metadata to the compiler and tools (`@Override`, `@Deprecated`, etc.).
- **Custom annotations**: user-defined metadata, often processed via reflection or frameworks.
