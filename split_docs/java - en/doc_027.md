## 1. What is an Exception?

An **exception** is an event that disrupts the normal flow of a program (e.g., invalid input, file not found, division by zero).  
Java uses exceptions to signal and handle errors in a structured way.

---

## 2. Checked vs Unchecked Exceptions

### Checked Exceptions
- Subclasses of `Exception` **excluding** `RuntimeException`.
- **Must** be either:
  - handled with `try-catch`, or
  - declared with `throws` in the method signature.
- Represent conditions that are often **recoverable** and are expected to happen (e.g., file not found, network issues).

**Examples:**  
`IOException`, `SQLException`, `FileNotFoundException`

**Example code (checked exception):**

```java
import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class CheckedExample {
    public static void main(String[] args) {
        try {
            String content = readFirstLine("input.txt");
            System.out.println("First line: " + content);
        } catch (IOException e) {
            System.out.println("Error reading file: " + e.getMessage());
        }
    }

    // Must declare throws IOException (checked exception)
    public static String readFirstLine(String fileName) throws IOException {
        try (BufferedReader br = new BufferedReader(new FileReader(fileName))) {
            return br.readLine();
        }
    }
}
```

**Sample input/output:**

- If `input.txt` exists and first line is `Hello`:

Output:
```text
First line: Hello
```

- If `input.txt` does not exist:

Output:
```text
Error reading file: input.txt (No such file or directory)
```

---

### Unchecked Exceptions
- Subclasses of `RuntimeException`.
- **Do not need** to be declared or caught.
- Usually indicate **programming errors** (bugs) such as:
  - invalid arguments,
  - null references,
  - logic errors.

**Examples:**  
`NullPointerException`, `ArrayIndexOutOfBoundsException`, `IllegalArgumentException`, `ArithmeticException`

**Example code (unchecked exception):**

```java
public class UncheckedExample {
    public static void main(String[] args) {
        int result = divide(10, 0); // will cause ArithmeticException
        System.out.println("Result: " + result);
    }

    public static int divide(int a, int b) {
        // No throws declaration needed (unchecked exception)
        return a / b;
    }
}
```

**Sample input/output:**

Input: (no user input; code calls `divide(10, 0)`)

Output:
```text
Exception in thread "main" java.lang.ArithmeticException: / by zero
    at UncheckedExample.divide(UncheckedExample.java:10)
    at UncheckedExample.main(UncheckedExample.java:5)
```

## 3. When to Use Which?

- Use **checked exceptions** for:
  - predictable, recoverable problems (I/O, network, database).
  - situations where the caller is expected to handle the problem.

- Use **unchecked exceptions** for:
  - programming errors (invalid state, wrong arguments).
  - conditions that usually cannot be meaningfully recovered from.

---

## 4. Basic Exception Handling Pattern

```java
try {
    // code that may throw an exception
} catch (SpecificException e) {
    // handle specific problem
} catch (Exception e) {
    // handle any other exception
} finally {
    // optional: always runs (e.g., close resources)
}
```

## 5. Custom Exceptions

You can define your own checked or unchecked exceptions:

**Custom checked exception:**

```java
class InvalidAgeException extends Exception {
    public InvalidAgeException(String message) {
        super(message);
    }
}

public class CustomChecked {
    public static void main(String[] args) {
        try {
            registerUser(15);
        } catch (InvalidAgeException e) {
            System.out.println("Registration failed: " + e.getMessage());
        }
    }

    public static void registerUser(int age) throws InvalidAgeException {
        if (age < 18) {
            throw new InvalidAgeException("User must be at least 18.");
        }
        System.out.println("User registered successfully.");
    }
}
```

**Sample output:**

```text
Registration failed: User must be at least 18.
```

**Custom unchecked exception:**

```java
class NegativeNumberException extends RuntimeException {
    public NegativeNumberException(String message) {
        super(message);
    }
}

public class CustomUnchecked {
    public static void main(String[] args) {
        int result = sqrt(-4); // will throw NegativeNumberException
        System.out.println("Result: " + result);
    }

    public static int sqrt(int x) {
        if (x < 0) {
            throw new NegativeNumberException("x must be non-negative");
        }
        // dummy implementation
        return (int) Math.sqrt(x);
    }
}
```

---

**Key idea:**  
- **Checked** = must handle or declare, for expected recoverable issues.  
- **Unchecked** = optional to handle, for programming errors and unexpected runtime issues.
