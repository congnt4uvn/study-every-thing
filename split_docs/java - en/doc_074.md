
Here’s a concise summary of **Spring Security: Authentication vs Authorization**, with simple code examples.

---

## 1. Core Ideas

- **Authentication**:  
  *Who are you?*  
  Verifies the user’s identity (e.g., username/password, token, OAuth2).

- **Authorization**:  
  *What are you allowed to do?*  
  Decides whether an authenticated user can access a specific resource or perform an action (based on roles/authorities).

In Spring Security:

- Authentication → produces an `Authentication` object stored in the `SecurityContext`.
- Authorization → uses that `Authentication` (especially its roles/authorities) to allow or deny access.

---

## 2. Basic Authentication Example

### Security Configuration (Spring Security 6+ style)

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable())
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/public/**").permitAll()      // no auth needed
                .requestMatchers("/admin/**").hasRole("ADMIN")  // authorization rule
                .anyRequest().authenticated()                   // all others need login
            )
            .httpBasic(Customizer.withDefaults()); // simple HTTP Basic auth

        return http.build();
    }

    @Bean
    public UserDetailsService userDetailsService() {
        UserDetails user = User.withUsername("user")
            .password("{noop}password") // {noop} = no encoding, for demo only
            .roles("USER")
            .build();

        UserDetails admin = User.withUsername("admin")
            .password("{noop}admin123")
            .roles("ADMIN")
            .build();

        return new InMemoryUserDetailsManager(user, admin);
    }
}
```

### How Authentication Works Here

- Client sends a request with HTTP Basic credentials (e.g., `Authorization: Basic ...`).
- Spring Security:
  - Extracts username/password.
  - Uses `UserDetailsService` to load the user.
  - If valid → creates an `Authentication` object and stores it in `SecurityContextHolder`.
  - If invalid → returns `401 Unauthorized`.

**Example request (input):**

```http
GET /hello HTTP/1.1
Host: localhost:8080
Authorization: Basic dXNlcjpwYXNzd29yZA==  // "user:password"
```

**Example response (output):**

```http
HTTP/1.1 200 OK
Hello, user!
```

If wrong password:

```http
HTTP/1.1 401 Unauthorized
WWW-Authenticate: Basic realm="Realm"
```

## 3. Authorization Example (URL-based)

From the config above:

```java
.requestMatchers("/admin/**").hasRole("ADMIN")
.anyRequest().authenticated()
```

- `/admin/**` → only users with role `ADMIN` can access.
- Any other endpoint → any authenticated user can access.

**Example:**

- Authenticated as `user` (role `USER`) and call `/admin/dashboard`:

```http
GET /admin/dashboard HTTP/1.1
Authorization: Basic dXNlcjpwYXNzd29yZA==
```

**Response:**

```http
HTTP/1.1 403 Forbidden
```

- Authenticated as `admin` (role `ADMIN`) and call `/admin/dashboard`:

```http
GET /admin/dashboard HTTP/1.1
Authorization: Basic YWRtaW46YWRtaW4xMjM=
```

**Response:**

```http
HTTP/1.1 200 OK
... admin content ...
```

Here:
- Both users are **authenticated**.
- Only `admin` is **authorized** for `/admin/**`.

## 4. Method-Level Authorization Example

You can also protect methods using annotations:

```java
@EnableMethodSecurity
@Configuration
public class MethodSecurityConfig {
}
```

```java
@Service
public class AccountService {

    @PreAuthorize("hasRole('ADMIN')")
    public void deleteAccount(Long id) {
        // only ADMIN can delete
    }

    @PreAuthorize("#username == authentication.name")
    public Account getMyAccount(String username) {
        // user can only access their own account
        return ...;
    }
}
```

- `@PreAuthorize("hasRole('ADMIN')")` → authorization based on role.
- `authentication.name` → current authenticated username.

---

## 5. Summary

- **Authentication**:  
  - Confirms identity (login, token validation).
  - Produces an `Authentication` object with principal, credentials, and authorities.

- **Authorization**:  
  - Uses the authenticated user’s roles/authorities to decide access.
  - Implemented via:
    - URL rules (`authorizeHttpRequests`)
    - Method annotations (`@PreAuthorize`, `@PostAuthorize`, etc.)

Both are tightly integrated in Spring Security:  
**First authenticate, then authorize.**
