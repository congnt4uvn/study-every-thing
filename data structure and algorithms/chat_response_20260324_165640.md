Certainly! The Fibonacci sequence is a classic example where dynamic programming can be used to optimize the computation by storing the results of subproblems to avoid redundant calculations. Below is a Java program that demonstrates this using dynamic programming with memoization.

```java
import java.util.HashMap;
import java.util.Map;

public class FibonacciDP {

    // Memoization map to store previously computed Fibonacci numbers
    private Map<Integer, Long> memo = new HashMap<>();

    // Main function to compute Fibonacci using dynamic programming
    public long fibonacci(int n) {
        if (n <= 1) {
            return n;
        }

        // Check if the result is already computed
        if (memo.containsKey(n)) {
            return memo.get(n);
        }

        // Compute the Fibonacci number recursively and store it in the map
        long result = fibonacci(n - 1) + fibonacci(n - 2);
        memo.put(n, result);

        return result;
    }

    public static void main(String[] args) {
        FibonacciDP fibDP = new FibonacciDP();

        // Example input
        int n = 50;

        // Compute and print the Fibonacci number
        long result = fibDP.fibonacci(n);
        System.out.println("Fibonacci number for n = " + n + " is " + result);
    }
}
```

### Explanation:

1. **Data Structure**: 
   - We use a `HashMap<Integer, Long>` called `memo` to store the results of Fibonacci numbers that have already been computed. This helps in avoiding redundant calculations.

2. **Algorithm**:
   - The `fibonacci` method checks if the Fibonacci number for a given `n` is already computed and stored in `memo`. If so, it returns the stored value.
   - If not, it computes the Fibonacci number recursively and stores the result in `memo` before returning it.

3. **Input/Output**:
   - The input is the integer `n` for which we want to compute the Fibonacci number.
   - The output is the Fibonacci number for the given `n`, printed to the console.

This approach efficiently computes Fibonacci numbers even for large values of `n` by reducing the time complexity from exponential to linear, thanks to memoization.
