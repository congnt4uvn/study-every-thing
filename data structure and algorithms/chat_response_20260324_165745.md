Certainly! The Rabin-Karp algorithm is a string-searching algorithm that uses hashing to find a pattern within a text. It is particularly useful for multiple pattern searches. Below is a Java implementation of the Rabin-Karp algorithm, including all the main functions, data structures, and sample input/output.

```java
public class RabinKarp {

    // Function to perform Rabin-Karp algorithm for substring search
    public static int rabinKarpSearch(String text, String pattern) {
        int m = pattern.length();
        int n = text.length();
        int prime = 101; // A prime number for hashing
        int hashPattern = 0; // Hash value for pattern
        int hashText = 0; // Hash value for text
        int h = 1;
        int d = 256; // Number of characters in the input alphabet

        // The value of h would be "pow(d, m-1)%prime"
        for (int i = 0; i < m - 1; i++) {
            h = (h * d) % prime;
        }

        // Calculate the hash value of pattern and first window of text
        for (int i = 0; i < m; i++) {
            hashPattern = (d * hashPattern + pattern.charAt(i)) % prime;
            hashText = (d * hashText + text.charAt(i)) % prime;
        }

        // Slide the pattern over text one by one
        for (int i = 0; i <= n - m; i++) {

            // Check the hash values of current window of text and pattern
            if (hashPattern == hashText) {
                // Check for characters one by one
                int j;
                for (j = 0; j < m; j++) {
                    if (text.charAt(i + j) != pattern.charAt(j)) {
                        break;
                    }
                }

                // If hashPattern == hashText and pattern[0...m-1] = text[i...i+m-1]
                if (j == m) {
                    return i; // Pattern found at index i
                }
            }

            // Calculate hash value for next window of text: Remove leading digit, add trailing digit
            if (i < n - m) {
                hashText = (d * (hashText - text.charAt(i) * h) + text.charAt(i + m)) % prime;

                // We might get negative value of hashText, converting it to positive
                if (hashText < 0) {
                    hashText = (hashText + prime);
                }
            }
        }

        return -1; // Pattern not found
    }

    public static void main(String[] args) {
        String text = "GEEKS FOR GEEKS";
        String pattern = "FOR";

        int result = rabinKarpSearch(text, pattern);

        if (result != -1) {
            System.out.println("Pattern found at index: " + result);
        } else {
            System.out.println("Pattern not found");
        }
    }
}
```

### Explanation:

- **Hashing**: The algorithm computes a hash value for the pattern and each substring of the text of the same length as the pattern. If the hash values match, it checks the actual characters to confirm a match.
  
- **Sliding Window**: The hash value of the current window of text is updated to the next window by removing the leading character and adding the trailing character.

- **Prime Number**: A prime number is used to reduce the probability of hash collisions.

### Input/Output:

- **Input**: The text "GEEKS FOR GEEKS" and the pattern "FOR".
- **Output**: "Pattern found at index: 6" (since "FOR" starts at index 6 in the text).

This implementation efficiently finds the pattern in the text using the Rabin-Karp algorithm.
