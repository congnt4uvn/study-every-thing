Certainly! The Activity Selection problem is a classic example of a greedy algorithm. The goal is to select the maximum number of activities that don't overlap, given their start and end times. Here's a Java implementation that includes the main function, necessary data structures, and the algorithm itself:

```java
import java.util.ArrayList;
import java.util.Arrays;
import java.util.Comparator;
import java.util.List;

class Activity {
    int start;
    int end;

    Activity(int start, int end) {
        this.start = start;
        this.end = end;
    }
}

public class ActivitySelection {

    // Function to perform the activity selection
    public static List<Activity> selectActivities(List<Activity> activities) {
        // Sort activities based on their end times
        activities.sort(Comparator.comparingInt(a -> a.end));

        List<Activity> selectedActivities = new ArrayList<>();
        int lastEndTime = 0;

        for (Activity activity : activities) {
            if (activity.start >= lastEndTime) {
                selectedActivities.add(activity);
                lastEndTime = activity.end;
            }
        }

        return selectedActivities;
    }

    public static void main(String[] args) {
        // Sample input: list of activities with start and end times
        List<Activity> activities = Arrays.asList(
            new Activity(1, 3),
            new Activity(2, 5),
            new Activity(4, 6),
            new Activity(6, 8),
            new Activity(5, 7),
            new Activity(8, 9)
        );

        // Get the maximum set of non-overlapping activities
        List<Activity> selectedActivities = selectActivities(activities);

        // Output the selected activities
        System.out.println("Selected activities:");
        for (Activity activity : selectedActivities) {
            System.out.println("Activity starts at: " + activity.start + ", ends at: " + activity.end);
        }
    }
}
```

### Explanation:

1. **Activity Class**: This class represents an activity with a start and end time.

2. **selectActivities Function**: 
   - First, it sorts the list of activities based on their end times. This is crucial for the greedy approach, as selecting the activity that finishes the earliest leaves the most room for subsequent activities.
   - It then iterates through the sorted list, selecting activities that start after or when the last selected activity ends.

3. **Main Function**:
   - Initializes a list of activities with their start and end times.
   - Calls the `selectActivities` function to get the maximum set of non-overlapping activities.
   - Prints the selected activities.

### Input/Output:

- **Input**: A list of activities with their start and end times.
- **Output**: The maximum number of non-overlapping activities, printed with their start and end times.

This code provides a simple and efficient solution to the Activity Selection problem using a greedy algorithm.
