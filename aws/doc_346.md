


Người hướng dẫn đang giải thích cách làm việc với định nghĩa OpenAPI trong API Gateway:

1. **Nhập (import) OpenAPI**  
   - Tạo một API REST mới, chọn chức năng “Import”.  
   - Chọn tệp định nghĩa API ở định dạng OpenAPI (có thể dùng API mẫu như “Pet Store”).  
   - Sau khi import, API mới (ví dụ Pet Store) sẽ được tạo đầy đủ tài nguyên, phương thức… dựa trên file OpenAPI.

2. **Xuất (export) API hiện tại thành OpenAPI**  
   - Mở API muốn xuất (ví dụ “API đầu tiên của tôi”), chọn một Stage (như “prod”).  
   - Trong phần hành động của Stage, chọn Export.  
   - Chọn định dạng: Swagger hoặc OpenAPI 3, JSON hoặc YAML.  
   - Có thể chọn thêm các tiện ích mở rộng cho API Gateway và Postman.  
   - Hệ thống sẽ tạo file định nghĩa để dùng lại hoặc import ở nơi khác.

3. **Lợi ích của OpenAPI**  
   - Từ file OpenAPI, có thể tự động tạo SDK cho nhiều ngôn ngữ (Android, JavaScript, iOS, Java, Ruby…).  
   - Ứng dụng có thể tương tác với API dễ dàng thông qua SDK này.  
   - Điều này thể hiện sức mạnh của việc dùng định nghĩa OpenAPI trong API Gateway.
