


Dưới đây là tóm tắt nội dung:

1. Bật demo CORS trên index.html  
   - Mở file index.html, bỏ comment phần demo CORS (xóa `<!--` trước `<div>` và `-->` sau `<script>`).  
   - Trang sẽ hiển thị “Hello world, I love coffee” và hình ảnh cà phê.  
   - Một script sẽ fetch thêm một trang HTML (subpage.html) và hiển thị bên dưới.

2. Trường hợp fetch trong cùng origin  
   - Upload 2 file: index.html và subpage.html lên cùng một S3 bucket (đã bật static website hosting).  
   - Mở URL website: nội dung chính + thông báo “additional page has been loaded successfully”.  
   - Điều này chứng minh fetch trong cùng origin hoạt động bình thường (không cần CORS).

3. Chuẩn bị origin khác để demo CORS  
   - Tạo thêm một S3 bucket khác (ví dụ ở region Canada), bật static website hosting.  
   - Đặt bucket ở chế độ public bằng cách chỉnh sửa bucket policy.  
   - Upload subpage.html lên bucket thứ hai.  
   - Kiểm tra URL object: subpage.html có thể truy cập công khai.

4. Chuyển subpage sang origin khác  
   - Xóa subpage.html khỏi bucket đầu tiên (chỉ còn ở bucket thứ hai).  
   - Refresh website đầu tiên: bị lỗi 404 vì không còn subpage.html trong cùng bucket.  
   - Lấy URL đầy đủ của subpage.html ở bucket thứ hai (origin khác).  
   - Sửa index.html ở bucket đầu tiên để fetch bằng full URL này (cross-origin request).  
   - Upload lại index.html lên bucket đầu tiên.

5. Lỗi CORS khi chưa cấu hình  
   - Mở website đầu tiên, bật Developer Tools (Chrome DevTools).  
   - Refresh: không thấy lỗi lớn, nhưng trong Console có thông báo:  
     - Cross-origin request bị chặn do thiếu header CORS `Access-Control-Allow-*`.  
   - Nguyên nhân: bucket thứ hai chưa cấu hình CORS nên trình duyệt chặn request.

6. Cấu hình CORS cho bucket thứ hai  
   - Vào bucket thứ hai → Permissions → CORS configuration.  
   - Dán cấu hình CORS JSON (đã chuẩn bị sẵn).  
   - Trong `AllowedOrigins`, đặt URL website của bucket đầu tiên (http, không có dấu `/` cuối).  
   - Lưu cấu hình CORS.

7. Kết quả sau khi bật CORS  
   - Refresh website đầu tiên: subpage (từ bucket thứ hai) được load thành công.  
   - Trong tab Network, xem request đến subpage:  
     - Response headers có `Access-Control-Allow-Methods: GET`,  
       `Access-Control-Allow-Origin: <URL bucket đầu tiên>`, v.v.  
   - Đây là các header cho phép cross-origin request.

8. Ý nghĩa  
   - Bài giảng minh họa rõ cách hoạt động của CORS giữa hai S3 static website khác origin.  
   - Mục tiêu: giúp hiểu khái niệm CORS để trả lời câu hỏi lý thuyết dễ dàng hơn, dù phần demo hơi nâng cao.
