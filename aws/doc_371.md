


Dưới đây là tóm tắt nội dung:

1. Tạo repository và domain trong CodeArtifact  
   - Tạo một repository mới tên “demo-repository”.  
   - Chọn một upstream repository là “pypi-store” để kéo các gói Python từ public PyPI.  
   - Tạo một domain mới (ví dụ: “my-company”) đại diện cho công ty, chọn khóa KMS (dùng AWS managed key) để mã hóa.  
   - Sau khi tạo xong, ta có:  
     - Repository demo.  
     - Repository upstream (pypi-store) có kết nối ra public PyPI.

2. Cách sử dụng repository với pip  
   - Mở AWS CloudShell để chạy lệnh.  
   - Trong phần “Connection instructions” của CodeArtifact, có lệnh CLI để cấu hình pip tự động.  
   - Tuy nhiên, trong CloudShell xảy ra lỗi “pip not found” (chỉ có pip3), nên cách tự động không dùng được.  
   - Chuyển sang cấu hình thủ công:
     - Dùng lệnh CodeArtifact để lấy authorization token (gắn với domain, owner, region).  
     - Token này có hiệu lực 12 giờ, hết hạn phải lấy lại.  
     - Sử dụng token này để cấu hình pip3 (sửa lệnh từ pip thành pip3) để kết nối tới repository CodeArtifact.

3. Cài đặt gói Python qua CodeArtifact  
   - Sau khi cấu hình pip3, chạy `pip3 install requests`.  
   - Pip kết nối tới CodeArtifact, lấy gói `requests` và các dependency của nó.  
   - Trong giao diện CodeArtifact:
     - Ở repository demo: thấy gói `requests` và các gói phụ thuộc đã được kéo về.  
     - Ở repository upstream (pypi-store): cũng có bản sao các gói đó (một bản ở repo demo, một bản ở repo upstream).

4. Quản lý phiên bản gói  
   - Có thể cài đặt phiên bản khác, ví dụ `requests==0.4.0`.  
   - Sau khi cài, refresh giao diện CodeArtifact sẽ thấy nhiều phiên bản (0.5.0, 0.4.0) của cùng gói trong repository demo.

5. Chính sách truy cập (Policies)  
   - Ở cấp repository:  
     - Có thể áp dụng repository policy để quy định quyền: read-only, read & publish, full access.  
     - Có thể chỉnh sửa policy để gán quyền cho các IAM principal.  
   - Ở cấp domain:  
     - Có thể áp dụng domain policy (ví dụ: quyền cộng tác viên, full domain access cho IAM user/role hoặc tổ chức).  
     - Dùng các mẫu policy có sẵn rồi chỉnh sửa cho phù hợp.

6. Kết luận và dọn dẹp  
   - Có thể tạo thêm nhiều repository, thiết lập upstream giữa chúng, rồi cấu hình pip (hoặc các tool khác) để lấy gói từ đó.  
   - Các dependency sẽ tự động được lưu trữ trong repository.  
   - Khi không cần nữa:  
     - Xóa các repository.  
     - Sau đó xóa domain.  
   - Kết thúc bài giảng với lời hẹn gặp ở bài tiếp theo.
