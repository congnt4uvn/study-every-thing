


Bài giảng nói về cách tích hợp Amazon X-Ray với Elastic Beanstalk:

1. **Beanstalk đã tích hợp sẵn X-Ray daemon**  
   - Không cần tự cài X-Ray daemon, vì nền tảng Beanstalk đã có.  
   - Có 2 cách bật daemon:
     - Bật trực tiếp trong Beanstalk console (phần Platform software → Amazon X-Ray → Enable daemon).  
     - Hoặc tạo file cấu hình `.ebextensions/x-ray-daemon.config` với 1 dòng để bật daemon.

2. **Yêu cầu về IAM Role**  
   - EC2 instance phải dùng IAM role có quyền X-Ray để daemon có thể:
     - Gửi trace data lên dịch vụ X-Ray.  
     - Lấy một số dữ liệu cần thiết.  
   - Ví dụ: role mặc định `aws-elasticbeanstalk-ec2-role` với policy `ElasticBeanstalkWebTier` đã có sẵn quyền X-Ray.  
   - Nếu dùng role tùy chỉnh, phải tự thêm các quyền X-Ray tương ứng.

3. **Yêu cầu về ứng dụng**  
   - Mã ứng dụng phải tích hợp SDK X-Ray để gửi trace (instrumentation trong code).  
   - Nếu chạy nhiều container Docker, phải tự quản lý X-Ray daemon (sẽ nói kỹ hơn trong bài về ECS).

4. **Demo cấu hình trong Beanstalk**  
   - Tạo môi trường Web Server mới trong Beanstalk.  
   - Giữ hầu hết cấu hình mặc định (single instance, không chỉnh VPC, scaling, traffic).  
   - Ở phần Platform software, bật Amazon X-Ray daemon.  
   - Dùng service role và EC2 role mặc định của Beanstalk (đã có quyền X-Ray).  
   - Deploy xong, X-Ray sẽ tự động hoạt động trên EC2 instances.

5. **Lưu ý cuối**  
   - Nếu ứng dụng có tích hợp X-Ray SDK, bạn sẽ thấy trace trong X-Ray console.  
   - Sau khi thực hành xong, nên terminate môi trường Beanstalk để tránh tốn chi phí.
