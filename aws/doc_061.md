


Dưới đây là tóm tắt ngắn gọn nội dung bạn đưa:

1. Khái niệm Application Load Balancer (ALB)  
- Là load balancer lớp 7 (HTTP/HTTPS).  
- Cho phép định tuyến đến nhiều ứng dụng HTTP trên các máy khác nhau, được gom trong “Target Group”.  
- Hỗ trợ HTTP/2, WebSockets, và chuyển hướng (ví dụ tự động chuyển HTTP → HTTPS).

2. Khả năng định tuyến linh hoạt  
ALB có thể định tuyến dựa trên:
- Đường dẫn (path) URL:  
  - Ví dụ: `example.com/users` → Target Group A  
  - `example.com/posts` → Target Group B  
- Tên host:  
  - `a.example.com` → Target Group A  
  - `b.example.com` → Target Group B  
- Query string và header:  
  - Ví dụ: `example.com/reserve?id=123&order=false` → Target Group khác.

3. Ứng dụng cho microservices và container  
- ALB rất phù hợp cho kiến trúc microservices và ứng dụng chạy trên container (ECS, Docker).  
- Hỗ trợ port mapping: có thể chuyển tiếp đến các port động trên ECS.  
- So với Classic Load Balancer (CLB):  
  - CLB thường phải có 1 load balancer cho mỗi ứng dụng.  
  - ALB có thể phục vụ nhiều ứng dụng/microservice phía sau một load balancer duy nhất.

4. Target Group là gì?  
Target Group có thể gồm:  
- EC2 instances (có thể nằm trong Auto Scaling Group).  
- ECS tasks (container).  
- Lambda functions (serverless).  
- Private IP (máy chủ on-premise trong data center riêng).  

Health check được cấu hình ở cấp Target Group, không phải ở cấp ALB.

Ví dụ:  
- ALB → Target Group 1 (EC2 trên AWS)  
- ALB → Target Group 2 (server on-premise, đăng ký bằng private IP)  

Có thể dùng quy tắc dựa trên query string để phân loại:  
- `?platform=mobile` → Target Group 1  
- `?platform=desktop` → Target Group 2  

5. Thông tin về hostname và IP client  
- ALB cung cấp một hostname cố định (giống CLB).  
- Backend (EC2, v.v.) không thấy trực tiếp IP client; ALB “kết thúc kết nối” và dùng IP riêng của nó để nói chuyện với EC2.  
- IP thật, port và protocol của client được chèn vào các header:  
  - `X-Forwarded-For` → IP client  
  - `X-Forwarded-Port` → port  
  - `X-Forwarded-Proto` → giao thức (HTTP/HTTPS)  
- Ứng dụng phía sau ALB phải đọc các header này nếu cần biết IP thật của client.

6. Kết luận  
- ALB là load balancer lớp 7, rất phù hợp cho microservices, container, serverless.  
- Cho phép định tuyến linh hoạt theo path, host, query string, header.  
- Có thể đứng trước EC2, ECS, Lambda, hoặc private IP on-premise.  
- Health check ở cấp Target Group, và client IP được truyền qua các header X-Forwarded-*.

Sau phần lý thuyết này, bước tiếp theo là thực hành tạo ALB đầu tiên.
