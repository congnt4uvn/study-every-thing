


Nội dung trên là một bài hướng dẫn thực hành triển khai ứng dụng serverless với AWS SAM (Serverless Application Model) trên Cloud Shell, cụ thể như sau:

1. **Khởi động môi trường & kiểm tra SAM CLI**  
   - Mở AWS Cloud Shell.  
   - Kiểm tra SAM CLI bằng lệnh `sam --version` (SAM đã được cài sẵn).

2. **Khởi tạo ứng dụng SAM**  
   - Dùng lệnh `sam init`.  
   - Chọn tạo ứng dụng từ “Quick start template”.  
   - Chọn mẫu “Hello World Example”.  
   - Chọn runtime: Python (ban đầu là 3.13, sau đổi thành 3.9).  
   - Tắt X-Ray, CloudWatch Insights, JSON logs.  
   - Đặt tên ứng dụng (ví dụ: `sam-app`).  

3. **Cấu trúc thư mục & các file quan trọng**  
   Sau khi `sam init`, vào thư mục ứng dụng (`cd sam-app`) và liệt kê file (`ls`):  
   - Thư mục `hello_world/`: chứa mã nguồn Lambda (file `app.py`).  
   - Thư mục `tests/`, `events/`: chứa test và file sự kiện mẫu.  
   - File `app.py`:  
     - Import `json`.  
     - Định nghĩa hàm handler Lambda nhận `event` và trả về chuỗi `"Hello world"`.  
   - File `requirements.txt`: khai báo các package Python cần thiết.  
   - File `samconfig.toml`:  
     - Lưu cấu hình SAM (tên stack, tham số build/deploy, v.v.).  
   - File `template.yaml`:  
     - Là AWS SAM/CloudFormation template.  
     - Sử dụng `Transform: AWS::Serverless-2016-10-31`.  
     - Định nghĩa resource kiểu `AWS::Serverless::Function` với các thuộc tính: `CodeUri`, `Handler`, `Runtime`, `Timeout`, v.v.  
     - Đây là “serverless application template” chính.

4. **Lỗi runtime & chỉnh sửa template**  
   - Khi chạy `sam build`, gặp lỗi vì runtime Python 3.13 chưa có trong môi trường (Cloud Shell chỉ có Python 3.9.23).  
   - Mở file `template.yaml` bằng `nano template.yaml`.  
   - Sửa `Runtime` thành `python3.9`.  
   - Lưu file (Ctrl+X → Y → Enter).  
   - Kiểm tra lại bằng `cat template.yaml`.

5. **Build ứng dụng SAM**  
   - Chạy lại `sam build`.  
   - Build thành công, tạo thư mục `.aws-sam/` chứa:  
     - Thư mục `build/`: mã đã build, dependencies, template đã xử lý.  
     - Trong `build/hello_world/` có mã `app.py` và các package Python.  

6. **Triển khai (deploy) ứng dụng**  
   - Dùng lệnh `sam deploy --guided`.  
   - Trả lời các câu hỏi:  
     - Tên stack (ví dụ: `sam-app`).  
     - Region (giữ mặc định).  
     - Xem trước resource và xác nhận (Yes).  
     - Cho phép SAM tạo IAM role (Yes).  
     - Không tắt rollback.  
     - Lưu cấu hình deploy vào `samconfig.toml` (Yes).  
   - SAM sử dụng CloudFormation để:  
     - Tạo stack “default” do SAM CLI quản lý (S3 bucket, IAM role, v.v.).  
     - Sau đó tạo stack ứng dụng chính (Lambda function, API Gateway, IAM permissions, v.v.).  
   - Có thể theo dõi tiến trình trong CloudFormation console hoặc trực tiếp trong terminal.

7. **Kiểm tra tài nguyên sau khi deploy**  
   - Vào AWS Lambda console, lọc theo tên/nhãn SAM, thấy hàm `HelloWorld` đã được tạo.  
   - Mã trong Lambda chính là `app.py` đã viết.  
   - Vào API Gateway console, thấy API ứng dụng SAM:  
     - Có resource `GET` trỏ tới Lambda `HelloWorld`.  

8. **Gọi thử API**  
   - Lấy endpoint “Prod” từ API Gateway.  
   - Dán vào trình duyệt → nhận phản hồi `"Hello world"`.  
   - Hoặc dùng `curl <URL>` trong Cloud Shell → cũng nhận `"Hello world"`.

9. **Kết luận**  
   - Chỉ với SAM CLI và vài file cấu hình (`template.yaml`, `samconfig.toml`, `app.py`, `requirements.txt`), có thể:  
     - Khởi tạo, build, và deploy một ứng dụng serverless hoàn chỉnh gồm Lambda + API Gateway.  
   - Đây là demo đơn giản, nhưng là nền tảng để phát triển các ứng dụng serverless phức tạp hơn với SAM.
