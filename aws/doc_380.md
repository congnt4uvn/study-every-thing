


Nội dung trên nói về các khả năng chạy **cục bộ** của AWS SAM:

1. **Chạy Lambda cục bộ**
   - `sam local start-lambda`: khởi động môi trường Lambda cục bộ, cung cấp các hàm Lambda như các endpoint trên máy.
   - Có thể chạy test tự động lên các endpoint này.
   - `sam local invoke`: gọi một hàm Lambda cục bộ với payload cụ thể, chạy xong thì thoát.

2. **Tương tác với AWS khi test cục bộ**
   - Nếu Lambda gọi các dịch vụ AWS (ví dụ DynamoDB), cần:
     - Khởi động dịch vụ tương ứng cục bộ, hoặc
     - Gọi với `--profile` phù hợp để chỉ định môi trường/tài khoản AWS.

3. **Chạy API Gateway cục bộ**
   - `sam local start-api`: khởi động một HTTP server cục bộ, host toàn bộ API và các Lambda liên quan.
   - Khi cập nhật mã Lambda, server sẽ tự động reload và API được cập nhật theo.

4. **Tạo sự kiện mẫu cho Lambda**
   - `sam local generate-event`: tạo payload mẫu cho các nguồn sự kiện khác nhau.
   - Có thể tạo sự kiện cho S3, API Gateway, SNS, Kinesis, DynamoDB, v.v.
   - Ví dụ: tạo sự kiện S3 với bucket và key cụ thể rồi dùng làm input cho `sam local invoke`.

Tóm lại, SAM cho phép:
- Chạy và test Lambda/API Gateway cục bộ.
- Gọi Lambda với payload tùy ý.
- Tạo sự kiện mẫu từ nhiều nguồn để mô phỏng môi trường thực.
