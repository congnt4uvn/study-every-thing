


Dưới đây là tóm tắt nội dung:

- Mục tiêu: Tạo 3 EC2 instance ở 3 region khác nhau và một Application Load Balancer (ALB) để chuẩn bị cho phần Route 53.

1) Tạo 3 EC2 instance  
- Region 1: Frankfurt (EU Central)  
- Region 2: N. Virginia (US East 1)  
- Region 3: Singapore (AP Southeast 1)  

Các bước cấu hình giống nhau cho cả 3 instance:
- AMI: Amazon Linux 2  
- Instance type: t2.micro  
- Không dùng key pair (sẽ dùng EC2 Instance Connect nếu cần)  
- Security Group: cho phép SSH (nếu cần) và HTTP (port 80) từ mọi nơi  
- Advanced details → User data: dán script “Route 53 user data”  
  - Script này hiển thị “Hello World” và kèm theo thông tin AZ (Availability Zone) của instance (ví dụ: EU Central 1b, US East 1a, AP Southeast 1b).

Sau khi khởi chạy:
- Lấy Public IP của từng instance, truy cập bằng HTTP để kiểm tra:
  - Frankfurt: “Hello World from EU Central 1b”  
  - N. Virginia: “Hello World from US East 1a”  
  - Singapore: “Hello World from AP Southeast 1b”  
- Ghi lại IP và region/AZ vào một file text để tiện theo dõi.

2) Tạo Application Load Balancer (ALB) ở Frankfurt  
- Chọn loại: Application Load Balancer  
- Tên: DemoRoute53ALB  
- Scheme: Internet-facing, IPv4  
- Chọn 3 subnet (AZ khác nhau trong region Frankfurt)  
- Security Group: dùng SG đã tạo trước đó (cho phép HTTP)  
- Listener:
  - Protocol: HTTP, Port 80  
  - Forward đến một Target Group mới (kiểu instance)  
  - Tên Target Group: demo-tg-route53  
  - Đăng ký instance EC2 ở Frankfurt vào Target Group  

Sau khi tạo:
- ALB được provisioned (có thể mất một chút thời gian).  
- Lấy DNS name của ALB, truy cập bằng HTTP:
  - Hiển thị “Hello World from EU Central 1b” (do ALB trỏ đến instance ở Frankfurt).  

Kết quả:
- 3 EC2 instance ở 3 region khác nhau hoạt động, mỗi instance trả về “Hello World” kèm AZ.  
- ALB ở Frankfurt hoạt động, trỏ đến instance Frankfurt.  
- Tất cả đã sẵn sàng để tiếp tục sang phần Route 53 trong bài giảng tiếp theo.
