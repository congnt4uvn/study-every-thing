


Nội dung trên nói về các tùy chọn bảo mật trong Amazon API Gateway:

1. **IAM Authorization cho Method**  
   - Trong từng method của API, phần “Authorization” cho phép chọn dùng IAM.  
   - Khi bật IAM, bạn có thể dùng **IAM policies** để kiểm soát quyền truy cập vào API (ai được phép gọi API).

2. **Resource Policies (Chính sách tài nguyên)**  
   - Áp dụng ở mức **toàn bộ API**, không chỉ từng method.  
   - Có thể:
     - Cho phép các **tài khoản AWS khác**, user/role ở tài khoản khác gọi API.  
     - Thiết lập **allow/deny theo dải IP** (whitelist/blacklist IP).  
     - Giới hạn truy cập theo **VPC nguồn** (chỉ cho phép truy cập từ VPC riêng, tạo API private).  
   - Có nhiều mẫu (template) chính sách có sẵn để tham khảo.

3. **Authorizer (Người ủy quyền)**  
   Nếu không dùng IAM làm authorizer, có thể dùng:
   - **Lambda Authorizer**  
     - Loại “Lambda” cho phép tùy biến logic xác thực/ủy quyền.  
     - Cần cung cấp hàm Lambda, role, payload sự kiện, và cấu hình việc “caching”/“nesting” kết quả.  
   - **Cognito User Pool Authorizer**  
     - Dùng **Cognito User Pool** để xác thực người dùng.  
     - Chỉ định yêu cầu nào (token, header, v.v.) được dùng để xác thực, sau đó người dùng đã xác thực sẽ được truy cập API.

Kết luận: API Gateway hỗ trợ nhiều cơ chế bảo mật khác nhau (IAM, Resource Policy, Lambda Authorizer, Cognito User Pool) để kiểm soát và giới hạn quyền truy cập vào API.
