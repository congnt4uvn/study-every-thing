


Nội dung trên hướng dẫn cách quản lý **nhiều tài khoản AWS** bằng **AWS CLI profiles**:

1. Mặc định, trong thư mục `.aws` có:
   - `config`: chứa cấu hình (region, output, …) cho profile `default`
   - `credentials`: chứa `aws_access_key_id` và `aws_secret_access_key` cho profile `default`

2. Khi có **nhiều tài khoản AWS**, ta không dùng chung profile `default` mà tạo **profile mới**:
   - Chạy:  
     ```bash
     aws configure --profile <tên_profile_mới>
     ```
   - Nhập:
     - AWS Access Key ID
     - AWS Secret Access Key
     - Default region name (ví dụ: `us-west-2`)
     - Default output format (có thể Enter bỏ qua)

3. Sau khi cấu hình:
   - File `credentials` sẽ có thêm một block mới:
     ```ini
     [my-other-aws-account]
     aws_access_key_id = ...
     aws_secret_access_key = ...
     ```
   - File `config` cũng có block tương ứng:
     ```ini
     [profile my-other-aws-account]
     region = us-west-2
     ```

4. Cách sử dụng:
   - Nếu chỉ chạy:
     ```bash
     aws s3 ls
     ```
     thì CLI dùng **profile `default`**.
   - Để dùng tài khoản khác:
     ```bash
     aws s3 ls --profile my-other-aws-account
     ```

5. Kết luận:
   - Mỗi tài khoản AWS → tạo một profile riêng.
   - Dùng `--profile <tên_profile>` với mọi lệnh AWS CLI để chọn đúng tài khoản.
   - Đây không phải nội dung bắt buộc cho kỳ thi, nhưng rất quan trọng trong thực tế khi làm việc với nhiều tài khoản AWS.
