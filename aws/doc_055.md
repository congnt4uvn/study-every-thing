


Dưới đây là tóm tắt nội dung bài giảng về Amazon EFS:

1. Tạo hệ thống tệp EFS  
   - Chọn VPC (dùng VPC mặc định).  
   - Chọn loại hệ thống tệp:  
     - **Regional (khu vực)**: trải trên nhiều AZ, độ sẵn sàng & bền vững cao, dùng cho production.  
     - **One Zone**: nằm trong một AZ, rẻ hơn, phù hợp dev/test, không nên dùng cho production.  
   - Bật **tự động backup** (khuyến nghị).  

2. Quản lý vòng đời (Lifecycle Management)  
   - Tự động chuyển dữ liệu giữa các lớp lưu trữ để tiết kiệm chi phí:  
     - Sau X ngày không truy cập → chuyển sang **Infrequent Access**.  
     - Sau thêm một khoảng thời gian (vd 90 ngày) không truy cập → chuyển sang **Archive** (rẻ hơn nữa).  
     - Khi tệp được truy cập lại → chuyển về **Standard**.  

3. Mã hóa & cài đặt hiệu suất  
   - **Encryption**: bật mặc định.  
   - **Throughput mode** (chế độ thông lượng):  
     - **Bursting (bùng nổ)**: thông lượng tỷ lệ với dung lượng lưu trữ (càng nhiều dữ liệu, thông lượng càng cao).  
     - **Elastic (đàn hồi)** – khuyến nghị: tự động mở rộng I/O theo nhu cầu, trả tiền theo mức sử dụng, phù hợp workload I/O khó đoán.  
     - **Provisioned (cung cấp)**: đặt trước thông lượng (vd 100 MB/s, burst 300 MB/s), trả tiền cho thông lượng đã đặt.  
   - **Performance mode**:  
     - **General Purpose**: độ trễ thấp, cho ứng dụng nhạy cảm với latency (khuyến nghị).  
     - **Max I/O**: cho workload song song rất cao, chấp nhận latency lớn hơn (big data, phân tích).  
   - Cấu hình khuyến nghị của AWS: **Elastic throughput + General Purpose**.  

4. Cấu hình truy cập mạng cho EFS  
   - Chọn VPC, subnet cho từng AZ (vì dùng loại Regional).  
   - Tạo **Security Group** riêng cho EFS (vd sg-efs-demo).  
   - EFS sẽ có các mount target trong mỗi AZ, gắn với subnet và security group.  

5. Tạo và cấu hình EC2 gắn với EFS  
   - Tạo **Instance A** (AZ eu-west-1a, Amazon Linux 2, t2.micro, SSH từ mọi nơi).  
   - Trong phần cấu hình EC2, chọn subnet cụ thể, sau đó:  
     - Thêm **EFS file system** ngay trong console EC2.  
     - Chọn hệ thống tệp EFS đã tạo.  
     - Đặt mount point: `/mnt/efs/fs1`.  
     - Console EC2 tự động:  
       - Tạo security group EFS (vd efs-sg-1, efs-sg-2).  
       - Thiết lập rule NFS (port 2049) cho phép từ security group của EC2.  
       - Thêm user data script để tự động mount EFS.  
   - Tạo **Instance B** ở AZ khác (eu-west-1b), dùng cùng EFS và mount point tương tự.  

6. Kiểm tra EFS từ hai EC2  
   - Kết nối vào Instance A (EC2 Instance Connect):  
     - `sudo su`  
     - `echo "hello world" > /mnt/efs/fs1/hello.txt`  
     - `cat /mnt/efs/fs1/hello.txt` → hiển thị “hello world”.  
   - Kết nối vào Instance B:  
     - `ls /mnt/efs/fs1` → thấy `hello.txt`.  
     - `cat /mnt/efs/fs1/hello.txt` → cũng thấy “hello world”.  
   - Chứng minh: EFS là một **network file system** dùng chung cho nhiều EC2 ở các AZ khác nhau.  

7. Dọn dẹp tài nguyên  
   - Terminate hai EC2 instances.  
   - Xóa file system EFS (nhập ID để xác nhận).  
   - Xóa các security group được tạo cho demo (efs-sg-1, efs-sg-2, v.v.).  

Ý chính cần nhớ cho kỳ thi:  
- Phân biệt **Regional vs One Zone** EFS.  
- Các chế độ thông lượng: **Bursting, Elastic, Provisioned**.  
- Performance mode: **General Purpose vs Max I/O**.  
- EFS có thể được mount đồng thời từ nhiều EC2 ở nhiều AZ, dùng NFS trên port 2049, và trả tiền theo dung lượng sử dụng.
