


Dưới đây là phần tóm tắt nội dung:

1. Khái niệm gọi đồng bộ (synchronous invocation)  
- Gọi đồng bộ nghĩa là client (người dùng/dịch vụ) gọi hàm Lambda và chờ kết quả trả về.  
- Mọi lỗi xảy ra trong Lambda phải được xử lý ở phía client (ví dụ: tự quyết định retry, backoff theo cấp số nhân, v.v.).

2. Ví dụ về gọi đồng bộ  
- Khi dùng:  
  - AWS CLI  
  - AWS SDK  
  - API Gateway  
  - Application Load Balancer (ALB)  
  - CloudFront với Lambda@Edge  
- Luồng: Client → (API Gateway/ALB/CloudFront) → Lambda → trả kết quả về cho client.

3. Đặc điểm chính  
- Đây là “lời gọi trực tiếp” và kết quả được trả về ngay cho bên gọi.  
- Nếu Lambda thất bại, client phải quyết định: retry, xử lý lỗi, v.v.

4. Các dịch vụ tích hợp đồng bộ với Lambda  
- Dịch vụ do người dùng trực tiếp gọi:  
  - Application Load Balancer  
  - API Gateway  
  - CloudFront (Lambda@Edge)  
- Các dịch vụ khác có thể gọi Lambda đồng bộ:  
  - S3 Batch  
  - Cognito  
  - Step Functions  
  - Lex  
  - Alexa  
  - Kinesis Data Firehose  

5. Trong khóa học này  
- Sẽ tập trung vào:  
  - Application Load Balancer  
  - API Gateway  
  - CloudFront  
  - Step Functions  
- Các dịch vụ khác chỉ được nhắc đến nhưng không đi sâu.
