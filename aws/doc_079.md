


Dưới đây là tóm tắt ngắn gọn nội dung:

1. **Tạo cơ sở dữ liệu RDS MySQL**
   - Vào Aurora/RDS → Cơ sở dữ liệu → Tạo cơ sở dữ liệu.
   - Chọn chế độ cấu hình đầy đủ, chọn engine **MySQL**.
   - Giữ phiên bản engine mặc định.
   - Chọn **Free tier** (gói miễn phí), triển khai **Single-AZ**.
   - Đặt DB identifier, username (admin), password (password – ví dụ).
   - Không dùng Secrets Manager, chỉ dùng xác thực bằng mật khẩu.
   - Chọn loại instance nhỏ (vd: db.t4g.micro), 20GB storage, có thể bật auto-scaling storage.
   - VPC: dùng VPC mặc định, không gắn EC2, giữ subnet group mặc định.
   - Cho phép **Public access**, tạo security group mới (vd: demo-rds).
   - Dùng port **3306**, bật giám sát tiêu chuẩn.
   - Đặt tên DB ban đầu là **mydb**.
   - Tạo cơ sở dữ liệu.

2. **Cấu hình kết nối & bảo mật**
   - Lấy **endpoint** và port 3306 từ trang chi tiết DB.
   - Kiểm tra **Security Group**: có rule inbound cho TCP 3306 từ IP của bạn.
   - Nếu không kết nối được, kiểm tra:
     - DB có public access không.
     - Security group có cho phép IP của bạn (hoặc 0.0.0.0/0) không.

3. **Kết nối bằng SQL Electron**
   - Tải SQL Electron (GUI client) cho Windows/Mac.
   - Tạo kết nối mới:
     - Name: RDSDemo.
     - Type: MySQL.
     - Host: endpoint RDS.
     - Port: 3306.
     - User: admin.
     - Password: password.
     - Database: mydb.
   - Test kết nối → Save → Connect.

4. **Thao tác SQL cơ bản**
   - Tạo bảng:
     ```sql
     CREATE TABLE mytable (
       name VARCHAR(20),
       first_name VARCHAR(20)
     );
     ```
   - Chèn dữ liệu:
     ```sql
     INSERT INTO mytable (name, first_name)
     VALUES ('marrek', 'Stephane');
     ```
   - Chọn dữ liệu để xem kết quả (Select rows).

5. **Tính năng quản lý của RDS**
   - Có thể tạo **Read Replica** để tăng khả năng đọc.
   - Có tùy chọn **Multi-AZ** cho mục đích HA/khôi phục.
   - Theo dõi các metric như:
     - CPU Utilization.
     - Số lượng kết nối DB.
   - Hỗ trợ **backup**, khôi phục:
     - Khôi phục tới thời điểm cụ thể.
     - Khôi phục sang region khác.

6. **Xóa cơ sở dữ liệu**
   - Vào **Edit** DB → tắt **Deletion protection**.
   - Apply immediately.
   - Sau đó chọn **Delete** DB:
     - Có thể bỏ qua final snapshot.
     - Xác nhận “Delete me” → Xóa.
   - Nhắc nhở: nên xóa DB sau khi thực hành để tránh tốn chi phí.

Toàn bộ nội dung minh họa cách tạo, kết nối, thao tác cơ bản với RDS MySQL và quản lý vòng đời (giám sát, backup, xóa), một số phần (SQL chi tiết) nằm ngoài phạm vi bài thi nhưng giúp hiểu bức tranh tổng thể.
