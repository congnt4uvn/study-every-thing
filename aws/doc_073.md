


Dưới đây là tóm tắt nội dung bài giảng:

1. Chuẩn bị  
- Dừng và chấm dứt tất cả EC2 instance đang chạy.  
- Mục tiêu: thực hành tạo Auto Scaling Group (ASG) gắn với Application Load Balancer (ALB).

2. Tạo Launch Template  
- Đặt tên: ví dụ “demo template”.  
- Chọn AMI: Amazon Linux 2 (x86, free tier).  
- Chọn instance type: t2.micro (free tier).  
- Chọn key pair: ví dụ “EC2 tutorial”.  
- Chọn security group sẵn có.  
- Giữ mặc định EBS (8GB gp2).  
- Thêm user data (script) để tự động cài web server và trả về “Hello World”.  
- Tạo launch template.

3. Tạo Auto Scaling Group (ASG)  
- Đặt tên: “Demo ASG”.  
- Chọn launch template vừa tạo.  
- Chọn VPC và nhiều subnet/AZ để phân tán instance.  
- Cấu hình phân phối AZ: cân bằng giữa các AZ.  

4. Gắn với Load Balancer  
- Chọn gắn ASG với target group của ALB đã tạo trước (ví dụ “demo-tg-alb”).  
- Bật health check của EC2 và health check của Load Balancer.  
- Giữ nguyên các thiết lập capacity: desired = 1, min = 1, max = 1.  
- Không cấu hình scaling policy, notification, tag (giữ mặc định).  
- Tạo ASG.

5. Hoạt động của ASG  
- ASG tự động tạo 1 EC2 instance để đạt desired capacity.  
- Trong “Activity history” của ASG có thể thấy sự kiện “Launching a new EC2 instance”.  
- Trong tab “Instance management” của ASG và trong EC2 console sẽ thấy instance mới.  
- Instance được tự động đăng ký vào target group của ALB.  
- Ban đầu health check báo “unhealthy” vì instance đang khởi động; sau một lúc sẽ thành “healthy”.  
- Khi healthy, truy cập ALB sẽ thấy phản hồi “Hello World”.

6. Cơ chế tự phục hồi  
- Nếu instance không bao giờ trở nên healthy (do lỗi security group, user data, cấu hình sai…), ASG sẽ chấm dứt instance đó và tạo instance mới.  
- Các sự kiện này được ghi lại trong “Activity history”.

7. Thử nghiệm scale-out (mở rộng)  
- Chỉnh sửa ASG: tăng desired capacity lên 2, đồng thời tăng max capacity.  
- ASG tạo thêm 1 EC2 instance mới.  
- Sau khi instance thứ hai healthy, nó cũng được đăng ký vào target group.  
- Khi truy cập ALB nhiều lần, sẽ thấy luân phiên 2 IP (2 instance) trả về “Hello World”.

8. Thử nghiệm scale-in (thu hẹp)  
- Giảm desired capacity từ 2 xuống 1.  
- ASG chọn một trong hai instance để chấm dứt.  
- Instance bị chấm dứt sẽ bị hủy đăng ký khỏi target group.  
- Cuối cùng chỉ còn 1 EC2 instance trong ASG.

9. Kết luận  
- ASG đảm bảo số lượng instance luôn khớp với desired capacity.  
- ASG tích hợp với ALB và health check để tự động thay thế instance không khỏe.  
- Bài giảng này dừng ở việc scale thủ công (thay đổi desired capacity); bài sau sẽ nói về auto scaling (tự động thay đổi theo tải).
