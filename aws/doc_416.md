


Dưới đây là tóm tắt nội dung đoạn lecture:

1. Cài đặt AWS Encryption CLI  
   - Giảng viên hướng dẫn google “install AWS Encryption CLI”.  
   - Cần có Python, dùng `pip install aws-encryption-sdk-cli`.  
   - Kiểm tra phiên bản bằng `aws-encryption-cli --version`.  
   - Việc cài và dùng CLI này không bắt buộc cho kỳ thi, chỉ để minh họa cách mã hóa/giải mã.

2. Chuẩn bị khóa KMS (CMK ARN)  
   - Sử dụng khóa KMS đã tạo ở bài trước.  
   - Lần này phải dùng **đầy đủ ARN** của khóa, không dùng alias.  
   - Xuất ARN đó vào biến môi trường (ví dụ `KEY=...`).

3. Tạo file cần mã hóa  
   - Tạo file `hello.txt` (ví dụ dùng `vi hello.txt`).  
   - Ghi nội dung “siêu bí mật” (giả sử file > 1 MB, nhưng thực tế chỉ là text).  

4. Mã hóa file bằng AWS Encryption CLI  
   - Chạy lệnh `aws-encryption-cli encrypt` với:  
     - `--input hello.txt`  
     - `--master-keys key=$KEY` (ARN KMS)  
     - `--metadata-output` để xuất file metadata  
     - `--output` để chỉ định file đã mã hóa (trong thư mục `output/`).  
   - Lúc đầu bị lỗi vì file metadata và file output không được nằm cùng thư mục; sau đó tạo thư mục `output` và chạy lại.  
   - Kết quả:  
     - `hello.txt` (file gốc)  
     - file metadata (JSON)  
     - file output (file đã mã hóa).

5. Xem metadata  
   - Dùng `jq` để xem file metadata rõ hơn.  
   - Metadata chứa:  
     - Thuật toán mã hóa được dùng  
     - Thông tin khóa dữ liệu (data key)  
     - Nhà cung cấp khóa (KMS)  
     - Encryption context (nếu có)  
   - Metadata chỉ mang tính mô tả, không bắt buộc phải giữ; có thể xóa.

6. Kiểm tra file đã mã hóa  
   - Vào thư mục `output`, xem file đã mã hóa (ví dụ `cat output-file`).  
   - Nội dung là “vô nghĩa” (binary/encoded) vì đã được mã hóa.  
   - Quan trọng: file được mã hóa bằng KMS CMK, một data key được tạo ra để SDK có thể giải mã sau này.

7. Giải mã file  
   - Chạy lệnh `aws-encryption-cli decrypt` với:  
     - `--input` là file đã mã hóa trong thư mục `output`  
     - `--master-keys` dùng cùng ARN KMS  
     - `--metadata-output` (nếu muốn)  
     - `--output` là thư mục `decrypted/`.  
   - Lúc đầu phải tạo thư mục `decrypted`, sau đó chạy lại.  
   - Kết quả: trong thư mục `decrypted` có file `hello.txt` (đã giải mã).

8. Xác minh kết quả  
   - Mở file `decrypted/hello.txt` để kiểm tra nội dung.  
   - Nội dung “siêu bí mật hơn một megabyte” được khôi phục đúng.  

9. Kết luận  
   - Chỉ với 2 lệnh (`encrypt` và `decrypt`), có thể mã hóa/giải mã file lớn bằng AWS Encryption CLI.  
   - Quá trình này gắn trực tiếp với AWS KMS (dùng CMK và data key).  
   - Phần này mang tính minh họa thực hành, không bắt buộc cho kỳ thi, nhưng giúp hiểu rõ cách KMS và data key hoạt động với file thực tế.
