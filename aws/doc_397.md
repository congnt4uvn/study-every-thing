


Người hướng dẫn đang demo cách tạo một State Machine trong AWS Step Functions dùng Lambda:

1. Tạo State Machine  
- Chọn “Create state machine” và dùng mẫu trống.  
- Có thể kéo thả “Lambda Invoke” trong giao diện thiết kế, cấu hình: tên state, loại tích hợp, tên hàm Lambda, payload, input/output, xử lý lỗi…  
- Thay vì cấu hình thủ công, chuyển sang tab Code, tải file `state-machine.json` (từ thư mục `0-hello-world`), copy toàn bộ nội dung vào để có workflow hoàn chỉnh.

2. Giải thích workflow  
- Có một bước `Lambda Invoke` gọi hàm Lambda.  
- Sau đó là một `Choice State`:
  - Nếu kết quả (output) của Lambda chứa chuỗi “Stephane” → chuyển sang state `Is Teacher`, output là “Woohoo!” (thể hiện giáo viên là Stephane).  
  - Nếu không → chuyển sang state `Not Teacher`, trả về mã lỗi: “Stephane, teacher not found in the Lambda Function output”.

3. Tạo hàm Lambda  
- Vào Lambda, tạo hàm mới tên `HelloFunction`, runtime Node.js.  
- Thay nội dung `index.js` (hoặc `function.js`) bằng đoạn code đơn giản:
  - Nhận event có key `who`.  
  - Trả về chuỗi “Hello, <who>!” (ví dụ: Hello, Stephane!, Hello, John!, Hello, Alice!).  
- Deploy code, sau đó test:
  - Tạo event test với `{ "who": "John" }` → kết quả “Hello, John!”.  
  - Thay `who` thành “Stephane” hoặc “Alice” → kết quả tương ứng.

4. Kết nối Lambda với State Machine  
- Copy ARN của `HelloFunction`.  
- Quay lại Step Functions, trong bước `Lambda Invoke`, dán ARN vào phần tên hàm.  
- Tạo (Create) state machine.  
- AWS tự động tạo một execution role cho state machine, có quyền gọi Lambda và dùng X-Ray nếu cần.

5. Chạy thử State Machine  
- Bắt đầu một execution với input: `{ "who": "Stephane Maarek" }`.  
  - Lambda trả về “Hello, Stephane Maarek!”.  
  - Choice State kiểm tra output, thấy có “Stephane” → đi nhánh `Is Teacher` → output “Woohoo!”.  
- Chạy execution khác với `{ "who": "John Doe" }`.  
  - Lambda trả về “Hello, John Doe!”.  
  - Choice State không tìm thấy “Stephane” → đi nhánh `Not Teacher` → trả về lỗi “Stephane teacher not found in the Lambda output”.

6. Quan sát lịch sử thực thi  
- Trong Step Functions, có thể xem từng bước: input, output, thời gian thực thi, nhánh đã đi… để hiểu rõ toàn bộ luồng hoạt động của state machine.

Tóm lại: bài giảng hướng dẫn tạo một workflow Step Functions đơn giản gọi Lambda, dùng Choice State để phân nhánh dựa trên nội dung output (có/không có “Stephane”), và minh họa cách test, quan sát execution.
