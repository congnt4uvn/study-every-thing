


Nội dung trên nói về cách tạo khóa để ký URL trong Amazon CloudFront, tập trung vào hai cách:

1. **Cách mới (được khuyến nghị): Trusted Key Groups (Nhóm khóa đáng tin cậy)**  
   - CloudFront hỗ trợ “public keys” (khóa công khai) và “key groups” (nhóm khóa).  
   - Bạn tự tạo cặp khóa RSA 2048-bit (private key + public key).  
     - **Private key**: đặt trên ứng dụng của bạn (ví dụ: EC2 instances) để ký URL.  
     - **Public key**: upload lên CloudFront. CloudFront dùng nó để xác minh chữ ký URL.  
   - Quy trình:
     1. Tạo cặp khóa RSA 2048-bit (có thể dùng công cụ online, nhưng trong thực tế nên dùng công cụ an toàn như OpenSSL, KMS, v.v.).
     2. Lưu trữ private key an toàn, dùng cho ứng dụng để ký URL.
     3. Vào CloudFront Console → Public keys → tạo public key mới, dán nội dung public key.
     4. Vào Key groups → tạo key group, thêm public key vừa tạo (mỗi nhóm tối đa 5 public keys).
     5. Phân phối CloudFront sẽ tham chiếu đến key group này để chấp nhận các URL đã ký.  
   - Ưu điểm:
     - Có API để quản lý, xoay vòng khóa.
     - Có thể dùng IAM để kiểm soát quyền quản lý public keys và key groups.
     - Không cần dùng tài khoản root, có thể dùng IAM users/roles.

2. **Cách cũ (không khuyến nghị): CloudFront Key Pair gắn với tài khoản root**  
   - Trước đây, CloudFront dùng “CloudFront key pair” gắn với **tài khoản root** để ký URL.  
   - Để tạo/ quản lý key pair:
     - Đăng nhập AWS bằng **root account**.
     - Vào “My Account” → “Security Credentials” → phần “CloudFront key pairs”.
     - Tạo key pair mới hoặc upload key pair.
     - Tải private key về, public key được lưu trong tài khoản.
   - Private key này được đưa cho EC2 để ký URL; CloudFront dùng public key tương ứng để xác minh.  
   - Nhược điểm:
     - Bắt buộc dùng root account (không an toàn, không khuyến khích).
     - Không có API để tự động hóa quản lý key pair.
     - Ít linh hoạt hơn, khó xoay vòng khóa.

**Kết luận:**  
- Trong các kỳ thi và thực tế, cách **được khuyến nghị** là dùng **CloudFront public keys + Trusted Key Groups**, vì:
  - Không cần root account.
  - Có thể dùng IAM để bảo mật và phân quyền.
  - Hỗ trợ API để tự động hóa việc tạo, xoay vòng và quản lý khóa.  
- Cách cũ với CloudFront key pair gắn root account chỉ mang tính lịch sử, không nên dùng trong hệ thống mới.
