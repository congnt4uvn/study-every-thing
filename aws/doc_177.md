


Dưới đây là tóm tắt nội dung:

1. Mục tiêu  
   - Hướng dẫn cách dùng AWS CLI và Docker CLI để kéo (pull) và đẩy (push) image lên Amazon ECR (Elastic Container Registry), thay vì dùng Docker Hub.

2. Đăng nhập vào Amazon ECR  
   - Dùng AWS CLI để lấy password đăng nhập ECR.  
   - Kết quả lệnh này được truyền (pipe) vào lệnh `docker login` với username là `AWS`.  
   - Sau khi `docker login` thành công, Docker trên máy bạn có thể kết nối với ECR private repository.

3. Tạo repository trong ECR  
   - Vào Amazon ECR, tạo một private repository (ví dụ: `demostephane`).  
   - Có các tùy chọn:
     - Tag immutability: ngăn không cho đẩy lại cùng một tag.  
     - Image scanning: quét lỗ hổng bảo mật khi đẩy image (tính năng cũ, nên dùng Amazon Inspector với scan filter ở cấp registry).  
     - Encryption bằng KMS (có thể bật/tắt).  
   - Phân biệt:
     - Public repo: ai cũng có thể pull.  
     - Private repo: chỉ tài khoản có quyền IAM phù hợp mới pull/push được.

4. Chuẩn bị Docker và CLI  
   - Cần cài Docker trên máy và đảm bảo Docker daemon đang chạy.  
   - Cần cấu hình AWS CLI để chạy được lệnh login ECR.

5. Quy trình kéo, gắn tag và đẩy image  
   - Ví dụ image ban đầu: `nginxdemos/hello` trên Docker Hub.  
   - Bước 1: Pull image từ Docker Hub:
     - `docker pull nginxdemos/hello`
   - Bước 2: Tag lại image theo định dạng ECR:
     - `docker tag nginxdemos/hello <aws_account_id>.dkr.ecr.<region>.amazonaws.com/demostephane:latest`
   - Bước 3: Push image lên ECR:
     - `docker push <aws_account_id>.dkr.ecr.<region>.amazonaws.com/demostephane:latest`
   - Docker sẽ dựa vào tên registry (`*.dkr.ecr.*.amazonaws.com`) để biết phải đẩy image lên ECR.

6. Quyền truy cập (IAM)  
   - Nếu không push/pull được image, nguyên nhân thường là thiếu quyền IAM phù hợp cho ECR.

7. Kết quả  
   - Sau khi push thành công, trong repository `demostephane` sẽ thấy image mới với tag `latest`.  
   - Image này có thể được dùng trong ECS task definition, và ECS sẽ kéo image trực tiếp từ ECR thay vì từ Docker Hub.
