


Dưới đây là tóm tắt nội dung:

1. Mục tiêu  
   - Dùng AWS CDK để triển khai hạ tầng gồm:
     - S3 bucket (chứa ảnh)
     - Lambda function (gọi Amazon Rekognition)
     - DynamoDB table (lưu kết quả)
   - Toàn bộ hạ tầng được định nghĩa bằng code (CDK) và triển khai qua CloudFormation.

2. Chuẩn bị môi trường CDK  
   - Cài đặt CDK library: `sudo npm install -g aws-cdk` hoặc `aws-cdk-lib` (tùy môi trường).  
   - Kiểm tra lệnh `cdk` hoạt động.  
   - Tạo thư mục dự án: `mkdir cdk-app && cd cdk-app`.  
   - Khởi tạo app CDK: `cdk init app --language javascript`.  
   - Kiểm tra stack: `cdk ls` → thấy `CdkAppStack`.

3. Chỉnh sửa mã CDK (stack)  
   - Vào thư mục `lib`, xóa file stack cũ, tạo file mới `cdk-app-stack.js`.  
   - Dùng `nano` (cài bằng `sudo yum install nano` nếu chưa có) để dán nội dung stack mẫu.  
   - File stack:
     - Import CDK và các construct: S3, IAM, Lambda, DynamoDB, v.v.
     - Tạo S3 bucket với policy xóa (destroy khi xóa stack) và output tên bucket.
     - Tạo IAM Role cho Lambda, thêm policy bằng `addToPolicy`.
     - Tạo DynamoDB table (khóa phân vùng, policy xóa, output).
     - Tạo Lambda function:
       - Chỉ định runtime, role, biến môi trường (tên bảng, tên bucket).
       - Gắn event source từ S3 (Lambda trigger khi có object mới).
       - Cấp quyền read/write cho Lambda trên S3 và DynamoDB bằng các helper method của CDK.

4. Tạo mã Lambda  
   - Từ thư mục gốc, tạo thư mục `lambda` (hoặc tương tự) và vào trong đó.  
   - Tạo file `index.py` và dán nội dung mẫu.  
   - Chức năng Lambda:
     - Nhận event từ S3 (object mới).
     - Gọi Amazon Rekognition để detect labels trong ảnh.
     - Ghi kết quả (labels, thông tin ảnh) vào DynamoDB.

5. Khởi động (bootstrap) CDK  
   - Từ thư mục gốc (có `cdk.json`), chạy: `cdk bootstrap`.  
   - Lệnh này tạo stack “CDKToolkit” trong CloudFormation:
     - SSM parameters, IAM roles, ECR repo, S3 bucket…  
   - Bootstrap chỉ cần làm một lần cho mỗi account/mỗi region.

6. Sinh template và triển khai  
   - `cdk synth`: sinh ra CloudFormation template đầy đủ (S3, Lambda, IAM, DynamoDB, SSM…).  
   - `cdk deploy`: triển khai template lên CloudFormation.
     - Xác nhận các resource sẽ tạo → `y`.  
     - CloudFormation tạo stack `CdkAppStack` với ~11 resource (S3 bucket, DynamoDB table, Lambda, IAM roles, v.v.).

7. Kiểm thử luồng xử lý  
   - Mở S3 bucket được tạo, upload ảnh (ví dụ `penguin.jpeg`).  
   - Lambda được trigger bởi sự kiện S3, gọi Rekognition để detect labels.  
   - Lambda ghi kết quả vào DynamoDB table.  
   - Vào DynamoDB → “Explore items”:
     - Thấy item mới với key là tên file (vd: `penguin.jpeg`).
     - Các label như: penguin, animal, bird, person, man, gloves, shoes…  
   - Upload thêm ảnh khác (vd: `kid_and_pigeons`, `swan`) → kiểm tra thêm dữ liệu trong DynamoDB.

8. Dọn dẹp tài nguyên  
   - Xóa toàn bộ object trong S3 bucket (bắt buộc trước khi xóa stack).  
   - Chạy `cdk destroy` để xóa stack (S3, Lambda, DynamoDB, IAM, v.v.).  
   - Sau đó môi trường trở lại sạch.

Tóm lại: bài giảng hướng dẫn trọn quy trình dùng AWS CDK để viết code hạ tầng (S3 + Lambda + Rekognition + DynamoDB), synth ra CloudFormation, deploy, test pipeline xử lý ảnh, và cuối cùng dọn dẹp bằng `cdk destroy`.
