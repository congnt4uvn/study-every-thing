


Dưới đây là tóm tắt ngắn gọn nội dung bài giảng:

1. **Lambda Triggers với Cognito User Pool**  
   - Cognito có nhiều trigger Lambda chạy đồng bộ ở các giai đoạn:
     - Trước/ sau khi xác thực (pre-auth, post-auth, pre-token generation).  
       → Có thể chấp nhận/ từ chối đăng nhập, ghi log cho phân tích, thêm/bớt claim trong token.  
     - Trong quá trình đăng ký (pre-signup, post-confirmation, user migration).  
       → Dùng để gửi thông báo chào mừng, ghi log sự kiện đăng ký, di chuyển người dùng.  
     - Tùy chỉnh nội dung thông báo gửi cho người dùng (email, SMS).

2. **Hosted UI (Giao diện đăng nhập được lưu trữ)**  
   - Cognito cung cấp sẵn giao diện đăng ký/ đăng nhập, không cần tự code UI.  
   - Hỗ trợ tích hợp sẵn các nhà cung cấp đăng nhập: Social, OIDC, SAML.  
   - Có thể tùy chỉnh CSS, logo để phù hợp với giao diện website/app.  
   - Có thể dùng **custom domain**:
     - Bắt buộc phải có chứng chỉ HTTPS trong **ACM**.  
     - Chứng chỉ phải được tạo ở **us-east-1**, dù User Pool ở region khác.  
     - Domain được cấu hình trong phần App integration của User Pool.

3. **Adaptive Authentication (Xác thực thích ứng)**  
   - Mục tiêu: phát hiện đăng nhập đáng ngờ và tăng cường bảo mật.  
   - Mỗi lần đăng nhập được đánh **điểm rủi ro**: thấp, trung bình, cao.  
   - Dựa trên: thiết bị, vị trí, IP, hành vi bất thường,…  
   - Nếu rủi ro cao:
     - Có thể chặn đăng nhập hoặc yêu cầu MFA (đa yếu tố).  
   - Có cơ chế bảo vệ takeover tài khoản, xác minh qua điện thoại/email.  
   - Hoạt động của adaptive auth được ghi lại trong **CloudWatch Logs** (số lần đăng nhập, điểm rủi ro, thử thách thất bại,…).

4. **JWT Tokens từ Cognito**  
   - Khi đăng nhập thành công, Cognito trả về **JWT (JSON Web Token)**:  
     - Gồm 3 phần: header, payload, signature (được mã hóa Base64).  
   - Để tin tưởng nội dung trong payload, cần **xác minh chữ ký** bằng thư viện JWT.  
   - Payload chứa thông tin người dùng:
     - `sub` (UUID) – ID người dùng trong User Pool.  
     - username, nhóm (groups), thời gian hết hạn token, v.v.  
   - Nếu cần thêm thông tin (email, tên, số điện thoại, thuộc tính tùy chỉnh), có thể dùng `sub` để truy vấn lại người dùng trong **Cognito User Pool**.

5. **Kết luận**  
   - Bài giảng tổng kết các tính năng chính của Cognito User Pool:
     - Lambda triggers  
     - Hosted UI & custom domain  
     - Adaptive authentication  
     - JWT tokens và cách sử dụng payload + truy vấn thêm dữ liệu người dùng.
