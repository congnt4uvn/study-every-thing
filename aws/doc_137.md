


Dưới đây là tóm tắt nội dung bài giảng:

1. Hiệu suất cơ bản (baseline) của S3  
   - S3 tự động scale để xử lý số lượng request rất lớn.  
   - Thời gian lấy byte đầu tiên (first byte) từ S3: khoảng 100–200 ms.  
   - Giới hạn hiệu suất theo *prefix* trong một bucket:  
     - 3.500 yêu cầu PUT/COPY/POST/DELETE mỗi giây mỗi prefix.  
     - 5.500 yêu cầu GET/HEAD mỗi giây mỗi prefix.  
   - Không giới hạn số lượng prefix trong một bucket → có thể tăng tổng throughput bằng cách phân tán object theo nhiều prefix.

2. Khái niệm prefix trong S3  
   - Prefix là phần “đường dẫn” giữa tên bucket và tên file.  
   - Ví dụ: `bucket/folder1/sub1/file`  
     - Prefix: `folder1/sub1/`  
   - Mỗi prefix có quota riêng (3.500 write + 5.500 read req/s).  
   - Nếu trải đều request lên nhiều prefix, tổng số request/s có thể tăng rất lớn (ví dụ 4 prefix → ~22.000 GET/HEAD req/s).

3. Tối ưu hiệu suất upload – Multipart Upload  
   - Nên dùng multipart upload cho file > 100 MB, và *bắt buộc* cho file > 5 GB.  
   - Cách hoạt động:  
     - Chia file lớn thành nhiều phần nhỏ.  
     - Upload các phần này song song lên S3.  
     - S3 ghép các phần lại thành một file hoàn chỉnh.  
   - Lợi ích: tận dụng băng thông tốt hơn, tăng tốc độ upload.

4. S3 Transfer Acceleration  
   - Dùng cho cả upload và download.  
   - Cách hoạt động:  
     - Client gửi dữ liệu đến một AWS Edge Location gần nhất (rất nhiều edge location trên thế giới).  
     - Từ Edge Location, dữ liệu được truyền qua mạng riêng AWS đến bucket S3 ở region đích.  
   - Mục tiêu: giảm phần đường đi trên internet công cộng, tăng phần đi trên mạng riêng AWS → tăng tốc độ truyền.  
   - Tương thích với multipart upload.

5. Tối ưu hiệu suất download – S3 Byte-Range Fetches  
   - Cho phép GET theo “phạm vi byte” (byte range) của file thay vì tải toàn bộ file.  
   - Có thể:  
     - Chia file thành nhiều đoạn (byte range) và tải song song → tăng tốc độ download.  
     - Retry từng range nhỏ nếu lỗi → tăng khả năng phục hồi.  
   - Use case:  
     - Tăng tốc download file lớn bằng cách song song hóa các GET.  
     - Chỉ lấy một phần file, ví dụ:  
       - 50 byte đầu tiên là header chứa metadata → chỉ cần GET byte range đầu tiên để đọc thông tin nhanh.

6. Kết luận  
   - Đã trình bày:  
     - Hiệu suất baseline của S3 và giới hạn theo prefix.  
     - Multipart upload để tăng tốc upload.  
     - S3 Transfer Acceleration để tăng tốc truyền qua mạng AWS.  
     - Byte-range fetch để tối ưu download.  
   - Các nội dung này quan trọng cho kỳ thi AWS Certified Developer – Associate.
