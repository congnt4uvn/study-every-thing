


Nội dung trên nói về các giới hạn (limits) quan trọng của AWS Lambda mà đề thi hay hỏi:

1. **Giới hạn thực thi (Execution limits)**  
   - RAM: từ **128 MB đến 10 GB**, tăng theo bước **64 MB**.  
   - Khi tăng RAM thì số vCPU cũng tăng theo.  
   - **Thời gian chạy tối đa**: **900 giây (15 phút)**.  
   - **Biến môi trường**: tối đa **4 KB**.  
   - **Dung lượng lưu tạm (/tmp)**: tối đa **10 GB** để lưu file tạm trong lúc hàm chạy.  
   - **Số lần thực thi đồng thời**: mặc định tối đa **1000**; có thể tăng nếu gửi yêu cầu, nên dùng “reserved concurrency” sớm.

2. **Giới hạn triển khai (Deployment limits)**  
   - Kích thước file zip **đã nén**: tối đa **50 MB**.  
   - Kích thước code **không nén**: tối đa **250 MB**.  
   - Nếu cần xử lý file lớn hơn (ví dụ vài GB), nên dùng thư mục **/tmp** hoặc giải pháp khác, không nhét hết vào package.

3. **Ý nghĩa cho kỳ thi**  
   - Nếu đề bài yêu cầu:  
     - 30 GB RAM,  
     - 30 phút thời gian chạy,  
     - file 3 GB,  
     thì phải nhận ra **Lambda không phù hợp** cho khối lượng công việc đó.
