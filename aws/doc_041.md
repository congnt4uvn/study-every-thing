


Dưới đây là tóm tắt nội dung:

1. **Lỗi timeout khi SSH**  
   - Hầu hết do **Security Group hoặc firewall**.  
   - Cần mở đúng port (thường là 22) và gán đúng Security Group cho EC2.

2. **Vẫn timeout dù Security Group đúng**  
   - Nhiều khả năng bị chặn bởi **firewall công ty hoặc firewall cá nhân**.  
   - Giải pháp: dùng **EC2 Instance Connect**.

3. **SSH không hoạt động trên Windows**  
   - Nếu báo `ssh: command not found` → Windows chưa có SSH client.  
   - Cần dùng **PuTTY** theo hướng dẫn. Nếu vẫn lỗi, dùng **EC2 Instance Connect**.

4. **Lỗi “connection refused”**  
   - EC2 truy cập được nhưng **dịch vụ SSH không chạy**.  
   - Thử **restart** instance. Nếu vẫn lỗi, **terminate** và tạo instance mới (Amazon Linux 2).

5. **Lỗi “Permission denied (publickey,gssapi-keyex,gssapi-with-mic)”**  
   - Có thể do:  
     - Dùng **sai key** hoặc không dùng key. Kiểm tra key pair gán cho EC2.  
     - Dùng **sai user**. Với Amazon Linux 2 phải dùng user `ec2-user` (ví dụ: `ec2-user@<public-ip>`).

6. **Không cách nào vào được**  
   - Đừng hoảng, hãy dùng **EC2 Instance Connect**.  
   - Đảm bảo đang dùng **Amazon Linux 2**.

7. **Hôm qua SSH được, hôm nay không**  
   - Có thể do **dừng rồi khởi động lại EC2**, làm **thay đổi public IP**.  
   - Cập nhật lại IP mới trong lệnh SSH hoặc cấu hình PuTTY.

Kết luận: kiểm tra lần lượt Security Group, firewall, key pair, user, trạng thái dịch vụ SSH, và IP; nếu vẫn không được, dùng EC2 Instance Connect.
