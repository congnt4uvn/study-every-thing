


Nội dung trên giải thích về việc **vô hiệu hóa (invalidation) bộ đệm trong CloudFront**:

- CloudFront lưu nội dung từ origin (ví dụ S3) tại các edge location với một **TTL** (thời gian sống của cache).  
- Khi bạn cập nhật file ở origin, các edge location **không tự biết ngay**; chúng chỉ lấy phiên bản mới sau khi TTL hết hạn.  
- Nếu bạn muốn người dùng thấy nội dung mới **ngay lập tức**, bạn có thể **vô hiệu hóa (invalidate) cache**:
  - Chỉ định các đường dẫn cần vô hiệu hóa, ví dụ:
    - `/index.html` để xóa cache một file cụ thể.
    - `/images/*` để xóa cache toàn bộ các file trong thư mục images.
  - Có thể dùng ký tự `*` để vô hiệu hóa nhiều file cùng lúc.

Cách hoạt động:
- Mỗi edge location có cache riêng (ví dụ chứa `index.html` và các hình ảnh) với TTL 1 ngày.
- Bạn cập nhật `index.html` và các hình ảnh trong S3.
- Bạn tạo một invalidation cho `/index.html` và `/images/*`.
- CloudFront yêu cầu các edge location **xóa các file này khỏi cache**.
- Lần truy cập tiếp theo:
  - Edge location thấy file không còn trong cache.
  - Nó gửi yêu cầu về origin (S3) để lấy phiên bản mới.
- Nhờ vậy, người dùng nhận được nội dung cập nhật **ngay**, không phải chờ TTL hết hạn.

Ý chính: **Invalidation cho phép bạn bỏ qua TTL và buộc CloudFront lấy lại nội dung mới từ origin sớm hơn.**
