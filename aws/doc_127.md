


Nội dung trên là một bài giảng thực hành về **EC2 Instance Metadata Service (IMDS)**, đặc biệt là **IMDSv2** trên Amazon Linux 2023.

Tóm tắt ý chính:

1. **Tạo EC2 instance**
   - Chọn AMI: Amazon Linux 2023 (AMI mới).
   - Trong phần “Advanced details” có mục **Metadata version**:
     - Có thể chọn: “V1 and V2” hoặc “V2 only”.
   - Với Amazon Linux 2023, mặc định phải dùng **IMDSv2**.
   - Tạo security group cho phép SSH từ mọi nơi.
   - Lúc đầu **không gắn IAM role** cho instance.

2. **Kết nối vào EC2 và thử IMDSv1**
   - Dùng **EC2 Instance Connect** để vào máy.
   - Mở tài liệu IMDS trên AWS để lấy lệnh mẫu.
   - Thử gọi IMDSv1 bằng `curl` tới URL metadata (169.254.169.254/...).
   - Kết quả: **HTTP 401 Unauthorized** → IMDSv1 không hoạt động trên Amazon Linux 2023 (khi chỉ bật V2).

3. **Sử dụng IMDSv2**
   - IMDSv2 yêu cầu **token**:
     1. Gửi request để lấy token:
        - `curl` tới endpoint `/latest/api/token` với header `X-aws-ec2-metadata-token-ttl-seconds` (ví dụ 21600).
        - Lưu token vào biến `TOKEN`.
     2. Khi gọi metadata, phải thêm header:
        - `X-aws-ec2-metadata-token: $TOKEN`.
   - Sau khi có token, gọi lại URL metadata:
     - Lần này **thành công**, không còn 401.
   - Giải thích:
     - Đường dẫn kết thúc bằng dấu `/` giống như “thư mục” (còn nhiều dữ liệu bên trong).
     - Không có `/` là một “giá trị” cụ thể.

4. **Ví dụ truy vấn metadata**
   - Lấy hostname:
     - Thêm `hostname` vào cuối URL → trả về hostname của instance.
   - Lấy địa chỉ IPv4 cục bộ:
     - Thêm `local-ipv4` → trả về IP nội bộ.
   - Có rất nhiều trường khác có thể thử.

5. **Lấy IAM credentials qua metadata**
   - Ban đầu instance **không có IAM role**, nên:
     - Truy vấn đường dẫn `identity/credentials/` → không có “security-credentials”.
   - Gắn IAM role cho instance:
     - Trong console: chọn instance → tab Security → Actions → Modify IAM role → chọn một role bất kỳ.
     - Đợi khoảng 30 giây.
   - Truy vấn lại:
     - Đảm bảo đường dẫn có dấu `/` cuối.
     - Vào `identity/credentials/ec2/security-credentials/` rồi đến tên role.
     - Nhận được JSON chứa:
       - `AccessKeyId`
       - `SecretAccessKey`
       - `Token` (session token)
       - `Expiration` (thời gian hết hạn)
   - Đây chính là cách **EC2 instance lấy temporary credentials** để sử dụng IAM role của nó.

6. **Ý nghĩa**
   - AWS CLI/SDK trên EC2 tự động:
     - Gọi IMDSv2.
     - Lấy token, lấy credentials.
   - Lập trình viên thường **không cần tự gọi** metadata service, nhưng hiểu cơ chế giúp nắm rõ cách EC2 nhận được quyền IAM.

7. **Kết thúc**
   - Sau khi thực hành xong, có thể **terminate** instance.
   - Bài giảng kết thúc tại đây.
