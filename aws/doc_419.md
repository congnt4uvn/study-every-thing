


Nội dung trên nói về một tính năng mới của Amazon S3 khi dùng mã hóa SSE-KMS:

1. **Vấn đề chi phí với SSE-KMS**  
   - Khi dùng SSE-KMS, mỗi lần S3 mã hóa/giải mã đối tượng sẽ gọi API tới KMS.  
   - Nhiều đối tượng → rất nhiều lệnh gọi KMS → chi phí KMS cao và có thể chạm giới hạn.

2. **Giải pháp: S3 Bucket Key**  
   - AWS giới thiệu **S3 Bucket Key** để giảm ~99% số lượng lệnh gọi API tới KMS.  
   - Cách hoạt động:
     - Khóa KMS (Customer Managed Key) chỉ thỉnh thoảng được dùng để tạo **data key** cho bucket.
     - Từ data key này, S3 tạo ra **S3 Bucket Key**.
     - S3 Bucket Key được dùng để mã hóa các đối tượng trong bucket (dùng cơ chế mã hóa phong bì – envelope encryption).
     - Nhờ vậy, không cần gọi KMS cho từng đối tượng, mà chỉ thỉnh thoảng khi xoay vòng khóa.

3. **Lợi ích**  
   - Giảm ~99% số lệnh gọi API tới KMS → giảm mạnh chi phí KMS.  
   - Giảm số sự kiện KMS xuất hiện trong CloudTrail.  
   - Vẫn giữ nguyên mức độ bảo mật (vì vẫn dùng KMS và envelope encryption).

4. **Cách bật trong Console S3**  
   - Khi tạo bucket mới:
     - Bật mã hóa mặc định: chọn **SSE-KMS** với **AWS managed key** hoặc CMK.  
     - Có tùy chọn **S3 Bucket Key**:
       - Mặc định hiện đang **bật**.
       - Có thể tắt nếu muốn mỗi đối tượng đều gọi trực tiếp KMS.
       - Nếu bật, S3 sẽ dùng Bucket Key để giảm chi phí và số cuộc gọi KMS.
   - Sau khi tạo bucket, các đối tượng tải lên sẽ được mã hóa bằng SSE-KMS với S3 Bucket Key.

Tóm lại: S3 Bucket Key là một cài đặt tối ưu hóa SSE-KMS, giúp giảm mạnh chi phí và số lệnh gọi KMS mà không giảm mức độ bảo mật.
