


Dưới đây là tóm tắt nội dung:

1. **Khái niệm AWS RDS & SQL**  
   - RDS (Relational Database Service) là dịch vụ cơ sở dữ liệu quan hệ được quản lý trên AWS.  
   - SQL là ngôn ngữ truy vấn có cấu trúc, tối ưu, chạy trên nhiều nền tảng.  
   - Với RDS, bạn tạo DB trên đám mây và AWS quản lý hầu hết phần vận hành.

2. **Các hệ quản trị CSDL mà RDS hỗ trợ**  
   - PostgreSQL  
   - MySQL  
   - MariaDB  
   - Oracle  
   - Microsoft SQL Server  
   - IBM DB2  
   - Amazon Aurora (CSDL độc quyền của AWS)

3. **Vì sao dùng RDS thay vì tự cài DB trên EC2?**  
   - RDS là dịch vụ **được quản lý**:  
     - Tự động triển khai DB.  
     - Tự động vá lỗi hệ điều hành nền.  
     - Tự động sao lưu, hỗ trợ **Point-in-Time Restore** (khôi phục về thời điểm cụ thể).  
     - Có dashboard giám sát hiệu năng.  
   - Hỗ trợ **read replicas** để tăng hiệu suất đọc.  
   - Hỗ trợ **multi-AZ** để tăng khả năng chịu lỗi và khôi phục sau thảm họa.  
   - Có **maintenance window** cho nâng cấp.  
   - Hỗ trợ **scaling**:  
     - Vertical scaling: nâng cấp loại instance.  
     - Horizontal scaling: thêm read replicas.  
   - Lưu trữ dựa trên EBS.  
   - Không thể SSH vào máy chủ RDS (vì là dịch vụ managed), nhưng đổi lại AWS lo toàn bộ hạ tầng.

4. **RDS Storage Auto Scaling (Tự động mở rộng lưu trữ)**  
   - Khi tạo DB, bạn chọn dung lượng (ví dụ 20 GB).  
   - Nếu bật Storage Auto Scaling, khi sắp hết dung lượng, RDS tự động tăng dung lượng lưu trữ mà không cần dừng DB.  
   - Cần đặt **ngưỡng dung lượng tối đa** để giới hạn mức tăng.  
   - Điều kiện tự động mở rộng:  
     - Dung lượng trống < 10% dung lượng đã cấp.  
     - Tình trạng thiếu dung lượng kéo dài > 5 phút.  
     - Ít nhất 6 giờ kể từ lần mở rộng gần nhất.  
   - Rất hữu ích cho workload khó dự đoán, hỗ trợ tất cả các engine RDS.

Tóm lại, RDS cung cấp một nền tảng cơ sở dữ liệu quan hệ được quản lý, giúp giảm công việc vận hành (cài đặt, vá lỗi, sao lưu, mở rộng, giám sát) và có thêm tính năng tự động mở rộng lưu trữ để tránh hết dung lượng đột ngột.
