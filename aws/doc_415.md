


Dưới đây là tóm tắt ngắn gọn nội dung:

1. **Giới hạn cơ bản của KMS**
   - API Encrypt/Decrypt của KMS chỉ mã hóa/giải mã trực tiếp dữ liệu tối đa **4 KB**.
   - Quy trình:
     - Gọi API Encrypt, chỉ định CMK, KMS kiểm tra IAM → nếu OK thì trả về dữ liệu đã mã hóa.
     - Gọi API Decrypt, KMS tự nhận ra CMK đã dùng, kiểm tra IAM → nếu OK thì trả về dữ liệu dạng plaintext.

2. **Vấn đề kích thước & nhu cầu mã hóa phong bì (Envelope Encryption)**
   - Khi dữ liệu > 4 KB (ví dụ file 10 MB), phải dùng **Envelope Encryption**.
   - API quan trọng: **GenerateDataKey**.

3. **Cách hoạt động của Envelope Encryption**
   - Ứng dụng gọi **GenerateDataKey** với CMK:
     - KMS kiểm tra IAM.
     - Nếu OK, KMS tạo **Data Encryption Key (DEK)** và trả về:
       - DEK dạng plaintext.
       - DEK dạng đã mã hóa bằng CMK.
   - Ứng dụng dùng **DEK plaintext** để:
     - Tự mã hóa file lớn ở phía client → tạo ra file đã mã hóa.
   - “Phong bì” cuối cùng chứa:
     - File đã mã hóa.
     - DEK đã mã hóa (do KMS trả về).
   - Gọi là “envelope” vì DEK đã mã hóa bao bọc quanh dữ liệu đã mã hóa.

4. **Giải mã với Envelope Encryption**
   - Từ file phong bì, lấy:
     - File đã mã hóa.
     - DEK đã mã hóa.
   - Gọi API **Decrypt** của KMS với DEK đã mã hóa (≤ 4 KB):
     - KMS kiểm tra IAM, nếu OK → trả về DEK plaintext.
   - Ứng dụng dùng DEK plaintext để giải mã file lớn ở phía client.

5. **Mục đích của Envelope Encryption**
   - KMS chỉ chuyên về:
     - Tạo khóa (DEK).
     - Mã hóa/giải mã DEK.
   - Việc mã hóa/giải mã dữ liệu lớn diễn ra ở phía client → giảm tải cho KMS.

6. **AWS Encryption SDK & Data Key Caching**
   - AWS cung cấp **AWS Encryption SDK** (CLI + SDK cho Java, Python, C, JavaScript) để tự động hóa envelope encryption.
   - Có tính năng **data key caching**:
     - Tái sử dụng DEK để giảm số lần gọi KMS → giảm chi phí.
     - Đánh đổi: giảm bảo mật vì cùng DEK dùng cho nhiều file.
   - Dùng **LocalCryptoMaterialsCache** để:
     - Quy định kích thước cache.
     - Tuổi tối đa của DEK.
     - Số byte tối đa hoặc số message tối đa mà một DEK được phép mã hóa.

7. **Các API quan trọng cần nhớ cho kỳ thi**
   - **Encrypt**: mã hóa trực tiếp tối đa 4 KB dữ liệu.
   - **GenerateDataKey**:
     - Tạo một DEK đối xứng.
     - Trả về:
       - DEK plaintext.
       - DEK đã mã hóa bằng CMK.
     - Dùng cho **envelope encryption** (trong đề thi hay hỏi).
   - **GenerateDataKeyWithoutPlaintext**:
     - Tạo DEK nhưng **không** trả về plaintext, chỉ trả về DEK đã mã hóa.
     - Dùng khi muốn lưu DEK để dùng sau; khi cần dùng phải gọi Decrypt thêm một bước.
     - Trong đề thi, nếu yêu cầu mã hóa phong bì “ngay bây giờ”, phải chọn **GenerateDataKey**, không phải GenerateDataKeyWithoutPlaintext.
   - **Decrypt**:
     - Giải mã tối đa 4 KB dữ liệu.
     - Dùng để:
       - Giải mã dữ liệu nhỏ.
       - Giải mã DEK trong envelope encryption.
   - **GenerateRandom**:
     - Tạo chuỗi byte ngẫu nhiên.

Tóm lại:  
- Dữ liệu ≤ 4 KB → dùng trực tiếp Encrypt/Decrypt.  
- Dữ liệu > 4 KB → dùng **Envelope Encryption** với **GenerateDataKey + Decrypt**.  
- Nhớ phân biệt **GenerateDataKey** và **GenerateDataKeyWithoutPlaintext** trong bối cảnh đề thi.
