


Dưới đây là tóm tắt ngắn gọn nội dung trên (quy trình demo CodeDeploy với EC2):

1. Tạo IAM Role  
   - Role 1: Cho dịch vụ CodeDeploy (CodeDeployServiceRoleForEC2)  
     - Chọn dịch vụ AWS → CodeDeploy → loại CodeDeploy EC2/on-premises.  
   - Role 2: Cho EC2 instance (EC2RoleForCodeDeploy)  
     - Chọn EC2 → gán policy AmazonS3ReadOnlyAccess để EC2 đọc code từ S3.

2. Tạo ứng dụng trong CodeDeploy  
   - Vào CodeDeploy → tạo Application tên DemoApplication.  
   - Chọn nền tảng: EC2/on-premises.

3. Chuẩn bị EC2 instance  
   - Vào EC2 → Launch instance tên DemoWebServer.  
   - Chọn Amazon Linux, t2.micro, mở port 22 (SSH) và 80 (HTTP).  
   - Không cần user data.  
   - Sau khi chạy, SSH vào instance.

4. Cài đặt CodeDeploy Agent trên EC2  
   - Cập nhật hệ điều hành (yum update).  
   - Cài ruby (yêu cầu cho agent).  
   - Tải script cài đặt agent từ AWS (theo region).  
   - chmod để script có thể thực thi, rồi chạy với sudo.  
   - Kiểm tra: `sudo service codedeploy-agent status` → agent đang chạy.

5. Tạo Deployment Group  
   - Trong CodeDeploy → tạo Deployment Group (ví dụ: DevelopmentGroup).  
   - Gán Service Role đã tạo cho CodeDeploy.  
   - Chọn loại triển khai: In-place.  
   - Chọn môi trường: Amazon EC2 (không dùng Auto Scaling, không on-premises).  
   - Dùng tag để chọn instance:
     - Vào EC2 → gán tag cho instance: Key = Environment, Value = Development.  
     - Trong Deployment Group, chọn tag Environment = Development → 1 instance được match.  
   - Chọn cấu hình triển khai (ví dụ CodeDeployDefault.AllAtOnce).  
   - Không bật tích hợp load balancer.

6. Chuẩn bị S3 và ứng dụng  
   - Vào S3 → tạo bucket cùng region với CodeDeploy (ví dụ eu-west-1).  
   - Upload file zip ứng dụng: SampleApp_Linux.zip.  
   - Nội dung quan trọng:
     - index.html: trang web đơn giản hiển thị thông báo “Ứng dụng này được triển khai bằng AWS CodeDeploy”.  
     - appspec.yml:  
       - Định nghĩa file xử lý: copy index.html vào `/var/www/html`.  
       - Định nghĩa các hook:
         - BeforeInstall: cài đặt dependency (httpd).  
         - ApplicationStart: khởi động httpd.  
         - ApplicationStop: dừng httpd trước khi cập nhật.  
     - Các script tương ứng rất đơn giản (install_dependencies, start_server, stop_server).

7. Tạo Deployment  
   - Trong CodeDeploy → Create deployment.  
   - Chọn Application: DemoApplication, Deployment Group: DevelopmentGroup.  
   - Chọn nguồn: S3 → nhập S3 URI của SampleApp_Linux.zip.  
   - Tạo deployment.

8. Lỗi và khắc phục  
   - Deployment lần đầu thất bại vì EC2 chưa gán IAM Role (EC2RoleForCodeDeploy).  
   - Vào EC2 → Actions → Security → Modify IAM role → gán EC2RoleForCodeDeploy.  
   - Re-deploy → lần này thành công vì EC2 có quyền đọc S3.

9. Kiểm tra kết quả  
   - Lấy Public IP của EC2 → truy cập `http://<public-ip>`.  
   - Thấy trang “Congratulations, this application was deployed using AWS CodeDeploy” → triển khai thành công.

10. Tổng kết  
   - Đã thực hiện: tạo IAM roles, ứng dụng CodeDeploy, deployment group, EC2 instance, cài agent, upload app lên S3, tạo deployment và kiểm tra.  
   - Có thể:
     - Cập nhật ứng dụng, upload zip mới lên S3, rồi redeploy.  
     - Tùy chỉnh Deployment Config (tỷ lệ server healthy, chiến lược triển khai).  
     - Đăng ký on-premises instances để CodeDeploy triển khai cả on-prem.  
   - Nhắc: sau demo nên terminate EC2 để tránh tốn chi phí.
