


Nội dung trên nói về cách triển khai và quản lý API trong Amazon API Gateway, tập trung vào khái niệm **stage** (giai đoạn) và **stage variables** (biến giai đoạn):

1. **Triển khai (deployment) là bắt buộc**  
   - Mọi thay đổi trong API Gateway chỉ có hiệu lực sau khi **deploy**.  
   - Nhiều người chỉnh sửa API nhưng quên deploy nên API không thay đổi như mong đợi.

2. **Stage (giai đoạn) trong API Gateway**  
   - Có thể tạo nhiều stage tùy ý, đặt tên tùy ý: ví dụ `dev`, `test`, `prod` hoặc `v1`, `v2`, `v3`.  
   - Mỗi stage có cấu hình riêng và có thể **rollback** vì API Gateway lưu lịch sử các lần deploy cho từng stage.

3. **Ví dụ v1 – v2 (tương thích ngược)**  
   - Stage `v1` trỏ đến Lambda function `v1`, khách hàng v1 dùng URL dạng: `api.example.com/v1`.  
   - Khi tạo Lambda `v2` với thay đổi phá vỡ (breaking changes), nếu deploy đè lên `v1` sẽ làm hỏng client cũ.  
   - Giải pháp: tạo stage mới `v2` trỏ đến Lambda `v2`, tạo URL mới `api.example.com/v2`.  
   - Trong một thời gian, `v1` và `v2` cùng tồn tại; dần dần chuyển client từ `v1` sang `v2`, rồi tắt `v1`.

4. **Stage variables (biến giai đoạn)**  
   - Giống như **environment variables**, nhưng dành cho **stage của API Gateway**.  
   - Dùng để thay đổi cấu hình **mà không cần redeploy API**, ví dụ:
     - ARN của Lambda
     - Endpoint HTTP backend
     - Tham số trong mapping template  
   - Truy cập trong API Gateway bằng cú pháp:  
     `"$stageVariables.tên_biến"`  
   - Các biến này được truyền vào **context** của Lambda, nên Lambda có thể đọc và log chúng.

5. **Liên kết stage variables với Lambda alias**  
   - Tạo biến giai đoạn để chỉ ra **Lambda alias** mà API Gateway sẽ gọi.  
   - Mỗi stage trỏ đến một alias khác nhau:
     - Stage `dev` → alias `dev` → 100% traffic đến phiên bản mới nhất.  
     - Stage `test` → alias `test` → trỏ đến phiên bản `v2`.  
     - Stage `prod` → alias `prod` → 95% traffic đến `v1`, 5% đến `v2` (canary/gradual rollout).  
   - Khi muốn thay đổi phân bổ traffic (tỷ lệ v1/v2), chỉ cần chỉnh alias Lambda, **không cần sửa API Gateway**.  
   - Mẫu này rất phổ biến để triển khai dần dần và quản lý version trong API Gateway.
