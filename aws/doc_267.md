


Nội dung trên nói về cách thực hiện **lời gọi đồng bộ (synchronous invocation)** với AWS Lambda:

1. **Qua giao diện người dùng (UI)**  
   - Khi chạy hàm Lambda từ console và chờ kết quả hiển thị ngay, đó là lời gọi đồng bộ.  
   - Dù hàm mất 2 phút, người dùng vẫn chờ đến khi có kết quả trong cửa sổ kết quả.

2. **Qua AWS CLI / CloudShell**  
   - Mở AWS CloudShell (hoặc dùng terminal với AWS CLI).  
   - Kiểm tra phiên bản CLI (sử dụng AWS CLI v2).  
   - Dùng lệnh `aws lambda list-functions` (kèm `--region`) để liệt kê các hàm Lambda trong vùng, xem tên hàm, ARN, runtime…

3. **Gọi hàm Lambda đồng bộ bằng CLI**  
   - Sử dụng script `sync.sh` (dành cho Linux/Mac) với lệnh `aws lambda invoke` để gọi hàm.  
   - Truyền:
     - `--function-name` (ví dụ: `lambda-demo`, không phải `hello-world` như nhầm ban đầu)  
     - `--payload` (dữ liệu gửi vào)  
     - file output (ví dụ: `response.json`) để ghi kết quả.  
   - Nếu dùng CloudShell, có thể bỏ `--region` nếu đã cấu hình sẵn; nếu không, phải chỉ rõ vùng (ví dụ `eu-west-1`).  
   - Sau khi sửa đúng tên hàm, lệnh chạy thành công, trả về mã trạng thái 200.  
   - Mở file `response.json` thấy giá trị trả về (ví dụ `{"value": 1}`), chứng tỏ lời gọi đồng bộ hoạt động.

4. **Kết luận**  
   - Bài giảng minh họa cách gọi Lambda đồng bộ cả từ UI và CLI, cách xử lý lỗi tên hàm, và xác nhận kết quả trả về.
