


Dưới đây là tóm tắt nội dung:

1. **Thắt chặt bảo mật với Security Group và ALB**
   - Ban đầu:
     - ALB (Application Load Balancer) có một Security Group riêng.
     - EC2 instances có Security Group riêng, cho phép HTTP từ “Anywhere” (0.0.0.0/0).
     - Có thể truy cập EC2 trực tiếp qua Public IP hoặc thông qua ALB.
   - Mục tiêu:
     - Chỉ cho phép truy cập EC2 thông qua ALB, không cho truy cập trực tiếp từ Internet.
   - Cách làm:
     1. Vào Security Group của EC2 (ví dụ: `launch-wizard-1`).
     2. Chỉnh sửa Inbound rules:
        - Xóa rule HTTP hiện tại đang cho phép từ mọi nơi (0.0.0.0/0).
        - Thêm rule HTTP mới:
          - Thay vì CIDR, chọn “Security group”.
          - Chọn Security Group của ALB.
     3. Lưu lại.
   - Kết quả:
     - Truy cập trực tiếp vào Public IP của EC2 sẽ bị timeout (bị chặn).
     - Truy cập qua ALB vẫn hoạt động, vì Security Group của EC2 cho phép traffic đến từ Security Group của ALB.
     - Như vậy, bảo mật mạng được siết chặt: chỉ ALB mới truy cập được EC2.

2. **Thiết lập Listener Rules trên Application Load Balancer**
   - Vào ALB → phần Listeners → chọn listener (ví dụ: HTTP:80).
   - Hiện tại:
     - Có một default rule: mọi request đều được forward đến target group “demo for my ALB”.
   - Thêm rule mới (DemoRule):
     1. **Điều kiện (Condition)**:
        - Chọn kiểu điều kiện là “Path”.
        - Đặt pattern là `/error`.
        - Nghĩa là: nếu đường dẫn request là `/error`, rule này sẽ áp dụng.
        - Ngoài Path, ALB còn hỗ trợ điều kiện theo:
          - Host header (tên miền),
          - HTTP method (GET/POST),
          - Source IP,
          - Query string,
          - HTTP headers.
     2. **Hành động (Action)**:
        - Thay vì forward đến target group, chọn “Fixed response”.
        - Cấu hình:
          - Status code: 404.
          - Body: “Not found, custom error” (hoặc nội dung tùy chỉnh).
     3. **Priority (Độ ưu tiên)**:
        - Đặt priority, ví dụ: 5.
        - Priority càng nhỏ thì ưu tiên càng cao (1 là cao nhất, có thể đến 50.000).
        - Nếu một request khớp nhiều rule, rule có priority cao hơn (số nhỏ hơn) sẽ được áp dụng.
     4. Lưu rule.
   - Sau khi tạo:
     - Có 2 rule: default rule và DemoRule.
   - Kiểm thử:
     - Lấy DNS name của ALB, truy cập:
       - `http://<ALB-DNS>` → đi theo default rule, forward đến target group (ứng dụng bình thường).
       - `http://<ALB-DNS>/error` → khớp rule DemoRule, trả về 404 với thông báo “Not found, custom error”.
     - Điều này chứng minh rule theo path `/error` hoạt động đúng.

Tóm lại:
- Phần 1: Cấu hình Security Group để EC2 chỉ nhận traffic từ ALB, tăng cường bảo mật.
- Phần 2: Cấu hình Listener Rules trên ALB để xử lý request theo điều kiện (ví dụ path `/error`) và trả về phản hồi tùy chỉnh (404 custom error).
