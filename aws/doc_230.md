


Dưới đây là tóm tắt nội dung:

1. Giới thiệu Kinesis Data Firehose  
   - Firehose nhận dữ liệu từ nhiều nguồn:  
     - Kinesis Data Streams (trong ví dụ là DemoStream)  
     - PUT trực tiếp qua Kinesis Agent, AWS SDK  
     - Các dịch vụ AWS khác: CloudWatch, IoT Core, EventBridge…  
   - Sau khi nhận dữ liệu, Firehose có thể:  
     - Gọi Lambda để chuyển đổi / lọc / xử lý bản ghi  
     - Chuyển đổi định dạng bản ghi (Parquet, ORC…)  
     - Ghi dữ liệu ra các đích: S3, OpenSearch (ElasticSearch), Redshift, HTTP endpoints, dịch vụ bên thứ ba.

2. Thiết lập Delivery Stream  
   - Chọn nguồn (source) là Kinesis Data Stream: DemoStream (nhập ARN).  
   - Tên delivery stream được tạo tự động.  
   - Tùy chọn bật Lambda để transform record (chuyển đổi, lọc, xử lý).  
   - Tùy chọn chuyển đổi định dạng bản ghi (record format conversion) – không đi sâu, chỉ cần nhớ là có thể làm.

3. Cấu hình đích (destination) S3  
   - Chọn bucket S3 đã tạo sẵn (ví dụ: demo-firehose-stephane-V3) hoặc tạo bucket mới.  
   - Không bật dynamic partitioning trong demo.  
   - Có thể cấu hình prefix cho dữ liệu và prefix cho lỗi, nhưng trong demo để trống cho đơn giản.

4. Buffering, nén (compression) và mã hóa (encryption)  
   - Buffer size: Firehose gom dữ liệu trước khi ghi ra S3.  
     - Mặc định 5 MB, có thể tăng (ví dụ 128 MB) để hiệu quả hơn hoặc giảm để ghi nhanh hơn.  
     - Trong demo đặt 1 MB để dữ liệu được ghi nhanh.  
   - Buffer interval: thời gian tối đa giữ dữ liệu trước khi đẩy ra S3.  
     - Ví dụ 300s (5 phút), 900s (15 phút).  
     - Trong demo đặt 60s (tối thiểu) để dữ liệu lên S3 nhanh.  
   - Nén: có thể chọn GZIP, Snappy, Zip… để giảm dung lượng lưu trữ.  
   - Mã hóa: có thể bật/tắt encryption cho dữ liệu.  
   - IAM Role: Firehose tự tạo một IAM role với quyền đọc từ Kinesis Stream và ghi vào S3.

5. Tạo và kiểm tra Delivery Stream  
   - Tạo delivery stream xong, có thể xem metrics, cấu hình, log đích (CloudWatch Logs).  
   - Để test, không dùng test data trong console mà dùng chính Kinesis Data Stream DemoStream.

6. Gửi dữ liệu vào Kinesis Stream  
   - Sử dụng AWS CloudShell và lệnh CLI để gửi dữ liệu vào DemoStream.  
   - Gửi 3 bản ghi: user registration, user login, user logout.  
   - Lưu ý: Firehose chỉ xử lý dữ liệu mới gửi sau khi delivery stream được thiết lập.

7. Kiểm tra dữ liệu trong S3  
   - Vào console S3, mở bucket firehose.  
   - Ban đầu chưa thấy object vì Firehose đang buffer (60s).  
   - Sau hơn 60s, refresh sẽ thấy file dữ liệu xuất hiện, được phân vùng theo ngày, v.v.  
   - Mở file (text) sẽ thấy nội dung 3 bản ghi: user registration, user login, user logout.  
   - Kết luận: Kinesis Data Firehose hoạt động đúng, dữ liệu từ Kinesis Stream đã được đẩy vào S3.

8. Dọn dẹp tài nguyên  
   - Xóa delivery stream (gõ đúng tên để xác nhận).  
   - Quan trọng: xóa luôn Kinesis Data Stream DemoStream để tránh tốn chi phí theo giờ.
