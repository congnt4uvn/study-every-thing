


Dưới đây là tóm tắt nội dung:

1. **Giám sát Lambda qua CloudWatch Metrics**  
   - Vào hàm Lambda (trigger từ S3) → tab Monitoring để xem:  
     - Số lần gọi (invocations)  
     - Thời gian thực thi  
     - Số lỗi, tỉ lệ thành công  
   - Biểu đồ có màu xanh (thành công) và đỏ (lỗi) vì đang theo dõi cả đích đến thành công và lỗi.  
   - Metrics hữu ích để theo dõi Lambda trong môi trường production, đặc biệt khi:  
     - Vượt giới hạn Lambda → async delivery thất bại  
     - Xử lý luồng (stream) với mức độ đồng thời (concurrency) cao.

2. **CloudWatch Logs cho Lambda**  
   - Mỗi lần Lambda chạy sẽ tạo log stream trong CloudWatch Logs.  
   - Log chứa:  
     - Request ID  
     - Các log do lập trình viên ghi (ví dụ log trong code)  
     - Báo cáo cuối cùng: thời gian chạy, chi phí, dung lượng bộ nhớ cấu hình, bộ nhớ tối đa dùng, duration cold start (nếu có).

3. **Bật AWS X-Ray cho Lambda**  
   - Vào phần Configuration → Monitoring and operations tools → Edit.  
   - CloudWatch Logs bật sẵn, có thể bật thêm X-Ray.  
   - Khi bật, Lambda sẽ ghi trace vào AWS X-Ray.  
   - Console có thể tự thêm quyền cần thiết vào IAM role của Lambda (quyền ghi X-Ray).  
   - Trong IAM role của Lambda có thể thấy quyền:  
     - CloudWatch Logs  
     - SQS (cho destinations)  
     - X-Ray (để ghi trace, segment, telemetry).

4. **Kiểm tra X-Ray với S3 trigger**  
   - Upload file lên S3 để kích hoạt Lambda.  
   - Lúc đầu Lambda lỗi → sửa code cho chạy thành công, deploy lại, rồi upload file mới.  
   - Sau khi Lambda xử lý các file, mở AWS X-Ray console → Service map.  
   - Service map hiển thị:  
     - Client gọi Lambda  
     - Lambda hoạt động, đôi khi lỗi (màu cam/đỏ), đôi khi thành công (màu xanh).  
   - Qua đó thấy được trace của Lambda trong X-Ray và cách tích hợp Lambda + X-Ray giúp quan sát, theo dõi lỗi và hiệu năng.

5. **Ý chính**  
   - CloudWatch Metrics: theo dõi số liệu vận hành (invocations, errors, duration, concurrency).  
   - CloudWatch Logs: xem chi tiết log và báo cáo thực thi.  
   - AWS X-Ray: xem trace, service map, lỗi, hiệu năng, thể hiện rõ luồng gọi và trạng thái hoạt động của Lambda.
