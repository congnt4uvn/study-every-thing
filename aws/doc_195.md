


Người hướng dẫn đang hướng dẫn thực hành AWS CloudFormation với ví dụ tạo EC2:

1. **Chọn vùng**  
   - Yêu cầu dùng vùng **US East (N. Virginia) – us-east-1** vì các template (đặc biệt AMI ID) được thiết kế cho vùng này.

2. **Làm quen CloudFormation & Application Composer**  
   - Vào dịch vụ CloudFormation, ban đầu có thể chưa có stack.  
   - Tạo stack mới bằng cách chọn một template có sẵn, ví dụ: **Multi_AZ_Simple, blog WordPress**.  
   - Không triển khai mà mở template trong **Application Composer** để xem:
     - Giao diện trực quan (Canvas) hiển thị các thành phần như: WebServerSecurityGroup, LaunchConfig, WebServerGroup, DB instance, DB security group, v.v.
     - Phần **Template** hiển thị mã dưới dạng **YAML hoặc JSON** (giảng viên ưu tiên YAML vì dễ đọc).  
   - Mục đích: hiểu rằng template CloudFormation (mã) tương ứng với các tài nguyên thực tế trên AWS.

3. **Giới thiệu template YAML đơn giản (0-just-ec2.yaml)**  
   - Mở file `0-just-ec2.yaml` trong thư mục CloudFormation.  
   - Cấu trúc cơ bản:
     - Phần **Resources** là bắt buộc.  
     - Định nghĩa tài nguyên đầu tiên:  
       - Tên logic: `MyInstance`  
       - Type: `AWS::EC2::Instance`  
       - Properties:
         - `AvailabilityZone: us-east-1a`
         - `ImageId: <AMI ID>` (AMI Amazon Linux 2023 cho us-east-1)
         - `InstanceType: t2.micro`  
   - Chỉ với vài thuộc tính này đã đủ để CloudFormation tạo một EC2 instance.

4. **Tạo stack từ template 0-just-ec2.yaml**  
   - Trong CloudFormation:
     - Chọn **Upload a template file** và tải `0-just-ec2.yaml`.  
     - Đặt tên stack: **EC2InstanceDemo**.  
     - Bỏ qua phần tags, permissions, các thiết lập nâng cao.  
   - Khi nhấn **Submit**, template được:
     - Upload lên **S3**.
     - CloudFormation dùng file trên S3 để tạo stack.

5. **Theo dõi quá trình tạo stack**  
   - Tab **Events** hiển thị các sự kiện:
     - `MyInstance` chuyển từ `CREATE_IN_PROGRESS` sang `CREATE_COMPLETE`.  
   - Điều này nghĩa là EC2 instance đã được tạo thành công.

6. **Kiểm tra EC2 instance được tạo**  
   - Tab **Resources** của stack hiển thị tài nguyên `MyInstance` với **Physical ID**.  
   - Nhấp vào Physical ID để mở trực tiếp console EC2:
     - Kiểm tra:
       - Instance type: **t2.micro** (đúng như trong template).
       - Availability Zone: **us-east-1a**.
       - AMI: đúng AMI Amazon Linux 2023 đã chỉ định.
     - Tab **Tags**:
       - CloudFormation tự gắn các tag:
         - `aws:cloudformation:stack-name = EC2InstanceDemo`
         - `aws:cloudformation:logical-id = MyInstance`
         - `aws:cloudformation:stack-id = <ID stack>`

7. **Các tab khác trong CloudFormation**  
   - **Stack info**: thông tin chung về stack.  
   - **Events**: lịch sử các sự kiện tạo/xóa/cập nhật.  
   - **Resources**: danh sách tài nguyên được tạo.  
   - **Outputs**: hiện chưa có gì (template chưa định nghĩa Outputs).  
   - **Parameters**: cũng chưa có gì (template không dùng Parameters).  
   - **Template**: chính là file YAML đã upload.

8. **Kết luận**  
   - Bài học minh họa sức mạnh của CloudFormation:
     - Có thể tạo tài nguyên (như EC2) hoàn toàn bằng **mã** (YAML/JSON), không cần cấu hình thủ công trên console.
     - CloudFormation quản lý tài nguyên theo stack, gắn tag tự động, và cho phép xem, theo dõi, quản lý tập trung.  
   - Bài tiếp theo sẽ tiếp tục đào sâu hơn về CloudFormation.
