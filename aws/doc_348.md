


Nội dung trên nói về cách cung cấp API cho khách hàng và quản lý việc tính phí/throttle thông qua **Usage Plan** và **API Key** trong API Gateway (AWS).

Tóm tắt:

1. **Usage Plan (gói sử dụng)**  
   - Xác định:
     - Ai được truy cập API (giai đoạn/stage nào, phương thức nào).  
     - Mức độ và tốc độ truy cập (throttle, quota).  
     - API key nào gắn với gói này để nhận diện khách hàng và đo lường việc sử dụng.  
   - Có thể cấu hình:
     - **Throttle limit**: tốc độ gọi (ví dụ số request/giây).  
     - **Quota limit**: tổng số request trong một khoảng thời gian (ví dụ 10.000 request/tháng, vượt thì phải trả thêm).

2. **API Key**  
   - Là chuỗi ký tự cấp cho khách hàng.  
   - Dùng để:
     - Xác thực và cho phép khách hàng sử dụng API qua API Gateway.  
     - Kết hợp với Usage Plan để kiểm soát và giới hạn truy cập.  
   - Các giới hạn (throttle/quota) được áp dụng ở **cấp API key**.

3. **Thứ tự các bước cấu hình**  
   Để mọi thứ hoạt động đúng, cần làm theo thứ tự:
   1) Tạo một hoặc nhiều API.  
   2) Cấu hình các phương thức yêu cầu **API key** (bắt buộc x-api-key).  
   3) Triển khai API lên các **stage**.  
   4) Tạo hoặc import **API key** để cấp cho khách hàng (developer ứng dụng).  
   5) Tạo **Usage Plan** với các giới hạn throttle/quota mong muốn.  
   6) Gắn **stage của API** và **API key** vào Usage Plan.  

4. **Cách khách hàng gọi API**  
   - Khi gọi API, khách hàng phải gửi **API key** trong header:  
     `x-api-key: <API_KEY>`  
   - Nếu quên bước này hoặc quên gắn stage/API key vào Usage Plan, hệ thống sẽ không hoạt động như mong muốn.

Mục tiêu: dùng Usage Plan + API Key để phân phối, giám sát và giới hạn việc sử dụng API cho từng khách hàng.
