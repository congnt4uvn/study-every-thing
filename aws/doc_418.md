


Dưới đây là tóm tắt nội dung:

1. Bối cảnh và vấn đề  
   - Tạo một hàm Lambda mới (Python) tên “Lambda KMS”.  
   - Cần lưu mật khẩu DB (db_password).  
   - Nếu viết thẳng mật khẩu trong code → ai có quyền xem code Lambda sẽ thấy mật khẩu → không an toàn.  
   - Nếu dùng biến môi trường (Environment Variable) dạng plaintext → ai có quyền xem cấu hình Lambda vẫn thấy mật khẩu → vẫn không an toàn.

2. Giải pháp: Mã hóa biến môi trường với KMS  
   - Vào phần cấu hình Lambda → Environment variables.  
   - Tạo biến DB_PASSWORD với giá trị “supersecret”.  
   - Vào phần Encryption configuration → chọn KMS key (khóa “tutorial” đã tạo trước trong KMS).  
   - Bấm Encrypt để mã hóa biến môi trường bằng KMS key đó.  
   - Sau khi mã hóa, giá trị biến môi trường trong console sẽ hiển thị dạng mã hóa.

3. Sử dụng KMS trong code Lambda  
   - AWS cung cấp sẵn đoạn code mẫu để giải mã biến môi trường.  
   - Sao chép đoạn code mẫu đó, dán vào hàm Lambda.  
   - Logic chính:
     - Lấy biến môi trường đã mã hóa (os.getenv("DB_PASSWORD")).  
     - Dùng AWS SDK (boto3 KMS client) gọi API `decrypt` để giải mã.  
     - Lấy kết quả plaintext, decode UTF-8 để có chuỗi mật khẩu.  
     - In ra giá trị đã mã hóa và giá trị đã giải mã (để kiểm tra).

4. Lỗi và cách khắc phục  
   - Lỗi 1: Timeout (3 giây không đủ) → tăng timeout lên 10 giây trong phần General configuration.  
   - Lỗi 2: AccessDeniedException khi gọi KMS decrypt.  
     - Nguyên nhân: IAM role của Lambda chưa có quyền dùng KMS key để decrypt.  
     - Cách xử lý:
       - Vào Configuration → Permissions → Execution role của Lambda.  
       - Thêm inline policy mới.  
       - Chọn service: KMS.  
       - Action: `kms:Decrypt`.  
       - Resource: ARN của KMS key (copy từ KMS console).  
       - Lưu policy (ví dụ đặt tên “allow-kms-decrypt”).  

5. Kết quả  
   - Sau khi thêm quyền, chạy lại hàm Lambda → thành công.  
   - Log cho thấy:
     - Biến môi trường được mã hóa (chuỗi ciphertext).  
     - Giá trị đã giải mã: “supersecret”.  
   - Ưu điểm:
     - Code không chứa mật khẩu.  
     - Biến môi trường trong console cũng ở dạng mã hóa; chỉ những ai có quyền dùng KMS key mới giải mã được.  
     - Mật khẩu DB được bảo vệ tốt hơn, thể hiện sự tích hợp chặt chẽ giữa Lambda và KMS.
