


Nội dung trên giới thiệu nhanh về PartiQL cho DynamoDB và cách dùng trong AWS Console:

1. PartiQL là gì  
- Là ngôn ngữ truy vấn có cú pháp giống SQL dùng cho DynamoDB.  
- Cho phép thao tác bảng DynamoDB bằng các câu lệnh kiểu SQL: SELECT, INSERT, UPDATE, DELETE.  
- Hữu ích cho những người quen với SQL, đồng thời hỗ trợ cả thao tác theo lô (batch operations).

2. Thao tác trên bảng trong Console  
- Trong phần Tables của DynamoDB Console có “PartiQL editor” ở bên trái.  
- Ví dụ tạo dữ liệu:
  - Bảng `users`: thêm item với `user_id = 123`, `name = Stephan`.  
  - Bảng `user_posts`: thêm item với `user_id = 123`, `post_id = 456`.  
  - Bảng `demo_indexes`: thêm item với `user_id = 123`, `game_timestamp = 2022`, `game_id = 456`.

3. Sử dụng PartiQL editor  
- Có thể chọn bảng `users`, dùng câu lệnh kiểu:  
  - `SELECT * FROM "users"` để quét (scan) bảng.  
- Kết quả hiển thị các item, có thể xem ở dạng “attribute view” (dạng JSON/Key-Value) giống như khi dùng trong code.  
- Có thể tải kết quả xuống file CSV.

4. Truy vấn phức tạp hơn và dùng index  
- Với bảng `demo_indexes`, có thể:
  - Scan toàn bộ bảng.  
  - Query với điều kiện, ví dụ:  
    - `SELECT * FROM "demo_indexes" WHERE user_id = 123 AND game_timestamp = ...`  
- Có thể chỉ định sử dụng index:  
  - `SELECT * FROM "demo_indexes"."TênIndex"` để truy vấn dựa trên index thay vì bảng chính.

5. Các thao tác khác với PartiQL  
- INSERT: chèn item mới (không được UI tạo sẵn, phải tự viết câu lệnh).  
- UPDATE (PUT/SET): cập nhật item cụ thể, đặt lại giá trị thuộc tính dựa trên partition key và sort key.  
- DELETE: xóa item cụ thể bằng câu lệnh `DELETE FROM ...`.

Kết luận: PartiQL editor trong DynamoDB Console là công cụ cho phép dùng cú pháp giống SQL để xem, chèn, cập nhật, xóa dữ liệu và truy vấn bằng index trên các bảng DynamoDB, rất phù hợp cho người quen làm việc với SQL.
