


Dưới đây là tóm tắt nội dung về AWS CodeBuild trong đoạn trên:

1. Nguồn mã và buildspec.yml  
- CodeBuild lấy source từ CodeCommit, S3, Bitbucket, GitHub.  
- File cấu hình build là `buildspec.yml`, bắt buộc phải nằm ở thư mục gốc (root) của repo.  
- Có thể cấu hình lệnh build trực tiếp trên console, nhưng tốt nhất là dùng `buildspec.yml` (quan trọng cho kỳ thi).

2. Kết quả build, log và giám sát  
- Sau khi build, log có thể lưu vào Amazon S3 và CloudWatch Logs.  
- CloudWatch Metrics dùng để xem thống kê build.  
- EventBridge có thể phát hiện build thất bại và kích hoạt thông báo (ví dụ CloudWatch Alarms).

3. Build project và CodePipeline  
- Dự án build có thể được định nghĩa trong CodeBuild hoặc trong CodePipeline.  
- CodePipeline có thể gọi các CodeBuild Project hiện có.

4. Môi trường build (build environment)  
- Có sẵn image cho Java, Ruby, Python, Go, Node.js, Android, .NET Core, PHP.  
- Có thể dùng Docker image tùy chỉnh để hỗ trợ bất kỳ ngôn ngữ/môi trường nào khác (tự chịu trách nhiệm cấu hình).

5. Quy trình hoạt động của CodeBuild  
- CodeBuild lấy source (ví dụ từ CodeCommit) + file `buildspec.yml`.  
- Tạo một container build từ Docker image (AWS cung cấp hoặc tự cung cấp).  
- Container tải source + `buildspec.yml` và chạy các lệnh trong đó.  
- Có tùy chọn cache file (thường là dependency) lên S3 để tăng tốc các lần build sau.  
- Log được gửi tới CloudWatch Logs và S3 (nếu bật).  
- Artifact (kết quả build) được trích xuất từ container và lưu vào S3.

6. Cấu trúc chính của buildspec.yml  
- Phải nằm ở root của source code.  
- `env`: định nghĩa biến môi trường  
  - Có thể là plain text.  
  - Hoặc lấy từ SSM Parameter Store / Secrets Manager (để không lưu mật khẩu, secret trong file).  
- `phases`: các giai đoạn build  
  - `install`: cài đặt dependency, chuẩn bị môi trường.  
  - `pre_build`: lệnh chạy ngay trước khi build.  
  - `build`: lệnh build chính (quan trọng nhất).  
  - `post_build`: bước hoàn thiện (ví dụ tạo file zip output).  
- `artifacts`: chỉ định file/thư mục nào trong container sẽ được xuất ra S3, có thể mã hóa.  
- `cache`: chỉ định file/thư mục (thường là dependency) được cache trên S3 để tăng tốc build.

Đây là các điểm cốt lõi về CodeBuild mà kỳ thi thường kiểm tra: vai trò của `buildspec.yml`, vị trí của nó, cách CodeBuild dùng container/image, log, artifact, cache và tích hợp với các dịch vụ khác như CloudWatch, EventBridge, S3, CodePipeline.
