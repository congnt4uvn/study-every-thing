


Dưới đây là tóm tắt ngắn gọn nội dung bạn đưa:

1. Khái niệm WebSocket  
- WebSocket là kênh giao tiếp hai chiều, tương tác, liên tục giữa trình duyệt (client) và server.  
- Server có thể chủ động “đẩy” dữ liệu về client mà không cần client gửi request trước.  
- Phù hợp cho ứng dụng thời gian thực: chat, cộng tác, game nhiều người chơi, trading platform,…

2. WebSocket với API Gateway (AWS)  
- Client kết nối tới WebSocket API trên API Gateway qua URL dạng `wss://...execute-api.<region>.amazonaws.com/<stage>`.  
- Khi kết nối được thiết lập:
  - API Gateway gọi Lambda `onConnect`.
  - Lambda có thể lưu `connectionId` vào DynamoDB kèm metadata (ví dụ: user nào).

3. Gửi và nhận tin nhắn  
- Client gửi tin nhắn qua kết nối WebSocket liên tục:
  - Mỗi tin nhắn là một “frame”.
  - API Gateway định tuyến frame này tới một Lambda (ví dụ `sendMessage`).
  - `connectionId` vẫn giữ nguyên trong suốt phiên kết nối.
  - Lambda có thể đọc/ghi dữ liệu từ DynamoDB, xử lý logic ứng dụng.
- Khi client muốn ngắt kết nối:
  - Sẽ kích hoạt Lambda `onDisconnect` để dọn dẹp (xóa connectionId, cập nhật trạng thái,…).

4. Server gửi dữ liệu ngược lại cho client  
- API Gateway cung cấp một “callback URL” dạng:  
  `https://...execute-api.<region>.amazonaws.com/<stage>/@connections/{connectionId}`  
- Backend (Lambda hoặc dịch vụ khác) có thể:
  - Gửi POST (ký IAM SigV4) tới URL này để đẩy tin nhắn từ server về client.
  - Gửi GET để lấy trạng thái kết nối.
  - Gửi DELETE để chủ động ngắt kết nối client.

5. Cơ chế định tuyến (Routing)  
- WebSocket API dùng “route selection expression” để quyết định tin nhắn sẽ đi tới backend nào.  
- Ví dụ: `routeSelectionExpression = $request.body.action`  
  - Nếu JSON gửi lên là:  
    ```json
    {
      "service": "chat",
      "action": "join",
      "data": "room1234"
    }
    ```  
  - API Gateway sẽ đọc trường `action` = `"join"`, tra trong “Route Key Table”:
    - Các route bắt buộc: `$connect`, `$disconnect`, `$default`.
    - Các route tùy chỉnh: `join`, `leave`, `delete`,…  
  - Nếu có route `join`, nó sẽ gọi đúng backend (Lambda hoặc dịch vụ khác) gắn với route đó.  
  - Nếu không khớp route nào, sẽ rơi vào `$default`.

6. Ý chính cần nhớ cho kỳ thi  
- WebSocket API dùng cho giao tiếp hai chiều, thời gian thực.  
- Có các route đặc biệt: `$connect`, `$disconnect`, `$default`.  
- Routing dựa trên “route selection expression” (thường là một trường trong JSON) để gửi tin nhắn đến đúng backend.  
- Kết nối là liên tục, `connectionId` được dùng để:
  - Lưu trạng thái/metadata (DynamoDB).
  - Gửi tin nhắn ngược lại cho đúng client qua URL `/@connections/{connectionId}`.
