


Dưới đây là tóm tắt nội dung bài giảng:

- **CloudTrail**  
  - Ghi lại **mọi lệnh gọi API** trong tài khoản AWS (từ user, service, hoặc AWS Console).  
  - Dùng để:
    - Kiểm tra, audit hoạt động.  
    - Phát hiện các lệnh gọi trái phép.  
    - Tìm **nguyên nhân gốc** của các thay đổi do lệnh gọi API gây ra.

- **CloudWatch**  
  - Dùng để **giám sát** hệ thống.  
  - Gồm:
    - **CloudWatch Metrics**: theo dõi các chỉ số (CPU, RAM, request count, v.v.).  
    - **CloudWatch Logs**: lưu trữ và xem **log ứng dụng**.  
    - **CloudWatch Alarms**: gửi cảnh báo khi số liệu vượt ngưỡng (ví dụ CPU quá cao).  
  - Tập trung vào **monitoring tổng thể**.

- **X-Ray**  
  - Dùng cho **phân tích trace** và **trực quan hóa bản đồ dịch vụ** trong hệ thống phân tán (microservices).  
  - Hữu ích cho:
    - Gỡ lỗi (debug).  
    - Xem **độ trễ**, **lỗi**, và phân tích lỗi chi tiết.  
    - Theo dõi đường đi của request qua nhiều service.

**So sánh ngắn gọn:**
- **CloudTrail**: Audit **lệnh gọi API**.  
- **CloudWatch**: Giám sát **metrics + logs + alarms**.  
- **X-Ray**: Phân tích **trace chi tiết** trong hệ thống phân tán.
