


Nội dung trên mô tả quy trình tạo và public một AWS Lambda Function thông qua Function URL:

1. Tạo Lambda function  
   - Tạo hàm mới, đặt tên là `lambda-demo-url`.  
   - Chọn runtime Python 3.9.  
   - Sau khi tạo xong, dùng phần “Test” để tạo một test event mới, đặt tên (ví dụ: `test`) và lưu.  
   - Chạy test, hàm trả về status code 200 với nội dung `"Hello from Lambda"` – tức là hàm hoạt động như một HTTP response.

2. Xuất bản phiên bản (version) và tạo alias  
   - Xuất bản hàm thành một phiên bản mới, đặt tên là `version 1`.  
   - Tạo một alias mới, đặt tên là `dev`, và trỏ alias `dev` đến `version 1`.

3. Tạo Function URL  
   - Ở phần cấu hình, mục “Function URL” ban đầu trống.  
   - Chọn “Create function URL”.  
   - Chọn loại xác thực (AuthType): IAM hoặc NONE. Ở đây chọn `NONE` để ai cũng có thể truy cập.  
   - Một Resource Policy được tạo tự động, cho phép mọi người truy cập URL của Lambda khi AuthType = NONE.  
   - Có thể cấu hình CORS (Cross-Origin Resource Sharing) để quy định origin, headers, v.v., nhưng trong ví dụ này không cần.

4. Truy cập Lambda qua URL  
   - Sau khi tạo xong, Function URL gắn với alias `dev` sẽ cố định (không đổi) cho alias đó.  
   - Mở tab trình duyệt mới, dán URL vào, sẽ thấy phản hồi `"Hello from Lambda"`.  
   - Trình duyệt có thể hiển thị thêm một số nội dung phụ, nhưng nội dung chính là `"Hello from Lambda"`.

5. Lưu ý về version và alias  
   - Khi quay lại phần “Aliases” và chọn `version 1`, nếu cố tạo Function URL trực tiếp cho version đã xuất bản, sẽ thấy thông báo: chỉ có thể thêm Function URL cho phiên bản chưa xuất bản (unpublished version – `$LATEST`) hoặc alias.  
   - Do đó, có thể tạo Function URL cho alias (như `dev`) hoặc cho phiên bản mới nhất (`$LATEST`), để mọi người truy cập phiên bản mới nhất của Lambda.

Kết luận: Bài giảng hướng dẫn cách:
- Tạo Lambda function,
- Test hàm,
- Xuất bản version và tạo alias,
- Tạo Function URL với AuthType = NONE,
- Truy cập công khai Lambda qua URL cố định gắn với alias hoặc `$LATEST`.
