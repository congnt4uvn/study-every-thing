


Dưới đây là tóm tắt nội dung về Amazon OpenSearch:

1. Khái niệm và bối cảnh  
- Amazon OpenSearch là dịch vụ kế thừa từ Amazon Elasticsearch (đổi tên do vấn đề giấy phép).  
- Thường dùng như một dịch vụ bổ trợ cho cơ sở dữ liệu khác (ví dụ DynamoDB), chứ không thay thế DB chính.

2. Khả năng tìm kiếm và phân tích  
- Khác với DynamoDB (chỉ truy vấn theo khóa chính/chỉ mục), OpenSearch cho phép:  
  - Tìm kiếm trên bất kỳ trường nào.  
  - Hỗ trợ tìm kiếm khớp một phần (full-text search).  
- Ngoài tìm kiếm, còn hỗ trợ truy vấn phân tích dữ liệu.  
- Có OpenSearch Dashboards để trực quan hóa và phân tích dữ liệu.

3. Cách triển khai (deployment modes)  
- Managed cluster: cụm được quản lý, bạn thấy và quản lý các instance.  
- Serverless: cụm không máy chủ, AWS tự động lo về scaling và vận hành.

4. Ngôn ngữ truy vấn  
- OpenSearch có ngôn ngữ truy vấn riêng.  
- Không hỗ trợ SQL gốc, nhưng có thể bật tương thích SQL qua plugin.

5. Nguồn dữ liệu đầu vào  
- Có thể ingest dữ liệu từ:  
  - Kinesis Data Firehose  
  - AWS IoT  
  - CloudWatch Logs  
  - Ứng dụng tùy chỉnh

6. Bảo mật  
- Tích hợp với Cognito, IAM.  
- Hỗ trợ mã hóa khi lưu trữ (at rest) và khi truyền (in transit).

7. Mẫu kiến trúc phổ biến với DynamoDB  
- Dữ liệu chính lưu trong DynamoDB (insert/update/delete).  
- DynamoDB Streams gửi sự kiện đến Lambda.  
- Lambda ghi dữ liệu vào OpenSearch theo thời gian thực.  
- Ứng dụng:  
  - Tìm kiếm (ví dụ tìm theo tên, khớp một phần) trên OpenSearch để lấy item ID.  
  - Sau đó dùng ID gọi DynamoDB để lấy bản ghi đầy đủ.  
→ OpenSearch dùng để tìm kiếm, DynamoDB vẫn là nguồn dữ liệu chính.

8. Mẫu ingest với CloudWatch Logs  
- Cách 1: CloudWatch Logs → Subscription Filter → Lambda (do AWS quản lý) → OpenSearch (thời gian thực).  
- Cách 2: CloudWatch Logs → Subscription Filter → Kinesis Data Firehose → OpenSearch (gần thời gian thực).

9. Mẫu ingest với Kinesis Data Streams  
- Cách 1: Kinesis Data Firehose → (tùy chọn Lambda transform) → OpenSearch (gần thời gian thực).  
- Cách 2: Lambda đọc trực tiếp từ Kinesis Data Streams → ghi vào OpenSearch (thời gian thực, code tùy chỉnh).

Tóm lại: Amazon OpenSearch là dịch vụ tìm kiếm và phân tích mạnh mẽ, thường dùng cùng với các dịch vụ như DynamoDB, Kinesis, CloudWatch Logs để cung cấp khả năng search và analytics gần thời gian thực cho ứng dụng.
