


Dưới đây là tóm tắt nội dung:

- Người hướng dẫn tạo một resource mới trong API Gateway tên là `mapping`, sau đó tạo một method `GET` cho resource này.
- Tạo một hàm Lambda mới tên `API Gateway Mapping Get` (Python 3.11), tích hợp với API Gateway nhưng **không** dùng Lambda proxy integration, mà là integration trực tiếp.
- Hàm Lambda rất đơn giản: trả về một JSON dạng:
  ```json
  {
    "example": "hello world"
  }
  ```
- Kiểm tra Lambda trực tiếp: kết quả trả về đúng `"example": "hello world"`.
- Kiểm tra qua API Gateway: response body là `{"example": "hello world"}`, status code 200.  
  Vì không dùng proxy integration nên Lambda chỉ cần trả JSON, không cần tự set status code hay headers.
- Mục tiêu tiếp theo: thay đổi nội dung response (thêm key mới, đổi tên key, …) mà **không sửa Lambda**, bằng cách dùng **Mapping Template** trên **Integration Response**.
- Trong Integration Response:
  - Tạo một mapping template với `Content-Type: application/json`.
  - Định nghĩa nội dung response mới, ví dụ:
    ```json
    {
      "myKey": "myValue",
      "myRenamedKey": "$input.json('$.example')"
    }
    ```
    - `myKey: myValue` luôn có trong mọi response.
    - `myRenamedKey` lấy giá trị từ trường `example` trong JSON mà Lambda trả về.
    - Cú pháp `$input.json('$.example')` dùng để truy cập giá trị của key `example` trong JSON input.
- Sau khi tạo mapping template, mọi response từ Lambda sẽ đi qua template này và bị “biến đổi”:
  - Kết quả trả về cho client sẽ là:
    ```json
    {
      "myKey": "myValue",
      "myRenamedKey": "hello world"
    }
    ```
- Kết luận:
  - Mapping template có thể áp dụng cho cả **Integration Request** và **Integration Response**.
  - Chúng cho phép chuyển đổi input/output của integration để phù hợp với API mà bạn muốn thiết kế.
  - Không cần nhớ chi tiết cú pháp cho kỳ thi, chỉ cần hiểu rằng mapping template là công cụ để biến đổi dữ liệu giữa API Gateway và backend (Lambda, v.v.).
