


Dưới đây là tóm tắt nội dung:

1. Tạo cụm ECS mới  
   - Vào Amazon ECS → Clusters → Create cluster.  
   - Đặt tên cụm, ví dụ: `DemoCluster`.  

2. Chọn hạ tầng (capacity provider)  
   - Tùy chọn chính:  
     - **Fargate**: mô hình không máy chủ, AWS quản lý hạ tầng.  
     - **Fargate + Managed Instances**: vẫn dùng Fargate, nhưng AWS tự động quản lý các EC2 instance phía sau.  
     - **Fargate + Self-managed instances**: bạn tự quản lý EC2 theo cách truyền thống (tự tạo Auto Scaling Group, chọn AMI, loại EC2, v.v.).  
   - AWS đang hướng người dùng sang mô hình không tự quản lý (Fargate/Managed).

3. Tạo IAM role cho ECS Managed Instances  
   - Tạo **instance profile** cho ECS (ví dụ `ecsInstanceRole`) để EC2 có thể đăng ký vào ECS.  
   - Tạo **infrastructure role** cho ECS Managed Instances.  

4. Cấu hình “Quick option” (tùy chọn nhanh)  
   - Có thể để ECS tự chọn tài nguyên dựa trên task definition và yêu cầu dịch vụ (phù hợp môi trường production).  
   - Hoặc dùng cấu hình tùy chỉnh:  
     - Chọn vCPU, RAM tối thiểu/tối đa.  
     - Giới hạn loại instance được phép (ví dụ chỉ `t3.micro`).  

5. Chuyển sang mô hình self-managed EC2 (để tương thích với các bài giảng sau)  
   - Tạo Auto Scaling Group (ASG) mới:  
     - Loại instance: `t3.micro`.  
     - Dùng vai trò mặc định cho EC2.  
     - Max 2 instance, không cần SSH.  
     - EBS root volume tối thiểu 30 GB.  
     - Giữ cấu hình mạng mặc định.  

6. Quan sát Auto Scaling Group  
   - ASG được tạo với tên kiểu `Infra-ECS-Cluster`, min=0, max=5, desired=0.  
   - Được tạo tự động bởi cụm ECS.  
   - ASG trải trên 3 Availability Zones → task ECS có thể chạy trên 3 AZ.

7. Xem cụm DemoCluster  
   - Vào cụm → hiện tại Services = 0, Tasks = 0.  
   - Phần Infrastructure cho thấy 3 capacity provider:  
     - **FARGATE**: chạy task Fargate.  
     - **FARGATE_SPOT**: chạy task Fargate trên Spot.  
     - **ASG provider**: chạy task trên EC2 thông qua ASG.  

8. Tăng desired capacity của ASG  
   - Chỉnh desired capacity = 1 → một EC2 instance sẽ được tạo.  
   - Instance này tự động đăng ký vào cụm ECS.  
   - Trong phần Container instances của cụm, thấy instance:  
     - Trạng thái Running.  
     - Ví dụ loại `t2.micro`.  
     - Có tài nguyên khả dụng: 1024 CPU, 982 MB RAM.  
   - Task ECS sau đó có thể được chạy trên:  
     - Fargate,  
     - Fargate Spot,  
     - Hoặc EC2 container instance trong ASG.

9. Kết luận  
   - Đã tạo xong một cụm ECS (`DemoCluster`).  
   - Đã cấu hình và hiểu 3 loại capacity provider (Fargate, Fargate Spot, ASG/EC2).  
   - Đã có một EC2 container instance sẵn sàng để chạy task.  
   - Bước tiếp theo (ở bài giảng sau) là chạy dịch vụ ECS đầu tiên trên cụm này.
