


Nội dung trên là một đoạn hướng dẫn thực hành tạo user IAM trong AWS, có các ý chính:

1. Vào dịch vụ IAM  
- Trên thanh tìm kiếm gõ “IAM” để vào IAM Console.  
- IAM là dịch vụ “global” nên không chọn region được; user IAM dùng được ở mọi region.

2. Lý do tạo IAM user  
- Hiện tại đang dùng tài khoản root (chỉ thấy Account ID ở góc trên phải).  
- Dùng root thường xuyên là không tốt về bảo mật.  
- Cần tạo user IAM (ví dụ user admin) để sử dụng tài khoản an toàn hơn.

3. Tạo IAM user mới  
- Vào mục “Users” ở menu bên trái, chọn tạo user mới, ví dụ tên “Stephane”.  
- Chọn cho phép truy cập AWS Management Console.  
- Chọn “Create IAM user” (không dùng Identity Center trong ví dụ này).  
- Đặt mật khẩu:
  - Với user khác: để auto-generate và bắt buộc đổi mật khẩu lần đầu.  
  - Với chính mình: đặt mật khẩu custom, bỏ chọn bắt buộc đổi mật khẩu.

4. Gán quyền cho user qua group  
- Thay vì gán quyền trực tiếp, tạo một group mới, tên “admin”.  
- Gán policy “AdministratorAccess” cho group admin.  
- Thêm user Stephane vào group admin.  
- Xem lại: user, group, tags.  
- Tags là metadata tùy chọn, ví dụ: Department = Engineering.

5. Kiểm tra user và group  
- Trong “Users”: thấy user Stephane.  
- Trong “User groups”: thấy group admin, trong đó có user Stephane.  
- Trong phần Permissions:
  - Group admin có policy AdministratorAccess.  
  - User Stephane có quyền admin thông qua group, không gán trực tiếp.  
- Giải thích: quản lý quyền qua group giúp quản trị dễ hơn.

6. Tạo alias và URL đăng nhập  
- Vào phần Account để xem Account ID và login URL.  
- Có thể tạo Account alias (ví dụ: aws-stephane-v5) để URL đăng nhập dễ nhớ hơn.  
- Alias phải là duy nhất trên toàn AWS.

7. Đăng nhập bằng IAM user  
- Nên mở thêm một cửa sổ trình duyệt ở chế độ Private/Incognito để đăng nhập IAM user, trong khi cửa sổ thường vẫn đăng nhập root.  
- Dán login URL, chọn đăng nhập bằng IAM user.  
- Nhập Account ID hoặc alias, sau đó nhập username (Stephane) và password.  
- Khi đăng nhập:
  - Cửa sổ root: góc trên phải chỉ hiện Account ID → đang dùng root.  
  - Cửa sổ IAM: góc trên phải hiện Account ID + IAM user → đang dùng IAM user.

8. Lưu ý bảo mật và sử dụng trong khóa học  
- Tuyệt đối không làm mất thông tin đăng nhập root và admin; nếu mất phải liên hệ AWS Support.  
- Khuyến nghị: dùng IAM user thay vì root cho công việc hàng ngày.  
- Trong khóa học, giảng viên sẽ chỉ rõ khi nào phải dùng root, khi nào dùng IAM user.  
- Yêu cầu giữ cả hai cửa sổ (root và IAM user) mở cho các bài học tiếp theo.
