


Dưới đây là tóm tắt nội dung bài giảng:

1. Cài đặt công cụ  
- Vào trang hướng dẫn Copilot (CLI) và cài đặt Copilot phù hợp với hệ điều hành (Mac, Linux, Windows).  
- Sau khi cài xong, chạy `copilot --help` để kiểm tra.  
- Đảm bảo đã cài đặt AWS CLI và Docker Desktop trên máy.

2. Chuẩn bị mã nguồn  
- Sao chép một repository mẫu đã được cấu hình sẵn để dùng với Copilot.  
- Trong repo có sẵn Dockerfile mô tả cách build image.  
- Đây là thư mục cơ bản để làm việc với Copilot.

3. Khởi tạo Copilot  
- Di chuyển vào thư mục ví dụ, chạy `copilot init`.  
- Đặt tên ứng dụng, ví dụ: `example-app`.  
- Chọn loại workload: ở đây chọn “Load Balanced Web Service” (dịch vụ web cân bằng tải).  
- Đặt tên dịch vụ, ví dụ: `front-end`.  
- Chỉ định Dockerfile để Copilot dùng.

4. Tạo môi trường và triển khai  
- Copilot hỏi về môi trường (ví dụ: `test`), nếu chưa có sẽ tạo mới.  
- Copilot dùng CloudFormation để tạo các stack:  
  - Stack quản trị (quyền, role, v.v.)  
  - Stack ứng dụng Copilot (dịch vụ thực tế).  
- Trong quá trình này, các tài nguyên như KMS key, S3 bucket, ECR repository… được tạo tự động.  
- Sau khi triển khai xong, Copilot cung cấp URL public.

5. Kiểm tra ứng dụng và tài nguyên  
- Mở URL được Copilot trả về, thấy ứng dụng demo (logo Copilot).  
- Vào CloudFormation để xem các stack và tài nguyên đã tạo:  
  - ECS Cluster, Load Balancer, Security Group, VPC, IAM Role, Lambda (nếu có), Log Group, ECS Service, v.v.  
- Khuyến khích vào ECS console xem cluster `example-app-test-Cluster`, dịch vụ, và cấu hình theo best practice.  
- Kiểm tra thêm Load Balancer trong console.

6. Cấu hình dưới dạng mã (Copilot folder)  
- Trong thư mục dự án, Copilot tạo thư mục `copilot/` gồm:  
  - `environments/` (ví dụ: `test/manifest.yml`) chứa cấu hình môi trường.  
  - `frontend/manifest.yml` (hoặc tương tự) chứa cấu hình dịch vụ UI.  
- Các file manifest này là nơi chỉnh sửa cấu hình (scale, env vars, network, v.v.), sau đó dùng Copilot để triển khai lại.

7. Dọn dẹp tài nguyên  
- Để xóa toàn bộ ứng dụng và tránh tốn chi phí, chạy:  
  - `copilot app delete` (hoặc `copilot delete app` tùy phiên bản).  
- Lệnh này sẽ xóa stack CloudFormation và toàn bộ tài nguyên liên quan, mất một chút thời gian.

Toàn bộ bài giảng minh họa quy trình: từ Dockerfile + Copilot manifest → Copilot CLI → CloudFormation → triển khai ứng dụng web trên AWS ECS/Fargate với Load Balancer, theo chuẩn best practice.
