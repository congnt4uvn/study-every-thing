


Dưới đây là tóm tắt nội dung:

1. Khái niệm chung  
- CodePipeline là công cụ workflow trực quan để điều phối quy trình CI/CD trên AWS.  
- Cho phép định nghĩa nhiều giai đoạn (stage) như Source, Build, Test, Deploy… với các hành động tuần tự hoặc song song.

2. Nguồn (Source)  
- Có thể lấy mã/tạo phẩm từ:  
  - AWS CodeCommit  
  - Amazon S3  
  - Amazon ECR (image Docker)  
  - Các nguồn bên ngoài: GitHub, Bitbucket…

3. Giai đoạn Build  
- Dùng để build mã nguồn.  
- Có thể dùng: CodeBuild, Jenkins, CloudBees, TeamCity, hoặc công cụ bên thứ ba khác.

4. Giai đoạn Test  
- Dùng để kiểm thử mã sau khi build.  
- Có thể dùng: CodeBuild, Device Farm (cho app iOS/Android), hoặc công cụ bên thứ ba.

5. Giai đoạn Deploy  
- Dùng để triển khai ứng dụng.  
- Hỗ trợ: CodeDeploy, Elastic Beanstalk, CloudFormation, ECS, S3…  
- Có thể dùng hành động Invoke để gọi Lambda hoặc Step Functions.

6. Ví dụ pipeline đơn giản  
- Các stage: Build → Test → Deploy lên môi trường Staging → Load test → Deploy lên Production.  
- Có thể chèn bước phê duyệt thủ công (Manual Approval) ở bất kỳ stage nào, thường là trước Production, để người chịu trách nhiệm xem kết quả test rồi “bật đèn xanh”.

7. Cách CodePipeline hoạt động bên trong  
- Mỗi pipeline tạo ra “artifact” (tạo phẩm) và lưu trong S3 để truyền giữa các stage.  
- Luồng ví dụ:
  - Dev push code lên CodeCommit.  
  - CodePipeline lấy code, tạo artifact và lưu vào S3.  
  - CodeBuild nhận artifact từ S3 (không cần quyền truy cập trực tiếp CodeCommit), build và tạo artifact triển khai.  
  - CodePipeline lưu artifact build vào S3 rồi chuyển cho CodeDeploy.  
  - CodeDeploy dùng artifact này để triển khai.

8. Giám sát & khắc phục sự cố  
- Dùng CloudWatch Events / EventBridge để theo dõi thay đổi trạng thái pipeline, stage, action (ví dụ pipeline fail, stage bị hủy) và gửi thông báo (email…).  
- Khi pipeline lỗi:
  - Xem trực quan trên console để biết stage/action nào lỗi.  
  - Kiểm tra IAM service role của CodePipeline nếu không thực hiện được hành động (gọi CodeBuild, lấy code từ CodeCommit…).  
  - Dùng CloudTrail để kiểm tra các lệnh gọi API bị từ chối hoặc lỗi trong hạ tầng.

Nội dung nhấn mạnh: CodePipeline là “bộ điều phối” kết nối các dịch vụ (CodeCommit, CodeBuild, CodeDeploy, S3, Lambda…) thông qua artifact trong S3, cho phép xây dựng quy trình CI/CD linh hoạt, có thể tự động hoàn toàn hoặc có bước phê duyệt thủ công.
