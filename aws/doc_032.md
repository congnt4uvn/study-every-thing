


Nội dung trên giới thiệu tổng quan về Amazon EC2 và EC2 User Data:

1. **EC2 là gì?**  
   - EC2 = Elastic Compute Cloud, dịch vụ hạ tầng dưới dạng dịch vụ (IaaS) của AWS.  
   - Cho phép thuê máy ảo trên đám mây, gọi là **EC2 instance**.  
   - Đây là dịch vụ rất phổ biến, nền tảng để hiểu cách hoạt động của cloud.

2. **Những thành phần chính liên quan đến EC2**  
   - **EC2 instances**: các máy chủ ảo.  
   - **EBS (Elastic Block Store)**: ổ đĩa/ổ cứng ảo gắn qua mạng.  
   - **Instance store**: lưu trữ gắn trực tiếp vào phần cứng.  
   - **Elastic Load Balancer (ELB)**: phân phối tải giữa nhiều máy.  
   - **Auto Scaling Group (ASG)**: tự động tăng/giảm số lượng instance.  
   - **Networking**: loại card mạng, IP công cộng, cấu hình mạng.  
   - **Security Groups**: “tường lửa” cho instance, quy định cho phép/ chặn traffic.

3. **Những lựa chọn khi tạo một EC2 instance**  
   - **Hệ điều hành**: Linux, Windows, hoặc macOS.  
   - **CPU**: số lõi, sức mạnh tính toán.  
   - **RAM**: dung lượng bộ nhớ.  
   - **Storage**:  
     - Lưu trữ qua mạng: EBS, EFS.  
     - Lưu trữ gắn phần cứng: instance store.  
   - **Mạng**: loại card mạng, IP, cấu hình kết nối.  
   - **Security Group**: quy tắc firewall.  
   - **User Data**: script khởi động để cấu hình máy khi mới tạo.

4. **EC2 User Data & Bootstrapping**  
   - **Bootstrapping**: chạy các lệnh khi máy khởi động lần đầu.  
   - **EC2 User Data Script**:  
     - Chỉ chạy **một lần** khi instance khởi động lần đầu.  
     - Dùng để **tự động hóa** các tác vụ khởi động:  
       - Cài đặt cập nhật.  
       - Cài phần mềm.  
       - Tải file từ Internet.  
       - Bất kỳ lệnh cấu hình nào khác.  
     - Chạy với quyền **root**, nên mọi lệnh đều có quyền sudo.  
   - Lưu ý: càng thêm nhiều lệnh vào User Data, instance khởi động càng lâu.

5. **Ý nghĩa**  
   - EC2 cho phép bạn “thuê” tài nguyên tính toán theo nhu cầu, rất nhanh chóng.  
   - Việc hiểu và sử dụng EC2 là nền tảng để làm việc với AWS và cloud nói chung.
