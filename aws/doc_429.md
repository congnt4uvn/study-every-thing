


Dưới đây là tóm tắt nội dung bài giảng về bảo mật trong CodeBuild:

1. **CodeBuild và VPC**  
   - Mặc định CodeBuild chạy ngoài VPC, nhưng bạn có thể cấu hình để chạy **bên trong VPC** (chọn subnet, security group) để truy cập tài nguyên nội bộ như RDS, EC2, v.v.

2. **Bí mật (secrets) trong CodeBuild**  
   - **Không** lưu mật khẩu/bí mật dưới dạng **plain text** trong environment variables (ví dụ: `DB_PASSWORD = "supersecret"`), vì dễ bị lộ.
   - Thay vào đó, có 2 cách an toàn:
     1. **SSM Parameter Store**
     2. **AWS Secrets Manager**

3. **Dùng SSM Parameter Store với CodeBuild**  
   - Vào **SSM Parameter Store**, tạo tham số, ví dụ:  
     - Tên: `/CodeBuild/DBPassword`  
     - Loại: `SecureString`  
     - KMS key: chọn CMK để mã hóa  
     - Giá trị: `"SuperSecret"`
   - Trong cấu hình CodeBuild:
     - Tạo environment variable `DB_PASSWORD`
     - Chọn loại là **Parameter Store** và nhập tên tham số `/CodeBuild/DBPassword`.
   - Khi chạy build, CodeBuild sẽ:
     - Tự động lấy giá trị tham số
     - Giải mã và inject vào container build dưới dạng biến môi trường.

4. **Dùng AWS Secrets Manager**  
   - Tương tự Parameter Store:
     - Tạo secret trong Secrets Manager.
     - Trong CodeBuild, tạo environment variable (ví dụ `DB_PASSWORD_ALT`) và chọn loại **Secrets Manager**, tham chiếu tên secret.
   - CodeBuild sẽ lấy secret và đưa vào container build.

5. **Quyền IAM**  
   - Role IAM gắn với dự án CodeBuild phải có quyền:
     - Đọc tham số từ **SSM Parameter Store**
     - Đọc secret từ **Secrets Manager**
   - Nếu không, CodeBuild sẽ không truy cập được các bí mật này.

Tóm lại:  
- Chạy CodeBuild trong VPC để truy cập tài nguyên nội bộ.  
- Không lưu bí mật dạng plain text trong environment variables.  
- Sử dụng SSM Parameter Store hoặc Secrets Manager + IAM role phù hợp để quản lý và inject secrets an toàn vào CodeBuild.
