


Dưới đây là tóm tắt ngắn gọn nội dung:

1. Mục tiêu  
- Sử dụng CloudFormation để tự động tạo hàm AWS Lambda (có bật AWS X-Ray) từ một file zip chứa mã nguồn được lưu trong S3.

2. Cấu trúc template CloudFormation  
- Có 3 tham số:
  - S3Bucket: tên bucket chứa file zip.
  - S3Key: đường dẫn/tên file zip (function.zip).
  - S3ObjectVersion: phiên bản đối tượng trong S3 (do bucket bật versioning).
- Tài nguyên chính:
  - IAM Role (Lambda Execution Role):
    - Cho phép Lambda assume role.
    - Policy cho phép:
      - Ghi log lên CloudWatch Logs.
      - Gửi trace tới AWS X-Ray.
      - Đọc dữ liệu từ S3 (Get*, List*).
  - Lambda Function:
    - Handler: index.handler.
    - Runtime: Node.js 14.x.
    - Timeout: 10 giây.
    - Code lấy từ S3 (bucket, key, object version từ 3 tham số).
    - Bật X-Ray bằng cách bật chế độ “Active tracing” trong cấu hình.

3. Chuẩn bị S3 và upload code  
- Tạo một S3 bucket mới (ví dụ: s3-cloudformation-lambda-demos-stephane) trong cùng region với Lambda.
- Bật versioning cho bucket.
- Tạo file function.zip (từ thư mục lambda, ví dụ trong CloudShell).
- Upload function.zip lên bucket S3.
- Lấy:
  - Tên bucket.
  - Tên file (function.zip).
  - Object version (từ tab Versions của đối tượng trong S3).

4. Tạo stack CloudFormation  
- Mở AWS CloudFormation console → Create stack.
- Upload template lambda-xray.yaml.
- Đặt tên stack (ví dụ: demo-lambda-cf).
- Nhập 3 tham số:
  - S3 bucket name.
  - S3 key: function.zip.
  - S3 object version: giá trị copy từ S3.
- Xác nhận CloudFormation được phép tạo IAM Role.
- Tạo stack và chờ trạng thái CREATE_COMPLETE.

5. Kiểm tra Lambda và X-Ray  
- Vào AWS Lambda console:
  - Thấy hàm mới được tạo bởi CloudFormation.
  - Console hiển thị thông báo hàm thuộc về một “application” (do tích hợp với CloudFormation).
  - Kiểm tra code đã được upload đúng.
  - Tạo test event đơn giản và chạy thử:
    - Hàm có quyền cần thiết (CloudWatch Logs, S3, X-Ray) nên chạy thành công, ví dụ trả về danh sách bucket.
- Vào AWS X-Ray console:
  - Thấy các trace mới được tạo nhờ active tracing.

6. Dọn dẹp  
- Nếu không cần nữa, xóa stack CloudFormation:
  - Tự động xóa Lambda function và IAM role liên quan.
- Như vậy toàn bộ cấu hình Lambda (code, role, tracing) được quản lý tập trung bằng CloudFormation.
