


Dưới đây là tóm tắt nội dung:

1. Tạo Kinesis Data Stream  
   - Mở dịch vụ Kinesis, chọn “Data Streams” (không dùng Firehose hay Analytics).  
   - Đặt tên stream là `DemoStream`.  
   - Chọn chế độ:  
     - On-demand: tự động scale, tối đa 200 MB/s ghi, 200k records/s, 400 MB/s đọc với Enhanced Fan-Out, tính phí theo thông lượng, không có free tier.  
     - Provisioned: phải chọn số shard, cũng không có free tier.  
   - Ví dụ chọn 1 shard (1 MB/s ghi, 2 MB/s đọc). Nhiều shard hơn thì thông lượng nhân lên tương ứng.  
   - Tạo stream và chờ trạng thái “ACTIVE”.

2. Thông tin về producer và consumer  
   - Producer: có 3 lựa chọn chính  
     - Kinesis Agent  
     - AWS SDK (mức thấp)  
     - Kinesis Producer Library (KPL – mức cao, API tiện hơn)  
   - Consumer:  
     - Kinesis Data Analytics  
     - Kinesis Data Firehose  
     - Kinesis Client Library (KCL)  
     - AWS Lambda (dù không hiện trong UI demo)  

3. Giám sát và cấu hình stream  
   - Có thể xem metric (số bản ghi, thông lượng) trong CloudWatch.  
   - Có thể thay đổi số shard để scale (ví dụ từ 1 lên 5).  
   - Có thể cấu hình Enhanced Fan-Out cho các consumer.  
   - Demo giữ đơn giản: chỉ đọc/ghi bằng SDK/CLI.

4. Sử dụng AWS CloudShell  
   - Mở CloudShell (biểu tượng cạnh cái chuông).  
   - CloudShell miễn phí, đã cài sẵn AWS CLI và dùng sẵn IAM credentials + region của bạn.  
   - Kiểm tra phiên bản CLI bằng `aws --version` (trong demo là AWS CLI v2.1.16).  
   - Có hai loại lệnh tùy CLI v1 hay v2; ở đây dùng CLI v2.

5. Ghi dữ liệu vào Kinesis (put-record)  
   - Dùng API `put-record`:  
     - `--stream-name DemoStream`  
     - `--partition-key user1` (cùng partition key → cùng shard; ở đây chỉ có 1 shard nên không khác biệt).  
     - `--data "user signup"` (dữ liệu text).  
     - Thêm `--cli-binary-format raw-in-base64-out` khi dùng CLI v2 với dữ liệu text.  
   - Sau khi chạy, CLI trả về `ShardId` (ví dụ `shardId-000000000000`) và `SequenceNumber`.  
   - Có thể gửi nhiều bản ghi: user signup, login, logout, v.v.

6. Mô tả stream để lấy thông tin shard  
   - Dùng `describe-stream` (hoặc `describe-stream-summary`) với `DemoStream`.  
   - Kết quả có `StreamDescription` và danh sách shard, ví dụ `shardId-000000000000`.  
   - Khi dùng CLI/SDK mức thấp, phải biết shard ID để đọc dữ liệu.  
   - Nếu dùng KCL thì thư viện tự xử lý việc chia shard.

7. Lấy ShardIterator để đọc dữ liệu  
   - Dùng `get-shard-iterator` với:  
     - `--stream-name DemoStream`  
     - `--shard-id shardId-000000000000`  
     - `--shard-iterator-type TRIM_HORIZON` (đọc từ đầu stream).  
   - Có thể dùng loại iterator khác để chỉ đọc từ “bây giờ trở đi”.

8. Đọc bản ghi từ stream  
   - Dùng `get-records --shard-iterator <ShardIterator>`  
   - Kết quả:  
     - Mảng `Records` chứa dữ liệu:  
       - `Data` (base64-encoded)  
       - `PartitionKey`  
       - `ApproximateArrivalTimestamp`  
     - `NextShardIterator` để lần đọc tiếp theo.  
   - Dữ liệu `Data` là base64, có thể copy lên một trang web decode base64 để xem nội dung text (ví dụ “user signup”, “user login”).  
   - Mỗi lần đọc tiếp, dùng `NextShardIterator` để tiếp tục từ vị trí đã dừng.

9. Ghi chú về chế độ tiêu thụ  
   - Cách đọc trong demo là “general purpose” (không dùng Enhanced Fan-Out).  
   - Với ứng dụng thực tế, nên dùng Kinesis Client Library (KCL) hoặc Kinesis Consumer Library để:  
     - Tự quản lý shard iterator, checkpoint, scaling, v.v.  
   - Ở đây chỉ minh họa cách làm mức thấp bằng CLI.

10. Kết thúc demo  
   - Đã demo:  
     - Tạo Kinesis Data Stream.  
     - Ghi dữ liệu vào stream bằng CLI.  
     - Đọc dữ liệu từ stream bằng CLI.  
     - Dùng CloudShell để thao tác.  
   - Giữ stream lại để dùng tiếp trong phần demo Kinesis Data Firehose ở bài sau.
