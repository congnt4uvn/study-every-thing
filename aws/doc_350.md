


Dưới đây là tóm tắt nội dung:

- API Gateway hỗ trợ CORS (Cross-Origin Resource Sharing) để cho phép chia sẻ tài nguyên giữa các domain khác nhau một cách an toàn trên trình duyệt.  
- CORS phải được bật nếu bạn muốn gọi API từ một domain khác (cross-origin).  
- Khi bật CORS, trình duyệt sẽ gửi một yêu cầu “preflight” (OPTIONS) trước khi gửi yêu cầu chính.  
- Yêu cầu preflight này chứa các header CORS như:
  - `Access-Control-Allow-Methods`
  - `Access-Control-Allow-Headers`
  - `Access-Control-Allow-Origin`
  … Các giá trị này có thể cấu hình trong console của API Gateway.  
- Ví dụ:  
  - Trình duyệt tải trang tĩnh từ S3 (vd: www.example.com).  
  - JavaScript trên trang đó gọi API ở domain khác (vd: api.example.com).  
  - Trình duyệt gửi yêu cầu OPTIONS (preflight) đến API Gateway.  
  - API Gateway trả về phản hồi CORS cho biết origin đó có được phép hay không.  
  - Nếu hợp lệ, trình duyệt mới gửi yêu cầu API thực sự và giao tiếp diễn ra bình thường.  
- Về phía kỳ thi: chỉ cần nhớ rằng CORS có thể (và cần) được bật trên API Gateway khi bạn thực hiện các yêu cầu cross-origin.
