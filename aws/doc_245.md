


Dưới đây là tóm tắt nội dung về CloudWatch Synthetics Canary:

- CloudWatch Synthetics Canary cho phép bạn chạy các script (Node.js hoặc Python) từ CloudWatch để giám sát API, URL hoặc website.  
- Script mô phỏng lại hành vi người dùng (ví dụ: vào trang sản phẩm, thêm vào giỏ hàng, thanh toán…) để kiểm tra toàn bộ luồng nghiệp vụ.  
- Nếu script thất bại, nghĩa là có vấn đề trong ứng dụng; mục tiêu là phát hiện lỗi trước khi khách hàng gặp phải.  
- Có thể kiểm tra:
  - Tính khả dụng (availability) và độ trễ (latency) của endpoint  
  - Thời gian tải  
  - Chụp ảnh màn hình giao diện người dùng  
- Ví dụ kiến trúc:
  - Ứng dụng chạy ở us-east-1  
  - Canary giám sát ứng dụng  
  - Khi canary thất bại → CloudWatch Alarm kích hoạt → gọi Lambda  
  - Lambda cập nhật bản ghi Route 53 sang phiên bản ứng dụng khác ở us-west-2 để chuyển hướng sang vùng đang hoạt động tốt.  
- Canary có thể:
  - Chạy một lần hoặc theo lịch (định kỳ)  
  - Sử dụng trình duyệt Google Chrome headless, nên có thể thực hiện hầu hết thao tác như người dùng thật.  
- AWS cung cấp nhiều “blueprint” (mẫu có sẵn):
  - Heartbeat Monitor: tải URL, lưu screenshot, lưu HTTP archive, kiểm tra hoạt động bình thường  
  - API Canary: kiểm tra các chức năng đọc/ghi cơ bản của REST API  
  - Broken Link Checker: kiểm tra các liên kết trong một URL để phát hiện link hỏng  
  - Visual Monitoring: so sánh screenshot hiện tại với screenshot chuẩn (baseline)  
  - Canary Recorder: ghi lại thao tác người dùng trên website, tự động tạo script để chạy lại trên Canary  
  - GUI Workflow Builder: tạo các luồng thao tác GUI (ví dụ: kiểm tra form đăng nhập hoạt động đúng).  

Tóm lại, CloudWatch Synthetics Canary là công cụ mô phỏng người dùng để giám sát chủ động ứng dụng/web/API, phát hiện lỗi sớm và hỗ trợ tự động hóa kịch bản giám sát.
