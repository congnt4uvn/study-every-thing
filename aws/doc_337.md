


Dưới đây là tóm tắt nội dung:

1. **Tạo REST API trên API Gateway**
   - Mở API Gateway console, chọn loại API là **REST API** (không phải HTTP/WebSocket).
   - Chọn kiểu endpoint là **Regional** (thay vì Edge-Optimized hoặc Private).
   - Tạo API mới với tên ví dụ: **MyFirstAPI**.

2. **Tạo hàm Lambda đầu tiên**
   - Vào dịch vụ **Lambda**, tạo hàm mới (ví dụ: `api-gateway-route-gets`), runtime **Python 3.x**.
   - Dán mã nguồn từ file `lambda-code.py`:
     - Trả về JSON với:
       - `statusCode: 200`
       - `body: "Hello from Lambda"`
       - Header `Content-Type: application/json`
   - Deploy hàm, tạo event test và chạy thử để xác nhận kết quả.

3. **Tích hợp Lambda với API Gateway**
   - Quay lại API Gateway, tạo **Method** mới cho resource `/`:
     - Chọn **GET**.
     - Chọn loại tích hợp là **Lambda Function**.
     - Dán ARN của hàm Lambda vừa tạo.
     - Bật **Lambda Proxy Integration** để toàn bộ request/response đi thẳng giữa API Gateway và Lambda.
   - Lưu ý: **Timeout của API Gateway tối đa ~29 giây**, dù Lambda có thể chạy lâu hơn.
   - API Gateway tự động thêm quyền vào **resource-based policy** của Lambda để cho phép API Gateway gọi hàm.

4. **Kiểm thử từ API Gateway**
   - Dùng chức năng **Test** trong API Gateway:
     - Gửi request GET.
     - Nhận response `"Hello from Lambda"`, status 200, content-type JSON.
   - Có thể xem **execution logs** của API Gateway để debug.

5. **Xem dữ liệu request trong Lambda**
   - Sửa code Lambda để `print(event)`.
   - Deploy lại, gọi lại qua API Gateway.
   - Vào **CloudWatch Logs** của Lambda để xem event:
     - Thấy các thông tin như `resource`, `path`, `httpMethod`, `headers`, `queryStringParameters`, v.v.
   - Điều này cho thấy toàn bộ request từ API Gateway được truyền vào Lambda.

6. **Tạo resource và Lambda thứ hai**
   - Trong API Gateway, tạo resource mới: **`/houses`**.
   - Tạo method **GET** cho `/houses`, tích hợp kiểu Lambda Proxy với một hàm Lambda mới.
   - Tạo hàm Lambda thứ hai (ví dụ: `houses`), runtime Python:
     - Trả về `"Hello from my beautiful house"` với status 200, content-type JSON.
   - Tích hợp ARN của hàm này vào method GET `/houses`.
   - Test trong API Gateway: nhận `"Hello from my beautiful house"`.

7. **Triển khai (Deploy) API**
   - Chọn **Deploy API**, tạo stage mới tên **`dev`**.
   - Sau khi deploy, API Gateway cung cấp một **Invoke URL**:
     - Ví dụ: `https://<id>.execute-api.<region>.amazonaws.com/dev`
   - Truy cập bằng trình duyệt:
     - `.../dev` → trả về `"Hello from Lambda"`.
     - `.../dev/houses` → trả về `"Hello from my beautiful house"`.
   - Nếu truy cập sai đường dẫn hoặc thiếu thông tin, có thể nhận lỗi như `"Missing Authentication Token"`.

8. **Kết luận**
   - Đã xây dựng một REST API trên API Gateway:
     - Root `/` map tới một Lambda.
     - Resource `/houses` map tới một Lambda khác.
   - Minh họa cách:
     - Tạo API, resource, method.
     - Tích hợp với Lambda (Lambda Proxy).
     - Thiết lập quyền, test, xem log, và deploy để truy cập từ trình duyệt.
