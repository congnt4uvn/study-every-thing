


Dưới đây là tóm tắt ngắn gọn nội dung bạn đưa ra:

1. **Vị trí mặc định của Lambda**
   - Mặc định, Lambda chạy *ngoài* VPC của bạn (trong VPC do AWS quản lý).
   - Vì vậy, Lambda mặc định:
     - Truy cập được internet công cộng, API bên ngoài.
     - Truy cập được các dịch vụ AWS công khai như DynamoDB.
     - **Không** truy cập được tài nguyên nằm trong VPC của bạn (RDS, ElastiCache, internal ALB, EC2 private, …).

2. **Cho Lambda vào VPC để truy cập tài nguyên private**
   - Bạn có thể cấu hình Lambda chạy trong VPC:
     - Chỉ định: VPC ID, subnet(s), security group(s).
   - Khi đó, Lambda sẽ:
     - Tự động tạo ENI (Elastic Network Interface) trong các subnet đã chọn.
     - Cần IAM role có quyền `AWSLambdaVPCAccessExecutionRole`.
   - Để Lambda truy cập RDS:
     - Lambda dùng ENI + security group của Lambda.
     - Security group của RDS phải cho phép inbound từ security group của Lambda (giống EC2).

3. **Lambda trong VPC và truy cập Internet**
   - Khi Lambda chạy *trong* VPC:
     - **Mặc định không có** truy cập internet công cộng.
   - Đặt Lambda vào **public subnet** cũng **không** làm nó có IP public hay truy cập internet (khác với EC2).
   - Để Lambda trong VPC truy cập internet:
     - Đặt Lambda trong **private subnet**.
     - Dùng **NAT Gateway** hoặc **NAT Instance** trong public subnet.
     - Route từ private subnet → NAT → Internet Gateway → internet.

4. **Truy cập DynamoDB từ Lambda trong VPC**
   - Cách 1: Qua internet
     - Lambda (private subnet) → NAT → Internet Gateway → DynamoDB endpoint public.
   - Cách 2: Truy cập riêng tư (không qua internet)
     - Tạo **VPC Endpoint** cho DynamoDB.
     - Lambda → VPC Endpoint → DynamoDB (trong mạng AWS, không cần NAT/IGW).

5. **CloudWatch Logs**
   - Dù Lambda ở private subnet và không có NAT/VPC Endpoint, **CloudWatch Logs vẫn hoạt động**.
   - Nghĩa là log của Lambda vẫn được gửi lên CloudWatch mà không cần bạn cấu hình thêm NAT hay endpoint.

Tóm lại:  
- Lambda mặc định: ra internet được, không vào VPC bạn được.  
- Muốn vào VPC (RDS, ElastiCache, …): gắn Lambda vào VPC, subnet, security group, ENI.  
- Khi đã vào VPC: mất quyền truy cập internet, muốn có lại phải dùng NAT.  
- Truy cập DynamoDB có thể qua NAT (public) hoặc VPC Endpoint (private).  
- CloudWatch Logs vẫn gửi được ngay cả khi Lambda ở private subnet không có NAT.
