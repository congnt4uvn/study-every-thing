


Nội dung trên là một đoạn bài giảng về việc dùng AWS SAM để triển khai một API serverless có tích hợp DynamoDB và cách chạy, test cục bộ. Tóm tắt:

1. Khởi tạo project với SAM  
   - Chạy `sam init`.  
   - Chọn mẫu “Serverless API” (tùy chọn số 7).  
   - Chọn runtime `nodejs22`.  
   - Tạo ứng dụng, ví dụ tên `sam-app-dynamodb`.  
   - Cấu trúc thư mục tương tự ví dụ trước, có thêm mã nguồn JavaScript.

2. File template.yaml (SAM template)  
   - Sử dụng `Transform: AWS::Serverless-2016-10-31`.  
   - Định nghĩa nhiều `AWS::Serverless::Function`:
     - Ví dụ: `getAllItemsFunction`, các hàm khác cho GET/POST.  
     - Mỗi hàm có: handler, runtime, memory, policies, events (API).  
   - Policy:
     - Dùng policy DynamoDB đơn giản (ví dụ `DynamoDBCrudPolicy`) tham chiếu đến tên bảng.  
     - SAM giúp tạo IAM policy dễ hơn nhiều so với CloudFormation thuần.
   - Events:
     - Định nghĩa các endpoint API rất đơn giản, ví dụ:
       - GET trên path `/` cho hàm lấy dữ liệu.  
       - POST cho hàm thêm dữ liệu, v.v.

3. Định nghĩa bảng DynamoDB trong template  
   - Có resource bảng DynamoDB với:
     - Khóa chính (partition key).  
     - Provisioned throughput: `ReadCapacityUnits` và `WriteCapacityUnits` là 2.  
   - Chỉ cần vậy là có thể triển khai một API hoàn chỉnh lên AWS, được backup bởi DynamoDB.

4. Khả năng chạy và test cục bộ với SAM  
   - Trong thư mục dự án có thư mục `events` chứa các file event mẫu.  
   - Có thể dùng:
     - `sam local invoke` để gọi từng hàm Lambda cục bộ với event mẫu.  
     - `sam local start-api` để khởi động API Gateway cục bộ.  
   - Tuy nhiên, trên Cloud Shell của giảng viên không chạy được vì thiếu Docker và các công cụ cần thiết.  
   - Người học có thể xem file `README.md` để biết lệnh cụ thể, tự chạy trên máy cá nhân:
     - Gọi hàm `putItem` với event mẫu.  
     - Gọi `getAllItemsFunction` với event mẫu để test.

5. Kết luận  
   - Ví dụ này minh họa:
     - Cách thiết lập DynamoDB bằng SAM.  
     - Cách định nghĩa API serverless đơn giản.  
     - Cách test Lambda/API cục bộ để lặp nhanh, sau đó mới deploy lên AWS.  
   - Giảng viên nhắc nếu triển khai thật thì nhớ xóa bằng `sam delete`.  
   - Kết thúc bài giảng và hẹn ở bài tiếp theo.
