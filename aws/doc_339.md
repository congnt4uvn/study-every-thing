


Người hướng dẫn đang minh họa cách dùng **Lambda versions + aliases** kết hợp với **API Gateway stage variables**.

Tóm tắt các bước chính:

1. **Tạo Lambda function và các version**
   - Tạo hàm Lambda `api-gateway-stage-variables-get` (Python 3).
   - Viết code lần lượt:
     - Version 1: trả về `"Hello from Lambda v1"`.
     - Version 2: cập nhật code trả về `"Hello from Lambda v2"`.
     - Bản mới nhất (chưa publish version): trả về `"Hello from Lambda DEV"`.
   - Publish:
     - v1: chứa `"Hello from Lambda v1"`.
     - v2: chứa `"Hello from Lambda v2"`.
     - Bản DEV là code mới nhất, không publish thành version.

2. **Tạo các alias cho Lambda**
   - Alias `DEV` → trỏ tới **$LATEST** (bản DEV).
   - Alias `TEST` → trỏ tới **version 2**.
   - Alias `PROD` → trỏ tới **version 1**.
   - Kiểm tra:
     - Gọi alias PROD → `"Hello from Lambda v1"`.
     - Gọi alias TEST → `"Hello from Lambda v2"`.
     - Gọi alias DEV → `"Hello from Lambda DEV"`.

3. **Kết nối Lambda với API Gateway**
   - Trong API Gateway, tạo method GET, integration type là Lambda proxy.
   - Dán ARN Lambda và thêm phần alias bằng stage variable:
     - `...:${stageVariables.lambdaAlias}`
   - API Gateway cảnh báo cần cấp quyền cho Lambda alias.
   - Sử dụng CloudShell chạy lệnh `aws lambda add-permission` (sửa lỗi ARN bị lặp, thêm DEV/TEST/PROD vào ARN) để:
     - Cho phép API Gateway gọi alias DEV.
     - Cho phép API Gateway gọi alias TEST.
     - Cho phép API Gateway gọi alias PROD.
   - Kiểm tra trong phần Permissions của alias PROD thấy policy resource-based đã được thêm.

4. **Kiểm tra bằng Test trong API Gateway**
   - Trong tab Test của method, có field `lambdaAlias` (stage variable).
   - Nhập:
     - `PROD` → trả về `"Hello from Lambda v1"`.
     - `TEST` → trả về `"Hello from Lambda v2"`.
     - `DEV` → trả về `"Hello from Lambda DEV"`.
   - Chứng minh stage variable đang điều khiển alias Lambda được gọi.

5. **Tạo các stage và gán stage variables**
   - Deploy API vào các stage:
     - Stage `DEV`:
       - Stage variable: `lambdaAlias = DEV`.
     - Stage `TEST`:
       - Stage variable: `lambdaAlias = TEST`.
     - Stage `PROD`:
       - Stage variable: `lambdaAlias = PROD`.
   - Mỗi stage có URL riêng, ví dụ:
     - `/DEV/stage-variables`
     - `/TEST/stage-variables`
     - `/PROD/stage-variables`

6. **Kiểm tra qua URL**
   - Gọi URL với stage:
     - `/prod/stage-variables` → `"Hello from Lambda v1"` (PROD → v1).
     - `/dev/stage-variables` → `"Hello from Lambda DEV"` (DEV → latest).
     - `/test/stage-variables` → `"Hello from Lambda v2"` (TEST → v2).

Ý chính:  
Người hướng dẫn cho thấy cách dùng **stage variables** trong API Gateway để ánh xạ mỗi stage (DEV/TEST/PROD) tới **một Lambda alias khác nhau**, từ đó mỗi môi trường gọi đúng version/alias của Lambda mà không cần tạo nhiều API hay nhiều hàm Lambda riêng.
