


Người tường thuật giải thích về việc dùng CloudWatch Agent để thu thập log và metric từ EC2 (và cả máy chủ on‑premise) đưa vào CloudWatch:

1. **Mặc định EC2 không tự gửi log lên CloudWatch**  
   - Muốn gửi log, phải cài và chạy **CloudWatch Agent** trên EC2.  
   - Agent sẽ đọc các file log bạn chỉ định và đẩy lên **CloudWatch Logs**.  
   - EC2 phải gán **IAM Role** cho phép gửi log lên CloudWatch Logs.  
   - Agent cũng có thể cài trên máy chủ on‑premise (VD: VMware), log vẫn lên CloudWatch Logs như bình thường.

2. **Hai loại Agent trong CloudWatch**  
   - **CloudWatch Logs Agent (cũ)**  
     - Chỉ gửi **log** lên CloudWatch Logs.  
   - **CloudWatch Unified Agent (mới)**  
     - Gửi **log** lên CloudWatch Logs.  
     - Đồng thời thu thập thêm **metric cấp hệ thống** (CPU, RAM, disk, network, process, swap…).  
     - Có thể cấu hình tập trung qua **SSM Parameter Store**, điều mà agent cũ không làm được.  
     - Vì gom cả log và metric nên gọi là **Unified Agent**.

3. **Metric mà CloudWatch Unified Agent thu thập**  
   - **CPU**: chi tiết hơn mặc định (active, guest, idle, system, user, steal, …).  
   - **Disk**: tổng dung lượng, dung lượng còn trống.  
   - **Disk I/O**: số lần đọc/ghi, byte đọc/ghi, IOPS.  
   - **RAM**: free, inactive, used, total, cached.  
   - **Netstats**: số kết nối TCP/UDP, số packet, byte mạng.  
   - **Process**: tổng số process theo trạng thái (dead, zombie, running, sleeping, …).  
   - **Swap**: dung lượng swap dùng/nhàn rỗi, phần trăm sử dụng.

4. **So sánh với metric mặc định của EC2**  
   - Metric mặc định của EC2 chỉ cho thông tin **CPU, disk, network** ở mức tổng quan.  
   - Không có chi tiết về **memory** và **swap**.  
   - Nếu cần metric chi tiết hơn, nên dùng **CloudWatch Unified Agent**.

Kết luận: CloudWatch Unified Agent là phiên bản mới, mạnh hơn, vừa gửi log vừa thu thập metric chi tiết, hỗ trợ cấu hình tập trung qua SSM, phù hợp khi cần giám sát sâu hơn so với metric mặc định của EC2.
