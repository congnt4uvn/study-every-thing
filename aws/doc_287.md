


Dưới đây là tóm tắt ngắn gọn nội dung:

1. Khái niệm “tùy chỉnh tại Edge”  
   - Khi dùng CloudFront, nội dung được phân phối từ các Edge Location gần người dùng.  
   - Nhiều ứng dụng hiện đại cần chạy một phần logic ngay tại Edge để giảm độ trễ.  
   - Phần logic này được triển khai dưới dạng “Edge Functions” gắn với CloudFront Distribution.

2. Hai loại Edge Functions của CloudFront  
   - CloudFront Functions  
   - Lambda@Edge  
   Mục tiêu là hiểu khi nào dùng loại nào và sự khác nhau giữa chúng.

3. Lợi ích chung của Edge Functions  
   - Không quản lý máy chủ (serverless).  
   - Được triển khai toàn cầu tại các Edge Location.  
   - Trả tiền theo mức sử dụng.  
   - Dùng để tùy chỉnh nội dung CDN từ CloudFront.

4. Các nhóm use case chính  
   - Bảo mật và quyền riêng tư.  
   - Ứng dụng web động tại Edge.  
   - Tối ưu SEO.  
   - Định tuyến thông minh đến nhiều Origin / data center.  
   - Giảm thiểu bot tại Edge.  
   - Chuyển đổi hình ảnh thời gian thực.  
   - A/B testing.  
   - Xác thực, ủy quyền người dùng, ưu tiên người dùng.  
   - Theo dõi, phân tích hành vi người dùng.

5. CloudFront Functions  
   - Là các hàm nhẹ, viết bằng JavaScript.  
   - Chỉ can thiệp vào Viewer Request và Viewer Response (tức là giữa client và CloudFront).  
   - Dùng cho các tùy chỉnh CDN quy mô rất lớn, nhạy cảm với độ trễ.  
   - Thời gian khởi động và thực thi cực nhanh (dưới 1 ms).  
   - Có thể xử lý hàng triệu request/giây.  
   - Được quản lý trực tiếp trong giao diện CloudFront.  
   - Dùng cho:  
     - Chuẩn hóa cache key.  
     - Thao tác header (thêm/sửa/xóa).  
     - Ghi lại hoặc chuyển hướng URL.  
     - Tạo và xác thực JWT để cho phép/từ chối request.

6. Lambda@Edge  
   - Viết bằng Node.js hoặc Python.  
   - Quy mô thấp hơn CloudFront Functions (khoảng hàng nghìn request/giây).  
   - Có thể can thiệp vào tất cả các giai đoạn:  
     - Viewer Request  
     - Origin Request (trước khi CloudFront gọi Origin)  
     - Origin Response (sau khi Origin trả về)  
     - Viewer Response  
   - Hàm được tạo ở region us-east-1, sau đó CloudFront tự sao chép ra toàn cầu.  
   - Thời gian thực thi dài hơn (5–10 giây), phù hợp cho logic phức tạp.  
   - Có CPU, RAM cấu hình được, có thể dùng thư viện bên thứ ba (ví dụ AWS SDK).  
   - Có thể gọi dịch vụ AWS khác hoặc dịch vụ bên ngoài (network access).  
   - Có thể truy cập hệ thống file và nội dung chi tiết của request để tùy biến sâu hơn.

7. So sánh chính  
   - Runtime:  
     - CloudFront Functions: JavaScript.  
     - Lambda@Edge: Node.js, Python.  
   - Quy mô:  
     - CloudFront Functions: rất lớn (triệu req/s).  
     - Lambda@Edge: nghìn req/s.  
   - Điểm kích hoạt:  
     - CloudFront Functions: chỉ Viewer Request/Response.  
     - Lambda@Edge: Viewer + Origin Request/Response.  
   - Thời gian thực thi:  
     - CloudFront Functions: < 1 ms, logic đơn giản.  
     - Lambda@Edge: đến 5–10 giây, logic phức tạp, nhiều tích hợp.

Tóm lại:  
- CloudFront Functions: cực nhanh, cực lớn, chỉ dùng cho viewer side, logic đơn giản.  
- Lambda@Edge: linh hoạt, mạnh, hỗ trợ nhiều giai đoạn request/response, phù hợp cho xử lý phức tạp và tích hợp với các dịch vụ khác.
