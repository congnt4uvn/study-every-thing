


Dưới đây là tóm tắt nội dung:

- Trong khóa học đến giờ chủ yếu dùng **REST API** của API Gateway, nhưng còn 2 loại khác:
  - **HTTP API**
  - **WebSocket API** (chỉ cần hiểu ở mức khái niệm cao cho kỳ thi, có thể xuất hiện).

**HTTP API**:
- Độ trễ thấp, chi phí rẻ hơn nhiều so với REST API.
- Tập trung vào **proxy**:
  - Proxy tới AWS Lambda
  - Proxy tới HTTP backend
  - Hỗ trợ tích hợp private
- Không có **data mapping** (mapping templates) như REST API.
- Hỗ trợ một số loại **ủy quyền**:
  - OIDC
  - OAuth 2.0
- Có hỗ trợ tích hợp cho **CORS**.
- Không hỗ trợ:
  - Usage plans (gói sử dụng)
  - API keys (khóa API)
- Đơn giản hơn, mới hơn, nhưng tên gọi dễ gây nhầm.

**REST API**:
- Có đầy đủ các tính năng đã học trong khóa:
  - Mapping templates, usage plans, API keys, v.v.
  - Hỗ trợ **resource policies** (chính sách tài nguyên).
- Không có **native** OpenID Connect / OAuth 2.0 như HTTP API.

**Điểm cần nhớ cho kỳ thi**:
- **HTTP API rẻ hơn nhiều** so với REST API.
- HTTP API và REST API khác nhau về:
  - Tính năng hỗ trợ (ví dụ: resource policies chỉ có ở REST API).
  - Cách ủy quyền và tích hợp.
