


Người hướng dẫn đang demo cách triển khai cân bằng tải (Application Load Balancer – ALB) cho 2 EC2 instance:

1. Khởi chạy 2 EC2 instance  
   - Chọn Amazon Linux 2, loại t2.micro.  
   - Không dùng key pair (không cần SSH, có thể dùng EC2 Instance Connect).  
   - Chọn security group “Launch Wizard 1” cho phép HTTP và SSH.  
   - Thêm user data để tự động cài web server (Hello World).  
   - Đổi tên instance thứ hai thành “My Second Instance”.  
   - Kiểm tra bằng cách copy IPv4 của từng instance, truy cập bằng trình duyệt, cả hai đều trả về “Hello World” (nội dung khác nhau ở phần cuối để phân biệt).

2. Mục tiêu: Có một URL duy nhất truy cập cả hai instance và phân phối (load balance) lưu lượng giữa chúng.

3. Giới thiệu các loại Load Balancer  
   - Application Load Balancer (ALB): cho HTTP/HTTPS.  
   - Network Load Balancer (NLB): cho TCP/UDP/TLS, hiệu năng rất cao, độ trễ thấp, xử lý hàng triệu request/s.  
   - Gateway Load Balancer: dùng cho bảo mật, IDS/IPS, firewall, phân tích lưu lượng.  
   - Classic Load Balancer: sắp bị loại bỏ nên không đề cập.

4. Tạo Application Load Balancer  
   - Đặt tên: DemoALB.  
   - Hướng ra Internet, dùng IPv4.  
   - Chọn tất cả Availability Zone để triển khai ALB.  
   - Tạo security group mới “demo-sg-load-balancer” cho ALB, cho phép HTTP (port 80) từ mọi nơi.  
   - Gán security group này cho ALB.

5. Cấu hình Listener và Target Group  
   - Listener: HTTP trên port 80.  
   - Tạo Target Group mới:  
     - Kiểu: Instances.  
     - Tên: demo-tg-alb.  
     - Giao thức: HTTP, port 80, HTTP/1.  
     - Health check giữ mặc định.  
   - Đăng ký cả 2 EC2 instance vào target group trên port 80.  
   - Liên kết listener của ALB với target group demo-tg-alb.

6. Kiểm tra hoạt động của ALB  
   - Đợi ALB ở trạng thái “active”.  
   - Copy DNS name của ALB, truy cập bằng trình duyệt.  
   - Trang “Hello World” hiển thị; khi refresh nhiều lần, nội dung thay đổi (do lần lượt đi đến 2 instance khác nhau) → chứng minh ALB đang phân phối lưu lượng.

7. Kiểm tra health check và tính sẵn sàng  
   - Vào Target Group, xem mục tiêu: cả 2 instance đều “healthy”.  
   - Dừng (stop) instance thứ nhất:  
     - Sau một thời gian, target group đánh dấu instance này là “unhealthy”/“unused”.  
     - ALB chỉ gửi lưu lượng đến instance còn chạy, nên khi refresh trang, chỉ thấy nội dung từ instance còn hoạt động.  
   - Khởi động lại instance bị dừng:  
     - Ban đầu ở trạng thái “initial”, sau đó trở lại “healthy”.  
     - ALB lại phân phối lưu lượng cho cả hai instance.

Kết luận: Bài demo minh họa cách:  
- Tạo 2 EC2 instance chạy web đơn giản.  
- Tạo Application Load Balancer, cấu hình security group, listener, target group.  
- Đăng ký instance vào target group và quan sát cách ALB tự động phân phối lưu lượng, phát hiện instance không khỏe và loại khỏi luồng, rồi thêm lại khi instance hoạt động bình thường.
