


Đoạn trên tóm tắt các khái niệm chính về AWS IAM:

- IAM User: đại diện cho người dùng thực trong công ty, có thể đăng nhập AWS Management Console bằng mật khẩu.  
- IAM Group: tập hợp nhiều user; có thể gán quyền (policy) cho group để áp dụng cho tất cả user trong group.  
- Policy: tài liệu JSON mô tả quyền truy cập; có thể gắn cho user hoặc group.  
- Role: một loại danh tính đặc biệt, thường dùng cho EC2 hoặc các dịch vụ AWS khác để chúng có quyền truy cập tài nguyên mà không cần dùng user/password.  
- Bảo mật: có thể bật MFA (xác thực đa yếu tố) và đặt chính sách mật khẩu cho user.  
- Truy cập lập trình:  
  - Dùng AWS CLI để quản lý dịch vụ qua dòng lệnh.  
  - Dùng SDK để quản lý dịch vụ qua các ngôn ngữ lập trình.  
  - Tạo Access Key để dùng với CLI hoặc SDK.  
- Giám sát & kiểm tra:  
  - Tạo IAM Credential Report để xem việc sử dụng thông tin đăng nhập.  
  - Dùng IAM Access Advisor để xem quyền nào đang được sử dụng.

Tóm lại, nội dung giới thiệu tổng quan về cách quản lý người dùng, nhóm, vai trò, quyền, bảo mật và cách truy cập AWS bằng IAM.
