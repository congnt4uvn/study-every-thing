


Người hướng dẫn minh họa cách tạo và dùng Access Key để làm việc với AWS CLI:

1. **Tạo Access Key**
   - Vào phần **Security credentials** trong tài khoản IAM user (Stephane).
   - Cuộn xuống, chọn **Create access key**.
   - AWS đưa ra các khuyến nghị:
     - Với CLI: nên dùng **CloudShell** hoặc **AWS CLI v2 + IAM Identity Center**.
   - Dù có khuyến nghị, người hướng dẫn vẫn tạo Access Key để minh họa.
   - Lưu ý: Đây là **lần duy nhất** có thể xem được Access Key và Secret Access Key.

2. **Cấu hình AWS CLI**
   - Chạy lệnh: `aws configure`.
   - Nhập:
     - Access Key ID.
     - Secret Access Key.
     - Default region (ví dụ: `eu-west-1`, người dùng chọn vùng gần mình).
     - Default output format (nhấn Enter để dùng mặc định).

3. **Kiểm tra CLI hoạt động**
   - Chạy: `aws iam list-users`.
   - CLI trả về danh sách user trong tài khoản, gồm:
     - Tên user (Stephane).
     - UserId, ARN.
     - Thời điểm tạo, thời điểm mật khẩu được dùng lần cuối.
   - Thông tin này tương tự như khi xem trên AWS Management Console.

4. **Minh họa quyền truy cập**
   - Người hướng dẫn dùng tài khoản root để:
     - Xóa user Stephane khỏi nhóm **admin**.
   - Kết quả:
     - Trên giao diện web (Console): làm mới trang sẽ báo lỗi không đủ quyền.
     - Trên CLI: chạy lại `aws iam list-users` bị **Access Denied**.
   - Kết luận: Quyền trên CLI **giống hệt** quyền trên Console; đều phụ thuộc vào IAM user/group.

5. **Khôi phục quyền**
   - Thêm lại user Stephane vào nhóm **admin** để lấy lại quyền quản trị.
   - Nhắc nhở: Đừng quên thêm lại user vào nhóm, nếu không sẽ mất quyền.

Tóm lại: Bài giảng hướng dẫn tạo Access Key, cấu hình AWS CLI, so sánh thông tin giữa CLI và Console, và minh họa rằng quyền truy cập của CLI phụ thuộc hoàn toàn vào quyền IAM giống như trên Management Console.
