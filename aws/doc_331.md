


Stephane đang giải thích các kiểu ghi (write) trong DynamoDB, tập trung vào vấn đề ghi đồng thời:

1. **Ghi đồng thời (Concurrent writes)**  
   - Có một item, hai người dùng cùng lúc cập nhật:  
     - User 1: đặt value = 1  
     - User 2: đặt value = 2  
   - Cả hai lệnh ghi đều “thành công”, nhưng thực tế chỉ giá trị cuối cùng (ví dụ 2) là còn lại, ghi sau sẽ ghi đè ghi trước.  
   - Đây là hành vi không mong muốn vì cả hai đều tưởng mình đã cập nhật thành công.

2. **Ghi có điều kiện (Conditional writes) – “khóa lạc quan” (optimistic locking)**  
   - Để tránh ghi đè, ta dùng điều kiện:  
     - User 1: đặt value = 1, **chỉ khi** value hiện tại = 0  
     - User 2: đặt value = 2, **chỉ khi** value hiện tại = 0  
   - Khi một ghi được chấp nhận (ví dụ User 1), value trở thành 1.  
   - Lúc đánh giá điều kiện cho User 2, DynamoDB thấy value không còn là 0 (mà là 1), nên ghi thứ hai **bị từ chối**.  
   - Nhờ vậy, tránh được việc hai người cùng “tưởng” mình đã cập nhật thành công.

3. **Ghi nguyên tử (Atomic writes)**  
   - Ví dụ:  
     - User 1: tăng value lên 1  
     - User 2: tăng value lên 2  
   - Cả hai ghi đều thành công, và kết quả cuối cùng là value tăng tổng cộng 3 (1 + 2).  
   - Đây là thao tác nguyên tử: các phép tăng được áp dụng một cách an toàn, không bị mất cập nhật.

4. **Ghi hàng loạt (Batch writes)**  
   - Cho phép ghi hoặc cập nhật nhiều item cùng lúc.  
   - Không liên quan trực tiếp đến vấn đề đồng thời, nhưng là một loại ghi khác trong DynamoDB.

Kết luận: Bài giảng phân biệt các kiểu ghi trong DynamoDB (ghi thường, ghi có điều kiện, ghi nguyên tử, ghi hàng loạt) và cách dùng ghi có điều kiện/khóa lạc quan để xử lý vấn đề ghi đồng thời.
