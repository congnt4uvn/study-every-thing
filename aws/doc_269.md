


Dưới đây là tóm tắt nội dung:

1. Tạo Lambda và ALB  
   - Tạo hàm Lambda tên `Lambda-alb`, runtime Python 3, với code đơn giản trả về statusCode 200 và chuỗi “Hello from Lambda”.  
   - Tạo Application Load Balancer (ALB) tên `demo-lambda-alb`, internet-facing, IPv4, triển khai trên 3 AZ.  
   - Tạo Security Group mới cho ALB (`DemoLambdaALBSG`), cho phép HTTP (port 80) từ mọi nơi (0.0.0.0/0).

2. Cấu hình Target Group cho Lambda  
   - Tạo Target Group kiểu Lambda, tên `demo-tg-lambda`.  
   - Chọn hàm Lambda `Lambda-alb` làm target.  
   - Gán Target Group này làm “default action” cho listener HTTP (port 80) của ALB.

3. Kiểm tra và thêm logging cho Lambda  
   - Test hàm Lambda trực tiếp trong Lambda console với một test event, nhận được “Hello from Lambda” và status 200.  
   - Thêm lệnh `print(event)` (ban đầu nhầm `console.log`) để log toàn bộ event vào CloudWatch Logs, giúp xem dữ liệu mà ALB gửi vào Lambda.

4. Xử lý phản hồi để hiển thị đúng trên trình duyệt  
   - Khi truy cập DNS của ALB, trình duyệt tải về file thay vì hiển thị nội dung, vì response từ Lambda chưa đúng format mà ALB yêu cầu.  
   - Tham khảo tài liệu “Using Lambda with an Application Load Balancer” của AWS để lấy cấu trúc response chuẩn:  
     ```json
     {
       "statusCode": 200,
       "statusDescription": "200 OK",
       "isBase64Encoded": false,
       "headers": {
         "Content-Type": "text/html"
       },
       "body": "<html>...Hello from Lambda...</html>"
     }
     ```  
   - Cập nhật code Lambda để trả về response theo format trên, đặt `Content-Type` là `text/html`.  
   - Sau khi deploy lại và truy cập ALB, nội dung “Hello from Lambda” hiển thị trực tiếp trên trình duyệt (do header Content-Type).

5. Xem event chi tiết từ ALB trong CloudWatch Logs  
   - Vào tab Monitoring/Logs của Lambda để xem log stream.  
   - Trong log, quan sát event JSON mà ALB gửi cho Lambda: chứa path, HTTP method, query string parameters, headers, v.v.  
   - Đây chính là “ALB request event” được mô tả trong tài liệu AWS.

6. Multi-value headers (tiêu đề đa giá trị)  
   - Trong Target Group, phần Attributes có tùy chọn bật “multi-value headers”.  
   - Khi bật, một header có thể có nhiều giá trị (ví dụ Accept, Host, User-Agent…).  
   - Nếu bật tùy chọn này, cần điều chỉnh code Lambda để xử lý và trả về headers theo dạng multi-value; nếu không dùng thì có thể tắt đi.

7. Quyền gọi Lambda từ ALB  
   - Trong phần Configuration → Permissions của Lambda, có resource-based policy cho phép ALB invoke hàm.  
   - Policy này chứa một statement cho phép service của ALB gọi hàm Lambda (đây là điều kiện cần để ALB có thể invoke Lambda).

8. Dọn dẹp tài nguyên  
   - Để cleanup, chỉ cần xóa ALB (và có thể xóa thêm Target Group, Lambda nếu không dùng nữa).

Tóm lại: Bài giảng hướng dẫn cách tích hợp AWS Lambda với Application Load Balancer, cấu hình Target Group kiểu Lambda, định dạng response đúng chuẩn để trình duyệt hiển thị nội dung HTML, xem event từ ALB trong CloudWatch Logs, và giới thiệu tùy chọn multi-value headers cùng với cơ chế quyền (resource-based policy) cho phép ALB invoke Lambda.
