


Nội dung trên nói về các tùy chọn cấu hình cho **Stage (giai đoạn)** trong Amazon API Gateway:

1. **Chỉnh sửa chi tiết Stage**  
   - Thay đổi mô tả giai đoạn.

2. **Bộ đệm (Caching)**  
   - Bật cache cho API để lưu yêu cầu/đáp ứng.  
   - Cần cấu hình thời gian sống (TTL) và các tham số liên quan.

3. **Điều chỉnh (Throttling)**  
   - Thiết lập giới hạn tốc độ (rate limit) và số lượng yêu cầu tối đa mỗi giây cho API.

4. **Tường lửa & chứng chỉ ứng dụng khách**  
   - Cấu hình firewall.  
   - Thiết lập client certificate để xác minh yêu cầu đến từ API Gateway.

5. **Ghi nhật ký và truy vết (Logging & Tracing)**  
   - Dùng CloudWatch Logs để:
     - Chỉ log lỗi, hoặc  
     - Log thông tin, hoặc  
     - Log đầy đủ request/response (phục vụ debug nhưng có thể lộ dữ liệu nhạy cảm).  
   - Bật/không bật số liệu chi tiết.  
   - Ghi nhật ký truy cập tùy chỉnh.  
   - Bật X-Ray để theo dõi (tracing) tích hợp giữa API Gateway và AWS X-Ray.

6. **Biến giai đoạn (Stage variables)**  
   - Dùng để cấu hình linh hoạt (đã được nhắc ở bài trước).

7. **Lịch sử triển khai (Deployment history)**  
   - Xem đã triển khai bao nhiêu lần vào stage đó.

8. **Lịch sử tài liệu (Documentation history)**  
   - Quản lý và xem lịch sử tài liệu API.

9. **Canary deployments**  
   - Tính năng triển khai canary (sẽ được giải thích chi tiết ở bài sau).

10. **Gắn thẻ (Tagging)**  
    - Gắn tag cho stage để quản lý, phân loại.

Bài học này chỉ giới thiệu tổng quan các tùy chọn; một số phần sẽ được giải thích chi tiết trong các bài giảng tiếp theo.
