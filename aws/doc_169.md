


Dưới đây là tóm tắt nội dung:

1. **Tự động mở rộng dịch vụ ECS (ECS Service Auto Scaling)**  
   - Có thể tăng/giảm số lượng task trong ECS Service tự động bằng **AWS Application Auto Scaling**.  
   - 3 metric chính dùng để scale:  
     - CPU Utilization (mức sử dụng CPU của ECS Service)  
     - Memory Utilization (RAM của ECS Service)  
     - ALB Request Count per Target (số request ALB trên mỗi target)

2. **Các kiểu Auto Scaling cho ECS Service**  
   - Target Tracking (theo dõi và giữ một giá trị mục tiêu cho 1 trong 3 metric trên)  
   - Step Scaling (tăng/giảm theo từng bước khi metric vượt ngưỡng)  
   - Scheduled Scaling (scale theo lịch, dùng cho các pattern tải dự đoán được)

3. **Phân biệt scale task và scale EC2**  
   - Tăng số lượng task trong ECS Service **không đồng nghĩa** với việc tăng số lượng EC2 instance trong cluster (khi dùng launch type EC2).  
   - Nếu không cấu hình auto scaling cho EC2, có thể thiếu tài nguyên để chạy thêm task.

4. **Fargate vs EC2**  
   - Với **Fargate**, không cần quản lý EC2 instance, nên việc thiết lập auto scaling cho ECS Service đơn giản hơn (serverless).  
   - Trong kỳ thi, Fargate thường được khuyến khích sử dụng.

5. **Khi dùng launch type EC2 – mở rộng EC2 backend**  
   Có 2 cách chính để scale EC2 instance trong cluster:  
   - **Auto Scaling Group (ASG)**:  
     - Scale ASG dựa trên metric như CPU Utilization.  
     - Khi CPU tăng, ASG thêm EC2 instance.  
   - **ECS Cluster Capacity Provider** (cách mới, thông minh hơn):  
     - Gắn Capacity Provider với một ASG.  
     - Khi cluster thiếu tài nguyên (CPU/RAM) để chạy task mới, Capacity Provider tự động scale ASG.  
     - Được xem là lựa chọn “thông minh” hơn so với chỉ dùng ASG thuần.

6. **Luồng hoạt động Auto Scaling của ECS Service**  
   - ECS Service A có 2 task, được cấu hình auto scaling theo CPU.  
   - Khi số lượng user tăng → CPU tăng → CloudWatch metric CPU Utilization tăng.  
   - CloudWatch Alarm kích hoạt → Application Auto Scaling thực hiện scaling action.  
   - Desired count của ECS Service tăng → task mới được tạo.  
   - Nếu dùng launch type EC2 và có ECS Capacity Provider:  
     - Khi thiếu tài nguyên để chạy task mới, Capacity Provider sẽ scale ASG → tạo thêm EC2 instance cho cluster.
