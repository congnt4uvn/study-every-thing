


Người hướng dẫn đang demo AWS Lambda và giải thích các ý chính:

1. Giới thiệu Lambda  
- Lambda là dịch vụ “serverless” của AWS, cho phép chạy code mà không cần quản lý server.  
- Hỗ trợ nhiều ngôn ngữ: .NET, Java, Node.js, Python, Ruby hoặc runtime tùy chỉnh.  
- Lambda có thể tự động mở rộng: khi có nhiều sự kiện hơn, số “instance” (bánh răng trong sơ đồ) tăng lên tương ứng.

2. Cách Lambda được kích hoạt  
- Lambda phản hồi các sự kiện từ nhiều nguồn:  
  - Dữ liệu streaming (ví dụ Kinesis)  
  - Thiết bị di động, IoT  
  - Ảnh hoặc file được đưa vào S3  
- Khi số lượng yêu cầu (invocations) tăng, chi phí cũng tăng, dù Lambda có free tier khá hào phóng.

3. Tạo một hàm Lambda (demo Hello World)  
- Sử dụng blueprint “hello-world” với runtime Python.  
- Đặt tên hàm (ví dụ HelloWorld).  
- Tạo một IAM execution role mới với quyền Lambda cơ bản.  
- AWS tự sinh sẵn đoạn code mẫu và handler (hàm được gọi khi có sự kiện).

4. Test hàm Lambda  
- Dùng giao diện “Test” trong console để gửi JSON mẫu (key1, key2, key3).  
- Hàm trả về value1 (giá trị của key1).  
- Nếu sửa JSON và xóa key1, code sẽ lỗi vì không xử lý ngoại lệ.  
- Có thể lưu lại “test event” để tái sử dụng nhiều lần.

5. Giám sát và log  
- Lambda tích hợp với CloudWatch để xem metric (số lần gọi, lỗi, thời gian chạy…).  
- Có thể mở “View CloudWatch logs” để xem log chi tiết:  
  - Giá trị input  
  - Lỗi xảy ra khi test  
- Dùng log để debug hàm.

6. Cấu hình Lambda  
- Trong phần Configuration có thể chỉnh:  
  - Bộ nhớ (memory)  
  - Thời gian timeout (thời gian tối đa hàm được phép chạy)  
  - Execution role (vai trò IAM của hàm)  
- Execution role mặc định cho phép ghi log vào CloudWatch Logs.  
- Nếu muốn truy cập S3, dịch vụ khác… thì thêm quyền vào role IAM này.

7. Quyền (Permissions) và Trigger  
- Tab Permissions hiển thị role và các quyền (ví dụ 3 hành động trên CloudWatch Logs).  
- Tab Triggers cho phép thêm nguồn sự kiện để kích hoạt Lambda:  
  - Nhiều dịch vụ AWS (S3, API Gateway, DynamoDB, v.v.)  
  - Một số nguồn đối tác  
- Ví dụ phổ biến: S3 bucket + loại sự kiện (upload file) để gọi Lambda.

Kết luận: Bài giảng giới thiệu tổng quan cách Lambda hoạt động, cách tạo, test, giám sát, cấu hình quyền và thiết lập trigger, nhằm giúp người học hiểu quy trình cơ bản từ đầu đến cuối.
