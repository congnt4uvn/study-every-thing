


Nội dung trên nói về cách đóng gói và triển khai hàm AWS Lambda khi có thêm thư viện/phụ thuộc:

1. Trong bài học trước chỉ dùng các hàm Lambda đơn giản, không có phụ thuộc ngoài.  
2. Trong thực tế, hàm Lambda thường cần thêm nhiều thư viện: X-Ray SDK, client database, các package khác…  
3. Khi đó, bạn phải:
   - Cài đặt các thư viện/phụ thuộc cùng với mã nguồn.
   - Nén tất cả (mã + thư viện) thành một file zip.
4. Cách cài phụ thuộc tùy ngôn ngữ:
   - JavaScript/Node.js: dùng NPM, thư mục `node_modules`.
   - Python: dùng `pip` với thư mục mục tiêu phù hợp.
   - Java: đóng gói các thư viện vào file `.jar` (hoặc kèm theo các jar liên quan).
5. Sau khi đóng gói:
   - Nếu file zip < 50 MB: có thể upload trực tiếp lên Lambda.
   - Nếu > 50 MB: phải upload lên Amazon S3 rồi tham chiếu từ Lambda.
6. Thư viện native (native libraries):
   - Phải được biên dịch trên môi trường Amazon Linux thì mới chạy được trên Lambda.
7. AWS SDK:
   - AWS SDK đã được tích hợp sẵn trong môi trường Lambda.
   - Nếu chỉ dùng AWS SDK, không cần tự đóng gói SDK vào trong mã.
