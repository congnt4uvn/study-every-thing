


Dưới đây là tóm tắt nội dung về AWS CodeDeploy trong đoạn trên:

1. Khái niệm chung  
- AWS CodeDeploy là dịch vụ tự động hóa việc triển khai (deploy) ứng dụng.  
- Dùng để nâng cấp ứng dụng từ phiên bản v1 lên v2.  
- Hỗ trợ triển khai lên:
  - EC2
  - On-premises servers (máy chủ tại chỗ)
  - Lambda functions
  - ECS services  

2. Tính năng chính  
- Tự động cập nhật và có khả năng rollback (khôi phục) nếu triển khai lỗi hoặc có cảnh báo.  
- Kiểm soát tốc độ triển khai (deployment configuration):  
  - AllAtOnce: cập nhật tất cả instance cùng lúc → downtime cao.  
  - HalfAtATime: cập nhật 50% số instance mỗi lần.  
  - OneAtATime: cập nhật từng instance → ít ảnh hưởng khả dụng nhất.  
  - Có thể định nghĩa tốc độ tùy chỉnh.  
- Sử dụng file `appspec.yml` để định nghĩa cách triển khai (các bước, hook, v.v.).

3. CodeDeploy với EC2 / On-premises  
- Hỗ trợ 2 kiểu deployment:
  - In-place deployment (tại chỗ)
  - Blue/Green deployment  
- Cần cài đặt CodeDeploy Agent trên các EC2/on-premises instance.  
- Agent:
  - Dừng ứng dụng, cập nhật code, khởi động lại ứng dụng.
  - Lấy bản build/bản sửa đổi ứng dụng từ S3.  
- EC2 instance cần IAM role đủ quyền truy cập S3 để tải bản ứng dụng.  

4. In-place deployment (ví dụ HalfAtATime)  
- Giả sử có 4 EC2 đang chạy v1:
  - 2 instance bị đưa vào trạng thái bảo trì, agent dừng app, cập nhật lên v2.  
  - Sau khi xong, 2 instance còn lại mới được cập nhật.  
- Không tạo instance mới, chỉ cập nhật trên chỗ.

5. Blue/Green deployment với EC2  
- Có một Application Load Balancer trỏ vào Auto Scaling Group (ASG) chạy v1.  
- CodeDeploy tạo một ASG mới chạy v2 (song song với v1).  
- Sau khi v2 sẵn sàng, ALB chuyển traffic sang ASG v2.  
- ASG v1 sau đó bị gỡ bỏ/terminate.  
- Gọi là blue/green vì có môi trường cũ (blue) và môi trường mới (green) chạy song song rồi chuyển traffic.

6. CodeDeploy với Lambda  
- CodeDeploy quản lý việc chuyển traffic giữa các phiên bản Lambda thông qua alias (ví dụ alias `PROD`).  
- Có v1 và v2 cùng tồn tại, alias PROD sẽ dần chuyển traffic từ v1 sang v2.  
- Được tích hợp sẵn trong AWS SAM.  

- Chiến lược chuyển traffic (thay đổi giá trị X – % traffic đến v2):  
  1) Linear (tuyến tính):  
     - Ví dụ `LambdaLinear10PercentEvery3Minutes`: cứ 3 phút tăng thêm 10% traffic đến v2 cho đến 100%.  
  2) Canary (hoàng yến):  
     - Ví dụ `LambdaCanary10Percent5Minutes`: 10% traffic đến v2 trong 5 phút, nếu ổn thì nhảy thẳng lên 100%.  
  3) AllAtOnce:  
     - Chuyển toàn bộ traffic từ v1 sang v2 ngay lập tức.

7. CodeDeploy với ECS  
- Dùng để triển khai định nghĩa task ECS mới.  
- Chỉ hỗ trợ kiểu Blue/Green.  
- Kiến trúc:
  - ALB trỏ vào một target group chứa các ECS task v1.  
  - CodeDeploy tạo target group mới cho v2 (từ task definition mới).  
  - ALB dần chuyển traffic từ target group v1 (blue) sang v2 (green).  
- Chiến lược chuyển traffic tương tự Lambda:
  - Linear
  - Canary
  - AllAtOnce  

Tóm lại:  
AWS CodeDeploy là dịch vụ triển khai tự động, hỗ trợ EC2/on-premises, Lambda và ECS, cho phép kiểm soát tốc độ triển khai, hỗ trợ rollback, và cung cấp nhiều chiến lược triển khai (in-place, blue/green, linear, canary, all-at-once) để giảm rủi ro khi nâng cấp ứng dụng.
