


Dưới đây là tóm tắt nội dung:

1. EBS (Elastic Block Store)  
   - Gắn vào từng EC2 instance (một-một), trừ trường hợp đặc biệt dùng Multi-Attach với io1/io2.  
   - Volume EBS bị giới hạn trong một Availability Zone (AZ): volume ở AZ1 không gắn trực tiếp cho EC2 ở AZ2.  
   - Để chuyển EBS sang AZ khác: tạo snapshot → từ snapshot tạo volume mới ở AZ khác.  
   - Với gp2: IOPS tăng theo kích thước volume.  
   - Với gp3, io1: có thể tăng IOPS độc lập với kích thước.  
   - Backup (snapshot) EBS dùng IO, có thể ảnh hưởng hiệu năng nếu chạy lúc ứng dụng đang tải cao.  
   - Root volume EBS của EC2 mặc định sẽ bị xóa khi EC2 bị terminate (có thể tắt hành vi này).

2. EFS (Elastic File System)  
   - Là hệ thống file mạng (NFS), mục tiêu là gắn cho hàng trăm EC2 trên nhiều AZ.  
   - Có các mount target ở từng AZ, nhiều instance có thể cùng chia sẻ một hệ thống file.  
   - Phù hợp cho các ứng dụng cần chia sẻ file, ví dụ WordPress.  
   - Chỉ dùng cho Linux (POSIX).  
   - Giá cao hơn EBS nhưng có các tier lưu trữ để tối ưu chi phí.

3. Instance Store  
   - Lưu trữ gắn vật lý vào host của EC2.  
   - Nếu mất instance (terminate, hỏng host), sẽ mất luôn dữ liệu trên instance store.

Tóm lại:  
- EBS: block storage, gắn cho từng instance, giới hạn trong AZ, có snapshot.  
- EFS: network file system, chia sẻ đa AZ, nhiều instance cùng dùng.  
- Instance store: lưu trữ tạm, mất instance là mất dữ liệu.
