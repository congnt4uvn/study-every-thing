


Dưới đây là tóm tắt nội dung:

1. Có 3 cách chính để truy cập AWS  
   - AWS Management Console (Bảng điều khiển quản lý): giao diện web, đăng nhập bằng username/password, có thể kèm MFA.  
   - AWS CLI (Command Line Interface): dùng lệnh trong terminal để tương tác với dịch vụ AWS.  
   - AWS SDK (Software Development Kit): thư viện cho các ngôn ngữ lập trình, dùng trong mã ứng dụng để gọi API AWS.

2. Khóa truy cập (Access Keys)  
   - Dùng cho CLI và SDK.  
   - Gồm Access Key ID (giống username) và Secret Access Key (giống password).  
   - Tạo và tải xuống từ AWS Management Console.  
   - Mỗi người dùng tự quản lý khóa của mình, không được chia sẻ cho người khác vì lý do bảo mật.

3. AWS CLI  
   - Là công cụ dòng lệnh để gọi trực tiếp các API công khai của dịch vụ AWS.  
   - Ví dụ lệnh: `aws s3 cp ...`  
   - Hữu ích để quản lý tài nguyên, viết script tự động hóa.  
   - Là mã nguồn mở, có trên GitHub.  
   - Có thể dùng CLI thay cho Console trong nhiều trường hợp.

4. AWS SDK  
   - Là bộ thư viện cho nhiều ngôn ngữ: JavaScript, Python, PHP, .NET, Ruby, Java, Go, Node.js, C++, v.v.  
   - Dùng trong code ứng dụng (không phải trong terminal) để truy cập và quản lý dịch vụ AWS theo chương trình.  
   - Có SDK cho mobile (Android, iOS) và IoT (thiết bị cảm biến, thiết bị kết nối).  
   - AWS CLI thực chất được xây dựng trên AWS SDK for Python (Boto).

5. Bài tiếp theo  
   - Sẽ thực hành thiết lập AWS CLI và xử lý Access Keys trên máy.
