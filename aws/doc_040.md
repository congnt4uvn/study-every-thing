


Dưới đây là tóm tắt nội dung:

1. Kiểm tra lệnh SSH trên Windows 10  
   - Mở PowerShell hoặc Command Prompt, gõ `ssh`.  
   - Nếu hiện thông tin trợ giúp của SSH → nghĩa là SSH đã được cài.  
   - Nếu không có → phải dùng phương pháp “vá lỗi” (cách cài SSH) đã nói ở bài trước.

2. Chuẩn bị thư mục và file `.pem`  
   - Mở PowerShell, chuyển đến thư mục chứa file `.pem` (ví dụ: `cd .\Desktop`).  
   - Kiểm tra thấy file `EC2Tutorial.pem` (file `.ppk` chỉ dùng cho PuTTY, không cần cho SSH).  
   - Đảm bảo Security Group của EC2 đã mở cổng 22 (SSH).

3. Cú pháp lệnh SSH vào EC2  
   - Dùng lệnh dạng:  
     `ssh -i EC2Tutorial.pem ec2-user@<IP_public_EC2>`  
   - Có thể dùng phím Tab để auto-complete tên file `.pem`.  
   - Khi kết nối lần đầu, SSH hỏi xác thực máy chủ, chọn “yes” để tiếp tục.  
   - Sau đó sẽ đăng nhập được vào máy EC2.

4. Xử lý lỗi quyền (permission) với file `.pem` trên Windows  
   - Nếu gặp lỗi quyền khi chạy lệnh SSH:  
     1. Chuột phải vào file `.pem` → Properties → tab Security → Advanced.  
     2. Đảm bảo Owner là chính user của bạn (nhấn “Change”, chọn user của bạn).  
     3. Tắt “inheritance” (vô hiệu hóa kế thừa quyền).  
     4. Xóa các principal khác (System, Administrators…) nếu có.  
     5. Thêm lại user của bạn, cấp “Full control”.  
   - Sau khi chỉnh xong, chỉ còn user của bạn có toàn quyền với file `.pem`.  
   - Chạy lại lệnh SSH, sẽ không còn lỗi quyền và không bị hỏi lại.

5. Dùng Command Prompt thay cho PowerShell  
   - Có thể mở Command Prompt, `cd` đến Desktop, chạy lại cùng lệnh SSH.  
   - SSH hoạt động cả trên PowerShell và Command Prompt.

6. Thoát khỏi phiên SSH  
   - Gõ `exit` hoặc dùng `Ctrl + D` để thoát khỏi máy EC2.  

Kết luận: Bài hướng dẫn cách SSH vào EC2 từ Windows 10 bằng file `.pem`, cách xử lý lỗi quyền trên file `.pem`, và xác nhận có thể dùng cả PowerShell lẫn Command Prompt để kết nối.
