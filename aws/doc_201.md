


Nội dung trên nói về phần **Outputs** trong AWS CloudFormation:

1. **Outputs là gì**  
   - Là phần *tùy chọn* trong template CloudFormation.  
   - Dùng để khai báo các giá trị đầu ra (ví dụ: VPC ID, Subnet ID, Security Group ID).  
   - Các giá trị này có thể:
     - Xem được trong AWS Console hoặc qua CLI.
     - Được *nhập* (Import) vào các stack khác.

2. **Mục đích chính**  
   - Giúp **liên kết nhiều stack** với nhau.  
   - Ví dụ:
     - Stack mạng (network stack) xuất ra VPC ID, Subnet ID.
     - Stack ứng dụng (application stack) nhập các giá trị đó để sử dụng.  
   - Đây là cách tốt để nhiều nhóm/ chuyên gia quản lý các stack riêng nhưng vẫn cộng tác được.

3. **Ví dụ cụ thể**  
   - Template 1:
     - Tạo một Security Group SSH.
     - Trong phần Outputs, tham chiếu đến Security Group này.
     - Dùng `Export` để xuất giá trị với tên (ví dụ: `SSHSecurityGroup`).
     - Tên export phải **duy nhất trong một region**.
   - Template 2:
     - Tạo một EC2 instance.
     - Dùng hàm `Fn::ImportValue` (ImportValue) để lấy Security Group ID đã export từ template 1.
     - Nhờ đó EC2 sử dụng được Security Group từ stack khác.

4. **Ràng buộc khi xóa stack**  
   - Nếu một stack đang **export** giá trị và stack khác đang **import** giá trị đó:
     - Không thể xóa stack export trước.
     - Phải xóa hoặc cập nhật các stack đang tham chiếu (import) trước.  

Tóm lại: Outputs cho phép xuất các giá trị từ một stack và nhập chúng vào stack khác, giúp tái sử dụng tài nguyên và cộng tác giữa các stack trong CloudFormation.
