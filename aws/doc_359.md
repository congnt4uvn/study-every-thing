


Dưới đây là tóm tắt nội dung:

1. Tạo repository CodeCommit  
- Mở AWS CodeCommit trong console (giao diện chung với CodeBuild, CodeDeploy, CodePipeline).  
- Tạo repository mới tên `my-nodejs-application`.  
- Sau khi tạo, màn hình hiển thị các bước kết nối (HTTPS, SSH, HTTPS GRC).  
- Lưu ý: Nếu đăng nhập bằng root account thì không thấy tùy chọn SSH; muốn dùng SSH phải dùng IAM User.

2. Khám phá giao diện CodeCommit  
- Vào repository `my-nodejs-application`.  
- Tạo file mới bằng cách upload file (chỉ upload được 1 file mỗi lần).  
- Upload file `index.html` (đã dùng trước đó với Elastic Beanstalk).  
- Điền thông tin tác giả, email, commit message (“first commit”) và commit.  
- Sau commit, repository chứa `index.html`, có commit ID, nằm trên nhánh `master` (nhánh mặc định).

3. Các chức năng chính trong CodeCommit  
- Pull requests: dùng để merge thay đổi từ nhánh khác vào nhánh chính (tính năng nâng cao, quan trọng cho kỳ thi DevOps).  
- Commits: xem danh sách commit, duyệt code tại từng thời điểm, so sánh commit.  
- Branches: hiện có nhánh `master` (default). Có thể tạo thêm nhánh như `dev`, `test`… (bài này không làm, chỉ dùng `master`).  
- Tags: không sử dụng trong phần này.

4. Settings (Cài đặt)  
- Thông tin repository: tên, ID, ARN, mô tả.  

5. Notification rules (Quy tắc thông báo)  
- Tạo rule mới, ví dụ: `DemoNotificationRule`.  
- Chọn mức độ chi tiết (Full/Basic), chọn các sự kiện sẽ kích hoạt thông báo:  
  - Comment trên commit/pull request  
  - Thay đổi trạng thái phê duyệt  
  - Pull request (tạo, cập nhật, merge, đổi trạng thái)  
  - Branches/tags (tạo, xóa, cập nhật)  
- Có thể chọn tất cả sự kiện.  
- Chọn “target” cho thông báo: SNS topic hoặc Slack chatbot.  
- Tạo SNS topic mới (ví dụ tên `codecommit-lab`) để nhận thông báo (dùng cho email alert, v.v.).  
- Sau khi tạo, có thể xem, chỉnh sửa, xóa rule trong phần Settings → Notification rules.

6. Triggers (Trình kích hoạt)  
- Tạo trigger mới, ví dụ: `DemoTrigger`.  
- Trigger tập trung vào các sự kiện “code-level”:  
  - Push lên nhánh hiện có  
  - Tạo/xóa branch  
  - Tạo/xóa tag  
- Có thể chỉ định nhánh cụ thể.  
- Chọn dịch vụ đích: SNS topic (có thể dùng lại topic vừa tạo) hoặc Lambda function.  
- Trigger sẽ gửi thông báo (hoặc gọi Lambda) khi có sự kiện tương ứng trong repository.

7. Repository tags  
- Có thể gắn tag cho repository, nhưng trong bài này không sử dụng.

8. Kết luận  
- Đã tạo repository `my-nodejs-application`, upload file `index.html`, tạo commit đầu tiên.  
- Đã cấu hình Notification rule và Trigger để gửi thông báo qua SNS khi có thay đổi trong CodeCommit.  
- Bài tiếp theo sẽ thiết lập thông tin đăng nhập để thao tác với CodeCommit bằng lập trình (CLI/Git client).
