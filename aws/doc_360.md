


Dưới đây là tóm tắt nội dung:

1. Mục tiêu  
   - Đẩy (push) mã trực tiếp từ máy local lên AWS CodeCommit bằng Git, không dùng giao diện upload file.

2. Thiết lập trên IAM  
   - Vào IAM → Users → chọn user → Security credentials.  
   - Có hai loại thông tin kết nối với CodeCommit:
     - SSH keys for AWS CodeCommit.
     - HTTPS Git credentials for AWS CodeCommit.
   - Nếu quen dùng SSH: upload public key, dùng URL SSH của repo.  
   - Nếu mới: dùng HTTPS Git credentials:
     - Tạo tối đa 2 cặp username/password Git cho CodeCommit.
     - Tải file chứa thông tin đăng nhập về máy.

3. Cài đặt Git trên máy  
   - Cài Git tùy hệ điều hành (Ubuntu/Windows/Mac).  
   - Kiểm tra bằng `git --version` để chắc chắn Git đã cài.

4. Clone repo từ CodeCommit  
   - Lấy URL HTTPS của repo từ CodeCommit (Copy HTTPS).  
   - Chạy: `git clone <HTTPS-URL>`  
   - Nhập username và password Git (từ HTTPS Git credentials IAM).  
   - Sau khi clone, có thư mục mới (ví dụ: `nodejs-app`) chứa file `index.html`.

5. Thêm file mới vào repo  
   - Sao chép các file từ thư mục khác (ví dụ `app.js`, `cron.yaml`, `index.html`, `package.json`) vào thư mục repo vừa clone.  
   - Kiểm tra: `ls` để thấy các file mới.

6. Dùng Git để theo dõi và commit  
   - Kiểm tra trạng thái: `git status` → Git báo có file mới chưa được theo dõi.  
   - Thêm file: `git add .`  
   - Kiểm tra lại: `git status` → file đã ở trạng thái staged.  
   - Commit: `git commit -m "added missing files to repository"`.

7. Push lên CodeCommit  
   - Chạy: `git push`  
   - Thay đổi được đẩy lên CodeCommit.

8. Kiểm tra trên CodeCommit  
   - Refresh trang repo trên CodeCommit:
     - Thấy các file mới đã xuất hiện.  
     - Có thể xem nội dung file (dù thực tế nên chỉnh sửa bằng code editor, không phải trực tiếp trên console).  
     - Xem lịch sử commit: thấy commit ban đầu và commit mới “added missing files to repository”.

9. Ghi nhớ  
   - CodeCommit hỗ trợ cả SSH và HTTPS.  
   - Có thể xem lại hướng dẫn kết nối trong console CodeCommit (link tới tài liệu).  
   - Quy trình chuẩn: chỉnh sửa code trên máy → dùng Git (add/commit/push) → đồng bộ với CodeCommit.
