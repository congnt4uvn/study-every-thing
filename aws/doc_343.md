


Dưới đây là tóm tắt ngắn gọn nội dung:

1. Các loại tích hợp của API Gateway  
   - **MOCK integration**:  
     - Trả về phản hồi giả, không gọi backend.  
     - Dùng cho phát triển, thử nghiệm, không dùng production.  
   - **HTTP / AWS (Lambda, SQS, …)**:  
     - API Gateway chuyển tiếp request đến backend.  
     - Có thể cấu hình **Integration Request** và **Integration Response**.  
     - Dùng **Mapping Templates** để chỉnh sửa request/response (đổi tên, sắp xếp, thêm/bớt dữ liệu).  
   - **Lambda Proxy (AWS Proxy)**:  
     - Request từ client được chuyển nguyên vẹn vào Lambda (event).  
     - Không dùng được mapping template để sửa request/response.  
     - Lambda chịu trách nhiệm hoàn toàn xử lý logic và trả về response (statusCode, headers, body).  
   - **HTTP Proxy**:  
     - API Gateway chỉ proxy request trực tiếp đến backend HTTP (ví dụ ALB).  
     - Không dùng mapping template.  
     - Có thể thêm header tùy chọn (ví dụ API key giữa API Gateway và backend, client không thấy).

2. Mapping Templates & VTL  
   - Chỉ dùng được với **AWS/HTTP integration không phải proxy**.  
   - Cho phép:  
     - Đổi tên/sửa tham số query string.  
     - Sửa nội dung body.  
     - Thêm/sửa header.  
   - Dùng **VTL (Velocity Template Language)**: hỗ trợ if, for, logic đơn giản để biến đổi dữ liệu.  
   - Cũng dùng để chỉnh sửa response (xóa bớt trường, đổi cấu trúc, …).  
   - Cần đặt Content-Type phù hợp (application/json, application/xml, …).

3. Ứng dụng điển hình của Mapping Templates  
   - **Tích hợp client REST/JSON với backend SOAP/XML**:  
     - Client gửi JSON đến API Gateway.  
     - Mapping template chuyển JSON → XML (SOAP message) để gọi dịch vụ SOAP.  
     - Nhận XML từ SOAP, mapping template chuyển XML → JSON trả về client.  
   - **Đổi tên query parameters trước khi vào Lambda (non-proxy)**:  
     - Ví dụ client gửi `?name=foo&bar=other`.  
     - Mapping template đổi thành `myFooVar`, `myOtherBarVar` trong JSON truyền vào Lambda.  

Tóm lại:  
- Proxy integrations (Lambda Proxy, HTTP Proxy): API Gateway chỉ chuyển tiếp, không chỉnh sửa nội dung.  
- Non-proxy integrations (HTTP/AWS): có thể dùng mapping templates (VTL) để biến đổi request/response, rất hữu ích khi tích hợp các hệ thống khác định dạng (REST ↔ SOAP, đổi query params, headers, body, …).
