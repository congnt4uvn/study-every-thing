


Dưới đây là tóm tắt ngắn gọn nội dung:

1. Tạo CodePipeline mới  
- Đặt tên pipeline: `MyFirstPipeline`.  
- Giữ chế độ thực thi là “Queued”, dùng các thiết lập mặc định.  
- Tạo một IAM service role mới cho CodePipeline (cho phép pipeline thực hiện các tác vụ cần thiết).  
- Giữ nguyên cài đặt nâng cao (artifact store, encryption key).

2. Kết nối nguồn GitHub (GitHub v2)  
- Chọn nhà cung cấp nguồn là GitHub version 2.  
- Tạo kết nối `MyGitHubConnection` với GitHub, cấp quyền cho AWS Connector for GitHub.  
- Cài GitHub App, chọn repo cần dùng (ở đây là `my-nodejs-app`) và nhánh `main`.  
- Định dạng output artifact dùng mặc định của CodePipeline.  
- Thiết lập trigger để mỗi lần push lên nhánh `main` thì pipeline tự chạy.

3. Bỏ qua bước Build, cấu hình Deploy lên Elastic Beanstalk  
- Tạm thời bỏ qua bước Build (CodeBuild/Jenkins).  
- Chọn nhà cung cấp Deploy là AWS Elastic Beanstalk.  
- Chọn Application và Environment (env) đã tạo trước đó.  
- Tạo pipeline: từ GitHub → Deploy thẳng lên môi trường Beanstalk (env).

4. Cấp thêm quyền cho service role của CodePipeline  
- Vào phần Settings của pipeline → mở service role.  
- Gắn thêm policy `AdministratorAccess-AWSElasticBeanstalk` (hoặc tương đương) để pipeline có đủ quyền deploy lên Beanstalk (phục vụ mục đích demo).

5. Kiểm tra lần deploy đầu tiên  
- Pipeline chạy, lấy mã từ GitHub và deploy lên Beanstalk.  
- Ứng dụng (index.html) có nền màu xanh dương được triển khai thành công lên môi trường env.

6. Mở rộng pipeline: thêm giai đoạn DeployToProd  
- Chỉnh sửa pipeline, thêm stage mới: `DeployToProd`.  
- Trong stage này, thêm:  
  - Action 1: ManualApproval (phê duyệt thủ công).  
  - Action 2: DeployToProdBeanstalk (deploy lên môi trường production).  
- Action DeployToProdBeanstalk dùng artifact `SourceArtifact`, cùng application nhưng chọn environment production.  
- Lưu pipeline: giờ pipeline có luồng:  
  Source → Deploy (dev env) → DeployToProd (ManualApproval → Deploy prod).

7. Thử lại pipeline với thay đổi mới  
- Sửa file `index.html`, đổi màu nền từ xanh dương sang đỏ.  
- Commit và push trực tiếp lên nhánh `main`.  
- CodePipeline tự động kích hoạt:  
  - Lấy mã mới → deploy lên môi trường dev Beanstalk.  
  - Kiểm tra dev: nền đã chuyển sang màu đỏ.  
- Ở stage DeployToProd:  
  - Thực hiện ManualApproval (ví dụ comment: “Màu đỏ trông đẹp đấy”).  
  - Sau khi approve, pipeline deploy lên môi trường production.  
- Kiểm tra prod: nền cũng chuyển sang màu đỏ → deploy thành công.

8. Xem lịch sử pipeline  
- Có thể xem lịch sử các lần chạy, thời gian, trạng thái, commit tương ứng, v.v.  
- Đây là ví dụ hoàn chỉnh về một quy trình CI/CD cơ bản:  
  - Tự động deploy lên dev → phê duyệt thủ công → deploy lên prod.
