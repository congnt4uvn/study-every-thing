


Dưới đây là tóm tắt nội dung:

1. Tạo SQS làm “destination”  
   - Vào SQS, tạo 2 hàng đợi:
     - `S3-success`: dùng cho các lần thực thi Lambda thành công.  
     - `S3-fail`: dùng cho các lần thực thi Lambda thất bại.

2. Cấu hình Lambda Destinations  
   - Vào hàm Lambda (được trigger bởi S3).  
   - Thêm destination cho **asynchronous invocation**:
     - Điều kiện: **On failure** → Destination type: SQS → Queue: `S3-fail`.  
   - Console tự động cập nhật IAM role của Lambda để có quyền gửi message vào SQS.  
   - Thêm destination thứ hai:
     - Điều kiện: **On success** → Destination type: SQS → Queue: `S3-success`.  
   - Kiểm tra trong IAM role thấy policy cho phép ghi vào 2 hàng đợi SQS tương ứng.

3. Kiểm thử trường hợp thành công  
   - Vào S3, upload một file (ví dụ `beach.jpeg`).  
   - Lambda chạy thành công, log vào CloudWatch.  
   - Destination gửi message vào hàng đợi `S3-success`.  
   - Vào SQS → `S3-success` → “Send and receive messages” → Poll messages:
     - Thấy message chứa:
       - Thông tin về event source (S3, event “ObjectCreated:Put”, v.v.).  
       - Thông tin về response payload (status code, body, …).  
     - Tức là message chứa cả ngữ cảnh sự kiện và kết quả thực thi.

4. Kiểm thử trường hợp thất bại  
   - Sửa code Lambda: thay vì `return`, dùng `raise Exception("boom!")`.  
   - Deploy lại Lambda.  
   - Vào S3, upload file khác (ví dụ `index.html`).  
   - Lambda được invoke async, nhưng lần này ném exception.  
   - Ngay lập tức vào SQS → `S3-fail` sẽ **chưa** thấy message:
     - Vì S3 gọi Lambda theo kiểu async, Lambda có cấu hình **retry** (ví dụ 2 lần).  
     - Chỉ sau khi hết số lần retry mà vẫn lỗi, destination “On failure” mới được kích hoạt.  
   - Sau vài phút, refresh SQS:
     - Thấy message trong `S3-fail`.  
     - Nội dung message cho biết:
       - Các lần retry đã hết, approximateInvokeCount ≈ 3.  
       - Event record gây lỗi (giúp debug).  
       - Response payload: message lỗi `"boom!"`, type là Exception, stack trace, v.v.

5. Ý nghĩa  
   - Lambda Destinations cho phép:
     - Gửi thông tin chi tiết về **thành công** và **thất bại** đến các đích khác nhau (SQS, SNS, Lambda, EventBridge).  
     - Dễ dàng debug, xử lý lỗi, hoặc xây dựng các workflow tiếp theo dựa trên kết quả thực thi.  
   - Trong ví dụ:
     - Thành công → `S3-success` queue.  
     - Thất bại (sau khi retry hết) → `S3-fail` queue.
