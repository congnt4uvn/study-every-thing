


Dưới đây là tóm tắt nội dung về Amazon RDS Proxy trong đoạn trên:

1. **Khái niệm & mục đích**
   - RDS Proxy là proxy cơ sở dữ liệu được AWS quản lý hoàn toàn cho Amazon RDS và Aurora.
   - Ứng dụng không kết nối trực tiếp tới RDS nữa mà kết nối tới RDS Proxy; proxy sẽ quản lý và gộp (pool) các kết nối tới DB.

2. **Lợi ích chính**
   - **Gộp & chia sẻ kết nối**  
     - Giảm số lượng kết nối trực tiếp tới RDS/Aurora.  
     - Giảm tải cho CPU, RAM, số kết nối mở và timeout trên DB.
   - **Cải thiện failover**  
     - RDS Proxy hoạt động đa AZ, tự động scale, không cần quản lý capacity.  
     - Khi RDS/Aurora failover (primary → standby), RDS Proxy xử lý chuyển đổi, giúp giảm thời gian failover tới ~66%.  
     - Ứng dụng chỉ biết tới proxy, không phải tự xử lý logic failover.
   - **Bảo mật & IAM**  
     - Hỗ trợ xác thực IAM tới DB thông qua RDS Proxy.  
     - Thông tin xác thực DB có thể lưu an toàn trong AWS Secrets Manager.  
     - RDS Proxy chỉ truy cập được trong VPC, không public qua Internet.

3. **Tương thích & triển khai**
   - Hỗ trợ: MySQL, PostgreSQL, MariaDB, SQL Server, Aurora MySQL, Aurora PostgreSQL.
   - Không cần thay đổi code ứng dụng đáng kể: chỉ đổi endpoint từ DB sang Proxy.

4. **Trường hợp sử dụng tiêu biểu: AWS Lambda**
   - Lambda có thể scale lên hàng trăm / hàng nghìn instance rất nhanh, mỗi instance mở kết nối riêng tới DB → dễ gây quá tải, nhiều kết nối mở, timeout.
   - Dùng RDS Proxy để:
     - Gộp các kết nối từ nhiều Lambda thành ít kết nối hơn tới DB.
     - Giảm vấn đề “connection storm” và quản lý kết nối hiệu quả.

5. **Ý chính cần nhớ cho kỳ thi**
   - RDS Proxy:
     - Gộp & chia sẻ kết nối, giảm tải DB.
     - Giảm thời gian failover (tới 66%).
     - Thực thi IAM authentication và tích hợp Secrets Manager.
     - Không public, chỉ trong VPC.
     - Rất hữu ích với các môi trường có nhiều kết nối ngắn sống như Lambda.
