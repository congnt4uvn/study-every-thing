


Dưới đây là tóm tắt nội dung:

1. Tạo Lambda và SQS  
   - Tạo một hàm Lambda mới (lambda-SQS, runtime Python 3.8).  
   - Tạo một hàng đợi SQS mới tên `lambda-demo-SQS` (Standard Queue).  

2. Gắn SQS làm event source cho Lambda  
   - Trong Lambda, chọn “Add trigger” → chọn SQS → chọn hàng đợi `lambda-demo-SQS`.  
   - Cấu hình:
     - Batch size: số lượng message nhận trong một lần gọi Lambda.  
     - Batch window: thời gian gom message trước khi gọi Lambda (tối ưu số lần invoke).  
     - Bật trigger.  

3. Xử lý lỗi quyền (IAM)  
   - Khi thêm trigger, xuất hiện lỗi: role thực thi của Lambda không có quyền `ReceiveMessage`/`GetMessage` trên SQS.  
   - Vào tab Configuration → Permissions → mở role IAM của Lambda.  
   - Gắn policy phù hợp, ví dụ `AWSLambdaSQSQueueExecutionRole` để Lambda có quyền đọc từ SQS.  
   - Quay lại Lambda, thêm lại trigger → lần này thành công.  

4. Viết code Lambda và kiểm thử  
   - Code Lambda: in (log) toàn bộ event nhận được, sau đó trả về “success”.  
   - Deploy code.  
   - Vào SQS → “Send message”:  
     - Body: `hello world`.  
     - Message attribute: `foo = bar`.  
   - Lambda sẽ tự động kéo message từ SQS và xử lý.  
   - Kiểm tra CloudWatch Logs:
     - Thấy event log, body = `hello world`, attribute `foo = bar`, thông tin nguồn sự kiện là SQS, thời gian gửi, v.v.  
   - Vào lại SQS → “Send and receive messages” → không còn message vì đã được Lambda xử lý.  

5. Tắt trigger để tránh chi phí  
   - Sau khi thử nghiệm, nên disable trigger SQS để Lambda không liên tục poll hàng đợi (tránh phát sinh chi phí).  

6. Giới thiệu trigger Kinesis  
   - Thêm trigger Kinesis để minh họa các tùy chọn:  
     - Chọn Kinesis stream.  
     - Chọn kiểu consumer (standard hoặc enhanced fan-out).  
     - Batch size: số record đọc mỗi lần.  
     - Batch window: thời gian gom record.  
     - Starting position: `LATEST`, `TRIM_HORIZON` (sớm nhất), hoặc timestamp cụ thể.  
     - Các tùy chọn nâng cao:
       - Dead-letter destination (nơi gửi record lỗi).  
       - Số lần retry, tuổi tối đa của record.  
       - Batching on error, số batch đồng thời trên mỗi shard.  
       - Windowing để tổng hợp và báo cáo lỗi.  
   - Hủy (cancel) vì chỉ xem tùy chọn, không cấu hình thật.  

7. Kết luận  
   - Đã minh họa cách dùng event source mapping (SQS, Kinesis) cho Lambda.  
   - Đã thấy cách bổ sung quyền IAM còn thiếu để Lambda đọc từ nguồn sự kiện.  
   - Khuyến nghị đọc tài liệu AWS để hiểu chi tiết các tùy chọn nâng cao.
