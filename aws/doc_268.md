


Dưới đây là tóm tắt nội dung bài giảng:

1. Mục tiêu  
   - Giải thích cách tích hợp AWS Lambda với Application Load Balancer (ALB) để đưa hàm Lambda ra Internet qua HTTP/HTTPS.

2. Cách hoạt động tổng quan  
   - Client gửi request HTTP/HTTPS đến ALB.  
   - ALB gọi hàm Lambda (theo kiểu đồng bộ) thông qua target group.  
   - Lambda xử lý và trả về kết quả cho ALB.  
   - ALB chuyển kết quả đó thành HTTP response trả lại cho client.

3. Chuyển đổi HTTP → JSON (request vào Lambda)  
   - ALB chuyển toàn bộ HTTP request thành một tài liệu JSON gửi cho Lambda.  
   - JSON chứa:
     - Thông tin ELB (ALB nào, target group nào).  
     - Thông tin HTTP: method (GET, POST, …), path (ví dụ /lambda).  
     - Query string parameters: được đưa vào JSON dưới dạng cặp key–value.  
     - Headers: cũng là cặp key–value.  
     - Body (cho POST/PUT…): được đưa vào JSON, kèm cờ cho biết body có được mã hóa Base64 hay không.

4. Chuyển đổi JSON → HTTP (response từ Lambda)  
   - Lambda trả về một JSON gồm:
     - Status code và mô tả.  
     - Response headers dưới dạng cặp key–value.  
     - Response body.  
     - Cờ cho biết body có được mã hóa Base64 hay không.  
   - ALB dùng JSON này để dựng lại HTTP response gửi cho client.

5. Tính năng Multi-value Headers (tiêu đề đa giá trị)  
   - Có thể bật trên ALB.  
   - Khi client gửi:
     - Nhiều giá trị cho cùng một query parameter (ví dụ: `?name=foo&name=bar`).  
     - Hoặc nhiều giá trị cho cùng một header.  
   - Thay vì chỉ giữ một giá trị, ALB giữ tất cả và gửi sang Lambda dưới dạng mảng trong JSON (ví dụ: `"name": ["foo", "bar"]`).  
   - Áp dụng cho cả headers và query string parameters.

Tóm lại: ALB đóng vai trò “dịch” giữa HTTP và JSON để Lambda có thể xử lý request web, và có thể hỗ trợ nhiều giá trị cho cùng một header/parameter khi bật multi-value headers.
