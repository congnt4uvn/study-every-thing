


Dưới đây là tóm tắt nội dung:

- Tác giả giới thiệu EC2 Instance Connect như một giải pháp thay thế cho SSH, dễ dùng hơn vì chạy trực tiếp trên trình duyệt.  
- Cách dùng:  
  - Chọn EC2 instance (ví dụ “My First Instance”) → nhấn “Connect”.  
  - Trong các tùy chọn kết nối, chọn “EC2 Instance Connect”.  
  - AWS tự điền sẵn username là `ec2-user` (phù hợp với Amazon Linux 2). Nếu đổi sang tên khác thì sẽ không hoạt động.  
- Khi kết nối bằng EC2 Instance Connect:  
  - Không cần tải lên hay quản lý SSH key thủ công.  
  - AWS tự động tạo và tải lên một SSH key tạm thời để thiết lập phiên SSH.  
  - Một tab trình duyệt mới mở ra, cho phép chạy lệnh trực tiếp trên Amazon Linux 2 (ví dụ `whoami`, `ping google.com`).  
- Phiên làm việc diễn ra hoàn toàn trên trình duyệt, không cần dùng terminal/CLI riêng hay các công cụ như PuTTY.  
- Trong khóa học, khi nói “dùng SSH”, người học có thể:  
  - Dùng terminal/SSH client riêng, hoặc  
  - Dùng EC2 Instance Connect trên trình duyệt, tùy hệ điều hành (Windows, Linux, Mac).  
- EC2 Instance Connect vẫn phụ thuộc vào SSH ở tầng nền:  
  - Cần mở cổng 22 trong Security Group.  
  - Nếu xóa rule SSH (port 22) khỏi inbound rules, EC2 Instance Connect sẽ không kết nối được và báo lỗi yêu cầu mở port 22.  
  - Cần thêm lại rule SSH (port 22) cho IPv4 (Anywhere) và, trong một số trường hợp, cả IPv6 (Anywhere) để đảm bảo hoạt động.  
- Kết luận: EC2 Instance Connect là cách nhanh, tiện, không cần quản lý SSH key, và sẽ được sử dụng nhiều trong khóa học.
