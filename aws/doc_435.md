


Nội dung trên là một bài giảng hướng dẫn cách dùng Amazon Athena để truy vấn log truy cập S3:

1. Cấu hình Athena và S3  
- Mở Query Editor của Athena.  
- Trước khi chạy truy vấn, phải cấu hình “Query result location” – chọn một bucket S3 để lưu kết quả truy vấn.  
- Vào console S3, tạo một bucket mới (tên phải là duy nhất), sau đó copy tên bucket và dán vào phần cấu hình kết quả truy vấn của Athena.

2. Tạo database trong Athena  
- Mở Query Editor, chạy câu lệnh SQL (lấy từ tài liệu S3 + Athena) để tạo database, ví dụ: `s3_access_log_db`.  
- Sau khi chạy, database mới xuất hiện ở panel bên trái cùng với database `default`.

3. Tạo bảng log truy cập S3  
- Chạy câu lệnh `CREATE TABLE` (cũng lấy từ tài liệu AWS) để định nghĩa bảng biểu diễn log truy cập S3.  
- Phần duy nhất cần sửa là `LOCATION`:
  - Điền đúng tên bucket chứa log.  
  - Nếu log nằm trong “thư mục” (prefix), thêm prefix và dấu gạch chéo.  
  - Đảm bảo có dấu gạch chéo ở cuối đường dẫn.  
- Sau khi chạy, bảng xuất hiện trong database, kèm theo các cột như: bucket owner, bucket, request datetime, IP, requester, request ID, v.v.

4. Xem dữ liệu log  
- Dùng chức năng “Preview table” để xem nhanh 10 dòng dữ liệu.  
- Thấy được các thông tin chi tiết về từng request trong log S3.

5. Chạy các truy vấn phân tích nâng cao  
- Tạo truy vấn mới để tổng hợp dữ liệu, ví dụ:
  - Đếm số request theo HTTP status và URI.  
  - Phân tích số lượng lỗi 404 (Not Found) để phát hiện vấn đề.  
  - Lọc các request có status 403 (Forbidden) để xem các truy cập trái phép.  
- Athena sẽ quét dữ liệu (ví dụ ~30 MB) và trả về kết quả thống kê.

6. Kết luận  
- Athena cho phép truy vấn dữ liệu trên S3 bằng SQL mà không cần thiết lập máy chủ (serverless).  
- Quy trình: tạo database → tạo bảng trỏ tới dữ liệu S3 → dùng SQL để phân tích.  
- Giảng viên nhấn mạnh đây là một dịch vụ rất dễ dùng và hữu ích của AWS để phân tích log và dữ liệu lưu trên S3.
