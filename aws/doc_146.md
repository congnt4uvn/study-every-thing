


Dưới đây là tóm tắt nội dung:

1. Tạo bucket S3 mới tên `demo-stephane-MFA-delete-2020` ở `eu-west-1` và bật versioning cho bucket.

2. Trong phần thuộc tính của bucket, mục “MFA delete” (xóa có xác thực đa yếu tố) đang ở trạng thái vô hiệu hóa và không thể bật/tắt qua giao diện console, nên phải dùng AWS CLI.

3. Điều kiện tiên quyết:
   - Tài khoản root phải có thiết bị MFA (virtual MFA device) được cấu hình trong IAM.
   - Lấy ARN của thiết bị MFA này để dùng trong lệnh CLI.

4. Cấu hình AWS CLI:
   - Tạo access key cho tài khoản root (chỉ dùng tạm thời cho demo).
   - Chạy `aws configure` để tạo profile mới, ví dụ `root-MFA-delete-demo`, nhập Access Key, Secret Key, region `eu-west-1`.
   - Kiểm tra bằng `aws s3 ls --profile root-MFA-delete-demo` để chắc chắn profile hoạt động.

5. Bật MFA Delete:
   - Dùng lệnh `aws s3api put-bucket-versioning` với cấu hình:
     - `Status=Enabled` (bật versioning).
     - `MFADelete=Enabled` (bật MFA delete).
     - Truyền ARN thiết bị MFA và mã MFA (6 số từ app) vào tham số `--mfa`.
   - Sau khi lệnh thành công, vào lại phần Bucket Versioning trong console sẽ thấy:
     - Versioning: Enabled
     - MFA Delete: Enabled

6. Kiểm thử:
   - Upload một file (ví dụ file JPEG) vào bucket.
   - Xóa file trong console: vì versioning bật nên chỉ tạo delete marker, thao tác này vẫn được phép.
   - Liệt kê các phiên bản object: thấy có nhiều phiên bản.
   - Khi cố gắng xóa hẳn một version cụ thể (permanent delete) trong console:
     - Console báo lỗi: không thể xóa vì bucket bật MFA Delete, phải dùng CLI hoặc tắt MFA Delete.

7. Tắt MFA Delete:
   - Dùng lại lệnh `put-bucket-versioning` nhưng đặt `MFADelete=Disabled`.
   - Cung cấp mã MFA mới (6 số) trong tham số `--mfa`.
   - Sau khi lệnh thành công, vào lại console sẽ thấy MFA Delete đã bị vô hiệu hóa.
   - Lúc này có thể xóa delete marker hoặc version object bình thường.

8. Cuối cùng:
   - Giảng viên nhắc phải xóa/disable access key của tài khoản root sau khi xong demo vì rất nguy hiểm nếu để lại.
   - Thực hiện deactivate và có thể xóa hẳn access key root.
