


Dưới đây là tóm tắt nội dung bài giảng về AWS SAM:

1. Khái niệm SAM  
- SAM = Serverless Application Model (Mô hình ứng dụng không máy chủ).  
- Là một framework để phát triển và triển khai ứng dụng serverless trên AWS.  
- Bạn viết code + file cấu hình YAML theo chuẩn SAM.  
- SAM tự động chuyển YAML đơn giản thành template CloudFormation phức tạp.

2. Quan hệ với CloudFormation  
- SAM hỗ trợ đầy đủ các tính năng của CloudFormation: Outputs, Mappings, Parameters, Resources…  
- Ở đầu template SAM có phần `Transform` để báo cho CloudFormation biết cần chuyển đổi template SAM thành template CloudFormation.  
- Thay vì dùng các resource CloudFormation chi tiết, SAM cung cấp các “cấu trúc” đơn giản:
  - `AWS::Serverless::Function` → Lambda  
  - `AWS::Serverless::Api` → API Gateway  
  - `AWS::Serverless::SimpleTable` → DynamoDB

3. Khả năng hỗ trợ phát triển  
- SAM có thể dùng CodeDeploy để triển khai Lambda.  
- Hỗ trợ chạy và debug cục bộ Lambda, API Gateway, DynamoDB.  
- Tập trung vào ứng dụng serverless, giúp phát triển, debug và triển khai nhanh hơn.

4. Quy trình build & deploy với SAM  
- Bạn có:  
  - Mã ứng dụng  
  - Template SAM (YAML)  
- Quy trình:
  1) `sam build`:  
     - Build ứng dụng cục bộ  
     - Chuyển template SAM thành template CloudFormation  
  2) `sam deploy`:  
     - Nén và upload code + template lên S3  
     - Tạo/ cập nhật CloudFormation stack (ChangeSet)  
     - Tạo các tài nguyên serverless: Lambda, API Gateway, DynamoDB…

- Trước đây phải dùng `sam package` rồi `sam deploy`, giờ `sam deploy` có thể tự đóng gói.

5. SAM Accelerate & sam sync  
- SAM Accelerate: tập hợp tính năng giúp giảm độ trễ khi triển khai lên AWS.  
- Lệnh chính: `sam sync`  
  - Đồng bộ dự án (template SAM + code) lên AWS.  
  - Nếu chỉ thay đổi code (không đổi hạ tầng), SAM có thể bỏ qua CloudFormation và gọi trực tiếp API dịch vụ → triển khai rất nhanh.  
- Cách dùng:
  - `sam sync`: đồng bộ cả code và hạ tầng.  
  - `sam sync --code`: chỉ đồng bộ code, không cập nhật hạ tầng → không dùng CloudFormation, cập nhật trong vài giây.  
  - Có thể chỉ định resource cụ thể (ví dụ chỉ một Lambda hoặc nhóm Lambda).  
  - `sam sync --watch`: chạy nền, theo dõi file, tự động sync khi phát hiện thay đổi.

6. Mục tiêu  
- Giúp bạn:
  - Viết ứng dụng serverless đơn giản hơn.  
  - Debug cục bộ dễ hơn.  
  - Triển khai nhanh hơn lên AWS (đặc biệt với SAM Accelerate).  
- Bài giảng sau sẽ có phần thực hành để hiểu rõ hơn cách hoạt động.
