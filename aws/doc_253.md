


Nội dung trên giải thích các API của AWS X-Ray ở mức độ cao, chia thành hai nhóm: phía ghi (write) và phía đọc (read), cùng với quyền IAM cần thiết.

1. **Phía ghi (Write side)** – dùng bởi X-Ray daemon để gửi dữ liệu lên dịch vụ X-Ray.  
   Managed policy: **XRayWriteOnlyAccess** (tên mô tả là “X-Ray write only access”), gồm các quyền chính:
   - **PutTraceSegments**:  
     - Gửi (upload) các tài liệu phân đoạn (segment documents) lên AWS X-Ray.  
     - Đây là API bắt buộc nếu muốn ghi dữ liệu vào X-Ray.
   - **PutTelemetryRecords**:  
     - Gửi thông tin telemetry về số lượng phân đoạn đã nhận, bị từ chối, lỗi kết nối backend…  
     - Dùng cho mục đích metrics/giám sát daemon.
   - **GetSamplingRules**:  
     - Cho phép daemon X-Ray lấy các quy tắc sampling hiện tại.  
     - Khi thay đổi sampling rules trong X-Ray console, daemon sẽ tự động cập nhật nhờ API này.
   - **GetSamplingTargets** và **GetSamplingStatisticSummaries**:  
     - Các API nâng cao, cũng liên quan đến sampling rules và cách áp dụng chúng.

   Kết luận: để ghi vào X-Ray, daemon cần:
   - **PutTraceSegments**
   - **PutTelemetryRecords**
   - Và quyền đọc các quy tắc sampling: **GetSamplingRules** (cùng với các API sampling liên quan).

   Đồng thời, phải gán **chính sách IAM phù hợp** cho X-Ray daemon để cho phép các lệnh gọi API này.

2. **Phía đọc (Read side)** – dùng để xem dữ liệu trong X-Ray (thường là khi truy cập console).  
   Managed policy: **XRayReadOnlyAccess** (mô tả là “read access”), gồm các API chủ yếu là “Get”:
   - **GetServiceGraph**:  
     - Lấy service map/graph chính hiển thị trong X-Ray console.
   - **BatchGetTraces**:  
     - Lấy danh sách các trace đầy đủ dựa trên các trace ID được chỉ định.  
     - Mỗi trace là tập hợp các segment document của một request.
   - **GetTraceSummaries**:  
     - Lấy danh sách trace ID và các annotation trong một khoảng thời gian.  
     - Sau đó, nếu muốn xem chi tiết, dùng các ID này gọi **BatchGetTraces**.
   - **GetTraceGraph**:  
     - Lấy service graph cụ thể cho một hoặc nhiều trace ID.

   Đây là các API cần thiết khi truy cập X-Ray console để đọc và phân tích trace.

3. **Ý chính cần nhớ (đặc biệt cho mục đích thi cử)**:
   - **Write**:
     - PutTraceSegments → ghi segment vào X-Ray.
     - PutTelemetryRecords → gửi telemetry/metrics.
     - GetSamplingRules (+ Targets, StatisticSummaries) → cập nhật quy tắc sampling cho daemon.
   - **Read**:
     - GetServiceGraph → service map.
     - GetTraceSummaries → danh sách trace (ID + annotation) theo thời gian.
     - BatchGetTraces → lấy chi tiết trace dựa trên ID.
     - GetTraceGraph → graph cho các trace cụ thể.
   - Cần nhớ **khi nào dùng API nào và tại sao**, và đảm bảo **IAM policy** phù hợp cho daemon (write) và cho người dùng/console (read).
