


Dưới đây là tóm tắt nội dung:

- Ta đã có một CloudFormation stack và muốn cập nhật nó. Cách cập nhật là thay thế template hiện tại bằng một template mới (không chỉnh sửa trực tiếp template cũ).

- Giảng viên chuyển sang file template mới `EC2-with-SG-EIP.yaml`. Template này là bản nâng cấp so với template trước:
  - Có thêm tham số (parameter) “Security Group Description”.
  - Có resource EC2 instance với nhiều tham số hơn.
  - Thêm Elastic IP (EIP) gắn với EC2 instance.
  - Thêm 2 Security Group: một cho SSH, một cho server (SSH + HTTP).
  - Các Security Group này sẽ được tạo và gắn vào EC2 instance.

- Khi cập nhật stack:
  - Chọn “Upload a template file” và chọn file `SG-EIP.yaml`.
  - CloudFormation yêu cầu nhập tham số “Security Group Description” (ví dụ: “Đây là một nhóm bảo mật thú vị”).
  - Ở phần “Change set preview”, CloudFormation hiển thị những thay đổi:
    - Thêm Elastic IP.
    - Thêm Security Group SSH.
    - Thêm Security Group server.
    - EC2 instance cũ sẽ bị “thay thế đúng” (true replacement): instance cũ bị terminate, instance mới được tạo.

- Trong quá trình update:
  - Hai Security Group mới được tạo trước.
  - EC2 instance được cập nhật bằng cách tạo instance mới rồi terminate instance cũ.
  - Elastic IP được tạo và gắn vào instance mới.
  - Tất cả resource đều được gắn tag tự động (logical ID, stack ID, stack name).

- Kiểm tra trên EC2:
  - Thấy instance cũ và instance mới (mới ở trạng thái “running”, cũ bị “terminated”).
  - Instance mới có public IP trùng với Elastic IP vừa tạo.
  - Instance mới có 2 Security Group:
    - SG SSH: inbound rule port 22.
    - SG server: inbound rule SSH + HTTP, phần mô tả (description) chính là giá trị tham số đã nhập: “Đây là một nhóm bảo mật thú vị”.

- Ý nghĩa:
  - Tham số (parameters) cho phép truyền giá trị động (như mô tả SG) vào template và dùng trong runtime.
  - CloudFormation tự động tìm cách từ trạng thái hiện tại đến trạng thái mong muốn (tạo, cập nhật, thay thế resource).

- Dọn dẹp (cleanup):
  - Không nên chỉ xóa EC2 instance thủ công vì SG, EIP vẫn còn.
  - Thay vào đó, xóa stack trong CloudFormation:
    - CloudFormation tự xác định thứ tự xóa: EIP → EC2 instance → Security Groups.
    - Xóa sạch toàn bộ resource do stack tạo ra.

- Kết luận:
  - Qua ví dụ này, ta thấy cách:
    - Tạo stack.
    - Cập nhật stack bằng template mới (thêm EIP, SG, tham số).
    - Xem change set và hiểu “replacement”.
    - Xóa stack để dọn sạch tài nguyên.
  - Đây là phần giới thiệu về sức mạnh của AWS CloudFormation trong việc quản lý hạ tầng bằng code.
