


Dưới đây là tóm tắt nội dung bài giảng:

- Khi dùng AWS CDK, bạn có thể kiểm thử code hạ tầng tương tự như kiểm thử code ứng dụng (Python, JavaScript, …).
- CDK có “mô-đun assertion” (cdk assertions) tích hợp với các framework test phổ biến như Jest (JS) hoặc Pytest (Python).
- Nhờ các assertion này, bạn có thể:
  - Kiểm tra tài nguyên (resource) được tạo đúng loại, đúng thuộc tính.
  - Kiểm tra các rule, condition, parameter trong template CloudFormation.

Có hai loại test chính trong CDK:

1. **Assertion chi tiết (fine‑grained assertions)**  
   - Kiểm tra cụ thể một resource có thuộc tính mong muốn hay không.  
   - Ví dụ:  
     - Hàm Lambda có đúng handler không.  
     - Runtime là `nodejs14.x` hay không.  
     - Subscription của SNS topic có tồn tại và đúng kiểu không.

2. **Test snapshot**  
   - So sánh toàn bộ template CloudFormation sinh ra với một “snapshot” (mẫu chuẩn) đã lưu trước đó.  
   - Dùng để đảm bảo template tổng thể không bị thay đổi ngoài ý muốn (ví dụ: bảng DynamoDB vẫn tồn tại với các thuộc tính cơ bản mong đợi).

Cách lấy template để test:

- **`Template.fromStack(myStack)`**  
  - Dùng khi stack được định nghĩa trong CDK.  
  - Import stack CDK, rồi tạo đối tượng template từ stack đó.

- **`Template.fromString(myString)`**  
  - Dùng khi template CloudFormation nằm ngoài CDK (ví dụ file YAML/JSON có sẵn).  
  - Đọc nội dung file thành chuỗi, rồi tạo template từ chuỗi đó để vẫn có thể chạy test.

Điểm cần nhớ cho kỳ thi: hai phương thức quan trọng là `fromStack` và `fromString`, dùng để tạo template phục vụ kiểm thử.
