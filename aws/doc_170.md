


Nội dung trên giải thích cách ECS thực hiện **rolling update** (cập nhật liên tục) cho dịch vụ khi đổi từ version 1 sang version 2, thông qua hai tham số:

- **Minimum healthy percent** (phần trăm khỏe mạnh tối thiểu)  
- **Maximum percent** (phần trăm tối đa)

Ý nghĩa:

- **Minimum healthy percent**: số lượng task tối thiểu (tính theo %) phải luôn chạy trong quá trình cập nhật. Nếu đặt < 100%, ECS được phép tạm dừng bớt task cũ xuống dưới số lượng ban đầu.
- **Maximum percent**: số lượng task tối đa (tính theo %) được phép chạy cùng lúc. Nếu đặt > 100%, ECS có thể chạy thêm task mới song song với task cũ.

Ví dụ minh họa với 4 task ban đầu:

1. **Min 50% – Max 100%**  
   - Có thể giảm xuống 2 task (50%) bằng cách dừng 2 task cũ.  
   - Tạo 2 task mới → quay lại 4 task (100%).  
   - Tiếp tục dừng 2 task cũ còn lại → còn 2 task (50%).  
   - Tạo thêm 2 task mới → 4 task mới (100%).  
   → Cập nhật bằng cách **giảm bớt task cũ rồi thêm task mới**, luôn giữ ≥ 50% công suất.

2. **Min 100% – Max 150%**  
   - Không được dừng task nào lúc đầu vì phải giữ 100%.  
   - Tạo thêm 2 task mới → tổng 6 task (150%).  
   - Vì đã vượt mức tối thiểu 100%, có thể dừng 2 task cũ → còn 4 task (100%).  
   - Lặp lại: tạo 2 task mới, dừng 2 task cũ…  
   → Cập nhật bằng cách **tăng thêm task mới rồi mới dừng task cũ**, luôn giữ ≥ 100% công suất.

Kết luận:  
Hai tham số **minimum healthy percent** và **maximum percent** quyết định chiến lược rolling update của ECS: hoặc giảm bớt rồi tăng (min < 100), hoặc tăng rồi giảm (min = 100, max > 100), đảm bảo dịch vụ luôn có đủ số task chạy trong quá trình cập nhật.
