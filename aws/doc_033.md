


Bài giảng mô tả quy trình khởi chạy và quản lý một EC2 instance đầu tiên chạy Amazon Linux bằng AWS Management Console:

1. Khởi chạy EC2 instance  
- Vào EC2 Console → Instances → Launch Instance.  
- Đặt tên (ví dụ: “My first instance”) và gắn thẻ nếu cần.  
- Chọn AMI: dùng Amazon Linux 2 AMI (free tier eligible), kiến trúc 64-bit x86.  
- Chọn loại instance: t2.micro (hoặc t3.micro nếu vùng không có t2.micro), phù hợp free tier.

2. Tạo và chọn key pair  
- Tạo key pair mới (ví dụ: “ec2-tutorial”) để đăng nhập bằng SSH.  
- Chọn loại: RSA.  
- Chọn định dạng:  
  - .pem cho Mac, Linux, Windows 10+  
  - .ppk cho Windows 7/8 (dùng PuTTY).  
- Tải file key pair về máy.

3. Cấu hình mạng và bảo mật  
- Giữ nguyên cài đặt mạng mặc định, instance sẽ nhận IP công cộng.  
- Tạo Security Group (launch-wizard-1) với các rule:  
  - Inbound:  
    - SSH (port 22) từ mọi nơi (0.0.0.0/0) để đăng nhập.  
    - HTTP (port 80) từ mọi nơi để truy cập web server.  
  - Outbound: cho phép toàn bộ traffic ra ngoài (để instance truy cập Internet).

4. Cấu hình storage  
- Dùng volume gốc mặc định: 8 GB gp2 (EBS).  
- Free tier cho phép tối đa 30 GB EBS gp2/gp3.  
- Tùy chọn “Delete on termination” mặc định bật: khi terminate instance, volume sẽ bị xóa.

5. Cấu hình Advanced – User Data  
- Trong phần Advanced, dùng mục User Data để dán một script.  
- Script này chạy tự động khi instance khởi động lần đầu (chỉ một lần):  
  - Cập nhật hệ thống.  
  - Cài đặt web server (httpd).  
  - Tạo file HTML đơn giản (trang “Hello World”).  
- Mục đích: tự động triển khai web server ngay khi instance khởi chạy.

6. Khởi chạy và kiểm tra instance  
- Review cấu hình, xác nhận free tier (750 giờ/tháng cho t2.micro, 30 GB EBS).  
- Launch instance.  
- Chờ trạng thái từ “pending” sang “running”.  
- Xem thông tin:  
  - Instance ID, Public IPv4, Private IPv4, AMI, instance type, key pair, security group, volume.

7. Truy cập web server  
- Lấy Public IPv4 address.  
- Truy cập bằng trình duyệt với:  
  - http://<public-ip>  
  - Lưu ý: phải dùng HTTP, không dùng HTTPS, nếu không sẽ bị treo tải.  
- Trang hiển thị “Hello World from …” kèm địa chỉ IP private (172.x.x.x) được script ghi vào.

8. Dừng (Stop) và chấm dứt (Terminate) instance  
- Stop instance:  
  - Vào Instance state → Stop instance.  
  - Khi stop, không bị tính phí compute, nhưng EBS vẫn tồn tại (vẫn bị tính phí storage).  
  - Web server không còn truy cập được (trình duyệt sẽ tải mãi).  
- Start instance lại:  
  - Instance state → Start instance.  
  - Private IP giữ nguyên.  
  - Public IP có thể thay đổi → phải dùng IP mới để truy cập.  
  - User Data script không chạy lại (chỉ chạy lần đầu).  
- Terminate instance (không thực hiện trong demo, chỉ giải thích):  
  - Instance state → Terminate instance.  
  - Instance bị xóa, volume gốc (nếu “delete on termination” bật) cũng bị xóa.

9. Ý nghĩa  
- Minh họa sức mạnh của cloud: có thể tạo, dừng, xóa nhiều server trong vài giây mà không cần sở hữu phần cứng.  
- Làm quen với các khái niệm cơ bản của EC2: AMI, instance type, key pair, security group, EBS, user data, public/private IP, start/stop/terminate.
