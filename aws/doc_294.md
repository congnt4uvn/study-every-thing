


Dưới đây là tóm tắt nội dung:

1. Gắn EFS vào Lambda  
   - Lambda có thể truy cập Amazon EFS nếu chạy trong VPC.  
   - Cần cấu hình Lambda gắn EFS vào một thư mục cục bộ (mount point) khi khởi tạo.  
   - Phải sử dụng EFS Access Point.  
   - Lambda phải nằm trong subnet private có kết nối tới EFS.  
   - Mỗi instance Lambda tạo thêm một kết nối tới EFS → cần chú ý giới hạn kết nối EFS và giới hạn “connection group” nếu nhiều hàm chạy đồng thời.

2. So sánh các tùy chọn lưu trữ cho Lambda  

   a. /tmp (Ephemeral storage)  
   - Dung lượng tối đa: 10 GB.  
   - Tính bền: tạm thời (mất dữ liệu khi instance Lambda bị hủy).  
   - Loại: hệ thống tệp, có thể đọc/ghi tự do.  
   - Giá: miễn phí đến 512 MB, trên đó thì trả thêm.  
   - Chỉ hàm Lambda đó truy cập được, không chia sẻ giữa các lần gọi khác nhau.  
   - Tốc độ truy cập: nhanh nhất.

   b. Lambda Layers  
   - Tối đa 5 layers/hàm, tổng kích thước ≤ 250 MB (giới hạn gói Lambda).  
   - Tính bền: lâu bền, bất biến (không sửa nội dung layer được sau khi publish).  
   - Loại: lưu trữ tĩnh, được gắn sẵn vào môi trường thực thi.  
   - Giá: đã bao gồm trong chi phí Lambda.  
   - Cần quyền IAM để truy cập layer.  
   - Tốc độ truy cập: rất nhanh, được chia sẻ cho tất cả lần gọi của hàm.  
   - Không thể sửa dữ liệu trong layer lúc runtime.

   c. Amazon S3  
   - Dung lượng: gần như không giới hạn.  
   - Tính bền: lâu bền.  
   - Loại: lưu trữ đối tượng, truy cập qua API S3 (GET, PUT, v.v.).  
   - Hỗ trợ hoạt động nguyên tử, versioning.  
   - Giá: trả cho lưu trữ + request + băng thông.  
   - Cần quyền IAM để truy cập.  
   - Lưu trữ dựa trên mạng, nhanh nhưng không nhanh bằng lưu trữ cục bộ.  
   - Chia sẻ cho tất cả các lần gọi Lambda (vì là dịch vụ bên ngoài).

   d. Amazon EFS  
   - Dung lượng: đàn hồi (tự mở rộng).  
   - Tính bền: lâu bền.  
   - Loại: hệ thống tệp, truy cập bằng các thao tác file system thông thường.  
   - Giá: trả cho lưu trữ, truyền dữ liệu, thông lượng.  
   - Được gắn như một network file system vào Lambda → truy cập rất nhanh.  
   - Chia sẻ giữa tất cả các lần gọi Lambda (vì là file system mạng).  

Nội dung chính: giảng viên giải thích cách Lambda gắn EFS và so sánh 4 lựa chọn lưu trữ (/tmp, Lambda Layers, S3, EFS) theo dung lượng, độ bền, kiểu lưu trữ, chi phí, tốc độ và khả năng chia sẻ, để chọn giải pháp phù hợp với từng tình huống.
