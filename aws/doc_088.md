


Dưới đây là tóm tắt nội dung bài giảng:

1. Khái niệm DNS  
- DNS (Domain Name System) là hệ thống dịch tên miền thân thiện với con người (ví dụ: www.google.com) thành địa chỉ IP của máy chủ (ví dụ: 9.10.11.12).  
- Nhờ DNS, trình duyệt có thể dùng tên miền để truy cập đúng máy chủ trên Internet.  
- DNS là “xương sống” của Internet vì hầu như mọi truy cập website đều đi qua DNS.

2. Cấu trúc phân cấp tên miền  
- Tên miền được tổ chức theo dạng phân cấp:  
  - Gốc (root): dấu chấm cuối cùng (.) thường bị ẩn.  
  - Tên miền cấp cao nhất (TLD): .com, .us, .in, .gov, .org, …  
  - Tên miền cấp 2: ví dụ amazon.com, google.com.  
  - Subdomain (tên miền phụ): www.example.com, api.example.com, …  
- FQDN (Fully Qualified Domain Name – tên miền đầy đủ): ví dụ api.www.example.com.  
- URL = giao thức (http/https) + FQDN + các phần khác.

3. Một số thuật ngữ DNS  
- Domain registrar (nhà đăng ký tên miền): nơi bạn đăng ký tên miền (Route 53, GoDaddy, …).  
- Bản ghi DNS (DNS record): A, AAAA, CNAME, NS, … dùng để ánh xạ tên miền tới IP hoặc tài nguyên khác.  
- Zone file: tệp chứa toàn bộ các bản ghi DNS cho một vùng (zone).  
- Name server (máy chủ định danh): máy chủ thực sự trả lời các truy vấn DNS.  
- TLD: .com, .org, .gov, …  
- Tên miền cấp 2: amazon.com, google.com, …

4. Quy trình phân giải DNS (cách DNS hoạt động)  
Ví dụ: truy cập example.com có IP 9.10.11.12  
- Bước 1: Trình duyệt hỏi máy chủ DNS cục bộ (thường do ISP hoặc công ty quản lý): “example.com là IP nào?”.  
- Bước 2: Nếu DNS cục bộ chưa biết, nó hỏi máy chủ DNS gốc (root server – do ICANN quản lý).  
  - Root server không biết example.com, nhưng biết máy chủ tên (NS) của TLD .com và trả về IP của name server .com.  
- Bước 3: DNS cục bộ hỏi name server của .com: “example.com là gì?”.  
  - Name server .com biết name server quản lý example.com (ví dụ IP 5.6.7.8) và trả về IP đó.  
- Bước 4: DNS cục bộ hỏi name server của example.com (thường do nhà đăng ký như Route 53 quản lý):  
  - Name server này có bản ghi A cho example.com trỏ tới 9.10.11.12 và trả về IP đó.  
- Bước 5: DNS cục bộ lưu kết quả vào cache (bộ nhớ đệm) để trả lời nhanh cho các truy vấn sau.  
- Bước 6: DNS cục bộ gửi IP 9.10.11.12 về cho trình duyệt, và trình duyệt dùng IP này để kết nối tới máy chủ web.

5. Kết nối với Route 53  
- Toàn bộ phần trên là kiến thức nền tảng về DNS.  
- Trong các phần tiếp theo, giảng viên sẽ dùng Route 53 để tự quản lý name server và các bản ghi DNS dựa trên những khái niệm này.
