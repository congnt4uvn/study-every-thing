


Dưới đây là tóm tắt nội dung:

1. Vào phần Security Groups  
   - Trong EC2 Console, vào mục “Network & Security” → “Security Groups”.  
   - Thấy 2 security group: “default” và “launch-wizard-…” (tạo khi khởi chạy EC2).  
   - Mỗi security group có một ID riêng.

2. Inbound rules (quy tắc vào)  
   - Inbound rules quyết định kết nối từ bên ngoài vào EC2.  
   - Ví dụ đang có 2 rule:
     - SSH (port 22) từ Anywhere (0.0.0.0/0).  
     - HTTP (port 80) từ Anywhere (0.0.0.0/0).  
   - Nhờ rule HTTP port 80 mà có thể truy cập web server qua IPv4 Public IP.

3. Minh họa lỗi timeout do security group  
   - Khi xóa rule HTTP (port 80) và lưu lại:
     - Truy cập lại website sẽ bị “loading” rất lâu rồi lỗi → đó là “timeout”.  
   - Bất cứ khi nào:
     - SSH bị timeout,  
     - HTTP bị timeout,  
     - hoặc bất kỳ kết nối nào đến EC2 bị timeout,  
     → gần như chắc chắn là do cấu hình security group (inbound rule) sai.  
   - Cách xử lý: quay lại security group, kiểm tra và chỉnh lại inbound rules.

4. Thêm lại rule HTTP  
   - Thêm inbound rule:
     - Type: HTTP → tự động port 80.  
     - Source: Anywhere (0.0.0.0/0).  
   - Lưu rule, refresh lại trang web → hoạt động bình thường.

5. Thêm các loại rule khác  
   - Có thể chỉ định port hoặc range port tùy ý (ví dụ 443 cho HTTPS).  
   - Có thể chọn sẵn từ dropdown (HTTPS → port 443).  
   - Source có thể là:
     - CIDR bất kỳ (custom),  
     - “My IP” (chỉ cho phép IP hiện tại của bạn).  
   - Lưu ý: nếu IP của bạn thay đổi, rule “My IP” sẽ không còn đúng → lại bị timeout.

6. Outbound rules (quy tắc ra)  
   - Ví dụ: cho phép “All traffic” IPv4 đến “Anywhere”.  
   - Điều này cho phép EC2 có kết nối Internet đầy đủ ra ngoài.

7. Gắn nhiều security group  
   - Một EC2 instance có thể gắn nhiều security group (1, 2, 3, …).  
   - Các rule của các security group sẽ cộng dồn (union).  
   - Một security group (ví dụ launch-wizard) có thể gắn cho nhiều EC2 instance khác nhau.

Tóm lại: Security Group là “tường lửa” cho EC2. Nếu bị timeout khi kết nối, hãy kiểm tra inbound rules (đúng port, đúng nguồn hay chưa).
