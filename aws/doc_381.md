


Nội dung trên nói về việc dùng file cấu hình `samconfig.toml` để quản lý nhiều môi trường (dev, prod) khi triển khai với AWS SAM CLI:

- Nhà phát triển có một template SAM và thêm file `samconfig.toml` (định dạng TOML).
- Trong file này, khai báo các cấu hình khác nhau cho từng môi trường/ngăn xếp:
  - Ví dụ: `dev.deploy.parameters` cho môi trường dev, trong đó cấu hình:
    - Tên stack
    - S3 bucket, prefix
    - Region
    - Capabilities
    - Tham số override (ví dụ đặt biến môi trường là “development”)
    - Tham số cho lệnh `sam sync` nếu cần
  - Tương tự: `prod.deploy.parameters` cho môi trường prod, với:
    - Tên stack, S3 bucket, region riêng
    - Tham số override để chỉ ra môi trường là “production”
    - Tham số cho `sam sync` cho prod

- Sau khi cấu hình xong:
  - Dùng `sam deploy --config-env dev` để SAM CLI tự đọc `samconfig.toml`, lấy đúng tham số và triển khai vào môi trường dev.
  - Dùng `sam deploy --config-env prod` để triển khai vào môi trường prod.
  - Có thể định nghĩa thêm nhiều môi trường khác nếu cần.

Mục đích: giúp quản lý và triển khai nhiều môi trường (dev, prod, …) dễ dàng, nhất quán, chỉ cần đổi `--config-env`.
