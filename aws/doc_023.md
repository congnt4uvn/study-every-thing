


Dưới đây là tóm tắt nội dung:

- CloudShell là một giải pháp thay thế cho việc dùng terminal cục bộ để chạy lệnh AWS CLI.  
- Biểu tượng CloudShell nằm ở góc trên bên phải console AWS, nhưng không khả dụng ở tất cả region; chỉ một số region hỗ trợ.  
- Khi dùng CloudShell:
  - AWS CLI đã được cài sẵn (ví dụ `aws --version`), và dùng chính credential của tài khoản bạn đang đăng nhập.
  - Region mặc định của CLI trong CloudShell là region mà bạn đang chọn trên console.
- CloudShell có một vùng lưu trữ “bền vững”:
  - Các file bạn tạo (ví dụ `echo "tests" > demo.txt`) vẫn còn sau khi khởi động lại CloudShell.
- Có thể cấu hình giao diện CloudShell:
  - Thay đổi kích thước font (nhỏ, vừa, lớn).
  - Thay đổi theme (sáng, tối, v.v.).
- Hỗ trợ upload/download file:
  - Bạn có thể tải file từ CloudShell về máy (download).
  - Bạn cũng có thể upload file từ máy lên môi trường CloudShell.
- Có thể mở nhiều tab hoặc chia cột để có nhiều terminal CloudShell chạy song song.
- Kết luận:
  - Nên chọn region có hỗ trợ CloudShell nếu muốn dùng.
  - Nếu không dùng được CloudShell, chỉ cần dùng terminal cục bộ đã cấu hình AWS CLI là đủ.
  - CloudShell đặc biệt hữu ích nhờ khả năng upload/download file và môi trường CLI sẵn sàng.
