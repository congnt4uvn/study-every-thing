


Người tường thuật đang hướng dẫn quy trình thiết lập CI/CD với AWS CodeBuild và CodePipeline cho một ứng dụng Node.js:

1. **Tạo file buildspec.yml cho CodeBuild**  
   - Thêm file `buildspec.yml` vào repo GitHub (tạo trực tiếp trên GitHub).  
   - Nội dung file định nghĩa:
     - `version: 0.2`
     - Sử dụng runtime Node.js mới nhất (ví dụ Node 20).
     - Các phase: `install`, `pre_build`, `build`, `post_build`.
   - Trong phase `build`, có lệnh test:
     - `grep -Fq "Congratulations" index.html`  
       → Build chỉ thành công nếu trong `index.html` có chữ “Congratulations”. Đây chính là bài test.

2. **Chạy CodeBuild và xem log**  
   - Commit file `buildspec.yml` lên nhánh main.  
   - Webhook GitHub–CodeBuild tự động kích hoạt một build mới.  
   - Build chạy qua các giai đoạn: submitted, queued, provisioning, download_source, install, pre_build, build, post_build, upload_artifacts, completed.  
   - Xem log chi tiết trong CloudWatch Logs:
     - Tải source từ GitHub.
     - Cài Node.js 20.
     - Phát hiện 4 phase trong buildspec (INSTALL 1, PRE_BUILD 1, BUILD 3, POST_BUILD 1).
     - Chạy các lệnh echo, rồi lệnh `grep` kiểm tra chữ “Congratulations” trong `index.html`.
     - Test pass → build thành công.

3. **Chuẩn bị tích hợp với CodePipeline**  
   - Sửa project CodeBuild: xóa webhook nguồn chính (GitHub) để sau này CodePipeline là bên kích hoạt build, không phải GitHub trực tiếp.

4. **Chỉnh sửa CodePipeline để thêm bước test**  
   - Mở pipeline `MyFirstPipeline` và chỉnh sửa:
     - Trước đây: Source → DeployToDev → DeployToProd.
     - Bây giờ thêm stage mới: `TestCode`.
   - Trong stage `TestCode`:
     - Thêm action `CodeBuildTest`.
     - Provider: CodeBuild.
     - Input artifact: `SourceArtifact`.
     - Project: `MyFirstBuild` (project CodeBuild đã tạo).
     - Output artifact: `OutputOfTest`.
   - Kết quả luồng mới:  
     **Source → TestCode (CodeBuild) → DeployToDev → DeployToProd**.

5. **Minh họa test thất bại**  
   - Mở `index.html` trong repo `my-nodejs-app`.
   - Thay nội dung từ “Congratulations” thành “Terrible. Let’s go.”  
   - Commit trực tiếp lên nhánh main.
   - Commit này kích hoạt pipeline:
     - Stage Source chạy.
     - Stage TestCode (CodeBuild) chạy, nhưng:
       - Lệnh `grep -Fq "Congratulations" index.html` thất bại vì chữ “Congratulations” đã bị xóa.
       - Build fail.
   - Trong CodePipeline:
     - Stage TestCode bị đỏ (failed).
     - Pipeline dừng, không chuyển sang Deploy.
   - Kết quả:
     - Dev/Prod không bị deploy phiên bản “Terrible”.
     - Khi refresh môi trường Dev, vẫn thấy “Congratulations” (bản cũ).

6. **Sửa lại để test pass và deploy**  
   - Sửa lại `index.html`:
     - Thay “Terrible” bằng “Congratulations CodeBuild” (vẫn chứa từ “Congratulations”).
   - Commit lên main.
   - Pipeline chạy lại:
     - Source → TestCode (CodeBuild).
     - Lần này `grep "Congratulations"` pass → build thành công.
     - Pipeline tiếp tục sang stage Deploy:
       - Deploy lên môi trường Elastic Beanstalk (Dev).
   - Vào Elastic Beanstalk, refresh:
     - Thấy nội dung mới: “Congratulations CodeBuild”.
   - Sau đó cần phê duyệt thủ công (manual approval) để deploy tiếp lên Prod.

7. **Ý nghĩa**  
   - Minh họa trọn vẹn quy trình CI/CD:
     - Mỗi lần code thay đổi, pipeline tự chạy.
     - Test tự động (CodeBuild) quyết định có cho phép deploy hay không.
     - Nếu test fail, thay đổi không được đưa vào môi trường Dev/Prod.
   - Người tường thuật nhấn mạnh tầm quan trọng của việc xem log (UI CodeBuild hoặc CloudWatch) để debug khi có lỗi, và kết luận rằng đây là sức mạnh của CI/CD: tự động hóa build, test, deploy.
