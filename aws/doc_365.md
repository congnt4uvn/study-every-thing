


Dưới đây là tóm tắt nội dung:

- Mục tiêu: Dùng AWS CodeBuild không phải để build sản phẩm, mà để kiểm tra xem thông báo “Congratulations” có còn xuất hiện khi triển khai ứng dụng hay không.

- Tạo Build Project:
  - Vào CodeBuild → Build projects → Create build project.
  - Đặt tên: `MyFirstBuild`.
  - Bỏ qua Description, Build badge, Tags.

- Cấu hình Source:
  - Chọn nguồn là GitHub.
  - Kết nối bằng OAuth → “Connect to GitHub” → ủy quyền cho CodeSuite.
  - Sau khi kết nối, chọn repository ứng dụng Node.js.
  - Build trực tiếp từ nhánh `main`.
  - Bật tùy chọn “Rebuild every time a code change is pushed to this repository”.
  - Chọn loại build là “Single build”.
  - Chỉ cấu hình trigger cho sự kiện `PUSH` (không cấu hình cho Pull request hay Release, nhưng giảng viên giải thích có thể dùng CodeBuild để build/test khi có Pull request – đó là một use case hợp lệ).

- Môi trường build:
  - Chọn “Managed image”.
  - OS: Ubuntu.
  - Runtime: Standard.
  - Image: `standard:7.0` (luôn dùng image mới nhất).
  - Tạo Service role mới: `CodeBuildDemoServiceRole`.

- Cấu hình bổ sung:
  - Timeout mặc định là 1 giờ, có thể chỉnh (ví dụ 10 phút).
  - Có thể gán VPC để build có thể truy cập DB hoặc tài nguyên trong VPC (ở đây không dùng, nhưng được nhấn mạnh là tùy chọn quan trọng).
  - Chọn kích thước compute: 3 GB RAM, 2 vCPU.
  - Có thể thêm Environment variables (hiện tại không cần).

- Buildspec:
  - Chọn “Use a buildspec file”.
  - Tên file: `buildspec.yml` (hoặc `buildspec.yaml`).
  - Cần đảm bảo file này nằm ở thư mục gốc của source code.

- Artifacts & Logs:
  - Hiện tại không cấu hình artifacts (không upload output lên S3 vì chưa build gì cụ thể).
  - Logs có thể gửi lên CloudWatch Logs hoặc S3.

- Tạo và chạy build:
  - Tạo xong project, bắt đầu một build.
  - Build thất bại (expected).
  - Lý do: Không tìm thấy file `buildspec.yml` trong repo, nên CodeBuild không có hướng dẫn để chạy.
  - Giảng viên nói sẽ thêm file buildspec trong bài sau.

- Kết luận:
  - Đã thiết lập xong CodeBuild project kết nối với GitHub.
  - Sẵn sàng cho bước tiếp theo trong bài thực hành (thêm buildspec và kiểm tra thông báo “Congratulations”).
