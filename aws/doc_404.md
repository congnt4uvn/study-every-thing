


Giảng viên hướng dẫn thực hành tạo một API với AWS AppSync như sau:

1. **Tạo API AppSync**
   - Chọn loại **GraphQL API** (không chọn API hợp nhất).
   - Thiết kế schema **từ đầu** (không nhập từ DynamoDB, không tạo API thời gian thực).
   - Đặt tên API, ví dụ: **My AppSync API**.

2. **Thiết kế mô hình dữ liệu (schema GraphQL)**
   - Tạo một **model** tên `Student`, được hỗ trợ bởi một bảng DynamoDB.
   - Các trường:
     - `id`: kiểu `ID`, bắt buộc.
     - `name`: kiểu `String`, bắt buộc.
     - `age`: kiểu `Int`, không bắt buộc.
     - `certified`: kiểu `Boolean`, không bắt buộc.
   - Cấu hình bảng DynamoDB:
     - Tên bảng: **StudentTable**.
     - Khóa chính: `id`.
     - Không dùng khóa sắp xếp.

3. **Tạo API và bảng DynamoDB**
   - Sau khi hoàn tất, AppSync tự động tạo:
     - **Schema GraphQL** tương ứng.
     - **Bảng DynamoDB** `StudentTable`.
   - Có thể vào DynamoDB để kiểm tra bảng đã được tạo.

4. **Sử dụng API (Queries & Mutations)**
   - Trong phần **Queries** của AppSync console:
     - Có các mutation như:
       - `createStudent`
       - `listStudents`
   - Ban đầu `listStudents` trả về rỗng vì chưa có dữ liệu.
   - Tạo dữ liệu bằng `createStudent`, ví dụ:
     - `name: "Mike"`, `age: 25`, `certified: true`
     - `name: "Alice"`, `age: 30`, `certified: true`
   - Sau đó chạy `listStudents` sẽ thấy danh sách sinh viên (Mike, Alice).
   - Kiểm tra lại trong DynamoDB sẽ thấy các item tương ứng đã được lưu.

5. **Ý nghĩa của AppSync + DynamoDB**
   - AppSync cung cấp một **API GraphQL đầy đủ** nằm trên bảng DynamoDB.
   - API này có thể được dùng bởi:
     - Ứng dụng web (JavaScript).
     - Ứng dụng di động (Android, iOS).
   - Mục tiêu: tạo một API thống nhất cho cả web và mobile.

6. **Cấu hình bổ sung trong AppSync**
   - **Caching**:
     - Có thể bật cache toàn bộ request, cache theo resolver, hoặc không dùng cache.
   - **Cài đặt API**:
     - Xem endpoint GraphQL, API ID, endpoint real-time, v.v.
   - **Chế độ ủy quyền (Authorization modes)**:
     - API Key (đang dùng trong ví dụ).
     - IAM.
     - OpenID Connect.
     - Lambda Authorizer.
     - Amazon Cognito User Pools.
     - Có thể cấu hình nhiều nhà cung cấp ủy quyền cùng lúc.
   - **Monitoring**:
     - Theo dõi số request, lỗi, v.v.
   - **Custom domain**:
     - Có thể gắn API với tên miền tùy chỉnh.

7. **Dọn dẹp tài nguyên**
   - Xóa API AppSync:
     - Vào AppSync, chọn API, nhập lại tên (ví dụ: “My AppSync API”) để xác nhận xóa.
   - Xóa bảng DynamoDB:
     - Vào DynamoDB, chọn `StudentTable`, chọn Delete table.
   - Sau khi xóa cả API và bảng là hoàn tất thực hành.

Tóm lại: bài giảng minh họa cách dùng AWS AppSync để tạo một API GraphQL đơn giản, được “backup” bởi DynamoDB, rồi sử dụng các mutation/query để thao tác dữ liệu, và giới thiệu sơ lược về các tính năng cấu hình, ủy quyền, cache, monitoring và cách dọn dẹp tài nguyên.
