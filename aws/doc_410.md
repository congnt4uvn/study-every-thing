


Dưới đây là tóm tắt ngắn gọn nội dung bạn đưa:

1. Microsoft Active Directory (AD) là gì  
- Có trên các máy chủ Windows chạy dịch vụ Domain Services.  
- Là cơ sở dữ liệu chứa các đối tượng: user, computer, printer, file share, security group…  
- Quản lý tập trung người dùng, quyền, bảo mật trong môi trường Microsoft.  
- Cấu trúc dạng cây (tree), nhiều cây tạo thành forest.  
- Domain Controller (DC) xác thực người dùng: mọi máy Windows trong domain đều gửi yêu cầu đăng nhập về DC để kiểm tra username/password.

2. AWS Directory Service – 3 “hương vị” chính  
Mục tiêu: cung cấp AD trên AWS, phục vụ các instance Windows/ứng dụng cần AD.

a) AWS Managed Microsoft AD  
- AWS triển khai và quản lý một Microsoft AD thực thụ trong AWS.  
- Bạn quản lý user cục bộ trong AD này, hỗ trợ MFA.  
- Có thể thiết lập quan hệ trust với AD on-premises (tại chỗ).  
  - AD trên AWS và AD on-premises tin cậy lẫn nhau.  
  - Người dùng on-prem có thể được dùng để truy cập tài nguyên trên AWS và ngược lại (tùy trust).  
- Phù hợp khi muốn quản lý user trên cloud, có MFA, và/hoặc cần tích hợp với AD tại chỗ.

b) AD Connector  
- Là một proxy/gateway, không phải AD độc lập.  
- Không lưu user trong AWS, chỉ chuyển tiếp (proxy) yêu cầu xác thực từ AWS về AD on-premises.  
- Người dùng chỉ được quản lý tại AD on-prem.  
- Hỗ trợ MFA.  
- Phù hợp khi đã có AD tại chỗ và chỉ muốn dùng nó để xác thực cho tài nguyên AWS, không muốn duy trì thêm AD mới trên cloud.

c) Simple AD  
- Thư mục được quản lý, tương thích API với AD, nhưng không phải Microsoft AD thật.  
- Không thể thiết lập trust với AD on-premises.  
- Dùng như một AD độc lập trong AWS, phù hợp khi:
  - Không có AD tại chỗ.  
  - Cần một directory đơn giản cho các EC2 Windows, chia sẻ thông tin đăng nhập, join domain…  

3. Các lựa chọn trong AWS Directory Service Console  
- AWS Managed Microsoft AD:  
  - Bản Standard: ~30.000 đối tượng.  
  - Bản Enterprise: ~500.000 đối tượng.  
- Simple AD: AD tương thích, độc lập, không trust on-prem.  
- AD Connector: proxy tới AD on-prem, có 2 mức: ~500 user và ~5.000 user.  
- Amazon Cognito User Pool: xuất hiện trong giao diện nhưng là dịch vụ khác, không tính là Directory Service truyền thống.

4. Cách ra đề trong kỳ thi (ý chính)  
- “Muốn ủy quyền cho user tại chỗ (on-prem)”: dùng AD Connector.  
- “Muốn quản lý user trên cloud, có MFA, có thể trust với on-prem”: dùng AWS Managed Microsoft AD.  
- “Không có AD on-prem, chỉ cần directory đơn giản trên AWS”: dùng Simple AD.
