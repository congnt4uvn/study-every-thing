


Dưới đây là tóm tắt nội dung bài giảng:

1. **Dọn dẹp (xóa dữ liệu) trong bảng DynamoDB**  
   - Cách 1: **Quét (Scan) toàn bộ bảng rồi xóa từng item**  
     - Rất chậm.  
     - Tốn nhiều RCU (đọc) và WCU (ghi) → chi phí cao.  
   - Cách 2: **Xóa (drop) cả bảng rồi tạo lại**  
     - Nhanh hơn, hiệu quả và rẻ hơn nhiều.  
     - Cần đảm bảo tạo lại bảng với đúng cấu hình như bảng cũ (key, index, throughput, v.v.).

2. **Sao chép (copy) bảng DynamoDB**  
   - Dùng **AWS Backup**:  
     - Tạo backup từ bảng nguồn.  
     - Khôi phục (restore) sang cùng tài khoản hoặc tài khoản khác.  
   - Dùng **AWS Glue (ETL)**:  
     - Glue tạo script để đọc dữ liệu từ bảng nguồn.  
     - Ghi dữ liệu đến nơi bạn muốn (bảng khác, dịch vụ khác, v.v.).  
   - **Tự viết code**:  
     - Dùng API như `Scan`, `PutItem`, `BatchWriteItem` để đọc và ghi dữ liệu.  
     - Linh hoạt nhưng phức tạp hơn so với dùng AWS Backup hoặc Glue.

Kết luận:  
- Để dọn sạch bảng: nên xóa và tạo lại bảng thay vì quét rồi xóa từng item.  
- Để sao chép bảng: ưu tiên dùng AWS Backup hoặc AWS Glue; tự viết code chỉ là lựa chọn cuối cùng.
