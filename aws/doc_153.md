


Dưới đây là tóm tắt ngắn gọn nội dung về CloudFront:

1. Khái niệm & mục đích  
- CloudFront là dịch vụ CDN của AWS.  
- Cải thiện hiệu suất đọc bằng cách cache (lưu trữ tạm) nội dung tại các Edge Location (điểm biên) trên toàn thế giới.  
- Nhờ đó người dùng ở mọi nơi truy cập với độ trễ thấp hơn, trải nghiệm tốt hơn.

2. Cấu trúc & bảo mật  
- CloudFront có hàng trăm điểm hiện diện (PoP), gồm Edge Location và Edge Cache.  
- Phân tán lưu lượng giúp giảm rủi ro và hỗ trợ bảo vệ trước tấn công DDoS.  
- Tích hợp với AWS Shield và AWS WAF để tăng cường bảo mật.

3. Cách hoạt động (caching)  
- Người dùng gửi request đến Edge Location gần nhất.  
- Edge kiểm tra cache:  
  - Nếu có sẵn: trả về ngay từ Edge.  
  - Nếu chưa có: Edge truy vấn Origin (nguồn gốc) → lấy dữ liệu → cache lại tại Edge → trả về cho client.  
- Ví dụ: S3 bucket ở Úc, người dùng ở Mỹ/Trung Quốc/Brazil sẽ truy cập qua Edge gần họ; lần đầu Edge lấy từ S3, các lần sau phục vụ trực tiếp từ cache.

4. Các loại Origin (nguồn gốc)  
- S3 bucket: phân phối file tĩnh, có thể upload qua CloudFront.  
- VPC origin: các dịch vụ trong VPC (ALB/NLB private, EC2 private, ứng dụng nội bộ).  
- Custom origin (HTTP):  
  - S3 static website (bật hosting tĩnh).  
  - Bất kỳ web server/Load Balancer công khai nào.  
- Kết nối CloudFront–S3 được bảo vệ bằng Origin Access Control (OAC) và policy của S3 bucket.

5. CloudFront vs S3 Cross-Region Replication (CRR)  
- CloudFront:  
  - CDN toàn cầu (~216 PoP).  
  - Cache file tại Edge trong thời gian ngắn (ví dụ ~1 ngày).  
  - Phù hợp cho nội dung tĩnh cần phân phối toàn cầu.  
- S3 replication giữa các Region:  
  - Phải cấu hình cụ thể từng cặp Region.  
  - Dữ liệu được sao chép gần như real-time, không phải cache.  
  - Thường dùng cho nội dung động cần cập nhật liên tục, chỉ đọc ở Region đích.  
- Mục đích khác nhau:  
  - CloudFront = phân phối & cache nội dung (CDN).  
  - S3 replication = sao chép dữ liệu giữa các Region.
