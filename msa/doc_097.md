
Dưới đây là tóm tắt nội dung:

1. Mục tiêu phần mới  
   - Tạm dừng các “thử thách” về microservices.  
   - Chuyển trọng tâm sang sử dụng MySQL làm cơ sở dữ liệu thực cho 3 microservice: accounts, loans, cards.  
   - Thay thế H2 (chỉ phù hợp cho demo, không khuyến nghị cho môi trường thực tế/production).

2. Chuẩn bị mã nguồn cho phần 7  
   - Trong workspace hiện có thư mục section 6 với v1 và v2.  
   - Tạo thư mục mới section 7.  
   - Sao chép mã từ v2-spring-cloud-config sang section 7.  
   - Mở section 7 trong IntelliJ và load tất cả các project Maven (trừ thư mục docker-compose chỉ chứa file compose).

3. Loại bỏ phụ thuộc RabbitMQ / Spring Cloud Bus  
   - Lý do:
     - Không muốn tiếp tục mang theo phụ thuộc RabbitMQ (spring-cloud-bus, config monitor) để tránh phải chạy thêm container, giảm tải cho máy yếu (8GB RAM).  
     - Đã hiểu cơ chế Spring Cloud Bus và Config Monitor, nên các phần sau không cần dùng nữa.  
   - Thực hiện:
     - Trong config server pom.xml:
       - Xóa dependency: spring-cloud-starter-bus-amqp.  
       - Xóa dependency: spring-cloud-config-monitor.  
       - Reload Maven.  
     - Trong config server application.yml:
       - Xóa các cấu hình liên quan đến RabbitMQ.  
       - Giữ lại các cấu hình actuator (management) và health (readiness, liveness).  
     - Trong accounts, cards, loans microservice:
       - Mỗi service: mở pom.xml, xóa dependency spring-cloud-starter-bus-amqp, reload Maven.  
       - Mở application.yml, xóa các thuộc tính liên quan đến RabbitMQ.  
   - Kết quả:
     - Không cần chạy container RabbitMQ nữa.  
     - Nếu cần refresh config, có thể gọi thủ công endpoint actuator /refresh trên từng instance.

4. Chuẩn bị MySQL bằng Docker thay vì cài trực tiếp  
   - Lý do:
     - Tránh cài MySQL trực tiếp lên máy (tốn thời gian và dung lượng).  
     - Dùng Docker để chạy MySQL như một container, dễ bật/tắt, xóa khi không dùng.  
     - Mỗi microservice nên có database riêng: accountsDB, loansDB, cardsDB.  

5. Tạo MySQL cho accounts microservice  
   - Đảm bảo Docker đang chạy.  
   - Mở terminal, chạy lệnh (ý chính):  
     - docker run -p 3306:3306 --name accountsdb -e MYSQL_ROOT_PASSWORD=root -e MYSQL_DATABASE=accountsdb -d mysql  
   - Giải thích:
     - -p 3306:3306: expose cổng 3306 của container ra cổng 3306 trên máy local.  
     - --name accountsdb: đặt tên container.  
     - MYSQL_ROOT_PASSWORD=root: mật khẩu user root là root.  
     - MYSQL_DATABASE=accountsdb: tạo sẵn database accountsdb trong container.  
   - Kiểm tra:
     - Container accountsdb đang chạy.  
     - MySQL trong container lắng nghe trên cổng 3306.

6. Kết nối MySQL bằng SQL Electron  
   - Giới thiệu SQL Electron: client GUI nhẹ, hỗ trợ nhiều loại DB, có thể kết nối DB local hoặc cloud.  
   - Tạo kết nối mới:
     - Name: LocalAccountsDatabase (hoặc tương tự).  
     - Type: MySQL.  
     - Host: localhost.  
     - Port: 3306.  
     - User: root.  
     - Password: root.  
   - Test connection → thành công → Save → Connect.  
   - Thấy database accountsdb đã được tạo, hiện chưa có bảng.

7. Tạo MySQL cho loans microservice  
   - Dùng lại lệnh docker run, nhưng:
     - MYSQL_DATABASE=loansdb.  
     - --name loansdb.  
   - Vấn đề:
     - Nếu vẫn dùng -p 3306:3306 sẽ lỗi vì cổng 3306 trên máy đã dùng cho accountsdb.  
   - Cách xử lý:
     - Đổi port mapping, ví dụ: -p 3307:3306.  
     - Nếu trước đó đã tạo container nhưng lỗi port, cần xóa container cũ rồi chạy lại lệnh.  
   - Giải thích:
     - Bên trong mỗi container, MySQL vẫn chạy trên 3306 (cổng nội bộ).  
     - Bên ngoài (host), mỗi container phải dùng cổng khác nhau: 3306, 3307, 3308…  
   - Kết nối loansdb bằng SQL Electron:
     - Name: LocalLoansDatabase.  
     - Type: MySQL.  
     - Host: localhost.  
     - Port: 3307.  
     - User: root, Password: root.  
     - Test → Save → Connect → thấy loansdb đã được tạo.

8. Tạo MySQL cho cards microservice  
   - Tương tự loans:
     - MYSQL_DATABASE=cardsdb.  
     - --name cardsdb.  
     - Port mapping: -p 3308:3306.  
   - Kết nối bằng SQL Electron:
     - Name: LocalCardsDatabase.  
     - Type: MySQL.  
     - Host: localhost.  
     - Port: 3308.  
     - User: root, Password: root.  
     - Test → Save → Connect → thấy cardsdb đã được tạo.

9. Trạng thái cuối cùng của phần này  
   - Đã:
     - Loại bỏ phụ thuộc Spring Cloud Bus và RabbitMQ khỏi config server và 3 microservice.  
     - Tạo 3 container MySQL riêng biệt bằng Docker:
       - accountsdb → exposed port 3306.  
       - loansdb → exposed port 3307.  
       - cardsdb → exposed port 3308.  
     - Kết nối và xác nhận 3 database tồn tại qua SQL Electron.  
   - Bước tiếp theo (sẽ làm ở bài giảng sau):
     - Chỉnh sửa code trong các microservice để:
       - Ngừng dùng H2.  
       - Kết nối tới các MySQL container tương ứng (accounts, loans, cards).
