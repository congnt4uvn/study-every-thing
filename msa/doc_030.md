
Dưới đây là tóm tắt ngắn gọn nội dung:

1. Trước đây:
   - Cùng một mã lỗi HTTP 500 (Internal Server Error) được dùng cho:
     - RuntimeException (xử lý trong GlobalExceptionHandler).
     - Thao tác update/delete khi thất bại (boolean trả về false).
   - Điều này gây nhầm lẫn cho client vì mọi lỗi đều là 500.

2. Thay đổi mới:
   - Không đổi gì trong GlobalExceptionHandler cho RuntimeException (vẫn 500).
   - Trong controller:
     - Với update:
       - Nếu boolean là false → trả về HTTP status 417 (Expectation Failed).
       - Tạo hằng số STATUS_417 và message riêng cho update: “Update operation failed. Please try again or contact development team.”
     - Với delete:
       - Nếu boolean là false → cũng trả về 417.
       - Tạo message riêng cho delete với nội dung tương tự.
   - Không còn dùng hằng số status 500 và message 500 cho các trường hợp update/delete thất bại (chỉ giữ lại nếu cần tham khảo).

3. Cập nhật ApiResponse (OpenAPI/Swagger):
   - Tất cả API đều có ApiResponse cho:
     - 500: dùng ErrorResponseDto (cho RuntimeException).
   - Riêng:
     - Update: có 3 response:
       - 204 (thành công, không nội dung),
       - 417 (update thất bại),
       - 500 (RuntimeException).
     - Delete: tương tự có 3 response: 204, 417, 500.

4. Vấn đề với ví dụ (example) trong ResponseDto:
   - Trong ResponseDto, ví dụ (example) cho statusCode là 200 và message là “request processed successfully”.
   - Khi hiển thị trên UI (Swagger), với response 417 vẫn thấy example là 200 → gây nhầm lẫn cho client.
   - Giải pháp: xóa các example này khỏi ResponseDto.
   - Sau khi xóa:
     - Swagger hiển thị kiểu dữ liệu chung (statusCode: String, statusMessage: String) mà không gắn cứng 200.
     - Tất cả API có hai dạng response chính:
       - ResponseDto cho thành công.
       - ErrorResponseDto cho RuntimeException.
       - Riêng update/delete có thêm 417 cho trường hợp thao tác không thành công.

5. Kết luận:
   - Phân biệt rõ:
     - 500: lỗi hệ thống / RuntimeException.
     - 417: thao tác nghiệp vụ (update/delete) thất bại.
   - Loại bỏ example cứng 200 trong ResponseDto để tránh gây hiểu nhầm trên UI.
