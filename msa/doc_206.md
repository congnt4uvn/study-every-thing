
Dưới đây là tóm tắt nội dung bài giảng:

1. Tạo project Spring Boot mới  
   - Vào trang start.spring.io.  
   - Chọn:  
     - Project: Maven  
     - Language: Java  
     - Spring Boot version: 3.1.2 (hoặc bản ổn định mặc định)  
   - Metadata:  
     - Group: `com.easybytes`  
     - Artifact & Name: `message`  
     - Description: “Microservice hỗ trợ nhắn tin trong EazyBank”  
     - Packaging: `jar`  
     - Java version: 17  

2. Thêm dependency Spring Cloud Function  
   - Tìm và chọn dependency “Function” (Spring Cloud Function).  
   - Dependency này thêm:  
     - `spring-boot-starter`  
     - `spring-cloud-function-context` (rất quan trọng để dùng Spring Cloud Function)  
     - Một số dependency test.  
   - Giải thích: Spring Cloud Function cho phép viết logic nghiệp vụ dưới dạng hàm, có thể chạy trên nhiều nhà cung cấp serverless khác nhau hoặc chạy như ứng dụng độc lập (mô hình lập trình thống nhất).

3. Chuẩn bị workspace và project  
   - Trong workspace, tạo thư mục mới cho “section 13” bằng cách copy thư mục “section 12” (chứa code phần trước) rồi đổi tên thành “section13”.  
   - Xóa thư mục `.idea`.  
   - Giải nén project Maven `message` vừa tải về và dán vào trong `section13`.  
   - Mở IntelliJ, chọn “Open”, trỏ đến thư mục `section13`, load tất cả project Maven trong đó.

4. Tạo DTO cho message service  
   - Trong project `message`, tạo package: `com.eazybytes.message.dto`.  
   - Tạo một `record` Java tên `AccountsMessageDto`.  
   - Các field trong record:  
     - `accountNumber`  
     - `customerName`  
     - `email`  
     - `mobileNumber`  
   - Mục đích:  
     - Service account sẽ gửi message lên message broker theo cấu trúc DTO này.  
     - Message service nhận DTO này để xử lý và gửi email/SMS.  
   - Giải thích về `record`:  
     - Tự động tạo getter, các field là `final`, dữ liệu bất biến sau khi tạo object.  
   - Thêm Javadoc để tránh cảnh báo.

5. Tạo lớp chứa các hàm (business logic)  
   - Tạo package: `com.eazybytes.message.functions`.  
   - Tạo class `MessageFunctions`.  
   - Đánh dấu class với `@Configuration`.  
   - Đây sẽ là nơi định nghĩa các hàm Spring Cloud Function để xử lý logic nhắn tin cho người dùng cuối.  
   - Phần viết cụ thể các hàm sẽ được trình bày trong bài giảng tiếp theo.
