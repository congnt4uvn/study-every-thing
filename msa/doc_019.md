
Dưới đây là tóm tắt ngắn gọn nội dung bạn gửi:

1. **AccountServiceImpl – tạo khách hàng và tài khoản**
   - Nhận `CustomerDto` từ client.
   - Dùng `CustomerMapper` để map `CustomerDto` → entity `Customer`.
   - Gọi `customerRepository.save(customer)` để lưu khách hàng vào DB.
   - Kết quả `save()` trả về `savedCustomer` có `customerId` được sinh tự động.
   - Tạo tài khoản ngân hàng mới cho khách hàng này:
     - Viết một method private `newAccount(Customer customer)`:
       - Lấy `customerId` từ `customer`.
       - Tạo số tài khoản ngẫu nhiên 10 chữ số (developer tự sinh).
       - Set `accountNumber`, `accountType` (ví dụ: SAVINGS), `branchAddress` (“123 Main Street, New York”).
     - Gọi `accountsRepository.save(account)` để lưu tài khoản.

2. **Ràng buộc số điện thoại di động & ngoại lệ tùy chỉnh**
   - Yêu cầu: mỗi `mobileNumber` chỉ được dùng cho một khách hàng.
   - Tạo package `exception`.
   - Tạo custom exception `CustomerAlreadyExistsException`:
     - Kế thừa `RuntimeException`.
     - Có constructor nhận `message` và gọi `super(message)`.
     - Annotate `@ResponseStatus(HttpStatus.BAD_REQUEST)` → trả về HTTP 400.

3. **Truy vấn khách hàng theo số điện thoại**
   - Trong `CustomerRepository`:
     - Định nghĩa method: `Optional<Customer> findByMobileNumber(String mobileNumber);`
     - Đây là “derived query method” của Spring Data JPA: dựa vào tên `findByMobileNumber` để tự sinh SQL.
   - Trong `AccountServiceImpl`:
     - Gọi `customerRepository.findByMobileNumber(customerDto.getMobileNumber())`.
     - Nếu `optionalCustomer.isPresent()`:
       - Ném `CustomerAlreadyExistsException("Customer already registered with given mobile number")`.

4. **Global Exception Handling**
   - Tạo lớp `GlobalExceptionHandler` trong package `exception`.
   - Annotate `@ControllerAdvice` để xử lý exception toàn cục cho tất cả controller.
   - Viết method, ví dụ:
     - `handleCustomerAlreadyExistingException(CustomerAlreadyExistsException ex, WebRequest request)`
     - Annotate `@ExceptionHandler(CustomerAlreadyExistsException.class)`.
     - Tạo `ErrorResponseDto` chứa:
       - `path` (lấy từ `WebRequest.getDescription(false)`),
       - `HttpStatus.BAD_REQUEST`,
       - `message` từ exception,
       - `timestamp` hiện tại.
     - Trả về `ResponseEntity<ErrorResponseDto>` với status 400.
   - Nhờ đó, mọi nơi ném `CustomerAlreadyExistsException` sẽ được xử lý tập trung, không cần try–catch lặp lại trong từng controller.

5. **AccountsController & dependency injection**
   - Trong `AccountsController`:
     - Khai báo field `private AccountService accountService;`
     - Dùng Lombok `@AllArgsConstructor` để tạo constructor injection (không dùng `@Autowired` trực tiếp).
   - Trong handler tạo tài khoản:
     - Gọi `accountService.createAccount(customerDto)`.
     - Nếu không có exception → trả về `ResponseEntity` với status 201 và message “Account created successfully”.
     - Nếu có `CustomerAlreadyExistsException` → không lên tới controller, mà được `GlobalExceptionHandler` xử lý.

6. **Xử lý lỗi thiếu trường audit (createdAt, createdBy)**
   - Lần đầu gọi API tạo tài khoản bị lỗi vì cột `createdAt`, `createdBy` không cho phép null.
   - Sửa trong service:
     - Trước khi `customerRepository.save(customer)`:
       - `customer.setCreatedAt(LocalDateTime.now());`
       - `customer.setCreatedBy("anonymous");`
     - Tương tự cho entity `Account` (set `createdAt`, `createdBy`).
   - Sau này có thể dùng tính năng auditing của Spring Data JPA để tự động điền các trường này.

7. **Kiểm thử với Postman**
   - API: `POST http://localhost:8080/api/create`
   - Body JSON (ví dụ):
     ```json
     {
       "name": "Madan Reddy",
       "email": "some-email@example.com",
       "mobileNumber": "1234567890"
     }
     ```
   - Tên field trong JSON phải trùng với field trong `CustomerDto` để Jackson map được.
   - Lần gọi đầu:
     - Nhận status 201, message “Account created successfully”.
   - Gọi lại với cùng `mobileNumber`:
     - Nhận status 400, body là `ErrorResponseDto`:
       - `path`: đường dẫn API,
       - `message`: “Customer already registered with given mobile number”,
       - `timestamp`: thời điểm lỗi.

8. **Kiểm tra dữ liệu trong H2 Console**
   - Mở H2 console, kết nối DB.
   - Query bảng `customers`:
     - Thấy bản ghi với `customerId`, `name`, `email`, `mobileNumber`, `createdAt`, `createdBy`.
   - Query bảng `accounts`:
     - Thấy bản ghi với `accountNumber` ngẫu nhiên, `customerId` liên kết với bảng customers, `accountType`, `branchAddress`, các trường audit.

9. **Định hướng tiếp theo**
   - Đã hoàn thành API “create” cho Accounts microservice.
   - Bài tập: tự triển khai logic tương tự cho microservice `cards` và `loans` (tham khảo code trên GitHub).
   - Bước tiếp theo trong khóa học: triển khai các API “read”, “update”, “delete” cho Accounts microservice.
