
Dưới đây là phần tóm tắt ngắn gọn nội dung bạn cung cấp:

1. Mục tiêu  
   - Xây dựng một REST API mới trong microservice Accounts.  
   - API này sẽ:
     - Nhận số điện thoại di động của khách hàng.
     - Lấy dữ liệu Account, Loan, Card tương ứng.
     - Hợp nhất tất cả thông tin và trả về cho client.  
   - Accounts không có dữ liệu Loan và Card, nên phải gọi sang microservice Loans và Cards.

2. Sử dụng Eureka và OpenFeign  
   - Các microservice (Accounts, Loans, Cards) đã đăng ký với Eureka Server.  
   - Để Accounts gọi nội bộ sang Loans và Cards, sử dụng OpenFeign (client-side load balancing + service discovery).  
   - Thay vì dùng RestTemplate/WebClient và tự viết logic URL, port, xử lý lỗi…, OpenFeign cho phép khai báo interface, phần triển khai được sinh tự động.

3. Thêm dependency OpenFeign  
   - Vào start.spring.io, chọn Maven, thêm dependency “OpenFeign”.  
   - Sao chép dependency vào pom.xml của Accounts service (sau Netflix Eureka Client).  
   - Reload Maven để tải thư viện Feign.

4. Kích hoạt Feign trong Accounts  
   - Mở lớp main của Accounts (AccountsApplication).  
   - Thêm annotation: `@EnableFeignClients` để bật chức năng Feign Client.

5. Tạo Feign Client cho Cards  
   - Tạo package mới: `com.eazybytes.accounts.service.clients`.  
   - Tạo interface `CardsFeignClient`.  
   - Thêm annotation:  
     ```java
     @FeignClient("cards")
     ```
     - "cards" là logical name mà service Cards đăng ký với Eureka.  
   - Trong interface, khai báo phương thức trừu tượng tương ứng với REST API trong CardsController (ví dụ: `fetchCardDetails`):
     - Chữ ký (input, output, HTTP method, path) phải khớp với API thật.  
     - Ví dụ:  
       ```java
       @GetMapping(value = "/api/fetch", consumes = "application/json")
       CardsDto fetchCardDetails(@RequestParam String mobileNumber);
       ```
   - Không viết code triển khai, Feign sẽ tự sinh.

6. Chia sẻ DTO giữa các service  
   - Accounts không có `CardsDto` và `LoansDto`.  
   - Sao chép `CardsDto` từ Cards service và `LoansDto` từ Loans service vào package dto của Accounts.  
   - Sau đó, dùng các DTO này làm kiểu trả về trong Feign Client.

7. Tạo Feign Client cho Loans  
   - Sao chép `CardsFeignClient` thành `LoansFeignClient`.  
   - Đổi annotation:  
     ```java
     @FeignClient("loans")
     ```
   - Đổi kiểu trả về thành `LoansDto`.  
   - Đảm bảo chữ ký phương thức khớp với API trong LoansController (ví dụ: `fetchLoanDetails`, nhận `mobileNumber`, path `/api/fetch`, v.v.).

8. Cách Feign hoạt động với Eureka  
   - FeignClient dùng logical name (“cards”, “loans”) để hỏi Eureka danh sách instance.  
   - Thông tin instance được cache khoảng 30 giây, trong thời gian đó Feign dùng dữ liệu cache để gọi API mà không hỏi lại Eureka.  
   - Feign tự xử lý việc chọn instance (client-side load balancing) và gọi API với request chứa `mobileNumber`.

9. Ý nghĩa  
   - Không cần viết code gọi REST thủ công (URL, port, RestTemplate/WebClient).  
   - Chỉ cần:
     - Khai báo interface với annotation Feign.
     - Khai báo chữ ký phương thức, path, kiểu request/response.  
   - Phần còn lại (triển khai, kết nối Eureka, load balancing) do OpenFeign đảm nhiệm.

Nếu bạn muốn, tôi có thể viết mẫu code hoàn chỉnh cho `CardsFeignClient`, `LoansFeignClient` và cách inject chúng vào service Accounts để gọi.
