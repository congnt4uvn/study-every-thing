
Dưới đây là tóm tắt nội dung bài giảng:

1. **Lệnh làm việc với image**
   - `docker images`: Liệt kê tất cả Docker image trên máy.
   - `docker image inspect <image-id>`: Xem chi tiết một image.
   - Có thể chỉ cần gõ vài ký tự đầu của ID (3–4 ký tự), Docker sẽ tự nhận diện.
   - `docker image rm <image-id>` hoặc `docker rmi <image-id>`: Xóa image (có thể xóa nhiều image cùng lúc).
   - `docker build -t <image-name> .`: Build image từ Dockerfile (Dockerfile phải nằm cùng thư mục).
   - `docker image push`: Đẩy image lên registry (ví dụ Docker Hub).
   - `docker image pull`: Kéo image từ registry về.
   - `docker image prune`: Xóa tất cả image “không sử dụng” (không có container nào, dù chạy hay dừng, dùng image đó).
   - `docker history <image-name>`: Xem các layer và lệnh đã dùng để build image.

2. **Lệnh làm việc với container cơ bản**
   - `docker run ... <image-name>`: Tạo và chạy container từ image (có thể kèm mapping port, v.v.).
   - `docker ps`: Liệt kê container đang chạy.
   - `docker ps -a`: Liệt kê tất cả container (cả chạy và đã dừng).
   - `docker container start <container-id>`: Khởi động lại container đã dừng (có thể nhiều ID).
   - `docker container stop <container-id>`: Dừng container “nhẹ nhàng”, cho ~5 giây để đóng tài nguyên.
   - `docker container kill <container-id>`: Dừng container ngay lập tức (không chờ).
   - `docker container restart <container-id>`: Khởi động lại container.
   - `docker container inspect <container-id>`: Xem chi tiết container.
   - `docker container logs <container-id>`: Xem log container.
   - `docker container logs -f <container-id>`: Theo dõi log liên tục (follow).
   - `docker rm <container-id>`: Xóa container (có thể xóa nhiều container).
   - `docker container prune`: Xóa tất cả container đã dừng.
   - `docker container stats`: Xem thống kê CPU, RAM, IO của container.

3. **Pause / unpause container**
   - `docker container pause <container-id>`: Tạm dừng container (không nhận traffic).
   - `docker container unpause <container-id>`: Tiếp tục container sau khi pause.

4. **Lệnh hệ thống tổng hợp**
   - `docker system prune`: Xóa container đã dừng, image không dùng, network không dùng, cache… để dọn dẹp hệ thống.

5. **Đăng nhập registry**
   - `docker login -u <username>`: Đăng nhập Docker Hub (CLI sẽ hỏi mật khẩu).
   - `docker logout`: Đăng xuất.
   - Nếu dùng Docker Desktop và đã login qua UI, CLI cũng tự được kết nối, thường không cần `docker login`.

6. **Chạy lệnh bên trong container**
   - `docker exec -it <container-id> sh`: Mở shell bên trong container đang chạy để gõ lệnh trực tiếp.

7. **Docker Compose**
   - `docker compose up`: Tạo và khởi động container theo file `docker-compose.yml`.
   - `docker compose down`: Dừng và xóa container (và các resource liên quan).
   - `docker compose stop`: Chỉ dừng container, không xóa.
   - `docker compose start`: Khởi động lại các container đã được tạo sẵn (không tạo mới).

8. **Lưu ý chung**
   - Nên tận dụng Docker Desktop vì có giao diện trực quan để xem image, container, log, stats.
   - Các lệnh này là những lệnh Docker quan trọng, dùng thường xuyên và hữu ích khi đi phỏng vấn.
   - Slide và repo GitHub của giảng viên có liệt kê lại toàn bộ lệnh để tham khảo.
