
Dưới đây là tóm tắt nội dung:

1. Mục tiêu  
   - Kiểm tra các thay đổi liên quan đến bảo mật bằng Docker container và Docker Compose.  
   - Bảo vệ các microservice nội bộ, chỉ cho phép truy cập thông qua API Gateway (edge server).  
   - Tất cả image microservice đã được build với tag `SQL` và đẩy lên Docker Hub.

2. Thay đổi trong Docker Compose  
   - Đổi tag image từ `S11` sang `S12` cho các dịch vụ (do copy từ phần trước).  
   - Thêm một service mới: `keycloak`:
     - Đặt tên service, image, container name.
     - Mapping port: `8080` trong mạng Docker, expose ra ngoài là `7080`.
     - Thiết lập biến môi trường: admin username và admin password cho Keycloak.
     - Thêm lệnh `start-dev` để Keycloak dùng DB nội bộ.
     - Đảm bảo Keycloak chạy trong cùng Docker network với các dịch vụ ngân hàng.

3. Các dịch vụ không thay đổi  
   - Không chỉnh sửa phần observability/monitoring: Minio, Prometheus, Tempo, Grafana, Gateway.  
   - Không thay đổi Config Server, Eureka Server.

4. Bảo mật các microservice nội bộ  
   - Đối với `accounts`, `loans`, `cards`:
     - Xóa port mapping ra bên ngoài (trước đây expose 8080).
     - Kết quả: các service này chỉ có thể được truy cập từ các service khác trong cùng Docker network qua tên service + port 8080, không truy cập trực tiếp từ bên ngoài.

5. Cấu hình Gateway với Keycloak  
   - Thêm biến môi trường mới cho Gateway:  
     `spring.security.oauth2.resourceserver.jwt.jwk-set-uri`  
   - URL này trỏ đến endpoint JWK của Keycloak:
     - Dùng tên service `keycloak` thay vì `localhost`.
     - Dùng port `8080` (port nội bộ trong Docker network), không phải `7080` (port expose ra ngoài).
   - Trong file `application.yml` của ứng dụng, vẫn dùng `localhost:7080` cho truy cập từ bên ngoài, nhưng Gateway nói chuyện với Keycloak qua service name + port nội bộ.

6. Đồng bộ cấu hình  
   - Các thay đổi trên được copy sang các profile/config khác như `default`, `scp`.

7. Khởi động hệ thống  
   - Trước khi chạy `docker compose up -d`:
     - Dừng tất cả instance đang chạy trong IDE.
     - Dừng mọi container liên quan đang chạy trên Docker Desktop (ví dụ container Keycloak cũ).
   - Sau đó chạy `docker compose up -d` để khởi động toàn bộ container, bao gồm Keycloak; mất khoảng 2–3 phút.

Kết luận:  
Tác giả đã:
- Thêm Keycloak vào Docker Compose.
- Cấu hình Gateway dùng Keycloak để xác thực JWT.
- Ẩn các microservice nội bộ (accounts, loans, cards) khỏi thế giới bên ngoài bằng cách bỏ port mapping, buộc mọi truy cập phải đi qua Gateway.
