
Dưới đây là tóm tắt ngắn gọn nội dung:

1. Mục tiêu  
- Thiết lập một cụm Kubernetes cục bộ để học và thử triển khai các microservices.  
- Lý do: học trên cụm cục bộ để tránh tốn chi phí khi chạy cụm Kubernetes trên cloud.

2. Lý do không dùng Minikube  
- Minikube giúp tạo cụm Kubernetes nhỏ trên máy local.  
- Tuy nhiên, một số lệnh của Minikube khác với lệnh dùng trên cụm production.  
- Để trải nghiệm gần với môi trường production hơn, tác giả chọn dùng Docker Desktop thay vì Minikube.

3. Bật Kubernetes trong Docker Desktop  
- Mở Docker Desktop → Settings.  
- Chọn mục Kubernetes ở thanh bên trái.  
- Tick “Enable Kubernetes”.  
- Không chọn “Show system containers” (để lệnh docker ps không liệt kê các container hệ thống của Kubernetes).  
- Nhấn “Apply & Restart” để Docker Desktop khởi động lại và tạo cụm Kubernetes cục bộ (single-node cluster).

4. Đặc điểm cụm Kubernetes cục bộ  
- Là cụm một nút (single-node): cùng một node đóng vai trò cả master và worker.  
- Phù hợp cho mục đích học và thử nghiệm.  
- Khi lên cloud, sẽ tạo cụm production-ready với nhiều worker node và một master node.

5. Thông tin thêm từ tài liệu Docker  
- Mặc định, các container hệ thống của Kubernetes bị ẩn khỏi lệnh docker ps.  
- Nếu muốn xem, có thể bật tùy chọn “Show system containers”, nhưng thường không cần.

6. Kiểm tra trạng thái Kubernetes trong Docker Desktop  
- Sau khi bật, trong Docker Desktop sẽ có thêm biểu tượng/trạng thái Kubernetes ở góc dưới, báo “Kubernetes is running”.  
- Trong phần Kubernetes của Docker Desktop cũng sẽ có các tùy chọn cấu hình, xác nhận cụm đã được tạo.

7. Thiết lập và kiểm tra kubectl  
- kubectl là CLI để tương tác với cụm Kubernetes.  
- Docker Desktop thường tự cài kubectl và cấu hình sẵn (trên Mac/Windows theo đường dẫn được nêu trong tài liệu).  
- Nếu lệnh kubectl không chạy được, cần thêm đường dẫn cài đặt kubectl vào biến PATH (tương tự JAVA_HOME, MAVEN_HOME).

8. Các lệnh kubectl cơ bản được dùng  
- `kubectl config get-contexts`  
  - Liệt kê các context (môi trường) mà kubectl có thể kết nối.  
  - Mặc định sẽ có context tên “docker-desktop”.  
- `kubectl config get-clusters`  
  - Liệt kê các cụm Kubernetes mà kubectl biết.  
  - Hiện tại chỉ có một cụm: “docker-desktop”.  
- `kubectl config use-context docker-desktop`  
  - Đặt context mặc định là docker-desktop (hữu ích khi có nhiều context, ví dụ Minikube + Docker Desktop).  
- `kubectl get nodes`  
  - Liệt kê các node trong cụm.  
  - Với cụm cục bộ hiện tại, chỉ có một node duy nhất.

9. Lưu ý cuối  
- Cần chạy các lệnh trên để xác nhận kubectl đã được thiết lập đúng.  
- Nếu lỗi, làm theo hướng dẫn trong tài liệu, chỉnh PATH, khởi động lại terminal rồi thử lại.  
- Nếu muốn tắt cụm Kubernetes cục bộ, có thể vào lại Settings của Docker Desktop và tắt Kubernetes.  

Tóm lại: nội dung hướng dẫn cách bật Kubernetes trong Docker Desktop, xác nhận cụm cục bộ đã chạy, thiết lập và kiểm tra kubectl, và giải thích lý do chọn Docker Desktop thay vì Minikube để có trải nghiệm gần với môi trường production.
