
Dưới đây là tóm tắt nội dung:

1. Trong Spring Cloud Gateway, hiện tại hệ thống đã cấu hình định tuyến (routes) bằng Java, sử dụng:
   - Predicate `path` để kiểm tra đường dẫn request.
   - Filter `rewritePath` để ghi lại (rewrite) đường dẫn trước khi forward đến microservice thực tế.

2. Sau khi filter `rewritePath` chạy xong, request được chuyển tiếp đến microservice tương ứng.

3. Bài giảng này giới thiệu cách:
   - Thêm nhiều filter hơn vào cùng một route (thay vì chỉ 1 filter như trước).
   - Cụ thể là thêm filter để chèn header vào response gửi về client.

4. Tác giả hướng dẫn cách tra cứu tài liệu chính thức của Spring Cloud Gateway:
   - Xem ví dụ cấu hình Java (Fluent Java Routes API) với `RouteLocator`.
   - Xem cấu hình YAML (route predicate factory, filter factory).
   - Nhấn mạnh: nên ưu tiên cấu hình Java vì linh hoạt hơn, dễ xử lý các yêu cầu phức tạp.

5. Filter mới được sử dụng là `AddResponseHeaderGatewayFilterFactory`:
   - Trong YAML có ví dụ sẵn, nhưng ở đây chuyển sang dùng cấu hình Java.
   - Filter này cần:
     - Tên header.
     - Giá trị header.

6. Tác giả cấu hình một header mới:
   - Tên: `X-Response-Time`.
   - Giá trị: thời điểm hiện tại (`LocalDateTime.now().toString()`).
   - Mục đích: giúp client biết thời điểm response được gửi, từ đó ước lượng thời gian xử lý (từ lúc gửi request đến khi nhận response).

7. Filter `AddResponseHeader` được thêm cho tất cả các route/microservice (ví dụ: loan, card, …).

8. Sau khi build và test (bằng Postman):
   - Trước đây response chỉ có 3 header mặc định.
   - Sau khi thêm filter, response có thêm header tùy chỉnh `X-Response-Time` với giá trị ngày giờ hiện tại.

9. Tài liệu Spring Cloud Gateway liệt kê rất nhiều filter có sẵn:
   - Thêm header request/response.
   - Thay đổi query param.
   - Cache request body.
   - Circuit breaker.
   - Fallback headers.
   - Modify request/response body.
   - Redirect, prefix path, v.v.

   Khuyến khích tận dụng tối đa các filter có sẵn để đáp ứng hầu hết yêu cầu nghiệp vụ.

10. Nếu không có filter dựng sẵn nào phù hợp với logic nghiệp vụ:
    - Cần tự định nghĩa “custom filter”.
    - Nội dung này sẽ được trình bày trong bài giảng tiếp theo.
