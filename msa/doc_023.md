
Nội dung trên nói về cách xử lý ngoại lệ toàn cục (global exception handling) trong Spring Boot REST API:

1. Hiện trạng:
- Ứng dụng có 4 REST API hỗ trợ CRUD.
- GlobalExceptionHandler hiện chỉ xử lý 2 ngoại lệ nghiệp vụ do người dùng định nghĩa: ResourceNotFoundException và CustomerAlreadyExistsException (ví dụ).
- Chưa xử lý các ngoại lệ runtime khác (NullPointerException, lỗi hệ thống, v.v.).

2. Mục tiêu:
- Xử lý mọi loại ngoại lệ runtime có thể xảy ra ở bất kỳ API nào.
- Trả về phản hồi phù hợp cho client (status code, message, path, thời gian…) để client biết yêu cầu đã thất bại và vì lý do gì.

3. Cách làm:
- Trong GlobalExceptionHandler, tạo thêm một phương thức xử lý ngoại lệ tổng quát:
  - Dùng @ExceptionHandler(Exception.class).
  - Đổi tên phương thức thành handleGlobalException().
  - Tham số ngoại lệ là lớp Exception (lớp cha của mọi ngoại lệ checked/unchecked).
  - Trả về ErrorResponseDto, trong đó có:
    - message: lấy từ exception.
    - path: đường dẫn API được gọi.
    - timestamp: LocalDateTime.now().
    - status: HttpStatus.INTERNAL_SERVER_ERROR (500).
- Ý nghĩa:
  - Spring Boot sẽ ưu tiên tìm handler khớp chính xác với loại ngoại lệ (ví dụ ResourceNotFoundException).
  - Nếu không tìm thấy handler cụ thể, nó sẽ rơi vào handler chung cho Exception.
  - Nhờ đó, mọi lỗi runtime đều được xử lý và trả về response có cấu trúc.

4. Mở rộng:
- Trong handler global, có thể:
  - Gửi email cho bộ phận vận hành.
  - Ghi log chi tiết vào database để báo cáo, thống kê.
  - Tùy biến theo yêu cầu nghiệp vụ.

5. Cách kiểm tra ngoại lệ runtime:
- Cố tình tạo lỗi trong AccountsController:
  - Xóa @AllArgsConstructor để Spring không tạo constructor có tham số.
  - Khi đó, autowiring không diễn ra, IAccountService sẽ là null.
  - Gọi bất kỳ API nào sẽ gây ra NullPointerException.
- Kết quả:
  - API trả về:
    - status: 500 Internal Server Error.
    - path: đường dẫn API.
    - message: thông báo lỗi runtime thực tế (ví dụ: object is null do autowiring thất bại).
  - Tất cả API (create, update, delete, v.v.) đều nhận cùng kiểu phản hồi lỗi nhờ global exception handler.

6. Kết luận:
- Kết hợp @ControllerAdvice và @ExceptionHandler là cách triển khai logic xử lý lỗi toàn cục cho tất cả Controller trong Spring Boot.
- Đây là kiến thức quan trọng, thường được hỏi khi phỏng vấn về Spring Boot và REST API.
