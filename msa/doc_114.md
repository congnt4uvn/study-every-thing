
Dưới đây là tóm tắt ngắn gọn nội dung:

1. **Mục tiêu**  
   - Dịch vụ Accounts cần gọi sang Loans và Cards, sau đó hợp nhất dữ liệu tài khoản, thẻ, khoản vay và thông tin khách hàng để trả về cho client.

2. **Tạo DTO hợp nhất**  
   - Tạo `CustomerDetailsDto` (copy cấu trúc từ `CustomerDto`).
   - Bổ sung các trường:
     - Thông tin khách hàng (name, email, mobile…)
     - `AccountsDto`
     - `CardsDto`
     - `LoansDto`
   - Cập nhật `@Schema` để mô tả đây là DTO chứa thông tin khách hàng + account + card + loan.

3. **Tạo CustomerController**  
   - Tạo `CustomerController` với:
     - `@RestController`, `@Validated`, `@RequestMapping` (copy từ `AccountsController` và chỉnh lại tên/mô tả).
   - Tạo API GET `/api/fetchCustomerDetails`:
     - Nhận `mobileNumber` (validate 10 chữ số, giống logic trong `AccountsController`).
     - Dùng `ICustomerService.fetchCustomerDetails(mobileNumber)` để lấy dữ liệu.
     - Trả về `ResponseEntity<CustomerDetailsDto>` với `HttpStatus.OK`.
   - Thêm các annotation OpenAPI (summary, description, responses 200/500).

4. **Tạo tầng Service cho Customer**  
   - Tạo interface `ICustomerService`:
     - Phương thức `CustomerDetailsDto fetchCustomerDetails(String mobileNumber);`
   - Tạo `CustomerServiceImpl`:
     - Implement `ICustomerService`.
     - Inject các dependency:
       - `AccountsRepository`
       - `CustomerRepository`
       - `CardsFeignClient`
       - `LoansFeignClient`
     - Dùng `@Service` và `@AllArgsConstructor` (Lombok) để tự động tạo constructor và autowire.

5. **Logic trong CustomerServiceImpl.fetchCustomerDetails()**  
   - Lấy dữ liệu từ DB:
     - Từ `CustomerRepository` tìm `Customer` theo `mobileNumber`. Nếu không có thì ném exception.
     - Từ `AccountsRepository` lấy `Accounts` theo `customerId`.
   - Mapping sang DTO:
     - Tạo phương thức mới trong `CustomerMapper`:
       - `mapToCustomerDetailsDto(Customer customer, CustomerDetailsDto dto)`  
         → ánh xạ dữ liệu `Customer` sang `CustomerDetailsDto`.
     - Gọi `CustomerMapper.mapToCustomerDetailsDto(customer, new CustomerDetailsDto())` để tạo `CustomerDetailsDto`.
     - Dùng `AccountsMapper.mapToAccountsDto(accounts, new AccountsDto())` rồi set vào `customerDetailsDto.setAccountsDto(...)`.

6. **Gọi các microservice khác bằng Feign**  
   - Gọi Loans:
     - `ResponseEntity<LoansDto> loansDtoResponseEntity = loansFeignClient.fetchLoanDetails(mobileNumber);`
     - Lấy `LoansDto` từ `loansDtoResponseEntity.getBody()` và set vào `customerDetailsDto`.
   - Gọi Cards:
     - `ResponseEntity<CardsDto> cardsDtoResponseEntity = cardsFeignClient.fetchCardDetails(mobileNumber);`
     - Lấy `CardsDto` từ `cardsDtoResponseEntity.getBody()` và set vào `customerDetailsDto`.
   - Trả về `customerDetailsDto` từ service.

7. **Kết nối Controller với Service**  
   - Inject `ICustomerService` vào `CustomerController` qua constructor.
   - Trong method `fetchCustomerDetails()`:
     - Gọi `customerService.fetchCustomerDetails(mobileNumber)`.
     - Trả về `ResponseEntity.status(HttpStatus.OK).body(customerDetailsDto)`.

8. **Chạy và kiểm thử**  
   - Khởi động lần lượt:
     - Config Server
     - Eureka Server
     - Accounts Microservice
     - Cards Microservice
     - Loans Microservice
   - Kiểm tra trên Eureka Dashboard: tất cả service đã đăng ký.
   - Dùng Postman:
     - Tạo dữ liệu account, card, loan với cùng `mobileNumber` (do dùng H2 nên phải tạo lại mỗi lần).
     - Gọi GET `localhost:8080/api/fetchCustomerDetails?mobileNumber=...`.
     - Nhận response hợp nhất: thông tin khách hàng + account + loan + card.

9. **Cơ chế phía sau**  
   - Accounts service dùng FeignClient gọi sang Loans và Cards.
   - FeignClient hỏi Eureka để lấy thông tin instance của các service.
   - Eureka trả về danh sách instance; Spring Cloud LoadBalancer chọn một instance để gọi.
   - Nhờ đó, không cần hard-code URL/DNS của các service; chỉ cần dùng service name đã đăng ký với Eureka.

Toàn bộ quá trình cho thấy cách:
- Thiết kế DTO hợp nhất,
- Tạo API mới trong Accounts service,
- Dùng Feign + Eureka để giao tiếp giữa các microservice,
- Hợp nhất dữ liệu và trả về cho client.
