
Nội dung trên hướng dẫn quy trình xây dựng (build) một Docker image cho microservice tài khoản:

1. **Kiểm tra Docker**  
   - Chạy lệnh `docker version` để đảm bảo Docker (client và server/daemon) đã cài đặt và đang chạy trên máy.

2. **Xây dựng Docker image**  
   - Sử dụng lệnh:  
     ```bash
     docker build -t <docker-username>/accounts:S4 .
     ```
     - `.`: thư mục hiện tại chứa Dockerfile. Nếu Dockerfile ở thư mục khác thì phải chỉ rõ đường dẫn.
     - `-t`: đặt tên (tag) cho image.
     - `<docker-username>`: tên tài khoản Docker Hub (ví dụ: `eazybytes`).
     - `accounts`: tên microservice.
     - `S4`: tag/phiên bản (ở đây dùng để chỉ “Section 4”).

   - Việc thêm `<docker-username>/...` là để sau này có thể push image lên Docker Hub mà không gặp lỗi.

3. **Quá trình build diễn ra**  
   - Docker server (daemon) nhận lệnh từ Docker CLI.
   - Bước đầu: tải image base `openjdk:17-slim` (từ `docker.io`).
   - Bước tiếp: sao chép file jar của service tài khoản vào image.
   - Kết quả: tạo image `docker.io/eazybytes/accounts:S4`.

4. **Kiểm tra image vừa tạo**  
   - Dùng lệnh:
     ```bash
     docker images
     ```
     - Tìm image tên `accounts`, tag `S4`.
     - Thấy được ID image, thời gian tạo, kích thước (ví dụ ~456MB).

5. **Xem chi tiết image**  
   - Dùng:
     ```bash
     docker inspect <image-id>
     ```
     - Chỉ cần vài ký tự đầu của ID.
     - Thông tin quan trọng:
       - `Author` (ví dụ: `eazybytes.com` – lấy từ `MAINTAINER` trong Dockerfile).
       - `JAVA_HOME` được thiết lập theo image base `openjdk:17`.
       - `Entrypoint`: lệnh chạy container, ví dụ `java -jar <accounts.jar>`.
       - Hệ điều hành: Linux (Docker dựa trên các cơ chế Linux như namespaces, cgroups).

6. **Xem trên Docker Desktop**  
   - Trong giao diện Docker, có thể chọn image `accounts:S4` để xem các layer bên trong.

7. **Kết luận**  
   - Image Docker cho microservice tài khoản đã được tạo thành công từ Dockerfile.
   - Bước tiếp theo (ở bài giảng sau): tạo container từ image này và thử nghiệm các tình huống.
   - Người học được nhắc rằng cứ luyện tập chạy các lệnh Docker nhiều lần sẽ nhớ dễ dàng hơn.
