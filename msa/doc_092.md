
Nội dung trên là một bài giảng về cách **tối ưu hóa file docker-compose.yml** bằng cách tách các cấu hình lặp lại ra một file chung `common-config.yml`.

Tóm tắt các ý chính:

1. **Mục tiêu**  
   - Giảm nội dung trùng lặp trong `docker-compose.yml`.  
   - Gom các cấu hình dùng chung (network, env, deploy, …) vào một file cấu hình chung `common-config.yml`.

2. **Tạo file `common-config.yml`**  
   Trong cùng thư mục với `docker-compose.yml`, tạo file mới `common-config.yml` và định nghĩa 3 service “mẫu”:

   1) `network-deploy-service`  
      - Chứa các cấu hình liên quan đến **network** (ví dụ network `eazybank`).  
      - Mục đích: tất cả service đều dùng chung network này, nên chỉ khai báo một lần ở đây.

   2) `microservice-base-config`  
      - **extends** từ `network-deploy-service`.  
      - Thêm:
        - Biến môi trường chung: `SPRING_PROFILES_ACTIVE`.  
        - Các cấu hình **deploy** (bị lặp lại ở tất cả microservice).  
      - Dùng cho các service cần network + deploy + profile chung.  
      - RabbitMQ không cần `SPRING_PROFILES_ACTIVE` và deploy config, nên không dùng service này.

   3) `microservice-configserver-config`  
      - **extends** từ `microservice-base-config`.  
      - Thêm cấu hình phụ thuộc vào **config server**:
        - Biến môi trường `SPRING_CONFIG_IMPORT: configserver:<endpoint-url>`.  
      - Dùng cho các microservice phụ thuộc vào config server (accounts, loans, cards).

3. **Chỉnh sửa `docker-compose.yml`**  

   - **RabbitMQ**  
     - Giữ nguyên các cấu hình riêng (image, hostname, ports, healthcheck).  
     - Xóa phần `networks` trùng lặp.  
     - Thêm:
       - `extends:`
         - `file: common-config.yml`
         - `service: network-deploy-service`  
       → RabbitMQ chỉ dùng cấu hình network chung.

   - **Config Server**  
     - Xóa các cấu hình lặp lại: network + deploy (và có thể cả `SPRING_PROFILES_ACTIVE` nếu muốn tách ra).  
     - Thêm:
       - `extends:`
         - `file: common-config.yml`
         - `service: microservice-base-config`  
       → Config server dùng network + deploy chung, kèm `SPRING_PROFILES_ACTIVE` (dù không thật sự cần, nhưng không gây hại).

   - **Các microservice: accounts, loans, cards**  
     - Xóa:
       - Cấu hình network lặp lại.  
       - Cấu hình deploy lặp lại.  
       - Biến môi trường `SPRING_PROFILES_ACTIVE`.  
       - Biến môi trường `SPRING_CONFIG_IMPORT`.  
     - Thêm:
       - `extends:`
         - `file: common-config.yml`
         - `service: microservice-configserver-config`  
       → Mỗi microservice tự động nhận:
         - Network chung.  
         - Deploy chung.  
         - `SPRING_PROFILES_ACTIVE`.  
         - `SPRING_CONFIG_IMPORT` trỏ tới config server.

4. **Kết quả đạt được**  
   - File `docker-compose.yml` ngắn gọn, dễ đọc hơn.  
   - Các cấu hình dùng chung (network name, deploy config, env chung) được tập trung trong `common-config.yml`.  
   - Khi cần thay đổi:
     - Ví dụ đổi tên network hoặc thay đổi resource limit, chỉ cần sửa một chỗ trong `common-config.yml`.  
   - Giảm rủi ro sai sót do copy-paste cấu hình giữa nhiều service.

5. **Hướng tiếp theo**  
   - Ở bài giảng sau, giảng viên sẽ tạo Docker images và kiểm thử các thay đổi trong `docker-compose.yml`.
