
Dưới đây là tóm tắt nội dung:

1. **Chờ hoàn tất cài đặt Helm chart**  
   - Sau khi cài đặt, nên chờ ít nhất 10 phút (thường 5 phút là xong nhưng nên đợi 10 phút trước khi kết luận có lỗi).

2. **Kiểm tra pod và node trong Kubernetes**  
   - Vào trang cụm Kubernetes → xem danh sách node → chọn node để xem tất cả pod đang chạy (Prometheus, Loki, Kafka, v.v.).  
   - Nếu tất cả pod có dấu tích xanh nghĩa là container đã triển khai thành công.

3. **Xác minh Gateway Server**  
   - Gateway Server rất quan trọng vì không có nó thì không test được các microservice.  
   - Dùng lệnh:
     - `kubectl get pods` để tìm pod tên Gateway Server.  
     - `kubectl describe pod <gateway-pod-name>` để xem chi tiết, trong đó có node mà pod được cài.  
   - Quay lại UI Kubernetes, tìm đúng node đó, rồi tìm pod Gateway Server, xem trạng thái, event, log để chắc chắn pod chạy ổn.

4. **Kiểm tra Workloads, Services, Secrets, ConfigMaps**  
   - Vào mục Workloads để xem tất cả Deployment đã được tạo và trạng thái của chúng.  
   - Vào mục Services để xem tất cả service:
     - Hầu hết là ClusterIP.  
     - Keycloak và Gateway Server là loại LoadBalancer để có IP công khai.  
     - IP công khai có thể được map sang domain/DNS.  
   - Kiểm tra thêm Secrets và ConfigMaps đã được tạo đầy đủ.

5. **Truy cập Keycloak**  
   - Sao chép URL service Keycloak (LoadBalancer) → mở trong tab ẩn danh → vào trang admin Keycloak.  
   - Đăng nhập bằng:
     - username: `user`  
     - password: `password` (theo cấu hình trong Helm chart).  

6. **Cấu hình client và roles trong Keycloak**  
   - Tạo client: `easybank-callcenter-cc`.  
   - Bật “client authentication”, tắt các tùy chọn không cần thiết, bật “service account roles”.  
   - Lưu lại, vào phần Credentials, sao chép client secret.  
   - Cập nhật trong Postman:
     - Dán client secret.  
     - Cập nhật URL token: thay `localhost` bằng IP công khai của Keycloak (ví dụ: `34.68.159.112`).  
   - Tạo các role: `account`, `cards`, `loans`.  
   - Vào client `easybank-callcenter-cc` → service account roles → gán các role `account`, `cards`, `loans`.

7. **Test API qua Gateway bằng Postman**  
   - Lấy IP công khai của Gateway Server (service LoadBalancer).  
   - Trong các request GET:
     - Thay `localhost` bằng IP công khai Gateway.  
     - Gọi các API:
       - Account service (ví dụ: contact info).  
       - Cards service.  
       - Loans service.  
     - Nhận được response thành công.

   - Với các request POST (tạo dữ liệu):
     1. Lấy access token từ Keycloak:
        - Đảm bảo client secret đúng.  
        - URL token dùng IP công khai Keycloak.  
        - Nhấn “Get New Access Token” trong Postman, rồi “Use Token”.
     2. Thay `localhost` bằng IP công khai Gateway trong URL API.
     3. Gửi request:
        - Tạo account → nhận HTTP 201 (account created successfully).  
        - Tạo card → nhận response thành công.  
        - Tạo loan → nhận response thành công.

   - Cuối cùng, gọi API `fetchCustomerDetails` (GET) của account service:
     - Cập nhật IP Gateway.  
     - Nhận response thành công, bao gồm thông tin account, cards, loans.

8. **Kết luận**  
   - Tất cả pod, service (Keycloak, Gateway, các microservice) hoạt động đúng.  
   - Xác thực, cấp token, gọi API qua Gateway đều thành công → xác nhận toàn bộ setup trên cloud hoạt động như mong đợi.  
   - Bài giảng tiếp theo sẽ kiểm tra các thành phần Grafana.
