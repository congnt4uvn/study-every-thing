
Nội dung trên mô tả cách dùng Spring Data JPA để tự động kiểm toán (auditing) 4 cột siêu dữ liệu trong tất cả các bảng của AccountsMicroservices:

- CREATED_AT  
- CREATED_BY  
- UPDATED_AT  
- UPDATED_BY  

Trước đây:  
- CREATED_AT, CREATED_BY được gán thủ công trong service.  
- UPDATED_AT, UPDATED_BY không được cập nhật khi update → gây lỗi.

Mục tiêu:  
- Giao toàn bộ việc cập nhật 4 cột này cho Spring Data JPA, vì framework đã quản lý toàn bộ CRUD SQL.  
- Không cần viết SQL thủ công.

Các bước thực hiện:

1. **Thêm annotation auditing vào BaseEntity**  
   Trong lớp BaseEntity (chứa 4 cột metadata), gắn các annotation của Spring Data:
   - `@CreatedDate` cho trường `createdAt`
   - `@CreatedBy` cho trường `createdBy`
   - `@LastModifiedDate` cho trường `updatedAt`
   - `@LastModifiedBy` cho trường `updatedBy`

   Đồng thời gắn:
   - `@EntityListeners(AuditingEntityListener.class)` để Spring Data lắng nghe và tự động set các giá trị này.

2. **Xác định “auditor” hiện tại (ai tạo/cập nhật)**  
   Spring có thể tự lấy ngày giờ hệ thống, nhưng không biết “ai” là người thực hiện.  
   Vì vậy tạo một lớp trong package `audit`:

   - Lớp: `AuditAwareImpl`
   - Implement interface `AuditorAware<String>`
   - Override method `getCurrentAuditor()` và trả về một giá trị cố định, ví dụ `"ACCOUNTS_MS"` (Accounts Microservice).
   - Gắn `@Component("auditAwareImpl")` (tên bean có thể tùy ý, nhưng sẽ dùng trong cấu hình).

   Ý nghĩa:  
   - Tạm thời dùng giá trị hard-code `"ACCOUNTS_MS"` cho `createdBy` và `updatedBy`.  
   - Sau này khi tích hợp Spring Security, sẽ thay logic này để trả về user/client đang đăng nhập.

3. **Bật tính năng JPA Auditing trong ứng dụng**  
   Trong lớp main Spring Boot (Application class):

   - Gắn `@EnableJpaAuditing(auditorAwareRef = "auditAwareImpl")`

   Điều này:
   - Kích hoạt cơ chế auditing của Spring Data JPA.
   - Nói cho Spring biết dùng bean `auditAwareImpl` để lấy auditor hiện tại.

4. **Xóa logic thủ công trong service**  
   Trong `AccountServiceImpl`:
   - Xóa các dòng code gán thủ công `createdAt`, `createdBy`, `updatedAt`, `updatedBy`.
   - Để Spring Data JPA tự động set khi insert/update.

5. **Kiểm thử**  
   - Gửi request tạo mới account/customer:
     - Kiểm tra DB: `createdBy` = `ACCOUNTS_MS`, `createdAt` được set tự động.
   - Gửi request update:
     - Kiểm tra DB: `updatedBy` = `ACCOUNTS_MS`, `updatedAt` được set tự động.

Kết quả:  
- Việc kiểm toán 4 cột metadata được tự động hóa hoàn toàn bởi Spring Data JPA.  
- Đây là một chuẩn tốt nên áp dụng cho REST API và microservices để quản lý lịch sử tạo/cập nhật bản ghi.
