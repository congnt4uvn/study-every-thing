
Dưới đây là tóm tắt nội dung bài giảng:

1. Mục tiêu  
   - Demo cách kích hoạt một profile cụ thể và ghi đè cấu hình bằng **các phương pháp cấu hình bên ngoài** trong Spring Boot:
     - Command-line arguments (đối số dòng lệnh)
     - JVM system properties (biến hệ thống JVM / VM options)
     - Environment variables (biến môi trường)

2. Sử dụng Command-line Arguments trong IDE  
   - Mở lớp `AccountsApplication` → “Modify Run Configuration”.
   - Trong ô “Program arguments”:
     - Dùng tiền tố `--`:
       - `--spring.profiles.active=prod`
       - `--build.version=1.1`
   - Chạy lại ứng dụng:
     - Profile `prod` được kích hoạt.
     - API contact cho thấy thông tin thuộc profile prod.
     - API build-info trả về `1.1` (giá trị `build.version` bị ghi đè so với `1.0` trong profile prod).

3. Sử dụng JVM System Properties (VM Options)  
   - Vẫn trong “Modify Run Configuration”.
   - Xóa program arguments.
   - Bật “Add VM options” và thêm:
     - `-Dspring.profiles.active=prod`
     - `-Dbuild.version=1.3`
   - Chạy lại:
     - Profile `prod` vẫn được kích hoạt.
     - API build-info trả về `1.3`.

4. Sử dụng Environment Variables  
   - Trong “Modify Run Configuration”, xóa VM options.
   - Sử dụng mục “Environment variables”:
     - Quy tắc:
       - Viết hoa toàn bộ.
       - Thay dấu chấm `.` bằng dấu gạch dưới `_`.
     - Ví dụ:
       - `SPRING_PROFILES_ACTIVE=prod`
       - `BUILD_VERSION=1.8`
   - Chạy lại:
     - Profile `prod` được kích hoạt.
     - API build-info trả về `1.8`.

5. Thử nghiệm mức độ ưu tiên (precedence)  
   - Đặt `build.version` đồng thời ở:
     - Environment variables
     - VM options
     - Program arguments
   - Kịch bản 1: Cả 3 cùng tồn tại  
     - Program arguments: `--build.version=1.3`  
     - VM options: `-Dbuild.version=1.1`  
     - Env: `BUILD_VERSION=1.8`  
     - Kết quả: `build.version = 1.3`  
       → **Program arguments có ưu tiên cao nhất**.
   - Kịch bản 2: Xóa program arguments, giữ VM options + env  
     - Kết quả: `build.version = 1.1`  
       → **JVM system properties (VM options) ưu tiên cao hơn environment variables**.
   - Kịch bản 3: Xóa luôn VM options, chỉ còn env  
     - Kết quả: `build.version = 1.8`.
   - Kịch bản 4: Xóa cả env, chỉ còn cấu hình trong profile prod  
     - Kết quả: `build.version = 1.0` (giá trị trong `application.yml` / profile prod).

6. Ý nghĩa và kết luận  
   - Có thể:
     - Định nghĩa nhiều profile (dev, qa, prod, …) trong `application.yml`.
     - Kích hoạt profile và ghi đè/ thêm thuộc tính bằng cấu hình bên ngoài.
     - Dùng cùng một Docker image cho nhiều môi trường, chỉ thay đổi cấu hình bên ngoài → ứng dụng trở nên “immutable”.
   - Tuy nhiên:
     - Đây chỉ là **cách tiếp cận cơ bản** để quản lý cấu hình.
     - Có nhiều **nhược điểm** (sẽ được giải thích trong các bài giảng sau).
   - Bài tập tiếp theo:
     - Áp dụng các thay đổi tương tự cho các microservice khác (cards, loans).
