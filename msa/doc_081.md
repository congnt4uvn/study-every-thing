
Dưới đây là tóm tắt nội dung bài giảng:

1. Đặt tên cho Config Server  
   - Mở `application.yml` của Config Server.  
   - Định nghĩa thuộc tính:
     ```yaml
     spring:
       application:
         name: configserver
     ```
   - Việc đặt tên ứng dụng Spring Boot là bắt buộc để Spring Cloud Config hoạt động đúng.

2. Tạo thư mục lưu cấu hình trong classpath  
   - Vào thư mục `resources`, tạo thư mục mới tên `config`.  
   - Toàn bộ file cấu hình của các microservice sẽ được đặt trong `resources/config`.

3. Di chuyển cấu hình của microservice Accounts  
   - Từ service Accounts, copy 3 file cấu hình tương ứng với 3 profile: `default`, `prod`, `qa` sang `resources/config` của Config Server.  
   - Để tránh trùng tên giữa các service, đổi tên file theo tên ứng dụng/microservice:
     - `application.yml` → `accounts.yml`
     - `application-prod.yml` → `accounts-prod.yml`
     - `application-qa.yml` → `accounts-qa.yml`
   - Xóa các cấu hình không cần thiết (như cấu hình DB, H2, server port…) vì sẽ dùng Spring Cloud Config, chỉ giữ lại các thuộc tính cần externalize (version, message, contact, support, v.v.).  
   - Không cần dùng `spring.config.import` nữa nên xóa phần này.

4. Di chuyển cấu hình cho Cards và Loans  
   - Làm tương tự cho microservice Cards và Loans.  
   - Tạo các file:
     - `cards.yml`, `cards-prod.yml`, `cards-qa.yml`
     - `loans.yml`, `loans-prod.yml`, `loans-qa.yml`
   - Đảm bảo tên file gắn với `spring.application.name` của từng service (cards, loans).

5. Cấu hình Config Server để đọc file trong classpath  
   - Mở `application.yml` của Config Server và thêm:
     ```yaml
     spring:
       profiles:
         active: native   # kích hoạt cấu hình 'native'
       cloud:
         config:
           server:
             native:
               search-locations: classpath:/config
     ```
   - `spring.profiles.active=native` cho biết Config Server dùng “native backend” (đọc file từ classpath/file system).  
   - `search-locations` chỉ ra thư mục chứa file cấu hình (`classpath:/config`).

6. Build và khởi động Config Server  
   - Build project, bật annotation processing (Lombok) nếu IDE yêu cầu.  
   - Chạy Config Server (port 8071).

7. Kiểm tra Config Server đã tải cấu hình  
   - Gọi các endpoint của Config Server:
     - `http://localhost:8071/accounts/prod`
     - `http://localhost:8071/accounts/qa`
     - `http://localhost:8071/accounts/default`
     - `http://localhost:8071/cards/prod`, `cards/qa`, `cards/default`
     - `http://localhost:8071/loans/prod`, `loans/qa`, `loans/default`
   - Config Server trả về JSON chứa:
     - Các thuộc tính từ profile mặc định (file `accounts.yml`, `cards.yml`, `loans.yml`).
     - Cộng thêm các thuộc tính từ profile tương ứng (`-prod`, `-qa`) nếu gọi theo profile đó.
   - Phát hiện lỗi: dùng `accounts_prod.yml` (dấu gạch dưới) khiến Config Server không nhận; sửa lại thành `accounts-prod.yml` (dấu gạch nối) là hoạt động bình thường.

8. Hành vi khi microservice khởi động  
   - Khi microservice khởi động với một profile (ví dụ `prod`), nó sẽ:
     - Gọi Config Server.
     - Lấy cấu hình từ file mặc định + file theo profile.
     - Ưu tiên giá trị trong file profile (`-prod`, `-qa`) so với file mặc định.

9. Gợi ý hiển thị JSON đẹp trong trình duyệt  
   - Để JSON hiển thị đẹp trong Chrome, cài extension “JSONView”.

10. Kết luận  
   - Đã tạo thành công Spring Cloud Config Server dùng “native profile”, đọc cấu hình từ classpath (`resources/config`).  
   - Tất cả cấu hình của các microservice (accounts, cards, loans) đã được tập trung tại Config Server.  
   - Bài tiếp theo sẽ thiết lập liên kết giữa từng microservice và Config Server để chúng tự động lấy cấu hình từ đây.
