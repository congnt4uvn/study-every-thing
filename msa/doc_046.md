
Dưới đây là tóm tắt nội dung:

1. Dừng microservice Accounts đang chạy bằng Ctrl + C và dọn màn hình console.  
2. Đảm bảo Docker Desktop đang chạy (kiểm tra biểu tượng Docker trên Mac/Windows). Mở dashboard để xem container và image hiện có (có thể trống). Điều này xác nhận Docker Engine hoạt động.  
3. Trong project Accounts Microservice, tạo một file mới tên chính xác là `Dockerfile` (không có phần mở rộng `.txt`, `.xml`, …).  
4. Bên trong Dockerfile, viết các chỉ dẫn để tạo Docker image cho Accounts Microservice:
   - Dùng lệnh `FROM` để chỉ định image cơ sở là Java, ví dụ:  
     `FROM openjdk:17-jdk-slim`  
     (image name: `openjdk`, tag: `17-jdk-slim`, tương ứng Java 17; có thể tra trên Docker Hub).
   - Dùng `MAINTAINER` để ghi thông tin người/đơn vị duy trì image, ví dụ:  
     `MAINTAINER easybytes.com`.
   - Dùng lệnh `COPY` để sao chép file jar đã build từ thư mục `target` vào image, ví dụ:  
     `COPY target/accounts-0.0.1-SNAPSHOT.jar accounts-0.0.1-SNAPSHOT.jar`  
     Ý nghĩa: lấy file jar trong thư mục `target` của project và đưa vào image.
   - Dùng `ENTRYPOINT` để chỉ định lệnh khởi chạy ứng dụng khi container chạy, ví dụ:  
     `ENTRYPOINT ["java", "-jar", "accounts-0.0.1-SNAPSHOT.jar"]`  
     Đây chính là lệnh `java -jar ...` dùng để chạy ứng dụng Spring Boot, nhưng viết theo dạng mảng JSON để Docker hiểu đúng các tham số.

5. Thêm comment trong Dockerfile để giải thích:
   - Bắt đầu từ image cơ sở chứa Java runtime.
   - Thêm jar ứng dụng vào image.
   - Lệnh entrypoint để thực thi ứng dụng.

6. Kết luận: Dockerfile đã được viết xong; ở bài giảng tiếp theo sẽ dùng Dockerfile này để build Docker image cho Accounts Microservice.
