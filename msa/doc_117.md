
Dưới đây là tóm tắt nội dung:

1. Chỉnh sửa docker-compose (cấu hình mặc định):
   - Xóa service RabbitMQ và các dependency liên quan trong Config Server vì không dùng nữa.
   - Cập nhật tag image từ `:s6` lên `:s8` cho các service.
   - Sao chép cấu hình Config Server, dán lên trên service accounts và đổi thành service Eureka:
     - Đặt tên service là `eurekaserver` (Eureka Server).
     - Đổi image thành `eazybytes/eurekaserver:s8`.
     - Giữ tên container là `eurekaserver-ms`.
     - Cập nhật port mapping sang `8070:8070` (Eureka chạy trên 8070).
     - Cập nhật lệnh health check: chỉ đổi port từ 8071 sang 8070.
   - Đảm bảo đã cấu hình các endpoint readiness/liveness trong `application.yml` của Eureka để health check hoạt động.

2. Cập nhật `commonconfig.yml`:
   - Xóa các biến môi trường liên quan đến RabbitMQ.
   - Thêm cấu hình phụ thuộc (depends_on) dùng chung cho các microservice:
     - Các service accounts, loans, cards sẽ phụ thuộc vào:
       - Config Server (service phải healthy).
       - Eureka Server (service phải healthy).
   - Thêm biến môi trường dùng chung cho các microservice:
     - `EUREKA_CLIENT_SERVICEURL_DEFAULTZONE=http://eurekaserver:8070/eureka`
       (dùng tên service `eurekaserver` thay vì `localhost`).

3. Cập nhật Eureka Server trong docker-compose:
   - Khai báo `depends_on` để Eureka phụ thuộc vào Config Server (Config Server phải healthy trước).
   - Thêm biến môi trường `SPRING_APPLICATION_NAME` cho Eureka, ví dụ `eurekaserver`, trùng với tên file cấu hình trong Config Server.
   - Không cần thêm biến môi trường khác vì Eureka không phụ thuộc DB hay microservice nào.

4. Cập nhật các microservice:
   - Cập nhật image của:
     - accounts
     - loans
     - cards  
     từ `:s6` lên `:s8`.
   - Các service này dùng chung cấu hình trong `commonconfig.yml`:
     - Phụ thuộc vào Config Server và Eureka Server.
     - Dùng URL Eureka từ biến môi trường đã cấu hình.

5. Đồng bộ cấu hình cho các môi trường:
   - Sao chép docker-compose.yml và `commonconfig.yml` đã cập nhật từ cấu hình mặc định sang:
     - thư mục `prod`
     - thư mục `qa`
   - Trong `commonconfig.yml` của từng môi trường, chỉnh `spring.profiles.active`:
     - default → prod
     - default → qa
   - Mục đích: tạo sự linh hoạt để sau này có thể cấu hình khác nhau cho từng môi trường.

6. Kết luận:
   - Đã hoàn tất cập nhật docker-compose và common config để:
     - Loại bỏ RabbitMQ.
     - Thêm và cấu hình Eureka Server trong môi trường Docker.
     - Đảm bảo các microservice chỉ khởi động sau khi Config Server và Eureka Server healthy.
   - Bài giảng tiếp theo sẽ chạy `docker compose` để khởi động tất cả container và kiểm chứng các thay đổi liên quan đến Eureka trong Docker.
