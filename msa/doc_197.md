
Dưới đây là tóm tắt nội dung bài giảng:

1. **Mục tiêu bài**  
   - Tạo **client** (ứng dụng khách) và **end-user** (người dùng cuối) trong Keycloak.  
   - Chuẩn bị để triển khai **Authorization Code Grant Flow** trong hệ thống microservice ngân hàng “easy bank”.

2. **Vì sao không dùng client cũ?**  
   - Client hiện có: `easy-bank-callcenter-cc` được tạo cho **Client Credentials Grant**.  
   - Để dùng **Authorization Code Grant**, cần tạo **client mới** với cấu hình phù hợp.

3. **Tạo client mới cho Authorization Code Flow**  
   - Vào mục **Clients** → **Create client**.  
   - **Client type**: OpenID Connect.  
   - **Client ID**: `easybank-callcenter-ac` (ac = authorization code).  
   - **Name/Description**: ví dụ “Easy Bank Call Center UI”.  
   - Bật **Client authentication**.  
   - Chọn **Standard Flow** (tương ứng Authorization Code Grant trong OAuth2).  
   - Bỏ chọn các flow khác (như Direct Access Grants, Client Credentials, etc.).

4. **Cấu hình URL & CORS**  
   - **Valid redirect URIs**:  
     - Là URL mà Keycloak sẽ **redirect** người dùng sau khi đăng nhập thành công (ví dụ: dashboard, profile page).  
     - Phải khai báo trước để tránh bị hacker chiếm đoạt access token bằng redirect đến site của họ.  
     - Tạm thời, vì chưa có UI thực tế, dùng `*` (chấp nhận mọi redirect URI).  
   - **Post logout redirect URIs**: có thể bỏ qua tạm thời.  
   - **Web origins**:  
     - Tạm thời dùng `*` để chấp nhận yêu cầu từ mọi domain/port.  
     - Trong thực tế nên chỉ định đúng domain nơi client được triển khai.  
     - Mục đích: xử lý **CORS** (Cross-Origin Resource Sharing) – cơ chế bảo mật của trình duyệt, mặc định chặn giao tiếp giữa các domain/port khác nhau nếu backend không cho phép.

5. **Lưu client và lấy thông tin**  
   - Sau khi lưu, có thể xem **Client ID** và **Client Secret** – đây là thông tin xác thực của client.

6. **Tạo người dùng cuối (end-user) trong Keycloak**  
   - Vào tab **Users**, hiện chỉ có user admin.  
   - Tạo user mới:  
     - Username: `madan`  
     - Email: `tutor@eazybytes`  
     - Enable **Email Verified** = Yes  
     - First name: `Madan`  
     - Last name: `Reddy`  
   - Sau khi tạo, vào tab **Credentials**:  
     - Đặt password: `12345` (nhập 2 lần).  
     - Bỏ chọn “Temporary” để user không bị bắt buộc đổi mật khẩu khi đăng nhập lần đầu.

7. **Trong thực tế ai tạo user?**  
   - Không phải admin lúc nào cũng tạo user bằng tay.  
   - Keycloak cung cấp **Administration REST API**.  
   - Bất kỳ ứng dụng nào được **xác thực đúng** có thể gọi REST API của Keycloak để:  
     - Tạo user, cập nhật user, v.v.  
   - Ví dụ:  
     - Ứng dụng web có trang **Register**.  
     - Người dùng nhập thông tin → ứng dụng gọi REST API của Keycloak → Keycloak tạo user ở backend.  
   - Tài liệu REST API: trên website Keycloak → mục **Documentation** → **Administration REST API** (mô tả endpoint, request format, response format, v.v.).

8. **Kết luận**  
   - Đã tạo xong:  
     - Client cho Authorization Code Flow (`easybank-callcenter-ac`).  
     - Người dùng cuối `madan`.  
   - Bước tiếp theo: sử dụng **Authorization Code Grant Flow** trong mạng lưới microservice ngân hàng “easy bank”.
