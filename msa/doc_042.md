
Bài giảng trên giải thích kiến trúc và các thành phần chính của Docker:

1. **Docker Client (Máy khách Docker)**  
   - Là ứng dụng mà lập trình viên dùng để ra lệnh cho Docker.  
   - Có 2 cách gửi lệnh đến Docker Server:
     - **Docker CLI** (giao diện dòng lệnh) – cách phổ biến nhất, dùng terminal để gõ lệnh `docker ...`.
     - **Docker Remote API** – gửi lệnh qua API (ít dùng trong khóa học này).

2. **Docker Server / Docker Daemon (Máy chủ Docker)**  
   - Được cài cùng với Docker Client trên cùng máy.  
   - Chạy tiến trình **Docker daemon**, luôn lắng nghe lệnh từ client.  
   - Nhiệm vụ chính:
     - **Tạo Docker Image**:  
       - Nhận hướng dẫn (dependencies, version Java, cấu hình…)  
       - Đóng gói ứng dụng (Spring Boot, Maven, microservice…) thành **Docker Image** – một gói chứa toàn bộ app + cấu hình + phụ thuộc.
     - **Tạo Container** từ Image:  
       - Không thể tạo container nếu không có image.  
       - Từ một image, Docker daemon tạo ra **container** – phiên bản đang chạy của ứng dụng.  
       - Khi container chạy, ta có thể truy cập API, endpoint, port… của ứng dụng.

   - Trong Docker Server sẽ lưu trữ **images** và **containers** trên máy cục bộ.

3. **Docker Registry (Kho lưu trữ Image)**  
   - Dùng để lưu trữ và chia sẻ Docker Image từ máy cục bộ lên kho từ xa.  
   - **Docker Hub** là registry do Docker cung cấp:
     - Có thể public image hoặc private (chỉ user được xác thực mới tải được).  
   - Ngoài Docker Hub còn có registry riêng của:
     - GitHub, AWS, GCP, Azure, v.v.  
   - Trong thực tế, tổ chức thường đẩy image lên registry riêng (ví dụ AWS) rồi triển khai từ đó.  
   - Trong khóa học, giảng viên dùng **Docker Hub**:
     - Tạo image trên máy cục bộ.  
     - Tạo container từ image đó để test.  
     - Sau đó push image lên Docker Hub.

4. **Luồng hoạt động cơ bản**  
   - Bước 1: Từ Docker Client, gửi lệnh cho Docker Server, ví dụ:  
     - “Chạy một container từ image X”.  
   - Bước 2: Docker Server kiểm tra:
     - Nếu image đã có trong máy cục bộ → dùng luôn.  
     - Nếu chưa có → kéo (pull) image từ Docker Registry (ví dụ Docker Hub).  
   - Bước 3: Từ image đó, Docker Server tạo container.  
   - Bước 4: Container chạy → ứng dụng sẵn sàng để sử dụng.

5. Ví dụ thực tế – MySQL  
   - Cách cũ: tải file cài đặt MySQL từ website, cài thủ công (phức tạp, mất thời gian).  
   - Với Docker:
     - Kéo image MySQL từ Docker Hub.  
     - Chạy container từ image đó.  
     - Lập tức có một MySQL server chạy trong hệ thống.

Kết luận:  
- Docker gồm 3 thành phần chính: **Docker Client**, **Docker Server (Daemon)** và **Docker Registry**.  
- Client gửi lệnh → Server tạo image/container → Registry dùng để lưu trữ và phân phối image.  
- Bài sau sẽ hướng dẫn cách cài Docker trên máy cục bộ.
