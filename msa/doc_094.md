
Dưới đây là tóm tắt nội dung:

1. Mục tiêu  
   - Tạo lại Docker image cho tất cả microservice: accounts, loans, cards và config-server (máy chủ cấu hình) bằng plugin Jib.  
   - Đẩy các image này lên Docker Hub để dùng về sau (qua docker compose hoặc cho người khác sử dụng).

2. Tạo Docker image với Jib  
   - Di chuyển đến thư mục từng service (accounts, cards, loans, config-server) – nơi có file `pom.xml`.  
   - Với accounts, cards, loans: đã có cấu hình Jib sẵn, chỉ cần chạy:  
     `mvn compile jib:dockerBuild`  
     → Jib tạo image rất nhanh (khoảng vài giây) với tag `:S6`.  
   - Với config-server: ban đầu chưa có plugin Jib trong `pom.xml`.  
     - Sao chép cấu hình plugin Jib từ một service khác.  
     - Chỉnh `image` thành `eazybytes/configserver:S6` (artifactId là configserver).  
     - Sau đó chạy lại: `mvn compile jib:dockerBuild` để tạo image cho config-server.

3. Dọn dẹp Docker local  
   - Mở Docker Desktop, vào phần Images:  
     - Xóa các image cũ không cần thiết (ví dụ các image S4, buildpacks, một số image loans, cards cũ…) để tiết kiệm dung lượng.  
   - Kiểm tra Containers:  
     - Chỉ còn container rabbitmq đang chạy → có thể xóa luôn vì khi chạy `docker compose` sẽ tự tạo lại.  
   - Sau khi build xong, có 4 image mới với tag `S6`: accounts, cards, loans, config-server.

4. Đẩy image lên Docker Hub  
   - Đảm bảo đã đăng nhập Docker Desktop và Docker daemon đang chạy.  
   - Từ terminal, lần lượt chạy:  
     - `docker image push eazybytes/accounts:S6`  
     - `docker image push eazybytes/loans:S6`  
     - `docker image push eazybytes/cards:S6`  
     - `docker image push eazybytes/configserver:S6`  
   - Mỗi lệnh sẽ đẩy image tương ứng lên Docker Hub.

5. Xác nhận trên Docker Hub  
   - Mở trang Docker Hub của tài khoản `eazybytes`.  
   - Trước đó đã có các image accounts, loans, cards (được đẩy từ 6 ngày trước).  
   - Refresh trang:  
     - Thấy các image mới nhất: accounts, cards, loans, configserver với tag `S6`.  
     - Ví dụ:  
       - `cards` có 2 tag: `S4` và `S6` (S4 là image từ phần 4, S6 là image mới).  
       - `configserver` chỉ có tag `S6` vì trước đó chưa từng đẩy image nào.  

6. Kết luận  
   - Đã tạo thành công Docker image cho tất cả microservice (accounts, loans, cards, config-server) bằng Jib.  
   - Đã đẩy toàn bộ image lên Docker Hub.  
   - Bài giảng tiếp theo sẽ kiểm tra `docker-compose` với cấu hình mới để xem mọi thứ hoạt động đúng hay không.
