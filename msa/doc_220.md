
Dưới đây là tóm tắt ngắn gọn nội dung bạn cung cấp:

1. Chuẩn bị Docker:
- Đã build image Docker cho 7 ứng dụng với tag S14 và đẩy lên Docker Hub.
- Cập nhật docker-compose để:
  - Dùng các image mới với tag S14.
  - Xóa cấu hình RabbitMQ.
  - Thêm cấu hình Kafka dựa trên hướng dẫn chính thức từ Confluent (docker-compose cho Kafka).

2. Cấu hình Kafka trong docker-compose:
- Đổi tên service từ broker thành kafka (service name, hostname, container_name).
- Đảm bảo mọi chỗ dùng hostname/ service name đều thống nhất là kafka (ví dụ: kafka:9092).
- Không thay đổi các giá trị bắt buộc phải là broker trong biến môi trường (vì đó là khái niệm broker trong Kafka cluster).
- Thêm healthcheck cho Kafka bằng netcat (nc) để kiểm tra cổng 9092 trên hostname kafka.
- Cập nhật phần depends_on để các service (accounts, messaging) phụ thuộc vào kafka thay vì rabbitmq.
- Cập nhật biến môi trường Spring:
  - spring.cloud.stream.kafka.binder.brokers=kafka:9092
- Thay toàn bộ tag S13 thành S14 trong docker-compose (cả phần profiles/default).

3. Khởi chạy hệ thống:
- Dừng tất cả container và dịch vụ đang chạy (Kafka, Keycloak, các service trong IntelliJ, Kafka local).
- Chạy docker compose up -d để khởi động toàn bộ stack (Kafka, Keycloak, các microservice).

4. Cấu hình Keycloak:
- Truy cập Keycloak Admin Console.
- Tạo client mới: easy-bank-call-center-cc.
- Bật “Client authentication”, tắt “Standard flow”, bật “Service accounts roles”.
- Lưu client, lấy thông tin credentials.
- Tạo Realm role mới tên account.
- Gán role account cho service account của client vừa tạo.

5. Kiểm thử với Postman:
- Lấy access token từ Keycloak.
- Gọi API (POST) của dịch vụ bằng token vừa lấy.
- Nhận phản hồi thành công.

6. Xác nhận giao tiếp qua Kafka:
- Xem logs của service messaging:
  - Thấy log gửi email, SMS, và thông tin offset/consumer → chứng tỏ đang dùng Kafka.
- Xem logs của service accounts:
  - Thấy log nhận yêu cầu, cập nhật trạng thái liên lạc → xác nhận giao tiếp bất đồng bộ giữa accounts và messaging qua Kafka.

7. Tổng kết thay đổi trong mã:
- Thay dependency Maven từ RabbitMQ sang Kafka.
- Cập nhật cấu hình kết nối Kafka trong ứng dụng accounts và messaging.
- Tất cả mã phần 14 đã được đẩy lên GitHub trong thư mục “event-driven microservices using Kafka, Spring Cloud Function & Stream”.

8. Kết luận khóa học:
- Đã xây dựng microservices theo hướng event-driven, sử dụng cả RabbitMQ và Apache Kafka.
- Học viên có thể dùng kiến thức này để trả lời phỏng vấn về event-driven microservices, Spring Cloud Function, Spring Cloud Stream.
- Giảng viên kêu gọi học viên để lại đánh giá trên Udemy hoặc gửi phản hồi qua Udemy, LinkedIn, email (thông tin liên hệ trên trang easytolearn.com).
