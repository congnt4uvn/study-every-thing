
Dưới đây là tóm tắt nội dung bạn đã trình bày:

1. **Dịch vụ Loans (khoản vay)**  
   - Chỉnh sửa `LoansController`:
     - Autowire các môi trường và `LocalContactInfoDto`.
     - Dùng `@Value` để đọc thuộc tính `build.version`.
     - Thêm constructor để inject `ILoanService`.
   - Tạo 3 REST API mới:
     1. **build-info**: trả về phiên bản build hiện tại (biến Java `buildVersion`), có mô tả `@Operation` và `@ApiResponses` cho Swagger.
     2. **java-version**: đọc và trả về biến môi trường `JAVA_HOME`.
     3. **contact-info**: đọc các thuộc tính trong `application.yml` (tiền tố `loans`) và map vào `LoansContactInfoDto` (record class với các field: message, contact, onCallSupport…).
   - Cấu hình properties:
     - Trong `LoansApplication` dùng `@EnableConfigurationProperties(LoansContactInfoDto.class)`.
     - Tạo thêm `application_qa.yml` và `application_prod.yml`.
     - `application.yml` chứa các thuộc tính chung cho Loans (build version, message, contact, support…) và đang hard-code kích hoạt profile `qa`.
     - `application_qa.yml` và `application_prod.yml` đều có các thuộc tính với tiền tố `loans`, nhưng giá trị khác nhau (ví dụ message, contact, support).
   - Kiểm thử:
     - Chạy ứng dụng Loans ở chế độ debug với profile `qa` (kích hoạt trong `application.yml`).
     - Gọi các API `build-info`, `java-version`, `contact-info` bằng Postman:
       - `build-info` trả về `2.0` (qa).
       - `contact-info` trả về message, contact, support từ profile `qa`.
     - Dừng server, sửa Run Configuration, thêm `--spring.profiles.active=prod`, chạy lại:
       - `contact-info` trả về dữ liệu từ profile `prod`.
       - `build-info` trả về `1.0`.

2. **Dịch vụ Cards (thẻ)**  
   - Câu chuyện tương tự Loans:
     - Chỉnh sửa `CardsController` để autowire, inject dependencies.
     - Tạo 3 REST API:
       1. **build-info**: trả về build version.
       2. **java-version**: trả về `JAVA_HOME`.
       3. **contact-info**: trả về thông tin từ `CardsContactInfoDto` (record class).
     - `CardsContactInfoDto` được bind với các thuộc tính có tiền tố `cards`.
     - Trong `CardsApplication` dùng `@EnableConfigurationProperties(CardsContactInfoDto.class)`.
   - Cấu hình:
     - `application.yml` kích hoạt profile `qa` mặc định.
     - Tạo `application_qa.yml` và `application_prod.yml` cho Cards, mỗi file chứa các thuộc tính `cards.*` khác nhau.
   - Kiểm thử:
     - Chạy với profile `qa`:  
       - `build-info` trả về `2.0`.  
       - `contact-info` trả về dữ liệu từ profile `qa`.
     - Dừng server, thêm `--spring.profiles.active=prod`, chạy lại:
       - `contact-info` trả về dữ liệu từ profile `prod`.
       - `build-info` trả về `1.0`.

3. **Lưu ý về YAML và profiles**  
   - Cần cực kỳ cẩn thận với khoảng trắng/indentation trong các file `.yml`.  
     - Nếu đặt các cấu hình sai cấp (ví dụ thụt lề sai dưới `spring` hay `config`), profile sẽ không được kích hoạt đúng.
   - Cách tiếp cận hiện tại:
     - Là cách cơ bản để quản lý cấu hình theo profile (qa, prod) trong từng microservice (accounts, cards, loans).
     - Phù hợp cho các ứng dụng nhỏ hoặc số lượng microservice ít.
     - Không phải là cách tối ưu cho tổ chức có hàng trăm microservice, vì có nhiều nhược điểm (sẽ được phân tích trong bài giảng tiếp theo).

4. **Kết luận**  
   - Đã hoàn tất các thay đổi tương tự trên cả 3 microservice: Accounts, Cards, Loans.
   - Nếu có thắc mắc, có thể tham khảo thêm trong repository GitHub.
   - Bài giảng tiếp theo sẽ nói về các nhược điểm của cách tiếp cận cấu hình hiện tại và hướng cải thiện.
