
Dưới đây là tóm tắt nội dung bài giảng:

1. Chuẩn bị workspace và dự án  
   - Tạo thư mục mới `section-6/v1-springboot`.  
   - Sao chép 3 microservice từ Section 4 (accounts, cards, loans) vào thư mục này.  
   - Mở thư mục `v1-springboot` bằng IntelliJ và load tất cả dự án dưới dạng Maven project.

2. Chuẩn hóa cách build Docker image với Google Jib  
   - Mục tiêu: tất cả microservice dùng chung một cách build image là Google Jib (không dùng Dockerfile, không dùng buildpacks).  
   - Trong service `cards`:  
     - Mở `pom.xml`, phần plugin Jib Maven.  
     - Đổi tên image (ví dụ thêm `s6` để phù hợp Section 6).  
   - Copy cấu hình plugin Jib từ `cards` sang `accounts` và `loans`:  
     - Xóa Dockerfile trong `accounts` (không cần nữa).  
     - Trong `accounts` và `loans`, xóa plugin `spring-boot-maven-plugin` và thay bằng plugin `jib-maven-plugin`.  
     - Maven reload để áp dụng thay đổi.  
   - Kết quả: cả 3 service (accounts, cards, loans) đều dùng Google Jib để tạo Docker image.

3. Định nghĩa thuộc tính trong `application.yml` (service accounts)  
   - Mở `application.yml` của `AccountsMicroservice`.  
   - Thêm thuộc tính:  
     ```yaml
     build:
       version: 1.0
     ```  
   - Mục đích: đọc giá trị `build.version` trong code Java và cung cấp một REST API cho client biết version hiện tại của service.

4. Đọc thuộc tính bằng `@Value` trong `AccountsController`  
   - Mở `AccountsController`.  
   - Thêm một field:  
     ```java
     private String buildVersion;
     ```  
   - Gán giá trị từ `application.yml` bằng `@Value` và Spring Expression Language:  
     ```java
     @Value("${build.version}")
     private String buildVersion;
     ```  
   - Nhờ đó, khi ứng dụng khởi động, `buildVersion` sẽ nhận giá trị `1.0` từ `application.yml`.

5. Tạo REST API trả về build version  
   - Trong `AccountsController`, tạo method:  
     ```java
     public ResponseEntity<String> getBuildInfo() {
         return ResponseEntity
                 .status(HttpStatus.OK)
                 .body(buildVersion);
     }
     ```  
   - Thêm mapping:  
     ```java
     @GetMapping("/build-info")
     ```  
   - Sao chép phần tài liệu OpenAPI (Swagger) từ một API GET khác, rồi chỉnh lại:  
     - `summary`: “Get build information”.  
     - `description`: “Get build information deployed in Accounts microservice”.  
     - Response 200: mô tả HttpStatus.OK.  
     - Response 500: dùng `ErrorResponseDto` như các API khác.

6. Lỗi khi chạy ứng dụng và cách sửa  
   - Khi chạy `AccountsApplication`, ứng dụng báo lỗi: thiếu bean kiểu `String` cho `AccountsController`.  
   - Nguyên nhân:  
     - Lớp `AccountsController` dùng `@AllArgsConstructor` (Lombok).  
     - Lombok tạo constructor với tất cả field: `IAccountService` và `String buildVersion`.  
     - Spring cố gắng inject cả hai, nhưng chỉ có bean `IAccountService`, không có bean `String`, nên lỗi.  
   - Cách khắc phục:  
     - Xóa `@AllArgsConstructor`.  
     - Tạo constructor thủ công chỉ nhận `IAccountService`:  
       ```java
       private final IAccountService iAccountService;

       public AccountsController(IAccountService iAccountService) {
           this.iAccountService = iAccountService;
       }
       ```  
     - Có thể dùng `@Autowired` trên constructor, nhưng không bắt buộc nếu chỉ có một constructor.  
   - Sau khi sửa, chạy lại ứng dụng, service accounts khởi động thành công trên port 8080.

7. Kiểm thử API bằng Postman  
   - Trong Postman, có collection cho service accounts (đã được cung cấp trên GitHub).  
   - Gọi API mới:  
     - Method: GET  
     - URL: `http://localhost:8080/api/build-info`  
   - Kết quả trả về: `1.0` (giá trị từ `application.yml`).

8. Đánh giá cách dùng `@Value`  
   - `@Value` rất tiện khi chỉ có 1–2 thuộc tính cần đọc.  
   - Nhưng với hàng trăm microservice và nhiều thuộc tính, việc tạo nhiều field và hard-code key như `build.version` trong từng controller là không khả thi.  
   - Do đó, cách này chỉ phù hợp cho số lượng thuộc tính ít.  
   - Các bài giảng tiếp theo sẽ giới thiệu cách tiếp cận nâng cao, phù hợp hơn cho nhiều thuộc tính và nhiều microservice.

Kết luận: Bài giảng minh họa cách:
- Chuẩn hóa build Docker image bằng Google Jib cho các microservice.  
- Định nghĩa thuộc tính trong `application.yml`.  
- Đọc thuộc tính bằng `@Value` và cung cấp REST API trả về thông tin build.  
- Chỉ ra hạn chế của `@Value` khi số lượng thuộc tính lớn, làm tiền đề cho cách tiếp cận thứ hai ở bài sau.
