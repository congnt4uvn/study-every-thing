
Nội dung bài giảng nói về việc **tối ưu hóa file docker-compose.yml** bằng cách tách các cấu hình lặp lại ra một file chung `common-config.yml`.

1. **Tạo file common-config.yml** (cùng thư mục với docker-compose.yml) và định nghĩa 3 service “mẫu”:
   - `network-deploy-service`: chỉ chứa cấu hình **network** dùng chung (ví dụ network `eazybank`) để không phải khai báo lặp lại trong từng service.
   - `microservice-base-config`:  
     - `extends` từ `network-deploy-service` → kế thừa cấu hình network.  
     - Thêm biến môi trường chung `SPRING_PROFILES_ACTIVE`.  
     - Thêm các cấu hình **triển khai** (deploy) lặp lại ở tất cả microservice (accounts, loans, cards, configserver).  
     - RabbitMQ không cần các cấu hình này nên sẽ chỉ dùng `network-deploy-service`, không dùng `microservice-base-config`.
   - `microservice-configserver-config`:  
     - `extends` từ `microservice-base-config` → kế thừa network, deploy, SPRING_PROFILES_ACTIVE.  
     - Thêm biến môi trường `SPRING_CONFIG_IMPORT=configserver:...` (URL endpoint) để cấu hình phụ thuộc vào config server cho các microservice như accounts, loans, cards.

2. **Chỉnh sửa docker-compose.yml**:
   - Dọn lại khoảng trắng cho dễ đọc.
   - Với **RabbitMQ**:
     - Xóa phần `networks` lặp lại.
     - Thêm `extends`:
       - `file: common-config.yml`
       - `service: network-deploy-service`
   - Với **configserver**:
     - Xóa các cấu hình network và deploy lặp lại.
     - Dùng `extends`:
       - `file: common-config.yml`
       - `service: microservice-base-config`
     - Chấp nhận việc nó nhận thêm `SPRING_PROFILES_ACTIVE` dù không cần, vì không gây hại.
   - Với các microservice **accounts, loans, cards**:
     - Xóa:
       - Cấu hình network.
       - Cấu hình deploy.
       - Biến môi trường `SPRING_PROFILES_ACTIVE`.
       - Biến môi trường `SPRING_CONFIG_IMPORT`.
     - Thêm `extends`:
       - `file: common-config.yml`
       - `service: microservice-configserver-config`.

3. **Kết quả**:
   - File docker-compose.yml ngắn gọn hơn, ít trùng lặp.
   - Tất cả cấu hình dùng chung (network, deploy, SPRING_PROFILES_ACTIVE, SPRING_CONFIG_IMPORT) được gom vào `common-config.yml`.
   - Khi cần đổi tên network, cấu hình deploy, hay biến môi trường chung, chỉ cần sửa ở một nơi.
   - Bài giảng kết thúc bằng việc hẹn ở bài sau sẽ tạo Docker image để kiểm thử các thay đổi trong docker-compose.
