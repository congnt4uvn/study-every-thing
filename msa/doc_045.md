
Dưới đây là tóm tắt nội dung bài giảng:

1. Cấu trúc thư mục & chuẩn bị dự án  
   - Trong workspace có thư mục `section-2` chứa 3 microservice: `accounts`, `cards`, `loans`.  
   - Tạo thư mục mới `section-4` bằng cách copy toàn bộ mã từ `section-2` sang, rồi đổi tên thành `section-4`.  
   - `section-3` chỉ là phần lý thuyết (scaling, service boundaries), không có mã.  
   - Mục đích: dễ upload mã theo từng phần, giúp học viên so sánh thay đổi giữa các phần.

2. Mở dự án trong IntelliJ  
   - Mở IntelliJ → Open → chọn `workspace/microservices/section-4`.  
   - Ban đầu trong IntelliJ tên dự án vẫn hiện `section-2` vì copy cả thư mục ẩn `.idea`.  
   - Xóa thư mục `.idea` trong `section-4`, sau đó mở lại `section-4` trong IntelliJ để IntelliJ tạo cấu hình mới.  
   - Khi mở lại, IntelliJ yêu cầu load các Maven build scripts → nhấn “Load” để nhận diện 3 Maven project: `accounts`, `cards`, `loans`.

3. Cấu hình Maven cho Accounts Microservice  
   - Mở `accounts/pom.xml`.  
   - Thêm (hoặc đảm bảo có) cấu hình:
     ```xml
     <packaging>jar</packaging>
     ```
   - Lý do: muốn đóng gói ứng dụng Spring Boot thành file `.jar` để tạo Docker image.  
   - Có thể đóng gói dạng `war`, nhưng `jar` là định dạng phù hợp và phổ biến cho Spring Boot + Docker.

4. Chuẩn bị thư mục target & giới thiệu lệnh Maven  
   - Vào thư mục `accounts/target` và xóa các file/class đã compile trước đó (nếu có).  
   - Mở terminal tại thư mục `accounts` (thư mục chứa `pom.xml`).  
   - Kiểm tra Maven đã cài đặt:
     ```bash
     mvn -version
     ```
   - Sau đó chạy:
     ```bash
     mvn clean install
     ```
   - Maven sẽ:
     - Biên dịch ứng dụng Spring Boot `accounts`.  
     - Chạy unit test.  
     - Nếu thành công, tạo file jar trong `target`.

5. Kết quả build & giải thích file jar  
   - Trong `accounts/target` xuất hiện file jar, ví dụ:
     ```
     accounts-0.0.1-SNAPSHOT.jar
     ```
   - Đây là “fat jar”: chứa toàn bộ dependency cần thiết (Spring Boot libs, embedded Tomcat, v.v.) ngoại trừ Java runtime.  
   - Tên jar được tạo từ:
     - `artifactId` (ví dụ: `accounts`)  
     - `version` (ví dụ: `0.0.1-SNAPSHOT`)  
   - Khuyến khích học viên dùng cùng convention để dễ so sánh và debug.

6. Chạy ứng dụng bằng Maven  
   - Trong thư mục `accounts`, chạy:
     ```bash
     mvn spring-boot:run
     ```
   - Plugin Spring Boot Maven sẽ tìm jar trong `target` và khởi động ứng dụng web.  
   - Ứng dụng chạy trên port 8080.  
   - Kiểm tra bằng Postman:
     - Gửi request tạo account → nhận response thành công.  
     - Gửi request fetch account → cũng nhận response thành công.  
   - Dừng ứng dụng bằng `Ctrl + C`.

7. Chạy ứng dụng bằng lệnh Java (không dùng Maven)  
   - Xóa console, sau đó chạy:
     ```bash
     java -jar target/accounts-0.0.1-SNAPSHOT.jar
     ```
   - Ứng dụng Spring Boot lại khởi động trên port 8080.  
   - Lý do trình bày lệnh Java:
     - Trong Docker image không muốn cài Maven.  
     - Docker container sẽ dùng lệnh `java -jar ...` để chạy ứng dụng.

8. Kết luận & bước tiếp theo  
   - Đến đây:
     - Đã cấu hình `packaging` là `jar`.  
     - Đã build được fat jar cho Accounts Microservice.  
     - Đã biết chạy ứng dụng bằng cả Maven và Java.  
   - Kiến thức này sẽ được dùng để viết Dockerfile:  
     - Dockerfile sẽ chứa các lệnh tương tự `java -jar target/accounts-0.0.1-SNAPSHOT.jar`.  
   - Bài giảng tiếp theo: viết Dockerfile và dùng nó để tạo Docker image cho Accounts Microservice.
