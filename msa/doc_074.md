
Dưới đây là tóm tắt ngắn gọn nội dung:

1. Vấn đề ban đầu  
   - Cấu hình đang được “hard-code” trong application.yml.  
   - Khi chuyển code giữa các môi trường (dev, test, prod) phải rebuild Docker image hoặc gói ứng dụng → trái với nguyên tắc 12/15-factor (config phải tách khỏi code).

2. Mục tiêu  
   - Tìm cách trong Spring Boot để kích hoạt cấu hình/profiles từ bên ngoài (externalized configuration), không cần sửa file yml và rebuild.

3. Các cách externalize configuration trong Spring Boot  

   a) Đối số dòng lệnh (Command-line arguments)  
   - Khi chạy jar:  
     `java -jar app.jar --property.key=value --another.key=value`  
   - Spring Boot tự động chuyển các đối số này thành cặp key–value và đưa vào Environment.  
   - Ưu tiên cao nhất: nếu trùng với cấu hình trong application.yml hoặc profile khác, giá trị từ command-line sẽ ghi đè.

   b) Thuộc tính hệ thống JVM (JVM System Properties)  
   - Cú pháp: dùng tiền tố `-D` trước tên thuộc tính:  
     `java -Dproperty.key=value -jar app.jar`  
   - Ưu tiên thấp hơn command-line arguments nhưng cao hơn application.yml.  
   - Nếu cùng một thuộc tính xuất hiện ở cả command-line và JVM properties → command-line thắng.

   c) Biến môi trường (Environment Variables)  
   - Được dùng rất phổ biến vì:  
     - Hỗ trợ rộng rãi trên mọi nền tảng/ngôn ngữ, không chỉ Java/Spring Boot.  
     - Phù hợp cho microservices, serverless, hoặc hệ thống không dùng JVM.  
   - Quy tắc đặt tên:  
     - Tất cả chữ cái viết hoa.  
     - Dấu chấm `.` trong tên thuộc tính được thay bằng dấu gạch dưới `_`.  
     - Ví dụ: `build.version` → biến môi trường `BUILD_VERSION`.  
   - Cách thiết lập:  
     - Trên Linux/Mac:  
       `BUILD_VERSION=1.0.0 java -jar app.jar`  
     - Trên Windows (qua command line): dùng cú pháp `env:VAR_NAME=...` rồi chạy lệnh java.

4. Thứ tự ưu tiên (từ cao xuống thấp)  
   1) Command-line arguments  
   2) JVM System Properties (`-D...`)  
   3) Environment variables  
   4) application.yml / application.properties và các profile khác  

5. Kết luận  
   - Spring Boot hỗ trợ nhiều cách externalize configuration: command-line, JVM properties, environment variables.  
   - Nhờ đó có thể kích hoạt cấu hình/profiles cho từng môi trường mà không cần sửa file yml hay rebuild ứng dụng, phù hợp với nguyên tắc 12/15-factor.  
   - Bài giảng tiếp theo sẽ demo thực tế cách dùng các phương pháp này để kích hoạt profile.
