
Dưới đây là tóm tắt nội dung:

1. Hiện tại các microservice đã expose metrics theo format Prometheus hiểu được (thông qua actuator/micrometer).

2. Thiết lập Prometheus:
   - Tạo thư mục `prometheus` trong thư mục `observability`.
   - Thêm file cấu hình `prometheus.yml`.
   - Trong `prometheus.yml`:
     - Cấu hình global: `scrape_interval` và `evaluation_interval` đều là 5 giây.
       - Cứ 5 giây Prometheus sẽ:
         - Lấy metrics từ các container microservice.
         - Đánh giá và cập nhật lại dữ liệu/biểu đồ.
     - Cấu hình `scrape_configs`:
       - Khai báo nhiều `job` tương ứng với các microservice: account, loan, card, gatewayserver, eurekaserver, configserver…
       - Mỗi job:
         - `metrics_path` là `/actuator/prometheus`.
         - `static_configs` chứa `targets` là tên service + port (ví dụ: `account:8080`, `loan:8090`).
       - Dùng tên service trong Docker compose vì tất cả chạy chung một network.

3. Tích hợp Prometheus vào Docker compose:
   - Mở file Docker compose (profile `prod`) trong thư mục observability.
   - Hiện file này đã có Loki, Grafana, Promtail, Minio và các service khác.
   - Thêm một service mới:
     - Tên service: `prometheus`.
     - Image: `prom/prometheus:<version>`.
     - Container name: `prometheus`.
     - Expose port 9090.
     - Mount file `prometheus.yml` từ host vào container (volume) để Prometheus đọc cấu hình.
     - Đảm bảo service Prometheus dùng chung network (`network-deploy-service`) với các microservice.

4. Kết nối Prometheus với Grafana:
   - Trước đây, trong service Grafana, dùng `entrypoint` để tạo file `ds.yml` chỉ chứa cấu hình datasource Loki.
   - Để tránh Docker compose quá dài, tạo thư mục `grafana` trong `observability`.
   - Trong thư mục `grafana`, tạo file `datasource.yml`:
     - `apiVersion: 1`.
     - Xóa mọi datasource trùng tên Prometheus/Loki trước khi tạo mới.
     - Khai báo 2 datasource:
       1. Prometheus:
          - `type: prometheus`.
          - `name: Prometheus`.
          - `uid`: một ID duy nhất.
          - `url`: `http://prometheus:9090` (dùng tên service và port đã map trong Docker compose).
          - Tắt authentication (hoặc cấu hình thêm tùy nhu cầu).
       2. Loki:
          - Khai báo tương tự, trỏ đến service Loki (như đã cấu hình trước đó).
   - Xóa phần cấu hình datasource Loki cũ trong `entrypoint` của Grafana (vì đã chuyển sang `datasource.yml`).
   - Trong Docker compose, thêm volume cho Grafana:
     - Mount `datasource.yml` từ host vào đường dẫn mà Grafana đọc datasource (ví dụ `/etc/grafana/provisioning/datasources/datasource.yml`).
   - Khi Grafana khởi động, nó sẽ tự động đọc `datasource.yml` và thiết lập kết nối với cả Prometheus và Loki.

5. Kết luận:
   - Đã hoàn tất:
     - Cấu hình Prometheus để scrape metrics từ các microservice.
     - Thêm Prometheus vào Docker compose, chạy cùng network với các service khác.
     - Cấu hình Grafana để dùng Prometheus và Loki làm datasource thông qua file `datasource.yml`.
   - Bài giảng tiếp theo sẽ demo thực tế Prometheus và Grafana (dashboard, metrics, v.v.) bằng Docker compose.
