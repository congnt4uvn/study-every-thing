
Dưới đây là tóm tắt ngắn gọn nội dung bài giảng:

1. **Từ image sang container**
   - Đã có Docker image: `eazybytes/accounts:S4` (microservice accounts).
   - Dùng lệnh `docker run` để tạo và chạy container từ image, tương tự như toán tử `new` trong Java (tạo nhiều instance từ một class → tạo nhiều container từ một image).

2. **Ánh xạ cổng (port mapping / port forwarding / port publishing)**
   - Container chạy trong **mạng riêng** của Docker, mặc định không truy cập được từ bên ngoài.
   - Cần dùng `-p` để ánh xạ cổng:
     - Ví dụ: `docker run -p 8080:8080 eazybytes/accounts:S4`
       - Cổng **thứ nhất** (trước dấu `:`): cổng trên máy host (máy cục bộ / mạng ngoài).
       - Cổng **thứ hai**: cổng bên trong container (nơi ứng dụng lắng nghe).
   - Ví dụ khác: `-p 8081:8080`
     - Container vẫn chạy ở 8080 bên trong Docker network.
     - Bên ngoài truy cập qua 8081.
   - Nhiều container có thể dùng cùng cổng **bên trong** (8080), nhưng cổng **host** phải khác (8080, 8081, …).

3. **Chạy container foreground vs detached**
   - `docker run -p 8080:8080 eazybytes/accounts:S4`
     - Chạy foreground, log hiển thị trực tiếp, không gõ lệnh khác được.
   - `docker run -d -p 8080:8080 eazybytes/accounts:S4`
     - `-d` = detached mode, container chạy nền, chỉ in ra container ID.
     - Có thể tiếp tục gõ lệnh khác trong terminal.

4. **Quản lý container bằng CLI**
   - `docker ps`: xem các container **đang chạy**.
   - `docker ps -a`: xem **tất cả** container (kể cả đã dừng).
   - `docker stop <container-id>`: dừng container.
   - `docker start <container-id>`: khởi động lại container đã dừng.
   - Có thể dùng vài ký tự đầu của ID container.

5. **Quản lý container bằng Docker Desktop**
   - Tab Containers:
     - Xem trạng thái (running/stopped).
     - Start/Stop/Delete container.
     - Xem log (log khởi động Spring Boot, log khi gọi REST API).
     - Tab **Inspect**: xem chi tiết (JAVA_VERSION, JAVA_HOME, cổng, cấu hình).
     - Tab **Files**: xem file hệ thống bên trong image/container (ví dụ file jar được copy vào root).
     - Tab **Stats**: xem CPU, RAM, tài nguyên sử dụng.
     - Tab **Terminal**: mở shell bên trong container đang chạy, có thể chạy lệnh (ví dụ kiểm tra thư mục hiện tại).

6. **Tạo nhiều container từ cùng một image**
   - Có thể chạy nhiều container:
     - `docker run -d -p 8080:8080 eazybytes/accounts:S4`
     - `docker run -d -p 8081:8080 eazybytes/accounts:S4`
   - Hai container dùng cùng cổng 8080 bên trong Docker network, nhưng host port khác (8080, 8081).
   - Kiểm tra bằng `docker ps` và Docker Desktop.
   - Gọi REST API trên từng container qua các cổng tương ứng (8080, 8081).
   - Đây là cách **scale** ứng dụng (tăng số instance).

7. **Quy trình đầy đủ chạy Spring Boot với Dockerfile**
   1. Chạy `mvn clean install` tại thư mục chứa `pom.xml` → tạo file jar trong thư mục `target`.
   2. Viết **Dockerfile** (copy jar, thiết lập JDK, lệnh chạy, v.v.).
   3. Build image:
      - `docker build -t eazybytes/accounts:S4 .`
   4. Chạy container:
      - `docker run -d -p 8080:8080 eazybytes/accounts:S4`
   - Có thể xóa các container cũ rồi tạo lại từ image.

8. **Lợi ích: portability & deployment**
   - Image chứa sẵn JDK, Spring Boot, Maven, jar… Người khác chỉ cần cài Docker, không cần cài JDK/Maven.
   - Cùng một image và cùng lệnh `docker run` có thể dùng:
     - Trên máy local, máy ảo, server cloud, v.v.
   - Giúp giải quyết:
     - **Khả năng triển khai**: cùng một artifact, cùng cách chạy ở mọi môi trường.
     - **Khả năng mở rộng**: dễ dàng tăng số container/instance.
     - **Tính di động**: chạy giống nhau trên mọi hệ thống có Docker.

Nếu bạn muốn, tôi có thể tóm tắt lại ngắn hơn nữa chỉ bằng các lệnh quan trọng và ý chính về port mapping.
