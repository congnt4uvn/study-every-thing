
Dưới đây là tóm tắt nội dung:

1. Mục tiêu  
   - Triển khai tất cả các microservice còn lại lên cụm Kubernetes.  
   - Các file manifest (YAML) đã được chuẩn bị sẵn trong thư mục Section 15.

2. Thứ tự triển khai đề xuất  
   1) Keycloak service (để tạo ConfigMap)  
   2) ConfigMap  
   3) Config Server  
   4) Eureka Server  
   5) Account service  
   6) Loan service  
   7) Card service  
   8) Gateway service  

3. Cấu trúc chung của các file YAML  
   - Mỗi dịch vụ đều có:  
     - Deployment  
     - Service (thường kiểu LoadBalancer)  
   - Nhãn (labels) trong Deployment và Service phải trùng nhau (ví dụ: app: keycloak) để Kubernetes mapping đúng Pod với Service.

4. Keycloak YAML  
   - Deployment:  
     - replicas: 1  
     - container name: keycloak  
     - image: keycloak (chạy với tham số start-dev)  
     - env: truyền biến môi trường từ ConfigMap:  
       - KEYCLOAK_ADMIN  
       - KEYCLOAK_ADMIN_PASSWORD  
     - Cách lấy giá trị env:  
       - env.name: tên biến môi trường trong container  
       - valueFrom.configMapKeyRef:  
         - name: easybank-configmap  
         - key: tên key trong ConfigMap  
     - Lưu ý:  
       - valueFrom phải nằm ngay dưới name trong env.  
       - Tên biến môi trường trong env phải đúng với tên mà ứng dụng Spring Boot/container mong đợi.  
       - Trong ConfigMap có thể dùng key khác (alias), nhưng để đơn giản nên giữ trùng tên.  
   - Service:  
     - type: LoadBalancer  
     - port: 7080  
     - targetPort: 8080 (trùng với containerPort của container).

5. Eureka Server YAML  
   - Deployment:  
     - image tag dùng S12 (section 12), không dùng S14, để tránh phụ thuộc Kafka/RabbitMQ trong phần này.  
     - env:  
       - Tên ứng dụng Eureka (lấy từ ConfigMap, key riêng, không dùng chung key SPRING_APPLICATION_NAME để linh hoạt cho các service khác).  
       - URL endpoint của Config Server (ví dụ: http://configserver:8071).  
   - Service:  
     - type: LoadBalancer  
     - cấu hình tương tự các service khác.

6. Account, Loan, Card service YAML  
   - Cùng cấu trúc như Eureka:  
     - image tag: S12  
     - env:  
       - Thông tin đăng ký với Eureka (EUREKA_CLIENT_SERVICEURL_DEFAULTZONE, v.v.)  
       - Thông tin Config Server  
     - Service: kiểu LoadBalancer.

7. Gateway Server YAML  
   - Cũng dùng image tag S12.  
   - env:  
     - Các biến giống các microservice khác (Eureka, Config Server).  
     - Thêm biến môi trường chứa URL Keycloak server (để Gateway lấy token/ chứng chỉ).  
     - Giá trị URL Keycloak cũng lấy từ easybank-configmap.

8. Lưu ý chung  
   - Tất cả các biến môi trường quan trọng (tên ứng dụng, URL Config Server, URL Eureka, URL Keycloak, user/pass Keycloak…) đều được lấy từ ConfigMap easybank-configmap.  
   - Cần đảm bảo:  
     - Cú pháp env và valueFrom.configMapKeyRef đúng.  
     - Tên key trong ConfigMap khớp với key được tham chiếu trong YAML.  
   - Mục tiêu của phần này là tập trung vào triển khai microservice lên Kubernetes, tạm thời bỏ qua phần event-driven (Kafka/RabbitMQ).

Trong bài giảng tiếp theo, các file YAML này sẽ được áp dụng (kubectl apply) lên cụm Kubernetes và kiểm tra việc triển khai các container.
