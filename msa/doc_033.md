
Dưới đây là tóm tắt ngắn gọn nội dung bạn gửi (bài giảng về microservice Loans):

1. Thiết lập dự án Loans microservice  
- Sử dụng Spring Boot, Java 17, Maven.  
- GroupId: `com.easybytes`, ArtifactId & name: `loans`.  
- Dependencies: Spring Web, Spring Data JPA, Validation, DevTools, H2 Database, Lombok, SpringDoc OpenAPI, Spring Boot Starter Test.  
- Cấu hình trong `application.yaml`:  
  - Cổng cố định: `8090` (phải giữ nguyên để không phải sửa Docker/Kubernetes sau).  
  - Cấu hình H2: driver, URL, username, password, dialect H2, `ddl-auto=update`, `show-sql=true`.  
  - Phần lớn copy từ microservice Accounts, chỉ đổi port.

2. Cấu trúc database & entity  
- File `schema.sql` định nghĩa 1 bảng duy nhất: `loans`.  
- Không tạo bảng `customers` vì đã có trong service Accounts; dùng chung `mobile_number` để liên kết logic giữa Accounts, Loans, Cards.  
- Các cột chính:  
  - `loan_id` (PK), `mobile_number`, `loan_number`, `loan_type`, `total_loan`, `amount_paid`, `outstanding_amount`, cùng 4 cột metadata (created_at, created_by, last_modified_at, last_modified_by).  
- Entity:  
  - `BaseEntity`: chứa các cột metadata + audit (createdAt, createdBy, lastModifiedAt, lastModifiedBy).  
  - `Loan` entity extends `BaseEntity`, dùng `@Id` + `@GeneratedValue` để Spring Data JPA tự sinh `loan_id`.  
  - Tên field trùng tên cột nên không cần `@Column`.

3. DTO, validation & OpenAPI  
- Loan DTO (LoanDto) chứa: `mobileNumber`, `loanNumber`, `loanType`, `totalLoan`, `amountPaid`, `outstandingAmount`.  
- Không đưa `loan_id` vào DTO để không lộ thông tin kỹ thuật cho client, chỉ trả dữ liệu nghiệp vụ.  
- Dùng các annotation:  
  - Validation: `@NotEmpty`, `@Pattern`, `@Positive`, `@PositiveOrZero` (chỉ cho phép số dương hoặc dương + 0, không cho số âm).  
  - OpenAPI: `@Schema`, `@Operation`, `@ApiResponses`… để mô tả API.  

4. Controller (LoansController)  
- Annotate với `@RestController`, `@RequestMapping("/api")`, trả JSON.  
- Inject `LoansService`.  
- Có 4 API chính:  
  1. **Create Loan**  
     - HTTP POST `/api/create`  
     - Nhận `mobileNumber` (query param).  
     - Tạo khoản vay mới cho số điện thoại đã dùng trong Accounts.  
  2. **Get Loan Details**  
     - HTTP GET `/api/fetch` (hoặc tương tự)  
     - Nhận `mobileNumber`, trả về LoanDto.  
  3. **Update Loan**  
     - HTTP PUT `/api/update`  
     - Nhận LoanDto trong request body, cập nhật thông tin khoản vay.  
     - Trả `200 OK` nếu thành công, `417 Expectation Failed` nếu lỗi nghiệp vụ.  
  4. **Delete Loan**  
     - HTTP DELETE `/api/delete`  
     - Nhận `mobileNumber`, xóa khoản vay tương ứng, trả `200 OK`.  
- Mỗi API có mô tả OpenAPI riêng (summary, description, responses).

5. Repository & Service layer  
- `LoansRepository` extends `JpaRepository<Loan, Long>` với 2 method custom:  
  - `findByMobileNumber(String mobileNumber)`  
  - `findByLoanNumber(String loanNumber)`  
- `ILoansService` interface: `createLoan`, `fetchLoan`, `updateLoan`, `deleteLoan`.  
- `LoansServiceImpl` triển khai:  
  - **createLoan**:  
    - Kiểm tra xem đã có loan cho `mobileNumber` chưa.  
    - Nếu có → ném `LoanAlreadyExistsException` (tương tự `AccountAlreadyExistsException`).  
    - Nếu chưa → tạo loan mới:  
      - Sinh `loanNumber` ngẫu nhiên (2 chữ số) + logic riêng.  
      - `loanType` mặc định: “Home Loan”.  
      - `totalLoan` mặc định: 100_000 (dùng underscore trong literal số – tính năng từ Java 7/8).  
      - `amountPaid = 0`, `outstandingAmount = totalLoan`.  
      - Gọi `save()` để lưu.  
  - **fetchLoan**:  
    - Tìm loan theo `mobileNumber`.  
    - Nếu không có → `ResourceNotFoundException`.  
    - Nếu có → dùng `LoanMapper` chuyển `Loan` entity → `LoanDto`.  
  - **updateLoan**:  
    - Tìm loan theo `loanNumber` (loanNumber không đổi).  
    - Nếu không có → `ResourceNotFoundException`.  
    - Dùng `LoanMapper` map từ LoanDto vào entity hiện có.  
    - Gọi `save()` → JPA nhận diện PK đã tồn tại nên thực hiện UPDATE.  
    - Trả `true` nếu thành công.  
  - **deleteLoan**:  
    - Tìm loan theo `mobileNumber`.  
    - Nếu không có → `ResourceNotFoundException`.  
    - Nếu có → `deleteById(loanId)`.

6. Exception handling & constants  
- Có file `Constants` chứa các chuỗi hằng dùng chung (message, code…).  
- Có package `exception` với `GlobalExceptionHandler`:  
  - Xử lý lỗi validation input.  
  - Xử lý `RuntimeException` và các exception nghiệp vụ (LoanAlreadyExists, ResourceNotFound…).  
- Cách làm tương tự microservice Accounts để giữ chuẩn coding thống nhất.

7. Chạy ứng dụng & kiểm thử bằng Postman  
- Class main `LoansApplication` có `@SpringBootApplication`.  
- Chạy ở chế độ debug trên port `8090`.  
- Dùng Postman:  
  - Import collection từ GitHub (collection “microservice”).  
  - Trong folder `Loans` có các request mẫu:  
    1. **Create Loan** (POST `localhost:8090/api/create`)  
       - Query param: `mobileNumber`.  
       - Dùng cùng số điện thoại với Accounts & Cards để sau này dễ tích hợp.  
       - Kết quả: HTTP 201, message “Loan created successfully”.  
    2. **Get Loan Details** (GET)  
       - Query param: `mobileNumber`.  
       - Trả JSON: `mobileNumber`, `loanNumber`, `loanType`, `totalLoan`, `amountPaid`, `outstandingAmount`.  
    3. **Update Loan** (PUT)  
       - Body: JSON lấy từ response của Get Loan, chỉnh sửa:  
         - `loanType` từ “Home Loan” → “Car Loan”.  
         - `amountPaid` = 10_000, `outstandingAmount` = 90_000.  
       - Gửi → cập nhật thành công, Get lại sẽ thấy dữ liệu mới.  
    4. **Negative cases**:  
       - Trong Update Loan:  
         - Nhập số âm cho `totalLoan`, `amountPaid`, `outstandingAmount`.  
         - Nhập `mobileNumber` dài hơn 12 chữ số.  
         → Nhận lỗi validation (do `@Positive`, `@Pattern`…).  
       - Trong Get Loan:  
         - `mobileNumber` chỉ 9 chữ số → lỗi validation.  
         - `mobileNumber` 10 chữ số nhưng không tồn tại → `ResourceNotFoundException` (“Loan not found with given mobile number”).  
    5. **Delete Loan** (DELETE)  
       - Query param: `mobileNumber` hợp lệ.  
       - Kết quả: message “Loan details deleted successfully”.  
       - Sau khi xóa, gọi Get Loan → `ResourceNotFoundException`.  

8. Mục tiêu sư phạm của khóa học  
- Giảng viên lặp lại cùng pattern với 3 microservice: Accounts, Loans, Cards.  
- Mỗi khái niệm (cấu hình, validation, exception handling, mapping, repository, service, controller, OpenAPI…) được lặp lại 3 lần để “in vào tiềm thức”.  
- Người học có thể:  
  - Xem đầy đủ cả 3 để luyện tay.  
  - Hoặc bỏ qua phần đã nắm rõ (ví dụ phần Cards) nếu thấy lặp.  
- Mục tiêu cuối: hiểu sâu và nhớ lâu các chuẩn coding & logic khi xây dựng microservices với Spring Boot & Spring Cloud.

Nếu bạn muốn, tôi có thể tóm tắt lại ngắn hơn nữa chỉ tập trung vào API và luồng nghiệp vụ chính.
