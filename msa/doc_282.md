
Dưới đây là tóm tắt ngắn gọn nội dung:

1. Tạo mô-đun con “common” trong dự án eazy-bom  
   - New Module: Java, Maven, groupId: `com.eazybytes`, artifactId: `common`, packaging: `jar`, JDK/Java 21.  
   - Thêm dependencies: Spring Web, Lombok.

2. Khai báo mô-đun trong pom eazy-bom (pom cha)  
   - Trong `pom.xml` của eazy-bom, thêm thẻ `<modules>` và khai báo `<module>common</module>`.  
   - Đảm bảo pom cha nhận biết mô-đun con.

3. Cấu hình lại pom của mô-đun common  
   - Thay phần `<parent>` từ Spring Boot starter sang `eazy-bom` (pom cha).  
   - Đưa version về dạng property trong pom cha, ví dụ:
     - Trong pom cha: `<common-jib.version>1.0.0</common-jib.version>`.  
     - Trong pom common: dùng `${common-jib.version}` thay cho version hard-code.  
   - Cập nhật `<description>` cho chuyên nghiệp hơn (ví dụ: “Common project for eazybank microservices”).  
   - Xóa các phần properties, build, test starter… đã được định nghĩa ở pom cha.  
   - Giữ lại Lombok, thêm dependency SpringDoc (vì DTO dùng annotation OpenAPI).

4. Di chuyển mã dùng chung vào mô-đun common  
   - Xóa class `CommonApplication` (không muốn chạy như ứng dụng Spring Boot, chỉ là thư viện).  
   - Tạo package `dto` trong common, copy `ErrorResponseDto` từ một microservice sang đây.  
   - Xóa `ErrorResponseDto` khỏi từng microservice (accounts, cards, loans) để tránh trùng lặp.

5. Thêm dependency common vào các microservice  
   - Trong `pom.xml` của accounts, loans, cards:  
     ```xml
     <dependency>
       <groupId>com.eazybytes</groupId>
       <artifactId>common</artifactId>
       <version>${common-jib.version}</version> (hoặc version tương ứng)
     </dependency>
     ```  
   - Reload Maven để cập nhật.  
   - Cập nhật lại import `ErrorResponseDto` trong các lớp (controller, GlobalExceptionHandler, v.v.) để trỏ sang package mới trong common.  
   - Build lại, sửa các lỗi import còn sót; sau đó build thành công.

6. Build Docker image và xử lý lỗi liên quan đến common  
   - Tạo Docker image cho account-service bằng lệnh:
     - `mvn compile jib:dockerBuild` (từ thư mục account-service).  
   - Nếu build Docker thất bại do không tìm thấy jar của common, cần publish common vào local Maven repo:
     - Từ thư mục eazy-bom/common (hoặc eazy-bom root): chạy `mvn clean install`.  
   - Jar của common sẽ được đặt trong local Maven repo (`~/.m2/repository`), từ đó plugin Jib lấy jar để build image.

7. Khuyến nghị kiến trúc  
   - Có thể tạo nhiều mô-đun con (security, logging, auditing, …) trong eazy-bom tùy nhu cầu.  
   - Nên chia nhỏ thành nhiều mô-đun thay vì một mô-đun Maven khổng lồ.  
   - Khi áp dụng, cần chú ý tránh tạo ra sự phụ thuộc chặt (tight coupling) hoặc khó khăn trong triển khai giữa các microservice.

Toàn bộ mã sẽ được đưa lên GitHub để tham khảo thêm.
