
Dưới đây là tóm tắt nội dung:

Giảng viên đang trình bày sức mạnh của Spring Cloud Function:

1. Viết logic nghiệp vụ dưới dạng hàm  
   - Toàn bộ logic nghiệp vụ được viết trong các hàm (ví dụ: `email`, `sms`).  
   - Ban đầu, các hàm này chỉ là các phương thức Java bình thường, không phải REST API.

2. Biến hàm thành REST API bằng Spring Cloud Function Web  
   - Mở `pom.xml` và thêm dependency `spring-cloud-starter-function-web` (thay cho `spring-cloud-function-context`).  
   - Reload Maven, build lại project.  
   - Chạy ứng dụng Spring Boot (`MessageApplication`) ở chế độ debug, ứng dụng chạy trên cổng mặc định 8080.  
   - Spring Cloud Function tự động phát hiện tất cả các hàm và log danh sách function catalog.

3. Gọi hàm qua Postman như REST API  
   - Tạo thư mục “message” trong Postman, có request `email`.  
   - Gọi `http://localhost:8080/email` với body là `AccountsMsgDto` (accountNo, name, email, mobileNumber).  
   - Đặt breakpoint trong hàm `email`, khi gửi request thì breakpoint dừng, chứng tỏ logic trong hàm được thực thi.  
   - Log hiển thị nội dung “sending email with details…” cùng dữ liệu request.  
   - Hàm trả về chính đối tượng đã nhận, nên response là dữ liệu đã gửi.

   - Tương tự với hàm `sms`:  
     - Gọi `http://localhost:8080/sms` với cùng `AccountsMsgDto`.  
     - Ban đầu lỗi do sai port, đổi lại 8080 thì chạy được.  
     - Breakpoint dừng trong hàm `sms`, log hiển thị “sending sms with details…”.  
     - Hàm `sms` trả về `accountNumber` kiểu `long`, nên response là số tài khoản.

   → Không cần viết controller REST; chỉ cần thêm dependency, các hàm tự động được expose thành REST API.

4. Lợi ích của Spring Cloud Function  
   - Cùng một hàm có thể:
     - Dùng như REST API.  
     - Tích hợp với event broker (RabbitMQ, Kafka).  
     - Deploy lên nền tảng serverless (AWS Lambda, v.v.).  
   - Rất phù hợp cho kiến trúc microservices hướng sự kiện: logic nghiệp vụ viết một lần, triển khai linh hoạt nhiều cách.

5. Kết hợp (compose) nhiều hàm thành một đơn vị logic  
   - Mục tiêu: có một REST API duy nhất gọi cả `email` và `sms` như một “chuỗi” logic.  
   - Mở `application.yml`:
     - Đổi tên file từ `application.properties` sang `application.yml`.  
     - Cấu hình:
       - `server.port: 9010` (tránh trùng với service Accounts chạy trên 8080).  
       - `spring.application.name: message`.  
       - `spring.cloud.function.definition: email|sms`  
         → Dùng ký hiệu pipe `|` để compose nhiều hàm thành một pipeline.

   - Build lại và khởi động ứng dụng, lần này chạy trên cổng 9010.  
   - Kiểm tra lại:
     - `http://localhost:9010/email` vẫn hoạt động.  
     - `http://localhost:9010/sms` vẫn hoạt động.  
     → Việc định nghĩa function pipeline không làm ảnh hưởng các REST API đơn lẻ.

6. REST API mới cho hàm đã compose  
   - Khi compose `email|sms`, Spring Cloud Function tạo một REST API mới với tên là chuỗi các hàm ghép lại: `emailSms` (không khoảng trắng, không ký tự đặc biệt).  
   - Gọi `http://localhost:9010/emailSms`:
     - Vì `email` là hàm đầu tiên, request body phải theo format mà `email` mong đợi (`AccountsMsgDto`).  
     - Response là `accountNumber` (do hàm `sms` là hàm cuối, trả về `long`).  
   - Trong log:
     - Thấy hai dòng: “sending email…” và “sending sms…”.  
     → Xác nhận cả hai hàm được thực thi như một đơn vị logic duy nhất.

7. Chuẩn bị chuyển sang Spring Cloud Stream + RabbitMQ  
   - Mục tiêu chính không phải là expose hàm dưới dạng REST API, mà là tích hợp với event/message broker (RabbitMQ).  
   - Để làm điều đó cần dùng Spring Cloud Stream kết hợp Spring Cloud Function.  
   - Giảng viên:
     - Comment dependency `spring-cloud-starter-function-web` (không muốn tiếp tục expose REST API).  
     - Giữ lại các cấu hình vì sẽ dùng khi tích hợp với event broker.  

8. Kết luận  
   - Spring Cloud Function cho phép:
     - Viết logic nghiệp vụ dưới dạng hàm.  
     - Tự động expose hàm thành REST API chỉ bằng một dependency.  
     - Compose nhiều hàm thành một pipeline logic.  
     - Tái sử dụng cùng logic cho REST, event streaming, hoặc serverless.  
   - Bài tiếp theo sẽ giới thiệu Spring Cloud Stream và cách tích hợp với RabbitMQ trong microservice “message”.
