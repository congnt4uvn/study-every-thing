
Dưới đây là tóm tắt ngắn gọn nội dung:

1. Mục tiêu  
- Chuẩn bị phần cấu hình và cơ sở dữ liệu trước khi xây dựng các REST API CRUD.  
- Sử dụng cơ sở dữ liệu H2 in-memory trong Spring Boot.

2. Chuyển từ application.properties sang application.yaml  
- Đổi tên file `application.properties` thành `application.yaml` để dùng định dạng YAML.  
- YAML được ưu tiên vì:
  - Dễ đọc, trực quan, tránh lặp lại khóa (ví dụ: `spring.datasource` chỉ viết một lần).  
  - Được dùng rộng rãi trong Docker, Kubernetes, các cloud provider (AWS, GCP, Azure).  
- YAML dựa trên thụt lề (indentation). Cấu trúc key–value được biểu diễn bằng:
  - `server:` rồi xuống dòng, thụt vào `port: 8080`  
  - `spring:` → `datasource:` → `url: ...`, `driver-class-name: ...`, `username: ...`, `password: ''`, v.v.  
- Cần giữ đúng khoảng trắng/thụt lề, nếu không file YAML sẽ lỗi.

3. Các cấu hình chính trong application.yaml  
- `server.port: 8080` (cổng mặc định, nhưng nên khai báo rõ).  
- Cấu hình H2 datasource:
  - `spring.datasource.url=jdbc:h2:mem:testdb`  
  - `spring.datasource.driver-class-name=org.h2.Driver`  
  - `spring.datasource.username` (mặc định)  
  - `spring.datasource.password` (để trống).  
- Bật H2 console:
  - `spring.h2.console.enabled=true` để truy cập H2 console qua `/h2-console`.  
- Cấu hình JPA/Hibernate:
  - `spring.jpa.database-platform` là H2 dialect.  
  - `spring.jpa.hibernate.ddl-auto=update` để Hibernate tự tạo/cập nhật bảng khi khởi động.  
  - `spring.jpa.show-sql=true` để in các câu SQL ra console, hỗ trợ debug.

4. Tạo schema.sql và (tùy chọn) data.sql  
- Trong thư mục `resources`, tạo file `schema.sql` để định nghĩa cấu trúc bảng.  
- Nếu muốn chèn dữ liệu mẫu, có thể tạo thêm `data.sql`.  
- Trong ví dụ, chỉ dùng `schema.sql` để tạo bảng.

5. Các bảng được tạo trong H2  
- Sau khi build và chạy ứng dụng:
  - Spring Boot đọc `application.yaml` và `schema.sql`.  
  - H2 console có thể truy cập tại `http://localhost:8080/h2-console`.  
  - Đăng nhập với driver, URL, username, password trùng với cấu hình YAML.  
- Hai bảng được tạo:
  1. `customers`  
     - Cột: `customer_id` (PK, auto-increment), `name`, `email`, `mobile_number`, và 4 cột metadata:  
       - `created_at`, `created_by`, `updated_at`, `updated_by`.  
     - Các cột metadata là chuẩn thường dùng trong dự án thực tế để biết ai tạo/cập nhật dữ liệu và khi nào.
  2. `accounts`  
     - Cột: `customer_id` (FK liên kết với `customers.customer_id`), `account_number` (PK), `account_type`, `branch_address`, cùng 4 cột metadata tương tự.  
     - Thiết kế đơn giản để tập trung vào khái niệm microservice hơn là mô hình dữ liệu phức tạp.

6. Lý do dùng script và cấu hình tự động  
- H2 là in-memory DB: mỗi lần tắt server, dữ liệu và bảng bị xóa.  
- Để tránh phải tạo bảng thủ công mỗi lần:
  - Dùng `schema.sql` + cấu hình JPA/Hibernate để Spring Boot tự tạo bảng khi khởi động.  
- `spring.jpa.show-sql=true` giúp xem các câu SQL mà Spring Data JPA thực thi, hỗ trợ debug.
