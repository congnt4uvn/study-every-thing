
Dưới đây là tóm tắt nội dung:

1. Mục tiêu  
   - Tạo Docker image cho tất cả microservice (config server, Eureka server, accounts, cards, loans) cho “Section 8”.  
   - Sau đó sẽ cập nhật docker-compose và kiểm tra việc service discovery/registration trong môi trường Docker.

2. Chuẩn bị  
   - Đảm bảo Docker daemon đang chạy trên máy local.  
   - Thêm plugin Google Jib vào pom.xml của Eureka Server (plugin đã có sẵn trong các microservice khác, ví dụ config server).  
   - Đổi tên tag từ S6 sang S8 trong pom.xml của:
     - Eureka Server  
     - Config Server  
     - Accounts service  
     - Cards service  
     - Loans service  
   - Lý do: mã được copy từ Section 6 (S6), không có Section 7, nên cần đồng bộ tag thành S8.

3. Build lại các project  
   - Chạy Maven build (clean build) cho tất cả microservice để đảm bảo mọi thay đổi được áp dụng.  
   - Dừng tất cả instance đang chạy trong IntelliJ.

4. Tạo Docker image bằng Jib  
   - Mỗi service được build image bằng lệnh:
     - `mvn compile jib:dockerBuild`
   - Thực hiện lần lượt (thứ tự không quan trọng) cho:
     - Config Server  
     - Eureka Server  
     - Accounts service  
     - Loans service  
     - Cards service  
   - Mỗi image được gắn tag S8.

5. Dọn dẹp image cũ  
   - Mở Docker Desktop (hoặc Docker UI) và xóa tất cả image liên quan đến S7 để tiết kiệm dung lượng.  
   - Kiểm tra lại, tìm với từ khóa S8 sẽ thấy đủ 5 image mới.

6. Đẩy image lên Docker Hub  
   - Dùng lệnh:
     - `docker image push docker.io/<docker-username>/<image-name>:S8`
   - Ví dụ: `docker image push docker.io/easybytes/configserver:S8`  
   - Đẩy lần lượt tất cả image (configserver, eurekaserver, accounts, cards, loans).  
   - Kiểm tra trên Docker Hub:
     - Thấy repo mới cho Eureka Server.  
     - Các repo khác (accounts, cards, loans, configserver) có nhiều tag: S4, S6, S7, S8.

7. Kết luận  
   - Đã tạo và đẩy thành công toàn bộ Docker image cho Section 8.  
   - Bước tiếp theo (ở bài giảng sau): cập nhật docker-compose và khởi động tất cả dịch vụ bằng một lệnh docker-compose để kiểm tra service discovery với Eureka trong môi trường Docker.
