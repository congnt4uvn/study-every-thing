
Dưới đây là tóm tắt nội dung:

1. Bối cảnh  
   - Đã thiết lập xong máy chủ quảng cáo (ad server) và máy chủ xác thực (authorization server).  
   - Từ phía client đã có thể lấy được access token.  
   - Phần còn lại là thiết lập máy chủ tài nguyên (resource server), cụ thể là biến Gateway Server thành Resource Server.

2. Chuẩn bị mã nguồn  
   - Tạo thư mục mới trong workspace, sao chép mã từ “Phần 11” sang và đổi tên thành “Phần 12”.  
   - Xóa các file ẩn như `.idea`.  
   - Mở project “Phần 12” bằng IntelliJ, load các project Maven và bật annotation processing.  

3. Thêm dependency bảo mật vào Gateway Server  
   - Mở `pom.xml` của Gateway Server.  
   - Thêm 3 dependency:
     - `spring-boot-starter-security` (bảo mật Spring cơ bản).  
     - `spring-security-oauth2-resource-server` (để Gateway hoạt động như OAuth2 Resource Server).  
     - `spring-security-oauth2-jose` (hỗ trợ JWT).  
   - Reload Maven để tải các thư viện mới.

4. Tạo cấu hình bảo mật (SecurityConfig)  
   - Tạo package `com.eazybytes.gatewayserver.config`.  
   - Tạo class `SecurityConfig` với:
     - `@Configuration` để Spring tạo bean.  
     - `@EnableWebFluxSecurity` vì Spring Cloud Gateway dùng WebFlux (nếu là ứng dụng MVC thì dùng `@EnableWebSecurity`).  

   - Tạo method:
     ```java
     @Bean
     public SecurityWebFilterChain springSecurityFilterChain(ServerHttpSecurity http) { ... }
     ```
   - Bên trong method:
     - Gọi `http.authorizeExchange(...)` với lambda `exchanges -> ...` để cấu hình quyền truy cập:
       - Cho phép tất cả các request HTTP GET (`HttpMethod.GET`) bằng `permitAll()`.  
       - Với các path pattern như `easybank/accounts/**`, `easybank/cards/**`, `easybank/loans/**` thì yêu cầu `authenticated()`.  
       - Lưu ý: do thứ tự cấu hình, rule cho GET (permitAll) được ưu tiên, nên các GET trong các path trên vẫn được phép không cần auth; các method khác (POST/PUT/DELETE) sẽ bị yêu cầu xác thực.

5. Cấu hình Resource Server dùng JWT  
   - Trong `SecurityConfig`, sau phần `authorizeExchange`, gọi:
     - `http.oauth2ResourceServer(oauth2 -> oauth2.jwt(Customizer.withDefaults()));`  
     - Điều này nói với Spring Security rằng Gateway là một OAuth2 Resource Server dùng JWT và dùng cấu hình mặc định cho JWT.

6. Vô hiệu hóa CSRF  
   - Gọi `http.csrf(csrfSpec -> csrfSpec.disable());`  
   - Lý do: không có trình duyệt tham gia, nếu không tắt CSRF thì các request POST/PUT/DELETE sẽ bị chặn.

7. Trả về SecurityWebFilterChain  
   - Cuối method, gọi `return http.build();` để tạo bean `SecurityWebFilterChain`.

8. Cấu hình Resource Server biết cách xác thực token  
   - Mở `application.yml` của Gateway Server.  
   - Thêm thuộc tính:
     ```yaml
     spring:
       security:
         oauth2:
           resourceserver:
             jwt:
               jwk-set-uri: <URL tới endpoint /certs của Keycloak>
     ```
   - `jwk-set-uri` là URL tới endpoint của Authorization Server (Keycloak) để Resource Server tải public key/JWK.  
   - Khi khởi động, Resource Server sẽ:
     - Kết nối tới Keycloak, tải public certificate/JWK.  
     - Dùng public key này để kiểm tra xem access token có thực sự do Keycloak phát hành và còn hợp lệ hay không.  
     - Keycloak giữ private key để ký token; Resource Server chỉ dùng public key để verify, không thể tự tạo token.

9. Tổng kết thay đổi  
   - Thêm dependency bảo mật vào `pom.xml`.  
   - Tạo class `SecurityConfig` để cấu hình WebFlux Security, Resource Server và CSRF.  
   - Thêm cấu hình `spring.security.oauth2.resourceserver.jwt.jwk-set-uri` trong `application.yml`.  
   - Sau các bước này, Gateway Server được bảo vệ và hoạt động như một OAuth2 Resource Server trong flow Client Credentials.  
   - Bài giảng tiếp theo sẽ kiểm thử các thay đổi này.
