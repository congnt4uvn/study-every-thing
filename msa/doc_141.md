
Bài giảng mô tả cách triển khai mô hình Circuit Breaker cho microservice Account đang dùng OpenFeign để gọi các dịch vụ phụ thuộc (Cards, Loans):

1. **Vấn đề ban đầu**  
   - REST API `fetchCustomerDetails` trong Account service gọi Loans service và Cards service.  
   - Nếu Loans hoặc Cards chậm, lỗi, hoặc có vấn đề mạng, lỗi sẽ lan sang Account service rồi đến API Gateway, gây RuntimeException cho client.

2. **Giải pháp: Circuit Breaker với Spring Cloud + OpenFeign**  
   - Kiểm tra tài liệu Spring Cloud OpenFeign: nếu có Spring Cloud Circuit Breaker trên classpath và cấu hình `spring.cloud.openfeign.circuitbreaker.enabled=true` trong `application.yml`, thì Feign sẽ tự động bọc các phương thức bằng Circuit Breaker.  
   - Thêm dependency vào `pom.xml` của Account service:  
     `spring-cloud-starter-circuitbreaker-resilience4j` (bản không reactive).  
   - Reload Maven, build lại.  
   - Trong `application.yml` của Account service:
     - Thêm cấu hình:
       ```yaml
       spring:
         cloud:
           openfeign:
             circuitbreaker:
               enabled: true
       ```
     - Sao chép thêm các cấu hình liên quan đến circuit breaker (tương tự như ở API Gateway) vào cuối file.

3. **Cơ chế fallback cho Feign + Circuit Breaker**  
   - Theo tài liệu, để dùng fallback:
     - Trên interface Feign, thêm thuộc tính `fallback` trong annotation `@FeignClient`.
     - Tạo lớp fallback implement cùng interface, override các method và viết logic fallback.

4. **Triển khai fallback cho Loans và Cards**  
   - Trong package `client` của Account service:
     - Tạo `LoansFallback`:
       - `implements LoansFeignClient`
       - Annotate `@Component`
       - Override các method, tạm thời trả về `null` (thay vì ném RuntimeException).
     - Tạo `CardsFallback`:
       - `implements CardsFeignClient`
       - Annotate `@Component`
       - Override các method, cũng trả về `null`.
     - Ghi chú: trong thực tế có thể trả về dữ liệu cache, dữ liệu từ DB khác, hoặc logic tùy ý.

   - Cập nhật interface Feign:
     - `LoansFeignClient`:
       ```java
       @FeignClient(name = "loans", fallback = LoansFallback.class)
       ```
     - `CardsFeignClient`:
       ```java
       @FeignClient(name = "cards", fallback = CardsFallback.class)
       ```

5. **Điều chỉnh service để xử lý null (từ fallback)**  
   - Trong `CustomerServiceImpl`, nơi gọi `LoansFeignClient` và `CardsFeignClient`:
     - Thêm kiểm tra null trước khi dùng kết quả:
       - Nếu `loansResponseEntity != null` thì mới xử lý và set vào response.
       - Tương tự với `cardsResponseEntity`.
   - Nhờ vậy, khi Loans hoặc Cards down:
     - Circuit Breaker + fallback sẽ trả về `null` thay vì lỗi.
     - Client vẫn nhận được thông tin Account (và dịch vụ còn lại nếu hoạt động), không bị RuntimeException.

6. **Kết luận**  
   - Đã tích hợp Circuit Breaker (Resilience4j) với OpenFeign trong Account service.  
   - Đã cấu hình fallback cho Loans và Cards, và cập nhật service để xử lý kết quả fallback.  
   - Kết quả: hệ thống ổn định hơn, tránh hiệu ứng lan truyền lỗi khi dịch vụ phụ thuộc gặp sự cố.
