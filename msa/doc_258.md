
Bài giảng trên nói về việc thiết lập Discovery Server trong Kubernetes và chuẩn bị chỉnh sửa mã các microservice:

1. **Lý do không dùng Helm chart cho Discovery Server**  
   - Discovery Server chỉ cần thiết lập một lần trong cụm Kubernetes, ít thay đổi, nên chạy thủ công manifest là đủ.  
   - Hiện chưa có Helm chart cộng đồng (kể cả Bitnami) cho Discovery Server này. Nếu tích hợp vào Helm chart của easybank sẽ tốn nhiều công sức, không đáng vì đây chỉ là thao tác một lần.

2. **Cách triển khai Discovery Server**  
   - Dùng lệnh: `kubectl -f kubernetes-discoveryserver.yaml`  
   - Manifest này tạo ServiceAccount, RoleBinding và Deployment.  
   - Kiểm tra trên Kubernetes Dashboard:  
     - Xem pod của Discovery Server.  
     - Mở log để xác nhận ứng dụng Spring Boot khởi động thành công (log cho thấy server chạy ~20 giây, có log Spring).

3. **Kết luận về Discovery Server**  
   - Discovery Server đã được thiết lập thành công và đang chạy ổn định trong cụm Kubernetes (ReplicaSet hiển thị màu xanh).

4. **Chuẩn bị chỉnh sửa các microservice**  
   - Mục tiêu tiếp theo:  
     - Loại bỏ các phần liên quan đến Eureka trong mã các microservice.  
     - Thêm các thay đổi để tận dụng Discovery Server dựa trên Kubernetes.  
   - Sao chép toàn bộ mã microservice từ Section 14 (phiên bản mới nhất):  
     - Copy: accounts, cards, configserver, gatewayserver, loans, messages.  
     - Không copy: Eureka Server (vì đang loại bỏ), Docker Compose (vì triển khai trên Kubernetes).  
   - Dán các thư mục microservice vào `section_17`.  
   - Sao chép thư mục `helm` hiện có sang `section_17` để sau này chỉnh sửa, loại bỏ cấu hình liên quan Eureka trong Helm chart.

5. **Mở và chuẩn bị dự án**  
   - Mở `section_17` trong IntelliJ IDEA.  
   - Tải các dự án Maven, bật annotation processing.  
   - Sau khi build xong, sẵn sàng thực hiện các thay đổi mã trong các bài giảng tiếp theo, với điều kiện Discovery Server phải đang chạy ổn trong cụm Kubernetes.
