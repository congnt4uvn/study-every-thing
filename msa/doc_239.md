
Dưới đây là tóm tắt nội dung bài giảng:

1. Kiểm tra cụm Kubernetes cục bộ  
   - Đảm bảo Kubernetes đang chạy (ví dụ qua Docker Desktop).  
   - Dùng lệnh `kubectl get services` để xác nhận cụm hoạt động bình thường.

2. Kiểm tra các release Helm hiện có  
   - Dùng `helm ls` để xem các release đã cài.  
   - Lúc đầu chưa có release nào vì chưa cài chart nào.

3. Cách Helm kết nối với Kubernetes  
   - Helm dùng cùng thông tin kết nối mà `kubectl` sử dụng.  
   - Thông tin này nằm trong thư mục ẩn `.kube/config` trong thư mục người dùng (trên Windows cũng vậy).  
   - File `config` chứa chi tiết kết nối tới cụm (ví dụ Docker Desktop cluster).

4. Tìm chart WordPress bằng Helm  
   - Sử dụng `helm search hub wordpress` để tìm các chart WordPress trên Artifact Hub.  
   - Có nhiều chart từ nhiều repo khác nhau.  
   - Tài liệu gợi ý dùng chart WordPress của Bitnami (repo nổi tiếng, sẵn sàng cho môi trường production).

5. Thêm repo Bitnami vào Helm  
   - Tìm hướng dẫn “install bitnami helm chart” trên Google.  
   - Chạy lệnh `helm repo add bitnami ...` để thêm repo Bitnami vào hệ thống cục bộ.  
   - Sau khi thêm, có thể dùng các chart trong repo này.

6. Cài đặt chart WordPress  
   - Dùng lệnh dạng:  
     `helm install <release-name> bitnami/wordpress`  
     (ví dụ: `helm install happy-panda bitnami/wordpress`).  
   - Helm tải chart WordPress từ Bitnami và triển khai lên cụm Kubernetes.  
   - Sau khi cài, Helm in ra hướng dẫn: cách lấy URL, user, password, lưu ý có thể phải chờ vài phút để LoadBalancer có IP.

7. Lấy URL và truy cập WordPress  
   - Chạy các lệnh mà Helm gợi ý để lấy URL WordPress (ví dụ http://localhost).  
   - Truy cập URL đó trong trình duyệt sẽ thấy site WordPress mẫu (blog demo).  
   - Nếu chưa truy cập được, chờ 1–2 phút và kiểm tra trạng thái trong dashboard Kubernetes/Docker.

8. Những gì Helm đã tạo trong Kubernetes  
   - Trong mục Deployments: có deployment cho WordPress và cho MariaDB.  
   - Trong mục Pods:  
     - Một pod cho MariaDB (database backend).  
     - Một pod cho WordPress (ứng dụng web).  
   - Trong mục Services:  
     - MariaDB: service kiểu ClusterIP (chỉ dùng nội bộ cụm).  
     - WordPress: service kiểu LoadBalancer (để truy cập từ bên ngoài cụm).  
   - Trong ConfigMaps: chứa các cấu hình môi trường.  
   - Trong Secrets:  
     - Secret cho WordPress (chứa mật khẩu admin).  
     - Secret cho database.

9. Đăng nhập trang quản trị WordPress  
   - Mật khẩu admin có thể lấy từ:  
     - Secret trong dashboard Kubernetes, hoặc  
     - Các lệnh mà Helm in ra (lấy username/password).  
   - Username thường là `user`, mật khẩu lấy từ lệnh/secret.  
   - Truy cập `http://localhost/admin`, nhập user/password để vào trang quản trị, tạo blog/post mới.

10. Ý nghĩa và sức mạnh của Helm  
    - Chỉ với một lệnh `helm install`, Helm đã triển khai hàng loạt manifest Kubernetes (deployments, services, secrets, configmaps, v.v.) theo chuẩn production.  
    - Giúp thiết lập một ứng dụng phức tạp như WordPress + database rất nhanh và nhất quán.

11. Vị trí lưu trữ chart Helm trên máy cục bộ  
    - Dùng `helm env` để xem các biến môi trường của Helm.  
    - Chart được lưu trong thư mục cache, ví dụ:  
      `~/Library/Caches/helm/repository/...`  
    - Trong đó có file nén chart WordPress; giải nén ra sẽ thấy cấu trúc chart và các file manifest.  
    - Bài giảng tiếp theo sẽ phân tích cấu trúc chart WordPress và các file manifest mà Helm đã chạy.

Tóm lại: bài giảng minh họa cách Helm sử dụng cấu hình `kubectl` để kết nối Kubernetes, cách tìm và thêm repo Bitnami, cài đặt chart WordPress, xem các tài nguyên được tạo trong cụm, đăng nhập trang admin WordPress, và chỉ ra nơi lưu chart trên máy để chuẩn bị phân tích cấu trúc chart ở bài sau.
