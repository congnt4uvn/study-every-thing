
Dưới đây là tóm tắt ngắn gọn nội dung trên:

1. Mục tiêu  
- Thay vì chỉ dùng `kubectl` trên terminal, ta cài đặt Kubernetes Dashboard (Admin UI) để quản lý và quan sát cụm Kubernetes dễ dàng hơn.

2. Cài đặt Helm  
- Dashboard mới được khuyến nghị cài qua Helm.  
- Helm là “package manager” cho Kubernetes (tương tự npm cho Node).  
- Cài Helm:
  - macOS: dùng Homebrew → `brew install helm`.
  - Windows: cài Chocolatey, rồi `choco install kubernetes-helm`.
- Kiểm tra: `helm version`.

3. Cài đặt Kubernetes Dashboard bằng Helm  
- Thêm repo Helm và cài chart Dashboard (các lệnh lấy từ trang chính thức).  
- Sau khi cài thành công, sẽ có thông báo “Congratulations, you have successfully installed…”.  
- Yêu cầu:
  - Docker Desktop bản mới.
  - Cụm Kubernetes local đang chạy.

4. Truy cập Dashboard  
- Chạy lệnh `kubectl` (hoặc lệnh Helm cung cấp) để mở proxy, thường trên cổng 8443.  
- Không đóng terminal đang chạy proxy, nếu không sẽ mất kết nối Dashboard.  
- Mở URL HTTPS (ví dụ `https://localhost:8443`), chấp nhận chứng chỉ không tin cậy → hiện trang login.  
- Dashboard cũ từng dùng cổng 8001, giờ dùng 8443, không cần lo về khác biệt cổng.

5. Tạo Service Account cho admin  
- Tạo file `dashboard-admin-user.yaml` với nội dung:
  - `apiVersion: v1`
  - `kind: ServiceAccount`
  - `metadata.name: admin-user`
  - `metadata.namespace: kubernetes-dashboard`
- Áp dụng:  
  `kubectl apply -f dashboard-admin-user.yaml`  
  → tạo Service Account `admin-user` trong namespace `kubernetes-dashboard`.

6. Tạo ClusterRoleBinding cho admin  
- Tạo file `dashboard-role-binding.yaml`:
  - `kind: ClusterRoleBinding`
  - `metadata.name: admin-user`
  - `roleRef`:
    - `kind: ClusterRole`
    - `name: cluster-admin` (role có sẵn, quyền cao nhất)
  - `subjects`:
    - `kind: ServiceAccount`
    - `name: admin-user`
    - `namespace: kubernetes-dashboard`
- Áp dụng:  
  `kubectl apply -f dashboard-role-binding.yaml`  
  → gán role `cluster-admin` cho Service Account `admin-user`.

7. Lấy token đăng nhập Dashboard (token ngắn hạn)  
- Lệnh (ví dụ):  
  `kubectl -n kubernetes-dashboard create token admin-user`  
- Copy token, chọn “Token” trên màn hình login Dashboard, dán token → đăng nhập.  
- Token này có thời hạn ngắn, hết hạn phải tạo lại.

8. Tạo token dài hạn bằng Secret  
- Tạo file `secret.yaml`:
  - `kind: Secret`
  - `metadata.name: admin-user`
  - `metadata.namespace: kubernetes-dashboard`
  - `type: kubernetes.io/service-account-token`
  - `annotations` trỏ tới Service Account `admin-user`
- Áp dụng:  
  `kubectl apply -f secret.yaml`  
- Lấy token từ Secret (ví dụ `kubectl get secret ... -o jsonpath=...`).  
- Lưu ý: kết quả có thể kèm ký tự `%` ở cuối, không được copy ký tự `%` này.  
- Token này cố định, có thể dùng lâu dài để đăng nhập Dashboard.

9. Khám phá Dashboard  
- Sau khi đăng nhập:
  - Namespace mặc định là `default`, có thể chuyển sang `kubernetes-dashboard`.  
  - Thấy các Deployment, Pod, ReplicaSet phục vụ chính Dashboard.  
  - Có thể xem:
    - Service Accounts → thấy `admin-user`.
    - Cluster Role Bindings → `admin-user` gán role `cluster-admin`.
    - Cluster Roles → `cluster-admin` có quyền trên mọi resource, mọi verb, mọi API group (siêu quyền).

10. Kết luận  
- Đã thiết lập:
  - Kubernetes Dashboard bằng Helm.
  - Service Account `admin-user`.
  - ClusterRoleBinding gán `cluster-admin` cho `admin-user`.
  - Token ngắn hạn và token dài hạn (qua Secret) để đăng nhập Dashboard.  
- Từ nay có thể dùng Dashboard để quan sát và quản lý cụm Kubernetes trực quan hơn.
