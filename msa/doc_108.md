
Dưới đây là tóm tắt ngắn gọn nội dung bài giảng:

1. Chuẩn bị mã nguồn  
- Tạo thư mục mới `section-8` trong workspace.  
- Sao chép toàn bộ mã từ `section-6 v2` (không dùng section-7 vì section-7 gắn với MySQL, còn phần này dùng H2).  
- Mở workspace section-8 bằng IntelliJ.

2. Loại bỏ Spring Cloud Bus & RabbitMQ  
- Mở từng project (config server, accounts, loans, cards).  
- Trong `pom.xml`: xóa dependency liên quan đến `spring-cloud-bus` và `spring-cloud-config-monitor`.  
- Trong `application.yml`: xóa các cấu hình liên quan đến RabbitMQ.  
- Làm tương tự cho tất cả microservice để tránh mang theo container/thư viện không cần thiết.

3. Tạo dự án Eureka Server  
- Vào start.spring.io:  
  - Project: Maven, Language: Java, Spring Boot: bản ổn định mới nhất.  
  - Group: `com.eazybytes`, Artifact: `eurekaserver`.  
  - Description: Service Discovery Agent for EazyBank microservices.  
  - Packaging: Jar, Java: 17.  
- Dependencies:  
  - `Eureka Server` (không chọn `Eureka Discovery Client`).  
  - `Config Client`.  
  - `Actuator`.  
- Generate project, giải nén và copy vào thư mục `section-8`.  
- Import dự án vào IntelliJ và đánh dấu là Maven project.

4. Cấu hình lớp main của Eureka Server  
- Mở lớp main (có `@SpringBootApplication`).  
- Thêm annotation `@EnableEurekaServer` để biến ứng dụng Spring Boot thành Service Discovery Agent dùng Eureka.

5. Cấu hình `application.yml` của Eureka Server  
- Đổi `application.properties` thành `application.yml`.  
- Thêm:  
  - `spring.application.name: eurekaserver`.  
  - `spring.config.import: optional:configserver:http://localhost:8071` để kết nối Config Server.  
- Thêm cấu hình Actuator (management endpoints, health, readiness, liveness) tương tự như trong Config Server để phục vụ health check (sẽ dùng trong Docker Compose).

6. Cấu hình trong GitHub Config Repo  
- Trong repo cấu hình (Eazybytes-config), tạo file `eurekaserver.yml` (tên phải trùng `spring.application.name`).  
- Các thuộc tính chính:  
  - `server.port: 8070`.  
  - `eureka.instance.hostname: localhost`.  
  - `eureka.client.fetch-registry: false`  
    - Vì Eureka Server không cần lấy registry của các service khác.  
  - `eureka.client.register-with-eureka: false`  
    - Để Eureka Server không tự đăng ký với chính nó.  
  - `eureka.client.service-url.defaultZone: http://${eureka.instance.hostname}:${server.port}/eureka`  
    - URL mà các microservice sẽ dùng để đăng ký và discovery.

7. Kiểm tra kết nối với Config Server  
- Khởi động Config Server.  
- Gọi `http://localhost:8071/eurekaserver/default` để kiểm tra Config Server đã load đúng cấu hình cho Eureka Server.

8. Khởi động và kiểm tra Eureka Server  
- Chạy lớp main của Eureka Server (có thể ở chế độ debug).  
- Ứng dụng kết nối Config Server, lấy cấu hình, và chạy trên port 8070.  
- Mở `http://localhost:8070` để xem Eureka Dashboard.  
  - Hiện tại chưa có instance nào đăng ký, nên dashboard trống.  
  - Dashboard này do Eureka cung cấp sẵn, không cần tự xây UI.

9. Kết luận  
- Đã thiết lập thành công Eureka Server làm Service Discovery Agent.  
- Bước tiếp theo (ở bài giảng sau): cấu hình các microservice (accounts, loans, cards) để đăng ký với Eureka và sử dụng nó để discovery.
