
Dưới đây là tóm tắt ngắn gọn nội dung bài giảng:

1. **Mục tiêu chung**  
   - Thêm và truyền “EasyBank-correlationId” (ID tương quan) xuyên suốt các microservice (Accounts, Loans, Cards) thông qua Gateway Server.  
   - Dùng ID này để theo dõi, debug luồng request giữa các dịch vụ.

2. **Thay đổi trong Accounts Service**  
   - Trong `CustomerController`, REST API `fetchCustomerDetails`:
     - Thêm tham số `@RequestHeader("easybank-correlation-id") String correlationId`.
     - Tạo `Logger` và ghi log giá trị `correlationId`.
   - Trong interface `ICustomerService`:
     - Thêm tham số `correlationId` vào phương thức `fetchCustomerDetails`.
   - Trong `CustomerServiceImpl`:
     - Cập nhật phương thức `fetchCustomerDetails` để nhận `correlationId`.
     - Khi gọi các microservice Loans và Cards qua Feign client:
       - Truyền `correlationId` làm tham số đầu tiên, sau đó mới đến `mobileNumber`.

3. **Thay đổi trong Feign Clients (Loans & Cards)**  
   - Trong các interface Feign client (LoanFeignClient, CardFeignClient):
     - Thêm tham số `@RequestHeader("easybank-correlation-id") String correlationId`.
     - Giữ `@RequestParam("mobileNumber") String mobileNumber` như cũ.
   - Logic nhận header tương tự như trong `CustomerController`.

4. **Thay đổi trong Loans Service**  
   - Trong `LoansController`:
     - API fetch (ví dụ `fetchLoanDetails`) thêm tham số `@RequestHeader("easybank-correlation-id") String correlationId`.
     - Tạo `Logger` cho lớp.
     - Ghi log ID tương quan trong phương thức fetch.

5. **Thay đổi trong Cards Service**  
   - Trong `CardsController`:
     - API `fetchCardDetails` thêm `@RequestHeader("easybank-correlation-id") String correlationId`.
     - Tạo `Logger` tương tự.
     - Ghi log ID tương quan trong phương thức fetch.

6. **Cấu hình logging trong application.yml**  
   - Gateway Server đã bật debug logging cho package `com.eazybytes.gateway`.
   - Thực hiện tương tự cho:
     - Accounts: `logging.level.com.eazybytes.accounts=DEBUG`
     - Cards: `logging.level.com.eazybytes.cards=DEBUG`
     - Loans: `logging.level.com.eazybytes.loans=DEBUG`
   - Mục đích: hiển thị các log tùy chỉnh (bao gồm correlationId) trên console.

7. **Quy trình build & khởi động lại**  
   - Lưu toàn bộ thay đổi, build lại project.
   - Khởi động lại lần lượt:
     1. Accounts service  
     2. Loans service  
     3. Cards service  
     4. Sau khi cả 3 đã đăng ký thành công với Eureka, mới khởi động lại Gateway Server.
   - Kiểm tra trên Eureka để đảm bảo tất cả dịch vụ đã đăng ký.

8. **Kiểm thử với Postman**  
   - Vì dùng H2 nên ban đầu không có dữ liệu:
     1. Gọi API tạo account qua Gateway (truyền `mobileNumber` trong body) → nhận 201.
     2. Gọi API tạo card qua Gateway → 201.
     3. Gọi API tạo loan qua Gateway → 201.
   - Sau đó:
     - Gọi `fetchCustomerDetails` qua Gateway:
       - Nhận response chứa: account details, loan details, card details.
       - Trong response headers có `EasyBank-correlationId`.
   - CorrelationId này có thể được khách hàng cung cấp lại cho EasyBank khi cần hỗ trợ, giúp dev truy vết log.

9. **Kiểm tra log**  
   - Trên Gateway:
     - Tìm theo correlationId, thấy log trong `RequestTraceFilter` (khi tạo traceId) và `ResponseTraceFilter` (khi thêm header vào response).
   - Trên Accounts:
     - Tìm correlationId, thấy log “easybank-correlationId found with value …”.
   - Trên Loans & Cards:
     - Tìm cùng correlationId, xác nhận request đã đi qua các dịch vụ này.
   - Nếu không thấy log ở Loans/Cards thì nghĩa là request không tới được các dịch vụ đó.

10. **Ý nghĩa & ứng dụng thực tế**  
    - Gateway Server đóng vai trò Edge Server, nơi triển khai các mối quan tâm xuyên suốt (cross-cutting concerns) như:
      - Logging, tracing, auditing, security, v.v.
    - Trong phần bảo mật microservice sau này:
      - Gateway sẽ được dùng để đảm bảo chỉ người dùng đã xác thực/được ủy quyền mới gọi được các microservice.
    - Slide bài giảng tóm tắt:
      - Tạo project Spring Boot cho Gateway.
      - Thêm dependencies, cấu hình `application.yml`.
      - Cấu hình routing.
      - Gọi các microservice qua Gateway (port 8072).
      - Áp dụng correlationId để trace request.

Tóm lại, bài giảng hướng dẫn cách:
- Truyền và log `easybank-correlation-id` xuyên suốt Gateway và các microservice (Accounts, Loans, Cards).
- Bật logging chi tiết để hỗ trợ debug.
- Kiểm thử toàn bộ luồng qua Gateway bằng Postman và xác minh bằng log.
