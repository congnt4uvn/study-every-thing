
Nội dung trên mô tả một vấn đề và cách khắc phục khi gọi API qua Gateway Server trong Spring Cloud Gateway:

1. Vấn đề:
- Khi gửi request đến Gateway bằng tên ứng dụng/dịch vụ viết HOA TOÀN BỘ (ví dụ: ACCOUNT, LOAN, CARD) thì hoạt động bình thường.
- Nhưng khi đổi sang chữ thường (lowercase) trong đường dẫn (path) thì bị lỗi 404 Not Found.
- Việc yêu cầu client luôn gửi tên ứng dụng hoặc microservice bằng chữ in hoa là rất bất tiện và không chuyên nghiệp, vì thông thường trong URL không nên dùng toàn chữ in hoa.

2. Giải pháp:
- Thêm một thuộc tính cấu hình mới trong file `application.yml` của Gateway Server, trong phần discovery locator.
- Thuộc tính mới là:
  `spring.cloud.gateway.discovery.locator.service-id-lower-case=true`
- Ý nghĩa: thông báo cho Gateway chấp nhận và xử lý tên service ở dạng chữ thường (lowercase).

3. Thực hiện:
- Cập nhật cấu hình trong IntelliJ (application.yml) với thuộc tính trên.
- Build lại ứng dụng Gateway.
- Thử lại trong Postman:
  - Khi dùng tên service viết thường trong URL, request trả về thành công.
  - Khi dùng chữ in hoa toàn bộ, có thể bị lỗi 404 tùy cấu hình.
- Tác giả cũng thử với API “fetch” (lấy thông tin theo số điện thoại):
  - Dùng chữ in hoa → 404 Not Found.
  - Dùng chữ thường → phản hồi thành công.

4. Kết luận:
- Nên dùng chữ thường trong đường dẫn URL cho chuyên nghiệp và đúng chuẩn.
- Thay đổi nhỏ trong cấu hình (`service-id-lower-case=true`) giúp Gateway hỗ trợ tên service viết thường, tránh yêu cầu bất thường là phải dùng chữ in hoa trong URL.
