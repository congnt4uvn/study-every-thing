
Dưới đây là tóm tắt nội dung:

1. Hiện trạng và vấn đề  
   - Trong môi trường cục bộ, các microservice kết nối được với container MySQL vì cấu hình `application.yml` dùng `localhost`.  
   - Khi chạy bằng Docker Compose hoặc Kubernetes, cấu hình này không dùng được do hostname bị hard-code là `localhost`.  
   - Giải pháp: dùng biến môi trường để truyền `datasource URL`, `username`, `password` thay vì cố định trong file cấu hình.

2. Chuẩn bị lại môi trường  
   - Dừng toàn bộ microservice đang chạy.  
   - Dừng và xóa tất cả container hiện có, vì sẽ tạo lại bằng Docker Compose, không chạy thủ công bằng `docker run`.  
   - Xóa bớt các Docker image cũ (phần 6) để giải phóng dung lượng.

3. Cập nhật phiên bản/tag và build lại image  
   - Mở các file `pom.xml` của từng service (accounts, loans, cards, config-server) và đổi tag từ `S6` sang `S7`.  
   - Refresh Maven.  
   - Build lại Docker image cho từng service bằng Maven + Jib:  
     - `mvn compile jib:dockerBuild -DskipTests` (ý chính là dùng `jib:dockerBuild`).  
   - Xác nhận các image mới đã được tạo trong Docker Desktop.  
   - Khuyến nghị đẩy image lên Docker Hub (hoặc registry riêng) để lưu trữ tập trung.

4. Cập nhật Docker Compose – phần database  
   - Mở file `docker-compose.yml` (mặc định).  
   - Xóa các service liên quan đến RabbitMQ vì đã bỏ Spring Cloud Bus.  
   - Tạo 3 service DB riêng:  
     - `accounts-db` (MySQL)  
     - `loans-db` (MySQL)  
     - `cards-db` (MySQL)  
   - Mỗi service DB:  
     - Dùng image `mysql`.  
     - Đặt `container_name` tương ứng (accounts-db, loans-db, cards-db).  
     - Map port:  
       - accounts-db: `3306:3306`  
       - loans-db: `3307:3306`  
       - cards-db: `3308:3306`  
     - Khai báo `healthcheck` (dùng lệnh `ping` hoặc lệnh từ tài liệu MySQL Docker) với `timeout`, `retries`, `interval`, `start_period`.  
     - Khai báo biến môi trường: `MYSQL_ROOT_PASSWORD`, `MYSQL_DATABASE`, …  
     - Gắn vào cùng network với các microservice (dùng cấu hình mạng chung từ `common-config.yml`).

5. Cập nhật Docker Compose – phần microservice  
   - Mở `common-config.yml` (hoặc file cấu hình chung) nơi đã có các service: accounts, loans, cards, config-server.  
   - Thêm biến môi trường chung cho tất cả microservice:  
     - `SPRING_DATASOURCE_USERNAME`  
     - `SPRING_DATASOURCE_PASSWORD`  
   - Với từng microservice, thêm biến môi trường riêng:  
     - `SPRING_DATASOURCE_URL`  
       - Lấy từ `application.yml` nhưng thay `localhost` bằng tên service DB trong Docker Compose:  
         - Accounts: host = `accounts-db`, port = `3306`, db = `accountsdb` (ví dụ)  
         - Loans: host = `loans-db`, port = `3307`, db = `loansdb`  
         - Cards: host = `cards-db`, port = `3308`, db = `cardsdb`  
   - Thêm `depends_on` cho từng microservice:  
     - Accounts service: phụ thuộc `accounts-db` với điều kiện `service_healthy`.  
     - Loans service: phụ thuộc `loans-db`.  
     - Cards service: phụ thuộc `cards-db`.  
   - Mục đích: Docker Compose chỉ khởi động microservice sau khi DB tương ứng đã “healthy”.

6. Giảm trùng lặp cấu hình DB trong Compose  
   - Nhận thấy nhiều cấu hình lặp lại giữa các DB service (image `mysql`, `healthcheck`, `MYSQL_ROOT_PASSWORD`, …).  
   - Tạo một service cấu hình chung trong `common-config.yml`, ví dụ:  
     - `microservice-db-config`  
     - Service này `extends` từ cấu hình mạng (deployment-network-config) và chứa các phần chung:  
       - `image: mysql`  
       - `healthcheck`  
       - `MYSQL_ROOT_PASSWORD` (và các biến chung khác).  
   - Trong `docker-compose.yml`, các service `accounts-db`, `loans-db`, `cards-db` sẽ:  
     - `extends: microservice-db-config`  
     - Chỉ giữ lại phần riêng: `container_name`, `ports`, `MYSQL_DATABASE`, …  
   - Thực hiện tương tự cho các file `docker-compose-qa.yml` và `docker-compose-prod.yml`.

7. Kết luận  
   - Đã:  
     - Build lại image với tag mới.  
     - Cấu hình Docker Compose để khởi động cả DB và microservice, liên kết qua network và hostname service.  
     - Dùng biến môi trường cho `datasource URL`, user, password.  
     - Thiết lập `depends_on` + `healthcheck` để đảm bảo DB sẵn sàng trước khi microservice chạy.  
     - Tách cấu hình chung DB vào `common-config.yml` để tránh trùng lặp.  
   - Bài giảng tiếp theo sẽ chạy `docker compose` để kiểm chứng các thay đổi.
