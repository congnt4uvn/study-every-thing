
Dưới đây là tóm tắt nội dung bài giảng:

1. **Mục tiêu**  
   - Tạo một REST API mới trong microservice Account để xóa thông tin khách hàng và tài khoản dựa trên số điện thoại di động.

2. **Thay đổi ở tầng Service**  
   - Thêm phương thức mới trong `IAccountService`:
     - `boolean deleteAccount(String mobileNumber);`
   - Trong `AccountService`:
     - Dùng `CustomerRepository.findByMobileNumber(mobileNumber)` để lấy `Customer`.
     - Nếu không tìm thấy, ném `ResourceNotFoundException`.
     - Lấy `customerId` từ `Customer`.
     - Xóa dữ liệu:
       - Gọi `customerRepository.deleteById(customerId)` (phương thức có sẵn từ framework).
       - Gọi `accountRepository.deleteByCustomerId(customerId)` để xóa các bản ghi account liên quan.

3. **Thay đổi ở tầng Repository**  
   - Trong `AccountsRepository` tạo phương thức:
     - `void deleteByCustomerId(Long customerId);`
   - Đây là phương thức custom, Spring Data JPA sẽ tự sinh câu lệnh `DELETE` dựa trên tên `deleteByCustomerId`.
   - Vì đây là thao tác thay đổi dữ liệu (delete), cần:
     - `@Transactional`
     - `@Modifying`
   - Hai annotation này đảm bảo:
     - Truy vấn chạy trong transaction.
     - Nếu có lỗi runtime, toàn bộ transaction sẽ rollback, tránh tình trạng xóa account mà không xóa customer (hoặc ngược lại).

4. **Lý do không cần @Transactional/@Modifying cho deleteById của CustomerRepository**  
   - `deleteById()` là phương thức có sẵn từ framework (CrudRepository).
   - Spring Data JPA đã tự xử lý transaction và việc modify dữ liệu cho các phương thức built-in này.
   - Chỉ các phương thức custom như `deleteByCustomerId` mới cần khai báo rõ `@Transactional` và `@Modifying`.

5. **Thay đổi ở tầng Controller**  
   - Tạo API mới với:
     - `@DeleteMapping("/delete")`
     - Tên phương thức: `deleteAccountDetails()`.
     - Nhận `mobileNumber` (query param).
     - Gọi `accountService.deleteAccount(mobileNumber)`.
     - Nếu trả về `true`: trả HTTP 200 (OK) với thông điệp “request processed successfully”.
     - Nếu trả về `false`: trả HTTP 500 với thông điệp lỗi “error occurred, please try again or contact dev team”.

6. **Kiểm thử API**  
   - Tạo mới một account (và customer) với số điện thoại cụ thể.
   - Gọi API GET để xác nhận dữ liệu tồn tại.
   - Gọi API DELETE `/api/delete?mobileNumber=...`:
     - Nhận phản hồi thành công.
   - Gọi lại API GET:
     - Nhận lỗi “not found” vì dữ liệu đã bị xóa.
   - Gọi lại API DELETE với cùng số điện thoại:
     - Nhận lỗi “not found” vì không còn dữ liệu để xóa.

7. **Kết luận**  
   - Đã hoàn thành đủ 4 thao tác CRUD cơ bản trong Account Service.
   - Bài giảng tiếp theo sẽ nói về:
     - Xử lý `RuntimeException` đúng chuẩn.
     - Kiểm tra các cột metadata với Spring Data JPA.
     - Ghi log cho REST API.
