
Dưới đây là tóm tắt nội dung bài giảng:

1. Chuẩn bị workspace và dự án  
- Tạo thư mục `section-nine` (phần chín) trong workspace.  
- Sao chép nội dung từ `section-eight` sang `section-nine`, đổi tên, xóa thư mục ẩn `.idea`.  
- Tạo một dự án Spring Boot mới trên start.spring.io với:  
  - Maven, Java, Spring Boot 3.1.2 (hoặc mới hơn), Java 17  
  - Group: `com.easybytes` (ví dụ), Artifact/Name: `gateway-server`  
  - Packaging: `jar`.

2. Thêm dependencies cho Gateway Server  
- Spring Cloud Gateway (gateway starter).  
- Eureka Discovery Client (để gateway kết nối Eureka).  
- Config Client (để gateway lấy cấu hình từ Config Server).  
- Actuator (để expose các endpoint quản lý).  
- DevTools (tăng năng suất, restart nhanh).

3. Import dự án vào IntelliJ  
- Giải nén file zip `gateway-server` vào `section-nine`.  
- Mở `section-nine` trong IntelliJ, để IntelliJ load tất cả các dự án Maven.  

4. Cấu hình Maven (pom.xml)  
- Mở `pom.xml` của một service (ví dụ account service), copy plugin Google Jib.  
- Dán plugin Jib vào `pom.xml` của `gateway-server` để build image Docker.  
- Đổi tên thẻ (tag) Docker từ `s8` sang `s9` trong tất cả các microservice (account, loan, card, config server, eureka server, v.v.) để đồng bộ version/tag.  
- Reload Maven và bật annotation processing.

5. Cấu hình application YAML cho Gateway Server  
- Đổi `application.properties` thành `application.yml`.  
- Copy nội dung `application.yml` từ một microservice (ví dụ account service) sang gateway, rồi chỉnh sửa:  
  - Giữ `spring.application.name` nhưng đổi giá trị thành `gatewayserver`.  
  - Xóa `server.port` (vì sẽ cấu hình trong Config Server).  
  - Xóa các cấu hình không cần cho gateway:  
    - Profiles, datasource, H2, JPA, v.v.  
  - Giữ `spring.config.import` để gateway kết nối Config Server (ví dụ `configserver:http://localhost:8071`).  
  - Giữ các cấu hình Actuator cần thiết (expose endpoints), nhưng xóa các cấu hình shutdown không cần thiết.  
  - Xóa các cấu hình Eureka trong file local vì sẽ đưa vào Config Server.  
  - Cập nhật thông tin `info` (name, description) cho phù hợp: ví dụ `gatewayserver`, `EasyBank Gateway Server`.

6. Bật endpoint gateway trong Actuator  
- Trong `application.yml` của gateway, thêm cấu hình để bật endpoint gateway:  
  - Dưới phần `management.endpoints.web.exposure.include`, thêm cấu hình:  
    - `management.endpoint.gateway.enabled=true` (hoặc tương đương trong YAML).  
- Mục đích: hiển thị các endpoint liên quan đến gateway trong Actuator.

7. Cấu hình trong Config Server (GitHub repo)  
- Trong repo cấu hình (GitHub), tạo file `gatewayserver.yml`.  
- Thêm các cấu hình chính:  
  - `server.port: 8072` để gateway chạy trên cổng 8072.  
  - Các cấu hình Eureka client để gateway đăng ký và lấy thông tin service từ Eureka (tương tự như account/loan/card services).  
- Nhờ đó, gateway sẽ biết endpoint thực tế của các microservice thông qua Eureka.

8. Kết nối Gateway với Eureka (Discovery Locator)  
- Trong `application.yml` của gateway, thêm cấu hình:  
  - `spring.cloud.gateway.discovery.locator.enabled=true`.  
- Ý nghĩa:  
  - Gateway sử dụng Discovery Client (Eureka) để tự động phát hiện các microservice.  
  - Gateway dùng thông tin này để định tuyến và chuyển tiếp request từ client bên ngoài đến đúng microservice.

9. Kết luận  
- Đã thiết lập:  
  - Gateway Server với Spring Cloud Gateway.  
  - Kết nối Gateway với Config Server và Eureka Server.  
  - Cấu hình Actuator và Docker (Jib).  
- Bài tiếp theo: khởi động tất cả microservice + Gateway + Eureka + Config Server, rồi test các microservice thông qua Gateway thay vì gọi trực tiếp.
