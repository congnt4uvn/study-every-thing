
Bài giảng trên nói về vấn đề khi dùng `docker run` thủ công cho nhiều microservice và cách giải quyết bằng Docker Compose:

1. **Vấn đề với `docker run`**  
   - Mỗi microservice (accounts, loans, cards) đã được đóng gói thành một Docker image.  
   - Để chạy, phải dùng `docker run` cho từng image, kèm port mapping.  
   - Nếu có nhiều service hoặc nhiều instance (ví dụ 100 microservice), việc gõ lệnh thủ công là không khả thi, rất phức tạp và tốn thời gian.

2. **Giới thiệu Docker Compose**  
   - Docker Compose là công cụ để định nghĩa và chạy nhiều container Docker.  
   - Dùng một file cấu hình YAML (thường là `docker-compose.yml`) để mô tả tất cả service.  
   - Chỉ cần một lệnh duy nhất để tạo/khởi động, dừng, rebuild, xem trạng thái, xem log, chạy lệnh một lần cho toàn bộ service.  
   - Có thể dùng trong mọi môi trường: dev, test, staging, production, CI.  
   - Khi cài Docker Desktop, Docker Compose thường được cài sẵn; có thể kiểm tra bằng `docker compose version`. Nếu chưa có, có thể cài theo hướng dẫn trên trang chính thức.

3. **Tạo file cấu hình Docker Compose**  
   - Tạo file `docker-compose.yml` (ở ví dụ là trong project microservice `accounts`, nhưng thực tế có thể đặt ở đâu cũng được).  
   - Cấu trúc YAML:
     - Gốc: `services:`  
       - `accounts:`  
         - `image: "eazybytes/accounts:S4"`  
         - `container_name: accounts-ms`  
         - `ports:`  
           - `- "8080:8080"`  
         - `deploy:`  
           - `resources:`  
             - `limits:`  
               - `memory: 700M`  
       - `loans:`  
         - `image: "eazybytes/loans:S4"`  
         - `container_name: loans-ms`  
         - `ports:`  
           - `- "8090:8090"`  
         - `deploy` … (giới hạn bộ nhớ tương tự)  
       - `cards:`  
         - `image: "eazybytes/cards:S4"`  
         - `container_name: cards-ms`  
         - `ports:`  
           - `- "9000:9000"`  
         - `deploy` … (giới hạn bộ nhớ tương tự)  
   - Lưu ý:
     - YAML rất nhạy với khoảng trắng/tab; phải thụt lề đúng.  
     - `container_name` giúp đặt tên container dễ nhận biết, tránh tên ngẫu nhiên khó phân biệt.  
     - `deploy.resources.limits.memory` dùng để giới hạn bộ nhớ (ví dụ 700M) cho mỗi container.

4. **Thiết lập mạng chung cho các microservice**  
   - Mặc định, mỗi service có thể nằm trên mạng riêng, gây khó khăn nếu cần giao tiếp lẫn nhau.  
   - Giải pháp: gán tất cả service vào cùng một network, ví dụ `easybank`.  
   - Trong từng service:
     - Thêm:  
       ```yaml
       networks:
         - easybank
       ```  
   - Ở cấp gốc (cùng cấp với `services:`), định nghĩa network:
     ```yaml
     networks:
       easybank:
         driver: bridge
     ```  
   - Điều này yêu cầu Docker tạo một network tên `easybank` với driver `bridge`, cho phép các container (accounts, loans, cards) giao tiếp với nhau qua cùng mạng.

5. **Kết luận**  
   - File `docker-compose.yml` đã sẵn sàng, chứa cấu hình cho 3 microservice, giới hạn tài nguyên và mạng chung.  
   - Ở bài giảng tiếp theo, giảng viên sẽ hướng dẫn cách khởi động tất cả microservice chỉ với một lệnh sử dụng Docker Compose.
