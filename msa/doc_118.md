
Bài giảng mô tả quá trình cấu hình và khởi động toàn bộ hệ thống microservices (config server, Eureka server và các microservice khác) bằng một lệnh `docker compose`.

Tóm tắt các ý chính:

1. **Mục tiêu**  
   - Khởi động tất cả microservice và config server, Eureka server bằng một lệnh:  
     - Điều hướng đến thư mục `Section8/docker-compose/default`  
     - Chạy `docker compose up -d` (trong bài giảng dùng “docker compose app -d”, ý là `up -d`).  
   - Trước khi chạy, dừng/xóa mọi container đang chạy hoặc đã dừng để tránh thiếu bộ nhớ.

2. **Sự cố ban đầu**  
   - Config server khởi động trước, chỉ khi ổn định thì Eureka server mới khởi động.  
   - Eureka server không khởi động được, log cho thấy không kết nối được với config server.  
   - Nguyên nhân: cấu hình Docker compose cho Eureka chưa đúng, đặc biệt là phần kế thừa cấu hình và biến môi trường.

3. **Tái cấu trúc cấu hình Docker compose**  
   - Ban đầu Eureka server mở rộng từ `microservice-base-config` (chỉ có cấu hình mạng & deploy).  
   - Giảng viên tạo thêm một service cấu hình riêng cho Eureka:
     - Tên: `microservice-eureka-config`.
     - Service này `extends` từ `microservices-configserver-config`.
     - Xóa các dependency không cần thiết (vì đã có trong service cha).
     - Giữ lại/định nghĩa các biến môi trường cần thiết cho Eureka (ví dụ `SPRING_PROFILES_ACTIVE`, `SPRING_CONFIG_IMPORT`, …) nhưng không dùng cấu hình dành cho client Eureka.
   - Trong config server:
     - Xóa dependency vào Eureka server.
     - Xóa các biến môi trường liên quan đến Eureka client (vì đã chuyển sang service cấu hình Eureka riêng).
   - Trong các microservice (accounts, loans, cards):
     - Cập nhật `docker-compose.yml` để:
       - Sử dụng `microservice-eureka-config` cho cấu hình Eureka.
       - Mở rộng `microservice-configserver-config` cho config server.
       - Loại bỏ các cấu hình trùng lặp vì đã có trong `common-config.yml`.

4. **Lỗi YAML (thụt lề biến môi trường)**  
   - Sau khi chỉnh sửa, Eureka vẫn không khởi động.  
   - Kiểm tra lại `common-config.yml` phát hiện lỗi:  
     - Các biến môi trường (environment variables) không được thụt lề đúng, không nằm dưới khóa `environment` như các phần tử con.  
   - Sửa lại thụt lề: mọi biến môi trường phải là child của `environment`.  
   - Lưu file, chạy lại `docker compose up -d`.

5. **Kết quả sau khi sửa**  
   - Eureka server khởi động thành công.  
   - Các microservice (loans, cards, accounts, …) lần lượt khởi động.  
   - Kiểm tra trong Docker logs: Eureka và các microservice đều chạy ổn.  
   - Mở trình duyệt truy cập `http://localhost:8070` (Eureka dashboard):  
     - Tất cả microservice đã đăng ký thành công với Eureka.

6. **Kiểm thử API với Postman**  
   - Thực hiện chuỗi API:
     1. Tạo account (API create account).
     2. Tạo card với cùng số điện thoại.
     3. Tạo loan với cùng số điện thoại.
   - Gọi API “get customer details”:  
     - Feign client sử dụng Eureka để gọi các microservice loans và cards.  
     - Nhận được response đầy đủ: thông tin customer, account, loan, card → chứng minh Eureka + Feign hoạt động đúng trong Docker.

7. **Đồng bộ cấu hình cho các profile khác (qa, prod)**  
   - Sao chép các thay đổi trong `docker-compose.yml` (profile default) sang:
     - `qa/docker-compose.yml`
     - `prod/docker-compose.yml`
   - Sao chép `common-config.yml`:
     - Vào thư mục `prod`, đổi profile từ `default` sang `prod`.
     - Vào thư mục `qa`, đổi profile từ `default` sang `qa`.
   - Đảm bảo tất cả file Docker compose cho `default`, `qa`, `prod` dùng chung cấu trúc đúng và hoạt động ổn.

8. **Kết thúc**  
   - Chạy lệnh Docker compose để dừng và xóa tất cả container.  
   - Nhấn mạnh tầm quan trọng của:
     - Cấu hình `extends` đúng giữa các service.
     - Thụt lề YAML chính xác, đặc biệt với `environment`.
   - Kết luận: các thay đổi Eureka hoạt động tốt trong Docker; người học được yêu cầu thử lại và báo lỗi nếu có.
