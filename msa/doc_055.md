
Bài giảng nói về cách dùng Docker Compose để khởi động và dừng nhiều microservice chỉ với một lệnh:

1. **Vị trí file docker-compose.yml**  
   - Khi ở thư mục microservice (ví dụ `cards-ms`) mà không có file `docker-compose.yml`, chạy `docker compose up` sẽ báo lỗi “no configuration file provided”.  
   - Cần chuyển về thư mục gốc (ví dụ `accounts`) nơi có file `docker-compose.yml` rồi mới chạy lệnh.

2. **Khởi động tất cả microservice**  
   - Dùng: `docker compose up` để khởi động tất cả container được định nghĩa trong file YAML.  
   - Để không chiếm terminal, dùng: `docker compose up -d` (chạy ở chế độ nền).  
   - Sau đó dùng `docker ps` để kiểm tra, thấy 3 dịch vụ chạy trên các port 8080, 8090, 9000.  
   - Trong Docker Desktop (UI), có thể thấy các container thuộc thư mục `accounts`. Tên hiển thị có thể hơi khác (chỉ hiện `ms` thay vì `accounts-ms`), tác giả cho rằng đây là lỗi hiển thị nhưng trên CLI vẫn đúng.

3. **Kiểm tra hoạt động của các dịch vụ**  
   - Gửi request tạo Account (port 8080) → nhận phản hồi thành công.  
   - Gửi request tạo Loan → thành công.  
   - Gửi request tạo Card → thành công.  
   → Xác nhận tất cả container đã khởi động đúng bằng một lệnh.

4. **Dừng và xóa container**  
   - Dùng: `docker compose down` để dừng và xóa tất cả container do compose tạo.  
   - Không dùng `-d` với `down` (không có ý nghĩa).  
   - Sau khi `down`, vào Docker UI sẽ không còn container nào.  
   - Nếu chỉ muốn dừng mà không xóa, có thể dùng `docker compose stop`, nhưng tác giả khuyến nghị thường nên dùng `down` để xóa luôn.

5. **Lưu ý về mạng và YAML**  
   - Sẽ có phần sau demo về mạng (network) `easybank` và cách các container giao tiếp qua bridge network.  
   - Có thể định nghĩa bất kỳ số lượng dịch vụ nào trong file docker-compose và quản lý chúng bằng một lệnh (up/down/restart).  
   - Cần chú ý cấu trúc và cú pháp YAML, đặc biệt là thụt lề (indentation); mỗi khoảng trắng đều quan trọng.  
   - File mẫu có thể xem trong GitHub repository.

Tóm lại:  
- `docker compose up -d` → khởi động tất cả microservice ở nền.  
- `docker compose down` → dừng và xóa tất cả container.  
- Phải chạy các lệnh này từ thư mục chứa `docker-compose.yml`.
