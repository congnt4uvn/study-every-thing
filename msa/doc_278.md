
Dưới đây là tóm tắt nội dung bài giảng:

1. Tạo dự án BOM:
- Tạo một dự án Spring Boot Maven mới tên là `eazy-bom` (BOM = Bill Of Materials).
- Ngôn ngữ: Java, Build: Maven, Group: `com.eazybyte`, Artifact: `eazy-bom`, JDK/Java 21, packaging ban đầu là jar.
- Không thêm bất kỳ dependency Spring Boot starter nào khi tạo.

2. Dọn cấu trúc dự án:
- Xóa toàn bộ thư mục source (không chứa code trong dự án BOM, chỉ dùng để quản lý dependency).
- Mở `pom.xml` và:
  - Xóa phần `<parent>` (không dùng `spring-boot-starter-parent`).
  - Cập nhật phần mô tả (description) để nói rõ đây là BOM chung cho các microservice.
  - Thêm các thẻ tùy chọn như: `url`, `licenses`, `developers`, `scm` (GitHub repo). Các thẻ này là optional, có thể bỏ nếu muốn.

3. Đặt packaging là `pom`:
- Thêm thẻ `<packaging>pom</packaging>`.
- Đây là điểm quan trọng: để POM đóng vai trò BOM (cha cho các microservice), packaging phải là `pom`, không phải `jar`.

4. Định nghĩa các thuộc tính (properties):
- Xóa phần properties cũ (ví dụ `java.version`) và thêm một loạt thuộc tính mới:
  - Phiên bản Spring Boot.
  - Phiên bản các thư viện bên thứ ba: SpringDoc, H2, Lombok, OpenTelemetry, Micrometer, v.v.
- Mục đích: chủ động kiểm soát version của các thư viện bên thứ ba, không phụ thuộc hoàn toàn vào version mà Spring Boot starter tự chọn.

5. Quản lý dependency:
- Xóa toàn bộ dependency hiện có (ví dụ `spring-boot-starter`, `spring-boot-starter-test`), để trống phần `<dependencies>`.
- Thêm phần `<dependencyManagement>` và định nghĩa các dependency quan trọng ở đây:
  1) BOM Spring Boot:
     - Import BOM của Spring Boot (`spring-boot-dependencies`) với:
       - `<type>pom</type>`
       - `<scope>import</scope>`
       - Version lấy từ thuộc tính đã định nghĩa.
  2) Lombok:
     - Thêm dependency Lombok như một BOM (type = pom, scope = import), version lấy từ properties.
  3) H2 Database:
     - Thêm dependency H2 (cũng quản lý version tại đây).
  4) SpringDoc:
     - Thêm dependency SpringDoc để hỗ trợ OpenAPI/Swagger.
  5) `spring-boot-starter-test`:
     - Thêm như dependency bình thường (không dùng type = pom, scope = import) vì đây là thư viện test nhỏ, không phải BOM.
  6) Spring Cloud:
     - Import BOM của Spring Cloud (type = pom, scope = import) để dùng các module như Eureka, Config Server, v.v.

6. Ý nghĩa:
- `eazy-bom` là một dự án Maven kiểu `pom` dùng làm BOM chung cho tất cả microservice.
- Tất cả version của Spring Boot, Spring Cloud và thư viện bên thứ ba được quản lý tập trung trong BOM.
- Các microservice con chỉ cần khai báo dependency mà không phải lặp lại version; version sẽ được lấy từ BOM.

7. Kết luận:
- Để tạo BOM:
  - Đặt packaging = `pom`.
  - Xóa `<parent>` Spring Boot.
  - Định nghĩa các thuộc tính version cần dùng.
  - Dùng `<dependencyManagement>` để khai báo các dependency (đặc biệt là các BOM khác) với type = pom, scope = import khi phù hợp.
- Bài tiếp theo sẽ hướng dẫn cách áp dụng `eazy-bom` vào từng microservice để thấy hiệu quả.
