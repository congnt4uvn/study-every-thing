
Dưới đây là tóm tắt nội dung:

1. Tổ chức cấu trúc thư mục:
   - Tạo thư mục `observability`.
   - Bên trong tạo thư mục `loki` và lưu file `loki-config.yml` vào đó.
   - Tạo thư mục `promtail` (trong video gọi nhầm là “promptaile”) và lưu file `promtail-local-config.yml` vào đó.
   - Hai file cấu hình này chính là những file đã được giải thích ở các phần trước.

2. Chỉnh sửa `docker-compose.yml` (môi trường prod trước, rồi sẽ copy sang môi trường khác):
   - Mở file `docker-compose.yml` trong thư mục `prod`.
   - Xóa container Redis (đã dùng trước đó cho rate limiting) để tránh có quá nhiều container không cần thiết.
   - Trong cấu hình `gateway-server`, xóa các biến môi trường và phụ thuộc liên quan đến Redis.
   - Tìm tất cả tag `s10` và thay bằng `s11`.

3. Thêm các dịch vụ observability (Loki, Promtail, Minio, Grafana):
   - Sao chép các dịch vụ từ tài liệu chính thức vào `docker-compose.yml`, nhưng:
     - Không sao chép dịch vụ `flog` (chỉ là app test, không dùng).
   - Điều chỉnh cấu hình Loki:
     - Cập nhật volume để trỏ đúng đến file `loki-config.yml` mới:
       - Từ vị trí cũ sang `observability/loki/loki-config.yml`.
       - Mount vào trong container tại `loki/config.yml`.
     - Cập nhật cấu hình mạng:
       - Thay network `loki` bằng network `easybank` (tên đã định nghĩa trong `common-config.yml`).
       - Giữ alias là `loki`.
       - Đặt tên anchor là `loki-dns` để tái sử dụng.
   - Phần Loki write:
     - Cập nhật volume tương tự như phần Loki read (copy cấu hình volume từ read sang write).
     - Đảm bảo biến `loki-dns` trỏ đến network `easybank`.

4. Cấu hình Promtail:
   - Cập nhật volume để trỏ đến file `promtail-local-config.yml`:
     - Từ vị trí cũ sang `observability/promtail/promtail-local-config.yml`.
     - Mount vào container ở đường dẫn phù hợp (như trong tài liệu chính thức).
   - Cập nhật cấu hình mạng:
     - Thay vì hard-code network `easybank`, dùng cấu hình mở rộng từ `common-config.yml`:
       - `x-extends` (hoặc tương đương) với service `network-deploy-service`.

5. Cấu hình Minio và Grafana:
   - Không cần thay đổi gì ngoài phần network.
   - Thay network cũ bằng cấu hình mở rộng từ `common-config.yml` (giống Promtail).
   - Sao chép cấu hình mạng tương tự cho `gateway` (cổng API).

6. Kết luận:
   - Đã:
     - Tổ chức lại file cấu hình Loki/Promtail.
     - Cập nhật `docker-compose.yml` để tích hợp Loki, Promtail, Minio, Grafana.
     - Loại bỏ Redis container không còn cần thiết.
     - Chuyển tag image từ `s10` sang `s11`.
   - Bước tiếp theo:
     - Build tất cả Docker image với tag `s11` ở chế độ nền.
     - Phần tiếp theo của bài giảng sẽ tiếp tục demo, đặc biệt là với các công cụ Grafana.
