
Dưới đây là tóm tắt nội dung:

1. Mục tiêu  
   - Thiết lập Grafana trong cụm Kubernetes trước khi triển khai toàn bộ các microservice.  
   - Sử dụng chart Helm Grafana từ thư mục Bitnami đã được commit lên GitHub, để tận dụng sẵn các chỉnh sửa trong values.yaml.

2. Cấu hình Grafana (values.yaml)  
   - Mở file values.yaml của chart Grafana để cấu hình sẵn các data source (Prometheus, Loki, Tempo).  
   - Tìm phần cấu hình `datasources` và sử dụng cơ chế `secretDefinition` mà chart cung cấp.  
   - Trong `secretDefinition`, khai báo:
     - `apiVersion: 1`
     - Xóa các data source cũ nếu trùng tên.
     - Định nghĩa 3 data source:
       - Prometheus: trỏ đến DNS service Prometheus trong cluster.
       - Tempo: trỏ đến service `grafana-tempo-query-frontend` (port 3200).
       - Loki: trỏ đến service gateway của Loki (port 80).
   - Cấu hình thêm phần “derived fields” để tích hợp Loki với Tempo (giống như khi test local).

3. Cài đặt Grafana bằng Helm  
   - Di chuyển vào thư mục chart Grafana.  
   - Chạy `helm dependency build` để build dependencies.  
   - Quay lại thư mục cha, chạy:
     - `helm install grafana Grafana`  
       (release name: `grafana`, chart name: `Grafana`).
   - Mặc định Grafana được expose dưới dạng `ClusterIP`.

4. Truy cập Grafana (port-forward)  
   - Dùng `kubectl port-forward` để truy cập tạm thời:
     - Ban đầu dùng port 8080 → xung đột với một service khác (LoadBalancer cũng dùng 8080).  
     - Dừng lệnh cũ, chạy lại port-forward với port 3000 → Grafana sẽ truy cập tại `localhost:3000`.

5. Đăng nhập Grafana  
   - Mở trình duyệt, truy cập `http://localhost:3000`.  
   - Username: `admin`.  
   - Password: lấy bằng 2 lệnh `kubectl` (theo hướng dẫn chart), copy giá trị mật khẩu và đăng nhập.

6. Kiểm tra data source  
   - Vào mục “Explore” trong Grafana.  
   - Kiểm tra dropdown data source: thấy đủ 3 nguồn Prometheus, Loki, Tempo → xác nhận cấu hình data source thành công.

7. Kết luận & bước tiếp theo  
   - Dừng port-forward; khi cần lại có thể chạy lại lệnh port-forward để truy cập Grafana.  
   - Chạy `helm ls` để xem tất cả release đã cài: Grafana, Kafka, Keycloak, Loki, Prometheus, Tempo.  
   - Bài tiếp theo: triển khai các microservice vào cụm Kubernetes.
