
Dưới đây là tóm tắt ngắn gọn nội dung bài giảng:

1. Các lựa chọn để triển khai **theo dõi phân tán** trong microservices:
   - **Spring Cloud Sleuth + Zipkin**  
     - Tự động thêm traceId, spanId vào log.  
     - Dev gần như không phải sửa code log, chỉ cần thêm dependency.  
     - Tuy nhiên, Spring Cloud Sleuth sẽ dừng ở phiên bản 3.1 và bị thay thế bởi Micrometer Tracing → trở nên lỗi thời, nên giảng viên không dùng.

2. **Micrometer Tracing**  
   - Cùng dự án với Micrometer Metrics (đã dùng với Prometheus).  
   - Hỗ trợ traceId, spanId, tích hợp với OpenZipkin, OpenTelemetry.  
   - Nhưng: cần khá nhiều cấu hình và thay đổi từ phía developer → giảng viên cũng không chọn cách này.

3. Lựa chọn được sử dụng trong khóa học: **OpenTelemetry**
   - Cũng cung cấp khả năng theo dõi phân tán như Micrometer, nhưng:
     - Là dự án **mã nguồn mở**, thuộc **CNCF**, không phụ thuộc nhà cung cấp.
     - Hỗ trợ rất nhiều ngôn ngữ: C++, .NET, Go, Java, JavaScript, Python, Ruby, Rust, Swift, v.v.
     - Tích hợp tốt với các framework phổ biến: Spring, ASP, Express, Quarkus…
   - Ưu điểm lớn:  
     - Dễ áp dụng cho nhiều ngôn ngữ khác nhau → phù hợp hệ thống microservices đa ngôn ngữ.  
     - Với Java, có thể dùng **Java Agent** để “tự động đo” (auto-instrumentation) mà không cần sửa nhiều code.

4. Cách dùng OpenTelemetry Java Agent (tự động):
   - Đảm bảo file jar agent OpenTelemetry có trong classpath.
   - Khi chạy ứng dụng Java:
     - Thêm tham số `-javaagent:/path/to/opentelemetry-javaagent.jar`.
     - Thiết lập thuộc tính/biến môi trường `otel.service.name` (hoặc tương đương) để đặt tên service (metadata cho trace).
   - Nhờ đó:
     - Không cần chỉnh sửa nhiều trong ứng dụng.
     - Chỉ cần cấu hình agent và tên dịch vụ cho từng microservice.

5. Kết luận:
   - Spring Cloud Sleuth: sắp lỗi thời → không dùng.
   - Micrometer Tracing: mạnh nhưng cần nhiều cấu hình → không dùng trong khóa học.
   - **OpenTelemetry**: giải pháp được chọn vì:
     - Hỗ trợ đa ngôn ngữ, đa framework.
     - Dễ triển khai với Java Agent.
     - Thay đổi tối thiểu nhưng vẫn triển khai được theo dõi phân tán cho các microservices.
