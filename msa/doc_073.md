
Dưới đây là tóm tắt nội dung bài giảng:

1. **Tạo nhiều file cấu hình (profile) cho microservice Account**  
   - Trong thư mục `resources`, tạo thêm:
     - `application_qa.yml`
     - `application_prod.yml`  
   - Tổng cộng có 3 file:
     - `application.yml` → profile mặc định (dev/local)
     - `application_qa.yml` → profile QA / test
     - `application_prod.yml` → profile prod

2. **Xác định thuộc tính nào thay đổi theo môi trường**  
   - Các thuộc tính **không đổi** giữa các môi trường:
     - `server.port` (cổng chạy ứng dụng)
     - Cấu hình DB H2  
   → Giữ trong `application.yml` (mặc định), không cần đưa vào file profile khác.
   - Các thuộc tính **có thể thay đổi**:
     - `build.version`
     - Các thuộc tính liên quan đến account (message, contact details, v.v.)

3. **Thiết lập giá trị khác nhau cho từng profile**  
   - Trong `application.yml` (dev/local):
     - `build.version: 3.0`
   - Trong `application_qa.yml`:
     - `build.version: 2.0`
     - Message: “Welcome to EazyBank accounts related QA APIs”
     - Thông tin liên hệ: Head QA (tên, email, số điện thoại khác với default)
   - Trong `application_prod.yml`:
     - `build.version: 1.0`
     - Message: “…production APIs”
     - Thông tin liên hệ: Product Owner (tên, email, số điện thoại)

4. **Khai báo profile trong từng file YAML**  
   Trong `application_qa.yml`:
   ```yaml
   spring:
     config:
       activate:
         on-profile: qa
   ```
   Trong `application_prod.yml`:
   ```yaml
   spring:
     config:
       activate:
         on-profile: prod
   ```
   → Khi profile `qa` hoặc `prod` được kích hoạt, các file tương ứng sẽ được Spring Boot load.

5. **Import các file profile vào `application.yml`**  
   Trong `application.yml`:
   ```yaml
   spring:
     config:
       import:
         - application_qa.yml
         - application_prod.yml
   ```
   → Spring Boot biết phải load thêm hai file cấu hình này.

6. **Kích hoạt profile**  
   - Nếu **không** kích hoạt gì, Spring Boot dùng **profile mặc định** (`application.yml`).
   - Để kích hoạt QA profile trong `application.yml`:
     ```yaml
     spring:
       profiles:
         active: qa
     ```
   - Khi đó:
     - Spring Boot load `application.yml` (default) trước.
     - Sau đó load `application_qa.yml` và **ghi đè** các thuộc tính trùng key (ví dụ `build.version`, message, contact info).

7. **Kiểm tra kết quả**  
   - Khi chưa kích hoạt profile:
     - API trả về `build.version = 3.0`
     - Message và contact info là của môi trường local/dev.
   - Khi `spring.profiles.active=qa`:
     - `build.version = 2.0`
     - Message là QA API, contact info là Head QA.
   → Xác nhận profile QA đã được kích hoạt và ghi đè giá trị mặc định.

8. **Vấn đề khi chuyển sang prod**  
   - Nếu muốn chuyển sang prod bằng cách sửa `spring.profiles.active` thành `prod` trong `application.yml`, ta phải:
     - Sửa code/config
     - Build lại, tạo lại Docker image  
   → Điều này làm cho codebase không còn “bất biến” (immutable) giữa các môi trường.

9. **Hướng mở cho bài sau**  
   - Bài tiếp theo sẽ hướng dẫn:
     - Cách kích hoạt profile bằng **tham số bên ngoài** (external parameters) khi khởi động ứng dụng.
     - Ví dụ: dùng tham số dòng lệnh, biến môi trường, v.v.  
   → Giúp thay đổi profile linh hoạt mà không cần sửa file cấu hình và build lại.

Nếu bạn muốn, tôi có thể giúp bạn viết cụ thể nội dung cho các file `application.yml`, `application_qa.yml`, `application_prod.yml` theo đúng cấu trúc mà bài giảng mô tả.
