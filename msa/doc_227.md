
Dưới đây là tóm tắt nội dung:

1. Kiểm tra trạng thái cụm Kubernetes trước khi triển khai  
   - Dùng các lệnh:
     - `kubectl get deployments`
     - `kubectl get services`
     - `kubectl get replicasets`
     - `kubectl get pods`  
   - Kết quả: trong namespace `default` chưa có deployment, replica set, pod nào; chỉ có dịch vụ mặc định của Kubernetes.  
   - Xác nhận tương tự trên Kubernetes Dashboard: mục Deployments, ReplicaSets, Pods, Services đều trống (ngoại trừ service mặc định).

2. Triển khai Config Server lên Kubernetes  
   - Đảm bảo đang ở đúng thư mục chứa file manifest: `config-server.yaml`.  
   - Chạy lệnh:  
     `kubectl apply -f config-server.yaml`  
   - Kết quả: deployment và service cho Config Server được tạo thành công.

3. Kiểm tra sau khi triển khai  
   - `kubectl get deployments`: thấy deployment mới tên `config-server-deployment`, desired = 1, current = 1 → container chạy bình thường.  
   - `kubectl get services`:  
     - Có service mới cho Config Server, kiểu `LoadBalancer`.  
     - Có ClusterIP dùng trong cụm.  
     - External IP là `localhost` (vì chạy trên máy cục bộ).  
     - Service được expose ra ngoài tại cổng 8071 (trong cụm có thể là một NodePort như 30153, sẽ giải thích sau).  
   - `kubectl get replicasets`:  
     - ReplicaSet cho Config Server, desired = 1, current = 1.  
     - Nếu current = 0, Kubernetes sẽ tự tạo container mới để thay thế.  
   - `kubectl get pods`:  
     - Thấy pod của Config Server, trạng thái `Running`.

4. Xác nhận trên Kubernetes Dashboard  
   - Mục Services: thấy service `config-server`, kiểu `LoadBalancer`, endpoint external là `localhost:8071`.  
   - Mục Deployments: thấy deployment Config Server, xem chi tiết image, labels, v.v.  
   - Mục Pods:  
     - Mở chi tiết pod, xem logs → thấy log khởi động Spring Boot của Config Server.  
     - Đây là nơi dùng để debug khi container gặp vấn đề.  
   - Mục ReplicaSets: xem desired/current, pod nào đang chạy, liên kết với service nào.

5. Kiểm tra truy cập Config Server  
   - Truy cập các URL ví dụ:  
     - `http://localhost:8071/accounts/prod` → nhận được các thuộc tính cấu hình cho microservice `accounts` (profile `prod` + `default`).  
     - `http://localhost:8071/loans/prod` → nhận cấu hình cho `loans`.  
     - `http://localhost:8071/eureka/default` → nhận cấu hình cho Eureka (chỉ dùng profile `default`).  
   - Điều này chứng minh Config Server đã được triển khai và hoạt động đúng trong cụm Kubernetes.

Kết luận: Bài giảng minh họa quy trình triển khai Spring Cloud Config Server lên cụm Kubernetes (cục bộ), kiểm tra tài nguyên bằng `kubectl` và Kubernetes Dashboard, và xác nhận Config Server hoạt động bằng cách truy cập các endpoint cấu hình.
