
Bài giảng giải thích các thuật ngữ/ vai trò cơ bản trong OAuth2 thông qua ví dụ Stack Overflow đăng nhập bằng GitHub:

1. **Resource Owner (Chủ sở hữu tài nguyên)**  
   - Thường là **người dùng cuối**.  
   - Là người sở hữu dữ liệu/tài nguyên trên hệ thống (ví dụ: email, tên hiển thị trên GitHub; ảnh trên Google Photos).  
   - Họ quyết định có cho phép ứng dụng bên thứ ba truy cập dữ liệu của mình hay không.

2. **Client (Ứng dụng khách)**  
   - Là **website, ứng dụng mobile hoặc API** muốn truy cập tài nguyên thay mặt cho Resource Owner.  
   - Ví dụ: trang web Stack Overflow là client, muốn lấy email và thông tin hồ sơ của bạn từ GitHub.

3. **Authorization Server (Máy chủ ủy quyền)**  
   - Máy chủ chịu trách nhiệm **xác thực Resource Owner** (đăng nhập) và **cấp Access Token** cho Client.  
   - Resource Owner phải có tài khoản trên server này (ví dụ: bạn phải có tài khoản GitHub).  
   - Trong ví dụ: GitHub đóng vai trò Authorization Server.

4. **Resource Server (Máy chủ tài nguyên)**  
   - Máy chủ **lưu trữ tài nguyên** của Resource Owner (email, profile, ảnh, repo…).  
   - Client dùng **Access Token** gửi đến Resource Server để lấy dữ liệu.  
   - Resource Server kiểm tra token, nếu hợp lệ thì trả về tài nguyên.  
   - Trong GitHub: server lưu email, profile; trong Google Photos: server lưu ảnh.

5. **Scope (Phạm vi)**  
   - Là **tập quyền cụ thể** mà Client yêu cầu, ví dụ:  
     - Chỉ đọc email người dùng.  
     - Đọc dữ liệu hồ sơ.  
     - Toàn quyền đọc/ghi repo (scope `repo` – rất mạnh, cần cẩn trọng).  
   - Authorization Server cấp Access Token kèm theo scope.  
   - Scope quyết định Client được phép làm gì trên Resource Server.  
   - Người dùng phải **đồng ý** (consent) với các scope này (ví dụ: GitHub hiển thị “ứng dụng này có thể đọc email riêng tư của bạn” và bạn chọn Allow/Authorize).

Ví dụ luồng Stack Overflow – GitHub:  
- Bạn (Resource Owner) chọn “Sign up with GitHub”.  
- GitHub (Authorization Server) xác thực bạn (nếu chưa đăng nhập) và hỏi bạn có cho phép Stack Overflow (Client) đọc email/profile không (scope).  
- Nếu bạn đồng ý, GitHub cấp Access Token cho Stack Overflow.  
- Stack Overflow dùng token đó gọi đến Resource Server của GitHub để lấy email, tên hiển thị và tạo tài khoản cho bạn.

Kết luận:  
- Nắm vững các thuật ngữ: **Resource Owner, Client, Authorization Server, Resource Server, Scope** là nền tảng để hiểu và giải thích OAuth2, đặc biệt khi triển khai bảo mật cho microservices hoặc đi phỏng vấn.
