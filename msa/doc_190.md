
Dưới đây là tóm tắt nội dung bài giảng:

1. Xác định token endpoint  
   - Vào Keycloak Admin → Realm Settings → phần Endpoints.  
   - Mở cấu hình OpenID Endpoint để xem danh sách URL.  
   - Tìm URL có tên `token endpoint` (ví dụ: `http://localhost:7080/realms/master/protocol/openid-connect/token`).  
   - Đây là URL mà client phải gọi để lấy access token.

2. Gửi yêu cầu lấy access token bằng Postman (Client Credentials Flow)  
   - Tạo request POST đến token endpoint.  
   - Phần Body chọn `x-www-form-urlencoded` (form-url-encoded).  
   - Gửi các tham số:
     - `grant_type`: `client_credentials` (luồng Client Credentials).  
     - `client_id`: ID của client đã tạo trong Keycloak.  
     - `client_secret`: bí mật của client.  
     - `scope`: `openid email profile` (các scope mà client yêu cầu).

3. Về scope trong Keycloak  
   - Trong Admin Console, mở client → tab Client Scopes.  
   - Có các scope mặc định: `address`, `email`, `phone`, `profile`, `roles`… do Keycloak cung cấp sẵn.  
   - Không thấy `openid` vì khi dùng OpenID Connect, scope `openid` được gán mặc định cho tất cả client.  
   - Có thể tạo thêm client scope riêng nếu cần và gán cho client.

4. Kết quả trả về từ token endpoint  
   - Khi gửi request đúng, server trả về:
     - `access_token` (JWT),  
     - `expires_in` (ví dụ 60 giây),  
     - `token_type` là `Bearer`,  
     - `id_token` (nếu có scope `openid`).  
   - Nếu bỏ `openid` khỏi scope, chỉ nhận `access_token`, không có `id_token`.  
   - Khuyến khích luôn dùng scope `openid` để có thêm thông tin về user/client.

5. Phân tích JWT token  
   - Copy `access_token` hoặc `id_token` → dán vào trang `jwt.io`.  
   - Xem:
     - Header: thuật toán ký, `kid` (key id)…  
     - Payload: thời gian hết hạn (`exp`), thời gian phát hành (`iat`), `iss` (issuer), `client_id`, roles, scopes, `preferred_username`, `email_verified`, v.v.  
   - `id_token` cũng chứa thông tin tương tự, có thể mở rộng thêm thông tin tùy nhu cầu.

6. Ứng dụng thực tế  
   - Trong bài giảng, Postman đóng vai trò client gọi đến Authorization Server (Keycloak).  
   - Trong dự án thực tế, client (Java hoặc backend khác) sẽ gọi token endpoint với cùng body để lấy access token.  
   - Access token sau đó được dùng để gọi Resource Server (các API bảo vệ bằng bảo mật).
