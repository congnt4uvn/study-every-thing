
Bài giảng nói về việc dùng **GitHub làm backend cho Spring Cloud Config Server** và lý do đây là cách tiếp cận được khuyến nghị.

1. **Vì sao nên lưu cấu hình trong GitHub**
   - Bảo mật tốt hơn (repo có thể để private).
   - Có quản lý phiên bản, lịch sử thay đổi, dễ kiểm tra lại cấu hình cũ (1–2 năm trước).
   - Các cách khác như file system, classpath khó theo dõi lịch sử version.

2. **Repo GitHub ví dụ**
   - Repo: `eazybytes-config`.
   - Chứa các file YAML cấu hình cho microservice: accounts, cards, loans, cùng một số file khác (Eureka, Gateway…).
   - Repo được để public cho sinh viên; trong thực tế nên để **private** và cấu hình **authentication** cho Config Server.

3. **Cấu hình Config Server dùng Git**
   - Trước đó dùng `native` profile (classpath/file system), giờ chuyển sang **git profile**.
   - Trong `application.yml` của Config Server:
     - Bật profile `git`.
     - Comment toàn bộ cấu hình liên quan `native` (search-locations, v.v.).
     - Thêm cấu hình:
       ```yaml
       spring:
         cloud:
           config:
             server:
               git:
                 uri: <https-url-repo-github>
                 default-label: main   # nhánh mặc định
                 timeout: 5           # timeout 5 giây khi kết nối GitHub
                 clone-on-start: true # clone repo ngay khi server khởi động
                 force-pull: true     # luôn ghi đè thay đổi local bằng bản trên GitHub
       ```
   - Ý nghĩa:
     - `default-label`: chỉ rõ nhánh để tránh nhầm lẫn khi có nhiều branch.
     - `timeout`: sau 5 giây không kết nối được thì ném exception, giúp phát hiện sớm lỗi (repo down, sai URI…).
     - `clone-on-start`: đảm bảo repo được clone ngay khi khởi động, tránh lỗi khi request đầu tiên đến mới clone.
     - `force-pull`: xóa/ghi đè mọi thay đổi local, luôn đồng bộ với repo GitHub.

4. **Khởi động và kiểm tra**
   - Dừng tất cả microservice và Config Server.
   - Khởi động lại Config Server với profile `git`.
   - Khởi động lần lượt: `AccountApplication`, `LoanApplication`, `CardsApplication`.
   - Kiểm tra Config Server:
     - Gọi `http://localhost:8071/accounts/prod`.
     - Thấy Config Server trả về cấu hình và hiển thị URL GitHub đang dùng → xác nhận đọc từ repo GitHub.
   - Kiểm tra tích hợp:
     - Gọi endpoint của service (ví dụ: thông tin liên hệ của cards).
     - Nếu trả về thành công → service đã lấy được cấu hình `prod` từ Config Server.

5. **Các backend khác & tài liệu chính thức**
   - Truy cập tài liệu Spring Cloud Config trên trang Spring:
     - Có phần **git backend**: hướng dẫn dùng repo private, truyền username/password, hoặc dùng SSH.
     - Có phần cho **AWS CodeCommit**, **Google Cloud Source**, **SSH config**, v.v.
     - Các backend khác:
       - File system backend.
       - Vault backend.
       - CredHub server.
       - AWS Secrets Manager, Parameter Store.
       - JDBC backend (lưu cấu hình trong DB).
   - Khuyến khích luôn đọc tài liệu chính thức khi gặp yêu cầu phức tạp hoặc chưa được đề cập trong khóa học.

6. **Thông điệp cuối**
   - Hãy chủ động đọc tài liệu chính thức để tự giải quyết các tình huống mới.
   - Số năm kinh nghiệm không quan trọng bằng kiến thức thực sự nắm được.
   - Khóa học chỉ giúp hiểu khái niệm; để lên mức senior, cần tự nghiên cứu thêm từ tài liệu chính thức.
