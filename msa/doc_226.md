
Dưới đây là tóm tắt ngắn gọn nội dung trên:

1. Cụm Kubernetes cục bộ đã chạy thành công. Bước tiếp theo là triển khai toàn bộ các microservice vào cụm để:
   - Kiểm chứng các tính năng hỗ trợ triển khai của Kubernetes.
   - Thay thế cách làm bằng Docker Compose (vì Kubernetes không hiểu file docker‑compose).

2. Để triển khai một microservice lên Kubernetes, cần:
   - Viết file cấu hình YAML theo cú pháp riêng của Kubernetes.
   - Mô tả cách triển khai (Deployment) và cách expose (Service).

3. Bài giảng tập trung triển khai Config Server trước:
   - Tạo thư mục `section_15/kubernetes`.
   - Tạo file `configserver.yaml`.
   - Trong file này chứa nhiều cấu hình Kubernetes (manifest) được phân tách bằng `---`.

4. Phần 1: Deployment
   - `apiVersion: apps/v1`, `kind: Deployment`.
   - `metadata.name: configserver-deployment`.
   - `metadata.labels.app: configserver` (nhãn rất quan trọng).
   - `spec.replicas: 1` (số bản sao/pod).
   - `spec.selector.matchLabels.app: configserver` để match với nhãn pod.
   - `spec.template.metadata.labels.app: configserver` (nhãn của pod).
   - `spec.template.spec.containers`:
     - `name`: tên container.
     - `image: eazybytes/configserver:s14` (image trên Docker Hub).
     - `ports.containerPort: 8071` (cổng container).
   - Nhãn `app: configserver` phải thống nhất ở mọi nơi để Kubernetes ánh xạ đúng Deployment – Pod – Service.

5. Phần 2: Service
   - `apiVersion: v1`, `kind: Service`.
   - `metadata.name: configserver` (tên này sẽ là hostname nội bộ trong cluster).
   - `spec.selector.app: configserver` để Service trỏ đúng tới pod của Deployment.
   - `spec.type: LoadBalancer` để expose ra bên ngoài cụm.
   - `spec.ports`:
     - `protocol: TCP`.
     - `port: 8071` (cổng bên ngoài).
     - `targetPort: 8071` (cổng trong container; phải trùng với containerPort).

6. Về file YAML:
   - Ba dấu gạch ngang `---` dùng để tách nhiều cấu hình (Deployment, Service, …) trong cùng một file.
   - Kubernetes sẽ xử lý từng khối cấu hình như các file YAML riêng.
   - Các file này thường được gọi là “Kubernetes manifest”.

7. Kết quả:
   - Với manifest `configserver.yaml`, Config Server sẽ được:
     - Triển khai thành pod/container trong cluster.
     - Expose ra bên ngoài thông qua Service kiểu LoadBalancer.
   - Bài giảng tiếp theo sẽ dùng file này để thực sự deploy Config Server lên cụm Kubernetes cục bộ.
